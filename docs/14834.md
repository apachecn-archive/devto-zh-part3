# 极端编程

> 原文:[https://dev.to/markpritchett/extremes-in-programming-4f2](https://dev.to/markpritchett/extremes-in-programming-4f2)

这里有很多关于各种编程语言/库的优秀文章，所以我想我应该写第一篇关于编程的另一个方面的文章——我称之为编程中的极端。

我看到了 Jen Simmons 的这条推文，我认为这是一个很好的时间来讨论为什么人们倾向于一种方法，经常宣称这是唯一正确的方法。

这些年来，我遇到过许多例子，这里只是几个:

*   我不会**使用 SPA 框架和计数器，我会**总是**使用 SPA 框架。**

*   我会**总是**使用存储过程，而我会**从不**使用存储过程。

*   我总是使用微软的产品，而不是 T2 的产品，我不会用驳船杠碰 M 美元。

在上面所有的例子中，以及任何其他的例子中，我们当然应该为正确的工作选择正确的工具。
