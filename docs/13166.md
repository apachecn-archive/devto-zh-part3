# 每周笔记#1

> 原文:[https://dev.to/sirtimbly/weeknotes-1-22ae](https://dev.to/sirtimbly/weeknotes-1-22ae)

我在其他一些博客上看到了这个“Weeknotes”的想法，并认为这是个好主意。有一段时间，我试图每周写一篇关于[发展到](https://dev.to/sirtimbly)的文章——但我错过了第 6 周，现在很难恢复这个习惯。也许如果我再给我的个人博客增加一些价值，我会更有动力去写。

## [](#monday)周一

看多了抽搐。Apex Legends 看起来太棒了。

通过 [Calibre](https://calibreapp.com) 设置一个 app 的性能监测——真的很棒。

查看了最新版本的 UI 引擎，看看它是否是我下一个设计系统的良好基础。它可以显示普通的 HTML 或类似分形的手柄模板，但它也支持显示 Vue 和 React 组件，所以这是一个很好的选择。最新版本在配置方面也有很大的改进。

## [](#tuesday)星期二

发现[正常运行时间机器人](https://uptimerobot.com)。好的免费计划。在 Remy Sharp 的博客上发现了一个[有趣的博客，关于在互联网 30 周年之际重建 CERN 的第一个网络浏览器。](https://remysharp.com/2019/02/12/cern-day-1)

## [](#wednesday)星期三

今天有很多代码审查。为客户寻找域名注册商，看看为什么他们的旧网站托管消失了。神秘。如果没有时光倒流机，我们会因为疏忽而失去更多的网站。

## [](#thursday)星期四

花了很多时间在 Zeit Now 上建立了一个初始原型 serverless+FRETS 应用程序。惊人的筹码。将不得不多写。完全可以部署用 TypeScript 编写的[无服务器函数，而无需预编译](https://zeit.co/blog/scalable-apps-with-typescript-and-now-2)。

## [](#friday)星期五

与我的学员讨论了 CSS 和 HTML。很多基本规则，比如何时使用`position`。选择器特异性和背景图像的概念。在 Chrome mobile viewport emulator 上查看 flex 元素时，发生了一件非常奇怪的事情。整个 flex 项目缩小到适合可用的 flexbox 空间…有趣。在 flex 项上设置一个像素值似乎可以解决这个问题。

看到这篇关于 UX 和布拉德·弗罗斯特的登录表单的文章。不要耍登录表单的小聪明。
