# 写作的简单性

> 原文：<https://dev.to/remotejavadev/simplicity-in-writing-1pb9>

写商务邮件时简洁的价值是非常重要的。此外，我们的效率取决于使我们的作品易于消化。一个远程 Java 开发人员可以通过出色的写作成倍地增加他或她对客户和雇主的价值。

> “商业写作需要清晰和说服力。主要的技巧是保持事情简单。简单的文字是有说服力的。一个五句话的好论点比一百句话的精彩论点能打动更多的人。别打了。”<cite>斯科特·亚当斯，呆伯特的创作者</cite>

### 常见场景

从成本的角度考虑我们发送的每封电子邮件是有帮助的。例如，一封写得不好的电子邮件会引发一连串的延迟和生产时间的损失。

想象一下一个常见的场景，您被分配了一个新功能请求的任务。最初，您只有最基本的需求，需要从产品负责人那里获得更多信息。然后由您来分析请求及其实现细节。接下来你要做的就是高价值开发者和一般开发者的区别。

一般的开发人员会等着和产品负责人开会讨论需求，而不会发送会前邮件。之后，通常会出现这样的情况:

会议时间到了，普通开发人员变得沮丧，因为对话变成了次要的细枝末节。这只是被一个健谈的非批判性参与者的习惯打破了，他习惯于进入不相关的意识流喋喋不休。这占用了会议的大部分时间，最后项目经理宣布你们下周需要再次会面，继续讨论需求。*(普通开发人员的大脑爆炸)*

### 高价值开发

相比之下，高价值的开发人员将分析特性请求，并写一封电子邮件，说明可能的用例及其含义。这给了产品所有者和开发人员一个共同的环境。随后，他们可以更直接、更清晰、更快速地迭代任何差异。

作为开发人员，我们价值的很大一部分是将模糊的业务请求转化为正式的构造。我们经常会发现产品负责人最初没有考虑到的边缘情况，反之亦然。通过允许我们在设计中考虑边缘情况，预先确定边缘情况可以节省时间。结果，这导致更健壮的软件并降低维护成本。这两者都受到不可避免的代码处理边缘情况的伤害，这些情况没有及时得到识别。

简而言之，高价值的开发

*   主动充实需求
*   用一封精心制作的电子邮件让每个人都在同一页上
*   通过避免上述的大脑爆炸式的会议来珍惜每个人的时间
*   不仅仅是编写一个规范

### 摘要

当在商业环境中交流时，简单应该是我们首先想到的。我们的工作性质使得写有效的电子邮件变得极其重要。尤其是在减少模糊性和简化复杂系统的时候。此外，通过做好这一点，您可以为您的公司节省数千美元的时间损失。总之，远程 Java 开发人员的写作甚至更重要，因为我们的异步交流技能应该使我们有别于我们的办公室同事。
