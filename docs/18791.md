# 你有好工作吗？

> 原文:[https://dev.to/antonfrattaroli/do-you-have-a-good-job-g46](https://dev.to/antonfrattaroli/do-you-have-a-good-job-g46)

当你只能和过去的工作做比较时，就不容易认识到工作的价值。过去糟糕的工作会让员工高估他们现在的工作。缺乏经验会导致低估一份工作的价值。好工作的定义并不总是一样的，但是让工作变好的责任总是落在管理者身上。一个好的经理知道是什么让一份工作变得好。

# [](#individual-attention-from-your-manager)来自经理的个人关注

在工作中取得成功是你和你的经理共同的责任。如果你不能定期与你的经理面对面交流，那么你就没有得到员工的尊重。在管理中，成功的一对一会谈是一个被广泛讨论和营销的话题。即使你在一对一的情况下沟通有困难，你的经理应该有一些工具和策略来开始并保持你的进步。你的经理应该让你感觉被倾听，而不是被遗忘。

# [](#career-progression)职业进展

如今，市场的剧烈变化司空见惯。因此，公司不能像过去那样保证优秀员工的终身雇佣。作为回应，企业应该努力保证增加就业能力。当你已经不满足当前职位的要求时，你应该具备下一步所需的技能和培训。在一个理想的世界里，你和你的经理会就这一点达成一致，公司应该在你离开的时候提升你，为你的离开做好充分的准备。

# [](#transparency-and-engagement)透明度和参与度

你理解并同意完成工作背后的原因吗？没有什么比被告知部分信息或被告知“只管去做”更损害你成功的机会了。如果工作的“为什么”没有被交流、理解和认同，那么它的重要性将会被那些做出贡献的人所忽略——导致糟糕的贡献。

# [](#professional-communication)专业交流

你的公司有讨论职业互动和工作方法的通用方式吗？许多系统看起来像愚蠢的个性测试，但却是为了给团队建设提供一种通用语言。加入团队可能需要很大的信心，但如果使用得当，它们可以增强你的团队相互沟通和同情的能力。

# [](#developer-experience)开发者体验

许多人认为用户体验是第一位的。当事情出错时，它就像一个情感安全网。在你和挖掘工作的实质之间有障碍吗？领导会想办法改进流程，让你的生产能力尽可能不受约束。

# [](#singlediscipline-teams)单专业团队

当你在一个团队中工作，练习同样的技艺时，合作、学习和分享的机会唾手可得。你的经理可以专注于管理一类员工的深层知识，而不是分散精力去追赶，以了解如何培养不同学科的员工。

# [](#team-performance)团队表现

您的团队需要商定一种简单的方法来集体衡量他们的绩效，以及您的经理如何与团队合作，随着时间的推移提高数量和质量指标。为这些共同的目标而共同努力有助于你理解你在公司的角色是什么，以及你在更大的计划中的位置。

# [](#regular-team-meetings)定期召开团队会议

除非你花时间，否则协作和持续改进不会发生。这个时间也可以用来刷新为什么工作是重要工作的原因，来重振和激励团队。

# [](#your-work)你的工作

仅仅因为你知道[在此插入传统技术]并不意味着你接受了这项工作。你的经理应该让你优先关注自己的核心职责，或者拓展到你想探索的领域。当你成功地做了你喜欢的工作，你的成就应该得到认可。

# [](#interteam-teamwork)团队间团队合作

对于一个团队来说，依靠另一个团队来取得进展可能是一个棘手的问题，即使他们反应迅速。你的经理应该通过更多的合作来取代对其他团队的依赖。如果一个开发团队定期向安全团队请求新的服务帐户，那么让开发人员创建他们自己的服务帐户，同时引入一种方法，允许安全团队审核帐户的合规性。

# [](#benefits)好处

健康，牙科，视力，401k 假期、休假和病假:相当标准的福利。高管们正在努力提供更多吗？灵活的(健康)支出账户、401k 匹配、奖金(利润分享、股票期权)、灵活的工作时间和远程机会、健身房准入以及产假/陪产假福利是公司可以在奖励员工贡献方面做得更多的一些领域。人力资源应该总是问“在这一点上，我们能承诺做得更多吗？”

# [](#people-leave-managers-not-companies)“离开管理者的是人，不是公司。”

确保你的职位是一份好工作是你经理的职责。我相信一份好的工作不需要在每一个方面都出类拔萃，但至少要解决每一个问题，并致力于不断提高。在一份真正好的工作中，你得到的总是比你投入的多。
