# 作为一名独立开发者的巨大困难

> 原文：<https://dev.to/markjohnson303/the-staggering-difficulty-of-being-a-solo-developer-2218>

六年前，当我与人合伙创办了一家电子商务/制造公司时，我开始了我的编码之旅。我们需要一个“技术人员”，而我是当时最接近的人。我远非专家，我在 Wordpress 上建立了我们的第一个网站，并努力让它正常工作。

久而久之，我积累了一点点知识。我开始思考 HTML/CSS，使用 Treehouse 学习基础知识。当我们把网站搬到 Shopify 时，我通过定制我们的主题学到了更多。不是很好。事实上，它充满了黑客和写得很差的垃圾(对以前的自己好一点！).在我从人类那里获得真正的代码培训之前，我自己又苦干了 3 年。

我很幸运能够去大会的全栈训练营。当我意识到自己独自工作所错过的一切时，我的心灵被打开了。让别人关注我的工作，哪怕只有 3 个月，也会让我的知识突飞猛进。

自从那次强化课程后，我就自己回来了。斗争是真实的。以下是我发现作为一个开发团队中最困难的一些事情:

## 你不知道你不知道的事

这对每个人都适用，尤其是当你独自工作的时候。在团队中工作，你会更频繁地接触到新的想法。你会听到你不知道的术语，你会得到关于做某事的更好方法的建议，你通常会得到更多的指导。

一个人工作？你大部分时间都会盲目飞行。如果你想得到关于某件事的反馈，你必须自己去寻找。你将不得不投入更多的时间进行研究，仅仅是为了获得对概念的基本认识。

## 你连 git 都怎么用？

不，真的，怎么会？我不能告诉你。我可以

`git add .`

和

`git commit -m "whats the point of this part"`

但是我不能告诉你为什么这对我很重要，除了我需要这样做来部署到 Heroku。显然对一个团队来说，这是至关重要的。一个人工作？嗯，我肯定有好处，但我肯定无法向你解释。

这是任何未来的开发人员都会讨厌我的众多原因之一，但事实就是如此。我不知道如何使用 git。

## 马虎起来容易多了

保持你的代码整洁是一个持续的挑战，尤其是当没有人让你对它负责的时候。你内心深处知道，你未来的自己会感谢你整理，但你也必须完成它。如果其他人将会看到它，你要让他们能够理解它。你一个人？这是一个全新层次的纪律要求！

## 你是自己的“第二双眼睛”

每个开发者都会在某个时候碰壁。你的代码不起作用，你就是弄不明白。如果你在一个团队中工作，你可以拍拍某人的肩膀，让他们看一看。自己一个人？谷歌。再次谷歌。问一个关于堆栈溢出的问题，祈求得到回应。发一条推特。再次谷歌。走开。回来吧。拔掉你的头发。然后找到丢失的括号。冲洗并重复。

## 孤独(哦上帝的孤独)

我的办公室在一个没有窗户的地下室里，这并没有帮助，但这种孤立感主要来自于知道公司里没有人真正知道我在做什么，我如何挣扎，或如何帮助。是真的。

## 保持动力

团队责任对保持你的动力大有帮助。当你自己编码时，这是你不能依赖的。确保完成任务并发布这些功能取决于您。

# 那么你能怎么办呢

如果有简单的解决方法，我就不会写这个了。我可以告诉你我在做什么，希望能有所帮助:找到你的部落。我很幸运生活在像丹佛这样的地方，那里的开发者社区很强大。我坚持尽可能多出去参加活动，结识新朋友(尽管这也带来了自身的动力问题)。我也试图更多地参与像这样的在线社区。每周一次的# DevDiscuss twitter 聊天帮助我减少了孤独感！

你是 solo 开发者吗？你的挣扎是什么，你是如何度过的？
