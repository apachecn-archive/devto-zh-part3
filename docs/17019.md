# 2018 年成就

> 原文：<https://dev.to/lukaszkuczynski/2018-accomplishments-3iha>

## 什么是过去

“从所有的辛苦工作中找到乐趣”是件好事。我能看到很多人在做他们的年度总结。显然，有比工作更重要的领域，但在这里我将只关注商业。有什么变化？

### 机器学习

今年学到了很多东西。我一直觉得这对我来说太多了。我一直喜欢数学，但当我开始挖掘机器学习背后的数学时，这项任务似乎令人生畏。但是 Udacity 和 Pluralsight 带来了很多好处。尤其是 [Udacity 一号](https://dev.to/lukaszkuczynski/machine-learning-with-udacity-549e)。凭借简单的例子和大量的图片，他们能够轻松地解释困难的事情。我立即想到了业务用例。所以有需求和动机。我收集了一些数据，并成功地将这些知识应用到我的两个项目中。两者都是关于文本分析的。第一个是有监督的机器学习，为我们的票务工具自动分配团队。我能够为有明确职责范围的团队实现相当高的准确性。第二个是应用聚类作为无监督学习的例子，使用著名的 K-Means 算法。当然，我不能分享这些数据。但是在玩我的机器学习基础知识时，我很高兴在我的 Kaggle 帐户中分享一些基于公共数据集的见解。你可以在这里看到我的一些内核。顺便说一下，了解 Kaggle 是我今年的另一件大事，因为这是一个分享我的知识和进步的好地方。但不仅如此，我还能从别人那里获得灵感。一些帖子证明了这一点，看看[这个关于巴西啤酒的](https://dev.to/lukaszkuczynski/regression-and-outliers-and-beer-57k8)。

### 云

今年，我理解了“一切都进入云”宣传的一些好的方面。当考虑一个人的本地或内部环境时，我可以看到可扩展性和缺乏流畅性的问题。当你不得不为每一个变化向一个专门的团队提出问题时，一切似乎都很难得到。当使用云基础架构时，这是多么不同啊！今年，我有机会作为 PaaS 与 Openshift 合作。您所需要做的就是关闭 Docker 映像中的依赖项，并在需要时重用它。

我也玩过市面上的大家伙:AWS 和 Azure。我甚至对他们部署相似基础设施的服务做了一点比较，因为我想比较他们的无服务器支持。我构建了一个简单的通知引擎，利用了 Azure 函数 T1 和 T2 AWS Lambdas T3。

我当前收费任务的主要部分是进行几项数据分析任务。我将在下一节详细介绍它，但我可以说，如果没有 Azure Databricks，这个旅程不会那么容易。

在云中，这是我的三个最爱:

*   它是无限的
*   即现即付制度
*   不关心基础设施

### 数据分析

正如我之前提到的，几年前我开始使用 Spark，但那时候——说得委婉一点——不太合理。那么最近发生了什么变化呢？当涉及到用户活动时，我被分配了几个任务来利用我们日志文件中的大数据，我注意到并行计算何时释放其威力:当容量大于兆字节时。当然，这还不是处理数据的 TB 数，但我们正在达到这一水平。当你在平台可用的地方运行 Spark 时，它很酷。在我目前的任务中，我可以利用 Databricks 的力量，因为我们有一些 Azure 帐户可以利用。就个人而言，数据块让事情变得简单快捷。为什么？

首先，我喜欢玩 Python 笔记本。它让你的工作变得如此具有描述性，我认为将代码与一些文本解释混合在一起的想法很棒。

其次，使用快速资源调配的灵活性非常好。您只需选择所需的集群，几分钟内即可完成。如果你不满意——或者你快没钱了！-只需解散一个。就这么简单。

我能够获得在利益相关者面前隐藏的非常好的见解，因此我们可以更好地做决策过程。我们可以回答以下问题:用户最喜欢的领域是什么？用户对新功能的反应如何？用户的位置是什么，它会影响他们的选择吗？

从数据中获得洞察力很重要，我认为在 Azure Databricks play 的帮助下进行数据分析是至关重要的。这项工作可以让决策者看到一些挑战或成功。它可以使复杂的数据变得简单。

## 接下来是什么？

我今年与业务相关的计划呢？在 2019 年，我不打算学习任何新的语言。我**不会**去改变我的方向。相反，我想继续我从 2018 年开始的进步。这一年我更加明白我从 Java 到 Python 的转变是一个好的选择。有了 Python，我觉得自己又年轻了，我喜欢编程和写代码，我现在觉得自己像个诗人，而不是像个被迫在月底前写几篇文章的记者。

我将阅读新书，我还不知道是哪本……但可以肯定的是，我有一个目标，那就是完成我最近开始阅读的关于用数据讲故事的巨著。哇，它真的打开了我的心扉，让我看到了绿色的田野。我想在 2 月份完成 Udacity，开始玩文字数据，说真的。顺便说一下，你可以在[专用 git 回购](https://github.com/lukaszkuczynski/ud120-projects/commits/master)上查看我的进度。

长话短说:

我想成为一个帮助者，并利用他人的帮助。

我想通过交流带来成果。

我想通过可视化让事情变得更简单。

我想自动化，使事情更快。
