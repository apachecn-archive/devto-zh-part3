# 什么是 Web 开发中的负载测试

> 原文:[https://dev . to/flipped coding/what-load-testing-is-in-web-development-55 H2](https://dev.to/flippedcoding/what-load-testing-is-in-web-development-55h2)

你知道你的网站是托管在一个服务器上的，不管它是在云中还是在办公室另一边的冷藏室里。在所有资源耗尽之前，服务器只能处理这么多流量。一旦这些资源达到最大容量，用户就会遇到网站问题，网站会运行得非常慢或崩溃。在你到达那种情况之前，考虑做一些负载测试。

## [](#what-load-testing-is)什么是负载测试

负载测试是一种特定的性能测试。您通常试图找出在真实用户条件下应用程序的最大运行能力。大多数网站所有者希望他们的流量随着时间的推移而增长，他们想知道他们的网站能够处理额外的流量。负载测试告诉您当一群用户访问应用程序时，应用程序的行为如何。

通常，负载测试用于基于 web 的应用程序，因为您正在挑战服务器的极限。这是一种非功能性的测试。通过功能测试，你可以预测你会得到什么样的结果，它们仅仅依赖于测试数据。对于像负载测试这样的非功能性测试，您永远不知道您的结果会是什么，它完全取决于系统中的用户数量。

## [](#why-we-do-load-testing-at-all)我们为什么要进行负载测试

你不希望你的网站在开始有大量流量的时候宕机，对吧？负载测试告诉您应用程序可以支持的并发用户的数量，并让您知道您的服务器是否有足够的资源来扩展和运行应用程序。大多数企业都想知道这类信息，因为他们想确保他们的网站始终处于运行状态。

当一个网站因为服务器无法处理用户负载而关闭时，企业会很快损失很多钱。即使网站加载，如果超过几秒钟，人们就会离开。负载测试向您保证，您能够处理任何遇到的用户。它还会告诉您客户机和服务器之间是否有延迟。

一旦您完成测试，您将能够看到您的系统可以改进的地方。这可能意味着购买更多的服务器资源，如内存，或者对软件进行性能增强。在你的网站吸引大批用户之前，你就应该了解所有这些，这样你才能更好地为他们做准备。

## [](#how-load-testing-works)负载测试如何工作

负载测试的细节取决于您正在使用的系统。大多数云服务都有自己的工具，但也有第三方工具可用。不管您决定使用什么工具，都有一些通用步骤可以用来执行良好的负载测试。

首先，您需要创建一个测试环境。这可能是开发或 QA 环境，如果你有权限使用它们的话。你永远不要仅仅因为有真实的用户在访问应用程序就在生产中进行负载测试。在您准备好测试环境之后，决定您想要使用哪种测试事务。

这包括您需要的任何数据、您希望系统模拟的用户数量、任何服务器配置、您希望针对的浏览器和操作系统以及连接速度。你需要知道的另一件事是你在看什么指标。您是想看看您的系统当前可以支持的最大用户数量，还是想看看内存容量？您需要知道您在寻找什么，否则您将不会从负载测试中获得太多的价值。

一旦您获得了所有这些信息，您就可以继续运行测试了。然后你将分析你的结果，并能够对系统中的哪些地方需要改进以及如何改进提出建议。基于你从其他人那里得到的反馈，可能是来自业务方面的反馈，你会对系统进行调整，直到它给你你想要的结果。

这可能被归类为 DevOps 的一项任务，但是开发人员的工具箱中也应该有这项任务。有人在运行负载测试时得到奇怪的结果吗？或者有谁有他们推荐的运行负载测试的特定工具吗？
