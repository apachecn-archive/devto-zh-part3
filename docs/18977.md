在我们的第一篇文章中，[“我们如何将最初的 JS/CSS 大小减少了 67%”](https://dev.to/goenning/how-we-reduced-our-initial-jscss-size-by-67-3ac0)，[，](https://dev.to/goenning)讲述了他为 Fider 削减前端负载大小的经历，使用了代码分割、延迟加载和转移到更小的依赖项等技术。Malik 和我讨论了图像的大小，由约束带来的创造性，以及“表演”的含义。

在[“你不知道你需要的 5 个 CSS 技巧”](https://dev.to/lynnewritescode/5-css-tips-you-didnt-know-you-needed-nb3)， [Lynne](https://dev.to/lynnewritescode) 涵盖了一些非常有用的 CSS 技巧，可以提升你的游戏水平，或者让你的脚变湿。我试图挑起一场关于瀑布的战争，我们谈论 CSS-in-JS-in-XSS-in-3EB 的价值。

我们接下来进入[“如何在照顾一个婴儿的同时用不到 13 Kb 编写一个游戏”](https://dev.to/vintharas/how-to-write-a-game-in-under-13-kb-while-taking-care-of-a-baby-4160)，其中(*回归冠军* ) [海梅·冈萨雷斯](https://dev.to/vintharas)...嗯，确实如标题所说。坦白地说，Malik 和我并没有对这篇文章进行深入的研究，所以去看看吧，因为它对你有好处。相反，我们惊叹于这样一个想法:有人真的可以照顾一个婴儿，谈论我们想写的游戏，以及如何处理多任务。

另一位*回归冠军*、 [Jenn](https://dev.to/geekgalgroks) ，在[“我与有毒团队的经历”](https://dev.to/geekgalgroks/my-experience-with-toxic-teams-6fk)中，带领读者了解她在工作场所看到的有毒行为、她对快速离开和永不回头的反思，以及各种危险信号。Malik 和我讨论了“文化反模式”、“有毒”一词的含义，以及最小破坏性毒性可能是什么样子。

接下来我们讨论[“揭秘长箭‘符’”](https://dev.to/somedood/demystifying-the-long-arrow-operator-4711)。[一些 Dood](https://dev.to/somedood/) 引导读者通过一种非常具体的方式编写递减循环，以及如何不被它难住，同时也解释了应该避免这种符号，因为它有点误导。我们的讨论把我们带到了最好的印第安纳琼斯电影(显然是水晶头骨)和我们试图避免的句法风格。

用户 [heftyhead](https://dev.to/heftyhead) 让我们继续上周在[的主题“让我们讨论一个不必要但流行的 Vue 插件”](https://dev.to/heftyhead/lets-talk-about-an-unnecessary-but-popular-vue-plugin-1ied)关于一个潜在不必要的模块导入。我们谈到了导入非常小的模块，权衡了模块的效用和添加它的风险，并再次提到了*主题*。

我们以一篇来自 *[我们光荣的霸主的帖子结束，祝福他的家，所有的光明和温暖永远从他那里散发](https://dev.to/ben)* 标题为[《论耻辱和一座热山》](https://dev.to/ben/on-gui-shaming-and-a-mountain-of-hot-takes-3oh0)。在一个充满阴影的世界里，真实的独特声音谈论的是在不贬低他人的情况下，对自己的工具链表示赞赏。我们谈论不同的工作流程如何意味着更多的人要做更多的工作，健康的社会压力和羞辱之间的区别，以及█████████████████████ *我们领导的话绝对正确。*
