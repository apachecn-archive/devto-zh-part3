# 我计划在 2019 年学习什么

> 原文：<https://dev.to/concreteniche/what-i-plan-to-learn-in-2019-59i>

又是一年，又一套闪亮的新技术诱惑着你！在 Cogent，我们最近问了对方 2019 年打算学什么。下面是我的列表:你会发现前端和后端技术的混合，以及一些营销相关的技术。

以任何方式学习任何看起来引人注目和有趣的东西都是很有诱惑力的。我试着变得更挑剔一些，选择一些成熟的东西。或者是新事物，我认为它会一直存在。

我渴望通过教学来学习。终于有了一个正常的博客，并制作视频。我会先写一篇博客，从人们那里获得反馈，然后记录一堂课或一次演练，在 YouTube 上分享。

当我教初级程序员的时候，我觉得我的很多知识都因为不得不清晰地表达那些我认为理所当然的概念而得到了加强。我很好奇事情通常是如何工作的，所以威尔想知道，但我总是忽略一些事情——“是的，这就是工作，我不在乎为什么或如何做我的工作”。所以坐下来研究这些概念，然后用一种不会导致 35 张困惑的脸的方式来解释它们是很棒的。

在考虑过我是否应该在博客上关注某个特定的话题后，我想我会写下我正在做的事情，而不用担心我的每一篇博文是否让“网络开发人员”或其他人满意。

## 我打算学什么

*   Cloudflare 工作人员及其最适合的职位
*   利用下一代 React 很好地设计应用
*   优化 web 应用程序，使其更快、更精简
*   GraphQL
*   Golang 和它最适合的，也许编译成网页汇编
*   谷歌应用引擎和数据存储
*   灵丹妙药和它最适合什么
*   Gigalixir 是否值得
*   也许 Websockets 和它们最适合什么
*   将再次尝试 CSS 网格
*   可能更多的是 iOS 应用开发
*   最新的快速变化
*   可能是机器学习
*   如何运行和启动开源项目
*   营销
*   如何计划、撰写和录制截屏视频——在教学后，我发现对我来说，向一群人展示比坐下来滔滔不绝地说要容易得多，也更有吸引力
*   如何不过度设计我的博客和使用像 Wordpress 这样无聊的东西

## 我打算什么*不*学

我发现有一个**否**列表也很有帮助:

*   没有理由
*   不生锈
*   不是安卓
*   除非是为了一个项目，否则可能不会有本地反应，那么是的
*   Less Node.js
*   不是电子

在教学时，我经常会列出“不”的清单，因为这有助于集中注意力。

2019 年你打算学什么？通过评论或推特让我知道！
