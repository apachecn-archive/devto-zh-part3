> 原文：<https://dev.to/ldevernay/haskell-whats-the-point-1jkj>

[![Haskell Logo](img/776a0ff0813699ad431c6bdf700b9d3b.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--wAQcQL6d--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/tqxznexz7xpmhxto2s1m.png) 
去年，我试图学习哈斯克尔。
这真的很难。
当然，我已经从事编码工作近 15 年，见过很多语言。但是这个...是的，很艰难。
老实说，学习 Haskell 是一次非常好(但痛苦)的经历。我的第一次尝试失败了，因为这种语言中有很多野蛮的概念(单子、幺半群、函子等)。我马上开始学习所有这些。

[![What did I expect?](img/eb3c5cc1f071829b64d75fe484715618.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--lAHJGUFj--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/lmcksg5gitgn19vo3swg.gif)

你猜怎么着？你不需要他们来开始。

因此，作为 2019 年的一个好的解决方案，我回到了基础，并在 codewars 上迎头赶上了一些 katas。一切看起来都更好。当然，我的解决方案并不是最佳的(一些推荐的解决方案真的很难理解)。但是我认为这是一个从基础开始慢慢学习难的部分的好方法。像[哈斯克尔快&硬](https://www.schoolofhaskell.com/school/starting-with-haskell/haskell-fast-hard)。
不久，我将再次开启[快乐学 Haskell](http://www.happylearnhaskelltutorial.com/) 和[学你一个 Haskell 为大好](http://learnyouahaskell.com/starting-out)。
但是现在， [Try Haskell](https://www.tryhaskell.org/) 有我需要的一切。

顺便说一下，还有一个很大的谜团:[这一切](https://www.youtube.com/watch?v=iSmkqocn0oQ)的意义是什么？
我知道有一些人出于工业目的使用 [Haskell 的例子](https://wiki.haskell.org/Haskell_in_industry)(脸书和它的反垃圾邮件工具，等等)。

但是为什么呢？怎么会？有人能帮我一下吗？
最有可能的是，我的下一步将是浏览一些 github 回购，但肯定有更好的方式。

有人能帮我吗？
