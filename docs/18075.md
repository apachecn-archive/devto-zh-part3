我相信 Java，*作为一个平台*，永远不会消亡。JVM 及其高质量库的生态系统，加上它在大规模企业环境中的深厚根基，使它成为一个值得开发的平台，并将继续存在下去。

然而，我确实相信 Java *作为一种语言*会越来越多地被 Kotlin 取代。其他 JVM 语言(Scala、Groovy、Clojure)可能试图取代 Java，但它们从未真正兑现承诺。他们都使用 Java 作为主干做了完全不同的事情，但是对于习惯于用“Java 方式”做事的开发人员来说就是不舒服。Scala 带来了纯函数式编程。Groovy 使它变得动态。但是 Kotlin 是第一个实际上是为了成为更好的 Java 而创建的，总的来说，如果你习惯于编写 Java8 风格的代码，你会很快爱上 Kotlin。

但是 Kotlin 成功的主要部分是它与 Java 的完美互操作。Kotlin 之所以伟大，是因为它只是 Java。Kotlin 和普通的 Java 没有什么本质上的不同。Scala 和 Groovy 都有自己的运行时，可以在 JVM 上运行，但并不完全兼容 Java。Kotlin 只是编译与 Java 完全相同的东西，这对于增量迁移来说是一个巨大的优势，尤其是在大型的、行动缓慢的企业中。

所以，再一次，Java 不会去任何地方。但我确实认为它正朝着 C 语言的方向前进，成为支撑所有其他语言的语言。就这一点而言，Javascript 也在朝着同样的方向发展。这是这种语言成熟的标志，它已经从一种直接使用的语言，成为下一级技术结构的一部分。

在这个问题困扰多年后，我开始...在:D 写东西

*   我在 GitHub 中创建问题，不仅是那些还不能工作的问题，也是我想在未来添加的功能问题
*   当引入代码变更时，我创建拉请求，即使没有人审查它们
*   通常在我开始工作之前，我会收集我的想法，看看我想要达到什么目标，我有什么痛苦需要解决

不要太详细，每个想法有几个要点通常就够了。

不需要太多时间，但是当我暂停几个月后回来的时候，我可以仔细阅读我最初的动机和计划，以及对我离开之前正在做的事情有一个大致的了解。

另外:不用担心忘记好的思路是一种非常令人欣慰的感觉🙂
