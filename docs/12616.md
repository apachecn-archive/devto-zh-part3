# 自定义开发人员设置

> 原文：<https://dev.to/joshbruce/custom-developer-setups-4mlj>

背景故事:我拥有一家名为[的公司。2018 年是真正开始认真对待事情的一年。我们已经扩大到 7 个从业者，但是我们声明的原则和实践有点松懈，没有被遵循。不是以疯狂的 Judy 法官的方式，但它确实导致基本上缩减到 2 个从业者，因为每个人都被要求真正反思他们与 8 倍的关系。事实证明，2019 年本质上是一个重置年。将在线业务扩展到基本静态的网站，取消了一系列域名的续费，并在运营领域全面紧缩开支。我也重新开始在平台上工作。](https://8fold.pro)

相关故事:我正在做我认为是用其他方式实现的奇怪的概念。我也在尝试 8fold 品牌指南中的理念:我们赞美个人，迎合最受限制的人。

所以，我们从无障碍运动中吸取了 UX 的概念，并将其应用于其他概念。

例如，不要在一个分组中有超过 100 个链接的页面，这不仅仅是因为 SEO 不好，还因为有认知问题的人可能无法一次记住那么多东西；见鬼，据我所知，一般人 7 岁左右就可以了。

[![Star Trek Enterprise](img/ec26183a1b8f73f54233ab561336c7b0.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--b5JUZgse--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/9bpev24nci9jlop00hc2.jpg)

为了扩大这一点，8fold 是一个面向微型企业的平台(我还不知道我对这个标签的感觉如何，但这是我所拥有的)。这并不意味着我们不能或不愿支持一个拥有 500 万员工、年收入 1000 亿英镑的“企业”，但这不是我们要迎合的人群。见鬼，这并不意味着我们不能或不愿支持一个 10 名员工、年收入 200 万至 300 万英镑的“小企业”，但这不是我们要迎合的人。相反，我们迎合好斗者和骗子。2010 年版的我，自由职业时住在车里([写了一本关于它的书，想看看吗](https://youtu.be/QfzDUpB88x4)？`<- funny video|book ->` [来了](https://leanpub.com/homeless-not-helpless)。

无论如何，我一直在研究平台代码，并将同样的约束应用到极致。制作一个关于使用 MAC OS split view 的快速[视频](https://vimeo.com/318621196)来创建一个不可维修的设置。我在进一步思考这个问题。

我希望提到的一件事是协作环境中的环境定制和认知负荷。

每当我进入一个环境，我永远不知道会发生什么。每个人都有自己的工作方式，这使得外人很难进来提供帮助。我记得有一次我在办公室的 iMac 上请别人帮忙设置一些东西。

他问:“你介意我安装一些东西吗？”

我说:“不尽然，只要我能完成我的工作。”

他进了临终病房。从记忆中键入了大约 20 个`curl`请求，并且不得不查找另一个请求，声明“我不经常使用那个”

这些都与我实际上做我需要做的事情无关，只是让他在我的香草世界里更加舒适。

完成后，他在一张纸上草草写下一些东西，这是一个“需要时使用”的终端命令他接着说，“但是你真的应该看看我安装的其他东西，它们都是非常好的东西，应该会减少你一天的工作时间。”

"它是从终端执行的吗？"

“是啊。”

“我不确定这怎么可能是真的，因为我每天都没有任何终端。”

“试一试。你会明白的。”

我也走过去帮助别人，我看着他们的想法，“搞什么鬼？玛格丽特。”这变成了几乎直接的结对编程，因为我不知道他们如何让他们的 IDE 看起来像那样，也不知道如何恢复它(就像在别人的车里调整最复杂的驾驶座)。

我最后会说，“打开这个文件。”然后看着他们做一些他们已经嵌入肌肉记忆中的键盘快捷键，文件就像大卫·科波菲尔做 NBC 特别节目一样出现。

同时我在想，这真的比*隐藏文件导航器并点击文件更快或更有效吗？*

我确实认为有些人这样做是为了排场，因为当他们把手放回键盘和鼠标上时，他们会让主音吉他[和弦变脸](https://youtu.be/hwjZ2vnDhdU?t=1m10s)，同时点头，手模挥动，仿佛在说，“看我刚刚做了什么。我打开了一个文件。”(不是所有人，但他们就在那里，你知道你是谁。)

当我进行数字组织时，也是同样的方式。我会问人们他们把行动清单放在哪里。没有人会使用提醒，即使它就在那里，并且做你需要它做的事情。没有判断，我喜欢听到所有的工具，只是让我着迷的是，尽管工具就在那里，但我们通过使用不同的工具解决了多少问题。当你定制你的世界时，记得留下足够的常态来允许其他人生活在其中。
