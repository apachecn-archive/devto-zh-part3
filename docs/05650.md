# 新播客:可维护

我一直在为一个新的播客做一些幕后工作，这个播客是关于可维护软件、遗留代码以及真正的工程团队每天面临的问题。它叫 [*可维护*](https://maintainable.fm/) ，主持人是《哦我的 Zsh[的创作者 Robby Russell 和我在](https://github.com/robbyrussell/oh-my-zsh)[星球氩](https://www.planetargon.com/)的队友。

有几百(几千？)的关于技术的播客，那么我们为什么需要另一个呢？我们不关注什么是闪亮的和新的，而是与经验丰富的从业者交谈，他们克服了巨大的挑战，以充分利用他们现有的技术投资。团队经常依靠“大重写”来解决他们的问题，但实际上这并不总是最好的解决方案。关于可维护性，我们将讨论这些决策以及它们如何影响团队。这是主持人 Robby 关于[为什么我们创造了可维护的](https://medium.com/planet-argon-tenets/hello-maintainable-software-podcast-90c97ec4236)的更多信息。

我希望你听一下，如果你听了，让我知道你的想法！如果你知道某人可能有有趣的故事要讲，请发邮件到 [hello@maintainable.fm](mailto:hello@maintainable.fm) 。

你可以通过搜索“可维护”来收听[苹果播客](https://podcasts.apple.com/us/podcast/maintainable/id1459893010)、[阴云](https://overcast.fm/itunes1459893010/maintainable)，或者任何你收听播客的地方。
