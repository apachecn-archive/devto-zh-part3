# 你能从现场程序员那里学到什么

> 原文:[https://dev . to/therealdanvega/what-you-can-learn-from-live-coders-3bp 5](https://dev.to/therealdanvega/what-you-can-learn-from-live-coders-3bp5)

2019 年似乎是直播编码器年。在这篇文章中，我将告诉你你能从现场程序员那里学到什么。我年轻的时候喜欢玩电子游戏。实际上，我现在仍然很喜欢它们，但是如果这些天我有空闲时间，我宁愿花在代码上。🤓

我记得我在圣诞节得到的第一台任天堂游戏机，我认为它是有史以来最酷的东西。每天放学后，我和我的朋友迈克都会玩我们当时想征服的任何游戏。当我们陷入困境时，我们无法求助于谷歌来寻找答案(是的，我已经那么老了)。我们不得不等待任天堂电源，或一个专门的游戏指南，或者作为最后的结果，我们会问我们的朋友第二天在学校。

## [](#live-streaming-gamers)直播游戏玩家

我提出这个问题是因为我们正处于一个游戏玩家直播的时代。Twitch 是最受游戏玩家欢迎的平台，他们在 2018 年看到了 220 万次广播和 1500 万次每日观看。该平台上一些顶级彩带的年收入超过 30 万美元，而最高收入者在 2018 年的收入超过 350 万美元。

这些数字让我震惊，也表明了当我第一次听到这个想法时，我是多么的不相信它。我最初的反应是“为什么会有人想看别人玩电子游戏”？想想我的反应很有趣，因为如果我开始思考，我年轻时也做过同样的事情，但规模小得多。

当我第一次接触电子游戏时，我们还没有能力和其他人在线玩游戏。如果我们想和最优秀的人比试我们的技能，我们必须去购物中心的拱廊。我真的很喜欢《街头霸王》和《真人快打》,曾经认为自己很棒，直到我去了购物中心，发现那里有一些更好的玩家。

在我被击败后，我会站在人群中观看真正优秀的选手比赛。不是因为我没有更好的事情可做，而是因为我想知道它们为什么这么好。他们用什么策略来控制任何与他们对抗的人？

当我第一次放弃直播的想法时，这是我错过的大画面点。我敢肯定，这些天还有其他的原因值得观看，因为很多赚大钱的人听起来很有趣，而且有很多娱乐价值。

你可能已经明白我的意思了，但是我认为把它和这次经历联系起来会更好。

## [](#live-streaming-coders)流媒体直播编码器

虽然 Twitch 的 99%(不是真正的%只是感觉如此)都在使用该平台直播流媒体游戏，但只有一小部分程序员使用它来编写代码。YouTube 和脸书也给了我们一个按钮就能在观众面前直播的能力。也有像 [Restream.io](http://restream.io) 这样的服务，可以让你一次流到多个平台。

这不是什么新鲜事，因为之前已经有开发者这样做了，但感觉 2019 年才是真正开始起飞的时候。今年我观察了一些开发人员，我真的很喜欢。

我开始把这件事告诉朋友和同事，我得到的第一反应是“为什么你想看别人的现场代码？”。听起来熟悉吗？大多数时候，它不像我最初的游戏评论那样不屑一顾，但通常会有一个眼球转动。

我得到的最大的争论是，为什么这些开发人员不创建一个视频，并编辑他们的错误，然后上传到 YouTube，就像我们多年来一直在做的那样。

### [](#problem-solvers)问题解决者

解决问题是所有开发人员入门时都需要的一项关键技能，也是他们在职业生涯中需要提高的一项技能。将一个问题分解成更小的问题来解决整个问题的能力是我每天都在做的事情。

所以当我听到像“我宁愿只看成品”这样的评论时，我不得不表示不同意。当另一个开发人员试图解决一个问题时，观察他或她的思维过程会涉及到很多东西。

这是面试过程的一部分，这是有原因的。你可能不总是有所有的答案，但我们正在寻找的是你将一个问题分解成小问题并一次解决一个问题的能力。

### [](#rock-star-developers-they-are-like-us)摇滚明星开发者(他们和我们一样)

观察别人编写代码的另一方面是理解我们都有同样的挣扎。情况并不总是这样，但通常现场编码的开发人员都有丰富的经验。

我认为我们对一些开发者有更高的标准，并认为他们没有和我们一样的挣扎。所以，当我们看到这些我们都敬仰的摇滚明星开发者做着和我们一样的事情时，这真的让我们对自己的能力有了一些信心。

当我看到有人不确定某个特定的 API 或者不知道如何做某事，看着他们在谷歌上搜索答案时，我想这让我们所有人都意识到我们都在同一条船上。当然，可能有一些非常优秀的工程师可以几个小时不查任何东西就编码，但我们其他人不会那样工作。

### [](#tips-and-tricks)提示和窍门

我认为观看现场编码员的另一个被低估的特点是获得开发人员认为理所当然的小技巧和窍门。你可能会因为头条新闻而开始观看，但你会因为提示和技巧而留下来。虽然开发人员可能会从头到尾进行实时会话来构建一个新的应用程序，但我总是发现小事情真的很有帮助。

我喜欢了解开发人员工具箱中的不同工具。即使这是一个我听说过的工具，甚至是在看到其他人如何使用它之前使用过的工具，也可能会启发我以一种我不会想到的方式使用它。我也喜欢看到开发人员如何设置他们的 ide，或者他们使用什么主题和扩展。

### [](#live-or-recorded)直播或录制

所有这些好处也可以通过观看录像获得。我需要确保我非常清楚这一点。我并不是说你需要看他们直播，但是看他们直播有一个很大的优势，那就是互动。许多现场程序员与观众互动非常好，并利用他们来推动教程和讨论，所以不要害怕谈论。

当谈到现场编码时，你当然可以观看录制，但我指的是，开发人员没有编辑视频以达到完美。当我谈论实时编码时，我想看到开发人员讨论问题并像我们一样犯错。

## [](#live-streaming-setup)直播设置

直播是我希望有更多时间去做的事情之一。我列出了大约 10 件我想看的事情，更不用说我有一大堆可以转换成视频的博客文章。

我总共做过 1 次直播，所以我不是询问直播设置建议的最佳人选，但我确实有一些资源给你。

Suz Hinton(在上面的列表中)是一个了不起的开发者，他在 Twitch 上做直播。她在她的[现场编码设置](https://medium.com/@suzhinton/my-twitch-live-coding-setup-b2516672fb21)上写了一篇令人惊叹的文章，如果你想进入这个领域，这是一篇必读的文章。

### [](#ecamm-live)Ecamm 直播

我还想提一下用于 mac 的一体化直播制作平台 [Ecamm Live](https://www.ecamm.com/mac/ecammlive/) 。如果你用的是苹果电脑，这是一款非常棒的软件，有一些非常棒的功能。我用它来播放我做的节目，但你也可以用它来录制视频。如果你有兴趣了解更多，请联系我并告诉我。我很想对 Ecamm Live 及其背后的优秀团队做一个详细的报道。
