# 为什么我认为它值得研究

> 原文：<https://dev.to/reg__/why-i-think-it-is-worth-studying-2737>

2018 年，距离我毕业[琴斯托霍瓦理工大学](http://pcz.pl/)，成为计算机科学硕士，已经过去 10 年了。我不后悔。关于是否值得在某所大学学习，还是尽早开始职业生涯并成为完全自学成才的专家更好，程序员之间存在着无休止的争论。以下是我个人对此的看法。(有点免责声明:我的观点可能有偏见，因为我父亲是电气工程教授，所以我从小就接触了很多科学界。)

一个经常出现的问题是拥有大学文凭对潜在雇主的意义。从我的经验来看，我会说，虽然正规教育不是获得程序员工作的必要条件，但对一些公司来说可能很重要。这一点很重要，尤其是在职业生涯的初期，那时你的简历中没有多少内容，因为在多年的专业工作之后，学历排在工作经验之后。较小的公司可能更青睐特定技术的技能和经验，不太需要学位，因为他们希望找到一个可以立即开始工作的员工。大公司可能更喜欢有学位的候选人，因为他们寻找有才华的人，这些人在被雇用后可以学习所需的技能，此外，他们通常有大量的人力资源部门，这些人不是技术人员，他们只根据关键词和简单的标准来筛选简历，比如大学毕业生。

另一个常见的问题是，大学里教的知识是否实用。当然，有许多教授特定主题的课程在你的职业生涯中可能永远不会有用。当然，在每一个工作场所，你都需要学习很多在学习中没有涉及到的东西。诚然，软件技术发展很快，所以你现在所学的可能在几年后变得无关紧要(尽管并不总是如此——例如，C 已经有 40 多年的历史，而且[它仍然是第二重要的编程语言](https://www.tiobe.com/tiobe-index/))。但是我认为学习给予的不仅仅是对未来工作有用的实用知识，或者一张文凭。

**1。它给你理论依据。**虽然你可以从一本书或网上找到的一些材料中学习一种新的编程语言或框架，但对于代数、微积分、物理、算法和数据结构、电子学、数字信号处理等更具理论性的学科来说，这并不容易。你会花很长时间自学这些，而不是专注于更实际的东西吗？我敢打赌，大多数人不会，但当你需要理解一些实际问题时，有一些理论基础是必不可少的，例如，在编写汽车模拟游戏时，速度是如何对时间的导数，加速度是位移的二阶导数。

**2。它教会你一切。**例如，如果你的专业兴趣涉及系统编程和 C 语言，或者数据科学和 Python，你可能更喜欢深化这方面的知识，而不是学习一些你不太感兴趣的东西。在大学里，你必须至少掌握各种学科的一些基础知识，无论是网站开发、数据库、计算机网络和安全、计算机硬件、操作系统还是 shell 脚本等等。这些学科中的每一个都可能在某一天变得出乎意料地有用，必要时，基础知识将是学习更多相关知识的良好起点。

**3。它教会你，并向别人证明，你有能力完成一个大项目。这本身就是一项重要的技能。有许多幻想型的人满脑子都是新的、高水平的想法，他们每天都开始一些新的事情，但从来没有完成任何事情，因为他们觉得做实际的工作很无聊或太难了。也有一些过度思考者，他们把每件事都理论化，计划到最小的细节，但是从来没有开始做。这些不是有效的员工。在大学，每门课程都像一个项目。通过每个学期的考试就像一个有严格期限的项目。最后，写论文和毕业也像一个大项目。这可能是你第一次取得如此大的成功，也是你职业生涯中挑战的良好开端。**
