# 我已经完成了编程课程，下一步是什么？导师的五条建议

> 原文：<https://dev.to/mkdev/i-have-finished-programming-courses-what-s-next-five-pieces-of-advice-from-the-mentor-1bj7>

[![](img/d0af14bc6ceceb6db141eb729efac949.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--AANk1o8w--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/mh8t3ev8jcwi85ul1seg.png)

很多时候，在完成课程后——在线的和离线的——我不得不回答学生的问题，这些问题通常被称为:下一步是什么？。

不同的人根据他们的脾气和心情对这个问题有不同的表述:下一步该学什么，我可以投简历了吗，如何找工作，现在该做什么——但是，以这样或那样的方式，他们中的许多人试图发现通往职业道路上的下一步。

由于我比较懒，为了不重复，我决定把我平时对这些问题的回答都收集在这篇短文里。(由于我极其懒惰，我已经写了半年了。)

所以，问题是:

> **我已经完成了课程(读了自学指南/写了我的第一个程序/看了 10 个视频教程)——接下来呢？**

#### 1。不要停下来

最重要的一点。学习不应该在获得一些证书、读过一本书(甚至获得一个学位)之后就停止。在我们这个领域，你应该不停地学习。这绝不意味着你应该学习所有的编程语言(甚至前 5 名)，以及所有的数据库，或者说，所有的前端框架。

但是不要停止意味着一直努力变得更好。

认为你已经足够了解 Ruby 和它所有的秘密特性了吗？它内部是如何工作的？它在其他语言中是如何工作的？还有有哪些方法可以让代码写得更短更清晰？或者为大项目建立一个合理的架构？以及如何评价代码效率并提高？

做了一个使用数据库的应用程序？你能用普通的 SQL 写一个查询吗？那么规范化和反规范化、查询计划及其优化呢？那么对于流行的数据库之间的差异的一般理解呢，什么是 NoSQL，为什么你需要一个？

有没有画出一个很酷的现代 UI 并让它动起来的？你对框架、ES6、HTML5、CSS3、通用 UX 和设计原则的理解是什么？..所以，我想你明白了。

#### 2。不要限制自己

忙于你进一步的自我教育，不要专注于一个太窄的领域，这很重要。在 2017 年，一个能做任何事情的 Ruby who 程序员很像一个拿着铲子能做任何事情的工人——本身就是一个劳动者。即使你是一个红宝石专家——通常它看起来就像一个工人拿着一把非常好的铲子。

了解你的编程技能的一些现代应用领域(更重要的是——对它们的好奇心)——这是你应该考虑的。

科技新闻的头条是什么？

游戏(永远！)、移动技术、虚拟现实、大数据、制图学、社会结构、机器人——这里有些东西会刺激神经元？买一本书，下载一门课程，试着做一个演示。即使看起来我甚至还不能很好地编程——通常作为一个大三学生更有用，他们可以说是的，顺便说一下，我知道一点关于神经网络的知识...只是读了一些书，但可以想象一个介于这和那之间，比做一个大三学生，只学了 Ruby on Rails+PostgreSQL+AngularJS 的例子。

#### 3。不要被吸引

...自我教育。

> (我听到一声安静的重击:这是我的同事和学生不省人事地倒下了。)

是啊，虽然从老师口中听到这样的话很令人惊讶。事实是，编程是一个非常实用的职业，从最低的层面来说——一般来说，是一种技能。一个已经完成了 184 门课程，熟记所有关于适当架构的经典书籍，但从未参与过真正的项目的程序员，就像一个在 Carpenting 大学学习了五年但从未在手中握过飞机的木匠。

很多技巧和方法都是在搏击中*才*磨砺出来的，导师挑的一个教学项目不太适合这个。不仅如此:一个木匠的学徒，如果对自己拿着飞机或学习如何制作这个凳子不感兴趣，那几乎是不可能的了。

然而，书籍、课程、文章、演示文稿都是有用的。如果不把它们与实践相结合，你可能一辈子都是木匠理论家。但是如果你完全忽视它们，你就可以一辈子做一个熟练的学徒。不是最糟糕的命运，在我们的职业中，即使学徒也赚得很多...但是有点丢人。这比你通过平衡的自我教育所能达到的要少得多。

#### 4。不要贪心

根据我的经验，在前面的建议之后，很多人通常会说是的，但是...—是的，我想参加一个真正的项目，但是我在哪里可以找到这样的项目？..没有人愿意带没有经验的大三学生去实习...

现在:**想赚经验——别自救！**

*   有没有一个天才的想法，你会在某一天以付费服务的形式实现，然后卖给谷歌，一辈子靠利息过活？把它做成某种开源库。有可能会有成千上万个这样的想法，而这一个将会非常过时。但如果不是——你在地上插了一根木桩，这很酷。
*   一个想法闪过，我想知道...–如果可以通过编程找到，那就继续找吧。赶上一个遥远的小镇罕见的门票？算算周边超市价格相差多少？比较并找出一级方程式赛车的平均速度在过去 20 年中是如何增加的？把你所有的家庭藏书保存到一个数据库中，并按流派分类？抓住一个想法——去做吧！很可能不会有超级有趣的结果，但你会获得一些数据经验。
*   你得到了一个免费的实习机会？确保公司很酷/项目很有趣，为之而跑！不需要专家来写邮件，专业人士不会免费工作。但是在没有经验的情况下参与一个行业项目是非常昂贵的。
*   有人正在寻找程序员-志愿者-说是。社会项目，学校网站，城市开放数据的黑客马拉松。准备好意识到没有人需要你的工作——除了你自己。但结果可能会相反。
*   最后(但事实上——首先)，有大量非常酷的开源项目——他们中的许多人准备好接受新手，看看他们的代码，礼貌地评论它，并给他们提供在战斗中成长的机会。如何找到他们？
    *   试着做点什么，然后思考——你用过的图书馆缺少什么？很可能没人想过，也没时间去做；
    *   有很多不同的计划在受欢迎的项目中雇佣学生；比如——[谷歌代码之夏](https://summerofcode.withgoogle.com/)。作为这个项目的一部分，谷歌甚至向学生支付奖学金，但即使是这也不是重点——重点是所有宣布的项目都相当受欢迎，但准备同时接受新手并教授他们。

最重要的事情:参加你的或别人的项目，而不是一次你会觉得你在浪费时间或(如果你幸运，但这种情况经常发生)玩得开心，而不是提高自己或找工作。不信。如果你正在写代码，那是需要的，至少*应该是*，出于某种原因，你正在做正确的事情。哪怕是核算附近幼儿园的锅的剧本，或者是你的色情片目录。

#### 5。不要放弃

一开始，一切都会很糟糕。几乎可以肯定。

如果你不是一个伟大的天才-人类的救世主(酷，如果是这样！)，那么很有可能是:

*   在面试中，人们会取笑你的经历两门课程，三本书，一个知识库；
*   在开源项目中，你的拉取请求会被 caddish 注释拒绝；
*   你自己的项目在最好的情况下会被忽略，但在地球上谁需要那种狗屎以及为什么它在最坏的情况下写得如此糟糕的精神下，嘲笑和肮脏的评论。

对程序员来说，最糟糕的事情可能是提前变成板凳上的老爷爷，他在 Habrahabr 刻薄的帖子上写为什么 X 技术是一堆狗屎，或者 Y 语言社区如何羞辱新手。这是一条通往没人需要、受人尊敬、但毫无用处的职业著名出版商的道路。

在第一轮拒绝、批评和忽视之后，最好的事情就是:考虑所有建设性的东西，忘记其他一切。

在这之前，记住。若干年后，当你的图书馆将在 github 上收集数百颗星星，招聘人员将排队，新技术将被征服——记住这段时间，善待新手。
