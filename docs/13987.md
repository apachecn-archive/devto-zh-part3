# 专用测试的价值

> 原文:[https://dev . to/jessekphillips/value-in-dedicated-testing-29 H2](https://dev.to/jessekphillips/value-in-dedicated-testing-29h2)

什么是软件测试？软件测试人员是做什么的？他们带来了什么好处？詹姆斯·巴赫一直试图通过将软件测试定义为一门手艺来拯救它。但是当你被雇佣去做软件测试时，你可能并没有被赋予定义好的技能。因此，我想把我认为我提出的质量改进放在桌面上，不管它的分类如何。

我认为我给团队带来的最大好处是调查技巧，它使重现问题和深入问题原因成为可能。这意味着问题已经被发现和识别，很多时候当我进入团队时，没有基础设施来处理观察和操纵系统以理解行为或注入已知正在发生的事件。

这种类型的价值与预期的角色任务非常不同。我应该在其他人看到问题之前找到它。我希望验证开发人员满足指定的要求。我需要检查所有先前指定的需求是否继续满足定义的行为。所有这些都是我在某种程度上参与的事情。然而，我正在关注这些变化和系统，并尽我最大的努力来涵盖这些变化所带来的关键风险领域。这意味着我花时间消除整个分类和测试区域，以便在合理的时间内满足合理的质量信心。

我还参与构建回归自动化。自动化有助于调用回归，但是我也相信我上面定义的某些焦点会降低这些回归测试的价值。然而，我最近决定让自己相信一些回归测试，我认为低风险仍然是提供信心的主要价值。这里的重点是，这些低风险测试提供了消除故障分类的覆盖范围，这样，当问题确实发生在一般区域时，焦点可以迅速从这些低风险区域转移到不太了解或覆盖的区域。

我最后写的自动化测试的另一个方面是，它提供了添加覆盖许多不同类型情况的回归测试的可能性。我仍然非常努力地将我的精力集中在已知有问题的地方，但挑战之一是让人们相信他们看到的症状并不总是需要关注的问题。

QA 不是找什么测试，而是找什么不测试。有如此多的事情构成了一个测试，搜索实际上能够消除什么可以留到发布。这可能来自风险、严重性、代码更改等等。
