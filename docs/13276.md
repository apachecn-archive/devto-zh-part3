# 什么是自托管，它有价值吗？

> 原文:[https://dev . to/mortoray/what-is-self-hosting-and-is-there-value-in-it-2p9p](https://dev.to/mortoray/what-is-self-hosting-and-is-there-value-in-it-2p9p)

我有机会与 CppCast *交谈，其中一个问题是自主主持。我想我应该很快地写点东西，让大家讨论它的价值。

抱歉，没有嵌入，因为它们似乎没有列在这里的播客上。😟

我当时在做 Leaf 编译器，我们在节目中讨论过，因为它是用 C++写的。有人问我，我的目标是不是自己主持。

自宿主意味着编译器是用语言本身编写的。在这种情况下，这意味着 Leaf 编译器是用 Leaf 编写的。很多语言都有这个目标，比如 Go 或者 C#。而像 Python 或 JavaScript 这样的语言不是自托管的。

我说过自托管从来不是 Leaf 的目标。我觉得那个回答有点出乎意料。当然，最终，它将是自主托管的，但我想不出它有什么短期价值。它没有给这种语言带来新的特性，却造成了一场麻烦。用 C++编写另一种语言不会伤害任何一个社区。就像用 C 语言编写 Python 并没有贬低 Python 或者 C 一样。

你认为拥有一个自托管编译器代表了语言的一种成熟吗？是自己吃狗粮的情况吗？或者它是不必要的，并且偏离了语言的主要目标？

你怎么想呢?
