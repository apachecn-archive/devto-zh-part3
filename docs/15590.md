# 本周我学到了:一点关于将应用程序发布到产品中的知识

> 原文：<https://dev.to/rachelsoderberg/this-week-i-learned-a-little-bit-about-releasing-an-application-into-production-505d>

在将敏捷应用到我的日常编程实践中时，我学到了许多有价值的经验，其中最具影响力的莫过于过去一周的经验法则“让部署过程成为常规的、可预测的、非事件性的”虽然我的公司没有正式使用任何敏捷实践，但我个人在日常工作流程中遵循了几个敏捷原则，当我采取措施将我的应用程序转移到处理生产销售订单时，我获得了回报。

这是我第一次真正的发布体验，我很高兴在我们运行第一个订单时只发现了两个小错误——在发送给销售助理的确认电子邮件上，总价是错误的。我们还捕获了一个小检查，用于处理一个有错误的订单，该订单需要在其销售订单 GUID 被分配后重新运行。

我和我的经理对总价字段包含错误的值并不感到惊讶。它在开发过程中没有受到应有的审查，我现在意识到这是一个错误-这封确认电子邮件被发送给销售助理并转发给他们的客户。它相当于一份账单或已收款项清单，并且绝对必须反映客户的期望。

我结束了一周的工作，回到应用程序中，开始着手处理定价问题。在与我的经理反复沟通后，我解决了这个问题；这是一个经过大量返工的过程。除了修复之外，我还在电子邮件确认方法之前添加了一个条件:如果我们的单价与 Salesforce 报价上反映的总价不匹配，则会向我自己和我的经理发送一封错误电子邮件，并且确认是*而不是*发送给销售员工。这可以防止将来任何潜在的单价错误到达客户手中。

我遵循的另一条规则是绝不在周五下午发布，所以我期待着在周一早上将应用程序重新投入生产...希望是最后一次！
