# 这是马拉松，不是短跑

> 原文：<https://dev.to/awwsmm/its-a-marathon-not-a-sprint-5568>

我讨厌在高中跑步。但是我们的父亲让我和我的兄弟参与体育运动，以培养健康的生活方式，这种生活方式一直伴随着我，直到今天。大二的时候，我是越野赛跑队的成员，但是我跳过了赛季的后半段，因为我懒得去做。我记得有一次比赛，在附近所有学区的几百名学生中，我是第二或第三个跑得最慢的。我只是没有练习的动力，这在比赛中表现出来了。我父亲很沮丧，他说我们要么做运动，要么找份工作(只要放学后我们能有一点空闲)。我立刻找到了一份工作，谢天谢地，我想我再也不用跑 5 公里了。

但去年，我报名参加了在爱尔兰西海岸美丽的康尼马拉(封面图片)举行的马拉松比赛。跑步可能是你能做的最简单和最便宜的个人运动，如果你不想按照团队运动的时间表工作，或者不想每周去健身房锻炼三到五次，跑步是一种很好的锻炼方式。马拉松比赛将在今年 4 月举行，我目前正在进行为期 16 周的训练计划，以保持体型。

高中让我厌烦的，我觉得是长跑的耐力方面。当然，可能会有一些风景，你可能可以听音乐，但在大多数情况下，当你跑步时，你会独自思考。当你一直在做的事情是左脚、*右脚*、*左脚*、*右脚*时，很难保持动力...

有时候，我对软件开发也有这种感觉。但是对于跑步和编程，我需要提醒自己...

* * *

### 你绝对必须练习

没有人在早上醒来后，没有经过训练就决定跑马拉松。

...或者，如果他们尝试的话，他们会经历一段非常艰难的时间，可能无法完成。软件开发也是如此——从来没有人在早上醒来后成为一名零实践的程序员。成为一个像样的程序员需要几个月的练习，成为一个真正优秀的程序员需要几年的练习。你只需要投入时间，这是没有办法的。当然，这可能会导致另一个问题...

### 有时候可能会很无聊

每天，新父母的生活都被他们的婴儿搞得天翻地覆。这是一个巨大的变化，从一个在特定时刻你可以(几乎)做任何你想做的事情的生活，到一个你的整个生活都围绕着照顾一个不能自理的婴儿的生活。这可能是难以置信的困难和迷失方向。但是，由于害怕公众羞辱或个人内疚，许多父母不愿意承认这有多难。

但是，承认你正处于困难时期，或者寻求帮助，或者承认你努力了很久的事情比你想象的要困难得多，或者乐趣少得多，这都是可以的(也是健康的)。承认你对自己的工作感到厌倦是可以的，即使你认为这是你梦想的工作。有时候，我们需要做一些无聊的，重复的事情。计算应该有助于减少这种情况发生的频率，但这些任务仍然存在，我们仍然不得不偶尔处理它们。

对我来说，写文档是一件极其乏味的事情。我知道这是必须要做的，但这可能会非常重复和无聊，而且我知道我的代码是如何工作的，所以这真的重要吗？马拉松训练要求你每周跑大约 30-40 英里，通常是在比赛前几个月。即使以合适的速度，10 英里的训练跑也需要一个半小时。一周多次这样做——尤其是在室内，在跑步机上，在冬天——可能会无聊到麻木。

你可以通过改变事情来补救。通过跑步，你可以跑不同的课程，或者你可以听播客，或者你可以在不同的时间间隔给自己计时。我在跑步机上要做的一件事是给自己设定 1/4 英里或 1/2 英里间隔的小目标。将一个目标分解成更小的步骤是让它更快实现的好方法。

你也可以在编程时进行切换。也许可以从日常工作中休息一下，探索一种新的编程语言。读一篇关于一款你听说过但还没用过的新软件的博文。绘制您的代码图以获得全局视图。你可以做很多事情来打破单调。如果你遇到一个真正适合你的方法，你可能会发现...

### 有时你会陷入一种状态

当你经常跑步时，你以每分钟的步数来学习你的配速。我的一般在 150-155 左右。像许多跑步者一样，我有一个适合我节奏的歌曲播放列表，我会做的一件事就是听这个特定的播放列表。过了一段时间，这些歌就过时了，但节奏恰到好处，让我专注于跑步，所以我可以忽略任何其他干扰。

有时候我也很擅长编程。我会发现*我只想完成这个模块*,这样我就可以将它合并到另一个模块中，这样我就可以添加这个 UI，这样我就可以测试这个反应式代码...一件事接着另一件事，不知不觉我一天就写了 1000 行。我认为需要一点时间来充分了解一门语言，以便像那样自如地从一点跳到另一点，但是一旦你到了那个水平，看着事情自然流动真的很令人满意。无论你是在编码的时候听音乐，还是让网飞做背景音乐，或者完全安静地工作，记住这一点...

### 没有“一条正确的路”

几个月前，我在都柏林的一家跑步用品店买了我的第一双昂贵的跑鞋。它们的价格大约是我之前一双鞋的 5 倍，是我从百货商店买的一些斯凯奇鞋。我得告诉你，我真的分不清两双鞋的区别。他们在跑步商店做了步态分析，并声称他们改善了我的状态，使我不容易受伤，但我没有感觉到任何不同，或更快，或更少疲劳。差别大概可以忽略不计。

我可以很容易地穿着我的又旧又便宜的鞋子跑步，并且可能取得大致相同的成绩。有些跑步者穿昂贵的鞋子，有些穿便宜的。有些人戴着奇怪的手套(单个脚趾)，有些人光着脚。这些选择并没有本质上的优劣。只要你的状态良好，你可以穿着大多数种类的鞋跑得一样快，一样安全。

编码也是如此。只要你没有做任何不安全的事情(没有 SQL 注入漏洞，没有无限循环，没有越界数组访问)，并且你正在完成工作，那么*如何*完成工作就没有什么区别。编写代码的方式越来越不优雅，针对特定任务的合适语言也越来越少(试着在[空格](https://en.wikipedia.org/wiki/Whitespace_(programming_language))处重写 Linux 内核)，但最终，程序员的工作是让最终用户的工作变得更容易。如果用户可以在合理的时间内做他们想做的事情，那么你使用`vim`或者 VS 代码或者制表符或者空格或者`for`循环或者 lambdas 都没有关系。最终结果才是重要的。做任何感觉正确的事。

但我认为，跑步能教会编程最重要的一点是...

### 这是马拉松，不是短跑*

假设你现在 30 多岁，从统计数据来看，你还有整整 40 到 60 年的时间。那有很多时间。没有人指望你在 20 岁、25 岁或 30 岁时就知道你所选择的语言或框架的一切。人们每天都在谷歌上搜索东西，甚至是*专家*。甚至*简单的*事情。我总是忘记，当在 Linux 中制作一个软链接时，它是`ln -s <src> <dest>`还是`ln -s <dest> <src>`。你知道吗，不用谷歌？

我有一大堆想学的东西。我想在哈斯克尔变得更好。我想学汇编。我想用一个反应式 JavaScript 框架来重建我的网站。我想看更多关于软件开发和 UI/UX 和设计的书。我想去旅行。我想提高我的法语水平。我想拥有一只*最小*** 的二狗。

但是我明天不会出去做所有这些事情。

这需要时间。你需要计划、练习和耐心。不要被大量你想学的东西压垮，享受学习的过程。一天一天，一步一个脚印。
