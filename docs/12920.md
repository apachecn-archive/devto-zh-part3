# 提高你的 Wordpress 网站速度的 9 种方法

> 原文:[https://dev . to/sebtoombs/9-ways-to-speed-your-WordPress-site-ohe](https://dev.to/sebtoombs/9-ways-to-speed-up-your-wordpress-site-ohe)

## 让你的 Wordpress 站点更快

没有人喜欢一个缓慢的网站。加载或呈现页面的速度太慢会惹恼用户，并导致糟糕的整体用户体验。不仅如此，研究表明，页面加载时间对页面浏览量、跳出率和转化率有着直接的、可衡量的影响。根据 Akamai 和 Gomez.com 的调查，47%的用户希望在 2 秒或更短的时间内加载一个页面。40%的人确实会放弃一个加载时间超过 3 秒的网站。

所有这一切意味着，如果你的网站运行缓慢，你不仅会失去访问者，你会失去转化率，你还会失去可能的推荐，因为你当前的用户体验不佳。

我们将会看到你可以加速 Wordpress 并帮助你的用户创造更好的体验的最好方法，这将有望带来更多的页面浏览量和更高的转化率！

## [](#test-your-site-speed-and-performance)测试你的站点速度和性能

在您开始对您的站点进行修复之前，测量一些因素并确定您的站点有多慢可能是个好主意。

要衡量的一些关键因素包括:

**页面加载时间:**

这实质上是加载和渲染一个交互页面的总时间。有几种方法可以测量，并且给出的结果都略有不同，所以做一些测量是个好主意。

**测一下:**我个人最喜欢的页面速度工具是 [Pingdom](https://tools.pingdom.com/) 。输入您的站点地址，选择测试位置并运行测试。

**服务器响应时间:**

这是服务器响应一个请求所花费的时间。

**测量它:**一个有用的工具是 [Bitcatcha](https://www.bitcatcha.com/report/) ，它将从全球几个地方测量您的服务器的响应时间。请注意，最相关的结果是离您的服务器最近的结果。

## [](#1-slow-hosting)1。慢速托管

没有人喜欢听到这个，但是一个慢网站的最大因素之一是便宜的共享 Wordpress 主机，这是加速 Wordpress 的最简单的方法之一。当我们优化 Wordpress 网站时，我们经常看到这种情况。一个托管在廉价共享服务器上的站点几乎总是会有缓慢的响应时间。这听起来可能不是一个大问题，但是考虑一下这个；假设一个用户直接在他们的浏览器地址栏中输入你的域名，而他们以前从未访问过你的网站(绝对是最坏的情况)。这是大致如下的事件顺序:(这里我们将使用 nimblewebdeveloper.com 作为例子)

*   用户在地址栏中输入“nimblewebdeveloper.com”
*   客户端(浏览器，用户的机器)通过 DNS 查找“nimblewebdeveloper.com”找出服务器的 IP 地址
*   如果网站托管在比如说美国，而访问者在澳大利亚，这种 DNS 查找本身可能需要半秒钟(0.5 秒)，尤其是在共享 cPanel 服务器上
*   然后，客户端从 DNS 返回的 IP 地址请求“nimblewebdeveloper.com ”,并等待服务器的响应
*   你的网站应该使用 HTTPS，所以 Wordpress 将返回一个 301(重定向)来将客户端指向“[https://nimblewebdeveloper.com](https://nimblewebdeveloper.com)
*   客户端请求"[https://nimblewebdeveloper.com](https://nimblewebdeveloper.com)
*   Wordpress 通常不是很聪明，如果你的服务器使用的是 apache(大多数 Wordpress 主机都是这样)，Apache 要求在所有非静态资产的 URL 末尾加一个斜杠，所以服务器会返回一个 301 重定向到“[https://nimblewebdeveloper.com/](https://nimblewebdeveloper.com/)
*   客户端最终可以向服务器请求页面
*   服务器构建页面时客户端等待
*   服务器返回页面，客户端可以开始下载页面和资产

我想现在你已经明白了，对于第一次页面加载，最坏的情况可能是在客户端开始下载任何东西之前需要几秒钟。我们见过这样的情况，仅 DNS 查找和重定向链就超过了 4 秒，然后服务器开始构建页面！

## [](#2-poorly-optimised-images)2。优化较差的图像

另一个提高 Wordpress 速度的最好方法是给你的用户提供更好的图片。我们看到大量的网站，一个巨大的图像，通常以全显示质量，被发送给所有的客户，而不管屏幕大小，连接速度等。

这里先从简单的开始；图像至少应该针对网络进行优化，并且只能以实际需要的最大尺寸交付。大多数时候，可能不需要传送 8000 像素宽的图像。优化你的网络图片的一个简单方法是通过像 [ShortPixel](https://shortpixel.com/) 这样的服务来运行它们。ShortPixel 甚至配备了一个 [Wordpress 插件](https://wordpress.org/plugins/shortpixel-image-optimiser/)，可以为你做所有的工作。当你上传图片，压缩它们，并把它们放回你的媒体库时，它会把你的图片传送到 ShortPixel。他们有一个免费层，可以让你每月优化 100 张图片，但你需要创建一个帐户才能使用这个插件。专业提示:我们建议在大多数情况下使用光泽压缩。

**额外好处** : ShortPixel 可以配置为传送 WEBP 图像，这可以显著加快现代设备上的图像加载速度。

## [](#3-lazy-loading-images)3。延迟加载图像

图片的延迟加载是提高你的 Wordpress 站点速度的一个非常简单的方法。延迟加载包括延迟图像的加载，直到它们在浏览器的视窗中。要理解这是如何工作的以及为什么工作，重要的是要理解浏览器会试图尽可能快地加载网页中所有 **img** 标签的源。不管图像是否在视窗外 2000 像素，或者在视觉上被隐藏，浏览器都会在到达它时开始加载它。

我们如何解决这个问题？有很多方法可以执行一个惰性加载策略，但是本质上(在一个非常基础的层次上)它包括以下内容:

*   用占位符图像替换图像 **src** (通常使用透明的 1x1 px PNG)。这将阻止浏览器立即加载图像。
*   将实际图像源添加到一个**数据-** 属性中。这允许 javascript 找到源代码。
*   javascript 加载到正确的源中，最好是图像在视窗中或视窗附近。

我们可以手动完成所有这些工作，但幸运的是，有相当多的解决方案可以为我们完成这项工作。如果你正在使用 Wordpress，我最推荐的是 [A3 惰性加载](https://wordpress.org/plugins/a3-lazy-load/)。这是 Wordpress 知识库中的一个免费插件，可以轻松加载图片、视频、iframes 等。它有一些配置设置，但在大多数情况下，默认应该只是工作。

## [](#4-remove-unnecessary-amp-out-of-date-plugins)4。移除不必要的&过时插件

Wordpress 最棒的一点是能够通过插件轻松快速地添加额外的功能。不幸的是，没有多少插件写得特别好，往往会导致页面加载时间缓慢。

我最近优化了一个表现不佳的电子商务网站，发现通过禁用 3 个糟糕的插件，页面加载速度提高了大约 1 秒。当然，一些插件提供核心或必要的功能，所以很难删除插件。最好的办法是检查你所有的插件，评估它们有多重要，以及它们是否是最新的。任何过时的插件显然都应该更新！

## [](#5-compress-amp-combine-assets)5。压缩&联合资产

网站加载缓慢的一个常见原因是加载了大量的外部资源(通常是由于插件)。用户的浏览器必须下载这些资产中的每一个，这会降低页面加载速度，特别是对于呈现阻塞的资产(注意，如果服务器和客户端都支持 HTTP/2，理论上这不是一个问题)。

样式表和脚本也可以被压缩和缩小以减小大小。压缩和缩小删除所有不需要的空白，重命名变量等。使样式和脚本尽可能小。我们不会真的在这里讨论如何做到这一点，因为对我们来说幸运的是有插件可以为我们做到这一点！如果你还没有使用过 W3 总缓存，我们很快会有一个教程。

在 W3TC 中配置缩小应该像安装插件一样简单，在**常规设置下启用缩小，缩小**。当然，事情不一定总是按计划进行，所以要谨慎使用并测试！如果需要，您可以在**缩小**特定设置下进一步调整设置。

## [](#6-enable-caching)6。启用缓存

当我们谈到 W3 总缓存时，我们可以启用一堆其他选项来进一步加速 Wordpress。如果您在上一步中安装了 W3TC，您可以轻松启用的其他选项有；

#### [](#page-caching)页面缓存

当我们谈论缓存时，页面缓存通常是我们谈论的内容。互联网上有很多关于缓存的解释，但简单来说；如果没有缓存，每当你的网站上的页面被请求时，Wordpress 就必须做大量的事情来构建页面，比如获取文章内容，获取标题等等。这种东西在技术上是动态的，但是，除非你更新你的页面或帖子，否则它不会改变。因此，缓存所做的就是构建页面一次，存储构建的 HTML，然后在后续请求中发送缓存的 HTML，而不是再次构建页面。这里有一个小小的折衷，在第一次页面加载时(构建缓存版本时)，页面加载会稍微慢一些。然而，在随后的加载中，它应该会快得多。

在 W3TC 中，开启应该和勾选**通用设置**中的**启用**一样简单。需要注意的事项；启用缓存后，你对内容的更改不会显示在前端，除非这些更改从缓存中被*清除*。帖子内容等。应该自动清除 W3TC 当你点击更新。

#### [](#database-caching)数据库缓存

数据库缓存也可以在 W3TC 的**通用设置**页面中轻松打开。数据库缓存只是存储频繁的数据库查询，这样就可以立即返回数据，而无需接触数据库。

#### [](#object-caching)对象缓存

对象缓存有点类似于数据库缓存。数据库缓存缓存原始的数据库请求和结果，对象缓存缓存使用这些结果的 Wordpress 查询。这可以显著提高重复查询的动态性能。同样，只需在设置中勾选这个就足够了。

#### [](#browser-cache)浏览器缓存

W3 Total Cache 中的浏览器缓存设置允许你的 Wordpress 站点压缩发送到浏览器的静态资产，并且还添加了告诉浏览器可以存储这些资产多长时间的标题。如果你不知道；大多数浏览器会缓存资产，甚至整个页面，以加速网站的重复访问。

## [](#7-dont-use-page-builders)7。不要使用页面生成器

页面生成器主题/插件(Divi，WP Bakery Page Builder，Beaver Builder 等)对于没有开发技能的人来说是一种建立和管理网站的简单方法。不幸的是，我从来没有遇到过表现出色的页面生成器。在构建真正高性能的东西和可以通过用户界面轻松编辑的东西之间总是有一个权衡。不幸的是，对于如此灵活的页面构建器来说，这是一个巨大的性能损失。如果再加上经常不太理想的编码，结果就是一个非常慢的页面。当然，你可以尝试这篇文章中的所有其他技巧，我相信你会看到一些性能，但我几乎准备打赌它仍然不是很好。

反而；使用构建良好的主题。如果你不想从头开始创建一个，从一个好的主题开始，然后从那里定制，以获得想要的结果。还是雇个好的开发商吧！

## [](#8-use-a-cdn)8。使用 CDN

CDN 可以对静态资产的交付产生巨大的影响。CDN 从地理上分散的服务器网络提供您的内容(图像、样式表、脚本等),这些服务器针对静态内容进行了优化。我们已经做了一个关于如何将 CDN 添加到 Wordpress 的快速帖子，,所以如果你不确定，就去看看吧。
