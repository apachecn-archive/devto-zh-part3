# 我们是不是自命不凡，狂妄自大？

> 原文:[https://dev . to/lucagrandicelli/are-we-自命不凡且狂妄自大-361l](https://dev.to/lucagrandicelli/are-we-pretentious-and-arrogant-361l)

前几天，我在脸书的一个小组和另一个开发者进行了一次对话，他声称如果你的网站不处理任何数据传输，你可以忽略 HTTPS。

我回答说，即使“技术上”这可能是真的，但由于种种原因，事实并非如此。主要是谷歌只是惩罚不在 SSL 上的网站。

在这一点上，这个家伙继续以一种非常令人讨厌的/侵略性的行为争论，声称他做了数百次从 HTTP 到 HTTPS 的迁移，我可能是那些认为安全只是 SSL 问题的傻瓜之一，等等...

最终其他人也加入了讨论，站在了我这边。不需要，但我提醒了。

然后是一个简单的想法。

这并不是第一次，当与其他开发者讨论 IT 问题时，事情开始变得激烈起来，以一种咄咄逼人、我无所不知的语气出现。

似乎很多开发人员都迫不及待地想要炫耀他们的知识，甚至没有试图去了解他们的产品是否有缺陷。

在这个行业工作多年后，我唯一知道的是，我什么都不知道...

你有没有遇到过这样的情况？你认为我们作为开发者，会因为我们特殊的技能而变得有点自私吗？

告诉我，开发人员。让我知道。
