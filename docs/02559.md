# 对 2019 年创建课程的思考

> 原文:[https://dev . to/just alever/a-reflection-on-creating-a-course-in-2019-3 mpd](https://dev.to/justalever/a-reflection-on-creating-a-course-in-2019-3mpd)

2018 年末，我开始创建一个名为 Hello Rails 的新课程。这是对 2019 年创建课程的反思，以及一些关于我为什么的背景。

### [](#backstory)背景故事

创建这个博客大约两年后，我开设了一个 YouTube 频道。这是我第一次接触视频，并成为我的主要关注点。

当我在寻找我的观众/内容路径时，一些视频在我的频道中脱颖而出。Affinity Designer 和 Photo 是我当时的新设计工具。我不想每月订阅 Adobe 产品，而 Affinity 的工具*足够好了*,而且花费少得多。

这些应用程序越来越成熟，但在 Adobe 主导的设计世界中并没有受到太多的关注。如今，每个月都会有一款设计工具冒出来。

我提到 Affinity Designer 和 Photo 是因为我在我的博客/频道上记录了一些“如何做”的教程，这些教程得到了相当多的关注。这种关注让我产生了在网上创建一个名为“亲和者”的空间的想法。这个网站将是一个针对 Affinity-Serif 产品的教程集合。

在我的蝙蝠洞里，我花了几个月的时间来设计和构建应用程序的 MVP。回想起来，我在这个阶段花了太多时间来验证我的目标受众是否会为这样的服务付费。经过几个月的努力，我推出了当天产品搜索排名第二的应用。这次发布很成功，但是它缺少的是付费用户。该应用程序具有按月、终身和自选支付模式。用户可以订阅/购买内容来下载资源和查看教程。

经过大约一年的时间和大约 200 个教程之后，我决定不再做亲和论者。这是一个艰难的决定，但最终，我创作内容的工作量让我筋疲力尽。这一切都发生在我做全职工作和抚养我年幼的儿子的时候。

#### [](#lessons-learned-the-hard-way)痛定思痛

尽管我将 Affinicasts 标记为失败，但它确实从社区获得了一些积极的反馈。在开发这个应用的时候，我也学到了很多关于 Ruby on Rails(我用它创建的 web 框架)的知识。

Ruby on Rails 触动了我渴望已久的神经。我是那些在 WordPress 工作多年的开发者/设计师之一。PHP 不是我最喜欢的语言，但它确实完成了工作。我想拥有比 WordPress 更多的控制权。由于厌倦了 PHP，我立刻喜欢上了 Ruby 和 Ruby on Rails。代码的语法和非常清晰的语言让人耳目一新。

#### [](#teach-what-you-want-to-learn)教你想学的东西

我总是听说要学好一件事，你需要教它。即使你不是 100%精通此事，你也可以磕磕绊绊地完成并得到需要得到的东西。这可以让你和你的观众一起更好地理解事情。

> 我总是听说要学好一件事，你需要教它

那时，我决定学习更多关于 Ruby on Rails 的知识，于是我开始在我的 YouTube 频道上教授这个框架。

这个框架成为焦点已经有一段时间了，并且在 YouTube 上受到了很多人的称赞。今天，我已经发布了 70 多个关于这个话题的视频，并继续制作更多的内容。这样做的时候，我注意到我的 YouTube 频道的订阅用户每天都在增加。我现在平均每天有 10 个用户，这对于我这样的无名小卒来说简直是疯了。如果你正在读这篇文章，并且你订阅了这个频道，我对你感激不尽！

### [](#why-a-course)为什么要上一门课？

现在是 2019 年。你看到的大多数待售软件都遵循 SaaS 模式。这通常意味着持续的每月价格标签。我本来可以尝试另一个 SaaS 应用程序，如 Affinicasts，但我渴望更容易接近的东西。SaaS 太难了。除非它是用户真正需要/使用的产品，否则每个月你都有流失(退订)的风险。试图兼顾这听起来非常有压力，并不是我想再次探索如此接近亲和日落的东西。

信息产品公司在网页设计和开发领域做得非常好。看到像[韦斯·博斯](https://twitter.com/wesbos)、[亚当·瓦森](https://twitter.com/adamwathan)和[史蒂夫·肖格](https://twitter.com/steveschoger)这样的人创作书籍、课程和内容，不仅帮助了社区，还为自己创造了事业，真是令人鼓舞。

我不想和他们一样大。其实我也不想当。我正在努力变得更加自立。创业是我的一大爱好。自己做主并从自己的错误中吸取教训是一件令人激动的事情。如果你能挥动它，它会使 9-5 少一点阻力。如果我说我不想成为一名全职企业家，那我是在撒谎。

我开始创建自己的课程，名为 [Hello Rails](https://hellorails.io) 。Hello Rails 是一门基础的 Ruby on Rails 课程，它通过例子向你教授框架。由于我的 YouTube 频道上的反馈/关注越来越多，我开始制作这门课程。我在 YouTube 上的内容足以满足你的胃口，但永远不会完整。该课程旨在采用一个概念并构建一个全功能的应用程序，该应用程序的部署就像一个真正的 Ruby on Rails 应用程序在野外一样。

### [](#but-thats-already-been-done)但那已经发生了

野外有大量的学习资源。臭名昭著的[railstutorial.org](https:///railstutorial.org)是大多数开发人员所知道的，并向其他人指出。gorails.com 的[是扩展框架知识的另一个重要资源。那么，为什么是另一门课程/资源呢？](https://gorails.com)

当我开始学习编码时，我渴望更深入的教程或课程，而不是编辑错误。一些视频教程非常精美，看起来就像一个机器人在教你很难理解的概念，但你很快就会忘记。

像杰弗瑞这样的老师在扩展硬概念方面做得非常好。他在 laracasts.com 的 T2 继续这样做。

我从他的风格中得到启示，在我的 YouTube 频道上加入了我自己的教程。我不想编辑大量耗时的东西，而是想捕捉每个教程中需要做的事情的“真实性”。我的课程也不例外。Hello Rails 包含 90 个视频，27 个模块，18.5 小时的内容，以及一个完整的图书版本。如果这还不够彻底，那我不知道该告诉你什么🙂

我希望我的风格和内容可以帮助学生更好地理解 Ruby on Rails 框架，并且带着信心离开课程去构建他们自己的应用程序。这种类型的感觉是我学生时代渴望的，但如果不努力建立自己的东西，我永远也找不到。

最终，构建更多的东西和实践将永远让你在软件领域更有知识。再多的课程、书籍、教程或讲座都无法让你为未来做好准备。从错误、失误和同伴中学习会给你留下印象，如果这些印象再次出现，你知道如何处理。

这就是教与学的意义所在。

### [](#the-road-ahead)前路漫漫

在撰写本文时，Hello Rails 仍未推出。我正在对我为这个课程构建的小框架做一些最后的调整，祈祷 Rails 6 很快正式发布。学生将有一个仪表板区域，他们可以在那里访问他们的课程材料。如果他们是一个团队，他们也可以邀请队友。这是一件有趣的事情，但需要一些关注。

这门课程花了很多时间和精力。我可能做得有点过头了。我从来没有真正打算创作一本书的版本，但我需要一个课程大纲。那份大纲变成了书面版本。我已经从这个过程中学到了很多，但对未来感到兴奋。我做的这个小框架有望被重新用于我将来可能尝试的课程理念。这就是建造它的初衷。

在努力变得透明的过程中，我的观众并不像一些大男孩那样多，但我对此没意见。现在，我的电子邮件名单上有 520 个订户，这还不错！

我很可能会打开这个列表，关注我的 YouTube 频道。然后，稍后做一个正式的产品搜索发布。营销将是关键，但我希望使这个过程更加“常青”。我的意思是保持它的当前性和长期性，这样将来任何人都可以利用它。稳扎稳打赢得比赛！

### [](#what-would-i-change)我会改变什么？

我可能会将课程的范围变得更小，或者以某种方式将它分成几个部分，这样人们就可以抓住一大块，学习它，然后继续下一课。

课程是巨大的。我认为它可能有点太大了。Ruby on Rails 是一个庞大的框架。这门课程仍然没有涵盖所有的内容。这为我将来的课程/教程打开了空间。

除此之外，我觉得我做得对。我没有向空空的观众发布，我围绕它建立了一些媒体(有很多事情要做)，并获得了一些明确的兴趣(仍然继续获得一些)。我不能要求更多了。好吧，也许一个平稳的发射…

我们拭目以待吧！
