# 拥抱复制/粘贴

> 原文：<https://dev.to/molly/embrace-the-copypaste-3epf>

我不止一次被问及从别人那里复制粘贴代码是否可以。我的反应通常是，“当然！”尤其是当你刚开始学习的时候，寻找代码片段来帮助你开动大脑，让你熟悉一门语言是一个很好的学习方法。即使代码没有完全按照您的要求运行，有时它也能帮助您找到正确的方向。

然而，当你找到一个代码片段时，不管它是来自 stackoverflow 还是你自己公司的代码库，在你使用它之前，一定要理解它在做什么！

[![](img/383f8106a03c2448289e693a7c07f214.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--24fyjEc9--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/2ktzwtymqp90ewx59qje.gif)

盲目地复制和粘贴代码并不能真正帮助你学习。你需要阅读代码并弄清楚它在做什么。刚开始的时候，阅读别人的代码对学习语法有很大帮助。我仍然通过阅读别人的代码来学习新的 Ruby 技巧。

## 并非所有代码都是生而平等的

同样重要的是要记住，并非互联网上的所有代码都是“好代码”

[![](img/2de7d38a0309f879574fc2ff3d8a0659.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--liM6bTQO--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/nhzjjaduk7ff3d118d52.gif)

令人震惊，我知道！但是说真的，当我说“好”的时候，我指的是一系列不同的事情。代码可能写得非常干净，但是性能很差。另一段代码可能看起来像是拼凑起来的一团乱麻，但该死的它工作得很好。如果你能理解一段代码是如何工作的，那将会更好地帮助你决定这段代码是否适合你。如果你不确定，问问别人！找一个同行，或者比你更资深的开发人员，让他们和你一起看代码片段。

如果您需要自己确定一段代码是否是您想要使用的代码，下面是一些基于代码片段来源的考虑事项。

*   **堆栈溢出:**
    *   问题有多老了？答案多大了？
    *   答案有多少上票数？
    *   问题或回答是否提到了使用或编写该代码的任何特定软件版本？
*   **Github Gist 或 Repo:**
    *   gist 或 repo 有几颗星？
    *   这段代码的最初目的是什么？
    *   有没有对 gist 或 repo 解释其用法的评论？
    *   维护者/作者是一种怎样的体验？
*   **博文:**
    *   代码是在什么情况下使用的？它是否符合您将如何使用它？
    *   博文是什么时候写的？
    *   博客的作者是谁？

如你所见，在决定一段代码是否适合你时，有一点趋势。

1.  它有多老了？
2.  最初的用途与你想做的事情一致吗？
3.  作者有什么凭证？
4.  你们的软件版本相似吗？

当您寻找第三方代码时，请记住这些问题，您可能会找到一些有用的代码片段😃

## 复制/粘贴不仅仅适用于初学者

我从 2012 年开始写代码，现在还在复制粘贴代码。我发现复制和粘贴代码最有用的时候之一是当你在使用一种你不太熟悉的语言或技术时。对于 HTML、CSS 和 Javascript 等前端技术，我经常这样做。我没有在正常的基础上与他们一起工作，所以我的语法知识很薄弱。对我来说，找到一个代码片段并调整它比从头开始编写自己的代码要容易得多。

我最近成功复制和粘贴的一个例子实际上是 bash 脚本。我需要一个 bash 脚本来重新索引一些 Elasticsearch 索引。我可以在睡觉的时候写出一个完整的 Ruby reindex 脚本，但是我没有写过很多 bash。所以我谷歌了一下，“Elasticsearch reindex bash script”然后你瞧，网上发来了！我发现了这个[方便的重新索引脚本](https://gist.github.com/hartfordfive/e507bc47e17f4e03a89055918900e44d)，我可以用它来做重新索引。我对脚本做了一些修改以适应我的需要，但除此之外，基本上还是照原样使用。我完全理解这个脚本在做什么，但是如果你让我在 bash 中从头开始写，我可能要花一整天的时间来解决格式错误。

## TL；速度三角形定位法(dead reckoning)

不要害怕复制和粘贴！拥抱它，把它作为你写代码时可以使用的另一个工具。像任何其他工具一样，聪明地使用它。如果使用得当，它可以教会你很多东西，让你更有效率！
