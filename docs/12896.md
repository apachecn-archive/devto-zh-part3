# 什么是公社？组和角色

> 原文:# t0]https://dev . to/limawi/cest-what-une-comm-groups-and-roles-2me 2

登录到服务后，您不一定是当前唯一的用户。如何正确管理您可以对服务执行的操作，以及其他用户可以(或不可以)对该服务执行的操作？

该服务实施访问控制。通常，它实施组或角色访问控制。对于社交网络，还有关系访问控制。

## [](#le-contr%C3%B4le-dacc%C3%A8s-de-groupe-group-based-access-control)群组存取控制(group based access control)

这是特里布。

此访问控制是一组有权访问若干资源的用户。

资源使用权限(权限)的集合。该组中的用户将继承每个权限。

## [](#le-contr%C3%B4le-dacc%C3%A8s-de-r%C3%B4le-role-based-access-control)角色访问控制(role based access control)

他是部落首领 shaman ...

此访问控制设置权限(权限)，这些权限将全部集合在一起，并授予特定用户。

通常，管理员角色具有所有权限。所有其他角色均由该管理员授予用户。用户不能为自己指定角色。

角色可以分层组织。上方角色包含下方角色的所有权限，并拥有其他权限。

## [](#la-diff%C3%A9rence-et-la-relation-entre-les-deux)二者的区别和关系

比较这两种类型的访问控制时，可能难以区分它们。根据不同的制度，这种区分并不一定适用。

但为了澄清这一点，我将尝试作出这种区分。

组通常用于对资源和用户进行分类。他们管理身份。该用户属于特定的组。

角色通常用于管理权限、权限。他们经营业务。这样的用户可以这样做，因为他们有这样的角色。

组和角色是相互关联的，因为它们是互补的。该组用于定义角色的权限范围，即角色可以操作的用户和资源。但是，一个角色可以存在于多个组中。

例如:

**脸书**

你可以成为树莓崇拜者团体的成员，喜欢*成熟的崇拜者网页*。从访问控制的角度来看，您是两个组的成员。

在*树莓粉丝群*中你是会员，在*成熟粉丝网页*中你是主持人。这是你的职责。

正如您所看到的，您的角色由组限定。你是*成熟仰慕者页面*的主持人，没别的事。但是，可能会有一位*森林迷团体*的主持人，他对该团体的权限与你在页面上的权限相同。因此，角色可以存在于多个组中。

同样，组成员资格也为您提供了默认权限。作为“*树莓仰慕者团体”*的简单成员，您有权访问该团体的出版物。如果您不是成员，则无法访问。

**一台电脑**

组和角色之间的分隔很模糊(特别是在 UNIX 上)。但是，您可以将计算机定义为自己的计算机-即使该计算机上的组和用户帐户也具有角色(通常一个是管理员，所有计算机上都具有该角色)。

## [](#contr%C3%B4le-dacc%C3%A8s-par-relation-relation-based-access-control)基于关系的访问控制

他们是你的朋友或者社交网络上的联系人

你和你的一个朋友之间的关系的存在给了你和这个*朋友*管理双方资源的权限。

关系管理权限。这些权限可以是对称的(“T0”朋友拥有与你相同的权利)，也可以是不对称的(“T2”朋友除非得到比你更少的权限，在这种情况下是加上*粉丝*或等级关系。

## [](#contr%C3%B4le-dacc%C3%A8s-par-aptitude-capability-based-access-control)基于能力的访问控制

这种类型的访问控制允许对权限进行更精细的管理。

权限附加至资源(物件)，而非资源群组(如同群组或角色存取控制)。“对象-权限”集合构成了一种技能。

如果用户具有相应的权能，则该用户可以使用设置的权限访问该对象。

这使得权限管理更加精细，因为权限管理是按资源(按对象)进行的。

此访问控制由[oath 2](https://blog.limawi.io/fr-fr/posts/les-social-logins-comment-ca-marche)(用于社会登录的机制)用于权限管理。例如，在网站上使用 Facebook 社交登录时，Facebook 会通知您该网站希望访问您的电子邮件、朋友列表或其他信息。

这是您是否将(或部分)权能委托给 Facebook 资源上的站点(在本例中是您的配置文件)。

在[myrypnet](https://mycrypnet.io/fr/)中，每个 myrypnet 访问都具有由 myrypnet 访问管理员创建的若干相关权限。然后，管理员将权能委派给用户。因此，用户可以使用管理员设置的权限管理访问。
