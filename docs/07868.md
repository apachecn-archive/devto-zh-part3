# 什么是版本控制？

> 原文:[https://dev.to/codetips/what-is-version-control-22d3](https://dev.to/codetips/what-is-version-control-22d3)

> 版本控制帮助您跟踪数据。

[![What is Version Control?](../Images/12fb3e367d2a765e3a5e7172e14dca77.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--uwe3lsIf--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://images.unsplash.com/photo-1486150702851-498e985c2efc%3Fixlib%3Drb-1.2.1%26q%3D80%26fm%3Djpg%26crop%3Dentropy%26cs%3Dtinysrgb%26w%3D1080%26fit%3Dmax%26ixid%3DeyJhcHBfaWQiOjExNzczfQ)

我们假设你写了一份简历。你重写或更新了多少次？

如果你想回到十年前的版本呢？你可以在每次做出改变的时候创建一个全新的版本，但是你最终会得到成百上千的简历。

如果你想比较你十年前的版本和你今天的版本呢？如果你已经分别保存了每个文件，你可以把它们并排放在一起打开，然后找出答案，否则，你就不走运了。

或者，您可以使用版本控制。

让我们假设你十年前创建了你的第一份简历。您将它保存在计算机硬盘上，并将其添加到您的版本控制系统中。

下次您需要进行更改时，您可以覆盖硬盘上的版本，并将最新的更改(带有解释您所做更改的消息)添加到您的版本控制系统中。

十年过去了，你已经这样做了几百次，但你仍然只有一个文档。因为您将每个变更添加到了您的版本控制系统中，所以您现在有了变更的“历史”。

```
| ID |       Date         |          Message                |
+----+--------------------+---------------------------------+
|2410|2019-04-01 09:12:00 | Added latest role/company       |
+----+--------------------+---------------------------------+
|2409|2019-03-21 12:25:00 |Fixed typo in personal statement |
+----+--------------------+---------------------------------+
|2408| 2019-01-11 19:01:00| Removed education section       |
+----+--------------------+---------------------------------+

......

|0001| 2009-04-13 15:29:00| First commit                    |
+----+--------------------+---------------------------------+ 
```

现在让我们使用版本控制系统，从头开始回答我们的问题。

**如果你想回到十年前的版本会怎样？**

版本控制系统将允许您返回到特定的版本。每个系统都有自己处理这种任务的方式，这里有一个使用[伪代码](https://codetips.co.uk/beginner/what-is-pseudo-code/)的例子:

```
checkout 0001 
```

这将把我们拥有的文档更新到 2009 年的确切状态(ID `0001`)。您可以使用相应的 ID 轻松地“签出”任何其他版本。

如果你想比较十年前的版本和今天的版本，会怎么样？

版本控制系统允许您查看单个文件或整个项目在两次保存之间的更改。同样，每个版本控制系统都有自己处理此类任务的方式，但这里有一个例子:

```
diff 0001 2410 
```

这将向您展示`0001`和`2410`之间的所有差异(添加、修改和删除)。这并不意味着过去十年中的每一个变化，只是两个版本之间的差异。

在我们结束这篇文章之前，我想澄清一件事。

`git`不是版本控制。许多人交替使用这两个术语，但是你应该知道它们的区别。

这是版本控制的一种形式。其实是非常流行的版本控制系统，但是还有其他的。

相比之下，谷歌 Chrome 是一款非常受欢迎的浏览器，但还有 ie、Firefox 等。

这不是一篇关于什么是`git`或者如何使用它的文章，这是一篇关于什么是版本控制的文章。
