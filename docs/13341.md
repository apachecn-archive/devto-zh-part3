# 阴影的危险

影子 IT 是一个术语，通常用来描述未经明确的组织批准而在组织内部构建和使用的信息技术系统和解决方案。它还与术语“隐形 It”和“客户端 IT”一起用于描述由 IT 部门以外的部门指定和部署的解决方案。一度，影子 IT 仅仅是员工在 Excel 中使用特殊或未经批准的宏来更快地完成工作。现在，影子 IT 可以让一小群员工安装他们自己的文件共享服务，而没有意识到他们正在让自己的公司面临重大诉讼和数据泄露。

公司如何养成自己的影子 IT 习惯？有时，最终用户会很固执，不愿意听从 IT 规则和政策。虽然有时可能会出现这种情况，但由于 IT 部门不能或者在某些情况下不愿意帮助解决其组织内的工作流和效率问题，影子 IT 更经常出现。如果一个部门或团队对他们当前的工具和工作流程感到足够失望，他们通常会找到一个变通办法来避免给它开罚单，因为他们觉得它不会得到答复。

如今，人们关注的一个主要领域是基于云的服务的快速采用。随着信息技术的消费化，影子 IT 的增长速度加快了。用户已经习惯于从云中下载和使用应用程序和服务来帮助他们工作。这在创意和产品部门中很常见，在这些部门中，不断有文件共享，而他们工作流程中的应用程序或服务不够充分。我已经数不清有多少次发现 XYZ 的公司资产通过约翰·史密斯的个人 Dropbox 账户被上传并分享给客户，这对公司和客户都是一个巨大的风险。

随着 IT 的消费化，典型的企业中使用了数百个这样的应用程序。缺乏对这些应用程序的可见性也是一个主要的安全漏洞。有许多文件共享应用程序和服务向最终用户工作站开放端口，使他们容易受到外部威胁，如果该工作站在公司网络上，则整个公司都会面临风险。虽然大多数应用程序是无害的，但有许多应用程序在未经最终用户同意的情况下添加后门甚至加密挖掘软件的报告。IT 和安全部门需要知道正在使用哪些应用程序，以及它们会带来哪些风险

为了解决这一问题，IT 部门需要与最终用户召开战略会议来解决棘手问题。最终用户正在规避 IT 管理，因为他们希望在高效、简化的工作流程中完成工作，但却缺乏适当的工具来提高工作效率。防止影子 IT 的一些步骤包括但不限于:

*   *首先确定导致用户需要走影子 IT 路线的政策和程序中的弱点。*
*   *与喜欢规避 IT 的部门主管和最终用户重新建立关系，并尝试确定需要补救的痛点。*
*   *强调 IT 是业务基础架构中技术的唯一把关者，在实施之前需要向他们提供解决方案和服务。*

采取主动措施将有助于减少影子 IT 实例，并降低贵公司的漏洞风险。不要等到为时已晚。如果你想不出解决方案，别人会的，而且这可能不是解决问题的安全方法。
