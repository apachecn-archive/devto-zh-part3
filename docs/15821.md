# 从 SQL 开始

> 原文:[https://dev.to/flippedcoding/starting-with-sql-2l1k](https://dev.to/flippedcoding/starting-with-sql-2l1k)

在 web 开发人员的一生中，您可能会看到或使用 SQL。你可能会更深入地研究它，或者你可能只在别人没有时间写它的时候才使用它。不管怎样，对这种数据库查询语言有一点了解是有好处的。

首先介绍一下数据库的背景。数据库用于保存不同的列和数据表。用户名、密码、生日、信用卡号和所有其他数据都存储在这里。有使用键在表之间建立连接的关系数据库，也有更多用于大数据的非关系数据库。

## [](#what-is-sql)什么是 SQL

SQL 是关系数据库的查询语言。它是一种语言，可以让您根据自己设定的需求获得所需的数据。几乎所有的关系数据库都使用 SQL。这意味着这是您在 MySQL、Microsoft SQL Server 甚至 Oracle 中使用的同一种语言。可能会有一些轻微的语法变化，但是一旦你知道了 SQL，你就不用花太多时间来运行任何东西。

## [](#what-does-it-do)它有什么作用

您可以只读取数据，可以创建新数据，可以更新数据，也可以删除数据。使用 SQL，您可以对数据库进行更改，并以任何方式使用信息。这就是当你在网站上忘记密码时，如何重置密码的方法。代码中的某个地方有一个 SQL 查询，用于查找您的特定数据并进行更新。

如果您需要在后端对数据进行过滤或某种计算，SQL 可以让您做到。您可以使用几个 SQL 脚本在后端完成排序和开票。它对于处理您可能必须向其他人展示的任何日常报告也非常有用。你甚至可以制作一个表的视图来得到特定的列和一些其他高级的东西。

## [](#when-is-it-used)什么时候用

一个缩写，API。对于那些负责为他们的应用程序编写 API 的人来说，SQL 是一个好朋友。有时，您希望前端开发人员能够处理一些数据，而不允许他们访问整个数据库。这就是 API 如此受欢迎的原因。您可以控制用户看到的所有数据，以及在数据到达他们手中之前如何处理这些数据。虽然 SQL 不仅仅用于 API。

如果你做任何报道，你肯定会看到它。在报告编写方面，SQL 几乎是无可替代的。有人会想知道某个客户在 3 个月内没有回复电子邮件的未结余额，或者其他类似的具体情况。大多数情况下，当需要回答一个问题时，您可以想到使用 SQL。从数据中得到的详细答案还是有点奇怪。

## [](#how-is-it-used)如何使用

它就像其他代码一样编写。有时候后端开发者表现得好像是别的什么东西。🙃但实际上，您可以在用于编写其他代码的编辑器中编写它。你需要注意的是你的数据模型。如果您知道哪些外键对应于哪些表，并且对数据库布局有一个大致的了解也不会有什么坏处，那么这将是值得的。

大多数情况下，如果有一个过程您会多次使用，您会将它保存为存储过程。这样你就有了一个小脚本，你可以在任何需要的时候运行它，也可以在其他脚本中使用它。试着把你的 SQL 写清楚，因为有一天你会回来看它，如果你能通读它并弄清楚你在做什么和你为什么这么做，它会有所帮助。

## [](#why-its-important)为什么很重要

我相信您可以看到 SQL 如此重要的许多原因。这就是你使用关系数据库的方式，到目前为止，关系数据库是数据库的主要类型。SQL 是一种让你在 Twitter 上创建新用户资料的语言。

它是显示你银行账户余额和所有交易的语言。这就是你如何删除你不再使用的奇怪的 Tinder 个人资料。实际上，正是这种语言让你不断地更改密码，因为你永远也不会记住它。这就是 SQL 如此重要的原因。
