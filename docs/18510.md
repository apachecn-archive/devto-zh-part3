# 什么是数据工程？

> 原文:[https://dev . to/the jessleigh/what-even-is-data-engineering-3g5m](https://dev.to/thejessleigh/what-even-is-data-engineering-3g5m)

早在 9 月份，我就开始了我作为数据工程师的第一份工作。我不太确定这份工作需要什么，但我准备转移我的注意力，学习新的东西。我花了四年时间进行后端应用程序开发，虽然我学到了很多东西，但我已经准备好脱离主要消费和构建 API 的工作。

当我面试这个职位时，我仔细阅读了 Dag、ETL 管道和特定的常见数据库模式，如[星型模式](https://en.wikipedia.org/wiki/Star_schema)。然而，在我的采访中，我被问到“如何优化这个 SQL 查询”和“假设您没有自动负载平衡器，您如何决定在哪个集群中创建新的数据库？”这些问题很有趣，所以我决定接受这份工作。但是，即使对其他工程师来说，也很难解释清楚我的工作范围。

我的团队负责数据库基础设施。我们确保数据库集群不会过载。我们关心内部数据安全。我们管理数据库用户和权限，以及来自不同团队的现场请求。我们是商业智能团队的信息收集部门；如果他们需要每 24 小时从特定的 API 转储数据，我们编写 ETL 管道来完成，并管理所有信息所在的数据仓库。我的工作就像是一个大杂烩，装满了需要完成的事情，但可能没有一个明确的主人。

我知道我可能没有很好地回答“什么是数据工程？”因为我不确定我了解自己！最近，我在当地的 Python 聚会上遇到了[阿里](https://dev.to/alysivji)，我们对数据工程学科似乎完全是虚构的这一事实产生了共鸣。

然而，在这个系列中，我将从数据工程的角度介绍我在工作中使用的不同工具。如果你有具体的问题，或者想了解数据工程师常用的具体工具，请在评论中告诉我。数据工程是软件开发中一个相当新的领域，其界限仍在划分中。让我们一起来找出答案吧！
