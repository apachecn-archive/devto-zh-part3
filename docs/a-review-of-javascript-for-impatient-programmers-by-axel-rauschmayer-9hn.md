# 评 Axel Rauschmayer 的《面向缺乏耐心的程序员的 JavaScript》

> 原文：<https://dev.to/zkadish/a-review-of-javascript-for-impatient-programmers-by-axel-rauschmayer-9hn>

这本书以一个关于的章节和 JavaScript 的历史开始，这是典型的 Brendan Eich 创建 JavaScript 的速度如此之快，以至于仍然让人头晕目眩。这样一来…

我的第一印象是，这本书内容广泛，在某种程度上揭示了那些你不会每天都遇到的边缘案例，但当你遇到时会很痛苦。对我们这些埋头苦读代码的人来说，这很好。你将会接触到日常工作中看不到的 JavaScript，这对程序员(我)来说是不可思议的。更令人兴奋的代码示例之一是通过对象方法使用 IIFE。是的，你可以通过一个对象方法建立一个生命…很有趣吧！

我喜欢这本书的另一个方面是练习是以单元测试的形式进行的。对我来说，这又是一个“哇，太酷了”的时刻。我没有编写过大量的单元测试，对于单元测试，我仍然抱着“我不想”的态度。完成这些练习将迫使像我这样的人在我职业发展的这个领域中成长。如果你喜欢单元测试...这是一个很好的方法…对吗？

整本书还有小测验。这些测验单独存在于另一个文件夹中，可以与阅读书籍分开进行。如果你没有耐心或者已经进步了，:D，你可以参加小测验，看看你的知识薄弱之处，然后只读那些章节。我认为这种格式很聪明…我正在阅读整本书…

Axel 也给了我们他对 JavaScript 的看法和观察。这是非常好的，它给阅读材料带来了个人的感受。例如，在解释“typeof”操作符时，他列出了相应类型的结果，并指出 null 返回“object”而不是 null。他将此描述为一个 bug，因为它不遵循其他类型的模式。他还注意到，当检查函数时，它返回“function ”,这打破了模式，并建议最好返回“object ”,因为这将遵循对象和数组设置的模式。

这本书确实从你可能称之为开始的地方覆盖了 JavaScript。它从值、类型和原语开始。你知道基础知识，然后一次一个主题地学习语言的其余部分。可调用值，你知道函数和作用域。这本书的流程很流畅，从简单的结构到每章复杂的结构。非常适合初学者！从那里开始，这本书涵盖了对象、原型和类。然后直接进入同步迭代，数组和析构。他还集成了所有的概念和词汇，你可以从一个经验丰富的 JavaScript 程序员那里得到。这本书的最后一个 JavaScript 部分介绍了调用堆栈、事件循环、承诺和异步函数。

他称最后一个部分为“杂项主题”，在那里他回顾了打字稿符号。他早先通过将 TypeScript 符号编织到他的例子中引入了 TypeScript 符号。在这里，他像在前面的 JavaScript 章节中一样介绍了它。学习 TypeScript 在我的任务清单上，所以这对我来说真是一种享受。

我发现当这本书让你做一个小测验时，小测验的名字却没有出现，这有点令人恼火。我发现自己不断地回卷章节，寻找测验的名称。不适合停下来，做些别的事情，然后重新开始。但最终“一旦我进入了书的流程，就很容易浏览了。

我更想要的是例子！我喜欢将编写单元测试作为学习材料的一种方法，我觉得有些测试和测验有点偏轻，我想要更多！

还没完呢！当我着手对这本书进行评论时，我就知道了这一点。然而，当你玩得开心的时候，你不想让它结束！

简单的描述和可读性很强的解释是业内最好的。这 40 块很值。我想要更多，并期待着这本书的下一个版本。爱死了。
