# 把敏捷做的很完美？那么，你做错了！

> 原文:[https://dev . to/dariusx/doing-agile-perfective-then-you-do-it-wrong-3 klf](https://dev.to/dariusx/doing-agile-perfectly-then-youre-doing-it-wrong-3klf)

很多团队都在销售敏捷。每个公司的做法都有所不同。有些人在深水区潜水，有些人则是小步前进。通常采用的是:短跑和站立/争球。接下来，也许是:整理故事，冲刺结束时的回顾。

围绕着敏捷的各个方面，已经有了很多很好的想法和技巧。例如，以 scrum 为例:它应该是一个快速的会议，为接下来的一两天设定团队的环境。这不是真正解决问题的会议。通常，团队中只有几个人直接涉及到一个问题。让其他所有人都监听可能是浪费时间。有道理。

然而，如果你过于严格地遵循这种“不讨论问题”的方式，你就没有做到敏捷。因为规则是过程的一部分，而敏捷与过程无关。如果规则是一个好主意，那么遵循它是有意义的...遵循它将是自然和富有成效的。如果你突然觉得需要打破规则，...如果此刻这对你有意义的话...你应该打破它。

当其他人违反规则时，您可能会认为“这不是敏捷”。停下来。抓住你自己。这种思维是面向过程的。相反，如果你认为“在这里讨论这个会浪费别人的时间”...那很好。

综上:**不要*做敏捷*，而要*做敏捷*** 。
