# 抛硬币:开发者应该设计吗？

> 原文：<https://dev.to/papaponmx/flipping-the-coin-should-developers-design-26ie>

我们都知道并且对设计师编码有不同的和松散的看法。但是反过来呢？

就我个人而言，我在学校和纳税系统中尝试导航和执行关键任务时都有过不舒服的互动。

当时感觉就像诺曼门，特别是当我被要求创建一个真正强大的密码，以保护关键信息。提交表单后，用户没有得到任何反馈。经过进一步检查，a 意识到向服务器发出了一个获取请求，它有大约 5 秒的延迟。如果事情出错了(他们确实出错了)，用户不会得到任何反馈。真的非常非常痛苦。

当你雇佣一个开发人员时，你并不期望她去设计，然而一些 UX 或用户界面设计总是很受欢迎的。你有什么看法？
