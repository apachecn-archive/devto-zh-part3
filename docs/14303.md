Glitch 是我在这个世界上最喜欢的应用之一。开玩笑都不行！如果你没有听说过 Glitch，你需要尝试一下。就像现在的*。Glitch 是一个让你轻松构建和发布网络应用的平台。它提供了一个学习和实验的绝佳机会。*

 *> 那么，如何开始使用 Glitch 呢？

1.  前往[https://glitch.com/](https://glitch.com/)
2.  从**新项目**中选择一个模板并编码！

[![](../Images/19d30ec6fb393aa2871c242b2ccba07c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--7SrFkBvq--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2ARrDT9wZA0CcGvLm0iYFZhQ.png)

等等，就这样？ ***是的*** ，这就是小故障的妙处。无需在本地下载、安装或运行任何脚本。你可以从一个模板开始，有数百个模板可供选择。它提供了一个不错的代码编辑器、控制台、日志和文件目录。你猜怎么着？我还没有登录！

[![](../Images/aa8349d39434c76a9570ecfb62fd45a3.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--e1iBIdFS--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2AaJQITFWPFmjy8_6CkpXVFw.png)

***但是等一下*** ，开发者难道不爱自己的本地 ide 吗？你是说我不能再用 VS 代码牛逼了！ ***嗯*** ，很高兴你问了。Glitch 中有一个非常方便的功能，可以让你将项目导出到 Github 或者下载到你的本地机器上。所以本质上，你可以在本地和云开发之间来回切换。
