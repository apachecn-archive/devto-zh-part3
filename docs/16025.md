# 编辑和校对的重要性

我经历过一次又一次，许多开发人员不校对他们的工作。我不知道为什么有些开发人员在没有完成编辑和校对这一重要步骤的情况下，就迫不及待地将他们的代码提交审查。

你还记得在学校时，当你完成论文或研究报告，你编辑和校对，然后提交对吗？我相信这也应该适用于我们的源代码。在某种程度上，源代码是开发人员的书面交流形式。

在完成准备提交或代码审查的部分工作后，考虑执行以下操作。

## [](#editing)编辑:

*   你的代码是否恰当地集成到整个系统中。打个比方，写下书的下一章或者文章的下一段。
*   使用正确的数据结构来提高影响和效率。
*   你是否清楚而中肯地表达了你正在做的事情。
*   尽可能使用最好的语言，例如，不要像 Java 一样编写 Swift 代码，或者像 C#一样编写 JavaScript 代码。
*   你的代码是否过于冗长或者过于紧凑。
*   你是否清楚地表达了你的意图，让其他开发者而不仅仅是电脑理解？

## [](#proofreading)校对:

*   你检查过拼写、语法和打字错误吗？林挺有所帮助，但你会惊讶地发现，人们经常忽视棉绒警告。
*   您是否遵从现有代码的风格，并且小心林挺工具可能捕捉不到的东西。
*   你检查了常见运行时错误的清单了吗？

我听说过不这样做的原因，大多数时候归结为“我想尽快提交我的代码。我不想等待，让我的代码与代码库的其他部分一起过时。”我们的源代码不仅是我们如何让计算机工作，也是开发者之间的一种交流方式。我鼓励所有的开发人员，尤其是新开发人员，养成与其他书面交流相同的习惯。提交前编辑和校对你的作品。
