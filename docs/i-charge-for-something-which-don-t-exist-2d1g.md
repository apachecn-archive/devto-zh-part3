# 我为不存在的东西收费...

> 原文：<https://dev.to/golangch/i-charge-for-something-which-don-t-exist-2d1g>

是的，我昨天在我的全栈开发者工作板上部署了一个新版本，在那里我为工作岗位引入了一个免费增值模式，对不存在的功能收取一些费用。

我是一个骗子，顺便说一句，我是一个疯子，是谁把这个贴在这里的？

[![](img/e7b7d7554e0f76666896ab11cb64fb77.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--piubWPYw--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/r9s8393v69j5s5ez7sny.jpg)

嗯，我想成为一个创造者/独立的领导者，这有点疯狂...但那是后话了。

重点是，我在 IH 和伟大的 Maker 社区页面 MakerLog 及相关网站上学到了很多，甚至在一天的工作中，我作为软件团队的领导者为电信行业做软件，那就是:

在实现功能之前先检查市场，我认为它们对我未来或现有的客户来说一定很重要。

因此...我的 Freemium 模型有两个包，免费的那个，和实际实现可以处理的功能一样。然后是新的付费专业套装，它提供了额外的功能，如显示公司标志，特色列表(在顶部 x 天)等。

[![](img/4827c4ab44be6bc23826523c003a001f.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--FUQ_bJE4--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/5uvw488anb1ujv7sxz6g.png)

其中一些特权必须首先实施。有些是简单的，比如显示 logo，有些不是很简单，比如基于时间的特性。

所以我决定不把它们作为我可以快速应用的一部分来实现，只要我有了第一个付费客户，我就可以先手动处理其他人，直到它实现。

关于报酬我做了同样的决定。我可以实现全部付款流程等。现在，或者像我一样，等待并手动处理第一批发送条形发票的客户。
