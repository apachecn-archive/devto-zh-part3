# Pravega, 2016

> 原文:[https://dev.to/laneone/pravega-2016-51j9](https://dev.to/laneone/pravega-2016-51j9)

Pravega 是 2016 年 1 月在班加罗尔的印度科学研究所举办的数据科学黑客马拉松。这是一个完全由微软赞助的活动，他们试图推动这个名为 Azure ML 的新平台，当涉及到机器学习时，它会处理大多数繁重的工作，并提供一个简单的用户界面，你可以使用它来学习/训练数据。

我们听了三个小时的讲座，内容是如何使用机器学习来完成初步任务，如线性回归情况下的预测和其他易于解释的例子。在讲座结束时，他们要求我们找到 http://data.gov.in 的数据集，并用这些数据向他们展示有价值的可视化效果。这是一个开放的话题，所以你可以选择任何你觉得舒服的话题，并就该话题得出结论。

Azure ML 平台从用户那里获取原始数据，并根据你决定的算法使用这些数据进行学习。然后，学习到的算法可以暴露给 web 服务，并托管在 Azure cloud 上，Azure cloud 可以随时运行。作为这次黑客马拉松的一部分，我们应该要求原始数据，从数据中学习，将算法暴露给 web 服务，可选地从瘦客户端调用 web 服务，并向用户显示可视化。

我们选择停电作为我们感兴趣的领域，我们查看了两个数据集，一个来自 data.gov.in，这是相当无用的，另一个来自 http://powercuts.in ，它不允许你下载报告。我们后来在 GitHub 的一个存储库中找到了一个相同的缓存副本。它只有大约 292 行数据，我认为这是算法预测的一个因素。

直觉是让机器学习影响印度停电的不同因素。许多因素，如计划内/计划外停电、停电的地理位置、日期、时间和其他因素，都作为数据集中的变量给出。我们使用 Azure ML 通过两类提升决策树算法对报告进行机器学习。因此，输入数据集被分为两部分，分别为 70%和 30%，70%用于训练模型和算法，30%用于测试算法并获得准确性分数，该分数可用于比较算法的工作情况。我们获得了 99.7%的准确率，这在当时似乎有点好得难以置信。我们计划将此公开给一个 web 服务，该服务监听时间戳和位置，并给出这两个变量，它将告诉您在接下来的 8 小时内是否可能会停电。

我们在这次黑客马拉松中没有赢得任何东西，但这是一次令人惊奇的经历，在整个黑客马拉松中我们得到的评委和导师都非常有帮助和足智多谋。我喜欢在这次黑客马拉松上度过的时间，并且学到了大量关于机器学习和 Azure ML 平台的知识。
