# 招聘经理的困境/视角？

> 原文：<https://dev.to/giorgosk/the-hiring-managers-dilemmaperspective--26i5>

我最近想知道，当一家公司正在寻找任何职位的开发人员时，是如何做出决定的，候选人以前工作过的环境会以什么方式影响决定。

在评估来自不同工作环境的候选人时会面临什么样的困境？开发人员的观点是什么，最重要的是招聘经理的观点是什么？

为了简单起见，我们假设只有 **2 名候选人**申请同一个职位。

候选人 1 (C1) 总是在先进高效的公司工作，有优秀的经理，工资总是绰绰有余，环境理想，公司支付教育、会议和聚会的费用，而且从来不用加班，因为一切都安排得很好，从来不用处理客户支持。

**候选人 2 (C2)** 总是在与前一个候选人完全相反的公司工作，没有既定的沟通或进度或报告渠道，单元甚至功能测试未知，过时的基础架构，人们通过电子邮件交流并附上 DOCX 修订版，花费大量时间支持客户，可能工作过度且报酬过低。

两人都参加了一个公司开发者职位的技术测试，该公司与 C1 以前的公司非常相似，C1 得分 97%，而 C2 得分仅为 85%。

从纯技术的角度来看，我认为 C1 应该得到这份工作。

我能想到的一些极端的观点

### C2 开发商的视角

努力成为最好的，但这总是一个艰难的过程。他/她在想，如果他/她成功地做到了 85%的称职，同时获得了报酬，X 会为他/她能取得的成就感到兴奋，如果他/她能找到一份报酬是 5X 两倍的工作。

### 招聘经理的视角

也许不存在两难问题，因为 C2 达不到公司的标准，不合适，所以 C1 会得到这个职位。

就我个人而言，如果让我来决定上述情况，我可能会聘用 C2。

我知道以上是一个非常极端的情况，但提出它只是为了作为讨论的基础。

如果任何人在开发人员或招聘经理方面有一些真实的(甚至没有)经验，我很想听听你的想法或总体意见。
