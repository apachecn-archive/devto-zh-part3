# TypeScript 是新的 C++

> 原文:[https://dev.to/stormingorman/typescript-is-the-new-c-4clf](https://dev.to/stormingorman/typescript-is-the-new-c-4clf)

采用 C++是一个必要的错误。为什么有必要？除非可以使用他们已经编写的代码和他们已经依赖的库，否则大量的 C 程序员不会采用 OOP。此外，不必学习全新语法的想法也是一个很大的优点。为了说服大多数编程社区采用 OOP，它需要感到安全和低风险。这些程序员知道他们需要一种方法来编写更好的代码。c 程序被像新 iPhone 一样脆弱的难以维护的代码所困扰。内存管理陷阱和缓冲区溢出会让任何正常的人质疑出生的好处。许多程序员肯定会在倾盆大雨中走出去，伸出双手指向天空，惊叫道:“你为什么要这样折磨我？你肯定还有别的东西要给我们用吧！给我发个签！”

不幸的是，宇宙的答案仍然是无符号的。

这种新语言是由比雅尼·斯特劳斯特鲁普开发的，被称为 C++。Stroustrup 指出，C++的目的是让程序员个人编写好的程序更容易、更愉快。当他设计 C++的时候，他在 C 语言中加入了 OOP(面向对象编程)的特性，而没有对 C 语言组件进行大的改动。因此 C++是 C 的‘相对’(称为超集)，这意味着任何有效的 C 程序也是有效的 C++程序。”[(c++)简史](https://mathbits.com/MathBits/CompSci/Introduction/history.htm)

对于开发者来说，一种带有面向对象的现代语言可以与其 C 代码和库无缝共存的想法就像是梦想成真。他们将获得多态性和封装的所有好处，同时仍然用熟悉的语法编写，与他们现有的代码配合得非常好！

C++在试图迫使注重过程的程序员笨拙地处理对象时，也有 C 的相同缺陷和警告。结果是产生对象的类的集合，这些对象只不过是结构函数库和数据对象。多态性通常被用作代码重用的巧妙工具，而不是创建真正的子类型。C++由于是 C 的子类**继承了*一切容易出错的东西，同时又未能*超越*破坏 OOP 设计的过程化思维。*

您的 JavaScript 语法在 TypeScript 中工作得非常好，事实上，您仍然可以使用所有您喜欢的 JavaScript 库！你不用学习全新的语法就能得到 OOP，并且仍然拥有现代语言的结构和特性！耶！

听起来熟悉吗？

TypeScript 据称最大的优势是它与现有 JavaScript 代码和库的易互操作性。我认为这实际上是它最大的弱点。TypeScript 仅仅是 JavaScript 的一个*超集*,会带来所有的 JavaScript 设计问题和陷阱。在 TypeScript 代码中，我经常看到到处都是“any”类型。这种互操作性导致了一种放任文化，这是 JavaScript 中存在的一个深层缺陷。当然，你可以遵守纪律，编写纯打字稿代码，但是你不需要*必须*这样做的事实使得你不太可能做到。
