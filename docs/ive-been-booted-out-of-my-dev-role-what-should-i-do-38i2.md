# 我被踢出了开发人员的角色！我该怎么办？

> 原文：<https://dev.to/jamesmh/ive-been-booted-out-of-my-dev-role-what-should-i-do-38i2>

我的[电子邮件简讯](https://eepurl.com/gdIV5X)的一位读者对一些绩效评估不佳的问题有疑问，这导致他被调到另一个部门。

他的问题是:

> 嗨，老兄，我刚刚订阅了你的时事通讯，并且不能停止阅读它。
> 
> 我是一名开发人员，3 个月前刚在*****的一家公司创业。我正面临着很多压力和麻烦，刚刚从我的主管那里得到了最糟糕的反馈....
> 
> 所以我现在做前端工程师，用 Angular，typescript 等技术。
> 
> 这是我前两次实习后的第三份工作。在我的第一份工作中，我大量使用 JavaScript，当涉及到 CSS 时，我们使用 bootstrap 和其他框架。
> 
> 当我开始新工作的时候，我的 CSS 技能并不是很好，这给我的工作带来了麻烦，因为我的同事和主管已经开始怀疑我作为程序员的所有技能。
> 
> 起初我被告知我可以提高自己，然后我被告知我一点都不好，即使我掌握了 css 技能，我也不能给公司带来任何价值。
> 
> 他们给了我一份 QA 测试员的工作，我非常失望和沮丧。
> 
> 我该如何克服被贬低和感觉自己毫无价值？这种态度让我什么都不想做。

我回答如下:

> 这听起来一点也不好玩。
> 
> 我以前见过这种情况，我想知道你是否处于类似的环境中？
> 
> 团队中有多少开发人员？公司有多大？你们是在开发新产品，还是在维护一个旧的或者更多的定制商店，为不同的客户开发大量的产品？
> 
> 我猜想答案将是 5-20 个 devs，大约 50-100 名员工的组织，在紧张的期限内建立一些新的东西....

他确切地证实了我的怀疑。

# 商业青春期？

为什么我会怀疑我在回答中列出的环境？

在这种情况下，我之前在一个队里。我确实看到许多人因为表现不好而突然被解雇，等等。

对高产出有一种非常有害的态度(即尽可能快地完成任务)。

不管任务完成得是否恰当，你完成的“东西”越多，经理就越高兴。然而，为了完成这些“事情”,仍然有许多“繁文缛节”和流程要走。

这种规模的组织有什么固有的问题吗？

我认为这是一个组织进入某种商业青春期的规模。

这是一家以快速发展、高产出的初创企业为战略的小企业和一家规模较大、发展较慢、考虑周到的组织之间的界限。

你可能会说这是一种身份危机？

我想我们的读者已经被夹在中间了！

# 我的建议

现在，读者可能真的不是一个伟大的程序员。但是，也许他是？

然而，这个故事更多地反映了这家公司，而不是他。

这是他的第一份非实习工作。应该给他指派一个导师。他应该处于职业生涯中的一个阶段，他仍在学习一些基本原理，并将它们付诸实践。

对于一个组织来说，仅仅告诉他“不够好”意味着公司不知道如何培训和培养自己的员工。这是一种认为员工大多是一次性的而不是可教的态度。

正因为如此，即使是高级开发人员也不会成长，因为他们没有导师、职业指导和规划，甚至没有成长的期望！人们只是希望他们完成所有分配给他们的任务。

我的建议是，你要做好准备，开始寻找其他地方。你甚至不在你被聘用的那个职位上！

我的职业时事通讯第一版或[这篇关于开发的文章](https://dev.to/jamesmh/transitioning-into-your-first-junior-developer-role-2n8d)中的一些提示可能有助于开始这种改变。

在写这个回复的时候，有人写了[一个非常相关的 twitter 主题](https://twitter.com/clairejlew/status/1092835636258660352)！好时机！

我认为对于读者来说，这是一个“好作品”的定义错位的例子。"

有人告诉过他好的工作到底是什么样的吗？

他有改进的计划吗？

没有。

同样，我会开始尝试把我的脚放在别的地方。

我不喜欢给这种建议。有时候最好再努力几个月，努力学习成长等等。

但是在这种情况下，提问者已经被踢出了他的 dev 角色。
