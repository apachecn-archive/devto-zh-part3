# 我从 Web 应用程序安全初学者指南中学到了什么

> 原文:[https://dev . to/chillsunfire/what-ive-learn-from-the-web-application-security-初学者指南-1dbd](https://dev.to/chillsunfire/what-ive-learned-from-the-web-application-security-beginners-guide-1dbd)

[![](../Images/cbf55359974c762776d37e7b0dd13d7f.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--jzj5JETs--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/http://blog.hitsaru.com/wp-content/uploads/webappsec-beginner-cover-300x294.jpg) 在过去的几周里，我一直拿着荧光笔通读 Bryan Sullivan 和 Vincent Liu (2012，McGraw Hill)的《Web 应用安全-初学者指南》。现在我已经读完了，我想记录下我对我所学到的东西的一些想法，以及我需要进一步探索的东西。

### [](#summary-points)总结要点

1.  **永远不要相信用户！**
2.  **白名单** >黑名单
3.  基于**用户角色和最低权限**设置访问控制
4.  对 SQL 查询使用**参数化存储过程**
5.  对整个应用程序使用 SSL，而不仅仅是敏感部分
6.  从开始到结束烘烤安全

### [](#lets-dig-deeper)让我们深入挖掘

##### [](#1-never-trust-the-user)1。永远不要相信用户

我在这里学到的最重要的事情是，“用户”不仅仅是屏幕/键盘另一边的人。它也是浏览器和任何其他可能调用我的网站/应用程序中的资源的网站或应用程序。离开我的服务器的任何东西在到达最终用户之前都可以被操纵。任何进入我的服务器的东西都可能在我和我的最终用户之间被操纵。因此，在处理输入服务器端之前，需要验证、清理和检验 access/authz。

##### [](#2-whitelist-is-better-than-blacklist)2。白名单比黑名单好

更有意义的是，只明确定义什么是可接受的，明确说明什么是允许发生的，而不是依赖于不断增长的不允许清单，因为不可避免地，某个地方的某人会想出一种方法来绕过你的黑名单，同时仍然不做你想要/期望的事情。指定“不要这个，不要那个，绝对不要这些”仍然会留下空间，而指定“只要这个，那个和那些”不会留下太多回旋的空间。

##### [](#3-set-access-control-based-on-user-role-and-least-privilege)3。基于用户角色和最低权限设置访问控制

如果应用程序只需要对数据库的读取权限，请在数据库中设置一个只具有读取权限的特定用户角色。不要对任何事情都使用管理员级别。这是我必须习惯的，因为我不常和数据库用户打交道。

##### [](#4-use-parameterized-stored-procedures-for-sql-queries)4。对 SQL 查询使用参数化存储过程

这对我来说是一件全新的事情，但似乎任何在正式环境中学习过 SQL 的人都应该接触过。不用说，我必须更深入地研究这一点，因为我不确定我是否完全理解语法或过程。

##### [](#5-use-ssl-for-the-whole-app-not-just-the-sensitive-parts)5。对整个应用程序使用 SSL，而不仅仅是敏感部分

这很有意义，并且正在成为整个互联网的标准。如果你想让用户信任你，你必须证明你是值得信任的。即使你不公开从你的访问者那里收集数据，他们可能也不会想给你任何不必要的信息，因为浏览器的 URL 栏上有一个大大的丑陋的“不安全”字样，对吗？当我开发哪怕是最小的应用程序和网站时，我都需要记住这一点，以加强这个好习惯。

##### [](#6-bake-security-in-from-start-to-finish)6。从开始到结束烘烤安全

采取全面的安全方法。这不仅仅是关于代码本身——在确定需求范围和进行设计时，需要考虑安全性。在考虑所有功能时，威胁建模应该成为第二天性(“这怎么可能被打破？”以及“应用程序应该如何应对此类问题？”).包括在项目的不同阶段进行测试。并且有一个处理突发事件的计划。虽然我想了解所有这些东西，但现在我需要专注于代码部分——我应该使用什么库和工具，以及如何使用它们(语法、位置等)。

### [](#thats-not-all)还没完呢！

这只是我巩固我所阅读的一些方面的方法，但是在这本书里有更多的内容。对于那些像我一样默认情况下没有安全意识的人来说，这是一个很好的介绍，让你像坏人一样思考，这样你就可以养成良好的习惯，防止坏人攻击你的产品。我期待着将这些东西付诸实践——事实上，当我升级我的喂蛇游戏以包括一个全球记分牌时，我已经列出了我需要做的事情，但我将把它留到另一篇文章中。
