# 开发、阶段和生产之间的生产环境差异

> 原文:[https://dev . to/flipped coding/difference-in-development-stage-and-production-d0p](https://dev.to/flippedcoding/difference-between-development-stage-and-production-d0p)

在某些时候，你可能会遇到三种不同的环境。每个环境都有自己的属性和用途，因此相应地使用它们很重要。一旦你知道环境是用来做什么的，它就会产生，因为为什么我们有这么多的环境。

主要的三个环境是:开发、阶段和生产。

## [](#development)发展

这是你电脑上的环境。这里是您进行所有代码更新的地方。它是你所有的提交和分支与你的同事一起生活的地方。开发环境的配置通常不同于用户工作的环境。

它将连接到某个本地数据库或虚拟数据库，这样您就可以在不弄乱真实数据的情况下编写代码。由于在开发环境中会有很多人工作，您还将处理任何分支合并。

您在开发环境中所做的任何事情都不会影响用户在打开网站时看到的内容。这只是为了让你和其他 web 开发人员看看新特性是如何工作的，并尝试改进。

在这种环境下会进行大量的初步测试。在确保代码至少能在本地工作之前，你不希望发布代码。尽可能彻底地检查你的代码，这样你就可以限制那些进入下一个环境的错误。

## [](#stage)阶段

舞台环境与生产环境尽可能相似。这一次，您将把所有代码放在服务器上，而不是本地机器上。它将连接到尽可能多的服务，而不会影响生产环境。

所有的核心测试都在这里进行。这里将测试任何数据库迁移以及任何配置更改。当您必须进行主要版本更新时，stage 环境可以帮助您找到并修复出现的任何问题。

如果你有一个客户，这是你能够给他们演示东西如何工作和外观的时候。他们将能够看到当他们使它运行时事情将如何工作，并且他们将能够给你任何你需要的反馈。把舞台环境想象成你做最后检查和润色的地方。

## [](#production)生产

每次你说让你的项目活起来，这就是你说的环境。生产环境是用户在所有更新和测试之后访问最终代码的地方。在所有的环境中，这一个是最重要的。

这是公司赚钱的地方，所以你不能在这里犯任何严重的错误。这就是为什么您必须首先完成其他两个环境的所有测试。一旦你进入生产阶段，任何遗留的 bug 或错误都会被用户发现，你只能希望是小问题。

有些人喜欢在生产环境中推广。这意味着他们向少数用户发布变更，并逐渐向所有人推广。并非所有的更新都必须同时发布到生产环境中。

在某些情况下，您可能会按计划推出您的更改，以处理负载问题或确保没有任何重大问题偷偷溜走。如果出现紧急情况，您可以发布对生产环境的更改，而无需通过 stage 环境，尽管这不是您想要经常做的事情。

不同的组织会以自己的方式应对这些环境。其中一些有不同的名称，或者可能有比这些更多的环境。不管怎样，你知道你需要做什么来充分利用它们。

顺便说一句，我知道舞台环境有时会很痛苦。建立这个假的生产环境并为其编写所有的测试需要做大量的工作。虽然值得花时间。最好继续下去，完成这一步，因为这将比你所知道的更多地拯救你的屁股。
