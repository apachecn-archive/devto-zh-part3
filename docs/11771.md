框架是做一些有用事情的程序的集合，你可以用它来开发你自己的应用程序。——朱利安·索洛萨诺
[看这里](https://www.quora.com/What-is-the-difference-between-a-programming-language-and-a-framework)

一些广为人知的框架包括但不限于:Bootstrap、Foundation、Angular、VueJs、Django、Laravel、Ruby on Rails。

React 开发人员可能想知道这里发生了什么😄。React 恰好是一个库，但是我们可以把库和框架放在一起，因为它们有相似之处。

框架提供了工具和结构，使得软件和应用程序的开发更快。这些工具包括从前端框架的浮动按钮到服务器端框架的 ORM(对象关系映射器)。结构可以是 MVC，MVVM 或类似的。

框架的本质是减少开发时间。这有助于开发人员满足项目期限。框架也给团队开发带来了一致性。每个成员都在很大程度上理解开发模式。

然而，当用户对框架所使用的底层语言知之甚少时，框架的优势很容易丧失。初学者陷入这个陷阱，因为他们想跟上新技术。

这导致的结果是，人们使用一个框架，却不知道为什么或背后有什么。如果有东西坏了，他们无法修复。进行重大修改变得困难或不可能。对于那些尝试的人来说，他们会花很多时间在谷歌或 Stackoverflow 上寻找基本问题的答案。

我对初学者和像我一样的不断学习者的建议是，在进入与核心编程语言相关的框架之前，花足够的时间学习这些语言。

不要被吓倒。不断学习。
