# 如何更快地加载网站

> 原文：<https://dev.to/thisisabdus/how-to-load-a-website-faster-33gh>

对于任何类型的网站来说，页面加载时间都是一个重要因素。加载时间越短，站点流量越高，跳出率越低。

如果你不知道什么是页面加载时间，那就是浏览器渲染一个网页所花的时间。这个过程包括:向服务器发出请求、下载资源、处理 DOM 元素和渲染。

为了减少一个网站的页面加载时间，你必须在建立/创建一个网站时遵循某些事情。

## 将脚本放在结束 body 标签之前

无论脚本做什么，它都应该放在的末尾。即结束的标签。脚本通常都很大，并且花费浏览器额外的时间来加载。此外，您不希望在 DOM 加载之前执行任何脚本，对吗？

## 缩小

是的，缩小样式表和脚本。minify 是一个从你的代码中删除空白和不需要的字符的过程。它将帮助您减少文件的大小，以便浏览器可以更快地下载它。

## 不要使用过多的重定向

每当我们重定向一个链接时，都会向服务器发出一个额外的请求。例如，如果你从*example.com*加载一些资源，这些资源被重定向到*example-2.com*。您正在迫使客户端(浏览器)提出额外的请求。因此，尽量减少使用重定向链接。

## 使用 CDN

CDN 或内容交付网络帮助您减少服务器负载，并根据用户的位置提供页面服务。例如，如果你在美国的某个地方，使用 CDN，网站将从最近的可用服务器提供服务，而不是从原始服务器。这种方法可以帮助你缩短资源的行程。

## 代码拆分

这对你来说可能是新的东西。这有助于您只加载呈现任何网站的第一个实例所需的代码。因此，如果你有一个很大的 CSS 文件，其中包含了许多在初始渲染时不需要的选择器，那么就删除它们，或者创建一个新的 CSS 文件，在以后的阶段加载它们。

## 脚本的  *异步*和*延迟*

如果你有一些你不马上需要的脚本，就把它们拿出来。`async`和`defer`告诉浏览器稍后加载这些脚本。例，

```
<script async src="example.com"></script>
<script defer src="example.com"></script> 
```
