# 用解决问题来克服我的挑战

> 原文:[https://dev . to/allysmirnov/克服挑战解决问题-9ik](https://dev.to/allysmirnov/overcoming-my-challenges-with-problem-solving-9ik)

在一月底的某个时候，我决定把恐惧和自我怀疑放在一边，申请 FullStack Academy。我已经涉足代码大约一年了，但仍然是(现在仍然是)一个初学者。

我通读了申请流程，注意到在书面申请被接受后，我需要参加一次编码评估，如果我成功通过，还需要通过 Skype 进行一次技术面试。那时，我以前从未解决过编码问题。FullStack 的申请页面包括一些资源，你可以用来评估你是否准备好接受他们的编码评估。我没有。

我不想让我不知道的事实阻止我申请，所以我给自己几个星期的时间来学习我所能学习的一切，以便能够解决代码挑战。

# [](#back-to-basics-kind-of)返璞归真(种)

我最大的障碍之一是我不知道足够多的 Javascript 来思考我需要的解决问题的方法。大约两个星期以来，这是我的主要关注点。我发现了一些很棒的资源，它们真的帮助我理解了比初级水平更深的概念。

### [](#programming-with-mosh-javascript-basics-for-beginners)用 Mosh 编程- [初学者 Javascript 基础知识](https://codewithmosh.com/p/javascript-basics-for-beginners)

虽然这个课程的开始对我来说有点慢，但我发现关于对象、数组和控制流的部分特别有用。一口大小的视频很容易消化，每个部分末尾的练习帮助我练习应用某些内置的 JavaScript 方法。每个练习视频还包括对解决方案的解释，如果我有不明白的地方，这非常有帮助。我只为这门课支付了 5 美元，但如果你喜欢他的教学风格，你可以每月支付 15 美元来学习他的所有课程。

### [](#fullstack-academy-bootcamp-prep-at-your-own-pace)富栈学院- [按照自己的节奏准备训练营](https://start.fullstackacademy.com/fullstack-academy-bootcamp-prep-at-your-own-pace)

这个训练营准备包括强制、数组和对象等基本主题，但也包括高阶函数和递归等非初学者主题。每个主题都有一个 20-40 分钟长的视频，每个视频后至少有 5 个练习题。这些练习肯定比上面的课程更有挑战性，我对其中的许多练习都感到困难。有几次我甚至觉得有点失败，但是我坚持下来了，我很高兴我做到了。它教会了我很多东西，并帮助我学会以一种更有成效的方式思考，即使我必须首先查看解决方案，以了解如何处理问题。这门课是免费的。

### [](#codesmith-csx)铁匠- [CSX](https://csx.codesmith.io/)

这是另一个伟大的免费资源。我没有完成整个 CSX 课程，但是我用它来帮助我理解更难的概念，比如闭包、递归和作用域。课堂上有完整的关于主题的讲座和练习题。讲座虽然很长，但却是我最喜欢的部分，因为我真的感觉就像和其他学生一起在课堂上一样。

Codesmith 还经常通过他们的名为 *JavaScript the Hard Parts* 的 [MeetUp 页面](https://www.meetup.com/Codesmith/events/)提供免费的在线讲座，我参加这个讲座是为了 Scopes 和 Closure。

# [](#practice-makes-perfect-no-really)熟能生巧(不，真的)

在每天下班后花大约 4 小时和周末 8 小时完成这些课程后，我对自己的 JavaScript 知识和解决问题的能力更有信心了。现在是我真正开始练习解决我可能在编码评估中发现的问题的时候了。

### [](#codewars-train-with-programming-challenges)Codewars - [用编程挑战训练](https://www.codewars.com/)

Codewars 提供了不同语言的编程挑战(或称之为 katas );从容易的级别(8kyu)到最难的级别(1ku)开始。给你一个要解决的问题，你的代码在提交之前必须通过所有的测试。回答完问题后，您可以查看其他人的解决方案，并与您自己的解决方案进行比较。您还可以在解决问题时进行排名，从而查看难度更高的问题的解决方案。

虽然在跳到解决方案之前，你应该总是尽最大努力尝试并解决问题，但我发现，有时当我感到完全迷失时，看一些解决方案有助于我认识到解决问题的方法和模式。在先看了一堆问题的解决方案后，我在解决问题方面变得更好了，所以当你刚刚开始解决问题时，不要因为偷看而责备自己。

一个类似 Codewars 的网站是 [HackerRank](https://www.hackerrank.com/) ，但是我没怎么用过，所以我不能提供任何有意义的信息。

### [](#coderbyte-coding-assessment-platform)Coderbyte - [编码评估平台](//www.coderbyte.com)

Coderbyte 是一个有趣的网站。我不是他们界面的粉丝，有时浏览起来有点困难，但那里的信息是黄金。这里有很多很好的例子，比如 [10 个常见的 JavaScript 面试问题](https://coderbyte.com/algorithm/10-common-javascript-interview-questions)和 [3 个常见的 JavaScript 闭包问题](https://coderbyte.com/algorithm/3-common-javascript-closure-questions)，它们帮助我了解我可能会遇到什么样的问题，以及如何解决这些问题的解释。这是一个付费网站，每月 35 美元，但对我来说，帮助我准备评估和参加 FullStack Academy 的面试是值得的。

不幸的是，没有快速的方法可以让你擅长解决问题。有很多事情我没有做，比如学习算法和数据结构，因为时间紧迫。如果你也在走类似的路，我可以给你一些建议:

*   用简单的英语写出来！直接跳到编写代码不是一个好主意，而且很容易陷入困境。写出你希望你的代码实现的目标将帮助你在开始写代码之前注意到你的逻辑中的任何缺陷或者你可能遗漏的东西。

*   在你的 IDE 中求解并经常使用调试器！当你的代码运行时，观察变量值的变化可以帮助你找出如果你的答案没有给出你想要的结果，你需要修正什么。

*   使用 Chrome 控制台！测试你操纵变量的方法。我花了很长时间才开始这样做，这对弄清楚我想做的事情是否会成功真的很有帮助。

*   **练习！练习！练习！最有可能的情况是，你做某件事的头几次都不会进行得很顺利。当我第一次开始在眼镜商的工作中修理镜架时，或者当我第一次尝试画眼线时，我不是很好。如果你经常练习，你**会**变得更好，即使一开始看起来不像**

变得沮丧是这个过程的一部分，只是要确保你不会沉湎于这种沮丧。当你感到不知所措的时候，休息一下是很好的，但是如果这个过程感觉很慢或者你没有马上找到答案，也不要气馁。通过努力和大量的练习，它会变得更容易，你会发现自己能够在一段时间后解决更复杂的问题。
