# 我是如何在 2 小时内学会成为一名开发人员的？

> 原文：<https://dev.to/simo97/how-i-learnt-accountance-being-a-developer-in-2-hours-3m4>

# 一点脉络

事实上，我是 Normalis Consulting 的一名 python 开发人员，我接受在那里工作，只是因为他们在很多事情上使用 python，他们实际上与 Odoo ERP 合作，因为这是我作为 Python 开发人员学习的技术路线图，我非常兴奋，因为我即将学习在其上创建模块、应用程序和其他东西，另一方面，CTO 对此非常擅长，所以我知道我找到了一位导师。

# 发生什么事了

该公司实际上为 SaaS 等多个客户提供 Odoo 服务，当有新客户时，我们只需配置软件以满足客户的企业需求，甚至在需要时创建新模块，本周我们为一家金融机构做了这项工作，我们的会计正在配置应用程序，他正在处理工资单模块的配置，他被困在定义工资计算的一些规则上(这里说的是， 薪资规则、类别、薪资结构等)，所以他打电话给我(我是技术人员)，我的工作是编写小的 python 代码(但对会计来说是高级的)来做一些计算，我很乐意这样做，但有一件事:**我是开发人员，不是会计**所以我必须学习这些概念。 我做什么？(因为一天下来，我对这方面有了一些了解)

# 我是怎么做到的

从他告诉我要做工资单，到他真的来到我的办公桌前，我一直在谷歌上搜索这样的东西:*用来计算工资的是什么？*、*什么是薪资规则？*等..我开始阅读许多关于这一概念的文章和书籍，我了解了分配、赔偿、基本工资、如何对工资征税等等...

要知道我要学的东西并不困难，我只需要导航到我在菜单上看到的 app(在 odoo 中)和*google-ed*标签，例如:*薪资规则*在 Google 上我输入:*什么是薪资规则？*等...

到最后，我对这些东西有了一个清晰的想法，会计过来给我解释了更多，我们准备好开始了，我做了他希望我做的工作，而且效果很好。我为自己感到骄傲。

# 我学到了什么

我从来没有做过这种面向商业的软件开发，我的一些朋友谈了很多，在 CS 学校我们学了一些关于会计和企业管理的基础知识，但我没有深入这些知识，但现在我准备重新打开我的旧书。

我发现，作为一名开发人员，我在谷歌上查找、学习和整合信息的能力非常发达。每次我们，开发人员，通过谷歌搜索和阅读文档来学习，甚至是全新的技术，这实际上可能是一种技能。

现在我知道了工资是如何详细计算以及它的构成。

# 未来

现在我准备学习企业管理的新部分，但我认为这将是一种边做边学或在过程的核心中学习...我面临着另一个问题，但我很冷静，将毫无问题地克服它们(我的首席技术官在那里；) ).
