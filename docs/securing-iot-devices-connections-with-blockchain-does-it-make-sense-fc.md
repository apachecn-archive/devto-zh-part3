# 保护物联网设备与区块链的连接:有意义吗？

> 原文：<https://dev.to/andrii_fedotov/securing-iot-devices-connections-with-blockchain-does-it-make-sense-fc>

**案例分析:**

X 公司部署了一批联网的物联网设备，这些设备具有硬件信任根的功能，其中私钥被集成到设备中。

1.  X 公司的系统管理员在区块链上注册了一个帐户。

2.  他将需要在物联网设备(即:闭路电视摄像机)上安装一个特殊的固件(或只是修改它的现有版本)。它应该在设备中生成一对密钥。

3.  此时，系统管理员获得设备公钥，并使用他的帐户将其存储在区块链上。该设备能够自己发送公钥，但是它需要访问帐户。作为替代，这可以通过预先安装区块链账户密钥的服务来完成。

4.  现在，系统管理员可以建立一个安全的加密连接，连接他需要的所有闭路电视摄像头。如果他需要撤销相机的公钥，他可以通过使用他的区块链帐户的私钥。

**区块链账户**

它有一对公钥/私钥，由用户在他的设备上用项目库之一生成。该帐户允许在区块链上存储公钥以及撤销公钥。仅使用一个帐户的密钥对，您可以在区块链上存储任意多的公钥。在区块链技术中，有几种方法可以安全地访问您的帐户，例如，多重签名。

没有 CA 签名，设备为什么要相互信任？

在这种情况下，管理员将向区块链发送一个请求，请求存储两台设备的公钥，并分别在每台设备上安装私钥。这样，管理员负责提供连接两个设备/应用程序的正确公钥。没有人能够存储先前添加到区块链中的公钥。
