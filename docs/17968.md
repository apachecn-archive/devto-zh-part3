# 我想要的分析工具

> 原文:[https://dev.to/ukutaht/the-analytics-tool-i-want-5f8j](https://dev.to/ukutaht/the-analytics-tool-i-want-5f8j)

在制作 [Gigride](http://gigride.live) 的时候，我们的营销主管要求我为我们的登陆页面整合谷歌分析。我的第一个想法是:

> 呃。我们可以用谷歌分析以外的东西吗？

当然，这变成了关于谷歌分析的优点和问题的半小时讨论。毫无疑问，GA 是一个强大的工具，十多年来为无数的 web 开发人员和营销人员提供了服务。尽管如此，每次我使用它时，它都能让我脖子后面的头发竖起来。我的反对意见可归结为以下几点:

*   **复杂性**:我觉得 GA 势不可挡，很难学。UX 可以做得更好，尤其是对新用户来说。
*   隐私:我不希望谷歌或任何人跟踪我的终端用户。你想过 GA 为什么可以免费使用吗？因为谷歌很乐意把他们的分析数据卖给广告商。
*   准确性:你必须学会忽略一些关于 GA 的报道，因为众所周知它们是不准确的。两个最惊人的例子是[会话长度](https://help.analyticsedge.com/article/misunderstood-metrics-time-on-page-session-duration/)和[直接流量来源](https://neilpatel.com/blog/what-is-dark-traffic/)
*   **重量**:谷歌标签管理器+谷歌分析总共重 73kb。这比我在 Gigride 上的时间还长。他们的代码片段塞满了很少有人使用的功能——最好保持简单小巧。

这些问题似乎很容易解决，不是吗？当我环顾四周，我完全准备好拿起另一个工具，并立即安装在 Gigride 上。

原来 Google Analytics 有一些[的替代品，但我没有发现其中任何一个有吸引力。](https://nomoregoogle.com/) [Simple Analytics](https://simpleanalytics.io/) 和 [Fathom](https://usefathom.com/) 是最接近我的理想的，但是对于我的用例来说，它们有点太简单了。例如，用户使用的浏览器版本对我来说非常重要——如果我使用 IE7 不支持的 css 规则，会影响多少用户？目前，无论是 Fathom 还是 Simple Analytics 都没有给出答案。

这似乎是一个构建我真正想要的工具的机会。我假设有其他人也有兴趣放弃谷歌分析，寻找更好、更道德的替代方案。那么，我对这个新平台有什么想法呢？

*   自动跟踪标准内容:页面浏览量、独立访问者、跳出率、会话长度、每页时间、热门推荐者、屏幕大小、热门页面、设备类型、操作系统版本、浏览器版本等。
*   它应该看起来很漂亮，易于使用。这很难，因为我的设计很烂。现在我会尽量保持简单，一旦产品适合市场，我可能会让设计师参与进来。
*   不要存储 IP 地址或任何个人可识别的信息。千万不要卖任何数据。旁注:我能证明这一点吗？开源是验证这些说法的唯一途径吗？
*   优化信噪比。用户应该只看到准确、有用和可操作的数据。例如，当显示独立访问者的月环比变化时，了解这是统计上的显著变化还是正常波动非常重要。据我所知，没有人做这件事。
*   最终用户下载的 Javascript 代码片段必须是轻量级的。

基本上，这可以归结为帕累托原则:我相信我们可以在遗传算法上获得 80%的收益，同时降低 20%的复杂性和成本。

在我的空闲时间里，我已经开发这个工具一个月了。我不能告诉你它叫什么，因为我还没有决定一个名字:)这个描述的一个非常精简的版本即将可用，我相信我可以在一月份推出一个私人测试版。
