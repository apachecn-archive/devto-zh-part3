# 让你的代码来说话

> 原文：<https://dev.to/dealeron/let-your-code-do-the-talking-5b0h>

作为 DealerOn 软件工程师的主要招聘经理，我经常收到招聘人员的推诿，以及候选人的提问，询问我们为什么要求候选人在参加面试之前完成编码测试。简单地说，我们认为利大于弊。但是它们是什么呢？让我解释一下…

* * *

在我的职业生涯中，我见过几家公司不同的招聘流程。都是从简历开始的。许多人要求求职信。你或代表你的招聘人员将这些提交给招聘公司的招聘经理或人力资源招聘人员，然后你就等着。如果你是少数幸运者之一，你将被安排与任何数量的技术和非技术人员进行面对面的面试，如果你得到这份工作，你将(大概)与他们一起工作。这是我职业生涯早期的做法。后来，我注意到一些公司在面试前增加了技术筛选电话，以淘汰任何不具备所需技术技能的候选人。

在我职业生涯的几次面试中，我被要求在白板上，甚至是用铅笔和纸来写代码。这对我来说总是很奇怪，因为[我们不会那样写软件](https://blog.usejournal.com/i-failed-my-effing-coding-interview-ab720c339c8a)。我们有编译器和功能丰富的 Intelli-sense ide 来防止我们犯愚蠢的错误，还有 Google 和 Stack Overflow 来查找我们可能知道但忘记了具体如何做的事情。这些方法也容易让候选人非常紧张，因为这让他们处于一种在实际工作环境中永远不会发生的尴尬境地。

DealerOn 要求的那种代码测试并不是在面对面的面试中完成的。我们提前几天提供问题，给你充足的时间以你希望的任何方式解决问题。我们提出的唯一要求是，你使用 C#或 Java，它可以编译，它不使用第三方库，你自己写。您可以使用任何您想要的在线或个人资源，结果可以是 web 应用程序、windows 应用程序或控制台应用程序。但是，请记住，作为我们面试过程的一部分，我们会询问候选人在代码测试中的设计选择，因此您需要理解您所做的选择。候选人可以完全自由地使用他们认为最适合问题的任何设计模式，并且可以按照他们的意愿编写简单或复杂的代码。我们的首选是 C#，因为这是我们在 DealerOn 主要使用的语言。然而，Java 在语法和结构上非常相似，所以用 Java 编写的测试也可以展示你的设计能力。

# 为什么不是所有公司都用一个代码测试？

不是所有的公司都使用代码测试。事实上，大多数人都不知道。原因很明显。

### 减少候选人数

一些候选人，尤其是那些非常抢手的候选人，觉得他们没有时间做代码测试。毕竟，他们有一些不需要代码测试的公司打电话给他们做面对面的面试。我敢肯定，从技术上来说，他们中的一些人是非常优秀的候选人，但是代码测试是我们文化的一部分，所以如果你不愿意花时间做测试，你很可能无法适应。

### 这会延长招聘过程

当市场火爆的时候，就像现在这样，时间往往不在我们这边。候选人可能会被其他公司抢走。更长、更复杂的招聘过程会让我们失去候选人。

我们决定将代码测试作为招聘过程的一部分，这是权衡利弊的结果。在这一点上，即使在劳动力市场紧张的情况下，我们仍然认为得到合适的候选人的好处是值得的，有时可能会失去其他人。

# DealerOn 为什么要用代码测试？

我们喜欢代码测试，因为我们觉得它利大于弊。我们看到的主要优势是:

### 它允许我们独立于简历来考虑候选人。

作为我们过程的一部分，审查代码测试的工程师对候选人没有任何预知。在评估开始前，他们不看简历，也不偷看代码。DealerOn 不会将初级开发人员限制在琐碎的任务上，我们也不会根据您在行业中的工作年限对您进行分类。如果你能写出高质量的软件，我们可以为你提供一个成长的地方。

### 这样做的考生是真心想在这里工作的。

想一想:如果每家公司都要求你完成代码测试，你会申请几个？通过要求进行代码测试，我们知道那些向我们提交代码测试的人确实有动力为我们工作。此外，我从许多候选人那里听说他们真的很喜欢做代码测试。这就是我们这里想要的那种人:喜欢写软件并渴望向我们展示他们能力的人。

### 这表明你的知识并不完全是理论上的。

是的，代码测试并不复杂。因此，根据您的提交，我们不能肯定地说，您可以设计高度可伸缩的网站和服务，可以在合理的时间内提供我们客户的数据。但它确实向我们展示了您有多在乎您编写的代码，并且您可能在这里获得成功。编码风格、可维护性、可测试性、代码清洁度、设计模式使用/知识都可以通过我们的代码测试完全捕获。

### 它给了候选人另一种让自己与众不同的方式。

我们如何在未来的雇主面前脱颖而出？我们简历上的经历是其中的一部分，简历的格式和设计也是如此，尽管招聘人员经常未经我们允许就重写简历，并使用他们的风格和公司信笺进行格式化。在面对面的面试中很好地展示自己当然是必不可少的，但是如果你从未被邀请参加面试呢？我们的代码测试允许候选人通过展示他们的编码技能来脱颖而出。我们已经看到了许多创造性的和优雅的解决我们测试的基本问题的方法。我们也看到代码测试错误百出，没有按照说明解决问题。在招聘过程中，我们掌握的信息越多，就越有可能做出正确的招聘决定，这是一件好事。

# 给潜在候选人的建议

无论你打算申请 DealerOn，还是另一家招聘过程中包含代码测试的公司，我都有一些建议:

### 认真对待

这似乎是显而易见的，但我无法告诉你有多少次我看到候选人在一个小时内“拼凑”出的提交材料。这是招聘过程的一部分。你在这里付出的努力越多，你得到这份工作的可能性就越大。你不会穿着睡衣去参加面对面的面试吧？提交代码测试而没有尽最大努力会给人同样的印象。拼写错误，不一致的变量和方法命名，或者非面向对象的代码表明你没有在工作中投入太多的思想和精力，这些都是危险信号。

### 遵照指示

你可能会惊讶这个问题出现的频率，但它是关键。如果你不能遵循我们提供的简单指示，我们怎么知道如果我们雇用你，你会遵循指示？忽略约束或给出无效输出的代码是巨大的危险信号。

### 卖弄一点

你的简历告诉我们你是 C#专家，或者你对工厂设计模式了如指掌。给我们看看！使用依赖注入，让单元测试脱颖而出！我们真的很喜欢看精心制作的解决方案。这表明你真的很喜欢你的工作，并且很可能会很适合这里。

# 结论

代码测试可以被视为招聘过程中一个繁重的额外步骤，但对于包括 DealerOn 在内的越来越多的公司来说，这是一种更好地了解你是谁、你是如何思考的以及你是一名多么熟练的开发人员的方式。不管你的经验如何，一份写得好、设计得周到的代码测试提交可以把你的简历推到最前面。花时间做好工作告诉我们[你很可能非常适合我们](https://www.dealeron.com/join-our-team/)。
