# 遵循此路径 swift 软件开发研究指南

> [https://dev . to/ronanodrigo/顺着-这条路---一个研究指南-关于开发-卸载软件-com-swift-1a0g](https://dev.to/ronanrodrigo/siga-por-este-caminho---um-guia-de-estudos-sobre-desenvolvimento-desoftware-com-swift-1a0g)

我在学习上一直面临的一个困难是优先考虑我应该学习的东西。大问题一直是“这重要吗？”"这很难吗？""这件事要花我很长时间吗？"。这些问题可能是拖延的燃料。

为了解决这个问题，我和一些朋友创建并组织了一个学习指南。它由 3 个级别组成，每个级别都有一部分任务需要完成。所有任务都按优先级排列，从最基本到最高级，从最重要到最不重要。本指南的目的是为新手和该领域的专业人员提供参考。

## [](#como-estudar)如何学习

由于看板帧的普及，设计了学习指南，并将其整理成了一种虚拟看板。对于不熟悉看板的人来说，看板是一种组织您的任务的方案/框架，因此您可以执行这些任务。并且本指南的工作方式与如此著名的 post its 框架相同。要使用此主板，您可能需要遵循一些规则:

*   永远不要跳过一个任务，总是从上到下拿第一个；
*   避免跳过级别(列)，仅在上一个级别完成后才前进到下一个级别；
*   开始学习科目(卡)时，将其移至学习栏；
*   在学习栏中只有一个主题；
*   使用“主题说明”字段记录有关主题的学习内容；
*   使用卡上附加的链接作为内容参考，并将它们的参考添加到卡上；
*   仅当您觉得谈论主题或应用所学内容很舒服时，才移动到 done
*   结合上述建议，创建自己的就绪定义。
