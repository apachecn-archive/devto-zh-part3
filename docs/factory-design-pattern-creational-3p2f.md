# 工厂设计模式[创造]

> 原文：<https://dev.to/itscoderslife/factory-design-pattern-creational-3p2f>

工厂方法模式允许调用者通过只知道协议或顶级基类而不知道确切的实现类来创建对象的实例。

*   调用者永远不会知道关于实现类的任何细节。
*   工厂方法包含了根据调用者提供的标准决定实例化哪个实现类所需的逻辑。
*   调用者不必知道所有的实现类，只需要知道协议或基类。
*   依赖组件必须只知道协议或基类和公开工厂方法的类。
*   该模式允许更改现有的实现类或添加新的实现类，而无需向依赖它们的组件公开这些细节。
*   *工厂法:传承创造。*原型:通过委托创作。

> 工厂方法根据调用者提供的标准选择要实例化的实现类。调用方必须只知道一种类型。

示例:调色板–用户需要提及调色板的名称，与该调色板相关的所有颜色都将被填充并准备好。

调色板-用户需要提及调色板的名称，所有与该设置相关的颜色将被填充并准备好。

如果我们想防止向调用者公开所有的实现类，工厂方法模式是一个很好的选择。

_ 请注意，当没有公共协议或共享基类时，不能应用工厂方法设计模式。_
