# 然而，劳里编码

> 原文:[https://dev.to/laurieontech/nevertheless-laurie-coded-2hnh](https://dev.to/laurieontech/nevertheless-laurie-coded-2hnh)

我 14 岁时开始为高中的一个班编程。我不认为这是“真正的”编码。整个大学期间，我继续学习一些编程课程，尽管走了一两次弯路，但最终还是找到了我的第一份开发工作。我爱我的故事，我为我的故事感到骄傲，它比我刚刚发布的要长得多。问题是，精彩的“我是如何开始的”故事并没有抓住这个行业代表性不足的现实。

我们每天都在增加不同申请者的数量，但我们不会因为他们没有留下来而改变主意。我想用今天的时间来谈谈这是如何影响我的，以及我认为我们在女性和非二进制编码者中共同分担的负担。

我喜欢漫威电影宇宙。请看下面昨天的证据！

漫威漫画中有很多女性超级英雄。令人难过的是，我们花了 15 部电影才最终得到一部由女性主演的电影。我今晚要去看那部电影，但是我已经在网上关注了几个星期了。那些因为被“强迫”去看一部他们认为不会与之相关的英雄电影而感到不安的人。那些花费时间和精力为一部尚未上映的电影提交糟糕评论的人。这种情况一直压在我身上，我想这是因为它离家太近了。

作为一名技术领域的女性，我经常看到其他人试图在这个行业为我们开拓空间。因此，男性不再是默认的，这样我们的参与就不会感觉像是被迫的、带角的代表，而是和其他人一样的有效的、聪明的声音。我们经常看到，一个代表性不足的工程师，基于她对行业的惊人贡献，得到的关注越多，她就越成为骚扰的目标，那些人希望不断质疑她的技术资质。

这很累人，有时候你只想坐着不动。但是很多天我们都想战斗。为我们的地位而战，证明那些混蛋是错的。

所以我们努力让自己有代表性。我们看着空间，那里没有足够的我们，我们试图加入他们。我们希望我们在这些空间的存在和贡献将鼓励其他人继续战斗，或者甚至改变想法。

对我来说，有太多的女工程师同事没有能力花自己的时间去填补这些空缺。所以我试着利用此刻我所拥有的时间，以及我所带来的经验。我一直专注于让自己更加引人注目，这样我就可以成为一个资源和代表。这意味着**在会议上谈论**从 DevOps 到 Javascript 应用，再到通信和技术领域的把关词汇。我在**写博客**的次数比以往任何时候都多，这是一个接触新技术和提高我简单表达技术话题的能力的机会。从管道开始到结束，我继续在我的当地社区做志愿者，支持我的同事和朋友。我已经开始尽我所能为开源项目做贡献。我**参加 Twitter 聊天**讨论相关话题，并支持那些刚刚踏上编码之旅的人。我会继续努力成为公司开发团队的优秀贡献者，提高我的技能和能力。

但我是一个人，我们都只是一个人。这个清单太长了。我已经投入了这么多的精力在这个行业中占据更多的空间，但感觉还是不够。这是被代表的诅咒，你希望到处都有代表，所以你试图填补空白。但现实是，我们谁都做不到这一切。所以今年我会不及格。我要伸展自己，功亏一篑，筋疲力尽，不管是什么。然后我会得到朋友们的支持，重新站起来，再试一次。或者他们会建议我辞职几周，休息一下。*拥有一个由和你有相似行业经历的人组成的支持系统是非常重要的！*

2019 年我还在继续编码，因为我看到那么多被吓退的人。我与初中和高中学习编程的女生一起工作，令我感到鼓舞的是，与前几十年的学生相比，她们这一代人将获得更多支持来追求 STEM。我之所以继续写代码，还因为我从那些才华横溢、充满激情的女性那里听到了一些故事，她们不再觉得自己在这个行业有一席之地，也不想再为开拓一个微小的空间而奋斗。

从那些对代码感兴趣的人，到那些多年来一直从事编码工作的人，令人沮丧的故事似乎往往比积极的故事要多。我每天都在为我的社区写代码和做贡献，我觉得我好像是又一个反对那些想把我们拒之门外的文化力量的女人。
