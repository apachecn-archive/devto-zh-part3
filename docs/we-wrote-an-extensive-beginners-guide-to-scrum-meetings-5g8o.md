# 我们为 scrum 会议写了一份广泛的初学者指南

> 原文：<https://dev.to/svikashk/we-wrote-an-extensive-beginners-guide-to-scrum-meetings-5g8o>

开发人员被拉进了太多的会议。

他们中的大多数人显然对此感到沮丧。

跟随 scrum 的团队会召开几次会议，帮助他们更快、更高质量地构建复杂的产品。

但问题是，我们大多数人出于各种原因讨厌会议——从计划外的议程到“我为什么要参加这个会议”，我们都听说过。

当然，你需要根据你的团队来塑造这个过程。但是很多时候，团队最终改变了太多的东西，并最终回到起点——太多的会议，惹恼了开发团队。

因此，我们决定编写一份深入的指南，详细介绍什么、谁、多长时间以及一些最佳实践。

这是指南的链接。

我很乐意接受您的反馈并改进指南，这样其他人也可以从中受益。
