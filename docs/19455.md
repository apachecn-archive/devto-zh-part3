# 你复制粘贴代码吗？

> 原文:[https://dev . to/flipped coding/do-you-copy-and-paste-code-1cm 4](https://dev.to/flippedcoding/do-you-copy-and-paste-code-1cm4)

你很难找到一个活着的没有复制和粘贴代码的开发人员。它可能是旧项目的代码，也可能是堆栈溢出的代码片段。你知道吗？这没什么不好。

知道如何复制和粘贴代码可以节省您数小时的时间。虽然复制和粘贴代码有一个微妙的技巧，每次都能吸引新的开发人员。在决定使用其他地方的代码片段之前，您必须注意一些事情。

## [](#you-need-to-understand-the-code)你需要理解代码

听我说完。即使你使用的代码有效，你也需要知道为什么。假设您后来发现了一个 bug，调用堆栈会将您带回代码块(这很可能会发生)。你是做什么的？为什么它不起作用？

这就是为什么你需要确切地理解代码在做什么，为什么。大多数代码片段需要稍加编辑，或者对项目的设置有所了解。有很多好的代码对你有用，但是你需要知道如何将它与你的本地代码集成。

## 留心潜在的错误或不相容之处

确保您复制和粘贴的代码符合项目的要求。检查它是否是最新版本，以及它是否与您的代码版本兼容。在 React 和 Angular 这样的框架中尤其如此。如果不小心，获得错误版本的代码可能会破坏整个项目。

还要注意那些奇怪的错误，它们让你无论如何都可以构建和运行项目。他们会在意想不到的地方引起混乱。当这些发生时，需要大量的挖掘文档来修复。如果你遇到这些问题中的一个，你可能想在深入之前看看其他的选择。除非你*有*使用那段代码。

## [](#dont-be-afraid-to-use-the-code)不要害怕使用代码

有时候，当你进入状态，代码从你的手指中流出，你忘记了使用代码来简化这个过程！编写自己的功能很酷。我认为这有助于拓展你作为开发人员的能力。只是当你有截止日期和要会见的客户时，你就没有那么多时间来开发自己的功能，除非它是一个超级定制的应用程序。

往前走，走捷径。如果你已经检查了以上两点，那么你就不会有任何问题了。如果有更好的东西出现，不要犹豫去测试它。不同的代码可以帮助您的应用程序运行得更好或更快。学习如何编码的一部分是知道何时何地使用它。不要害怕。

所以继续复制粘贴吧。一个额外的好处是，当你复制和粘贴代码时，你正在学习如何阅读别人的代码和他们是如何写的。这将帮助你成长为一名开发人员。通常，如果一个代码片段被频繁使用，它会是一个非常可靠的代码片段。这意味着是一个非常可靠的开发人员写的。那才是你要学习的人。

你觉得使用复制的代码怎么样？有些人讨厌其他开发人员只是复制和粘贴代码。
