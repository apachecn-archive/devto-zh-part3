# 当然，HTML 是一种编程语言

> 原文：<https://dev.to/mortoray/of-course-html-is-a-programming-language-4ll5>

HTML 是编程语言吗？我可以对这个问题表示一点震惊，甚至沮丧，但相反，我会尝试一种更巧妙的方法。在某种意义上，这可能是一个合理的问题，在这种情况下，它应该得到一个公正的答案。从另一个意义上说，这可能是一种守门的尝试。我想这个问题也必须解决。

我首先关心的是，说它不是编程语言有什么价值？它是否让我们对其目的有所了解？HTML 被程序员用来创建 ui 还有疑问吗？HTML 有源代码有疑问吗？我们是否怀疑 HTML 源代码像其他源代码一样受制于相同的发布系统、需求管理、版本控制、同行评审和测试？我不确定说它不是编程语言会带来什么样的清晰度。

然而，在我猜测我们还能问什么之前，让我们从更技术性的角度来看。

HTML 是一种声明式编程语言。与告诉计算机如何做事的[命令式编程语言](https://mortoray.com/2017/05/10/what-is-imperative-programming/)不同，声明式语言告诉计算机结果应该是什么。处理代码的引擎找出一种方法让系统处于这种状态。

有许多特定领域的声明式编程语言。配置文件通常也属于这一类。选项列表声明您希望如何配置系统。在 DevOps 中，配置始终被称为代码。

偶尔我们会在讨论中看到图灵完备性。图灵机是分析计算和一般算法的极好工具。它们被用在谈论可计算性，我们是否能找到一个问题的答案。虽然它们可以模拟许多语言和机器，但它们是现实世界实际计算的可怕模型。他们模拟，但不能建模，许多我们期望的特性-包括一些基本的东西，如随机访问列表。

如果我们问什么是编程语言，我认为我们需要停留在实践层面。让我们把图灵完备性留给理论。在这个领域进行适当的讨论需要更多的细节。我没有忽视它。我觉得这很有趣，但与我们的问题无关。

什么是实用的定义？编程语言是一组明确指示计算机应该做什么的代码。HTML 指导计算机如何为用户创建可视文档或应用程序界面。Python、Java 或 C++给出命令序列，告诉计算机如何操作内存、修改文件和在网络上交谈。哈斯克尔或奥凯姆尔创造了计算结果的方程结构。

> 我可能需要小心“明确”这一点，因为许多语言都有黑暗的角落:语言中无意的缺陷，有意的未指定行为，以及完全令人讨厌的未定义行为。所以，我们将不得不忍受大部分的不含糊。

不同的语言对计算机的指令不同。我们认识到它们并不适合所有的情况。再也没有全领域通用的语言了。

那么回到问题，说 HTML 不是编程语言会带来什么价值？恐怕答案与守门有关。有些人觉得有必要根据一些人为的目标来区分程序员的素质。它有助于将程序员的角色人为地保持为精英——就精英的某些定义而言。

这并不像“程序员”这个标签神奇地让一个人有资格做所有类型的编程。这是一个很大的领域。我们有很多工具和方法。只懂 HTML 的人不太可能被雇佣来编写游戏引擎或嵌入式传感器的程序。专攻 NumPy 的数据分析师也不适合移动界面。

但即使如此，谁只知道 HTML？也许我们谈论的是使用它来创建原型的 UI/UX 设计师。这些人将与程序员一起工作，帮助他们得到他们想要的东西。我不认为他们自己吵着要被承认为程序员？虽然我认为他们是开发者；我的书中有一章讲述了为什么我认为这很重要。

否则，我见过的大多数做 HTML 的 web 程序员似乎也在做 CSS 和 JavaScript。只使用 HTML 很难编写界面；它不可避免地与其他技术相结合。不一定，但这是最常见的情况。如果有人想否认 JavaScript 是一种编程语言，那么，让我们忽略这个愚蠢的想法。

总之，HTML 是一种声明式编程语言。它不能做所有的事情，但是没有一种语言可以。说它不是编程语言没有实际价值。虽然只懂一种语言的程序员会受到限制，但这适用于所有语言，包括 JavaScript、C#、Python 等等。我们可以说这个理论上只知道 HTML 的人不是一个全面的程序员，但不是说他们根本不是程序员。与其试图排斥他们，为什么不与他们合作，鼓励他们学习所有其他可以做的美好的事情？
