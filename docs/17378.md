# 10 年专业开发经验。

> 原文:[https://dev . to/hugao marques/10-years-as-a-professional-developer-12n 5](https://dev.to/hugaomarques/10-years-as-a-professional-developer--12n5)

2018 年 12 月 8 日，我做了 10 年的软件工程师。这篇文章收集了我这段时间的记忆和学习。

很久以前，确切地说是 10 年前，我曾住在巴西。我刚刚被一家跨国咨询公司聘为实习生。我们的分公司很小，我的主要工作是支持和修复电信服务中的错误。我最自豪的时刻是，该服务无法处理客户订单来创建 31，000 条新线路。处理超时。我发现这个问题是由两个嵌套循环引起的。我能够重构代码来提取这两个循环，这使得我们能够完成处理。为什么我以此为荣？那是我第一次在大学之外用算法分析解决现实世界的问题。在埃森哲学到的最好的经验是如何与不同的人沟通和合作。

从 2010 年到 2011 年，我离开了我的家乡，在不到 7 个月的时间里在两家不同的公司工作，然后回到家乡，在埃森哲工作了整整一年。在那一点上，我对我的职业感到非常失望，主要是因为我从事的是 CRUD 应用程序，没有学到任何东西，也没有有趣的挑战。

2012 年，我们搬到了巴西首都巴西利亚，那里有很多适合像我这样的 Java 开发人员的合同工工作。在那里，我找到了第一份承包商的工作。这是一个新的开始，我感到焕然一新，重新开始投资我的职业生涯。那一年我参加了 3 个 Java 认证考试(Java 程序员、Java Web 开发人员和企业 Java 开发人员)和 1 个 IBM 认证(面向对象的分析和设计)。我参加了 hibernate/JPA 培训，并成为团队中 ORM 方面的专家。我主要使用 Java EE 堆栈和 Flash ActionScript (RIP)。也是在那个时候，我对干净的代码、编码最佳实践和软件架构产生了兴趣。

2013 年，我去了一家初创公司工作，但那真的没有回报。CTO 的期望和我自己的期望明显不匹配。从那个时期吸取的两个教训是:

1.  这里学到的一个教训是:永远不要在你是局外人的家族企业中工作；
2.  总是提前调整你的期望。

同年，我在巴西教育部签了一份合同。我喜欢那份新工作。我作为该系统报告模块的主要所有者，参与了一个允许巴西学生出国留学的项目。我负责需求、设计、开发、测试和监控部署。在那份工作中，我学会了拥有自己的产品。

然后在 2013 年，我收到一封来自亚马逊招聘人员的电子邮件，要求参加亚马逊在巴西圣保罗的招聘活动。我加入招聘过程时认为自己没有机会，我对算法从来都不擅长，但撇开冒名顶替综合症不谈，我决定尝试向前迈进。接下来的 3 周，我日以继夜地解决问题，重温所有我认为有用的数据结构和算法:列表、树、集合、地图、排序、搜索、图表。长话短说，2014 年 10 月，我通过了招聘环节，收到了亚马逊的邀请，来到华盛顿州西雅图市。

从 2014 年开始，我一直在亚马逊工作。迄今为止，我已经在 3 个不同的团队工作过:亚马逊印度、客户评论和现在的 AWS 构建工具。在亚马逊，我学到了各种有趣的东西:如何查看指标，如何进行监控和报警，如何协调和部署与多个服务通信的解决方案。在那里，我能够编写我的第一个 API，供全世界数百万人使用，我用各种不同的语言编写和阅读代码。如今，在 AWS Builder Tools，我开发工具来帮助亚马逊工程师提高工作效率，交付更高质量的解决方案。

那是我职业生涯的一万英尺视角。从我家乡一个小办公室的实习生到世界上最大的科技公司之一的软件工程师。我希望这篇文章能帮助其他人，激励他们去尝试和寻找自己的道路。

干杯！
