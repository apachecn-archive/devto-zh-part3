# 聚焦

请务必注意以下几点:

*   理论
*   例子
*   动手实践(小问题)
*   全面、真实的项目

还是别的？

# 格式

这是我真正希望你发挥创造力的地方。有一些成熟的编程教学格式:

*   文本+图像/图表(阅读)
*   视频(讲师教学内容)
*   练习题(计算机检查)

但这也让人觉得在这个领域还有更多实验的空间。

我一直在探索的一种可能性是“视频”(讲师的音频+动画文本编辑器)，其中涉及教师展示一些东西，但它会暂停并变得互动(你可以在编辑器中键入)，在视频继续之前完成一个挑战。

我很乐意听到关于这个想法的反馈，以及你能想象到的任何东西！不要害怕用这个变得滑稽。如果够疯狂的话，我可能会去做。；)
