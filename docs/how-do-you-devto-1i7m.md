# 你怎么去？

我最近加入了 DEV.to 社区，我很好奇你们每个人是如何使用这个平台的。我猜我的故事和你的大部分相似，所以这是一个谈论它的好地方。

我是一名拥有近 20 年经验的软件开发人员，我写博客已经有很长时间了。我发现自己最近在自己的平台上写博客越来越少，创作的视频越来越多。这让我有点难过，因为我喜欢写作，除了编码，这是我最喜欢的事情之一。

我只在这里贴了一篇文章，就已经喜欢上了。我收到了很多评论和反馈，这似乎是一个和像我一样的开发者交流的好地方。

[![Code](img/de88f9ed0bbaa05ff1fce49148f2fb72.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--nxdFd3yF--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/imi6vg61bhukgtmgr3e9.jpeg)

所以我的问题是:对于那些有个人网站和博客的人来说，你发布新内容的策略是什么？你在这里和你的博客上发帖吗？我在独立黑客播客上听到本谈论媒体的问题。我对媒体和其他平台最大的问题是，你没有自己的内容。

当我在我的博客上发布内容时，我不仅试图帮助人们，而且试图树立我的名声。我希望建立我的邮件列表，当我要求付费内容时，我希望我的订户知道它来自一个好地方。

我应该把我所有的内容都放在这两个博客上，还是你决定把一些内容放在个人博客上，也许先用这个平台测试一些内容？很好奇大家都是怎么使用平台的。
