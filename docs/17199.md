# 解码前端面试过程

> 原文:[https://dev . to/emmabostian/decoding-the-front-end-interview-process-14dl](https://dev.to/emmabostian/decoding-the-front-end-interview-process-14dl)

作为前端开发人员，难免会遇到前端技术面试(或者十个)。不幸的是，前端开发面试的过程可能有点不清楚，并且会因公司而异。

对这个过程有一个坚实的理解会帮助你在面试中感觉更舒服。面试是一项需要练习的技能，但是洞察面试过程可以让你平静下来。

所以让我们来看看前端开发面试流程，并学习一些成功面试的技巧。

[![Process](../Images/fde6c8c65113c7d673488c5d50c2cf6e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Xg2IzMhT--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/2000/1%2AyQ66ke-0OnlzZj4q3P3RvA.jpeg)

我有过很多前端工作面试。事实上，面试是大多数人认为理所当然的技能，前端开发面试是最困难的面试之一。

它们要求你从概念上理解你使用的语言(HTML，CSS，JavaScript)，就地编码(eek)，利用计算机科学中的排序和搜索算法(冒泡排序，合并排序，二分搜索法等)。)，并了解许多相关的技术和领域，如 HTTPS、REST 和软件开发方法，如敏捷。

根据我的经验，在美国各地面试工作时，以及在德国，这是大多数公司的一般面试流程。

[![Diagram of interview](../Images/4fd17086c0662af5c05308a007dc652b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--kgjg7szG--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/2000/1%2AQBQ-xzE7Y-gwSVWBCJD9Bg.jpeg)

所以让我们来分解一下前端开发面试的不同组成部分。

[![Cultural Interview](../Images/48897e0cb7f40aeb6b370ee70524f687.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--HO4eYKn_--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1600/1%2AfcZywk0H7Od5QMt1RTdAvQ.jpeg)

文化面试通常是应聘者在公司的第一印象，所以不要轻视这些面试。虽然典型的文化面试不像编码面试那样可怕，但是面试官可以决定你申请的成败。

最有可能的是，这次面试将由招聘人员主持。他们可能有，也可能没有开发经验，但他们应该能够就前端技术进行明智的对话。

给招聘人员留下深刻印象是很重要的，因为他们的意见会对你的申请产生巨大的影响。如果你进入了第二轮或第三轮面试，并且公司对是否让你通过持观望态度，招聘人员可能是你最大的支持者。所以一定要留下深刻印象。

典型的文化面试以“告诉我关于你自己的情况”开始。对这个问题必须有一个深思熟虑的回答。

例如，让我们检查两个响应:

然而，和佩吉在一起时，她更善于言辞。她不仅能够简洁地总结自己的经历，而且她的回答引发了对她之前角色的情感和自豪。此外，加入一两句关于她的爱好的话，可以让你了解她是什么样的人。

### [](#tips-for-the-cultural-interview)文化面试小贴士:

*   准备一个小故事来说明你是谁
*   用 1-2 个故事展示你最大的成就和你解决的一个难题
*   对你的技能要诚实。当被问及 Redux 的细节时，如果你只是为了进入下一轮面试而抛出这个流行词，你不会想陷入尴尬的境地。
*   不要抨击你现在或过去的雇主。当你对你以前的公司说负面的话，这表明缺乏尊重。未来的雇主需要知道你会对他们忠诚，即使你的工作已经结束。
*   为你的面试官准备一些问题，展示你对公司和你申请的职位的洞察力。在面试快结束的时候，你的面试官应该问你是否有问题要问他们。有两个很有见地的问题。一个好的格式应该是:“我看到你的公司刚刚做了 X…这将如何影响 Y？”这表明你研究过这家公司，并且了解科技行业的最新动态。你也可以问招聘人员在公司工作最喜欢的一点是什么。你可能会感到惊讶！
*   记笔记。你可能会有很多面试，重要的是保持所有面试都直截了当！
*   不要问薪水或福利的细节。一旦你有了录取通知书，这些细节就可以解决了。但是太快询问工作的额外津贴会把你描绘成一个只为钱而来的人。
*   做你自己。这次面试是双向的。你要确保公司文化与你的个性和谐一致。所以不要装模作样；做你自己。

### [](#tips-for-the-coding-interview)编码面试提示:

*   大声思考。面试官听到你的思维过程很重要！
*   如果你对其中一个方向不清楚，就去问。有时候面试官甚至会给你提示。
*   经常快速失败。总是尝试你的想法，当它们不起作用时，从中学习并迭代。
*   如果你以前收到过面试问题，并且知道答案，你应该让你的面试官知道。如果你一步一步地知道答案，面试官会告诉你你查过答案(或者之前就知道了)。诚实永远是最好的策略。
*   如果您有解决方案，请尝试提高性能或运行时间。优化是关键。
*   如果你不知道答案，没关系。我经常承认:“老实说，我不确定，但如果我必须做出一个有根据的猜测……”我见过候选人在一个问题上完全胡说八道，并假装完全正确。在我看来，当你不知道的时候最好承认，但是给出一个有根据的猜测。

[![Take Home Challenge](../Images/af76512186e2c92ecc527dc86fdacb1a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--7HIualmu--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1600/1%2AeJOHIN5rXy6D9pJr7ZyQgA.jpeg)

有些工作可能不需要你完成带回家的挑战。但是如果他们这样做了，这是你展示你令人敬畏的开发技能而没有被监视的压力的机会。

带回家挑战的类型可能有所不同。您可能会得到一个简单的“创建待办事项列表应用程序”提示。这是一个巨大的挑战，因为它允许你使用 JS 框架，如果你愿意的话，但是也可以用普通的 JS 来完成。另外，完成起来也不太费时间。

你可能会得到一个你必须在线完成的大会挑战的链接。这些是偶尔定时的，但是不要让它吓到你。

或者，您可能会被要求重新创建公司应用程序的一部分。

无论挑战是什么，尽你所能，写出干净的代码。以下是我带回家挑战通常遵循的流程。

### [](#questions-to-ask-yourself-before-submission)提交前要问自己的问题:

*   我是否满足了所有的项目要求？
*   我的解决方案是否有效？
*   我的解决方案响应迅速/移动友好吗？
*   我的解决方案可用吗？
*   我的解决方案吸引人吗？
*   我的代码干净高效吗？
*   我对这个解决方案感到自豪吗？

如果以上任何一个问题的答案是否定的，你可能会想回去重新做一些事情。

### [](#tips-for-the-take-home-challenge)带回家挑战小贴士:

*   不要花超过一天的时间。如果你在这上面花了几天时间，最有可能的是你应该回去继续你的一些技术技能。
*   把自己从候选人名单中抽离出来是没问题的。我过去参加过面试，我不得不给招聘人员发电子邮件，告诉他们我的技能不在他们需要的地方。这向他们展示了一定程度的自我意识。我甚至有几次让招聘人员来找我，看我是否仍然对面试感兴趣。当你有需要改进的地方时，承认是可以的。此外，你也不想因为太难的挑战而给自己压力，因为当你得到一份工作时，很可能这份工作也会给你同样大的压力。
*   明确项目需求。他们要你用 JS 框架吗？你能使用 CSS 框架来节省样式的时间吗？
*   如果你需要额外的时间，尽管开口。招聘人员知道你很忙，因此生活会妨碍面试。因此，不要对你的“带回家”挑战敷衍了事，给面试官发电子邮件，让他们知道这对你来说极其重要，你希望投入适当的时间来完成它。大多数公司更希望你在评估上多花几天时间，而不是牺牲质量。

你是个坏蛋，公司知道这一点！如果你收到一份工作邀请，请记住以下几点:

*   问问你能考虑多久？大多数经理会给你一周左右的时间来给出答案。
*   说不可以，如果你觉得这份工作不会让你开心，那就不要接！
*   如果与你的工作岗位和工作地点的平均工资相称，你可以要求更高的工资。
*   询问是否有搬迁援助。如果你像我一样，穿越整个国家(或世界！)，缺乏经济援助可能是一个交易破坏者。
