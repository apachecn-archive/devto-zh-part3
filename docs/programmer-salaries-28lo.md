# 程序员工资

> 原文：<https://dev.to/kspeakman/programmer-salaries-28lo>

今天，我偶然发现了这句话，我将首先对它进行外科手术式的修改。

> 现在，人们经常会听到这样的观点，即 __ ___ _______ 编程是一个报酬过高的职业，在未来几年，程序员的工资可能会下降。

你可能认为这句话可以在今天的讨论中找到。但是没有。引文中缺少的词是“在六十年代”。这段话来自 Edsger W. Dijkstra 在 1972 年接受 ACM 图灵奖时发表的题为[谦逊的程序员](https://www.cs.utexas.edu/~EWD/transcriptions/EWD03xx/EWD340.html)的演讲。

有趣的是，讲座中对硬件成本下降的猜测已经成为现实，当你考虑到相对于计算能力的成本时，这几乎是不可想象的。但是软件开发的成本似乎并没有减少。事实上似乎相反，发展中国家现在比过去得到了更好的报酬，甚至需要更多我们这样的人。科技行业已经基本上接受了成本范式翻转——即(定制)软件现在的成本远远高于硬件。
