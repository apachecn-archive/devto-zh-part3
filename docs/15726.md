首先让我们了解什么是认证和授权。

# 认证

身份验证是验证声明的属性是否真实的过程。在 IT 环境中，一台机器或一个人声称自己是某个人，例如某个实体，并向某个人，例如他想从中获取某些资源的服务器证明
他实际上就是他所声称的那个人。在认证过程中，不仅涉及索赔，而且涉及索赔的确认。因此，如果确认成功发生，则认证成功，否则认证失败，因为声称拥有身份的人未能证明它。

每次登录网站时，您都使用身份验证向网站服务器证明您确实是您所声称的那个人。没有安全认证，每个人都可以自称是别人。

# 授权

身份验证过程成功完成后，授权过程将决定您可以访问哪些资源。因此，授权基本上是关于分配角色、权利和特权，并确保没有人能够看到或修改超出其范围的资源。

因此授权可以确保只有你和网站的管理员可以修改你发布的内容。想象一下，在成功的认证过程之后，每个人都可以修改网上的内容。不仅仅是混乱。限制资源的读取也非常重要。不是每个人都应该看到私人共享内容或敏感数据。

# 结论

身份验证和授权对于在整个 IT 中保护资源和限制访问非常重要。
