# 领先程序员的有点傻的历史

> 原文:[https://dev . to/just zeros/a-稍微有点傻的领先程序员历史-15d8](https://dev.to/justzeros/a-slightly-silly-history-of-leading-programmers-15d8)

很久很久以前，在一个很远很远的地方，经理们告诉人们该做什么。这看起来很自然，因为经理比工人更聪明，知道的更多。很少有工人对这一点提出异议，他们从婴儿起就接受“家政经理”(又名“父母”)、在学校接受“教室经理”(又名“老师”)以及各种低工资工作(又名“塔可钟经理”)的培训。)尽管管理者在成为管理者时经历了类似的条件反射，但这些想法很快被一种感觉所取代，即他们是“精英”，已经“超越”了普通员工。

最终，那些身居公司最高层的人决定，他们需要用一个不同的名字称呼自己，以显示他们比经理们更优秀。他们自己选择了“执行者”这个词，这个词的词根是“执行”，这是一种特别暴力的杀人方式。这个词的选择并非出于偶然，它有助于在经理和员工心中制造适当的恐惧。

(当然，一些工人需要感觉比某人优越，所以管理层为最不重要的工人发明了头衔:“实习生”(军事囚犯的另一种说法)、“见习员”(训练中的新手)和一直流行的“初级”(更年轻或更小的同义词)是常见的选择。这让没有这些头衔的*员工感觉良好，尽管他们仍然不适合做经理。但是我跑题了。)*

当管理人员从工人的行列中毕业时，他们接受了如何安排人员、衡量绩效、纠正行为和解雇人员的培训。许多经理很惊讶他们接受了培训，认为如何做这些是与生俱来的，而不是学来的。新经理们怀疑他们与工人没有太大区别，任何人都可以学习成为一名经理。聪明的人保持沉默，以免显得不那么善于管理。

新经理们很快发现，他们必须采取两种截然不同的立场才能获得成功，而且通常是同时采取。这有时需要一点扭曲。在他们的团队面前，他们采取了*“我们在这里都一样”*的姿态，所以人们会喜欢他们。这让他们的团队感觉他们是“帮派中的一员”

但是对于他们的老板，经理们采取了“我们与工人不同”的立场，所以他们的老板会觉得他们适合这份工作。如果一个人不适合做经理，他们可能会被降职或解雇，或者在某些奇怪的情况下，被提升为高管。规则不明确，但总体来说是有风险的。

在这两种姿态之间跳舞是有压力的，但却是生存所必需的。放弃任何一种立场或者忘记使用哪一种立场都会让管理者面临巨大的风险。

随着时间的推移，这些经理中的一些人不断晋升，发现不再需要对工人采取*“我们都一样”*的立场，因为他们现在只与经理联系在一起。这是一种解脱，因为所有的扭曲、跳舞和摆姿势都让人筋疲力尽。经理们可以专注于他们的*“我们不同于工人”*立场，只要他们记得他们也不同于高管。

数百年来，这种体系一直延续着:经理们知道得最清楚，工人们按照他们被告知的去做，每个人都害怕高管。

但是后来，事情发生了变化。电脑出现了，让一切都陷入了混乱…
