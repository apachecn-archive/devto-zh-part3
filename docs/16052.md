啊，一月。

一个乐观的时代，一个重新开始的时代。

在 web 开发世界中，为了跟上时代的步伐，我们要着眼全局，决定我们需要学习什么。

### [](#deciding-what-to-learn)决定学什么

因为事情变化如此之快，决定*关注什么*是不被淹没的关键。

我在[之前已经介绍过我是如何决定学什么的](https://dev.to/kball/learning-web-development-in-2018-2knc)，用一个寻找动机、动力和金钱的“3 个月”框架。

这个想法的基本框架是寻找我感兴趣的话题，这将为我未来的学习提供一些动力，并且我可以获得报酬或有明确的技能市场。

下面我根据我现在在 web 开发生态系统中所看到的来应用这个框架，对于每个领域，我都推荐了一些资源来帮助你开始。

对于每个主题，我都提供了一系列免费和付费的资源。作为一个参考，一些付费资源确实使用附属链接，但我已经尽了最大努力不让这影响我的推荐，并确保我始终有免费的选择。

## [](#1-typescript)1。以打字打的文件

TypeScript 是 2018 年最令人惊讶的增长故事之一。npm 调查发现惊人的 46%的 npm 用户正在使用 TypeScript。它现在不仅是使用 Angular 的默认语言，也是 React 中常用的选项，而且是 Vue 中越来越常用的选项。

已经宣布 Vue 3 正在用 TypeScript 编写[，Node.js 的创建者正在为](https://medium.com/the-vue-point/plans-for-the-next-iteration-of-vue-js-777ffea6fabf)[开发一个新的类似 Node 的项目](https://deno.land/)，它首先是 TypeScript。

现在 [Babel 支持 TypeScript](https://blogs.msdn.microsoft.com/typescript/2018/08/27/typescript-and-babel-7/) ，你甚至不需要改变你的构建系统。

因此，如果你一直坚持己见(老实说，像我一样)，2019 年绝对是学习 TypeScript 的一年。

它有发展势头，越来越多的工作需要它，这很明显是 JavaScript 生态系统的发展方向。

## [](#2-react)2。反应

对于你们中的许多人来说，React 已经是旧闻了，但是我出于两个原因再次将它放在列表的前列。

首先，它仍然非常肯定地赢得了“金钱”和“动力”的游戏。在前端世界里，React 的工作和合同比其他任何东西都多，所以如果你还没有选择它，你一定要选择它。

第二——它一直在变化，一些新特性(尤其是[钩子](https://reactjs.org/docs/hooks-intro.html)，还有像上下文 api 这样的东西)有望彻底改变我们编写 React 代码的方式，所以即使你已经在使用 React，你也应该重温一下。

### [](#resources-and-articles)资源和文章

*   [React 手册](https://medium.freecodecamp.org/the-react-handbook-b71c27b0a795)。Freecodecamp 博客上关于 React 的 80/20 介绍，旨在让你快速开始使用 React。
*   [勾一瞥](https://reactjs.org/docs/hooks-overview.html)。针对那些已经熟悉 React 的用户的钩子文档；可能是学钩子最好的地方。
*   [全栈反应](https://www.fullstackreact.com/articles/)。一个博客和一份时事通讯都有很好的深入的 React 文章
*   [React 播客](https://reactpodcast.simplecast.fm/)。如果你对播客感兴趣，这个由[陈宸](https://twitter.com/chantastic)又名 Chantastic 制作的很棒。

## [](#4-css-grid)4。CSS 网格

CSS Grid 已经达到了跨浏览器的临界质量，几乎可以满足任何你想要满足的用户，这是从以前的布局技术向前迈出的惊人的一步。

如果您仍然在使用来自 Bootstrap 或 Foundation 等 UI 工具包的重量级网格框架，那么您就落伍了。CSS Grid 以更少的标记和复杂性为您提供更多的功能。

唯一的障碍是学习。这也是这篇文章的重点。因此，如果你仍然坚持己见，那就把这一年作为你学习 CSS Grid 的一年。

### [](#free-resources)免费资源

*   [CSS 网格初学者指南|免费代码营](https://medium.freecodecamp.org/a-beginners-guide-to-css-grid-3889612c4b35)。一个伟大的，简单的，非常直观的 CSS 网格逐步指南。
*   [Grid | CSS 窍门的完整指南](https://css-tricks.com/snippets/css/complete-guide-grid/)CSS Grid 最全面的在线资源之一，展示了所有与网格相关的属性、它们的潜在价值，以及所有这些价值的可视化演示。
*   CSS 网格布局的最佳实践| Smashing Magazine CSS 网格已经存在并被使用了足够长的时间，我们开始看到一些最佳实践的凝结。Rachel Andrew 通过自己的经验(她是 CSS Grid 的最重要的倡导者和教师之一，并且参与了规范的制定)和对使用 Grid 的开发人员的调查了解了其中的一些内容。
*   关于 CSS Grid | CSS 窍门的另一个有趣事实集合一组关于如何使用 CSS Grid 的经验教训，一些速记属性，以及处理显式和隐式行和列的方法。充满了基于 codepen 的示例，因此您可以深入研究并随心所欲地修改代码。
*   [通过实例网格](https://gridbyexample.com/)。一组精心策划的 CSS 网格示例。
*   [CSS 网格花园](https://cssgridgarden.com/)。一个伟大的基于游戏的学习工具。

### [](#free-course)免费课程

*   [CSS 网格| Wes Bos](https://cssgrid.io/) 。很棒的入门课程，由 Wes Bos 教授，Firefox 赞助，对你我完全免费。

## [](#5-graphql)5。GraphQL

GraphQL 是一个大有前途的东西，我认为 2019 年可能是它真正开始成为主流的一年。

当然，它已经引起了很多热议，尤其是在 react 生态系统中，但是 [npm 的数据](https://slides.com/seldo/npm-and-the-future-of-javascript#/55)显示，buzz 的使用也在以极快的速度增长。

我不确定有多少工作还需要 GraphQL，所以这可能不会马上赚钱，但是从动机和动力的角度来看，现在学习它会让你在未来处于一个非常好的位置。
