# 我的 2019 年成为更好的开发者的计划。

> 原文:[https://dev . to/carloteran 19/my-plan-to-be-a-better-developer-in-2019-4li 7](https://dev.to/carloteran19/my-plan-to-become-a-better-developer-in-2019-4li7)

我知道你可能在想什么:“已经快 2 月了，陈词滥调的新年决心帖子还在我的时间线上滚动(现在在 dev.to 上)。”我不怪你！但即使我通常不参与新年决心热(通常我会因为没有坚持下来而感到沮丧)，今年我觉得我需要一个，如果我想在年底前看到切实的结果，我需要制定一个好计划。

## [](#the-reason)原因

回顾 2018 年，这是我在洛杉矶一家小型 it 咨询公司担任网页开发员的当前(第一份)工作的一年，我熟悉了许多新的网络技术和“最佳实践”，我是获得 Udacity 颁发的[移动网络专家纳米学位](https://www.udacity.com/course/mobile-web-specialist-nanodegree--nd024)全额奖学金的谷歌申请人的一员。

尽管这一年听起来并不糟糕，但缺乏切实的成果让我感到挫败。我仍然没有**一个像样的作品集**，我没有完成我开始的大部分**在线课程**，我没有参与**开源项目**，我不太了解 **TDD** ，我没有写足够的**博客文章**等等...

只有我有这种感觉吗？其他出色的开发人员是如何做到的？要成为一名优秀的开发人员，我需要花多少年的时间在周末编码？

我知道我必须用不同的方式做事，但是怎么做呢？

然后，我做了一些研究...

首先，我发现我不是唯一有这种感觉的人。Erik Trautman 在本帖中解释了学习如何编码的多个阶段和困难。我意识到我目前正处于他称之为**绝望沙漠**的阶段:

> 当你进入第三阶段时，知识的范围会扩大。你现在需要了解使用什么工具，学习什么语言，CS 的基础知识，如何编写模块化代码，面向对象，良好的风格，以及如何寻求帮助(仅举几例)。每次访问谷歌或黑客新闻都会让你陷入另一个兔子洞，被更多你不知道但感觉应该知道的事情淹没...
> 
> 即使你能够运行一些应用程序，也很难不在成为真正专业人士的大计划中感到迷失。很难衡量你的进步。你怎么知道你需要学习什么，或者你是否正在学习正确的东西？

此外，昆西·拉森在他的[学习编码的故事](https://medium.freecodecamp.org/a-cautionary-tale-of-learning-to-code-my-own-eddb24d9d5a7)中完美地描述了我的情况:

> 但是作为个人，有这么多选择，真的很难决定学什么。我知道是因为我是那条在十字路口绕圈的狗，追逐着一辆又一辆的车。一个好的程序员的技能组合可以用 T 型来形容，在很多领域知识浅薄，但至少在一个领域知识深厚。但是经过几个月的压力和努力工作，我变得更像一条下划线。

第二，我发现有出路。幸运的是，埃里克·特劳特曼和昆西·拉森不仅完美地描述了我的处境，还提出了一条出路。这是他们的建议:

1.  **有目标**。我意识到我试图一次学太多东西。我需要写下我的目标，并学会对那些有意思的事情说“不”,这些事情可能是无益的和令人分心的。

2.  **寻找路径(写计划)**。我需要写一份能引导我实现目标的计划。这样我就可以决定遵循哪个课程并坚持下去。

3.  **焦点**。我相信这篇文章会帮助我保持理智，即使遇到困难也要坚持这个计划。我需要不断地相信我的判断，记住我今天为什么要写这篇文章。通过与你分享，我公开承诺遵循这个计划，并在年底写下结果。

## [](#the-goal)目标

回到 T 型程序员。我想对一般的 Web 开发概念和技术有一个很好的理解，但是我想对 Ruby on Rails 有更深的了解。

我选择 Ruby on Rails 的原因有很多，但为了长话短说，我只需要说，与 Ruby 和 RoR 一起工作让我意识到我想以写代码为生(即使在完成 CS 学位后)。关于 Ruby on Rails 如此有趣的原因，你可以看看 DHH 的 [Rails 教条。](https://rubyonrails.org/doctrine/)

总之，我的目标是成为一名更好的 Ruby on Rails 开发人员，并最终[获得一份 Rails 工作](https://thoughtbot.com/upcase/videos/landing-a-rails-job)(远程、兼职或代理工作)。

## [](#the-plan)计划

我决定把计划分成几个部分。这样，它将帮助我专注于我需要学习的东西，并对我想要完成的项目有一个良好的视觉。

1)我将从一些 Web 开发概念开始，大部分是我去年留下的未定论的概念。2)和 3)我将使用 Thoughtbot 的 [Upcase](https://thoughtbot.com/upcase/) 和他们的[学徒计划](https://github.com/thoughtbot/apprenticeship/blob/master/onboarding-packet.md)作为指导，开始在 Ruby on Rails 上工作。4)我将对 Vue.js 有所了解，并在最终项目中将其整合在一起。
