# 有抱负的 JavaScript 开发人员的 8 大读物📚

> 原文:[https://dev . to/jerod Santo/8-great-reads-for-aspirating-JavaScript-devs-3 BPA](https://dev.to/jerodsanto/8-great-reads-for-aspiring-javascript-devs-3bpa)

在 JS Party #73 的最后一部分，Divya 、 [Emma](https://twitter.com/EmmaWedekind) 和我就**的书**进行了一次闲聊。

那次会议的推荐书目太好了，不能只存在于第中，所以它们在这里:一些<mark>有抱负的 JS 开发者必读的</mark>书！

* * *

## [](#1-refactoring-ui)1。[重构 UI](https://refactoringui.com/book)

推介:

> 学习如何使用从开发者的角度解释的特定策略自己设计漂亮的用户界面。

艾玛推荐。达薇亚添加了她的`+1`，说道:

> 太棒了。对于不认为自己是设计师，但想学习更多的人。太好了。

## [](#2-the-pragmatic-programmer)2。[务实的程序员](https://pragprog.com/book/tpp/the-pragmatic-programmer)

安迪·亨特和迪夫·托马斯的经典之作。推介:

> 《实用程序员:从熟练工到大师》一书直接从编程的角度切入现代软件开发日益专业化和技术性的领域，考察核心过程——如果你想创建易于使用并对用户有益的软件，作为个人和团队，你该怎么做？

杰罗德是怎么说的:

> 一本引人入胜的书，每个开发人员都值得一读。

## [](#3-the-mythical-manmonth)3。[神话中的人月](https://en.wikipedia.org/wiki/The_Mythical_Man-Month)

推介:

> 一本关于软件工程和项目管理的书，由 Fred Brooks 于 1975 年首次出版...它的中心主题是“给一个后期的软件项目增加人力会让它更晚”。这种想法被称为布鲁克斯定律，并与第二系统效应和原型的倡导一起提出。

杰罗德说:

> 可以用一句话概括。“9 孕妇 1 个月生不了孩子。”把这句话应用到软件开发中。

## [](#4-you-dont-know-js)4。[你不知道 JS](https://github.com/getify/You-Dont-Know-JS)

推介:

> 这是一个探索 JavaScript 部分的系列丛书，我们都认为自己了解这些部分，但实际上并不了解。

艾玛说:

> 凯尔·辛普森是我发现的最多产的老师之一..他所有的书都是免费的，这令人难以置信...直到他解释了闭包，我才明白。

## [](#5-professional-javascript-for-web-developers)5。[面向网络开发者的专业 JavaScript】](https://www.amazon.com/Professional-JavaScript-Developers-Nicholas-Zakas/dp/1118026691)

推介:

> 这本书提供了开发人员级别的介绍，以及 JavaScript 的更高级和有用的特性。

艾玛说:

> 这是一本相当大的书。我把它拼凑起来，并且乐在其中。

## [](#6-learn-javascript-amp-jquery)6。[学习 JavaScript & jQuery](http://javascriptbook.com)

推介:

> 我们知道现在各种各样的人都在建网站(而且他们并不都有计算机科学学位)。所以，如果你想用 JavaScript 让你的网页更具交互性、趣味性和可用性，我们可以帮你。即使你从来没有做过任何编程。

艾玛说:

> 这个更适合初学者...我只读 JavaScript 部分。他们使用这些漂亮的图形来解释范围和事件委托等概念。它对初学者非常友好。

## [](#7-code-complete)7。[代码完成](https://www.amazon.com/Code-Complete-Practical-Handbook-Construction/dp/0735619670)

推介:

> 一本由 Steve McConnell 撰写并于 1993 年出版的软件开发书籍，鼓励开发人员继续过去的代码和修复编程以及预先设计和瀑布模型。它也是软件构造技术的概要，包括从命名变量到决定何时写子程序的技术。

达薇亚说:

> 当我思考我所做决定的总体哲学时，我总是会回到这本书上。这本书很长，但可以在网上免费阅读。

## [](#8-learning-javascript-design-patterns)8。[学习 JavaScript 设计模式](https://addyosmani.com/resources/essentialjsdesignpatterns/book/)

推介:

> 设计模式是软件设计中常见问题的可重用解决方案。在任何编程语言中，它们都是令人兴奋和令人着迷的话题...在本书中，我们将探索如何将经典和现代设计模式应用到 JavaScript 编程语言中。

艾玛说:

> 如果你想学习更多有效设计代码的方法，强烈推荐。

达薇亚补充道:

> 可以说，那是我不使用框架时的 goto 圣经。我只是在做香草风格的事情...那本书帮助我理解了什么是单身。
