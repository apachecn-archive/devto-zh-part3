# 3 本推荐的编程书籍

> 原文:[https://dev . to/samatkins/3-推荐-编程-书籍-gfk](https://dev.to/samatkins/3-recommended-programming-books-gfk)

当我反思 2018 年时，有三本编程书籍让我喜欢阅读，并从中受益匪浅。我想我会分享我对每本书的总结。

### [](#the-pragmatic-programmer-by-andrew-hunt-and-david-thomas)《实用主义程序员》作者安德鲁·亨特和戴维·托马斯

老板推荐给我的，这是一本永不过时的编程指南。编程和技术发展很快，尽管这本书是在 1999 年写的，但它仍然适用于今天。在我看来，这一事实表明了这本书教授的原则是多么重要。这本书真的帮助我理解了编程最佳实践，以及如何将这些最佳实践融入到我的日常工作中。随着我在编程生涯中的进步，这是一个不断回归的参考。

### [](#coders-at-work-by-peter-seibel)工作中的编码员彼得·高顺

我老板的另一个推荐，这是对不可思议的程序员的一系列采访。阅读这些程序员在工作中采用的不同方法是很有趣的。

特别有趣的是，在讨论开发 JavaScript 语言时，阅读了布伦丹·艾希和道格拉斯·克洛克福特的对比和互补的哲学和方法。

我也喜欢阅读关于 Brad Fitzpatrick 和他令人难以置信的编程技巧和商业/开源敏锐度的文章，LiveJournal 和 Memcached 就是很好的例子。

从更广泛的行业角度来看，这是一个非常热门和重要的主题，阅读法兰·艾伦关于该行业如何成为男性主导的经历以及她对增加多样性重要性的看法。

### Gene Kim、Jez Humble、Patrick Debois、John Willis 撰写的《DevOps 手册》

这可能是我去年读过的最喜欢的编程书籍。它的全称是 DevOps 手册::如何在技术组织中创造世界级的敏捷性、可靠性和安全性

在转行之前，我的背景是供应链和运营(在制造业中)，包括精益和流程优化工作。这本书是一个真正的大开眼界，从生产运营的角度来构建软件。这本书解释了 DevOps 文化如何看待人、过程和工具，可以极大地提高软件公司在上市速度和软件弹性方面的表现。

基础设施作为代码实际上是使用版本控制的最重要的方面(而不是我之前假设的开发代码)，这是一个巨大的观点，但一旦消化就有意义了。如果你对 DevOps 和团队如何构建软件感兴趣，我强烈推荐这本书。
