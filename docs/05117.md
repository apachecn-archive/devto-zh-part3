# 了解托管数据库

> 原文:[https://dev . to/digital ocean/understanding-managed-databases-43 a9](https://dev.to/digitalocean/understanding-managed-databases-43a9)

## [](#introduction)简介

安全、可靠的数据存储几乎是每个现代应用程序的必备条件。然而，对于许多团队来说，自我管理的本地数据库所需的基础设施可能非常昂贵。同样，拥有有效维护生产数据库所需的技能和经验的员工也很难找到。

云计算服务的普及降低了提供数据库的门槛，但许多开发人员仍然缺乏管理和调整数据库以满足其需求所需的时间和专业知识。出于这个原因，许多企业开始求助于*托管数据库服务*来帮助他们构建和扩展数据库，以适应他们的增长。

在这篇概念性的文章中，我们将讨论什么是托管数据库，以及它们如何为许多组织带来好处。我们还将讨论在托管数据库解决方案上构建下一个应用程序之前应该考虑的一些实际问题。

## [](#managed-databases-in-a-nutshell)概括地说托管数据库

托管数据库是一种云计算服务，最终用户向云服务提供商支付访问数据库的费用。与典型的数据库不同，用户不必自己建立或维护托管数据库；相反，监管数据库的基础设施是提供商的责任。这使得用户可以专注于构建他们的应用程序，而不是花时间配置他们的数据库并使其保持最新。

提供托管数据库的过程因提供商而异，但总的来说，它类似于任何其他基于云的服务。注册帐户并登录仪表板后，用户查看可用的数据库选项，如数据库引擎和集群大小，然后选择适合他们的设置。提供托管数据库后，您可以通过 GUI 或客户端连接到它，然后开始加载数据，并将数据库与您的应用程序集成。

托管数据解决方案简化了配置和维护数据库的过程。不用从终端运行命令来安装和设置，只需在浏览器中点击几下，就可以部署一个生产就绪的数据库。通过简化和自动化数据库管理，云提供商让任何人(甚至是数据库新手)都可以更轻松地构建数据驱动的应用程序和网站。这是几十年来简化、自动化和抽象各种数据库管理任务的趋势的结果，这本身就是对数据库管理员长期感受到的痛点的回应。

## [](#pain-points-of-onpremises-and-selfmanaged-databases)内部和自我管理数据库的痛点

在云计算模式兴起之前，任何需要数据中心的组织都必须提供建立数据中心所需的所有时间、空间和资源。一旦他们的数据库启动并运行，他们还必须维护硬件，更新软件，雇佣一个团队来管理数据库，并培训员工如何使用它。

随着云计算服务在 2000 年代越来越受欢迎，配置服务器基础设施变得更加容易和经济实惠，因为它所需的硬件和空间不再必须由使用它的人拥有或管理。同样，完全在云中建立数据库也变得简单多了；企业或开发人员只需申请一台服务器，安装和配置他们选择的数据库管理系统，并开始存储数据。

虽然云计算确实简化了建立传统数据库的过程，但它并没有解决所有的问题。例如，在云中，在开始收集数据之前，仍然很难确定数据库基础设施的理想大小。这一点很重要，因为云消费者是根据他们消耗的资源付费的，如果他们提供的服务器比需要的大，他们可能要支付比他们需要的更多的费用。此外，与传统的内部数据库一样，在云中管理数据库是一项成本高昂的工作。根据您的需要，您可能仍然需要雇用一名经验丰富的数据库管理员，或者花费大量的时间和金钱来培训您现有的员工，以便有效地管理您的数据库。

对于较小的组织和独立开发人员来说，这些问题中有许多是复杂的。虽然大型企业通常有能力雇用对数据库有深入了解的员工，但较小的团队通常可用的资源较少，留给他们的只有现有的机构知识。这使得复制、迁移和备份等任务变得更加困难和耗时，因为它们可能需要大量的在职学习以及反复试验。

托管数据库有助于解决这些棘手问题，为企业和开发人员带来诸多好处。让我们看看这些好处，以及它们如何影响开发团队。

## [](#benefits-of-managed-databases)托管数据库的优势

托管数据库服务有助于减少许多与供应和管理数据库相关的难题。首先，开发人员在托管数据库服务的基础上构建应用程序，以大大加快数据库服务器的供应过程。对于自我管理的解决方案，您必须获得一台服务器(本地或云中)，从客户端或终端连接到该服务器，对其进行配置和保护，然后安装和设置数据库管理软件，然后才能开始存储数据。有了托管数据库，您只需决定数据库服务器的初始大小，配置任何额外的特定于提供商的选项，您将有一个新的数据库准备好与您的应用程序或网站集成。这通常可以通过提供商的用户界面在几分钟内完成。

托管数据库的另一个吸引力是自动化。自我管理的数据库会消耗组织的大量资源，因为其员工必须手动执行每项管理任务，从扩展到执行更新、运行迁移和创建备份。然而，使用托管数据库，这些和其他任务可以自动或按需完成，这显著降低了人为错误的风险。

这与托管数据库服务有助于简化数据库扩展过程这一事实有关。扩展自我管理的数据库可能非常耗费时间和资源。无论您选择分片、复制、负载平衡还是其他方式作为您的扩展策略，如果您自己管理基础架构，那么您有责任确保在此过程中没有数据丢失，并且应用程序将继续正常工作。但是，如果您将应用程序与托管数据库服务集成在一起，您可以按需扩展数据库集群。您不必事先计算出最佳的服务器大小或 CPU 使用率，而是可以快速地动态配置更多的资源。这有助于你避免使用不必要的资源，也就是说你不会为你不需要的东西买单。

托管解决方案往往具有内置的高可用性。在云计算的环境中，如果一个服务是稳定的，并且可能长时间无故障运行，那么这个服务就被称为*高度可用*。大多数著名的云提供商的产品都有一个*服务水平协议(SLA)* ，这是提供商和其客户之间的承诺，保证他们服务的可用性和可靠性。典型的 SLA 将指定客户预期的停机时间，并且许多 SLA 还定义了如果未达到这些服务级别时对客户的补偿。这为客户提供了保证，他们的数据库不会崩溃，如果崩溃了，他们至少可以从提供商那里得到某种补偿。

一般来说，托管数据库简化了与供应和维护数据库相关的任务。根据提供商的不同，您或您的团队可能仍然需要一定程度的数据库使用经验，以便在构建和扩展应用程序时提供数据库并与之交互。不过，最终，管理托管数据库所需的特定于数据库的经验将比自我管理解决方案少得多。

当然，托管数据库不能解决所有问题，对某些人来说可能不是理想的选择。接下来，我们将讨论在配置托管数据库之前应该考虑的一些潜在缺点。

## [](#practical-considerations)实际考虑

托管数据库服务可以减轻部署和维护数据库的压力，但是在使用托管数据库服务之前，仍然需要记住一些事情。回想一下，托管数据库的一个主要优点是，它们抽象掉了数据库管理中大多数比较繁琐的方面。为此，托管数据库提供商的目标是交付一个满足最常见用例的基本数据库。因此，他们的数据库产品不会提供大量的定制选项或更专业的数据库软件中包含的独特功能。因此，您将没有太多的自由来定制您的数据库，并且您将受限于云提供商所能提供的。

托管数据库几乎总是比自我管理的数据库更昂贵。这是有意义的，因为您是在为云提供商付费来支持您管理数据库，但是对于资源有限的团队来说，这可能是一个值得关注的问题。此外，托管数据库的定价通常基于数据库使用多少存储和 RAM、它处理多少读取以及用户创建多少数据库备份。同样，任何使用托管数据库服务来处理大量数据或流量的应用程序都将比使用自我管理的云数据库更加昂贵。

人们还应该思考转换到托管数据库将对其内部工作流产生的影响，以及他们是否能够适应这些变化。每个提供商都不一样，根据他们的 SLA，他们可能只负责一些管理任务，这对于寻求全面服务解决方案的开发人员来说是个问题。另一方面，一些提供商可能会有非常严格的 SLA，或者让客户完全依赖相关提供商，这种情况称为[供应商锁定](https://en.wikipedia.org/wiki/Vendor_lock-in)。

最后，也许也是最重要的，人们应该仔细考虑他们正在考虑使用的任何托管数据库服务是否能满足他们的安全需求。所有数据库，包括本地数据库，都容易受到某些安全威胁，如 SQL 注入攻击或数据泄露。然而，对于云中托管的数据库来说，安全动态是非常不同的。托管数据库用户不能控制其数据的物理位置或谁可以访问数据，也不能确保符合特定的安全标准。如果您的客户有更高的安全需求，这可能会特别成问题。

举例来说，假设您受雇于一家银行，开发一个应用程序，客户可以在其中访问财务记录并进行支付。银行可能会规定应用程序必须具有[静态数据加密](https://en.wikipedia.org/wiki/Data_at_rest#Encryption)和适当范围的用户权限，并且必须符合某些监管标准，如 [PCI DSS](https://en.wikipedia.org/wiki/Payment_Card_Industry_Data_Security_Standard) 。并非所有托管数据库提供商都遵守相同的监管标准或维护相同的安全实践，他们也不太可能只为他们的一个客户采用新的标准或实践。出于这个原因，确保您为这样的应用程序所依赖的任何托管数据库提供商能够满足您的安全需求以及您的客户的需求是至关重要的。

## [](#conclusion)结论

托管数据库有许多吸引各种企业和开发人员的特性，但是托管数据库可能无法解决所有问题或满足所有人的需求。有些人可能会发现，托管数据库有限的功能集和配置选项、增加的成本和降低的灵活性超过了它的任何潜在优势。然而，易用性、可扩展性、自动化备份和升级以及高可用性等引人注目的优势已经导致托管数据库解决方案在各行各业得到越来越多的采用。
