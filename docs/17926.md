# 什么是接口？

> 原文:[https://dev.to/flippedcoding/what-is-an-interface-3ggh](https://dev.to/flippedcoding/what-is-an-interface-3ggh)

如果你打算做任何后端开发，了解接口是很重要的。起初它们看起来有点奇怪，但是一旦你明白它们是什么，你就会变得比以前更强大。

你会在 C#中看到很多接口，因为它不允许你继承多个类。这就是接口发挥作用的地方。它们允许您在不同的类中实现相同的方法和变量。接口有点像你的类*拥有*的东西的列表。

当你在你的类上使用一个接口时，你必须在你的类中编写方法的实现。因此，您可能想知道，如果您仍然必须为每件事情编写单独的实现，为什么还要为接口费心。

这实际上是因为在 C#中不能继承多个类。如果你只需要继承一个类，你可以使用类似抽象类的东西。接口之所以如此强大，是因为在同一个类中可以有多个接口。这样，当你的代码被编译时，它就知道在运行时会发生什么。

让我试着用一个接口的例子。假设你有一个厨师班。它应该有一个名为 Cook 的方法，可以用来烹饪任何一类实现了 ICookable 接口的食物。Chef 类不应该对每一种具有相同基本操作的类都有一个 CookItalian、CookThai 或 Cook(X)方法。

大多数人对接口感到困惑的另一件事是它们是否应该由多个类来实现。你们中的一些人可能会说这不是真的，但是请听我说完。有一股奇怪的力量推动你尽可能地去分离每一点代码，我们通过使用接口来做到这一点。

如果你仔细阅读一些代码，我敢打赌你会发现很多类只实现了一个接口，而且没有一个接口是相同的。你想过这是为什么吗？如果只有一个类使用一组方法，为什么还需要一个单独的接口来实现呢？

用于测试和依赖注入！这就是为什么我们使用界面的方式。这不一定是使用它们的“正确”方式，但它能完成工作。接口确实使测试变得容易，因为你可以模拟东西，而不需要去除大量的类依赖。至于依赖注入，它用于控制应该使用哪个接口实现。

接口有点晦涩难懂，直到你使用了一段时间，即使这样，它们仍然很奇怪。我只是希望这有助于你对它们是什么以及我们为什么使用它们有一点点的了解。
