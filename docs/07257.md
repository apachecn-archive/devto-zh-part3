# 任务冒名顶替:我如何应对我的冒名顶替综合症

> 原文:[https://dev . to/am lane/mission-imposter ble-how-I-m-coping-with-my-impostor-syndrome-2li 2](https://dev.to/amlane/mission-imposterble-how-i-m-coping-with-my-impostor-syndrome-2li2)

在我的第一篇讲述我在兰姆达学校第一周的博客文章中，我没有提到的是我是如何与[冒名顶替综合症](https://en.wikipedia.org/wiki/Impostor_syndrome)作斗争的。即使我在吸收资料和完成我的指定项目，被这么多有才华的人包围着，他们学习代码的时间要长得多，这让我很快失去了理智。我想和你们分享是什么帮助我超越了自我怀疑。

你的使命，如果你选择接受它，尝试这些简单的想法，并与他人分享对你有用的东西:

## [](#pair-program)配对程序

结对编程不仅仅是一个编程学习小组(尽管那些也很有趣！).结对编程是指两个开发人员一起开发一个项目。第一个程序员是“驱动者”，解释他们希望看到发生什么。第二个程序员将打出驱动程序想要发生的代码。

闯入结对编程的一个好方法是让另一个开发人员查看您正在进行的项目，并让他们告诉您他们希望看到什么变化。下面分别是我的项目的前和后:

[![before and after photo of a survey form project](../Images/aabeb6acbaf4b8bd34f7a96c1829a032.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--m1lXtGDJ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/i95vo9lvwsdktjwliocj.PNG)

这个练习的目的不仅仅是让我的项目“看起来更好”通过让另一个学生告诉我做一个我从未想过要尝试的改变，它帮助测试和增长了我的 CSS 属性知识库。

## [](#put-in-the-reps)放入 reps

`if (my code knowledge < I want it to be) { return `Study more!`}；`

当我对某事感到不安全时，我会提醒自己，这是来自我潜意识的信息，告诉我它担心我们的生存。当我对自己的身体不自信时，我开始遵循健康的锻炼/饮食习惯。用同样的逻辑，如果我对我所拥有的知识感到不安全，如果我想改善我生活的这一部分，我应该练习良好的学习习惯和有一个良好的日常生活。

承认你的弱点并努力克服它们是个人发展的缩影。

## [](#accept-that-time-travel-doesnt-exist)接受时间旅行不存在

我因为没有在学习代码的道路上走得更远而自责不已。有好几次我差点去学校学计算机科学，但都没有成功。我意识到，直到现在我还没有准备好，无论是精神上还是身体上，到现在为止我所做的一切实际上都是为了让我充分利用这段经历。

## 每天至少做一次改善心情的事情

我明白了我的心态才是真正的罪魁祸首。我必须意识到，我 100%地控制自己对冒名顶替综合症的反应，以便真正消灭它。这是你必须自己通过尝试&错误来发现对你有效的东西。

在 [Twitter](https://twitter.com/MandaLane_) 上与我联系，和我一起参加#100DaysOfMotivation 挑战(或者你想承诺的任何天数)。让我们互相扶持吧！

## [](#understand-that-everyone-struggles-with-it)明白每个人都在纠结

谁不会纠结于自己不够好或不属于自己的感觉呢？我知道我有。甚至当我现在写这篇文章的时候，我还在纠结于我写得很糟糕的感觉，以及为什么会有人想看我的博客文章？(我正在写我通常的第三稿)...你能做的最好的事情就是表达这些感受。在拉姆达学校(T2)上完我的第一堂 JavaScript 课后，我的大脑真的很痛。我觉得我周围的每个人都在完美地遍历数组，而我却在为基本的语法问题而挣扎。**(仅供参考:后来我发现我的同学也在挣扎，但都装出勇敢的样子。)**
