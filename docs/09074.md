# 什么是伪代码？

> 原文:[https://dev.to/codetips/what-is-pseudocode-4p09](https://dev.to/codetips/what-is-pseudocode-4p09)

> 伪代码是代码的表示。

[![What is pseudocode?](../Images/cddda56debf05492df5e1aa09c1914da.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--PSFJEL0L--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://images.unsplash.com/photo-1516894039413-0ee3187dbd9d%3Fixlib%3Drb-1.2.1%26q%3D80%26fm%3Djpg%26crop%3Dentropy%26cs%3Dtinysrgb%26w%3D1080%26fit%3Dmax%26ixid%3DeyJhcHBfaWQiOjExNzczfQ)

当你试图解释一个编程概念，或者描述你计划如何在你的程序中实现某些功能时，使用伪代码，而没有使它在语法上正确的约束。

在这个网站上几乎任何以前的文章，你都会找到伪代码。在[什么是循环](https://codetips.co.uk/beginner/what-are-loops/)中，我们引入了列表和循环的概念:

```
-- list --

recipients = James, John, Jimmy, Jane, Jake, Jemma, Joanne, Juliet, Jessica, Josh

-- Loop --

for $name in recipients {
    Dear $name

    .... body of letter ....
} 
```

据我所知，这两者在任何编程语言中都是无效的，我的意思是，如果你试图使用它们，就像上面写的那样，你会得到错误，程序不会运行。

它们被添加到文章中，因为这使得解释过程变得更加容易。他们说一张图胜过千言万语，我认为代码示例也是如此。想象一下，在没有伪代码的情况下阅读前面所有的例子，你能理解它们吗？

你现在可能想知道为什么要使用伪代码。如果代码示例使这个过程变得如此简单，为什么不直接编写语法正确的可执行代码呢？外面有很多编程语言。我们可以用一种编程语言来解释这些概念，但是它只适用于这一种语言。我们可以用所有的编程语言来解释这些概念，但是这些文章会变得很长而且重复。

因此，我们使用伪代码向您展示这些概念，以便您可以专注于这些概念，而不是某个特定的编程语言实现。
