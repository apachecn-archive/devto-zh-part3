# 现在就认真对待数据治理，否则未来我们所有人都会为此付出代价。

> 原文：<https://dev.to/alexantra/get-serious-about-data-governance-now-or-it-will-cost-all-of-us-in-the-future-4jdb>

这些运营公司声称他们认真对待，但实际上并不重视。

说他们这样做看起来很好，但实际上，他们半推半就的后果并不可怕，而且/或者不会被那些可以追究他们责任的人发现。

这就是为什么他们会倒在路边。

直到有事情发生，颠覆了这个等式，一个公司才会真正投入正确的时间、精力和文化来做好这件事。

过去十年的一个很好的例子是信息安全。曾几何时，这是公司半吊子的事情。他们确保他们有几个防火墙，并继续他们的一天。接着，索尼遭到黑客攻击，几百万张信用卡的详细信息被窃取，突然间，信息安全得到了应有的重视。

他们的一些概念看起来将永远处于这种不稳定状态。例如，信息管理很重要，就像信息安全一样，但它的后果还不够可怕。让黑客进入你的网络，他们不仅会造成巨大的损失，而且法律追索权以及客户信任的丧失会让一家公司破产。未能恰当地实施信息管理，说来也怪，员工士气高涨。(我爱信息管理 btw -在这里阅读[我的文章](https://dev.to/ronsoak/master-information-management-by-understanding-this-core-principle-31g5)。)

目前，数据治理属于这一类。每个人都声称他们做了，但很明显，没有人做得好。这是口头承诺。

看看每周不断出的新闻故事就知道了。

*   Snapchat 员工使用内部工具监视他们想要监视的任何人。
*   脸书允许新合伙人获取客户的私人信息。
*   一个 Instagram 插件在 AWS 上托管了著名 Instagram 用户的个人信息，没有密码或加密。
*   互联网提供商在未经你同意的情况下出售你的数据。

很明显，应该站在良好数据治理前沿的公司并没有正确地做到这一点，这让我对我们其他人产生了怀疑。

不仅仅是糟糕的数据安全性，公司的行为要么完全不道德，要么选择生活在灰色地带，从不询问他们是否应该，只是测试他们是否可以。

现在，公司以合乎道德的方式处理数据比以往任何时候都更加重要。立法还没有赶上我们在 2000 年初发明的技术，更不用说最近几年了。GDPR 是一个伟大的开始，就像 CASL 一样，但这只是大计划中的一小步，本不应该发生。我们应该自我调节，而不是傲慢的管理机构去干预。

随着数据科学被推到前沿，开发人工智能的压力不仅仅是一船的逻辑规则，我们需要公司认真对待他们正在用自己的数据做什么。不只是对所有人撒谎，希望什么都不会泄露。

这意味着数据治理得到了正确的实施，因此可以认真地实践它。

是的，他们更多的是数据治理，而不是安全和道德使用，但如果处理不当，他们会带来最严重的后果。如果在 AWS 上以纯文本形式存储而没有密码，那么经过清理的数据集有什么用处呢？嘿，你的数据目录是准确的，这很好，但你的用户将如何处理他们现在可以访问的数据，比以前更好？

这些公司没有意识到的事情是，如果没有正确地进行数据治理，您就违背了您已经实施的数据治理流程。糟糕的数据治理流程会成为您的员工忍受、忍受并经常投入时间解决的事情。一个糟糕的实现增加了风险和开销，所以你为什么要在第一时间烦恼。

我满怀信心地说，在未来十年，数据治理将受到重视，但要实现这一目标需要什么呢？谁会拉屎拉得那么惨，让我们都注意到？

因为这就是我们现在生活的现实。目前，你们都在口头上支持数据治理。客户不知道你在用他们的数据做什么，也无法发现，所以你认为你是安全的。说你做了而不去实践更容易，所以你和你的整个公司就是这么想的，灌输一种变通的文化，挑选和选择要遵循的规则，更不用说合乎道德的行为现在并不能为你赚钱。你们中有多少人通过了 ISO 27001 标准考试？真的吗？

难道你看不到这个泡沫即将破裂吗？有人会把事情搞得如此糟糕，以至于政府和公众会有所反应，而你将不得不争先恐后地把事情做好。就像公司不得不争夺 GDPR 一样。这是一个迹象，表明我们的行业已经失败，当立法必须制定在我们身上。因为法律不是来自经验，而是来自一群六个月前对我们的领域一无所知的人，他们被迫做出影响所有科技公司的决定，无论是现在还是未来。他们的立法没有放之四海而皆准的东西，它会扼杀未来的创新和机会，因为它必须适用于一群缺乏行业经验的人设定的一系列要求。

但与 GDPR 不同，你不能坐等它强加于你，你需要现在就接受它。

许多公司对 GDPR 的实现是混乱的，在最后一分钟才实现，而系统中从来没有这个需求。作为事后的想法加上去的。这导致了更差的整体体验，降低了效率，我不太敢说这是一种好的文化。同样的事情也会发生在数据治理上。它需要适当地实施，有足够的准备时间、文化和资源。如果它到了立法强迫你正确做的地步，你会再次半途而废，像事后思考一样把它钉在你当前的技术上，然后你会马上回到你现在的位置。一个让你的公司慢下来的系统，没有员工的支持，并且会增加出错的风险。

现在就正确地进行数据治理，因为这不是一件可以仓促完成的事情。适当的数据治理可降低风险并提高效率，半托管则适得其反。如果你等到它立法到你身上，那么你将别无选择，只能半途而废，减慢你的公司，增加一些不好的事情发生的风险，除非到那时，他们将立法来惩罚你。
