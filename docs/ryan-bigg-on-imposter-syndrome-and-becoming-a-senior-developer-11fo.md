# Ryan Bigg 谈冒名顶替综合症和成为高级开发人员

> 原文：<https://dev.to/jacobherrington/ryan-bigg-on-imposter-syndrome-and-becoming-a-senior-developer-11fo>

最近，我有机会与我的个人英雄之一瑞安·比格(Ryan Bigg)交谈，他也被称为雷达(radar)。

Ryan 是一个多产的软件工程师，他对 Rails 做出了一两个贡献，基本上是 T2 构建了我今天维护的开源项目。

我们深入讨论了 Ryan 与冒名顶替综合征的个人经历，他甚至在今天仍能感受到它的影响。这句话出自一个有着十多年经验的开发人员之口，让我印象深刻:

> 我有[冒名顶替综合症]，也许不是每天都有，而是每周都有？当然可以。

Ryan 的诚实令人耳目一新，因为它有助于澄清一种误解，即资深工程师已经解决了整个冒名顶替综合症的问题。事实上，几乎每个人都会在某个时候怀疑自己的能力或价值。这并不意味着你是个糟糕的工程师；瑞安就是活生生的例子。

我们花了相当多时间的另一件事是学习在团队中工作。Ryan 现在的工作是为 Culture Amp 的一群初级开发人员提供领导和指导(这听起来是一份非常酷的工作)，但曾经有一段时间他不重视团队活力。

瑞安承认，即使在今天，他也很难做出被动反应。这很讽刺，因为他作为导师和老师很受尊敬。这正好说明，意识到你的挣扎有助于成功。
