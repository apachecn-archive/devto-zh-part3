门户网站很疯狂，因为它们让你很难跟踪哪个网站在为你提供内容。即使 twitter 视频中突出显示了当前的 URL，也很难准确理解所有内容的来源。

起初我发现这令人担忧，但随着时间的推移，我开始相信门户网站正确地认识到，在一般情况下，通过 URL 组织网络浏览对用户没有用处。(当然，URL 非常适合分享页面链接。但是当点击时，它们远没有我们想象的那么重要。)

最近我想了很多关于分组想法的正交方法，以及为什么明显的分组并不总是最有用的。在前端世界中，我们共同意识到按语言分组不如按组件分组明智:

对我来说，门户似乎是同一类型的(有争议的)转变。我们选择根据内容所属的领域对内容进行分组，而不是根据内容所属的领域。这意味着将不再需要链接，即互联网的原始“字符串”,来将不同的位置联系在一起。取而代之的是，相似内容的片段将被自然地分组，而不管其来源。

我认为，如果没有某种大规模的起义，这种变化不会发生。我们在 JSX 看到了它，我相信我们会再次看到它。但是我还想说，门户网站将迫使网站所有者和用户更少地关心内容托管在哪里，而更多地关心内容如何连接，从而极大地改善互联网体验。
