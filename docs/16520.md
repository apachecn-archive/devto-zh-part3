# 最后，我需要的一体化工具:VSC！

> 原文:[https://dev . to/Joe hobot/finally-all-in-one-tool-for-my-needs-VSC-5ga 8](https://dev.to/joehobot/finally-all-in-one-tool-for-my-needs-vsc-5ga8)

我是一个终端迷，我每天做的大部分工作都是在终端(zsh)完成的..运行脚本、编码、编辑、配置等..

在糟糕的日子里，我最终会打开 Sublime Text 中的文件，在那里编辑它，然后回到我的终端，继续我的工作。

几天前，我看到了一个关于 VSC(Visual Studio 代码)的帖子，我听说过它，但我从未真正研究过它，因为我在 Mac 上运行微软的产品听起来很奇怪。

好吧.....

现在，我在 VSC 跑了 48 小时，我必须说，我爱上了它，因为我真的有一个一体化的“窗口”。

Terraform、Terminal(zsh)、GitHub、Docker、Ansible 等..它支持所有这一切，我得到了所有安装没有任何问题。

所以昨天我花了一整天的时间在终端上运行一些 kubernetes 测试，看着一个只有几个标签的屏幕。

在上面的 img 中，从我的一个屏幕中，您可以看到我的 tf 模块，在下面的终端中，您可以看到我的运行和一些 aws 命令，以及 github 分支、kubernetes 名称空间和 kubernetes 集群名称。

所以，如果你在 Mac/PC 上做一些和我类似的事情，给 VSC 一个机会。
