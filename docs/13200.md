# 经理的道路回顾

> 原文：<https://dev.to/horia141/the-managers-path-review-3fc>

又是一个月，又是一篇书评[1]。这次是卡米尔·福涅尔的《经理之路》。这是一本很好的读物，我觉得我从中受益匪浅。它似乎经常出现在初次接触管理人员的阅读列表中，最近也在工作中出现。主要是因为现在有很多我们这样的人。

这本书旨在成为软件工程团队的管理手册。它有一个简单的结构:几个章节，每个章节涵盖一个典型的工程管理阶梯的一个层次。因此，从个人贡献者开始，然后是首席工程师、经理、经理的经理/主管，最后是执行角色:副总裁和首席技术官。

每个层次都有一个共同的结构——典型的期望、要避免的陷阱、案例研究等等。大部分的东西都是从我(超级有限的)经历中产生的共鸣。有几件事也许值得重复一下。

第一，这或多或少是软件工程的转行。不是晋升，不是“修正者”(除了在一些公司的初级水平)。照这样对待它，看它是不是你想做的事情。如果你只把它视为你工作的公司唯一的自然发展，它就不会达到你的期望。你最好去一家有稳定的个人贡献者职业轨迹的公司

第二，考虑到第一点，你仍然是*工程*经理。所以你需要知道足够多的信息，如果工作的技术部分有所贡献的话。在较低层次，通过代码审查、架构建议等，在较高层次，通过技术选择、战略决策等。

无论如何，这本书里有大量的额外信息，从如何管理你的职业生涯，到如何与其他部门互动。非常值得你花时间。
