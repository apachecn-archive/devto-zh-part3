# 星期五前端:高级 JavaScript 模式版

> 原文：<https://dev.to/kball/friday-frontend-advanced-javascript-patterns-edition-2ab0>

希望你这周过得愉快，如果你在美国，祝你周末假期愉快！有一组文章对一些高级 JavaScript 特性和模式进行了更深入的探讨。代理、生成器和功能模式。还有一些精彩的 CSS 文章，一份关于 2019 年互联网健康状况的精彩报告，等等。尽情享受吧！

最佳，

来自 ZenDev 的 KBall

#### CSS&SCS

##### [调试 CSS 网格:第二部分:什么 Fr(动作)？](https://css-irl.info/debugging-css-grid-part-2-what-the-fraction/)

上周我们在隐性轨道上包括了第一部分；这是在 CSS 网格中使用新的`fr`单元的一个很好的演示。我不得不承认，0.5fr 的行为不是我所期望的，但在读完这篇文章后，它变得非常有意义。

##### [这个“内幕”问题](https://css-tricks.com/the-inside-problem/)

Chris Coyier 深入探讨了 web 开发中最普遍的设计情况之一，即填充内容的全出血背景。他着眼于常见的解决方案，缺点，并暗示一些真正有创意的解决方案，如果你发现自己陷入困境。

##### [中文窗棂和 CSS](https://yuanchuan.dev/2019/05/15/window-lattice-and-css.html)

不实用，使用非标准属性，对专业开发没有什么用处。但我还是忍不住看着它。一个利用网络作为媒介创作艺术的惊人例子。反射和嵌套创建展开模式的简单能力。太美了。

##### [让你生活更轻松的 CSS 规则](https://medium.freecodecamp.org/css-rules-to-live-by-962a051e1eb2)

一组可靠的建议将帮助你保持你的 CSS 有序和良好的状态，防止你做一些你以后会后悔的事情。有些与可访问性有关，有些与一致性有关，有些只是与简单的可维护性有关，但至少考虑一下这些都是好的。

##### [流体网页排版的状态](https://betterwebtype.com/articles/2019/05/14/the-state-of-fluid-web-typography/)

深入研究几种不同的响应式排版方法和所谓的“流动”(不断变化的尺寸)排版方法。对不同方法的利弊进行良好的分析和评估。

##### [CSS 能做到吗？](https://dev.to/ananyaneogi/css-can-do-that-18g7)

本周的 CSS 额外文章——一个很棒的快速点击列表，列出了你可以用 CSS 做的很酷的事情，每个都有一个 codepen 示例。一个伟大的快速扫描，如果你看到新的东西，你可以去挖掘更深。

#### JavaScript

##### [Javascript 代理实用指南](https://dev.to/tombarr/a-practical-guide-to-es6-proxy-fln-temp-slug-8739819)

代理功能非常强大，但不常用。我认为其中一个潜在的原因是很难让你明白到底该用它们做什么。我特别喜欢这篇文章的一点是，它使用了一种“实例教学”的机制，不仅揭示了如何使用代理，还展示了无数强大的用例，这些用例将激发你的创造力。

##### [用 JavaScript 优雅地处理错误](https://jrsinclair.com/articles/2019/elegant-error-handling-with-the-js-either-monad/)

如果你不是一个函数式编程大师(我不是)，这将是一个完全的精神弯曲。但是它也做了非常非常好的工作，为错误处理的功能/纯方法建立了解释和案例。有一些很酷的火车图片帮助我理解。:)

##### [函数式 JavaScript](https://medium.com/@bluepnume/functional-ish-javascript-205c05d0ed08)

与上一篇文章相比，我更倾向于编写这种类型的 JavaScript。JavaScript 融入了一些函数概念，使得管理、理解和维护变得更加容易，但并不是一直都是函数。这篇文章提出了一个令人信服的理由。

##### [JavaScript 生成器基础知识](https://dev.to/treyhuffine/the-basics-of-javascript-generators-1n8-temp-slug-872349)

生成器是我非常想研究和使用的语言特性之一，因为它们看起来非常强大和有用，但我还没有。但是我有几次差点就成功了…第一次是在 JSConf Hawaii 的一次演讲(遗憾的是视频还没有发布),第二次是这篇博客。看看这个。

##### [用 React 多选复选框](https://tj.ie/multi-select-checkboxes-with-react/)

好的，它使用类和生命周期方法来代替钩子，但是这是一个很好的逐步构建中等复杂的 React 组件的例子。深入其中的思考，以及如何一次构建一个方法。

#### 其他牛气

##### [很复杂:Mozilla 2019 年互联网健康报告](https://internethealthreport.org/2019/)

一组有趣的研究和故事探索了 2019 年互联网的健康状况，涉及安全、开放和中心化等问题。我计划花很多时间挖掘这个。非常非常酷。

##### [介绍 GitHub 包注册表](https://github.blog/2019-05-10-introducing-github-package-registry/)

如果你错过了，上周 github 引入了一个捆绑到 GitHub 的新的包注册表。除了作为对 NPM 的警告之外，这里还有一些真正有趣的潜力——在不需要另一个服务提供商的情况下在公司建立内部注册表的能力，以及 GitHub 能够进行端到端分析并验证您正在查看的代码是否是您的包中的代码的可能性。非常非常有趣，值得密切关注。

##### [深入了解图像和帧的本地惰性加载](https://css-tricks.com/a-deep-dive-into-native-lazy-loading-for-images-and-frames/)

有趣的是，新的(仍在浏览器中实现的)本地延迟加载功能。一些非常有趣的花絮——我不知道浏览器会试图在懒惰加载的图像中嗅探维度，以避免重新布局。整洁！

##### [2019 年 GraphQL 趋势](https://nilan.netlify.com/posts/graphql-trends-in-2019/)

GraphQL 在 2019 年变得疯狂，这篇博客文章是对正在发生的许多事情的一个很好的总结。
