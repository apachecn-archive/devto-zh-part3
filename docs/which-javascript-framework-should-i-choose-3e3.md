# 应该选择哪个 JavaScript 框架？

> 原文：<https://dev.to/niharikapujari/which-javascript-framework-should-i-choose-3e3>

简介:

当新的 Javascript 框架以极快的速度发展时，设计前端基础设施并不容易。随着新框架进入市场，以及现有框架的趋势性新更新，让我们来分析一下现有流行 JS 框架的一些优点和局限性。

# 棱角分明

Angular 是一个已知的基于类型脚本的 JavaScript MVVM 框架，成立于 2009 年，由 Google 支持，用于开发高度交互式的 web 应用程序。它被谷歌、福布斯、WhatsApp 等众多世界 500 强企业广泛使用。从一开始，AngularJS 就在工程师中非常受欢迎。一个强大的社区提供了足够的培训材料、讨论和第三方工具来开始使用 AngularJS，并为几乎所有出现的问题找到解决方案，这种趋势甚至在 Angular 2+的今天仍在继续。

## 好处:

*   基于组件的体系结构提供了更高质量的代码:组件可以被看作是接口中相互独立的小部分。

*   可重用性:开发人员可以在应用程序的不同部分重用组件。这在使用类似元素(如搜索框、日期选择器、排序列表等)的企业级应用程序中特别有用。

*   可读性:封装还确保了新开发人员——他们最近加入了一个项目——可以更好地阅读代码，并最终更快地达到他们的生产力平台。

*   单元测试友好。组件的独立性简化了单元测试，即旨在验证应用程序最小部分(单元)性能的质量保证程序。

*   可维护性:容易相互解耦的组件可以很容易地被更好的实现所替代。

*   RxJS: RxJS 是一个通常与 Angular 一起使用来处理异步数据调用的库。它允许独立地并行处理事件并继续执行，而不需要等待某个事件发生，也不会让网页无响应。RxJS 有一个陡峭的学习曲线，但是一旦你掌握了它，它会让生活变得更容易。该库使用 Observables 操作，这是一种描述数据流如何组合以及应用程序如何对这些流中的变量做出反应的蓝图。基本上，一旦你掌握了可观测量，你就可以像乐高积木一样轻松地重用、管理和组合它们，从而降低诸如构建拖放功能、处理大量数据等编程操作的复杂性。您可以将 RxJS 与其他支持前端(React)的框架结合使用。JS)或后端操作(Node。JS)，但是在 Angular 中 RxJS 是必需的，我们也将在缺点部分讨论。

## 局限性:

*   将遗留系统从 AngularJS 迁移到 Angular 需要时间:AngularJS 和 Angular 之间的差异很大，从过去迁移到未来的路径也很大。不像从 Angular 5 升级到 Angular 6，这不会是一件轻而易举的事，尤其是如果你正在处理一个遗留的怪物。有不同的方法可以做到这一点，其中之一就是使用混合方法。它需要在增量更新整个产品的同时让新旧 Angular 同时运行。这不仅需要时间，你还必须复习许多工具，过渡到一种新的语言，并处理更重的应用程序。

*   陡峭的学习曲线
    与类似的 React 或 Vue 入门相比，熟悉 JavaScript 的新开发人员学习和使用新 Angular 将面临挑战。涵盖的主题和方面很多:模块、依赖注入、组件、服务、模板等等。另一个障碍是 RxJS，一个用于异步编程的反应式编程库。学习它，至少在基础水平上，对于使用 Angular 是强制性的。工程师们抱怨错误消息过于隐晦，如果没有额外的研究和反复试验的操作，很难理解。
    TypeScript，正如我们在优势部分已经提到的，也是争论的焦点。虽然 TypeScript 提高了代码的可维护性，但学习它并不能使曲线变得更平缓。

# 做出反应

React 由脸书维护，是一个 JavaScript 库，用于为 web 应用程序构建 UI 组件。它被脸书广泛使用(咄！)，优步，网飞，乌代米和梅莫尔。

## 好处:

*   更新流程得到优化和加速。
*   JSX 使组件/块代码可读。它显示组件是如何插入或组合的。
*   React 的数据绑定为创建动态应用程序创造了条件。
*   提示渲染。使用 comprises 方法来最小化 DOM 操作的数量有助于优化更新过程并加速它。
*   可测试。React 的本地工具用于测试、调试代码。
*   搜索引擎优化友好。React 通过服务器端呈现和连接用户端的事件处理程序来呈现首次加载体验:1。在服务器上调用 React.renderComponentToString。2.在客户端调用 React.renderComponent()。3.React 保留服务器端呈现的标记，附加事件处理程序。
*   最新的。脸书团队支持图书馆。脸书社区可以提供建议或代码样本。
*   使用 React+ES6/7，应用变得高科技，适合高负载系统。

## 局限性:

*   学习曲线。作为非全功能框架，将用户界面自由库集成到 MVC 框架中需要深入知识。
*   面向视图是反应堆的缺点之一。应该找到“模型”和“控制器”来解决“视图”问题。
*   不使用同构方法开发应用程序会导致搜索引擎索引问题。
*   许多开发人员不喜欢 JSX React 的文档，手册对新手来说很难理解。
*   React 的大尺寸库。

# 视图

Vue.js 是市场上的新产品，是讨论最多、发展最快的 JavaScript 框架之一。发布于 2014 年 2 月，你可以使用 HTML、CSS 和 Javascript 构建交互式 ui。被阿里巴巴广泛使用。Gitlab，并被全球许多开发人员采用。

## 好处:

*   文档:Vue.js 有着令人难以置信的精心设计的文档，非常全面，写得非常好。初学者编写第一个应用程序所需要的只是一些基本的 Javascript 和 HTML。

*   尺寸:我发现 Vue 的一个最重要的特点是它的尺寸。Vue.js 的生产就绪版本在 gzipping 后仅重 18kb，相比之下，minified jQuery 重约 80kb(取决于您使用的版本), gzipping 后重 29kb。还需要注意的是，Vue.js 生态系统也是小而快。使用 Vue.js，用户可以将模板到虚拟 DOM 编译器甚至运行时间分开。

*   可伸缩性和多功能性:它既可以作为一个库，也可以作为一个成熟的框架。我的下一个项目将涉及一个几乎纯粹的使用 Wordpress REST API 的 Vue.js web 应用程序。因此，它为我提供了我所需要的开发速度，同时为我公司中技术含量较低的内容创建者提供了他们信任的熟悉度和易用性，让他们可以轻松地使用 Wordpress 这样的 CMS。

*   完美主义者的可读性:我发现 Vue.js 非常容易阅读。函数的可访问性令人难以置信，如果你的命名约定是正确的“this function will dot this ”,那么你就不会错得太多。例如，对于表单提交，我喜欢将每个字段验证分解成它自己的功能。当设计师要求我将全名改为名和姓字段时(反之亦然),有一个单独的功能是值得的。

*   比$jQuery 更容易学习:这有点主观，但我发现用 Vue.js 开始构建基础要比 jQuery 等更传统的 Javascript 框架容易得多。它非常平易近人——文档非常出色，虽然有些人发现社区主要是非英语社区，但我发现社区非常支持我。

## 局限性

*   缺少一些通用的可靠插件/组件:由于 Vue.js 相对较老，所以众所周知缺少一些通用插件，使得使用各种工具更加容易。谷歌地图是一个很常见的例子。我还没有在 Vue.js 中使用过谷歌地图，总是求助于普通的 Javascript。

*   快速发展:许多 Vue 学习者提到 Vue.js 发展相当快，所以你在网上找到的很多例子可能已经过时了。我会一直投资于一个可靠的在线课程，一个好的文档阅读和一个可靠的书籍启动。

*   iOS 和 Safari 的小问题:我发现了旧的 iOS 和 Safari 设备的小问题；但是，完全诚实地说，这些问题通常是可以解决的，尽管对一些问题需要更多的挖掘。

*   反应性警告:如果您阅读文档，会发现有一些反应性警告，比如直接从数组中设置一个项(例如 this.items[key]=value)或者添加一个新的数据属性。这可能是好的也可能是坏的，取决于用例。然而，在大多数情况下，最好在数据中预先指定所有内容。

# 结论:

根据上面列出的比较，哪一个是最好的 JavaScript 框架？我认为框架的选择取决于你的偏好、需求和用例。然而，如果你正在开发一个大型的应用程序，并且正在寻找一个非常稳定的平台来支持它，那么你可以选择 Angular，前提是你非常习惯使用 Typescript。另外，如果你正在寻找面向对象的编程或者有一个庞大的开发团队，那么 Angular 是非常值得推荐的。如果你不想使用 Angular，但想开发巨大的应用程序，那么 React 是你的最佳选择。该平台不仅有利于大型团队，而且有一个巨大的生态系统围绕着它。最后，如果你愿意尝试一个非常有前途的框架，这个框架正在稳步增长并且很容易上手，那么 Vue.js 是你最好的选择。该应用程序将有更干净的代码，可以由一个小团队开发。
