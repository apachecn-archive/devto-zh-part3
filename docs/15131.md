## [](#what-is-model-view-controller)什么是模型、视图、控制器？

创建单独的代码段来处理程序的特定目标是很重要的，这被称为**关注点分离**。理解 MVC 和我们的关注点分离的一个很好的例子是 HTML、CSS 和 JavaScript。HTML 将处理内容，即**模型**，CSS 处理样式，即**视图**，JavaScript 处理逻辑，即**控件**。它们都处理程序的特定部分，这是关注点的适当分离。

为了进一步扩展 MVC 各个方面的 web 开发功能，**模型**属于我们应用程序的实体。对我来说，解释实体的最好方法是举个例子。一个单独的人可以被想象成一个模型，通常一个人总会有一个名，一个姓，和一个喜欢的颜色。我们可以把所有这些加在一起，创建一个模型，看起来像这样:

```
{
firstName: 'Peter',
lastName: 'Parker',
favoriteColor: 'red'
} 
```

模型包含与应用程序相关的最重要的信息之一:**要处理的信息**。与此相对应的是控制器，它处理**如何**处理信息。视图只是我们从模型中呈现数据的不同方式。在一个网站的例子中，我们的**控制器**处理用户请求，调用适当的路线和模型，并设置适当的视图。例如，在 Express 中，当用户单击表单提交按钮时，将调用控制器来呈现新页面。

## [](#advantages)优点

使用 MVC 架构的一些优点是代码可重用性、同步开发和易于修改。代码可重用性是任何程序或应用程序都非常需要的特性，甚至有它的首字母缩略词！干，不要重复自己，是编程中的一个关键原则，如果程序员认为他们可能会重复自己甚至两次，强烈建议他们重组代码块，以便它可以重复使用，因为如果应用程序规模更大，它可能会重复使用。同步开发允许多个开发人员在同一个项目上工作，但是它的不同方面允许更有利的结果。这方面的一个例子是，一个开发人员处理网站的 JavaScript，而另一个开发人员处理 HTML 和 CSS，同时保持通信以防止语法和参数错误。

采用 MVC 架构无疑会提高任何程序的效率。它允许代码的可重用性、可伸缩性和关注点的分离。尽管如此，不管代码有多难，它总是容易的一天！
