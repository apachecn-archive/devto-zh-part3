# 网络工程师的 14 个有用工具

> 原文:[https://dev . to/rafaelmonteiro/14-网络工程师有用工具-2d0l](https://dev.to/rafaelmonteiro/14-useful-tools-for-network-engineers-2d0l)

有趣的是，一个网络专业人员知道一些工具，可以提高他的生产力，使他的工作更容易。

虽然一些工具——尤其是 ARPSpoof、nmap、TCPTraceroute 和 AirCrack——可以在恶意环境中使用，例如执行侦察和探测弱点以准备攻击——但它们对于合法目的也有价值。

**AirCrack** -可以揭示谁在使用无线网络，并可用于解决问题。它也是发现附近无线网络的好工具。

**ARP 欺骗** -黑客利用它发送欺骗的 ARP 请求，试图将联网设备的 MAC 和 IP 地址配对。但它也可用于创建中间人活动监控，例如，无需在路由器的 span 端口上安装设备。

**Cacti** -它收集并绘制一段时间内的 SNMP 值，给出设备利用情况的图片。

这个工具基本上在服务器之间来回移动数据，它在测量网站的响应时间方面非常有用。

这是一个搜索服务器，可以与 Logstash 和 Kibana (ELK)配对，收集日志数据并创建仪表板。Elasticsearch 提供搜索功能，Kibana 呈现数据来创建仪表板。

**fprobe** -监听指定的接口并收集有关流量通过的网络流数据是 fprobe 的主要目的。它可用于检测不需要的流量类型，例如企业环境中的视频流服务。

**iperf** -该工具测量吞吐量、数据包丢失和抖动，并支持 UDP 和 TCP 数据包，以确定网络上设备之间的连接质量。它可以将收集到的数据绘制成图表，以了解网络状况如何随时间变化。

**nfdump** -由 fprobe 收集的流信息可以被导出到 nfdump，nfdump 将其存储在文件系统中，它可以读取并使用该文件系统来基于协议显示数据并对顶级用户进行排名。例如，一种应用是发现一天中的拥塞问题。

这是一个强大的网络、设备和服务发现工具，可用于网络扫描和执行安全审计。它可以扫描特定的端口，并确定它们是否打开。它可以按子网扫描设备，并提供有价值的信息，如设备输出的流量类型。除了发现网络上的设备之外，Nmap 还可以扫描活动的服务，并执行指针记录查找和反向 DNS 查找，这可能有助于识别找到的设备类型。

OpenNMS -这个工具监视设备和服务，当它们关闭时发出警报，并可以在设备上写可用性报告。

**冒烟** -它测量延迟和数据包丢失，可以随着时间的推移进行分析，以揭示延迟的变化，从而用于故障排除或网络规划。它通过定期发送 Ping 数据包并记录响应时间来实现这一点。所收集数据的图表上显示的峰值表明响应时间问题何时出现，并有助于缩小对其原因的调查范围。

这是一个入侵检测 ID 工具，可以用来实时监控网络，但也可以用来将规则应用于一组捕获的跟踪文件。它可以与 ElasticSearch 和 LogStash 等日志工具配合使用，收集的数据可以进行分析，并设置规则来寻找特定的条件并发送警报。

这个工具使用 TCP 而不是 ICMP 来追踪网络中的路径。这有助于发现是什么阻塞了传输中的流量，例如配置为阻塞流量需要使用的端口的防火墙。

Wireshark -它捕获并分析数据包，以发现格式错误的帧、无序的数据包等。例如，用户可以编写规则来只捕获无线、TCP 或 http 之类的特定协议，以解决服务器响应时间慢的问题。
