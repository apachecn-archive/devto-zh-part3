# 我在学习软件开发时遇到的两个最大挑战

> 原文:[https://dev . to/wangonya/my-two-great-challenges-while-learning-software-development-25eg](https://dev.to/wangonya/my-two-greatest-challenges-while-learning-software-development-25eg)

> "我们的知识增长得越多，我们的无知就展现得越多."约翰·肯尼迪

我在学校学的是计算机科学。专攻 it 的软件方面。毕业后，我觉得自己懂得不少。我在课堂上做过几个项目，还有一些兼职项目，所以编码对我来说并不陌生。现在我可以靠写软件赚钱了，对吧？不对。

我所不知道的是，软件开发不仅仅是写有用的代码。我需要学习的还有很多。我是说很多。

## [](#oop)OOP

我在学校讨厌 Java 和 C++。你会问为什么？类、对象、方法。我从来不理解他们，也不在乎。为什么要把代码复杂化呢？只需编写一个函数来处理您需要的东西。你还需要别的东西吗？再写一个函数。这可能是因为我的第一门语言是 C，但是用传统的自顶向下的结构化编程方法编写代码对我来说很有用。在 C 语言之后，我坚持使用 Javascript，因为我知道我可以坚持使用函数😅。

长话短说，我没能坚持多久。采用面向对象的方法有很多好处。理论上，我知道这一点，但当我在 bootcamp 项目中用 Python 开发 api 时，它真正打动了我。我意识到我的代码中有太多的重复。文件变得越来越大，事情变得难以控制。我想，一定有更好的办法。班级。方法。遗产。我不得不学习 OOP。

理解这一点并不容易，但自从我理解了这一点，编程就变得前所未有的愉快。我觉得我对代码库有了更好的理解，所有的东西都很好地结合在一起。这让我想到了下一件事:

## [](#tdd)TDD

是的，我在学校学过考试。黑盒测试。白盒测试。所有的理论。但是我从来没有从测试驱动开发的角度考虑过测试。我以为测试是给用户的。就写软件，看看行不行。如果是的话，就发布它，并在以后进行改进。如果有人说行不通，那么…

[![works-on-my-machine](../Images/5a6f8be06933104330d56fc8dabd77a2.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--R0uRWYvW--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ve03cwgb0dwprntk76cz.jpg)

再次，我听说从零开始学习这个。因为对我来说完全是新的，所以相当紧张。我只是不能遵循这个过程:写失败的测试，写代码让测试通过，折射者。我更喜欢我自己的方式:写代码，写测试通过，折射。但是当我发现即使输出是错误的，我的测试还是通过了，我发现这种情况并不太好。这些系统的存在是有原因的。

我认识到，首先编写测试确实给了我手头任务的整体概述，有点像我必须用我编写的代码实现的目标。这有助于我的生产力，因为我必须一次专注于一件事，使我的代码更整洁。

## [](#in-conclusion)总之…

我每天都在学习更多的东西，但我要说我学到的最重要的事情是，如果你从来不去学习新的东西，你将永远不会知道你不知道的东西。如果我从来没有强迫自己从事比我习惯的项目更大的项目，我永远不会明白 OOP 的重要性。如果我不推动自己进入职业发展，我永远不会看到考试的重要性。你可能只是坐着认为你知道事情是如何运作的，但是总有更多，更多你可能不知道的。如果你觉得你有一段时间没有学到新的东西，你可能会变得平庸。从不和解。继续成长。
