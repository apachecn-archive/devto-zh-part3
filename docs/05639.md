# 计算机科学硕士没有教你的东西

> 原文:[https://dev . to/Healey codes/what-they-don-t-teach-you-at-a-a-computer-science-masters-hmn](https://dev.to/healeycodes/what-they-don-t-teach-you-at-a-computer-science-masters-hmn)

我的文学学士学位是创意写作。我申请攻读计算机科学硕士学位，以便更好地掌握理论，填补知识空白，并成为一名更好的软件工程师。这里有一些课程上没有的东西。

### [](#markdown)降价销售

使用 [markdown](https://github.com/adam-p/markdown-here/wiki/Markdown-Cheatsheet) 很简单，可以在五分钟内学会。然而，有效地使用它有点困难。撰写好的技术文档是与计算机科学不同的技能。你是否曾经访问过 GitHub 上的一个知识库，而 README 文件恰好与你合拍？您的问题:已回答。你的眼睛:奖励。

指引访问者的视线并塑造他们对你的文本的体验是一件轻描淡写的事情。我被要求审查投资组合，并在 wazoo 中发现了非常棒的库，其中有教科书式的干净代码和模式。但是所有这些都隐藏在一个很少格式化的冰冷的减价页面后面。几个简短的句子挤在一起，对再点击几下美丽的代码毫无意义。大多数招聘经理都会止步于此。

在学术界学习计算机科学与编程的现实是完全分开的，编程是关于人与人之间的联系。我们为人类而不是机器写代码。没有人想在没有介绍的情况下开始阅读代码。一份精心标记的自述减轻了这种打击，并将帮助你得到那份工作。

### [](#clean-code)清理代码

随着截止日期的临近，整洁的代码得不到奖励，很难学会干净地编写代码。尽管在我的第一份工作中经历了一些评估过程，但我很快就适应了。为教授写代码和为你关心的团队写代码之间有很大的区别——唯一的要求是它能工作并且“可读”。学术界的项目大部分都是建立起来然后扔掉的。我们都在抱怨一些精彩论文背后的代码。(虽然，我的一些同学确实开源了我们项目的[部分)。](https://github.com/healeycodes/websocket-benchmarker)

人们对干净代码有自己的定义。对我来说，我感兴趣的是写一些可以在三到四个月内轻松扩展或修改的东西。代码只在必要的地方加注释，缓存变量来帮助内聚，所有东西都按照样式指南格式化，然后进一步提高可读性。我希望有人在学校告诉我:不要忘记人类。

### [](#code-review)代码审查

虽然有一两次在团队中工作的机会，但是建立一个代码审查过程并不是一件很重要的事情。所有编码——从[发射宇宙飞船](https://coder.today/tech/2017-11-09_nasa-coding-standards-defensive-programming-and-reliability-a-postmortem-static-analysis./)到编写 iOS 应用程序——都将涉及审查他人的代码，并为审查准备自己的工作。这里有两部分，检讨和被检讨。

审查代码是建立心智模型和运用同理心的一种实践。如果你适当地回顾，你不仅关心代码如何逐行阅读，而且关心它如何影响今天和明年这个时候的代码库。你必须带着最大的同理心提出问题并提供反馈。你不是一个被鄙视的戏剧评论家——你正试图帮助你的队友改进他们的代码。要有礼貌，宁可有礼貌。不过，当你们关系融洽的时候，在这些 gif 上涂上厚厚的一层。

为评审准备代码本身就是一种技能。你的评论应该是一个逻辑块。越小越好。描述你的代码正在解决的问题或故事也很重要(我个人在这方面使用 markdown)。它将武装你的伙伴，让他们在你的第一个 bug(抱歉，杰米)尝试的深处投入战斗。这会让他们不用把椅子挪到你身边，也不用再去考虑那个评论了。

### [](#the-edges-of-languages)语言的边缘

在学校，我们应用 Java、JS 和 Python 等语言来解决算法问题。我们构建了数据库、管理系统、导航迷宫，还接触了一些完整的堆栈。我们优化了。我们用核心工具完成了这一切。数据结构，算法，面向对象的编程，以及一些数学知识。然而，我不得不自己学习`setTimeout`、`delete`和`this`如何在产品代码库中交互。没有人告诉我关于装饰者，或者 ES6/7/8 是如何把我们带入未来的。我喜欢带我去我以前没去过的地方的虫子——当我打开 StackOverflow 的脊柱时，那些深奥和未知的相互作用。对于 JavaScript，我可以推荐[你不知道的 JS](https://github.com/getify/You-Dont-Know-JS) 和[雄辩的 JavaScript](https://eloquentjavascript.net/) (都是免费的)。对于 Python: [所以你想成为 Python 专家？](https://www.youtube.com/watch?v=cKPlPJyQrt4)。

### [](#)~

尽管我很努力，但通过尝试在网络上变得更有创造力，进入 T2 的游戏堵塞，以及破解奇怪的项目，我学到了更多。我很认为自己是自学成才的，除非我足够幸运地在简历上写上*理学硕士*这几个字母。我过去认为最重要的技能已经消失了。我曾经认为，成为一名程序员只需要灵光一现，在你的大脑中收集一个算法库。

当我听说有新人加入我们的团队时，我不会担心他们在编写代码方面有多优秀。我想知道他们会在评论中听我的评论(并告诉我我错在哪里！).我希望能够与他们就技术问题进行交流。我想向他们学习，但不仅仅是代码相关的东西。最重要的是，我希望和他们一起工作很有趣。
