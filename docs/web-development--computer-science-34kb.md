# Web 开发！==计算机科学

> 原文：<https://dev.to/flippedcoding/web-development--computer-science-34kb>

让我们坚强地开始。Web 开发不是计算机科学。他们不一样。有些人可能会说，web 开发是计算机科学的一种应用，这是比较准确的说法，但仍然不完全正确。这两者之间有一些主要的区别需要讨论。

## 计算机科学更多的是基于数学和硬件。

如果你以前上过计算机科学课，你就会知道实际上涉及了多少数学。你必须确保你设计的算法不会把大 O 增加到某个荒谬的数字，要做到这一点，你至少需要懂一些代数 2。

这是大多数复杂的机器学习算法的来源，相信我，它们需要更多的数学知识，而不仅仅是代数。你上一次在 web 开发中真正不得不做复杂的数学运算是什么时候？web 开发人员很少需要处理贝叶斯统计。

对此要有所保留，因为计算机科学不仅仅是数学。你必须为你的程序创建架构，这可能依赖于你使用的硬件。

## Web 开发更多的是基于应用。

如果不接触 IDE 并编写一些代码，你就不可能成为一名 web 开发人员。再多的理论也无法建立一个网站。其实不写代码连 web 开发都学不会。作为一名开发人员，有一些理论上的东西会对你有所帮助，但它们并不总是与计算机科学相关。

如果计算机科学家的算法和架构被传递给开发人员，他们可能永远也不会接触一段代码。大多数计算机科学家知道如何编写代码，但有很大一部分人并不擅长。作为一名 web 开发人员，编写和调试代码只是生活中的一部分。

还是那句话，对此不能全信。Web 开发人员专注于开发人们日常使用的应用程序。你写的所有代码都围绕着人们将如何使用你的应用，以及如何防止他们破坏它。有时候你必须做的方式并不是最有效的，但是你需要它在截止日期前有效。

* * *

大多数时候，由于学位要求，计算机科学家比其他人更容易成为 web 开发人员，尽管这种情况确实存在。然后，web 开发是软件工程的一个子集，这是一个完全不同的论点。

请记住，一般来说，web 开发比计算机科学更专业。你可能想知道的主要问题是工资。根据经验水平和专业，是的，计算机科学通常支付更多。

问题是你仍然需要一个学位才能被称为计算机科学家。如果你在找一份薪水很高的工作，而且技术上不需要学位，那么网络开发可能就是你要找的。

另一件要记住的事情是，一个不一定比另一个更好。它们都有各自的用途，也有各自的挑战。然而，归根结底，我们还是技术人员。

我知道这会引发一些激烈的争论，所以我想听听你的想法。计算机科学和 web 开发之间的差异是在缩小还是仍然有天壤之别？
