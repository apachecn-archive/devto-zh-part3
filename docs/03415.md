# [](#dev-rel-developer-relations)Dev Rel =开发者关系

首先，我有一些我在加入 [DataStax](//httsp://www.datastax.com) (大约 10 个月前加入)后的 6 个月里对一些事情的观察。在这篇文章中，我详细介绍了我对 DevRel 的许多方面、角色、部门、组织结构和相关元素的一些想法、观察和想法。

# [](#refining-the-definition-of-developer-relations)细化开发者关系的定义

在过去的几个月里，出现了很多关于 DevRel 在组织结构中隶属于市场部的时刻和对话。这让我重新思考这个问题，“DevRel 是什么，我们又该做什么？”以一种自由和开放的方式问这个问题，会带来许多答案和想法，围绕着不同的 DevRel 团队，甚至一个 DevRel 团队中的团队可能有什么任务。让我们打破这一部分，只是想通了定义。DevRel 包括的或者与之密切合作的一些其他组我也会包括在内。

## [](#developer-advocates)开发商拥护者

DevRel 的核心是对开发人员的支持。这种倡导伴随着一种隐含的观念，即倡导者将带来坚实的技术细节。然后，这些细节被带到工程中，在许多情况下，甚至以某种技术方式促进生产的进步和发展。这种情况总是发生在拥护者中吗，可悲的诚实的答案是不，但这是另一个博客条目。在这一点上，让我们用一个简单的定义来工作，开发者关系的倡导者从技术的角度工作，把产品和实践带给社区中的开发者。然后将从这些互动中获得的经验和了解开发人员社区正在做的事情带回工程和产品，以帮助开发产品，进而传递信息。为了清楚起见，我再次强调这一点:

> “倡导者从技术角度出发，将产品和实践带给社区中的开发人员。然后将从这些互动中获得的经验和了解开发人员社区正在做的事情反馈给工程和产品部门，以帮助开发产品，进而传递信息。”

我觉得，即使是那个冗长的定义也有一些关键词。首先，当我在这个定义中写社区时，我有一个具体的和包容性的上下文来使用这个词。它包括客户，但也非常具体地包括非客户、类似竞争产品的用户、潜在客户以及所有对产品或产品相关主题感兴趣的人。除此之外，产品需求在这个定义中也有明确的范围。产品意味着，例如在弹簧框架的情况下。产品不会仅仅局限于 Spring 及其代码库和构建的框架产品，它还会包括框架如何与其他产品交互或不交互。这将包括需要至少一个短暂的熟悉，以及在出现问题时能够更深入地研究围绕 Spring 框架的整个生态系统的外围技术。

如果这个定义有任何不合理的地方，我很好奇你的想法。这个定义好吗？在所用单词周围添加具体细节有帮助吗？如果你对这件事有想法，我希望你的想法，观察，想法，尤其是任何意见和热点！

## [](#curriculum)课程设置

课程任务:如何有效地学习和分享产品知识

开发人员关系团队通常包括课程开发，可能是课程开发的一部分，或者以其他方式与课程开发紧密结合。课程开发是一个创造性的、系统化的过程，它决定了如何呈现学习和教授产品和产品生态系统的材料，这一点极其重要。除非你卖的是一个简单的按钮，否则这个星球上几乎所有实用的产品或服务都至少需要一些教育材料。我们都是从对某个主题一无所知开始的，这个团队的目标是让一个新的学习者从零知识到精通尽可能好的方法。倡导者或专门的教师可能会负责提供这些材料，有时它的组织方式略有不同，但无论如何，理解课程发生了什么都是极其重要的。

让我们以 DataStax 的课程团队为例。他们构建材料，为我们的研讨会、全天教学会议、DataStax 学院材料等提供途径。有时倡导者会参与进来，帮助组织材料，有时是工程师和其他人。他们做了扎实的工作，我非常感谢他们的支持。它给老师们，在很多情况下是我们的拥护者，一条没有确定道路开销的道路。

然而...

这仍然是非常重要的，就像倡导者以有效的方式将社区反馈带到工程中一样，我们需要将学生的反馈和想法带回课程团队，以提高课程的有效性。在我们教学的同时，我们发现了呈现信息的新方法和帮助学生尝试和实验概念和想法的新方法。因此，再一次，倡导者完全符合两个群体之间的沟通任务。确保这种交流和课程材料一样有效是开发者倡导者的许多核心技能之一。

在下一篇关于提炼、定义和学习 DevRel 最佳操作方式的文章中，有一些主题想法:

*   抽搐流-它是如何工作的，它给了我什么？它给潜在客户、社区和相关的想法带来了什么。
*   Github -从 DevRel 的角度来看，使用 Github 最有效的方法是什么？显然，代码在这里，但还有其他方式——我们应该大量使用维基，用 Github 页面构建页面以提供额外的信息，应该为 repos 提供单独的域名，还有其他什么要问的吗？如此多的问题，再一次，一个似乎没有从 DevRel 的角度经常探索的空间。
*   Twitter——这似乎是许多思想聚集、碰撞并以积极和消极的方式引发混乱的中心。在 DevRel 中，有哪些方法可以最大限度地利用 Twitter，当 Twitter 成为一个标准的、基本的、各种各样的家庭实用工具时——它仍然带来了什么价值？
*   LinkedIn——这是一个寻找工作、与他人联系、与他人讨论话题的好地方，同时也是一个充满热情和粗鲁的招聘人员的沼泽。一个人如何获得价值或增加价值？
*   StackOverflow、Hacker News 和其他媒体——对于 DevRel 希望联系的社区中的人们来说，还有哪些资源有利于消息传递、讨论、学习和相关工作？
*   DevRel 的价值——DevRel 为社区和产品的潜在客户提供了很多价值。但是什么为我们提供了价值呢？这是一个很少被触及的问题，更不用说回答了。
