本周 Glitch 是一个常规系列，在这个系列中，我们总结了一些了不起的创作者在 Glitch 上完成的令人惊叹的项目。以下是本周吸引我们眼球的几个项目。

## [了解调制器](https://glitch.com/~modulation-viz)

首先是何塞·布兰科·佩拉斯的“项目”[调制——即](https://glitch.com/~modulation-viz)。这是一个交互式的可视化展示，显示不同的调制器如何影响信号载波的参数。通过观察打开和关闭不同调制器的影响，可以帮助您了解周期波的调制方式。更好的是，它是用纯 JavaScript 编写的，如果你想深入研究数学，你可以[查看源代码](https://glitch.com/edit/#!/modulation-viz)。

[https://glitch.com/embed/#!/embed/modulation-viz?previewSize=100&path=index.html](https://glitch.com/embed/#!/embed/modulation-viz?previewSize=100&path=index.html)

## [试出千伏存储](https://glitch.com/~rollup-built-in-modules)

Chrome 正在试验一种称为内置模块的新功能，他们推出的第一个功能是名为 KV Storage 的异步键/值存储模块。这是一个简单而缓慢的替代方案。其工作原理在这篇[博客文章](https://developers.google.com/web/updates/2019/03/kv-storage)中有所解释，但演示应用“[内置模块汇总](https://glitch.com/~rollup-built-in-modules)展示了如何在自己的应用中试用内置 KV 存储模块。

该演示由谷歌网络平台工程师 Philip Walton 创建，与 Rollup 捆绑在一起，因此它使用渐进式增强，并退回到为尚不支持 KV 存储的浏览器使用 KV Storage polyfill。

[https://glitch.com/embed/#!/embed/rollup-built-in-modules?previewSize=100&path=index.html](https://glitch.com/embed/#!/embed/rollup-built-in-modules?previewSize=100&path=index.html)

## [在设计中加入移情](https://glitch.com/~mood-ai-design)

[Olesya Chernyavskaya](https://glitch.com/@monolesan) 创建了“ [mood-ai-design](https://glitch.com/~mood-ai-design) ”，这是一项设计实验，旨在通过使用人工智能和相机来赋予设计同理心，以确定你的情绪，使用 [face-api.js](https://github.com/justadudewhohacks/face-api.js) 。

通过检测 7 种情绪(高兴、惊讶、中性、悲伤、恐惧、愤怒和厌恶)中的一种，Olesya 的原型会相应地改变所呈现网页的文字、语气和颜色。这是一种有趣的方式，看看我们是否可以通过赋予它感知我们的情绪并基于此改变内容的能力来“人性化设计”。

[https://glitch.com/embed/#!/embed/mood-ai-design?previewSize=100&path=index.html](https://glitch.com/embed/#!/embed/mood-ai-design?previewSize=100&path=index.html)
