# 也许我们应该暂时远离网络世界

> 原文:[https://dev . to/zellwk/maybe-we-should-step-away-the-online-world for-bit-7gj](https://dev.to/zellwk/maybe-we-should-step-away-from-the-online-world-for-a-bit-7gj)

我们开发人员已经变成了一群有毒的人。我们在所有媒体上制造激烈的“辩论”。推特，脸书，无论我们在哪里。

我们谈论 CSS 如何糟糕(以及它们如何不糟糕)。

我们谈论可访问性和性能(并抱怨做得不好的公司)。

我们讨论框架和非框架。React vs Vue vs 香草 JavaScript。以及为什么应该学习框架和为什么不应该学习框架。

我们还讨论了一些技术是如何“死亡”的(即使它们仍然继续存在相当一段时间)。

大家都有看法。这些意见大多是抱怨。我们在交流中传播愤怒、恐惧和担忧。每天地；天天地

这应该停止(但不会，因为你无法控制人们说什么或做什么)。你能做的就是休息一下，忽略其他人要说的话。

## 我很擅长忽略别人的意见

应该学习框架吗？该不该学框架？

Web 组件已经死了！CSS 烂透了！React 烂透了！

如果你在建一个球场，你应该制作视频！纯文本不行！人们对视频的反应更好！而且还有更多感知价值！

每个人都有自己的观点。我们有权发表自己的意见。我们也有权表达我们的观点。但它们只是观点。

它们不是事实。

我很擅长忽略别人的意见，因为在我的一生中，我忽略了很多意见。

还在上学的时候(我是商科学生)，同龄人都想去大公司工作；或者银行。我不喜欢那样，于是我开始编码。

大多数人会认为自由职业是艰难和可怕的。在你有了几年的经验(和好客户)之前，你不应该这么做。但是我继续前进，在学习代码不到一年的时间里开始了自由职业。

学习代码两年后，我写了第一本书。我凭什么写书？我没有权威，没有经验，也没有写作的天赋。大多数人都会这样想，我也是。但我还是写了(也卖了)它。

我做的特技很冒险。我不否认这一点。

但是公众的意见并不重要。重要的是我们给自己讲的故事。

也许你没有能力做我年轻时做过的事情。没关系。但也许你的“我做不到”只是你告诉自己的一个故事。可能是假的。

我从没想过我能在编码两年后写一本书。我没有勇气做这件事。我只是在改变了我的叙述之后才能够写这本书(通过经历一个生活指导课程)。我鼓起勇气开始写作。

**我改变了我告诉自己的故事**。

你也可以改变你告诉自己的故事。第一步是离开你创造的舒适区。

## [](#stepping-out-of-the-comfort-zone)踏出舒适区。

走出舒适区的最佳方式是**尝试新事物**。一些你从未想过会做的事。

我做了一些我从未想过会做的事:

1.  市场营销(必须学习，因为我需要推销我的课程)。
2.  治疗(想学，因为我厌倦了总是感觉疲倦和无精打采)。好家伙，这项工作是有益的和有趣的！如果我们有机会见面的话，也许我可以在你身上下功夫。会很有趣的。
3.  刚刚拿起我尘封了十年的吉他(因为我意识到我仍然想学弹吉他)。

还有，今年五月，我要去参加一个关于萨满教的课程。

我做的这些额外的事情大大扩展了我的世界观。当我离开我创造的科技泡沫时，我意识到**这个世界比我想象的要多得多。**

编码不是一切。

你不必编码。见鬼，如果你愿意的话，你甚至可以不去设计或者与网络世界有任何关系。人们在你想象不到的地方生存(甚至繁荣)。

所以生活没有固定的方式。没有人知道答案。

可以选择编码。但是这些激烈的争论都无关紧要。

重要的是你想要过的生活。

*   你想干嘛?
*   你想怎么生活？
*   你想成为谁？
*   你想关注什么？
*   你想帮助谁？
*   你想承担什么问题？

或许是时候靠边站，喘口气了。深入挖掘，回答那些问题。每隔几个月休息一下，再看看你的答案。会改变的。

也许有些事情会召唤你，也许和 web 开发无关。但是你可以走了。尽情享受吧。
