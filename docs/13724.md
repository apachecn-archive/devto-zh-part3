# 如何开始？

> 原文:[https://dev.to/pranavkhurana/how-to-begin-with-2pa1](https://dev.to/pranavkhurana/how-to-begin-with-2pa1)

*“如何开始？”*，也许是最难的问题，只能说是因为我的经历，我对研讨会的经历，我对如何开始写这个的经历。

所以实际上我最近做了一个关于“Python 入门”这个主题的实际操作会议，这是我第一次站在这个舞台上，就像所有人一样，我也经历了同样的阶段，一次又一次地害怕☹和那种焦虑，直到这一切开始并顺利结束。

我们探讨了一些基本方面，如 Python 中的变量、Python 列表、切片和循环。我也学到了一些新东西，探索了更多关于语言的东西，在那次会议中有一个时刻，当我讲述所有关于[身份运算符、](https://www.tutorialspoint.com/python/identity_operators_example.htm)的内容时，我开始讲述一些错误的东西，但后来我意识到，没有人会取笑我的错误，如果我在某个地方出错，每个人都会帮助我并纠正我。除此之外，一切都很好，我明白了如果我们能帮助别人，那是一种极大的快乐。

此外，我的一个朋友问我，我是否可以写博客或其他东西来分享我的经历，但我无法找到，如何开始，如何开始写作。

我花了 3 天时间思考如何开始，但没能得到它，所以今天我只拿了一支笔和一张纸，不假思索地开始了它，同时我意识到，我们只需要开始时不要想太多，一切都会自动继续流动，显然会有错误，但会被忽略，并进一步纠正，就像你们在阅读时忽略语法错误，只是继续流动。☺
