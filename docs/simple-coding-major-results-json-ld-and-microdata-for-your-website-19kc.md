# 简单的编码，主要的结果:JSON-LD 和微数据为您的网站

## 什么是结构化数据

谈到 SEO，许多人都忘记了一个事实，即谷歌通过使用一种算法来抓取网站，内容的质量与人类的感知并不严格相关，但一切都围绕着多个数字连接和提及而波动。为了更好地理解这一点，链接实际上是谷歌在质量和相关性方面看到的主要排名原因。结构化数据遵循一种[模式标记](schema.org)，并将其与 JSON-LD 脚本相结合，以便被臭名昭著的 Googlebot 正确抓取。

## 我应该如何处理我的结构化数据？

应用结构化数据非常容易，因为它创建了一个脚本，该脚本独立于所有其他功能，事实上，它被视为编码的基本形式，可以通过使用 [schema app](https://www.schemaapp.com/) 安装在编码站点和基于 CMS 的站点中。鉴于模式社区中有大量可用的标记，而且由于它是开源的，更多的标记正在开发中，因此使用最常用的标记很重要，比如组织、网站和产品标记。

## 好处

结构化数据是 SERP 中丰富片段的主要信息来源。这不仅从 UX 的角度来看非常重要，而且在许多营销案例研究中也涉及到了这一点，因为我们可以看到某些严重依赖 SEO 和有机流量的 iOS 应用开发公司。
最重要的是，整体的标记策略肯定会提高有机流量，因为获取过程将被简化。

## 最后

在你的网站上安装结构化数据肯定可以改善你的网站的抓取过程，如果做得好，还可以确认你的品牌在网上的提及(例如，通过使用“sameas”属性)。
