# 标准化的重点是什么

> 原文：<https://dev.to/jessekphillips/what-is-important-to-standardize-32ch>

标准化很重要，我可能会误解这篇评论，但是我想继续完善我的想法。

我认为推动标准化的重点是任何不同的东西，然后要求是使它不不同。挑战在于你不想困住别人。

一个糟糕的标准是每个人都应该使用的文本编辑器。他们的许多工具都属于这一类。这些是你的工具箱工具，你已经学会了如何使用它们，它们在你的环境中工作，影响那些试图触摸你键盘的人。

仍然存在支配这些工具的吸引力。例如，eclipse 和 NetBeans 以不同的方式管理您的项目构建和结构。这两者都有文本插入字段，但是在这种情况下，您的选择会影响到每个人对代码的理解。类似地，如果你在一个集成环境中模仿服务，那么这个服务是共享的，你可以共享模仿定义，人们可以理解。

这些工具的另一个价值是在培训时。我不知道有多少次我试图用 Git 帮助别人，每个图形工具都被安装，因为他们从那些使用不同工具的人那里获得帮助。因此，标准化用于培训的工具，无论是 firebug、chrome 调试器、postman 还是 art。

尝试标准化沟通，如何而不是什么。你通过会议、合并请求、代码注释来交流吗？事情并不完全是你所想的那样，但首先；

跨部门沟通的内容很重要。这就像你的公共 api，在你的部门内部，你应该有流动的变化，而跨部门的应该是更静态的。
