# 州规则！

> 原文:[https://dev.to/dlomedes/state-rules-5doa](https://dev.to/dlomedes/state-rules-5doa)

国家规则

*   不要混淆道具和状态
*   “State”是一个 JS 对象
*   仅适用于基于类的组件
*   创建组件时必须初始化状态
*   从不直接更新状态。
*   状态只能用 setState 更新。
*   更新状态会导致组件重新呈现。

状态对象是在构造函数中定义的。
使用状态(this.state)管理动态数据。

构造函数注释
React 的基类。组件有自己的构造函数，通过定义新的构造函数，它会被重写。但是为了确保“父”构造函数被调用，我们使用了 super(props)。Super with props 是对 React 内部父构造函数的引用。组件功能。
