# 求社区发帖

> 原文:[https://dev.to/rhymes/ask-the-community-for-a-post-1a1p](https://dev.to/rhymes/ask-the-community-for-a-post-1a1p)

DEV 的社区充满了对各种不同主题充满热情的人们，以及那些愿意投入更多时间来写关于软件开发的博客的人。

感谢 [@jess](https://dev.to/jess) 和开发团队，现在有了一个官方标签来互相推动:

#RequestForPost is the perfect way to ask the community to write an article you would really like to read

您可以使用它从社区请求帖子，将您的愿望发送出去，就您想写但目前没有时间的主题提出想法，就您了解的主题或您可能想到的任何其他主题请求不同的观点。

这与`#explainliveimfive`的区别如下:

[![rhymes profile image](../Images/557aa2636ae2993dfd13f29772ceb666.png) ](/rhymes) [ rhymes ](/rhymes) • [<time datetime="2019-01-02T21:57:20Z"> Jan 2 '19 </time> • Edited on <time datetime="2019-01-02T21:57:40Z">Jan 2</time>](https://dev.to/rhymes/comment/7pfb) 

我想过这个问题，但我认为有一个微妙的区别。

`#explainlikeimfive`是要求社区向你或任何正在阅读的人解释概念

这应该是把你的兴趣和想法放在那里，并希望有人来拿起它们；-)

所以，不要说“给我解释一下单子，因为它们很复杂”，而是“我一直想读一些关于单子的东西”或者“我想写一些关于延续传递风格、文件系统和循环的东西，但是我没有时间”。

我承认这可能行不通，因为对我来说也不是 100%清楚，我们只能试试:D

希望细微的差别是显而易见的...

标签侧边栏中的一些示例想法和一个由 [@helenanders26](https://dev.to/helenanders26) 提醒我的想法:

*   未来的浏览器是一个深远而全面的开发者平台
*   “我职业生涯中的一天”(例如，询问一位从事与你完全不同工作的专业人士)
*   奇妙的并发模型以及何时使用它们
*   窗口函数以及为什么它们是现代 SQL 中最强大的特性

问吧！
