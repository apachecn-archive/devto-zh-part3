# Git 中级-提示

> 原文：<https://dev.to/vitalipom/git-intermediate-tip-120g>

我只想让你知道两个 git 命令，使用 git 的人和其他不真正使用 git 的用户并不知道，因为大多数 git 基础和 git 提交是互联网上人们谈论的唯一话题。

那么，你有五分钟的时间在你的项目之间检查额外的材料吗？
慢慢来，复习一下`git rebase`和`git cherry-pick`命令。`git-rebase`不仅仅是简单的分支扁平化，而且`git-cherry-pick`允许你进行大量的提交，并将它们从一个分支转移到另一个分支。因此，如果你有一个“神奇地工作”的特性，并且你想在把它推向`origin`之前理解它，你通常会关注如何重新绘制你的 git 树，使它看起来更漂亮。在任何情况下，这两个命令对你都是有用的。
