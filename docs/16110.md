# HTML 标题:不仅仅是“大、更大、最大”

> 原文：<https://dev.to/sublimemarch/html-headings-not-just-big-bigger-biggest-1km>

我看到大多数初级 web 开发人员(以及许多非初级人员)的一个常见错误！)make 误用了标题标签。最近，我为一个正在训练营进行前端开发的朋友审阅了一些代码，并提醒他们尊重标题标签的语义规则，提到这对于可访问性非常重要。

他们的回答非常鼓舞人心:“我没有意识到除了喜欢之外还有语义上的原因(标题标签)——更大更大更大！所以这很酷。”

等等，如果不是为了让一些文本比其他的大，那么标题实际上是干什么用的呢？回答:结构和语义！[语义 HTML 元素](https://www.w3schools.com/html/html5_semantic_elements.asp)，如`<form>`、`<table>`和`<h1>`为浏览器和辅助技术(如屏幕阅读器)明确定义了它们的内容。这些元素的目的不仅仅是视觉上的，尽管很容易假设一些元素(尤其是标题标签)基于默认的浏览器样式具有纯粹的视觉目的。

# 那么标题标签的用途是什么呢？

标题标签为页面提供了一个结构化的层次结构，就像一个轮廓。一级标题标签(`<h1>`)是最重要的，它描述了整个页面。它通常应该非常类似于页面的``属性。`<h2>`标签描述内容的大段，`<h3>`标签描述子段，`<h4>`标签描述子段等。每个页面需要恰好有一个`<h1>`。您不必使用其余的标题元素，但是您不能跳过任何一个，并且它们不能是无序的。例如，如果你使用一个`<h3>`元素，你必须在它上面的某个地方有一个`<h2>`。

# 但是如果我想要一个更大的标题，我该怎么办？

使用与你想要的文本大小相对应的任何标题标签可能很有诱惑力，但是记住标题标签与文本大小无关，这就是 CSS 中的`font-size`的用途。前端语言遵循[关注点分离原则](https://en.wikipedia.org/wiki/Separation_of_concerns)。也就是说，代码的每个部分都有不同的目的。在前端开发中，HTML 是关于结构和内容的，CSS 是关于表现的，JS 是关于交互的。这意味着我们的语义 HTML 元素与表示无关，因为那是我们 CSS 的工作。如果我们希望标题标签看起来像某种特定的方式，我们使用 CSS 来实现它。

# 这和可达性有什么关系？

使用辅助技术(如屏幕阅读器)的人使用标题标签来导航和快速跳转到页面区域。这类似于视力正常的用户通过标题和其他文字浏览网页。标题是网页可访问性不可或缺的一部分，在网页内容可访问性指南(WCAG)的 [1.3.1:信息和关系](https://www.w3.org/WAI/WCAG21/Understanding/info-and-relationships)和 [2.4.6:标题和标签](https://www.w3.org/WAI/WCAG21/Understanding/headings-and-labels.html)中有所讨论。
