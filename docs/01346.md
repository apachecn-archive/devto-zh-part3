## [](#suggestion-2-headline)2️⃣建议 2 -头条

除了标题“Web 开发人员”或“软件工程师”之外，可以使用任何其他名称它没有告诉我们任何关于你的事。我注意到来自印度和非洲的资料使用了“IT”这个词，即使他们想要描述完整的 web 开发经验。“IT”贬低了 web 开发人员的价值，因为“IT”是一个支持性的工作，所以我建议如果你想成为一个全栈开发人员，避免使用“IT”这个术语。同样的道理也适用于使用“网站”这个词，而不是说“网络应用”

## [](#suggestion-3-profile-summary)3️⃣建议 3 -概要总结

大多数个人资料要么什么都没有，要么有一篇论文的有价值的信息，伴随着一长串他们曾经读过或简单涉及过的每项技术的技能。

我在上一篇关于 DEV.to 的文章中说过，我谈到了你应该如何避免这些名单，因为它们会降低你的个人资料的价值，因为招聘人员可能会将你归类为缺乏经验。

您通常可以发现这些列表，因为它们似乎总是伴随着可怕的 ASCII 箭头或示例:

```
► Linux, Unix, Windows, FreeBSD, Solaris, Temple OS

► C, C++, C#, Lua, Python, Ruby, Go, Rust, Java, ColdFusion

► Rails, Laravel, Spring, Flask, Meteor, Wordpress 
```
