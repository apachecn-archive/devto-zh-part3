# 注入代码是什么意思？

> 原文：<https://dev.to/flippedcoding/what-does-it-mean-to-inject-code-50d6>

在科技行业，你不可能不听说代码注入。你会被期望在没有被确切地告知做什么或如何做的情况下注入代码，这没关系，因为我们将会讨论一下什么是代码注入。

所有代码注入的意思就是你把代码传递给其他代码，就像你把参数传递给函数一样。以动态网页为例。根据您设置的一些用户权限，不同的人会看到不同的布局。这是因为您将拥有基于这些用户权限运行的不同代码。

代码注入的好处是，您可以通过一些简单的逻辑非常容易地重用代码块。您可能有一些需要在用户每次登录时执行的验证代码。代码注入将帮助您将验证与您需要运行的所有其他事情分开。

代码注入还有助于自动化。如果你有一个你想让人们使用的试用版，这可以帮助你控制哪些用户仍然可以使用这个试用版。

现在不全是阳光了。代码注入会使调试花费大量时间。这是因为很难判断错误是从哪里开始的，因为许多执行的代码是匿名的。

如果不小心，代码注入也可能是一个安全风险。黑客寻找漏洞，他们可以戳进他们的代码，你不想给他们留下任何机会。只要确保你使用良好的安全措施。

当然，还有一个更技术性的解释，尽管希望这能让你对代码注入的工作原理有一个更高层次的了解。不管你是否知道，你可能已经这样做了一段时间。编码术语总是让我觉得我是一个糟糕的开发人员，直到我开始明白我知道如何做正确的事情。不要让行话让你沮丧。
