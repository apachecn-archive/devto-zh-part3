# 我花在开源上的时间是每小时 1 美元

> 原文：<https://dev.to/kossnocorp/i-pay-1-every-hour-i-spend-working-on-open-source-27e7>

开源为现代公司节省了一大笔钱。它被用于各个层面，从支持基础设施的软件到开发人员用来构建公司产品的库。然而，维护一个开源项目无疑是我们行业中报酬最低的工作。如果平均每小时工资超过一美元，我会感到震惊。

对我来说当然更少。其实是负面的。我花在 [date-fns](https://date-fns.org/) 上的每一个小时都要支付 1 美元，这是一个 JavaScript 库，用于处理 npm 上每月下载量超过 800 万的约会。[我们对 Open Collective 的年度预算估计为 371 美元](https://opencollective.com/date-fns)。平均来说，我每月工作 20 个小时，每月支付大约 50 美元的网站托管费。我甚至没有谈论我的时薪和我投入开发的数千美元，这样我们就可以与 Moment.js 功能并驾齐驱。

> 美国劳工统计局的报告显示，平均每个家庭每年在外出就餐上花费 3008 美元

如果一个家庭在外面吃饭花费了 3K 美元，那么如果平均小费金额的 20%，那么一个成年人至少要花 300 美元。那为什么每个开发者不每月花 25 美元给开源团队或个人捐款呢？考虑到开发人员的平均工资至少是美国家庭平均收入的两倍，而且我们使用低技能劳动力作为例子，这应该是最小的。

此外，如果你试图计算公司使用开源软件节省了多少钱，如果这个数字超过了公司的总资金，我不会感到惊讶。然而，大多数公司都不喜欢回馈开源。为什么会这样？我觉得问题又出在开发商身上。

众所周知，开发人员几乎不擅长评估任何事情，从他们开发一个功能所需的时间开始，到他们给公司带来的价值结束。所以难怪他们不重视开源贡献者的时间。当一个开源维护者要钱的时候，他们要么无视他们，要么[给他们贴上乞丐](https://twitter.com/ITBeHa/status/1041955686240985089)或者[推销员](https://twitter.com/MartinLClifford/status/1066758300241281024)的标签。[一些开发者认为雇主应该资助开源](https://github.com/moment/moment/issues/3979#issuecomment-304533266)，但是他们从一个微软员工的角度看待问题。结果，他们得到了像[事件流](https://github.com/dominictarr/event-stream/issues/116)这样的东西。我们需要看多少部剧才能开始清醒？

开发人员负责传达他们从开源中获得的价值，并说服他们的经理为他们使用的开源付费。管理者不会自己意识到，除非把它当成一个营销机会。

有了[开放集体](https://opencollective.com/)，他们的[回你栈](https://backyourstack.com/)和 [Patreon](http://patreon.com) 支持开源简直易如反掌，那么你还在等什么？

去给你最喜欢的项目和维护者捐款，然后去烦你的经理，直到他们也付钱。让我们让开源变得可持续。
