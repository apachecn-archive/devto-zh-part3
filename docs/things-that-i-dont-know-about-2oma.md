# 我不知道的事情

> 原文：<https://dev.to/jonoyeong/things-that-i-dont-know-about-2oma>

现在是 2019 年，还有很多事情我不知道 2018 年将会发生什么。我想把这些写在某个地方，这样我可以在一年中参考它。意识到我不知道的东西有多少是令人畏惧的。但是我喜欢承认我的知识差距。通过努力工作，我可以在 2019 年结束时知道更多一点。以下是我仍然不知道的一些事情:

*   加密货币(Cryptocurrency)——每个人都在谈论它，我有点明白它是如何工作的了。但我不知道它是如何建造的，也不知道如何开采。
*   **机器学习/人工智能** -我知道一些基本理论，但我从未成功地构建过一个让*学会*任何东西的程序。这个领域似乎是一个正在发展的趋势，所以对我来说，建立一个使用机器学习的项目可能是值得的。
*   **网站设计** -我还是不知道该怎么做。我不知道如何使用工具，也不知道如何让一些东西看起来更好。看到一个好的网站我就能说出来。但是我如何翻译成我自己的网站呢？
*   **函数式编程语言**——Javascript 算一个吗？我不知道。什么是函数式编程语言？为什么大家都告诉我应该学一个？我想更好地学习语言，所以为什么不从功能性语言开始呢？
*   容器(Containers)-Kubernetes，Docker，他们有关系吗？我不知道这些是如何工作的。容器为什么好？它们能包含什么？
*   本地应用程序开发——我一年只能学到这么多。但我确实想在某个时候学习应用程序开发。
*   graph QL——我见过 Github 和其他公司用这个。但我不知道这是怎么回事。我认为它是 REST API 的替代品。但是我怎么实现呢？为什么它是好的？
*   Javascript 前端框架 -现在有这么多的框架。我对他们一无所知。我听说过关于 Vue 的好消息，但老实说，我不知道他们试图解决什么问题。
*   **节点** -这有助于建造服务器？我知道它是用 Javascript 写的，过去 Node 也有过戏剧性的变化。我如何使用它进行 web 开发？
*   **花哨的 AWS 玩意儿**——我听人说起过雅典娜和胶水。但是我不知道他们是什么或者他们达到了什么目的。AWS 生态系统非常庞大，我想了解更多。

这只是我所不知道的事情的皮毛。可能还有很多。但随着我在 2019 年继续工作，我想花时间研究其中的一些主题。希望到今年年底，我会知道更多一点。
