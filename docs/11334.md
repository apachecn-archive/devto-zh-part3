# 闯入 DevRel 的 7 个技巧

2021 年 1 月标志着我成为一名开发者拥护者的第三年。这段时间我一直很忙。在 DevRel 中有很多乐趣，当然也有很多工作。对我来说，这是我做过的最有成就感的工作&我期待着在可预见的未来呆在这个领域。

## [](#1-learn-how-to-write)1。学习如何写作

当我说学习如何写作时，我并不是说你需要成为一个老练的完美主义者，从不犯拼写或语法错误。

我的意思是，你应该学会非常舒服地把你的想法写在纸上。对我来说，这需要大量的练习。我养成了记录我所学到的东西的习惯，既有博客的形式，也有 GitHub 中 READMEs 的形式。

我们写得越多，就越舒服。你将学会不再是一个完美主义者&你的写作风格将有机地具体化。

写作也让你在更深层次上理解思想。自己能够做或理解某件事是一回事，但真正教别人&向别人解释那件事是另一回事。

写作或教学会让你解决那些你还不知道答案的问题。它迫使你深入主题&获得你可能没有的知识。

你写的越多，向世界发布的越多，你就越有可能在网上出现。

#### [](#communication)沟通

在 DevRel 中最重要的事情之一就是交流。您将与您的团队、其他开发人员、社区领导、会议组织者和无数其他人交流。

更有可能的是，这种交流将以推文、电子邮件、直接信息或某种书面交流的形式出现。

舒适的写作会扩展和提高你的沟通能力，反过来让你更适合这个角色。

## [](#2-build-bridges)2。建造桥梁

我见过的最有影响力的演讲之一是来自 2015 年 NG Vegas 的 [Aaron Frost](https://twitter.com/aaronfrost) (是的，我曾经是 Angular 开发者)。

这个演讲叫做[搭建桥梁](https://www.youtube.com/watch?v=E8TkRKWGSVc)。在书中，他陈述了这样一个事实:别人为他的成功铺平了道路，&现在他有机会把它向前推进，他正在这样做。

我们中的许多人能够走到今天的唯一原因是因为其他人所做的工作，他们所做的教程，他们所创建的开源项目，以及他们孜孜不倦的贡献。这个想法也是建立桥梁&一旦我们有能力这样做，就向前支付。

根据你的自信和专业水平，搭建桥梁可能看起来像很多事情。对我来说，一对一的指导是免费的，就我所学到的东西写博客，主持并支付当地会议超过 3 年，回答关于堆栈溢出的问题等等。

这种哲学就是开发者关系的全部。

## [](#3-dont-be-married-to-a-specific-technology)3。不要拘泥于特定的技术

许多人进入 DevRel 是因为他们是专家。对我来说，我是一名 React Native 专家&现在仍然活跃在 React Native 社区。但这是你需要控制的事情。

当你进入 DevRel 时，最重要的一个方面是建立信任。这种真实性是没有办法伪造的。

过于忠于特定的技术或既定的做事方式会让你失去对技术总体目标的洞察力。我们的目标不是展示我们对我们选择的框架或语言的永恒的爱。

学会对竞争技术保持好奇，而不是你最熟悉的技术。很多时候，你会发现你可以从他们做事的方式中学到很多，或者你更喜欢他们做事的方式，以至于你改变了你的专业。

我最引以为豪的事情之一是 React 本地广播播客，我已经运行了 3 年多。我特意展示了像 Ionic、Xamarin、Flutter & NativeScript 这样的竞争技术。我还帮助发布了 OpenGraphQL 出版物，以便向 GraphQL 社区开放一些想法，而不是那些主要在其他领域推动自己议程的人所提倡的想法。

当你运营播客、时事通讯或社区财产时，你是在要求人们给予他们信任。当你在自己的观点中表现出偏见或者阻止别人的观点和想法被听到时，你就是在对你的社区造成伤害。

同样，我们的目标不是推销你喜欢的东西，而是让人们能够创造东西&提升自己。要做到这一点，要对你所擅长的技术之外的其他技术持开放态度，并向它们学习，随时准备转换。

拥有一个关注社区和受信任的记录将会大有帮助。

## [](#4-speak)4。说话

这似乎是一个显而易见的权利，但我们如何开始呢？

我开始在密西西比州的一个当地集会上演讲。这些都是完美的起点，因为支持 T2 的人群通常很少。他们通常不会投资很多钱来参加，所以如果你有一个小问题，就不会那么武断了。

也有本地的(虽然乏味&令人敬畏🤩Toastmaster 小组旨在帮助人们练习公开演讲&提高沟通技巧。这些都是开始&试水的好地方。

闪电谈话也是很好的开始，因为花 5 分钟时间准备材料和练习比花 50 分钟要容易得多，压力也小得多。你也将获得在“真正的”活动中演讲的经验&如果你愿意，你可以利用这个演讲参考来参加更大的活动。

为了进入你的第一个大型活动，我建议组织一次我称之为“生产就绪”的谈话。一些你练习过的东西，就像你的手背一样，一个你花了很多时间润色标题和描述的东西。你可以用这个演讲去参加几十/几百个活动，如果你被不止一个活动接受，你就不会有太多的压力去做同样的演讲，但是你会从中获得经验。

#### [](#nerves)神经

这里有一个关于公开演讲的秘密:它永远不会变得容易。T2 的确变得更容易了，但你可能永远不会在人群面前谈论你真正兴奋的事情，而至少 T4 会有些焦虑。

我知道这一点，因为我和许多其他有同样感觉的人谈过。当然，可能有例外，但这是规则。就像耐克说的那样，你需要**去做**。一旦你张开嘴&开始说话，焦虑就会开始消失。

当你开始说话时，这里有一些提示:

1.  准备和练习(大量)
2.  谈论一些你真正感兴趣的事情
3.  从小处着手

当面试这个角色时，你通常会被期望至少有一些经验。即使在聚会/小型活动中有一些轻松的经历也足够了。

## [](#5-teach-as-you-learn)5。边学边教

我清楚地记得我在 Medium 上写的第一篇“真正的”博客。这是 Webpack 的初学者指南。我正在为我的公司创建新的 React 原型(这是在 create-react-app 出来之前)&我正在努力学习 [**webpack**](https://webpack.js.org/) 。

随着我的学习&浏览文档，开始在这篇博文中写下我所知道的&在我知道之前，我觉得我有值得分享的东西。请记住，当我写这篇文章时，我绝不是专家！当时，有几十个人联系我，告诉我他们从中学到了多少。那篇帖子，我的第一篇帖子，现在已经被全世界超过 485，000 人阅读了！

这又回到了第一点，教授或写一些东西会迫使你以一种有时令人不舒服的水平去理解它，如果你的目标也是帮助别人理解它，这在一天结束时是有好处的。

## [](#6-encourage-people)6。鼓励人们

我们能做的最简单的事情之一，尤其是在当今世界充满负面情绪的情况下，就是鼓舞和鼓励人们。随着你的人际网络不断扩大，这一点变得尤为重要&你有机会帮助那些没有发言权或特权较少的人。

成为那些不断提供帮助和鼓励的人的一员，不仅会让你作为社区成员受益，而且你会真正帮助人们克服精神障碍、土地工作、土地自由职业合同——甚至帮助他们最终分享他们的知识和想法。

作为 DevRel 的一员，需要记住的最重要的事情之一是，你的主要工作是代表其他开发者和社区成员，而不是不可避免地雇佣你的公司。

## [](#7-act-like-one)7。表现得像一个

我想说所有这些中最重要的一点是开始成为一个人，不管你现在的角色是什么。跟随别人做你想做的事，向他们学习，并开始做他们正在做的事。

如果你这样做了，你就已经在获得这个角色的路上了。当你申请或面试的时候，如果你已经有了某种记录，这是很容易的。
