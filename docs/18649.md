# 吉拉是一个反模式吗？

> 原文:[https://dev.to/itsasine/is-jira-an-antipattern-1ob5](https://dev.to/itsasine/is-jira-an-antipattern-1ob5)

这篇文章分享在我的工作 Slack 上，非常棒: [JIRA 是一个反模式者](https://techcrunch.com/2018/12/09/jira-is-an-antipattern/) - TechCrunch

这显然是一篇观点文章，似乎是为了引发讨论，尽管只有另外一个人想在#general 上谈论它

最吸引我的是:

> 听起来是不是有点…嗯…像瀑布一样？

看起来吉拉对小事情的过度关注鼓励了瀑布式的方法。每一期都会经历一个瀑布。如果一个 sprint 在 x 周内完成了瀑布，那么它就是成功的。

你怎么想呢?吉拉是否帮助开发人员忽略了全局，或者制作可行的问题片段是否更值得？
