### 自我介绍！你是谁？你在哪里工作？

嗨！我是 Daniele，是 Twitter 的高级合作伙伴工程师。我在数据企业解决方案团队工作，帮助我们的客户利用实时和历史社交数据推动他们的大规模业务。

### 是谁或什么让你开始编程的？

我 8 岁的时候，我的堂兄弟住在附近。其中一个有一个我们经常玩的准将 64。任何有 Commodore 64 恋物癖的人都会记得任何加载屏幕上的模糊颜色——只要看一眼就会让我开心一天。

我的另一个表兄有一台飞利浦 MSX VG-8020。尽管这是一台更强大的机器，但这台电脑没有 Commodore 64 那样受欢迎。(公平地说，在过去，它被称为“穷人的准将”)。MSX 是微软开发的一个平台，所以它会附带一个相当标准的 BASIC 实现——就像你在第一代个人电脑中看到的一样。理论上，这将允许你为 Intel 8088 编写一个基本程序，并在 MSX 的 Z80 处理器上无缝运行。

MSX 可能是第一台让我意识到我想制造东西的电脑。我记得盯着它的蓝色屏幕，输入了我脑海中的第一个计算机程序:

"绘制:观察面部"

可怜的 BASIC 翻译会绝望地回答:

"语法错误"

电脑附带了一本有趣的基础手册；在我的表兄弟们玩完 MSX 上的[枪战](https://en.wikipedia.org/wiki/Gunfright)后，我会重置它，这样我就可以调用 BASIC 并尝试键入几行代码。运行完成的代码将是最有收获的感觉；我想那时我发现了我的激情。我会把我的堂兄弟姐妹想象成屏幕前的人。我想成为屏幕后面的那个人。(我知道我是个古怪的孩子。)

但我认为我激情背后真正的杀机是我的父亲。他把我推向了软件开发，可能我们都没有意识到我们的共同兴趣将会决定我的职业生涯。作为一名发电厂主管，他会监控发电机的生命体征，并直接从电脑屏幕上操作它们的设置。他的工作电脑将配备像 Lotus Symphony 这样的应用程序，其中包含一个非常复杂的自动化宏和公式环境。作为一个好奇的修补匠，我想他很快就被这个软件及其无限的应用所吸引。到 1994 年，我们拥有了第一台个人电脑——一台快速、闪亮、(几乎)全新的英特尔 80386。正式来说，那是我的第一份圣餐礼物。我们知道那是我爸爸给自己的礼物。他会使用 Microsoft Excel for Windows 3.1 和前面提到的 Lotus Symphony 来计算他生活的许多方面——包括他的轮班表和退休计划预测。他甚至会设计出一套复杂的宏指令，运行一整夜来生成可能中奖的彩票号码(他有时确实会中奖。)

当他把那台电脑带回家放在我们客厅的桌子上时，他会在电脑外壳上贴一张便利贴来完成安装。便条内容如下:

绝对不要以任何理由打字:

*   `FORMAT C:`

*   `DEL \*.\*`

*   `FDISK`

可悲的是，我运行它们，抹去他所有的宝贵工作。直到今天，我们仍然不谈那件事。

### 作为合作伙伴工程师，你如何找到客户端交互和编码的平衡？

将客户交互和编码结合起来绝对是我工作中最棘手的部分。许多公司没有明确区分内向和外向职责。我最好的方法是无情地封锁我的日程表，至少留出 20-30%的时间用于内部任务，包括构建产品和编写文档，如果这样做有意义的话。在任何一个季度末，我的时间几乎全部用于客户互动，因此我可以通过几乎实时地提供解决方案、架构蓝图和技术指导来帮助销售组织。

也有内部时间与面对客户的互动交织的时候；当客户需要定制的解决方案或技术指导时(例如，当我需要审查一个集成时)，就是这种情况。在这种情况下，我分配时间与客户的开发人员聊天，并保留内部时间来调查和原型解决方案或修复(如果有 bug)。这样，我的内部时间将被夹在两个短时间的外部沟通窗口之间——迫使我对我的技术任务进行优先排序和时间限制，并努力向客户更新我的进展。

### 在你的工作岗位上，你经常与许多非技术人员一起工作，你是如何发现这一点的，你对有效的工作关系有什么建议？

我绝对喜欢和非技术人员一起工作；我发现这方面的工作几乎让人上瘾。我必须用英语将一行行代码翻译成现实世界中的例子，英语不是我的第一语言。这让我保持思维活跃，也是我对这个角色充满热情的原因！

与非技术人员的理想关系建立在同理心、技术领导力和信任的基础上。当非技术人员参与一个技术项目时，他们可能会被他们认为不会掌握的大量细节所淹没。理解他们对项目感到不安的原因是至关重要的，你必须承认他们的感受。也许他们觉得自己的技术背景不如同行强大；或者他们只是需要有人解开一些最复杂的行话，而仅仅是在说代码的人和说语言的人之间充当翻译。不管是什么原因，同理心让你成为一名具有与客户互动能力的工程师，以及技术项目经理或软件工程师。

给谈话带来一定程度的安慰也将为更顺畅的沟通和互利打开大门。例如，非技术人员可以帮助你驾驭一个大组织，但这只有在他们确信你的技术领导是可靠的时候才能实现。我不害怕说我在职业生涯开始时就在这方面挣扎过。我原以为技术领导仅仅意味着通过煞费苦心地列出一个项目的所有细节来展示你的知识，但是我很快发现这个方面实际上被认为是过分热心了。技术领导实际上是利用你的最佳判断将最困难的方面浓缩成一个简单明了的陈述；这也意味着隐藏你知道不需要项目的关键利益相关者达成广泛共识的细节。鉴于他们的经验和背景，合作伙伴工程师可以轻松评估任何给定任务的时间和工作量，并可以创建一个流畅的叙述，以在大型项目中达成广泛的共识。作为一名合作伙伴工程师，人们相信你的判断，因为你有能力与你的团队产生共鸣，你有扎实的技术能力和软技能。

### 你认为广告业在未来几年会有怎样的变化，这对你的角色会有怎样的影响？

社交媒体彻底撼动了广告业。移动时代显然给行业带来了另一场变革浪潮。它创造了一个全新的生态系统——应用程序开发，进而成为一个全新的广告垂直市场。尽管如此，大多数广告平台都在努力寻找一个完善的移动战略。由于我们仍然生活在这个变革的时代，我的工作与测量和战略解决方案有很大关系。

预测未来相当困难，但在这个行业，性能和隐私将决定技术和广泛的对话。GDPR 等法规现在要求营销人员仔细考虑他们的数据架构和流程。看看测量行业的发展方向，你就会找到答案。

### 在担任领导一段时间后，你最近回到了个人贡献者的角色。这是一个艰难的决定吗？

绝对不行。这很有趣——我们将领导力视为一种进步，但对我来说，这是一种与你的日常职责并行的不同工作。在团队中有一个高级职位，我可以继续做导师，但更重要的是，我可以继续向我的团队学习。总的来说，有时在一家更知名的公司担任更重要的角色会更有意义，这也是我选择这份工作的原因。与你的经理谈论你想要的职业道路是很重要的(我们已经广泛讨论过这个问题)。

### 迄今为止，在你的软件生涯中，你学到的最艰难的一课是什么？

我有两条:永远摆脱技术债，永远不要满足于一个看不到你真正价值的公司。有意思的是，有时候这两件事是交织在一起的。

技术债务的积累有很多原因，但如果我们深入挖掘，那些无力偿还(或者更糟的是，他们故意选择不偿还)的公司是那些努力在他们的行业中产生重大影响的公司。当这种情况发生时，他们无法吸引优秀的人才，他们的领导层似乎不知道何时解决这种缺乏认可的问题。你可能最终会在一个工程团队中工作，而这个团队总是不能很好地更新你公司的堆栈。同时，首席技术官和/或工程主管可能会推迟这些关键基础设施的选择，以加快交付更多面向客户的交付品。这不可持续。以我的经验来看，这样的模式会让公司陷入困境，但不会让它彻底沉没。这是一种缓慢而痛苦的死亡。

由于公司是由人组成的，投资于公司技术的雇主就是投资于公司员工的雇主。有两种类型的公司:认为你决定公司成功的公司和其他所有人。就像尊重一样，认可是二元的:要么存在，要么不存在。如果你的影响只是得到了部分认可，那就根本没有得到认可。

认可不花钱，也不花时间，这就是为什么看到如此多的雇主不重视他们的员工是如此令人沮丧。绝大多数公司认为，认可意味着创造一个社会事件或活动的结构，所有这些都是以你公司的名字命名的。公司 Acme，Inc .将邀请你参加他们的“Acmagic”派对(这是*他们的*派对，而不是*你的*派对)——你听起来熟悉吗？如果这只是另一个快乐时光，那么它有什么“神奇”的呢？

领导可以花 5 分钟的时间来感谢你的努力工作。它应该是自然的，它应该是发自内心的，或者至少是合理的同理心。“我只是想提醒你们，能和你们一起工作是我的荣幸”这句话就能达到目的。

相反，他们的陈述听起来是音盲和被迫的——更糟糕的是，有时他们是以蹩脚的诗歌形式从一张纸上读出来的，或者他们是作为你知道你无论如何都会得到的奖励而呈现给你的，如果你等待足够长的时间并且没有太多抱怨的话。但是你猜怎么着:这与竞争无关；这是关于为了完全相同的原因呆在同一家公司，原因是你对你所做的事情绝对感到惊讶。我觉得我的影响在没有颁奖的地方比在我得到一个或多个奖项的地方更被认可。

### 对于成功的软件职业生涯，你的第一条建议是什么？

永远保持专注，永远亲力亲为。当你专注于建造某样东西时，时间过得真快。有时我意识到我找到了一种我不相信自己能达到的孤立状态。我相信每个人都有自己的建议。让你的环境隔音对我来说真的很好。投资好的降噪设备真的可以帮你减少很多分心的事情(有时候我只是调一下耳机，不放任何音乐。)

更抽象地说，保持专注也意味着牢记我们工作的大背景。

最后，作为软件工程师，我们应该永远感谢好奇心这份礼物，它让我们能够将一个问题分解成更小的块，我们可以单独解决，这样它们就可以组合成一个解决方案。将这份礼物运用到我们的日常生活中
