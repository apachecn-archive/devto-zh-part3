# 你对类型有什么看法？

> 原文：<https://dev.to/solkimicreb/what-do-you-think-about-types-eh7>

> 我不小心在周日贴了一小段时间，就在承诺继续作为每周一系列之后。如果你看到了那个帖子或者评论了它，我很抱歉。这是最终的再版版本。

随着大量教程的出现，一些事情正在成为 JavaScript 中事实上的解决方案，但没有太多关于实际好处和原因的讨论。这个系列的目标就是谈论这些话题。

规则是:

*   谈论为什么，而不是如何。包含一些喜欢的库和模式是可以的，但是这不应该是你评论的主要焦点。

*   不要谷歌！我对你的观点、经历、轶事和直觉感兴趣。

*   如果你没有要求，你的评论可能会在每周总结中被链接或引用。我从不在负面环境中引用评论。

* * *

### 本周的话题是**类型**

*   你喜欢强类型语言还是弱类型语言？

*   你有最喜欢的语言类型吗？是什么让它比其他的更好？

*   你能举个例子说明使用类型为你发现早期错误的情况吗？

*   TypeScript 与组项目有什么关系？与香草 JS 相比，它使进入和合作的障碍更简单还是更困难？

*   无论你想到什么类型。
