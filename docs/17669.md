# 假期项目时间，或者也许不是？

> 原文：<https://dev.to/mfahlandt/holidays-side-project-time-or-maybe-not-2jcf>

[![alt text](img/cd5eba84887315bc64525effd0e05b94.png "By Nathan Dumlao")](https://res.cloudinary.com/practicaldev/image/fetch/s--WrObd3OO--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/9r0npe7taqckvt12horg.png)

今年我花了 247 天写代码，并且可能会在接下来的两天里继续写。在我的 twitter 时间表和我阅读的许多博客文章中，假期是兼职项目时间。4-5 年前我会同意这一点，但现在不同了。

两年前，我的父母从一日游的距离搬到了需要一些组织和至少 4 天才能到达的距离。所以我们一年才见他们一次。当我想起过去的那些日子，我把时间浪费在我父母身边的一些小项目上，而不是花一些时间在一起，这让我感觉很糟糕。是的，你没看错，我写的是浪费。我们每个人在一年中的大部分时间里都以这样或那样的形式编写代码，这应该是有趣的，而且应该保持有趣，关键是:在某一点上你将不再有快乐，在某一点上它将变得必要。对我来说，这将是我一生中最糟糕的时刻，我不得不做一份不再给我带来快乐的工作。我可以听到强烈抗议，哦，你不是一个充满激情的软件开发人员，但老实说，我看不出来。我们所有人最重要的工作是照顾好自己和我们所爱的人。尤其是这些日子是为它而生。大多数你爱的人都有时间，你可以聚在一起玩棋盘游戏，散步，一起看电影，最重要的是让你的大脑从一整年必须做的事情中得到休息。
我经常听到的另一个观点是:兼职项目是我尝试新事物的唯一途径。这是一个让我一听就抓狂的论点。原因很简单。作为一名雇主，我的员工的知识是我最宝贵的财富。因此，如果现在有任何开发人员告诉我，假期是一年中唯一可以提升我的技术知识的日子，这意味着开发人员没有从他的雇主那里获得学习新东西的机会和时间。要求你的雇主抽时间尝试新的东西。如果答案是肯定的，你知道你的知识是有价值的，如果答案是否定的，也许是时候找份新工作了。

如果你在假期没有做兼职，即使你没有和你爱的人共度美好时光，也不要难过。即使你有空闲时间，只是想通过阅读非科技书籍、玩游戏或其他方式来放松，也可以看一部电影或一系列你在今年的压力中错过的节目。

退后一步休息一下，你不会错过任何东西。让你的心态达到这样一种状态:见鬼，我希望我能重新回到工作中去编码。

现在你的方式是什么，你正在为假期做一个兼职项目吗？还是你给了自己一天的时间。不管怎样，节日快乐
