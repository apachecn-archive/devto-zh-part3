# [](#what-is-it)什么事？

让我们用简单的话来说，服务只是一个有目的的东西。在说这是垃圾之前，请抓住我。

服务于制作软件的目的意味着它为一个类做一些该类本身不打算做的事情(遵循单一责任原则)。现在，类可以是任何框架中任何东西，例如 Rails 中的模型类和 Angular 中的组件类。

# [](#why-we-need-them)我们为什么需要它们？

实际上，我们只在以下情况下需要它们:

*   如果在不止一个地方使用相同的逻辑。我们在多个地方修改同一个东西的能力并不太好。
*   如果应用程序将被更改或修改。我想会的。

因此，如果我们的代码将要被修改/扩展，并且我们想要遵循 DRY 和 SR 原则，那么我想我们需要这样做。

# [](#how-we-do-them)我们是如何做到的？

服务什么都不是，只是另一个类。也围绕一个单一的责任做一些事情。例如:一个处理 Twitter API 调用的类。

现在，在您需要的任何类中创建一个此类的对象，并使用它来做它的事情，而您调用它的类将只使用此服务对象的输出。

> 现在，你如何在不同的语言/框架中实现它们是你的下一个搜索，我只是给了你一个搜索的理由。
