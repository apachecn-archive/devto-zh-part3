# 超越完美

> 原文：<https://dev.to/bencodezen/progress-over-perfection-3k6f>

对于那些已经跟随我一段时间的人来说，这大概是为我的编码生活建立一个坚实的写作仪式的第 8 次尝试。虽然我有一部分对这个事实感到羞耻，但这是我在过去两年中失败的又一次尝试，这个事实概括了今晚帖子的本质。

## 我遇到的障碍

你在这里看到的每一次我未能建立一个适当的博客习惯的尝试，都有太多我数不清的错误开始。虽然有很多原因导致我的努力总是失败，但下面是我遇到的一些最大的障碍:

*   害怕批评:我过去(现在仍然)担心我可能会写一些会招致社区或同行批评的东西。如今，人们很容易断章取义，很多人在没有了解全局的情况下就妄下结论。

*   **其他人写道**:每次我想到一个概念，我都会看到一篇标题或概念相似的文章，然后放弃这个想法，因为我觉得人们会说我抄袭了那篇文章，或者说我不是原创。

*   但是我需要网站上的 X 功能:我的博客会被我的同行看到的想法经常让我麻痹，因为我担心它会被评价为:

    *   设计美学
    *   表演
    *   易接近
    *   技术架构
    *   我不擅长的前端专业

我上面提到的障碍只是我想到的三个最大的障碍。在我继续讲述如何处理这些问题之前，我想强调的是**我今天仍然在处理这些问题**。对我来说，这场战斗还远未结束。

## 处理那些障碍的技巧

这些技术绝不是刀枪不入的，但我希望它们能在您处理类似问题时起到灯塔的作用。

1.  “原创”这个概念其实并不存在。如果你从写书的角度来考虑，每个故事都包含了过去某个时候存在的其他书或故事的一些元素。

2.  重要的是你的视角。你是独一无二的。即使一个概念可能有一百种不同的解释方式，你必须记住的是，你的方式可能会引起完全不同的观众的共鸣。

3.  记住零对一的重要性。对于那些没有读过[《从零到一》](https://amzn.to/2LpdaE4)这本书的人来说，简而言之，从零到一的影响是任何其他增长都无法比拟的。所以当你面临功能瘫痪的时候，最重要的是把你的想法表达出来。随着时间的推移，你可以迭代其余的，但是如果你的博客永远不会上升，人们阅读你的想法的机会是零。

由于我仍在正式确定我的应对批评恐惧的技巧，我会建议你关掉对博客帖子的评论。这有许多好处:

*   博客帖子上的评论很难实现，而像 Disqus 这样相当容易集成的评论有一些可疑的跟踪策略，这让我不太愿意让我的读者接受。
*   你不必管理评论和随机发送你的评论的机器人
*   你避开了那些想躲在屏幕后面而不是和你进行有意义的谈话的人

## 欢迎来到我的写生簿

我相信我以前曾以某种身份写过这方面的文章，但正如戴夫·鲁伯特恰当地指出的，博客最好被视为一个“草稿”文件夹。面对我们处理的令人难以置信的大量噪音，我们很容易觉得博客帖子应该是这些经过完美编辑的文章，可以提交普利策奖。

也许有些人想创建完美的博客，但从个人经验来说，这通常会导致失望和沮丧。对大多数人来说，写作本身就已经很难了，所以给自己压力去写宏伟的艺术作品通常会阻止这个想法实现。

坦白地说，我厌倦了我的想法被追求某种理想版本的博客帖子扼杀。所以我只是要跟随我的直觉，按照我受到的启发去写作。这意味着:

1.  很可能会有语法错误。如果有，请随时告诉我！我欢迎这样的反馈，因为我想在写作方面做得更好！

2.  有时候想法是不成熟的。就像一个艺术家有一个素描本，记录有朝一日可能成为真正的艺术作品的概念一样，把我网站的博客部分当成我的素描本。

虽然有些人可能会继续写获奖的博客文章，但我们必须记住的是，几乎没有人会从那里开始。所以作为对我自己而不是对你的提醒，记住:

> 写作的行为比写出完美的文章更重要

将会有一个时间和地点让我来策划完善的文章和教程，但这将不是这个空间。这是我自由写作和分享想法的地方。期待你与我同行。

干杯！🥂
