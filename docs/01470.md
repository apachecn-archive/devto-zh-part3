# 成为最好的项目经理

> 原文：<https://dev.to/flippedcoding/being-the-best-project-manager-31ej>

在某些时候，你可能会发现自己处于项目经理的角色。你可能已经做了几年的开发人员，所以你以前和项目经理打过交道。好的和坏的之间有着难以置信的差别。现在你站在了另一边，记住几件事让你的团队开心。

## 预先把一切都组织好

在你开始分配任务之前，应该已经考虑好时间估计。如果没有，花几分钟时间和你的团队一起做一个，这样他们就对他们应该做什么有一个大致的概念。能够向你的团队解释任务，如果不能，准备去寻找答案。设置您想要使用的通信系统，然后向您的团队展示如何使用它。

如果可以，在他们开始设置项目时，为他们的本地机器准备好数据库备份。您希望您的开发人员为他们准备好开始工作所需的一切。在你和你的团队讨论之前，你的组织水平为项目或者冲刺的进展定下了基调。一旦你把工作交给你的团队，他们将准备好开始工作，当他们有问题时，他们会看着你。

## 愿意回答问题

令人惊讶和难过的是，许多项目经理试图避免与他们的团队交谈。他们给出含糊的指示，然后几天都不会被打扰。不要成为那样的人。当开发人员对某个特性或用户故事有疑问时，你是他们的主要联系人。准备好用有用的答案回答你得到的每一个问题。

即使你目前不知道答案，也要让开发者知道你会为他们找到答案。只要让他们知道，当他们有问题时，你会在他们身边，并且你会对他们做出回应。如果需要在小组之间共享公共文档，可以创建一个新的 Slack 通道或其他东西。对于任何项目经理来说，拥有一个团队查看文档和相互交流的中心位置都是至关重要的。

## 知道每个人应该在什么时候做什么

每周至少和每个人一起检查一次任务清单。这样，每个人都知道本周任务列表的优先顺序，他们可以互相帮助解决问题。一开始就明确最后期限和任何期望。没有人应该想知道他们什么时候需要准备好他们的拉取请求。

确保任务的优先级被定义，这样人们就知道在他们完成当前任务后要做什么。只要对真实发生的事情保持透明，你的团队就会尊重你。如果每个人都知道每个人的期限和任务，那就不是问题。大多数时候这样更好。

## 如果需要的话，愿意加入这个项目

如果有一个部署需要在周五晚上进行，那么愿意和你的团队呆在一起直到完成。当你的团队在某个问题上陷入困境时，试着去帮助他们解决这个问题。项目经理能做的最糟糕的事情就是在团队最需要他们的时候抛弃他们。你的工作是确保团队按时完成工作，这意味着你可能需要进去帮忙。

即使你不知道如何编写代码，你也可以在那里回答问题，或者确保开发人员能够获得他们需要的资源。如此多的项目经理在非正常时间提出荒谬的要求，然后离开，这开始在团队和经理之间建立一些紧张关系。你希望你的团队听你的，对你有信心，最好的方法之一就是和他们并肩作战。

我不能说我已经做了很长时间的项目经理(或者我是否会一直做下去)，但是这些是我从一些优秀的项目经理那里学到的一些东西。有人有更多的项目管理经验吗？你从开发者转到管理者了吗，你喜欢还是想回去？
