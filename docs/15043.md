# 小心你为谁工作！

> 原文：<https://dev.to/hpez/be-careful-who-you-work-for-1o7c>

这是我在这里的第一个帖子！这是这样的:

去年我在一家公司工作了 4 个月，有一个大错误让我这几个月过得很艰难，那就是没有注意我在和谁/什么人一起工作。

### 面试官

首先，不要相信你在面试中听到的话，这可能意味着两件事:
1-面试官可能对你不那么诚实，或者，
2-他/她可能没有像你应该的那样看待这份工作(从技术角度来看)。

### 什么时候应该开始寻找更多

当你到了知道自己在做什么的阶段，除了薪水、合作经验和其他显而易见的东西之外，还有一些东西你应该弄清楚。他们正在使用什么样的软件堆栈，该堆栈是否适合您使用
2。他们在管理他们雇佣你的职位方面有多有经验

### 它能引起什么

不知道上述内容可能会导致我的经历，一个糟糕的软件栈导致我失去了编码热情，因此没有交付我应该拥有的质量。第二部分导致无法管理任务，不知道从开发人员那里能得到什么；这在一个相当小的团队中，会搞乱整个开发团队，并给开发人员带来很大的压力。

所以总而言之，小心你为谁工作！
