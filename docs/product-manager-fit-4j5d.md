# 产品经理适合度

> 原文：<https://dev.to/grantgumina/product-manager-fit-4j5d>

我最近面试产品经理候选人的经历告诉我，成为一名项目经理对不同的人意味着不同的事情。因此，为了更好地了解候选人已经做了什么，以及他们对下一份工作的期望，我会问他们以下问题:

“你最喜欢在哪里工作；一个公司的特色、项目、产品或业务水平？”

没有单一的正确答案，但候选人的回答通常会告诉我很多关于他们如何看待自己以及如何才能最有影响力的信息。

## 特征

特性项目经理是主要与工程团队一起工作的人，通过现有产品中的特性来实现功能。他们将大部分时间用于筛选从销售团队、客户支持单以及与客户的理想互动中收集的客户需求，并将它们反馈到开发流程中。

特性 PM 的范围很深，但不宽。通常，较大的公司雇佣大量的特性项目管理来确保他们发布的所有特性的一致性。一个很好的例子可能是像 Windows 这样的技术产品，它使用非常复杂的 SMB 文件协议，一个特性 PM 可能拥有特定类型的操作(多通道读取、写入租用等)。)并向负责整个协议 PM 领导报告。

一个不太技术性的例子可能是脸书如何在他们的 Messenger 产品上雇用大约 20 名项目经理，所有人都组织在一个层级中，Messenger 的副总裁在最高层。在最低级别上，功能 pm 负责确保 Messenger 端点都是同等可访问的，并且符合可访问性法规。这些人必须成为他们产品领域的专家，随着时间的推移，他们将选择寻求更广泛的领域，或者成为更复杂主题的专家。

据我观察，大多数特性项目经理都有技术背景，并且擅长探索复杂的架构实现决策。因为他们所做的大部分是在团队之间协调复杂的技术项目，同时确保在开发的每一步都考虑到用户的需求，所以特性项目经理开发技能，与他们的利益相关者交流技术复杂性。

鉴于其深度，功能项目经理是产品团队的一个很好的补充，他们希望将特定的优势添加到他们的名册中。如果产品中有一个快速增长的领域需要某人对问题空间(例如，协议、同步或数据一致性)有深刻的技术理解，那么特性 PM 可能是团队的一个重要补充。

值得一提的是，我已经采访了一些希望拥有一个产品或跨团队参与项目的特性项目经理。通常他们来自一家大公司，他们在面试中面临的最大挑战是:

1.  将复杂性引入决策并使其悬而未决的倾向
2.  从以企业或投资者为中心的角度思考产品决策的不安
3.  缺乏单独生产工作与协调或记录已完成工作的一致性和跟踪记录

然而，即使你需要候选人能够在更高的水平上工作，重要的是不要马上拒绝一个特色项目经理。有很多聪明、有才华的人担任专题节目经理，只要有机会，他们就会成功。找到一个合适的可能很难，但是风险值得回报。

## 项目

来自咨询背景的申请 PM 工作的候选人通常会回答许多问题，并给出他们在公司被分配的各种项目的例子。我通常认为这些候选人是最适合在复杂的项目级别工作的人，即使他们没有这样回答我的问题。

这些人有快速学习新产品或市场的丰富经验，并有在有限的时间内完成一个明确任务的授权。在大公司中，喜欢在项目层面工作的项目经理可能会从一个部门转到另一个部门，解决类似的问题。

例如，在 GDPR 法案在欧洲生效之前，微软部署了一个由项目经理、律师和工程师组成的团队，与各个业务部门合作，以确保 GDPR 法案的合规性。这些人在 GDPR 成为了专家，因为他们花了几个月的时间进入团队，执行一个计划，使他们遵从。

当我面试在一个组织的项目层面工作过的人时，特别是当他们来自咨询背景时，我面试他们是为了看看他们是否能像一个所有者一样行事。是的，顾问不关心他们的客户是一种糟糕的刻板印象，但在某种程度上，如果你每隔几个月就从一个项目转移到另一个项目，就很难对某件事投入 100%的感情。

大多数时候，候选人表现出他们非常有能力担当起所有者的角色。事实上，许多有咨询背景的候选人正在寻求改变，因为他们想要更多的所有权，所以他们几乎总是非常适合。

很多时候，担任项目 pm 的顾问都是多面手。鉴于他们丰富的经验，他们可以被派到公司的任何部门，评估情况，提出或执行建议。

## 产品

除非你是一家非常早期的公司的创始人，否则几乎不可能拥有一个完整的端到端产品。然而，在大多数后期阶段的公司里，需要一个产品经理参与进来，并负责整个产品开发周期。

这意味着负责最初的客户发现，定义技术需求，与工程师和设计师一起迭代以交付产品，并最终定义和执行上市战略，并对其成败负责。

大多数项目经理，包括我自己，都是随着时间的推移成长为这个角色的，或者是从以前的创业或领导经历中成长起来的。在产品层面上工作，需要在与客户和工程师交谈时，能够在复杂程度上上下波动，还需要能够以引起组织中所有利益相关者共鸣的方式快速吸收和综合数据。对于求知欲强的人来说，如果他们愿意忍受大量的环境转换，这可能是一份理想的工作。

在产品层面工作最令人沮丧的方面可能是你有多少利益相关者。公司会将你视为领导者，但你需要说服你的老板、他们的同事以及你在产品/销售/营销/财务方面的同事来执行你的计划。为了有效地做到这一点，你必须建立的社会资本数量可能会显得很消耗，对一些人(包括我自己)来说，这部分工作可能会妨碍创造一些令人敬畏的东西。

重要的是不要陷入孤独天才的神话，并且记住创造性的过程在大多数时候也是合作的过程。在你的组织中发展牢固的关系是这份工作的一个关键部分，随着这些关系的发展，你的创造能力也会增强。

然而，面试时要注意的一件事是，那些宁愿花时间建立社会资本，成为每个人的朋友，也不愿做出艰难的产品决策的项目经理们。PM 反模式存在，其中与内部利益相关者的关系优先于为客户做正确的事情。

不幸的是，这种特质几乎不可能在面试中被发现。

## 业务

大多数时候，在“业务”级别工作的人在他们的组织中处于某种领导地位。我没有那么高的职位去面试领导职位的人，所以我还没有听到一个候选人这样回答。

然而，在我看来，所有候选人都需要展示出协调客户和业务需求的能力。他们能根据某种考虑到业务现实和支持数据的框架来权衡决策的利弊吗？或者，他们是否会根据单个嘈杂的客户反馈做出过多的决策？更糟糕的是，他们会让自己对产品的个人感觉来驱动产品开发吗？

产品管理是一个奇怪且不断发展的学科。这些“层次”都是重叠的，项目经理需要同时在每一个层次上运作才能成功。然而，我发现这个框架在评估候选人如何最有影响力方面很有用，我希望你也是如此。
