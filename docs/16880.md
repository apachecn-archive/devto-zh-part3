# 抓住机会，脱颖而出

> 原文:[https://dev . to/Nicky tonline/take-chances-and-outstand-because-who-known-3k h6](https://dev.to/nickytonline/take-chances-and-standout-because-who-knows-3kh6)

这是我第一次写关于职业建议的帖子。这种建议绝对不是一刀切的。每个人都有自己的经历、环境等。，但希望我讨论的一些内容能引起你的共鸣。

2016 年 5 月，我决定开始创业，并在巴塞罗那住了一个夏天，担任前端开发人员，使用 React、TypeScript、Node 和其他工具。我和我的老板(未来的蒙特利尔办事处)在巴塞罗那学习业务，帮助西班牙办事处配备人员，然后计划返回蒙特利尔开设蒙特利尔办事处。

我面临的风险是，尽管我有一年多的专业类型脚本经验和丰富的 JavaScript 知识，但我没有 React 或 Node 的专业经验(除了一些构建工具之外),并且只在。网络生态系统。在过去，当这样的机会出现时，我会对自己说，你不是他们要找的人。但是我已经面试了 JS 全栈和前端的角色，所以我知道这是我想要的。所以我决定冒险一试。这可能是我职业生涯中最好的决定，因为它让我走上了我真正想走的路。

尽管我没有专业的 React 经验，但我已经开始为一个非常受欢迎的 React/Redux 样板文件做贡献，这个文件叫做 [React Slingshot](https://github.com/coryhouse/react-slingshot/commits?author=nickytonline) (最终我被[邀请成为一名维护者](%5Bhttps://github.com/coryhouse/react-slingshot/pull/298#issuecomment-258616537%5D)并被接受了😉)来帮助我学习 React 和[丹·阿布拉莫夫](https://dev.to/dan_abramov)的令人敬畏的免费 egghead.io Redux 课程。带着这个想法，我去了巴塞罗那。

我遇到了很多很棒的人/和他们一起工作，去了几次伦敦，和我的英国同事一起参观/工作，并在西班牙海岸度过了一段时间，还在安道尔的山区看了一场免费的太阳马戏团 [Scalada : Vision](https://visitandorra.com/en/agenda/scalada-vision-by-cirque-du-soleil-2016/) 户外演出，结束了这个夏天。

在我下一个职位的面试中，英特尔(现在是迈克菲)脱颖而出...说来话长，谷歌一下就知道了)。该角色是一名前端开发人员，负责他们的密码管理器浏览器扩展产品。我从未在浏览器扩展领域工作过，所以这引起了我的注意。面试进行得非常顺利，接下来是完成带回家测试的时候了。我是做一个基本的密码管理器浏览器扩展。

其中一个开发人员表达了对 [TypeScript](https://dev.to/nickytonline/why-you-might-want-to-consider-using-typescript-6j3) 的兴趣，正如我提到的，我已经有了经验，所以我用我的 React、Redux 和 TypeScript 技能构建了一个浏览器扩展。

扩建部分很有意思。我把它修好了，包括热重装。你可以登录大型网站，推特，脸书等等。该扩展将保存您的登录/密码。如果您注销并重新访问其中一个网站，您会自动登录。此外，还有一个额外的家庭作业，制作一个基本的管理页面来修改你每个域的凭证，这也是我做的。我甚至添加了 [Storybook](https://dev.to/nickytonline/getting-started-with-react-storybook-9jh) 来展示扩展使用的一些组件。我对自己建造的东西非常满意。当我和我未来的同事交谈时，他们说除了能力之外，我带回家的测试也很突出。其他人都做了最低限度的工作。成功...录用了！💯

去年夏天/秋天，当我在几个地方面试新工作时，可汗学院似乎是一个有潜力工作的有趣地方。然而，我又一次看了需求，我想，我不认为他们会对我感兴趣。我实际上推迟了大约 5 周申请。最终，我申请了一封我认为独一无二的求职信，还有一些平常的东西，比如我的 GitHub 简介。

我没想到会收到他们的回复，但最终我还是收到了。我最初通过电子邮件与他们的一位人才招聘人员联系，他为我安排了一次与我潜在的工程经理的面试。那次采访似乎进行得很顺利，我提到我已经为他们的 t0ta11y 项目做了一次公关。

## 架构概述

tota11y 中的大部分功能来自它的**插件**。每个插件在 [`plugins/`](https://github.com/Khan/tota11y/tree/master/plugins) 中获得自己的目录，维护自己的 JavaScript、CSS 甚至手柄。[下面是简单的 LandmarksPlugin 的样子](https://github.com/Khan/tota11y/blob/master/plugins/landmarks/index.js)。

[`plugins/shared/`](https://github.com/Khan/tota11y/tree/master/plugins/shared) 包含各种插件的共享实用程序，即[信息面板](https://github.com/Khan/tota11y/tree/master/plugins/shared/info-panel)和[注释](https://github.com/Khan/tota11y/tree/master/plugins/shared/annotate)模块，用于在屏幕上报告易访问性违规。

[`index.js`](https://github.com/Khan/tota11y/blob/master/index.js) 汇集了这一切。

tota11y 使用了多种技术，包括 [jQuery](https://jquery.com/) 、 [webpack](https://webpack.github.io/) 、 [babel](https://babeljs.io/) 和 [JSX](https://facebook.github.io/jsx/) 。没有必要知道所有的事情

事情进展到与未来潜在同事的第三次面试，然后从那里开始，事情就结束了。不幸的是，事情最终没有解决，但即使我很失望，这仍然令人惊讶，因为我得到了我认为是他们 a11y 工具的一个很酷的浏览器扩展的 PR，我得到了与 John Resig 交谈整整一个小时，只有我们两个人。

所以请记住:

*   抓住机会…真的。
*   出众。
    *   每个人都会在某个时候做课后测试。让你的与众不同。
    *   为开源做贡献。有些人会因为工作/生活平衡而不同意，但以我的经验来看，这有助于让你与众不同。
    *   以另一种你认为能让你从其他候选人中脱颖而出的方式脱颖而出。
*   申请职位，即使你认为自己不够好，因为你永远不知道。即使在某个地方面试不成功，你仍然可以对自己或工作有所了解。谁知道呢，也许一路上你会有一些有趣的对话。
