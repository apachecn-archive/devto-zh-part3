# 黑客是如何创造知识的

> 原文：<https://dev.to/metamn/how-hackers-are-creating-knowledge-2e4g>

在阅读了七个有超过 1200 条评论的黑客新闻帖子后，我发现没有任何工具可以让我立即投入到创造个人知识中去。

其中五个帖子是最近的——2018 年的夏天和秋天——揭示了社区遭受的一个明显的痛点。

它们不仅包括知识创造，还包括正确的笔记和目标跟踪、信息组织和搜索、通用同步和访问(移动)、互操作性(数据可移植性)和工具稳定性(几年后停止开发)。

有超过 50 个关于使用工具的建议——就像这些由 [SaasHub](https://www.saashub.com/reviews/post-news-ycombinator-2018-07-15-ask-hn-favorite-note-taking-software) 从一个线程中收集的。人们的共识是，必须使用一套工具，而不是一个通用的工具，每一套工具都是为不同的任务定制的。

很明显，Evernote 错过了这趟列车，Emacs 的 Org 模式是赢家，还有像黑客攻击纯文本/降价文件和文件夹这样的个人解决方案。

许多黑客依赖其操作系统默认提供的工具，如谷歌 Keep 和 Suite、微软 OneNote 和苹果 Notes。其他人正在使用像 Simplenote，Zim Wiki，conception，Workflowy，颤，Vim Wiki，Notational Velocity 和 Zotero 这样的工具。

有些人甚至仍然使用纸和笔。安奇只被注意过几次。只有半打的人被发现真正钻研这个创造知识的课题。

看起来这些任务和事情都很难，而且与过程中使用的工具高度个人化。

我们都有不同的数据使用模式，这需要高度灵活、可定制的工具和服务，但没有人能在一个包中提供。

一些工具有利于存档、标记、分类，而另一些工具有利于提炼知识、创造知识和记忆事物。有时候搜索比组织更重要。

在这个混乱的顺序中——在最后——发现了一些好的建议。我们可以说，这是一天的普遍规律。

1.  确保组织知识本身不会成为目标。首先获得一个用例，然后开始围绕它组织信息。
2.  区分被动信息和主动信息。档案信息和为你工作的知识信息。
3.  不要过度工程化。
