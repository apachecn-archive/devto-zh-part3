### [](#a-little-perspective)一点透视

在过去的 12 年里，我一直坐在面试桌的两边。遗憾的是，在这几轮谈判中，重点总是放在 Javascript 上，而另外两种重要的语言——*、HTML* 和*、CSS*——通常没有得到同等的重视。

尽管 Javascript 很难作为一个整体来介绍，因为它已经从 DOM 操纵框架内部发展到任何人们想要的东西。更不用说在漫长而寒冷的冬天之后，大量的 Javascript 库和框架像一群兔子一样四处涌现——春天来了！！

抛开所有的笑话不谈，你可以在一次采访中彻底颠覆它，感觉自己掌控了局面，或者在另一次采访中被狠狠地击倒。

整个面试过程都被破坏了，这让求职者和试图招聘合适人才的公司都感到沮丧。

我决定贡献我的微薄之力，我希望能成为解决方案的一部分。我一直在认真阅读数百个常见的面试问题，感觉像是**回顾**和**重构**这些是解决更大问题的关键要素，但我需要你的帮助:)

我将与大家分享我挑选、改编、有时甚至创建的一份精选清单，这些清单不仅与候选人的准备最相关，也与公司对候选人的评估最相关。这显然是非常固执己见的，但有希望在 JS 社区的帮助和投入下变得有价值，也许会成为一个标准。

下面是第一批 4 个，读一读，让我知道你对评论中的全部或部分的感觉。我强烈建议你在评论中提出一些可能的面试问题。请与任何能增加讨论内容或对讨论有益的人分享。

### [](#1-the-famous-fizzbuzz)1。著名的嘶嘶声

**最适合:**初级|高级- **阶段**:第 1 轮|所有回合

##### [](#1a-the-challenge)1A。-挑战

写一个程序打印从 1 到 100 的数字。但是对于三的倍数而不是数字印刷嘶嘶声和对于五的倍数印刷嘶嘶声。对于 3 和 5 的倍数的数字，打印 FizzBuzz

##### [](#1b-the-context)1B。背景

FizzBuzz 挑战并不是 JavaScript 特有的，几乎在每一种编程语言中都是编码面试过程的一部分。这通常是评估候选人基本编程本能的快速检查，但如果面试官决定这样做，也可以在深入知识的评估中进行。

这通常是在屏幕共享时进行的轻量级首次技术面试的一部分。非 JavaScript 程序员也喜欢询问并快速评估您的技术知识和方法。

在一个 Javascript 环境中，期望演示对以下一些或所有概念的熟悉程度:

*   逻辑运算符
*   环
*   虚伪的价值观
*   三元运算符
*   类型强制

### [](#2-array-method-polyfill)2。阵列法聚合填充

**最适合:**初级|高级- **阶段**:第 2 轮|现场

##### [](#2a-the-challenge)2A。挑战

您可能需要将一组数字中的所有项目相加。您一直在关注 JS 中的新内容，但是不确定 **sum()** 是数组原型方法的一部分。

编写一个小程序，将一个数字数组中的所有项目相加，并使用 Array **sum** 函数(如果存在)或使用您的自定义函数返回总数。

```
 // should output 21
[1,2,3,4,5,6].sum();
// should be chainable and also output 21
[1,2,3].concat([4,5,6]).sum(); 
```

##### [](#2b-the-context)2B。背景

在 JS 圈子里，扩展原生对象通常是不被认可的。这应该会激起一场关于这样做的利弊的“健康”辩论。最后，它应该强调候选人防范潜在的现有和未来功能覆盖的意识。我认为这是一个很好的代码筛选问题，也是现场面试的第一个问题。

### [](#3-calculate-employee-gross-pay)3。计算员工总工资

**最适合:**高级|专家- **阶段**:第 2 轮|第 3 轮|现场

##### [](#3a-the-challenge)3A。挑战

作为 Kora 公司发展部的新成员，你的新任务是帮助人力资源部快速计算员工的总工资。以下是每个角色的薪酬模式:

```
const associate = { roleId: 3, rate: 12.5,overtime: 18.75 };
const supervisor = { roleId: 2, rate: 15,overtime: 22.5 };
const admin = { roleId: 1, rate: 30,  overtime: 0 }; // salary no overtime 
```

编写一个**基函数**，它以一个**角色(对象)**、**小时数**(数字)和**加班数**(数字)作为参数，返回雇员的工资总额。使用此基本函数，**创建三个部分应用程序函数**，它们在被调用时将分别计算员工、主管或管理人员的总薪酬。

##### [](#3b-the-context)3B。背景

乍一看，用一个函数或程序来计算总工资非常简单。在这里，当我筛选候选人时，我会密切注意她/他是否花时间阅读所询问的解决方案的特殊性。

我预计会有很多关于基函数和局部应用概念的问题。我也不会要求深入了解**函数式编程概念**，比如**奉承**，但我希望对这些有一个大致的了解。

### [](#4-rolls-of-coins)4。成卷的硬币

**最适合:**初级|高级|专家- **阶段**:第 2 轮|第 3 轮

##### [](#4a-the-challenge)4A。挑战

下班时，Amina 的小费罐里装满了硬币。她需要一些帮助来清点和储存她辛苦赚来的战利品。她想把硬币卷成一卷，这样便于带回银行。

编写一个程序，帮助她快速找出每种面值的硬币有多少卷，以及每种硬币上的剩余部分。

下表列出了每卷每种面额的硬币数量:

| 硬币 | 每卷硬币 | 价值 |
| --- | --- | --- |
| 便士==> 1 英镑 | Fifty | $0.50 |
| 镍==> 5 | Forty | $2.00 |
| 10 美分 | Fifty | $5.00 |
| 四分之一==> 25 | Forty | $10.00 |

你的程序将接受一组未分类的硬币。你可以假设每个硬币要么是 1，5，10，要么是 25。它应该打印出如下所示的消息:

便士:10 卷- 39 左

镍币:25 卷- 0 左

一角硬币:12 卷- 49 左

四分之一:2 卷- 20 左

##### [](#4b-the-context)4B。背景

这是一个轻微的扭曲和变化的 [Socker 商业挑战-黑客等级](https://www.hackerrank.com/challenges/sock-merchant/problem)。

这是一个作为练习的频率计算，因为它可以通过无数种方式接近和解决，这是一个完美的选择，让候选人向我们介绍他们解决问题的方法和迭代过程。

我非常支持“先让它工作，再优化和重构”的方法。我会密切关注整体可读和自我记录的代码。
