# 开发者颂

> 原文：<https://dev.to/mcelhinneyjames/ode-to-developers-536g>

这是很多事情的高潮，

在众多的谷歌搜索中，

许多推特漫谈。

这是很多浏览博客帖子和 YouTube 视频的机会，

筛选例子和演示，

在源代码中爬行，

费力地阅读文件，

非常沮丧，

但是报酬很高。

多啊哈！瞬间，

以及大量的好奇心。

这些我都不是在教室里学的(不是说你不应该学)。

我从来没有拿到过 CS 学位(不是说你不能)。

我从来没有老师(除非你把浏览器算进去)。

我从来没有任何同学(除非你算上其他用户)。

我从来没有一个课程表(除非你把训练营、模拟辩论赛和[开发到](http://dev.to)推荐也算在内)。

我从来没有一个阅读清单(除非你把 Twitter 上的推荐也算上)。

我从来没有一个小组项目(除非你把拉请求也算在内)。

我从来没有进行过班级讨论(除非把懈怠和话语也算在内)。

但是，当然，你**做**计算所有这些。

谁说它必须有四面墙和一块白板才能称之为教室？

谁说他们必须在礼堂展示一副牌才能成为你的老师？

谁说你需要一件文具来证明你学到了东西？

谁说浏览器不是教室？

用户不是你的同学？

推特不是你的课堂讨论？

GitHub 不是你的团队项目吗？

如果“教育是伟大的均衡器”，

那么技术就是这句话的体现。

偏见仍然存在，技术也不能幸免，否则就太天真了。

不考虑知识和专长，有些仍然会被忽视，

被男人，被常春藤联盟，被优等生。

但是技术能做的是，

它确实为你提供了工具，

做一些事情的工具。

如果有人不让你玩他们的沙箱，你可以创建自己的沙箱。

创建自己的沙盒。
