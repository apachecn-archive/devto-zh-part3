# 如何与 JavaScript 生态系统保持同步？

> 原文:[https://dev . to/sergiodxa/how-to-keep-updated-with-the-JavaScript-ecosystem-51fh](https://dev.to/sergiodxa/how-to-keep-updated-with-the-javascript-ecosystem-51fh)

很多人问我，我是如何不断更新所有新的 JavaScript 东西的，我是如何知道要学习什么以及在哪里可以找到信息的。我发现这个问题在开始学习如何编码的人之间很常见，不管是前端还是 JavaScript。行业内起步的人。

这也是为什么我要分享你如何学习该学什么，在哪里可以找到信息。

*最初发布于[https://sdx . im/articles/how-to-keep-updated-with-the-JavaScript-ecosystem](https://sdx.im/articles/how-to-keep-updated-with-the-javascript-ecosystem)*

## [](#find-information)查找信息

有很多地方可以找到信息。我发现我经常去一些地方获取更多的信息。

### [](#mozilla-developer-network)[Mozilla 开发者网络](https://developer.mozilla.org/en/)

也称为 MDN。这个网站是网络技术的文档。如果你想了解一个 HTML 标签，一个 CSS 属性或者任何 JS 相关的东西，你必须去那里。

### [](#caniuse)[CanIUse](http://caniuse.com/)

这是一个非常好的地方，可以检查一个特性在多个浏览器中的支持。它也给你关于那个浏览器使用的信息(个人建议:如果一个浏览器没有超过 5%的使用率就不要支持它)

### [](#specific-documentations)具体单据

通常你只使用特定的技术，你的个人堆栈。经常查看该技术的文档以了解更多信息，通常他们有博客、twitter 账户、Github 组织等。你可以用它来获得更多的知识。

### [](#awesome-lists)[牛逼排行榜](https://github.com/sindresorhus/awesome)

Awesome 是由 sindresorhus 提出的一个想法。它们是关于任何技术的有用链接的列表。主列表(链接在标题中)是其他牛逼列表的列表，如[牛逼-javascript](https://github.com/sorrycc/awesome-javascript) 、[牛逼-反应](https://github.com/enaqx/awesome-react)、[牛逼-redux](https://github.com/xgrommx/awesome-redux) 等等。这个列表对于查找图书馆、文章、演讲、视频等非常有用。等等。任何特定的技术。

## [](#follow-developers)追随开发者

几乎每个开发人员都有一个 Twitter 账户，他们分享许多他们正在从事或关注的有趣项目。最重要的是，他们分享想法并进行讨论，你可以通过阅读学到很多东西。

### [](#devto)[开发到](http://dev.to/)

Dev.to 是一个面向开发者的社区和博客平台。它类似于 Medium，但只用于开发文章。在这里你可以关注标签或人物，阅读大量关于技术和发展的文章和观点。

### [](#personal-blogs)个人博客

我过去读了很多个人博客，我现在仍然关注和阅读一些博客，比如 [PonyFoo](https://ponyfoo.com/) 或 [2ality](http://2ality.com/) 。他们中的大多数都迁移到了 Medium 或 dev.to，这意味着你可以在那里获得更多的文章。

### [](#development-blogs)开发博客

除了 Medium 和 Dev.to，还有更多拥有许多作者的博客，如 [CSSTricks](https://css-tricks.com/) 、 [Codrops](https://tympanus.net/codrops) 、 [Treehouse 博客](http://blog.teamtreehouse.com/)、[NPM 博客](http://blog.npmjs.org/)、[GitHub 博客](https://github.com/blog)、 [SurviveJS](http://survivejs.com/) 、 [Smashing Magazine](https://www.smashingmagazine.com/) 、 [Scotch](https://scotch.io/) 、 [React 博客](https://facebook.github.io/react)或 [Mozilla Hacks](https://hacks.mozilla.org/) 。

### [](#echojs)[EchoJS](http://echojs.com/)

EchoJS 是一个与 HackerNews 有着相同想法的网站，任何人都可以发布有趣的链接，唯一的规则是他们需要与 JS 相关并且是英文的。你可以关注他的 [Twitter 账户](https://twitter.com/echojs)，访问[网站](http://www.echojs.com/)或者关注 [RSS 源](http://www.echojs.com/rss)来分享所有链接。

我发现这个网站是我关于 JS 和相关技术的链接和文章的最大来源之一。你可以有一个单独的精选文章列表，而不是关注很多博客。

### [](#reddit)[Reddit](https://www.reddit.com/)

特别是 [JS 子编辑](https://www.reddit.com/r/javascript/)，以及其他具体的子编辑如 [React](https://www.reddit.com/r/reactjs/) 、[前端](https://www.reddit.com/r/Frontend/)、 [React Native](https://www.reddit.com/r/reactnative/) 、 [Redux](https://www.reddit.com/r/reduxjs/) 、 [GraphQL](https://www.reddit.com/r/graphql/) 等。这和 EchoJS 的想法是一样的，echo js 是一个有管理的链接源。

## [](#watch-talks)观看会谈

我在上面多次提到会谈。观看关于 JS 的谈话和会议，社区是巨大的，我们有许多关于 JS 的会议。JSConf 和特定国家的 JSConf 总是有令人惊讶的对话。

不仅要在 Youtube 上观看演讲，还要帮助他们！很多国家都有 JSConf，比如 [JSConf AR](https://www.jsconfar.com/) ， [JSConf UY](https://jsconf.uy/) ， [JSConf CO](https://jsconf.co/) ， [JSConf US](http://jsconf.us/) ， [JSConf EU](http://jsconf.eu/) 等等！而且不仅仅是 JSConf、 [NodeConf](http://nodeconf.com/) 、 [NodeSummit](http://www.nodesummit.com/) 、 [ReactConf](http://conf.reactjs.org/) 、 [▲ZEIT Day](http://zeit.co/day) 等。

### [](#assist-to-meetups)协助聚会

通常会议每年举行一次。而且只在首都。但是关于任何技术的聚会可以是每月一次的，以辅助聚会。许多城市都有聚会，如果你所在的城市没有聚会，你可以开始！带上其他开发者，开始在你的城市里聚会。

## [](#participate-in-communities)参与社团

有很多社区，他们通常有一个免费的 Slack 团队，你可以加入。 [▲ZEIT 社区](https://zeit.chat/)、 [ElmLang 社区](https://elmlang.herokuapp.com/)、[react flux(react、RN、redux、GraphQL、Jest、Relay 等。)社区](https://www.reactiflux.com/)等等。许多开发社区都有自己的 Slacks、Discord 或 Gitter 聊天室，你可以加入。

开始参与可以帮助你与创建你所使用的库和框架的人以及与你处于相同位置的人交流。

Youtube 上有很多频道，其中一些与开发有关。你可以关注的一些很酷的渠道有[funfunfunfunction](https://www.youtube.com/channel/UCO1cgjhGzsSYb1rsB4bFe4Q)、[脸书开发者](https://www.youtube.com/channel/UCP_lo1MFyx5IXDeD9s_6nUw)、[网飞 UI 工程](https://www.youtube.com/channel/UCGGRRqAjPm6sL3-WGBDnKJA)、[谷歌开发者](https://www.youtube.com/channel/UC_x5XG1OV2P6uZZ5FSM9Ttw)和 [node.js](https://www.youtube.com/channel/UCQPYJluYC_sn_Qz_XE-YbTQ) 。

## [](#follow-and-participate-in-github-repositories)关注(并参与)Github 资源库

你可以关注和参与许多 Github 库，这将让你了解你日常使用的技术的未来。您甚至可以参与发送 Pull 请求，报告或发现他们问题中的 bug(希望还有 bug 的解决方案),他们是获得关于您使用的框架或库的更多见解的绝佳场所。

## [](#conclusion)结论

有很多方法可以跟上技术的发展。记住，开发(尤其是 JavaScript 和前端)是一条不断学习的道路。你总是会学到越来越多的东西，这是没有尽头的。所以，拥抱它，继续学习吧！
