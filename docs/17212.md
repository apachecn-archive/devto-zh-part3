# 什么是导师制，作为潜在学员的发展会员计划是什么样的，可能是什么样的

> 原文：<https://dev.to/tiagomagalhaes/what-is-mentorship-and-what-the-devto-membership-program-look-like-as-a-potential-mentee-and-what-could-look-like-533b>

我们都是开发者。我们编码，我们用我们的机器让事情发生，我们帮助把想法变成现实。我们为他人(和他人一起)工作，或者为自己工作。有时两者都有。在我们做事情的过程中，有时我们手忙脚乱，我们伸出手去解决自己的问题，有时帮助别人解决问题。

这种帮助/被帮助的社会活动可以被定义为指导。这可能是最简单的定义，根据 macOS 中的字典，导师制被定义为:

> 导师提供的指导，尤指公司或教育机构中有经验的人:他因其导师和解决问题的能力而受到员工的尊敬。
> 
> *   一个人接受导师指导的一段时间:在一家科技初创企业中与一名企业家进行为期两年的指导。

后一个定义的最后一部分谈到了持续时间。所以它是可变的。因此，我们可以适应所有那些 1 分钟的错误修复情况，那些围绕遗留应用程序的漫长“旅程”，或者前端和后端开发人员之间的会谈，以了解之前达成的关于前端的 API 协议将在两周前与后端对话。因此，就导师将指导学员的情况的复杂性而言，导师制可能有所不同。

这些情况可能发生在办公室环境中，甚至在为公司远程工作时。但是当我们作为一名自由职业者或者兼职时——项目或者潜在的创业想法——会发生什么。我们求助于网络。我们看文档，教程，QA 社区网站(我正在看你)，Slack 和 discord rooms，Repo issues 等。

这些都是或多或少与传统定义紧密相关的导师形式。当我们在 SO 上发布一个希望得到答案的问题时，我们正在寻找一个熟悉该技术的人，为解决您的问题提供一些有用的指导。或者您想使用另一种类型的数据库并查看“入门”指南。

但是，如果我们需要比上一个例子更高层次的指导，我们去哪里找呢？

我们可以在 Craiglist 上寻找产品，或者在谷歌上寻找一些平台，让人们展示自己是导师，让其他人寻找他们。当导师列出他们的专业领域或他们使用的技术时，这可以进行定制，从而允许潜在的学员进行筛选。

关于这一点，我将作为一个没有填写申请的潜在学员谈论“发展到导师”计划，以及如何改进该计划，如果这个 **beta** 计划能够/将会使**稳定**。

关于我作为一名开发人员的生活，我有几个问题充斥着我的大脑，一个是关于我的能力，另一个可能是关于我自己工作的附带项目。我可以在以导师为导向的平台上寻找顾问或导师。但 dev.to 还提供为导师和学员牵线搭桥的服务。我认为 dev.to 社区是一个受欢迎和有益的社区。我可以把自己列为学员。

当我读到这一页的时候。有一些表格，一个登记我作为导师，另一个作为学员，这两个表格都有一些有用的描述，说明在这些表格中应该填写什么。但是在我读完之后，我可以使用一些指导的主题不符合那些描述。这与工具或技术无关。如果我仍然提交申请，是否有人可以就我的需求提供指导，或者我可以盲目提交？

我想讨论的是，国家对发展会员计划的态度如何，以及如何改进该计划，以更清楚地了解它能为社区提供什么，因为它可能会增加导师、学员和访问者。
