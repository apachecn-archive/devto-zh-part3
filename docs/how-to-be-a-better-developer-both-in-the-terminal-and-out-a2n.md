# 如何成为一个更好的开发者(终端内外都有)！

> 原文：<https://dev.to/robertfish/how-to-be-a-better-developer-both-in-the-terminal-and-out-a2n>

我想先给每样东西都加上前缀。这都是主观的，我自己的看法。无论你在哪里看，都会有一些共享的技巧和新的，所以要有所保留。

## 三思而后行

一个简单但有用的提示。由于时间限制(相信我，我也有过这样的经历)和缺乏前瞻性思维，这条建议被忽视了很多，但丑陋的事实是。如果没有计划，那些截止日期将会更快地逼近你，所以提前想一想，稍微计划一下，你需要做什么，以及这件事将如何与其他事情一起工作。

## 环视四周

当你看着一个按钮切换某个东西时，想的很简单。另一方面，用户的想法恰恰相反。这是因为你必须考虑产品是如何制造的，以及产品将如何被使用。如果互动是笨拙的和不可用的，那么你没有做好你的工作。

## 编写解决问题的代码，而不是制造问题

这是一个相当简单的概念，但我们很多人都犯了这个错误。我们编写的代码解决了一个问题，但同时也产生了三个问题。对于每一行代码，想想这是如何解决问题而没有任何影响的，如果你从这个练习中出来而没有给别人或你自己带来麻烦的话。那你就做对了你的工作。

## 将它记录下来，人们就会跟随它

我们很多人都讨厌写文档。我知道你在想什么。无聊，没用，浪费时间。我明白，但同时，你知道什么是好产品吗？很好的文档！如果你的产品可以在你的文档中很容易地解释自己，那么更多的开发者会采用它并使用它。可以是几条注释，也可以是解释一个函数的整句话。不管是什么，你都在帮助你的开发伙伴理解你的代码，这让他们的生活变得更容易，你也可以放心地知道你已经编写了高质量的代码和文档。
