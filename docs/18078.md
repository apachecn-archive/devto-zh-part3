# 建筑产品-从零开始

> 原文:[https://dev.to/gonedark/building-products-从广场开始-0-3217](https://dev.to/gonedark/building-products---starting-from-square-0-3217)

最近我一直在准备打造一款名为 [Optionality](https://optionality.app) 的 SaaS 新产品。我想记录下这个过程，不仅仅是讨论代码，还包括产品决策和产品开发中的市场营销。

所以在这第一篇文章中，我想从头开始。这很重要，因为我很少从写代码开始。作为开发人员，当我们想到一个想法时，我们会迷失在技术细节中。我们开始预架构、横向扩展服务器，并选择最新的技术堆栈来构建这一切。

虽然这当然很有趣，但你不能马上开始写代码。人们，不仅仅是开发人员，而是所有人，团队，公司都直接投入到建设中。这是一个错误。

事实是你所拥有的只是一个想法。真的没什么。你的大脑中最多只有几个孤立的离子。你必须一点一点地把这个想法带到这个世界，一路证明和塑造。

比如有了[得到 Git](https://gettinggit.com) ，我在录制视频课程之前就领导了会议培训。如果我没有在多个会议上给所有听众讲过这个，我就不会上这门课。事实上，我原本计划录制第二个课程，但在继续这个想法之前，我正在等待第一个课程的回应。

凭借我的书 [BaseCode](https://basecodefieldguide.com) ，我在会议上发言，但也做了一系列[推文](https://twitter.com/i/moments/948577473742233600)来缩小主题。这些推文是证明这个想法的一种方式，相对而言，获得了大量的喜欢和转发。

在那里，我创建了一个邮件列表。我想是贾斯廷·杰克逊说的，如果你没有 500 个订户，你可能就没有好主意。其他人如[内森·巴里](https://nathanbarry.com/)和[亚当·瓦森](https://twitter.com/adamwathan)不断地谈论登陆页面对建立你的观众群的重要性。这是 *BaseCode* 的关键，并且无疑导致了比*获得 Git* 更成功的产品。

现在，这个新的 SaaS 将更具挑战性，因为它是一个不同的行业。自从我开始编程以来，我就一直在投资股票市场。在过去的几年里，我进入了期权交易领域。我注意到的是缺乏投资组合跟踪和分析的工具。

经纪人在非常基础的水平上提供这种服务，但是他们的工具更侧重于交易。作为一些投资社区的成员，似乎大多数人都使用某种电子表格——这是一种糟糕的手工体验。

所以我确定了这个差距。但同样，这并不是证据*如果我建造了它，他们就会来*。我只是证明了一种需要。就这一点而言，最好的想法是从满足个人需求开始的。如果我不会用它，其他人也不会用它。你必须愿意*吃你自己的狗粮*。如果没有，你就不会用心去建设它，你也更有可能放弃。

现在，在我跑去我的办公室建造这个之前，我想证明市场更多。为了做到这一点，我需要增加我的观众。我创建了一个单独的 [Twitter 句柄](https://twitter.com/TraderJMac)。因为穿越，我立刻获得了大约 100 名追随者。这些开发人员也分享了我对投资的热情。然而，我需要培养我真正的投资者观众。

我开始每周写线程推文。不幸的是，它没有增加我的追随者，因为，嗯，这是一个新帐户，没有人知道我到底是谁。我的新计划是利用这些线程化的推文，添加一些图表和图像，将它们转化为我可以在 [Optionality](https://optionality.app) 上发表的文章。然后我会在媒体、投资社区等交叉发布。

记住，我没有观众。所以我甚至还没到可以开始向潜在用户验证我的想法的地步。我甚至不在*广场 1* ，我在*广场 0* 。我必须首先努力扩大观众群。这需要在深入编写代码或构建这个投资组合跟踪平台之前发生。这并不意味着我们不会写任何代码。但它不是你可能认为的应用程序代码。事实上，我需要编码的第一件事是一个博客来保存这些文章。

我要用拼图游戏来做这件事。这是一个静态站点生成器。没什么特别的，只是把 markdown 转换成 HTML 并把它放在一个站点模板中。没有理由安装一些沉重的软件或手工编码我自己的博客。我总是想做最简单的事情。什么都是 MVP。

可以说，最简单的事情就是在*媒体*上发布一个投资标签。但是，我需要开始建立一个搜索引擎的存在。乐观地说，当我做这个产品时，我希望人们被引导到[选择性](https://optionality.app)，而不是*媒介*。因此，虽然直接使用*媒体*并链接回 *Twitter* 会更简单，但以很少的额外成本集中内容是有价值的。

另一点是我本可以使用[杰基尔](https://jekyllrb.com/)或 [GitHub 页面](https://pages.github.com/)。本质上，Jekyll 是 Ruby，Jigsaw 是 PHP(带有 Laravel 的暗示)。虽然我对 Jekyll 很熟悉，但一般来说，我更熟悉 PHP 和 Laravel。

这是很重要的一点，因为开发人员经常试图将新想法与新技术结合起来。这增加了不必要的复杂性，最终降低了你成功的概率。你不仅在学习你的想法，你也在学习技术。

最后，我平衡了 MVP 方法，同时注意到每一次迭代都让我走在通向最终目标的道路上。不一定每次迭代都能让你更快或完全地到达那里，但是它应该让你保持在这条路上。

现在，在我开始这一步之前，我想再明确两件事。首先，我的时间线。时间表非常重要。它们强制实施问责制并强化 MVP 方法。你不可能永远达到你的目标。第二，我需要列出我的目标的可衡量的目标。我在尝试做什么，我如何验证它的工作。

对于 *Square 0* ，我的目标是增加我的观众。我会给自己 4 周的时间来做这件事。我计划通过以下可衡量的目标来实现这一目标:

*   每周博客文章交叉发布到媒体、Twitter 和投资社区。
    *   **子目标:**在媒体上发布联合广告。
*   根据 Google Analytics 推荐人和内容报告，增加 *optionality.app* 的流量。
*   将推特粉丝增加到 250 人。

我会说，有些是一种延伸。但是我必须从某个地方开始，让自己对测量负责。因此，我们将在 4 周内看到这些目标是否达到，我应该继续开发我的产品。
