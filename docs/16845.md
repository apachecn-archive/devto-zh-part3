## 查找信息

有很多地方可以搜索信息。虽然有一些我想咨询的时候总是拜访的。

### [Mozilla 开发者网络](https://developer.mozilla.org/en/)

也称为 MDN。该网站基本上是 web 技术的文档。如果您要阅读有关任何 HTML 标签、CSS 属性或任何与 Js 相关的内容，这就是位置。

### [CanIUse](http://caniuse.com/)

一个很好的地方来回顾多个浏览器对某项功能的支持。您还拥有有关每个浏览器使用情况的信息(个人推荐:如果浏览器的使用情况不超过 5%，请勿支持该浏览器)。

### 具体文件

通常一个人使用某些技术，你*个人栈*。查看这些技术的文档以了解更多总是很重要的，他们通常有博客、Twitter 账户、GitHub 组织等，可以用来了解更多信息。

### [利塔斯牛逼](https://github.com/sindresorhus/awesome)

Awesome 是由[信噪比](https://github.com/sindresorhus)发起的一个想法。它们是关于任何技术的有用链接的列表。主列表(标题链接)是其他 awesome 列表的列表，如 [awesome-javascript](https://github.com/sorrycc/awesome-javascript) 、 [awesome-react](https://github.com/enaqx/awesome-react) 、 [awesome-redux](https://github.com/xgrommx/awesome-redux) 等。这些列表对于查找有关任何技术的书店、文章、讲座、视频等特别有用。

### [EchoJS](http://echojs.com/)

echoJs 是一个想法与 HackerNews 相同的网站，任何人都可以发布有趣的链接，唯一的规则是它们必须是关于 js 和英语的。你可以跟随他的[Twitter 账户](https://twitter.com/echojs)，访问他的[网站](http://www.echojs.com)，或者跟随他的[RSS](http://www.echojs.com/rss)feed，了解他们共享的所有链接。

我意识到这个网站是我关于联署材料和相关技术的链接和文章的主要来源之一。与其跟踪一堆博客，倒不如给你一个治愈的文章列表。

## 会看演讲

我在上面提到过几次谈话。他会看关于 JS 的演讲和讲座，社区是巨大的，我们有很多 Js 讲座。 [JSConf](http://jsconf.com/) 除了各国的具体情况外，总是有精彩的演讲。

仅[在 YouTube 上观看演讲](https://www.youtube.com/user/jsconfeu)我参加了！很多国家有很多，如 [JSConf AR](https://www.jsconfar.com/) 、 [JSConf UY](https://jsconf.uy/) 、 [JSConf CO](https://jsconf.co/) 、 [JSConf US](http://jsconf.us/) 、 [JSConf EU](http://jsconf.eu/) 等！而不仅是 JSConf，还有[no conf](http://nodeconf.com/)、[no summit](http://www.nodesummit.com/)、[reactionconf](http://conf.reactjs.org/)、[day](http://zeit.co/day)等。

### 去开会

会议通常每年举行一次。而且只限于首都。但是任何技术的会议都可以每月举行，我参加了会议。很多城市都有集会，如果你的城市没有一个可以发起。和其他开发者一起，开始在你的城市举行会议。

## 我跟着(并将参加)Github 存储库

您可以跟踪和参与许多存储库，这样做有助于您了解您每天使用的技术的未来。您甚至可以通过发送拉式请求、报告或查找错误(并希望找到这些错误的解决方案)以及它们的问题来参与其中，这是了解您使用的框架和库的更多信息的好地方。

## 结语

有多种方法可以保持最新。记住，开发(更具体地说是 JavaScript 和前端)是一条不断学习的道路。你会越来越学越来越多。**拥抱变革，继续学习！**
