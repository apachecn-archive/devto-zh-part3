# 香农·肯德尔论从表演到编码

> 原文:[https://dev . to/samjarman/Shannon-Kendall-on-acting-to-coding-159m](https://dev.to/samjarman/shannon-kendall-on-acting-to-coding-159m)

Dev Chats 第二季又回来了！在这一系列中，我大约每周都会与一位出色的开发人员或技术人员交谈。你可以在这里阅读更多。如果你知道下一个我应该和谁聊天，请告诉我。

### [](#introduce-yourself-who-are-you-where-do-you-work)自我介绍！你是谁？你在哪里工作？

我叫香农·肯德尔。我是一名纽约的开发人员，在转行做编程之前，我在电影和电视行业做了七年的演员。我目前在 Lifion by ADP 工作，是一名应用程序开发人员。

### 是谁或什么让你开始编程的？

我真的很喜欢表演。我决心继续比赛，直到成功。然而，七年后，我开始认识到，我在演艺圈呆的时间越长，我就越觉得自己离我最爱的东西越远。我意识到，作为一名演员，日复一日的经历让我深感不快。当我意识到不快乐的来源时，我决定是时候改变职业了。我要确保它能让我每天都开心。

我知道我想进入 STEM 领域，我想以一种非常真实的方式(而不是“当我最终足够出名时”)成为变革的一部分。我来自一个对科技友好的家庭；我父亲是硬件方面的计算机工程师，我哥哥是游戏方面的技术艺术家。当我告诉他们我正在考虑转行学习编程时，他们都很鼓励我，并给我发来了免费在线 javascript 课程的链接。我试了一个，马上就迷上了。从那以后，一切都是连续不断的运动。

### [](#acting-to-programming-a-bit-of-a-change-how-have-you-found-the-transition)演技到编程，有点变化！你是如何找到转变的？

进入办公室工作对我来说有点冒险。对我和我的 ADD 来说，习惯一天参加多个会议是一个挑战。还有很多事情是我没有预料到的，比如和我的经理定期进行一对一的谈话。我从未做过我的经理关心我的担忧或我认为我做得如何的工作。我仍然觉得绩效评估是最奇怪的过程之一。

就这份工作而言，我真的很喜欢。我喜欢在一个专注的团队中工作——我总是在合作中工作得最好。我非常幸运地被安排在一个完全由女性开发人员组成的团队中，我们都有完全不同的解决问题的方法。我非常感激我所有的表演训练，它教会了我当有人和我交流时，不仅仅是听话语。但我最感激的是每天都有挑战我的事情，给我当所有的事情都在一起时的那种奇妙的感觉。

### 你在富栈学院的经历如何？你会推荐新兵训练营作为职业过渡的方式吗？

绝对的。训练营真的很难，要想被录取，你必须证明你有足够的技能从那里出来准备工作。但是我已经准备好改变我的生活，我不想为了获得另一个学位而负债累累。

我被 Fullstack Academy 的格蕾丝·赫柏项目录取，这是一个与他们常规的全日制沉浸式课程平行的课程，但只对女性和非二进制个人开放。该项目包括延期付款计划，允许学生在找到工作前延期付款。我非常喜欢在那里的时光——以至于我写了两篇关于那里的文章中的一篇。

当它真正归结到整个训练营的辩论时，对我来说真正重要的是编码训练营使技术职业变得容易。他们给那些原本没有机会在这个行业找到一席之地的优秀人才提供了机会。正因为如此，它们对科技行业有利。它们是急需的视角的来源。他们给这个行业注入了新的想法和经验，而这些正是这个行业所缺乏的。我花了七年时间找到技术作为职业，我不知道如果没有训练营的帮助，我是否会成功。

### 你认为科技行业在未来几年会发生怎样的变化，这会对你的角色产生怎样的影响？

这是一个大问题。我在这个行业的时间还不够长，不足以反思变化的模式。当一个行业的基本技能成为学生课程的一部分时，这个行业会发生什么？

这太疯狂了。这个行业让我惊叹。我真的不担心*是否*会改变，因为科技行业*正在*改变。它正在挑战我们认为可能的边缘。它是认识到我们的懒惰是极其聪明的。它是运用理性的、批判性的思维来创造性地解决我们的问题，不管这些问题有多小。我就是为此而来的。

### 迄今为止，在你的软件生涯中，你学到的最难的一课是什么？

科技界有很多人对新人很苛刻。关于 stackoverflow 上的人有多可怕的笑话的模因数量说明了我的观点。我有一部分相信，为了证明我实际上是一个软件开发人员，为了被认为不仅仅是一个新奇的女演员变成的“开发人员”，我需要他们的认可。我觉得我需要知道一切，甚至获得入学资格。我不知道软件世界到底有多大，也不知道任何人不可能知道所有的事情。我觉得我需要吃饭，呼吸，做梦软件来赶上进度。

但我没有。我花了一些时间对自己的技能建立信心，但是在我不自信的时候，我仍然在创作。我变得自信，因为我看到自己一遍又一遍地做这件事，每一次，我都比上一次做得更好。你总会犯错，但你只有停止前进才会失败。

### 对于成功的软件职业生涯，你的第一条建议是什么？

接受自己的错误。它永远不会停止发生，因为你是一个人。作为一名开发人员，您花费了大部分实际开发时间来诊断您做错了什么。但也要接受你的想法是错误的。解决一个错误的想法是你找到正确想法的方法。作为开发人员，我们经常出错，以至于很难与有问题的人合作。

### 工作之外，你有什么爱好吗？你认为它们对你的科技事业有任何帮助吗？

我有如此多的爱好。我真的很喜欢做东西——食物、配件、架子。我喜欢制作我送给别人的礼物，因为这对我来说更有个性，所以随着时间的推移，我涉足了很多爱好。我最近学会了编织，因为我可以在通勤时随身携带，所以我不会每天玩两个小时的糖果粉碎游戏。我发现它真的很放松，是我焦虑的一个很好的出口。我喜欢尝试新的烘焙技术，或者组装一个梳妆台。我对事物的运作方式和原因很好奇，一旦我足够理解了，我也很喜欢发表自己的见解。成为一名创造者肯定会影响我在工作中处理项目的方式，因为它让我不断磨练每个部分如何融入并影响整体的感觉。

### 你会向那些想走上与你相似道路的人推荐哪些书籍/资源？

我真的很喜欢看 youtube 教程，当谈到学习一个新的发展概念。当我想了解 JavaScript 的新进展，或者更新一个我很久没用过的概念时，FunFunFunctions 是我的首选渠道。我还看了很多[网络忍者](https://www.youtube.com/channel/UCW5YeuERMmlnqo4oq8vwUpg)如何一步一步地演示如何使用库(我比训练营的其他同学领先了一步，因为我已经看了他关于使用 Node.js 的前 30 个左右的教程)。

如果你只是想试水，我在学习 JavaScript 时尝试了几个免费教程。对于非常初学者，推荐可汗学院的 JavaScript for Animation 教程。这是我对 cod 的介绍，没有什么比看到你的代码让一个愚蠢的小飞船穿过屏幕来确认你做了这件事更好的了。如果你已经掌握了窍门，如果你能负担得起每月 30 美元的费用，我绝对推荐 CodeSchool。课程的视频和练习册风格让人感觉非常透彻。然而，如果这不是一个选项，我建议将 Codecademy 的免费课程与 Youtube 上的视频结合起来。由于 Codecademy 只是文本，当他们完成练习时，有另一种解释来更全面地掌握这个概念会非常有帮助。

最后，大声喊出来！你想让读者去看什么？

这可能看起来有点跑题，但我上下班时最喜欢听的播客之一是关于农业的，由我的表妹莎拉·泰伯主持。它叫做[农场到泰伯，](https://www.farmtotaber.com/)，这本身就是一个听下去的极好理由。我不知道她有一个播客，直到我终于有了一个 twitter 账户，并开始关注她。她拥有土壤科学博士学位(她总是这样称呼这个学科)，并且已经在农业领域工作了大约十年。我不知道农业会如此迷人。她才华横溢，风趣幽默，能够在她采访的每一位客人身上找到农业和工业之间的直接联系或相似之处。这不是什么硬科技，但我发现她对农业影响的观察为我们今天在科技世界看到的东西提供了惊人的洞察力。
