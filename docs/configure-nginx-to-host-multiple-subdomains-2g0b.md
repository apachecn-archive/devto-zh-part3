我假设你在 linux 机器上，你知道 nginx 的意思。

首先，安装 nginx -

```
sudo apt-get install nginx 
```

从上面你可以归纳出以下几点-

*   这告诉 nginx“嘿，我认为你应该这样配置服务器”
*   `listen 80` -翻译为“监听端口 80，这是 web 客户端的默认端口”
*   `root /var/www/html;` - nginx 知道可以提供来自该位置的 HTML 文件
*   `index index.html index.htm index.nginx-debian.html;` -这表示这类文件应该按照优先级的降序(从左到右)提供服务。
*   `server_name _;` -这表示`_`是服务器名。假设我们的域名是`example.com`，服务器名将是`example.com`。这是因为当收到对`example.com`的请求时，nginx 应该决定是否处理它。
*   `location / {}` -这意味着所有请求都必须由这个服务器配置来处理。还有一个处理特定请求的选项。例如，以`/server_1`开始的请求可以加载`server_1.html`，所有其他请求可以加载`default.html`。

下一个明显的问题是-我如何配置子域？

没那么复杂。它包括 7 个步骤-

*   复制默认配置，并将其重命名为`subdomain1.example.com`。

```
cp /etc/nginx/sites-available/default /etc/nginx/sites-available/subdomain1.example.com 
```

Enter fullscreen mode Exit fullscreen mode

注意:不一定要给文件命名为`subdomain1.example.com`。如果你遵循一个让每个人都容易参与的惯例，事情会变得更简单。

*   打开它，去掉`listen 80`旁边的术语`default_server`和它下面的线。这是至关重要的。
*   将`server_name`更改为`subdomain1.example.com` &更改正在使用的 HTML `root`
*   然后将该文件链接到 nginx 的站点启用目录中的一个文件。这是为了让 nginx 了解配置中启用了配置`subdomain1.example.com`。

```
sudo ln -s /etc/nginx/sites-available/subdomain1.example.com /etc/nginx/sites-enabled/subdomain1.example.com 
```

*   在`/var/www/html`中为`subdomain1.example.com`编写一个 HTML 文件
*   重启 nginx 服务器
*   如果你使用的是像 Cloudflare 这样的 DNS 提供商，用 A 记录`subdomain1.example.com`把机器的 IP 加进去。如果没有，那么将机器的 IP 和 subdomain1.example.com 添加到/etc/hosts。
*   在浏览器上加载`subdomain1.example.com`来检查它是如何加载的。如果一切正常，您将看到您为这个子域编写的 HTML。

为您需要的任何其他域或子域扩展相同的方法。
