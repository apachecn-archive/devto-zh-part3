# 领先程序员的有点傻的历史——第 3 部分

> 原文：<https://dev.to/justzeros/a-slightly-silly-history-of-leading-programmers-part-3-4je5>

欢迎来到我们故事的中场休息。

([这个故事的第一部分在这里](https://dev.to/justzeros/a-slightly-silly-history-of-leading-programmers-15d8)

第二部分到了。)

除了生物课间休息，让我们停下来想想我们为什么在这里。

我是来帮助提高我们的技术管理和领导技能的。

你在这里有许多不同的原因，现在有许多你(超过 5300！).如果我认为你们都是出于同样的原因来到这里，那我就太傻了。

更愚蠢的是，认为你们每个人都在同一个地方，需要听到同样的建议，或者吸取同样的教训。

你们每个人都是独立的个体，就像我一样。

很容易认为自己是这个团队的“领导者”，但事实上，我只是一个拿着麦克风的人。

今天，拿着麦克风的人不知道接下来的故事该何去何从。

我知道，当领导者被难住时，他们不应该承认。但我确实是，所以为什么不承认呢。

**我们开始这趟旅程的原因**

你看，我可以想象这个故事从这里转向许多不同的方向，每一个转向都带来不同的教训。

所以让我回到我写这个故事的最初原因，我还没有与你分享。

当我问自己“马库斯，我想知道为什么管理程序员这么难，为什么管理对程序员来说这么难？”

我想到的一个答案是，管理科学和实践对计算机的影响和复杂性毫无准备。

也许计算机，尤其是计算机人，打破了传统的管理方法。

我有点喜欢这个想法——我们打破了这个系统。也许这就是我叛逆的一面。

**我们摆弄它，直到把它弄坏**

这也可能给我们一个提示，为什么管理程序员是困难的，以及为什么管理对程序员来说是困难的。

也许管理程序员如此困难是因为…他们讨厌被管理。他们可能不介意被领导，只要他们相信领导者。

而且，仅仅是可能，管理对程序员来说如此困难，因为…嗯，同样的原因:他们讨厌被管理。他们发现传统的管理理念野蛮、野蛮、专横。

我们可能会想象，电脑人走进古老的、令人肃然起敬的大教堂，贴上“管理”的标签，在地基上放置炸药，然后炸毁大楼。相当革命性的想法，对吧？

然后，我们骄傲地站在废墟周围，环顾四周，问道:“好吧……现在，它的位置在哪里？”

发生了一次集体耸肩，我们回到了编码上。_(ツ)_/

我必须承认，我的大部分工作都是在向旧体制投掷更多的炸药，将那些想法击得粉碎。

这很好……但是我几乎没有做什么来代替它。

这个故事，以及在哪里拍摄的选择，向我揭示了这个模式。令人惊讶的是，生活从最意想不到的地方给我们上了很多课。

这涉及到了程序员(以及设计师、开发人员、UX 人、产品所有者和一般的“制造者”)难以管理的另一个原因:**如果我们已经摧毁了传统的管理，它的位置上还剩下什么？**

通过写这篇文章，这真的是与我自己的对话，我现在明白了，我想要建立新的想法，而不是简单地推倒旧的。

我不太清楚故事将会如何发展，但这是我前进的方向。

**先睹为快**

我先给你一个提示:我认为答案是“领导力”，一个叫做“关系型领导力”的想法对我来说效果最好。

感谢你们加入我的旅程，并在幕间休息时一直坐着。
