毫无疑问，如果你想成为一名优秀的(可雇佣的)开发人员，你需要知道如何写代码。这是显而易见的——你不会雇佣一个对汽车一无所知的汽车修理工，对吗？

然而，除了编写干净的代码之外，还有其他技能可以将优秀的开发人员与伟大的开发人员区分开来。来自 freeCodeCamp 的这篇文章概述了程序员不需要写代码就能掌握的一些重要技能，但是我想补充一些我自己的。

### 谷歌搜索技巧

开发者经常在网上查找资料。即使是拥有多年经验的资深开发人员，也会花费大量时间在互联网上搜索答案或信息，无论是在文档、论坛还是搜索引擎中。

知道如何搜索以找到你想要的东西是一项重要的技能，可以帮助所有水平的开发者。像 web 开发中的大多数事情一样，您的“谷歌技能”会随着经验的增加而提高，但是您可以做一些事情来加快这个过程，更快地找到正确的答案:

*   在你要查找的内容前加上“mdn”。例如，如果您想知道遍历数组中各项的最佳方式，可以尝试搜索“mdn 遍历数组”这将从 [MDN Web Docs](https://developer.mozilla.org/en-US/) 返回一堆结果，这是所有 Web 开发的首选资源。MDN 文档对初学者来说可能有点吓人，但是请相信我——随着您向成熟的程序员迈进，这些文档将成为无价的资源。
*   无论你搜索什么，你几乎肯定会得到一个来自最大的开发者在线社区 [Stack Overflow](https://stackoverflow.com) 的结果。这是一件好事，也是一件坏事。好的一面是，您可以从全球数百万开发人员的集体知识中受益。坏的一面是，为了找到适合您的用例的东西，您将不得不费力地通过一些不相关或无效的解决方案。
*   有些开发人员对此不屑一顾，但我一直认为 W3Schools 是一个有用的资源，尤其是作为一种解码 MDN 上一些更复杂文档的手段。我推荐使用大多数教程中包含的交互式示例。它们非常适合以实际操作的方式反复强调家庭概念。

### 策划技巧

作为开发人员，您参与的每个项目都应该包含某种程度的规划，以便为成功绘制路线图。即使你随着进展调整路线(你几乎肯定会)，拥有仔细规划初始路线的技能有助于使你作为开发人员的生活变得容易得多。

规划也不一定意味着在开始编码之前要多次迭代线框和模型。它可以简单到在纸上勾画出你的想法，然后花一些时间思考如何用代码执行你的计划。

克里斯·费迪南迪，香草 JS 学院的负责人，[就是这么做的](https://gomakethings.com/how-to-plan-out-your-javascript-project/)。“在我打开文本编辑器或浏览器之前，我会在纸上计划好我的脚本。它帮助我考虑全局，在我陷入实现代码所需的具体的*方法*和*策略*之前，仔细考虑我的代码的*逻辑*。”

克里斯·费迪南迪在写一行代码之前，先在纸上规划好他的脚本。

### 决断技能

这一点很重要，尤其是对于努力掌握复杂编程概念的初学者。别搞错了，*学习如何编码是很难的*！要成为一名优秀的开发人员，你需要强大的决心技能来帮助你坚持度过那些困难时期(相信我，会有很多困难)。

有时候，我会花上几个小时，甚至几天时间来解决一个我无法解决的问题。我在谷歌上查找可能的解决方案，搜索 MDN 的文档，阅读博客文章，测试东西，失败，询问同事，测试他们的建议，再次失败，离开一两天，回到问题，测试更多的东西，失败，测试，失败，测试…

这就是成为一名 web 开发人员的意义所在。你需要坚持项目和解决棘手问题的决心，直到你最终偶然发现你一直在寻找的解决方案。这可能需要你 5 分钟、5 小时或 5 天的时间。没关系。重要的是你知道，不管需要多长时间，你都会有最终找到答案的决心。

### 商业技能

无论你是一名独立的自由职业者还是一个庞大的软件开发团队的一员，商业技能都是非常有价值的。有很多优秀的开发人员和工程师写出了完美的代码，但是却没有任何商业头脑。他们不能批判性地思考他们正在构建的产品或功能，这意味着他们不能有效地传达他们的想法、关注点等。营销人员、经理和利益相关者。

对于开发人员来说，这是一个巨大的错失机会，他们可以让自己成为团队和雇主不可或缺的一员。建立一个坚实的商业技能库——沟通、谈判、网络、营销和销售、领导力、金融知识、客户服务等。-为了补充你的编程技能，你会走上工作保障的快车道吗(可能会更快)。

我认为，科技行业中真正的独角兽不是全栈开发人员或设计师/开发人员，而是那些可以毫不费力地在编写代码、给团队成员起草内部电子邮件、为即将推出的功能开发新的促销文案以及回复客户咨询之间穿梭的人。我的朋友们，那是一只独角兽。
