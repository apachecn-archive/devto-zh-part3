# 杂志是面向代码新手的开放教育资源

> 原文:[https://dev . to/Daniel Harding/zines-as-open-educational-resources-for-code-newbies-2he 8](https://dev.to/danielharding/zines-as-open-educational-resources-for-code-newbies-2he8)

创建开放教育资源(OER)是测试或更新你的知识同时帮助他人学习的好方法。

在 10 月份的 mozz fest(T1)上，我有幸顺道参观了 T2·éléonore(T3)和 T4·马尔文(Malwine)的杂志展(T5)。我错过了他们的会议，但他们启发的迷你杂志展示了任何主题，从复杂的数据科学到城市指南，都可以分解成小规模的独立出版物。你所需要的只是一张纸，一支笔和一个分享的话题。

液体错误:内部

几个月后，经过几次测试，创建一些与编码相关的东西似乎是一个不错的主意。改变现有资源的用途效果很好，数字化设计意味着它们可以快速调整或更新，同时还可以在线分享。

液体错误:内部

忠于“杂志精神”意味着避免重量级的 DTP 工具、学习曲线或需要图形设计学位(尽管如果你做了，你可能会创造一些令人敬畏的杂志)。你只需要谷歌幻灯片、PowerPoint 或任何其他能让你组合基本形状、文字和颜色的软件包。

更重要的是，给你的作品添加一个开放许可(例如 [Creative Commons](https://creativecommons.org/share-your-work/) )会给其他人一个机会来将你的想法重新混合或发展成他们自己的东西。尤其是当它以一种易于编辑的格式发布时。大家都赢了！

通过与更有经验的开发人员交谈，很容易忘记同时学习这么多新事物有多难。但是即使作为一个初学者，我仍然能够编写一个 CSS Grid 入门指南，其中涵盖了一些基本原则。这就是杂志的魅力所在，新手和专家都可以看。
