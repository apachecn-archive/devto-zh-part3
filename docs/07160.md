自从几年前第一次听说开源的 Apache Kafka 服务平台以来，我就被它迷住了。部分原因是为了实现它设定的目标——让微服务互相对话，但也是因为它的名字。

为什么取名卡夫卡？这确实是我感到好奇的事情。当一个软件库或平台走得比常规更远，选择了一个更不寻常的名字时，会有某种阴谋感。在这种情况下，卡夫卡这个名字远远超出了日常生活的琐碎，进入了一个超现实的文学领域。

我全心全意地欢迎任何与软件和技术无关的事物的联系，比如阿帕奇卡夫卡的文学世界。作为一名软件工程师，我工作了十多年，有时开始把软件和技术看作是一种静态的、非人性化的数据和算法空间，在那里没有什么事情发生，平庸和平凡的事物蓬勃发展。我并不是指用反乌托邦式的技术奇点的方式去人性化，用人工超级智能控制世界。我的意思是以一种纯粹现代的方式，作为一个在软件领域谋生的人，这个领域通常缺乏人类的互动。

根据[维基百科条目，](https://en.wikipedia.org/wiki/Apache_Kafka)Kafka 平台以作家 Franz Kafka 的名字命名，这是因为，作为一个为写作而优化的系统，创作者认为使用作家的名字是合适的。此外，Kafka 是一个很好的软件库的名字——简短，有趣，容易记忆。但是 Kafka 也恰好是一个非常合适的名字，对于一个用消息队列和事件操作的系统来说，原因不仅仅是 Franz Kafka 是一个作家。

碰巧的是，他对信息和交流有一种特殊的亲近感。1917 年，卡夫卡写了一篇名为《来自皇帝 的 [*消息》的短篇小说，其中一个主题被委托在一个垂死的皇帝的床边传达一条紧急消息。在这篇不到一页长的寓言中，信使是一个“不知疲倦的人”，不知疲倦地穿过人群，然后是楼梯和庭院，一次又一次，几千年来，从未到达目的地。*](https://www.nybooks.com/daily/2011/07/01/message-emperor-new-translation/)

信息也是卡夫卡其他一些作品的主题，如《流放地的[】中的](https://en.wikipedia.org/wiki/In_the_Penal_Colony)，一个残酷的机器将信息刻在一个囚犯的背上，或者《T2》中的，故事发生在一个超现代的纽约市，在那里交流是故事的核心。

到现在为止，您可能已经得出结论，这篇文章不会深入研究 Apache Kafka 软件平台的工作原理。如果你是一名软件工程师，当你正试图更熟悉这个系统的时候，你碰巧在读这本书，你可能会把这整个故事看作是某种文学上的胡说八道，与你要找的东西完全无关。如果你是一个以文学为导向的人，你可能同样会失望，尽管你很可能会小心不要买票去看卡夫卡峰会。

现在你可能会感到有点恼火，甚至可能会感到厌恶。如果我告诉你我从未使用过阿帕奇卡夫卡，那就更是如此。我使用 RabbitMQ 已经有一段时间了，我相信这是一个竞争对手的平台，但我不是这方面的专家。就此而言，我也不是弗朗兹·卡夫卡文学作品的专家。

[随笔——在卡夫卡的阴影下——挥手还是溺水:卡夫卡与意义——BBC 的声音](https://www.bbc.co.uk/sounds/play/b05tq3rg)

另一方面，翻译兼牛津大学现代德国文学教授卡伦·里德(Karen Leeder)是公认的专家，她在 2015 年为 BBC 广播 3 台(T3)撰写的一篇关于 T2 的文章中写道，卡夫卡确实写了很多信，他对这些信是否会到达目的地充满疑虑。这与软件平台 Apache Kafka 有些不同，Apache Kafka 尽一切可能发送这些消息，并具有复杂的机制来保证它们的到达和确认。

但是弗兰茨·卡夫卡会怎么看待自己的名字被一个软件平台使用？毫无疑问，他会觉得这非常有趣，因为他在晚年确实发现了自己的作品。虽然不是特别滑稽，但其中一些确实具有超现实或荒谬的一面，可以被视为有趣。或者也许他会认为整件事是卡夫卡式的。如果你仍然不相信，可以考虑一下*皇帝*的最后一句话，这句话是突然针对读者的:“*然而，当夜晚来临的时候，你坐在你的窗前，梦见这条消息”。*
