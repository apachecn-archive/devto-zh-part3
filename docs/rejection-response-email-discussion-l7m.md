# 拒绝回复电子邮件讨论

> 原文：<https://dev.to/kevindsteeleii/rejection-response-email-discussion-l7m>

我最近收到了一个公司拒绝我后写的回复的反馈。我会粘贴信的相关部分，而不会暴露公司或好心给我反馈的人的身份。

> 凯文，
> ...
> 
> 我们发现你拥有广阔的技术领域——很明显，你对技术的热情让你学到了很多东西，但我们发现这种广度可能是以深度为代价的。不仅重要的是要知道我们如何使用某个东西，无论是框架、语言还是库，还要知道通过什么模式以及底层技术如何真正发挥作用——当事情不像它们应该的那样工作时，这具有巨大的价值。
> 
> 我认为你正在解决这个问题——我发现你使用 Node/Express 而不是 RoR 令人印象深刻，我知道这不是 Flatiron 教你的。我将专注于使用这一堆栈的最佳方式，并查看正在出现的不同模式(容器化微服务、MVC 设计、Restful 原则等)。这种深度将作为一种核心知识很好地服务于你，并给你更大的背景，让你融入更多的广度。

在这一点上，他说再见，并写下他的名字，这将违反互联网授予的匿名，所以我不会张贴这些信息。他很友好地给了我这个建议，现在我必须为此做些什么。但是什么？任何建议，无论是视频，讲座，书籍阅读等。非常欢迎。任何关于这类事情的个人经历或故事也很棒。我不应该是唯一从中受益的人。
