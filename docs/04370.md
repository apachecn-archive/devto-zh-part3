# 设计技巧如何帮助开发人员更好地为产品做出贡献

> 原文:[https://dev . to/param Harrison/how-design-skills-help-developer-to-contribute-better-to-the-products-5b C9](https://dev.to/paramharrison/how-design-skills-help-developer-to-contribute-better-to-the-products-5bc9)

UI 和 UX 设计技能被开发人员所忽视，尤其是在大型组织中。在 UI 和 UX 设计的早期，通过有效的反馈来构建更好的产品和帮助产品团队是一项重要的技能。

## [](#feedback-early-on-usability)可用性早期反馈

开发人员是第一批用户，他们在将设计付诸实施之前先查看设计，并有机会对任何模型的整体可用性做出贡献。

### 什么是对用户界面设计有用的反馈

*   排版问题以及子内容与标题内容分离的问题
*   色彩对比问题。关于文本颜色和背景颜色重叠的反馈。色盲是一个真正的问题，很多人都有不同程度的色盲，所以要确保用户界面能自己说话，即使颜色只是灰色、黑色和白色。
*   组件响应的可用性，提供更简单的解决方案/想法，如果有的话
*   总是要求设计师首先提供小屏幕的反馈模型，然后是大屏幕，而不是相反。它有助于简化用户界面，而不是用没有可用性的功能来束缚页面
*   如果设计中包含文案，与文案相关的问题。如果一个开发者第一眼看不明白这个副本，那么用户很可能也不会明白。确保副本总是简单的
*   就布局和间距问题提供反馈
*   检查主要活动和次要活动。如果设计没有区分主要动作，就要引起关注。
*   检查定制组件，看看是否有任何简单的替代标准 HTML 组件的样式。
*   UI 的不同状态。设计不需要覆盖所有状态，但应该覆盖一些默认状态，如空状态和完成状态。其他中间状态可以在开发过程中导出。

### 对于用户界面设计来说，什么是最没用的反馈

*   像素问题。我们经常走极端，看到像素问题，并认为我们需要坚持实物模型的精确像素。不，没有这样的规定。往往一个 500px 的容器和 520px 的容器在大屏幕下看起来差别不大。
*   根据你在开发中使用的东西，画一个假想的网格系统，并让所有的设计都为这个网格服务。网格被高估了，不要总是认为 12 列或 16 列是标准的设计方式。它只是在开发过程中定义的一个有用的工具，可以轻松地创建 UI。设计总是与网格值保持一致，只要这样使用网格，给设计师更多的自由，也让你自己远离无用的规则。

## [](#what-to-learn-in-design-skills-from-developer-perspective)从开发者的角度来看，在设计技巧中需要学习什么

*   排版——不同屏幕的字体大小、粗细和行高
*   不同屏幕的间距和大小
*   颜色，如何在不同的 UI 上下文中正确使用颜色
*   可读性和可访问性最佳实践
*   图像和图标的使用。由于现在越来越多的设计是图像和图标重，你应该学习如何利用不同的图标，图标字体和图像对比度，图像颜色等最佳做法。

希望这有助于你了解如何更好地做出贡献，并作为一名开发人员在设计过程的早期给予反馈😎
