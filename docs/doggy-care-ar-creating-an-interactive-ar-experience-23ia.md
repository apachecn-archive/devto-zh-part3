# 狗狗护理增强现实:创造互动增强现实体验

> 原文：<https://dev.to/higherpixel/doggy-care-ar-creating-an-interactive-ar-experience-23ia>

# 简介:头脑风暴

在项目开始时，我从一开始就知道我想使用我的新三星 S9 手机作为目标平台来创建一个软件应用程序。与我以前的手机相比，这是一个巨大的升级，但最重要的是，它支持 AR。增强现实将现实世界与数字世界相结合，为用户创造独特的体验。在 YouTube 上看了几个狗的视频后，我开始考虑养一只狗。当时我养不起狗，所以我决定退而求其次，创造一只我可以照顾的 AR 狗。我开始记下我在成长过程中玩的游戏，这些游戏有一个很酷的宠物模拟。像任天堂、电子鸡和神奇宝贝让我们去皮卡丘和 Eevee 这样的游戏具有用户可以与虚拟宠物互动的机制。

这个项目背后的目标是制作一个互动游戏，用户可以在手机上照顾虚拟宠物，并使用 AR 技术将其投影到现实世界中。我认为 Unity 是这个项目的最佳选择。Unity 是一个游戏开发软件，可以为包括 android 在内的许多不同平台创建游戏。谷歌有一个用于 Unity 的 SDK，叫做 AR Core。这用于使在 Unity AR 中创建的项目能够运行。这个项目被分成 4 个冲刺阶段。前两次冲刺是为了原型化想法和奠定基础，一次冲刺是为了固化机制和实现 UI 元素，最后一次冲刺主要是修复 bug 和写这个报告。除了最后一次，每次冲刺都是两周。最后一次冲刺总共 7 周的进度只有一周。

# 冲刺#1:原型设计的第一阶段

现在想法已经在脑子里了，目标也开始了，我在 Unity 中创建了一个新项目，并将其标记为 Creature Care，并从 Google 开发者网站导入了 Google AR Core SDK。它包括一个名为 Hello AR 的示例场景。场景中有一个 AR 控制器，这样当玩游戏时，它会用手机的摄像头绘制出环境，并在摄像头收集的点上放置一个 3D 平面。在这里，我添加了一个脚本，用于在用户触摸到检测到的平面时生成狗屋，并在一段设定的时间后，在检测到的平面上生成狗，以给出真实世界中显示的虚拟宠物的效果。在这个阶段，狗模型只是一个 3D 球体，狗屋是一个 3D 盒子。我也给了 3D 平面草的纹理，所以它看起来像草。下一步是让狗在飞机上移动。我使用了一种光线投射，如果手指触摸与检测到的平面对齐，则光线从手机射向检测到的平面。这个 sprint 的最后一部分是为移动命令、激光笔命令创建一个基本的 UI，并为测试创建一个低多边形狗。激光笔命令与移动命令完全一样，只是狗会移动到手机在检测到的平面上指向的位置。如果狗在动或站着不动，它会播放动画。该应用程序的基本功能已经实现，并完成了原型的第一部分。

# 冲刺#2:原型开发的第二阶段

基本的基础是在上一个 sprint 中设置的，所以现在在这个 sprint 中，目的是创建另一只具有更好模型的狗，这样可以使用更多的动画，狗可以更加流畅。如果用户离它足够近，并且在靠近它的时候摩擦手机屏幕，这只狗现在可以成为宠物。下一个添加是选择狗和房子的缩放比例。这个游戏有两种缩放模式。一种是桌子模式，允许在小范围内看到狗，而另一种是房间模式，允许在大范围内看到狗。

增加了新的交互，例如在手机指向的方向发射一个球的按钮，让狗捡起球并带到狗屋将球还给主人，在房间模式下手机位于房间模式下的按钮和在桌子模式下位于房子前面的按钮，让狗吃东西。还为狗实现了一个按钮来执行翻转。这些按钮位于 UI 的侧边栏上。随着所有这些功能的实现，原型阶段即将结束，现在的重点是完成 UI 和实现狗的健康系统。

# 冲刺#3:打磨时间

在这个 sprint 中，目标是教玩家如何与狗互动，所以重点是 UI 和演示。在狗的每一个动作之后，在屏幕的顶部都有一个 UI 滚动条，告诉玩家接下来会发生什么。与其说它在移动，不如说它在追逐一束激光、一个球或食物，跑马灯会告知用户这个动作。与此同时，用户界面得到了急需的改进，所以符号更容易理解。这些附加功能让用户清楚地知道该做什么以及游戏中正在发生什么。

狗也接受了健康系统。通过移动狗，狗最终会累，并会开始哭。如果用户在一定时间过去后没有抚摸狗，狗就会开始哭。这是玩家为了维护和照顾狗而必须记录的两个数据。当狗哭的时候，眼泪的粒子系统出现在狗身上，当狗抚摸的时候，心的粒子系统出现在狗的上方。用户也可以创建一只狗，选择颜色和选择它是否有斑点，并在主菜单上给它一个名字。我还在这次冲刺中加入了音效和音乐。现在这已经完成了，项目的其余部分主要是修复 bug 和准备结束。

# 冲刺#4:最后冲刺

这个冲刺阶段主要包括修复 bug，从校园内不同的人那里获得反馈，以及准备发布演示。游戏演示将在我的 Itch.io、谷歌 Play 商店和 GitHub 上推出。在校园里四处走动向人们展示游戏时收到的反馈对我来说是一个巨大的推动。观看人们玩游戏并与狗互动对我来说是一个巨大的推动，我能够发现像狗被卡在探测到的飞机下，球扔出去后消失，以及其他小的奇怪的错误。我得到了很多有用的反馈，并能够添加有用的附加功能，如音乐静音按钮和宠物指示器，让玩家知道他们是否足够接近宠物。此外，当游戏处于房间模式时，玩家可以激活狗屋，并将其变成一个玩家和狗都可以在其中活动的大房间。利用人们的反馈，我能够按时完成项目，并在演示发布后创建一个我想添加的东西的列表。

# 失败:没有按计划去做什么

在这个项目中，我经历了两个主要问题。一个是在 Unity 中设置 IBM Watson 语音到文本 SDK 的失败。我们的目标是获取手机麦克风中所说的内容，通过使用语音到文本转换器，我可以说出命令并将其翻译成文本。如果翻译的文本是像移动到这里，翻转，滚动等关键词。，狗会执行给定的命令。我不能让 SDK 在手机上工作，但我可以让它在 Unity Editor 上工作，因为我找不到修复它的方法。我不得不割掉它。

另一个主要的失败是狗模型。在这个项目中，我在 blender 中制作了所有的 3D 模型。我做的狗模型没有最好的索具或骨骼位置，所以一些为狗设计的技巧被删除了，比如翻身、装死和便壶。虽然它没有包含在这个演示的发布中，但是一个新的狗模型将会在未来的更新中制作。当手机连接到电脑时，我在测试手机上的应用程序时遇到了问题。我手机的显示屏会出问题，我不得不重置手机和电脑来解决这个问题。这只是一个恼人的错误，困扰着我整个应用程序的创建。

# 未来更新:游戏的下一步是什么

现在试玩版已经发布了，我可以开始考虑有哪些新特性是我希望在未来看到的。我心中的一个目标是增加更多种类的狗和颜色。目前只有一种狗有几种颜色，新的狗和颜色的加入将允许更多的定制。一个新的狗模型带来了更好的动画和更好的狗身体，所以这允许狗有更多的方式向用户传达感情。这将允许我带回在项目失败时讨论过的剪辑动画的想法。我将回顾测试人员给出的反馈，并在未来制定新的目标。
