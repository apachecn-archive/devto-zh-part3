# 开发者训练营结束后要做什么

> 原文:[https://dev . to/kev _ mcg/what-to-do-after-the-developer-boot camp-4p ca](https://dev.to/kev_mcg/what-to-do-after-the-developer-bootcamp-4pca)

在编程训练营学习可能是一种旋风般的体验——这是一种快节奏、富有挑战性和身临其境的环境，推动你进入技术职业的世界。完成一门课程后的一段时间是利用学习动力培养习惯的重要时间，这些习惯将长期维持你的编程实践——只要专业开发人员在改进他们的技术，他们就会一直这样做！这里有三个步骤可以让你马上迈出正确的一步。

## [](#learn-something-completely-different)学习完全不同的东西

利用训练营的动力，继续自己的探索。开始学习一门你在课程中根本没学过的语言或框架。你会惊奇地发现，与你刚开始学习编程时相比，这种体验是多么的不同。你仍然会陷入困境，有很多问题，但你会有一层基本技能，可以加速这个过程，加深你的知识。许多技能在不同的框架和语言之间转移，所以开始探索一些新的东西，看看你已经走了多远，作为一个学习者你有多独立。如果这与你真正想做的个人项目有关，效果会更好，因为这会给你动力，让你专注于在可能性的海洋中接下来要学什么。

学习不应该局限于在学校的几年或训练营的几个月——继续学习是开发人员成功职业生涯的必要条件。bootcamp 体验中最重要的部分不是你学习的特定语言或框架，而是无论你的技术探索将你带到职业生涯和个人项目的什么地方，你都可以使用的编程基础。所以马上使用它们，并获得另一个领域的经验！

## [](#connect-with-other-developers)联系其他开发者

确保你有办法定期花时间和训练营体验之外的其他开发者在一起。成功和持续学习的关键是花时间和其他人一起做同样的事情。训练营的环境为你提供了这些机会，但在课程结束后，如何将这些机会融入你的生活取决于你自己。这可能意味着聚会、喝咖啡时的信息采访、社交媒体(Twitter 和 Slack 是与其他开发者交流的好地方)或者与你的导师/老师/训练营的其他学生保持联系。作为一名导师，我喜欢听学生们在训练营结束后学到了什么！

此时很容易出现冒名顶替综合症——你可能会觉得自己并不适合或者不能有效地与其他开发人员交流。我和其他开发人员交谈过，我完全听不懂他们在说什么！最好的办法是完全坦诚地说出你在哪里。你可以这样说，“我刚刚完成了一个训练营项目，在那里我学到了 X、Y 和 z。现在我正在探索 A、B 和 C，我正在寻找与 foo、bar 和 baz 相关的机会。”插入这些变量，无论你在和谁交谈，他都会对你的经验水平和你下一步想学的东西有一个很好的了解，这使得他们更有可能提供足够的支持和建议，把你和能帮助你下一步的资源和人联系起来。

## [](#share-what-you-learn)分享你学到的东西

将分享融入你的日常生活。写下你正在学习的东西，寻找帮助新开发人员的机会，并在对话中分享你的观点。可能感觉不到，但是你已经比许多想学习 web 开发的人更有经验了，很快你就会成为这个房间里更有经验的开发人员之一，人们会向你寻求指导。现在就养成这个习惯——它将加速你自己的学习，并随着时间的推移建立更多的联系。

* * *

这三个习惯是开发者无论在职业生涯的哪个阶段都必备的技能。如果训练营是一次短跑，那么这些慢跑习惯会让你不断前进和学习。因此，选择一个项目，尝试一下，并在聚会上告诉我们所有关于它的事情！
