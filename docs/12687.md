# 无服务器词汇表

> 原文：<https://dev.to/paulswail/serverless-glossary-3a9g>

这个词汇表提供了在构建基于服务器的应用程序时使用的术语和短语的定义。在我们开始之前，如果你不确定术语“无服务器”是什么意思，请先阅读[这个](https://winterwindsoftware.com/serverless-definitions/)。

这些是我个人的定义，受我最熟悉的生态系统(如 AWS)的影响。它们不一定代表通常理解的定义。

## API 网关

AWS 提供的完全托管服务，允许开发人员创建、发布、维护、监控和保护 API，这些 API 可自动与 Lambda 和 SQS 等其他 AWS 服务集成。[(官方网站)](https://aws.amazon.com/api-gateway/)。

## 构架师框架

一个开发框架，用于使用定制的高级声明性语法在 AWS 上构建和部署无服务器应用程序。[(官方网站)](https://arc.codes/)。

## Azure 函数

微软的 FaaS 产品。[(官方网站)](https://azure.microsoft.com/en-us/services/functions/)。

## 云原生

一种构建完全由云供应商提供的服务和基础设施组成的软件应用程序的方法，而不是使用内部资源。

## 云端开发

一种工作流，开发人员在云中运行他们正在进行的代码，作为他们标准开发流程的一部分，而不是在完成开发任务之前只在本地计算机上运行。例如，开发人员在云 AWS 帐户中部署和调用 Lambda 函数。在构建无服务器应用程序时，这种工作流程几乎是强制性的，但在更传统的应用程序中并不常见。一个优点是，它迫使开发人员在几乎与早期生产环境相同的环境中运行他们的代码，减少了集成错误的风险。缺点是在开发人员可以测试功能之前，当前工具在部署功能方面的时间开销，以及开发人员不能离线工作的事实。

## 冷启动

第一次调用云[函数](#faas)时出现的现象，导致它比后续(热)调用花费更长的时间来完成。这个预热时间是由几个因素造成的，包括云提供商提供托管功能的底层[容器](#container)所需的时间，以及将任何特定于功能的依赖项加载到内存所需的时间。

冷启动是所有 FaaS 提供商的通病，也是 FaaS 不适合持续要求极低延迟的工作负载的常见原因。

## 配置管理

(由开发人员和 ops 工程师)在应用程序中指定、版本化和分发组件资源的配置属性的实践。

## 容器

> “容器是一个标准的软件单元，它将代码及其所有依赖项打包在一起，以便应用程序能够快速可靠地从一个计算环境运行到另一个计算环境。” ( [来源](https://www.docker.com/resources/what-container))

在无服务器环境中，云提供商使用容器来提供和运行[功能](#faas)。一个容器被重用来服务多个函数调用，并且通常保持运行几个小时(尽管不能保证)，以最小化[冷启动](#cold-start)的影响。

## 部署框架

一个开发框架，使得[配置](#configuration-management)和部署云资源更加容易，比如[功能](#faas)，API 和事件触发器。

📖**相关: *[为什么无服务器新手要用部署框架](https://winterwindsoftware.com/serverless-newbies-should-use-framework/)。***

## DynamoDB

由 AWS 提供的完全托管的 NoSQL 数据库服务。
它的主要优势集中在其内置的[受管扩展](#managed-scaling)、安全性、备份/恢复和高吞吐量上，这消除了操作传统数据库带来的大量管理任务。

DynamoDB 是 AWS 专有的，因此不能在其他云平台上使用。
[(官方网站)](https://aws.amazon.com/dynamodb/)。

## 事件驱动架构

一种架构模式，软件系统通过这种模式对自己的事件做出反应并创建自己的事件。这种模式几乎是无服务器系统的现成模式。例如，AWS Lambda 使您能够配置不同类型的事件(例如，添加到 S3 存储桶的文件、计划的 CRON 作业)，这些事件会自动触发函数的调用。通常，云提供商管理用于监听事件的钩子，开发人员负责将这些钩子连接到动作(例如 Lambda 函数)。

## FaaS

“功能即服务”是一种在云中运行定制代码的机制，无需首先提供专用服务器来托管代码。FaaS 函数非常类似于更一般的编程意义上的函数，因为它将输入作为参数(通常是事件负载)，并允许开发人员执行他们的自定义代码，并将响应发送回调用者。

FaaS 功能不需要服务器进程持续运行，这使得云提供商可以独立地自动扩展它们。

FaaS 的实现包括 [AWS Lambda](#lambda) 、[谷歌云功能](#google-cloud-functions)和[微软 Azure 功能](#azure-functions)。

术语“无服务器”经常被错误地与 FaaS 混为一谈，但前者是一个超集，包括非定制代码托管服务。

## 全托管服务

完全托管的服务(在无服务器云环境中)是由云供应商提供的服务，提供特定的功能集，其运营问题(如供应、修补和扩展)全部由云供应商管理，并从开发人员处抽象出来。AWS 生态系统中完全托管服务的例子有 S3、DynamoDB 和 Cognito。

## 功能并发

在负载下， [FaaS](#faas) 函数需要被并发调用。并发调用将导致[冷启动](#cold-start)，因为云提供商需要启动一个新的底层容器来服务每个新的并发执行。AWS 对函数数量的软限制是 1000(实际数字因地区而异),在限制任何后续调用之前，它将允许并发运行这些函数，并将错误返回给调用/触发服务。

📖**相关:*[λ缩放计算器](https://winterwindsoftware.com/lambda-scaling-calculator/)。***

## 谷歌云功能

谷歌的 [FaaS](#faas) 产品。[(官方网站)](https://cloud.google.com/functions/)。

## 基础设施为代码

作为代码的基础设施是使用代码文件管理与软件系统的基础设施(服务器、网络、托管服务等)相关的所有配置的学科，这些代码文件可以存储在版本控制系统(如 Git)中。

它的实践有几个好处，包括允许使用一致的配置轻松设置多个管道[阶段](#stage)，促进灾难恢复，以及在将变更集应用到生产环境之前对其进行同行审查。

IaC 实现的例子包括 AWS CloudFormation 和 Terraform。

## λ

AWS 的 [FaaS](#faas) 产品。[(官方网站)](https://aws.amazon.com/lambda/)。

## 托管缩放

托管服务的一种属性，工程师无需提前为服务调配特定级别的容量，因为云提供商会为他们进行扩展。具有可管理伸缩性的服务示例有 Lambda、API Gateway 和 DynamoDB。

有些云服务确实提供了扩展挂钩，但不是自动的，因此需要工程师做一些容量规划，以便在需要时触发扩展，例如 AWS Elasticache、AWS ElasticSearch 服务。在这方面，可管理的扩展可以被视为一个频谱，而不是一个离散的状态。

## 微服务

微服务是一种软件架构方法，通过这种方法，系统被划分为独立部署的小功能块，并且通常通过 HTTP 相互通信。

在 FaaS 应用中，默认情况下您几乎可以获得微服务架构，因为部署单元(功能)本来就很小，可以独立部署。

📖**相关:*[AWS 的无服务器微服务模式](https://www.jeremydaly.com/serverless-microservice-patterns-for-aws/)。***

## 巨石

单片软件系统在一个大型的单一部署单元中维护应用程序的所有功能。

使用 FaaS(例如，一个处理所有 API/web 应用程序请求的通用函数)开发整体应用程序是可能的，但这种方法并不常见，通常被认为是不好的做法。它与微服务架构相反。

## 多云

多云是一种在同一分布式应用程序/工作负载中使用来自不同供应商的云服务的架构方法。

这种方法的主要好处是减少对任何单一云提供商的依赖，并降低围绕[供应商锁定](#vendor-lockin)的风险。这种方法的一个主要缺点是，不同云上的类似服务之间缺乏标准化，导致实施成本高。

## PaaS

平台即服务是云服务的一个类别，它允许开发人员将其应用程序部署到云中，而无需管理部署代码的基础架构。这类似于无服务器应用程序，但与 PaaS 的主要区别在于，云提供商实际上提供虚拟服务器来托管这些应用程序，因此应用程序的可扩展性受到限制。另一个关键区别是 PaaS 服务的定价模式往往是每小时[一次](#paying-for-idle)而不是每次使用[一次](#per-usage-pricing)。

PaaS 服务的例子有 AWS Elastic Beanstalk 和 Heroku。

## 按使用量定价

每次使用定价是指根据云客户使用特定服务的次数而不是固定的时间段来收费。例如，AWS Lambda 根据每个函数执行的次数和持续时间收费。

📖**相关: *[如何计算将一个 EC2 app 迁移到 Lambda](https://winterwindsoftware.com/ec2-to-serverless-migration-calculator/) 的计费节省。***

## 为闲置买单

为闲置资源付费是云客户在使用 EC2 等服务时必须做的事情，这种服务根据用户的资源对服务请求进行计费，即使它当前没有接收任何流量。

## 机密管理

机密管理是[配置管理](#configuration-management)的一个子集，涉及如何安全地存储机密(如密码、API 密钥、数据库连接字符串)并将其分发给系统内的资源。

## 无服务器应用模式

SAM 是 AWS 开发的代码为 T1 的基础设施框架，专门关注无服务器应用。

## 无服务器应用程序库

AWS 提供的存储库，允许开发人员发现可重用的无服务器应用程序(或应用程序的组件)并将其部署到他们自己的云帐户中。
[(官方网站)](https://aws.amazon.com/serverless/serverlessrepo/)。

## 无服务器框架

无服务器框架是一个代码为框架的[基础设施，专门针对无服务器应用。它允许开发者使用 YAML 或 JSON 来配置功能和其他资源，并将它们部署到 AWS、Azure 或谷歌云。](#infrastructure-as-code) [(官方网站)](https://serverless.com)。

## 无服务器优先架构方法

“无服务器优先”是一种默认情况下通过使用 FaaS 和云服务来构建软件应用的方法，除非出现需要使用基于服务器的系统来满足特定功能或运营需求的限制。

## SSM 参数存储

这是 AWS 提供的一项服务，用于帮助[配置管理](#configuration-management)。它提供了一个集中的键值存储，允许安全地存储基础设施和应用程序配置数据(包括机密)，并通过基础设施作为代码(在部署时)和应用程序代码(在运行时)进行访问。

## 阶段

阶段是整个软件系统的独立部署，代表系统发布周期中的特定点。这个术语经常在自动化持续集成/持续交付管道的上下文中使用，并且有时被称为“环境”。阶段的例子有开发、测试、试运行和生产。

## 步进功能

Step Functions 是 AWS 提供的一项服务，它提供了一种编排由多个 Lambda 函数或云服务事件组成的通用工作流的方式，而无需编写管道代码来处理工作流状态和重试逻辑等。[(官方网站)](https://aws.amazon.com/step-functions/)。

## 总拥有成本

软件应用程序的 TCO 是在其生命周期内构建和运行它所需的所有成本的总和。它包括支付所有工程师(开发人员和运营人员)和支持人员的费用，以及云账单和任何其他运营费用。

要考虑的成本应该是实际实现的成本，以及给定具体设计决策的未来成本的潜在风险。

📖**相关: *[你在想无服务器成本都错了](https://theburningmonk.com/2019/01/you-are-thinking-about-serverless-costs-all-wrong/)。***

## 无差别的重物搬运

无差别的繁重工作是一个术语，通常指耗时和/或需要昂贵技能的工程活动，但这些活动不会使公司在其业务的关键领域与竞争对手区分开来。这种繁重的工作通常被视为开展业务的必要成本，包括管理内部或云中的服务器等任务。

📖**相关: *[在一个没有服务器的世界里消失的担忧](https://winterwindsoftware.com/concerns-that-serverless-takes-away/)。***

## 供应商锁定

供应商锁定是指软件应用程序的设计使得架构的某些组件是为特定的云提供商设计的，并且会产生与转移到另一个云提供商相关的巨大成本。

与传统的基于服务器的应用程序相比，无服务器应用程序的风险略高，因为它们倾向于更多地使用专有云服务和数据库，而这些服务和数据库无法在另一个提供商的云上完全复制。

📖**相关: *[FaaS 和厂商锁定——真正的担忧还是 FUD？](https://winterwindsoftware.com/faas-vendor-lockin/)。***

## 工作量

在云提供商上运行的具有特定功能和操作需求的软件工件。工作负载在某种程度上与术语“应用程序”同义，但更为概括，不包含任何面向用户的组件(例如 UI、API)。
