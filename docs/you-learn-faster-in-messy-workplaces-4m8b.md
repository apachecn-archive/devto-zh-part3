# 你在混乱的工作场所学得更快

> 原文：<https://dev.to/marek/you-learn-faster-in-messy-workplaces-4m8b>

作为一名开发人员，我很高兴能与一些令人惊叹的团队在一些漂亮的项目上合作。他们的入职流程很好，文档很完整，有一整套自动化测试、持续集成和代码审查。

这种项目是一种乐趣。

但这不是这篇文章的内容。

我做过一些完全相反的困难项目。没有测试，没有自动化，文档在 6 个月内没有更新，没有人知道发生了什么变化，因为没有拉取请求或代码审查。

这种情况通常有多种典型原因。通常是来自管理层的压力或在最后一刻发生的不切实际的范围变更。一些团队试图使用 Scrum 过程，并成功地进行了冲刺和每日站立会议，但这并没有让事情变得更好。

根本问题不仅仅是缺乏过程，而是典型的缺乏资源和过度依赖技术债务。

## 最佳实践在真空中毫无用处

我对那些糟糕的项目没有最好的记忆，但同时我意识到我学到了很多重要的教训。

我学到了一些东西，在我从事组织良好的项目时，我甚至没有意识到这些东西是重要的。

当我们从理论上思考“最佳实践”时，很容易在没有现实世界压力和限制的真空中思考。但在现实中，每个项目都是施加于其上的不同压力之间的权衡和妥协的集合。永远不会有无限的时间和资源。你必须做出许多艰难的选择。

大多数技术债务并不是因为无知或无能而发生的。这是因为选择，因为外界的限制和压力，必须做出牺牲。

## 为创业公司工作

创业公司是一个可以找到很多经验教训的好地方。我曾为一些初创公司工作过，其中很多现在已经倒闭了。我认为这对我来说是一个幸运的结果，因为它让我看到了作为一名企业家应该做什么，不应该做什么。

我见过创业公司内爆的各种原因:资源管理不善、创始人纠纷和戏剧性事件、对目标市场缺乏了解、等待发货时间过长。

有趣的是，在我见过的案例中，软件开发本身似乎是最不可能导致失败的原因。客户几乎从不关心代码。他们关心你为他们解决的问题。

与此同时，在我工作过的那些确实成功的初创企业中，虽然我获得了许多关于创业的洞察力，但我没有学到多少深刻而艰难的教训。

可能是因为创业成功的方式真的只有一种。做一些为真正的客户解决真正问题的东西。但是失败的方式有很多。有很多错误的方法，所以有很多潜在的教训。

## 做一块海绵

我喜欢把这看作是学习的海绵原则。

在工作中学习的最好方法是什么？吸收你能吸收的一切。像海绵一样。

海绵在哪里吸收最多？在潮湿脏乱的地方。有东西吸收的地方。

相比之下，想象用海绵在干净干燥的台面上擦拭来吸收任何东西。你不会从中得到什么的。

做一块海绵，从你周围发生的事情中学习。不仅仅从你自己的工作中学习:向你的同事、向你的经理学习。向公司创始人学习。了解市场营销和客户支持领域的动态。

# 如果有疑问，赶紧离开

最后一件事。所有这些都需要提醒。不要在一个糟糕的工作场所呆太久。能走就走。如果你再也学不到任何东西，一定要离开。

小心那些谈论公司忠诚度或者告诉你在换工作前应该等待 X 时间的人。除了工作/工资交易，你不欠雇主任何东西。如果很糟糕，就离开那里。倦怠从来都不值得。
