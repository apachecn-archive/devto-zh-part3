四月刚刚结束，意味着夏天要来了！德克萨斯州的四月意味着大量的雨水，夏天第一个炎热的日子，以及每条高速公路旁盛开的蓝帽子。对于兰花来说，这也是一个相对缓慢的月份，但许多真正酷的东西也即将在这里绽放！

这是一份关于兰花的月度时事通讯，兰花是 JVM 最新最好的静态站点生成器。越来越需要让社区了解有关兰花的最新消息，在这里我将分享兰花在上个月的进展！跟随本系列，了解兰花的最新特性，跟踪 Github 上的采用情况，并了解谁在使用兰花！

# Github 上的[](#on-github)

截至发稿时，兰花在 Github 上的排名是 196 位。离下一个 200 颗星星的里程碑还有 4 颗星星，你会帮助我们到达那里吗？

# [](#whats-new)有什么新鲜事？

兰花现在的版本是 0 . 16 . 10，这个月发布了一些小的更新。

*   Orchid 不再包含任何 Lombok 代码，为了更稳定的代码库和更好的将剩余 Java 代码转换为 Kotlin 的路线图。
*   现在支持相对基本 URL
*   添加了`baseUrl`和`homepageUrl`过滤器，以便在格式化临时 URL 时更加宽容
*   更新到 Pebble 3.0.9，其中包括我为该项目贡献的 [`embed`](https://pebbletemplates.io/wiki/tag/embed/) 标签。这将对改善《兰花》的主题发展大有帮助
*   改进的错误处理

尽管如此，我在下一个主要版本 0.17.0 上也取得了很大的进步！这将是对兰花中的 wiki 和部署特性的重大重构和改进，以允许与所有主要 Git 托管平台更好地集成，并为将来更容易集成类似特性铺平道路。

这是一个非常重大的改变，需要做很多工作，但是我的目标是在这个月完成它！以下是您可以在此版本中预期的功能的完整列表:

*   Netlify 为真正的大型网站进行了优化部署，能够上传[无服务器功能](https://www.netlify.com/products/functions/)
*   将站点部署到 GitHub、Bitbucket、GitLab 或 Azure DevOps(目前仅支持 GitHub)
*   直接从 GitHub Wikis 导入 wiki 内容
