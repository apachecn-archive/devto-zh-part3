# 我如何应对编码挑战

> 原文：<https://dev.to/craignicol/how-i-approach-coding-challenges-301>

去年的这个时候，我正在为工作面试做大量的测试，并且和 https://adventofcode.com 一起练习

我知道有各种各样的数据结构和算法可以让这些变得更容易(我希望在以后的帖子中回到我最喜欢的一些)，但我采取的一种通用方法对我有帮助，也可能对你们有些人有帮助。

## 使用 TDD

我一直是测试驱动开发的粉丝，这些类型的难题非常适合 TDD 思维。通常会提供一些例子来形成初始的测试套件。该算法通常被分成几个步骤(有时明确地“在每个时钟周期…”)

养成边走边查的习惯，充分利用提供给你的信息。在编写任何代码之前，首先考虑问题，以及输入和输出之间的映射。

检查简单解决方案的所有内容。

## 有模板

大多数挑战都涉及一些常见的任务。从文件中读取输入(忽略空行)，创建…列表，以 CSV 格式输出，等等。

其中一些在你选择的语言中会有标准化的模式，你应该使用它们。有些可能需要一些错误处理。有些需要从标准库或第三方库中加载。所有这些都需要一个测试框架。

当你这样做的时候，建立一个你知道如何使用的函数、类和包的实用程序列表，足够健壮并且有最小的混乱。它们并不意味着除了你之外任何人都可以使用，但是你会经常使用它们。收藏并珍惜它们。

## 认为简单

一般来说，技术测试和编码挑战都被设计成简单的解决方案。如果你需要求解 P == NP 来使你的代码更快，很可能你使用的算法并不合适。找出另外两种解决问题的方法，并选择更简单的一种。

## 比那更简单

想一想最简单可行的解决方案，然后简单地写下来。

使代码可读。现在不是寻求聪明解决方案的时候。当一个输入失败时，让它容易改变。

## 但没那么简单

您仍然需要考虑边缘情况、内存使用和无数其他资源约束。在这个难题的范围内，想想什么可能出错。《代码的来临》有一个特殊的诀窍，可以在每天挑战的第 2 部分中检测代码中的边缘情况。

## 想得快

小心过早的优化，但是如果使用简单的实现，会有很长的输入和很多循环。学习新的算法和数据结构。
