不久前，云计算、人工智能(AI)、机器学习(ML)、区块链等技术。以令人难以置信的能力征服了我们。

> ### By 2020, artificial intelligence (AI) and machine learning will be the main catalysts to promote the popularization of cloud computing.

人工智能、人工智能和云计算的使用正呈爆炸式增长。我们离 Alexa 只需要几个字就能独立为我们构建移动应用的那一天不远了。人工智能和人工智能模型正在改变今天的技术范式，包括云计算和管理空间。然而，它还不能在这个领域开发具有人类认知水平的系统。

今天，一些云管理服务利用机器学习或人工智能。正如我们所知，ML 模型需要在一个定义的环境中的大型数据集来进行分析和预测人类认知水平的结果。

因此，即使在云计算的利基空间，人工智能或基于人工智能的解决方案也只有在大数据集可用时才能创造奇迹。 像大型企业一样，他们以一种有组织的方式收集数 Pb 的度量数据——从虚拟机的内存、CPU、磁盘利用率到安全威胁。因此，在这样的设置中，处理大数据集的指标并预测结果更容易。

与大型组织不同，中小型企业的指标数据较少。因此，使用人工智能或基于人工智能的解决方案并不能提供预期的有效结果。

在这种情况下，他们需要构建超复杂的数据模型来获得有效的结果，这是一个相当大的挑战。标准化 AI/ML 模型的可用性要完全解决这些问题还有很长的路要走。这可能是孤岛式管理工具或控制台仍然广泛使用的原因之一。

## **行为 AI 的力量:解决 AI/ML 过度依赖数据的方案**

解决人工智能/人工智能在云世界中过度依赖数据的一个似乎可行的解决方案是行为——人工智能用于实时战略(RTS)游戏——想象一下像魔兽争霸和星际争霸这样的游戏。在这里，预测和决策更多地基于实体的行为分析。

[![Clash of Clans Behavioral AI](../Images/cdeab07c41752b40855f12d412cc2961.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--xE34WFNo--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/61wl3sru229pq65doros.jpg)PC:[https://www.imore.com/clash-clans-tips-tricks-and-cheats](https://www.imore.com/clash-clans-tips-tricks-and-cheats)

每个云服务都表现出一种标准的行为。举个例子，亚马逊 S3 桶以某种方式运行，而 Kubernetes 集群有它们的行为，谷歌可抢占虚拟机有它们的特点。因此，RTS 游戏中的决策概念可以系统地应用于预测和建模云计算资源的动态性和特性。

此外，由于标准云 API 已经可用，这些预测可以自动化，并通过调整已经制作的 ML 模型来共同利用以构建下一级智能。

> ## "When behavioral artificial intelligence is put to good use on cloud resources, we can build an intelligent cloud platform, which can provide visualization as human imagines, analyze as human thinks, and make correct decisions as human acts."

在 TotalCloud，我们正朝着建立一个智能云平台的方向慢慢前进，这个平台具有人类思维的先天偏好。

## 前路:先睹为快

在 2019 年初，我们将推出三项关键功能:

#### **强化世界观**

这将是 AWS 所有区域的资源库存和成本数据的统一零步骤视图。您可以同时获得与资源库存相对应的 AWS 云支出的顶层视图。

#### **标签视图**

该视图将能够实时提供 AWS 标签模式。使用这个视图，您将能够根据您为您的应用程序、环境、版本等创建的某些标签来查找资源。一目了然。

#### **AWS VPC 流量日志分析**

AWS VPC 流量日志在检测安全漏洞和网络行为方面非常有用。这项即将推出的功能将帮助您实时监控所有数据包流，并帮助您更快地查明架构中的问题。
