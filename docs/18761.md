# 在不知情的情况下拿着一把上膛的步枪

> 原文:[https://dev . to/hagailuger/holding-a-loaded-rifle-without-known-it-31i 5](https://dev.to/hagailuger/holding-a-loaded-rifle-without-knowing-it-31i5)

两个故事:

1.  当我在军队服役时，一个不太有趣的故事发生在我的一个朋友身上:
    他因为无聊而玩他的 M16(那是 2000 年。没有智能手机...).以某种奇怪的方式，他在不知情的情况下给自己的枪上了膛。
    那我怎么知道呢？
    因为一名军官检查了他的枪...(我们都很幸运)。

2.  在使用 SQL SERVER 时，我的一个朋友(总是朋友。从来不是我...)在数据库的本地副本上工作。或者这就是他认为自己在做的事情。
    实际上他在 SSMS 打开了两个标签:
    一个代表他当地的环境。
    一个代表我们的产品。
    你能猜到发生了什么吗？

您可能是对的:
生产数据库被删除了。

好吧，让我们 3 个小时后，当所有的经理从家里回来，因为没有工作。虽然是在 SQL AZURE 上，它有很好的备份，但他设法完成了所有步骤，避免了恢复到时间点的正常方式。

我们是如此的无助，以至于我们试图将缓存中的数据恢复到数据库中！最后，Azure 的人来帮助我们，给了我们需要的东西。

我从中学到了什么？似乎比我想的要少。
因为昨天差点又发生在我身上！

进入这些地方太容易了。

唯一的办法就是纪律。如果你不用，任何花哨的工具都救不了你。

你制定规则，你深刻理解它们为什么存在(在一些职业中，他们说:“规则是用血写的”)。

尽量不要拿着一把上膛的步枪，如果你必须拿着它，至少要意识到这一点。
