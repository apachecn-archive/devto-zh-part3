### 什么是 JavaScript 疲劳？

在研究 JavaScript 时，我注意到了术语 **JavaScript 疲劳**；在过去的几个月里，我花了很多时间研究不同的相关 JavaScript 主题，当然我并不是一个人在做这个任务，看起来每次我需要学习或理解这个主题的新东西时，它都会把我带到不同的路线上(我很喜欢)。

**JavaScript 增长**

JavaScript 疲劳是 JavaScript 生态系统增长的结果，其不断增长的受欢迎程度和全栈开发人员的需求以及不同的编码训练营教育和教授编码已经影响了使用、应用和需求，不仅需要了解 JavaScript，还需要了解可以增强应用和网站开发的工具。

**太多的工具和框架**

事实上，你可以用 JavaScript 构建任何东西，是的！但是借助 ES6，jQuery，React，Redux，Webpack，Babel，TDD，JEST，Git，async，OOP，functional-style，npm，Node.js，Yarn，SQL，NoSQL，Graph databases，MongoDB，Angular 等等。等等等等。

使用不同的框架和库来更好地适应你正在开发的应用程序将会增强你的代码工作流程。即使你可以用普通的 JavaScript、HTML 和 CSS 做很多事情；现代 JavaScript 为您提供了丰富开发工具的巨大选择。

尤其是对初学者(包括我)来说，一开始看起来有些不知所措，但是理解不同工具和库的工作方式会让你在求职过程中脱颖而出，成为专业人士。

**收益**

使用不同工具和框架的好处是，有些是样板库，您可以不受限制地使用，或者不需要对原始版本做太多更改。

**结论**

JavaScript 是许多其他工具、框架和库的生态系统，因此，如果您想在开发应用程序或与其他开发人员合作时使用最新的和新的附加功能，了解这些技术的重要性要求您关注围绕这些工具的所有不同的更新和变化。
