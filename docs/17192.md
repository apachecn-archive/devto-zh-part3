# Linux 桌面的定义

> 原文:[https://dev . to/dmerejkowsky/a-definition-of-the-Linux-desktop-19 F8](https://dev.to/dmerejkowsky/a-definition-of-the-linux-desktop-19f8)

*最初发表于[我的博客](https://dmerej.info/blog/post/a-definition-of-the-linux-desktop/)。*

很久以前，我在 twitter 上看到加里·伯恩哈特的一个帖子。

与 macOS 和 Windows 相比，以下是他对 Linux 的评价:

> 不像其他两个操作系统，你有控制权。但是没有什么是可靠的，尤其是在升级的时候。当它破裂时，社区会把它框定为你自己的道德失败。

作为一个几乎在所有事情上都使用 Linux 的人(无论是在工作中还是在家里)，你可能会认为我不会同意。

但是不，我同意几乎所有的观点，除了我要重新措辞最后一句。让我解释一下。

# [](#control)控制

让我们从头开始。根据您使用的操作系统，您有什么样的控制？

## MAC OS 上的控件

你几乎无法控制硬件。你通常在“苹果商店”购买苹果制造的硬件。

大多数应用程序都有最低限度的配置，像防止笔记本电脑在合上盖子时挂起这样简单的事情很难实现 <sup id="fnref1">[1](#fn1)</sup> 。

最后，你经常无法控制自己的数据。在非苹果设备上观看你在 iTunes 上购买的电影是非常困难的。

你可以把你所有的东西都放在 iTunes 或者 iCloud 上，但是如果你失去了访问你的账户的权限，你就失去了一切。 <sup id="fnref2">[2](#fn2)</sup>

## Windows 上的控件

相比 macOS，你对硬件的控制力更强。例如，当你使用的电池出现故障时，你可以购买一个新电池，而不是购买一个全新的设备。

但是，你对更新什么和什么时候更新的控制越来越少，而且我们不要谈论隐私…

## Linux 上的控件

你控制了(几乎)一切。

你可以选择你的发行版。你可以选择它的稳定性。例如，在 Debian 上，你可以在*稳定*、*测试*或*不稳定*之间进行选择。

你可以选择你的桌面环境:KDE，Gnome，LXDE，XFCE，Mint，甚至根本不用桌面环境。

你可以选择你的窗口管理器，你的初始化系统，你的软件包管理器…

你可以选择更新的时间和内容(不像 Arch Linux 这样的“滚动发行版”，尽管如果特别小心的话还是有可能的……)等等。

控制无处不在，无处不在，甚至令人恐惧。但这是免费的:除了时间和精力，没有什么能阻止你去尝试。

# [](#when-it-breaks)当它断裂时

让我们来谈谈当你正在使用的软件崩溃时会发生什么。

在 macOS 和 Windows 上，你能做的就是祈祷，找一个有解决办法的人。当然有 bug 追踪器(但是在 macOS 上你只能看到你打开的 bug)。

如果你升级 Windows，一个第三方驱动停止工作，你基本上就完蛋了。因为这个，我看到我妈妈几个月来没有使用它的扫描仪。

在 Linux 上，事情*会*更频繁地出错。他们需要更少的人来做 QA，有更多的未知，大量不同的配置(正如我们在“控制”部分看到的)，以及经常发布和保持“前沿”的压力。

几乎所有的东西都是由无偿志愿者编写和维护的(顺便说一下，他们的工作非常出色)。

升级是巨大而可怕的，所以东西经常会坏掉。没必要讨论这个。

几个月前，我在笔记本电脑上升级了 Arch Linux 后，亮度控制按钮停止了工作，我不知道为什么。

但老实说，我没有花时间去寻找解决方案，这将引导我们进入下一部分。

# [](#your-own-moral-failure)自己的道德失败

让我们看看普通用户之间的一些交流，他们有一个问题，从社区中寻找答案，例如在公共 bug 追踪器或论坛中:

> 问:我已经把 *foo* 升级到 3.0 版本了，现在我不能再刷了
> 答:你在升级前看过变更日志了吗？

> 问:我买了一台新打印机，现在只能黑白打印！在购买新打印机之前，你研究过与 Linux 的兼容性吗？

让我们试着设身处地为那些用户着想。他们可能认为自己在某些事情上失败了，甚至认为他们甚至不应该问这种问题 <sup id="fnref3">[3](#fn3)</sup> ，这与加里所说的“道德失败”是一致的。

# [](#rephrasing-the-last-part)改写最后一部分

那么我哪里不同意呢？嗯，我不喜欢的是“框架”部分。我想把最后一句话改写为:“当它坏了，你(或社区)有责任修理它”。

我来详细说明一下。

当 Linux 上的东西坏了，你可以做点什么。

您可以降级，您可以切换到其他发行版，您可以在论坛上讨论问题，您可以向邮件列表发送电子邮件，您可以在 bug tracker 上打开一个问题。

你甚至可以拿到代码，自己打补丁，如果你不怕的话。有时候也没那么难。

所以我不认为社区在设计它。我认为这就是 Linux 世界的工作方式。你不能指望志愿者总是免费修复所有的错误。而且你不能指望质量和由专业人员做的质量一样。

所以，是的，作为控制自由的交换，你得到了一些责任。当事情发生时，你的职责是帮助自己和他人。顺便说一句，教用户在购买硬件之前阅读变更日志和做研究是很好的建议，*如果*他们愿意接受的话。

对我来说，这些责任只是小小的代价，因为比起方便，我更看重控制和自由。如果您愿意接受这些职责来获得这种自由，并且不使用 Linux，那就试试吧！

如果你正在使用 Linux，并且想要“改变”其他人(比如朋友或家人)，向他们解释这一点，这样他们就不会生你的气。准备好帮助他们不懂的事情。你甚至可以教他们一些东西，或者教他们如何学习。大约 15 年前，我就是这样开始使用 Linux 的。

如果你不想在东西坏了的时候试着去修理，那就继续用 macOS 或者 Windows。这没什么不好。

干杯！
