# 我不研究框架

随着从事软件开发工作的时间的推移，我过去常常研究一些库和框架，以便更加精通这种特定的方法。

如今，我不会花一分钱去研究任何框架或库，尤其是关于 web 应用程序架构的。

当我们谈论 web 应用程序时，通常公司想要的工作流是这样的:

CRUD 服务和显示信息的地方。

#### [](#crud-service)Crud 服务

这一代喜欢使用微服务架构的计算一代基本上是来自某个地方的 crud。它可以是数据库、队列、文件、其他服务或任何其他数据管理系统。

#### [](#a-place-to-display-the-info)显示信息的地方

随着互联网上多个平台的连接(笔记本电脑、个人电脑、移动设备、微波炉、汽车、医疗机器人，甚至我们身体的某些部分都可以连接到互联网上)。我们需要一种漂亮的方式来正确显示信息。它可以是一个队列，一个单页应用程序，一个嵌入式组件，一个汽车上的显示器，一个外科医生的显示器，或者任何一种我们的想象力可以创造的疯狂的东西。

考虑到这一切，我为什么要花时间和一些人(我的，公司，朋友..)花钱去学习如何用不同的方式创造这些东西？这是我想要的成长方式吗？通过成为工具使用者？或者我想知道如何实施这些解决方案，然后选择我必须采取哪些措施来完成工作？

[![Lazy](../Images/7bf15b551d48a0b21eef7312823f1c7e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--EN08zToA--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://static.techspot.com/images/teaser/lazy-coder.jpg)

让我们举一个例子:

你必须开发软件，帮助一些研究人员了解巴西特定地区的癌症增长情况。

您必须创建一个 web 应用程序，通过导入一些已经由数据科学团队分析过的 CSV/excel 文件来收集这些信息。您的应用程序还必须为数据工程师团队提供一个 Restful API，以便放置和参考来自管道作业的信息。

你知道这时候该做什么。你需要建立一些微服务和一个 spa。您的微服务将使用 MVC 设计模式还是发布-订阅方法？该公司需要哪种编程语言的解决方案？Java？Python？仙丹？Golang？Php？Nodejs？

[![](../Images/40023366e3bc056f2c26642e26b615d2.png)T2】](https://i.giphy.com/media/lgRNj0m1oORfW/giphy-downsized-large.gif)

您知道这些解决方案必须如何工作:它必须有一些 HTTP 操作将访问的资源，必须有一些身份验证/角色。让我们用 Java 作为我们的编程语言。

通过简单的谷歌搜索，我们得出了今年最好的 web 框架，对于创建 Restful Apis 服务非常有用。

所以在这一点上，我必须知道如何正确地使用 Java，如果我这样做了，在一周内学会如何使用一个框架就不会那么痛苦了。通常，框架都有很好的文档和入门指南，这将帮助你在短时间内精通。

所以我用我的时间来提高我使用 Java 的技能，而不是学习一些我可能有一天会用到或者用不到的框架。

好吧，但是你没有从这篇文章中学到任何东西。好吧，这个结论对提高你的编程技能很重要。如果你花时间去学习一些工具，比如框架，你最好马上使用，否则，你将花费精力在一堆知识上，而这些知识在你需要的时候可能会过时，此外，现在有大量类型的框架，我相信你不会因为学习它们而破坏你的能力，
你需要做的只是:

*   了解哪些框架试图解决什么问题。
*   深刻理解你的编程范例的概念
*   知道在哪里搜索
