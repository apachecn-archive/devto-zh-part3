# 过渡到您的第一个初级开发人员角色

> 原文:[https://dev . to/jamesmh/transitioning-into-your ' s first-junior-developer-role-2n8d](https://dev.to/jamesmh/transitioning-into-your-first-junior-developer-role-2n8d)

[在之前的文章](https://dev.to/jamesmh/the-stages-of-your-software-development-career-2dd)中，我们看到了在思考一个典型的开发人员如何在他/她的职业生涯中过渡时，我觉得有助于用作通用模板或框架的各个阶段。

第一个转变是从程序员转变为初级开发人员。

在这里，我将为那些可能面临这种转变的人提供一些建议。

此外，如果你个人已经“超越”了职业生涯的这个阶段，重温一下仍然是有帮助的，因为这些建议中的许多仍然是有帮助的。

当你指导或者只是想帮助其他处于这种情况的开发人员时，拥有这些知识也是有帮助的😁。

# [](#coder)编码器

一个程序员基本上是一个爱好者。他们不是拿工资的专业人士，但喜欢在业余时间编写代码。

通常情况下，他们独自完成他们的项目。

一个编码者可能有以下一些特征:

*   可能精通几种编程语言
*   可能对语法有深入的了解
*   能够解决小型项目中出现的问题

# [](#junior-developer)初级开发人员

当有人在第一个有报酬的职位上进入这个领域时，这种转变就开始了。

开发人员很快就会了解到，作为一名开发人员不仅仅是“编写代码”。

你将被期望接受业务需求，并将它们转化为解决现实世界问题的应用程序和工具。

以下是我对那些面临这种转变的人的各种提示和建议:

## 你现在是一个团队的一员了！

没错。没错。人们。很多人。

这增加了一个全新的维度！

以下是一些关于成为优秀团队成员的建议:

### [](#communicate-clearly)沟通清楚

在讨论中，你需要意识到什么是重要的。

例如，如果你要告诉每个人你一天在做什么，不要深究你正在做什么的细节。

相反，简要地说明你正在做什么工作，以及你是否需要帮助:

*“我将修复一个与我们手机应用程序主菜单相关的错误，然后我会继续开发新功能 x。”*

我见过很多开发人员直接投入到他们工作的技术细节中。那只是浪费大家的时间和精力！

### [](#emotional-intelligence)情商

我们都有糟糕的日子。

你的队友也是人。

像对待人一样对待他们。

尽可能鼓励他们。

如果你真的有问题，开始一对一地解决问题。但是明智地选择你的战斗。如果弗雷德今天过得不好，那么最好是顺其自然。

如果一种态度等等。会影响整个团队或在很长一段时间内，您的经理也会帮助您解决问题。

## 对我的小脑袋来说太大了！

现在来谈谈一些更普遍的建议！

你会很快发现自己工作的代码库与你在学校时的不一样。你可以把这些装进你的脑袋里。

在现实世界中，你将会面对庞大的代码库，而这些代码库是不可能一次全部装进你的脑袋的🤯。

当我作为一名初级开发人员开始工作时，我被期望开始处理一个拥有 700 多个表和数千个函数和存储过程的数据库。作为一个刚刚起步的人，我真的不知所措！

你也会不知所措。

我们都经历过。

只是需要时间。

我的建议是:**在你接触代码之前，把一切都写下来！**

先在“纸上”想出策略。

把问题分解成小步骤。写下每一步。

这里有一些你可能想包括的东西:

*   哪些数据库表/列需要更改？
*   您需要更改或调查哪些代码文件？
*   有没有解决核心问题的已知设计模式？
*   是否有任何未回答的问题或不明确的要求？
*   等等。

你的同事会非常感激你花时间去分解每件事。

如果你确实需要帮助，他们会非常乐意帮助你。

这对每个人来说都更容易——即使是高年级学生也不能记住所有的事情，并把它们都记在脑子里！🤣

## [](#keep-an-achievements-library)保留一个成就库

记录下你完成的每一项重要任务。如果可能，包括可能相关的指标。

例如:

*“我将一个关键 SQL 查询的速度提高了 200%”*

*“完成特性 X，在未来 6 个月内为公司带来 X 个新客户。”*

*“了解了 SQL 应用操作，并使用它将 X 公司的数据迁移速度提高了 50%。”*

这将有助于你私下回顾自己的成就。我们都会有情绪低落的时候，会陷入感觉自己没做什么好事的季节。

但是，当你重新写简历的时候，这也会有所帮助。你会有很多弹药！

## [](#learn-how-to-research)学会如何研究

爱因斯坦曾说过一句话，大意是“永远不要记住你能查到的东西。”

这是软件开发者的口头禅！

即使是高级开发人员也必须每天研究东西！

我的建议:**阅读。读了很多。**

阅读那些稍微超出你的舒适区和当前理解范围的东西。

那会使你伸展。最终，你应该能够读得更快，甚至学会快速浏览材料。

这可以区分一个平庸的开发人员和一个超级开发人员。

如果你能很快找到你不知道怎么做的事情的答案，那么你会被尊重为能很快回答别人问题的人。

你将成为提升他们所在团队的人。

## [](#community)社区

我个人后悔没有早点做的一件事是参与开发者社区。

这将帮助你找到工作，结交新朋友，找到导师，并让你感觉自己不仅仅是你自己和你工作的公司的一部分。

找出谁是你特定领域和兴趣的“最佳”开发者。

在 Twitter 上关注他们，并开始阅读和评论他们的博客！

你最终可以涉足的其他领域:

*   开始为开源项目做贡献(甚至只是文档！)
*   开始一个开源项目
*   参加本地用户组、聚会和会议
