# 好代码

> 原文：<https://dev.to/anortef/good-code-5bcd>

我总是听到开发人员说“我们必须写出好的代码”或者“那些代码是坏的”，但是什么是好代码，什么是坏代码呢？

首先，我们是工程师，什么是工程？一个很基本的描述就是:用最先进的技术以较低的成本解决一个问题。

现在我们要找到软件开发的成本在哪里，成本主要分为两部分:创建和维护。维护是数量级最大的一项，因此我们需要降低维护成本。

我们发现，TDD 和其他一些被称为“良好实践”的方法，通过降低重构风险或构建一个结构良好的类系统，有助于降低软件维护成本。因此，具有 90%代码覆盖率和非常低的[圈复杂度](https://en.wikipedia.org/wiki/Cyclomatic_complexity)的代码库并不是好代码，因为它拥有这些，但由于这些，它保持了较低的维护成本。

综上所述，我们可以说好的代码是生产成本最低的(维护成本在生产成本之内)，好的工程师是能够生产好代码的人。
