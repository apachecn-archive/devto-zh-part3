# 跟上

> 原文：<https://dev.to/burdettelamar/keeping-up-506e>

在我之前的帖子“[退休，显然是](https://dev.to/burdettelamar/retired-apparently-1210)”中，安德烈·佩纳评论道:

> 我真的很有兴趣了解你认为你的认知能力是如何随着时间的推移而变化的，以及你是如何在工作行业中保持相关性的。我认为“随着年龄的增长，我的职业生涯会发生什么”是大多数开发人员担心的事情。

有很多关于[保持](https://www.google.com/search?client=firefox-b-1-d&q=keeping+up+in+software+development)的文章和帖子。

我在这里要做的是写下我个人所做的事情。

这是:

### 视频、音频、迪斯科(拉丁语意为“我看、我听、我学”)

“光看就能观察到很多。”- *约吉·贝拉*

在作家(小说、诗歌、散文)中，据说你应该为你试图写的每一个人读 100 部作品。

在开发世界中，几乎没有人阅读足够的代码。

*   我在博客、书籍和 GitHub 项目中阅读代码。
*   当我在谷歌上搜索技术诀窍时，除了规格和想法，我还会搜索*示例代码*。
*   我看别人写代码(并提出问题)。我总是总能学到一些东西。

### 定期复习重点书籍

我说的*复习*，不是指重读。我的意思是我翻阅，寻找以前对我不重要的东西(或者至少*看起来不重要)，但现在可能变得重要了。*

在汽车服务站等候，翻阅着*红宝石食谱*我了解到了令人惊叹的红宝石[合同](https://egonschiele.github.io/contracts.ruby/)。

我的最爱:

*   *务实的程序员*。
*   *代码完成*。
*   *Ruby Cookbook* (您的语言可能会有所不同)。

### 在工作中学习新技能

(重点是用老板的钱学习。)

*   我当数学老师的时候，学过 Fortran，开发过数学考试生成器。
*   当我还是一名文档管理员时(在现代工具出现之前)，我开发了一些实用程序来:
    *   创建目录和索引。
    *   扫描关键代码文件的变化。
*   当我是一名文档员时，我学习了 C 和 M4，来开发一个标记处理器。
*   当我还是一名手工测试人员时，我学习了 Perl 来开发自动化测试。
*   当我还是一名 C#自动化开发人员时，我学习了 Ruby 和 Watir 来测试 web 应用程序。

### 自愿承担新的责任

当我还是一名运行时库维护者时，我自愿接管一个新生的矢量并发数值函数库的开发，所有这些都是用 [i860 汇编器](https://en.wikipedia.org/wiki/Markup_language)编写的。很难说在接下来的几年里我学到了多少。

### Scribo(拉丁语意为“我写的”)

我把代码和文本放到网上。

我在我的(现在处于休眠状态的)博客，T2 的 GitHub ，T4 的 RubyGems ，还有 Reddit 的[，当然还有 DEV](https://www.reddit.com/user/BurdetteLamar) 的[。这样做:](https://dev.to/burdettelamar)

*   我学到了新东西:降价，红宝石。
*   我从其他人那里得到了宝贵的反馈。

### 拥抱潮流，但不是赶时髦

不要第一个尝试新事物，也不要最后一个抛弃旧事物。- *亚历山大教皇*

要知道潮流和时尚之间的区别并不容易，但这很重要。

提示:如果在会议上他们兜售你从未听说过的东西，要小心。

对我来说:

*   我开始使用 ide 时已经太晚了。
*   我花了太多时间探索关键字驱动测试，这不适合我测试的复杂应用。
*   我很快就要睡觉了。
