# 如何写 bug 报告？

> 原文:[https://dev . to/rlxdprogrammer/how-to-write-a-bug-report-37bh](https://dev.to/rlxdprogrammer/how-to-write-a-bug-report-37bh)

这不是一个真正的博客帖子，只是一些简短的我想到的和我想与你分享的东西。

不幸的是，我收到的错误报告给我带来了非常糟糕的体验。我经常收到这样的东西:

*   “软件不工作”= >好的，不工作是什么意思？你忘记启动它了吗？
*   “我不能点击按钮”= >你的鼠标坏了还是怎么的？
*   “它没有保存文件”

在这种错误报告的情况下，可能需要几个小时才能弄清楚您真正的意思并重现它。所以这只是浪费时间，你也需要等待更长的时间来修复你的错误。

相反，请尝试详细说明如何重现您的错误以及为什么它的行为不正常？

错误报告应始终包含以下信息:

*   你有什么工作环境(硬件，什么操作系统，其他相关软件(例如。web 应用情况下的浏览器))等。
*   您正在使用软件的哪种配置/设置？
*   重现问题的步骤是什么(请从启动软件开始)？
*   你在这里会期待什么样的行为？
*   当前行为是什么(当然和预期的不一样，这就是为什么叫 bug)？

如果你以这种方式报告你的 bug，开发人员会更高兴，你的 bug 会更快修复！
