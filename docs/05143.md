## [](#3-angularjs-to-angular-7)3。角度 j 至角度 7

Angular 是 Google 在 2010 年创建的基于 JavaScript 的开源前端 web 框架。它用于应对开发单页面应用程序时遇到的许多挑战。

到目前为止，我们已经观察到了从 Angular 1 到 7 的许多版本，这在某种意义上阻止了许多公司和项目采用它。

让我们了解一下 Angular 7 的新功能:

*   更新 CLI 提示:使用`ng new`或`ng add`，您可以选择路由，并与 CSS，SCSS，更少的样式创建。

*   增加了 ScrollingModule，允许实现虚拟滚动。例如，当元素变得可见或不可见时，DOM 会添加或删除它们。

*   为拖放功能增加了 DragDropModule:列表重新排序，在列表之间移动元素等。

*   更新依赖关系:Angular 现在使用 Typescript 3.1，RxJS и Node.js 10。

## [](#4-graphql-instead-of-rest-api)4。GraphQL 代替 REST API

许多人认为 GraphQL 将在 2019 年成为 REST APIs 的替代品。是什么，为什么更好？

[![](../Images/c6871a32d4afaabd923f3979966dfd07.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--uotcQv4p--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/kiwbji1wvormcxzmh02n.jpg)

GraphQL 是一种 API 查询语言，也是一种用现有数据完成这些查询的运行时语言。GraphQL 为 API 中的数据提供了完整且易于理解的描述，使客户能够准确地要求他们需要的东西，使 API 更容易随时间发展，并支持强大的开发工具。

以下是 GraphQL 解决的三个最重要的问题:

*   需要多次请求数据。GraphQL 允许您通过对服务器的一个请求获得所有必要的数据。

*   客户端对服务器的依赖性。使用 GraphQL，客户机以通用查询语言进行通信，这消除了服务器严格指定返回数据的结构或组成的需要。

*   无效的发展方式。在 GraphQL 中，开发人员使用声明性方法描述接口所需的数据。开发者关注的是他们想要得到什么，而不是如何去做。

## [](#conclusion)结论

在 JavaScript 库和框架中，社区选择了 React。这个库在与 Vue 和 Angular 等竞争对手的竞争中遥遥领先。许多开发人员选择 React 是因为它是一个有前途的、高潜力的、可伸缩的选项。

但是，你也要考虑到正在稳步快速发展的 Vue.js。它已经领先于 Angular，但这个选项有一些功能需要改进和更新。在我们看来，Vue 还需要几年时间才能成为第一。

考虑到服务器和前端的连接，我们认为您应该关注 GraphQL，它是 REST APIs 的一个很有前途的替代品。
