# 别把自己压死了

> 原文:[https://dev.to/kovah/stop-stressing-yourself-to-death-l2j](https://dev.to/kovah/stop-stressing-yourself-to-death-l2j)

我们确实生活在一个每秒都在加速发展的现代世界里。每件事都需要做得越来越快，最好是现在就做。我们的技术在发展，每一次发展我们都会在计算中发现新的可能性，这进一步提高了对新可能性的期望。这听起来有点像科学的事情，例如，在我们最终越来越多地放弃化石燃料之后，现在希望能够使用冷聚变，可以被分解为对信息技术特定领域的非常详细的观察:网络，它的发展和在这一领域工作的人。作为一个从事 web 开发的人，我看到越来越多的问题出现在这种越来越快的竞赛中。

# [](#the-problem-with-burnout)倦怠的问题

几十年来，一些人(但不是很多人)遭受了一种有点幽灵般的奇怪状况，倦怠或倦怠综合征在 1974 年由赫伯特·弗罗伊登伯格首次描述，他研究了一家诊所的工作人员，称之为“工作过度要求导致的疲惫”。这也是你能找到的最好的简短描述。今天，倦怠通常是在人们崩溃后被诊断出来的，这是由长期的大量压力造成的。在最好的情况下，这些人不会崩溃，在为时已晚之前被诊断出来。大多数受影响的人不想，或者无法意识到他们即将崩溃。有些人可能能够掩盖他们的压力水平，有些人只是减少他们的社会交往，呆在家里。就像所有其他疾病一样:症状会增加，直到病因被消除或减轻。这是这里的一个主要问题:倦怠不是一种疾病，而是一种暂时的状况，因此大多数人不会做任何事情来对抗它。但是接受你生病了——不是身体上的，而是精神上的——是恢复健康的第一步。

# Web 开发的实际问题是什么？

倦怠与 web 开发有什么关系？很多。其中一个主要原因可能是，世界变得越来越快。不断涌现的新趋势、新技术和更高的期望推动着互联网应用程序或网站的开发。这导致了不同的问题:

## [](#develop-fast-to-match-trends-and-current-expectations)快速发展以符合趋势和当前期望

潮流来来去去，随之而来的是新概念的出现。我刚刚从 Nextpage 上读到一篇非常有趣的[文章，内容是关于网络的发展以及我们最终要进入的下一代:web 3.0。虽然这不是官方术语，但你可以清楚地看到，随着 CSS Grid 和 WebAssembly 等令人兴奋的新事物的出现，我们达到了新的高度，客户也期待着更好、更令人兴奋的网站和应用程序。对最终可能发生的事情感到有点不知所措是很正常的。如果你的客户对某件事情应该如何运作有一个非常具体的想法，而你刚刚开始考虑它，你已经稍微提高了你的压力水平。](https://nicepage.com/doc/article/20348/web-design-3-0-when-your-web-design-really-matters)

## [](#learn-more-to-develop-faster)学得越多发展越快

[![Laptop in the Dark](../Images/dafae6990696d903c523eb7bc24899ce.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--6gdgZ3DM--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/n66n9qfsojvq0i24glee.jpg)

最后一点是，随着网络生态系统的不断发展，包括数以百万计的开源库和工具，我们能够更快地开发新的东西，因为引导变得更容易，我们不必为显示图表或制作精美的动画编写自己的实现。然而，这导致了一个问题，您应该准备好使用所有这些库和工具，这涉及到不断的学习。学习新事物，因为你要么必须使用新的东西来发布令人兴奋的东西，要么你的旧设置已经不足以满足期望。

## 网络真是快得出奇

虽然汽车制造商有更多的时间来制造他们的汽车，因为你不可能在一周内造出一辆新车并把它运出去，但 web 开发人员使用的是一种非常快速的媒介。不仅如此，新的设备实际上能够处理这些更高的期望(像做沉重的动画或加载高分辨率的图像，因为网络速度上升)。网站可以一秒进来，下一秒就走，这也是事实。(大多数情况下)只需点击一下就能把一个网站放到网上。一个命令，一个按钮，你能想到的。除此之外，网络是大众保持联系和获取信息的媒介。因为这个原因(也因为我们就是我们)，互联网上的东西很快就过时了。激怒了成千上万用户的新闻故事可能几天后就无关紧要了。

结果是一个永无止境的更快的循环:开发者试图多交付 1%来满足他们客户的期望，而客户因为一天后在另一个网站上看到的东西而提高他们的期望。这个循环导致了 web 开发领域的工作人员面临持续的压力。

# 不要把自己压死

嗯，说实话，没那么容易。事实上，要让自己的肩膀不承受太大的压力真的很难。关于这个话题的文章即使没有上千篇，也有数百篇。每一个都讲述了不同的故事。因此，我会试着告诉你我遇到了什么问题以及如何解决它们。

## [](#learn-the-essentials-first)先学要领

这是我可以向任何新开发人员或任何想成为开发人员的人宣扬的一件事:学习要点。本质是基本架构系统或编程语言本身，而不是库或框架。为什么？因为这些基本要素实际上会帮助你更容易地进入任何遇到你的新事物，然后适应你的需要。如果一个人甚至不能将带有自定义回调的事件监听器附加到单个元素上，他为什么要开始学习 Vue.js 呢？

## [](#quit-open-source)退出开源

尤其是开源项目似乎就像是精疲力竭开发人员的孵化器。开源本身就是一个问题，无论你做什么，你都无法解决它。因此，放弃吧。如果你是一个维护者，并且觉得这个项目与其说是快乐的源泉，不如说是一个负担，那么就像我为 InvoicePlane 所做的那样，寻找一个新的维护者[。长话短说:不值得为那些不给你报酬的人损害你的健康。就这么简单。如果你经营一个基于你的开源项目的生意，要么卖掉它，要么找人帮你经营。](https://community.invoiceplane.com/t/topic/7114)

## [](#switch-your-employer)切换你的雇主

我们开发人员幸运地属于一个缺乏专业人员的行业。如果你觉得你现在的雇主低估了你的价值(这也可能导致很大的压力)，或者强迫你加班更长时间而不给你补偿，那么辞职，寻找新的公司。如果你已经快累垮了，那就远离创业公司和代理公司。找一家信誉卓著的公司，提供固定的工作时间、充足的假期，或许还有其他与工作相关的福利。

## [](#trends-are-not-worth-following)趋势不值得追随

这可能是一个有点争议的观点，但我坚持了几年，现在我自己的项目。巨幅横幅图片？文字文章谁需要 4000px 大图？古怪的颜色和全幅渐变？可能与你项目的设计语言不匹配。在匆忙将这些新奇的东西运用到你的下一个项目中之前，想想它们。

而且，如果你相信这一点或现在，你可以建立漂亮的，现代的和交互式的网站，它们也很快，并且不需要在你的用户设备中使用四核 CPU，不需要使用任何臃肿的库，如 Bootstrap，React 或 Angular。

## [](#take-your-time-off)请假

离开我的开源项目 InvoicePlane 后，背后开始思考改善“内心生活”的方法。而且是一些小事。休假并不意味着暂停工作三个月，探索越南丛林，或者与外界隔绝，与西藏僧侣生活在一起。休假意味着在上班的路上把笔记本电脑放在包里。像我一样阅读一本书(最好是非技术性的)。狂看一系列关于网飞的节目。请几个朋友一起吃饭。有太多的事情不涉及编码。只要做就好，哪怕只有一个小时。你可以以后再编码。

## [](#stay-away-from-challenges)远离挑战

另一个有争议的观点，但我真的不喜欢所有这些编码挑战，因为#100DaysOfCode 现在很流行。嗯，有些人会说这些挑战激励人们编写更多的代码，并最终取得一些成就。但是问问你自己，你是否必须有所成就。在我看来，这些挑战毫无价值。因为没人在乎你是否完成了一项挑战，也没人在乎你是否还没开始。

# [](#stay-healthy)保持健康

如果我想到其他事情，我可能会在以后扩展这篇文章。现在，问题已经列出来了，我给了你一些思考的要点。保持健康，开发伙伴。
