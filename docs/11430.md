# 拥有你的故事

> 原文：<https://dev.to/mercier_remi/own-your-story-52mo>

这个帖子对我来说真的很重要。这件事在我心里已经有一段时间了。这是一个关于你生活中的巨大变化的故事，你如何在这个过程中迷失并找到回来的路。希望你会喜欢！👏

* * *

当你把自己放在那里——写一本书，运送一个产品，改变你的生活——你会得到关注。有些是令人愉快的。有一部分，没那么多。无论你得到什么样的反馈，保持头脑清醒都不容易。

在过去的 15 年里，我做过彩色玻璃制造商、插图画家、画廊老板、营销人员等。不用说，我遇到过一些不可思议的人，但也有一些糟糕的同事(有时，我自己也很糟糕)。所以当我开始寻找我的第一份开发工作时，我知道我会对团队超级挑剔。

我想加入一个团队，在那里我可以不时感到脆弱，问愚蠢的问题，打开蹩脚的拉请求，而不会被有毒的混蛋鞭打。

在过去的四个月里，我已经和 20 家公司谈过话了。作为一名开发人员，我想更好地了解就业市场，所以我不会回避与讨人喜欢和不讨人喜欢的人会面。在展示我的作品时，我得到了很多反馈。这让我意识到我知道的是多么少，我是多么的新手。

尽管我对自己日益增长的无知感到满意，但我最终还是被我的知识和我的对话者之间的差距弄得晕头转向。每次采访都带来了我不知道的技术内容。我已经决定我会保留一份这样的事情的清单，以便以后挖掘。但是很快，名单变得如此之长，让我心灰意冷。更糟糕的是，我越是处理这些不相关的事情，我的日子就越觉得乏味。就像又回到了学校。将所有东西捆绑在一起的总体产品曾经很有趣，但很快就失去了吸引力。

作为新兵训练营的一员，我也受到了很多指责。我在采访中听到的一些评论:

*   “我以为你最多能在 5 分钟内写完这段代码”(我花了 45 分钟🤷‍♂️).
*   “为什么你不能解释这个算法是如何工作的”。
*   “所以你不知道什么是本征类”。

一位首席开发人员甚至给我上了 45 分钟的课，讲述新手如何缺乏适当的 CS 知识，以及如何成为一名真正的开发人员(TL；博士:读黑客新闻)。

所有这些采访都是在被评判中度过的，我经历的反弹最终将恐惧带回了我的生活。害怕比较。害怕不能作为开发人员工作。害怕成为一个彻头彻尾的骗子。仅仅几个月我就完全变了。从大胆，我变得恐惧。我不想这样开始我的开发生涯。

当我开始找工作时，我戴着最薄的社交面具参加面试。我穿着破旧的牛仔裤，谈论精疲力竭，没有简历，说了很多“他妈的”(但我仍然要进行技术测试)。然而，在几次采访后，我意识到这种行为会吓坏人，所以我尝试了一种更传统的方法。

我更新了我的简历，把我能找到的每一行代码都塞进去。问题是，冒名顶替综合症已经出现了。听说过想变得像牛一样大的青蛙吗？👋

**我刚才说的:**

> 开发了一个潜在客户生成基础设施，年收益 60 万美元。使用 API 集成了几个营销 CRM:映射数据库设计以确保数据完整性。使用开源 AngularJS widgets 库和外部 API 编码的 [opendatainception.io](https://opendatainception.io/) 。

**IRL:**

> 使用 Hubspot 生成销售线索，并通过内置集成将数据传递给 Salesforce。实际编码 [opendatainception.io](https://opendatainception.io/) 使用开源的 AngularJS widgets 库和外部 API。也厌倦了琐碎的工作、办公室政治和过多的会议。

我非常想看起来像一个真正的开发者，以至于我不自觉地夸大了事实，并为此受到了惩罚。还记得我说过的那些下流的评论吗？看过我的简历后，人们对我的期望很高，因为我的简历让他们相信我更有经验。他们的失望是合理的。

在找工作的间隙，我和尼古拉斯·泽马蒂谈论了我的经历。尼古拉斯让我意识到我只是在假装成另一个人。我关闭了自己的特性。所以我试着重新思考我当初为什么决定学习 web 开发。回到我自己的故事，以及我是如何结束编写[哈利](http://harry.team)、[一击即中](https://strike-it.herokuapp.com)和[一些非常令人讨厌的东西](%7Bsite.url%7D/wtf-time-complexity)的感觉很好。

我喜欢做东西。我的职业生涯始于制作彩色玻璃。我尽可能在我的阳台上制作木制家具。我画画。做了几年市场营销后，我对被排除在产品建设之外感到厌倦。这就是为什么我想回去运输东西。

我也通过小门进入了技术领域:管理社区、设计信息图、定制 Wordpress 主题、学习 DB 设计、编写 Javascript 垃圾程序等等...我没有计算机学位(将来也不会有)。我将成为这些开发人员中的一员，他们推动着[“标准 rails 混乱”](https://dev.to/ben/ah-to-be-the-justin-bieber-of-open-source-4jog)，却不知道[如何在白板上写一个冒泡排序](https://twitter.com/dhh/status/834146806594433025)。我也将成为这些开发者中的一员，享受:

*   建造人们使用的东西，
*   与他人分享他们所知的一点点，
*   努力建设更具包容性和支持性的工作场所。

这是我的故事。找到你的，拥有它，找到欣赏它的公司/客户/人。
