[![molly profile image](img/5c34463ea6ee7c5f65b3228848d6db47.png) ](/molly) [ Molly Struve (she/her) ](/molly) • [<time datetime="2019-01-03T21:57:46Z" class="date-short-year"> Jan 3 '19 </time>](https://dev.to/molly/comment/805k) 

对我来说，最大的障碍是当一家公司设定了他们希望你在办公室的“固定时间”。我个人是一个早起的人。我喜欢在早上 7:45 到办公室，我在早上工作效率最高。然后我在下午 4/4:30 左右回家。一些公司对办公室的朝九晚五很严格，这对我来说从来都不好。

@raviipradhan 写了一篇精彩的文章来回答 **[你的编码起源是什么？](https://dev.to/kelly/whats-your-coding-origin-story-544m)** 。看到不同人的编程之路总是很棒:

[![ravii profile image](img/0e423388853a1b039bad514f7f511671.png) ](/ravii) [ Ravi 🐧 ](/ravii) • [<time datetime="2019-01-03T18:45:01Z" class="date-short-year"> Jan 3 '19 </time>](https://dev.to/ravii/comment/8034) 

大约在 2017 年初，我开始对我在商业和社会研究领域的工作感到厌倦。恰好在这个时候，学校的一个伙伴想要提高他的编程指导和训练技能。他想找个人做他卑微的实验品，考虑到我的情况，我很乐意帮忙。因此，通过 Codewars katas 进行结对编程和 TDD 是我进入这个世界的切入点。

我想最先打动我的是这一切是多么有趣和容易理解。除了一般的 IT 课程，当我在学校的时候，在英国的课程中并没有特别强调编程；因此，我经常认为这是一个深奥的职业，只有少数人才能做到。出于享受的考虑，我决定继续进行指导——主要包括每周下班后喝几杯啤酒后的结对编程会议。不久后，我递交了辞呈，并在 2017 年 4 月带着些许惶恐辞去了研究工作，专注于学习全职编码。

从那时起，我对 web 开发产生了兴趣，并在指导下主要致力于迷你项目、FreeCodeCamp 和 YouTube 教程。我也去了伦敦内外的聚会。这个城市有一个很棒的社区，我发现能够让人们交流想法，听开发者分享他们的经验非常有帮助。

我进入了这些活动的节奏，在 2017 年底，在发出几份申请后，我获得了一个职位，是一家位于该市的初创公司的初级前端开发人员。那是我目前的工作地点。这很有趣，而且附近有一个很棒的开发人员社区，我觉得从事编程工作很舒服。绝对觉得自己的飞跃是正确的！

[![abraham profile image](img/46dbdfe87e9cd0d86c6f8df2a1690fd0.png) ](/abraham) [ Abraham Williams ](/abraham) • [<time datetime="2019-01-02T22:54:07Z" class="date-short-year"> Jan 2 '19 </time>](https://dev.to/abraham/comment/7pg5) 

在过去的几年里，我一直在使用 Atom，并且对它非常满意。圣诞节期间，我切换到了 [VS 代码](https://code.visualstudio.com/)，我不认为我会回头。代码速度更快，延迟明显更少。我搬家的原因之一是看到新的包出现在代码上，而把 Atom 远远甩在后面。

在动荡的一个月里，我使用了 T2 的安卓工作室，发现它臃肿而复杂。从那以后，我也将 Flutter 开发转向了代码。

[![httpjunkie profile image](img/bc7c6b268071a03f2f2acebef036a9ee.png) ](/httpjunkie) [ Eric Bishard ](/httpjunkie) • [<time datetime="2019-01-02T18:16:01Z" class="date-short-year"> Jan 2 '19 </time> • Edited on <time datetime="2019-01-02T18:16:45Z" class="hidden m:inline-block date-no-year">Jan 2</time>](https://dev.to/httpjunkie/comment/7pc2) 

现在前端面试很疯狂，所以很难学习。我第一次开始面试前端机会是在 2014-2015 年。在这一点上，如果你对 JS、CSS 和 HTML 有扎实的理解，你就不会有很多问题。

但在 2018 年的最后一年，我面试了 Paypal、亚马逊、LinkedIn、脸书、Airbnb、Cruise、谷歌，我确定我错过了一些。面试都不一样。Airbnb 真的很难，他们开门见山，他们有一种非常具体的使用类和对象处理 JS 的方式，尽管有其他所有前端技能，但我在第二次面试时被踢出了这个过程，因为我不太了解他们处理 JS 的具体风格。

他们可能不太关心我在 UI 领域很强，我是 JavaScript 社区的成员，我已经和 JS 一起工作了 20 年并且学得很快。他们并不关心我是否有全面的技能，相反，重要的是我能够回答一个特定的问题，这个问题与他们在那个部门的工作方式非常相关。

我和一个人说话，也被那个人拒绝了。

脸书实际上有一个很好的方法，随着面试的进行，他们越来越难，他们能够准确地知道你作为前端开发人员的位置，他们对你的优势和劣势有很好的了解，他们准备好帮助和理解你可能没有在整个夏天接受他们非常具体的做事方式的培训。

今天的前端面试可以涵盖所有领域。这篇文章中的建议很好，但我警告人们，前端现在可能比几年前更复杂。祝你好运！
