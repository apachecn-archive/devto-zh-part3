# 测试你的代码很重要

> 原文：<https://dev.to/flippedcoding/its-important-to-test-your-code-3lid>

有时我们作为开发人员会忽略最明显的错误。有时我们知道他们，并说“我们会回来”。然后是时候部署网站或为客户做演示了。他们打开网站，很快就会看到一条难看的服务器错误消息。你还记得你现在忘记的那件事吗？

这就是测试的用武之地。它就像你最好的朋友，让你在付钱给你的人面前不显得愚蠢。虽然就像一个最好的朋友一样，但很容易认为这是理所当然的，因为它总是在那里。(去告诉你的朋友谢谢你)为了让他们明白测试代码的重要性，这里列出了一些测试可以帮你避免的事情。

## 向客户展示他们无法运行的网站

没有什么比让所有东西都在本地工作，然后让网站运行起来更糟糕的了，没有什么东西像它应该的那样工作。当你客户的钱岌岌可危时，他们不会在乎你是否让网站在你的机器上运行。他们希望它能在客户的机器上运行。

您可以通过提前部署网站并自己做一些测试来避免这种情况。可能有一个连接字符串或 CORS 策略没有正确的值。这也是您进行功能检查的机会。用户不会知道你让网站运行的小技巧。确保他们不需要。

即使提前部署需要额外的时间，这也是值得的。你不仅可以快速修复你发现的任何明显的错误，还可以展示你工作的真实质量。

## 浪费大量时间

拥有多个客户、项目或角色会分散你的时间，所以你需要确保尽可能有效地利用时间。起初，测试似乎花费了太多的时间，尤其是对于那些超级小的单元测试。为他们腾出时间。

与浏览整个应用程序相比，在处理特定的代码块时更容易跟踪 bug。错误有传播的趋势，如果你有匿名函数，在调用栈中丢失是正常的。这就是为什么单元测试很重要。

它们让你看到最小的东西，以确保它们在多种情况下都能正常工作。你知道用户可以找到破解你的代码的方法，所以你要在他们有机会之前就这么做。这将节省你抱怨的时间，在压力下挖掘代码的时间，以及处理别人的监督的时间。

## 编写糟糕的代码

测试你的代码教你如何写出好的代码，因为你必须修复所有的坏代码。当你不得不检查并修复你自己的错误时，你会学到不同的解决问题的方法。破解代码的方法数不胜数，这也是它如此有趣的原因。

查找和修复错误有助于您知道在编写代码时首先要注意什么。这就是你获得网络开发经验的方式。你写代码并修复它。你做的测试越多，你就越容易接触到不同种类的错误。

你看到的错误越多，你就越不会再写了。因此，你会成为一个强大的开发者。测试是你在整个职业生涯中都要坚守的老师，因为你从中学到了很多。即使你不想学习。

这并不是说测试可以让你远离一切。服务器可能存在您无法控制的问题，或者浏览器更新可能会导致一些问题。尽管如此，测试对于避免你遇到的常见问题或者那些你忘记“回头再来”的事情是很重要的。

所以请不要跳过测试。这不仅仅是为了你，也是为了那些在你之后还得为你工作的开发人员。不管刻板印象是什么或者社会怎么说，网络开发是你和其他人一起做的事情。测试让你的生活更轻松，也让别人的生活更轻松。做吧，因为这是对的。:)
