# 软件工程师的编程技巧

> 原文：<https://dev.to/thefern/programming-tips-for-software-engineers-5gki>

我不久前在 imgur 上写了这篇文章，有近 5000 名 imgur 人喜欢这篇文章。我真的很自豪，甚至几个软件工程师都称赞这个帖子。这真的让我大吃一惊，因为我是自学的。从那以后，我就没怎么写过技术文章，尽管我更多地参与了开源社区。我想是时候让我站出来做出更多的贡献了。以下是我给初学程序员的建议，没有特定的顺序。我对我最初的帖子做了一些小的编辑。今年我将完成我有史以来最大的项目，名为 **CrystalPlot** ，这是一个工业自动化绘图工具，更多内容将在以后的文章中介绍。

## 0。有学习编程的愿望，并且明确知道为什么

知道你的目标。不管是做一个游戏，一个手机 app，一个网站，不管你的目标是什么，一开始就要明确。一个好的计划是制作一个日历，在日历上添加章节，并努力坚持这个计划。

## 1。在编程的早期阶段学习单元测试

我是自学的，我不知道有多少书根本没有谈到单元测试。我去了很多年，对单元测试一无所知。一旦你学会了，你就会慢慢地停止在你的代码中使用这么多的 print 语句。单元测试是一种针对不同场景测试代码的简单方法。

## 2。选择一种编程语言，然后跟着它走

新手的一大困境。有很多选择，你的选择可能会受到朋友或同事的很大影响。每个人都是不同的，我们都用不同的方式思考。您对语言的选择应该受到您手头的资源以及您的编程目标的影响。例如，如果你计划开发 iPhone 应用程序，那么 objective-c 应该是你的选择，或者是现在的 swift。如果你想在 Unity3d 中构建游戏，那么就用 c#或 javascript。顺便说一句，一旦你学会了一门语言，其他的也会学得很快。这里有一个小提示，仅仅因为你只做控制台应用程序，并不意味着没有好处。图形用户界面(GUI)将稍后推出，不要担心。

专业提示:猛禽流程图，如果你想试试 Python 也是个不错的选择。

## 3。用你所学的语言决定一个中小规模的项目

这也是一个艰难的选择，在我卑微的开始阶段，我不知道在学会一门编程语言后该做什么。一定要选择你热爱的事情。库存系统、银行系统或解析库。此外，你也可以贡献给一个知识库，同时学习。不管是什么，不要想太多，建就好。相信我，我等了很久，认为我需要学习和学习更多。

不要谷歌如何构建一个完整的项目，而是在纸上设计你的程序规范。写下里程碑的清单。用小步骤写你的程序。我认为对于试图一次拿太多的初学者来说，这是一个巨大的问题。如果你在路上遇到困难，寻求帮助，StackOverflow，YouTube，Google。很可能你要找的东西已经在那里了。

## 4。学习 Git 或其他类型的版本控制

如果你正在做任何类型的编程，Git 是必须的。你可以跟踪你的项目，也许还能让其他人对你的项目感兴趣。Git 的好处在于，您可以跟踪所有的修订，并为将来的特性或当前正在处理的 bug 创建分支。

## 5。向他人学习

在 GitHub 中你可以浏览别人的代码并从中学习。在编程中，你通过写代码、读代码和修改代码来学习。导师也确实令人鼓舞，但我从未找到过，或者说从未寻找过。

## 6。了解您的调试工具

不言而喻，一个好的程序员不会做出一个没有 bug 的程序，而是知道如何修复它。学习调试是你工具箱中最大的武器。

## 7。学习设计模式

不同的问题需要不同的方法。也许你听说过 MVC、Facade、Singleton、Observer 等术语。有三种基本的设计模式，结构的、创造的和行为的。好好学，你以后会感谢我的。

## 8。数据结构和算法

大多数程序都有某种类型的数据。并且它应该被有效地处理，这是 DS 和有效算法到位的地方。UNSW 电子教学是一个很好的起点。理查德·巴克兰教授很棒，他的课通常很有趣，也很有互动性。

## 9。学习面向对象编程范例

这是一个给定的，一旦你学会了基础知识，书籍会抛出像多态性，封装等术语。不要害怕。面向对象是获得干净和可维护代码的好方法。现在大多数现代语言都支持 OOP。然而，还有其他的编程范例，请看下面的链接来了解它们。
[编程范例](https://en.wikipedia.org/wiki/Comparison_of_programming_paradigms)

## 10。一路帮助你的计划

一开始，你只需要一个文本编辑器。然后学习如何从终端/命令提示符编译和运行程序。随着程序的增长，我建议切换到 IDE，但是两种方法都尝试一下，看看哪种最适合你。我使用 Intellij、Sublime Text 和 terminal。

## 11。证明文件

学习文档方面非常重要，SDD，软件设计文档。并且很好地注释你的代码。

## 12。记录

随着项目的增长，ole 打印使得跟踪代码变得更加困难。几乎任何编程语言都有大量的选择。着色、过滤、级别，只是一个好的日志框架的几个优点。

归根结底，就是找到你的定位，并把它带到下一个层次，无论是建立网站，游戏，移动应用程序，控制台应用程序，还是桌面应用程序。我希望这个小小的列表能让你对我认为的一些很棒的建议有所了解。祝你在软件开发生涯中一切顺利。
