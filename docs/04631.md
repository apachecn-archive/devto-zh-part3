# 为什么我要把我的博客和作品集转换成盖茨比

> 原文：<https://dev.to/ashleemboyer/why-i-m-converting-my-blog-and-portfolio-to-gatsby-4ac6>

你也可以在我的个人博客上找到这篇文章[。](https://ashleemboyer.com/why-i-m-converting-my-blog-and-portfolio-to-gatsby)

为什么，是的，我想完成我的网站的设置，这样我就可以更专注于我想在我的投资组合上发布的副业项目。

*你*昨天不是刚 *[发了关于完美主义](https://ashleemboyer.com/neither-you-nor-your-work-has-to-be-perfect)的帖子吗？*

当然有！但这不一样。软件将永远是一条有许多曲折和山丘的道路。😉

严肃地说，在发布我的网站的最新版本之前，我没有试用过盖茨比。在过去的这个周末，我终于屈服了，决定试一试。就这样。好玩。**首先**，他们的文档非常优秀。它们很好地解释了事情，任何水平的编码经验的人都可以知道该做什么。

其次，这个框架非常容易使用。在一个命名良好的目录中创建一些文件，你就有了一些神奇的页面。你也可以动态地制作页面*，这也是我换页面的原因之一。*

 ***最后**，使用他们的插件做任何你想做的事情太容易了。比如用 [`gatsby-plugin-react-helmet`](https://www.gatsbyjs.org/packages/gatsby-plugin-react-helmet) 插件，可以动态处理 SEO *和*打开图形元数据。这意味着当你在推特上分享你的帖子时，你可以拥有带图片的酷酷的推特卡。这让我大吃一惊！*(这在 Angular 有可能吗？对我来说太晚了，但也许其他人可以利用这些知识。)*

老实说，我只是想多陪陪盖茨比。你会看到很多人在他们的博客网站上使用它，所以我想利用这些信息来挑战自己，做一些真正脱颖而出的东西。💪🏻

那么，你呢？你在你的网站上使用盖茨比吗，或者你打算这样做吗？我很乐意在评论中或者在 Twitter 上听到你的项目。
