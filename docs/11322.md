# [](#why-did-we-build-it)我们为什么要建？

如果您曾经使用过 Krypton 来认证 SSH 会话、签署 Git commit 或通过 WebAuthn/U2F 使用 2FA，或者甚至使用过 Duo——您就会知道在提示符下点击“Approve”按钮，然后立即登录或授权某些操作是多么令人满意。这比在每个网站上输入看起来不一样的 6 位数字代码要甜蜜得多。

氪解决*开发者* **签到**需求。

我们想超越开发者，超越登录。通过 ApproveAPI，我们为任何类型的验证或批准工作流的每个人提供相同的**一键批准** UX。

## [](#rated-e-for-everyone)给所有人评定为 E

每个人都有手机，但不是每个人都愿意下载应用程序。无处不在的解决方案必须适用于任何地方。ApproveAPI 的一个目标是开箱即用地为每个人工作。

ApproveAPI 允许您在第一天就为您的所有用户创建相同的授权 UX，并通过电子邮件和短信提供可定制的批准提示。

不仅仅是电子邮件和短信。从氪星，我们知道基于应用的体验可以有多好，但是为什么不是每个*应用都支持这个？通过 ApproveAPI，您可以在您自己公司的应用程序中添加**相同的**批准提示 UX，这是我们与苹果和谷歌的推送通知服务集成的简单推送 SDK。*

## [](#beyond-just-logins)超越仅仅登录

点击-批准工作流程如此有用，为什么只局限于两个因素呢？

ApproveAPI 的目标之一是使开发人员能够轻松地为任何工作流请求实时用户批准和二次验证——从确认可疑的欺诈交易、使用双因素验证对用户进行身份验证、为客户支持验证身份，到具有主管批准和审计日志记录的内部合规性用例。
