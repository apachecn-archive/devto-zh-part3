# 2018 年回顾过去，2019 年展望未来

> 原文:[https://dev.to/gonedark/2018-lookbehind-lookahead-2019-4pnn](https://dev.to/gonedark/2018-lookbehind-lookahead-2019-4pnn)

每年我都会制定目标。我的大部分目标都集中在工作上，但我也会挤出一些个人目标。去年，我公布了我的 2018 年发展目标。所以，在谈论我的 2019 年目标之前，让我们回顾一下 2018 年。

## [](#2018-goals)2018 年目标

### [](#write-a-book)字本

我在 7 月 13 日发布了基础代码的早期版本，并在 9 月 12 日发布了完整版本。不完全是这样，最终的*退出*章节是 12 月 3 日发表的。

在写 *BaseCode* 之前，我已经有了代码样本、资源和实践。即使所有这些都准备好了，它仍然比我预期的要长。对我来说，写作并不是最难的部分，它是把它们放在一起:校对，格式化。

另一个延迟我们都是附加的。在最初的版本中有更完整的版本会更好。当我发布*基本代码*时，我卖出了不同的*套件*。这给了我很大的压力，要我尽快完成。不一定来自别人，而是我自己。

BaseCode 在财务上没有我预期的那么好。我仍然相信 *BaseCode* 填补了中级开发者的空白。所以有观众。但是营销是另一种野兽。我这次做得更好，发了简讯和推文。我没有注意到的是发布。尤其是让有影响力的人帮助我及时、集中地接触到更多的受众。虽然这部分是故意的。我知道别人愿意帮忙，但我想看看我自己能做多少。最终这些类型产品的销售只是有最初的流行，但也可能有一个长尾。我预计*的 BaseCode* 将在未来几年带来销售额，并可能最终达到其财务目标。

### [](#learn-python)学习 Python

我没有完成这个。我选择 Python 的原因是为了数据处理。我的意图是用它来做一个投资分析工具，但我一直没有时间去做。否则我没有必要每天使用 Python，因此我没有达到这个目标。

有一天我可能还会学习 Python，但这是出于需要。面对我的其他目标，我真的没有理由强迫花时间去学习它。

### [](#expand-shift-services)➖扩大班次服务

我在 2018 年增加了更多班次，我必须诚实地说，我没有按照我想要的方式在技术上扩展服务。这仍在进行中，但它不会进入 2018 年，并将作为部分目标结转到 2019 年。

特别是，我想添加一个订阅来允许用户运行班次，作为他们持续集成工具的一部分。这需要订阅计费、额外的班次、网络挂钩、API 密钥、容量。因此，无论如何，这是威尔很难在 2018 年完全完成的事情。它甚至可能会持续到 2019 年以后。

### [](#speak-at-five-conferences)✅在五次会议上讲话

2018 年，我在 13 场会议上发言。13!我粉碎了这个目标。不会再有这样的一年了。虽然我喜欢每一次会议，但这太多了。2019 年，我将更有选择地提交哪些会议，提交哪些主题，并最终接受——理想情况下，只在与我的产品和服务或理想地点直接相关的会议上发言。

### [](#reach-10k-twitter-followers)❌推特粉丝达到 10k

就像学习 Python 一样，我也非常失败。2018 年，我的 Twitter 粉丝数量甚至没有超过 2017 年。目前，我刚刚完成 10k，有 4800 名粉丝。

实现这一目标需要持续的存在和精心制作的内容。虽然我总是尝试做后者，但前者是我失败的地方。这是我将继续关注的事情，但可能没有成为目标的压力。我会把它看作有希望有机发生的事情。

## [](#2019-goals)2019 年目标

### [](#redesign-blog)重新设计博客

我重新设计我的博客已经有几年了。它看起来很过时了。这导致我在 Dev.to 和 Medium 等其他网站上发布内容，而不是首先在这里发布。我还打算把域名换成我在黑色星期五打折时买的 jasonmccreary.me。

诚然，这是一个相当容易的目标。但是，以一些快速的胜利来开始这一年的势头是很好的。我发现当我试图用一个大目标开始新的一年时，我会全力以赴。

### [](#less-talks-more-workshops)少开讲座，多开研讨会

正如我之前提到的，我计划少在会议上发言。但是我仍然喜欢演讲，并且想继续下去。我认为我擅长发现知识上的差距。这通常不是最新的花哨代码或趋势，而是需要注意的更基本的主题。这些被忽视了，但实际上是我们每天都在使用的东西，并且会带我们走得更远。

2017 年，我举办了几次在线研讨会。大多数是免费的，一些是付费的。2019 年打算办几个。

### [](#contribute-to-open-source-projects)为开源项目做贡献

虽然我主要通过教育为社区做出贡献，但我也想在 2019 年通过代码做出贡献。显而易见的候选人是为拉勒维尔做更多贡献。我想对这个框架的几个方面给予一些关注。所以我计划学习更多关于当前代码的知识，看看我能在哪些方面做出一些贡献。

### [](#improve-frontend-technologies)提高前端技术

我一直坚信前端和后端的分离——特别是通过 API。我认为这是一个既定的趋势，并将继续下去。2019 年，我计划在任何新项目上“API 优先”，并可能重构一些现有项目。这并不一定意味着单页 app，但我有机会学习更多的技术，如 Vue，React 等。

### [](#build-optionality)建立可选性

学习 Python 的根本目的是构建一个投资分析工具。该工具已转向*可选性*。构建[可选性](https://optionality.app)是一个故意缓慢、细致入微的 MVP 过程。然而，这是我个人打算使用的工具。虽然我计划尊重 MVP 方法并分享我的进步，但这是我仍将建立的东西。不为别的，只为搔搔自己的痒处。就像速战速决一样，我相信你也需要一个宠物项目。
