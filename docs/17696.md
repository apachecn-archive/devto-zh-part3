# 角都要在 30 天内反应过来

> 原文:[https://dev.to/httpjunkie/angular-to-react-in-30-days-p5k](https://dev.to/httpjunkie/angular-to-react-in-30-days-p5k)

这将是一篇短文，重点介绍我作为一名 Angular 开发人员在一个月内学习 React 的经历。

2017 年 6 月，我被解雇了，在那之前，我一直梦想着在特斯拉担任 UI 工程师。我不想把重点放在这一点上，但应该注意的是，这就是为什么我有一些空闲时间，并且需要通过学习另一个框架来进一步提高我的技能。

为什么我觉得有必要学习 React？

我列了一张清单，上面列了所有我可能会为之工作的公司。他们所有人，FTMP，都用过 React。我做了功课，我认为如果我只面试那些寻找棱角分明的开发人员，我就会把许多我想去的公司从名单上去掉。我希望能够申请前端工程师，而不是 Angular 或 React 开发人员。当谈到他们将使用什么时(几乎在每种情况下都是这个或那个)，我知道我可以明智地就每个框架进行对话，并在任一框架中工作。

在旧金山湾区，我离所有好的聚会和活动都很近，所以我参加了更多的 React 聚会，并开始在家工作，从头开始学习 React。我想督促自己像了解 Angular 一样学习 React。我惊讶于 React 是如此的简单和平易近人。我也在很短的时间内了解到 React 正在经历一些成长的痛苦，即将出现的一些变化不仅会改变 React 开发人员编写组件的方式，而且这些变化将是框架中的一个重大发展。

我想从测试驱动的角度开始学习 React 的基础知识，为此，我使用 Chris Minnick 的 React 从[测试驱动开发开始学习 React。既然我有时间，我认为从测试驱动的角度学习 React 或者至少尝试一下是非常重要的。学习这门课程很容易，之后，我相信我可以在大部分 React 代码中实现单元和集成测试。我还认为从一开始就适应每个框架的默认测试模式是很重要的。](https://app.pluralsight.com/library/courses/test-driven-development-react/table-of-contents)

这两个框架确实彼此不同，但从 UI 的角度来看，它们有很多共同点，Angular 给了我作为 UI 工程师的大部分东西，React 也允许我做，只是不同。

在 Angular 中，如果您安装了 CLI，Jasmine 和 Karma 分别用于开箱测试。在 React 中，与 CLI 相当的是 create-react-app，我发现开始测试的最简单的方法是 Enzyme 和 Jest，大多数人都使用它们。因此，本课程有助于解释 React 中这些工具的安装和使用。

我在学习 React 的路上感觉很好，如此渴望和自信，以至于我想让 Redux 成为我学习的下一件事。作为一名为几家大公司工作的 Angular 开发人员，我看到他们经常因为没有正确管理状态而陷入糟糕的实践中，我想把这作为重中之重，但是我知道这不会像跳进 Redux 那么容易。我强迫自己先学习一点反应式编程，使用过滤器、贴图和 reduce。我用普通的 JavaScript 实践了这些概念，就像我学习如何实现 Flux 和 Redux 模式一样。

使用普通的 JavaScript 来学习这些概念，我认为是学习状态管理时最重要的事情。因为你学习的模式应该能够被带到 Angular，React 或 Vue 或任何其他框架或库。在您选择的框架之外理解这些概念可以确保您对技术的理解不局限于一种做事方式。

尽管如此，我不确定我是否会将 Redux 用于 Angular，因为我认为很容易将我对 Redux 的了解转化为与 Ngrx/Store 等技术一起工作，并且我认为在 Angular 中使用它有很大的好处，但是我明确意识到将 Redux 与 Angular 一起使用是完全可以的，并且知道我可以决定其中之一，因为我新发现的 Redux 知识使我成为一名开发人员。

老实说，我仍然在学习状态管理，并弄清楚如何最好地使用 Redux 和 React，我知道这将是一个漫长的旅程，需要一些时间才能变得更好，但我在 Egghead.io 和 Pluralsight 找到了很好的课程让我开始学习:

[Redux 入门](https://egghead.io/courses/getting-started-with-redux)
[Redux 基础](https://app.pluralsight.com/library/courses/redux-fundamentals/table-of-contents)
[高级 Redux](https://app.pluralsight.com/library/courses/advanced-redux/table-of-contents)

如果我不得不为开始 React 和 Redux 之旅的人推荐其中一门课程，那就是上面由 Dan abra mov(Redux 的作者)链接的“Redux 入门”课程。事实上，我会按照我列出的顺序推荐上面的课程，我还会注意到 Pluralsight 之外还有其他非常好的 Redux 课程，但我也知道很多人可以通过他们的公司免费订阅 Pluralsight。

我还想提一下 Redux 的一些免费课程，我是 Traversy Media 的 [YouTube 频道](https://www.youtube.com/user/TechGuyWeb)的超级粉丝，他的 Redux 课程是免费的。

[用 React 还原速成班](https://www.youtube.com/watch?v=93p3LxR9xfM)

事实上，这个家伙在 YouTube 上有很多很棒的内容，还有很酷的波士顿口音。

我还会推荐[自由代码营](https://www.youtube.com/FreeCodeCamp)，因为他们有大量的免费网络内容，包括 React 上的许多东西。我知道有这么多，有些谁知道可能会更好，但至少你有一些想法，从哪里开始寻找付费和免费课程。

因此，尽管 Redux 方面有更多的工作要做，但我仍然想学习另一个主题，以确保我可以开始 React 构建自己的应用程序。幸运的是，Egghead.io 有一门很棒的路由课程。

[使用 React 路由器 v4 向 React 应用添加路由](https://egghead.io/courses/add-routing-to-react-apps-using-react-router-v4)

这是一门很棒的 [React Router](https://github.com/ReactTraining/react-router) 4 入门课程，他的教学风格是重点和快节奏，同时超级简洁。我喜欢他带你一步一步了解你需要了解的各种不同的链接、导航链接和路由器配置。

了解如何使用其他 JavaScript 框架是非常好的。谁知道你会不会真的在这个过程中被改变。尤其是对我来说，当我今年夏天面试工作时，我需要知道，如果我有一个在 React 工作的好机会，我就像在 Angular 一样了解它。与此同时，我也学到了很多关于 JavaScript 和 React 在创建组件时利用的新功能的知识，以及这使它与路由器一起工作变得多么容易，只需用最少的代码就可以获得一个基本的应用程序。

我创建了一个使用 react-router-dom 的 repo，如果你想克隆它，看看它有多容易使用。

[https://github.com/httpJunkie/create-react-app-with-routing](https://dev.tocreate-react-app-with-routing)

最后，我不仅找到了一份与 React 一起工作的工作，而且为一家对 React 领域也很陌生的公司工作，我将能够与他们一起从初学者的角度围绕使用 React 创建内容，并作为 React 开发人员成长，并教他们的客户如何使用 React 和他们的 React 相关组件库。所以我开始继续学习(对我来说这并不像是工作)，花时间学习 React 的前沿技术，并构建很酷的演示和 web 应用程序来帮助其他人学习和推广我们的产品。这家公司正在[进步](https://www.progress.com/kendo-ui)，我为他们的 [Telerik 团队](https://www.telerik.com)工作，代表 React 社区和他们的 [KendoReact 组件库](https://www.telerik.com/kendo-react-ui)进行宣传。

在我开始写的 Telerik 博客文章中，我关注的一个领域是 React Hooks。我开始写关于这个主题的文章，我和 React 团队都相信这是一个真正的特性，Hooks 将引入一种新的方式来构建组件，并以一种更具功能性的方式将它们组合在一起。

快进到我进入 React 的旅程中的几个月，我还将继续参加另一个 React 基础课程[泰勒·麦金尼斯的 React 基础课程](https://tylermcginnis.com/courses/react-fundamentals)，我仍然在学习，你看我在开始学习 Hooks 之后又回到了基础课程，原因是因为我学习了很多新的做事方式，我的新工作需要我教授这些概念，但是如果我没有完全理解 Hooks 正在取代什么，我不想把自己放在教授 Hooks 的位置上。如果我要论证钩子为什么如此神奇，我需要能够详细地解释为什么新的方法与以前的方法相同或者更好。

祝 2019 年愉快，继续学习 JavaScript，我们都知道要保持相关性，我们必须不断学习，所以想办法让学习变得有趣和享受吧！
