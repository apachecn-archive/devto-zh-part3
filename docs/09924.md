随着 C# 8 的到来，我想这是一个很好的时机来反思一下这种语言最近的新增内容。有些很棒的改进你可能错过了，有些我真的很喜欢使用，有些我认为已经达到了规范使用状态，我认为这些都值得反思。

早在 2016 年，随着 Roslyn 的发布，一个用 C#编写的完整重写 C#编译器平台，我们开始看到自那以来从未见过的语言功能的增加。Net 4.0(大概)。大多数人可能还记得`async` `await`，然后语言特性开始大量涌现。

2016 年我们看到了 C# 6 和 Visual Studio 2015 的发布。这有一些很棒的特性，期待很快再次访问。两年后，Visual Studio 2017 发布了 C# 7，紧随其后的是 7.1、7.2 和 7.3，以及 Visual Studio 2017 的 8 个次要版本中的三个。这是过去 4 年来需要学习和了解的一些特性。

现在我们有了 Visual Studio 2019 的预览版(实际上是三个)，这让我们有机会在预览版中使用 C# 8 功能。有很多很棒的博客在讨论这些新东西，所以我会尽量避免重复，可能会在实际开发生活中使用这些特性几个月后再来看看。

同时，我希望回顾一下你已经拥有的，并且今天就可以开始使用的东西会对你有所帮助。

首先是猫王操作员。敬请关注。
