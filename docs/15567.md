# 成为发货的人

> 原文：<https://dev.to/vinthanedar/be-the-one-who-ships-1hcj>

作为黑客、工程师、企业家和兼职项目执行者，我们往往是完美主义者。我们关心我们发布的工作产品，并希望确保展示我们最好的工作。

然而，我们大多数人可能都有半途而废的项目，我们的想法似乎永远不会实现，或者项目进展如此缓慢，我们还不如从未开始。我自己去过那里无数次。

我们并不是看不清问题所在——完美是足够好的敌人。当有时间和资源限制时尤其如此。

在过去一年半的时间里，我记录了一些我们做得对的事情和我们避免的陷阱。

你可能会涉及到这些，我希望你下次发现自己犯下这些(非受迫性)错误时，能抓住自己，并尽快纠正自己的错误。

我最初把这个贴在我的个人博客上-[https://vinthanedar.com/2019/01/17/teams-that-ship/](https://vinthanedar.com/2019/01/17/teams-that-ship/)

希望你喜欢阅读它，并希望听到你的经历！

TL；博士；医生

无论是兼职项目，还是在公司开发软件:

建立并保持动力，就像你的生命依赖于它一样
*一次前进一点点，不管这一步有多小。*

进步>润色
*这是一个更好的方法，但是在经历过战斗之后。*

**做你自己的客户**
*狗粮你的产品，让进步更快。*

小心兔子洞。他们是瘟疫。
*做需要做的事要无情，而不是想做的事。*
