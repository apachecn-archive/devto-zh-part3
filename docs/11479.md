尽一切努力保持心流。流就是敲代码。不流动意味着谷歌搜索，使用你的鼠标，切换窗口，喝咖啡休息。

以下是常见的中断和修复方法

*   谷歌搜索/阅读文档
    *   将文档下载/集成到您的 IDE 中(参见 devdocs)并使用手册页。此外，检索 dep 的源代码，并确保您有自动完成功能(ctags)。尽可能避免谷歌搜索——这是一个坏习惯。
*   等待构件
    *   缓存您的工件以加速您的构建，或者投资于 CI
*   测试您的软件
    *   确保您可以使用 CLI 而不是交互式行为快速测试。这通常意味着您可以运行一个单元套件或命令行实验。例如，如果你正在构建一个服务，写一些你可以用断言击中的曲线，以确保它在你构建时工作。如果是 UI，编写 selenium 测试。
*   编码本身很慢
    *   激活 ide 中的自动完成和代码片段。如果你是这门语言的新手，做形来练习常见的积木。你会打字吗？如果没有，学习并玩 typeracer 来加速
*   分心(社交媒体、聊天)
    *   尝试重新安排时间来识别陷阱和阻止分心

**TL；假装你的鼠标通电了。任何时候你触摸你的鼠标，想一想为什么，让它自动化。**

我做了 3 年的“卫星工作者”，我可以证明这是非常困难的。对于在办公室工作过的人来说，有没有注意到这种情况？你要开会做决定；会议结束后，你开始与一些同事交谈，不知不觉中，你就在白板上画画，获得了一些关于这个话题的详细信息。任何远程的人都将完全错过这次对话，除非有人决定让他们进行视频通话。作为一个遥控器，即使你知道它正在发生，如果没有人在看聊天，你也无法引起任何人的注意，因为这是你的主要沟通方式。

如果团队遵循一些规范，成为卫星工作者可以工作，但我同意这个帖子的说法，这非常困难。我目前正试图在一个完全分布式的团队中找到工作，希望那会更好。

[![sleepyfran profile image](../Images/23ac07e01f16e843aa19685434038507.png) ](/sleepyfran) [ Fran González ](/sleepyfran) • [<time datetime="2019-02-27T10:09:00Z" class="date-short-year"> Feb 27 '19 </time> • Edited on <time datetime="2019-02-27T16:22:06Z" class="hidden m:inline-block date-no-year">Feb 27</time>](https://dev.to/sleepyfran/comment/94a3) 

好文章！然而，我觉得有必要提一下，spread 操作符不**而不是**对对象执行深度克隆，而是执行浅层复制。这意味着 spread 操作符将复制简单的值(如整数、字符串等)，但对对象保持相同的引用。比如:

```
const obj = { a: 1, b: { a: 1 } };
const copy = {...obj}
copy.b.a = 2

console.log(obj.b.a)
console.log(copy.b.a) 
```

这会打印出 2 和 2，因为属性 *b* 被浅复制，这使得 *copy.b* 指向与 *obj.b* 完全相同的对象。
