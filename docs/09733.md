# 没有跨浏览器测试测试不完整的 5 个原因？

> 原文：<https://dev.to/pcloudy/5-reasons-why-testing-is-incomplete-without-cross-browser-tests-3549>

在过去的几年里，网络的使用有了巨大的增长。根据互联网世界的统计，互联网用户从 2017 年的 35 亿增加到 2018 年的 41 亿。此外，根据 internet live stats，今天在万维网上大约有 15 亿个网站。开发团队在构建网站时投入了大量的精力，因此网站应该在所有浏览器上无障碍地工作是非常重要的。此外，几乎每个人都依赖于互联网上的内容。最重要的是，这些内容可以在 3000 多种不同的支持互联网的设备上获得，这也是本文的重点。更多设备，更多浏览网站的方式。所以，这里真正的问题是——所有使用不同设备的用户都能够以预期的方式浏览网站吗？

什么是跨浏览器测试？随着网络的发展，构建网站的新技术和工具也同时被引入。但并不是所有的都是浏览器兼容技术。因此，对于浏览器兼容性测试，开发人员需要执行跨浏览器测试。无非就是对 web 应用进行多浏览器测试，检查其与不同操作系统、屏幕分辨率、设备的兼容性，看是否符合开发者的预期。当涉及到 web 应用程序时，一项重要的任务是确保它们在多浏览器环境中完全可用，并为用户提供良好的体验，而不管他们使用的是什么设备和浏览器。浏览器测试是“必须的”,因为一旦用户遇到任何问题或没有获得良好的网站体验，他们可能不会回来。

让我们更深刻地理解为什么跨浏览器测试是重要的，它的目的是什么。

1.  每个浏览器都有不同的解释:
    每个浏览器都是不同的，都有不同的解释能力，也就是说，不同的浏览器对所有代码的理解是不同的。因此，一个程序员必须确保所有的浏览器都以相似的方式解释命令，以通过网站给预期的用户体验。此外，由于有多个不同版本的浏览器，要知道网站是否在所有版本上都运行良好变得很有挑战性。跨浏览器测试工具有助于测试多种浏览器及其不同版本。

2.  不同的移动平台:

交叉浏览测试变得重要的一个主要原因是因为今天有不同的移动平台。每年都会推出多种移动设备，因此移动网站必须能够在每种设备上正确打开。用户偏好不同的浏览器组合，因此组织可以专注于在移动设备上正常运行的那些组合。测试团队需要在不同的移动浏览器版本上进行测试，因为了解用户使用的版本变得很困难。甚至 CSS 在每个版本上的运行也不同，所以跨浏览器测试工具可以很容易地检查相同的内容。

1.  更少的错误:跨浏览器测试是 QA 团队必须执行的严格任务之一，但同时引入这一实践是一个伟大的举动，因为这将减少同时跨不同浏览器测试的时间。为了使每一分钟的组件和功能正常工作，并避免任何最后一分钟的复杂性和错误，应该进行跨浏览器测试，以确保所有浏览器及其版本在所有平台上都能正常工作。Web 流量数据有助于了解我们的用户经常使用什么样的浏览器，这些数据可以用作信息，只关注测试的重要部分。

4.可重用测试:
使用跨浏览器测试，不需要为每个浏览器平台创建和维护单独的浏览器测试。借助于不同的测试自动化技术，这些测试可以在不同的支持浏览器中重用。除了自定义解决方案失败的情况之外，跨浏览器测试可以由自定义脚本创建。

1.  可用的免费工具:有许多免费工具可用于执行 web 和移动自动化，如 Selenium WebDriver 和 Appium。大多数跨浏览器测试自动化工具都基于开源自动化工具，即 Selenium，它有一个很好的用户界面，可以更容易地在多个浏览器上执行跨浏览器测试。有许多广泛使用的网络浏览器，但并不是所有的浏览器，以及这些浏览器的版本，都是一样的。一个网站在每一个网站上的外观或功能可能都不一样，这可能会导致企业的收入损失。我们在 pCloudy 解决了从以前可用的固定浏览器集中进行选择的问题。考虑到我们用户日益增长的需求，我们为我们的 Android 用户引入了[启动和上传浏览器功能](https://www.pcloudy.com/mobile-application-testing-documentation/manual-app-testing/web-testing.html)，这将使他能够自由选择他想要使用的浏览器版本。这将在很大程度上节省他们的测试时间和精力，并为他们的跨浏览器测试提供更好的结果。欲了解更多信息，请访问 [pCloudy](https://www.pcloudy.com)
