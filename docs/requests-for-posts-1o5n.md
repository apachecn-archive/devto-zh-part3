如果你想写一篇文章，我会给你留下一个请求:你对 Web APIs 未来的看法。其余的未来在哪里，GraphQL 如何发展，像 [capnproto](https://capnproto.org/) 这样的原缓冲区的替代品，以及任何你想谈论:D 的东西

在这本你谈论过去的书中，我很想读一本关于现在和未来的书:)

Dev.to 有很多软件开发领域的专家，有时我会看到开发人员问*“我应该写些什么？”*。

我们都知道，提出想法并不总是容易的。

因此，我提议:

我们为什么不创建一个标签，人们可以用它来请求社区中的帖子呢？

有意义吗？

可以简单到*“嗨，朋友们，我想知道更多关于 HTTP/2 的服务器推送”*或者可以是*“我想写一篇关于这个或那个主题的帖子，但目前我没有时间，所以你来了”*

很明显，对任何人都没有义务，这也可能只是一种了解人们想了解什么的方式。

我承认这个想法还没有完全形成，但这是今年的第一天，我只是要把它放在那里😜

你怎么想呢?
