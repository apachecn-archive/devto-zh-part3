# 创造不完美的东西是有原因的

> 原文:[https://dev . to/Perkins Jr/create-something-that-is-impaired-for-a-reason-3 lmg](https://dev.to/perkinsjr/create-something-that-is-imperfect-for-a-reason-3lmg)

## [](#problem)问题

我创建并支持一些开源软件 [regexits](https://www.npmjs.com/package/regexits) 和[they up](https://www.npmjs.com/package/aretheyup)并且还有两个大型项目在进行中。一个 HQ 视频到 HQ GIF，适用于 Windows、Linux 和 Mac，以及 IQFINITY，这是亚马逊和 ebay 卖家的仪表板，都是开源的。处理所有这些事情并创造一些由用户驱动的东西是一件很费事的事情，它会耗费精力并导致精疲力竭。

## [](#solution)解

我故意创造“不完美”的网络应用、演示甚至 cli 应用。让我解释一下，我创建的应用程序运行，做我想做的事情，就是这样。通常它只关注一个元素

例如[血腥天气](http://weather.jamesperkins.io)它有一个操作。键入一个美国邮政编码，并返回带有英国俚语的天气信息...它不漂亮，不符合我通常从事的项目的水平。我从周日开始，在项目工作的 30 分钟间隔内开始。

这是它在手机上的样子

[![home](../Images/d29664eac9ffa594fa875d8bf331b21b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--QcZrPBRm--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/nslrdi0exvw7yojp54vi.png)

它不是非常漂亮，而且有缺陷，例如，如果你在手机上打字，屏幕会移动，导致重叠。我当然能修好它，但那不是我的目标。目的是通过不担心造型和响应式设计来减压。

## [](#sum-up)总结

试试这个应用程序..该 API 是一个开放的天气，有一些使用邮政编码的问题，但该应用程序开始很小，接下来我会添加城市，当我需要一些解压。你应该找一个小项目，你不要把高标准的压力，只是让它流动，它有助于突破其他工作的挫折。
