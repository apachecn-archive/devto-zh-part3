什么是插座？简单地说，套接字是数据发送和接收的端点。套接字被用作同一台计算机或同一网络上的不同进程之间的通用通信手段。因为基于 Unix 的操作系统中的一切都是一个文件，或者可以表示为一个文件，所以进程通过它们的文件描述符来识别套接字。

插座有不同的风格。每种口味都规定了给予的保证和宽大处理。从现在开始，这将被称为沟通风格。接下来，使用什么协议？这是底层细节和机制告诉数据如何传输和接收的地方。

通信风格可以被认为是一系列的问题，根据套接字的通信风格给出不同的答案。

*   使用的单位？

有些风格认为数据只是没有任何结构的字节。其他人认为数据被分组到由样式决定的结构中。

*   传输过程中的损耗？

偶尔丢失一个数据包会影响整体吗？另一端收到的数据包的顺序是否与发送时的顺序相同？有些样式保证所有传输的数据包都将按相同的顺序检索，如果数据包被丢弃，它将被重新发送。其他类型的通信可能会丢弃一些数据包。数据包也可能不止一次到达，而且是无序的。使用这种套接字的程序员必须在他们的代码中放置安全措施，以适应这种通信方式的不可靠性。

*   搭档？

传统上认为套接字类似于客户机-服务器关系，双方可以自由地相互交换数据。还有一种片面的版本:在地址上盖章，然后在途中发送数据。

我们如何为一个套接字设置端点？每个套接字都与一个套接字地址配对，或者换句话说，与 IP 地址和端口号配对。设置套接字的地址称为绑定。
