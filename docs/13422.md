# 全栈工程对你意味着什么？

> 原文:[https://dev . to/amandasopkin/what-do-full-stack-engineering-mean-to-you-107e](https://dev.to/amandasopkin/what-does-full-stack-engineering-mean-to-you-1o7e)

我进入软件工程领域时是一名全栈工程师，一名多面手，希望能够从事从低级到高级以及两者之间的各种任务。

现在，当我继续我的软件之旅时，我发现自己在怀疑自己是否还符合那个模式。专攻软件世界是有好处的，Android、React 和无数其他技术学科的专家能够达到更高的深度，并经常作为他们领域的专家上升到更高的水平。

另一方面，虽然在全栈工程的一般领域获得专业知识可能更困难，但大多数公司都在寻找全栈/一般软件工程师——一大群。

就我而言，我更喜欢后端工程，但想知道现在专业化是否为时过早。因为特别是对于缺乏经验的工程师来说，大多数公司都希望人们能够承担你可能交给他们的任何任务。

谈到面试，许多工程师在这个问题上有分歧。有些人将全栈工程视为邀请提出任何问题，有些是具体的，有些是宽泛的。其他人认为，询问高度专业化的前端问题，如如何在 CSS 中设计样式，对全栈工程师来说是不公平的。就在本周，我筛选了一名候选人，他关心的是，对于一名全栈式工程师，可能会问什么样的问题。一般的前端问题还是比较具体的？

你呢？你觉得哪些面试问题适合全栈工程师？如果你从全栈工程转向更专业的东西，那会怎么样？
