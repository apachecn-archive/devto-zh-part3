# 展望未来

> 原文：<https://dev.to/arleneandrews/looking-forward-355e>

[![](img/eb173711d7a694a168c2d553af43a705.png)](https://blog.internetcreations.com/wp-content/uploads/2015/12/Working-Remotely-Cloud.png)[https://blog.internetcreations.com](https://blog.internetcreations.com)

我最近参加了[在线测试会议](http://www.onlinetestconf.com/)，不仅享受了我参加的第一次专业级会议，而且大部分会议都是信息丰富和令人愉快的，尽管对这个世界来说有点难以理解。

有几次会谈是关于未来的展望。当然，这些都是我焦急等待的——还有什么比确保自己有价值更好的方法来确保自己有一个职位呢？有很多东西需要学习，最后一个简短的演讲提供了更多的见解。

但是他们错过了一些东西，就像所有的预言家一样。

在兴奋地找到一个适合自己的地方之后，我仔细思考了一些事情，最终我和一位导师进行了交谈，并最终找出了一些我觉得被错过的东西。

开发者世界正在进入一种模式，这种模式可能有助于平衡大城市的拥挤:远程工作。这是一个激动人心的创新时代，选择和挑战成为焦点。测试人员有了一个新的地方，他们可以将自己插入到正常的工作流程中。

然而，这意味着“要求完成代码”，或者“与团队的其他部分配对”，不再像找到正确的办公室或者发送内部通知那样简单。即使在同一栋大楼里，这些都是问题；在展示结果、获得反馈和鼓励质量时，我们必须考虑这些因素。

我对我加入的公司的最初计划不得不改变:我仍然会问我是否可以和每个部门的人一起吃饭或喝酒(请给我咖啡或茶)。但我现在不得不假设，这可能涉及到我通过电脑与他们交谈，而且可能是在尴尬的时候。但是从长远来看，与开发组的成员进行社会接触仍然是有回报的。我可以倾听他们正在做的事情的挑战和成功，并从中获得一些见解。我的目标是给他们留下这样的印象:我是他们愿意一起工作的人，并向他们提供想法。

远程工作只是必须考虑的事情之一，也是必须考虑的想法之一。其他一些被忽略的领域，但对我来说很重要:

*   代码共享工具——这些工具的安全性是否达到了 NDA 和商业信心所希望的水平？

*   现在，人们越来越意识到设备对资源和健康的影响，这正在改变我们的产品目前的使用方式，随着新市场的开放，旧市场被迫改变以参与竞争，这种改变可能会更大。

*   监管和用户人口统计的变化——年龄范围的两端都在更多地使用我们制造的东西。

*   工作流程正在向*应该是*的方向转变，一个更稳定的反馈流:这将如何影响我们质量所需的传统手工和自动化测试？

*   接下来，我们使用的设备现在跨越了几个化身——这增加了所有潜在的访问我们所构建的东西的方法的质量对等的负担。

我们今天构建和学习的东西能够经得起未来考验吗？

是的，可以。但现在，一些企业一直遵循的理论将开始显示出压力的迹象——作为一个致力于确保我们的客户获得最高质量产品的团队，我们需要能够与其他团队一起将我们的重点转移到下一阶段。

一如既往，抛弃和重新学习的能力是软件生命中的一个主要部分。我们必须学习和了解的内容使现在成为进入这个行业的最佳时机——可以帮助我们将我们的客户和产品提升到一个新的水平，使迎接下一次重大变革变得不那么困难。
