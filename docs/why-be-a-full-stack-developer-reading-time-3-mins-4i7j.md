# 简介

现在似乎有很多人讨厌全栈开发人员。这通常是非常吸引人的负面标题，如“全栈开发人员是一个骗局”，“全栈开发人员之死”。

所以我写在这里来解决这种消极。希望它能鼓励更多的开发人员认为这个角色不仅仅是一个多面手角色。

# 为什么对全栈开发者恨之入骨？

如果让我总结一下，可以归结为两件事。

第一个是由于**工资原因**作为一个**通才**，你做多种不同类型的工作，工资都很少。

然而，成为一名专家，也就是前端的 T1 或后端的 T3，可以为单一的专业获得更多的报酬。

然而，作为一个全栈开发人员，作为一个前端或后端开发人员，掌握每一项所需的知识要困难得多。

# 技能叠加

我真的很喜欢这个术语**技能堆积**，我是从**乔丹先驱秀**中听说的。

因为它包含了作为一个全栈开发者的真实感受。也就是**发展技能**将**融合**它独特地让自己置身于人群之外。

举个例子，你可能是排名前 10%或前 20%的最好的前端开发人员，他们专注于因受欢迎而竞争激烈的 React。

然而，对你来说，融合另一种 80%的顶级技能要容易得多，比如进行**设计冲刺**或建立**设计系统**。

瞬间由于你对它的理解，让你从前端开发者的拥挤领域中脱颖而出。

使用**设计冲刺**或在不同公司创建**设计系统**的顾问来提供 React 原型，以扩展他们的设计团队。

根据我的计算，由于你成功地将这两种技能**融合**，这让你得到了非常**不同**的报酬。

如果你不相信，看看数据科学领域的数据科学家的工作就知道了。

这被认为是 21 世纪性别歧视者的工作，这是这些专业混合的直接结果，人们会为此支付额外费用。

# 样样精通，样样精通

对我来说，成为一名全栈开发者通常需要一项核心专业技能。

这通过使用**技能堆叠**允许他们提供**独特视角**来开发和发布软件。

我可以自己为一个 web 应用程序构建前端，或者委托给我的前端开发人员。

在反应和角度方面，谁能比我更快更好地完成它呢？

这解放了我自己，让我在创建 REST APIs 的过程中变得更好。

使用 **Postman** 和 **OpenAPI** 等工具在 **Flask** 或 **Django REST** 框架中原型化并创建 API。

# 结论

我希望将来会有更多关于全栈开发者的文章关注于**技能栈**。

显然，我不是唯一的全栈开发者。因为我的工作和在新加坡帮助开发者聚会。

我也遇到过许多技能堆积者。来自一位专注于提供各种会议视频记录的高级开发人员，名为**工程师。SG** 帮助提升开发人员的技能。

另一个是我工作中的同事，由于她以前在**法律**专业的知识，创建了优秀的文档，并且可以轻松处理 **openssl** 库。
