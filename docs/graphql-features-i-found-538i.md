# 我找到的 GraphQL 特性

> 原文：<https://dev.to/tigerdevloper/graphql-features-i-found-538i>

现在自己在学 GraphQL。起初，我不明白它是什么，为什么需要它。
我们已经有了 REST API，为什么还需要 GraphQL？
我一遍又一遍地阅读这些文档，现在，我终于知道它是什么以及如何使用它了。
当然，我承认它比 REST API 强大和有用得多。
我很确定它迟早会取代其他所有的 API。

那么我发现的主要特征是什么呢？我发现 GraphQL 有两个最有用和强大的特性。

一个是它使用一个端点。由于这个特性，使用 GraphQL 查询非常灵活。

另一个是它有订阅功能。今天，许多应用程序的一个重要需求是实时连接到服务器，以便立即获得重要事件的通知。
对于这个用例，GraphQL 提供了订阅的概念。
它只是打开一个新的稳定的到服务器的连接，每当一个特定的事件发生，服务器就把相应的数据推送到客户端。

我还是 GraphQL 班的新手，所以希望大家对我的文章提出意见。谢谢。
