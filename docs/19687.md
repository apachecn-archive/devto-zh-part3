# 正如一个死去的企业开发人员在树旁所说

> 原文:[https://dev . to/vivainio/as-speed-by-a-dead-enterprise-developer-by-the-tree-1nh 2](https://dev.to/vivainio/as-spoken-by-a-dead-enterprise-developer-by-the-tree-1nh2)

程序员说，

**1。就像一个好的牧羊人温柔地纠正一只任性的羔羊的路线一样，应用程序的牧羊人将代码引向更安全的路径，而不是强迫或责备代码。**

**2。**有人告诉你，在 Windows 中应该使用反斜杠作为路径分隔符。但是要知道反斜杠在我家是很讨厌的，只应该作为转义字符使用。让没有反斜杠就崩溃的 Windows 软件崩溃吧，让我们不要再谈论那个软件了。

**3。想得太多的人看着一个想法，认为他可以做得更好，花上几个晚上为完美的实现而喋喋不休，却不输入程序。但是他聪明的邻居实现了他的第一个想法，看到了它是如何运作的，并相应地修改了它。他的邻居被赋予财富和尊重，而他却挨饿受冻。**

**4。你说你应该依赖于接口而不是实体。然而，您将接口与实现放在同一个 dll 中，就像一个醉汉把裤子也弄脏了一样。要知道，如果不拖动实现就不能使用接口，那么接口是没有用的。**

**5。如果你注入你的依赖项，你可以在那些依赖项中有不同的生命周期和状态。但是当你足够仔细地观察时，瞬态依赖几乎就像一个静态函数。**

程序员说，

**6。你被告知要编写许多单元测试。然而，告诉你的那个人不会在自己家里写单元测试。言语是风，判断一个人是看他的行为，而不是他的言语。**

**7。**傻青年看到新图书馆，知道长辈不知道。因此，为了获得妻子，年轻人炫耀他的知识超过他的长辈，因为他肯定比他们更聪明。相反，一个聪明人知道《图书馆》是另一个失恋的年轻人写的，也是为了打动别人。年轻人从智者那里学到了这一点，在 JavaScript 之外学习计算，在他的时代赢得了一个体面的妻子和一个好房子。

**8。愚蠢的老人想教他村子里的年轻人。但是他们不会听他的，因为他老了，破产了。不要做那个老人。相反，做一个聪明的老人，倾听年轻人的心声，因为他们走得更远，有自己的战争和征服故事要讲。然后看看哪些是你能用的，哪些是你必须丢弃的。**

程序员说，

**9。有人说面向对象是垃圾，函数式编程太难。但是聪明人知道，适度地进行面向对象编程并不是垃圾，适度地进行函数式编程并不难。而在 Go 中编写简单程序的乡村白痴，对两者都嗤之以鼻，肚子里的肥肉来自丰富的食物和美好的生活。**

10。但是要知道 Haskell 注定要默默无闻，因为默默无闻它被写出来，在默默无闻中它茁壮成长。

**11。**文本是字节，写成 UTF-8。它的开头不应该有字节顺序标记，因为这是令人讨厌的。不要再说那件事了。

程序员说，

**12。许多你尊敬的人使用铁锈，你也渴望如此。但是在他们修复他们的编辑器支持之前，你不应该使用 Rust，因为编辑器支持仍然很差。**

13。智者看代码，寻找表现力和清晰度。但是同样聪明的人接受丑陋的语言，如果工具是由对质量有狂热追求的富有的主人制造的。美丽的程序不会在冬天的微风中温暖你，但快速的编译器可能会。
