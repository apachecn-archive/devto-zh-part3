# 写一份不烂的简历

> 原文:[https://dev . to/recursivefaults/write-a-resume-that-suck-9ko](https://dev.to/recursivefaults/write-a-resume-that-doesnt-suck-9ko)

对于今天的开发人员来说，他们在申请最初的几份工作时有很多事情要考虑。他们建立了一个伟大的投资组合吗？他们会投资策划一个伟大的 GitHub 档案吗？他们会更新简历吗？他们应该起草一份理想的求职信吗？

底线是，几乎在每种情况下，开发人员都必须为一份工作投简历。如果他们有一个很好的 Github 个人资料，他们将不得不发送一份简历。即使是公司内部的朋友在推荐，简历也必须寄出。有一个华丽的投资组合？太好了，请发送您的简历，开始这一过程！

写一份好的简历是求职过程的基础。它是一种被传递的货币，可以把你排除在机会之外，把你从一堆其他简历中区分出来。

## [](#my-story)我的故事

我完成了计算机科学的学士学位，然后去了日本一年。当我回来时，我开始在软件开发领域找工作。没人会雇我。

我有一份我在大学某堂课上写的简历。我的简历是一些糟糕的东西，我拼凑起来完成一项任务。我开始努力让我的简历不那么令人讨厌。

几个月后，几乎每次我投简历都会收到面试的请求。这是没有软件经验的。

## [](#know-your-audience)了解你的受众

人们会阅读一份长达 90 秒的简历。通常他们会看起来比这少。这样考虑这个挑战:你有 90 秒的时间说服他们面试你而不是其他人。

你必须说服人力资源部的人和开发经理。

你有 90 秒的时间说服他们花时间去发现你是否值得。

当你阅读我的建议时，想想这些人可能想在简历中看到什么。你能让他们对你说“好的”有多容易？

## [](#skills)技能

我多年来使用的基本结构以突出我的技能的部分开始。

我从这里开始，因为第一个看我简历的人往往是人力资源部门的人，他们不懂软件开发。他们有一份由其他人写的工作清单，他们试图与之匹配。他们会在很浅的层次上这样做。这意味着他们会寻找匹配的关键字。

如果一份招聘启事列出了 Java，那么它在我的技能列表中排在第一位。

当人力资源部的人阅读我的简历时，他们会匹配大多数关键词，这将把我放在一堆要传给下一个人的人中。

当你写简历的时候，把你认为重要的关键词放在技能部分。

也许可以作为一个注解，告诉你什么时候可以，什么时候不可以把这些技能写进简历。没有一个神奇的人允许你在简历上写东西。想放什么就放什么。如果你撒谎，你会被抓住的。

如果你撒谎被抓，它会一直跟着你。在你的职业发展过程中，你会发现我们只有这么多人，而且我们会很容易记起那个谎报技能的孩子。当我们再次看到它们时，这是我们首先想到的。在他们第一次撒谎后，我会考虑再花时间在他们身上吗？

我并不是说你必须成为你所列举的每一个领域的专家，但是你确实需要在这些技能上展示一些能力。

## [](#experience)经历

简历的第二部分是我的经历。开始这可能是相当薄或真的很难变戏法。

对于你列出的每一份工作或每一段经历，包括这些元素:公司名称、工作年限和你创造的价值。

注意，我没有说要包括你的头衔。我也没说要列出你在工作中做了什么。

很多人陷入了在经历中罗列工作职责的陷阱。猜猜软件开发人员的工作职责是什么？建筑软件。跳过那个。跳过你做建筑的部分。跳过你记录、设计、测试、重构的部分。这确实是每个人简历上的内容。它还告诉我，除了你去上班，尽了你最起码的义务之外，什么也没有。

相反，你所写的是你所产生的影响，你所交付的成果，或者你所带来的价值。这需要一些实践来想出。这里有一个人为的例子:

来自:“使用 react、node 和 Postgres 构建了一个 web 应用程序。”
To:“构建了一个应用程序，带来了 120 万英镑的收入。”

想猜猜根据这两个陈述谁会被叫去吗？不会是上市的人反应过来的。

随意强调你想引起开发经理注意的某些事情。将你的技能融入到这些事情中，但是要专注于结果。在这些要点中列出一项技能是为了让他们大吃一惊。例如，如果您知道他们正在使用特定的技术，并喊出了扩展，那么这是一个绝佳的机会来描述您使用该技术解决的扩展问题。

否则，保持关注价值。

你可能已经注意到我在那个例子里放了一个数字。数字能吸引人们的注意力。他们会在面试中想知道背后的故事。这很棒，有两个原因。首先，你得到了他们的关注，这增加了你面试的机会。其次，你增加了你知道他们在面试中可能会问你什么的机会。你可以在预期的问题中练习你背后的故事。

至于我为什么建议不列书名。招聘人员可能会要求你把它们加进去。我建议不要把它们列出来，除非有人问我，因为不是所有的头衔对同一家公司来说都意味着同样的事情。如果我的公司没有初级开发人员这个头衔，但这正是你的头衔，我现在必须调和这个概念。这个学妹会是谁呢？他们没准备好吗？

至于每个经验模块要添加多少项目符号，我认为每个公司应该添加 3 到 5 个。

## [](#nondevelopment-experience)非发展经验

那种与软件开发无关的体验呢？好吧，那些工作教会了你一些有价值的东西，这些东西将伴随你的余生。无论你走到哪里，你传递价值的能力都值得发扬光大。把那些早期的非发展性工作看作是突出你为这份工作带来的其他非发展性东西的机会。

你有没有一份可以用来突出在高绩效团队中工作的工作？太好了！你有没有一份可以谈论完成一个雄心勃勃的目标或项目的工作？太好了！找到了一份可以突出沟通、促进、教学、指导、设计、营销、广告、金融等技能的工作？太好了！把那个放下。向他们展示你是独一无二的，并带来比写代码更多的东西。

## [](#summary)总结

通过将你的简历聚焦于你的受众，并在 90 秒内最大化你的影响力，你可以在面试中脱颖而出。

让你的技能很容易被人力资源发现，这样他们就可以把你传递下去。包括你可以讲述或展示的技能。

为了你的经验，列出公司和日期。用 3-5 个项目符号来展示你的价值和成果。忘记放下你的职责和任务。

不要害怕列举非开发经验。这是一个展示你发展能力之外的所有其他技能的机会。关注价值和结果。

我使用了我提到的每一种技巧来建立我的简历。即使在我没有“经验”的时候，这些技巧也帮助我不断地获得面试和工作。
