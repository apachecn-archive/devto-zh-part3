# 数据可视化术语

> 原文：<https://dev.to/stereobooster/data-visualization-terminology-3okl>

**尺寸** -参数，特性。在几何、平面、绘图等背景下的轴。列，在 RDBMS(关系数据库管理系统)和 OLAP(在线分析处理)的上下文中。例如，年龄、性别、身高、颜色、坐标、时间等。

**基数** -维度中元素的数量(最初在一个集合中)。例如，性别是一个低基数的维度，坐标是一个高基数的维度。

**连续，离散** -维度是连续的，如果它可以被无限分割成更小的部分，通常只适用于柏拉图式的想法。例如，被认为是连续的空间和时间。实际上，人类进行的所有测量和我们可以存储在计算机中的信息都是离散的，因为它受到测量工具精度的限制。因此，当我们在计算机中处理连续维度时，我们实际上是在处理离散近似(通常具有高基数)。

**可比较的，不可比较的** -数字维度是可比较的，因为数字可以被比较并排序。但是有一些特征是不可比较的——你不能说一件物品更大(更亮、更重等等)。)比其他。例如，性别是一个不可比较的维度。你可以按字母顺序对性别进行排序，但这没有意义，而且在不同的语言中，排序会发生变化。

**分类** -当你处理高基数(高于你需要的)的维度时，你可以通过分组条目来降低基数。例如，您可以创建两个类别:大于或等于 50 岁，小于 50 岁，而不是为所有年龄值构建图表。

**笛卡儿积** -(源于集合论)，二维所有可能的值的组合。例如，几何平面由可以用两个坐标轴(因此是二维或笛卡尔坐标系)表示的点组成。或者我们有两个低基数的维度，我们可以用笛卡尔积来创建一个维度。例如，我们有关于人口性别(男性、女性、非二进制)和年龄的数据，使用年龄分类(> = 50，< 50)和笛卡尔乘积，我们可以实现一个有 6 个值的维度。

**坐标系** -一个或多个维度的组合。例如，笛卡尔坐标系(用于简单绘图)、极坐标系统、地理坐标系(纬度和经度)等。

**投影**——是一种在不同的维度系统中表现一种维度系统的方式(变换)。比如二维空间有表示三维空间的方式，像
多视图投影，轴测投影；有很多方法可以在地图上表现地球表面，比如墨卡托和其他的 T4。投影会导致某种信息损失(如轴测投影)或失真(如墨卡托投影)。

**比例** -是原始坐标相对于给定图形显示的大小的参考。例如，地图中使用的线性比例或数量变化很大时使用的对数比例，但我们对大数量和小数量的细节感兴趣。

离群者或异常者(Outlier 或 anomaly)——在一个特定的群体、阶层或类别中是非典型的人或事物。这个术语适用于数据而不是可视化，但是我们可以在图上单独标记异常值。

**图**——由线连接的点的网络(数学术语)。在数据可视化中，我们讨论的是数学术语的可视化表示。图形的主要思想是显示元素之间的关系。从这个意义上说，它与其他主要目的是显示维度之间关系的可视化没有什么不同。
