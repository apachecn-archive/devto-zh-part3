# [](#programmers-love-their-names-and-acronyms)程序员喜欢他们的名字和缩写

也许比其他行业更重要的是，我们以命名我们的程序为荣，并从中获得乐趣。名字很重要，因为它们不仅是我们谈论代码的方式(很明显)，还可以添加重要的上下文信息
以 Ruby 和 Ruby Gems 为例。Ruby 是编程语言，Ruby Gems 是 Ruby 的库。宝石让你想到什么？一些小的，独特的，有价值的东西，对吗？我们可以对图书馆说同样的话！它们是“小”的代码集合，做一些对我们有价值的独特的事情。

有人可能仅凭名字就能猜出 ruby 的宝石是什么。

再拿撬来说。Pry 用于“窥探”我们的代码，并查看其内部工作情况。

或者以 Windows 为例，它是一个窗口应用程序的集合！

或者 ISF -交互式着色器格式，这是一种交互式的着色器格式！

这并不意味着你不能从你的名字中得到乐趣，许多公司喜欢在他们的名字中加入参考，愚蠢甚至递归！看看流行的 linux 操作系统 Ubuntu 的创造者给他们的版本起了什么名字...这些名字可能看起来很傻(因为它们确实很傻！！)但是很好地服务于目的，因为它们是独特的并且容易记住。比起 Ubuntu 19.04.01 LTS，你可能更容易记住“Disco Dingo ”,另外“Disco Dingo”本来就是 Ubuntu，而 19.04.01 LTS 可以很容易地引用另一个操作系统。

# 那么，为什么这些都很重要呢

那里有很多代码，我们需要能够清楚有效地谈论它们。当您根据代码的名称和使用的上下文来猜测代码是做什么的时候，这将非常有帮助。

如果你的名字很难发音，人们会叫它别的名字，你可能不想这样

如果你的名字永垂不朽呢？我们还是说说 FORTRAN 吧(最新稳定发布是 2018 BTW！)尽管那是 20 世纪 50 年代制作的。

如今，社交媒体和搜索引擎优化被捆绑在一起，这很糟糕，但如果你的代码有一个独特的名字，人们可以很容易地记住和搜索，它可能会更受欢迎。SEO 很重要！我甚至不记得上一次去谷歌搜索的第二页是什么时候了

## [](#but-is-it-actually-important)但这真的重要吗？

不见得...人们会使用你的代码，如果它是好的，不管它的名字是什么，JavaScript 是世界上最大的实现语言之一。

# [](#what-makes-a-good-nameacronym)什么是好名字/缩写？

使其独特/显著，这并不太重要，但一个独特的易于搜索的名称将使其更加显著，并提高搜索引擎优化

要说得简单，不要给你的代码起一个很难发音的名字，否则人们会叫它别的名字。

让它主题化，就像 Ruby - Gems 一样，让相关的软件在一个主题中命名可以使它们的关系内在地有意义。

让它有意义。如果你调用 Student_controller，它应该控制学生，而不是处理服务器安全

专业一点。说脏话或者其他不合适的话。尽管 Brainfuck 或者 My Little Pony interpreter 很简洁，但我可能不会和我的老板谈论这件事。

你想怎么做就怎么做。说真的，在一天结束的时候，你可以随意命名你的代码。如果人们想使用你的代码，他们会不管它的名字，我的意思是...JavaScript 是世界上实现最多的语言之一。

但是帮助全世界的程序员，让你们的名字好听！这延伸到代码中的任何东西，从类到方法到变量到应用的实际名称。在一个专业的环境中，不是你的人会阅读你的代码，进行编辑、实现、篡改，谁知道还有什么。将来甚至可能是你！帮助你自己和你的同事，有一个好名字
