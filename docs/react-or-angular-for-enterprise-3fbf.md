# 对企业有反应还是有角度？

> 原文：<https://dev.to/embiem/react-or-angular-for-enterprise-3fbf>

最近，我和同事们讨论了 React 和 Angular 在企业应用中的区别。我也想把这个讨论带到开发者社区。

核心论点是:

1.  React 自带的内置工具比 Angular 少。因此，React 设置需要通过安装开源依赖项来补充 vanilla React。这使得新开发人员加入使用 React 构建的项目变得更加困难，因为任何未知的依赖项都需要首先了解。而 Angular 有必要的概念和内置工具，Angular 开发人员已经知道这些概念和工具，因此可以更快地进入新项目。

2.  Angular 默认情况下使用 Typescript，并且在构建时就考虑到了 Typescript。React 不是。React 中的某些模式很难或很难正确输入。由于在企业项目中，Typescript 通常比 JS 更受青睐，Angular 是更好的选择。
