# JS 界是不是忘了聪明不聪明？

> 原文:[https://dev . to/polymorphiprod/did-the-js-community-forget-smart-is-not-smart-1bnl](https://dev.to/polymorphicprod/did-the-js-community-forget-clever-is-not-smart-1bnl)

首先，我想对社区说几句感谢的话。如果没有那些贡献工具和脚本与世界分享的少数人的努力，这一切几乎都不可能实现。

也就是说，我个人认为他们现在同样的努力可能正是正在毁灭/破坏一切的东西。

# 聪明对社会有害，但这没关系。

每个人都在某个时候发现了一个需要艰难解决的难题。你可以说，今天问题的增长速度超过了解决问题的速度。我也坚信，有时候你只需要让它工作，那就可以了。有鉴于此，你应该努力让事情保持原样，即使它们是“坏”的，那也没关系。显然，完美是好的敌人。既然我们都认为聪明的事情有时会发生，如果真的发生了，我们也不应该感到难过，让我们继续吧。

# [](#why-clever-is-not-smart)为什么聪明不聪明？

聪明应该不难。聪明是简单的，聪明是容易的，它对所有感知它的人都有意义。它之所以聪明，完全是因为它有效、可重复且简单。所有这些都与聪明背道而驰。

# JS 社区为什么要这么巧？

首先，我不是 js 开发专家。见鬼，我甚至都不是一个真正的中级开发人员。不过，我敢打赌，我代表了社区中的很大一部分人，他们觉得由于过去 5 年来一大批聪明的开发者，他们已经完全被排除在这个行业之外了。这并不是说所有的新事物都聪明或不聪明。例如，“析构赋值语法”是惊人的，非常聪明。几乎每个人都马上明白了，而且很有效。遗憾的是，对于我们社区的其他主要进步却不能这么说。让我们看一下 Webpack。虽然它无处不在，每个人都在使用它，但我敢打赌不是每个人都马上得到了它。我打赌你们很多人还是不明白。然而，它就在这里，很可能在你现在正在做的至少一个项目中。好吧，我们不要责怪 Webpack，它只是每个人都在努力做的事情的反映。它试图解决困难的问题，所以自然有时会有困难的解决方案，我们都同意这没什么。

这里到底发生了什么？首先，每个人都在提高自己的标准，这很好，对吗？你们都知道我在说什么。

*   “您不需要为所有事情都使用 jQuery”
*   "你今天应该使用 ES6(巴别塔)"
*   “命令式 vs 声明式”
*   电休克疗法..

因此，问题自然会发生变化，通常会变得越来越难，解决方案也是如此。

让我们通过一个流行的库来看另一个例子。

http://photoswipe.com/可以说是照相馆的最佳选择。自然，这意味着大多数新开发人员会像我一样在谷歌上查找如何在 js 中添加照片库。由于它支持纯 js，它自然会赢得 lightbox 的支持，我认为 light box 仍然在使用 jQuery，因为这是我们告诉社区要避免的。

因此，当新开发人员去阅读示例代码，了解他们如何将这个库集成为自己的照片库时，他们会遇到类似这样的情况。

```
var closest = function closest(el, fn) {
    return el && ( fn(el) ? el : closest(el.parentNode, fn) );
}; 
```

当我们坚持认为您应该对这些代码足够熟悉，甚至可以开始使用这个库时，我们向社区传递了什么信息？

这是试图初始化该库的示例的一部分！

当然，他们注意到在 read me 中使用这个库需要一个学习过程，但是，这是他们的意思吗？

我恳求你们所有人去观察你们所认为的聪明，并真正地问自己，你只是在变聪明吗？如果是这样的话，没关系，但是，让我们做更多的事情，不要把它作为标准，因为这会伤害我们整个社区。
