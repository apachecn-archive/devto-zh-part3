# 生成器设计模式[创建]

> 原文:[https://dev . to/its coders life/builder-design-pattern-creation-3gj 7](https://dev.to/itscoderslife/builder-design-pattern-creational-3gj7)

何时使用构建器模式？

*   当我们需要创建需要许多配置值的复杂对象时，使用构建器模式。
*   当要创建的对象通常可以使用一些预定义的默认值，并且很少被修改时，这是一个很好的选择。
*   呼叫者可以自定义这些默认配置值，但大多数情况下默认值保持不变。
*   这种模式封装了默认的配置值和在构建器类中创建对象所需的设置逻辑。
*   构建器模式将对象的配置与其创建分开。
*   它简化了需要许多默认值的实例的创建，这些值在创建新对象时通常不会频繁更改。

> 构建器模式将对象的创建与其配置分开，从而简化了复杂对象的创建。

示例:

*   web 应用程序或移动应用程序的主题生成器——背景颜色、导航栏颜色、文本颜色大小等。深色主题或浅色主题都有自己的一组值，您可以根据需要进行自定义。主题构建器用于代码编辑器和移动应用程序中
*   网络会话管理器–在这里，会话以一组缺省值开始。
*   麦当劳里的一个食物构建器——用户只需说出食物的名称，食物的所有部分都会被构建出来，用户无需参与复杂的创建过程。然而，用户可以根据需要定制。

**总结**:当实例意味着复杂的配置步骤，并且在创建新对象时不需要频繁更改设置时，构建器设计模式是最佳选择。当您需要将对象的配置与它们的创建分开时也是如此。调用者只提供最小的配置值。builder 类包含构建对象时设置默认配置值所需的所有逻辑。
