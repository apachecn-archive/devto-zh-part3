# 在 dotConnect 中支持 PostgreSQL 11，用于 PostgreSQL、LinqConnect、SSIS 组件和由 Devart 开发的实体

> 原文:[https://dev . to/devart software/support-for-PostgreSQL-11-in-dot connect-for-PostgreSQL-linq connect-ssis-components-and-entity-developer-by-devart-4p3i](https://dev.to/devartsoftware/support-for-postgresql-11-in-dotconnect-for-postgresql-linqconnect-ssis-components-and-entity-developer-by-devart-4p3i)

数据库连接解决方案和数据库管理与开发工具的捷克软件提供商 Devart 在新发布的 dotConnect for PostgreSQL 7.12 和 SSIS 数据流组件 1.10 以及 LinqConnect 和实体开发人员产品中引入了对 PostgreSQL 11 的支持。现在可以用 Devart 产品连接到 PostgreSQL 11，并使用它的新特性，比如通过散列键对数据进行分区，在 WHERE 子句中对表达式进行 JIT 编译，等等。

此外，Entity Developer、LinqConnect 和 dotConnect 完全支持 PostgreSQL 11 存储过程，可用于映射等。此外，与存储函数相比，在 dotConnect for PostgreSQL 中执行存储过程有一些独特之处。

dotConnect for PostgreSQL 是一个支持 ORM 的增强型 PostgreSQL 数据提供程序，它建立在 ADO.NET 技术的基础上，为开发基于 PostgreSQL 的数据库应用程序提供了一个完整的解决方案。

SSIS 数据流组件允许通过 SQL Server Integration Services(SSIS)集成数据库和云数据，并使用 SSIS ETL 引擎以及高性能数据加载、便捷的组件编辑器、对云数据源的 SQL 支持和大量数据源特定功能，提供易于设置的经济高效的数据集成。

LinqConnect 是一个快速、轻量级且易于使用的 LINQ 到 SQL 兼容 ORM 解决方案，支持 SQL Server、Oracle、MySQL、PostgreSQL 和 SQLite。它允许您为您的使用高效和强大的数据访问。NET Framework、Metro、Silverlight 或 Windows Phone 应用程序支持代码优先、模型优先、数据库优先或混合方法。

Entity Developer 是 ADO.NET 实体框架、NHibernate、LinqConnect、Telerik 数据访问和 LINQ 到 SQL 的强大 ORM 设计器。它引入了设计 ORM 模型的新方法，提高了生产率，并促进了数据库应用程序的开发。
