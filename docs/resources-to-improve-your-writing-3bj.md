经常有人问我，“有哪些学习技术写作的好地方？”这个页面将是一个不断发展的文档，列出了论坛、组织、甚至书籍(喘息！)帮助我提高了比赛水平。我可能会包括一些看起来很优秀的人，即使他们在我的特定职业生涯中并不重要。

## 对你的写作进行可靠、自动的反馈

*   海明威应用程序旨在提高你的写作水平。一般来说，它似乎有助于你写出非常干净、生动的或多或少适合科技写作的文章。它给出了许多关于如何写出好的、干净的散文的好建议。

*   [Readable.io](///conteimg/2017/06/logo.io.png) 根据不同的学术尺度给你各种我不懂的评分来评估文本。但我所理解的是它给出的简单的字母等级。它也给你一些统计上的反馈，比如你的单词中副词的百分比。

## 书籍和在线指南

### 英语和科技通讯用法

*   [切题:简明写作词典](https://smile.amazon.com/Point-Dictionary-Concise-Writing/dp/0393347176/ref=sr_1_1?s=books&ie=UTF8&qid=1497132167&sr=1-1&keywords=to+the+point+a+dictionary+of+concise+writing)。如果你能把每个短语从 3 个减少到 2 个，你就为你的读者节省了 1/3 的时间，他们会因此而喜欢你的。查找一个短语，这本漂亮的书提供了一个更简短的选择。当我感觉到我内心的诗人在推动我的技术交流工作时，我就用它。你用得越多，你需要的就越少。

*   [微软风格手册](https://www.amazon.com/Microsoft-Manual-Style-4th-Corporation/dp/0735648719/ref=sr_1_1?s=books&ie=UTF8&qid=1497131810&sr=1-1&keywords=microsoft+manual+of+style+5th+edition)适用于更具体的软件文档和格式用法的所有语言。

*   在线的《芝加哥文体手册》或[购买一本](https://www.amazon.com/Chicago-Manual-Style-16th/dp/0226104206)来解决所有关于语法和用法的问题。

*   风格的要素，因为它便宜、简洁，是如何写出干净简单的散文的绝佳指南。如果你没有读过，你可能不擅长写作。开玩笑的。算是吧。

*   [完整的普通文字](https://www.amazon.com/Complete-Plain-Words-Ernest-Gowers/dp/1567922031/ref=sr_1_1?s=books&ie=UTF8&qid=1497132032&sr=1-1&keywords=complete+plain+words)，基本上是一种更长的形式*的风格要素*并且——我不是在开玩笑——适合日常阅读。它教给你的更多的是一种心态，而不是记忆规则，并帮助你学会同情你可怜的读者。

### 科技通讯的手艺

*   [开发质量技术信息](https://smile.amazon.com/Developing-Quality-Technical-Information-Handbook/dp/0133118975/ref=mt_paperback?_encoding=UTF8&me=)

*   [现代技术写作](https://www.amazon.com/gp/product/B01A2QL9SS/ref=kinw_myk_ro_title)专注于软件文档，是一本书的瑰宝。

*   由 Madcap Flare 的人编写的创建 API 文档的权威指南，这是技术写作的行业标准。

## 组织

*   [技术交流协会](https://www.stc.org/)是一个现实生活中的组织，旨在帮助技术交流人员建立网络并改进我们的技术。该协会制作工资指南、同行评审期刊和在线杂志，以及许多活动和年度会议。声明:我是会员。警告:不便宜。建议:我发现这是值得的，否则我不会继续更新。

*   从他们的网站上写文档:“写文档是一系列的会议和本地聚会，关注所有与软件文档相关的事情…我们认为每一个关心交流、文档和他们的用户的人都是我们社区的成员。这些人可以是程序员、科技作家、开发者支持者、客户支持者、营销人员，以及任何希望人们拥有出色软件体验的人。”
