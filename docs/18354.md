# 别人的问题

> 原文:[https://dev.to/craignicol/somebody-elses-problem-3d82](https://dev.to/craignicol/somebody-elses-problem-3d82)

有时候划分任务会造成效率低下。当一个人既做饭又打扫卫生时，他们比分工负责时更少洗碗，因为洗碗变成了“别人的问题”。但是，一个人洗，一个人晾，负荷是相当的。当排水器是空的并且洗衣机试图从锅上去除顽固的烧焦的部分时，这种设置可能容易停止，此时任何好的 ToC 实践者都会要求干燥机帮助洗涤以保持产量。

当团队按层而不是按特性划分时，他们之间的界面就变成了裂缝，因为每个团队都缺少其他团队的上下文。数据具有错误的形状，验证需要无法捕获的字段。如果你两边都建，过渡是平滑的。

当开发人员测试时，他们可以关注更小的、独立的块，这意味着更少的测试和更大的覆盖率。为复杂的计算编写集成测试既痛苦又缓慢。测试与 xUnit 理论相同的计算更干净、更快。

专家很棒，但是专业化导致骨折和更多工作在哪里？
