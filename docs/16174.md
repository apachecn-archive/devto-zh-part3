# 你如何弥补知识差距？

> 原文:[https://dev . to/maniframes/how-do-you-approach-knowledge-gap-44n 4](https://dev.to/maniflames/how-do-you-approach-knowledge-gaps-44n4)

当你试图构建一些东西时，有时会遇到困难，这是很正常的。然而，当我一次又一次地犯同样的错误时，我知道这与我对某个主题缺乏了解有关。

当它是一个特定于问题的框架或库时，答案很可能在文档、堆栈溢出或教程中。然而，当它与语言功能或理论知识有关时，你可能更难发现你不知道的东西。公平地说，当我需要的时候，我偶然发现了正确的信息。它通常隐藏在一次谈话或一篇写得很好的文章或帖子中。

我所说的“问题”有两个真实的例子:

#### [](#1-understanding-js)1。了解 JS

当我开始使用更多的异步 JS 时，我试图写的东西的期望和现实并不总是匹配的。一些修补或变通通常会解决问题。直到一个非常热心的 JS dev 建议我去看 event loop 上的一个讲座，我才意识到自己的知识差距有多大。我不知道 JS 是如何在幕后处理异步操作的，也不知道为什么有些事情行不通。

如果你没有看过，并且你喜欢 JS，看看菲利普·罗伯茨的“到底什么是事件循环”。他谈论的不仅仅是异步编程，很多与 JS 相关的流行词汇在听完这个演讲后都会变得有意义。

#### [](#2-understanding-rust)2。了解铁锈

我现在正在努力解决的是 Rust 中的所有权和借款问题，尽管这本书有整整一章的内容。它基本上与你的代码所使用的内存以及你是否尝试使用真实的值，引用或者拷贝有关。

当我试图在 Rust 中做一些事情时，我经常与借检查器发生冲突。经过一些练习，我开始变得好一点了，至少可以理解为什么我会停滞不前。我看到了这篇文章，如果你碰巧在所有权问题上挣扎，这篇文章将会很有帮助。

## 文章不再可用

公平地说，Rust 中大多数与内存相关的东西现在对我来说都是个问题，因为我对它了解不多。我想这是我必须理解的事情，至少要更好一点，才能真正理解我在做什么。

你如何揭示你的知识差距是什么？你有没有最喜欢的方法来学习更理论化的东西并加以应用？我很想听听其他开发者是如何处理这个问题的😄。
