# 软件 CSI，未来的工作

> 原文:[https://dev . to/damn Jan/software-CSI-a-job-of-the-future-36d 8](https://dev.to/damnjan/software-csi-a-job-of-the-future-36d8)

这一定是未来的工作职位，也许它不会被称为 CSI SE，但它会像在这个帖子中描述的那样。有人必须调查所有的犯罪现场，肮脏的黑客，深夜补丁，死代码，不良做法。我们在大多数地方都有足够的开发人员，但是由于缺乏良好实践和纪律，我们浪费了他们的力量来创建漂亮的结构。相反，软件开发人员习惯于创造更多的坏代码。

### [](#how-i-come-with-the-idea-of-crime-scene-investigation-software-engineer)我是如何产生犯罪现场调查软件工程师的想法的

<figure>

我妈妈问我在新的工作场所每天的任务是什么，我的工作岗位是什么。我可以告诉她，我是一名高级软件开发人员，每次换公司时，这都是一个相当乏味的回答，但这次感觉不一样了。我的工作不仅仅是跳代码，朝九晚五地工作。我得到了一份工作，因为我能给公司带来一些特殊的技能，一些特殊的知识，以及对他们需要的新想法的渴望。该公司已经开发了软件，但它不在任何部署的管道中，没有经过测试，非常脆弱，并以某种方式进行维护以保持其活力。必须有人跳到那里，观察哪里有错误，发生了什么软件犯罪，将来如何避免等等。
回我妈。她以前看过 CSI Miami(以及其他 CSI 相关的电视剧)，对概念、犯罪现场、调查都相当熟悉。

### [](#csi-se-definition)CSI SE 定义

定义 1(长一点):
CSI SE 是 scrum 高手但不组织团队，那么对于组织代码
定义 2(长一点):
CSI SE，短来自犯罪现场调查软件工程师是能够在软件中识别、探索和检查犯罪现场的人。一个理解并已经经历了所有不良实践、反模式、不良工程行为和可重复错误的人。
CSI 工程师表现得像一个**软件法医**，他看到死代码、意大利面条代码、脆弱和肮脏代码的犯罪现场，他试图首先了解原因，然后提出解决方案。

### [](#why-we-need-this-position)我们为什么需要这个职位

<figure>

我们到达了一个点，找到一个开发职位的人不再那么困难。但是要得到一个好的，那是非常困难的。此外，许多公司与他们的开发力量斗争，他们有足够的开发人员，高薪，工作时间似乎是公平的，但仍然，他们与质量斗争。编写糟糕的代码，匆忙地编写代码并部署它，只让东西工作，但不遵循任何模式或实践，这变得可以接受。
所有这些都发生了，因为不久前，我们的工程师严重短缺，所以每个人都得到了工作，不管他们拥有什么技能或正规/非正规教育。好的实践或模式没有被强制执行，因为员工害怕他们会失去开发人员。
所以每个键入代码的人都称自己为开发人员，即使他们学得很差或者从来没有学过，从来没有测试过，并且继续努力回避他们的软件，但是似乎没有人发现问题。
这就是我们对 CSI SE 的需求。一个新的工作岗位，为现有的开发团队带来知识、实践、纪律和指导方针。犯罪现场调查软件工程师应该能够立即发现上述所有故障，并提出一个团队可以接受的解决方案。

### [](#what-is-a-benefit-for-the-company)什么是对公司有利

雇佣 CSI SE 的公司在短期和长期都有优势。优秀的 CSI SE 可以立即向团队提出一些好的练习和实践。
例如，如果有许多地方有“死代码”，那么 CSI SE 可以提出，所有的“死代码”都应该以特定的方式处理([参见此处的示例](https://dev.to/damnjan/dead-code---alive-problem-2job))。
如果问题是不清楚的 git 消息、太不频繁的提交或不一致的命名约定，也可以应用容易接受的解决方案和实践，这对每个人来说都是快速的胜利。
从长远来看，CSI SE 应该发现开发技能、开发实践和工作流中的弱点，并引入学习或持续运动来改进。学习可以是会议、结对编程或培训，以及研讨会，而通过持续的运动，我考虑在我们想要去的地方和我们必须实施的步骤建立策略和计划。

### [](#is-this-an-executive-role)这是执行角色吗？

不一定。CSI SE 可以像团队中的任何其他开发人员一样，只是有一个特定的辅助角色。如果我回到这个定义:CSI SE 是一个 scrum 大师，但不是组织团队，那么对于组织代码来说，它清楚地表明 CSI SE 不是一个团队领导或 CTO 的角色。CSI SE 只是团队的一部分，其额外任务是确保代码持续改进并消除障碍。
CSI SE 还需要能够向团队中的其他成员传授知识。如果她/他是同一层级团队的一员，那么在飞行中教导他人会容易得多。

### [](#conclusion)结论

我预计未来几年我们将对 CSI SE 有更高的需求，也许行业不会这样称呼它，但我想描述会像我刚才描述的那样。消息对于一个想跳进这个浪潮的人来说，学习好的 CI/CD 实践，学习 TDD，学习好的敏捷和极限编程实践，学习如何转移你的知识，将会有很高的应用它的能力的需求的人。
