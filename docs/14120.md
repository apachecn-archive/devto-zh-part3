# 像蜜獾一样运送它

> 原文:[https://dev.to/honeybadger/ship-it-like-honeybadger-27jb](https://dev.to/honeybadger/ship-it-like-honeybadger-27jb)

很久以前，我和我的朋友遇到了麻烦。我们几乎每天用来为客户调试软件的一个应用程序基本上已经停止工作了，这让我们所有人都很头疼。当时市场上没有真正可行的替代品，所以我们决定自己造一个。三个月后，我们的应用——蜜獾——发布了。

在接下来的 5 年里，我们把 Honeybadger 打造成了我们真正引以为豪的东西，成千上万的开发者用它来调试异常和监控他们的 web 应用程序的错误。我们还建立了一家我们非常想去工作的公司，[拒绝了一个诱人的收购提议](https://joshuawood.net/badger-life)。

#### 我们自己完成了这一切，没有资金，只有 3 名开发人员。

如果你不相信我，这里有一张最近公司聚会的照片(我们都很遥远)。如果你想知道我们为什么穿蓝色，那是为了自闭症意识月:

当我告诉其他开发人员我们的公司有多小时，我通常会得到同样的反应:“不可能，我以为你只有 20 个人”(有一次有人说 100 个🤔)，“我希望有一天能[做我自己的了不起的事情]”。

#### 我的回答总是一样的:如果我们做到了，你也可以。

构建蜜獾是一项艰巨的任务——我们的时机有点幸运，但我们并不特别。事实上，我们相当普通(我甚至没有完成大学😬).

我能想到一些我们在 Honeybadger 上做对的事情，但是有一件非常基本的事情我把我们的成功归功于它:**我们发布了一个零碎的版本 1。**

它并不完美，也没有做到我们的客户今天喜欢我们的所有事情。不过，这没关系，因为这足以让我们开心，事实证明，还有其他像我们一样的开发者。如果我们浪费时间试图让它变得完美，它就永远不会起飞，我也不会在这里告诉你——不管它是什么——你也可以把它运出去。

发货并不意味着你必须启动自己的应用程序(虽然你当然可以)。你可能想写博客，录制视频，为开源做贡献，或者只是在你的工作/职业中踢屁股。你可以做所有这些事情——基本原理是一样的。

#### [](#in-hindsight-it-sounds-too-simple-oh-yeah-just-ship-it)事后看来，这听起来太简单了...“哦，对了，就装船吧。”

我知道比那更难。有很多时候你没有开始，因为你不知道如何开始，或者不相信你能开始。每当你想到最终执行你的计划时，那种令人麻痹的恐惧就会降临。每当你接近终点线时，总有一个愤世嫉俗的声音在低声说“这永远行不通”。

如果你把所有这些借口加起来，你就会得到塞斯·戈丁 所说的“阻力”从 10，000 英尺的角度来看，你所知道的大部分东西都在阻碍你，让你无法创作出最好的作品。要想出货，需要克服阻力。

#### 你可以通过小的、反复的改变来战胜阻力——是时候升级了。

无论你在职业生涯的哪个阶段，总会有下一个阶段。随着时间的推移，当你做一些小的改变时，构建软件就变得更容易了(没有人想要一个大的重写)。习惯也是如此。你运送的东西越多，运送就变得越容易。为什么不从今天开始呢？

选择你现在能做的最小的事情*，然后去做。发送拉取请求。与他人分享你知道的东西。研究一种新的工具或技术。读赛斯·戈丁的 [*【关键*](https://www.amazon.com/Linchpin-Are-Indispensable-Seth-Godin/dp/1591844096/) (我之前提到的那本书——挺好的)。*

 *养成运送小件物品的习惯；当大的到达时，你也要运送那个。🚀
