# 保存您的更改

> 原文:[https://dev.to/jessekphillips/save-your-changes-3l02](https://dev.to/jessekphillips/save-your-changes-3l02)

Git 喜欢变化，它希望你的变化安全。当改变分支时，git 会在切换过程中保持工作树的变化，但是如果它需要修改其中一个文件，它不会切换分支。

Git 希望保证您的更改安全，所以它强调您提交、隐藏(您也可以这样做)。它提供了使你的提交清晰的工具。互动 rebase 是你最好的朋友。

因为 git 非常了解文件树，所以您永远不需要克隆一个干净的文件夹。几乎总是有一个命令来清理你制造的混乱。当有疑问时，提交并切换分支。事实上，git 在这方面做得很好，如果你添加一个完全不相关的 repo 作为遥控器，你可以在一个文件夹中工作。不推荐。

不要使用 GUI，它们只是在没有警告的情况下破坏更改时弹出对话框。在命令行中，您运行安全操作，然后在验证销毁是所期望的时，使用- force 重新运行。
