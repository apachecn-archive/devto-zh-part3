# 修改日志:调整你关注的标签的权重

> 原文:[https://dev . to/dev team/changelog-adjust-the-weights-of-tags-you-follow-571k](https://dev.to/devteam/changelog-adjust-the-weights-of-tags-you-follow-571k)

新功能！

您现在可以调整您关注的标签，使某些主题更有可能出现在您的订阅源中。

### [](#you-can-adjust-your-tags-at-dashboardfollowing)您可以在[/仪表盘/跟随](https://dev.to/dashboard/following)处调整您的标签。

这是我现在的一个例子:

[![](../Images/07e496074a2ddd2dc5353ec74b115824.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Nx2kHRkD--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cl.ly/b329a5f8a8ea/Image%25202018-11-30%2520at%25204.03.13%2520PM.png)

我决定跟随最强壮的`meta`，同时也更紧密地跟随`ruby`和`rails`。`smarthome`和`management`似乎也是我不想错过内容的标签。

随着全局算法随着时间的推移而修改，这对提要的影响可能会有所不同，但它应该有助于人们从平台中获得他们正在寻找的体验。

随着我们不断发展标签的用途和功能，这应该是人们如何在平台上组织自己的一个中心点。如果你想在一段时间内更多地了解某个话题，那就加大它的权重。

此时，你*可以*减少你不想看到的标签，但是所有想要的行为还没有实现。你跟随一个标签的信号可能是正信号也可能是负信号。它通常会帮助你少看到一些东西，但完整的行为将很快得到解决。

很多用户可能不会去碰这些 nob，这没什么。我们不想创造过于复杂的体验。

> 简单的事情应该简单，复杂的事情应该可能

我认为这种改变有助于使复杂的事情成为可能，同时保持相同的基本默认设置简单。

## [](#whats-possible)有什么可能

这种改变应该使某些行为成为可能，比如发布到更多的“利基”标签。目前，如果你想让别人看到你的 JavaScript 帖子，你应该发布到`javascript`标签。这一改变应该有助于促进发布到像`advancedjs`或`functionaljs`等标签的想法。较小的社区可以在某些标签周围弹出，并确保他们集体更有可能看到该标签上的动作。你可能*而不是*想要发布一些更“主流”的标签。

随着时间的推移，这将如何体现还有待观察，但我希望我们可以添加一些功能来支持这种行为。
