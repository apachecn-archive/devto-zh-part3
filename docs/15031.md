# C 语言中的套接字编程:通信风格

> 原文：<https://dev.to/noah11012/socket-programming-in-c-communication-styles-27fc>

由于有多种不同类型的插座可供我们使用，我们将简要介绍其中一些。当我们开始创建套接字时，第二个参数接受一个表示通信方式的数字。每个数字都有一个符号常数。

## 1。袜子 _ 流

通过电缆或管道等连接传输数据的套接字。它像字节流一样可靠地传输数据。

## 2。SOCK_DGRAM

这种类型的套接字没有连接，而是每个数据包被单独寻址和发送。而且，这是一种不可靠的交流方式，不像 SOCK_STREAM。写入这种套接字的每个数据被转换成一个包。这个套接字给你的唯一保证是它会尽力传输你发送的每个数据包。

## 3。袜子 _RAW

大多数用户级程序不会使用这种通信，因为它提供了对协议和接口的低级访问。
