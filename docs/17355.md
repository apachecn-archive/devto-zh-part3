# 将 Pytest 结果与 GitHub 集成

> 原文:[https://dev . to/tryexceptpass/integrating-pytest-results-with-github-49mo](https://dev.to/tryexceptpass/integrating-pytest-results-with-github-49mo)

当加入一个新的工程团队时，我做的第一件事就是熟悉开发和测试流程。尤其是用来执行它们的工具。在过去 5 年左右的时间里，我注意到许多组织仍然使用尚未发展到支持现代实践的旧工具。即使是纯粹开发软件的团队也会发现他们在繁琐的系统周围工作，这些系统阻碍而不是促进。

我这么说是什么意思？这些工具中很少包含有用的接口来利用与其他系统的集成(比如 REST APIs)。大多数人没有现代开发实践的概念，如持续集成或容器化。几乎所有的人都想一步一步地记录通过/失败，就像你在手工执行一样。绝大多数都是围绕测试和开发的分离而构建的(有些人甚至强调了这一点)。并且他们中的许多要求组织雇佣“专家”来为团队“定制”工具。在我看来，这些类型的系统迫使组织强调责备而不是质量，强调团队界限而不是生产力。

我已经在几个组织中非常成功地构建了这些系统的长期替代方案。我已经做了足够多的工作，知道哪些特性值得包含，哪些应该留给测试/开发工程师，尤其是在持续集成和交付出现之后。
