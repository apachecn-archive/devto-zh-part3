我意识到我很少使用书签，但我仍然会用书签标记有趣的网站、项目或工具。我猜这是因为我的浏览器在建议中首先显示书签页面，当你只知道标题的一部分或一个领域时，这可能非常有用。

[![Search bar with bookmark suggestions.](../Images/dd70f8329f04988bfa9a5de296f49c6d.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--wNu37ZfA--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/r731nuj0lv73h5j0dqbo.png)

由于我只使用了几十个书签链接，我想在清理它们之前，我可以与社区共享导出的书签文件。这里有许多有趣的、有用的和不太有用的链接。你可以[下载文件](https://www.silvestar.codes/bookmarks/bookmarks.zip)或者[在你的浏览器](https://www.silvestar.codes/bookmarks/browser/)中看到它。

如果您认为这个列表非常庞大，那么您应该看看从我的 Pocket 应用程序导出的内容。事实上，你可以在这里看到它。2012 年开始存链接，去年就停了。在那段时间里，我已经收集了超过两千个的*链接。有些在今天已经过时，但有些却是常青树。有些是垃圾，有些是黄金。列表太大了，无法查看。令人震惊的是，我已经阅读了这些链接的绝大部分。😱*

我喜欢[主演 GitHub 项目](https://github.com/maliMirkec?tab=stars)、[拯救 CodePens](https://codepen.io/CiTA/pens/loved/?grid_type=list) 、[拍中故事](https://medium.com/@malimirkeccita/has-recommended)、[喜欢 Tweets](https://twitter.com/malimirkeccita) 、[爱情曲目](https://www.last.fm/user/maliMirkec/loved)也是。此外，我还是一个优秀的追随者:我在 CodePen、Medium 和 Twitter 上关注了 3000 多位作者。如果你写了，我读了，你会得到通知的。我也喜欢在 Twitter 上分享我所有的发现。我必须说，当我需要找到一篇具体的文章时，这种做法帮了我好几次。但是这种情况很少发生。除了获得知识和经验，我还没有找到一个有用的方法来从这些链接中获益。🤷‍♂
