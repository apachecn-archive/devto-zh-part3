## 解决现实问题

公司希望雇佣那些能够证明自己能够解决实际问题的人。提问者明白这一点——否则，他们不会问这个问题。😜

你需要成为一个问题解决者——恰好知道如何构建软件来解决这些问题中的一些问题。

那么你的任务就是找到一些问题——或大或小。那就造点东西解决。

让我给你举几个例子。

### 科拉韦尔

有一天我有了这个想法,“我希望。NET 开发可以像使用 Laravel 一样简单。

。NET 非常健壮，但是缺乏其他语言的框架所具有的易用性——比如 [Laravel](https://laravel.com/) 。

所以，我创造了一个解决这个问题的方法！这就是科拉维尔诞生的原因。

如果你把这样的东西变成 GitHub repo，那么你可以展示这样的东西:

*   了解如何通过构建软件来解决现实生活中的问题
*   能够编写非常清晰的文档
*   主动和热情去建造它

### 建个 App！

另一个选择是构建一个完整的应用程序！

几年前，当我学习如何使用 Laravel *构建 web 应用时，我就这么做了。[这只是一个小型社交媒体恶搞应用。](https://social.jamesmichaelhickey.com/)*

这个应用前端使用 Vue.js 和 SASS，后端使用 Laravel。

# 懂 X 种编程语言是否让你成为一名开发者？

不。这只是意味着你可以做别人让你做的东西。

可悲的是，许多公司只会根据无关的技术知识进行面试(这不会测试你是否能独立思考等等。)

你需要成为能够首先发现公司或社区内部问题的人。然后，您可以决定构建软件是否适合解决这个问题。

有哪些其他的技能和品质可以帮助你成为一名优秀的开发人员？

*   能够以积极和鼓励的方式与团队合作
*   将不同系统连接在一起的经验
    *   数据库后端
    *   API 的前端
    *   API 的后端
*   了解一般编程原则和最佳实践
    *   固体
    *   认识和了解“代码味道”的修复方法
*   代码组织
    *   业务/产品应该如何影响你的代码结构
    *   了解组织代码的不同方式以及它们之间的权衡
*   建筑模式
    *   手动音量调节
    *   MVVM
    *   洋葱建筑
    *   六角形建筑
*   编程范例
    *   面向对象
    *   功能的
    *   声明性与程序性

这绝不是一个详尽的列表——但是成为一个优秀的开发人员的基础。

如果你不知道这些话题中的大部分，那么我建议你从总体上对每一个话题都了解一点。

然后，挑选几个真正投入进去。

真正了解这些**中的一些**可以帮助你在同龄人中脱颖而出。
