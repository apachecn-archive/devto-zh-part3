我的很多潜在(希望是未来)客户使用 Wordpress，我所有的测试版客户都使用 WordPress。我所在的领域要求我的客户有某种类型的网络存在，基本上是一个“小册子”网站，当然这是一个 WordPress 似乎覆盖得很好的领域。

我的问题是关于为我的客户的网站编写一个插件，与我提供的服务集成。我有大约 12 年的 PHP 经验(最后一次接触 5.3 是在 2009 年左右)，我知道它已经改变了，但对我来说编写它应该不是问题。我确实有疑问:

*   我应该瞄准哪个版本的 Wordpress？4.9(这需要 PHP 5.2...)

*   我应该瞄准哪个版本的 PHP？我知道这与上面的问题直接相关，但是 5.4.x 已经死了，不再受支持了，对吗？

*   客户升级 Wordpress 版本有多难？我似乎记得升级 Drupal 6.x 网站是一场噩梦，你最好准备好你的备份。

*   PHP 7.3 和 Wordpress 4.9 兼容吗？

基本上，如果有人能在 2019 年写一个 greenfield Wordpress 插件，我会很激动。
