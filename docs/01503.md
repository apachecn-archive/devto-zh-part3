# 给新前端开发人员的 7 个提示

> 原文：<https://dev.to/douglasamarelo/7-tips-to-new-front-end-developers-3em>

[![7 tips to new Front-end developers.](img/d362e4b600e960a4648198902b6c8e0a.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--yNbToJFh--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/j50j6abjti2ofkjb9ojr.jpg)

这几天我在 LinkedIn 上收到一条消息，来自一个刚刚开始前端开发领域的家伙。他很高兴我在我的社交网络上接受了他，他给我写了这个:

> “谢谢接受。有一天我希望成为像你一样的专业人士。”

当我读到他的留言时，我想用一些建议来回答他。当我开始的时候，没有人告诉我，我必须随着时间的推移，通过实践，特别是通过错误来学习。一些我希望有人在我开始时告诉我的建议。

当我想到要说什么的时候，我想到了七条建议，我把它们放在了下面。

1.  一直学习。即使只有一小会儿，但是每天都要看**学习**。让学习成为你日常生活的一部分。试想，如果每天学习一件事情在短期内看起来微不足道，但在中期/长期内，它将对你是谁和将成为什么样的人，你能做什么和将做什么，以及你已经达到和将要达到的任何目标产生巨大的影响。

2.  不要放弃一个问题，但是要意识到什么时候需要停下来，散散步，喝喝咖啡。有时我们甚至会撞车，即使在这方面有多年的经验，这种情况也会继续发生。

3.  寻求帮助之前先研究一下。独自找到解决方案带来了非常好的感觉。但是，如果你找不到解决办法，不要犹豫，寻求帮助，永远不要感到羞耻。这不是弱点，而是知道如何认识到自己的局限性，如果你以前做过调查，就能更好地了解谁会帮助你。

4.  尽你所能帮忙。我们所有人，无论级别和/或经验如何，在某些时候都需要帮助。如果你总是乐于帮助别人，别人也会乐于帮助你。

5.  总想提高，想变得更好！说真的，一直都是！不要满足于你已经知道的，认为这将永远足够。这种情况不会发生。

6.  你要为你的进化负责。不会总有人看着你的代码并提出改进建议，所以你必须自己努力。你有责任不断进步，成为一名更好、更合格的专业人士。不要外包这项工作。

7.  利用你所拥有的知识，在你所处的地点和条件下，尽你所能做到最好。我的意思是，环境和/或公司和/或客户不应该干涉你的工作质量。无论是为了“你叔叔”项目，还是为了一家跨国公司，永远保持优秀。你的工作质量与你的性格直接相关，这不应该因为项目/客户的大小而改变。

**额外提示:**访问您的一些旧代码。这将帮助你了解你是如何进化的。有时候，在很短的时间内。

最后，我想说的是，这篇文章并不打算成为“初学者的开发人员手册”。肯定还有数百万其他有用的建议我没有在这里引用，还有数十亿我仍然不知道。

我希望以某种方式帮助那些刚刚起步的人，并能够为帮助过我并继续多次帮助我的开发社区做出贡献。

祝大家大获成功！
