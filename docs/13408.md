# 为什么我不参加黑客马拉松

> 原文:[https://dev.to/mattdark/why-i-dont-attend-hackathons-1i27](https://dev.to/mattdark/why-i-dont-attend-hackathons-1i27)

我在大学的时候从来不知道黑客马拉松，因为我居住的城市没有组织这种活动。这可能是我不参加黑客马拉松的主要原因，但我会试着多解释一点。

我第一次接触软件开发是在 15 年前，当时我还在上高中。那一刻我学会了 HTML，CSS，JavaScript。后来我决定学计算机系统工程。

在大学里，我学习了 C++、Visual Basic(我使用微软的技术最少，所以之后我没有编写 Visual Basic 代码)、Java(我自己学的，但不喜欢)、HTML、CSS 和 JavaScript(现在我又专注于 web 开发)，最后在一次会议上知道了 Python，并开始自学。

我喜欢在做项目时投入足够的时间，思考如何解决问题，在笔记本上做笔记和写想法，在需要时学习新技术，阅读文档，编写代码和尝试。

我需要几个小时，一天，几天，一个月等等来完成一个项目，这取决于我必须工作的时间。花一个周末做一个项目，与其他人合作，这听起来不是一个坏主意，但有时不睡觉或在活动现场睡觉，喝咖啡或任何精力充沛的饮料，吃像披萨这样的食物，对我和我的健康来说不是一个好的选择。

过去，我常常几天睡不好或者几天醒着，吃不健康的食物，做一个项目，阅读我感兴趣的技术，准备一个会议，等等。所有这些都在某种程度上影响了我的健康。

现在我关心我的健康。我试着多睡一会儿，不花几天醒着做任何事情。我遵循健康的饮食(大部分时间)，经常锻炼，我是那种不喝咖啡就写代码的开发人员(我两年前就不喝咖啡了)。

事实上，我特别向学生推荐参加黑客马拉松，因为这是他们提高编程技能、向导师学习、结识有趣的人并有机会认识未来的联合创始人的一种方式。我也组织过几个。

我通常不是一个喜欢社交的人，但我最喜欢参加本地或国外的会议，是结识有趣的人，了解其他城市和文化。我相信我在参加黑客马拉松后会做的事情是睡觉，不做任何其他活动。

所以我决定黑客马拉松不适合我。但我愿意作为导师参加黑客马拉松，至少几年前我是这么想的。

我联系了一些黑客马拉松和创业活动，邀请我作为导师参与，但他们从未回应。我开始认为我没有能力成为一名导师，即使我有多年的经验为自由/开源软件项目做出贡献，参与并共同创建技术社区，编写代码多年，并在会议上发言。

是的，冒名顶替综合征，它多次让我对向 CFP 提交提案或参加活动产生怀疑。但是现在我知道我在错误的地方寻找分享我的专业知识。

去年，我在 2017 年 12 月从第四轮毕业后，作为第五轮的特色专家和第六轮的导师加入了 Mozilla Open Leaders([@ MozOpenLeaders](https://twitter.com/MozOpenLeaders))计划。今年我将作为第七轮的特色专家回来。

去年 12 月，我加入了 PyCon USA 导师计划，并与 Jacob Kaplan-Moss 一起帮助审查并反馈了提交给 CFP 的几份提案。

我也是技术社区和一些朋友、开发者和设计师的导师。无论如何，虽然我不会参加黑客马拉松，但我可以在你的活动中发言和指导。

记住，这只是我个人的看法。这取决于你去尝试和参与，我鼓励你去做。
