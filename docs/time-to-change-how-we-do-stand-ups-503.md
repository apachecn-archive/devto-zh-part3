# 是时候改变我们表演单口相声的方式了

> 原文：<https://dev.to/justinctlam/time-to-change-how-we-do-stand-ups-503>

我们都很熟悉站立会议，在这种会议中，每个人都按照预先确定的节奏开会(每天一次，每周三次，等等)...)来回答 3 个重要问题。

1.  我昨天做了什么帮助开发团队实现了 sprint 目标？
2.  我今天将做什么来帮助开发团队实现 sprint 目标？
3.  我是否看到了任何阻碍我或开发团队实现 sprint 目标的障碍？

我们称之为站立会议，因为与会者应该站着回答 3 个重要问题。站着不舒服是为了缩短会议时间。我们试图回答这三个重要问题的原因是为了项目能够及时成功地完成。

然而，我在软件行业工作了很多年，也参加过无数的团队，我觉得我们严重滥用了单口相声的初衷，只是“因为我们一直在这么做”才这么做。

我注意到传统的单人脱口秀有以下问题。

1.  缺席的与会者错过了所有介绍的信息。这种情况比你想象的更常见。例如，“我早上需要修理我的车，今天会错过站起来”，“猫在地毯上尿了一地，所以我需要清理它”，“今天会错过站起来”，“约了牙医，今天会错过站起来”，等等...

2.  人们经常迟到，这不可避免地延长了站立会议的时间。

3.  人们几乎总是进入对团队其他成员不重要的无关紧要的讨论。我开始认为这是人类的天性。

4.  我听说过其他持续 45 分钟到 1 小时的单口相声。那支队伍也为此感到非常自豪。

5.  强制时间限制几乎从来不起作用。有些人就是喜欢描述他们工作中的每一分钟细节，即使 3 分钟计时器在响。

6.  人类的记忆力很差。

我想建议我们用群聊软件(微软团队，Slack 等)代替面对面的 15 分钟站立会议...).这确保了 3 个重要的问题被记录和搜索。不再有遗漏的信息，不再有他/她说的误解和线索可以回复和引用回来。

但是我从队友那里听到反馈，他们希望面对面地交流，而且他们“据说”从超长的站立会议中获得了价值。翻译:我们喜欢浪费时间谈论周末或我们昨晚做了什么。嗯，我可以为你介绍“我们一起去喝咖啡会议”来满足那些社交需求吗？
