# 我学到了在没有设计师的情况下构建一个附带项目

> 原文:[https://dev . to/Ryan _ dunt on/what-I-learned-build-a-side-project-without a-designer-1al 3](https://dev.to/ryan_dunton/what-i-learned-building-a-side-project-without-a-designer-1al3)

## [](#the-project)项目

我喜欢奥斯卡。当我说我爱奥斯卡时，我真的爱奥斯卡。我和我的室友每年都会观看所有 24 个奖项的奥斯卡提名。这是我最喜欢的传统之一，然而，我们通常通过从物理列表中划掉我们看过的电影来跟踪我们的进展。

周末的时候，我决定做一个网站来帮助我们跟踪我们的进展。

最终产品:[www.theoscarwatch.com](http://www.theoscarwatch.com/)

## [](#round-1-myself-as-a-designer)第一轮:自己当设计师

我在构建网站的过程中得到了很多乐趣，但是作为一名 React 开发人员，我所关心的是如何在网站中移动数据。最后，该说的都说了，该做的都做了，这是结果的一个快照。

[![first try](../Images/8a7426122d411b5aeaf64c70fdaff6db.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--sgx7-MgI--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/fr25ehpcdho8ky3plm9w.png)

我真的为这个网站的快速和简洁感到骄傲。然而，我的朋友群中的其他人(非开发人员)的反应却不冷不热。多条评论“我们能把它弄漂亮点吗？”或者“哦，哇，干得好”。最后，由于对网站的外观不满意，我在周一，也就是提名公布的前一天，请了我的一个设计师朋友来帮我一把。

## [](#round-2-working-with-a-designer-electric-boogaloo)第二轮:与设计师合作:电动布加洛

我让我的设计师朋友在前一天晚上给他发了一个链接后过来了。他为如何改变事物设计了一个快速的模型。第一个是取消显示数据的模式，支持下拉菜单。第二个是缩小滚动区域。第三个是增加一个倒计时器。第四是修改了配色方案。

最终结果看起来是这样的:

[![final](../Images/bedcb9a1c4104bc881c4caa06366dd05.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--s60_7mp_--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/bhv1lq6zxfcyk5ulmybc.png)

更干净，更好看，最重要的是...非开发人员更容易接受！

整个事情让我想起了硅谷的一幕，在那里，没有一个非工程师理解压缩应用程序的神奇之处。

长话短说，作为开发人员，我们有时认为设计师是理所当然的，我们需要欣赏他们能够带来我们甚至没有想到的价值。
