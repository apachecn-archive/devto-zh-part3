# 骨架和堆积物——我如何解释网站

> 原文：<https://dev.to/geekgalgroks/skeletons-and-caboodles---how-i-explain-websites-4ghl>

作为一名前端开发人员，我经常与从未写过代码或害怕编程的人一起工作。我曾经和属于这一类的经理、沟通者和商业分析师一起工作过。许多行业标准术语对他们来说毫无意义。解释几个关键概念极大地改善了我的工作流程，让他们能够提出更好的问题，他们对流程有了更多的参与和理解。

## 摇头晃脑的骷髅

我能画出一个可辨认的卡通骨架(他的名字叫鲍勃)。每当我解释 HTML、CSS 或 JavaScript 时，我经常从在白板或纸上画一个骨架开始。这总能引起一些笑声，让他们感觉更舒服。

### HTML

骨架对于解释基本的 HTML 非常有用。他们有头、脚和身体。骨架中各部分的顺序与页面的顺序相同。我将在这部分的后面写 HTML 的例子。

```
<head>
    Skeleton explanation
<head>

<body>
    <header>
        <h1>Bob the skeleton</h1>
    </header>
    <div>Shirt</div>
    <p>Pants</p>
    <footer>
        <p>Boots</p>
    </footer>
</body> 
```

### CSS

我的骨架有衣服。我会在骨架上画一个 t 恤，给它上课`.shirt`。我通过添加其他类，如`.red`、`.long-sleeved`或`.striped`，完成了基本的 CSS 特性和继承。然后我在骨架上穿裤子或短裤重复这个过程。

```
.shirt {
    background: blue;
}

.red {
    background: red;
}

body .green{
    background: green;
} 
```

### JavaScript

这部分我表演出来。我伸出我的手臂，前后摆动，说:

> JavaScript 就是让骨架动起来的肌肉。

根据他们的问题，有更多糟糕的表演和解释。

Bob 帮我解释了为什么要求新的行为(JavaScript)比改变标题的颜色(CSS)要花更多的时间，因为改变肌肉的工作方式比改变衬衫的颜色更难。

## 一堆堆

Caboodles 在 80 年代和 90 年代非常流行。它们被用来存放和整理化妆品。把它们想象成粉色、紫色或蓝绿色的化妆工具箱。

我在解释内容审计时使用 caboodles。每当改变一个网站的框架或平台时，我都坚信要进行内容审计。网站经常会出现没人看或关心的陈旧内容。在我看来，如果不需要，为什么要移动这些内容呢？

这个例子中的堆就是网站。每当你搬到一个新的地方，你会把旧的东西都拿出来，注意到你有 17 支胡椒博士唇彩，其中 5 支还没有打开。你需要把所有 17 个放在你的新堆里吗？你有地方容纳他们吗？你还喜欢唇彩的味道吗？

这个例子可以很容易地适用于鱼饵(渔具盒)或手工工具(工具箱)。
