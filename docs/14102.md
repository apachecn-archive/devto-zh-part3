# 技术面试者的谦逊训练

> 原文：<https://dev.to/triplebyte/a-humility-training-exercise-for-technical-interviewers-96n>

***TL；谦逊是技术面试者的重要品质。我们的数据显示，对自己能力非常自信的面试官给出的面试分数不太一致。相比之下，意识到自己弱点(以及面试有多吵)的面试官会给出更一致的分数。我们开发了一个练习来帮助培训这方面的面试官。*T3】**

**编程面试很吵。即使是在同一个特定领域，两位面试官对同一个应聘者的技能也会得出截然不同的结论。这种噪音是采访准确性的一个重大障碍。减少这种噪音是培训技术面试官的主要目标之一。**

# 培训面试官

我们在 Triplebyte 培训了很多面试官。我们雇佣了一个由 40 名经验丰富的工程师组成的团队，在候选人通过我们的平台时对他们进行面试。当我们培训这个团队的新成员时，我们关注几件事。我们确保面试官在他们将要评估的领域是强大的和最新的(有时，很难区分一个因为是某个领域的专家而给出不同寻常答案的候选人和一个因为不知道自己在说什么而给出不同寻常答案的人)。我们确保面试官对他们评估的技能有明确的指导方针(这是对面试官模式匹配偏见的最好防御)。然而，我现在认为同样重要的是培养面试官的谦逊<sup id="ref1">[【1】](#fn1)</sup>。

对自己的弱点缺乏认识是面试噪音的一个主要来源。这是真的，因为过于自信的面试官对应聘者的评判过于苛刻。软件工程的领域非常广阔，没有一个工程师能够完全掌握它。然而，我们都相信我们已经掌握的领域是最重要的。而且我们不会完全尊重另一个工程师，如果他们在一个我们很强的领域很弱(即使他们在其他领域非常强)。在面试中，这表现为对技术实力与面试官不同的候选人的偏见。我们通过让多个面试官观察同一个面试并打分来衡量这一点。即使当面试官给与他们自身优势无关的领域打分，甚至当他们使用结构化评分标准时，这种影响依然存在。**面试官只是给不喜欢他们的候选人较低的分数。**这是噪音。这使得采访不太准确，我们需要减少它。

我们发现，解决办法是训练面试官的谦逊态度。那些意识到自己弱点的面试官(也意识到面试会有多吵)会较少受到他们应该评估的领域之外的领域的影响，给出更一致的分数。T3】

# 建立谦逊的一种练习

**那么，如何培养面试官的谦逊呢？**怎样才能让自己变得更卑微？我认为，答案是体验一个候选人所经历的事情。求职面试是令人羞愧的。你被拷问。你必须记住那些你已经很多年没想过的事情。聪明人会指出你的逻辑和代码中令人尴尬的缺陷。你永远不会知道你以为你知道的那么多。几乎每个人都有很大一部分面试失败。

我们开发了一个练习，用来让我们的面试官体验成为候选人的过程。起初，我们尝试简单地要求面试官互相面试。然而，这并不奏效，因为他们不能给出诚实的反馈。如果你采访了你的同事，最后觉得他们有点糟糕，你会诚实地告诉他们吗？大多数人在这种情况下不会。我们需要想办法解决这个问题。

解决方案不期而至。这个练习实际上是在我试图为我们的面试团队招聘员工时开始的。我使用的评估过程的一部分是要求候选人面试我(它变得非常元)，并且，为了使这些面试更有趣，我给出了一个好的和坏的混合答案(我的尝试)。我立即注意到这是如何让他们给出诚实的(和谦逊的)反馈——甚至是对我的“好”答案。从这次经历中，我们开发了以下练习来训练所有面试官的谦逊态度:

*   和一个同事配对，让他们问你一些他们最喜欢的面试问题。
*   提前告诉他们，你会故意回答一些糟糕的问题(弱势候选人可能会给出的角色扮演答案)。
*   然后，随着面试的进行，这样做。大约一半的时间给出你最好的答案。另一半时间故意给出一个拙劣的答案。
*   面试结束后，让你的同事评论你的答案。

这样做可以让你的同事变得 100%诚实。他们不知道面试的哪些部分是你真正想表现好的。此外，他们会注意到你给出的糟糕答案。如果你故意给出一个糟糕的答案，而他们没有“抓住”，他们看起来有点糟糕。因此，他们会诚实、详细地讲述他们的感受。

这项练习要小心！我已经做过很多次了，这让我深感惭愧。它几乎总是导致你尊敬的人指出你不擅长的事情。这有可能引发冲突。我认为这可能只应该在内部信任度很高的团队中进行(危险是让团队成员相信其他团队成员不是很好)。但结果是强大的。它清楚地凸显了优秀的工程师在某些领域的薄弱程度，以及面试官对候选人的意思妄下结论的程度。我认为每个指导面试官的人都应该通过这个练习来锻炼自己。

# 结论

技术面试官保持谦逊是很重要的。这为候选人创造了更好的体验，也使面试更加准确。最好的面试官意识到自己的局限性，并对面试过程的反复无常有着正确的认识。为了在这些方面做得更好，面试官需要花更多的时间作为候选人，亲自接受面试。很难在员工中创造这种体验，但我们想出了一个(危险！)这种锻炼效果很好。

如果人们更广泛地尝试这个练习，我会很高兴。我认为这可能会成为大多数公司面试团队的标准。如果你想试一试，给我发电子邮件到[ammon@triplebyte.com](//mailto:ammon@triplebyte.com)，让我知道进展如何。
