# 5 小时内创建 app 的 7 件重要事情

我和朋友合作的第一个项目花了 5 个月才完成。这次我从开发过程中移除了耗时的元素。
因为我有花 5 个月时间创建 web app 的经验，所以我知道什么是耗时。我介绍了 7 件你应该做或不应该做的事情，以令人难以置信的速度创建应用程序。

## [](#1-no-login-feature)1。没有登录功能

登录功能非常耗时。你需要在大范围内实现代码。社交登录看起来很酷，但你需要实现 OAuth 并获得 APIkey，这是非常累人的工作。考虑到成本，登录功能在项目的第一阶段并不是必需的，不是吗？

## [](#2-reduce-usecases-to-the-limit)2。将用例减少到极限

“改变世界”只让你做两件事。

*   邮政
*   获取帖子列表

我只压缩了两个用例，这大大降低了开发成本。当你创建一个服务时，你会想出各种有用的特性，但这会导致用例的增加。你应该挤出两三个重要的。登录功能也是用例增加的一个因素。

## [](#3-dont-write-css-as-possible)3。尽可能不要写 CSS

使用 CSS 框架，尽量不要自己写 CSS。你可以把时间花在做好设计上，但让我们专注于你服务的主要核心。看谷歌服务。他们好像没那么在乎设计，但他们是世界上最好的服务。

## [](#4-use-boilerplate)4。使用样板文件

构建开发环境非常耗时。我使用样板文件，在 3 分钟内完成了 dev env 的构建。样板文件就像一个已经构建好的模板，它包含了你需要开发的所有库。

## [](#5-serverless)5。无服务器

许多云服务竞相创造创建应用程序的最简单方法。无服务器是充分利用这些服务的方式。Firebase 和 Netlify 非常方便。您可以通过这些服务在 3 分钟内部署您的服务。

## [](#6-ignore-small-parts)6。忽略小零件

忽略小零件。您可以在发布后修复它们。功能的改进、重构、设计的微调，各种各样的事情把你从你需要投入的主要核心中诱惑出来。完全忽略这些东西。在某个地方做个笔记可能是个好主意，但是在完成之前你不应该碰它们。

## [](#7-dont-use-technologies-you-dont-know)7。不要使用你不知道的技术

出于研究目的，在你的项目中引入新技术是一个非常好的主意。但是如果你想在 5 个小时内创建 app，你不应该完全使用它们。你需要付出昂贵的学习成本来学习新的东西，你会在某个地方陷入陷阱。如果你阻挠并停止项目，那是没有意义的。让我们考虑只发布你的应用程序。
