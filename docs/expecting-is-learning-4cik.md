# 期待就是学习

> 原文：<https://dev.to/xowap/expecting-is-learning-4cik>

> *免责声明* —这个故事听起来好像我是个迂腐的混蛋。我知道。但是我尽量保持事实。我希望它保持在合理的范围内:)

当我完成我的第一个开发实习时，团队中一些初级到中级的开发人员问我一个令我惊讶的问题:“但是你是怎么知道所有这些的？”。

事实上，我已经在许多问题上帮助过他们，但大多数情况下，我事先并不知道答案。相反，我在他们面前做了一些很好的互联网搜索，并从结果中找出了一个解决方案。

现在，每个人都能够复制/粘贴一个堆栈溢出回复——特别是与一个没有经验的实习生相比——因为他们看到了整个过程，我不禁问自己，他们不明白其中的什么？

多年后，在我看来，答案实际上是，虽然我不知道答案，但我知道要寻找什么。理论是这样的:

> 学习新事物的关键是期待它们的存在

假设你的设计师交给你这个下拉菜单:

[![](img/3f6f78745c7b7e09e76bd7d782307b3e.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--Q_dHzFuV--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/76f6dpvrqxd0n0yiuvpa.png)

它非常复杂，因为它有一个带阴影和圆角的箭头，这在纯 CSS 中是不可行的(据我所知😊).无论如何，你都要使用背景图片。但是内容大小不是静态已知的，所以还需要扩展它。简而言之，如果你以前从未做过，解决方案可能并不明显。你是否:

*   看着它的形状挠头？
*   将背景切割成 Macromedia 风格？(高五 2000 的开发者！)
*   浏览 MDN 直到找到 [`border-image`](https://developer.mozilla.org/en-US/docs/Web/CSS/border-image) ？

其中一个解决方案比其他的简单得多，但是如果你以前不知道它，如果你不希望 CSS 有这个特性，你真的找不到它。

在我看来，对事物存在的期待是你最初几个月停滞不前和每天学习的区别。

但是我很想听听对这个理论的看法。你对此有共鸣吗？你觉得是对还是错还是不完整？如何培养自己的「期待感」？
