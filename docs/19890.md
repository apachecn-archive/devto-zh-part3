# 法国的开发者怎么了？

> 原文:[https://dev . to/kevinvavelin/法国开发人员的遭遇- 4gko](https://dev.to/kevinvavelin/what-happened-to-developers-in-france---4gko)

声明:这篇文章仅仅基于我的个人经历和我这一年在一些公司的工作经历。

今年是主要关注 web 开发的一年，这不是我特别喜欢的领域，但如果项目足够有趣，我可以做(不是我可以在互联网上找到并花一天时间做的事情)。一开始我知道现在业界爱 JavaScript 框架，React，Angular，Vue 等。在开发人员社区中有很多投票。但是我注意到一些我不能理解的事情…

## [](#imposter-)冒名顶替？

它始于一个年轻的学生，他想学习更多关于 web 开发的知识，但很快我意识到他可以在 React 中编写代码，但对 JavaScript 本身一无所知！怎么会这样？如果某个东西不在 React 上，他就迷失了，他没有意识到 React 只是一个库，如果 React 不这么做，可能是因为 JavaScript 做了…他不知道链接函数调用、匿名函数、什么是闭包甚至承诺，但他使用了它，可以为它编写 sugar 语法等等。他写的代码，他不知道这些代码是如何工作的，但是他能写出来，因为他学会了，看到了，复制了，并且知道它是这样工作的。当时我感到很失落，但是，为什么不呢，毕竟他还年轻，他还可以学习。

然后我的团队来了一个新的开发人员，他有更多的经验，不是那么多，而是更多。他来了，说他知道一些关于 Angular 和 React 的东西，所以我很自然地给他这个项目的 Angular 方面。当我看到代码时，我注意到如果不是在一些文章、论坛或教程上，他不能在 Angular 中做任何事情。即使类本身的文档显示了他的工作的一些样本代码…我们在法国亲爱的开发者发生了什么？

最近，我有一些开发人员学习了 docker，如何将 Docker 容器部署到云提供商，猜猜会发生什么？就凭这一点，他们现在认为自己了解云计算的一切！我知道他们有一个术语，叫做“冒名顶替者”。

## [](#the-danger-of-pression-to-developer)对开发者的威胁

我继续工作，我要求我的同事写一些测试。我们的代码需要测试，这是一个优先事项。你猜怎么着？他们没有，他们总是有借口，没有足够的时间，因为客户想要一个新的功能，我们有太多的错误等待修复，等等。但他们无法假设的真正原因是:他们不知道如何去做。他们只是不知道如何测试他们的代码。我认为这是一个大问题。

说到压力，它来自金字塔的顶端。管理层不关心开发人员被什么卡住了，或者为什么他会试图改进它的代码，他们只希望产品被发送，客户付款，如果还有一些 bug，他们就继续工作。毕竟，他们出售他们自己的技术债务的维护，这是客户没有要求的…

这在法国成为了一种经典的软件开发方式，他们找到了向客户出售自己技术债务的方法，而客户对此一无所知。我认为这是偷窃和对客户的不尊重。

但是由于开发人员不想丢掉工作，他们一次又一次地做着同样的事情，如果没有人说实话，这种事情可能永远不会停止。因此，我们有越来越多的开发人员感到疲惫、不开心，并且不愿意正常工作，因为他们知道他们不会被他们的层级所倾听。

## [](#what-about-manager-)经理呢？

哦……经理……到目前为止，我见过两种类型的经理:

*   那些假设他们不了解技术方面，并且当他们说不的时候信任他们的开发者的人。
*   那些不关心技术方面的人，开发人员没有话说，必须按他们说的做，因为客户不会看到技术方面，他只是想要一些工作的东西，如果他们仍然有 bug，那就更好了，因为他们可以赚更多的钱。

在我看来，第二种类型是最糟糕的。他们越来越突出，所以如果你对一个任务说“不”，经理让你别无选择，换个公司。

## [](#thanks-to-the-developers-community)感谢开发者社区

我看到越来越多关于如何对付冒名顶替者，我们如何一起成为更好的开发人员的博客帖子，对此我表示真诚的感谢。在 LinkedIn 上，我看到越来越多关于开发人员厌倦了一个接一个地写代码的帖子，因为他们没有做正确的事情。
我非常喜欢这个开发者社区，他们热爱自己的工作，并与他人分享。
