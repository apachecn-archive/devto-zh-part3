我以前在会议上讲过话，开过研讨会，但从来不是关于 React 或前端开发的。在我的日常工作中，我大部分时间都在使用 Docker、Kubernetes、Golang、服务网格、无服务器功能等后端。

然而，在我的空闲时间，我做了 Javascript，React，我喜欢它。我来自后端世界，我总是想知道前端开发人员对了解 Docker、Kubernetes 和服务网格有多大兴趣？

给你一个关于这些技术的基本概述的会议会引起人们的兴趣吗，特别是当它用实际例子演示如何使用这些东西的时候？例如，一个带有几个后端 API 的 React 应用程序，可能是 GraphQL 和一个数据库？你怎么想呢?

这是我想出来的简短摘要:

你听说过 Docker、Kubernetes 和 Service Meshes 吗？有没有想过这有什么大惊小怪的，以及作为 React 开发人员如何使用它？这个会议将是对 Docker、Kubernetes 和服务网格的一次快速而实用的浏览，它将为您提供，并有希望激发您作为 React 开发人员如何使用这些技术的想法。

你怎么想呢?只剩下几天了...⏰
