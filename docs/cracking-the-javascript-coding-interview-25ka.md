# 用 Javascript 破解 CS 代码面试

> 原文：<https://dev.to/learnersbucket/cracking-the-javascript-coding-interview-25ka>

对于初学者来说，接到面试电话是最令人兴奋的经历之一，但是一旦你在最初几轮中失败了，这种兴奋就会消失。

任何典型的 javascript 面试都由几轮组成。

1.  电话。
2.  初步筛选。
3.  任何编码网站上的挑战，比如 Hackerrank，Hackerearth，Codechef。
4.  现场面试。
5.  提议。

在这五轮面试中，前两轮并不难，因为你只需要向面试官解释你的兴趣、知识和能力。

但第三轮是你现场面试的通行证，这可能会改变游戏规则，因为现在是时候证明你确实有能力担任这个角色了。

耶，来吧，我准备好了！。😎。
新手有时会有这种感觉，但坚持住，你会大吃一惊的🎁。

你需要解决所有测试案例都要通过这一轮的算法(也有一些其他的典型问题，取决于你面试的角色)。

现在所有解决的算法例子都在 **Java** 、 **C++** 或 **Python** 中，但我只知道 javascript(因为我是初学者)😔。
