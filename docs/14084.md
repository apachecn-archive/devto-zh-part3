# [](#we-want-to-learn-from-you)我们要向你学习！

作为开发者倡导者，我花了很多时间在这个领域，参加会议讨论，倾听世界上发生的事情。我知道有些人在标记系统方面是冷酷的专家。我们想给你一个理由，让你聚在一起，与我们以及彼此谈论你对规模、技术债务和最佳实践的发现。我们看到 Gremlin 的 ChaosConf 和 Honeycomb 的 o11ycon 为从业者提供了一个交换信息和交流思想的机会，我们希望继续下去。我坚信，把有共同兴趣的人聚集在一起，会让他们谈论的整个类别发生飞跃。看看 DevOpsDays！

# 人们渴望最佳实践

当然，我可以告诉你什么样的旗帜命名方案最有效，但是难道你不想听一个已经大规模使用了很多年的人说吗？让人们聚在一起，从初学者，到构建自己系统的人，到大规模使用我们的客户，给我们所有人一个学习和整理最佳实践的机会。在 LaunchDarkly，我们认为今年是特性管理采用的转折点，作为该行业的一部分，我们希望帮助最佳实践尽可能广泛地应用。我们希望《轨迹》能让我们讨论如何加速测试、设计、开发、发布和销售，并使之更加强大。你如何达到逃逸速度？我们想听听。

# 我们认为你很棒，有很好的想法！

我们已经计划好了未来的发展方向，以及我们的愿望清单上的下一步是什么，但我们希望确保它能为您、我们的客户和潜在客户服务。创造一项伟大的技术成就却不能真正帮助人们，这一点都不好玩。

当人们问我伊迪丝和约翰是如何想到将功能标志作为服务的，我总是告诉他们伊迪丝告诉我的，她参加了太多的深夜、周末部署会议，以至于她开始讨厌他们，并最终建立一个产品，这意味着人们可以在没有深夜戏剧或压力的情况下启动和测试他们的软件。

我们一直希望做到这一点，让您的生活变得更轻松——所以告诉我们怎么做吧！我们列了一个希望能被提出的演讲清单，但我打赌你对你学到的东西有想法，你可以谈论或写下来。以下是我们想到的一些事情:

*   转型或灾难故事——了解我们需要新的软件在这个新世界中安全运行
*   我们如何解构一块巨石
*   如何打破文化壁垒，让 DevOps/feature flag 大规模采用
*   GDPR——一年后
*   棕地开发的新技术组合
*   如何在生产中测试
*   持续交付的途径
