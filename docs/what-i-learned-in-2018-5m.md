# 2018 年学到的东西！

> 原文：<https://dev.to/buphmin/what-i-learned-in-2018-5m>

我们开发人员倾向于告诉自己，我们不够好，太懒了，因为我们没有学到所有的东西。对我来说，我当然觉得今年我几乎什么也没学到，直到我把它分解开来，逻辑地分析了这一年。我看了看我在正常工作和生活时间表之间学到的所有东西，并编制了一份清单。

事不宜迟，以下是我学到的东西:

## Webpack

我可能学得晚，但我花了一些时间学习 webpack，并在我的工作和个人项目中实现了它。我喜欢捆绑和将 js 代码写成模块的能力。此外，能够一起使用 typescript 和 vue 是一个很大的好处。

### 我能改进什么

有一件事我不明白，为什么 webpack 在捆绑过程中添加了这么多额外的代码。在我做的大部分工作中，这并不十分重要，因为我们有一个局域网速度的内部应用程序，但对于一个实时网站来说，这可能很重要。

## 打字稿

Typescript 很棒。慢慢地，我们已经将旧代码转换为使用 typescript，这是一个好处。重构代码变得更加容易，理解更大程序的流程也变得更加容易。

### 什么能变得更好

泛型有时会很棘手，有一些事情需要我花一些时间才能弄清楚(我在看你蓝鸟承诺+ async/await)。

## Golang

我利用工作机会为一个项目学习 golang，这个项目可以通过线程大大加快速度。我发现围棋非常直观，也很容易学习。我当然能理解为什么有些人选择迁移以满足高性能需求。

### 事情改善

我仍然不太流利，所以在我感觉我能处理任何事情之前，我需要更多的练习。另一件事是，我在 1.11 中非常关注 go 模块，所以我对经典 Go dev 方法论的理解还需要一些努力。

## 我将列举的其他一些事情

让我们看看我还能记起什么...

*   webdev 的 Node.js(以前在 kafka 事件系统中使用)
*   Adonis 框架
*   改进的 JS/PHP 基础
*   C++的一些基础知识

## 2019 年做什么？

我不是一个很有新年决心的人，但我确实有一些关于 2019 年我想学什么的想法。

*   Golang api 网站开发
*   Vue 服务器端渲染
*   渐进式网络应用

好了，伙计们，这就是这篇文章的内容。希望我已经启发了你，让你回顾一下自己一年来的工作，为你所取得的成就给自己一个鼓励！
