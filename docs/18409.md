## [](#requirements-or-maybe-restrictions)要求(或者也许是限制)

每个项目都需要一些要求，所以我应该在这个博客中强调这些要求。我想在这种情况下工作，这样我就不会继续做一些以后会后悔的事情。

1.  **易写**。我不想在每次写文章的时候都一遍又一遍地经历“写-构建-部署”的循环。这个过程应该像在我的编辑器中输入，看到预览和上传文件一样简单。就是这样，没有预处理，没有构建，最少的东西。
2.  **无 bullsh*t 特性**。功能集应该是最小的。因此，像评论、反应或分享帖子这样的事情不会被包括在内。任何添加的新功能都会有一个解释。
3.  **账单更容易**。这不会让我赚到什么，所以我想尽可能的轻松。这也可能促使我寻找更新的解决方案(希望如此)。

## [](#going-about-it)四处奔波

这就是我决定的原因。

1.  **储存在天蓝色的水滴里**。这将是存储原始博客降价文件的地方。这将保留一个备份，并为原始源文件创建一个目标。
2.  **上传**时自动处理存储的文件。我设置了这个，文件在没有我的参与下被处理。后端使用这些经过处理的文件来响应请求。
3.  **前端和后端分开住**。这更多的是个人选择，而不是实际选择。我将在 react 中写博客，我需要一个后端来查询我的博客。
4.  **起后端**的作用。与永远在线的服务器相比，它们的运行成本非常低。
5.  **托管 Netlify 上的前端**。我听说过 netlify 的好处，想尝试一下。它的免费层很棒，我可以免费添加我的自定义域名。
