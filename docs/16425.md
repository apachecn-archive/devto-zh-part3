# IDE -初学者的陷阱！

> 原文：<https://dev.to/yafkari/ide---the-beginners-trap-eg2>

大家好，今天我们来谈谈 IDE！

但是首先..

## 什么是 IDE？

对我来说，IDE 是一个软件，它允许我们在一个舒适的环境中为我们正在使用的语言编写代码。我们可以做得更快更好。
我甚至可以说，它不是一个软件，而是一套软件，提供了许多工具，如语法高亮显示、调试器、代码完成、代码重新格式化、...

## 我们为什么爱 IDE 的？

你不认为拥有一个 IDE 可以改变生活吗？我们总是在评论和论坛中捍卫我们的 IDE 是最好的。
为什么？因为它真正改变了我们的生活和我们编码的方式。**我们做事更快更好**。

无论是*自我完成*、*报告我们代码的问题*、*重新格式化我们的代码*等等...我们的 IDE 有时会把我们从许多挣扎中拯救出来。

它不仅是一种工具，也是一种工作方式

但是对于初学者来说，IDE 可能是一个**双刃工具**(也是一个非常锋利的工具)，因为我们将在本文中讨论一些东西，如果你不介意的话。

## 为什么我说那是个陷阱？

我在整篇文章中所说的都是我个人经历的结果，也是我周围的人(同学，同事，...).不要犹豫，说出你的观点，分享你的观点😁。

对我来说，初学者可能陷入的第一个陷阱是使用 IDE 来学习一门新语言。因为，对我和大多数人来说，通过使用最大限度的感官**来学习更容易，只读一门课程的事实不足以正确和有效地学习。比如，阅读课程，复制代码，甚至看一段视频，让你所有的感官都工作起来，这样会更好。(视觉，听觉，...)**

使用 IDE 会导致这种情况，因为即使你复制了你在教程或课程或任何其他地方找到的代码，**IDE 会帮助我们，并自己完成一些关键字**，这就是我的同学在课堂上用 Netbeans 学习 java 的方法，但他们必须编写一个简单的代码，显示**“你好，世界！”**他们无能为力，因为他们不知道如何准确地编写这段代码，因为 IDE 自动生成了一些元素作为*public static void main(String[]args){ }*和**这是 100 个其他例子中的一个！**我过去也陷入了这个陷阱，现在，当我学习一门语言时，我倾向于**复制记事本上的所有关键词**，以便**更精确地记住这门语言的每个特征**。

IDE 对于过于完美主义的人来说也是一场灾难。我知道有些人花更多的时间为 vs 代码寻找一个美丽的主题，而不是学习编码，这让我时常发笑😆。

我也有机会见到一些人，他们只知道如何在他们使用的 IDE 上可用的按钮的帮助下使用 git，你知道，那些同时自动提交和推送的按钮。然后，**当他们改变他们的 IDE 时，他们迷失了**，因为他们不知道我们如何这样或那样。**他们太依赖他们的工具了！**

## 我们能做什么？

这就是为什么，我认为在你可以用 IDE 使你的生活变得更容易之前，你必须经历一个没有它的学习阶段，并且这样**确切地理解已经被使用的工具和它背后的技术**。

有一句话告诉我们，先要走，才能跑，我觉得用在这里也可以。

如果你知道你在用什么，你会更有效率，这是我们都在寻找的。**生产力。**

因此，在没有 IDE 的情况下开始学习，花一些时间学习内部的工具，并尝试使用相同的工具。

就像 a 之前说的，试着用一段视频学习然后重现代码，然后对代码坐立不安。这是理解一个元素的最好方法。
