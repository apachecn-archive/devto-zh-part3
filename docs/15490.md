# 如何撰写技术书籍的技巧

> 原文:[https://dev . to/matheus/tips-on-how-to-write-a-technical-book-23k 4](https://dev.to/matheus/tips-on-how-to-write-a-technical-book-23k4)

我在工作中做了一个关于 React 的演讲，每个人都非常喜欢它的内容和呈现方式。我对反馈很满意，并受到鼓舞继续学习和教授这个主题。那时我有了写一本书的想法。

我对教育和软件开发充满热情，我每天都在努力学习新的东西。正因为如此，我觉得我不能把我学到的东西只留给我自己，我觉得我必须以最好的方式分享它，这样更多的人才能学习和提高他们的技能。

今天，我有大约 4 年的职业经验，我担心人们是否会对我的书感兴趣，因为我在社区中没有这么强的名字。我也不知道适当的写作，格式，出版和定价。

我在考虑一个大的教程，但是是用章节组织的，有清晰的解释。这种“教程”的心态对一本书来说也是一样的吗？

欢迎任何建议！
