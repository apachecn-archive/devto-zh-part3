# 你的 dev 超能力是什么？

作为开发人员，我们经常争论学习的最佳方式或构建应用程序的最佳框架。我认为所有的方法都是有效的。我相信有不同的开发风格。我们的学习和职业经历以及个人偏好塑造了我们的风格，并帮助我们在发展的某个方面出类拔萃。

一些开发人员擅长分解复杂的理论，并用简单的术语解释它。而其他人擅长为服务于数百万用户的 web 应用程序优化代码。有些人理解框架的细微差别，可以构建下一个最好的框架。我们都有超能力。

我的超能力是我可以快速学习新的编程语言和框架。我也可以用一个没有文档的现有应用程序来弄清楚它在做什么。

我是如何获得这些超能力的？
我在大学里学的是计算机科学。我们从头开始构建数据库和编译器。当我开始找工作的时候，我意识到没人关心这个。他们想让我构建 web 应用程序。所以，我开始为一个非营利组织做志愿者，并在一个朋友的创业公司工作。我开发的第一个 web 应用程序是用 PHP 和 jQuery 编写的。因此，我通过调整应用程序来学习，并随着学习内置更复杂的功能。我还学习了一些 voicexml，并为一家初创公司的拼车服务建立了一个自动语音系统(有点像优步池。我们来得太早了。那时智能手机还不算什么东西😆).

在我的第一份全职工作中，我为科学家和研究人员开发 web 应用程序。我采用传统的 web 应用程序，并使用现代框架重新构建它们。我把复杂的数据库和电子表格简化，然后在不同的系统中重新构建。开发这些应用程序的人已经不在那里工作了。因此，我重新构建了这些应用程序，没有任何文档，也没有任何人提问。我曾经用过一个 30 页的 SQL 触发器，里面有很多业务逻辑，用 Python 实现了几个函数。

我也曾为一家初创公司工作过。我必须使用在不同框架下编写的应用程序。他们还发明了自己的编程语言(没有文档)🙄。

这并不容易，但这就是我如何逐渐建立起我的超能力的。

凭借我的超能力，我可以用任何框架或语言快速构建应用程序。我可以查看一个我不熟悉的现有应用程序，并修复一个错误或为其添加新功能。我擅长为时间紧迫的初创公司工作，也擅长为预算紧张的政府机构工作。

告诉我你的超能力。这是什么？你是怎么发展出这种超能力的？
