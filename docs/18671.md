# 敏捷中的 MVP 是什么，为什么它很重要？

> 原文:[https://dev . to/maxchurilov/test-automation-strategy-how-we-put-product-quality-on-auto pilot-1c9b](https://dev.to/maxchurilov/test-automation-strategy-how-we-put-product-quality-on-autopilot-1c9b)

任何创业在初期都是无利可图的。要挣钱，需要获得一定的规模，获取客户。当开发一个软件产品时，从一开始最主要的就是不要失去焦点，及时认识和承认错误，不要害怕做出困难的决定。

这就是为什么 MVP 在敏捷产品开发中如此重要。

今天几乎所有的软件都是使用敏捷方法开发的，这种方法预先假定将项目分成几个冲刺阶段。它允许降低风险并快速响应所需的变更。

构建最小可行产品(MVP)有助于企业家向早期采用者传递想法，并获得有价值的反馈，以帮助开发和避免不知情的决策。

你今天使用的大多数应用程序都使用 MVP 方法来使他们的产品成熟(Airbnb、Dropbox、Buffer 等等)。这些例子表明，在发布一个全功能的解决方案之前，你绝对应该考虑制作一个 MVP。
