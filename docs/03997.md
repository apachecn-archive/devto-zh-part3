# 人工智能在开发运维转型中的作用

> 原文:[https://dev . to/pcloudy/the-role-of-the-artificial-intelligence-in-transforming-devo PS-1g 6](https://dev.to/pcloudy/the-role-of-artificial-intelligence-in-transforming-devops-1g6)

DevOps 帮助企业以最快的速度和最少的问题构建软件。在自动化工具的帮助下，上市时间加快了，并且在持续部署中错误修复得更快了。人工智能与 DevOps 非常一致，因为主要焦点是自动化过程，而利用人工智能，系统可以识别模式，预测问题并提供解决方案。主动方法提高了软件开发生命周期的整体效率。所以我们来看看 [AI 是如何改造 DevOps](https://www.pcloudy.com/moving-beyond-traditional-app-testing-with-ai-and-devops/) 的。

**反馈循环和关联数据**
devo PS 的主要作用是在流程的每个阶段持续获取反馈。人们通常使用性能监控工具来获得运行应用程序的反馈。这些工具以日志文件、数据表、性能矩阵和其他类型的形式收集大量信息。监控工具使用机器学习来及早识别问题并提出建议。DevOps 团队使用这些建议对应用程序进行必要的改进。很多时候，团队使用两种或更多种工具来监控应用程序的健康状况，来自所有平台的数据可以在机器学习的帮助下进行关联，以更深入地了解应用程序的功能。

**软件测试**
人工智能正在通过改进软件开发流程和[提高测试效率来改变 DevOps】。无论是回归测试，用户接受度测试，还是功能测试，这些都会产生大量的数据。人工智能可以找出以结果形式收集的数据中的模式，并识别出产生大量错误的糟糕的编码实践。DevOps 团队可以使用这些信息来提高他们的效率。](https://www.pcloudy.com/5-ways-ai-is-changing-test-automation/)

异常检测
开发安全是软件开发的一个重要方面，因为安全是任何成功的软件实现的关键。分布式拒绝服务攻击越来越多，企业需要做好准备保护自己的安全系统免受黑客攻击。 [DevSecOps](https://dzone.com/refcardz/introduction-to-devsecops?chapter=1) 可以使用人工智能来增强安全性，通过中央日志记录架构来记录威胁，并运行基于机器学习的异常检测。这将有助于企业主动减弱黑客和 DDOS 的攻击。

**警报**
开发运维方法可能会导致团队收到大量没有优先级标签的警报。这将在团队中制造混乱，因为在持续开发的环境中处理所有的警告是非常困难的。在这种情况下，人工智能可以通过标记警报并对其进行优先排序来提供帮助，以便可以立即处理紧急事件。

**根本原因分析**
要永久解决问题，根本原因分析是必要的。尽管与用补丁解决问题相比，这可能要花一些时间，而补丁可以提供即时的解决方案。为了找到问题的根本原因，开发人员将不得不花费时间来推迟产品的发布。人工智能可以通过在收集的数据中寻找模式来加速这个过程，并实现修复根本原因。

通过实现 AI，可以使用收集的数据来找到模式，并加快开发过程。有组织的数据更有用，并使预测成为可能。最佳实践是使用机器学习来自动化耗时的任务，这将确保 DevOps 团队的平稳有效运行。
