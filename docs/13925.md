# 介绍 DocsThursday:每月一次的文档“聚会”

> 原文：<https://dev.to/felicianotech/introducing-docsthursday-a-monthly-documentation-party-2fmb>

我喜欢开源文档，如果你正在阅读这篇文章，也许你也一样。我有一个“激励、教育、激励”的个人口号，文档正好符合这个口号。
文档使我们能够**教育**他人，达到近乎无限的规模。

你知道，我的职业生涯部分归功于免费且可访问的 Linux 服务器文档，这些文档可以在 Linode Docs 网站上获得。
我是 Ubuntu 的铁杆用户，但我无法告诉你无数次 [Arch Linux Wiki](https://wiki.archlinux.org/) 帮助我走出困境。

文件是无价的。

我想**激励**更多的人做出贡献，并互相学习如何才能做出更好的贡献和更好的文档网站。
[DocsThursday](https://DocsThursday.com) 是一个聚会、活动、研讨会或任何你想叫它的东西，在每个月的第三个星期四创建和贡献开源文档。

## 为什么 DocsThursday？

周四没什么特别的。我本可以在周三或周五做一个文档，但这并不吸引人，而且我们都知道周二是墨西哥卷日。那就周四吧。

DocsThursday 的灵感来源于 DigitalOcean 的 [Hacktoberfest](https://hacktoberfest.digitalocean.com/) 、 [24 拉请求](https://24pullrequests.com/)，最重要的是，写文档的[写作日](https://www.writethedocs.org/conf/portland/2019/writing-day/)。我正试图利用写作日的能量和生产力，在你当地的聚会上，与你公司的同事，甚至是在线黑客马拉松上，更频繁地复制它。

让我们花 30 分钟，一个小时甚至更多，仅仅一个月一次，来给开源文档一些爱。

感兴趣吗？查看网站，[DocsThursday.com](https://DocsThursday.com)。

## 你愿意帮忙吗？

DocsThursday 是全新的，我还在考虑如何使用它。
[网站是开源的](https://github.com/felicianotech/docsthursday.com)，由[雨果](https://gohugo.io/)打造。
请随时打开 PRs 来改进网站，或打开问题来提问或开始讨论。
你也可以在这里开始讨论/提问，或者在 Twitter 上给我发[消息](https://twitter.com/FelicianoTech)。

如果你有这些感觉或类似的想法，请说出来！让我们一起来做这件事。😄
