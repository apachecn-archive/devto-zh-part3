# 如何将 git 分支用于您一直想要的有组织的代码库

> 原文：<https://dev.to/codemochi/how-to-use-git-branches-for-the-organized-code-base-you-always-wanted-2m41>

在这个视频博客中，我将回顾为什么我们使用 git 来管理我们的项目，为什么把你所有的代码提交给一个主分支是绝对疯狂的，并且是你应该*永远不要*做的事情。相反，您应该创建特性分支——它们允许您创建离散的代码块，您可以一次一个特性地将这些代码块添加到您的登台分支中，直到您有了可以部署到主版本的版本。

您将看到这是如何让我们始终拥有一个准备就绪的生产分支，一个拥有所有最新功能但尚未准备好投入使用的登台分支，以及一个我们可以在不影响任何其他人的情况下开发和破坏的功能分支。
