# 不需要编码的技术领域

> 原文:[https://dev . to/hack sultan/tech-fields-that-do-require-coding-1epe](https://dev.to/hacksultan/tech-fields-that-do-not-require-coding-1epe)

名词（noun 的缩写）b:你一定是个了不起的读者和优秀的思想家。(不要害怕，不用写那些你觉得很无聊的代码，你仍然可以在科技领域工作)

1.  UX/用户界面设计师:前端/后端开发人员令人惊叹，公司雇佣专业人员从用户的角度思考，你将负责设计用户界面，了解终端用户如何与功能和产品联系起来。这个领域是任务型的，你实际上需要很多设计技巧才能做到完美。

2.  软件质量测试员:公司不先测试就不推出 app 或功能；自动或手动。尽管自动化测试很好/更快，但最终用户的感觉和体验是需要的，而令人惊奇的人可以担当这个角色。你可以提出意见和改进建议💪

3.  SEO 分析师/专家:无论一个产品(网站/应用程序)多么神奇，如果人们没有机会使用它，它就没有用。不要把这个领域和市场营销混为一谈，尽管你会像市场营销一样思考，不同的是你也会和程序员一起思考和研究👌

4.  数据分析师:这个职位需要惊人的数学计算、统计和分析技能，你需要精通各种数据处理应用程序。相信我，你的业绩有时会决定公司的发展方向。💪一些核心数据分析需要掌握 Python 或 R 的知识，但是你可以很好地利用这些软件。同样的，我也不会调用 Matlab 编程。

5.  项目/计划经理:这实际上并不等同于计划高于项目。你需要让自己适应一个特定的领域，你知道你不是在写代码，但是你应该知道为什么你希望你的开发者使用 GraphQL 而不是 REST，为什么 Angular 不是 React。你会读很多书，理解技术术语，这个角色的工资比编程领域高很多倍，尽管很多公司要求你有 CS 学位，你可以参加纳米学位课程，成为这个角色的完美人选。
