# 追踪你的进步以提高你的自信

> 原文:[https://dev . to/codeidoscope/tracking-your-progress-to-improve-your-confidence-12lh](https://dev.to/codeidoscope/tracking-your-progress-to-improve-your-confidence-12lh)

我几乎每天都为我编写的代码创建一个日志，从 12 月份开始就这样做了。我这样做主要是为了记录我的进步，这样我就可以看到我已经走了多远，也可以阻止自己认为我做不到。编程很难，反思我们在编程时学到了多少很重要，因为它帮助我们意识到我们已经走了多远，并提醒我们付出的所有努力。

通常，我们认为别人知道的比我们多得多，或者他们天生比我们更擅长编程。然而我们看不到的，是他们付出的努力，遭遇的失败，面对的艰辛。我们通常看到编程中毫不费力的部分，当我们做的时候感觉不一样时，就会感到没有动力。

通过每天记录我的工作，我可以清楚地看到过去三个月我做了什么，以及我走了多远。它帮助我意识到，一点一点地，我在成长，变得越来越好，它也帮助我对自己的编码能力更加自信。我今天写关于日志的文章，希望它能帮助那些像我一样，可能正在努力看到他们所取得的成就的人。

## [](#what-is-logging)什么是日志记录？

日志记录只是记录一些您想要跟踪的信息。日志是一个文件，你用一种你觉得有用的格式写下了对你来说重要的事情。我将详细说明我是如何记录日志的，所以请继续阅读！

## [](#what-are-the-benefits-of-logging)日志记录有什么好处？

#### [](#keeping-track-of-your-progress)记录你的进步

通过记录你每天做的事情，你可以看到自从你开始记录以来，你学到了多少。它让你很容易看到你什么时候学到了什么，那是多久以前的事了，而且如果你定期写日志的话，看你投入了多少时间、工作和努力也是很好的。

#### [](#keeping-track-of-the-issues-youve-encountered)记录你遇到的问题

当我的代码有问题时，我有时会把它写在日志中。这迫使我想办法把我的问题解释清楚，这有助于我更快地找到解决方案。就像橡皮鸭一样！如果我需要问别人，那么我确信我可以很好地表达我的问题，我问的人可以帮助我。

#### 记录你找到的解决问题的方法

一旦我写下了我的问题，我会写下问题的解决方案，这样当我再次遇到它时，我就可以在我的日志中找到它，并更快地解决它，而不是从头再搜索一遍。

#### [](#keeping-track-of-the-resources-you-used-when-solving-a-problem)记录你解决问题时使用的资源

我保留了一个博客帖子、Stack Overflow 答案、Safari O'Reilly 书籍等的链接列表。帮助我解决了一个问题或者让我对一个话题有了更多的了解。这样，我就可以在我试图解决的问题的上下文中找到这些链接，这样，如果我需要重新访问它们，我就可以刷新我的记忆。

#### 记录你想在以后读到的更多话题

通常，我会遇到一个新的想法、概念或技术，我想了解更多，但在工作时没有时间。所以我把它记在我的日志里，以后有时间的时候再挖出来。

## [](#so-how-do-i-log)那么，我该如何日志呢？

我已经使用下面的模板几个月了，它对我来说效果很好。日志的美妙之处在于，你可以让它成为你自己的，决定你想要记录什么，所以如果有什么对你没有帮助，那就改变它！

```
# Codelog #X - XXXX-XX-XX

-----

## Plan for the day

-----

## To investigate

-----

## Learned today

-----

## Done today

-----

## Plan for tomorrow

-----

## Resource list 
```

通过“为一天做计划”，我设定了一天的目标，并决定了我将做什么，这让我有责任感并帮助我设定目标。另一方面,“为明天做计划”,我会把我在某一天可能无法完成的任务放在那里，或者我会为第二天制定计划，当我在那天开始写日志时，我会回顾这些计划。

我在“调查”中记录我想阅读的内容，在“今日知识”中记录我的问题及其解决方案，在“资源列表”中记录我找到的有用链接。

## [](#where-do-i-log)我在哪里登录？

我曾经把我的日志放在我的博客上，但是我已经把它们转移到了他们自己的仓库里，因为它们太多了，很难跟踪。

**编辑**:如果你想要自己的日志库，我已经创建了一个[模板库](https://github.com/codeidoscope/codelogs-template)供你随意使用。

您可以在 Word/Google Docs 文档、笔记应用程序、笔记本、网站或任何您想去的地方记录日志。只要确保当你需要回顾你所拥有的东西时，它很容易被找到和导航。

## [](#conclusion)结论

过一会儿，你会看到你的日志收集增长。你将能够重温你的日志，回忆起你解决棘手问题的时候。你将能够更快地找到你已经有过的问题的答案，并且你不会告诉自己你做不到，或者你不够聪明，或者你知道得不够多。

因为你能做到，而且你足够聪明，学习是一个终生的过程！所以祝你在日志方面好运，我希望你和我一样觉得它很有用！
