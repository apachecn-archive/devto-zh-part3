# 2019 年软件自动化测试人员职业概述

> 原文:[https://dev . to/dohertykonelia/career-overview-of-software-automation-testers-in-2019-1bm3](https://dev.to/dohertykornelia/career-overview-of-software-automation-testers-in-2019-1bm3)

在这篇文章中，我将向您展示，如果您想作为一名测试人员保持市场地位，为什么获得测试自动化技能对于您的质量保证职业是必需的。如果您刚刚开始测试自动化，我将告诉您需要学习哪些技术技能和自动化工具。

## [](#why-2019-is-a-great-time-to-be-an-automation-tester)为什么 2019 年是做自动化测试人员的大好时机？

在过去的几年里，QA 和测试分析师的招聘广告强调了测试自动化技能的需要。四年前，招聘人员明确建议测试人员获得技术和测试自动化技能。从那以后，对测试自动化的需求持续增长，现在一些团队正在完全淘汰手工测试人员，只雇佣具有测试自动化技能的测试专业人员。

如今，越来越多的生命周期组件正在被自动化，并且关注于整个生命周期的自动化。应用程序和新版本数量的增加也提高了自动化的重要性。根据 2018-2019 年世界质量报告，测试自动化仍然是生成和执行功能测试用例的最受欢迎的方法。到 2018 年底，18%的功能测试用例是使用测试自动化工具生成的，16%是使用测试自动化工具执行的。类似地，所有安全性测试中有 16%是使用自动化工具执行的，自动化也应用于所有性能测试用例中的 16%的执行。非常令人鼓舞的是，15%的端到端业务场景也是使用测试自动化工具执行的。

由于测试自动化是目前世界上发展最快的职业领域之一，很少看到测试角色的工作列表中没有对测试自动化和至少一种脚本语言的理解的要求。在我写这篇文章的时候，实际上大约有 16，955 个测试自动化工程职位，30，705 个 Glassdoor 职位，5，580 个 LinkedIn 职位。这些数字表明如今对这种技能的需求很高，因此你不必太担心转行的风险。请记住这样一个事实，作为一名自动化测试员，你总是被需要的，并且会发现不缺乏领先的公司尽最大努力来招募你，并且在你的职业生涯中不断有机会成长到管理级别。

## [](#what-do-you-need-to-prepare-for-your-automation-testing-career)自动化测试职业需要准备什么？

为了在这一领域出类拔萃，测试人员必须了解测试行业中使用的自动化工具和现代技术。这并不意味着你被要求成为程序员，但是它确实要求自动化测试人员有编程背景。编程技能可以帮助测试人员更好地与开发人员交流，并参与单元测试。由于自动化测试与代码密切相关，团队使用的编程语言知识是测试人员的关键要求之一。如果产品是用 Ruby 编写的，那么测试人员需要在简历上有 Ruby 的经验。除了明确的编程语言，如 Java、C/C++、Ruby、Python、Perl 之外，其他一些必需的技能包括了解 SQL 和关系数据库，具有 HTTP/HTTPS、HTML、CSS、JS、XML 和测试 API 的经验，了解其中一个框架(如 Cucumber for Ruby)绝对是一个优势。

您可以通过参考一些流行的软件测试博客和论坛，如 Utest、软件测试帮助、Stickyminds、测试部等，开始您的测试自动化之路。这些网站帮助教育和激励各种类型和规模的团队来测试自动化。

正如我提到的，你拥有的技术和测试自动化技能越多，你作为测试人员就越有市场。然而，任何认真对待测试职业的人都应该选择一个强大的自动化工具。市场上大多数可用的测试自动化工具都是用编程思维开发的。一些最流行的工具:

*   加泰罗尼亚工作室
*   UFT

在这些工具中，我发现 Katalon Studio 是最用户友好的自动化软件测试工具。您可以轻松地自动化 web 服务和移动测试，并在您想要的平台上执行脚本。最有趣的功能是间谍工具，只需一次点击就可以捕获网络和移动对象。Katalon Studio 足够智能，可以用所有可能的定位器捕捉您的元素。如果你刚开始学习自动化测试的编程，请在你的项目中找到需要重复一千次的最简单的测试场景。然后创建一个简单的自动化脚本来运行它并检查测试的结果。这些简单的测试将激励你创建越来越多的脚本，尽可能避免手工操作。

获得测试自动化方面扎实知识的其他方法是参加当地的聚会和会议，例如 STAREAST、STARWEST、敏捷测试日和 QualityJam。上个月，我有机会参加了由 Katalon Studio 和 ToolsQA 共同主办的关于 2019 年软件测试趋势的网络研讨会。我觉得这个网上研讨会很棒，也很实用，我建议你尝试注册类似的活动。
