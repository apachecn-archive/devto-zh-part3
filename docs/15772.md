# 解释脑外科手术，以便一个五岁的孩子可以理解它

> 原文：<https://dev.to/camainc/explain-brain-surgery-so-that-a-five-year-old-child-can-understand-it-4o4i>

声明:这篇文章是关于我的一个好奇心。我没有和任何特定的人说话，所以如果你因为做了我要说的事情而感到内疚，请不要介意。不是关于你的。

我好奇的是，为什么人们总是要求解释一些事情，“好像是给一个五岁的孩子解释一样。”

这有什么用？

一个五岁的孩子没有参照系来理解我们作为开发人员所做的工作。即使有人能以一个五岁小孩能理解的方式解释软件开发概念和技术，这对其他开发人员又有什么帮助呢？

我们让人们解释一项技术，让一个高中生也能理解它，怎么样？至少一个高中生会有一个关于计算机的参考框架，以及从用户的角度看它们能做什么，他们中的一些人甚至上过基础编程课。

我得到了能够用简单的术语解释非常复杂的事情的概念，我认为这是一个有帮助的想法。但是，至少在我看来，试图把一些东西降低到一个五岁小孩的水平有点牵强。

"向一个五岁的孩子解释脑外科手术."

“有时人们会伤透脑筋。所以脑外科医生打开那个人的脑袋，改变他们的大脑，这样它就不会再坏掉了。”

好吧，假设一个五岁的孩子知道什么是大脑，这可能有助于他们理解脑外科手术，但这将如何帮助一个崭露头角的医科学生呢？

我认为这和试图在一个五岁的水平上解释软件开发概念和技术是一样的。这可能有助于人们理解一些最基本的东西，但这将如何帮助一个崭露头角的软件开发人员呢？

无论如何，那只是我今天好奇的话题。如果你认为在一个五岁的水平上解释事情是有帮助的，请在评论中畅所欲言。

谢谢，祝您愉快。
