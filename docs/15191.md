# 开发者生活 2019 年 1 月

> 原文:[https://dev.to/tomavelev/developers-life-january-2019-17ib](https://dev.to/tomavelev/developers-life-january-2019--17ib)

这个月我开始制作几个视频，展示如何使用我的工具 [GeneratorApp](https://www.youtube.com/playlist?list=PLD7vZwzN2iX7hdFq83NCX6HLKDMfasWZR) 。我可能会在某个时候用视频截图和描述写一篇博文。在制作视频的过程中，我发现需要再次验证我的工具程序是否正常工作，我设法修复了几个错误和不一致之处。

我还为非开发人员发布了一个早期版本的 GeneratorApp。它在我的网站上缓存模型，Jenkins VM 的任务是每半小时扫描一次新的模型定义——构建它们并上传 APK 文件。在我充分改进 android 版本后，我正在考虑对桌面数据库应用程序的 Jar 文件做类似的事情，最终目标是使它们在没有任何第三方服务、计算机或任何东西的情况下同步-使用代码和我从过去的应用程序中获得的一些经验，如笔记、预算、文件。如果我达到这一点——这一次，我的工具的输出将生成不限于我认为有用的数据模型的应用程序。在整个过程中，所有的更改都将在 GeneratorApp 内部进行，因此任何人都可以根据任何业务需求生成和修改代码。

不幸的是，我没有花太多时间。我已经完成了代码级(lint 和 stuff)的优化和改进，我正在为设备位置访问制作一个 API 这将使用现有的 API。

我也开始研究将我的网站[http://kakvoiadesh.com/](http://kakvoiadesh.com/)与谷歌的语音助手和亚马逊 Alexa 整合的可能性。
8 年前，我尝试过实现语音识别软件。然后，这项研究太技术性了，试图理解语音识别的数学，试图建立一个需要语法短语定义的框架，但失败了。

今天，大公司提供 SDK 和工具，用于建模和实现与人类的实际对话，并负责随着时间的推移不断发展和改进将语音转换为可理解字节的技术挑战。是的，你(和我)最终做的事情是公司的财产，但如果它增加了我自己软件的覆盖范围，那也不是太坏。凡事都有利弊。

我和助手做了一些概念验证。有一些技术挑战需要解决，如果我完成了这些挑战，这将释放出一种潜力，为任何工具、网站或软件添加与谷歌语音软件交互的可能性。之后，我也可能会尝试 Alexa。与机器“交谈”在西方世界开始变得非常普遍。不用再坐着，站在任何尺寸的显示器前执行任务。正如一位营销人员所说，速度和时间节约是任何技术最有价值的特征之一(即使是从财务角度来看)。
