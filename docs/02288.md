# 跟上科达希人

> 原文:[https://dev.to/molly/keeping-up-with-the-codeashians-534c](https://dev.to/molly/keeping-up-with-the-codeashians-534c)

这些天来，科技世界以令人眼花缭乱的速度前进。感觉就像几乎每天你都会听到一种“每个人”都在使用的新型数据库或语言。试图跟上这一切的压力可能是无法承受的。我写这篇文章的灵感来自@emmawedekind 的一条推文，这条推文真的引起了我的共鸣(如果你看了所有的回复，显然还有许多其他人！).

那种你永远跟不上的感觉对我来说太真实了。通常，当别人开始在谈话中抛出新的技术时，我不得不偷偷谷歌他们提到的每个名字，同时我点头附和，因为我不知道他们在谈论什么。这就是冒名顶替综合症开始蔓延的时候，我感到内疚。我认为我不太像一个开发者，因为我不了解所有这些新技术。

让我感到沮丧的一部分是，那些一直谈论这些新技术的同事们，正是那些在空闲时间里研究这些东西的人。不幸的是，我没有时间做那件事。除了编程，我还有第二个爱好，那就是骑马。

[![](../Images/d7f12ad1fe2a19a946763b04af8de58d.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--I21nSOks--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/fhm9mco43sk8ew9ln6o0.jpg)

我不是说一周一次，我一周骑 4-5 天，有 5-6 匹马要骑。我工作之外的大部分空闲时间都用来骑马了。我不会用我在马鞍上的时间来交换任何东西。它帮助我充电，给我带来很多快乐。但有时我还是会想，我是否应该在业余时间做更多专注于技术的事情。

我知道内心深处的答案是响亮的“不！”但这需要努力让自己确信这是正确的决定。看到艾玛的推文和阅读回复是超级令人欣慰的。我终于觉得我不是唯一一个在这方面挣扎的人。这里有几个回答，我也认为是对这场斗争的描述。

在阅读所有回复的同时，我还通过 Twitter 收到了来自一位网站可靠性工程师同事的 DM。他读了我写的关于自信的《工程基础知识》一文，想知道如何找到自己的位置。

> 对我来说，这就像是 SRE 每周都在变化，老实说，这让我感到焦虑，而且几乎增加了我花时间了解新技术的速度，我从经验以及其他人那里知道，这是不可持续的，会导致倦怠。

🙌哈利路亚，一个和我有同样感觉的人！我给他的建议是:

> 试着抑制住想要抓住最新事物的冲动，找到对你有用的东西并坚持下去。深入了解它能做的所有事情。当你掌握它的时候，你可能会错过一些转瞬即逝的新鲜事物，是的，但我认为这是值得的。

我不得不不断重读这个建议，提醒自己不可能什么都学会。给自己施加压力，让自己认为我们需要跟上并不是答案。我们需要接受一些事情从我们身边经过。只要我们拥有做好工作所需的技术，我们就万事俱备。
