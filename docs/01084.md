# 激励、授权和自动化:如何建立梦之队

> 原文：<https://dev.to/yanchauskayte2/motivation-delegation-and-automation-how-to-build-a-dream-team-c4p>

亚历克斯·卡塔耶夫领导斯凯恩公司的开发团队。

我不想自吹自擂，但我真的相信我成功地创建了一个梦之队，全公司最好的团队。从那以后，我试着教其他人如何把他们的团队变成梦之队。

以下是你也可以做到的方法！

# 遇见蒂姆

让我们想象一个团队领导。我们叫他蒂姆。蒂姆是个好员工，他接受任务并完成任务，因为这是他工作的主要内容。但是团队的其他成员——产品经理、CTO、开发人员——总是需要 Tim 的一分钟时间。所以蒂姆开始慢慢耗尽精力。

他没有足够的睡眠，24/7 都在想工作，他被任务淹没，其中一些最终被遗忘或淹没在积压的工作中。蒂姆很快会发生什么事？他要么辞职，要么犯一个会让公司损失很多钱的错误。

[![](img/a315908ff063c5417f9fca76782b45d4.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--LrBJoHQW--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/o6mlgyhi35l3v0veu6ky.jpg)

让我们看看蒂姆在我工作的公司会如何工作。目前，我们有 17 个开发团队，每个团队通常由产品经理、团队领导和开发人员组成。

产品经理设定业务目标，管理我们的收入和支出，与客户和顾客沟通。团队领导——在我们的例子中是 Tim——是首席工程师。

他负责技术战略、技术债务、技术解决方案——一切技术事务。但他也是一名项目经理，监督团队内部的所有流程——雇用和解雇人员，教导和指导团队，领导和激励团队。

我们怎样才能让蒂姆的生活更轻松？你的第一个猜测是将所有的小任务委派出去，这样蒂姆就可以专注于真正重要的事情，比如投资和管理。但是这个不行。为什么？因为授权需要时间，你需要详细解释你希望事情如何完成。但是蒂姆已经一天工作 10 个小时了，他没有时间做那个，他宁愿自己做所有的事情。

让我们试着创造一种新的、更健康的授权方式。

# 一个频道回答所有问题

首先，蒂姆需要找到简单、容易完成的任务。大多数情况下，你会知道这些是什么。如果你不知道，看看工作日志。我一般会把白天做过的事情都写下来，然后回顾一下。这些小任务中的大部分是解决来自开发者、产品经理、CTO、客户和顾客的问题。我的解决方案是为所有问题创建一个宽松的渠道。

开发人员和 QA 工程师轮班待命，回答所有的问题。如果他们没有答案，他们只是标记开发人员在产品的这一部分工作。我不在私信中回答任何问题。然后我们写了一个机器人，解释你打开频道的规则。顺便说一下，我们已经任命员工或可信的自由职业者来编写机器人，团队开发人员不必这样做。这就是我如何消除大量小而耗时的任务。它还可以进一步优化，例如使用 FAQ。

# 小型任务的内部外包

我们假设的蒂姆摆脱了大部分日常任务。下一步是什么？

[![](img/7b56924f4a6a0c81c0d2f93a0bc41544.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--Zyz7DqEe--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/pyo6ertj0l717zeds6yp.jpg)

Tim 拥有一支高素质的专业团队。但这个项目时不时需要做一些小而简单的事情，比如传输数据、写字幕或整理文档。蒂姆不想把这些琐碎的工作交给高薪的员工，所以他自己做。

对于这样的事情，斯凯恩有一个私人助理部门。它的工作方式类似于公司内部的 YouDo 服务(一种为 mircojobs 雇佣人员的俄罗斯服务——译者注)。助理可以接触到所有的文件，他们接受质量控制的监督，这些人不是随机的，他们是经过精心挑选和训练的。公司里的任何人都可以使用模板卡通过 Trello 为某项任务任命助理。

# 故障和危急情况

现在，当大部分工作完成后，蒂姆不会被小事分心，并且确信负责这些小事的人不会把事情搞砸。但是还有一个方面需要考虑——危急情况。生产服务器半夜崩溃怎么办？去叫醒蒂姆来修理它？不，如果我们需要一个休息好的团队第二天领导。所以我们设置了一个机器人，如果服务器崩溃，它会打电话。但它不打电话给 Tim，而是打给一个随叫随到的开发人员。开发人员轮流值班，可以访问所有的诊断工具和所有的服务器。Tim 不在那个团队中，只有当开发人员没有醒来调查为什么会发生这种情况时，他才会参与进来。

此外，我们还编写了一个应急文档，解释在所有东西都崩溃时该怎么办。

# 委派会议

蒂姆睡得很好，现在准备好工作和参加会议。他有很多。他们可以被授权吗？蒂姆在团队中有合格的人，但他不能任命任何人，这是行不通的。

我做了什么？我们以技术评论为例。首先，我总结了我所有的专业知识，开发了一个技术综述的算法。它涵盖了许多方面，比如我们为什么要这样做，如何进行结构化审查，讨论的生活窍门，任务和投票的模板，时间表和样本。我与团队分享了这份文件，我们一起工作，现在我们有了 Skyeng 专业知识的总结。

然后我问是否有人想负责技术审查。事实证明，每个人都渴望这样做。我们制定了一个时间表，现在轮流做技术评估。我也这么做，因为我需要留在圈子里。会议结束后，每个参与者都要完成一份调查问卷，以评估会议进行得如何以及主持人做得如何。这种反馈有助于改进流程。我们还为每个团队制定了基本原则，即使没有我的意见，也能帮助他们做出决定。这些包括公开讨论问题和风险，重视高质量的结果多于快速的结果，不外包任务。原则可能会因团队而异，例如，你可能需要外包任务或更喜欢快速而不是质量。

按照这个算法，Tim 可以限制他的会议次数。

# 蒂姆是做什么的？

现在，当蒂姆从他的大部分工作中解脱出来时，他可以做一些别人做不了的事情。它们是什么？

首先，是团队建设和保持团队精神。其次，雇佣和解雇人。第三，在会议上发言，分享他的经验。我相信你花在会议上的时间是对你团队未来的投资。

我所描述的这些都不是一朝一夕可以完成的。从我的经验来看，你至少需要一年的时间来设置流程和建立一个负责任的团队。但是无论你是在办公室还是在网上工作，使用相同的原则是完全可能的。
