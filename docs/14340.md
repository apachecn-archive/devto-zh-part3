# Web 辅助功能入门

> 原文:[https://dev . to/vorillaz/getting-started-with-web-accessibility-4k 82](https://dev.to/vorillaz/getting-started-with-web-accessibility-4k82)

在过去的几年里，web 行业正在快速变化:我们有了新的工具、框架和库，使我们能够创建丰富而有创造性的用户界面。自 1997 年以来，万维网联盟(W3C)推出了网页可访问性倡议(WAI ),从那时起，我们就有了强大的工具集和大量资源来帮助我们构建一个更易访问的网站。
我们的工作影响的人比你想象的要多，全球超过 [5 亿网络用户](https://www.sitepoint.com/how-many-users-need-accessible-websites/)正遭受某种残疾。在这篇文章中，我将介绍一些关于可访问性的资源和基本工具。

## [](#resources)资源。

WAI 官方网站有许多关于可访问性问题的支持资源。 [Mozilla Foundation](https://developer.mozilla.org/en-US/docs/Web/Accessibility) 和【Google web fundamentals】网站([https://developers . Google . com/web/fundamentals/accessibility/](https://developers.google.com/web/fundamentals/accessibility/))也是一个很好的起点，上面有大量关于可访问的 web 界面和应用程序的实例和教程。

## [](#case-studies)案例分析。

让你的雇主、同事甚至你自己相信可访问性真的很重要可能会很难，甚至令人沮丧。除了人道主义的角度，可访问的用户界面可以增加你的产品或系统的收入和访客。

*   [NBC 学](http://nbclearn.com/accessibility/)已经达到了[第 508 节的标准](https://www.access-board.gov/guidelines-and-standards/communications-and-it/about-the-section-508-standards)
*   粉碎杂志封面 [BBC iPlayer 案例研究](https://www.smashingmagazine.com/2015/02/bbc-iplayer-accessibility-case-study/)
*   BBC 也有一个很棒的资源库，里面有很多实用的案例研究和用户关于他们工作的故事。
*   [是的！你的网站可以(也应该)被访问，劳拉·卡瓦亚尔分享了许多关于《金融时报》向更容易访问的网站转变的见解。](https://youtu.be/tGF4-lPTJR0)
*   W3C 也有一个知识库，里面有一些来自大公司和组织的案例研究。

## [](#getting-actually-started)开始实际操作

像往常一样，一下子打破一个庞然大物是低效的，需要大量的计划和努力。虽然你可以马上开始，但是调整产品的几个部分可以显著改变用户的体验。

*   W3C 为开发团队提供了一些很棒的技巧。
*   A11Y 项目是一个社区驱动的知识库，拥有大量的资源和支持文章。对于你的项目，还有一份很棒的[清单](https://a11yproject.com/checklist)。
*   Emily Mears 为新来者分享了一些很好的起点。
*   我最亲爱的朋友和同事格里戈里娅·庞蒂克写了一篇关于[无障碍低挂水果](http://www.grigoria.gr/2018/08/25/accessibility-low-hanging-fruits.html)的指南
*   Juan David quiones 也有一些关于如何开始的见解。

## [](#final-thoughts)最后的想法

开始使用 web accessibility 可能是一种思维模式，需要改变你的开发和设计习惯，而不是分解和重构你的整个代码库。在接下来的教程和帖子中，我将讲述更多关于建立开发环境的技术细节，并自动过渡到更易访问的代码库。
