# 了解物联网技术在交通制造业中的风险

互联技术正以飞快的速度改变着运输行业。火车上的微型传感器捕捉旅行数据以提高运营效率，自动驾驶汽车已经从科幻小说转向城市街道，以实时交通数据为特色的导航系统通知我们的路线。

如果设备和通信机制正常工作，物联网(IoT)交通应用可以提高便利性、安全性和生活质量。如果该技术出现故障，甚至暂时离线，可能会导致财产损失，并对乘客和旁观者造成严重伤害。物联网制造商和软件公司可能会被追究责任。

物联网生态系统中的每一家公司都应该意识到他们可能会发现自己负有责任的方式，并采取行动将这些风险降至最低。以下是以技术为中心的公司在为交通行业开发物联网应用程序时应该了解的四个关键风险类别。

*   **财产损失**。如果物联网设备存在缺陷或无法按预期运行，可能会导致财产损失。这些风险可能包括个人财产(如笔记本电脑)或不动产(如办公场所)的使用损失或物理损坏。**例如**，一个旨在捕捉机车数据以通知预测性维护的物联网传感器可能无法准确读取机车的性能。这可能导致车辆在没有及时维护的情况下行驶，如果维护失败导致机车与其他财产相撞，则可能导致数十万美元的可预防损失。设备制造商可能会因其设备发送不准确的信息而被起诉。
*   **身体伤害**。如果一个设备在任何时候都不能像广告宣传的那样工作，设备开发者可能要对由此造成的任何伤害负责。物联网制造商和软件公司可能面临人身伤害风险，这可能使他们在发生故障时承担责任。**例如**，由自动驾驶汽车制造商和物联网软件开发公司合资生产的自动驾驶卡车可能有软件漏洞。如果这个错误导致迎面而来的交通事故，那么受伤的司机和乘客可以起诉软件开发商，要求赔偿由此产生的医疗费用。
*   网络风险。网络风险可能是由网络罪犯的攻击、无效的云安全政策、IT 安全软件故障或心怀不满的员工的破坏造成的。如果信息系统中的数据保护不当，可能会导致潜在的财务损失、声誉受损或身份被盗。公司有责任确保他们的物联网存储和通信功能遵循可靠的数据实践，但如果软件设备制造商的设备不能像宣传的那样工作，他们仍然可能要承担责任。**例如**，如果网络窃贼拦截了一家物流公司的传感器和云之间不安全的机器对机器(M2M)通信，窃贼就可以将数据出售给竞争对手。该设备制造商可能会因未能保护 M2M 通信而被起诉。

### 最大限度降低物联网风险

*   考虑适当的质量和风险管理系统，以帮助确保产品始终符合要求和规格。
*   将网络安全构建到设备中，包括服务器机房和数据中心的物理安全，以及对云数据存储的全天候监控。
*   评估公司合同实践，探索合同风险转移和防御/赔偿条款，这些条款可以将风险转移给其他方。

与代理人或经纪人讨论相关的保险范围也是有帮助的。财产、汽车责任、汽车物理损坏、一般责任、技术错误和遗漏(E&O)责任以及与网络相关的第一方保险可以帮助防范潜在的责任。
