# 我们能避免扯淡泡沫吗？

> 原文:[https://dev . to/fpuffer/how-can-we-how-can-avoid-the-bullsh-bubble-4fng](https://dev.to/fpuffer/how-can-we-avoid-the-bullshit-bubble-4fng)

许多软件开发处理将一些设计糟糕的系统与另一个设计糟糕的系统集成，导致一个更难使用和维护的系统。至少这是我在过去几年中观察到的，主要是在所谓的企业软件领域。

这是为什么呢？我不确定，但我猜这是因为它产生收入。从经济角度来看，这是有道理的。对我们中的一些人来说，垃圾软件只是一种可持续的收入来源。

但是这对开发者或用户来说是健康的吗？尽管从事这种类型的项目有时可能非常具有智力挑战性，但我想我们大多数人都不会对做另一份无聊的工作感兴趣。此外，这种影响对我们整个行业的声誉也是不利的。

你也做了同样的观察吗？有出路吗？

(顺便说一句，幸运的是，我自己的大部分项目都不属于这一类，但其中一些项目属于这一类。)
