# 自动化测试的机器学习

> 原文：<https://dev.to/lambdatest/machine-learning-for-automation-testing-17og>

我们在这里通过使用机器学习实现自动化测试的目标是，通过数据挖掘他们的日志和他们在要编写测试的应用程序/服务上的行为，基于用户交互动态编写新的测试用例，进行实时验证，以便在对象被修改或删除或其他一些变化(如“拼写修改”)的情况下，如大多数 IDE 以智能感知的形式(如 Visual Studio 或 Eclipse)所做的那样。

“测试自动化”中的机器学习可以帮助防止以下一些但不限于此的情况:

1.  节省了[编写测试用例](https://www.lambdatest.com/blog/17-lessons-i-learned-for-writing-effective-test-cases/)的手工劳动
2.  测试用例是脆弱的，所以当出错时，一个框架很可能要么放弃测试，要么跳过一些可能导致错误/失败结果的步骤。
3.  除非测试运行，否则测试不会被验证。因此，如果编写一个脚本来检查“OK”按钮，那么在我们运行测试之前，我们不会知道它的存在。

机器应通过应用模糊匹配来帮助从运行中的测试中恢复，这意味着如果一个对象被修改或删除，那么程序和脚本必须能够找到与它正在寻找的对象最接近的对象，然后继续测试。例如，如果一个 web 服务最初有“小型、中型、大型”选项，并且脚本是根据这些选项编写的，如果添加了另一个选项，即“超大”，那么脚本必须能够适应这种情况并预见到这种变化，以便测试运行可以继续运行而不会失败。

[了解更多关于跨浏览器测试的信息](https://www.lambdatest.com/)。

惠普统一功能测试是市场上用于自动化测试的知名工具之一。它有一个 GUI 界面，除此之外还有其他工具，如 selenium libraries(用几种语言实现，如 Java、C++、C#、Python 等)。)和黄瓜。像这样的工具允许人们编写自己的脚本，并让脚本通过运行几个案例来完成测试工作。

在开始测试之前，系统需要学习案例。我们需要给它一些东西作为开始。在触发培训部分之前，我们必须设置系统，在定制的时间窗口内向普通用户显示一些广告，并在此期间收集和记录日志，这有助于我们生成观看特定广告的人的性别比例和年龄组。目标是找出哪个年龄组和性别的用户在观看了呈现给他们的特定广告后被刺激从网站购买某些东西。结果被保存为训练数据，以便可以对它们进行测试。

机器经过训练，可以根据收集到的信息编写测试用例。训练数据每隔一段时间更新一次，以便可以根据不同的人口统计数据对最新数据进行测试，并向潜在客户投放相关广告。如果这是手动完成的，那么每次趋势改变或网站改变时，人们可能不得不修改脚本或手动添加测试用例。

### ML 算法的自动化测试

SVM:它属于 ML 算法的“线性分类器”家族，试图找到一个(线性)超平面，将不同类别的样本分开。在学习阶段，SVM 将训练数据视为 k-1 维的向量。目标是找到最大余量(距离)。SVM 技术主要用于二元分类。除此之外，我们还有 MartiRank，一个排名算法，处于学习阶段。它需要若干轮，在每轮/迭代期间，数据被分解成 N 个子列表，每个子列表包含设备/应用故障总数的 1/n。

在回归测试的情况下，需要开发一套测试用例，MartiRank 的开发是一个持续的过程，并已用于检测新的 bug。例如:开发人员可能已经重构了应用程序中的一些代码，并放入了一个新的函数。回归测试向我们展示了产生的模型与之前的不同。

当我们编写测试用例时，我们测试软件在理论上应该如何运行，我们没有真实的数据，一些测试用例可能永远不会在现实生活中使用，而一些错过的测试用例可能是最重要的，这就是为什么数据挖掘日志并让机器根据这些日志自动编写测试用例节省了大量的工时并有助于实际测试。像 HockeyApp & TestFlight 这样的服务正在提供自动化的移动应用测试服务。

至于 GUI 测试，有一些研究论文谈到了测试自动化的深度学习和强化学习。正在测试的系统首先进行数据挖掘，以获得 GUI 界面上有意义的点击、文本和按钮按压，这产生了大量的训练数据。这些数据随后被用来对软件进行几个小时的测试。最好的部分是，不需要编写模型或测试案例，随着时间的推移，错误会被发现，但一些案例没有被测试，这可能是因为缺乏训练数据。强化方法改进了测试，因为它们通过多次迭代运行。

英特尔和英伟达一直在大力投资硬件解决方案，这些解决方案可以帮助深度学习和相关算法更快地取得成果。从移动第一世界到人工智能第一世界。我们知道，对于测试某个产品，无论是一个小计算器，永远不会有足够数量的正确测试用例，这就是为什么鼓励开发人员和测试人员编写越来越多的测试用例，以使他们的产品更加稳定。保罗·格拉厄姆曾经建议使用贝叶斯过滤器来过滤垃圾邮件，成千上万封电子邮件被输入系统，然后对训练数据进行学习和测试，以确保过滤器不会出错。网络爬虫在不同的网站间移动，寻找 404 或其他错误，实时更新它们的索引和测试用例。
