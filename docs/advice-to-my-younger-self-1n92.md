# 给年轻自己的建议

> 原文：<https://dev.to/arvidquarshie/advice-to-my-younger-self-1n92>

你好世界，

在过去的几个月里，我一直试图找出我能给“年轻的程序员”什么重要的建议。所以我选择强调一些我认为在开始这次旅程或冒险时非常重要的事情。通过分享这篇文章，我希望能帮助其他人在他们的软件开发生涯中导航(最好的情况:)

## 与他人友好相处

作为一名软件工程师，有趣的事情是，我觉得我们都是在操场上玩耍的孩子。但是为了让我们玩得开心，我们需要知道如何互相“玩”。这就是通常所说的成为**团队成员**。我认为一个软件工程师或任何人都应该具备的最重要的技能之一是与他人良好相处的能力。这听起来很容易也很简单，但是要和别人“玩得好”需要付出很多努力，但是这将最终决定你将来能有多成功。所以花一点时间阅读关于不同性格类型的书籍，以更好地了解人们，同时阅读关于人的书籍，这会给你真正有用的见解。
*亲提示:检查你的自我；)*

## 自我价值

[![bald-blur-boy-236302.jpg](img/de4e9e70b7e860a9a068c5606305f841.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--GOrwpnPV--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.dropbox.com/s/5luvfjptukn5cdt/bald-blur-boy-236302.jpg%3Fdl%3D0%26raw%3D1) 
另一个宽泛的话题但是总结一下，你需要有一种价值感(以健康的方式)。这不同于拥有自我，自我价值只是简单地**知道你是谁，你带来什么价值，以及你在未来哪里看到自己**。当你在软件工程的世界中成长时，拥有一个健康的自我形象有助于你和他人。这也有助于你处理可能影响你的有毒情况。这并不像听起来那么容易，所以在身体上、精神上和情感上努力对保持健康的平衡很有帮助。

## 导师导师导师！！

[![athlete-athletic-baseball-264337.jpg](img/03b30279fec4c0a40921d250d959f212.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--eJvW35eN--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.dropbox.com/s/3zyioh425b8i4ne/athlete-athletic-baseball-264337.jpg%3Fdl%3D0%26raw%3D1) 
是的，我不得不说三遍。你需要导师，最好是来自你希望从事的领域，甚至是生活中的其他领域。只要你能从中学习，这并不重要。
**免责声明:不是每个人都想指导你，这完全没问题；你可以有不止一个导师，只要你能够跟踪你的目标和优先事项，这是完全没问题的。有一个导师是让你在过程中保持克制和负责的好方法。想想看，大多数超级英雄都有某种形式的导师。(*蝙蝠侠、超人、蜘蛛侠、潜水侠、钢铁侠甚至夏洛克·福尔摩斯)。得到它；你可以在网上查找如何找到导师，然后选择一个适合你的策略。我还知道，并不是所有的导师都是完美的，他们也是会犯错甚至伤害你的人。(当然不是故意的)。这就是为什么有一个健康的自我价值感会让我们很容易处理这样的事情，继续前进而不会感到失望。这是生活的一部分，你越早做好准备，你就会越好。***

## 不总是关于代码

[![blurred-background-cellphone-coffee-842554.jpg](img/651cbfbf4bf88e1718080c5d71a3f2f9.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--HGtClunv--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.dropbox.com/s/hv2u9ktnewln5dw/blurred-background-cellphone-coffee-842554.jpg%3Fdl%3D0%26raw%3D1) 
嗯，我认为所有的开发人员都为他们各自的堆栈感到骄傲和依恋(当然是以一种好的方式:)
首先，重要的是代码背后的人。正如你知道如何在 x 数量的算法中排序，加上给出它们的复杂度符号，这是**非常重要**。你应该能够用一种让人们听起来有趣的方式来表达自己。就像*“你好，我的名字是 John Doe，我喜欢狗，这让我在我通常使用的普通 java/kotlin 的基础上，使用 flutter(我正在研究的一个新框架)创建了一个连接狗的应用程序”*，你看这让我对你感到好奇，显然不能否认你知道你的东西。
**免责声明确保你确实熟悉你的堆栈/领域，并把你的热情作为你为什么编码的支点:)**
同样，在解决问题时，首先要考虑可能的情况，而不是直接进入编码会话。当然，在选择特定的方法之前，您可以尝试各种可能的解决方案。相信我，这可以节省你最后一分钟进行调整和修复的时间，或者让你的拉取请求被拒绝几次:(

## 版本控制

[![ai-artificial-intelligence-blur-546819.jpg](img/3c38b1ba3775c7d3ec405228cff0ebba.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--0Suxn730--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.dropbox.com/s/s796i5abg4zinfp/ai-artificial-intelligence-blur-546819.jpg%3Fdl%3D0%26raw%3D1) 
关于这个没什么好说的，其实我有很多话要说，但简单来说就是学习 git 以及为什么版本控制在软件开发中很重要。相信我，你越早知道如何使用它，在团队中工作就越容易。还记得**吗“和别人友好相处..这是其中之一，学习 git 时期；)"**
把它想象成编码的**瑞士军刀(专利申请中；) )，*看似无关紧要，但当你能够有效利用它的时候，你会惊讶于你获得了多大的力量*。**

## 敏捷

大多数团队在敏捷环境下工作，所以了解一些框架是很重要的，比如 scrum、看板。超过 40%的团队在他们的项目中使用敏捷的衍生产品。所以知道基本原理也无妨。过会儿谢谢我:)

## 信任流程

[![active-adult-bicycle-1005803.jpg](img/f77b5f99c2a3dd4b37f3768748228cb8.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--rH5C9US3--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.dropbox.com/s/jlgktpxkvrwkc1w/active-adult-bicycle-1005803.jpg%3Fdl%3D0%26raw%3D1) 
这有点宽泛，但基本上可以归结为从容不迫。不要对自己太苛刻，尽管你很想成长和提高，但这需要时间，事情就是这样。没有捷径，你需要投入时间和努力，最终你会开始进步。有了频繁的里程碑和检查点，你就能够评估你的进展与你想要达到的目标之间的关系，所以你需要制定一个**计划**
基本上就像骑自行车一样，你会摔倒几次，但最终你会享受这一过程。

## 玩得开心

[![audience-band-concert-1105666.jpg](img/66abda415da4c04ad6ae8355f5b7d10d.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--AOZbT6f9--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.dropbox.com/s/n6y1v2gzksr27kp/audience-band-concert-1105666.jpg%3Fdl%3D0%26raw%3D1) 
我花了一段时间才做到这一点，但是的，开心是应该的，这是你应得的(也许)。在我职业生涯的早期，我认为长时间和超负荷工作在软件工程领域是很正常的事情。老实说，是的，有几个例外可能需要很长时间才能解决，但我意识到的是，通过适当的规划和准备，我们可以最大限度地减少这种情况。(作为一名软件工程师，研究如何避免精疲力尽)
所以请走出去参加社交活动，作为一个内向的人，我知道社交活动会耗尽你的精力，但这不是长时间呆在屏幕前的借口。你需要设计一个健康平衡的生活，所以不要打着软件工程师的幌子躲起来。
***与人交谈你会惊讶地发现，他们也会回嘴。；)***
