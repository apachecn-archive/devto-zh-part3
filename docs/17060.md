## 让我们来谈谈试剂吧

React 是一个开源 JavaScript 库，旨在创建用户界面。

如今，许多大型网站都加入了 React，因为它使开发人员能够在网站、平台、页面等内部以分段方式创建组件。

此外，它还简化了向导的创建过程。通过 reactor，您可以创建简单的视图，以便在数据更改时对其进行渲染和更新。

好的，这个开发者创造界面，然后呢？

那就是不和的地方。

## O 设计操作

还记得卓别林的故事吗？-是啊。

长期以来，设计者和开发者的角色都很明确。各就各位。每个人都有自己的

一个人开发，另一个人创造设计。不同的功能，往往是相互冲突的想法。

但是，再一次思考时代和事物的演变，负责视觉活动的人意识到，重视发展过程的方法是很重要的。该项目虽然是由各方创建的，但需要保持一致。这些部分最终将成为一个整体。

对于 18 世纪的行业来说，职责分工加快了流程，带来了活动类型所需的统一性。但说到创造，这种观念可能会成为一个大问题。

因此，不仅最终结果必须是相互关联的，而且建造它的人也必须是相互关联的。

于是，一个已经存在的职业得到了加强，并获得了一个名字:DesignOps。

此专业人员配置文件负责组织和设计团队内的操作。或者，团队可以担当此角色，帮助构建企业中的组织流程。

对于 devs，此 DesignOps 与开发团队一起，有助于创建和开发项目。
