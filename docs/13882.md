# 你可以打破自己的建议，也可以改变主意。

> 原文：<https://dev.to/joshhadik/youre-not-a-hypocrite-if-you-dont-always-stick-to-your-own-advice-1m5n>

在像这样的社区(或任何社区)，人们都有一种共同的恐惧，那就是如果他们说了什么，他们就必须去实践。

如果他们想尽办法给出某种技术建议，他们需要确保他们在所有公开回购的所有代码中 100%地坚持这一点。

如果他们不这样做，他们会被贴上伪君子的标签。

这是一个谎言。

你不需要 100%遵循自己的建议，如果你不这样做，也不应该害怕分享你的观点。

我过去抽烟，(而且我可能仍然沉迷于偶尔的香烟。)

这并不意味着如果有人直截了当地问我是否应该吸烟，我必须说是。我的回答是不，他们当然不应该！这是一个坏习惯，浪费你的时间，耗尽你的银行存款，让你闻起来像垃圾，并破坏你的健康。

我可以这么说，尽管我有时仍然吸烟。

给予技术建议也是如此。即使你不总是言行一致，说出你的想法也是可以的。

没有人知道。

看在上帝的份上，这是编程，大多数人 90%的时间都不会遵循自己的建议。

永远不要觉得你不能分享你的意见，如果你不总是坚持它，永远不要觉得有义务总是遵循你的建议，只是因为你分享了它！

哦，还有一件事...

错了也没关系。

改变主意也没关系。

许多过去鼓吹干燥的人现在鼓吹潮湿。

许多曾经是 100%测试覆盖率狂热者的人现在说，试图测试所有那些千载难逢的边缘案例是浪费宝贵的时间。

人们学习。人会变的。人会成长。

我讨厌想到你们中的一些人有独特的、有价值的想法，害怕分享它们，只是因为他们不总是坚持自己的想法，或者因为他们不是这个主题的绝对专家。

如果这听起来像你，我在这里告诉你，这是确定的！像这样的社区依靠不同经验水平的人聚在一起分享他们的经验和观点而蓬勃发展。

所以分享你的想法，如果你改变主意，没有人会评判你！
