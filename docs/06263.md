#### CSS&SCS

##### [](#art-direction-for-the-web-using-css-shapes)[使用 CSS 形状的网页艺术指导](https://www.smashingmagazine.com/2019/04/art-direction-for-the-web-using-css-shapes/)

通过在 CSS 形状周围环绕文本实现的一些设计可能性的迷人外观。Safari、Firefox 和 Chrome 现在都支持 Shapes 随着 Edge 转向基于 Chrome 的方法，我们很快就可以在任何地方使用它。这使用一些非常简单的属性打开了一组非常强大的布局可能性。

##### [](#follow-these-steps-to-become-a-css-superstar)[遵循以下步骤成为 CSS 巨星](https://medium.freecodecamp.org/follow-these-steps-to-become-a-css-superstar-837cd6cb9b1a)

这个标题有点夸张，但是对于那些 CSS 新手来说，这实际上是一个非常扎实的学习序列。他们本可以在链接资源方面做得更好，但即使只是知道先搜索什么也是非常好的。(顺便说一句，如果你确实想要一套很棒的资源，看看我去年整理的[CSS 学习终极指南](https://dev.to/kball/the-ultimate-guide-to-learning-css-9le-temp-slug-359113)。)

##### [](#css-masonry-with-flexbox-nthchild-and-order)[CSS 用 flexbox 砌筑，:n-child()，以及顺序](https://tobiasahlin.com/blog/masonry-with-css/)

非常有趣的砖石布局的纯 css 方法。强调了 flexbox 在创建适应其内部内容的布局方面的一些功能。

##### [](#underengineered-toggles)[设计中的触发器](http://adrianroselli.com/2019/03/under-engineered-toggles.html)

这是一个鼓舞人心的例子，展示了如何从一个简单的复选框开始，使用渐进式增强在视觉上构建一个完整的切换组件…这也是我尽可能使用 UI 工具包/组件库的一个非常有说服力的原因。做好一个组件需要做很多事情。本文将带您一步一步地思考和实现。

##### [](#58-bytes-of-css-to-look-great-nearly-everywhere)[58 字节的 css 看起来棒极了几乎无处不在](https://jrl.ninja/etc/1/)

人们很容易被 CSS 布局的可能性淹没，但是这篇文章强调了获得一个跨设备看起来不错的基本布局是多么简单。

#### [](#javascript)JavaScript

##### [Vue 中命名转场的力量](https://css-tricks.com/the-power-of-named-transitions-in-vue/)

这是一篇很棒的文章，展示了使用 Vue.js 进行 CSS 过渡是多么容易。爱死了！

##### [](#the-pitfalls-of-asyncawait-in-array-loops)[数组循环中 Async/Await 的陷阱](https://medium.com/dailyjs/the-pitfalls-of-async-await-in-array-loops-cf9cf713bfeb)

异步是 JavaScript 中最难理解的概念之一。这篇文章短小精悍，强调了一个常见的错误及其解决方案。

##### [](#state-management-with-react-hooks%E2%80%8A%E2%80%8Ano-redux-or-context-api)[带 React 挂钩的状态管理—无冗余或上下文 API](https://medium.com/javascript-in-plain-english/state-management-with-react-hooks-no-redux-or-context-api-8b3035ceecf8)

构建定制钩子的另一个很好的例子——这个例子创建了一个共享的状态管理解决方案，其中可以有一个单独的状态和一系列相关的侦听器。这个解决方案如此简单却又如此强大，它真正凸显了钩子作为一种范式的优势。

##### [](#what-are-controlled-components-in-react)[有哪些受控成分在起反应？](https://www.robinwieruch.de/react-controlled-components/)

一个常用描述的好的基本解释者。受控组件经常被提及，但很少被解释。这个例子是在 React 中(我认为这是这种描述最常用的地方),但是这个概念是跨框架相关的，

##### 凯尔·辛普森:我忘记的 JavaScript 比大多数人学过的都多

标题有点…挑衅…但这是对《你不知道的 JS》及相关书籍作者的实实在在的采访。有趣的是，我们看到了一个最受欢迎的谈论 JavaScript 的作者是如何看待这种语言的。

#### [](#other-awesomeness)其他牛气

##### [](#using-the-web-speech-api-for-multilingual-translations)[使用网络语音 API 进行多语言翻译](https://css-tricks.com/using-the-web-speech-api-for-multilingual-translations/)

好吧，这完全让我大吃一惊。这就像在浏览器中内置了一个多语言的屏幕阅读器，但是对于你想给它的任何文本。我的意思是，是的，谷歌已经能够做这种类型的东西有一段时间了，有音频版本的翻译，但这意味着它可以为每个网络开发人员所用！太神奇了。

##### [](#frontend-developer-handbook-2019)[前端开发者手册 2019](https://frontendmasters.com/books/front-end-handbook/2019)

FrontendMasters 每年发布的大量前端学习资源。它几乎是百科全书，有大量高质量内容的链接。这是一个需要反复阅读的资源，而不是一次性阅读。

##### [](#netlify-dev-the-power-of-netlify-on-your-local-computer)[网络生活开发——网络生活在你本地计算机上的力量](https://scotch.io/tutorials/netlify-dev-the-power-of-netlify-on-your-local-computer)

Netlify 因其强大的 JAMStack 支持和在世界范围内快速部署静态站点和 JAMstack 应用程序的能力而备受关注。他们有很多工具来帮助解决这个问题，比如无服务器功能等等，但是使用这些工具的开发经验具有挑战性。现在，他们已经发布了一个本地开发环境，让你可以在你的台式机或笔记本电脑上测试和复制你的无服务器功能&整个环境。

##### [](#top-5-questions-asked-in-accessibility-trainings)[无障碍培训中问得最多的 5 个问题](https://www.deque.com/blog/top-5-questions-asked-in-accessibility-trainings/)

一组最常见的可访问性问题(我知道我曾经问过这些问题中的大部分)和深入的回答。
