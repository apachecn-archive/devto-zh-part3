# 拉动式请求清单

> 原文：<https://dev.to/idoshamun/the-pull-request-checklist-2opk>

我有一个想法，为我们的开发人员制作一个清单，这样我们就可以保持我们的工程标准高。目前这份清单很简单，我正在寻找更多可以添加的想法。

*   每个第三方呼叫都是:
    *   由断路器完成
    *   受时间限制
    *   正确处理错误
*   每个承诺/回调/异步都正确处理错误
*   每个边缘案例都被测试所覆盖
*   没有关于数据完整性/一致性的假设
*   考虑了潜在的性能缺陷
*   提交消息遵循约定

当然，我们的开发方法要广泛得多，但是在这个列表中，我只想保留最重要的东西。
