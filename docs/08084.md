# 教孩子们编程

> 原文:[https://dev.to/walkingriver/teaching-kids-to-code-3gb4](https://dev.to/walkingriver/teaching-kids-to-code-3gb4)

*交叉发布自 [WalkingRiver 博客](https://walkingriver.com/teaching-kids-to-code/)T3】*

当我的大儿子 11 岁时，我雇他和我一起做一个 ASP.NET 的项目。这不是我计划要做的事情，但我有一些单调的布局工作，他渴望帮助。事实证明，他有相当的诀窍。15 岁时，他得到了第一份“真正的”工作，为我的一个朋友维护一个现有的 ASP.NET 网站。如今，他正在完成软件工程学位，同时在一家金融服务公司做带薪实习，编写应用程序。在本文中，我将尝试记录这一经历，并希望找出一些可重用的模式来教授他人。

## [](#background)背景

像过去 20 年左右出生的大多数孩子一样，我的儿子一直对电脑表现出极大的兴趣。早在 4 岁的时候，在假期拜访家人时，他经常会被要求为他们安装软件。他最先认识的一些单词是`Start`、`OK`和`Cancel`。

## [](#from-web-controls-to-sql-queries)从 Web 控件到 SQL 查询

2006 年春天，我和一个朋友被转包给新罕布什尔州康科德的一家小公司，为其创建一个网站。这将是一个中等规模的网站，有一个大的 SQL Server 后端。该项目的承包商是一名 SQL Server 专家，没有前端编码经验。

经过一番考虑，我们决定用 ASP.NET 网络表单编写它，使用微软的新 Ajax 控件工具箱。我们认为我们三个人可以在大约六个月内完成整个项目，谈好价格，然后开始。

初夏的某个时候，当 UI 接近完成时，客户问我们是否愿意用一个[日历控件](http://www.ajaxtoolkit.net/Calendar/Calendar.aspx)来增强整个应用程序中每个与日期相关的文本框。这将提供现在常见的自动显示日历来选择日期的能力，而不是让用户手动键入日期。

添加 calendar 控件的过程很简单:将控件图标从可视化工具箱拖到 web 窗体设计图面上，然后将文本框的控件 ID 添加到 calendar 控件中，将它们链接在一起。这并不难，但会很乏味。有几十个这样的字段分散在应用程序的许多页面中。向客户收取几个小时拖放工作的费用似乎很愚蠢。

我一时兴起，问儿子是否有兴趣赚几美元。他问要多少钱，要做什么。我给了他 10 美元/小时，并计算出我给客户开的账单总共不超过一个小时，不管他花了多长时间。他同意了，于是我向他展示了如何将日历控件添加到文本框中，并解释了哪些文本框需要它。他走了。

几个小时后，他宣布自己完成了，并开始向我展示他的作品。它完美无缺。他接着问，“好吧，接下来呢？”

整个夏天，我给他越来越多的工作做，主要是清理 UI 布局，确保页面之间的风格一致，以及其他类似的小事。

一天，他问他是否可以尝试从头开始创建一个页面。我有一个次要的页面，我们只有在有时间的情况下才会做，所以我让他去做。它有一个数据网格控件，简单地反映了数据库表中的值。

控件完成了大部分工作。它只需要知道数据库的名称、要显示的表以及每个表记录中的值。他用其他页面作为例子，包括另一个已经有数据网格控件的页面。

当他在研究它的时候，他想知道更多关于它在幕后是如何工作的。所以我向他展示了网格是如何连接到数据控件的。然后介绍数据控件如何连接到数据库和表、它的列，以及如何指定搜索和排序选项。这变成了对基本 SQL 语法的讨论。没有比`SELECT * FROM Customers`类型的东西更有趣的了。他似乎明白了，然后继续工作。

完成后，他来到我面前，向我宣布他已经改进了它。他问我是否知道也可以在数据网格控件中添加内嵌编辑和删除功能。他解释道，你所要做的就是提供适当的 SQL `UPDATE`和`DELETE`语句。他非常兴奋，向我展示了一个完全可用的可编辑数据网格。

他对基本概念和主动性的理解给我留下了深刻的印象。然而，它确实引发了对“特性蔓延”的危险的讨论

回到 2006 年，当大书店还是个东西的时候，我曾经花很多时间在新罕布什尔州纳舒厄的巴诺书店里，浏览他们的电脑书籍。有一天我偶然发现了 [LINQ 的袖珍参考文献](https://amzn.to/2I5gDIp)。我听说过 LINQ，喜欢奥莱利袖珍参考资料的风格和简洁。我买了一本带回家。

正如我经常做的那样，我把这本书忘了几天。它放在我桌子的一边，没动过。然后有一天，我需要查找一些东西，但这本书已经不在那里了。事实上，我的几本书不见了:LINQ 的书和另一本 C#的书。我找遍了房子里通常我可能会放书的地方，但没找到。

当我儿子放学回家时，我问他是否见过他们。他两个都有。他看到它们放在我的桌子上，觉得它们看起来很有趣，于是决定读一读。“你有吗？”我问。“是的，”他回答道。“你理解他们吗？”他声称他知道，所以我让他给我解释一个 lambda 表达式。

## [](#whats-a-lambda-expression)什么是 Lambda 表达式？

当时我和 LINQ 之间的一个矛盾是 lambda 表达式的概念。出于某种原因，我花了很长时间将传递一个函数到另一个函数的想法内化。Lambda 表达式通过将未命名的整个函数作为参数传递，使这变得复杂。称它们为“lambda 表达式”并没有帮助。

所以我让他给我解释一下。假设您有一个客户列表，并且您想将该列表过滤为名字为“Mike”的客户，并按姓氏排序。

他想了一会儿，想到了这个

```
var results = Customers
  .Where(x=>x.FirstName=="Mike")
  .OrderBy(x=>x.LastName); 
```

## [](#a-credit-card-and-an-eagle-scout)一张信用卡和一个老鹰童子军

在接下来的几年里，只要我有我认为他能处理的工作，我就继续支付我儿子的时间。随着他的经验和理解的增长，这项工作变得越来越复杂。

他 15 岁时，我的一位前同事，一家小电信公司的首席执行官，在他的公司给了我一份全职工作。他需要有人来维护和改进我在过去两年中为他构建的应用程序。

碰巧的是，我刚刚接受了另一家公司的全职职位。失望之余，他问我是否认识其他人可以处理一些工作。犹豫了一下，我问他是否会考虑给我儿子一个尝试。我向他介绍了这篇文章中概述的大部分背景，并向他保证，如果事情没有进展，他不会感到难过。

他知道我的儿子在他的高中是优等生，并且正在积极争取他的童子军鹰级军衔。作为一名前童子军领袖，他知道成为一名鹰级童子军需要付出多少努力和奉献。他同意试一试，并提出以每小时 20 美元的价格让他维护应用程序 UI，这是一个与 SQL Server 数据库相关的 ASP.NET MVC 2 应用程序。

他的工作得到了首席执行官的称赞，他继续向任何愿意倾听的人讲述这个故事。在几年后的一次贸易展览会上，有人问他开发这个应用程序需要什么。首席执行官笑着说:“我们用信用卡和老鹰童子军兼职建造的！”

## [](#software-lead-and-devops-engineer)软件主管兼开发工程师

在接下来的几年里，整个高中期间，我儿子都在继续开发 web 应用程序，并扩展到公司的其他应用程序。最终，他成为了公司的首席开发人员。

他负责创建一个大规模的自动化测试套件，一个多头詹金斯 DevOps 系统；更新应用程序的数据层；以及实现在 NodeJS 和。网芯。

他在多个软件开发领域变得非常熟练，包括移动开发。他在苹果、谷歌和亚马逊的应用商店都有应用。最近，他在谷歌的 Firebase 上部署了一个渐进式网络应用。

## [](#software-engineering-major)软件工程专业

今天我的儿子是软件工程专业的学生，正在攻读学士学位。他 10 多年的付费软件开发经验帮助他在一家金融服务公司找到了一份带薪实习工作。当他一年左右离开大学时，他将没有学生贷款，并有足够的积蓄开始他的成年生活。我肯定他找一份全职工作不会有问题。这一切都是从一个视觉设计工具、一些业余时间和一点好奇心开始的。

## [](#can-this-scale)这能秤吗？

我写这篇文章的目的是试图确定我所做的具体事情，这些事情可能会在未来被其他孩子复制。

我试着对我的另外两个孩子重复这个过程。到目前为止，他们对追随哥哥的脚步没有表现出任何兴趣。那很好，因为我不想催他们。

在我的影响范围内，其他想要学习的年轻人呢？更重要的是，有没有什么别人或许可以套用在青春上的东西？怎样才能帮助其他孩子，就像我对我儿子那样？一些对我来说很突出的事情，可能也适用于其他人:

*   认识到欲望。如果一个孩子对你所做的任何方面表示出兴趣，鼓励他。通过给他们提供一个小任务来帮助他们。看他们怎么处理。帮助他们度过难关。
*   早点开始。如果你当时问我，我会说 11 岁可能太年轻了。显然，我错了。
*   用书。是的，他们仍然存在。主动提出借给他们他们感兴趣的任何主题的书。
*   给他们一个机会。当你有一个真正的项目要做时，学习新的东西会更容易。如果我的首席执行官同事没有给我儿子一个机会，他的兴趣可能在他还在上高中的时候就已经减弱和消失了。
*   即使他们失败了，也要继续鼓励。我儿子在最初的几个月里犯了很多错。他学习，解决问题，提高。

你还能想到什么？在软件开发或任何其他类型的努力中，你做了哪些事情来帮助教育和鼓励年轻人？我很乐意在 Twitter 上听到你自己的故事。我是 [@walkingriver](https://dev.to/walkingriver) 。
