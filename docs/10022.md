# 冒名顶替综合症是真的

> 原文：<https://dev.to/elimerrell/imposter-syndrome-is-real-dol>

我是熨斗学校软件工程专业的学生，我的同学最近已经完成了课程的一半。令人难以置信的是，在过去的七周里，我学到了很多东西。我有机会学习 Ruby 和 JavaScript 两种语言，并通过与我出色的同学一起构建项目来应用我的新技能。你可以说，我在刚刚过去的这个周末正昂首阔步；一切似乎都很顺利，我对自己的新能力非常有信心。我决定报名参加在西雅图举办的“技术为善”黑客马拉松，因为我认为这将是一个极好的机会，可以结识我当地开发社区的其他成员，并有可能为一个令人满意的项目做出贡献。接下来发生的事情是，冒名顶替综合症像一列货运列车一样袭击了我。

周六早上，我一大早就到了，渴望尝试一下“黑客马拉松”的事情。大约有八个不同的小组在上午展示了他们的项目，并讨论了与会者如何贡献他们的才华。我对一个人行道地图项目很感兴趣，该项目试图使用开放街道地图来绘制城市周围的人行道，以更好地记录残疾人的步行能力。我自愿与这个小组一起工作，几个非常有才华的开发人员也加入了进来。在讨论这个项目时，我决定深入研究 PostGreSQL 数据库表，尝试头脑风暴如何改变模式以适应项目的新目标。我们使用的代码库是用 Python 编写的，并且使用 Flask 作为 web 框架。我对所有这些技术都没有什么经验，我很快就被弄糊涂了。

当恐惧和焦虑开始建立时，我开始搜索代码库，试图找到数据库模式上的信息。由于接受过 Rails 培训，我习惯于依赖每次运行数据库迁移时生成的 schema.rb 文件。我努力在 Flask 目录中找到一个类似的文件，并开始像一个疯子一样搜索它。与此同时，一个在我旁边开店的队友正以我从未见过的速度飞快地通过代码。他在他的 Linux 机器上的几个终端窗口之间疯狂地切换，以使程序在他的本地环境中运行。我敬畏地看着他快速浏览终端命令和 Python 代码片段，同时快速排除任何出现的错误。

随着时间的推移，情况变得大同小异。我的团队中有人会问我是否曾经使用过技术“x”或者我是否知道如何做一些编程任务“y ”,我的回答总是一样的:没有。我惊讶于我的团队成员在我所有的“不”中表现出的耐心和友善。我从来没有因为不知道而受到批评，并且经常得到有益的解释或鼓励的话语。即使有所有鼓励的话语和有益的指导，还是很难不感到格格不入。意识到我要学的东西太多了，这让我既兴奋又害怕，压力也很大。

尽管我参加黑客马拉松时感觉自己已经牢牢掌握了一些编程概念，但现在我感觉自己一无所知。我觉得自己是一个彻头彻尾的骗子，冒牌货，骗局，不管你怎么称呼它。就好像我突然置身于一个超越我的浩瀚知识海洋中。我对自己不知道的东西的无知已经逐渐消失，我突然意识到我仍然处于我的编码之旅的开始。

既然我已经第一次真正尝到了冒名顶替综合症的滋味，我该何去何从？答案是我必须不断学习和进步。虽然我在熨斗学校的时间将在仅仅七周内结束，但我知道编码将是一个终身学习的努力，这令我兴奋。随着我不断学习新技能和新技术，我迫不及待地想看看我能创造出什么。冒名顶替综合症是不可避免的，所以我建议正视它，并把它作为一次学习经历。我祝每个人在你的编码之旅中好运！
