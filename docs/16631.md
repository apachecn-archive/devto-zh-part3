# 运行自己的开源项目

> 原文:[https://dev.to/meanin/running-own-opensource-project-2o8](https://dev.to/meanin/running-own-opensource-project-2o8)

我一直想开始一个开源项目。大约一年前，我和几个朋友试图写一个评价精酿啤酒的门户网站。有很多这样的应用程序，但这应该是学习新事物的一种方式。事实证明，除了我之外，没有人有时间专注于这个项目。在 3-5 周内，我们放弃了整个想法。我开始怀疑在业余时间这样工作是否有意义。

# [](#change)变化

换了工作后，我开始有更具挑战性的任务。他们中的一些人让我创建了一个 PoC 项目，我把它放在了我的 github 上。随着时间的推移，我回到了 dev.to 上的写作。在那段时间里，我创建了一个[项目](https://github.com/meanin/vsts-tasks)来支持 Azure DevOps 上的 CI/CD 流程，该门户网站上也有相关文档:)。然后我想到——让我们再试一次。所以我问自己，有没有什么东西能对我的日常工作有所帮助，而这正是我所缺少的？最近，我开始接受技术采访。我是通过填写一些 excel 文件来做这件事的。这是令人讨厌的，无效的等等。这就是我如何发现我的开源项目的想法。

# [](#technology)技术

我主要是后端。net 开发者。我对蔚蓝云有所了解。很明显，我的项目将基于这项技术。一个基于云的 web 应用程序，支持技术面试。好吧，我决定后端，但它也需要一些前端。我应该使用 React 吗？还是棱角分明？还是其他什么？啊啊啊。网络摇滚，对吧？最近发现了一篇关于. net SPA 框架的文章。我深入研究了一下。它目前是一个不受支持的实验性 web 框架，但是，嘿，它是 dotnet，这将是我想使用的东西。这个框架叫做 [Blazor](https://blazor.net/) 。正如医生所说:`Browser + Razor = Blazor`！

# [](#get-the-fellows-on-board)让伙伴们上船

我当时没有任何期待。我不想强迫任何人捐款。我独自开始这项工作，并采取了一些初步措施，让一个脚手架应用程序的工作。我在 GitHub 上创建了一个知识库，下载了前面提到的框架，开始学习。很少有朋友在 GitHub 上关注我。他们开始询问这个项目，因为他们也听说了一些关于这个框架的事情，并且想从头开始学习。

过了一段时间，我开始关注 GitHub 的新特性。其中之一是项目委员会。它有助于组织项目工作。我创建了一个项目来了解它是如何工作的。它看起来像一块看板。我想我们可以把它作为一个 scrum 板来使用。所以我创造了一个。用 GitHub 问题来填充它，描述我想通过这个项目实现的目标。

当 tasks ( `issues`)到达时，我问我的朋友们是否愿意帮忙。目前，我们已经非常接近我们应用的 1.0 版本了。他们乐于参与。他们完成了一些任务，我也完成了。这确实是朝着正确的方向发展。
