# 使用 Aurelia 的 4 个理由

> 原文:[https://dev.to/bigopon/4-reasons-to-use-aurelia-5e0p](https://dev.to/bigopon/4-reasons-to-use-aurelia-5e0p)

我是一个试过 AngularJS，Angular，React，Vue，Aurelia 的开发者。到目前为止，Aurelia 是最强大、最优雅、最直观、最令人愉快的开发框架。在这里，我将与你分享一些我认为你应该帮自己一个忙并尝试一下的理由。

# [](#developer-friendliness)💭开发者友好性

从 Aurelia 开始，就像在应用程序中添加一个脚本标签一样简单。添加新组件只是一个导出的 JavaScript 类、一个 HTML 文件和一个 html require 元素:`<require from="./path/to/element"></require>`。增加一条新的路线是完全一样的。奥雷利亚没有 vDOM。元素、属性之间的绑定，无论是单向的还是双向的，都是一个简单的命令就可以完成的。它可以直接、直观地处理您所熟悉和喜爱的 DOM 对象。由于没有 vDOM，给你的应用程序添加一个第三方插件是一个非常简单的任务。

# [](#power)⚡的力量

Aurelia 不仅为您的应用提供了令人惊叹的性能，使其他流行的框架及其 vDOM 望尘莫及，而且还提供了一个简单而自然的数据绑定心理模型，将为您提供军队的力量。您将享受成为突破业务需求、完成工作并发展公司的人，同时利用标准 JavaScript 和 JavaScript 对象释放您的新能力，而不是将时间浪费在框架专有特性上。无论你做什么，它都会起作用。此外，Aurelia 生态系统还配备了对话、验证、国际化、状态管理和 http 等关键工具。

# [](#intuitive-and-simple-dependency-injection)👌直观简单的依赖注入

曾经想要学习依赖注入(DI ),但是因为只有很少的地方可以使用它，而且似乎没有一个地方可以提供简单的东西来帮助你获得“点击”时刻，所以感觉被推迟了？Aurelia DI 就不一样了，它是 JavaScript 类上的一个语句`static inject ...`,你已经声明了一个自定义元素或者自定义属性的依赖项，它们会在构造函数中直接给你。不再有古怪或繁琐的声明或用法。这只会帮助您驯服应用程序，不管它增长多少。

# [](#self-progression)📚自我进步

在学习 Aurelia 的同时，你将学会如何回归 JS，回归自我，并进步到你学习中的下一件重要的事情:你的事业。您将会了解到，拥有一个不为您想做的事情操心的框架，甚至是双向数据绑定，是多么美好的事情。一切都如你所料地联系在一起。经历这些会给你一个“只是工作”的心理模型，并帮助你建立下一个每个人都可能依赖的令人敬畏的东西。
