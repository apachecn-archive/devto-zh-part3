我很清楚现在是三月，离假期还远着呢，明天我们会在四旬斋前的四旬斋前一天结束我们的饕餮之旅。或者明天对你来说只是一个普通的星期二。恭喜你熬过了周一。

无论你在 2019 年的什么地方，我都想以我有史以来最喜欢的一些回购的形式给你一份小礼物。把它当成“ [lagniappe](https://en.wikipedia.org/wiki/Lagniappe) ”吧，这是一点额外的东西，因为奥普拉在下半年对最喜欢的东西有着非常坚定的把握。🎁

#### [T1。gitignore 模板](#gitignore-templates)

我不知道你的情况，但是我每周，有时每天都在不同的语言之间转换。Unity 项目创造了一大堆你不一定想与世界分享的东西，所以我强烈推荐看一看这些。

github/gitignore

专业提示——你在 Unity 编辑器中写的任何东西都会作为参数出现在场景中。所以如果你和我一样傻，在 2018 年初把一堆凭据放在里面，下一节就是给你的！

#### [](#github-flavored-markdown-cheatsheet)GitHub 风味的降价备忘单

说到 Git/GitHub，我是那种真的永远不会记住我所学的东西的人。这包括我每天做的事情，一天多次。

我分不清左右，当我伸出手的时候，我会过度思考“L”的方向，你怎么会认为我会记得如何在 markdown 中链接某个东西？括号前的括号还是括号前的括号？

[adam-p/markdown-here](https://github.com/adam-p/markdown-here/wiki/Markdown-Cheatsheet)

还有其他备忘单类型的东西吗？绝对的。但是我已经参考这个项目的维基很多年了。

#### [](#scrub-your-repo-for-secrets)为秘密擦洗你的回购

由于不理解 Unity 如何处理场景，我在 GitHub 和 GitHub Enterprise 上有大约 6 个需要大量清理的回购。这些不是无关紧要的“Hello World”类型的转发，这些转发是你可以去喝咖啡，回到你的办公桌，也许他们已经完成了克隆或提交或任何你要求他们做的事情。

[rtyley/bfg-repo-cleaner](https://github.com/rtyley/bfg-repo-cleaner)

我只使用它从非常特定的文件中删除凭证值，但它工作得很好，我没有看到任何不想要的副作用。

不要忘记删除任何秘密、令牌、密码等。你可能打扫过了。

#### [](#automated-accessibility-testing)自动化可访问性测试

回到我编写内部企业 web 应用程序的时候，我参加了一个简短的演示，演示者是一个设计 web 应用程序并使用屏幕阅读器测试它们的人。听到对使用屏幕阅读器的人来说这种经历是多么痛苦是令人震惊的可怕。

虽然我并不认为自己是一个明确的可访问性倡导者或拥护者，但我确实读到过自动化的可访问性测试工具可以帮助你从几乎不可访问到基本可访问。像 AccessSniff 这样的工具可以帮助你实现 70-85%的目标，尽管你的里程数可能会有所不同。

[yargalot/AccessSniff](https://github.com/yargalot/AccessSniff)

#### [](#diverse-speakers)多样演讲者

现在是 2019 年，我厌倦了成为你阵容中唯一的女性会议发言人。去年，我在这方面变得更加积极，今年你可以期待更多。

这里有一个伟大的女性发言和组织活动的名单。请记住，这只是多样性的一部分。

[fempire/women-tech-speakers-organizer](https://github.com/fempire/women-tech-speakers-organizers)

这是另一个作品，科技领域的黑人扬声器。

[Samantha bretous/black-speakers-in-tech](https://github.com/samanthabretous/black-speakers-in-tech)

这绝不是全面的，但它是一个很好的起点。让另一个回购充满不同的发言者？让我知道，我很乐意添加它。
