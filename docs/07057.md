# 控制台应用程序伟大的 4 个原因

> 原文:[https://dev . to/claudiobernasconi/4-reasons-why-console-applications-are-great-3pnd](https://dev.to/claudiobernasconi/4-reasons-why-console-applications-are-great-3pnd)

在人人都在谈论用户体验、人工智能(AI)和 web 应用的 2019 年，你为什么要关心编写一个控制台应用程序？我给你四个理由。

## [](#simplicity)简朴

创建控制台应用程序比构建具有丰富用户界面的应用程序更简单。你不需要知道或使用任何用户界面语言、工具或框架。您可以使用简单的 C#编程实现控制台应用程序。

(开源)库帮助你完成重复的任务，如参数解析、输出格式化、菜单创建等。你不必从头开始。如今，根据您想要实现的业务逻辑构建控制台应用程序比以往任何时候都简单。

## [](#robustness)稳健性

控制台应用程序不需要随着每一个新的闪亮的框架或工具而改变。

控制台应用程序一旦创建，就不必每天都进行更改。当然，如果需求改变或者如果你想增加一个新的特性，你必须改变代码或者增加新的代码。

通常在用户界面中添加新功能会迫使你改变一些东西，并考虑进一步的改变来提供更好的用户体验。

[![Console application on notebook](../Images/d98873591d39a9a15e946c464375c540.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--TidNA3rp--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.claudiobernasconi.ch/wp-content/uploads/2019/04/conosle-app-on-notebook.jpg)

您可以将更多的精力放在应用程序提供的功能上，而不是构建人们不需要的用户界面。当然，它并不适用于所有用户和所有程序，但例如对开发工具是有效的。

开发人员希望尽快完成他们的工作。这就是可以从命令行执行的控制台应用程序在开发人员社区中流行的原因。

## [](#low-development-cost)开发成本低

与包含用户界面的应用程序相比，创建控制台应用程序要便宜得多。如果你想为一个有用户界面的应用程序建立良好的用户体验，有很多问题。控制台应用程序为用户提供了一个简单而有效的界面。

利用库可以节省时间和成本，同时您可以专注于需要完成的工作。它不仅使您的应用程序代码更加简单；这也有助于在更短的时间内完成项目。

## [](#great-for-automation)伟大的自动化

如今，在持续集成和持续部署的时代，自动化变得非常重要。如果您需要执行自定义代码，控制台应用程序通常允许您在自己喜欢的环境中编写所需的代码。

默认情况下，控制台应用程序可以从命令行执行，这允许简单的服务器端自动化或基于客户端的脚本。我喜欢当我能够写一个为我完成一系列任务的脚本的时候。

自动化不仅有助于加快工作流程，还有助于避免错误。手动执行的任务总是有风险的。你不知道是谁做的，如果这个人对这些任务没有经验，就会导致错误。通过自动化任务，你可以让你自己和你的团队避免犯可避免的错误。
