# 使用屏幕截图和视频差异加快 UI 代码审查

> 原文:[https://dev . to/evandeaubl/faster-ui-code-reviews-using-screen-and-video-diffs-5177](https://dev.to/evandeaubl/faster-ui-code-reviews-using-screenshot-and-video-diffs-5177)

代码审查是多个开发人员在同一个项目上工作的一个不可或缺的部分，无论您是在一个开发团队中，还是对公共开源项目做出贡献。无论是哪种情况，审阅者都需要对您的更改感到满意，然后才能合并它们。

这就导致了代码评审的一个法则:**评审者越清楚你做了什么更改，评审者就能越快地评审和合并你的更改。**

不幸的是，当你开发 iOS 用户界面时，这变得很复杂。无论您的团队选择了故事板、编程 ui 还是两者的组合，阅读故事板 XML 或代码差异都很难想象除了琐碎的更改之外还有什么。更糟糕的是，如果你有故事板和代码的组合——这是常有的事——你不一定能相信故事板之前和之后看起来就像应用程序做的那样。

为了使您的更改意图清晰，您需要另一种方式来显示这种差异。对于较大的 UI 更改，将这些差异表示为截图或视频是使这些更改尽可能清晰的关键。

* * *

如果通过用户界面的前后截图很容易表达你的观点，那么通过模拟器或设备也很容易做到。

模拟器:按 Command-S 截图。

在设备上:按下睡眠按钮，然后快速按下主屏幕按钮(或者在没有主屏幕按钮的设备上调高音量)。

如果你有一个多步骤的过程，或者需要展示一些动态的东西，视频是一个不错的选择。Quicktime Player 可以通过模拟器或设备录制您与应用程序互动的视频。

模拟器:

*   打开 Quicktime Player，选择文件->新屏幕录制
*   在模拟器屏幕上拖出一个方框，然后单击开始录制。
*   完成后，点按菜单栏中的停止图标、Touch Bar 上的停止按钮，或者按下 Command-Control-Esc。

在设备上:

*   通过 lightning 电缆将您的设备连接到 Mac。
*   打开 Quicktime Player，然后选择“文件”->“新电影录制”
*   点按“录制”按钮右侧的箭头，并选择您的设备作为摄像头和麦克风。
*   单击录制按钮。
*   完成后，单击停止按钮。

您现在有一个. mov 文件，但是没有一个流行的 git 服务或 bug 追踪器接受视频附件。然而，他们会接受动画。gifs，所以上传前的最后一步是转换你的截屏。幸运的是，有一个快速的开源工具可以做到这一点。

下载 [Drop 到 GIF](https://github.com/mortenjust/droptogif) ，安装，打开。这是一个简单的拖放界面，虽然你可能想点击齿轮配置设置，使文件更小。我一般选择不选匹配原件，设置 FPS 为 10。质量滑块对尺寸没有很大的影响，保持这些图像的保真度相当重要，所以我通常将质量设置为最高。

一旦你有了。如果你使用 GitHub、Gitlab 等，你可以很容易地拖放到你的 Pull 请求中。，或者上传到您选择的错误跟踪工具中。这是一点额外的前期工作，但是通过使您的代码评审对评审者来说更快，对您来说更快和更成功，这是值得的。
