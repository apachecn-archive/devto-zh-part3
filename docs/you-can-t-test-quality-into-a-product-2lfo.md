# 你不能测试产品的质量

> 原文：<https://dev.to/lastzero/you-can-t-test-quality-into-a-product-2lfo>

QA 测试和评审是在开发完成后进行的活动。您不能在构建功能之前测试它，也不能在编写代码之前检查代码。

[![](img/2b2946f15ad5842a962f5ce446d8289f.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--AZ2RNcnm--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/8pj98qubgwlw90w1ed8s.png)

越早发现潜在问题，管理缺陷和重新实现功能的开销就越低。

质量保证是从开发开始的积极主动的团队活动。它需要设计严密的反馈回路，以确保质量首先被构建到产品中。
