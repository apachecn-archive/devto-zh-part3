当你认真对待它时，学习 React(或任何新事物)都是关于实践的。当然，你需要先阅读一些教程和文档。也许看几个视频。

但是后来呢？一旦你陷入每次都用谷歌搜索答案的窠臼，这可能是一个很难打破的习惯。你发现自己在谷歌上搜索每一件小事，然后——下一次你遇到同样的问题——你不得不再次在谷歌上搜索*。每次我不得不退出 vim 时，我仍然会这样做。请派人来帮忙。*

 *事实是，当你不理解底层概念的时候——状态如何工作，如何传递道具，T2 如何用组件思考，从头开始构建一个应用是非常困难的。

即使你知道你可能会绞尽脑汁想出来，但是谷歌一下会更快。

你可以找到一系列的 [React 项目创意](https://daveceddia.com/react-practice-projects/)，但即使是这些也可能不是你想要的。也许它们没有涵盖你想学的技能，或者它们对你来说不有趣，尽管我很努力地列出了这个清单。

那么，你怎样才能提出自己的项目创意呢？

## [](#invent-your-own-project-ideas)发明自己的项目创意

好的项目创意来自哪里？

就我个人而言，我倾向于接触模仿现有应用程序或现实世界物体的东西。我尽量避免太过原创的东西。这可能看起来很奇怪，但是原创增加了复杂性。

如果这个项目是为了实践，那么你的目标是实践 React(或 Redux，或 GraphQL，或其他)——而不是同时实践设计、用户体验和产品开发(当然，除非是这样)。

这里，重点是关键。把问题缩小到最低限度。最低可行实践项目。一个微小的原子。整个生产应用程序全部功能的一小部分。

## [](#what-do-you-want-to-learn)你想学什么？

在你聚焦之前，你需要弄清楚你想把什么聚焦在上。选择一件事，即使你有一大堆技术要学。

想学 React？然后只专注于反应，完全靠它自己(我在我的书里教它的方法)。先不加 Redux。不要添加你还不了解的其他技术(无论是 GraphQL、AWS Lambdas、服务器端渲染，还是其他什么)。

[想学 Redux](https://daveceddia.com/how-does-redux-work/) ？希望你已经理解了 React 的基本知识——太好了！现在创建一个小小的 React 应用程序，并在其中添加 Redux，这样你就可以专注于 Redux 带来的新事物:商店、动作、[reducer](https://daveceddia.com/what-is-a-reducer/)、 [thunks](https://daveceddia.com/what-is-a-thunk/) 等。

想学 GraphQL？太棒了。确保你明白反应。然后创建一个包含 React + GraphQL 的小 app。现在，您还必须为后端找到或创建一个 GraphQL 服务器。

你可以找一个公共的——像 GitHub 的 GraphQL API(,然后专注于学习客户端……或者你可以建立自己的小 graph QL 服务器来玩。

### [](#minimize-and-simplify)最小化和简化

重点是这样的:*更多的活动部件*意味着*更多可以破坏*的东西，以及更大的调试难题。任何曾经在谷歌上搜索过[“对 CORS 错误做出反应”](https://daveceddia.com/access-control-allow-origin-cors-errors-in-react-express/)的人都知道，想到问题出在*他们关注的一项*技术上，而根本原因却完全是其他东西，这是多么痛苦。

所以:选择自己的难度设置。你不必一次全部学会，并行地。你可以及时了解这一切。一块一块的，像砌砖墙一样。

只要有可能，减少不属于你正在努力学习的内容的数量。

## [](#brainstorm-project-ideas)头脑风暴项目创意

看看你使用的应用程序——手机和桌面。看看你访问的网站。在你的房子周围寻找你可以用数字表现的实物。

选择你感兴趣的东西，或者与你的某个爱好有某种联系的东西。

进木工？做一个“截单”app。

进航空？也许“日志”会是一个有趣的项目。

放音乐？如何建立一个五分之一圈计算器。

为提前退休存钱，钱胡子先生？也许汽油里程计算器会有帮助。(或自行车里程，就此而言)。

以下是我在为一门新课程进行头脑风暴时想到的一些想法。希望它能激发你的创意。

*   恒温器。它是一个有控件和状态的物理对象；基本上是一个“计数器”加上一个计时器)
*   面包机。一些状态——开/关，亮/暗。一个计时器。
*   红绿灯。红/黄/绿灯和一个计时器。
*   活动跟踪器。最基本的，它只是你做的事情的列表。你也可以把它设计成一个基于链条的习惯跟踪器。
*   播客播放器。仿照 iTunes，或阴天，或任何你喜欢的。
*   天气应用。这可能非常简单，只需要几天时间…也可能非常复杂，就像一个完整的天气网站。
*   笔记记录器。学习 Web 音频 API，制作简短的录音，然后存储在列表中。添加/重命名/删除录像。
*   购物车购物网站。这是产品列表，也是购物车中的商品列表。添加/删除/更新。你可以让它只在客户端(静态数据)或者与服务器集成。
*   带有图形的可定制仪表板。每个人都喜欢好的仪表板。这里有一个学习 D3.js 并将其与 React 整合的机会。
*   网站分析。大多数情况下，它是一个页面名称和浏览量的列表。但是如果你想用图表来描述它，你可以在 D3 里工作。要真正做到这一点，您需要一个完整的后端，以及一些简单的 JS 代码来跟踪页面浏览量。
*   博客+编辑。基本就是 Wordpress。克隆他们的 UI 设计。
*   某种类似于 TypeForm 的测验/调查生成器(这将是获得正确的小 UX 细节的伟大实践)
*   起下封隔器。这几乎是一个待办事项列表，但是被重新组织了。你可以保存多个列表，比如“去度假”和“工作旅行”。
*   电子邮件客户端。仿照 Gmail 或 Fastmail 开发一款应用。使用静态数据的最佳实践。
*   松弛克隆。在 Slack 或者 IRC 之后建模一个 app。频道和消息。很好的 Redux 练习。
*   课程平台。仿照 Teachable 或 Podia 或 [egghead](https://egghead.io) 制作，边上是课程清单，中间是内容。良好的 CRUD 实践。
