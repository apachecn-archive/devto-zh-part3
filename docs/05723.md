在这篇文章中，你将知道应用程序如何监控性能，这样你就可以充分利用你的智能手机软件。

APM 集成在应用程序的整个生命周期中，包括部署前和部署后阶段。因此，这对于开发人员、测试人员和业务团队来说非常重要。

由于与 API、服务器或浏览器相关的一系列问题，加载缓慢可能会很普遍。APM 技术有助于查明实际问题。

基本的可用性监控包括测试 IP 协议和网络服务，但是用户需要建立一些最佳性能阈值和实时警报来监控和管理速度和可靠性。

**监控应用性能的提示**

为了优化用户体验，性能监控必须考虑某些因素。这些包括从用户的角度进行测量，即以用户感知的方式实时测量性能、稳定性和资源利用率。

背景也很重要。确定有多少用户受到了速度下降的影响，以及特定操作设备上的什么操作导致了中断，这一点非常重要。要真正体验“移动时刻”，你应该遵循这些提示来:

**测量帧渲染时间**

仅仅测量网络时间并不能满足用户的需求，因为用户对网络何时恢复以及何时响应都很感兴趣。

通过设备或操作系统测量响应时间，产品经理可以一次专注于一个平台，响应性能问题，并相应地部署资源。

**测量电池使用量**

测量移动应用的电池消耗以及数据计划影响和内存使用非常重要。如果应用程序耗尽了你的电池，你应该让它符合最佳实践。

还可以最大限度地减少频繁更新，从而提高用户满意度。

**测量小区数据消耗**

Dimensional Research 最近的研究表明，20%的受访者停止使用大量使用数据的应用程序。这表明您应该按应用程序版本、用户总数、每分钟消耗的数据量(kb)、每次点击传输的数据量以及每次点击消耗的数据百分比来衡量消耗。

**测量误差**

需要捕获来自某些 URL 的 HTTP 错误。这将有助于跟踪收到错误的用户数量、导致错误的操作、应用启动时出错的用户数量、导致至少一个错误的操作的百分比以及应用版本。
