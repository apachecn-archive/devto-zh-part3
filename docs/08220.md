这就是我(最终)进入软件开发的方式。完成了我的学位，在图书馆工作。25 岁时，我意识到自己被困在农村一份枯燥的工作中。所以，我离开了图书馆，一无所有地搬到了伦敦。在金融危机开始的时候。那很难。

最终找到了写文案的工作，这种工作转变成了营销职业。换了公司，发现自己在一个不好的地方。讨厌我的老板，讨厌公司，极度不开心——所以我什么都没准备就辞职了。

做了一点临时工，找到了一家可爱的公司，他们对我很好，我喜欢为他们工作。但我想彻底改变职业——我不喜欢营销，我想对自己的工作更加满意。所以冒险去了一个为期 12 周的训练营。

这很难；没有钱，没有保障，不知道接下来会发生什么。我很幸运，在离开后一个月左右找到了工作。那是大约四年前的事了。一直没回头。

是什么导致了这些决定？必要性。我有必要逃离这些事情，否则我会很痛苦，没有时间像现在这样做。

老实说，没有我的妻子，我不可能做到这一点。她在经济上和情感上都支持我。

[![thorstenhirsch profile image](../Images/43e9e167c96a7a04938ba4473bf4df26.png) ](/thorstenhirsch) [ Thorsten Hirsch ](/thorstenhirsch) • [<time datetime="2019-03-30T12:38:17Z" class="date-short-year"> Mar 30 '19 </time>](https://dev.to/thorstenhirsch/comment/9mjb) 

我所知道的最糟糕的经理是犹豫不决型的。他对球队没有目标，甚至连能持续几周的小目标都没有。他总是重复上级告诉他的话，并要求整个团队立即处理每一件小事。进展完全停滞了，但他今天仍然是负责人。

我所知道的第二糟糕的经理为自己制定了长期目标，但却对每件事都进行微观管理。更糟糕的是，她还向她的上司兜售一切都是她自己做的，甚至在文件和演示文稿中用她自己的名字代替她的团队成员的名字。最后，她团队中的大多数人突然退出，她被降级了。

我所知道的最好的经理有很深的工程知识，但根本不是微观管理。他花了大部分时间保护团队，这是保持生产力的一个很好的方法。他也遵循“公开表扬，私下批评”的规则。没有人想离队。

我在上班的第一周就用叉子叉了一个 GitHub 回购，然后当我意识到我用叉子叉错了一个时，我不得不删除它。嗯，尽管 GitHub 一再警告，我还是设法删除了 *source* repo，而不是 fork。至少有 6 个开放的 PRs 正在针对它进行开发，而且没有人有完整的本地克隆可以用来恢复它。值得庆幸的是，GitHub 的支持非常有用，并且恢复了它。

最糟糕的是，一些同情的同事幽默地解释说，所有 GitHub n00bs 在某个时候都会发生这种情况。问题是那时我已经使用 GH 至少 5 年了，我应该更了解它。
