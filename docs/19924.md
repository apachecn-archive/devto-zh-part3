# 我们分享得不够

我刚从一个会议中走出来，我们正在讨论一些内部系统的路线图。我有一个本可以解决一个特别困难的问题的想法，但只是搁置了一下，直到会后才说出来。

我把我的经理拉到一边，和他分享了这个想法。“太有才了！你怎么不早说？”。我只是回瞪着他——我以为自己会因为一个坏主意而受到惩罚。他开始追踪参加会议的每个人，和他们谈论这件事，并告诉他们我已经想到了。我不知所措，但这并没有阻止我在那天剩下的时间里昂首阔步。

# 你为什么要说出来

你们中可能有人有过类似的经历。我经常在其他工程师那里看到这种情况——尤其是现在我有了自己的团队。

我不知道是什么让我们不去分享或者说得更多。也许这是冒名顶替综合症，也许我们中的一些人觉得存在等级制度，初级工程师不应该说出来，或者也许我们只是假设其他人已经考虑过了。真的没关系——最终结果是我们分享的不够多。

我在这里告诉你，无论出于什么原因，你和我都不说话，这是错误的。对于那些在有毒的工作环境中提供解决方案不受重视的人，尽可能找份新工作。那种环境会拖你的后腿，反正你的生活中不需要那种消极。

你是一名工程师，你的雇主雇用你是因为他们认为你有所贡献。就看你去送了。这意味着不仅要编写代码和实现别人的解决方案，还要提出你自己的解决方案，大声说出来，并与他人分享！

## 你应该分享想法的 4 个理由:

1.  你对世界的看法不同。忽略“你和其他人一样独一无二”这句暗示你和其他人没有什么不同的话。你的经历和视角有一种独特的旋转，让你看事情和别人不一样。你有所贡献，但老实说，更糟糕的是，你有了解决方案，而你的经理却对你的解决方案感到恼火(这是我的人生故事，相信我)。
2.  你可能会学到一些东西。想法总是被否决，这完全没关系。我从和我交谈过的一些高级工程师身上学到了很多，他们向我展示了解决问题的更好方法。如果你愿意倾听，他们通常有东西可以教你。这样做足够多，你会发现你会成为一个有东西可教的人。
3.  你的想法是拼图的一部分。你在小组中头脑风暴的一个主要原因是，即使是最有知识的人也不知道所有的事情。你们中有多少人因为高级工程师或经理没有“明白”而被激怒？作为一名工程师，你需要学会如何帮助他们理解(同样，如果你处在一种很难做到这一点的文化中，就要找到一份新工作或新角色)。你不必成为所有答案的英雄。你会惊讶地发现，当你的想法有助于填补空白时，人们会有多感激。
4.  你的职业发展会加速。当你分享想法时，其他人会学习你的思维方式，并开始理解你和你的专业知识。他们会更愿意和你一起工作，和你一起集思广益。很快你将指导他人，并建立一个你可以联系的工程师网络来帮助完成工作。你甚至会发现有人走过来对你说“我听说你是某方面的专家”。

# 找到前进的方向

我知道我们很多人都不像喜欢技术那样喜欢与人交流。如果你努力的话，你会收获很多，不管你是想进入管理层，还是想从事建筑工作，还是想成为一名高级工程师，成为很多人的导师。就拿这个害羞内向的人来说吧，他沉溺于冒名顶替综合症，在显示器后面比在一群人中间更快乐。如果你想在事业上有所进步，想把事情做好，分享想法会起到事半功倍的效果。

如果你不知道从哪里开始，试试这里的帖子吧！分享你最近学到的东西，展示你正在做的项目，或者提出一个值得与他人讨论的想法。不一定要惊天动地。即使是简单的想法或学习也需要分享。很有可能其他人需要你一直在学习的东西。

诚然，这是我在 dev.to 上的第一篇帖子。我已经沉默了一段时间，终于开始吃自己的狗粮，走出自己的壳，并试图回馈这个社区。自己试一试，自己写帖子。你可能会惊讶有多少人认同你的想法。你甚至会发现有人评论说“太棒了！为什么以前没人这么说？”
