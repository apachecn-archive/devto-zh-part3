从我几个星期到几个月的紧张经历中，我建议如果你想尝试人工智能，并且它不是基于 Java / JVM 的，不要回避它。最好从你所拥有的高级语言开始，当你对这个主题有了足够的理解后，试着用你熟悉的语言来应用它们，不管是 Java 还是你可能知道的任何其他 JVM 语言。我并不声称我了解他们，只是分享我的里程。

跟随推动创新和灵感的社区和工具，在选择的主题上变得更好。在这种情况下，它适用于[人工智能](https://en.wikipedia.org/wiki/Artificial_intelligence)及其变种[2](https://en.wikipedia.org/wiki/Machine_learning)[3](https://en.wikipedia.org/wiki/Deep_learning)。

# [](#summary)总结

所以总结一下，我们在 [LJC Unconference 2018](http://unconf.londonjavacommunity.co.uk/) 上的讨论，我们提到了其他语言像 ***Python，R，Julia，Matlab*** 之类的，对 AI，ML，DL 的贡献比另一种编程语言更大。

[![](../Images/52c2ea3fc8ad83e0e860ced7758550ad.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--quqRrp9R--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://d2mxuefqeaa7sj.cloudfront.net/s_99A6B7BF29C0506FDFCFF787852D9B1FB1990DE18F39F5A1E21435041E3E8B6E_1545442687987_AI-ML-DL.png)

我知道这样说不会让我受欢迎，但我对所有开发人员的谦卑请求是，不要认为或期望一种编程语言有所有可能。任何语言，在这篇文章的上下文中，Java 和其他 JVM 语言都是为一个目的而写的，毫无疑问，我们可以在 [Java/JVM 语言](https://en.wikipedia.org/wiki/List_of_JVM_languages)中复制其他语言所做的努力。

但是，归根结底，它们都应该被视为工具，并在适当的时候使用。我希望这篇文章中分享的一点点仍将有助于激励 Java / JVM 社区，尤其是那些对诸如[人工智能](https://en.wikipedia.org/wiki/Artificial_intelligence)、[机器学习](https://en.wikipedia.org/wiki/Machine_learning)和[深度学习](https://en.wikipedia.org/wiki/Deep_learning)等主题有浓厚兴趣的人。
