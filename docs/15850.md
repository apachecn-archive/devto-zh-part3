## [](#the-story)故事

所以 2019 年 1 月 13 日我刚满 36 岁。这意味着我已经正式编码 25 年了。

但在那段时间里，我意识到我所有的辛勤工作和努力，就我的投资组合而言，我没有什么可以展示的。

我只有客户网站，其中大部分已经不存在了；或者我作为员工工作过的企业。

对此我做了大量的自我反省，至于为什么我没有表现出来，这是什么原因造成的。

它始于我的一个朋友，他曾经向我伸出手来学习编码，一天晚上打电话给我说“嘿，看看我的网站！我设的！”。

他购买了一个域名，一个 Wordpress 模板和一些便宜的主机。他设置了它并填写了内容。这是现场直播。

它看起来做得很好。我的偏好将不再是 Wordpress，因为我更喜欢使用 PHP 或 Node 构建完整的应用程序。但是他不知道如何编码，所以 Wordpress 对他来说是最好的解决方案。

在我教了他基础知识后，他在一周内就创建了一个很棒的网站，这让我感到很嫉妒。

这并不是嫉妒他的技能，因为我知道他只做了一两天就能学会的基础知识。

但事实是他开始了这个项目，并在一周内完成了 90%。

他在这么短的时间内就有了有形的产品。

他是怎么做到的？！他是如何克服发布不完美网站的恐惧的？他是如何找到为网站创建内容的动机的。他是如何保持注意力集中，不允许过度设计，让他陷入一个兔子洞？

我必须对自己诚实。我开始意识到我生活在对自己技能的怀疑中，害怕人们的评判。当涉及到我的项目时，我是一个完美主义者，所以我会开始他们，花大量的时间思考如何使它成为我能做到的最好的应用程序，但最终永远不会完成。

我唯一一次完整地完成项目，是在有了老板并为我的工作获得报酬的时候。我的个人项目会完成 50%到 90%,然后我会感到厌烦或者忘记它们。

在内心深处，我知道我做得很好！实际上，我每天都在学习，并且在工作中从事大项目。我在这个领域已经超过 15 年了。我和一些最好的开发人员一起工作，甚至向他们挑战。

我做过的软件/应用程序能带来数百万美元的收入，拥有数十万用户。所以在工作的时候，我是可靠和安全的。

那么我为什么要纠结于个人工作呢？这几乎就像人格分裂。

我意识到我必须回到我开始的日子。" K.I.S.S "简单点，笨蛋！

我也开始更加关注发展的社会方面。我又加入了几个团体。其中一个是初学者小组，允许我像一个新的程序员一样重新思考。

我注意到有些程序员只有几个月的工作经验，他们提供建议，甚至全力以赴。令我震惊的是，有那么多人不仅感兴趣，还感谢那个人。新的编码员正在培养一批追随者。

对我来说，这太令人震惊了。他们是如何做到这一点的，当人们几乎不知道他们在做什么的时候，他们又是如何关心他们的意见的呢？

我想我没有意识到有多少人在寻找比他们知道的多一点的人。他们不在乎那个人是否比他们多受 25 年或 25 分钟的教育。他们只在乎能不能从这个人身上学到东西，和他们建立起沟通的线路。

我很快意识到，即使在这里，我只张贴了几篇文章，我已经有了大量的追随者。唯一阻碍我的是我自己。

今年我想强迫自己改变。我决定一步一步来。

首先，我得出的结论是，由于从事兼职项目，我完成任何个人项目的压力都很大。我一直都是全职工作，在我人生的大部分时间里，我都有一些兼职客户来赚外快。

我决定，如果我赚不到足够的钱让我进入下一种生活方式，那就不值得我浪费时间。

我对此保持简单的思考。如果我没有从全职工作中获得至少 50%的收入，那么我就不应该做兼职。

所以我放弃了我的客户。压力的释放是惊人的。也没有更多的人来回答。只是一份正常的工作，我的家庭和我的副业！

我做的第二件事，是强迫自己构建一个快速的项目并发布它，不管我觉得它可能有多糟糕。

我也这么做了！你可以到[[https://Gidgitz.com](https://Gidgitz.com)去看看。

下一个目标是继续给这个网站添加功能。学习框架，包和设置，我可以很快掌握，并会很高兴用于未来的项目。

我一开始真的很强...但是后来我在 refactor tangent 上坚持了 3 个月。3 个月过去了，每天我都兴奋地编写新的 api 后端代码。

三个月前我意识到我在做我一直在做的事情。我必须把一切都复杂化，而且必须完美。我不想发布我所拥有的，因为我必须完成 x，y，z。

所以今晚，当我意识到我又一次犯了同样的错误，让自己陷入同样的糟糕模式时，我强迫自己接受目前所拥有的，并打破未完成的过剩。今晚，我发布了我能做的改变。

所以我做了！我对此感觉很好。不是很多，但已经启动并运行了。它可能不完全是我想要的样子，但它有一百万个可以改进的地方。但我这么做了，为了证明把它公之于众比把它放在我的笔记本电脑上更好，因为没有人有机会看到它。

我必须努力避免我的旧习惯。直面我对审判的恐惧。认识到即使没有人看到它，至少这个项目是存在的。这不是让它变得完美，而是让它变得可用，然后得到反馈并做出改进。

我的下一步是完成我的后端 api 设置，每隔几周发布一次小的更新。一旦我对我的设置感到满意，是时候创造我的下一个想法了。

最重要的是，我打算把自己放在那里。冒险向人们展示我的想法。把旧项目拿出来，让其他人也能看到。

在接下来的几周里，我会把它们贴在这里。

这是一个学习的过程。25 年的编码生涯，我现在才刚刚开始。

祝我好运！如果你读完了所有这些，你应该受到表扬！
