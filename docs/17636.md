# 编程语言概述(观点)

> 原文：<https://dev.to/donaldsebleung/an-overview-of-programming-languages-opinion-59b3>

作为一名从编码训练营出来的 Web 开发人员，后来我决定主修计算机科学(目前我正在学习第二年)，这些年来我使用过许多不同的编程语言，从初学者友好的脚本语言，如 JavaScript 和 Python，到行业标准的 C++/Java，再到利基函数语言，如 Haskell，以及最近的基于一阶逻辑的 Prolog。这篇文章旨在从我的角度介绍一些我已经学习或使用过的语言，包括我对它们学习曲线的难度以及掌握它们的难易程度。

### JavaScript

初始学习曲线:简单
掌握的障碍:高

JavaScript 是一种非常宽松的编程语言，几乎不需要样板代码，是许多初学者的首选语言。对于未经训练的程序员来说，它的动态、弱类型系统和隐式转换似乎非常有意义，比如完全没有整型值和浮点型值之间的区别(毕竟，数字只是数字——为什么`1`应该与`1.0`有任何不同？)或者双向常见的数字和数字串之间的隐式转换，例如，`3 - "2"`是`1`，而`"The number is: " + 42`只是`"The number is: 42"`。它的基于原型的对象模型乍一看也非常简单直观，并提供了很大的自由度——您可以用它来实践经典的继承或混合匹配，动态地向对象添加属性和方法，以及以普通的经典 OOP 可能无法实现的方式关联不同的对象。然而，掌握 JavaScript 完全是另一个问题。首先，它的动态弱类型系统意味着几乎所有的错误都被委托给运行时，所以一个有效的 JavaScript 程序随时都可能出错。其次，大量的隐式强制意味着即使是经验丰富的 JavaScript 程序员也会对看似无意义的行为感到惊讶，例如`"" == false`和`false == " "`但是`"" != " "`(也就是说，JavaScript 的松散等式操作符`==`是不可传递的(并且在`NaN`上是非自反的)！)，意味着它不能准确地描述等价关系)。最后，原型继承的概念本质上很难掌握，因为它提供了太多的自由——这就是为什么 ES6 和以后的版本“被迫”引入类似 Java 的`class`语法来隐藏它下面的原型 OOP 的原因(结果导致了更多关于 Java/JavaScript 的混乱:p)。

### 巨蟒

初始学习曲线:容易
掌握障碍:中等(？)

Python 在很多方面都很简短、简洁和可读，无论是在函数定义方面(例如，使用比`function`更短的`def`关键字)，还是在列表理解方面(`[x ** 3 in range(1000)]`描述了从`0^3`到`1000^3`的立方数的有序列表)。再者，Python 是“包含电池”的。想要列出一个列表的所有可能排列？使用图书馆。想要以一种酷的、实用的风格来映射、过滤和减少列表吗？使用图书馆。想要使用复杂的机器学习技术来分析您大学中数千名学生的成绩并预测他们未来的 GPA 吗？用库！Python 在重载操作符方面也相当宽容，对于你自己的类也是如此。可能唯一让 Python 难以掌握的是它所包含的内置库的数量——有可能有些库你甚至在使用 Python 一两年后也从未接触过。

### C

初始学习曲线:困难
掌握的障碍:容易

如果你在编程生涯中只使用过(非常)高级语言，那就准备好用 c 吧，在这种编程语言中，许多你最喜欢的内置语言根本不存在。想要连接两个字符串吗？抱歉，字符串没有内置的`+`操作符——可以考虑使用库函数`strcat`(它会进行适当的连接)或者使用自己的函数。想求一个数组的长度？抱歉，数组上没有这样的内置属性/方法——你必须自己跟踪数组长度。想要像在 Python ( `list.append(elem)`)或 JavaScript ( `arr.push(elem)`)中那样将一个元素追加到一个现有的数组中吗？如有必要，您必须自己重新分配内存，并确保不会超出范围。作为一门低级语言(至少与 JavaScript/Python 甚至 C#/Java 相比)，要准备好调试一大堆你在其他语言中做梦都想不到的神秘错误——悬空指针、越界数组索引、内存泄漏...更糟糕的是，大多数时候你不知道哪个错误是哪个——许多不同类型的错误(比如悬空指针和越界数组索引)都以 segfaults 的形式出现。甚至指针本身的整个概念(这是 C 编程语言的基础)也可能需要你花很长时间才能掌握。C 的好处是它的速度快得惊人——在几乎任何其他语言中(可能除了 C++ ),你都不可能获得与高效的 C 程序相差甚远的速度。此外，一旦你真正理解了 C 语言中指针(和结构/联合)的概念，就没有更多的东西要学了——你唯一需要关注的是更好地掌握它。

### C++

初始学习曲线:中等
掌握的障碍:非常高

乍一看，C++似乎并不太糟糕 Hello World 程序可能看起来有点奇怪:

```
std::cout << "Hello World!" << std::endl; 
```

Enter fullscreen mode Exit fullscreen mode

但是后来你慢慢就习惯了。拥有比 C 语言更多的内置功能，比如一个`std::vector<T>`类型(像一个动态数组)、一个`std::map<T1, T2>`和一个`std::string`，这意味着开始编写小而不平凡的 C++程序并不需要花费太多精力。诚然，开始使用 C++可能不像开始使用 JavaScript、Python 或 Ruby 那么容易，但是，嘿，你不得不称赞这种已经有 35 年历史的编程语言！

问题是，如果你以这种方式“学习”C++，你将对 C++实际上是如何工作的一无所知，你在 C++中能做的事情将变得非常有限——如果你甚至不理解构造函数/析构函数是如何工作的以及它们的用途，你将永远无法定义有趣的、重要的用户定义类型。但是为了让*以那种方式了解* C++，你基本上必须学习*关于 C 的一切*(指针、结构、联合、静态/动态内存分配/释放)**加上**c++带来的所有那些额外的特性——类、构造/销毁的顺序、继承、静态/动态绑定、抽象类、虚拟和纯虚拟函数、通过模板的泛型编程...你说吧。有传言说，C++多年来积累了如此多的特性，以至于一个拥有五年经验的高级 C++程序员可能会遇到一个他/她根本无法理解的程序。也许关于 C++的一个主要事情即将改变——有传言说，C++委员会正计划从 C++20 开始逐步淘汰原始指针之类的东西，这样未来的 C++程序员就可以免去手动内存管理和悬空指针的痛苦(这只是 C++的很小一部分)。

### Java

初始学习曲线:中等
掌握的障碍:中等

如果你以前从未编程过，Java 可能会给你一个相当令人生畏的编程印象。考虑最小的 Hello World 程序:

```
// Program.java
public class Program {

  public static void main(String[] args) {
    System.out.println("Hello World!");
  }
} 
```

Enter fullscreen mode Exit fullscreen mode

你的第一个问题可能是:什么是`class`？那些关键词`public`、`static`、`void`都是什么？`System.out.println("Hello World!");`里的那些圆点是什么？这一切看起来太可怕了！Java 教授/老师/教程说“现在忽略所有的样板文件——我们稍后会解释它”似乎没有帮助但是如果你设法克服了这一点，你可能会意识到 Java 是一种相当简单的语言(比如说，与 C++)有一些基本的数据类型来处理数字(`byte`、`short`、`int`、`long`、`float`、`double`)、字符(`char`)和逻辑值(`boolean`)，其他的都是对象(引用)。此外，Java 的类层次结构相当简单——每个对象(引用)类型都有一个基类`Object`,然后每个类都直接或间接地继承它。也没有奇怪的事情，比如多重继承(如 C++中所见)或任意运算符重载(所以`+`只意味着数字相加或串联`String`)。此外，Java 可能相当严格，拒绝在早期编译您的许多程序，但这意味着您遇到的最糟糕的运行时错误可能是`NullPointerException`——没有诸如 segfaults、悬空指针/引用或内存泄漏之类的奇怪东西。此外，从 Java 8 开始，可以使用 Java 8 流提供的函数式编程技术编写(相对而言)简短但简洁(而且通常高效)的 Java 代码。学习整个 Java 规范的一个相当大的子集并不需要太多的努力，尽管随着越来越多的特性和语法糖被添加到 Java 中(由于 Java 旨在保持向后兼容性，很少会被弃用/过时)，随着时间的推移，这可能会变得更加困难。

### 哈斯克尔

初始学习曲线:非常高
掌握的障碍:高(？)

如果你已经习惯了用“传统的”编程语言(几乎都是经典的 OOP 语言)编程，并且想要学习 Haskell，那么就准备好重新学习四分之三的编程(本身！).由于严格遵循函数范式，Haskell 将迫使你重新思考编程意味着什么。不再有循环，不再有可变的对象，不再有命令式的代码(“做这个，做那个”)——一切都必须按照方程和纯函数来完成，通常涉及到高级编程技术，如递归。这仅仅是开始。准备好接受代数数据类型、递归数据类型、单子、幺半群、函子的挑战吧...强烈建议学习 Haskell，要有很强的数学背景和数学思维方式。

### 序言

初始学习曲线:非常高
精通门槛:中等(？)

Prolog 的整个语言都是基于 Horn 子句的，Horn 子句是形式一阶逻辑中谓词逻辑的一个子集(以及被称为“modus ponens”的推理规则)，所以如果你没有上过离散数学课程，那么即使是最简单的 Prolog 程序，你也可能会感到吃力。然而，如果您设法理解了这一点，您可能会发现 Prolog 非常简单、优雅、富于表现力且功能强大——编写解析器来分析简单的英语句子(如“一个人吃了一个苹果”)并不需要专家的帮助当然，这可以很容易地扩展到包括分析适当的编程语言(通过扩展，包括处理它们的编译器/解释器)。在 C/C++/Java 甚至 Haskell 这样的传统编程语言中，这是很难实现的。
