# 重新设计一个 app:重构还是从头开始？

> 原文:[https://dev . to/bridgerputnam/re designing-an-app-refactor-or-start-from scratch-2m 29](https://dev.to/bridgerputnam/redesigning-an-app-refactor-or-start-from-scratch-2m29)

重新设计我的副业项目[阅读长度](https://www.readinglength.com)，已经在我的雷达上很久了——当我点击“部署”的那天，我知道在我可以实现我想要的一些特性之前，需要做出重大的改变。问题是，我对 JavaScript 知之甚少，不知道从哪里开始，也不知道最好的方法。

就这样，学习 JavaScript 的漫长旅程开始了。现在我已经了解了 Node、React、NoSQL、GraphQL 和一大堆其他东西的来龙去脉，我需要做一个决定:

## [](#do-i-refactor-what-i-have-or-start-from-scratch)我是重构现有的，还是从头开始？

在我继续删除我的代码库之前，我有一个主要的考虑:**首先写它要花很长的时间**。重写我所拥有的东西，使之更清晰，或者把它分解成适合我的愿景的碎片，这值得吗？

我的一部分想要[深入回购，尽我所能重构](https://github.com/Putnam14/ReadingLength/tree/refactor)。另一方面，我上过商学院，知道沉没成本谬论。

对代码库的客观观察表明，在意大利餐馆出售它可能不会受到惩罚。当我对 JavaScript 几乎一无所知时，我写了这篇文章。JavaScript 之所以有意义，是因为它似乎是未来的发展方向，它比 WordPress 能给我更多的控制权。Node 看起来很酷，所以我选择了它，并摸索着找到了一个可用的网站。

一年后，我对 JavaScript 了解得更多了。我读过书，上过一些很棒的课程，建过一些很棒的网站。我实现了一个风格指南。我学过面向对象的概念。我现在拥有的代码并不代表我的技能，我现在肯定会以不同的方式处理这个问题。那么为什么要把自己绑在这个旧代码上呢？

我将从头开始重新设计，并在这里从头到尾记录这段旅程。
