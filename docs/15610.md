# 如何开始 Web 开发

> 原文:[https://dev . to/aryanjnyc/how-to-get-started-in-web-development-j8l](https://dev.to/aryanjnyc/how-to-get-started-in-web-development-j8l)

# [](#capture-your-why)捕捉你的为什么

讽刺的是，你已经开始了你的 web 开发之旅。你决定适可而止，是时候开始了。你在网上搜索如何开始的指南，偶然发现了这篇文章。

审视一下你现在的状态。是什么在给你加油？学习成为一名网页开发者会给你带来什么？注意这种燃料，捕捉你内心的火焰。带着这些燃料到处走。在艰难时期你会需要它，因为学习 web 开发将是一个漫长(但值得)的旅程。当你发现自己燃料耗尽时，提醒自己为什么。

# [](#find-your-roadmap)找到你的路线图

要到达目的地，你需要一份路线图。幸运的是，有大量的地图可用。例如， [freeCodeCamp](https://learn.freecodecamp.org) 和 [Odin Project](https://www.theodinproject.com/courses) 课程都是很好的开始(选择一个，不需要两个都做)。如果你想要更正式的课程，看看 Coursera 的[全栈网络专业化](https://www.coursera.org/specializations/full-stack-mobile-app-development)(你可以免费旁听课程！).浏览地图，了解一下会发生什么。

收集补充资源，以便在 web 开发过程中使用。例如，Codecademy 提供了一个 [HTML & CSS 课程](https://www.codecademy.com/catalog/language/html-css)，它补充了你将在 freeCodeCamp 或 Odin 项目中做的任何 HTML 和 CSS 练习。

# [](#slowly-and-steadily-learn)慢慢地、稳步地学习

一旦你创建了你将遵循的课程和你将使用的补充材料的地图，就开始吧。趁火还在燃烧，马上开始吧。

同时，不要太快耗尽自己的精力。在无数的场合，我看到人们在学习 web 开发的第一周每天投入八个小时，但在第二周却累得没有时间投入。

保持热情的更好方法是从更小的承诺和习惯开始。James Clear 在《原子习惯》一书中描述了小习惯是如何带来轻松胜利的，这给了我们培养这些小习惯所需要的正面强化。所以，不要每天八个小时，只需要每天 15 到 30 分钟。一旦你养成了坐下来锻炼你的 HTML 和 CSS 的习惯，把你的时间延长到一个小时。如此等等。

一旦你对 HTML 和 CSS 中的练习感到舒适，是时候用你新开发的技能建立一个项目了。

# [](#build-projects)构建项目

项目是你学习 web 开发的面包和黄油。做练习和教程的唯一原因是能够编写自己的项目。此外，雇主正在寻找有经验的开发人员，而不是成功完成教程的开发人员。不要让*而不是*被[的教程跑步机](https://www.thewebdevcoach.com/blog/2018-12-14-learning-to-code-get-off-the-tutorial-treadmill/)缠住。

选择一个合适的项目。作为人类，当挑战太容易时，我们会感到厌烦，当挑战太难时，我们会感到沮丧。如果你正在学习 HTML 和 CSS，不要试图创造下一个脸书。相反，[建立一个致敬页面](https://learn.freecodecamp.org/responsive-web-design/responsive-web-design-projects/build-a-tribute-page)。如果你正在学习 React，就造一个[随机报价机](https://www.youtube.com/watch?v=3QngsWA9IEE&index=3&list=PL3cz80ehFCandt8KYS1NRzfjjIqpeu5c4)。确保你选择的项目与你的技能相匹配。
