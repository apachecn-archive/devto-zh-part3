# 对如何教学生一门困难学科的见解

> 原文：<https://dev.to/tomasmuzas/insights-on-how-to-teach-students-a-difficult-topic-4bca>

# 一点点脉络

我自己也是学生。在我们大学，我们有一个由学生推动的倡议，称为“学生对学生”——高年级学生自愿给低年级学生讲授困难的科目，消除了标准讲座的所有形式。

第一学期，来了一群一年级的学生。他们只被介绍给可怕的集会🙂之前没有经历过大学讲座，在 ASM 开始第一年的学习时，一些完全不同的东西让新来的人感到困惑。这就是我们介入的地方😁

尽管我主要是组织讲座和安排礼堂，但我自己也做过一些讲座(如果有人问，那是关于中断、MPL 和条件操作的)。

在这篇文章中，我想分享我的经验，什么有效，什么无效。

# 举出现实生活中的例子

在我的案例中，比较汇编代码和 C 代码非常有效(在立陶宛，C 是学校教授的标准语言)。c 代码是所有学生都熟悉的，因此更容易理解和阅读汇编代码。

有时不可能将它转换成 1 比 1，但结果证明这样更好。ASM - > C 有时需要 C 中的额外步骤，这反过来解释了 ASM 的底层逻辑。

这样，通过看到 ASM 旁边已经熟悉的代码，他们可以更好地理解事情。

# 不要忘记上下文

当进入一个未知的主题时，知道事物为什么以特定的方式工作或实现比知道它们在做什么要有用得多。

这就是为什么我不仅仅谈论我的话题。例如，当谈到条件操作时，我必须解释内存是如何工作的，命令是如何作为机器代码构造的，它们是作为一个接一个的序列编写的，等等。

当你知道命令被一个接一个地写入内存时，机器代码中的`jmp`和它们的偏移量就变得更有意义了。

# 使用简单的语言

这可能是显而易见的，但使用简单的语言来解释困难的事情是至关重要的。避免使用任何缩写，直到你解释了它们，并确保学生理解了它(下面会有更多)。

# 自己做学生

这不是字面上的意思。你需要知道，学生们还不是严肃的科学家，他们注意力不集中，喜欢开玩笑。不要过度使用笑话或迷因，当它来自年长的人并被过度使用时，他们会觉得很别扭😁使用非正式语言，使用学生俚语。

# 确保题目理解

如果听不懂，讲课就没有意义。对问题持开放态度，主动提问，甚至强迫别人回答。尤其是睡觉的人😁当然，不要期待完美的答案。如果他们的答案是错的，就向他们提问，了解他们的思维方式并填补空白。

在转移到另一个话题之前，如果大多数人已经理解了一个观点，那么要进行两到三次检查，并且在这个话题上停留更长的时间，即使你不得不再次回到绝对的基础。

# 做出停顿

停顿真的很有必要。在一个科目上集中注意力一个半小时真的很难。给学生五分钟的休息时间来恢复他们的注意力，思考你一直在谈论的话题，或者只是抽根烟😁

# 让你的讲座互动起来

不要只是给他们你所有的信息。让学生自己解释事情。问问他们，为什么某些东西会这么叫。问问他们，他们的看法是什么。问他们控制问题，看他们是否跟随你。让他们猜，让他们多次失败，从自己的错误中吸取教训。

# 有实践，没有理论

给学生一些练习来加深对主题的理解是有好处的。有人解决了问题后，让他们向同伴解释解决方案。这对他们和其他学生都有好处。

这些是我的教学心得🙂请分享你的技巧和诀窍，这样我和其他读者可以提高我们的教学技能！
