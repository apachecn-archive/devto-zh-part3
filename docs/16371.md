# 为什么我热身(也许你也应该！)

> 原文：<https://dev.to/gmeluski/why-i-warm-up-and-maybe-you-should-too-3ci7>

*本帖最初发表于[meluski.com](https://meluski.com/engineering/2019/01/11/why-i-warm-up-and-maybe-you-should-as-well.html)T3】*

在过去的一年里，我养成了早上第一件事就是跑步的习惯。不要拉伸——只要醒来，走到外面去跑步。很奇怪。我在高中时参加过越野赛跑，每次训练我们都要做至少 15 分钟的热身。这让整个努力看起来很沉重，最终让我不想再跑了。

去年我读了《和海豹一起生活》,意识到作者训练的非常健康的人在跑步前没有热身。一点也不。我可以理解。现在，如果不冲浪或游泳的话，我每周至少跑步五次。不远，但这是新的一天的开始。

我永远不会为工作热身，在工作强度和工作量方面，我几乎无法控制。相比之下，没有热身的跑步仍然让我能够控制速度，并让我能够在事情准备就绪时做出最佳判断。编码可能是脑力劳动，当与典型开发角色的所有其他类型的思维结合在一起时，您可能会谈论许多上下文切换和不间断的精神紧张期。没有任何借口就投入工作实际上看起来有点疯狂——你可以很容易地有一个突然的、戏剧性的加速度，能够转向多个方向。

“干净的程序员”中有一个金块，关于每天开始和结束时做十五分钟的热身和放松。这引起了共鸣。所以我开始每天花时间做编码练习来热身。

我真的很喜欢它，有几个原因:

我每天至少有一部分时间专注于我的编程技能。有些日子，我几乎不写代码，也不深入思考编程问题。过去，如果我一整天都在开会，或者没有着手解决某个问题，我会很沮丧。编码对我和我的大脑来说是一个熟悉的空间，在这里我可能会找到一种流畅感和节奏感。我的工作并不经常欠我这个，所以我可以得到我的满足，即使这一天即将排除了腾出时间来编码的可能性。

热身往往是算法问题，这使精神肌肉保持敏锐。在一个典型的星期里，我有多少次挑战自己去用算法思考？答案是不多。

就我个人而言，我认为做更深入的编程工作并使代码尽可能的干净和高效是一种特别的快乐。我意识到，只有在我找工作或者偶然遇到一门引起我兴趣的课程时，我才会有意识地提高自己在算法思维方面的技能。那不是很傻吗？对于一个程序员来说，那是很傻的。

最近我一直想学习 Rust，我有很大的意图:我会浏览代码库和编译东西，我会到处阅读文章，但我在这方面停滞不前。所以我决定用铁锈作为我热身的首选语言。我利用这段时间学习一门新语言的复杂性和弱点，而不是我不太熟悉的那门语言。使用一种不同的语言有助于对我在 Javascript 中认为理所当然的事情进行反思。我很感激。

最后，我认为在更高的层面上把前面的观点联系起来:生活应该是关于成长的，你不能总是指望你的工作来挑战或成长你。有时候，推动项目或公司的发展比你渴望去做的事情更重要。围绕这份工作的核心前提进行一个小练习有助于在这段时间里保持动力。从本质上来说，热身是程序员自我指导的一种形式，是一种值得开发的技能。
