这是关于 Angular 的起步、发展和基本特征，没有过多的细节。

* * *

**2009** - Angular 最初由 Misko Hevery 于 2009 年开发。
2009 年底的某个时候，他工作的一家公司决定把代码放到 Git 上，成为开源。

**2010** -最初的 rebase 是在 2010 年，也是谷歌维护最多的一次。
AngularJS 是一个基于 JavaScript 的框架，工作方式类似于具有 MVC 架构模式的单页面应用程序。

**2013**——2013-14 年出现了其他 web 框架和工具，最著名的是 React，由脸书开发者开发。

**2016**——从一开始，React 就显示出良好的结果和更好的性能。angular 团队需要改变一些东西，他们决定不升级 angularjs。
相反，他们决定从一开始就开发一个新的框架。只保留名字的第一部分，砍掉‘JS’，添加版本 2，2016 年 9 月 Angular 2 创建。同一个团队开发 AngularJS 和 Angular 2。

**2018 年 11 月** -当前版本:Angular 7

**角度 2 基本特征**

*   组件-MVC 模式
*   每个组件类都是一个独立的单元，具有高度的可重用性。
*   每个组件都需要逻辑、视图和数据作为一个整体。也就是说组件使用 MVC 模式，框架使用组件->组件-MVC 模式。
*   向现有应用程序中添加更多组件很容易。
*   使用 Angular 2 组件建造水疗中心就像使用乐高积木建造房子一样。只需添加更多的乐高积木，让你的建筑更大。

* * *

*   Angular 2 框架由几个库组成，有些是核心的，有些是可选的。
*   这些库中有许多是模块(即 FormModule、HttpModule 和 RouterModule)。

* * *

*   Angular CLI 代表 Angular 命令行界面。
*   建议使用 angular CLI 创建 angular 应用程序，因为您不需要花费时间安装和配置所有必需的依赖项，也不需要将所有东西连接在一起。

* * *

*   使用 Typescript，我们可以使用数据模型和类型安全。

* * *

*   Angular 团队在文档方面也做得很好，有强大的社区和 Google 的支持。
