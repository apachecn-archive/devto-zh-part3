# 我是一名前端开发人员。还是我？

> 原文：<https://dev.to/lynnewritescode/im-a-frontend-developer-or-am-i-3gkh>

作为一名前端开发人员，我总是发现很难在前端和后端开发之间划清界限。显然我不是一个人在这样做！

我在一家数字代理公司的第一份工作是使用 HTML、Sass 和 Coffeescript 来构建网站，尽可能与设计团队提供的设计相匹配。除此之外，我还会使用 Ruby on Rails，我们的定制 CMS 就是在它的基础上构建的，将数据拉进视图。

在那份工作中，我有一个很好的导师，他教会了我很多关于 Sass 和代码组织的知识，我今天仍然对此充满热情(西蒙，我对你感激不尽！).大三的时候，我陷入了困境，对我来说，这是最好的学习方式。

在我看来，这是一个前端开发人员的角色，在当时感觉定义得很清楚。

### 新篇章

2015 年，我转到了一家不同的数字机构。前端开发人员编写 HTML、Sass 和一些 JavaScript，然后将它们全部传递给后端开发人员以连接到 CMS。

从那时起，事情发生了很大的变化，现在我们的前端使用 CMS 和 C# Razor 标记，并在需要时使用 React 和 Vue。但是似乎由于前端框架越来越受欢迎，前端开发者实际做什么的定义变得更加模糊。

其他人有时很难知道哪些任务是前端的，哪些是后端的。有时我觉得人们似乎没有完全意识到前端开发所涉及的工作量。

我们处于一种混合角色，在设计师和后端开发者之间工作，以确保你的网站看起来很棒，表现良好。

招募新的前端开发人员也变得很困难。作为一家公司，我们以建立尽可能符合我们设计的网站为荣。人们会直接告诉我图标是否超出了几个像素。我知道你在想什么，但这不是坏事。它让我在工作中变得更好，像这样最小的变化也会对最终产品产生巨大的影响。公司的每个人都知道这个。这是一个团队的努力，是有建设性的。没人会把它当回事。每个人都非常关心我们在创造什么。

招聘新前端的困难在于，我们需要一个在 HTML、Sass、样式和设计方面非常有能力的人——同时也是一个优秀的 JavaScript 开发人员。如今，他们似乎比以往任何时候都更加分裂。

我对开发人员社区的问题是:

*   有没有其他人也有这种想法，还是只有我有这种想法？😁
*   作为一名前端开发人员，你认为你的角色是什么？你如何定义它？
*   你觉得“前端开发人员”这个头衔适合它所涵盖的大量技能吗？
*   在招聘方面，你有过类似的经历吗？在这个职位上找到一个和你有同样期望的人？

我认为这是一个有趣的讨论，我听说过前端开发人员感到不被重视的例子，因为人们“并不真正知道他们在做什么”。此外，与后端开发相比，他们需要掌握大量的技能。
