# 加速你的普通 PHP 编码测试

> 原文:[https://dev . to/shahriarsiraj/speed-up-your-vanilla-PHP-coding-test-3hl 6](https://dev.to/shahriarsiraj/speed-up-your-vanilla-php-coding-test-3hl6)

首先，我给大家讲一个关于我自己的小故事。我是一个住在瑞典的全栈开发人员，使用 PHP 开发 web 应用程序，特别是 Laravel 框架已经超过 2 年了。最近，我在瑞典申请网页开发工作，几乎所有的申请，我都通过了编码测试，有资格进入下一阶段。你猜怎么着？所有的编码测试都需要开发一些简单的东西，但是使用原始的(或普通的)PHP，不使用任何外部库。

听起来很简单吧？我也有同感，想到一夜之内完成编码测试。但是由于我使用 Laravel 进行开发工作，当我实际开始为测试编码时，我意识到那些简单的事情变得非常复杂。在 Laravel 中，我不需要考虑漂亮的 URL、动态路由、sql 注入、视图和逻辑分离等等。但是使用原始 PHP，现在我需要处理所有这些，我不能使用任何框架或包。

所以，我开始写代码，在每次编码测试时都牢记这些。经过几次测试后，我意识到我已经下意识地创建了自己的框架，因为我在接下来的测试中一次又一次地使用相同的东西。然后，我想，为什么不创建一个我可以为将来的编码测试克隆和扩展的存储库呢。

**这里是框架:**【https://github.com/ssnigdho/vanilla-php-framework】T2

如果你和我一样，需要加速你的原始 PHP 开发，请随意使用它。
