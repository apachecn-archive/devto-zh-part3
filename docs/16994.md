# ReactJS vs Angular:谁会赢得这场战斗？

> 原文:[https://dev . to/hoanglamyeah/react js-vs-angularjs-who-will-win-the-battle-3kd 6](https://dev.to/hoanglamyeah/reactjs-vs-angularjs-who-will-win-the-battle-3kd6)

随着我们的成长，技术在不断变化，事情变得越来越现代，但也越来越麻烦。对于 web 开发领域来说，这无疑是正确的，专业人员需要对每个新项目有新的想法。企业也在思考新的商业推广方式。

移动应用是每个品牌都需要的最时尚的功能。但是，拥有一个好的应用程序的主要需求是:使用最好的技术、平台和框架。React 和 AngularJS 是 JavaScript 框架的两个流行名称，具有独特而强大的特性。

如果有人会问技术上，有几个共同的特点，包括前端使用 CSS3，HTML5，纯函数，继承和许多类似的。

但是，当比较这两者时，几乎没有什么方面存在。让我们阅读下面的内容来进一步了解。

## [](#react-and-angular-two-different-teams-with-varied-skills)反应和棱角分明:两个不同的团队，技能各异

这场战斗是在两个领先的名字之间进行的，Angular 是一个成熟的框架，React 是一个库。

AngularJS 于 2009 年推出，主要由谷歌支持。它的一些主要优势是双向数据绑定、编写自定义指令和使用指令来扩展 HTML 功能。

ReactJS 库在 2013 年发布了第一个操作系统，得到了脸书和 Instagram 的支持。React 允许轻松实现 web 组件来扩展 HTML 文档。它的显著特点是虚拟 DOM，能够改变组件的状态以修改视觉呈现，并下载数据流，其中父数据与子组件保持不变。

让我们深入探究区分这两者的变体。

**部署**

Angular 有相对较少的工具来以我们想要的定制方式打包代码。这里，HTML 模板在大多数情况下被用来给代码一个古老的外观。

另一方面，React 从原生 JavaScript 的角度工作。这里，像 RequireJS 这样的库可以用来加载代码。还有许多其他类似的解决方案，React 都支持开箱即用。

**流程**

AngularJS 是一个强大的开源框架，用 CSS іn JavaScript 构建由一个 HTML 页面组成的 web 应用程序。它拓宽了基于 MVC 的浏览器应用，简化了开发和测试过程。有了 AngularJS，你就不允许拘泥于一个应用程序结构，一个惰性初始化被用来缩短加载时间。

Angular 中的数据表示具有非常紧凑的视图，但是与 React 不同，它似乎缺乏处理大量数据的能力。在需要存储的数据较少的情况下，这种方法效果很好。

ReactJS 允许创建无需重新加载就能随时间更新的应用程序。它是一种局部更新技术，提供了一套开发单页面应用程序的工具。虚拟 DOM 有助于提高开发速度。它呈现大量数据的能力使它成为领先的 JavaScript 框架。

**架构**

这两种框架的应用程序架构存在很大的差异。按照惯例，AngularJS 使用被称为 MVC(模型-视图-控制器)的架构模式。然后切换到 MVVM，缩写为模型-视图-视图模型。

如果我们选择 ReactJS，它只使用视图模式，因为它主要关注数据呈现，而将所有其他组件留给开发人员。

**脚手架**

当涉及到为您的生产环境捆绑东西时，对于模块和依赖关系管理有一场艰苦的斗争。带有 Angular 2 的 Angular-cli 提供了一种漂亮的搭建方法，可以生成样板文件供您开始使用。

React 在口袋里有一个很棒的工具，叫做 webpack，简单易用。它有丰富的功能，如代码分块和生产环境压缩，以方便应用程序的发布。

*移动-接近*

同时创建网络和移动应用就像是锦上添花。React 和 Angular 都可以帮助你创建移动应用，但方式不同。离子 1 和 2 分别适用于角度 1 和 2。

当把 React Native 考虑在内时，人们注意到，与利基市场中的其他名字相比，React Native 非常受欢迎。

从技术角度来看，Ionic 和 React Native 的不同之处在于，Ionic 用于构建使用 AngularJS 架构和 Cordova 的混合 web 应用程序，而 React Native 则提供完全的原生体验。

**技术**

AngularJS 利用指令，使用 HTML、CSS 和 JS 代码创建高质量的应用程序。然而，React 仅使用带有 JSX 语法的 JavaScript 将 HTML 插入 JavaScript 代码。

开发 AngularJS 的目的是使编写应用程序测试变得容易。当模块按照官方文件使用时，可以期望得到最好的结果。
