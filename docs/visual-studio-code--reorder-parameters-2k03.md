## 问题

有时候你会写代码，把参数放错顺序，比如我的 System.assertEquals，这里，我把 actual 放在了期望值的上面，把 expected 放在了实际值应该在的地方。我可以通过复制粘贴或删除其中一个值并重新添加来解决这个问题。

我还打乱了参数的顺序，我想让`one`在第一个，而不是`two`。

## 换档器

我希望`two`在这里，我可以再次做整个复制粘贴的事情，或者，这是一个顶点，我可以得到一个名为移位器的扩展，我会在下面提供一个链接，但移位器，只是安装它，现在我可以使用`ctrl + alt and left`或`ctrl + alt and right`，并重新排序我的参数。

它也适用于方法的参数定义。不需要复制粘贴任何东西。这很简单。所以它在 apex 中工作。我们也可以在 JavaScript 中使用它。

所以在我们的 JavaScript 文件中，如果我们有一个有两个参数的方法，如果我们有一个数组，并且它们没有按照正确的顺序排列，我们可以重新排列这些参数。

我们也可以在数组中使用它。所以我们可以在方法的定义中重新排序，我们可以重新排序数组的值，我们也可以在调用方法时重新排序。
