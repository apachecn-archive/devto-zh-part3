# JavaScript 开发人员(以及其他任何人)的薪资谈判

> 原文：<https://dev.to/kahlil/salary-negotiations-for-javascript-developers-and-anybody-else-40f6>

对我来说，工作开始时的薪资谈判总有点像一场战争。

每一方都试图站在更高的位置，以获得相对于另一方的战术优势。

典型的情况是，其中一方准备不足。一般来说，这就是未来的员工。

奇怪的是，你与未来雇主的第一次互动是他们试图从你身上得到最好的东西。默认情况下，公司应该支付公平市场价格，但他们没有。他们想尽可能便宜地得到你。

我从一个朋友那里学到了一些关于薪资谈判的事情，从我的经历中，从这个 [twitter 帖子](https://mobile.twitter.com/filosottile/status/1070099435085066241)中，我看到了[这篇关于这个话题的惊人的 7k 单词 Kalzumeus 文章](https://www.kalzumeus.com/2012/01/23/salary-negotiation/)。

以下是我学到的三件最重要的事情。

## 1。永远不要给出数字

如果他们想知道你想做什么，他们想利用你来降低你的要价，不要告诉他们。

而是告诉他们:

> 首先，我感兴趣的是我们是否相互适合。我很乐意在稍后的过程中讨论财务问题。

如果他们一直问，就说这个。

> 现在钱对我来说不是那么重要，我想先看看我们是否合适。我确实希望得到公平的薪水。

如果他们想知道你目前挣多少钱，他们想获得对你的优势，看看他们是否能压低你的价格，不要告诉他们。

而是告诉他们:

> 出于对现任雇主的尊重，我不喜欢谈论我目前工作安排的内部情况。

或者干脆直言不讳地说:

> 我不明白我目前的薪水如何影响这些讨论。先看看我们是不是相互适合。"

目标是让他们先出价。这个报价将在他们能给你的价格范围的中间，你可以从那里开始谈判。

## 2。总是在谈判

永远协商。不一定是因为你需要钱，而是因为它代表了你在公司的价值。这是一个尊重的问题，它会影响你在公司的形象。

## 3。当你收到报价时开始谈判

因为你没有给他们一个数字，他们会提供一个报价。这是你开始谈判的地方。在这一点上，他们不太可能拒绝你的谈判。他们已经在你身上投入太多了。慢慢来，也许说你必须和你的配偶讨论一下，或者你需要平静地读一遍。

然后你可以说这个提议“很有意思”，但是还没到可以完成的程度。询问这个数字是否有弹性。他们可能会给你另一个更高的报价，而且不能再高了。这是你可以告诉他们的地方，如果他们能多给几天假期或类似股票期权等，这个提议将会起作用。正如@patio11 在他的文章中所说:

> 你有一个多维偏好集。使用它。

这种策略会让你达到你职位可能达到的最高水平。这会让你感觉良好，并从一开始就巩固你在公司的地位。

* * *

老实说，我认为这是系统中的一个缺陷，你与新雇主的第一次互动就是这种性质的，真的很不舒服。

很少有不同的做法。例如，Basecamp 根据职位和当前的市场形势向每个人支付同样有竞争力的薪水。你应该读一读他们关于这件事的文章，它对这件事的观点令人耳目一新。
