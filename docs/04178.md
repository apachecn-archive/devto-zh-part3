# 探索任何 JS 框架的完美的第一个项目

> 原文:[https://dev . to/ashleemboyer/the-perfect-first-project-for-exploring-any-js-framework-48ip](https://dev.to/ashleemboyer/the-perfect-first-project-for-exploring-any-js-framework-48ip)

完成一个教程最糟糕的部分是不知道下一步该做什么。你*可以*去网上查找一些应用程序的想法，但是你不厌倦制作你可能不会用到的待办事项列表和计算器吗？大概感觉没什么好做的了。

[![Jack Tripper saying, "Well, as a matter of fact, there is something"](../Images/3626908c6b4b0575545e0b47ad6241d1.png)T2】](https://i.giphy.com/media/FeiQbJ2BjZLoHdD5oB/giphy.gif)

简历呢？我花了一整天的时间用 Gatsby 编写我的代码(我真的需要使用字体和图标插件)，然后我做了一个快速的小`ctrl-p` >“保存到 PDF”。对我来说，这样做比在 LibreOffice Draw 中做一些东西要容易得多。这个程序似乎总是扭曲颜色之类的东西。

因此...是什么让一份简历成为完美的项目？

*   只有一页——工作量不算太大。
*   如果你像我一样使用两列，你可以练习用 CSS 布局。
*   你可以练习各种类型的元素:标题、段落、链接和列表。
*   同样，如果您使用两列，您可以练习按照正确的顺序放置元素以便于访问。
*   您可以使用像 [Lighthouse](https://developers.google.com/web/tools/lighthouse/) 这样的工具进行额外的练习，测试可访问性、性能和最佳实践。
*   完成后，你可以把它添加到你的个人/作品集网站上，有一些好看的东西可以炫耀。😉
