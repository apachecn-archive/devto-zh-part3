# 什么是专业软件开发？

> 原文：<https://dev.to/jautero/what-is-professional-software-development-8df>

你听说过乔尔测验吗？这是 12 个**是**或**否**问题的简单列表。不，*等等！不要点那个链接！它已经有将近 20 年的历史了，几乎与现代软件开发没有任何关联。*日常在 *CI/CD* 的世界里是谁打造*？代替*规格*和*进度*的是*积压*和*冲刺*。我有时会考虑为敏捷世界写新版本的乔尔测试 T21。*

我一直在思考这个问题，因为一个多月前，我开始与一个小团队合作，他们的工具和实践让我想起了 21 世纪初人们开发网站时使用 Java 和 PHP 的事实。我不禁要问，是他们不专业，还是我把自己的标准定得太高了？

其中一个问题是 web 造就了许多从未使用过 IDE 或调试器的开发人员。有多少使用 Vim 的人配置了它来让代码导航工作，而不是仅仅从命令行使用 grep？有人知道如何使用调试器吗？你知道实际上有 PHP 的[调试器吗？还有](https://xdebug.org/)[你可以从 Vim](https://mutelight.org/minimal-guide-to-debugging-php-with-xdebug-and-vim) 使用？

我认为你要么在*开发一个项目*中的软件，该项目在*上工作，在*发布*时开发一个规范*，要么你使用 *sprints* 或*看板*实现*产品积压*中的*项*。其他的都太没有计划性了，不够专业。我认为太多的开发人员认为项目计划或产品积压只是经理的簿记，而不是组织日常工作的方法。这有两个后果。首先，它将计划或待办事项从实际工作中分离出来，将它们变成一个虚构的东西，只给人一种控制的错觉。其次，开发人员必须使用其他方式来组织他们的工作。最有可能的是，他们根本没有组织它，结果是“该准备的时候就准备好了”。
