# 我很高兴我并不总是对我的代码满意

> 原文：<https://dev.to/miguelrodoma95/as-a-junior-developer-im-happy-that-im-not-always-happy-with-my-code-1em9>

今年对我来说很神奇，尽管我毕业时是生物医学工程师，但我有机会进入编程的世界，并发现了我作为一名 Android 工程师开发移动应用程序的真正热情，我不得不说...我从来没有想到我会在工作中如此快乐。

作为初级开发人员，我们必须意识到，我们很可能不是办公室里最聪明(或更有能力)的人，这没关系，我认为每个人都会在某个时候经历这种情况。我们必须利用我们作为初学者的“地位”,尽可能多地从其他开发人员那里学习，当然，也从我们自己的错误中学习。利用这段公司认为你是初级/初学者的时间去尝试不同的事情和失败，并利用这个教训来不断改进。

在这篇文章中，我将阐述承认即使在我们最好的工作中，甚至在最简单的实现中，总有一些事情我们可以做得更好，更有效，更可读，或者只是更清晰的架构是多么重要。这种认识将使我们作为开发人员成长得更快更好。请随意评论和/或讨论你如何同意或不同意我写的内容。

## 确保你**理解了**那个教程

作为一名自学成才的开发人员，我不得不通过观看教程和直接编码来学习。看教程可能会有欺骗性。是的，按照教程进行一些实现将有助于你完成工作，但是我们初学者可能犯的一个常见错误(或者至少是我犯的错误)是，只是复制教师编写的代码，而不理解程序实际上是做什么的。确保你理解正在发生的事情，如果你不理解，那就更深入，开始阅读和学习你写的那个方法或类，但是不知道它实际上做了什么。当您需要实现类似的东西时，您将知道如何去做，它是如何工作的，现在您将能够探索其他选项并比较它们以选择一个使您的代码在各个方面都更好的选项。

## 曾经为你过去的工作感到羞耻吗？完美。你比你写那个的时候好多了

你曾经回顾过你过去的工作吗？一些你可能没有完成的过去的项目？有时间读一点你写的代码吗？如果你有，很可能你对此感到羞耻(我有)。对你过去的代码感到糟糕并不是坏事，相反，如果你，一个初级开发人员，能够认识到你的错误，并且你觉得有能力改正它们，那么我有好消息告诉你，你正在进步。重要的是对自己诚实，学会承认你做的事情很糟糕，需要清理，或者还可以，可以做得更好。

## 阅读并学习如何做得更好

既然您已经确定了代码可以改进的地方，那么是时候开始学习了！我们很有可能犯同样的错误，快速实现一个解决方案，只是为了纠正或完成一些功能，这将导致我们在未来讨厌我们的代码，所以我们可以通过检查 Youtube、Stack Overflow、Github 等资源中的正确实现来避免这一点。或者问我们更有经验的同事他们会怎么做，为什么。记住理解我们写的东西是很重要的！

## 传授你所学

尤达大师的至理名言。既然你有能力认识到并改正自己的错误，是时候帮助别人度过难关了。作为初级开发人员，我们并不总是有机会帮助我们的同事，因为当他们需要帮助时，我们不是他们的第一选择。但是当你有机会的时候，尽你所能帮助他们。能够从经验中帮助他人是我们能做的最好的事情之一，它让我们感觉到我们每天都在变得更好，它让我们的同事和经理意识到我们有能力承担更多的责任和挑战。执行、失败和学习的循环将继续下去。

有一种方法可以做得更好——找到它。
——托马斯·A·爱迪生
