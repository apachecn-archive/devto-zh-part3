# 纯函数式语言中的不确定性？

> 原文：<https://dev.to/drbearhands/nondeterminism-in-purely-functional-languages-3lg0>

这是我一直想知道的事情。在纯函数式语言中，非确定性函数有一席之地吗？我所说的非确定性是指[非确定性算法](https://en.wikipedia.org/wiki/Nondeterministic_algorithm)，而不是 [NTMs](https://en.wikipedia.org/wiki/Non-deterministic_Turing_machine) 或[非确定性编程](https://en.wikipedia.org/wiki/Nondeterministic_programming)。对同一输入可能返回不同输出的函数。

传统上，人们会使用 RNG 的种子来实现伪随机函数的不确定性。每次调用函数时，种子都需要增加，因此会有副作用。

但是让我们假设我们的 CPU 有一个量子随机数生成器:它可以量子纠缠并坍缩 2 个 q 比特来生成一个真随机数。据我所知，这将是真正的副作用免费。或者，我们根本不关心 random 函数的底层实现，假装它可以工作。

非确定性函数的使用显然破坏了参照透明性。我们应该关心吗？它确实使某些函数无法记忆，但如果非确定性可以被静态识别，这不会影响任何理论上可以记忆的函数。

对于懒评应该无所谓。随机'选择'是随机的，不考虑顺序，一个真正的随机函数不会关心它何时被调用。

它不应该影响库里-霍华德对应，因为它没有改变类型系统。

它不应该真正影响与范畴理论的同构，因为虽然我们可能不再有许多情况下的值相等，但我们仍然有等价。两个随机数是等价的，因为它们都是随机数。

然而，我不禁觉得我在这里错过了一些重要的东西。我还没能找到任何关于这个主题的学术文献，但话说回来，我也不擅长找学术文献。
