#### CSS&SCS

##### [如何学习 CSS](https://www.smashingmagazine.com/2019/01/how-to-learn-css/)

一个极好的和彻底的关于 CSS 的基本原则的介绍，每一个都有链接到更详细的地方去学习。正如作者所说:学习 CSS 不是为了记忆属性，而是为了理解语言的工作方式。你可以随时查找财产细节，但这会给你一个很好的事情是如何运作的基本理解。

##### [来自战壕的萨斯技术](https://css-tricks.com/sass-techniques-from-the-trenches/)

有很多关于如何像 BEM 一样做 CSS 架构工作的文章，也有关于 Sass 特性的文章，但是我认为这篇文章是我见过的最好的文章，它将两者联系起来，并展示了 Sass 的特性如何帮助您编写出干净的 CSS 代码。

##### [简单 CSS 动画教程](https://medium.com/@js_tut/simple-css-animation-tutorial-c4ad941a5d5c)

我有点希望这能更进一步。第一部分很棒，做了很好的介绍，然后就在最后，他们提出了一些更先进的技术和想法，但没有深入细节。也就是说，理解不同动画属性如何工作的图表是高质量的。

##### [关于设计系统的快速 CSS 审计和一般说明](https://css-tricks.com/a-quick-css-audit-and-general-notes-about-design-systems/)

尽管这篇文章写得很快，但它看起来很长，尤其是当你试图通读 CSS 审计的所有截图时。也就是说，它也给了你一个很好的感觉，当你审计你的 CSS 时，你可能想要寻找的东西的类型，以及让你知道一个非常酷的工具。

##### [前端项目常见 CSS 问题](https://www.smashingmagazine.com/2018/12/common-css-issues-front-end-projects/)

作为前端开发人员，我们必须处理一系列常见的情况，以及在开发过程中可能会遇到的一些小问题。没有什么惊天动地的东西，但是有一些很好的提示，包括如何修复的例子和注释。

#### JavaScript

##### [2019 年 Kickass JavaScript 开发者 9 招](https://dev.to/treyhuffine/9-tricks-for-kickass-javascript-developers-in-2019-in8-temp-slug-5180369)

这里有一些非常好的东西可以提高您的 JavaScript 技能，绝对值得通读。也就是说，我对建议 6 有点谨慎——虽然我*喜欢*可选链接所带来的简洁(我以前在 ruby 中使用过),但要谨慎的是，这仍然是第一阶段的建议，不是 JavaScript 语言的正式部分，在最终确定之前可能会有所变化。

##### [《React 手册》](https://medium.freecodecamp.org/the-react-handbook-b71c27b0a795)

非常长，彻底的资源旨在成为学习反应的“快速通道”。它无疑为您提供了在一个地方高效反应所需的几乎所有信息。如果您刚刚开始使用 React，值得浏览一下。

##### [了解虚拟 DOM](https://bitsofco.de/understanding-the-virtual-dom/)

使用虚拟 DOM 已经成为 JavaScript 框架中事实上的规范，为模板和应用程序提供了极其强大的声明性方法。如果您曾经想知道这到底意味着什么，什么是虚拟 DOM 以及它是如何工作的，这是一个很好的分解。

##### [了解节流和去抖](https://blog.bitsrc.io/understanding-throttling-and-debouncing-973131c1ba07)

当处理 UI 交互、事件和触发 API 调用时，理解如何节流和去反跳是关键。这是一个很好的介绍，既介绍了何时需要使用这些工具，又介绍了如何使用 lodash 工具快速完成这些工作。

##### [与 JavaScript 的数据角力:打印！](http://www.the-data-wrangler.com/data-wrangling-with-javascript-printed/)

这不是超级前端的重点，但我想强调一下，因为它展示了如何将 JavaScript 带入一个我认为很多人为了支持 python 和其他语言而回避的领域:数据争论。我们已经看了很多使用 tensorflow.js 等工具进行机器学习模型的客户端渲染，但 JavaScript 能够处理数据管道中的一切。我通读了这本书的一些早期版本——如果你对数据漫不经心，这是非常值得的。

#### 其他牛逼

##### [2018 员工最爱| CSS 招数](https://css-tricks.com/2018-staff-favorites/)

我很享受这种漫步式的、异想天开的、但也是发人深省的回顾，回顾了这一年的许多重大话题。由 CSS 窍门作者编译，但大部分文章都不是关于 CSS 窍门的。

##### [UI 工程的要素](https://overreacted.io/the-elements-of-ui-engineering/)

一篇很棒的文章，列举了前端 web 开发中不同类型的挑战。如果你曾经努力解释为什么前端如此具有挑战性，这将给你大量的素材。如果你想提高自己的水平，请通读这篇文章，并开始思考如何自己解决这些问题，或者在你选择的框架内。

##### [飞 CDN](https://github.com/superfly/cdn)

用 typescript 编写的可定制的开源 CDN。你为什么想要这样的东西？因为它可以让你编写各种各样的定制逻辑，并将其推送到 CDN 边缘服务器，在那里它可以超级接近你的用户运行。如果你正在拥抱 JAMstack 和 Edge computing 之类的东西，这是你工具箱中的另一个工具，虽然它针对发布它的公司进行了优化，但该项目声称可以在任何支持 edge service worker 的平台上工作，所以你应该能够在 Cloudflare workers、Lamda@Edge 或任何你可能使用的 Edge 平台上运行它。

##### [2018 年的牛逼演示](https://tympanus.net/codrops/2018/12/27/awesome-demos-from-2018/)

来自 codrops 的一组非常有趣的演示，每一个都在一个 codepen 中，所以你可以看到它是如何完成的，如果它让你觉得有趣，你可以使用它。不过要小心——如果你点击迷幻波演示，你可能会发现自己被困的时间比你想象的要长，因为你的大脑会爆炸。或者可能只是我。:P

##### [JavaScript 的熔炉](https://increment.com/development/the-melting-pot-of-javascript/)

对当今围绕 JavaScript 的生态系统和文化的深刻思考。是的，它是以 JS 为中心的，但我把它放在“其他令人敬畏的”部分，因为它更多地是关于人、文化和生态系统，而不是语言或任何特定的工具。
