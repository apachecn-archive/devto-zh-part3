# 培养人才，而不是技术

> 原文：<https://dev.to/guifroes/build-people-not-tech-31kk>

回顾、构建-测量-学习和持续改进都是公司及其敏捷团队为支持项目交付和产品开发而进行的活动和实践。

我认为这是错误的。

这些东西太重要了，不能仅仅用来支持业务。

不久前，企业使用技术来加速、自动化、以某种方式改善自己。团队将会形成，技术产品将会被制造出来，企业将会继续它的生活。

现在不同了。**业务就是技术。**企业的生存依赖于技术的持续生产和运营。

正因为如此，公司不应该把重点放在生产技术上，而应该放在不断培养人的能力上。

回顾、构建-测量-学习和持续改进必须放弃支持活动的位置，占据业务的关注中心。团队和他们的学习能力——而不是他们的产品——是企业的竞争优势。这应该反映在投资、招聘和晋升上。

科技公司必须转型为知识技能型公司。
