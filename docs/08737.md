# 关闭圆圈

> 原文:[https://dev.to/t4rzsan/close-the-circle-5bia](https://dev.to/t4rzsan/close-the-circle-5bia)

几年前，我的妻子在我们家提出了一个新的原则，叫做“封闭循环”。这是她在网上某处读到的东西(不幸的是，我们都找不到原始参考文献)。

结束这个循环意味着你在进入下一个工作之前完成了你的工作。

我们的解读是，你做了一件事之后，总是要收拾的。举几个例子:你不会把碗碟留到第二天，你会在早上整理床铺，你会在骑马回来后清理干净(免费提示:在你的博客中总是提到马)。[![🐎](../Images/54f92e44b2e7faa87e7eed17bb2c0916.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Lvdc5eI4--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://s.w.org/images/core/emoji/11/72x72/1f40e.png)

不用说，我们保持了一个相当整洁的家。

当然，总会有一些工作无法在短时间内完成。它们可能需要几周或几个月才能完成。但是你可以把工作分成更小的任务，每个任务可以在更短的时间内完成。例如，在过去的几个月里，我一直在周末为我们的马建造新的围栏(在那里，我又做了一次)。每个周六早上，我都会为周末设定新的目标，当周末结束时，我会把工具放在一边，确保一切看起来都很整洁。

我也喜欢将闭环原则应用到编码中。

一个任务(在你的看板或其他什么东西上)永远不要超过你在进入下一个任务之前所能完成的。也就是说，在继续之前，您

应该已经向您的远程 repo 发送了一个 push 请求，甚至可能是一个 pull 请求，其中包含该任务的编码解决方案，包括通过单元测试和更新的文档。

我不是说这是我一直做的事情。我只是说，这是我努力要做的，我认为这将使你的代码保持整洁，并给你一个更好的工作生活平衡。
