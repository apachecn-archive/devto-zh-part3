# 重新认识威亚姆

> 原文:[https://dev . to/ryanrousseau/getting-re-acquidated-with-wyam-52gl](https://dev.to/ryanrousseau/getting-reacquainted-with-wyam-52gl)

我参加了。网拆箱回到 2015 年。在那里，我遇到了戴夫·格里克，他为他的项目 [Wyam](https://wyam.io/) 做了一个简短的演讲。

Wyam 是一个由 Roslyn 驱动的静态*内容*生成器。静态站点是最常见的情况，但是 Wyam 可以使用模块生成其他类型的内容。

那次闪电谈话是我第一次介绍静态站点生成器。我立刻被这个想法吸引住了。从那以后，我一直关注着 Wyam，并对它进行了一些测试，但是直到今天我还没有发布过任何使用它的站点。

对我来说，两个新特性是[食谱和主题](https://wyam.io/recipes/)。您可以使用配方来减少生成站点所需的配置和输入。blog recipe 将生成一个包含存档、标签和提要的完整博客。您可以将菜谱与主题相结合，以自定义布局和风格。这些命令将使用默认主题生成一个空博客，然后在本地提供服务。

```
> wyam -r Blog
> wyam preview 
```

[![Blank blog generated by Wyam](../Images/d759805ad325d15e68901ef082429db9.png "Blank blog generated by Wyam")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--QZigd3iT--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/wgixdkwtwnvxc04awm6q.png)

这是一个很好的起点。我通读文档，寻找如何添加帖子。blog recipe 在输入文件夹下的 posts 文件夹中查找文件。它获取这些文件并运行它们来生成 post html 文件。我创建了 posts 文件夹，并添加了 getting-re-acquidated-with-wyam . MD 和一些占位符内容。

```
Title: Getting Reacquainted with Wyam
Lead: Creating a blog with Wyam in 2019
Published: 3/2/2019
Tags:
  - wyam
---

Content goes here. 
```

我喜欢恒星模板的外观，所以我在重建时使用了它。

[![Blog with post generated by Wyam](../Images/6c3a284472a6998c39ca3fb55cd20e53.png "Blog with post generated by Wyam")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--q5kqVjQ7--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ejo0970wd9paskzbwd8t.png)

看起来还不错，但是我想换个背景图。您可以通过将新版本放入输入文件夹来覆盖模板使用的文件。我从 [GitHub repo](https://github.com/Wyamio/Wyam/tree/develop/themes/Blog) 中抓取了一份主 CSS 文件，放在 input/assets/css 中，并进行更新。

图片来自于 Wyam 文档中的一个资源 [Unsplash](https://unsplash.com/) 。我需要归功于摄影师，所以我复制了页脚模板，并添加了学分。

[![Blog with footer updates" title="Blog with footer updates](../Images/2afe5049963e5b764a32053a83981f4f.png "Blog with footer updates" title="Blog with footer updates")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--WeVVwWND--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/xsxre0mbf0b6e0ewzz69.png)

在图片的映衬下，文字的可读性不是很好，但这是我以后可以改进的地方。初始版本的最后一个变化是站点的标题和描述。

这些可以通过 config.wyam 文件进行设置。

```
Settings["Title"] = "blog.rousseau.dev";
Settings["Description"] = "Ryan Rousseau";
Settings["Intro"] = "Tech, Comedy, Cats"; 
```

我还修改了 navbar 布局，在第一个链接中使用“Home”而不是博客标题。

[![Blog with all updates](../Images/2f36de3ccaf57bc1bea86c82474a8c4c.png "Blog with all updates")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--UciDGVyA--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/mhuuecrcjo2hxvsvuyum.png)

我可能会添加一个评论系统，但这最好是留给另一天。总的来说，我做了以下的改变来建立并运行一个博客网站。

*   安装 Wyam
*   自定义一个 CSS 文件和两种布局
*   拍几张截图
*   在 Markdown 写一篇文章
*   使用菜谱和主题建立网站
*   部署到我的主机

我在 [Firebase](https://firebase.google.com/) 上主持。我已经有几个项目托管在那里，所以这是一个很容易的选择。我现在手动部署。我将在以后的文章中介绍自动部署 Firebase。
