# 柔术如何让我成为更好的工程师

> 原文:[https://dev . to/alexkondov/how-JIU-jitsu-make-me-a-better-engineer-1 pai](https://dev.to/alexkondov/how-jiu-jitsu-made-me-a-better-engineer-1pai)

每年年初，我们开始回顾前一年我们完成了什么，还有什么没有完成。我们回顾我们设法实现的决议和我们在第二周放弃的决议。

去年 1 月，我坐下来思考了一下，就我的职业生涯而言，我想在 2018 年完成什么。我首先想到的是“嗯，我想成为一名更好的开发人员！”。当然，我怀疑从事软件工程的人不想提高自己的技术。

但是成为一个更好的软件开发者到底意味着什么呢？为了更上一层楼，一个人需要具备什么样的品质和技能。任何高级工程师都具备对基础知识的深刻理解。同时，跟上新技术、趋势和建议也很重要。换句话说，学习应该是重中之重。

然而，学习而不实践只能让你到此为止，对不对？我们都知道，在业余时间做点什么或者解决一个具有挑战性的问题是提高的最好方法之一。这将带你进入未知领域，推动你的创新和创造力。

好了，我们对开发人员如何在他们的工作中进步有了一个大致的概念。但是当我们把编码放在一边时，仍然有许多活动据说对我们的改进有益。教学、演讲、写博客等。你从哪里开始？

## [](#so-what-makes-us-better)那么，是什么让我们变得更好呢？

今年早些时候，我看到了一篇令人惊叹的文章，主题是如何提高软件工程师的素质。更准确地说——工程师如何停止改进。本质是，一旦我们找到了一种对我们有用的方法，并且得到了我们想要的结果，我们就会倾向于坚持下去，直到我们达到一个稳定的状态。

既然我们对这种方法如此投入和自信，那么就很难改变我们的思维和习惯。作者强调了这对陷入陷阱的组织和程序员的职业生涯的影响。

这篇文章让我反思了我在职业生涯甚至爱好中所做的决定。找到一种让我们表现良好的方法并停止寻找其他替代方法是非常容易的。

## [](#jiujitsu-and-programming)柔术与编程

当我读到那篇文章时，我的思绪立刻回到了我生命中的前一刻，起初这与编程毫无关系。我完全理解作者的意思，因为我以前也陷入过这个陷阱。以前我训练巴西柔术，有点像摔跤，但目标是让你屈服。

我爱上了这项运动，享受着令人疲惫、有时甚至是痛苦的训练过程中的每一刻。然而，在我的柔术之旅的早期，当我与强壮的家伙对打时，我很挣扎，因为他们可以用他们的力量打破我的提交。

这是令人沮丧的，因为即使我正确地应用了我的技术和知识，有些人也能靠蛮力找到出路。腿锁被证明是解决我的问题的有效方法，因为它们是如此的快速和出乎意料。对于脚踝来说，力量也没那么重要。

虽然有时钻不同的位置和逃生看起来很平常，但腿锁很酷，有时很花哨。我一直在偷偷摸摸地提交文件，与更大的对手战斗不再是一场斗争。

我不知道如此关注这些技术对我的发展弊大于利。

但是一旦你有了锤子，所有的东西看起来都像钉子。当我理解了腿锁的有效性，我开始忽视其他提交或位置的机会，只要有机会就抓住脚踝。我用这种方法相当成功，并在我的游戏中获得了很多信心。

然而，把你所有的注意力都集中在一个技能上而忽略了其他的，就像把你的龙与地下城角色的所有分数都放在力量上而没有放在智力上。

当我开始与更好的知道如何防御腿部攻击的柔术练习者对练时，我所有的计划都崩溃了。因为我唯一的注意力是攻击脚踝，如果我没有得到它，我获胜的机会就会大大下降。

换句话说，我在面对更好的人时表现不好。我意识到我忽略了训练中的其他基本要素，但我已经深深陷入了我的舒适区，在练习我迫切需要提高自己的基本技术时感到不舒服。

## [](#dont-make-the-same-mistake)不要犯同样的错误

读了我上面提到的文章，意识到我过去所犯的错误，我知道我必须改变我对进步的态度。虽然学习下一个新奇的技术肯定会给我带来短期利益和成就感，但从长远来看，回去填补空白对我来说更好。

很酷的新 JavaScript 框架和库是我编程生涯的绊脚石。我又一次变得过于专注于学习新奇的新技术，并保持在潮流的前沿。我知道每一个新的工具和库，但是在开发人员需要的核心知识方面有很大的差距。

此时，我已经获得了作为一名软件工程师的自信。我觉得我处理问题的方式是正确的，至少可以说，改变我的心态对我没有吸引力。毕竟，我所知道的起作用了。如果它没坏就不要修好它。

另一种解释这种心态的方式是通过[爱因斯坦效应](https://en.wikipedia.org/wiki/Einstellung_effect)。这是指我们有时倾向于以特定的方式解决问题，即使有替代的和可能更好的解决方案。

## [](#feeling-confused-is-okay)感觉迷茫没事

在编程社区中，我崇拜的一个人是 MPJ——Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun Fun 在他的视频中所有有价值的教训中，有一个最闪光，那就是感到困惑是正常的。

当处理一个具有挑战性的问题时，尴尬的感觉实际上是我们应该追求的。这意味着我们已经走出了安全港，也就是我们的舒适区，我们实际上正在学习。

当我们长时间使用一种特定的方法或技术时，我们会对它和它的功能感到舒适。我们往往会忘记第一次尝试使用它时的尴尬和困惑。

这使得向新事物的过渡比以前更加困难。如果我们长期尝试新事物，我们的自我会尽力保护我们。

## [](#looking-for-comfort-in-uncomfortable-situations)在不舒服的情况下寻找安慰

我从柔术中学到的最有价值的一课是不要避免不舒服和尴尬的情况。不管怎样，你将处于不利的地位，你将不得不寻找出路。

也许你可以避免一段时间，但总有一天它会发生，这是必然的。让自己为这样的时刻做好准备的唯一方法就是有意识地将自己暴露在它们面前。

虽然这听起来像典型的“走出舒适区”的建议，但我发现有不同的方式来看待它。如果一种特定的语言是你的避风港，你仍然可以通过给自己施加一些约束来让自己放松下来。

你可以禁止自己使用这种语言的某些特征，试着想出解决问题的替代方法。也许你决定不使用类。也许你可以修改所有的函数，或者限制它们的长度。

一种完全不同的方法是给自己更多的选择，而不是限制它们。对我来说，这是将 TypeScript 引入我的一个副业项目。当我还在使用 JavaScript 时，静态类型将我习惯的编程引入了一个全新的层次，我从中学到了很多。

作为一个每天与 React 和 Redux 一起工作的人，我在这个生态系统中感觉很舒服。然而，在我的 Redux 流程中引入类型安全动作，并从 Sagas 和 Thunks 切换到 Observables，让我学会了一种包装正常编程流程的新方法。

## [](#summary)总结

问题是学习新的东西既困难又不舒服。当我们转换环境并尝试使用其他工具时，有时我们会在基本操作上遇到困难。当我在用 Observables 寻找我的方法时，我总是有这样的想法:我已经知道了一种有效的方法。为什么要纠结这个？每次我想到回到我所知道的地方，我就会想起我是如何停滞不前的，因为我没有把目光放得比过去更远。

作为软件开发人员，我们有机会在一个永无止境的学习领域工作。我们不应该为了保护自我而剥夺这份礼物。
