# 没有全栈开发者这种东西

> 原文：<https://dev.to/tevko/theres-no-such-thing-as-a-full-stack-developer-1m3i>

**如果你是一个拥有完整头衔的多面手，这篇文章不是攻击你对团队的宝贵贡献。

### TLDR:直接跳到我猜测的结论

## [T1】简介](#intro)

这个术语已经流行了很长一段时间了，它是有意义的，因为在理论上它听起来是一个好主意。作为从事网络相关技术工作的人，你希望被视为精通所有可能的领域，从而增加你的可雇佣性。作为一个领导工程团队的人，你希望能够雇佣能够对你的代码库的每个领域做出无缝贡献的人。虽然听起来像一个伟大的想法，这个概念是站不住脚的几个原因。

### 每一叠都不一样

没有一套特定的技术用于创建端到端的 web 应用程序。有无数的编程语言、服务器框架、数据库平台、预处理程序和包管理器可供选择，所有这些都是为了解决特定的问题，这些问题根据您要构建的应用程序的类型以及负责维护应用程序的团队的经验而有所不同。最好的情况是，作为一个全栈开发人员，只能意味着您能够熟练地为您过去使用过的特定栈的每个级别做出贡献。任何招聘经理寻找一个全栈开发人员实际上只是在寻找一个使用过招聘公司目前用来交付 web 应用程序的每一项技术的人。同时，他们忽略了那些关注其他技术和实践的人，这些技术和实践可能会极大地提高他们当前应用程序的可用性、功能性和能力。

### 专业化很重要

即使在一个自称全栈开发人员的地方，也有一些技术我们会比其他人更适应，反之亦然。一个拥有 CS 学位的开发人员有时可能比一个自学成才的开发人员对 GO、Rust 和 Node 更感兴趣，这个自学成才的开发人员为了改变他们的 WordPress 博客的背景颜色而过渡到学习 CSS 的领域。虽然这些差异绝对没有错(因为两者都服务于必要的目的)，但期望两个开发人员以同等的熟练程度为代码库的相同部分做出贡献是完全不公平和不负责任的。即使是那些拥有全套头衔的人，在加入工程团队后也倾向于他们最熟悉的技术。关注全栈开发人员往往会忽略这些差异，这反过来会对应用程序的健康产生负面影响。

### 满栈不利于你的应用

一个全栈开发人员的职位发布倾向于吸引特定类型的候选人。通常，这是一个熟悉高度编程语言，同时对 CSS、HTML 语义、可访问性和前端性能有一点经验的人。在 Javascript 逐渐成为主流编程语言的环境中，这是有意义的，新开发人员都是从学习 Javascript 开始的。因此，web 开发的其他同样重要的方面退居次要地位，并由大量旨在使这些不太受关注的领域更容易管理的框架和工具来补充。在一个精通 Javascript 可以让你赚很多钱的世界里，在你整个 web 开发生涯中只专注于 Javascript 有时可能是一个好的决定。这种模式导致了一些不幸的趋势，这些趋势可以在大量的 web 应用程序中找到。

由于对全栈开发人员的关注，出现了最大的(没有双关语)问题是性能问题。web 应用程序不必要地在第一页加载时就向用户交付成百上千字节的图像、CSS 和 Javascript，这种情况已经司空见惯。大多数情况下，这些性能问题仅仅是由于对 web 开发的这些领域不感兴趣的开发人员缺乏对前端性能的关注。作为一名招聘经理，你的团队中有不关注前端表现的开发人员并没有错。然而，如果你的团队中没有人关注它，那就有问题了。

另一个没有得到应有关注的问题是可访问性。让一个应用程序对残疾用户来说是可访问的，从 HTML 层开始，这是大多数进入招聘市场的全栈开发人员高度忽视的领域(这不是开发人员的错，它经常被学校和编码训练营等忽视)。如果您领导一个工程团队，让您的应用程序为最大数量的用户服务是一个显而易见且有利可图的策略，如果团队中没有开发人员关注可访问性，这是不可能实现的。

为了简洁起见，我要谈的最后一个问题是前端技术中的新兴特性和功能。服务人员、响应图像、新的 CSS 布局模块、推送通知和后台同步功能都是前端 web 今天能够处理的功能，但大多数全栈开发人员几乎闻所未闻，因为它们需要在全栈 web 开发人员招聘中提到的传统技能之外的特殊关注水平。同样，不熟悉这些技术也没什么不好。当管理应用程序的团队中挤满了不擅长这些新兴领域的全栈开发人员时，有理由认为应用程序会输给竞争对手。

### 满仓不利于你的招聘渠道

我参加过几次面试，技术评估只包括一些算法挑战和 Javascript 框架特定的任务(比如用 React 实现一个可排序的表)。最近，这些是我唯一接受的技术评估。它强调全栈驱动的思维模式，这种模式强调后端功能而不是用户界面开发。如前所述，这会对应用程序的整体健康状况产生负面影响。招聘应该更加全面，关注个人的优势和劣势，同时考察候选人对团队做出贡献的整体潜力。

### 结论

《金钱球》是一部好电影。这给我们上了宝贵的一课，它不仅仅适用于棒球:个人贡献者带来了不同的优势和劣势，最终形成了一个胜过所有 MVP 的团队。成为一个完整的 web 开发人员在理论上听起来很棒，但实际上，这通常意味着团队缺乏应用程序设计和开发特定领域的知识。专门化和平衡可以解决这个问题，并改善应用程序的整体健康状况。招聘经理、团队领导、编码训练营和个人贡献者应该放弃全栈的想法，因为栈非常大，非常复杂——当你雇佣一个 CSS 专家和一个 Kubernetes 老手时，你可能会做出一个更好的应用，而不是两个还不知道你的栈是什么的全栈开发者。
