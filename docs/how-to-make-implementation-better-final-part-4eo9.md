# ***总之第 1 部分***

在第一部分，我分享了我的编码之旅，如何开始，如何使代码更有效。并讨论了一些基本的关键概念，如程序、程序员、指令、代码、语法等。什么是标准定义，这些关键概念的真实含义是什么，或者你可以说区分标准定义和真实含义。

现在让我们来到现在，讨论一些更有一些重要作用的事情。但是在进入主题之前，我有一个你必须问自己的问题清单。这些问题不是基于语言或特定领域的。你不需要在任何地方发帖，所以不要担心，只要回答这个问题就可以了。

***   Are you ready to learn programming/coding?*   If you are a beginner, which language would you choose and why?*   Why learn programming/coding? Do you want to learn because it sounds cool or are you really interested? How does your code help our daily life? Can you relate what you have learned to your daily life?*   When you encounter a programming problem, how will you deal with it? How did you overcome this problem? What are some everyday examples, such as computer software or application names written in this language? How are you going to use it to your advantage?**

诸如此类的问题你必须问自己，这些基本的关键概念将帮助你成为一名优秀的程序员或使你的实现更好。一般来说，当我们得到一个问题或一个想法时，我们会直接开始思考问题的解决方案，比如我们讨论 C 语言中的 2 个数字相加的问题或与设计相关的问题时，我们会直接开始思考在 C 语言中我们将使用多少变量，或者如果我们讨论设计，我们会直接开始设计，而不用制作蓝图或测试设计。这是初学者通常会犯的最大的错误，这就是为什么他们不能有效地实现代码，而计算机会给他们一个额外的结果，即错误或者输出一个糟糕的设计。这也造成了混乱，有时由于混乱，他们不能写完整的代码，也就是说，在几个步骤之后，他们开始思考接下来的步骤，这将导致恼怒。鉴于此，从我的角度来看，这是一个糟糕的选择或方法。尽管如此，你们大多数人认为为什么这个方法是错误的？如果这种方法是错误的，那么什么是正确的方法，并告诉我们这种方法的缺点是什么？首先，我们来看看这种方法有什么缺点。

***缺点*** :

1.  当我们用这种方式直接在计算机上写代码时，我们一次做两件事。首先，我们考虑逻辑或设计，同时我们也实现逻辑，这造成了混乱，有时这是一个断点，我们不知道下一步会是什么。
2.  如果你认为这种方法能让你成为一名优秀的程序员，那你就错了。因为第一，它使你的调试技能变弱，当你得到一个错误时，你不知道错误在哪里，如何修复它。
3.  在许多公司的面试中，他们不会给你任何笔记本电脑或计算机，他们会给你纸和笔，让你在纸上写代码或设计。所以现在这样你不能更好地执行，犯一些愚蠢的错误，因此有可能被淘汰。
4.  当你在纸上写代码时，你总是会有困惑。即缺乏信心。

***现在来了你要做什么或者你要如何进行*** :

因此，如果遵循这种方法，这些是你可能面临的一些最重要的缺点。当你遇到问题时你要做的是。

**DOs** :

*   当你遇到任何问题时，你必须阅读第一个问题两遍，以明确概念，然后开始为该问题建立逻辑，你必须如何进行，然后以你自己的方式在纸上写下逻辑，即以你自己的伪代码，然后键入逻辑并运行你的程序。

如果你是一个初学者，不能写或不能理解伪代码，那么我有另一种方法或解决方案。

在这种情况下，我会建议你用自己的方式编写代码，或者自由地进行设计，只是想一些日常生活中的例子。但是同时尝试理解或尝试学习如何编写标准伪代码。我建议你使用伪代码，因为通过这种方式，你可以尝试构建逻辑，因为通过这种方式，你不必编写完整的代码。只需几个步骤，你就能给出整个想法，这将有助于你未来的工作，比如你将为任何组织工作，或者参加网上竞赛。
