# 每日脚本——一个简单的练习来建立你的创造技能

> 原文：<https://dev.to/kev_mcg/daily-scripts-a-simple-exercise-to-build-your-creative-skills-56hm>

吉姆是我们家的老朋友，他是个修补匠。他利用自己无穷无尽的好奇心和智慧，收集旧收音机和坏电话，把它们拆开，看看是否能修好。或者训练他的狗做更细致、更有挑战性的动作。或者在他的院子里种樱桃树。我们一会儿再来讨论吉姆。

《艺术家之路》是一本关于创造力的经典书籍，作者是朱莉娅·卡梅隆。书中的一个练习已经成为作家和艺术家的标准练习:每日一页。这个想法很简单。每天，你醒来写三页纸。规则是你必须坐下来，开始写，直到写满三页才停下来。这个练习启发了我最喜欢的网站之一，750words.com 的。

就像吉姆的《好奇的流浪》一样，我认为这种练习是对头脑的一种修补。修补意味着你的探索有些漫无目的，但不是无用的。你通常在整理和组织事情，但是你可能对你正在做的事情没有一个具体的计划或者宏伟的目标。对于一个作家来说，这种流浪有很多好处。当你不想动它的时候，它让你的笔动起来，它建立了持续进步的日常习惯，它揭示了隐藏的思想和联系，如果没有练习你可能不会发现。

正如我的绘画老师会说的，画笔移动的次数应该比它不动的次数多。不停写的规则会让你对你写的东西产生怀疑。停下来无所事事地思考，对作家或画家没有任何好处，只会阻碍创作。

作为编程练习，这可能看起来违反直觉。编程*就是*思维。这是缓慢而仔细的思考，翻译成计算机可以理解的指令。一个程序员当然需要思考！我们花在编程上的时间太多了，无论是作为一名受雇的开发人员，还是作为一名学生或业余爱好者，都花在了一个特定的项目上。当你坐下来的时候，你心里已经有一个目标了。但是我不认为这是编写代码的唯一方式。代码也可以用来随意修补。我们可以写每日脚本，而不是每日页面，只要对练习稍加调整，我们就可以获得与作者在每日页面中发现的相同的好处。

试试这个。在你开始一天的“工作”之前，写一个脚本。你可以在你的文本编辑器中，在像[glitch.com](//www.glitch.com)或 [codepen.io](//www.codepen.io) 这样的网站上，或者在你的语言的命令行界面中这样做。不要设置要写的代码行数，也不要设置一旦开始就不停止输入的规则，而是将时间限制在 20 分钟。一开始，没有目标地写代码可能会感觉很奇怪。但是花二十分钟修补代码。你可能会探索一种你已经使用过的语言特性，但并不完全熟悉。您可能最终会编写一个脚本来自动化您每天都要做的手动任务。你可能最终会发现你在“真正的工作”中正在解决的问题的解决方案你可能会发现一个很酷的模式，以后可能会有用。你可能最终得不到任何有用的东西。或者你可能会写下一个新的非常有用的开源库或有趣的项目的前 50 行，你已经安静地思考了几个月，但从来没有机会探索和认识。

一天 20 分钟的修修补补似乎根本不算什么。但是就像刷牙一样，日积月累，这个日常习惯会有很大的好处。就像写作一样，编程可以是一种探索和发现表面之下尚未表达的想法的方式。当你试图完成真正的工作时，你的发现甚至可能比你写的任何东西都更有价值。或者至少，如果你在工作中感到筋疲力尽，它可以让你重新认识到为什么代码是有趣的，或者如果你完全不知道从哪里开始，它可以让你有一个新的想法。
