# [](#where-to-start)从哪里开始？

随便挑个东西。但是如果你想做 web 开发，你最终将不得不与 HTML、CSS 和 JavaScript 纠缠在一起。如果听到前端、后端或全栈之类的东西让你焦虑，我建议从前端开发开始。前端是一个很好的起点，你可以从制作静态网站开始。所以一开始没有复杂的 JavaScript 或任何东西。只需担心构建结构、定制布局以及外观。

### [](#ides-repls-amp-text-editors)IDEs，REPLs &文字编辑

要编写代码，你需要某种交互式开发环境或 IDE，或者像记事本这样的文本编辑器。但是我建议你使用代码编辑器。

REPL 代表读取、评估、打印、循环，还有一些在线的简洁工具，可定制用于您正在使用的任何环境。如果您需要测试一个函数或代码片段，请使用它们。

### [](#html-amp-css)HTML & CSS

所以我们从 HTML 和 CSS 开始。如果你看到类似 HTML5 的东西，不要惊慌，那只是 HTML 或*超文本标记语言*的最新版本。CSS 代表*层叠样式表*。你会听到一些关于 css 框架和其他东西的事情，但是首先要学会足够简单的 CSS 来对页面进行基本的修饰和布局更改。我**强烈**建议你在使用 CSS 框架或库做任何除了表面上的改变(可能会发生不可预见的副作用)之前，先学习以下内容:

*   [位置](https://developer.mozilla.org/en-US/docs/Learn/CSS/CSS_layout/Positioning)
*   [z 轴索引](https://developer.mozilla.org/en-US/docs/Web/CSS/z-index)
*   [显示](https://www.w3schools.com/css/css_display_visibility.asp)
*   [柔性盒](https://www.w3schools.com/css/css3_flexbox.asp)
    *   [深度指南](https://css-tricks.com/snippets/css/a-guide-to-flexbox/)
*   [css 网格](https://www.w3schools.com/css/css_grid.asp)
    *   [深度网格指南](https://css-tricks.com/snippets/css/complete-guide-grid/)
