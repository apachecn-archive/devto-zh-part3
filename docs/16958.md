# 2018 年我捡到的 Dev 原则

我想通过强调我在 2018 年的工作流程中学习并严格采用的几个开发原则来结束这篇文章。

*   没有测试的代码**和没有代码的**一样好****
*   **不**抽象 **>** **错误**抽象
*   你提出的大多数优化策略要么导致**过早优化**要么导致**过度优化**。**仅在绝对必要时进行优化**。
*   对问题/解决方案有一个更广阔的视野，但是要把它分解成**个小部分**递增
*   如果你添加了一个 todo 来改进，你将永远不会再回到它。所以，**现在就做**。
