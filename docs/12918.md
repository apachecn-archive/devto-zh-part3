## [](#what-the-internet-is)互联网是什么

互联网让网络世界成为可能。从本质上来说，互联网只是一个巨大的计算机网络，相互通信。真的就这些了。互联网是连接计算机并确保它们保持连接的一种方式。

## [](#how-the-internet-works)互联网如何运作

有许多方法可以连接计算机，使它们相互通信。您可以使用蓝牙、Wi-Fi，甚至以太网线。这将在那些计算机之间创建一个网络。最好的部分是，你可以添加更多的连接到这个网络。您可以添加任意数量的计算机。使计算机易于识别的是它们的 IP 地址。

IP 地址是计算机的唯一标识符。每当您的电脑连接到 Wi-Fi 或任何其他类型的网络时，它都会被分配一个 IP 地址。这就是为什么你能在网上找到任何网站。网站所在的服务器有自己的 IP 地址。我们通常不需要记住 IP 地址，但是我们知道我们想要连接的网站或计算机的域名。

一旦有了你想去的域名，就可以使用 HTTP 协议通过互联网发送请求。这个协议允许浏览器和服务器在互联网上相互通信。当有人键入访问您网站的 URL 时，他们会向服务器发送一个 HTTP 请求，以加载他们需要查看网站的文件。

HTTP 请求是客户端和服务器之间的临时连接。在服务器响应了客户端的请求后，连接被关闭。每次发出新的 HTTP 请求时，都必须建立新的连接。这有助于防止安全风险，例如错误的人访问通过持续连接发送的信息。

## [](#below-the-surface)在表面之下

现在您需要知道当服务器接收到 HTTP 请求时会发生什么。下一层是 TCP 层。这就是 HTTP 请求被发送到服务器上正确位置的方式。TCP 层接收 HTTP 请求，将其分成不同的块，并向每个块添加自己的信息。

然后，根据 TCP 层中添加的信息，将数据块发送到服务器上正确的端口号。端口号用于将数据路由到计算机的正确部分。这就是为什么你可以在谷歌上查找资料的同时阅读邮件。这两个应用程序使用计算机上不同的端口。

您通常会听到 TCP/IP，因为这两者是相辅相成的。IP 地址只是一个地址。它不关心数据是如何处理的，但 TCP 关心。TCP 确保 HTTP 请求中的所有数据块以正确的顺序被接收，并被发送到正确的位置。

拥有所有这些不同的层和协议可能看起来有些多余，但它们是我们所知的互联网的主干。如果没有这些，互联网就不会那么容易使用。数据将在错误的时间被接收，这将导致错误的 HTTP 请求和许多 404 错误。我们甚至可能需要记住 IP 地址，它们看起来都像这样:109.435.424.42 当然我们可以这样做，但这会限制在线获取的知识量。

作为一名 web 开发人员，你可能永远不会深入到 HTTP 请求。TCP/IP 之类的东西和所有的网络都不在 web 开发人员的范围之内。这是最好了解的事情之一，以防问题真的到了那个程度。Web 开发人员需要理解的不仅仅是代码，即使他们不是其他方面的专家。它有助于了解幕后的细节。
