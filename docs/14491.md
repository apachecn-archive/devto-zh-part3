# 扩展贪吃蛇游戏以包括全球记分牌——第 1 部分

> 原文:[https://dev . to/chillsunfire/expanding-the-snake-game-include-a-global-score board-part-1-20dh](https://dev.to/chillsunfire/expanding-the-snake-game-to-include-a-global-scoreboard---part-1-20dh)

正如你可能已经在我的帖子[喂蛇游戏](https://dev.to/chillsunfire/feed-the-snake-game-5hlo)中看到的，我正试图通过改进我从教程中构建的东西来学习更多关于 web dev 的知识。

这款游戏使用浏览器的本地存储来记录用户的高分。但是试图打破自己的分数并不像试图打破别人的高分那样有趣。

现在，我必须承认，由于我想从一开始就专注于安全的编码实践和学习如何最好地构建安全的应用程序，这感觉真的令人生畏，我已经推迟了一段时间。但是正如我被提醒的那样，最好的学习方法是进去，做，失败，然后找出正确的方法...

因此，虽然我知道这个版本还不安全，但我花了几天时间才获得分数(用 JS 添加到页面中)并将其与名称(用户在游戏结束时输入)结合，然后通过 PHP 将所有内容发送到我的小 db。

虽然仅仅是让它工作起来就让人感觉很了不起，但真正让我震惊的是，我已经忘记了 web 开发的一些核心原则，即记住处理不同文件的顺序，以生成浏览器中显示的网站。我不得不离开家，在公共汽车站冻得耳朵发凉，记住一旦 PHP 处理完，它就消失了，如果我想在页面加载后获取 JS 生成的东西并发送给 PHP，它必须通过 AJAX 调用。操作顺序。

正如我所说的，我知道它可能容易受到诸如篡改甚至 XSS 之类的攻击，但由于我还不太明白这些攻击是如何工作的，我不能很好地编写代码来防止它。我考虑过各种方法来处理接受用户输入可能产生的问题，但在我看来，我首先需要确保我的数据进入数据库，然后将最高分显示给每个人看。因为如果我做不到那么多，那么我就无法测试其他方法的安全性，对吗？

我还没准备好让你帮我戳它，所以我不会分享链接。但是如果你有任何可以帮助我学习更安全地编写应用程序的技巧、建议和指导，请分享。谢了。
