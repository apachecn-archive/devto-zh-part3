# 基数——为什么我们是可扩展的？

> 原文:[https://dev.to/radixdlt/radix-why-are-we-scalable-3nng](https://dev.to/radixdlt/radix-why-are-we-scalable-3nng)

# [](#inventing-tempo)发明节奏

为了实现速度、可伸缩性和效率，Radix 创建了一个名为 Tempo 的组合分布式数据库架构和共识算法。它是 Radix 平台的核心。它使用逻辑时钟生成事件的部分顺序，并使用向量时钟根据分布式系统中的因果关系创建事件的绝对顺序，以检测和防止违反协议。

# [](#scaling-distributed-ledgers)缩放分布式总账

为了解决可扩展性问题，Radix 从这种规模所需的数据架构开始。我们意识到，要服务于这个世界，你需要一个可以让数十亿人甚至更多设备同时使用的数据架构。这将需要难以置信地容易索引(使数据结构可用)，并且[大规模分片](https://www.radixdlt.com/post/what-is-sharding)(切成片段)，以便随着网络需求的增长，可以添加更多的计算机来增加网络的吞吐量。

我们意识到临时分割数据是行不通的。相反，你需要知道数据将如何被分割。您应该能够查看任何一段数据，并知道它在数据结构中的位置，而不必在每次添加大量新数据时重新索引。基数的这个特性是通过一个确定性的过程实现的(这个过程给定一组输入，无论何时计算，总是输出相同的答案)。我们通过[将平台的数据结构预分割成 18.4 万亿个碎片(2^64)](https://www.radixdlt.com/post/sharding-in-radix) 来实现这一点。然后，我们使用键引用字段，比如一个公钥，作为一种确定特定数据片段在碎片空间中的位置的方法。

这种创新的结果是一种数据结构，这种数据结构是分片的，能够线性扩展，没有开销，从微小的数据集到大到足以服务世界上每个人和每个企业的数据集。网络的总吞吐量仅受网络中参与节点数量的限制。

# [](#designing-an-efficient-and-secure-consensus-algorithm)设计一种高效安全的一致性算法

高效处理大量数据只是分布式账本技术(DLT)真正主流应用必须解决的问题的一部分，另一部分是确保系统中不存在冲突数据，而不会被快速标记和删除。这种对数据冲突进行比较、排序和达成一致的系统被广泛称为“共识”

共识系统可以大致分为许可共识和非许可共识。许可共识是指参与决定冲突条目的所有各方都是可识别的(例如，特定企业)，并且被给予明确的许可来修改和拒绝系统中的信息。这些系统在管理分布式数据库方面已经存在很长时间了，并且仍然在许多新的许可系统中使用，如 Hyperledger 和 R3 的 Corda，并且建立在 20 世纪 70 年代的许可共识系统上，如 PBFD 和 Raft。

他们的缺陷是共识是不安全的，它只是依赖于保护选择被允许管理数据的各方的过程，而不是使达成共识的方法本身安全和容错。

[无许可共识](https://www.radixdlt.com/post/why-consensus-algorithms-are-like-military-generals)是 2008/2009 年创造比特币的创新。它没有选择参与达成共识的各方，而是想出了一种确保达成共识过程安全的方法；这意味着不需要中央权威来选择和审查网络参与者。取而代之的是，计算机能力被用作一种方式，让每个人都可以找出网络中大多数人认为正确的真相，将其组织成事务块，然后将这些事务块链接在一起，每个新事务块都引用上一个事务块，从而形成了区块链。

使用计算机能力来保护这一过程被称为[工作证明(PoW)](https://www.radixdlt.com/post/what-is-proof-of-work) ，并且仅与用于决定接受什么数据和拒绝什么数据的总计算能力的一半一样安全。这对于非常大的网络可能是有效的，在这些网络中有明显的加密经济激励来提供不断增加的计算能力，例如比特币，但是对于小型或私有网络来说效果不好，在这些网络中获得超过 50%的网络计算能力是微不足道的。

然而，它有一个非常重要的特点——它不依赖于单个网络成员的安全，而只依赖于整个网络的安全。如果以正确的方式部署，这个特性使整个系统比许可系统更有弹性和更安全。

Radix 还通过创建一个无需许可的共识系统来解决这些问题，该系统可以在小型和大型网络中安全使用，但受到一个不能在 Radix 中轻易购买或伪造的属性的保护:逻辑时间的流逝。除此之外，与其他 DLT 体系不同，它并不对每一个事件都适用共识，只对那些有冲突的事件适用。这使得整个系统非常高效，即使是在大规模的情况下。

逻辑时间的流逝依赖于一个叫做[逻辑时钟](https://youtu.be/wfsZuN6NaJo)的概念——它使用这些时钟来生成分布式系统中事件的部分排序，以检测和防止基于 [Leslie Lamport 的逻辑时钟理论](https://lamport.azurewebsites.net/pubs/time-clocks.pdf)的因果违反。简单来说，逻辑时钟就是一个计数器。Radix 网络上的每个节点(节点是维护网络包含的部分或全部状态数据的计算机)都有自己维护的逻辑时钟。它必须遵循的唯一规则是，它必须严格向上计数，并且每次只能递增 1。

在 Radix 中，当一个节点看到一个它以前从未见过的有效事件时，这个计数器被触发。节点必须简单地检查它自己的数据库，然后增加它的逻辑时钟，用新的时间戳和 ID 标记事件，然后[把它](https://youtu.be/9kPyyOIn6Bo)传给网络。它必须定期做出[承诺](https://youtu.be/jTYGOuS2xvg)，以确保它不能对历史事件的顺序撒谎。

由于数据结构的原因，这种简单可靠的共识机制现在只有在任何节点不同意相关事件的正确排序时才会被触发。这种被动共识机制既非常可靠，在大量不诚实或有故障的节点上也非常有效，因为除了解决冲突之外没有额外的能量消耗。

# [](#tempo-fast-amp-scalable)节奏快&可伸缩

Tempo 既是“异步的”，意味着没有阻塞时间，也是拜占庭故障检测，意味着它可以在一个无权限的系统中检测和停止协议违反，具有 99%的容错一致性。Tempo 通过保持事件的总顺序，允许无信任的价值转移，时间戳和其他功能来做到这一点。它将状态转换信息仅限于那些需要它的网络成员(碎片)。这减少了开销，并实现了快速终结时间，满足了大规模采用所需的快速和可伸缩的[前两个属性[([https://docs . radix DLT . com/alpha/learn/platform # how-is-radix-different](https://docs.radixdlt.com/alpha/learn/platform#how-is-radix-different)]。

# [](#tempo-efficient-amp-secure)周迅&安稳

节奏数据架构的属性允许建立对等状态机，该状态机验证状态转换而不是计算状态转换，并且不需要大量计算能力(PoW)或大量资本(PoS)来保护它，因此降低了进入的障碍，并且增加了去中心化的程度，同时具有能量效率。

它既使用拜占庭容错来防御 51%的网络分割攻击，又使用拜占庭故障检测来检测和丢弃违反协议的行为，如双重开销，并具有 99%的容错能力，从而实现最高的安全性。

阅读这篇解释者博客文章，了解如何与 99%不诚实的演员达成共识

它适用于公共和私有部署，无需修改，也不需要特殊的硬件或设备。

# [](#tempo-decentralized-and-fair)节奏——分散而公平

Radix 通过简化节点部署降低了准入门槛。它运行在标准的商业硬件上。它不需要高计算能力或资本来保护网络，从而避免了网络集中化。它不会通过使低功率设备能够根据其资源容量部分地支持网络来歧视低功率设备，并提供获得奖励的同等机会。

# [](#tempo-ease-of-use)节拍——易用性

Radix 网络是一个由计算机组成的对等网络，这些计算机共同构成了一个分散的状态机。与以太坊/EOS 等其他公共网络相比，最大的区别是节点验证状态转换，而不是计算状态转换。这减少了错误并允许协议可编程性。开发人员使用他们首选的编程语言(如 Java、Kotlin 和 JavaScript)构建应用程序，以使用本机协议 API 调用与 Radix Alpha 网络进行交互，而不使用智能合约语言或需要图灵完整计算机器。

因此，Radix 平台是比特币和以太坊应该有的样子；快速、可扩展、安全、完全分散并由可用且相对稳定的加密货币提供动力。

# [](#how-does-it-work-simplified)它是如何工作的？(简体)

Radix 是基于一种叫做 Tempo 的组合架构和共识算法构建的，Tempo 白皮书对此进行了全面概述，可在此处找到:[http://bit.ly/2ADK8LA](http://bit.ly/2ADK8LA%E2%80%8B)

Tempo 的目标是创建一个不可信的、分散的共识算法，可以作为一个公共网络可靠地工作。为此，必须:

异步-网络上的任何进程都可以在需要时开始或结束，例如，它不需要等待“块”被挖掘。

高度并发——许多流程可以同时进行，没有瓶颈——例如，可以放入“块”的事务数量没有限制。

可扩展-没有限制网络总吞吐量的上限，如 CAP 定理对区块链和 Dag 的限制。

为了实现这一点，Tempo 依靠两个简单的想法和一个逻辑上的飞跃:

使用称为“逻辑时钟”的基本数字钟

告诉你的邻居发生了什么，这个过程被称为“八卦”

为了防止碎片重复花费，你的碎片地址应该基于你的钱包公钥

逻辑时钟是一个计数器，每当有新的事情发生时，它就加 1。在 Radix 上，“新东西”是指 Radix 网络中的一个节点与另一个节点对话。这为每个节点提供了基于网络活动的相对时间，节点可以用它来创建简单的事件顺序。

Gossip 协议在计算机科学中已经得到了很好的确立，并且是通过网络可靠地共享信息的最快方式之一。它的工作原理很简单，一个节点选择几个其他节点来告诉一些新的东西，然后这些节点再告诉另外两个节点，如此循环往复。这导致信息以指数速度传播。

在 Tempo 上，我们将节点逻辑时钟(由相关节点签名)添加到它们在网络中传播的流言中。这使得每个人都可以看到新信息，以及该信息在什么逻辑时钟时间被网络的其他成员看到。

为了实现高可伸缩性，Tempo ledger 被分割成一个非常大的碎片空间，允许大规模的并发。为了避免在任何碎片上重复花费，钱包所在的碎片由它的公钥决定。这确保了从钱包中的任何花费将总是在同一个碎片上开始。

当与逻辑时钟和八卦相结合时，Tempo 能够创建所有相关事件的总排序，允许网络快速检测和消除重复花费。
