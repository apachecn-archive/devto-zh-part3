# 为他的团队服务

> 建房是我唯一的借口，因为我年纪大了
> *【那是我的人──ntm*的最高境界】

作为一名技术领先者，这不是授予开发人员的特权，也不是授予微型经理同事的荣誉。我认为，他的作用是为他的团队提供服务，帮助他们实现卓越的技术，改进项目组织，并帮助他们发展软技能。帮助团队成员发展技能是朝着正确的方向前进。

# 技术领先≥超级英雄

武术练习者会告诉你:一个学生在开始训练时，是根据掌握和发展基础的能力来判断的。接下来，由他帮助成长的学生的素质来判断

我认为，一个好的技术领先企业必须以发展其他技术领先企业为目标，因此最终目标必须是不再扮演这一角色。必须避免培养超级英雄开发者的神话，这种神话集中了信息和技能。

作为一个技术领导，我很想参与建立一个能够产生质量代码的自主组织。这就要求该小组取得进展并承担责任，以便其成员能够在不同专题上发挥领导作用。

# 团队的律师和发起人

> 知道吗，我想要的是我的人呼吸着
> *【那是我的人 ntm 最高者】*

“创新团队”是指采取主动行动的团队。说实验就是失败，直到找到正确的公式。在这些失败的阶段，必须记住，这是一个正常的创新过程，必须从这些失败的举措中吸取经验教训。在企业组织中，这些故障有时可能会发出超出预期的噪音，在这种情况下，作为技术领先公司，为团队辩护是很重要的，这样，环境噪音就不会干扰团队，甚至使团队失去动力。

当团队实现目标时，必须强调他们必须作为技术领先公司彻底退出。我的角色也是我的团队的一名新闻干事，每次获奖，我都想确保他们的努力得到应有的重视。

# 技术领先，团队中最好的开发者？

简单的问题，复杂的答案。你认识什么？一个好的开发人员？这是他写复杂代码的能力吗？还是简单的代码？他与同龄人合作的能力？让他的同事进步？使用新的框架？

想知道作为 Tech Lead，我们是否比其开发者同事更好是毫无意义的。Tech Lead 的作用并不要求在任何时候都比其他人更好，这甚至是空想。技术领导也是一个开发人员，因此也需要不断培训。技术领先公司并不总是想培养一种比别人更好的感觉，而是可以主要参与创造一种环境，在这种环境中，它承认自己没有对所有事情作出反应，合作、互助和信任是取得进展的最佳途径。

# 创造和维护友好环境

> 走着去，多少人会跟你说同样的话呢？
> 我守护你的背影，你守护我的
> *我的──shurik’n*

开展创新项目需要有一个有利的环境，在这种环境中，所有团队成员都能不断改进。必须设法恢复这样一种观念，即项目中的任何人都不会在早上起床，并思考如何破坏他人的生活。

当然，在我的经验中，我关注的重点是技术卓越，同时也关注自主性的发展、与业务团队、客户和产品所有者的联系-我...。对我来说，重要的是创造一个人与人之间建立积极联系的环境。
