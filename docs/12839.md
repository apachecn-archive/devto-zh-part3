# 作为一名软件开发人员，如何在同行中脱颖而出

> 原文:[https://dev . to/jamesmh/how-to-stand-out-of-your-peers-fi8](https://dev.to/jamesmh/how-to-stand-out-among-your-peers-fi8)

我的职业生涯始于为一家公司工作，该公司为汽车制造商搭建平台，整合并运行销售分析等。

我开始不得不使用我还没有经验的技术和框架(ASP。NET，T-SQL 等。)

最初几个月，我感到不知所措。

我花了很长时间来做一些简单的事情，比如在网页上添加一个复选框，然后保存在数据库中！

也许你也去过那里！

# [](#my-first-specialization)我的第一次“特殊化”

几个月后，我会不时地构建 SQL 脚本，将大量数据从一家汽车制造商的旧平台迁移到这个新平台。

想想梅赛德斯-奔驰美国公司的所有数据(销售数字、库存等)。)针对美国所有经销商，为期十年。大量数据。

迁移数据的一个核心问题是，例如，某些代码(财务帐户、SKU 等。)在另一个系统中可能看起来完全不同。它可能根本就不存在！

我们必须通过解析模式或处理旧代码来提取正确的代码。

现在，在大学期间，我爱上了正则表达式。所以我真的很擅长它们(不再那么擅长了！😂).

我的同事们——在这个行业有十多年经验的同行——看到我的剧本时会非常惊讶。我基本上只是使用正则表达式和 SQL 来操作这些数据集。我的脚本通常只需要几条 SQL 语句就可以完成这项工作。

另一方面，我的同事正在用 C#构建单独的命令行可执行程序，这些程序首先从数据库中获取一些数据，遍历这些数据，做一些事情，然后将一些数据放回数据库。

我的解决方案需要几个小时或几分钟，而他们的需要几天。

很快，每当团队遇到一些很难提取的数据时，我就成了“关键人物”!这都是因为我非常了解正则表达式！

真不敢相信

**信任。**

# [](#some-tips)一些提示

现在，我不是要告诉你学习正则表达式😂。

但是，对我来说，它是从正则表达式开始的。然后是前端开发。然后是建模业务规则和系统架构。

我的建议是:在你的团队或公司中找出差距，这有助于解决你的公司面临的一些重要问题。

但是你在想- *“说起来容易做起来难！”*

实际上，如果你处于职业生涯的早期，只要留意你的同事是否对你的技能或做事方式感到惊讶。

这可能表明一个差距，可能有助于您的组织！

对于那些更有经验的开发人员来说，应该更清楚需要填补哪些空白。而且，你越有经验，你就越应该自然而然地成为指导他人的人。

一旦你经验丰富，你可以使用这个原则，并将其应用于全球社区-而不仅仅是你的内部公司。试着找出哪里有需求或者某些趋势似乎指向哪里。
