# 所有程序员至少应该做一次的事情 Pt 1

## 用汇编语言写一个小程序

前几天，我发表了我的“赤脚在雪地里双向爬坡”/“回到我的时代”编程演讲，这让我想到了初出茅庐的程序员的道路。我现在在编程界看到的挑战是，我们正站在巨人的肩膀上，而这些巨人正站在其他巨人的肩膀上，而这些巨人也正站在..良好的..你明白了。在这一堆巨人的最底层是最糟糕的母亲。比其他的更快更强几个数量级。她就是那个向中央处理器耳语未经过滤的魔法的人。我担心这些天她越来越孤独了。她花时间与 bios 工程师、固件人员和硬件爱好者进行无聊的对话。所有程序员都应该花一点时间和她在一起，学会一些崇敬。他们需要知道在关键时刻，项目是如何真正完成任务的。

早在 1984 年我的旅程开始时，我正在 [BASIC](https://en.wikipedia.org/wiki/BASIC) 的 [Commodore 64](https://en.wikipedia.org/wiki/Commodore_64) 上编写通信软件。由于小处理器和低效的语言，有时程序很难跟上 300 波特的连接。我发现唯一的解决办法是用[机器码](https://en.wikipedia.org/wiki/Machine_code)编写输入和输出例程。为此，我花了 14 岁夏天的第一段时间使用一个机器代码监视器来研究如何简单地将我的名字输出到屏幕上。几天后，我完成了这个看似无关紧要的任务，但想到所有与 BASIC 语言接口和读写串口的工作，我就有一种可怕的沮丧感。然后事情发生了..一个周末，在我们疯狂的软件交易聚会上，我的朋友 Mike 给了我一个盗版的 [Commodore 宏汇编程序开发系统](https://www.c64-wiki.com/index.php/Macro_Assembler_Development_System)。*这个。变了。一切。* [汇编语言](https://en.wikipedia.org/wiki/Assembly_language)给了我一套全新的工具集。现在，我不仅仅是在一个盒子里输入数字，我还可以在一个文本文件里写出看起来模糊不清的英语指令。我可以给变量取名字，写注释。太神奇了！为了把这个文件转换成我需要的超快的机器码，我不得不学习编译和链接，这对于一个在那之前只会在基本解释器中键入指令和键入“run”的孩子来说是非常陌生的东西。

自从我第一次接触到 [6510](https://en.wikipedia.org/wiki/MOS_Technology_6510) 汇编程序以来，我一直在 x86 和 m68k 处理器以及各种嵌入式芯片组上使用汇编程序。每当我这样做时，我的思绪就会回到那个我必须做很多事情才能得到很少结果的地方，但代价是那件很小的事情跑得非常非常快……* vee 非常快！*除非你用汇编语言写了东西，否则你永远不会知道你的电脑到底能跑多快。

我的第一次组装经历让我受益匪浅。

*   我了解了处理器消耗的单个指令。

*   我学习了寄存器。

*   我学到了[累加器](https://en.wikipedia.org/wiki/Accumulator_(computing))的魔力。

*   我学会了分配、访问和释放内存。

*   我学到了比较和分支的核心。

*   我学习了编译成目标代码，以及为什么存在链接器。

我旅程的下一大步是学习 C 语言。我用宏汇编程序学到的所有东西都给了我巨大的优势，因为它让我理解了 C 语言是如何在非常低的层次上工作的。

学习用汇编语言编写程序教会我的最重要的事情是，除了一份粗制滥造的手册和几个聪明的朋友之外，如何用很少的资源做一些非常困难的事情。
