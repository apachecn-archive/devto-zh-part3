# 现有功能的设计评审

> 原文：<https://dev.to/lgranger/design-reviews-for-existing-features-3j3g>

最近，我接手了一个项目，帮助组织我团队的设计评审过程。我们想要做的一部分是对我们现有的功能进行“设计评审”,其结果将是关于该功能的设计评审会议以及关于该功能的新文档。

目前，我们要求为现有功能撰写设计评论的人回答以下问题:

*   这是什么特点？

    *   它最初的设计目的是什么？
    *   客户希望它做什么？
*   它是如何工作的？

    *   这个特性的组成部分是什么？
    *   他们使用什么技术？
    *   他们如何一起工作来创建功能？

你的团队如何保持对你现有特性的了解？在没有更新的(或任何)文档的情况下，您如何解决这些问题呢？在现有功能的设计评审中，您还会问其他问题吗？
