# 如何为你的网络开发客户处理托管

> 原文：<https://dev.to/raddevon/how-to-handle-hosting-for-your-web-development-clients-364i>

根据**你建立了什么**和**你想在托管**中参与的程度，你有几种选择来托管你客户的网站。我将分享几个选项，并告诉你**我如何为我的客户**托管网站。

## 为客户提供托管的方式

### 他们“宿主”

客户端实际上并没有托管应用程序，但是**他们拥有在**托管应用程序的账户。如果你根本不想参与主持，这是你的麻烦。

这并不意味着你不能从这笔交易中赚些钱。你为主人提供了一个有价值的资产:你的顾客。你已经花时间和他们建立了关系和信任。大多数供应商都愿意付钱给你，让你接触到那个客户。当你的客户注册时(或者他们已经被托管了几个月)，你将获得一次性推荐费用。

不要因为一家供应商提供的报酬最高，就把你的客户介绍给他们。未来你与他们的关系远比短期内赚快钱更有价值。如果他们与你推荐的供应商有过糟糕的经历，你花时间培养的信任可能会在瞬间消失。我的方法是首先为客户*研究最好的选择*，然后只为*和*寻找联盟计划。这样的话，我的选择就不会受到推荐费的影响。

要找到这些项目，请寻找“推荐”、“合作伙伴”或“附属”项目。你通常需要注册并获得批准，所以现在就做，而不是等到最后期限。

### 他们主持(这次为真)

不，真的。这一次，客户端实际上托管了您为他们构建的东西。总的来说，这让大家的日子都不好过了。**一些企业客户会要求应用在本地托管**(在本地，意味着它们不能在云中托管)。他们可能想要自己部署应用程序，或者他们可能想要让您远程访问部署应用程序。企业客户待遇不错，所以这些麻烦应该是值得的。

### 你主持

您可以通过拥有托管或云帐户来托管客户的站点，您的客户的应用程序就驻留在该帐户中。你将支付费用，并把它们开给你的客户，通常会有一点加价，包括维护、支持或任何你想提供给客户的经常性服务。这创造了一些经常性收入，帮助你远离自由职业者不得不面对的盛宴和饥荒。

### 一种混合方法

一些主机提供**白色标签**。这允许他们主机/你主机的混合。使用白色标签，您可以转售提供商的服务，就像您托管一样。然而，客户不是付钱给你，而是直接付钱给主人。你通常要么赚取一定比例的月费，要么选择增加月费，从主人那里收取差额。

客户直接向主机付款，但主机的标识不会出现在发票、管理面板或其他任何东西上。(这是白标部分。)你可以在任何东西上贴上你自己的商标。这允许你拥有与你的客户的关系，而主机照顾所有的主机。

## 我*做什么*做什么

虽然经常性收入很诱人，但我从来不想在凌晨 2 点接到电话，因为一个网站或一个应用程序关闭了。我想确保我的客户了解谁负责确保他们的网站正常运行，这样他们就知道当网站关闭时该找谁。这种移交是通过推荐的方式进行的。他们注册我推荐的服务级别，我为他们部署网站。

我考虑过尝试“托管”，甚至走白色标签路线，但无论哪种情况，我都将是我的客户托管的代言人。我喜欢构建软件，但不喜欢支持基础设施。我已经有意识地选择做我喜欢的事情，而不是赚最多的钱。

这就是我为静态网站、WordPress 和其他 PHP CMS 支持的网站所做的。在其他情况下，我的客户已经有了他们想要部署应用程序的云帐户。如果我遇到这种情况，我会利用合作伙伴计划，这样我就可以获得介绍费，不再支持基础架构。

我仍然想尽我所能帮助我的客户。如果他们打电话给我，我会尽我所能帮忙。我采用这种方法的目的只是确保我不是我的所有客户在他们的网站宕机几分钟时第一个想到的人。

这种模式非常适合我。我可以做我喜欢的事情，当我把一个客户介绍给一个主持人时，我可以得到报酬。我可能会留一些钱在桌上，但这是我愿意做出的牺牲。
