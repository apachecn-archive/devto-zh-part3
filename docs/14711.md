# 你会给人类编程吗？

> 原文：<https://dev.to/mortoray/would-you-program-a-human-57gb>

随着 Crispr 婴儿的故事出现在新闻中，我一直在思考给人类编程的意义。这可能是也可能不是我最近看的一部犯罪小说的灵感，它就围绕着这个话题。如果给我一份编辑人类基因组的工作，我会接受吗？

我理解这个问题在道德问题较少的中国更容易。公开的目标听起来也不可怕:保证超过 10 亿人的受抚养人不会生来就患有遗传疾病。谁不想这样呢？然而，我们的伦理委员会不太可能批准在人身上进行实验——至少在我们看到中国领先之前不会。

为了进行思想实验，让我们假设伦理委员会已经批准了这项工作，我被聘为程序员。它是如何开始的？

作为一名优秀的程序员，我从用户故事开始，描述相关人员以及他们想用软件完成什么。当我从北京的一个咨询会议回来时，我写下了这个用户故事:

> 作为一个繁荣、人口众多的国家的民选领导人，Xi 正面临着具有挑战性的稳定决策。他非常关心市民，知道他们在受苦，也不安心。他担心人们会沉迷于毒品、酒精、个人主义。他成功地启动了监测项目来识别这些人，并提醒邻居注意他们的困境。现在他希望更进一步。他在寻找一种基因解决方案来根除毒瘾。

太好了，听起来是个高尚的事业。有了这种认识，我现在可以推荐各种可能的解决方案。自然，我先从最新版本的 CRISPR 基因编辑技术说起。

我安装工具包。

很快，我就不感兴趣了。文档很乱。已经过时了，大部分 API 只是空白。StackOverflow 充满了关于程序随机崩溃的问题，以及自鸣得意的贬低海报的回答。我将不得不盲目地四处寻找有用的东西。

你会笑，但这是运行你的电话、汽车、医疗设备和军事硬件的软件的真实情况。我们真的期望我们会更严格地对待人类编程吗？我们不能因为还没搞懂编程就停止技术的发展。我们也不能用“但他们是人”这样的论点这对像脸书和谷歌这样的巨头所做的决定没有什么影响，他们已经通过自己编写的软件控制了我们的生活。

好吧，我有一些代码。

我不能就这么部署。我需要测试一下。我想知道这将如何工作。我们有模拟器吗？也许吧，但它看起来有问题。我看到印度提出外包我的测试。他们有网络表单和上传代码的能力。我最好不要去想后端会发生什么。只要我得到我的结果，那是他们的事，不是我的。

我付钱给印度测试的承包商。我已经给了他们直接访问我的问题系统的权限，所以他们可以提交他们发现的任何缺陷。

```
Issue #18: Results in random clucking like a chicken
> CLOSED: not reproducible
> COMMENT: Second case of clucking
> REOPENED: Confirmed
> SEVERITY: Cosmetic, PRIORITY: Low 
```

也许这种有价值的副作用在编程中不会经常出现。但是，所有那些医学研究人员可能希望分享他们对被埋没、破坏和/或锁定在付费墙后面的研究数量的看法。如果我在赶时间，或者试图挽回面子，我可能会低下头，专注于完成手头的工作。

这就提出了一个问题:什么程度的完整性是可以的？目前不可能在软件中设置一个截止日期和一个计划的特性集。这不是缺乏规划能力，而是这个职业如何运作的根本不确定性。没有理由假设人类编程会有任何不同。

谈到基因编辑，我们愿意接受哪些缺陷？如果这个项目是为了治愈脑瘫，我想随机咯咯叫是可以接受的。但是道德判断会允许讨论副作用吗？高智商和其他神经疾病之间有一些关联。冒着其他遗传疾病的风险，编辑基因以换取更高的智商，这是可以接受的吗？

如果我想到自动驾驶汽车的讨论，我们根本缺乏处理道德和软件的知识。并且反响将在高政治层面上展开，它甚至可能永远不会涉及到程序员——尽管他们对答案是必不可少的。

尽管如此，当争论结束时，我继续编码。我不会推迟发布代码。除非我被明确告知不要这样做，否则我会继续工作。另外，我觉得很安全。我个人不会对这些缺陷负责。现在没有人对任何类型的缺陷负责。考虑大规模数据泄露；那些公司受到的惩罚有多严厉？即使在医学界，也有一连串的药物存在可疑的副作用，但这些制药公司的担忧仍然存在。

我可能正在描绘一幅基因编辑的惨淡画面。它有各种积极的用途，包括根除遗传疾病的能力。但是，实际上，我们如何回答关于测试和缺陷的问题呢？我们应该争论这是否是编程吗？很难不这样称呼它，因为我们有一个名为“生成编程”的范例，已经使用了半个多世纪。

除了问题什么都没有。

你呢？你会接受人类编程的职位吗？
