# 是什么造就了一家技术领先的公司？

> 原文:[https://dev.to/nprimak/what-makes-a-tech-first-company-16aa](https://dev.to/nprimak/what-makes-a-tech-first-company-16aa)

我同意没有大的影响力和/或没有增长是开发者离开公司的主要原因，但我认为这背后有更深层次的原因。在我看来，有些公司是“技术第一”的，而有些公司的技术和资源根本不在优先考虑之列。

这一切都取决于公司从哪里获得收入，以及收入与技术质量的紧密程度。这最终决定了管理层对保持技术团队和开发人员满意的重视程度。

下面是一些例子(如果你已经明白我在说什么，你可以跳过下面两段)

* * *

例如，一个政府承包公司在一年内为不同部门创建了许多应用程序，这不太可能是技术优先，因为他们从来没有很多时间来润色和维护他们的产品，因为他们从一个项目跳到另一个项目太快了。他们的首要任务是按时交付给政府，并继续赢得新合同。

另一个例子是一家制造医院内容管理系统的公司。他们的首要任务是让客户满意，但他们很少想要修改界面或对系统进行重大更改，因为这将直接影响试图治疗患者的医生，他们没有时间学习如何使用新的 CMS。

* * *

从长远来看，我认为在一个不以技术为先的公司里，开发人员更难发展壮大。但我想知道，除了技术优先，是否还有多个层次。有“技术第二”和“技术第三”的公司吗？有多少公司说他们是技术第一，但实际上不是？许多公司希望被视为技术第一，但在表面之下可能并不那么光彩。

几年前，仅仅通过询问诸如敏捷之类的事情，并观察他们的反应，就可以很容易地识别出那些认真对待他们技术的公司。然而，我认为随着敏捷方法变得更加广泛，这变得更加困难。你用什么策略来判断一家公司是否是技术领先的？你认为开发人员能在不以技术为先的公司里茁壮成长吗？为什么？有很多问题需要解决，我希望听到社区关于这个主题的意见。
