# 思考我作为软件开发人员的职业道路

> 原文:[https://dev . to/halldorstefans/estimating-on-my-career-path-as-a-a-software-developer-44 a3](https://dev.to/halldorstefans/contemplating-on-my-career-path-as-a-software-developer-44a3)

我做软件开发已经快三年了。在我最近的评估面试中，我的经理和我讨论了我想在职业生涯中采取什么步骤，以及朝什么方向发展。面试前我一直在思考这个问题，但还没有得出结论。我发现问题是有多少条路线可用。我害怕选择一条路，在这条路上投入时间，然后发现自己不喜欢。

作为一名开发者，你可以专注于 web 开发、移动应用、区块链、物联网等等。你甚至可以在 IT 部门内部进行转换。专注于云计算、计算机网络、安全，成为解决方案架构师或设计师。我们还可以进一步关注项目管理、分析师和系统管理员。可能性似乎无穷无尽，势不可挡。

这让我开始思考，“我怎样才能把范围缩小一点？”。我不一定想找到确切的职位名称，但可能会对我想去的地方有更好的想法。我列出了我觉得有趣的事情和可能是有趣的项目。例如，更新我的网站，使用机器学习，玩 AWS，以及任何我觉得有趣的事情。当我想不出有什么新的东西可以添加进去时，我想把它简化成更容易管理的东西。我决定创建一个包含 10 项任务的清单，按难度排序。所以我会从简单的事情开始，让球滚动起来。我也想有一些我可以很容易地从清单上划掉的项目，以获得一些轻松的胜利。

我不打算详细介绍这份清单，但它进行得很顺利，我已经完成了一半。我经常使用 Udemy 来帮助我学习，他们那里有一些很好的课程。谢天谢地，女朋友在读大学。所以我们都在晚上学习，我不觉得分配这些时间来学习新技能有什么不好。我的目标是，当我完成这个列表时，我将创建一个新的列表，它可以比第一个更具体。一旦我这样做了几次，我希望我能更好地理解我喜欢做什么，不喜欢什么。另外，我会在这个过程中创造一些很酷的项目。
