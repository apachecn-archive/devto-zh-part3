### @马 _ 电子书

@Horse_eBooks 是一个最初开始推广电子书的账号。是的，我们过去称之为电子书。根据@Horse_eBooks [维基百科页面](https://en.wikipedia.org/wiki/Horse_ebooks)的说法，它之所以出名，是因为它经常在推特上发布完全断章取义、完全荒谬的电子书广告。传说这是为了避免垃圾邮件检测，而不是为了搞笑。

### [](#horsejs)@horse_js

这个帐户已经成为 JavaScript 社区中最保密的秘密之一。每个人都认为他们肯定知道那是谁，每个人都认为是别人。一些人说只有一个人，而其他人确信这是协调的努力，像光明会一样隐藏在匿名中。

回到 2018 年上半年，我的朋友 [Jasmine Greenaway](https://twitter.com/paladique) 和我有一个想法:**我们是否可以使用现代机器学习技术，这种技术可以对庞大的数据集(如 Twitter 数据)进行预测，以确定@horse_js 的真实身份？**。我们可以仅仅使用 Twitter 数据来解决 JavaScript 社区中最大的谜团之一吗？

在接下来的几个月里，我们在时间允许的情况下断断续续地工作，处理大量的 Twitter 数据，进行不同的分析，并尽可能地利用机器为我们完成工作。

2018 年 8 月，我们在 JSConf US 上做了一个演讲，谈论了我们的一些工作。在这次谈话中，我们故意不在最后透露我们的答案。

[https://www.youtube.com/embed/s0A0hL7nJXw](https://www.youtube.com/embed/s0A0hL7nJXw)

几个月后，我们将我们的数据汇集到一个网站上，从我们的嫌疑人那里得到许可发布我们的信息，今天我们发布了[【https://whoishorsejs.com】](https://whoishorsejs.com)。这是我们冒险解决现实生活中“谁干的”的故事。这比我们想象的要困难得多，我对那些专业破案的人有了新的尊重。像机器学习这样的新工具可以处理大量的数据，甚至可以在我们不确定解决问题的算法是什么的时候解决问题。

但是它能解决**这个**问题吗？
