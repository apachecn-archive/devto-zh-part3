# 我最喜欢的设计资源

> 原文：<https://dev.to/sunnysingh/my-favorite-design-resources-4ele>

> 本文原载于[我的简讯](https://sunnysingh.io/news)。

虽然我不认为自己是一名设计师，但我认为设计是一项极其重要的技能，无论你是在开发应用程序、创建品牌资产、制作内容，还是制作支持当地活动的传单。以下是一些我最喜欢的与设计相关的资源。

## 🖼创作&编辑图片

[![My personal brand logo](img/eb4424d01b4488d54541940800c96673.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--pvh-_Rnn--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/b9pj2ueloryhpc5bi7zj.png)

当谈到从头创建图像或对现有图像进行润色时，我推荐 [Figma](https://www.figma.com/) 和 [Pixelmator](https://www.pixelmator.com/pro/) 。

过去我用过 Gravit Designer，它仍然是像 Adobe Illustrator 和 T2 Sketch 这样的矢量设计软件的很好的替代品。Figma 已经慢慢流行起来，因为它是基于网络的，所以开始创建徽标和原型用户界面几乎是微不足道的。不幸的是，Pixelmator 是一个 Mac 专用的桌面应用程序，但我喜欢它，因为它比 Photoshop 便宜得多。

我自己用 Gravit 设计了我的个人标志(如上所示),最近我在推特上发布了一个新的播客作品,这是我和 Figma 合作的。

**奖励工具**:

*   Pablo:我用 Pablo 为社交媒体快速创建图片，比如每当我分享我的一篇新文章时。
*   这是一个从照片中移除背景的令人兴奋的工具。
*   双色调:如果你想快速创建一个迷人的背景，双色调让你上传一张照片，并添加不同的效果。
*   我强烈建议你使用像 Squoosh 这样的图像压缩工具来保持你的图像尺寸较小，尤其是当嵌入到网站上的时候。

## 👤图标和插图

[![My 404 page illustration](img/65d038ea785f8a323d69c6e1d4ee31bb.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--L7mylp6u--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/qhwphf630jigw8ml92u0.png)

我绝对缺乏的一件事是能够设计图标或绘制插图。值得庆幸的是，有来自其他设计师的大量资源，他们免费为你提供这些类型的资产。

*   羽毛图标:一个干净简单的图标集，我用它来做任何事情。你可以使用 [React Feather](https://github.com/feathericons/react-feather) 包轻松地将它包含到 React 应用程序中，或者只需下载原始的 SVG 图标。
*   [React Kawaii](https://react-kawaii.now.sh/) :如果你需要一个快速的可重用组件包来为你的应用程序增加一些可爱，这就是它。我在我的网站上的几个地方实现了这一点，比如我的 [404 页面](https://sunnysingh.io/404)。
*   [OpenMoji](http://www.openmoji.org/) :喜欢表情符号？不要求助于原生的苹果，我推荐你使用这个统一的套件，它也提供了搜索和开源许可。
*   [unDraw](https://undraw.co/) :一个*的大规模*列表，为不同的用例预先制作插图。
*   人类:另一个伟大的资产包，可以混合搭配不同的场景和角色。

## 🔼提升您的设计水平

正如我之前提到的，我不认为自己是一名设计师，但我仍然非常喜欢愉快和直观的设计。对某件事充满热情可以让你从周围的世界中获得灵感，比如从苹果的极简设计模式中。

极简设计的伟大之处在于它并不那么难学。从用户界面中去掉不太重要的元素，添加空白，使用一致的小调色板，这样你就离干净的界面不远了。

我在去年年底重新设计了我的个人网站，[写了我的设计决定](https://sunnysingh.io/blog/revamping-personal-site-gatsby)。那篇文章中提到的是令人惊奇的[重构 UI](https://refactoringui.com/) 资源，我从保持我的设计简单易读中学到了很多。我也强烈推荐史蒂夫·舒格的这些 [UX 建议。](https://twitter.com/i/moments/880688233641848832?lang=en)

学习如何提高你正在构建的网站或应用的 UX(用户体验)需要时间。学习如何恰当地树立品牌和发展个人品牌也需要时间。重要的是想要提高，希望我在这里分享的一些资源可以帮助你。
