#### [](#automate-everything-you-can)尽你所能自动化一切

[![](../Images/587b8c33809925e4e84e50bf733e1698.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--m4a5cIIB--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2ADrKc7wEs4O5q6mwUCy0dCg.jpeg)

作为一个 web 开发人员，你必须懒惰。每个痛苦的、重复的任务都可以自动化。试着列一个清单来识别那些重复性的任务。

你的项目里没有 CLI 吗？创建一个。您可以创建一个脚本，专门为您的项目生成目录和文件。

如果你是一名前端开发人员，尝试使用 Node.js，制作一个小脚本，或者如果你对另一种语言感兴趣，这将是学习新东西的机会。

#### [](#know-how-things-work-under-the-hood)知道幕后的事情是如何运作的

[![](../Images/7f47e9f01e41f833fb58b4500a911c79.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Rr0Ek18O--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2Ae9xZLyReopougN3kd_Nl8Q.jpeg)

有了 Github 这样的平台，你可以访问重要的免费文档。所以要好奇你的模块是如何工作的，尤其是大模块。

例如，你可以检查 [Redux-saga 的储存库](https://github.com/redux-saga/redux-saga)。他们使用 ES.next 令人兴奋的特性。

你必须进入源代码，阅读代码或者克隆库，然后使用它。虽然这可能很吓人，但定期做这个过程对你来说会更容易和自然。

#### [](#explain-a-concept)解释一个概念

[![](../Images/0df28e201a1496fd833d0f5903266541.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--50e51dZM--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2Aa2cFEVU4AnvOmFqcvzE4kQ.jpeg)

> 凡是想得好的，都说得清楚。说出来的话也流畅自如。

这句话是法国作家[尼古拉斯·布瓦洛](https://en.wikipedia.org/wiki/Nicolas_Boileau-Despr%C3%A9aux)说的。事实上，非常有趣的是，如果你不能清楚地解释一些事情，你可能没有很好地理解这个概念。

进行类比，使用视觉图像，绘制图纸。

用这种方法，你可以看到你不清楚的地方。这是一个极好的练习。试着和你的同事甚至你的朋友，家人一起，像这样，你会看到你是否可以庸俗化。
