# [](#an-archer-class-and-unexpected-complexity)一个弓箭手阶层和意想不到的复杂性

我制作了一个弓箭手职业来配合战士和法师职业。

[![Archer class attack](../Images/1e01fd5d9bc94712d3803901a9f359a5.png "Archer class attack")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--2NY4IXkS--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/kuxh5zfdrcnmg7iap5s7.gif)

在上面的截图中，弓箭手的攻击未能伤害其中一个目标。弓箭手的攻击失败可能是因为游戏更新率低结合弓箭手的抛射攻击速度高。因此，对于当前的配置和设计，禁止高速攻击。

由于最近的经历，我第一手发现，开发一个使用动作战斗的多人游戏比开发一个像大多数 MMORPGs 一样使用 tab 键定位的多人游戏要复杂、困难和昂贵得多。所以这个游戏不会是动作 RPG 游戏。

[![Level up game mechanic](../Images/62e21b5dc700c07610a26e0a7df8ab6e.png "Leveling up mechanic")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--I2sUmoOX--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/l4cnyqli1ccjxz52v5vk.gif)

# [](#an-entitycomponent-system)一个实体-组件系统

当我最初开始这个 3D 多人游戏时，我选择了禁止自己使用实体组件系统的规范。原因是我希望这个游戏产品足够简单，不需要实体组件系统。

但我在这里，创造了这样一个东西，这是意想不到的。

# [](#socioeconomic-aspects)社会经济方面

我做的这个产品并不完全是技术上的努力，因为它是一个需要满足某种市场需求的游戏。尽管如此，在这个十年结束之前，我将最终成功制造出这种产品，并通过实现产品与市场的契合来创造收入。

此外，这个开发日志并不是一个真正的高效用因素，因为我的潜在和真正的目标受众甚至不消费这样的内容。依靠这个博客/法典来营销这个产品是一个负面的效用因素。因此，我将减少这些开发更新的数量，这样我就可以为产品的成功投入更多的精力。
