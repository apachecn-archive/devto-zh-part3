# CSS 全年

> 原文：<https://dev.to/bassemibrahim/full-year-of-css-5757>

在我职业生涯的第一年，我在一家数字营销机构工作，是一名全栈 web 开发人员。在那个领域，主要的焦点是用户体验、好看的设计和 SEO。我花了一段时间才熟悉了 web 开发的方法。起初，我不知道如何构建一个完全响应的页面。但在第一年快结束时，我为大品牌开发了几个网站。

在这篇文章中，我想和你分享我写 CSS 的规则。

**CSS 的力量💪:**
我以前觉得 CSS 没什么大不了的。任何有规则表的人都能做得很好。感谢上帝，我没花这么长时间就知道自己大错特错了。

CSS 和其他语言一样复杂。其实 CSS 有不同的方法论。响应式设计有不同的方法，CSS 也有基于它的扩展语言，如 [Sass](https://sass-lang.com/) 。有些人正在使用普通的 CSS 创建令人惊叹的创意动画和效果。

我真正喜欢 CSS 的是，代码质量、付出的努力和开发人员的技能在最终的设计中可以清晰地看到。

我遵循的 CSS 准则🙄:
**好好想想你的选择器🤔当添加一个新的 CSS 规则时，给自己一点时间，想想你将如何编写你的选择器。想想它的影响。想想你要设计哪些元素的风格。想想你想避免哪些元素。我经常发现自己在解决新规则带来的问题。那是一个冗长而令人沮丧的过程。了解您的选择器的全部影响将节省您的时间和精力。**

只添加你真正需要的东西🧐
人们通常利用试验&误差来润色和实现恰到好处的设计。这是一个很好的方法，我一直都在用。但是你必须确保清理你的 CSS，删除你在这个过程中添加的每一个无效的或者不需要的规则。它们可能暂时没有效果，但它们可能会导致将来难以跟踪的问题。一个好的建议是使用浏览器的开发者工具来尝试任何新的规则。完成后，只将布局所需的规则应用于 CSS 文件。

**忌！重要的是不惜一切代价✋**
你有没有遇到过无法改写 CSS 规则的情况？嗯，我有，这是非常诱人的刚刚使用！重要。但是你真的不应该这样做，因为这很可能只是推迟问题的解决。更好的是，退一步，从根本上解决这个问题。然而，有时在处理第三方库时，不使用就很难覆盖！重要。尤其是当他们使用 javascript 或内联样式添加规则时。只是把它作为最后的手段。

如果可以用 CSS 来做，就用 CSS 来做👌
无论你想做什么，如果你能用 CSS 实现的话，它可以是一个动画或者一个简单的弹出窗口。用 CSS 做。我知道有些人更喜欢 javascript，它可以给你更多的控制，甚至可以更容易。但是 CSS 弹性较小，优化更好，最重要的是，它的加载速度比 javascript 快得多。在添加另一个 javascript 插件之前，请考虑这一点。

就是这样。基本上，CSS 就像任何其他编程语言一样。编写更干净的代码需要练习和专注。但是它提高了代码的质量和可读性，从而在以后添加或修改时节省了大量的时间和精力。遵循上面的指导方针在我的旅程中帮助了我很多，我希望它们也能帮助你。让我知道，好吗？🙄🙄

**快乐编码🔥🔥**
