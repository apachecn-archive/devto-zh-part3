# [](#what-is-hash-table)什么是*哈希表*？

哈希表是一种数据结构，其使用哈希函数来有效地将键映射到值，以进行有效的搜索/检索、插入和/或移除。

哈希表是计算机科学中许多数据结构之一，其目的是使用各种管理和存储格式来组织数据，使我们能够有效地访问和修改它们。

还有许多其他的数据结构，如数组、链表、二叉树等。

在这篇文章中，我将重点介绍哈希表，它广泛应用于各种计算机软件，尤其是关联数组、数据库索引、缓存和集合。

# [](#how-do-you-use-it-)怎么用？

像任何数据结构一样，哈希表用于帮助解决数据问题。

基本上你能在时间和空间上有多高效....

1.  搜索项目
2.  插入一个项目
3.  删除项目

哈希表的工作方式是使用**键= >值查找**

### [](#a-you-take-input-keys-run-it-through-a-hash-function-where-it-will-map-those-input-keys-into-numbers-which-corresponds-to-number-of-indexes-in-an-array)A .)你拿**输入键**通过**散列函数**运行它，在那里它将把那些输入键映射成对应于数组中索引数的数字。

假设我有一份名单，上面有迈克尔、鲍勃、杰克、贝基。(这些是我的输入键)

### b .)通过散列函数运行这些输入。

有许多方法可以编写一个散列函数，但本质上散列函数应该接受一个输入字符串(我们列出的名称)，将它们转换成一个整数，然后将该整数重新映射到数组中的一个索引。

### [](#c-boom-we-have-associative-values-where-we-can-now-search-insert-and-delete-our-data-by-using-the-array-index)C.) Boom 我们有联想价值。在这里我们可以使用数组索引来搜索、插入和删除数据。

注意:存储哈希表数据的数组可能总是比所有可用的潜在哈希代码或键输入小得多。

### [](#d-example)D .)举例

让我们想象一下，我们将前面列出的名字放入一个散列函数中，这个函数的最大索引为 7。

[undefined, [Bob], undefined, undefined,[Becky], [[Michael], [Jack]], undefined,]

因此，您可以看到数组中有 7 个索引，所有这些名称都映射到一个特定的索引。

我们现在可以搜索一个特定的名字，比如 Bob 在 array[1] Becky 在 array[4]等等。

*但是存放迈克尔和杰克的数组[5]呢.....多么完美的赛格威进入我的-

“最后但并非最不重要的”注意:由于有限的索引，这为“冲突”打开了大门，当两个或更多的哈希代码被映射到同一个索引中。

有不同的方法来解决冲突，一个非常常见的方法是将它们存储在一个链表中，而不仅仅是字符串。可以把它想象成给每个名字分配一个 id，然后把它存储在一个散列表中。

因此，当循环通过一个有多个键的索引时，你可以用它们链接的 id 来引用你想要的特定名称。

# [](#why-do-you-need-to-know-hash-table-)为什么需要知道哈希表？

当然是为了在技术面试中表现出色....

从技术上来说是的，但是也要记住像散列表这样的数据结构是有效搜索、插入和删除数据的方法。哈希表是最受欢迎的方法之一，因为它们非常高效。
