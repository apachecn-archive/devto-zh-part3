# Ruby 2019 的状态:它正在消亡吗？

> 原文:[https://dev . to/vinibrsl/the-state-of-ruby-2019-is-it-dying-35a 0](https://dev.to/vinibrsl/the-state-of-ruby-2019-is-it-dying-35a0)

有很多关于 Ruby 语言消亡的讨论。嗯，这个胡言乱语已经流传了一段时间了，3 年前就有帖子说 Ruby 已经死了，但事实上，我们在这里。

谷歌趋势搜索显示，人们对 Ruby 的兴趣并没有随着时间的推移而减弱，而是遵循着类似技术正在追踪的相同模式。此外，永远记住**编程语言是解决特定问题的工具**，这就是为什么没有最好的编程语言这种东西。

[![](../Images/0078bc3ca11bc896cd4eaf32817a316a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--y1rmvZAU--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/3166/1%2A2_WZXzwiIIiUDnd95Mwa-g.jpeg)

在过去的一年里，Ruby 经历了很多重大的变化，它当然没有消亡，而是在成长和进化。2019 年对 Ruby 社区来说意义重大。检查以下一些要点:

## [](#ruby-26-and-its-new-jit-compiler)Ruby 2.6 及其新的 JIT 编译器

Ruby 的新版本已经在 2018 年 12 月发布。这个版本包括许多性能改进和新特性，包括 JIT(实时)编译器的初始实现。

老实说，这个新的 JIT 编译器还没有达到它的最佳性能，正如这里的所提到的[，但是它有很大的前途。Ruby 社区认为这次发布开启了 Ruby 的**新时代。**](https://medium.com/@k0kubun/ruby-2-6-jit-progress-and-future-84e0a830ecbf)

> # Although I failed to get Ruby's JIT production ready in 2.6, I don't regret what we did this year. I need to develop or lead all of the above for Ruby 2.6, but I believe I can mainly devote my time to the performance improvement of Ruby 2.7\. — [@k0kubun](https://dev.to/k0kubun)

除了这个新的 JIT 编译器之外，Ruby 的这个新版本还将[的性能提高了 1.7 倍](https://gist.github.com/k0kubun/d7f54d96f8e501bbbc78b927640f4208)，除了比 Ruby 2 . 5 . 0 提高了[5–10%的性能。](https://www.ruby-lang.org/en/news/2017/12/25/ruby-2-5-0-released/)

## [](#ruby-on-rails-new-release-is-getting-close)Ruby on Rails 新版本即将发布

[![](../Images/f407426e9832d11a0385cbf7ba0c050f.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--vJd8xYLO--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/2560/0%2AWxkqQXrHAe5OL1sl.jpg)

Ruby on Rails 6 已经很接近了。这个版本包含了许多很酷的特性，这些特性将使 web 开发变得更加容易，就像 Rails 多年来一直在做的那样。

2018 年推出了主要版本 Rails 5.2.x，包括:

1.  **主动存储**，一种将文件直接上传到云端的现代方法。

2.  **重定向缓存商店**

3.  **HTTP/2 早期提示**

4.  **凭证**，一种安全存储秘密的新方法

现在，Rails 6 提供了许多令人兴奋的功能，例如:

1.  **Action Mailbox** ，一种服务于[的新方式，将收到的电子邮件](https://github.com/rails/actionmailbox/blob/master/lib/action_mailbox/base.rb#L6)路由到类似控制器的邮箱，以便在 Rails 中进行处理

2.  **动作文本**，将[特里克斯编辑器](https://trix-editor.org/)带到框架中

3.  **并行测试**，允许你使用分叉或者线程来并行化你的测试

4.  **Webpack** 作为默认的 JavaScript bundler

## [](#ruby-isnt-dying-its-maturing)红宝石不是垂死，而是成熟

是的，Ruby 很糟糕，Rails 很慢。但这在 2018 年是不正确的，在 2019 年也不会。Ruby 并不慢，Rails 也可以扩展。如果它不适合你，也许你选择了错误的工具来解决你的问题。

2019 年，让我们一起享受 Ruby 2.6 和 Rails 6 的编程吧！新年快乐
