# 快速失败是不够的

> 原文:[https://dev.to/pbeekums/failing-fast-is-not-enough-1fo0](https://dev.to/pbeekums/failing-fast-is-not-enough-1fo0)

到目前为止，快速失败已经成为一个常见的比喻。这在某些情况下很有意义。害怕失败会导致许多问题，其中最大的问题是害怕尝试新事物。即使一个新事物只有 1%的机会教会你有用的东西，不尝试也让你 0%的机会学到有用的东西。害怕失败会阻碍长期的成功。克服这种恐惧并不容易，但是可以做到。

克服这种恐惧并不容易，但是可以做到。快速失败可以帮助人们习惯失败。他们开始意识到失败并不是世界末日，抓住机会可以收获很多。他们还通过快速失败的“快速”部分来最小化冒险的成本。

然而，失败的关键部分是能够从失败中吸取教训，以增加以后成功的机会。在这方面，失败得太快了。能够告诉自己你“失败得很快”会产生一种虚假的安全感。与其彻底分析失败，不如对失败不屑一顾，不认真尝试从中吸取教训。

一个完美的例子是软件开发过程。没有一个过程是完美的，每个过程都需要在团队的基础上发展。然而，对可能出现的第一个问题反应过度是非常容易的。“看看我们是怎么让这一只虫子通过的！这个过程烂透了。我们需要从头开始。”

这个场景中的问题是它跳过了对故障的深入分析。也许正确的选择是从头开始。或者对现有的流程进行调整也是可行的。直觉反应不会做出正确的决定。失败需要讨论和剖析。否则失败对你毫无益处。快速失败变成了为了失败而失败，而不是为了未来的成功而失败。

还有一个问题是，在大多数情况下，你可以尝试的事情的数量几乎是无限的。打个比方:比方说，我正在写一篇博客，我试图找到正确的表达方式。有几十万到几百万字。当考虑包含许多单词的短语/句子时，这个数字会非常快速地增长。不是每种排列都有用。语言方面的经验提供了一个粗略的过滤器，可以过滤掉绝大多数的排列。对以前作品的分析为其余作品提供了一个更好的过滤器。

但是如果没有这种分析，我将只是盲目地尝试随机单词。那不是成功之路。那简直是失败。
