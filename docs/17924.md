# 当 PHP 框架烂了系列:工作面试中的框架话题

> 原文:[https://dev . to/damn Jan/when-PHP-framework-sucks-series-framework-topic-on-a-job-interview-2fjg](https://dev.to/damnjan/when-php-framework-sucks-series-framework-topic-on-a-job-interview-2fjg)

如果你是有任何经验的 PHP 开发人员，并且至少参加过一次工作面试，那么你会被要求用一个特定的框架来描述你的知识。这个问题变得如此普遍，以至于我们认为它是 PHP 职位面试的标准。为什么公司和招聘人员如此坚持要知道你在特定框架中的熟练程度？如果你在特定领域的经验不足，为什么你不能通过面试？

#### [](#a-framework-is-a-tool-not-a-skill)框架是工具，不是技能

每个人都应该将框架视为一种工具，而不是一种技能。在招聘过程中，将编程技能与特定框架的知识分开尤为重要。许多优秀的候选人被拒绝，因为他们没有进入特定框架的经验。这可能意味着两件事:

1.  公司把自己过多地束缚在框架上
2.  一个正在做招聘流程的人对其他框架一无所知或知之甚少。

我已经讨论了第一个话题，并奉献了整个[博文](https://dev.to/damnjan/when-php-framework-sucks-series-how-not-to-shape-your-app-in-the-shape-of-the-framework-582m)。对于第二个问题，除了教育人们更好地理解框架目的之外，没有简单的解决方案。

#### [](#tools)工具

工具是框架、库、IDE、计算机、版本控制系统、虚拟化软件等。工具是我们用来制造产品的东西。我们可以用各种工具的组合来构建我们的产品。在大多数情况下，工具并不能确定某件事情是否可行。公司有时会把自己过多地束缚在框架上，以至于他们需要只能在框架范围内思考的技术人员。

#### [](#skills)技能

技能是干净的代码、文档、解决问题、测试策略、架构技能等。开发人员可能有也可能没有技能。熟练的开发人员，或者渴望学习的人，绝对可以使用任何工具(在这种情况下是框架),并构建一个好的产品。

#### [](#conclusion)结论

工具有螺丝刀、电钻和水平仪。

#### [](#common-question)常见问题

这个问题我已经在这里和我的演讲中遇到过好几次了，所以我会在这里简单回答一下。如果一家公司有[Symfony|Zend|Laraver]应用程序，那么他们需要[Symfony|Zend|Laraver]开发人员，对吗？
举例来说，如果你的业务是“音乐流媒体服务”，而你的应用是“Laravel”应用，那就大错特错了。你的应用必须是“音乐流媒体应用”，而不是不惜任何代价的“Laravel”应用！
原因是:

1.  如果您解除了与框架的绑定，那么您可以放心地升级到下一个版本。
2.  您将能够在其他环境、库和框架中重用您的代码。
3.  你不必每次 framework 弃用它们的一些库时都要修改你的应用程序。

#### [](#once-again-a-framework-is-a-tool-not-a-skill)还是那句话，框架是工具，不是技能。
