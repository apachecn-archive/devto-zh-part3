# 作为软件开发人员的个人博客

> 原文：<https://dev.to/abhishekcghosh/on-personal-blog-as-a-software-developer-ngm>

我最近一直在为自己建立一个博客。这是一个交叉帖子，也是我第一次在 [dev.to](https://dev.to) 上发布。

* * *

我是一名软件工程师。我现在已经入行五年了。我曾是一家“五大科技公司”的一员，从事的工作经常给我机会来满足我作为一名开发人员的智力需求和好奇心。一切都很棒。然而，当我看着一般的开发者社区时，我基本上不存在。

对于一个人来说，五年是学习、成长和立足的重要时期。并不是说我没有在职业上成长或确立自己的地位，事实上，恰恰相反，我可以自信地说，一直以来，我学到了很多东西，对此我非常感激。很多技能的学习和表达，不管是好是坏，都停留在这个直接的环境和其中的人的范围内，在那里我可以舒适地生活在它自己的世界里！然而不管这有多好，在全球开放的社区中，我仍然不太相信*存在*。

如果我没有说我试图将其归咎于这样一个事实，即我一直专注于茧中存在的技术和项目，或者我觉得我总是有更多的事情要做，而不是有时间坐下来与社区分享我的想法或那些我认为为什么有人会关心的小项目，那我就是在撒谎。但那不一定是真的。我没有到处去，因为我自己都懒得去。不知何故，我不觉得有这个必要。回想起来，那有点遗憾。

那么，为什么这种新发现的渴望现在存在呢？这肯定不是为了向世界展示我有多“棒”。这不是一个展示或投资组合。或者我并不真的在乎。我肯定也不会去寻找工作机会。我不会天真地不同意，拥有一个伟大的社区存在和一个谈论你的工作、技能或兴趣的媒体肯定会增加你的职业生涯，但这不是为什么我相信我有这种挥之不去的感觉，我需要把自己放在那里。这不是一场自私的生存危机。这是一种顿悟——很大程度上——这种情况，这种模式——事实上，是普遍的，令人悲伤的事实。

我个人认识很多很棒的人，他们能提供很多东西。就我个人而言，我不知道有多少人积极地发现并决定成为全球软件开发社区的一部分，并定期为其做出贡献。当然，每个人都有权选择，但对我来说，我只能真诚地希望我没有站在这个宇宙的错误一边。

> 
> “如果我看得更远，那是因为我站在巨人的肩膀上”
> 
> ——艾萨克·牛顿

人们普遍认为，软件工程行业在广度和深度上都是如此巨大，并且一直呈指数增长，以至于任何人都不可能真正掌握和了解其中的大部分内容。不过，这并不是这个领域的特别之处。这是任何值得了解的事物的普遍特征。艾萨克·牛顿发现了运动定律，发明了微积分，并从根本上改变了我们当时对物理世界的认识，甚至他也被不止一次地引用过类似于 T2 的观点。看到[软件开发社区遭受冒名顶替综合症](https://www.google.com/search?q=imposter+syndrome+in+software+development&num=50)的折磨就不足为奇了。

我不禁觉得，这也许是为什么我，以及像我一样的成千上万的其他人，可能没有足够的关心去把自己放在那里的最大原因之一。

但是我会给你一个相反的观点。事实上，这正是*为什么*我们应该关心把自己放出去的原因。外面有太多的东西需要了解和分享，事实上，我们都只是男孩、女孩或其他人，在众所周知的海边玩耍。多找几块鹅卵石玩，只会让大家更开心。

我们都有自己独特的进入软件开发世界的经历，我们自己独特的经历使我们获得了知识和从获得的知识中得到的*实现*的独特组合。

并不是所有这些都特别适用于博客(我们可以尝试很多其他途径)，但在这次对话的背景下，这是我将诉诸的程度。就我个人而言，我不认为我需要再一次宣讲和和为什么作为开发者拥有自己的博客是一件很棒的事情，以及它对你(和社区)有什么好处。我相信很多人已经做了一项了不起的工作，解释了这么做的所有好处。

不过，我会强烈要求的是渴望得到它。

所以看起来，这篇文章显然是对我的内在自我，然后是对所有那些人的一个大喊！你有足够的物质来分享。在外面的某个地方，可能有人在拼命寻找一种有效的方法来获取你可能如此不经意地拥有的知识。当然，并不是所有的信息都需要公开自由流动，这是公平的。但是，积极的可能远远不够。

*所以试试吧*。

如果没有别的，这个尝试至少会为你自己创造一个表达和确认的媒介。希望这对一些人来说是足够的动力。

所以这个博客是我尝试实践我所宣扬的东西，一个大洋岸边的小鹅卵石！
