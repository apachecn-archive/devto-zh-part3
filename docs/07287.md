# 提高编码效率的 4 个简单步骤

> 原文:[https://dev . to/rahulchowdhury/4-simple-steps-to-more-productive-coding-sessions-2e2d](https://dev.to/rahulchowdhury/4-simple-steps-to-more-productive-coding-sessions-2e2d)

不管你有多喜欢编码，总会有一天你会因为很多原因而濒临崩溃。

这可能是因为你陷入一个问题太久了，或者只是因为你编码太久了，没有给你的身体一个应有的休息。

这在每个开发人员的生活中都很常见。去过那里。 *Pfft* 。

别担心，一切都还没完。你只需要对你的编码习惯做一些调整，你很快就会回到司机的位子上。

* * *

# [](#1-apply-the-pomodoro-technique)1。运用番茄工作法

这项技术的基本概念是在*短脉冲*中工作，然后休息一会儿。尽管这种技术主要应用于像写作这样的任务，但它也适用于编码。

番茄工作法要求你:

*   不间断工作 25 分钟
*   然后休息 5 分钟，让你的头脑清醒一下
*   之后，回到另一个不受干扰的 25 分钟工作中
*   在 4 次训练后，也就是工作 1.5 小时多一点，休息 15 分钟

[![Pomodoro technique](../Images/0ee6c0632842dc11213e0f1a0a36d3a9.png)](https://i.giphy.com/media/l2SpW9nzXFVDw2nAs/giphy.gif) 
简单，但*相当有效*。我利用这种技术已经很多年了，从我的经验来看，我只能说这很有效。

然而，我通常对此半信半疑。我不一定每次编码或者写的时候都放一个定时器。我通常把我的任务分成很小的工作单元。

当我完成任务后，我会快速休息一下。

让我们看一个例子，好吗？

假设您是一名前端开发人员，使用 React 创建一个 web 应用程序。您可以将此任务分成以下工作流:

*   编写一个组件
*   快速休息一下
*   为新组件编写测试
*   快速休息一下
*   继续下一个组件

这样你就不会因为超过了 25 分钟的时间限制而被迫中途离开工作。

# [](#2-kick-out-the-distractions-with-some-chill%C2%A0beats)2。用寒冷的节拍赶走干扰

这是我完成工作的首要因素。不管你是在办公室工作还是独自在家，干扰无处不在。

有时是你的电话在响，有时是有人来到你的办公桌前说“ *Hi* ”。

这些都是让你脱离心流的干扰。一旦失去了这些，就很难再回到你所处的环境和内心深处。

正如[卡尔·纽波特](http://calnewport.com/)在他的著作[深作](http://calnewport.com/books/deep-work/)中所言:

> “因此，如果你服务于低影响力的活动，你就占用了本可以花在高影响力活动上的时间。这是一个零和游戏。”

当你对*低影响力*的活动说“是”的时候，比如查看你的脸书通知或参加一次饮水机谈话，你就*错过了*让自己参与提高自己并帮助你在职业生涯中进步的活动的时间。

编码需要*注意*。编码需要深刻的思想状态。分心的想法会剥夺你充分利用编码会议的能力。

现在的问题是，在一个充满杂念的世界里，你如何避免这些琐碎的杂念。

对此最简单的回答可以是*到*:

*   带上一副耳机
*   放一些好听的音乐
*   抵制对你周围发生的任何事情做出反应的冲动

这种活动的一个常见术语是“*在*中连线”。

[![Richard from Silicon Valley - Wired in](../Images/4e8f7614c14d6e1e0670afec4f2ec58c.png)T2】](https://i.giphy.com/media/yKHUU78VzxAg8/giphy.gif)

我喜欢连线。我就是这样完成大部分工作的。当我连线时，我周围的人很难引起我的注意。

我记得有一次，我团队中的一个人通过各种渠道给我发信息，包括 Slack、Hangouts、SMS、Email 等等，但还是不得不在我面前挥动他们的手来吸引我的注意力。

这就是我在这里谈论的焦点。

听音乐*在这里不是*的要求。这有助于将自己隔离在一个非常嘈杂的环境中。然而，如果你不介意戴上耳机并达到我所说的那种专注，那么当然，不要戴。

当你工作的时候，尽你所能去培养一种深层的精神状态。这是你施展各种魔法，创造美好事物的时候。

# [](#3-have-a-todo%C2%A0list)3。有一个待办事项列表

长期以来，我一直在为有一个待办事项清单而挣扎。主要是因为每天在我的清单上添加待办事项本身就像是一件苦差事。

在阅读了大量赞扬拥有待办事项清单的习惯的文章，并尝试了所有流行的待办事项应用后，我最终决定使用 Todoist 来维护待办事项清单。

这是迄今为止我用过的最直观的待办事项应用。这款应用让添加待办事项变得轻而易举。可能这就是我现在能坚持这个习惯相当一段时间的原因吧。

爱马仕

[![Todoist karma section](../Images/409776cd0cd88d6ecc2bde6535e0789a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--QfgV5T7b--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ptrdht1yoc2xjzajumea.jpg)

是的，这是我使用该应用程序完成的大约 **2，861** 个任务，为我赢得了大约 **19K** 的因果积分。

如果你已经开始怀疑，这是*而不是【Todoist 的广告。我之所以称赞这个应用，是因为它真正帮助我养成了保存和遵循待办事项清单的习惯。*

列出待办事项清单的主要原因是，一旦你准备好了当天的任务清单，你就不会再想接下来需要做什么了。都在那了，按优先级排序。

只要照着单子做，把事情做好。*周期*。

如何保持一份待办事项清单而不打破你的脑袋:

*   晚上睡觉前列出你第二天的任务
*   把你的任务分解成不需要太长时间就能完成的小任务
*   按优先级给你的任务排序( *p1* *用于即时和重要的任务，* *p3* *用于不重要的杂务*)
*   每天坚持这个习惯*不要失败*

最初的几天会很艰难。你会反抗。你会错过的。继续前进。一旦你养成了习惯，它就会成为你的第二天性。

截至目前，我甚至不需要记得维护我的待办事项列表，它对我来说是一种例行公事。每天晚上，我都会列出第二天需要做的所有事情，然后一整天都照着做。

# [](#4-revisit-problems%C2%A0later)4。以后再讨论问题

显而易见，我们花了太多的时间去烦恼由于一些小错误而陷入的问题。

有时候，很难发现你正在犯的一个错误，因为你的大脑被无用的信息堵塞了。我们花了几个小时才发现我们又回到了起点*。*

[![Calculating mentally](../Images/b41bae3299278ad5829b6c85bde8728f.png)T2】](https://i.giphy.com/media/14snHgzbVpgZIQ/giphy.gif)

在这种情况下，你能做的最好的事情就是暂时放弃这个问题，把注意力集中在其他任务上。这可能是其他编码问题，也可能与编码完全无关。

稍后重新审视原来的问题。*最佳*，睡了一夜好觉之后。你的潜意识是一个未知奥秘的百宝箱。它能够解决你的意识无法解决的问题。

> "要知道在你的内心深处有无限的智慧和力量."约瑟夫·墨菲

给它时间和机会来解决你的问题。这就是为什么睡觉是解决问题的普遍建议。

我记得我熬夜到凌晨 4 点，一直被一个编码问题困扰着，我无法让 XML 和改型正常工作。只是在那天晚上我睡觉后，我才能够在早上醒来后马上解决这个问题。

我突然有了那种灵光一现的感觉。

一步一步来，掌握它，然后继续下一步。
