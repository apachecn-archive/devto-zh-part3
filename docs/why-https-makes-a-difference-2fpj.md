# 为什么 HTTPS 与众不同

> 原文：<https://dev.to/flippedcoding/why-https-makes-a-difference-2fpj>

现在几乎每个网站都在使用 HTTPS。如果不是，谷歌会在左上角用一个难看的“不安全”符号来标记它。大多数网站使用 HTTPS 协议的原因有很多，这就是我们将要讨论的。

## HTTP 和 HTTPS 的区别

让我们从 HTTP 和 HTTPS 的区别开始。HTTP 是超文本传输协议，HTTPS 是超文本传输协议安全。对于常规的 HTTP 请求，拦截互联网流量相对容易。这就是公共 Wi-Fi 网络上的人可以看到您的信息的方式。

HTTPS 解决了这个问题。数据被加密，而不是发送原始请求。这就是 SSL 证书的用武之地。该证书包含网站所在服务器的公钥。公钥与用户共享，这就是安全连接的建立方式。

即使有人拦截了流量，如果不知道私钥，他们也无法从中获得任何有用的信息。HTTPS 实际上稍微降低了网站的速度，因为用户和服务器必须在传输任何真实数据之前使用证书交换密钥。对于那些可能关心的人，HTTP 使用端口 80，而 HTTPS 使用 443。

## 为什么安全很重要

这似乎是一个小差异，有这么多大惊小怪，但所有的大惊小怪都是值得的。保持在线数据安全是一个日益突出的问题。人们一直在寻找破解的新方法，他们会一直这样做下去。至少当你使用 HTTPS 协议时，你让他们很难弄清楚你的数据包含了什么。

HTTPS 加密向他们显示混乱的字母、数字和字符，如果不知道解密密钥，这些字符毫无意义。从技术上来说，有人仍然可以找到读取数据的方法，但他们将不得不付出更多的努力。HTTPS 是一种快速而简单的方法来确保你的网站上的数据得到安全传输。

这样，用户的信用卡、电子邮件地址、用户名和密码都更安全。如果这是你的一个大问题，它也有助于 SEO(搜索引擎优化)。另外，当人们使用你的网站时，它会给他们信心，因为他们知道他们的信息是安全的。这样想想，如果在角落里有一个写着“不安全”的红色 x 的信息，你还会觉得从亚马逊订购一样舒服吗？(也许你会，但你知道我的意思)

## 如何升级到 HTTPS

如果你对你正在使用的服务器有所了解，更新你的网站以使用 HTTPS 并不难。你可以去 https://certbot.eff.org 这样的网站获取命令行所需的代码。或者您可以自己更改文件。

如果你使用的是托管包，并且有 cPanel 之类的东西，升级到 HTTPS 协议应该会非常容易。它们中的大多数都有某种自动功能，可以为您进行更新。了解你的服务器，如果你需要做这个更新，你将能够从那里找到细节。

考虑到它每天保护的数据量，HTTPS 是强大的。作为一个 web 开发人员，了解一些网络方面的东西也无妨。当你遇到一个你以前没有见过的问题时，这些额外的知识真的很有帮助。最起码，它确实给了你另一个寻找 bug 的地方。

作为一个警告，在你做这种更新之前，一定要备份你的网站。有时会发生奇怪的事情，它们可能会破坏你所有的文件或一些关键的文件。先花时间做个备份就好。
