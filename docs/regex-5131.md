# 正则表达式…

> 原文：<https://dev.to/flippedcoding/regex-5131>

正则表达式是您可以在 web 开发中使用的最强大但复杂的东西之一。他们天生就很神秘，需要一段时间才能弄清楚他们到底在做什么。这就是为什么在您最终得到一个无法使用的文件之前，了解它们是如何工作的非常重要。

首先，正则表达式通常用于字符串操作、验证以及最重要的搜索。它们帮助您快速浏览字符串值，并以狙击手的准确度从它们那里获得您需要的信息。不要误解我，它们不能解决你所有的问题。肯定有使用正则表达式的时间和地点。

正则表达式通常用于电子邮件验证。既然你迫不及待地想看看这个会是什么样子，那就来吧:

```
/^(([^<>()\[\]\\.,;:\s@"]+(\.[^<>()\[\]\\.,;:\s@"]+)*)|(".+"))@((\[[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}])|(([a-zA-Z\-0-9]+\.)+[a-zA-Z]{2,}))$/ 
```

这个正则表达式符合互联网人设定的所有安全要求。不要被这个吓倒。这需要一个有经验的 web 开发人员带着任何理解通读这个字符串。请记住，这只是 JavaScript 的正则表达式。如果你用另一种编程语言如 C#来做，你可能需要用不同的方式格式化正则表达式。

正则表达式最强大的用途之一是能够加快搜索速度。如果你能写一个好的正则表达式，你可以梳理数百万个字符串，找到你需要的确切信息。

这也是谷歌能这么快找到你要找的东西的原因之一。另外，有不同的正则表达式会返回相同的结果。这意味着可以编写正则表达式来提高搜索效率。

您可以使用 regex 查找数字和字母的任意组合。如果您需要处理日期、密码、IP 地址或 URL，正则表达式也可以处理它们。他们可以帮助你抓取网页或者只是简单地验证表单输入。这里有几个简单的正则表达式，用于日常编码事务:

```
Numbers: /^[0-9]+$/

Letters: /^[a-zA-Z]+$/

Numbers and letters: /^[a-zA-Z0-9]+$/ 
```
