# 如何在一个小时内找出你的知识差距

> 原文：<https://dev.to/ashleemboyer/how-to-identify-your-knowledge-gaps-in-under-an-hour-1b06>

*最初发布于[我的个人博客](https://ashleemboyer.com/how-to-identify-your-knowledge-gaps-in-under-an-hour)。*

科技世界的广博知识可以让人不知所措。人们总是在 Twitter 上抛出流行词汇:Kubernetes、Rails、React、Angular、Elixir 等。这些都是什么，其他人是怎么知道的？

如果你像我一样，不知道你所在领域的许多人属于同一个行业，那么通过定期讨论，很难弄清楚每个人都在炒作什么框架。不过，没必要那么难。互联网就在这里！你可能会想，“但是我怎么知道用什么词来搜索我不知道的东西的结果呢？”

太简单了，你脑子里已经有答案了。去一个求职网站(我推荐 [Glassdoor](//www.glassdoor.com) ，因为它能帮助你更好地了解薪水以及公司是否可信)，搜索你目前的职位或者你感兴趣的职位，然后抓取一些好的职位列表。

你会想看看比你当前职位更高的职位列表。比如作为一个初级的前端 web 开发人员，我可能会看一些高级的职位。再举一个例子，我真的很想学习更多关于 web UI/UX 的知识，但是我不知道任何术语，也不知道这方面的工作是什么样的。在这种情况下，我可能会搜索“UX 研究员”、“UX 设计师”或“UX 实习生”。

利用所有这些华而不实、令人生畏的工作清单来为自己谋利。流行语越多越好。此外，不要害怕梦想成真。谁知道你会遇到什么？
