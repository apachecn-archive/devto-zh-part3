# 抽象的力量

> 原文:[https://dev.to/acoh3n/the-power-of-abstraction-1gpa](https://dev.to/acoh3n/the-power-of-abstraction-1gpa)

如果有人问我“在所有软件开发中，最重要的概念是什么？”我会用一个词来回答:“抽象”。

抽象就是把一个想法简化到它的绝对本质。

我们来看一个例子:以跑步这个活动为例。如果你仔细看看你跑步时身体里到底发生了什么，你的头会旋转起来:无数的肌肉保持你的身体平衡，你的心脏疯狂地泵血，你的肺全速工作以保持你的血液氧化，你的汗腺向你的皮肤分泌汗水以保持你的凉爽。

但对你来说，这只是简单的*运行*。你的头脑简单地从跑步的概念中抽象出或去除了不必要的细节，这样你就可以很容易地思考它。

让我们看另一个例子。下面是一些代码:

```
var splitString = str.split("");
var reverseArray = splitString.reverse();
var joinArray = reverseArray.join("");
return joinArray;
```

现在让我们添加抽象:

```
function reverseString (str) {
  var splitString = str.split("");
  var reverseArray = splitString.reverse();
  var joinArray = reverseArray.join("");
  return joinArray;
}
```

在第一个例子中，我们必须按顺序阅读每一行代码，并跟踪正在发生的事情，以便理解它。如果这是一段更大的代码的一部分，我们可能必须在软件的整个生命周期中一遍又一遍地做这件事。

然而，在第二个例子中，我们给这段代码起了一个名字，让我们忽略内部细节并记住基本概念:反转字符串。啊哈！

这个想法对计算机(实际上也是对生活)如此重要，以至于当你思考一会儿，你会意识到你每天使用的硬件和软件是由软件的“层”(或抽象)组成的。从硅片到机器码，到汇编代码，到操作系统，再到你的应用程序代码，每一层都可以很容易地推理出来，而不用让我们的脑袋被细节弄得爆炸。

在软件中创建清晰的抽象是软件设计的本质。这是可理解的、易于维护的软件和需要处理的噩梦之间的区别。
