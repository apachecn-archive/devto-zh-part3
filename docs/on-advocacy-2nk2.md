# 关于宣传

> 原文：<https://dev.to/zellwk/on-advocacy-2nk2>

当我们谈论我们关心的事情时，我们是激烈的和保护性的。作为开发人员，我们非常关心这些主题:

1.  易接近
2.  Web 性能
3.  半铸钢ˌ钢性铸铁(Cast Semi-Steel)
4.  Java Script 语言
5.  结构
6.  包容性和平等

表达你的观点没有问题。我们都有自己的观点，我们希望别人听到我们的观点。我理解并尊重这一点。

但是我们应该注意我们说话的方式。如果我们以抱怨、辱骂和羞辱的方式表达我们的观点，我们的观点不会被听到。不会发生任何变化。它们只会造成担忧、仇恨和焦虑的恶性循环。

如果我们改变表达意见的方式，我们就能带来真正的改变。我们的文化会变得更好。

## 恐惧和变化

如果你公开叫某人出来，他们会有什么感觉？

惭愧。愤怒。被误解了。

他们不同意被叫出去。骂人和欺负人差不多。你用同龄人的压力迫使他们屈服。

它不起作用。

如果你想让人们听你说话，他们首先需要知道你理解他们。这就是骂人无济于事的原因。它把你放在了你应该在的相反的方向。

骂人的另一个问题是。

那些真正想做出改变的人变得害怕这样做。他们不敢寻求帮助，因为如果他们做得不好，可能会被公开羞辱！

我们的文化受到冲击。社区受到了打击。

与其恃强凌弱，更好的办法是温柔地、悄悄地去 ping 人，问他们是否需要帮助。如果他们不想要帮助，那就这样吧。

## 什么是倡导

我认为倡导不是抱怨或羞辱。它是关于**给予指导和指示**。这是关于耐心和真诚。这是关于**帮助人们朝着你倡导的方向前进。**

许多优秀的人在这方面做得很好。举几个我心目中的英雄:

1.  关于 CSS 和布局的瑞秋·安德鲁
2.  海登·皮克林、[莱奥妮·沃森](https://twitter.com/leoniewatson)和[斯科特·奥哈拉](https://twitter.com/scottohara)谈可及性和包容性。
3.  杰克·阿奇博尔德和 T2【艾迪·奥斯马尼谈 JavaScript/Web 性能。

当然，还要特别提到我的好友静，她谈论了全世界的 CSS 和排版。

(外面有很多很棒的人。很抱歉我不能包括大多数人)。

## 对明目张胆的冒犯者

大多数人想要并寻求改变。我们只需帮助他们清除面临的障碍。

不幸的是，世界上的一些大公司确实滥用他们的权力。这些明目张胆的犯罪者不接受真正的帮助——他们认为自己不需要帮助。

如果大公司继续滥用他们的权力，我们的世界将在我们知道之前就变成废墟。在这些情况下，我们有责任站出来与他们作斗争。我们必须通过骂人来做到这一点。我们甚至可能不得不组织一次抵制(如果我们有能力的话)。据我所知，这是不可能的，这很不幸。

但是对于想要改变的普通大众来说，温和的版本更有效。

所以请注意你在和谁说话，你在延续什么。

文字有力量。

我想用电影《惊奇 T1》中的一句话来结束我的演讲。我喜欢这条规则，并一直尝试着去应用它。

> 善待他人，因为每个人都在打一场硬仗。如果你真的想知道人们是什么样的人，你要做的就是...就是看。

要善良。
