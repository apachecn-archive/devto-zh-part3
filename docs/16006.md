# 你还用 macOS 应用启动器吗？

> 原文:[https://dev . to/preslavrachev/do-you-still-use-a-MAC OS-app-launcher-2 NPD](https://dev.to/preslavrachev/do-you-still-use-a-macos-app-launcher-2npd)

曾经有一段时间，我可以发誓效忠一个或另一个应用程序启动器，只要它不是 Spotlight。起初，我相信，几年前我开始使用免费版本的 Alfred ，但是当我意识到 Alfred 仍然非常依赖 Spotlight index 做任何事情时，我转向了 [Launchbar](https://obdev.at/products/launchbar/index.html) 。当时，为了节省 CPU 周期和硬盘空间，我已经手动[关闭了](https://www.iclarified.com/49187/how-to-disable-and-reenable-spotlight-indexing-on-your-mac)spot light 索引程序(是的，那是很久以前的事了)。Launchbar 建立了自己的索引，它非常快，而且精确到一个应用程序启动器需要的程度。

我仍然使用 Launchbar 作为我的日常应用启动器，尽管我已经多年没有禁用 Spotlight 索引程序了。我仍然不喜欢 Spotlight 的用户界面，但我意识到除了应用程序本身之外，索引还被更多的应用程序使用。

我有兴趣知道你是否还在使用 macOS 的自定义应用启动器。如果是的话，我想听听你的理由，也许还能从你那里得到一些新的有趣的自动化技巧。请在评论中给我留言。
