# 拥抱技术而不是解决问题

> 原文:[https://dev . to/bel hassen 07/enchanging-technologies-而不是-solvingproblems-lfa](https://dev.to/belhassen07/embracing-technologies-rather-than-solvingproblems-lfa)

我职业生涯中犯过的最严重的错误之一是，我如此热爱技术，以至于我成了趋势和酷技术的学习机器，尽管我没有看到他们试图解决的问题，但他们围绕着他们大肆宣传。

[![Another Javascript framework and blockchain based product](../Images/f57b474903b4c13ddd1df3147811a9b9.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--R2nfpNMW--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/800/0%2Akp5gK8vqZxSdms-x)

我认为这是有害的，原因有二:

*   首先，这将使你作为一个终身学习者的学习之旅变得压力重重。我敢打赌，你确实看到了你需要学习的技术是多么的无穷无尽，而且它正在把你拆散。
*   第二，你很可能会制造以技术为中心的产品，这些产品不能解决用户需要解决的特定问题，而是围绕着新技术的宣传的不太有趣的产品。

## [](#so-what-should-we-do%C2%A0)那么我们该怎么办呢？

对于学习困境这一部分，接受这样一个事实:你不可能学会即将到来的每一项技术，无论是编程语言、库、软件、工具还是其他任何东西。即使它是一项技术也没关系，它可能是一些概念。你不可能学到所有的东西，这没关系。事实上，你不应该这么做。

接受这样一个事实，你正试图解决一个特殊的问题来缓解某人的生活。这听起来很理想化，但却是真实的。有时很明显，但有时你必须深入挖掘才能发现。

事实上，我们所做的事情并没有人性化的一面，它的核心是人性化的，这对于作为产品设计师的我来说很自然，但我相信无论你在做什么，你总能被重新发现。

因此，从现在开始，如果一项技术出现或成为趋势，问自己这个特定的问题:
这将帮助我解决我感兴趣的产品类型，还是解决我遇到的特定问题，例如我的工作流程？
如果不是，就不要学，专注于你已经知道的技术或工具。不要什么都懂，什么都不懂！我将在下一篇文章中讨论这个问题。
