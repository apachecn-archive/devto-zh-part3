# 按功能封装与按模块封装

> 原文：<https://dev.to/rcosteira79/package-by-feature-vs-feature-by-module-5387>

大家好，

最近我一直想开始一个新的个人 Android 项目，在参加了 GDG 开发者大会后，我终于找到了这样做的灵感。

我习惯于按照“按特性打包”的逻辑来组织一切，但是我最近读了一些文章，在这些文章中，人们在谈论将每个特性放在自己独立的模块中。

现在，从架构上来说，这似乎是一个非常好的选择(高内聚、松耦合、特性重用等等)，但可能是因为我习惯了单一的 Android 应用程序(尽管组件之间有明确的界限)，我很难不认为这是一个过度的选择。

你们谁有这方面的经验吗？或者你能看出什么缺点吗？我认为依赖注入和导航可能会成为一种挑战。

我可能最终会自己尝试一下，但我想听听你对此的想法。谢谢！:)
