# 当转行从事软件开发时，如何给自己树立品牌

> 原文：<https://dev.to/jamesmh/how-to-brand-yourself-when-switching-careers-into-software-development-2ajb>

几周前，我收到了一个来自 [*导航您的软件开发职业*](https://eepurl.com/gdIV5X) 时事通讯的读者的问题，是关于当转行成为一名软件开发人员时如何定位或标记自己的:

> “我已经阅读了您关于作为开发者在 Dev.to 上进行在线展示品牌化的帖子💪伟大的阅读！
> 
> 我可以问你的推荐吗？
> 
> 我假设你的帖子是为已经从事专业工作的开发人员准备的。
> 
> 对于完全是初学者的代码新手来说——刚刚开始学习编码，并想写一个博客(无论是自己托管的，还是使用 Dev。to / Medium)，在从非技术领域过渡到软件工程的同时，如何给自己‘打上品牌’？"

爱它-伟大的问题！

# 非技术背景

读到这里的许多人可能已经是开发人员了——但是我确信你们中的许多人在成为开发人员之前实际上已经有了“前世”。

举个例子，当我回到大学学习如何编码时，我才在 25 岁左右开始真正的编码！

我在十八九岁的时候使用过计算机，玩过 Unix 机器和一些 bash 脚本——但仅此而已。

我的背景实际上是音乐和哲学🤔。

在开始编码之前，我计划完成我的学业...直到我认为再花 7 年时间去获得博士学位不值得花时间、金钱等等。

所以我不得不考虑其他可能的职业:

*   不需要大量的学校教育
*   我热衷于
*   可以研究我住在哪里
*   总的来说报酬不错

那正好是编程！

# 简答

这是我对这个问题的回答:

将自己定位/标榜为一个正在将职业生涯转向软件开发的人。

嗯（表示踌躇等）....🤔

# 长回答

这是我在 LinkedIn 上看到的一个很好的例子——亚历克斯·奥尔蒂斯。

他最近一直在博客上写他尝试进入更多技术/开发职位的旅程——作为一名非开发人员。

我认为这是一个奇妙的方法。

## 战术

当你正在学习做出改变时，你可以通过以下方式公开你的旅程:

*   博客
*   不时更新短视频
*   在推特上发布一些小的更新，比如你正在学习什么，什么是难的，什么是容易的，等等。

通过给自己打上一个传统营销者(如果这是你的领域)的烙印，你将会转向开发职业，你可以这样推销自己:你是什么！

毕竟品牌是关于你的声誉的，你聪明吗？激情？有帮助？值得信任吗？正宗？诚实吗？你帮助谁？

关键是围绕你是谁，你去过哪里，你要去哪里建立一个社区和声誉。

注意:一个长期的方法实际上应该围绕着其他人在哪里以及你能在哪里找到他们。但是现在，开始吧，专注于你自己的旅程是一个自然而伟大的方式。

那些正在经历同样过程或者考虑从一名护理人员转行成为一名开发人员的人将会**吞噬**你的内容！

有很多人都在这样做——从非开发人员的角色转换到学习和开始软件开发的职业生涯。我认为这是新兵训练营如此受欢迎的原因之一。

我从与美国一个主要的新兵训练营的交谈中了解到，他们有很高比例的学生从以前的职业轨道转向其他职业。

# 下一关

如果你真的很想建立一个声誉，并试图帮助其他人实现这一转变，你也可以开始电子邮件通讯。

每周或每两周发送一次关于你的旅程、建议的更新，甚至采访其他已经成功完成相同类型转变的人。

最终，你也可以在这个市场上自己出版一本书。你已经有一群想要消费你的内容的追随者了！

# 一些最终的想法

最后，作为一项要求，大多数公司都在寻找熟练的开发人员。

但最终得到工作的人是那些能够向潜在公司证明自己价值的人。拥有良好的声誉和追随者真的可以帮助你做到这一点，脱颖而出。

通过这种方式，你将获得更多的联系，如果你在旅途中有了追随者，你将处于进入这个领域的最佳位置。
