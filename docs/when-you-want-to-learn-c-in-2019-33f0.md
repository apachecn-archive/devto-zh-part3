# 2019 年想学 C 的时候

> 原文：<https://dev.to/benjcal/when-you-want-to-learn-c-in-2019-33f0>

我们中有多少人有过这样的美好回忆:敢于编写超过 20 行的代码而不进行编译，只是为了在`segmentation fault`下面看到一个嘲讽的闪烁的 bash 光标。哦，怀旧！

但是说真的，C 语言似乎对你不利！它用微不足道的 32 个关键词诱惑你，然后用`*`和`&`拍你的后背。

我不知道这是因为我喜欢痛苦，还是因为我仍然崇拜在 Slackware 上运行的`vi`时代，在内核 2.2 和 ext2 不断崩溃的时代，但我最终决定跳到 c。

我一直在“发现”的是，C 的美妙之处在于它不会试图对你运行程序的系统隐藏许多细节。它假设你知道你在做什么，你知道什么东西可以存储在 L1 或者内存中！我想权力越大，责任越大是真的。

之所以要学 C(我是一个爱上 JS 和 React 的 web dev，Rust 让我心潮澎湃！)是因为 Linux。我已经使用 Linux 很多年了，现在我想更深入地了解系统调用和内核等等。即使在更高的层次上，我们仍然是 gstreamer、Glib、Gtk+和 libssl、readline 和 wayland 的 C 领地...

在许多低级的 Linux 世界中，c 仍然称王称霸。

在 2019 年学习它似乎有点像一个挑战，因为你似乎会发现两个阵营，一个说“C 是朋克摇滚”(21 世纪 C)，而另一个阵营说在阅读 K&R2 之前不要害怕键盘。一个希望你用 C 和一堆库开发高级的东西，而另一个告诉你如何在 200 倍于 RAM 的 L1 上存储东西。

不管怎样，我真的很喜欢从这本神奇的 tlpi 书中学习更多关于系统的知识。欢迎任何建议、想法或资源！
