# 管理你作为一个网页开发者的声誉

> 原文:[https://dev . to/flipped coding/managing-your-reputation-as-a-web-developer-2714](https://dev.to/flippedcoding/managing-your-reputation-as-a-web-developer-2714)

作为一名网站开发者，你必须关注自己的声誉，因为大多数时候这是你的全部。有一些认证，但不是所有的公司都承认它们，这使得证明你了解自己的东西变得有点困难。这里有几件事可以让你得到技术社区和付费用户的青睐。

## [](#produce-quality-code-consistently)产生一致的质量代码

你的代码说明了一切。如果你的网站和应用程序很烂，那就意味着你的代码很烂。能够说服客户和雇主雇佣你始于你向他们展示的东西，这将是与你的代码相关的东西。即使他们只是问你编程问题，你的代码仍然是相关的，因为那是你汲取经验的地方。

不应该有某些人谈论你在一个项目上的工作有多棒，而另一些人谈论你的编码技能有多糟糕。一致性是编写高质量代码和被视为编写高质量代码的人的关键。把一个项目做得非常好并不是逃避下一个项目的借口。保持一致。

保持最新是保持代码高质量的另一种方法。编程语言的一些更新修复了安全问题，或者使编写某些功能变得更容易。你的代码应该表明你能够跟上时代，并且你正在努力继续学习。

## [](#talk-to-people)与人交谈

参加专业活动是建立你作为开发人员的声誉的最好方式之一，尤其是如果你在就业市场上。网络是大多数人不喜欢做的奇怪的事情之一，但它确实有效。只要坚持出现几次，如果你不先说话，就会有人和你说话。

这是你了解该地区 web 开发情况的方法，你可能会遇到一些最终与你一起工作的人。与他人交谈并倾听他人的意见会教会你作为一名开发人员真正需要知道的东西。你还会教其他人你知道的事情，这对他们的帮助可能比你知道的更多。

过一会儿，人们会开始认出你，然后你知道，你就是人们要找的人。很多时候不是你认识谁，而是谁认识你。确保你因为正确的原因而出名。

## 永远完成你说过的话

不要承诺你无法完成的最后期限。不要接你应付不了的客户。接受挑战和让自己失败是有区别的。接受挑战会将你推向极限，但是在七天的期限内接受一个你从未见过的语言的项目是非常失败的。

你的声誉很大程度上取决于你的客户和雇主，因为他们几乎在整个过程的每个阶段都会看到你的工作。当你坐下来告诉他们你能做什么时，这正是他们所期望的。当你开始错过交付日期或提交半成品时，你就得罪了不该得罪的人。

了解自己的技能水平，并据此制定计划。我的一位导师总是告诉我，少承诺多兑现比多承诺少兑现要好。接受它的价值。🤷‍♀️，不要贪多嚼不烂，你在这里会过得很好。另一方面，不要忘记不时接受挑战！

## [](#keep-growing)保持增长

作为一名网络开发人员，你永远不会停止学习新的东西。将会有新的编程语言、新的框架和我们还没有想到的新东西。这是 web 开发令人兴奋的部分。它永远不会变老，你也不会。

跟上时代是必要的，它显示了你的适应能力。这就是为什么你的声誉取决于它。如果你只向人们展示你 10 年前做的工作，他们会认为这就是你的症结所在。学习一些较新的框架，因为它会派上用场。

重点是向人们展示你在不断学习，并且乐在其中。一旦人们看到你喜欢不断前进和成长，他们会觉得你知道自己在做什么，并开始对你有信心。

作为一名 web 开发人员，你的大部分声誉将来自于你的工作和与你一起工作的人。你可以在没有太多名声的情况下完成你的职业生涯，但当你有一个好名声时，事情会简单得多。🙂其他人有过好的或坏的网站开发者声誉的经历吗？
