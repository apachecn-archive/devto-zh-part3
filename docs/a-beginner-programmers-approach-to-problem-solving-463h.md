# 初级程序员解决问题的方法

> 原文：<https://dev.to/neumand/a-beginner-programmers-approach-to-problem-solving-463h>

我想分享我那位经验丰富得多的计算机科学朋友给我的一些关于用编程方式解决问题的建议。希望你和我一样觉得它很有帮助！

作为一名面临需要解决的入门问题的初级开发人员，您可能会发现自己会立即考虑如何实现代码来提出解决方案。例如，在我即将到来的训练营的准备模块中，我需要解决经典的“嘶嘶”问题。如果你不了解它，你可以在这里找到它的例子[。](https://www.hackerrank.com/challenges/fizzbuzz/problem)

您的第一反应可能是考虑如何实现满足所有需求的功能。我是否运用了我所学到的关于循环的知识？我知道这里有很多场景...如何在我的代码中实现条件？

虽然你可能会问自己这些有效的问题，但这里有一个重要的要点:**花一些时间思考这个问题，不要在脑海中编码**。思考问题更广泛的含义，而不是一头扎进编码中找出解决方案。把它想象成在你采取行动之前制定一个计划。这样做会帮助你建立一个解决问题的框架。对于“fizzbuzz”问题，您可能会认为这有点过头了(不管怎么说，我是这么认为的)。但我知道我的朋友在压力很大的算法面试问题中使用了这种方法，并取得了巨大的成功。

我知道这对你们中的许多人来说可能是常识，但是我发现这个建议对于初学者来说非常有价值。希望你也是。
