### [](#what-to-do-after-the-coding-academy)考上编码学院后做什么

大家好！大约 5 年来，我拥有并经营着一家开发商店，在那段时间里，我有幸看到许多开发人员在进入工作场所的过程中提高了他们的技能(无论是专业技能还是个人技能)。然而；我很快注意到一些来自编码学院和训练营的个人的趋势。

在教我们年轻的学徒时，我很快发现大多数编码学院的课程都缺少很多信息。我把这归因于程序太短，试图一次把这么多知识塞进他们的大脑。大多数人获得了“什么是”软件开发的知识，但是缺乏“如何”使用它的智慧。

如果这是你(或你的一个团队成员)——你并不孤单。在 3-6 个月内成为一名完全成熟的软件工程师几乎是不可能的，尽管像 NYDCA 这样的地方希望你相信并非如此。或许这就是他们关闭的原因。但不要害怕；有希望！编码学院是你旅程的一个很好的*起点*，只要接下来有正确的指导和指引。

所以，我想就“下一步做什么”提供指导和指引。我希望这篇文章能吸引毕业生，以及那些负责培训他们的高级开发人员和经理。我们都在这份事业中，所以让我们互相帮助，让这份事业对每个人都很棒。

*附:如果有人有你认为有用的附加物品；我将更新这篇文章，使之成为最佳实践/智慧的结晶。*

### [](#step-1-learn-the-true-fundamentals)第一步——学习真正的基本面

你可能认为你在你的编码学院学到了基础知识；但是遗憾的是，你实际上学习了 web 开发的基础知识。你还没有学到的是计算机科学的“如何”和“为什么”的基础知识。诸如“什么是 IP 地址？”，“什么是 var(引擎盖下)？”，以及“我如何逐步调试这个问题？”从来没有人问过也没有人教过，所以我们有责任去解决这个问题。

为了获得这种理解，我建议回到过去，这样你就能理解未来。脚本语言不是学习基础知识的地方；然而，C++是建立在基础之上的。你可能再也不会使用这种语言了——但是我希望你花 2-3 天的时间来完成下面的教程。我保证它们是有益的——以下是步骤

1.  下载 Visual Studio 代码
2.  [带自己浏览一下这个 C++教程](http://www.learncpp.com/#Chapter0)
3.  完成第 0、1、2、4、5、6、7 章:
4.  完成这些章节时，请回答以下问题:
    *   你从这一章中学到了什么？
    *   在这一章中，你遇到了什么问题/困难？
    *   在这一章的最后，你想尝试什么问题或事情？

### [](#step-2-create-a-very-simple-c-application)第二步——创建一个非常简单的 C++应用程序。

我建议如下:

创建一个应用程序

1.  当通过命令行启动时，会问“你好，你想创建什么？”1:一只狗”，“2:一只猫”
2.  当用户键入“1”并按回车键时，就会创建一个 Dog 类。
3.  在 Dog 类的构造函数中，有一个打印函数“我是一只狗，我出生了”
4.  在猫类构造函数中，有一个打印函数“我是猫，我出生了！”
5.  抽象狗和猫类成为动物类的一部分，并从它继承。
6.  完成其他一些小的改动，以反映您刚刚在教程中学到的内容

### 步骤 3 -将 C++应用程序翻译成 Web 应用程序。

现在，使用您在编码学院学到的技术创建一个 web 应用程序——但不是命令行，而是通过 Web 浏览器的 GUI。Angular 应用程序或 Rails 应用程序都可以，然后回答以下问题:

*   C++和 javascript/ruby 相比如何？
*   有什么区别？
*   哪些部分比较容易？哪个部分更难？
*   让一个朋友在两个方面都介绍一个错误，但不告诉你在哪里。现在，试着调试两者的错误。你在调试中看到了不同吗？

### [](#step-4-understand-the-internet)第四步——了解互联网。

大多数人不明白互联网到底是什么；这真的很神奇。你知道在网络上发送的每个数据包中都隐藏着这些数据吗？你知道你的 ISP 是怎么工作的吗？好吧，首先让我们阅读一下斯坦福大学关于互联网到底是什么的白皮书。

### [](#step-5-understand-your-environment)第五步——了解你的环境。

这一步是有争议的一步，最好在你不再使用的旧电脑上完成。如果你没有，确保备份你当前系统上的所有东西。

1.  (假设你正在创建一个 Angular 应用程序)下载并安装家酿、节点、NPM、鲍尔和咕噜。
2.  尝试安装并运行[一个平均堆栈示例应用](http://mean.io/)
3.  卸载节点的当前版本，并安装旧版本。通过 [NVM](https://github.com/creationix/nvm) 进行版本控制
4.  尝试运行相同的应用程序。
5.  现在，在别人的电脑上做同样的事情。不管是什么类型，所有操作系统都有学习功能:)

你可能有问题，也可能没有——猜猜是什么；对于许多开发人员来说，版本依赖是一个巨大的挑战。5 使用 MAC 的人可能都有不同的环境——这还不包括所有的 Windows / Windows + Ubuntu / Linux 发行版。

1.  (奖金)擦你的电脑-并重新安装一切。这是有争议的部分，但通常当你在做开发时，你不太理解文件路径和 chmod/sudo 结果，你会弄乱你的开发环境。你可以花几个小时来调试它，或者你可以删除并重新启动-你的选择。

现在，您一定在想“一定有一种标准化环境的方法”。是的，有很多方法。一个众所周知的方法是一个叫做 [Docker](https://medium.com/phytochemia-tech-blog/how-docker-can-improve-your-development-environments-731cdfda0b9a) 的工具。我会让你读完剩下的。

### [](#step-7-practice-being-systematic)第七步——系统地练习

注意这一点——这是我能给你的最重要的建议。

当你在处理一个特性、调试或者任何类型的开发时——在每一个有意义的改变之后提交。不要等待特性/bug 完成。

计算机科学是一门*科学*，科学是系统性的。我们尝试，测量，然后得出结论。我向你保证，如果你把大的提交和大的问题分解成超级小的问题；你会以 10 倍的速度完成任务，并因此而更加快乐。
