# 提示我希望在开始学习软件工程时就知道

> 原文：<https://dev.to/christopherkade/tips-i-wish-i-knew-when-starting-studying-software-engineering--4pkk>

今天早些时候，我在我的[网站](https://christopherkade.com/student-tips)上发布了这篇文章，并希望在这里分享，以获得大家的一些反馈。我希望你喜欢！

* * *

总的来说，计算机编程是一个奇怪的领域，不是吗？我们可以在很多方面提高自己，然而，学生们往往会忽略一些重要的技能，而这些技能将在他们早期的职业生涯中改变游戏规则。

正是这种心态促使我在 5 年前开始攻读软件工程硕士学位时，给年轻时的自己写了一封信。一些可能会让我在课堂内外占上风的建议。

我希望这篇文章能被那些正在开始软件工程这一令人惊奇的有趣、令人沮丧和有益的旅程的人所阅读，同时记住我只是从专业角度开始。这意味着下面所有的提示都来自于我与天才工程师的互动，这些互动推动我成为一名更好的开发人员、教师和导师。

## 提示 1:你可以不断提高你的技术

持续改进是我们工作的一部分，而置身事外会很快让我们不知所措。尤其是在 web 开发这样的领域，每周都会有新的框架和库出现。

提高你已经知道的东西的一个方法是通过结对编程或 mob 编程向更有经验的开发人员学习，参加会议和看/读文章(你已经在正确的轨道上了！).

但是没有什么比做一个有实际最终目标的具体项目更好的了。始终努力改进你的代码、设计模式和选择。如有疑问，请随时在 [StackExchange](https://stackexchange.com/) 等平台提问。

## 秘诀 2:虚心接受(建设性的)批评

当我遇到断然拒绝建设性批评的开发人员时，我总是很沮丧(不管它是如何提出的)。他们的思维方式不允许他们改进或发现解决问题的新方法，他们往往是一个痛苦的工作对象。

以开放的心态去批评，把你的骄傲从你的代码中分离出来，这能帮助你提高，并让你立刻成为一个愉快的共事者。

作为一个轶事，我曾经和另一个开发人员一起做一个大的年终项目，姑且叫他约翰吧。约翰固执己见，不听我们对他的设计选择提出的意见。所以最终，我们停止了和他一起做代码评审(哦，我希望我们没有)。这导致我们在交付前两天重构了他的代码库。两个漫长的日日夜夜，充满了大量的咖啡和沮丧，如果约翰接受建设性的批评，这些都可以避免。

请记住，不是每个人都知道如何建设性地提出批评，你会遇到说话严厉的人，他们觉得有必要通过质疑你所做的每一个选择来夸耀自己的经验和才华。你最终会不可避免地遇到这样的人，你只需要学会处理它。谢天谢地，以我的经验来看，大多数开发者都是好意。

## 小贴士 n 3:不要害怕提问

同样，你应该**永远不要**害怕问问题。起初，它们可能听起来模糊不清，但是随着你知识的增长，你的问题的质量也会提高。

如果你周围都是良师益友和有经验的人，这一点就更适用了，因为他们中的大多数人都非常乐意在具体问题上分享他们的知识。

相信我，下一次你有技术问题时，四处问问，看看人们会如何团结在白板前帮你解决。你最终会获得更多的知识，更重要的是，你会知道如何综合你的想法。

这让我们想到...

## 技巧四:学会如何综合你的想法

能够综合你的想法意味着只给出必要的信息就能直奔主题。这将有助于你的日常生活，正如你在本文中可能已经理解的那样，与他人交流是必不可少的。

但是合成当然很难，真的。尤其是在试图对复杂的概念进行分析时。我发现自己试图解释问题或练习，却看到我前面的人眯着眼睛试图理解。

提高你的综合技能的一个好方法是观察别人是如何做的，并做笔记。另一种方法是向不从事软件工程的人解释技术概念，这将迫使你找到相关的例子让他们理解。

## 提示 5:开始为开源做贡献

我写了一篇关于如何开始为开源软件做贡献的短文，这可以让你走上正确的道路，但为什么贡献对学生有益，你将能够:

*   提高你现有的技能
*   结识新朋友
*   寻找导师
*   提高你的软技能

另外，为这个生态系统做贡献真的很有回报，招聘人员和科技公司也很认同这一点。

## 秘诀六:保持好奇

我的意思是，永远探索新技术。

你的研究可能给你机会发现软件工程的各个方面:系统、编译器、嵌入式、web、应用程序等。这很好，这是决定你未来专业的最佳方式。但是，即使在你的学习之外，也要试着跟随教程学习新的语言、框架和库(这最终会带来一个完整的个人项目)。

获得灵感的好地方是 Github 的[探索](https://github.com/explore)页面。

最后，但并非最不重要...

## 提示 7:寻找个人项目去做

我已经能听到你说“*但是克里斯，我是一名学生，我没有足够的时间做兼职项目*”，我真的听到了你的话。但这次相信我:为他们腾出时间。

就像开源贡献一样，副业是提升自己的最好方式之一，也有助于未来的招聘者注意到你的好奇心、技能和热情。 [Github](https://github.com/christopherkade) 是一个展示你的副业项目的好地方，谁知道呢，也许你的某个项目会成功并产生影响！

毕竟，这一切都始于一个想法和一些动机。
