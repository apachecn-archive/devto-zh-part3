# 用 Octobox 解开你的 GitHub 通知

> 原文:[https://dev . to/octo box/untangle-your-github-notifications-with-octo box-5 CIA](https://dev.to/octobox/untangle-your-github-notifications-with-octobox-5cia)

作为 GitHub 上许多开源项目的积极贡献者和维护者，我努力跟上各种正在进行的问题，并在多个存储库中拉请求，发现很难看到我所参与的一切的概览。如果你管理或参与 GitHub 上的任何项目，我敢打赌你可能也有同样的问题。

这是因为 GitHub 通知被标记为已读，一旦你加载页面或查看通知的电子邮件，它就会从列表中消失。这使得你很难掌握哪些通知你还需要跟进。

大多数开源维护者，甚至一些 GitHub 员工最终都使用 Gmail 中过滤器和标签的复杂组合来管理他们收件箱中的通知。如果像我一样，你试图避免电子邮件，那么你可能想试试 [Octobox](https://octobox.io) 。

[![](../Images/52fd0647ff847a7580bd558546e364d1.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Bwdr98y---/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/2800/1%2AJ2VYzaXsvfrGPzjcZVxB0w.png)

Octobox 的设计正是考虑到了这个问题。首先，它为每个通知添加了一个额外的“存档”状态，以便您可以将其标记为“完成”。如果线程/问题/pr 上出现新活动，相关通知将会弹出到您的收件箱中。您还可以启动专门针对您的通知。

这使得何时清除通知的控制权回到了你的手中，你可以按照自己的节奏处理收件箱。这也意味着你总是可以找到旧的通知，在 GitHub 上，在你阅读它们几天后就会消失。

GitHub power 用户面临的另一个问题是他们收到的通知数量，有些用户每天会收到 200 多个通知。Octobox 通过多种方式帮助解决这个问题:

*   过滤器:你可以用任何你能想到的方式过滤收件箱里的通知；按存储库、组织、类型、操作、状态、CI 状态和原因，并将来自机器人的通知与您的常规标签、作者和受托人一起保存。

*   多选:一次将一大堆通知归档并静音，让您轻松掌控最嘈杂的存储库。

*   搜索:结合各种强大的搜索过滤器，帮助您直接获得您正在寻找的通知，并专注于您所需要的，然后将您最喜爱的搜索锁定在侧边栏，以便于访问。

*   键盘快捷键:使用受 Gmail 启发的键盘快捷键，像专业人士一样快速导航、分类和管理您的通知，无需鼠标。

Octobox 于 2016 年 12 月作为一个辅助项目诞生，从那时起，它已经发展到帮助超过 10，000 人管理超过 400 万个通知，并且该项目已经被拥有自己版本的开发者下载了近 50 万次。它现在是 GitHub 上最受欢迎的开源工具之一。

上个月[本和我宣布](https://medium.com/octobox/goodbye-tidelift-hello-octobox-60c4eafc4507)我们将开始全职工作，使 Octobox 成为一个真正可持续的开源项目，能够在经济上支持它自己和社区。

今天，我们正在扩大 Octobox 和[* *在 GitHub Marketplace](https://github.com/marketplace/octobox) **上发布的范围，在 [Octobox.io](https://octobox.io) 上为私人存储库提供新的付费增强功能，有两周的免费试用期。

你也可以通过[捐赠或成为 Open Collectiv](https://opencollective.com/octobox#) e 的赞助商来获得私有存储库的相同增强，价格相同，但支持直接提供给社区。

*   稍后再休眠通知

*   Gmail 风格的自动通知过滤器

*   本地化和国际化

*   团队讨论和直接的用户对用户信息传递
