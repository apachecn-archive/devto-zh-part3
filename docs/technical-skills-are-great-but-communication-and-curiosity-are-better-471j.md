招聘和面试开发人员的话题已经在主要舞台上出现了很长一段时间。招聘和面试过程是建立多元化高效团队的关键。但是这个话题也有一些非常不好的宣传。

有时候面试过程充满了奇怪的技术挑战，这些挑战在现实生活中没有任何立足点。其他时候，面试过程过于松懈，没有把重点放在了解个人或他们是如何工作的。在我看来，这两种选择都不好。

在前一种情况下，你是在根据假设的情况排除候选人。这些情况实际上并没有评估和他们一起工作会是什么样子。向人们抛出奇怪的技术挑战很可能会失败。这是因为它不是基于团队需要什么和他们如何融入团队来评估人。

后者也好不到哪里去。软件开发需要一种想要学习和挖掘问题根源的诀窍。它还需要能够沟通和表达你正在做的事情。因此，评估一个人的学习和交流能力与评估他们的技能一样重要。

这就是为什么艾米丽·弗里曼的这条推文引起了我的注意。

她雇佣的三个特质是我们在整个面试过程中都应该评估的。

技术技能很棒，软件开发也需要。但是，沟通和好奇心是可以为团队增加指数级技能的重要技能。

但是，这是为什么呢？

### 好奇心培养技能

学习编程是你开发生涯的第一步。

技术技能会随着时间而改变。科技领域可能在几周内发生变化，因此人们今天拥有的技能可能在下个月就会过时。

要想在职业生涯中获得成功和发展，你需要能够适应这些变化。对正在发生的变化和原因保持好奇。好奇心能让你快速启动自己的学习。

好奇心不仅仅是对新兴技术感到好奇或感兴趣。这也意味着你很想了解你遇到的某个特性、错误或支持问题背后的*为什么*。

在我做软件工程师的时候，和我一起工作过的最好的人在务实和好奇之间有着难以置信的平衡。他们走在过度工程化和规模足以制造伟大产品之间的界线。他们还有一种天生的好奇心，想在更深层次上理解概念、问题和特性。

毫不奇怪，这些工程师也有广泛的技术技能。了解和尝试不同方法的好奇心推动了技术技能的进步。

雇佣那些有足够好奇心去深入挖掘的人。这些人非常有能力学习你的团队需要的任何技术技能。他们也会有好奇心去理解事物为什么会是现在这个样子，并帮助提出不同的解决方案。

### 传播传播知识

在软件开发团队中，20-25%的时间用于编码，70-75%的时间用于交流。

交流有多种形式，有太多不同的受众。这可能是面对面的，你正在对一个棘手的 bug 进行编程。或者你正在给你的管理团队写一封电子邮件。无论哪种方式，清晰简洁的交流都是至关重要的。

能够调整与不同受众的沟通也很重要。有时团队领导或产品经理不需要精细的技术细节。你必须能够在微观技术层面和宏观层面与事物对话。

沟通做得好，增加了团队的知识。也许这是一个快速的`README`来配置你的开发环境的必要步骤。或者它可能是 Slack 上的一篇关于新兴技术 X 为什么有趣的快速帖子。

这听起来像是一个相当简单的技能，但是沟通就像其他技能一样是可以学习的。因此，有些人很擅长，有些人很糟糕。当它存在的时候，团队可以茁壮成长，当它消失的时候，他们可能完全是垃圾箱火灾。

寻找能够交流想法和思维过程的人。确认他们能够以易于理解的方式向不同层次的人传达复杂的想法。

### 结论

当涉及到开发团队的招聘时，我们往往会非常关注技术技能。但是还有一系列其他的技能更加重要。比起你是否了解 framework X 或者 tool Y，交流和好奇心是我更看重的两个技能。

为什么？因为这些技能会直接影响我和团队。

有好奇心深入挖掘问题并探索新想法的人会让其他人在其他问题或想法上做同样的事情。但这只是等式的一半。他们还必须能够交流他们的发现、想法和想法，以便其他人可以使用它们。

这两项技能突出了一个人在开发团队中是负责任的，有责任感的，并且能够交付的。
