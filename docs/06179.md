# 故事 1:#工作中的开发人员。请注意！

> 原文：<https://dev.to/vjnvisakh/developer-at-work-attention-please-5a2k>

[![](img/392bc7bd88e524f57a7273784499a1f8.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--e4KSd75I--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/9p2qdycpc3mgf5cuc1mj.png)

我们开发人员是如此沉迷于我们所做的事情，以至于我们很少注意我们在一天中实际说了什么。如果你注意的话，这里有一些你能听到开发者说的最重要的事情。

1.  你合并我的请求了吗？
2.  这不是一个 bug，而是一个新特性。
3.  这太愚蠢了。这是什么样的测试案例？
4.  我们以后会处理的。
5.  戴夫:不！这是不可能的。TL:一个小时之内就可以完成。
6.  为什么互联网不能工作？
7.  我需要再来一杯咖啡。
8.  工作差不多完成了。我明天会把它送去测试！
9.  让我们再优化一下。
10.  那一点也不有趣。
11.  没有日志文件。
12.  点击 Fiddler 并检查 API 是否工作。
13.  它在我的机器上工作。
14.  更新您的浏览器。
15.  USB 线松了。
16.  编辑又挂了。
17.  请删除 node_modules 文件夹，然后重新安装。[阿德勒·科埃略·桑托斯]
18.  还有很多很多...

如果你也有一些有趣的，请加入这个列表！
