# 书评:Kurt Gunteroth 的优化 C++

> 原文：<https://dev.to/sandordargo/book-review-optimized-c-by-kurt-gunteroth-3ad0>

在我参加了一个关于优化 C++的培训后，我觉得我有兴趣更深入一点。至少要多读一点这个话题。所以我向培训师要了一些他推荐的关于优化的书。一个是 [Kurt Guntheroth 的优化 C++](https://amzn.to/2vGyHRT) 。我一读完[敏捷开发人员必备技能](https://devreads.sandordargo.com/essential-skills-for-the-agile-developer/)，就开始读这本。

我喜欢这本书，但老实说，到最后我觉得有点失落。这只是意味着这本书从简单的想法开始，走向复杂的想法。显然，我没有花足够的时间去理解最后两章关于并发和内存管理的内容。我没有在一个需要那里描述的技术提供的好处的环境中工作，这是我的借口。

另一方面，在这本书的其余部分，我发现了许多建议，这些建议现在或在不久的将来对我有用。Guntheroth 解释了为什么优化很重要，何时应该开始优化，以及应该如何做。他详细介绍了不同排序和搜索算法、动态变量分配、数据结构等的成本。他从最常见的解决方案走向罕见的解决方案。

完全说得通。很多时候你不需要什么花里胡哨的东西，只需要复习一下你的算法。我记得在我编程生涯的最开始，我用 O(n**4)而不是 O(logn*n)做一些事情。当这个应用程序非常慢的时候，我被叫了出来。我做了一些测量，结果发现我们花在那里的时间不到 1%，正如我所料，而超过 95%的时间是在第三方库中，该库没有得到很好的记录，几周后我们意识到我们没有正确清理它，事实上，我们清理得太频繁了。

这是非常重要的一点。不要徒劳地优化，当你优化的时候，衡量一下效果。我非常喜欢这本书的一点是作者告诉我们他的假设和失败。他解释说，他预计一种数据结构会比另一种数据结构好几个数量级，但结果只是快了一点。或者在其他情况下，甚至更慢。这给了他可信度，也强调了实验的重要性。

我想我会把[优化的 C++](https://amzn.to/2vGyHRT) 放在我的(虚拟)书架上，每当我遇到需要更高性能的热门代码部分时，鉴于这本书的清晰结构，我会知道在哪里打开它以获得一些好的建议。
