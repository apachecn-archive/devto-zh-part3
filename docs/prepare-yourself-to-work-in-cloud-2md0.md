# 准备好在云环境中工作

> 原文：<https://dev.to/cmendibl3/prepare-yourself-to-work-in-cloud-2md0>

如今，很明显，无服务器计算、容器化、云原生应用、DevOps 人工智能、机器学习以及混合和多云解决方案等 it 趋势正在成为主流或“新常态”。所有企业，无论大小，都在寻找在这些热门趋势中拥有关键技能的云专家。使用无服务器架构，应用程序被分成由不同类型的事件触发的各种独立功能。这改变了我们通常对应用程序的看法，也改变了我们在监控性能、跟踪、扩展和高可用性方面处理运营的方式。对 [AWS Lambda](https://aws.amazon.com/es/lambda) 、 [Azure 函数](https://azure.microsoft.com/es-es/services/functions/)、[谷歌云函数](https://cloud.google.com/functions/)和事件驱动架构的良好了解是雇主们将寻找的技能。

容器化和云原生应用都是关于使用公共云提供商的服务和技术，如 [Docker](https://www.docker.com/) 和 [Kubernetes](https://kubernetes.io/) ，部署和运行分布式应用和微服务。这种结合提高了工作效率和质量，使企业变得更加敏捷，通过自动化、应用生命周期管理、服务供应和同质运营培养开发运维文化。请注意，您的潜在雇主将寻求精通微服务、容器和容器编排引擎(如 Kubernetes)的 IT 专业人员。

在企业范围内执行人工智能、数据分析和机器学习需要 CPU 能力和存储，而这种能力和存储通常无法在内部提供。这是采用公共云的主要驱动因素之一，当然也是您在规划职业道路时需要牢记的领域之一

如今，世界各地的企业都在寻找新的方法，以变得足够灵活，根据总体成本、位置、数据保护法、安全性、涉及的技术和许多其他因素来选择在哪里运行工作负载。这就是为什么许多公司将赌注压在混合云和多云模型上，部署公共云和私有云的组合。因此，组织正在寻找能够使用各种云提供商和自动化工具的管理语言的 IT 专业人员，如 [Terraform](https://www.terraform.io/) 或 [Ansible](https://www.terraform.io/) 。

作为这种情况的结果，你必须建立或更新你的技能，以跟上这些永无止境的技术和文化变革的步伐，而你选择的道路将取决于你当前的技能和经验。巴里·布里格斯(Barry Brigs)和爱德华多·卡斯纳(Eduardo Kassner)在他们的著作《企业云战略》(Enterprise Cloud Strategy)中写道，在这一演变过程中，你的“现有 IT 技能将保持不变，但价值将低于以云为中心的新技能”。

对于如何实现这种转变，没有经验法则，但是如果你是一名开发人员，你可能会专注于开发业务架构师、解决方案架构师或 DevOps 相关的技能。业务架构师计划技术资产和业务流程的发展。解决方案架构师专注于设想和增强专注于特定领域的应用程序，同时为新特性和功能的开发提供监督和指导。DevOps 技能与一场关于合并开发和运营团队的运动有关。

如果你是一名测试人员或运营专业人员，你可能会发现自己在创建自动化配置、部署或监控脚本、编排流程、使用云提供商工具，甚至编写云应用程序，在某种程度上，开发人员、测试人员和运营角色之间曾经清晰的界限开始变得模糊。

如果您是 DBA、统计师或数据分析师，您会希望专注于数据科学家或信息架构师的角色。大数据是一个巨大的话题，如果你有兴趣学习，请查看奥里奥尔·托雷西拉的帖子:[如何成为大数据专家](https://blog.everis.com/en/blog/technology/how-become-big-data-expert)。
