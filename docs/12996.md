# 超越教程:程序员解决问题的课程

> 原文：<https://dev.to/aspittel/moving-past-tutorials-a-course-on-problem-solving-for-programmers-3oa4>

在我与初级和中级程序员一起工作的经验中，编程最困难的部分是把程序的各个部分放在一起，并知道实际使用哪个部分。而且，大部分编程课程都不教这部分。他们单独教授这些作品，然后期望学生们把它们组合在一起。所以，我正在建立一个多媒体课程，它将完全专注于解决问题，如何超越过去的教程，以及为什么在编程时应该做出某些选择。

本课程将采取开发系列的形式，因此它将完全免费，任何人都可以访问。我将每周发布一个新帖子，这样我就有时间从上一周的帖子中获得任何反馈，并将其纳入下一篇帖子，同时平衡创建内容和我的其他工作。

我坚信要让不同学习风格的人都能接触到内容，所以这门课将会非常互动，并使用不同的媒介来强化材料。大部分的概念将会被书写和图表化。然后会有概念中的例题让你解决，最后会有视频解释你自己尝试后如何解决练习题。

这门课基本上是语言不可知的，尽管我会用 Python 来解决我的问题，因为如果你懂不同的编程语言，这是很容易理解的。不过，这门课不会教授编程的基础知识。在开始之前，我会学习基础知识，至少是最基本的能力。在本课程之前需要的基础知识是:变量、列表/数组、哈希表/字典/对象(无论它们在你选择的编程语言中被称为什么)、循环、条件和函数。如果你想开始学习，这里有一个免费的学习资源列表。

主题将包括分解编码难题和更现实的业务问题、调试、抽象思维、伪代码、优化代码时要考虑什么、重构、解决问题、边缘案例，以及在代码之外要学习什么来帮助你编码。
