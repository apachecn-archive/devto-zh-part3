# 这一领域经历了一年左右的风风雨雨

> 原文：<https://dev.to/leppercameron/a-whirlwind-year-or-so-in-the-field-1mep>

我一直在积极阅读戴夫。在 Twitter 和网站上，有一段时间了，但这将是我的第一篇文章。所以记住这一点，请温柔一点！

我想分享我的故事会很有趣，就像我看到其他人做的那样，回顾我在计算行业的第一年(或者说，前 13 个月)。希望对于那些开始进入我们这个奇妙领域的人来说，这至少是一本有趣的读物。

关于我的一些背景，我学习了计算科学的本科荣誉学士学位，然后是商业计算的研究生硕士学位。本科期间，我在一家中型 IT 公司完成了为期十周的实习，在那里我做了一些软件开发。我不会在这里讨论这些，而是专注于我全职工作的第一年。

# 开端

2018 年 1 月，我是一家工程公司智能技术部门的应届毕业生。最初，我被聘为 Azure BI 开发人员，但随着最初几个月的过去，T2 BI 部分被放弃，我开始在 Azure Cloud 中开发软件。这是我开始意识到云计算，特别是微软 Azure 平台，真的让我很感兴趣。

随着我对云计算兴趣的增长，公司开始过渡到 Azure 的需求也在增长。这是一个互利的安排；该公司需要迁移到 Azure，我想学习如何坚持使用 Azure。我开始深入参与和接触很多新的 Azure 技术和工作方式；尤其是围绕微服务方法、容器编排和 CI/CD(使用 Azure DevOps/VSTS)。随着我在这个领域知识的增长，我的信心也在增长。去年 12 月，我们部门成立了一个新团队，主要致力于 Azure 开发。他们要求我加入他们的行列，我欣然同意。

自从加入这个团队以来，我的责任与日俱增，我已经深入参与了许多 R&D、创新，并试图高效地实现 Azure。直到本周一，我一直处于这个位置...

# 要约

几周前，一家公司的招聘人员找到了我，他们在这个领域非常有名，问我是否有兴趣跟他们聊一聊，解释一下一个空缺的职位，看看这个职位是否适合我。我解释说，我在目前的职位上很开心，获得经验对我很重要，所以必须有令人信服的理由才能跳槽。然而，我和招聘人员通了一个电话，发现确实有很多令人信服的理由要搬家。

提供的角色是 DevOps 工程师角色，开始专注于 Azure，但慢慢地纳入 GCP 和 AWS(因为他们是三大巨头的合作伙伴)。我在 3 天内总共接受了 5 次面试:

*   初次电话体验面试
*   一个小时的视频通话，探索一些技术示例并更深入地讨论背景
*   半天，大约 3 个小时，包括技术白板面试、背景和经验面试以及文化契合度面试。

面试过程非常有趣，令我惊讶的是，我觉得我已经完成了技术评估。本周我收到了一份工作邀请，将于三月中旬开始我作为 DevOps 工程师的新工作。我不能再快乐了。对于我来说，这是一个令人兴奋的新机会，可以在一个被云计算专家包围的环境中发展我的职业生涯。学习机会将会非常丰富。我等不及要开始了。

# 惊喜

我患有一种我喜欢称之为“年轻人综合症”的疾病，我的意思是我经常对自己的能力缺乏信心，通常是因为我觉得自己缺乏经验。直到我收到 DevOps 工程师技术面试的反馈，我才真正意识到，我似乎，不可思议地，实际上知道我在做什么。令人难以置信的是，一年过去了，我已经从一个完全的菜鸟变成了至少在一个非常具体的领域的技术面试中能够坚持自己的知识。

我这样说是因为，作为一个已经在这里阅读过这类博客一段时间的人，这类帖子是一种保证，表明我会喜欢开始我的技术职业生涯。我还是觉得自己很‘小三’(当然我也是)——但是知识来源于经验和应用，一年教会你很多东西。我很高兴能继续学习和成长，但我真的开始反思我在过去的一年里成长了多少。

# 群众演员

## 侧面项目

除了工作经验之外，我还有半年的志愿者经历，作为*founders 4 schools*initiative 的一部分，我可以走进学校，举办研讨会，激励下一代人参与 STEM 学科。这不仅是有益的，也是有益的——我与人们的交流和信心都得到了极大的提高，这是这种演示的直接结果。

## 提示和窍门

我做过的最好的事情之一就是为我大学毕业的朋友们创建了一个 Slack 页面。当我们有编码问题，或关于某件事的一般性问题时，我们互相标记并公开讨论。从我自己组织的泡沫之外的人那里学习是很棒的，从全国各地的资源中汲取真的帮助我对解决方案保持开放的态度。

# 结论

这些都是轶闻，所以不要把它们当成福音，但这是我要说的:

*   如果你刚刚开始，我能建议的最重要的事情是真正地把自己投入到这个角色中去，并且在你前进的过程中尽可能多的学习。不可能什么都知道。如果你有一个支持你的团队，你会在第一年成长很多。
*   找到一种方式来表达自己的想法。这是我在商业环境中与人打交道时增长自信的好方法。
*   你可能会发现，在早期，你最初的兴趣领域发生了变化。这是可以的。
*   有一天你会醒来，意识到有那么一瞬间，你会说‘哦，我的上帝，我似乎真的有了这个领域的线索。“这是怎么发生的，”。享受那一刻。
*   尽最大努力学习和参与 R&D 的活动。如果你在工作中没有机会，试着每周至少花几个小时在晚上/周末玩玩你感兴趣的技术。

在科技行业的第一年，我以一种前所未有的方式爱上了软件开发，发现并爱上了云计算和 DevOps 方法，并获得了宝贵的经验，这些经验帮助了我的技术能力，也帮助我成长为一个人物。

我非常高兴成为这个行业的一部分，现在正式成为 Dev 的活跃成员。敬社区。

感谢你花时间阅读我的第一篇博文。放轻松。

卡梅伦。
