# 个人开源项目如何帮助我学习围棋

> 原文:[https://dev . to/kilgaloon/how-personal-opensource-project-helped-me-learn-go-54b 9](https://dev.to/kilgaloon/how-personal-opensource-project-helped-me-learn-go-54b9)

我喜欢用 C 语言工作，我把 C 语言看作是对我个人的挑战，但一年前我想学习一些看起来和感觉上都像 C 语言的新东西，就在那时我发现了围棋。

我开始阅读并浏览围棋，但我觉得这还不够，如果你不采取实际的方法，做出一些东西，解决一些问题，思考解决方案并自己获得一些想法，你将如何学习。

这是我决定创造[小妖精](https://github.com/kilgaloon/leprechaun)的时刻，它最初是作为一个小项目开始的，你可以在我发布的第一个版本 [0.3.0](https://github.com/kilgaloon/leprechaun/tree/v0.3.0) 中看到，我不记得为什么我从这个数字开始，但管它呢。

这些是第一步，你还没有进入社区，所以你要订阅 reddit，去 youtube 看视频(对 Francesc Campoy 和他的 youtube [频道](https://www.youtube.com/channel/UC_BzFbxG2za3bp5NRRRXJSw)大喊，他帮助我更好地理解 GO，我强烈推荐它)。

此时，一个全新的世界展现在我面前。我开始发现如何构建 go 项目，你可以看看这里的，我开始在 GitHub 上查看一些 GO 项目，希望了解更多。

不久前，我注意到 GO 和我正在使用的其他语言之间的巨大差异，我在官方 GO 文档页面上花的时间比在其他任何地方都多，GO 的一个优点是非常非常好的文档，它非常容易阅读，我很高兴，我的脑海中没有 StackOverflow 的部分解决方案和想法，我不认为这是坏的，但看到的不能是看不见的， 尤其是当你刚开始时，你可能会被一些问题想法和解决方案束缚住，而你可能很难抛弃这些想法和解决方案，重新思考。

花更多的时间在 Go 文档上，我有更多的时间去思考我自己的解决方案，尝试它们，经历错误和尝试不同的方法，在开始时，我不考虑方法是好是坏，我试图让我的项目工作。

探索 GitHub 上的另一个项目我开始注意到我自己的项目的不同之处，我注意到我喜欢一些方法和想法，所以我以我自己的风格接受它们，实现它们并进行尝试。当你像我一样处于起点时，我可以说没有什么真正糟糕的路可以走。把每一个信息都当作储备，也许有用，也许没用，随着你的进步，你会发现的。

在我的整个项目历史中，我对一些包进行了彻底的检查，因为我提高了我的围棋技能，我得到了更好、更有效的想法，这是因为我不断地阅读和观看与围棋相关的东西，我不断地在我的项目上工作。

然后时间到了，当我的程序开始按我预期的那样工作时，是时候请某人给我一些提示或任何能帮助我的东西了。这是我第一次在 reddit 上发布的时刻，向人们征求代码审查、诚实的意见和任何可以帮助我的东西。我不害怕一些不好的评论，我是新来的，我准备把一切都看作是建设性的批评，我想利用这一点学习更多。

当我得到我的代码的第一次审查时，我开始改进它，一切都在不断改进，我把自己奉献给了这个项目，它只占用了我很少的一部分时间，但它给了我更多，让我学会了用新鲜和新的东西编程。

随着时间的推移，我开始享受与 GO 一起工作，每当我需要用 GO 做一些事情时，我都很期待，每当我学到一些关于 GO 的新东西时，我都感觉很好，语言很容易掌握，我开始觉得我知道它已经很多年了，而不是几个月了。

## [](#summary)总结

开始你的新项目会让你经历一些不好的东西、错误和实践，这将让你在未来的项目中变得更聪明和更强大，让你更明白在其他代码背后发生了什么。它将在你和你的项目之间建立一种纽带，并总是推动你做得更好。你将成为社区的一部分，在那里你可以体验其他项目并从其他人的错误中学习，找到一些好的内容，有时在空闲时间你可能会想“让我们找到一些与围棋有关的好资源”。毕竟，对我来说，我个人的成功是有人会发现你的项目很有用，并会使用它。
