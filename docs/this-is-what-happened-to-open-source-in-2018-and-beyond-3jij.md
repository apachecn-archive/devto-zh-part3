# 这就是开源在 2018 年(及以后)发生的事情

> 原文：<https://dev.to/davedri/this-is-what-happened-to-open-source-in-2018-and-beyond-3jij>

[![](img/cb4fd7041d8de25bf9828759bac88e69.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--Hu09C4nA--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/880/1%2AgA8Xjgtvl9h5nZbQ9EQdgQ.png)

看看最近的开源事件给了我们一些有趣的未来线索。尤其是 2018 年是开源成立以来最重要的一年之一。

许多高峰时刻都是新芽，表明开源将以有趣和不可预测的方式成长。同时也作为企业和社区创新的一种稳定形式扩展(和补充)了它的根基。

这些也是反复弹出的话题。在 2019 年初，我写道我将开始研究开源的未来。在这两个月以来进行的采访中，我不断听到同样的主题。

每个人似乎都深深地关注着这些主流中的至少一个。所以我认为挖掘不断涌现的十大主题是值得的——并询问 2018 年开源的重大事件对未来一年(或几年)可能意味着什么。我们走吧！

[![](img/ef46bc17ed36dc8a6bf11cc6ac33a222.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--ikpdbPhc--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/880/1%2AA2SGth6af4RDyVj-BGqlCw.png)

### 开源公司以创纪录的金额收购

Salesforce 在 2018 年 3 月宣布[以 65 亿美元](https://techcrunch.com/2018/03/20/salesforce-is-buying-mulesoft-at-enterprise-value-of-6-5-billion/)收购 Mulesoft。然后轮到 Adobe[宣布以 16.8 亿美元收购 Magento](https://techcrunch.com/2018/05/21/adobe-to-acquire-magento-for-1-6-b/) 。然后 [SUSE 以 25 亿美元卖给了 EQT](https://techcrunch.com/2018/07/02/micro-focus-sells-suse-for-2-5b/) 。这可是好几十亿美元。

如果这还不足以让机械键盘发出黑客新闻的话，IBM 宣布以 340 亿美元收购 Red Hat。在这些较大的数字中或周围是相对较小的开源公司的收购。2018 年初,[传出 CoreOS 收购](https://www.forbes.com/sites/janakirammsv/2018/01/30/red-hat-acquires-coreos-for-250-million/#64dfd6f12ecb)的消息，感觉已经是一千年前的事了。

所有这些活动为开源创造了积极的情绪——如果不是在这个过程中以溢价直接定价市场的话。就像 1999 年 Red Hat 创纪录的首次公开募股一样，水涨船高。尽管我们可能会忽略接下来的 2000 年到 2002 年。

### 重要的首次公开募股使更多的资本可用

说到 IPO，那里也有一些值得注意的活动。近 20 年来，红帽一直保持着[最大开源 IPO](https://www.cnet.com/news/red-hat-shares-triple-in-ipo/) 的记录——34 亿美元并非微不足道。直到 [Pivotal 在 2018 年的 IPO](https://www.forbes.com/sites/alexkonrad/2018/04/20/pivotal-software-ipo/#1a7c21c734c3)创下 39 亿美元的新高

有趣的是，这是一个完全不同的时代。红帽公司的联合创始人鲍勃·扬曾经向我解释过当年围绕他们首次公开募股的歇斯底里。当然，这是一家有着坚实数字的伟大公司——但它是[第一个筹集风险资本和首次公开募股的开源品牌](http://www.linux-mag.com/id/348/)。他们是一个全新垂直开放的象征。不难想象，开源世界内部的积极情绪与从外部发现新领域(和贪婪)的感觉交织在一起。市场做出了相应的反应。

更多的资本表面上是一件好事。在空闲时间编写的自由软件是伟大的。但是，拥有一份已经与开源使命一致的稳定工资对整个生态系统来说几乎是不可估量的重要。

同样，增加资本意味着获得必要技术的能力更强。虽然 Linux 获得了开源的历史性认可，但如果没有中间件、虚拟化或存储，很难想象过去十年的开源前景。这些知识产权投资中的一些可能不太像预期的那样是一场革命([你好，大数据](https://www.wired.com/insights/2014/04/big-data-big-hype/))，但其他如集装箱化等投资正在实时重写规则手册。

### Github 被微软收购

2018 年收购开源公司是一件大事。开源社区越来越依赖的平台的收购也是如此。当微软宣布以 75 亿美元收购 GitHub 时，我们看到了三件意料之中的事情发生。

第一个是社区中的一些人的立即反应，他们担心邪恶的微软会染指如此多的开源之家。这是一个合理的担忧[，但它并没有真正发生](https://www.techrepublic.com/article/pretty-much-no-one-quit-github-over-the-microsoft-acquisition-heres-why/)。

这可能与微软当初收购他们有很大关系——GitHub 不仅仅是一个存储库，还是一个工作流和社区。正如我们在 SaaS 生态系统中所说，这是一个非常粘的产品。由于所有这些开发者都坚持不走，微软突然宣布了收购后的第一个邪恶举动，即……[免费开放私有库](https://www.businessinsider.com.au/github-adds-free-private-repositories-2019-1?r=US&IR=T)。

接下来是 [关于微软拥有 GitHub](https://www.quora.com/Will-Microsoft-ruin-Github) [将意味着什么的问题](https://www.zdnet.com/article/microsofts-github-acquisition-its-all-about-developer-relationships-influence/)和[热点](https://www.reddit.com/r/github/comments/8og3xf/why_is_microsoft_acquiring_github_bad/)。一些伟大的见解，但随着时间的推移会泛出来。

最后是来自 GitLab 的[投资](https://about.gitlab.com/2018/09/19/announcing-100m-series-d-funding/) [公告](https://techcrunch.com/2018/09/19/gitlab-raises-100m/)(跟随收购定价的机会，为竞争市场注入活力)。这很有趣，因为 GitLab 不仅仅是一个竞争性的代码库，而是一个针对整个 DevOps 生命周期的产品。和开源。注意这个空间。

### 对可持续商业模式的探索势头强劲

除了 GitLab 的融资，我们还看到了其他值得关注的风险举措。Tidelift [刚刚在 5 月为其开源管理](https://news.crunchbase.com/news/tidelift-raises-25m-series-b-just-seven-months-after-last-funding/)[订阅服务](https://tidelift.com/subscription)筹集了 1500 万美元，又宣布了 2500 万美元。

这是开源的可持续性话题中为数不多的好消息。早在 2016 年，Nadia Eghbal 就称这个问题为“[互联网最大的盲点](https://medium.com/@nayafia/how-i-stumbled-upon-the-internet-s-biggest-blind-spot-b9aa23618c58)”。纳迪亚[清晰地谈论着这个话题](https://omny.fm/shows/future-of-coding/31-sustaining-the-underfunded-nadia-eghbal)，她的“柠檬水摊”是[开源财政支持的便利指南](https://github.com/nayafia/lemonade-stand)。

Nadia 的工作之所以突出，是因为她在多个角度的经验，从 VC 到 GitHub 的一部分。我自己在 [SustainOSS 峰会](https://sustainoss.org/about/)上的观点，由我在报告中提交的[项目总结而成。像 Tidelift 这样的其他公司正在采取企业的方式来提供社区资源。](https://sustainoss.org/assets/pdf/SustainOSS-west-2017-report.pdf)

这并不像人们想象的那样普遍。Open Collective [的 Xavier Damman 写了一篇很棒的文章，解释了支持一个项目的个人和品牌激励。当然也不乏善意的](https://medium.com/open-collective/a-new-way-to-fund-open-source-projects-91a51b1b7aac)[小费罐](https://gratipay.com/) [平台](https://liberapay.com/)、赞助和 [bug 奖金](https://www.bountysource.com/)。总比没有好。

但是 Tidelift 的例子反映了开源在企业中的兴起。如果不是 Red Hat 有能力将 Linux 提升到一个值得信任的企业关系中…这个行业会有今天吗？红帽公司从企业领域获得$3B 收入的例子是不是比制造 30 亿小费罐更好的目标？

无论维护开源世界的方法的最终组合是什么，毫无疑问的是[风险投资产品线从中受益](https://medium.com/@nayafia/open-source-was-worth-at-least-143m-of-instagram-s-1b-acquisition-808bb85e4681)(但不支持它)，答案很可能是积极参与市场，而不是向它寻求施舍。搜索仍在继续。

### 莱纳斯退出 Linux

开源并不总是一个很好的工作场所。邮件列表甚至会议都是令人沮丧和疲惫的事情。敬业的贡献者甚至来自文化内部的[都可能被耗尽](https://news.ycombinator.com/item?id=8415603)和[关掉](https://modelviewculture.com/pieces/leaving-toxic-open-source-communities)——更不用说好奇或者[偶然的探险者](https://www.quora.com/Why-are-open-source-projects-hard-to-understand)。尤其是那些来自创世纪文化规范之外的人。

这其中很大一部分来自于第一波开源和自由软件身份的浪潮，它在一个反文化的时代(作为一种行为)塑造了自己的形象。这样一种新兴的文化需要塑造一种身份。这种围绕着教条的正义之战建立的文化通常会这样做，而且[反对一个明显的恶棍](http://techrights.org/2010/05/12/microsoft-subverts-open-source-label/)。

然而，我们现在处于一个战争已经结束的时代，[开源已经赢得了](https://www.wired.com/2016/08/open-source-won-now/)，相互之间成为一个混蛋可能有点不太好。

莱纳斯·托沃兹在 2018 年 9 月宣布他将“休息一段时间，并在如何

理解人们的情绪并做出适当回应方面获得一些帮助”时承认了一些事情。

可以想象，人们对此会有不同的感受。但是世界并没有停止，最终…

### Linux 探索新的行为准则

托瓦尔兹的声明发布后，新的行为准则出台了。这些反应在预期的行为范围内。我们生活在有趣的政治和社会时代，谈话反映了这些极端。

这是一个比博客帖子上的摘要列表更值得细致入微的话题。但是它与流行的政治，也许还有政治疲劳是分不开的，这种方式将在未来一段时间内影响开源文化。

### Redis 尝试更便捷的“开源”

从表面上看，这个问题似乎很简单。云供应商被指控从开源项目中获利，因为他们充实了自己的产品并争夺主导地位。特别是亚马逊。

再强调一下，如果基于开源开发模式的公司对亚马逊这样的公司使用他们的代码库而不付费或贡献感到恼火，这是可以理解的。尤其是当云领域日益增长的垄断给那些运营上游社区的公司带来压力的时候。

从这个角度来看，对开源许可证进行修改以停止免费下载是有意义的。也就是【Redis 在 2018 年做的事情。[种](https://www.theregister.co.uk/2019/02/22/redis_labs_changes_license_funding_60m/)。此举[在开源支持者中并不特别受欢迎](https://www.techrepublic.com/article/why-redis-labs-made-a-huge-mistake-when-it-changed-its-open-source-licensing-strategy/)。[完全没有](https://www.zdnet.com/article/the-battle-between-real-open-source-vs-faux-open-source-heats-up/)。

另一个观点是[这种感觉一直存在于开源软件中](https://www.itworld.com/article/2749046/are-platform-vendors-stealing-linux-.html)。其他人指出，专门用于阻止云供应商的许可证只是一家商业公司利用开源社区反对另一家。问题不在于 AWS 本身——[在于开源收入模式](https://www.techrepublic.com/article/open-source-has-a-problem-with-monetization-not-aws/)。

特别是考虑到 Redis 刚刚完成了一轮 6000 万美元的 E 轮融资，高盛私人资本投资公司和贝恩资本风险投资公司也参与其中。不一定是以热情的开源倡导而闻名的组织。

所有这些观点都有一个有趣的观点。这个问题的细微差别使得开源生态系统本身成为一个有趣的案例研究。比如…

### 开源双倍于被开源

关于像 Redis 这样的公司调整他们在开源许可上的立场的有趣的事情不仅仅是围绕它正在进行的讨论。这是开源本身核心的自我反省的严肃工作。

在这种情况下，由开源倡议发布的开源定义的[确认。这是一本引人入胜的读物。如果你想知道自由软件和开源之间的区别是什么，这就是你的入门书。特别是关于“信任”和“商业”的主题。](https://opensource.org/node/966)

这个问题有趣的地方在于，人们越来越意识到开源正在发展。总有一天，当前的许可证会更新以与之匹配。但是正如 OSI 所指出的，开源作为商业上成功的合作创造的推动者，其存在依赖于信任和在整体的更大范围内满足个人目标的能力。

这并不是说云供应商是吃白食的(尽管[他们现在比其他任何人](https://www.infoworld.com/article/3343038/cloud-vendors-are-whos-powering-open-source-now.html)贡献更多)。只是免费下载不是开源的一个 bug 它实际上是一个特性。虽然其中一个分布不均。

### 微软开源其专利(某种程度上)

对于一个从昏迷中醒来的人来说，关于微软开放其 6 万项专利的头条新闻会非常令人困惑。微软把这一切都给了开源？这个微软被这个家伙经营的[？](https://www.youtube.com/watch?v=EHCRimwRGLs)

谢天谢地，微软已经发生了变化(不仅仅是股价)。但还没到他们放弃专利的程度。为了让[确切地理解发生了什么](https://hackernoon.com/did-microsoft-really-just-open-source-all-its-patents-3e419ae1a439)，这个细微的差别需要更多的挖掘。

微软加入[开放发明网络](https://www.openinventionnetwork.com/)意味着微软正在制定一个公开的(和合法的)协议“不使用它的 60，000 项专利中的任何一项来反对其他 OIN 参与者使用或传播 Linux 系统’技术”。

关于微软有趣的事情是，它实际上“T0”听取客户和开发者的意见，并坚定地致力于 Linux 和其他开源程序。这是公司副总裁兼首席知识产权顾问[埃里希·安德森](https://twitter.com/erichandIPG)在宣布这一消息的新闻稿中引用的话。

这是一个微软拥抱开源，追逐亚马逊的云领导地位，在真正的重大创新转变之外[几十年的歌舞之后](https://www.youtube.com/watch?v=lAkuJXGldrM)重新聚焦，甚至令人信服地支持创业公司。不是以大多数项目运行的“哦哇免费积分”的方式— [，而是专注于联合销售](https://techcrunch.com/2018/02/14/microsoft-revamps-its-startup-programs-with-500m-commitment-and-new-co-selling-program/)。

对于开源社区来说，微软可能是一个理想的反派，可以找到它的力量和声音与之对抗。但是现在看起来他们正在扮演一个令人印象深刻的领导角色。

### 开源不再由开源社区控制

[软件吃了天下](https://a16z.com/2011/08/20/why-software-is-eating-the-world/)。开源是整个软件开发体验的支柱。开源的定义可能最近才庆祝了它的 20 岁生日，但它早已长大并搬出了家。

在以后的文章中，我会更深入地探讨这个话题。承认这里最大的困难是知道对开源的未来进行更广泛的讨论的需要会引起一些开源社区本身的反对。尤其是现在开源是为…所有人开放的。

像吉姆·怀特赫斯特的《精彩的 T2》和《开放组织》这样的书展示了开源作为一个社区模型是如何被战略性地应用到一个组织框架中的。然后是由政策和计划定义的 CapitalOne 的[开源文化](https://thenewstack.io/open-source-culture-starts-with-programs-and-policies/)。或者澳大利亚联邦科学与工业研究组织(CSIRO)的数字部门 data 61[发布了开放银行业的指导方针](https://www.computerworld.com.au/article/649169/data61-releases-draft-open-banking-apis/)。还是在澳大利亚，我们看到[开放政府伙伴关系](https://ogpau.pmc.gov.au/)发布[开放数据](https://data.gov.au/)，该数据在 2014 年的估值约为[240 亿澳元](https://blog.data.gov.au/about-open-data)。

这些努力似乎偏离了核心定义和起源或开源。但是，在构建协作和大规模贡献方面，它们都围绕着同一个重心运行。所有这些都是开放源码课程的适用性的例子，远远超出了文化的起源。
