# 面向开发者的播客

> 原文：<https://dev.to/nurul3101/podcasts-for-developers-3i79>

这是我关注和偶尔收听的播客的详细列表。

## [Syntax.fm](//syntax.fm)

作者@ [韦斯·博斯](https://dev.to@wesbos)和斯科特·托林斯基([@斯托林斯基](https://dev.to/stolinski)在推特上)
这是我个人最喜欢的一张。他们每周上传两个播客。
周一一个，20 分钟左右(急餐)
周三一个，1 小时左右。

## [The React 播客](https://reactpodcast.simplecast.fm/)

迈克尔·杰克森和陈宸
他们邀请了各种各样的 JS 名人和开发者，讨论 React 的未来

## [Devchat.tv](https://devchat.tv/)

他们主持各种与技术领域相关的播客，其中一些是

*   [我的 JavaScript 故事](https://devchat.tv/my-javascript-story/)
*   [JavaScript 咆哮](https://devchat.tv/devchattv_show/javascript-rants/)
*   [反应综述](https://devchat.tv/react-round-up/)
*   [JavaScript Jabber](https://devchat.tv/js-jabber/)

## [FreeCodeCamp 播客](https://freecodecamp.libsyn.com/)

Quincy Larson(FreeCodeCamp 的创始人)邀请了不同的开发者和工程师，他们分享他们的故事，讨论最新的技术进步和关于 Javascript 的一切

## [Codenewbie 播客](https://www.codenewbie.org/podcast)

它包含了人们在编码之旅中的故事

## [软件工程日报](https://softwareengineeringdaily.com/category/all-episodes/exclusive-content/Podcast/)

这些播客每天都有，涵盖了与软件工程相关的一切

## [现代网](https://www.thisdot.co/modern-web)

他们在罗利和湾区主持网络平台上各种主题的播客和活动

## [前端欢乐时光](https://frontendhappyhour.com/)

与前端相关的一切

## [全栈电台](http://www.fullstackradio.com/)

一个面向对构建优秀软件产品感兴趣的开发者的播客。每一集，亚当·瓦森都会有一位嘉宾加入，谈论从产品设计和用户体验到单元测试和系统管理的一切

## [JS 党](https://changelog.com/jsparty)

关于 JavaScript 的一切

## [JAMStack 电台](https://www.heavybit.com/library/podcasts/jamstack-radio/)

JavaScript-API-标记
