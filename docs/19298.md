# 从前端框架中挑选樱桃

> 原文：<https://dev.to/proticm/cherry-picking-from-front-end-frameworks-2neo>

你好，我的开发伙伴，欢迎阅读另一篇文章。

作为前端开发人员，我相信你已经是了，或者考虑到你正在阅读这篇文章，你有兴趣成为一名前端开发人员，当开始一个新项目时，人们通常会有许多疑虑和问题，但是最近的一个主要问题是这样的:*“哪个框架对我来说是最好的，它适合我的项目需求吗？”*

JavaScript 世界有很多有用的工具，但有时这是最让我们头疼的部分，因为它让我们不确定应该为我们的项目选择什么。

坚持到这篇文章结束，我们将尝试回答第一个问题。

在撰写这篇文章时，我想到了以下框架相关的里程碑，它们应该有助于我们决定选择哪一个:

1.  学习资源及其可用性
2.  背后的社区
3.  特征
4.  可用性和第三方库

在这一点上，我认为满足所谓的里程碑至少应该足以让您开始选择框架，但是在它们实现之前，特别是如果您是前端世界的新手，您可能会一头撞在桌子上。

# 学习资源及其可用性

打开这一章意味着你已经过滤掉了我们最喜欢的工具。

通常，每个框架都有自己的文档网站，这是我们学习新东西时首先访问的地方。主页可能很吸引人，设计也很好，但是你真正感兴趣的是文档是如何编写的。

此外，仅有干燥的文档是不够的。你仍然需要额外的教程、书籍和相关主题的视频。来自不同作者的文章也很受欢迎。

这里需要指出的是，如果您最终没有找到有用的东西，您正在尝试学习的框架可能是新的，还没有被社区采用。

如果我们看一下几个例子，我们可以说 EmberJS 是有据可查的。附加的代码片段很好地描述了这些特性。

Angular 和 React 遍布网络。有一些面向前端的网站没有至少一篇关于这两者之一的文章。就个人而言，我发现 React 文档网站更好，更容易导航，以便找到我们需要的东西。

另一方面，Vue 介于两者之间。它并没有遍布网络，但它可能会到达那里。文档网站写得非常好，并指导我们一步一步地从头开始，指出如何使用该框架。

你应该尽可能多地接触不同的资源。通过这种方式，你可以从同一主题的不同角度增加信息输入。有选择总是好的。

# 背后的社区

在决定为您的项目选择哪个框架时，这一部分可以发挥重要作用。从开发人员的角度来看，从最新的技术中学习一些新奇的东西可能很有趣，很酷，这很好，但是从业务的角度来看，这有点复杂。

原因是你的公司或客户更喜欢已经存在一段时间的东西。这样他们就可以确信这个框架已经被很好的测试过了，并且没有错误。

另一个与商务相关的问题是，例如，如果你决定辞职并换工作怎么办？在这种情况下，他们会被一个除了你之外没有人熟悉的框架所困。

这些问题可能会改变公司的游戏规则，但是对于开发人员来说，广泛采用的框架也可以带来一些好处。例如，如果您正在开发一个特性，并且遇到了困难，您可以很容易地从社区中找到指导和帮助。我的意思是，如果你选择 React，你肯定会找到各种各样的文章，以某种方式帮助你。

此外，如果你学习并擅长一些流行的东西，你会看到光明的未来，那里会有很多工作机会。

[![job](img/1e2329b114b214fa1843744b2ff7b487.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--SJxmKT-b--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/http://mediabites.com.pk/wp-content/uploads/2018/02/feat-img_job.jpg)

但是要小心，因为只学习角度或反应只会让你的职业生涯到此为止。

还有其他伟大的框架，它们的存在是有原因的。你应该在业余时间试着了解它们。这样你将获得有价值的知识和新鲜的空气，特别是如果你已经用同一个框架工作了相当长的时间。

# 特性

在这一部分，您实际上需要深入思考您的项目以及您打算通过构建它来解决的问题。如果你做错了这一部分，你最终会得到一个错误的框架，从长远来看，这会扼杀这个项目。

您可以选择一个还没有被采用的不太流行的框架，或者一个没有被很好地记录的框架，但是最糟糕的事情就是选择一个最终没有提供您项目需要的核心特性的框架。

这将迫使您编写导致错误和不兼容的解决方法。你可以用第三方库来解决，但是当然，所选择的框架必须足够灵活，允许在不中断工作流的情况下集成它。

熟悉框架的核心特性至关重要。你可以从阅读文件开始。通常有一个快速入门指南页面可以帮助你开始。您需要大致了解这个框架是关于什么的，以及它能提供什么。

如果你是一个有经验的开发人员，这部分应该相对容易，因为所有主要的框架都提供相似的核心特性。想想吧。作为一名开发人员，你的日常工作是什么？你用什么包？

在文档中，您应该搜索诸如组件、混合、HTTP 通信、模板和绑定、表单处理等主题。基本上，你作为一个开发者每天所做的一切。

如果我们看一下三个最流行的框架，React，Angular 和 Vue，它们都符合我们的要求。

值得一提的是 EmberJS 有自己的对象模型和一个叫做 Ember Data 的东西。这是 EmberJS 应用程序的数据层。类似的包可以在 ExtJS，Sencha 框架中找到，但是让我们不要走得太远，就此打住。

# 可用性和第三方库

最后，选择了所需的框架。现在，我们需要挖掘它，让它变得有用。这意味着我们实际上需要写一些代码。

[![usability](img/b8fd8dad7f43c6304e97bd326470504a.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--AuC6aT2x--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://encrypted-tbn0.gstatic.com/images%3Fq%3Dtbn:ANd9GcSm9FUHGUjNMPf7JVisluWn6-FDDJ_l1EwpsVyx62bubQuhzOyS)

您可以观看大量视频，并花几个小时阅读文档，但真正的见解只有在您亲自编写代码后才能提供。弄清楚是什么让一个框架变得可用是非常简单的。

在做一个项目的时候，花点时间看看花了多长时间让它开始运作。构建框架并作为项目包运行需要什么？实现一个特定的功能需要多长时间？它需要第三方库吗？

至此，您收集了关于框架及其设置的经验。如果它达到了您所要求的水平，那么您很可能会在即将到来的项目中再次使用这个框架。

最受欢迎的框架与框架所有者或社区构建的许多助手工具和包一起发布。随着框架越来越受欢迎，您将更容易找到这些工具。

重要的是要指出，不管你选择的框架有多流行，不管它有多少特性，你都有可能面临一些必须用某种外部工具来解决的问题。

集成外部库可能会很麻烦，这取决于你对框架和框架本身的了解。但是有一点是肯定的，如果每次当你的项目需要它的时候，你都花费数小时来集成它，那么它可能不是最佳的选择。

# 结论

选择前端框架是处理项目时最重要的事情。不管你是在做一个小项目还是大项目，不管你是独自工作还是作为团队成员，你都必须把这篇文章中描述的里程碑算进去。从长远来看，这可能会省去你许多麻烦。
