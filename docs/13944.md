# 单一设计模式[创造]

> 原文:[https://dev . to/its coders life/singleton-design-pattern-creation-209h](https://dev.to/itscoderslife/singleton-design-pattern-creational-209h)

动机:当你需要一个独特的对象

小心:在多线程中访问它的情况下

*   单例模式确保了对象的唯一性。
*   如果只能创建给定类型的一个实例，并且该实例不能被克隆，那么您就正确地实现了该模式。
*   单例实例必须是线程安全的，因为许多不同的组件可能同时访问它。
*   您必须为单例的初始化阶段以及对其公共属性或方法的任何访问实现防止并发使用的保护。

> 单例模式确保给定类型只有一个实例，并且该实例不能被复制或克隆。单例必须是线程安全的，因为共享实例可以被几个线程同时访问。

目标是创建一个只能有一个实例的类。但是在实施的时候也需要注意。我们必须禁止复制或克隆单例实例。我们只能使用引用类型，也就是类，因为值类型在赋值给新变量时会被复制。singleton 实例表示一个共享资源，依赖于给定类型的所有组件都可以使用该资源。这些组件可以从不同的线程同时访问单例实例。因此，单例类必须是线程安全的。我们必须防止并发访问、其实例化以及对其公共成员和方法的任何访问。

单例的例子:

*   文件管理器
*   记录器系统
*   分析系统
