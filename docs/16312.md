# 为什么你需要一个基本的 ML 理解

> 原文:[https://dev . to/Chris hunt/why-you-need-a-basic-ml-understanding-1 ngi](https://dev.to/chrishunt/why-you-need-a-basic-ml-understanding-1ngi)

在你职业生涯的某些时刻，你会发现一个软件、一个库或一项技术，而你希望几年前就已经知道了。你可以看到在以前的工作或项目中，它是如何节省你的时间，让你的生活变得更加轻松。

对我来说，Docker 和 MongoDB 都属于这一类。尽管我参加这两个聚会的时间相对较早，但它们对我日常工作和交付能力的影响迅速改变了我的工作方式。

机器学习是最新的。直到最近，我可能还有点天真地认为，ML 是那些拥有巨额预算和可以访问大型服务器的专业 ML 员工的大公司的专属。这是大多数与 ML 相关的科技新闻报道描述的情况，虽然在光谱的顶端这可能是真的，但它是光谱，基本知识并不难理解。

要明确的是，机器学习(及其所有子类)是一个庞大的主题，我并不是说你可以在几个小时内训练你的汽车走向无人驾驶。我的建议是，作为一名开发人员，你应该意识到它能做什么和不能做什么，它能给你(和你的公司)带来什么好处，如果你想实现一个基本的 ML 解决方案，你需要走什么路线才能实现。或者，换句话说，当你的经理/主管/项目经理不可避免地加入到对话中时，你应该有能力围绕 ML 进行对话，“我听说过很多关于机器学习的事情，我认为我们应该使用它”。

## [](#we-dont-need-machine-learning)“我们不需要机器学习”

也许你在一家你认为 ML 不能提供任何价值的公司工作。你不制造无人驾驶汽车，也不运行 Pb 级搜索引擎。然而，如果您的公司处理大量数据，ML 可能会让您受益。

让我给你举个例子，解释一下 ML 是如何打开其他思路的。

假设您的电子商务公司的销售部门要求在新客户第一次购买时发出警报，该警报带有一个标志，表明该客户是否可能成为长期客户，以便销售团队可以亲自跟进。我们所能做的只是几万现有客户的现有数据。

一年前，作为一名没有 ML 知识的经验丰富的工程师，我已经开始在现有客户中寻找模式，第一次购买，潜在的他们的位置和他们未来的历史。我希望编写一个包含多个`if`语句的函数来确定它们的状态。任务完成。

ML 的方法也是从查看数据开始的。我们分析数据，并将其整理成我们的模型能够读取的方式(“特征”是 ML 中使用的术语)，以及他们是否成为长期客户的结果(“标签”)，我们将了解现有客户的情况。然后，我们可以训练一个机器学习模型，将这些特征与标签关联起来。对于成千上万的客户，这种培训在一台像样的 PC 上不会太昂贵(可能与我们在媒体上给 ML 文章的印象相反)，然后我们可以测试这个模型以发现它的准确性。如果足够，我们的模型可以预测新客户是否有可能成为长期客户。

在这一点上，我们已经用两种不同的方法处理了同一个问题，并可能产生两种可行的解决方案。然而，六个月后，销售部门通知你，旗帜不像以前那么准确了。市场发生了变化，消费者的习惯也发生了变化，这需要调整。用我们的工程方法，我们可能不得不从头开始重写我们的函数。然而，使用我们的 ML 解决方案，我们可能只需要用新的、更新的数据来重新训练我们的模型，以便提高它的性能。随着时间的推移，我们可以用新数据来训练我们的模型，这样你的模型就会一直保持最新。

所以我说起来容易，举个例子就行了，但我知道你在想什么....

## [](#i-dont-have-the-skills)“我没有技能”

我也不知道。我不是一个数据科学家，我对 ML 的经验不超过每天与数据科学家一起工作，以及在 Udemy 上用 Python 学习的[基础 ML 课程，如果你想有一个基本的开始，我强烈推荐你。然而，我所拥有的是关于 ML 能解决的问题的知识。这是我建议所有开发人员都能受益的工具。有了这个，你就能认识到什么时候 ML 解决方案可能对你有用，可能对你更有效。](https://udemy.com/python-for-data-science-and-machine-learning-bootcamp/learn/v4/overview)

## [](#summary)总结

如果我已经说服你看一看 ML，我会推荐这个视频给你一些非常基本的代码示例。Laurence 在介绍 ML 方面做得很好，没有进入复杂的数学公式。他还很好地解释了 ML 相对于“传统编程”的症结所在。

最终，传统编程接受规则和数据，并产生答案。机器学习接受答案和数据，并产生规则(ML 模型)。

当需求变得越来越复杂或模糊时，这种方法尤其有用(正如在简短的电子商务示例中所检查的)。

将所有这些放在一起，我真的相信，那些能够围绕机器学习进行可靠对话的开发人员，因为他们了解机器学习可以做什么，将比那些不了解机器学习的开发人员在大多数工作场所以及进一步进入就业市场时具有显著的优势。这是一个快速发展的行业。
