# 开发自己的 CSS 框架教会了我什么

> 原文：<https://dev.to/jdriviere/what-developing-my-own-css-framework-taught-me-4hbk>

### **开头**

是的，我同意:这是另一个类似的帖子。是的，我知道:已经有很多这样的 CSS 框架了。是的，我知道:越来越多的网页设计者和开发人员正在远离这些框架，因为 CSS 语言的不断发展(随着变量和网格系统的引入)，以及越来越多的定制可能性，通过像 Wordpress、Wix 或 Squarespace 这样的平台。但这是我的旅程，我觉得有必要与大家分享。它是如何开始的？什么时候开始的？

在获得数字艺术学位后，除了绘画和制作 3D 模型，我一直对在媒体中运用我的创造力感兴趣。我热爱编码，我想把我在艺术方面的知识与它结合起来。因此，我的目光转向了网页设计。我在一个暑假的时间里自学了 HTML 和 CSS，并创建了一些非常简单的网站。随着我的进步，我注意到我的大部分代码越来越重复；这变得很烦人(当时我还不知道 CSS 框架的存在，比如 Bootstrap 和 Foundation)。只是后来，在那个夏天，当我搜索如何设计按钮时，我偶然发现了 Bootstrap！

起初，我爱上了 Bootstrap 但是这个框架下载起来太大了；随着时间的推移，我不得不不断下载 jQuery 来使用它的一些特性，这让我失去了兴趣。我开始探索其他人创建的其他框架，后来发现了布尔玛，以及其他一些不太为人知的纯 CSS 框架。但是我没有使用它们中的任何一个，而是考虑创建自己的框架。

### **级数**

[Sparkle.css](https://sparklecss.herokuapp.com) 的第一次迭代开始时是一系列类，用于风格化按钮和创建基于块的网格系统(很像`W3.css` -直到我了解到 flexbox 的存在)，用原始 css 编写，然后演变成一个更复杂的库，用于用 SASS 编写的 web 设计的多个方面。

我在这个项目上工作得越多，就越开始欣赏它。一旦我完成了我的第一个版本，我就想出版它。为了实现这个目标，我不得不自学如何使用 Git(hub)、Node、Gulp、Jade/Pug，以及一些`travis.yml`。

### **当下**

是一个我近期不打算放弃的激情项目。在整个旅程中，艰苦的工作和学习过程，错误和成功，挫折和宽慰，对我来说都太宝贵了。直到今天，我仍然致力于它，不断寻找方法来完善它，修复它的错误，改进它的文档，测试新功能，评估它的内容，确保它的一致性，改进它的工作流程，等等。最重要的是，用它来构建网络应用和网站！

### **未来**

我打算通过将它用作 React.js 和 Vue.js 的组件来进一步扩展`Sparkle.css`，我也已经有了一个使用我的框架的项目列表，你可以在这里找到[；我做的项目越多，我在页面上添加的内容就越多。](https://github.com/sparkle-css)

### **结论**

总之，我可以宣布我为我的项目感到骄傲。开始它可能会令人困惑，碰到路障可能会令人沮丧；但总的来说，它得到了回报——现在仍然如此！我仍在学习诀窍(尤其是在管理 bug 报告、pull 请求和贡献时)，我希望交付更好的东西。

如果你有足够的好奇心去尝试我的[框架](https://github.com/jdriviere/sparkle-css)，在你的下一个项目中，不要犹豫通过链接与我分享。而且，如果你允许的话，我甚至可以在`Sparkle.css`的官网上展示！:)
