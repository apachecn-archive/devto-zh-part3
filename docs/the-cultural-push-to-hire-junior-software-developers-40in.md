# 雇用初级软件开发人员的文化推动

> 原文：<https://dev.to/amberleyjohanna/the-cultural-push-to-hire-junior-software-developers-40in>

最近一个朋友和我聊天，有一个有趣的问题。

> 行业内正在兴起的“雇佣初级工程师”的背后是什么？

我们进行了一次长谈，我决定将其转化为书面想法。

## 行业背景

### 对高级人才的高需求，低供给

对软件开发人员的现有供应和不断增长的需求之间存在巨大差距。软件开发人员的职位空缺[比合格的人要多](http://www.arcgis.com/apps/MapJournal/index.html?appid=b1c59eaadfd945a68a59724a59dbf7b1)。美国劳工统计局预计，到 2020 年，T2 将有一百万个软件职位空缺。

对软件开发人员的需求比所有职业的平均增长速度都要快。这部分是因为在一个日益数字化的世界里，除了传统的“科技”公司，金融、零售、制造和其他传统行业
[的需求也在上升。](https://www.glassdoor.com/research/app/uploads/sites/2/2017/12/Final_GD_ResearchReport_5Disruptions2018.pdf)

这些统计数据支持了我敢说的大多数工作软件开发人员经常亲自观察和听到的轶事。尤其是通过 LinkedIn。

### 初级人才饱和

我们缺乏合格的人才来满足现有的需求。

回到 2014-2015 年，获得计算机和信息科学学士学位[的人数为 59581 人](https://nces.ed.gov/programs/digest/d16/tables/dt16_322.10.asp?current=yes)。这甚至还不足以缩小供应缺口。

我们现在的情况是，每个人都想雇用高级员工，但已经没有足够的高级或中级人才来满足需求。与此同时，供应并没有增长以满足需求，更不用说与需求增长成比例地增长了。

代码训练营的激增本身就证明了有一个市场可以填补这个空白。相比之下，由于训练营和寻求满足需求的事物，我们有饱和的渴望，初级人才。与此同时，找到一份高质量、初级、入门级的工作也是一个挑战。他们很难得到，那些在那里的人似乎在寻求远高于“进入”的经验。

## 我的职业背景

从我的角度来看，对我的职业道路有一点小小的改变。

进入这个领域需要敲开许多门，并试图弄清楚我需要做什么才能最终让我的开发人员 Card™️.进来

我甚至不记得我是什么时候开始摆弄静态网站的。当我工作的非营利组织(当时从事营销、品牌和平面设计)有一个网站职位空缺时，我欣然接受了。我得到了它——至少部分地，我相信——因为离开的人(造成空仓)为我主张，知道我能做什么。这并不是说我不应该得到这份工作——我有资格得到这份工作——而是因为非营利组织中没有人真正理解这份工作。)

这很好，但最终我因为没有人可以学习而感到沮丧。这是我自己能想出来的。我开始申请前端开发人员的职位。

在一家我很看好的公司，我面试了一个前端开发的职位。最终，因为我“全面的背景”，我得到了一个“技术战略家”的职位，部分是业务开发，部分是客户管理，部分是项目管理。我认为这意味着我还没有准备好成为一名专业的 web 开发人员(尽管这是我目前的全职工作。认知失调是惊人的)，这将是一个很好的垫脚石。在我开始的时候，这个机构没有女性开发人员。

这个职位没有满足我的要求——我知道这很令人震惊。当我在那里的时候，他们确实雇用了一个女性开发人员，她是一个了不起的、支持我的导师和朋友。她曾是一名社会工作者，通过参加训练营改变了职业。她给了我信心，让我做出艰难的转变，迈出这一大步。我离开了 DC，搬回了德克萨斯，去了奥斯丁的一个代码训练营。(Makersquare，在被 Hack 反应器获得之前，在被镀锌获得之前🙃).

在整个叙述过程中，我因不了解自己所不知道的事情而痛苦。我不知道如何到达我想去的地方，甚至不知道那个职位是什么样的。我从来没有想象过我现在的处境。

## 为什么推聘晚辈？

考虑到这种情况，我认为有一些主要的原因促使雇佣初级开发人员。

### 因为我们需要新的人才来发展我们的行业

高级工程师不会像成熟的雅典娜那样从以太中自发出现。

> ![Kate Heddleston profile image](img/9c3a8c60d16f69ac8c2062232d9a651d.png)！2018 年 02:17AM-13 Sep[![Twitter reply action](img/44d8b042100e231770330321e5b63d65.png)](https://twitter.com/intent/tweet?in_reply_to=1040061861817868288)[![Twitter retweet action](img/93d9c70ccc54851d2e8e881b53c21dae.png)](https://twitter.com/intent/retweet?tweet_id=1040061861817868288)[![Twitter like action](img/2e93f7775eadefab8bcd34a4542cc5a7.png)](https://twitter.com/intent/like?tweet_id=1040061861817868288)

我们需要培养初级人才，以满足现有和未来的需求。每个行业都需要这样才能长久。考虑到对高级人才的需求和短缺，我们希望在哪里填补这一缺口？

在谈到雇用初级员工时，我听到过这样的话:“如果我雇用一个初级员工，他们在获得一些经验后就会离开，那还有什么意义呢？”在我看来，这是一个无效的方式来提出这个问题。相反，也许是“我们如何才能创造一个如此充实和充满活力的工作环境，让一个低年级学生想要继续成长并从内部成为一名领导者？”制度知识是无价的，培育一种维持这种知识的文化是有意义的，而不是确保自我实现的预言得以实现。如果你营造了一种文化，让你的初级人才愿意留下来，并成长为高级职位，你可能也会更好地留住现有的高级人才。

这也与一个更大的行业问题有关:

因此，将这一点与围绕“什么是初级、中级或高级开发人员”的含糊不清结合起来。没有通用的资格，也没有公认的能力标准。

然后再结合这个统计数据:“男性只满足 60%的条件就申请工作，而女性只有满足 100%的条件才申请工作”

你可以说，无论如何申请都是女性的责任，但我认为这忽略了许多现存的歧视性社会现象。例如，哈佛大学、麻省理工学院和沃顿商学院的研究人员进行的一项研究发现，投资者更喜欢男性企业家的推介，即使推介的内容相同。

所以很多时候，男人会向上申请，女人不会。或者，即使两人都申请了，而且资格相当，男性申请人可能被认为更有资格或更有前途。

我认为这导致了“资历”的不均衡加速，但也导致了进入该行业的不均衡加速。

### 这很有商业意义

但是“做正确的事”并不总是符合“对企业最有利”的要求，不是吗？

这是谈话的一部分，它总是提醒我在证明网页可访问性时遇到的障碍。在[网页可访问性发起网站](https://www.w3.org/WAI/)中有一整块区域专门用于[网页可访问性的商业案例](https://www.w3.org/WAI/business-case/)，以帮助教育人们制作网页可访问性的经济案例。

除了表明潜在经济机会损失的英特尔研究之外，雇用初级员工还有其他好处:

#### 更易理解的代码开始发布

就像写一篇演讲迫使你在更深的层次上理解你的主题一样，和其他人，尤其是比你级别低的人解释和讨论事情，迫使你更好地理解它，并发现你的实施和思考过程中的漏洞。更简单的代码更容易维护，错误更少。在绝对*至少*的情况下，这是一个有价值的现场“橡胶鸭”的情况。

#### 更智能的任务分配

## TL；博士；医生

我认为，雇用和培养初级人才的努力不能与推动技术领域多元化的努力相分离，后者具有社会、文化和经济效益。我们行业的健康发展和我们所创造的产品的质量取决于该领域不断增加的多样性，以及创造一个更大的人才库。
