# 不要害怕问问题

> 原文：<https://dev.to/ashleemboyer/don-t-be-afraid-to-ask-questions-5g99>

没有愚蠢的问题。可能会有一些愚蠢的时机，比如一个学生在老师已经宣布信息的 5 分钟后问考试时间。但这仍然是一个好问题，他们只是在一个可怕的时间问了这个问题。

一般来说，提问没有太多限制。很大程度上是关于它们的计时。例如，如果你还在试图找出客户对潜在产品的需求，你就不要问太多关于实现细节的具体问题。他们的需求*可能*会在你每次与他们交谈时发生变化，而过于严格地计划实施可能是浪费时间。

那么，打破你对提问的恐惧，直接去做有什么好处呢？

### 学习，学习，学习

在某种程度上，谁会在乎他们对你问一堆问题的感受呢？你的职业是你自己的，只有你有责任让它更上一层楼。

### 结交新朋友

我无法告诉你在过去的一个月里，我通过 DEV 社区网站和 Twitter 认识了多少人。如果你把自己放在那里，你能在这个行业找到多少支持，这真的很令人兴奋。

### 树立你的信心

有时候问问题最可怕的部分是知道如何问问题。你可以尝试谷歌搜索 10 次，谷歌不会因为你这么做而评判你。对一个人重复几遍问题，以确保你得到了正确的信息？这可能会让你们两个都感到沮丧。但是好消息！你问的问题越多，你就越会措辞。
