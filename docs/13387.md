# 你在栈溢出上问过最疯狂的问题是什么？

> 原文:[https://dev . to/Josh hadik/what-the-crazy-question-you-even-question-on-stack-kic](https://dev.to/joshhadik/whats-the-craziest-question-you-ever-asked-on-stack-kic)

作为已经走出“初学者”阶段的开发人员，我们经常低估我们已经走了多远。这就是编程的本质。曾经看起来是世界上最复杂的事情变成了第二天性，所以我们把它们一笔勾销。

慢慢地，随着时间的推移，我们成为越来越好的开发者。而随着我们的成长，我们往往会忘记自己所经历的痛苦，这往好了说会导致我们忽视自己的天赋和价值，往坏了说会让我们对新的开发者产生敌意(“你连 X 是干什么的都不知道？你怎么能称自己是程序员！”)

我认为，对于我们这些已经“存在”了一段时间的人来说，回顾一下我们关于堆栈溢出的一些最古老的问题，并分享一些我们曾经问过的“最疯狂”的事情，将是一个有趣的练习。这些东西现在对我们来说是如此的基本和根本，但在某一点上对我们来说就像量子物理学一样有意义(或者其他一些疯狂复杂的概念，如果你碰巧是这个领域的专家的话)。)

如果你是一个阅读这篇文章的初学者，我希望这篇讨论能帮助你认识到我们都经历过，如果我们能做到，你也能！

如果你已经过了初级阶段，我希望这个练习能帮助你认识到自己技能的价值，让你对新开发人员多一点理解和宽容。

所以开始了。
