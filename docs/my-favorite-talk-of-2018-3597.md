# 2018 年我最喜欢的演讲

> 原文：<https://dev.to/wiredferret/my-favorite-talk-of-2018-3597>

这是我今年看到的改变我想法最多的演讲，也是我最常提到的。

大卫·海涅梅尔·汉森

这是一个小时，但这一个小时是值得的。以下是一些亮点/时间戳。

2:15–20 年后，我不再是年轻时的无知和热情，而是被一种历史感所驱使。我们工作的核心是关系数据库。

12:50–这是一种模式–我们有时可能需要深入了解过去的历史，但现在不再需要了。Ruby 把我们所有的垃圾收集问题都给了@tenderlove。太好了！这是真正的进步。尽管我们所做的许多事情的基础是关系数据库，但这并不像我们写 SQL 了。

17:30–概念压缩。为了使用它，我们不需要了解我们的全部堆栈。概念压缩是我们在 Rails 开发中做的最重要的事情。如果我们不压缩我们理解的概念，我们就没有空间容纳新的概念。

当你可以买下整个玩具的时候，为什么你要用乐高建造一个千年隼呢？因为你投资建造它，但它不是更好的产品。这是自我放纵。

24:00–今天，一个经验较少的程序员可以开始并完成比以往任何时候都多的事情，因为我们已经在概念上压缩了这么多。

30:00–我们越能降低进入门槛，就有越多的人能与我们合作，这很好。如果你想留着它，滚开！

37:30–我们希望它易于学习，也希望它易于发布。你需要操作和处理的概念越少，你就能越早开始。

42:45–如果我们想要更广泛的人制造技术，我们必须降低准入门槛，允许更多类型的人改变我们的系统。我们不能把今天的人应用到明天的问题上，得到一个不同的世界。

我们应该利用过去来为未来的选择提供信息，我们必须回顾过去，确保我们利用自己的技能至少站稳脚跟。

我们在倒退。该行业有很多概念上的扩展，但我们没有将其压缩到可用的程度。现代 web 应用程序比过去难多了，它要么拒绝人们，要么推动了大量的专业化。

51:30–当我们高度专业化时，我们就会与劳动产品疏远。当我们的影响力很小的时候，我们不明白我们是如何改变世界的。

即使我们可能不同意马克思主义的处方，症状的描述是重要的，尤其是有意义的。

引用哈里·布雷弗曼的话。

(歇斯底里咯咯笑的是我，因为我看到了他要去的地方)

我们越疏远我们的劳动产品，我们就越容易犯软件医源性错误——通过努力做好事，我们会造成伤害。软件从未像现在这样伤害更多的人。它也帮助了很多人，但这些事情并不总是联系在一起的。

你越专业，你的蒙蔽就越强，就越难看清大局，也就越容易为你一直在做的坏事辩护。

这尤其是因为我们已经被殖民化了——互联网已经被征服者征服了，他们通过攫取我们的注意力和隐私变得异常富有。我们并不完全理解这种交易——他们带着廉价的镜子和劣质的梳子出现，用我们人性中最有价值的一面来交易我们。

互联网过去不像现在这样被殖民。

皮凯蒂。资本回报率大于经济增长率。

作为这些系统的实施者和发明者，我们不能忽视我们所创造的后果。

我们的使命应该是解放最好的想法。用最好的想法武装反叛者，并在概念上压缩它们，使它们更容易传播。

软件正在吞噬世界，但是谁在编写软件呢？你是。所以，你怎么能没有道德和伦理义务来引导它呢？

* * *

所以，是的，这就是我的演讲。如果你只想要最精彩的部分，你可以从 40 分钟左右开始，获得关于概念压缩的部分，降低进入门槛，拒绝与我们的劳动产品疏远，并对我们作为软件创造者的行为负责。
