# 我喜欢软件工程的什么

最近，我开始意识到我最喜欢我的工作和兼职项目的地方:解决迫使我看得更远的问题。

我真的喜欢开发软件，但是像所有的活动一样，有时有趣，有时不有趣。编程可能会很无聊，真的很令人沮丧。调试问题或寻找构建东西的好方法可能会令人厌倦。

我总是想到分层编程。程序员在他们感到舒适的空间工作。当你用 HTML、CSS 和 Javascript 构建网站时，你通常不会过多考虑操作系统或压缩算法。但是有时候会出现一些 bug 或者问题，需要你更深入的去寻找。这就像开车一样。您需要查看引擎盖下以解决问题或进行维护工作，但通常您并不关心里面有什么。我最近做的一些工作迫使我审视以前不理解的东西。起初，面对一个在自己的舒适区内无法解决的问题是令人沮丧的，但经过一些研究和许多问题之后，它会变得非常有趣和有益。你学到了许多新东西，这有助于你更好地理解你以前做过的工作或将来会做的工作。

这也是我再次意识到开源的美妙之处。如果你使用可以拆卸的技术，当你遇到问题时，你有很好的机会帮助自己。我喜欢看我正在使用的开源项目的代码，并通过 Github 和 Stackoverflow 寻找解决方案。如果幸运的话，我可能会找到负责我正在做的事情的那部分代码。最好的部分是理解问题，并可能找到解决方案。如果这导致了补丁、拉取请求或解决方法，那将是有趣且有益的。帮助其他人解决同样的问题也是一个不错的奖励。

我喜欢在这些想象的层之间跳跃。对我来说，这就是编程和开源的精神。找到有趣的东西，研究它，并在这个过程中学习。
