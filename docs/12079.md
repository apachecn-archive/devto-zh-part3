# 我不能全部做到:我的倦怠故事

> 原文:[https://dev.to/molly/i-cant-do-it-all-my-burnout-story-1e54](https://dev.to/molly/i-cant-do-it-all-my-burnout-story-1e54)

在评论了最近的一篇帖子[你如何处理倦怠](https://dev.to/j0nimost/how-do-you-handle-burnouts-2e6k)之后，我认为分享我的倦怠故事可能会有所帮助。希望这能帮助其他人认识到自己或周围人的倦怠迹象。

# [](#i-can-do-it-all)我都能做到

我已经在我的公司工作了 3 年多了。当我开始的时候，我们是一个 30 人的小团队。现在，我们有超过 120 个。因为我已经工作了很长时间，所以我有很多关于我们整个应用程序的领域知识。我个人已经编写了大量的应用程序代码，所以我了解它的所有细节是有意义的。直到 6 个月前，每当有人需要帮助或出现呼叫问题时，我都是他们的首选。每个人都知道我可以最快地解决任何问题，所以他们来找我。

起初，这是令人振奋的！谁不想一直感觉被需要呢？！我欣然接受了自己作为关键人物的角色。当有人需要帮助时，我会立即投入。我从没拒绝过。我是肯纳的奥利维亚·波普！

在早期，这是可持续的，因为应用程序和团队相对较小。然而，随着团队的成长和应用程序的扩展，它开始对我产生影响。

更糟糕的是，我不是第一个注意到的人。我的团队领导和我周围的人首先注意到了这一点。他们开始问我是否需要帮助，或者我是否想减轻一些负担。每次我都回答，“不，没关系，我能应付。”最终，它吸干了我的生命。我对工作和同事变得更加易怒。但是，我一直努力做到这一切。我是那种不会放弃的人。我是一个讨人喜欢的人，如果可能的话，我总是想帮忙。

# [](#intervention)干预

最终，我不可思议的同事们介入了。一位同事让我坐下来，用一个非常准确的类比向我展示了它。他说，“莫莉，你是个瘾君子。”我笑着回应，但他的脸仍然非常严肃，他继续说。“每次有人向你求助，你都会兴奋不已。这和有人吸毒没什么区别。当他们受到打击时，他们会立刻感觉良好，但这会对他们的身体造成伤害。你全天候“随叫随到”的状态让你付出了代价，这种情况必须停止。”

在这次谈话之后，我被强行踢出了用于随叫随到问题的空闲频道，所以我甚至无法忍不住去帮忙。几个人接替了我的下一轮值班，每个人都制定了一条规则，除非网站瘫痪，否则不允许任何人向我寻求帮助。我是最后一道防线。

最后成功了！对我来说这不是最容易的转变。一开始，我觉得我忽视了我的工作。但是几个星期后，事情变得容易了，很快我就恢复了原样。你知道最棒的是什么吗？因为我让其他人解决出现的问题，他们对我们的应用程序更加了解。现在，应用知识更加分散。最近几次大的随叫随到，我都没参与。令人难以置信的自由。过了假期，我去度假了，第一次，笔记本电脑一次都没开！！！那对我来说是闻所未闻的。

# [](#burnout-advice)倦怠建议

回顾整个形势，我的建议是:

> 不仅要意识到你自己，也要意识到你周围的人。

你总是想知道自己的习惯和情绪，但也要密切关注你的同事。如果你觉得他们“不在”，就和他们联系一下。有些人可能需要你介入，允许他们休息一下。

另一方面，如果你需要帮助来应对倦怠，就去问吧！许多编码人员对他们所做的事情有着强烈的热情，这种热情导致我们中的许多人在某个时候筋疲力尽。你周围的人可能已经经历了你正在经历的事情，他们会很乐意以任何方式帮助你。

我的 DM 永远向任何感受到压力并需要一个共鸣板的人开放🤗
