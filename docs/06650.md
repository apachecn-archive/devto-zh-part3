# [](#am-i-qualified)我合格吗？

你可能会想，“我没有足够的经验和知识来指导别人！”

成为一名导师可能并不像我们大多数人认为的那样宏伟。

做导师其实很简单。

导师只是一个可以帮助建议和指导别人的人。

有时候正式比非正式多，直接比不直接多，等等。

但是说到底，做一名导师其实就是通过提问、回答问题、提供指导、鼓励等方式来帮助他人。

通常，导师对他们提供指导的特定领域有一些经验或知识。

这只是在帮助别人。

# [](#how-do-i-become-a-mentor)我如何成为导师？

我认为导师分为两大类:

*   正式的
*   非正式的

## [](#formal)正式

例如，这可能发生在工作场所。你可能会被正式要求给公司的新员工授课和提供指导。

或者，你可能想为某一特定人群提供服务，帮助他们实现目标。

一个很好的例子就是史蒂夫·史密斯和 DevBetter 合作的项目。

Steve 是代码架构和领域驱动设计领域的专家。

他的服务可以帮助开发人员找出需要关注的技能，帮助提高这些技能，提供职业指导等。

另一个例子是 [Coading 蔻驰](https://codingcoach.io/)。你可以找到一个导师，并要求开始免费的指导！

在这种情况下，导师和学员的目标和意图更加集中。通常会有一个正式的计划，由导师来充实。

因为它非常耗费时间和资源，所以通常是有偿服务——无论是由你的雇主还是学员直接支付。

## [](#informal)不拘礼节

非正式是没有报酬的，通常不涉及一个坚实的计划或地图。

我认为您最感兴趣的问题是:

*   这到底是怎么回事？
*   如果没有计划，我能帮什么忙？

# [](#why)为什么？

我的一个遗憾是没有早点加入开发者社区。

我觉得我没有太多可以提供的，只是害怕把自己“放在那里”的想法。

但真正发生的是，你会遇到这么多了不起的人！

你可以接触到新的思维方式，并抛弃与你不同的人的想法。

今年年初，我觉得我想开始做一些事情来帮助人们。不只是“制造东西”，而是真正帮助他人。

这就是我的时事通讯的诞生。

我接触了几个月的一个人向我咨询了他的第一份开发工作。

几周前，他同时收到了 3 份工作邀请！他得到了他的第一份开发工作！

能够像这样帮助别人带来如此多的满足感和成就感！

如果你真的想帮助别人，建立关系，建立联系，那么你可能会考虑导师(即帮助别人😋).

# [](#how-does-it-happen)它是如何发生的？

通常，非正式的师徒关系涉及两个人:导师和学员。

当以下任一情况发生时，它才真正开始:

*   学员提出一个问题或寻求一些指导
*   导师寻求为学员提供价值或帮助

我想重点谈谈第二点。

如果你是一名已经在这个领域工作多年的软件开发人员，你在职业生涯中可以采取的最令人惊讶的步骤之一就是**帮助他人。**

如果你看到在你的领域有人正在写很棒的内容:**让他们知道并鼓励他们！**

如果你在 Twitter 上看到有人在问一个关于你熟悉的技术的问题:**回答他们的问题，或者如果你还不知道的话，找到答案！**

当然，可以说，把答案放在“银盘”上交给别人，可以鼓励人们在试图自己找到答案之前先去寻求答案。

但是总的来说，试着回答这些问题吧！

# [](#pro-tips)亲提示

以下是我认为对你有帮助的一些建议:

## [](#let-people-know-youre-available)让别人知道你有空

如果人们不知道你愿意...那他们就不会来找你了。

## [](#try-to-build-a-relationship-privately)尝试私下建立关系

首先:不要让人毛骨悚然👀。

你想先与他人进行一些公开接触(也就是说，应该有一定程度的信任和关系)。

通过 DM 等建立关系。有几个优点:

*   有时候帮助别人就是指出哪些地方可以改进！最好私下进行😉
*   这表明你确实在花时间和对这个特定的人感兴趣。这有助于建立真正的关系！这让他们知道你是真心想帮助他们。
*   记录你说过的话/做过的事很容易。你可以查看一些建议等。你过去给过的。对于学员来说，他们可以很容易地找到你提供的帮助。

## [](#your-main-goal-should-be-to-encourage-support-and-guide)你的主要目标应该是鼓励、支持和引导

每当你的学员做得好的时候，你就在那里鼓励他们！

你不是去指出每一个可以改进的地方。那没有帮助。

想象自己是一个可以提供建议或回答问题的啦啦队长！

## [](#promote-their-strengths)宣扬自己的长处

如果你看到一项技能，经验等。这一点让你印象深刻——告诉你的学员！

如果你认为他们有一个鼓舞人心的独特的故事要分享，鼓励他们分享！

如果你认为他们写的主题会吸引某一部分人，告诉他们！

鼓励他们写一本书，建立那个项目，寻找新的工作，等等。

拥有智慧和洞察力可以大有作为！

# [](#bonus-dualmentorship)奖励:双师徒！

我还想快速提一下，许多这样的关系可以是双向的指导。

你可能有某种技术的经验。另一个人可能擅长你需要帮助的其他事情。

这些关系都很棒！

我发现这实际上就是发生在我身上的事情。

每个人的故事或技能都有一些特别或独特的地方，你也可以从中学习。

你只需要问一下对方就可以了！
