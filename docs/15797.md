# 一天，不，甚至不到一个小时，生活在栈溢出

> 原文:[https://dev . to/dexygen/a-day-no-not-even-hour-of-life-on-stack-overflow-1g4b](https://dev.to/dexygen/a-day-no-not-even-an-hour-of-life-on-stack-overflow-1g4b)

自从 Stack Overflow 推出几个月以来，我一直是它的用户，看到了很多变化。这更像是你可能从新手那里听到的抱怨，但即使对我来说，它似乎也不值得再参与了。

所以今天一个低代表(~1000)问了一个问题，但是给了它一个不合适的标签。一个 100+K rep 的用户抱怨，但没有删除标签，而我只用 7K rep 就能做到。

顺便问一下，你要花多少时间处理堆栈溢出才能获得超过 100K 的 rep？说到这样的用户，当我实事求是地评论说，我的代表级别低得多，我可以删除标签时，我收到了来自另一个 100+K 代表用户的尖刻评论。

很好。

此外，问题中的问题是由印刷错误引起的，所以我投票关闭它，但人们仍然回答了它。所以我不得不浪费掉代表否决的答案，这些答案本不该被公之于众。

我参加这个活动只是为了获得 100 个好问题的“黄金”(顶级)“苏格拉底”徽章，我已经完成了 90%的工作。顺便说一下，我问的问题是我回答的问题的 2/3，我也不理解除了回答什么都不做的用户。

如果你对编程没有疑问，那你就是做错了。
