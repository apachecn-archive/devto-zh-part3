编码和我一言难尽…
Turbo Pascal，VB6.0 和多年一无所获…
1 年前我偶然看到了何塞·波尔蒂利亚的 Udemy 课程:完整的 Python Bootcamp:在 Python 3 中从零到英雄，我承认这个标题立即吸引了我。

我一直热爱编码，我记得我为了一个小的保险客户管理软件用 VB 6 度过的不眠之夜。每当我学到新的东西，我绝对必须在当天晚上实施它。因此，可以说，这个乌迪米的课程将我推进了一个我想重温的怀旧的过去。但问题是:38 岁的我还能保持这样的速度吗？我的回答是，为什么不呢。

python 社区

通过给我儿子报名 Udemy 课程，让他开始学习 Python，我发现了不和谐。在那里，我发现了一个社区，充满热情的人们，已经集成或将集成 school 42 的人们，决定完全转向 Linux 的老年人(他们可能一开始就知道所有的限制)。年轻人激励年长的人，从不认为他们比他们优秀(即使他们是！).

在 Discord 社区，我发现了 Git、Github、GitLab、Heroku 和 Codewars 的挑战。这些挑战让我出汗，我几乎掉光了头发。但是当你找到解决方案时(用最少的代码行，因为否则你不会觉得不同)，你会说“太好了！”。

当你进入一个不和谐的世界时，你进入了另一个国家，在那里，你不理解每两个单词的语言技术术语，你从未见过的表情符号。这个社区[对他们来说，通过终端输入指令是一件轻而易举的事情]是为了帮助你，他们很有耐心，因为你不知道如何解释你的问题，答案对他们来说是非常愚蠢的，对你来说也是如此，在你发现这个问题后(你忘记了一个括号或冒号)，他们会友好地回应你。

总有一天，你要帮助别人理解数字阵列中的切片，保持谦逊，因为这是这种不和谐的精神。

经验

编码真的很费时间，很难停下来。

我晚上不出去，或者只是在必要的时候才出去。这是我下班后做的第一件事。然后你会记起你也有家人，需要关注的孩子。所以我放慢了脚步。

然后到了一个阶段，一切都出错了，什么都没用。你在 conda 库中有太多的冲突(太多:pip install …)，以至于当你的深度学习脚本工作时，你的 opencv 脚本就不工作了。你安装了这么多东西，这么多库，你的电脑就成了一个真正的储藏室。您可以使用一种解决方案，格式化笔记本电脑，然后从头开始。
去学习虚拟环境以避免库冲突，打开虚拟机如果你想在 Linux 上尝试你的代码等等…

我经历了太多类型的错误，花了太多时间在堆栈溢出上，以至于一切都变得越来越简单。我不需要阅读论坛的所有内容就可以获取我想要的信息。它变得几乎是自然的！
