## [](#introduction)简介

我记得我第一次开博客的时候，我不知道从哪里开始，做什么，我应该怎么做，我也没有开发人员和博客的朋友。

除了著名的博主，我一直在读一位来自[的**约翰·桑梅兹****简单程序员**T5】和**丹尼尔·罗伊·格林菲尔德**来自](https://simpleprogrammer.com/)[T9】py Danny](https://www.pydanny.com/)他们写了一本在这方面相当成功的书[**【Django 的两勺】**](https://twoscoopspress.com/products/two-scoops-of-django-1-11) 。

如果你现在正在读这篇文章，你可能正在考虑为自己开一个博客，或者想了解并利用盖茨比为你建立网站。

这样你就可以使用这项技术，通过 Gatsby 很容易地创建网站或博客。

## [](#why-a-developer-blog)为什么要开开发者博客？

你应该创建开发者博客的理由是什么？这里列出了开发人员博客的理由。

*   从你的博客中赚钱
*   记录您作为开发人员的学习历程
*   分享你对世界的看法
*   作为开发人员，控制您的叙述

对我来说，你应该有一个开发者博客的主要原因是让自己成为专家。

## [](#the-3rd-door-to-awesome-work)牛逼作品的第三扇门

[![steven-garcia-260350-unsplash](../Images/28892d9ddc0c84835f1d4ad365643a32.png)T2】](//images.ctfassets.net/ly2f59p4unnn/2zoaFNNdqs86IQ6GgqKAqk/a8b05baed4991462d2eaa82e30e684f5/steven-garcia-260350-unsplash.jpg)

我敢肯定，作为一名开发人员，你经历过找工作的痛苦。通过求职门户网站找工作，成功率很低，这是一种持续的挫败感。

如果你不是在寻找工作却没有太大的成功，而是你未来的雇主来敲你的门并给你提供工作，那会怎么样？

这就是为什么你需要创建一个博客来区分你作为一个开发人员，这样你就可以做你想做的工作，并作为一个特定领域的专家获得更高的薪水。

## [](#selecting-your-niche)选择你的利基

选择你的利基可能是一项艰巨的工作，因为如果你像我一样有广泛的兴趣和大量的专业知识，我想涵盖。

因此，我希望你花一些时间来思考你希望你的博客是什么样的。

这样你就能更好地迎合你想要的读者类型。对我来说，我知道我的博客将会涉及创业、网络开发和 python。

你可以看看个人开发者的几个开发者博客，像 [**API 布道者**](https://apievangelist.com/)[**real Python**](https://realpython.com/)[**全栈 Python**](https://www.fullstackpython.com/) 或者 [**简单却复杂**](https://simpleisbetterthancomplex.com/) 之类的。

## [](#buying-a-domain)购买域名

一旦你为你的博客选择了合适的位置，现在是时候考虑你博客的域名了，也就是你主页的 URL。

你可以使用你的名字或者一个能让你的读者了解你将在博客上发表的主题的名字。

一旦你选择了你的域名，前往 [**NameCheap**](https://www.namecheap.com/) 或 [**GoDaddy**](https://www.godaddy.com/) 注册你的域名。

无论何时注册域名，都可以选择在 WHOIS 数据库中不公开显示个人数据。

## [](#setting-aside-time-to-write)留出时间写作

[![green-chameleon-21532-unsplash](../Images/2d5beb577657f8c058397d8e4ec5fb3e.png)T2】](//images.ctfassets.net/ly2f59p4unnn/b8I8adi65MyUUWuMoeamw/9e92c03aea596fe273057b362bca5b5e/green-chameleon-21532-unsplash.jpg)

找时间写作是一项艰苦的工作，尽管写了一段时间，我仍然发现很难写。

我建议任何想开博客的人开始留出 **3-4 个小时**来写一篇**每周博文**。

我把写作的时间一分为二，留出前**个半小时**来创作博文的**初稿**，剩下的时间用来编辑草稿。

关键是坚持发表你的博客文章，选择一个你喜欢的日期，并努力在那个日期和时间以特定的频率发表。

## [](#gatsby)盖茨比

首先去盖茨比网站教程，花些时间完成教程，对盖茨比有个大致的了解，以便将来编辑你的博客。

我建议你使用 **CSS** 浏览一下关于**插件** &的章节，因为你将使用 [**Gatsby Starters**](https://www.gatsbyjs.org/starters/?v=2) 为你的博客制作插件模板。

## [](#markdown-editors)降价编辑

根据你使用的 markdown 编辑器的类型，大多数编辑器会遵循特定的格式来发布你的博客文章。

要开始，去 Github 找一个叫做 [**掌握 Markdown**](https://guides.github.com/features/mastering-markdown/) 的向导。

## [](#conclusion)结论

总的来说，开一个博客可能是一项令人生畏的任务，所以我希望上面的建议能给你指明方向，帮助你创建一个新的博客。
