# 个人项目！是还是不是？

> 原文:[https://dev.to/gac/personal-projects-yes-or-no-586](https://dev.to/gac/personal-projects-yes-or-no--586)

作为一名程序员，我喜欢尝试新的编程语言和技术。这就是我们大多数人如何在日常工作和生活中发现自己是思考者的一个新的副业。像我们中的许多人一样，我也有许多甚至从未完成或见过天日的兼职项目，但这并不意味着我停止工作或寻找新的项目。从那以后，我开始从事业余 android 手机开发。

像大多数人一样，当我开始一个新的个人项目时，我总是感觉很好，因为我知道我没有严格的截止日期或客户会在项目中期改变他/她的想法，让我从头开始，即使我对自己这样做，我仍然会享受它，因为我会认为这是一个新项目，而不是旧项目的返工。这些项目最大的好处是，我可以在任何时候，任何我想做的时候去做。但是关于我们的大部分副业，事实是它们迟早会死去，被留在一些外部硬盘或女贞路的仓库里。在一些后方的情况下，一个人可能会完成并活着看到白天的光，并形成一些他们将被证明比他们目前的工作更有利可图，或者只是出版，但没有太多的活动。

一个很好的例子就是我的手机应用程序，名为 [Banja Luka WiFi](https://play.google.com/store/apps/details?id=com.gac.banjalukawifi) 。我创建它是出于无聊和纯粹的渴望，想看看我是否真的能做到。这是背后的真实故事:

一天，我躺在床上，看着一个越来越受欢迎的 facebook 群组。它的目的是持有我所在城镇附近咖啡店的 WiFi 密码，这样人们就可以连接到这些咖啡店，而不必进去点餐，就可以从服务员那里获得密码。但是，我意识到这是一个好主意，它有它的流程，例如，为了让我连接到某个网络，我需要记住所有这些密码，或者上网并查看该组中的帖子，以便能够连接到网络，这样我就可以上网(看到漏洞了吗？).因此，我认为尝试创建一个移动应用程序会很有趣，它将拥有一个存储所有这些网络的远程服务器，并且用户拥有一个移动应用程序，该移动应用程序将在上线后连接到该服务器，并将所有这些密码存储在用户设备上，并允许用户在需要时搜索某个密码。

我就是这样开始学习 java 和移动开发，并发布了我的第一个移动应用程序。但是我也有很多从未完成的项目，而且很可能永远也不会完成，尽管它们很有趣，看起来对其他人而不仅仅是我有用。

但我也读过其他开发者的故事，他们实际上把他们的副业项目变成了全职工作，甚至围绕他们开发了完整的业务(以脸书为例，但还有更多)。尽管起初我们中的大多数人会认为这些辅助项目不会做太多，或者会存在足够长的时间以至于有人实际使用它们，但这可能会在一夜之间改变，它们可能会成为每个人都在使用或谈论的下一个大东西。或者，这可能不会发生，但您的项目仍然可以看到阳光，一些人发现它是有用的。

不管是什么情况，我的故事的结论是，如果有人问你正在做什么，副业应该总是有一个肯定的回答。因此，它们可以帮助你保持原有的技能，如果你想用一种新的编程语言来做的话，还可以帮助你学习新的东西，还可以帮助你放松，把你的注意力从日常工作中的问题和斗争上转移开来(参见[CFH.net](https://clientsfromhell.net/)或[发展趋势](https://devrant.com/stories)中的一些笑话和真实故事)。

我已经决定把我的博客从我的个人网站搬到这里，因为我认为它可能会找到更好的受众:)
