## 书执行

### 写作的准备工作

在你开始写一本书之前，你能做的最好的准备就是已经成为你想写的主题的专家或知识渊博的人。这会让你召集所有的知识和经验，并立即开始写作。否则，你需要聚集你所有的耐心和好奇心，深入你感兴趣的话题。

一旦你觉得你找到了你书的主题，那么你就准备好开始写作了。在这一点上，你不需要知道所有的事情，当你开始写每一章时，会有时间做更多的研究，但重要的是，你在这个主题上有很强的知识，以便能够为这本书写一个一致的大纲，并通过一些早期的胜利保持你的精神。

在 JavaScript-mancy 系列的案例中，我有很多经验。虽然我不知道 ES6 Reflect API 的所有细节，但我知道很多我认为 C#开发人员对 JavaScript 非常感兴趣的东西。

### 写作的技巧

我写一本书的一步一步的过程如下:

1.  为这本书写一个大纲。这意味着写一份我希望这本书包含的所有章节的清单。
2.  为每一章 a. **写提纲**为第二章 b. **不停地写**从头到尾关注内容而不是风格 c. **回顾**着眼于提高章节的清晰度和风格 d .回顾章节大声朗读 e. **回顾代码示例** f .写故事 g .写练习
3.  包含技术评审者改进的特别评审
4.  **复习全书**。聚焦**清晰度**。
5.  **在 kindle 上阅读完整的书籍**。专注于**让每一章变得特别**。
6.  **胜利**

让我们深入这些步骤。

#### 总的轮廓

在你开始写作之前，这本书的大纲会让你对整本书有一个大概的了解。我通常使用以下结构:

```
- title of chapter
- small description of chapter content 
```

总纲的目的有两个:

*   给你一个想法，让你思考这本书的内容和范围
*   塑造书的高层次结构，使其最有利于学习

请注意，在这一点上没有什么是一成不变的，当你开始写真正的章节时，你可以添加/删除或移动你认为合适的章节。当你对这本书的高层次结构感到满意时，移到第一章，开始写作。耶！

#### 写一章

在写每一章的时候，我都遵循我之前提到的过程:

1.  **为章节写提纲**
2.  从头到尾不停顿地写作，关注内容而不是风格
3.  **回顾**章节，重点改进**的清晰度和风格**
4.  大声朗读复习章节
5.  **审查代码样本**
6.  写故事
7.  写练习

我首先写下整个章节的大纲，它基本上由一系列标题组成，这些标题是我设想的这一章的不同部分和小节。之后，我从顶部开始写。

没有什么比一张白纸更可怕的了(除了恐怖电影中令人毛骨悚然的小孩子)，所以我只是开始写作，不要求太高，也不追求完美。在第一遍中，比起风格和结构，我更看重内容。我就是写着写着写到最后。如果你想成为一名优秀的作家，这一点非常重要。不要盯着被挡住的空白页，不要停下来思考几分钟解释 x 的最佳方式。只管写，即使你知道你以后需要改进它。

我通常会编写**代码样本**，因为我需要它们来支持我所写的不同概念和技术。我会使用像 [jsBin](http://jsbin.com/?html,js,output) 、 [jsFiddle](https://jsfiddle.net/) 或 [codepen](https://codepen.io/) 这样的 JavaScript 原型工具，在编写和编码之间跳转。只有使用我不熟悉的新 API 或者更复杂的例子，我才会在写真正的章节之前开始写代码。

当我完成这一章时，我通常会进行三次审查，前两次集中在提高清晰度和风格上，最后一次集中在代码样本的正确性上。

##### 回顾并关注清晰度和风格

为了清楚起见，我遵循一些启发性的方法:

*   将大段分成小段。我觉得这给页面增加了很多喘息的空间，也让文字更容易消化。
*   将大句子分解成小句子。类似上面。如果我找到任何一个大句子，我会把它分成由点分隔的小句子。我发现这篇课文更容易理解。
*   选择主动语态而不是被动语态。感觉更直接，更近，更对话。

为了时尚，我做了几件事:

*   我大声朗读这一章,这有助于我让每一章更具对话性。我的意图是让这一章看起来像是我亲自和你一起解释一些事情。
*   **我喜欢加入笑话、幻想参考和古怪的文字来打破阅读的单调，让书读起来更有趣**。
*   **一般来说，避免在连续的句子中重复单词**。
*   试着用代词或冠词代替它们所替换的单词。当你这样做的时候，文本有时会变得更清晰。但其他时候，它可能会变得非常重复，所以要小心，使用你的蜘蛛感官。

##### 审核代码样本

当涉及到代码样本时，我会额外关注它们。我首先检查它们的正确性，一切都应该完全按照代码示例中的描述运行，然后我:

*   **解释代码示例**。我更喜欢过度解释一个例子，而不是假设读者的知识和背景，让他/她感到困惑。
*   **喜欢小例子而不是大例子**。一般来说，我会将一些代码和一些文本一步步地解释它们。
*   **喜欢有意义的例子，不喜欢捏造的例子**。如果例子是有意义的和相关的，而不是人为的、简单的和捏造的，那么它们总是最有效的。问题是找到一个好的例子需要时间。不要偷懒，找个好榜样，会有回报的。好的例子是学习的极好工具。

##### 上帝保佑技术评论者

不管你复习一章多少遍，花多少时间校对，多仔细看代码样本，**你都会犯错**。

拥有第二双、第三双眼睛是非常宝贵的。如果可以的话，找一些技术审稿人帮你保证书的质量。理想情况下，他们对你所写的主题有不同程度的专业知识，他们能够从初学者和专家的角度给你反馈。

#### 习题

锻炼还是不锻炼，这是个问题！我通常不做任何在编程书籍中找到的练习。同时，我发现互动练习是一种不可思议的学习工具。没有什么比尝试更能真正学到东西了。你可能认为你已经通过观看视频或参加 Pluralsight 课程学习了 X 技术，但你没有。除非你自己试验过这项技术，否则你不会。

对于 JavaScript-mancy 系列，我决定写一些练习来巩固每章中的所有主要概念。这给章节的制作增加了大量的时间，但我认为他们表现得很好。我喜欢练习是开放式的，我只提供一个解决方案，不是作为真理的唯一来源，而是作为可能的解决方案之一，鼓励读者去尝试。

如果你决定为你的书写练习，那么你可以考虑自己被警告了。这将是很多额外的工作。

#### 再次回顾

当我写完所有的章节后，我会做两次最后的评论。这些最后的评论非常有趣，主要是因为一个原因:通常，自从我写了第一章以来，已经过了很长时间，感觉就像在读别人的作品。这种与自己作品的距离是令人敬畏的，因为它给了你一个更客观的观点和更好的批判眼光。对于校对来说也很棒。

首先，我做了一个回顾，再次关注清晰性和结构。我可能会做一些小的修改，比如打字错误或者简化句子，或者做一些大的改动，比如重组整个章节。

在第一次审查后，我对这本书的每一章都很满意，也对不同章节作为一个整体的契合度很满意。下次回顾！

为了期末复习，我用 kindle 阅读了这本书。这让我可以回顾这本书在 kindle 中的感觉，也让我充满了自豪。这很奇怪，但这本书在 kindle 上看起来比你在电脑上以 pdf 格式阅读时更像一本真正的书。在这最后一遍中，我把重点放在风格上，让每一章都流行起来。

在这两篇评论结束时，我应该感到兴奋和自豪。书准备好了！

### 作家的工具

为了从写作中获得最大的收获，你需要使用你熟悉的、舒适的、支持你写作需求的工具。在我的例子中，因为我喜欢它的乏味，我使用了 **vim** 和 **markdown** 。我有几个 vim 插件来实现一个不错的*无干扰*类型的东西:

*   GoYo 将您正在书写的文本置于屏幕中央。
*   通过调暗其余段落来突出你在某一时刻正在写的段落

差不多就是这样。Vim 有一个内置的拼写检查器，您可以通过一个简单的命令`:set spell spelllang=en_US`启用它。

我使用 [leanpub](https://leanpub.com/) ，它允许我将我的书放在 git repo 中，在 markdown 中编写书和章节，每次我向资源库推送更新时，leanpub 都会生成 pdf、epub 和 mobi 格式的书，并将生成的文件放入 dropbox 文件夹供我查看。我觉得这非常古怪和方便。

### 论富有成效

如果你开始写一本像我这样的技术书籍，你可能会把它作为一个兼职项目。也就是说，你很可能不会辞职或者休假去写书。这意味着你会在工作和生活之外的宝贵时间里写书。因为你没有那么多时间来写作，所以充分利用这段时间是很重要的。这里有一些提高工作效率的建议，可以帮助你完成这本书:

*   **养成写作的习惯**。这是什么意思？经常写信，尽可能经常写信。你做某事越多，你做得越多。这就像一个自我滋养的美德圈。当你从某事中养成一个习惯时，你就消除了大部分阻止你做这件事的心理障碍，从而减少了拖延。
*   减少你对社交网络的使用。社交网络就像裂缝。它们非常容易上瘾。如果你不小心，你会浪费大量的时间漫无目的地浏览这些人渣和邪恶的蜂巢，并刷新饲料。去除坏习惯的最好方法是创造一条阻力最大的道路。或者**为你想要鼓励的行为创造一条阻力最小的路径**。这些例子包括:从你的智能手机上删除 twitter、facebook 和 reddit 应用程序，或者安装一个浏览器扩展，比如 [focus.me](https://chrome.google.com/webstore/detail/focusme/jdkgogfonlaifeigkgbkfamcihablemc) ，它可以很好地避免默认情况下进入社交网络的快速反应。
*   **仅仅 5 分钟**。每当你感到疲倦或没有动力时，试试 5 分钟技巧。就写 5 分钟。只写 5 分钟是一个非常小的承诺，没有理由不写。你会发现一旦你开始写作，就很容易继续下去。做出这些小小的承诺是打破阻碍我们完成任务的惰性的好方法。
*   **试试[番茄工作法](https://cirillocompany.de/pages/pomodoro-technique)T3。番茄工作法是一个简单的生产系统，旨在教你专注于手头的活动。它包括把任何一项任务分成 25 分钟全神贯注的小块，然后休息 5 分钟。它有很大的副作用，比如了解你自己，你是如何失去注意力的，你工作的环境，给你持续改进的空间等等。当我感到注意力不集中或者不想做某项活动时，我发现它非常有效。**
*   **休息一下**。时不时休息一个周末，养精蓄锐。做一些其他能给你能量的事情，然后精力充沛地回来。
*   **做一次冲刺**。在花了一些时间写完你的书后，当你有最后的 20%来完成它时，你会达到 80%/20%的点，但是进展会感觉超级慢(还有 80%的努力)。当你处于这种情况下，一个很好的前进方式就是冲刺。预定一个周末，休几天假，把书写完。或者尽可能多的阅读这本书。
*   **对其他事情说不**。如果你想完成你的书，现实是你需要牺牲其他东西。你将不能看一眼那个闪亮的新框架，或者有时间去学习那个很酷的编程语言，或者去参加那个会议。这是你的决定，但是承担其他的责任会证明是令人疲惫、紧张和沮丧的。
*   有些时候，好必须足够好。最好是做得尽善尽美。有些时候你必须正视那一章，告诉他*“你完了，我的朋友”*然后继续前进。

如果你想了解更多关于我用来完成工作的[生产力系统，那么看看这篇文章中我描述它的地方](https://www.barbarianmeetscoding.com/blog/2015/06/04/want-to-get-stuff-done-behold-the-ultimate-system/)。

### 保持你的动力

写书是一场马拉松。最困难的事情之一是在整个过程中保持你的动力和精神，这样你就可以看到它结束。这里有一些事情可以帮助你保持这种动力。

*   **记住你的为什么**。首先要经常记住你为什么要写这本书。
*   **练习可视化**。想象一下当你读完这本书时的感觉。想象一下，平装本终于到了你的手中，看到你的书完成时你的骄傲、幸福和喜悦。想象一下，你的一个读者正在浏览你的书，并且非常喜欢。想象一个与你的“为什么”相关的未来情景。是要被认可吗？想象你的同龄人正在庆祝有史以来最棒的书。
*   经常进步，庆祝小小的胜利。写完了一章？庆祝一下！完成了全书的 25%？庆祝一下！做了你的第一笔生意？庆祝一下！不要等到完成这本书才庆祝。
*   **公开承诺**。我们是群居动物，当我们公开承诺做某事时，我们更有可能去做。一个很好的方法是遵循精益出版模式，在你的书完成之前出版它。当你有了第一笔销售的时候，你就有了一个付费的顾客，他期待着阅读你的书。这是一个强大的动力。
