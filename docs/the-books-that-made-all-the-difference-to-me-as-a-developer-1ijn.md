# 作为一名开发人员，这些书对我来说非常重要

> 原文：<https://dev.to/antjanus/the-books-that-made-all-the-difference-to-me-as-a-developer-1ijn>

这个问题会出现在各种媒体上。“要成为一名更好的开发者，应该看哪些书？”。我已经回答过无数次了。在 Twitter 上，在 Dev 上。到，Reddit，和其他任何地方。

我觉得把清单放在一个我可以随时链接的帖子里是个好主意。

**注意:**我*正在*使用我链接到书籍的亚马逊联盟链接。

## 干净的代码:敏捷软件工艺手册

**我不再推荐这本书**

作者对种族、女性等问题有很多不确定的看法。请*不要*购买这本书或向作者投稿。

## [务实的程序员:从记者到大师](https://amzn.to/2Crujc4)

这本书我还看了一半。这本书对于任何开始从事开发工作的人来说都是令人惊奇的——对任何已经进入开发工作的人来说也是如此。我发现它展示并解释了你在现实世界中发现的各种场景，然后讨论了解决它们的策略。它提供了大量关于如何保持代码清晰分离、如何考虑并发性、如何处理需要自我清理的代码等等的想法。

大概这本书我最喜欢的部分是围绕“原型”是干什么用的，如何使用等等的讨论。然后，它讨论了“追踪子弹”的想法(我在我的帖子[中讨论了我在一家初创公司运行开发的经验](https://antjanus.com/blog/thoughts-and-opinions/experience-running-development-startup/))并完全赢得了我的心，改变了我做开发的方式。

## [游戏编程模式](https://amzn.to/2VW0bib) - [GMP 网上免费阅读](http://gameprogrammingpatterns.com/contents.html)

在我开始之前，你可能会想，但是为什么要进行游戏编程呢？我是一名网络开发人员，这本书对网络开发人员和游戏开发人员同样重要。到目前为止，这本书是我所找到的讨论常见(也是众所周知的)编程模式的最佳资源——从它们的优点到它们的缺点、权衡以及它们的细节。所有都写得很清楚，易于理解。

有几本书是我绝对推荐阅读的:

1.  对象池
2.  肮脏的旗帜
3.  singleton——因为这就是节点导入的工作方式！
4.  观察者

如果你喜欢游戏，我还建议你阅读一下`game loop`和`update method`模式。

## [非暴力沟通](https://amzn.to/2tj3gLZ)

这可能看起来不太合拍，但尽管这是一本非编程书籍，我发现它在沟通和将同理心应用到我的工作方面非常有用。不仅如此，我发现它在我与队友和其他同事的互动中非常有用。它涵盖了几个关键概念:

1.  如何确保双方完全理解一个概念
2.  在紧张的情况下运用同理心
3.  诚实的价值
4.  表达自己需求的价值

所有这些都是我职业生涯中的关键。这里有一个肮脏的秘密:我自己的团队非常注重彼此之间以及与我们经理之间的沟通。改进的沟通会带来更大的工作满意度(个人)，但也会带来更好的代码、特性发布、围绕开发路线图的更好的对话等等。我们没有使用 NVC，但是我们在组织方式上做了很大的改变

## 我还没读过的书(还没有)

我的 GoodReads 里有一大堆我想读的书，甚至还有一大堆我保存在脑子里的书。我个人不能保证这些，但我确实看到这些在 Twitter、这里和其他地方反复出现:

### [CSS 机密](https://amzn.to/2N2fv8E)

Lea Verou 所著的《CSS 秘密》一书一经问世便引起了轩然大波，这本书重点介绍了解决 CSS 问题的技巧和方法，以及如何实际编写 CSS。如果你不确定这是否适合你，[看看她众多演讲中的一个](https://www.youtube.com/watch?v=8BXQ3zCihYM)。

### [重构 UI](https://refactoringui.com/book/)

Adam Wathan 和 Steve Shoger 所著的《重构用户界面》一直在我的脑海中盘旋，我认为这是一本值得一读的书。我个人没有时间去查看，但我听说过人们喜欢的 Adam 的课程，我密切关注着他的 CSS 框架 [Tailwind CSS](https://tailwindcss.com/docs/what-is-tailwind/) 的开发。
