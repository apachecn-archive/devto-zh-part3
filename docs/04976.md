# 为什么 Web 开发人员关注 Web 可访问性很重要

> 原文:[https://dev . to/flipped coding/why-it-s-important-for-web-developers-to-focus-on-web-accessibility-37n 3](https://dev.to/flippedcoding/why-it-s-important-for-web-developers-to-focus-on-web-accessibility-37n3)

互联网被称为“伟大的均衡器”。全世界的人们都可以获得信息，而这些信息曾经只限于那些负担得起的人。现在你可以在网上免费获得大学素质教育。作为 web 开发人员，我们还可以做更多的事情，让互联网世界变得更加公平。它涉及残疾人的无障碍环境。

## [](#accessibility)可达性

网页可访问性致力于让每个人都能使用互联网，不管他们有什么残疾。例如，有些人是盲人或很难看到你的网站，所以他们使用屏幕阅读器或其他设备。这些设备的有效性很大程度上依赖于网页的布局方式。

网页可访问性是有标准的。官方消息来自 W3C 的人和 https://www.w3.org/WAI/standards-guidelines/wcag/的人。这些标准并不特别复杂，只是很多开发人员不了解它们，或者他们继承了旧的项目。

## [](#how-to-make-an-accessible-website)如何制作无障碍网站

让你的网页具有可访问性是相对容易做到的。大部分都可以归结为使用通用的最佳实践和注意语义。自从 HTML5 更新以来，语义在可访问性中的作用变得更加重要。诸如 main、section 和 article 之类的元素在辅助设备上都有不同的解释。

这些设备寻找特定的标签来识别页面上的重要内容，决定它们应该在哪里寻找内容，以及如何将内容呈现给用户。因此，如果你在一个页面上使用多个 h1 标签用于格式化，停止它。这是辅助设备用来帮助理解页面内容的另一个元素。这也违背了最佳实践。

另一个简单的改变是在你所有的图片、视频和任何你需要看到的东西上添加替代文字。如果一个盲人来到你的网站，但没有一张图片有替代文字，他们会错过一些重要的信息，因为屏幕阅读器不会发图片的音。添加一些真正描述图片或视频要点的词语，这样他们就能很好地理解图片所描述的内容。

这就引出了你可以为可访问性做的其他事情。为网站上的任何视频添加字幕或文字记录。也有听力受损的人会错过视频中的大部分价值，如果它没有某种伴随的文本。

其他要考虑的事情是更新你的网站导航来使用导航标签。这有助于残疾用户找到他们想要的东西。通过将任何子页面与父页面相关联，确保您的导航有意义。这有助于用户了解他们在你网站上的位置。它还帮助辅助设备知道当用户完成一个页面时它们应该去哪里。

你能做的一件大事就是构建真正好的表单。如果需要，使用标签，将输入分组在一起。确保人们可以通过 tab 键输入，或者他们可以通过按 enter 键提交表单。我们的目标是让人们无论在什么样的设备上都能更容易地使用。

## [](#why-you-should-bother-with-accessibility)为什么您应该为可访问性而烦恼

一个严峻的事实是，许多 web 开发人员在紧张的时间限制下工作，大多数公司没有考虑语义工作的时间因素。在你的设计中包含网页可访问性需要一点额外的时间，但是这并不难做到。不要到处使用 div(我有时也会犯这种错误)，继续使用真正的语义元素。这就是它们的用途。

更新旧网站以遵循所有的可访问性标准是很难的，但是在你可以的时间和地点做出改变。不过，对于新网站，你可以从记住标准开始。你需要在你的设计中考虑可访问性，这样当要写代码的时候你就不会感到惊讶了。

你花费额外的几个小时让你的网站更容易访问，这可能是成千上万的人能够找到他们需要的信息的区别。起初这听起来很琐碎，但是想象一下，如果你在经营一家企业，并且它吸引了很多视障用户。如果你的网站无法访问，你就失去了潜在的大量客户。

最重要的是，让你的网站可以访问，这将决定一个孩子是了解大学的工程选项，还是决定他们不能这样做。网页可访问性不仅仅是网页开发者需要做的事情。网页可访问性是我们所需要的，以确保互联网能够真正成为一个公平的领域，让很多人的生活变得更好，或者至少稍微更有趣。

我过去一直认为网页可访问性的更新要花很多时间，直到我开始为自己寻找标准。这些标准迫使我们在创建网站时更加自律，这是一件好事。它强化了所有的最佳实践，并帮助我们思考人们将如何使用网站。
