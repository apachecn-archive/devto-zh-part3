# [](#payments)支付款项

[![Stripe Homepage](../Images/b1fcb941955108f744f43ffbafe0870e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--qEQLnSS---/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/00wylla1g1fgrp7my8ry.png)

如果你需要给信用卡充值，你会想使用像 [Stripe](https://stripe.com/) 这样的支付网关服务。通过直接向他们发送支付信息，您甚至可以避免看到客户的支付信息，将他们最重要的财务安全交给 Stripe 经验丰富的团队。它们提供了一个优雅的仪表板，易于使用的退款或处理争议交易的工具，与数百种其他服务的集成(包括本列表中的其他服务！)，以及一个开发人员友好的 API，用于将他们的平台连接到您的项目。

**其他伟大的解决方案:**

*   在线支付巨头 PayPal 提供了一个类似于 Stripe 的 API 和大量的集成。一个关键的区别是，与 Stripe 不同，你需要在结账时从客户那里收集支付细节——这可能会给你带来安全问题。
*   Braintree 是 Stripe 的另一个有价值的竞争对手。它在接受的支付种类上击败了 Stripe，包括 Venmo，但没有与许多其他服务集成。与 PayPal 的解决方案不同，你不需要接触客户的支付信息。

## [](#ecommerce)电子商务

[![Snipcart Homepage](../Images/a05e5d26bd6be2485f9484f328a8d91c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--rxKSoXiA--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/sb0wc2n5eqqc9uqknxzy.png)

有许多巨型电子商务平台可以为你提供一个简单的店面，但如果你想要更多的控制或需要在现有的网站上进行电子商务，那么这些平台可能不是最佳选择。像 [Snipcart](https://snipcart.com/) 这样的服务可以成为快速将电子商务添加到 JAMstack 站点的绝佳选择，因为它提供了简单设置和深度定制之间的平衡。使用 Snipcart，您将获得一个基本的购物车和结账界面，还可以深度定制外观和行为。您还将获得一个用于管理订单的仪表板和更多高级功能，如购物车恢复和库存管理。此外，它们集成了上面列出的所有支付网关，因此您将能够快速安全地获得支付。

**其他伟大的解决方案:**

*   Gumroad 是一个为创作者设计的简单、不干涉的解决方案。无论你是销售艺术品、应用程序或文章的独立人士，你都可以在 Gumroad 上快速建立一个商店，将他们的购买按钮添加到你的网站上，然后继续工作。然而，尽管 Gumroad 的小工具设计精良且极具吸引力，但如果你想定制结账的外观和感觉，那么你会希望使用不同的服务。
*   另一方面，[福克西](https://www.foxy.io/)更倾向于深度定制而不是快速设置。它是这里列出的最强大和可定制的解决方案，但是更适合开发者而不是商家。如果你需要的不仅仅是一个购物车和一个收银台，那么你可能想去别处看看。

## [](#memberships)会员

[![Memberful Homepage](../Images/f58d6c83f7717d3d97d2750c484529f4.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--lRtG5ZV4--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/7avkpv2tk4vgi5pzbj7f.png)

虽然电子商务网站提供了销售商品的能力，但销售网站订阅可能要复杂得多。如果你想为订阅者提供特殊的时事通讯，访问付费内容，等等，你会想使用像 [Memberful](https://memberful.com/) 这样的专业订阅服务。其核心是，Memberful 将简单的订阅流和管理与最常见的内容创建和管理工具(如 Wordpress、Mailchimp 和 Discourse)相结合。一个有形客户，[大众科学](https://massivesci.com/)，为管理他们社区实现了成员功能。Memberful 为他们提供了用于订阅新成员和成员访问其帐户的简单插件、用于实现付费墙和论坛认证的更高级的 OAuth APIs，以及用于维护特殊成员专用邮件列表的新闻稿集成。

**其他伟大的解决方案:**

*   如果你正在构建一个 web 应用，那么你可能想考虑一个更加 API 驱动的订阅管理器，比如 [Chargebee](https://www.chargebee.com/) 和 [Recurly](https://recurly.com/) 。虽然他们希望您提供自己的前端订阅界面，但他们会处理所有复杂的后端逻辑，以便向您的订户收费，并提供分析来帮助您减少客户流失。
