# 为什么你应该了解科技作家

> 原文:[https://dev . to/sales forceng/why-you-should-get-know-tech-writers-4hh 4](https://dev.to/salesforceeng/why-you-should-get-to-know-tech-writers-4hh4)

“等一下——你的团队中有 200 多名作家可以向其他人解释所有这些技术？我们怎么不知道这件事！?"

这是我的一位 Salesforce 同事提出的一个合理的问题。他和他的团队的任务是编写关于新功能的内容，他不知道内容和通信体验(CCX)团队可以提供帮助。他听说过技术作家。但他不知道和我职位相当的人是做什么的，也不知道为什么公司会雇佣拥有我技能的团队。这不是他的错。

历史上，技术作家在软件开发团队的阴影下工作。我们写的内容是大多数人试图避免的，除非他们陷入困境——在线帮助、发行说明、实施指南、API 开发人员指南等等。我们就像嵌入式记者，从产品设计过程的开始到编码周期的结束，询问关于功能行为的棘手问题，并提出解决内容问题的方法。我们创建 UI 文本、图形、视频、弹出窗口、演练、交互式教程以及更多提供清晰性或指导性的屏幕机制。最终，我们试图写出尽可能简洁的内容。(因为谁有时间去读那些东西？)我们与设计师、产品经理和工程师合作，构建需要更少解释的直观功能，并帮助客户完成工作。

有了那段经历，我们能提供的比言语更多。如果你在一家有科技作家的公司工作，这里有一些你应该了解他们的理由:

1.  它们可以帮助你以简单易懂的方式理解复杂的主题。在为各种技术开发内容时，技术作家采访专家——工程师团队。通常，工程师是如此的聪明和注重细节，以至于他们的答案看起来冗长或者难以理解。科技作家简化信息。为了尽可能写出最好、最简洁的内容，科技作家们把他们从工程师那里学到的东西拆开来消化，自己也变成了科技专家。结果是，在他们努力创建清晰内容的过程中，他们可以像学校里最好的老师一样解释最复杂的主题。你可以从科技作家那里了解科技，因为他们已经为你做了所有的研究。
2.  科技作家可以为你提供你不知道的有用内容。在技术作家开始内容创作过程之前，他们通常会收集与他们被分配到编年史的功能相关的每个内部规划文档。规范、代码样本、测试或商业案例——技术作家已经探索了所有这些。技术作家还帮助定义或定位最佳实践、疑难杂症、先决条件或用户故事，以阐明关于新产品或技术的最复杂(或不相关)的细节。此外，一旦客户可以使用该功能，技术作家就会跟踪客户反馈，并在可能的情况下帮助解决问题。通常情况下，技术作家对他们正在研究的技术有一个 360 度的视角，他们可以给你发送你正在努力寻找或不知道可用的内容。
3.  他们和其他能帮助你的人有关系。即使科技作家不能回答你的问题，他们也知道有人能回答。类似于电影被认为是一种合作艺术，技术也是如此。许多人和团队在幕后一起工作来创建一个功能，一个技术作家与他们所有人建立关系来跟踪他们编写内容所需的信息。通常，作者是团队之间的粘合剂，帮助他们理解局限性或沟通差距，因为他们的工作需要他们收集跨专业角色的见解。无论是产品营销，客户支持，产品管理，用户体验(UX)，产品管理，还是开发人员，科技作家都与他们合作，所以他们可以让你接触到正确的人。
4.  科技作家可以给你一个现实检查。当一项新功能或技术被开发出来时，关于它能解决什么问题的谣言比比皆是。幸运的是，科技作家是真理的来源。技术写作的目标和技巧是传达准确性——高于一切。不同于其他用来激发情感或吸引你注意力的写作形式，科技写作的目的是交流事实。科技作家是活的百科全书。他们可以准确地告诉你为什么要建造某样东西，以及它到底是如何工作的。

事实并不总是有趣的，这导致了技术写作有一个枯燥的名声。但是科技作家是一群活跃的人物。在 Salesforce，我们有为剧院缝制服装的作家，在世界各地演奏巴尔干或蓝草音乐的作家，出版推理小说和先锋诗歌的作家，跑 100 英里超级马拉松的作家，以及在希腊群岛之间游泳寻求刺激的作家。正如一位同事最近告诉我的，“你的团队太酷了，为我们节省了这么多时间……为什么我们不早点知道你呢？”
