# CSSConf EU 2019 在一个简短的观点回顾中

> 原文：<https://dev.to/viniciuskneves/cssconf-eu-2019-in-a-short-opinionated-recap-1ien>

这是在 [CSSConf EU 2019](https://2019.cssconf.eu/schedule/) 期间发生的关于其会谈的总结，供那些错过或到过那里并希望回顾的人参考。我还没有看完所有的演讲，所以我会对我看过的和我认为“必须看”的⭐的演讲发表一些看法。

我还添加了来自 [@lisi_linhart](https://twitter.com/lisi_linhart) 、 [@wilfredspringer](https://twitter.com/wilfredspringer) 、 [@clairikine](https://twitter.com/clairikine) 和 [@SarahOnDaCloud](https://twitter.com/SarahOnDaCloud) 的笔记，这些笔记很好地总结了每次演讲的内容。

如果你有一些额外的材料，你想在这里添加，让我知道！可能是额外的素描笔记、视频、评论、推文...你所想的一切都可以加入到这个列表中！

希望你喜欢！

#### [为什么我们称 HTML 标签为“标签”？通过 csS、数学和 CS 的历史来寻找答案！](https://2019.cssconf.eu/speakers/andres-cuervo/#talk)作者[安德烈斯·库埃沃](https://twitter.com/acwervo)

演讲的名字本身就说明了这一点。没什么技术含量，但是很有趣。如果你想知道为什么我们称它为标签，这是正确的说法。安德烈斯讲述了历史以及术语*标签*是如何产生的。【https://slides.cwervo.com/mdx-slides/css-conf-eu-2019/?】此处提供幻灯片:[mode=#0](https://slides.cwervo.com/mdx-slides/css-conf-eu-2019/?mode=#0)

#### ⭐ [班🥊碰撞](https://2019.cssconf.eu/speakers/alex-tait/#talk)由[亚历克斯·泰特](https://twitter.com/AT_Fresh_Dev)

你听说过函数式 CSS 吗？如果不是，这是一个很好的介绍。这个想法是让类做单一的工作，这意味着向元素添加一个特定的 CSS 属性，而不是一个有一堆属性的类，这样很难重用它。很多例子和它解决的问题的答案。Alex 提到[超光速粒子](https://github.com/tachyons-css/tachyons)是一个很好的功能 CSS 库。你也可以在这里了解更多:【https://rangle.io/blog/styling-with-functional-css/】T2
