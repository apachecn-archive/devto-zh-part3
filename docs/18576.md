# 工作时写日记

> 原文:[https://dev.to/kmcginn/journaling-at-work-1ef0](https://dev.to/kmcginn/journaling-at-work-1ef0)

我的记忆力不是最好的。或者说，最起码，我不能一直把工作中做的每一件事都记在脑子里。所以我把事情写下来！几年来我一直在调整这个系统，它对我来说效果很好。也许对你也有帮助！

> 注意:我一直使用 OneNote 做笔记(这是一个来自微软员工的大惊喜)，这对我组织事情的方式产生了影响。如果有人想把这个应用到其他系统中，请在评论中加入进来！

# [](#what-did-i-do-today)我今天做了什么？

[![gif of man staring out a rainy window](../Images/9d53742e4db7cb25bc8d3802822c1bd3.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--z1A1Tz8V--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://tse2.mm.bing.net/th%3Fid%3DOGC.46556737b9557bb8d8f45e2c4d52edcc%26pid%3DApi%26rurl%3Dhttps%253a%252f%252fmedia.giphy.com%252fmedia%252fObIZ5V7xGUhvW%252fgiphy.gif%26ehk%3DH52%252bJu%252bpr19niAbqxvlJCQ) 
有时候，我会在离开工作的日子里感到有些失望；看起来我没有完成任何事情，特别是当我做的一些事情不容易在标准工作项中捕获时。

我发现了一个帮助反击的窍门，那就是在我完成一件事情后，立即写下它的要点。令人惊讶的是它是如何开始增加的，尤其是当回顾整个星期的时候！

我的结构是在我的笔记本上有一个部分来写我的日记。我每个工作周都有一页，每个日期后面都有一个我已经完成的事情的列表。随着这本日记的增长，我将旧页嵌套在今年的标题页下。

一旦你这样做了一段时间，你就会开始注意你经常做的小动作的模式，或者人们问你的常见问题——这些都是自动化或共享文档的重要目标！

# [](#lets-get-together)聚一聚吧！

[![gif of Unikitty saying Business, business, business. Numbers.](../Images/7acd68fe58c431e240dbd22fb0e35e5c.png)T2】](https://i.giphy.com/media/Rju9mallzrqDK/giphy.gif)

唯一会让冗长无聊的会议变得更糟的是，会后你会意识到你已经完全忘记了会上讨论的一半内容。

我用我的会议记录部分来对付这个问题。每个会议都有一个页面，从日期、主题和与会者名单开始。随着会议的进行，我会写下一小段重要的内容，以及是谁说的。

我尽量不去担心纠正拼写或语法错误；与其在会议中修改文本时错过了一些新的东西，不如在会后花点时间清理一下。如果小组决定了一项行动，我会用粗体字表示，这样我以后可以更容易地找到它。

如果有人提供了一个链接或演示文稿，我会试着事后获取并把它放在笔记中。类似地，如果会议被录制，当它准备好时我会添加一个链接。

这不仅对我自己的记忆，而且对整个团队来说都是一个很好的资源，可以确保每个人都在同一页上！

# #作品

[![Schuyler sisters from Hamilton saying WORK](../Images/abfb30fc2fed78d914aa07521af22b96.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--LGFpXLsO--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://tse1.mm.bing.net/th%3Fid%3DOGC.1f2f23084d4f667179c47d87f2d6190b%26pid%3DApi%26rurl%3Dhttps%253a%252f%252fmedia.giphy.com%252fmedia%252fnTORsqR4gia6Q%252fgiphy.gif%26ehk%3DzEfXqnZWC9rPX40us6THmA)

现在是组成我的笔记本的大部分的部分:工作本身！

我的笔记本上有两个部分:活动工作和关闭工作。我的每个工作项都有自己的页面，有一个链接指向适当的工作项/bug/ticket/等等，以便正确关联。然后文字流就开始了！

当我写工作笔记的时候，我是以一种意识流的叙事风格来写作的。我说我发现了什么，我是如何发现的，以及我认为它在那一刻意味着什么。把这些部分敲出来的行为帮助我更清楚地处理信息，并找出下一步该去哪里。

这些注释不同于我在工作项本身中所做的更新，因为这些注释都是关于我的思考过程和细节的。对工作项的更新更多的是交流进展和突出关键发现，一目了然；回顾我最近的笔记也可以使这些工作项的更新变得相当容易！

对我来说，一个关键的概念是永远不要从我的笔记中删除文本，除非我几秒钟前刚刚写了它。如果我之前写错了什么，我会应用删除线样式，但我会将文本留在原处，以便我可以注意到我的错误，并在以后搜索笔记时从中吸取教训。

我发现在我的笔记中包含各种“工件”非常有用:源代码的相对路径、屏幕截图、代码片段、附件、电子邮件、表格或任何有助于我捕获和处理问题的东西！

每当我不得不在另一天回到某个特定的作品时，我会插入日期，然后继续写。这有助于打破文本，也为我将来回顾笔记时提供了有用的背景。

一旦一项工作完成(或转移/暂停)，我就将页面移动到关闭的工作部分。理想情况下，关闭的工作区将继续增长，而活动的工作区保持较小的大小。如果活跃工作开始增长，这可能是随机化的迹象！

# TL；速度三角形定位法(dead reckoning)

如果你想采用我的系统，这里有一个快速总结:

*   为你的日志创建一个部分，每周创建一页，用要点记录你每天做了什么。全天保持更新。

*   为会议笔记创建分区，并为您参加的每个会议创建页面。记笔记，突出行动项目，任何对你有帮助的事情！

*   为活动工作创建一个部分，并为您处理的每个工作项创建页面。记录下你在完成过程中的发现和尝试。

*   为已完成的工作创建一个区段，当您完成(或暂停)工作项目时，将页面从活动工作区段转移到此处。然后，您可以使用您在这一页中的粗略笔记来帮助您在工作项本身中生成解决方案的摘要。

假设你的笔记软件有很好的搜索能力，你会慢慢产生一个很好的资源——不用再为你几个月前实施的解决方案绞尽脑汁。只要搜索关键词就能找到！
