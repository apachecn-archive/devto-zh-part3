# 代码模型:编程行业角色模型的 Twitter 列表

> 原文：<https://dev.to/jsjoeio/code-models-a-twitter-list-of-role-models-in-the-programming-industry-3n3c>

在学校和运动中，我的老师和教练鼓励我去寻找一个“榜样”——一个我可以尊敬的人。一个我想成为的人。

它给了我方向，因为我告诉自己，

> “我长大后想成为他们那样的人。我想像他们一样积极地影响这个世界。”

这种寻找我们钦佩的人的做法可以激励我们“成为世界上[我们]希望看到的变化。”

我认为采纳这个想法并创建我称之为“代码模型”的东西会很有趣——这是编程行业中的一系列角色模型。

### Twitter 上的列表是什么，为什么要用它？

我直接引用推特:
[![A List is a curated group of Twitter accounts. You can create your own Lists or subscribe to Lists created by others. Viewing a List timeline will show you a stream of Tweets from only the accounts on that List.<br>

这和只是跟着那些人走的区别，本质上就是“把人分在一个地方”。

当你查看你的时间线时，你会看到你关注的每个人的推文，而列表**只显示列表**上的人的推文。

[![sheep giving a thumbs up](img/1a4764b8eea48a3053275373a0522038.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--6cKB8alj--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://i.giphy.com/media/tIeCLkB8geYtW/giphy.webp)

我觉得为这个做网站没什么意义。反正我们大多数人已经花时间在 Twitter 上了。

当时最简单的解决办法是在 Twitter 上创建一个列表，这样你就可以很容易地添加你崇拜的人。这将有助于您在一个集中的位置查看他们在做什么和共享什么。

我开始制作我的[清单](https://twitter.com/jsjoeio/lists/code-models)，你可以订阅，但我建议你自己制作！😄
