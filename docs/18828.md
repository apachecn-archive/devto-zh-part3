# 1000 个字符的突变测试

> 原文:[https://dev . to/schreiber _ Chris/1000-characters-193 a](https://dev.to/schreiber_chris/mutation-testing-in-1000-characters-193a)

代码覆盖率是衡量代码质量的常用指标，但它不能保证测试真的测试了预期的行为。

突变测试可以被描述为测试的测试。它包括向您的代码库引入突变，并检查每个突变是否至少有一个测试失败(在这种情况下我们说突变被杀死，否则它存活):

*   条件反转
*   改变边界条件(
*   改变计数器的增量
*   更改数学运算符
*   返回 null 而不是返回值
*   不要调用 void 方法、构造函数...
*   许多其他类型转换

如果一个突变被杀死了，那意味着你的测试覆盖了一些边缘情况，给你更多关于测试质量的信心。

突变测试可能是一个繁重的过程，所以它不需要像单元测试那样频繁地执行，但是它可以是您的发布过程中的一个步骤。

Java 项目可以使用 [Pitest](http://pitest.org) 库。
