# 道路和桥梁

我最近读了 Nadia Eghbal 的《道路与桥梁:数字基础设施背后看不见的劳动》的前半部分，作为我自由/开源软件课程的作业，我真的很喜欢。我计划下周有时间的时候完成论文的另一半，但现在我必须专注于我的功课。Eghbal 是一名软件开发人员，也是开源软件的倡导者和研究者。

她的论文是关于开源软件的影响和世界对工具的依赖，这些工具是由无偿的程序员出于热情或必要而开发的。这是一篇关于自由/开源软件的流行以及伴随而来的好处和缺点的很好的读物。我真的很喜欢文章中关于缺乏组织和贡献者的讨论，以及描述不同开源工具的小图，这些工具都有简短的关于它们是如何开发的以及由谁开发的简介。虽然它非常冗长，但我真的认为这篇文章值得一读。

Eghbal 对开源软件的未来提出了一些非常有趣的问题，在我到目前为止阅读的章节中，她没有给出明确的答案。我们如何计划激励更多的人来贡献和维护代码，以帮助修复贡献者和用户之间的偏差？她指出，现在大量使用开源工具的人远远超过了为其做出贡献的人，这让项目所有者有太多的工作要做。她还问开源项目从没有投资回报计划的风险投资中拿走大量资金是否合乎道德或逻辑，她将许多这样的项目描述为“亏损领袖”。她为 OpenSSL 等项目描绘了一幅黯淡的画面，这些项目很快就会耗尽资金或让维护者退休。

# npm

我还被要求阅读 npm 5.7.0 上的一个[问题线程](https://github.com/npm/npm/issues/19883)。这篇文章非常清楚地说明了用户遇到了什么问题，以及他们做了什么导致了这个问题。npm 5.7.0 导致了文件权限的改变，用户只是简单地报告了这个问题，并告诉团队如何重现它，但评论线程是一个巨大的垃圾箱火灾。显然，试图更新 npm 包的人得到了一个显然不工作的预发布版本。这个帖子很快变成了一场巨大的争论，争论的焦点是这是用户的疏忽，还是维护人员没有明确标记这是预发布版本的错误(运行`npm install`将安装 5.7.0)。人们开始讨论 Edgal 在她的论文中讨论的一个话题:只有两个人维护这个被无数人使用的重要工具。线程变得如此糟糕，以至于维护人员不得不限制对线程的访问。npm 的新回购甚至没有一个问题部分可以发布，而旧回购现在有 2000 多个问题。难怪他们看起来不知所措。
