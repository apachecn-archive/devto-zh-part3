# 动荡的一个月

> 原文:[https://dev.to/abraham/a-month-of-flutter-47c4](https://dev.to/abraham/a-month-of-flutter-47c4)

*最初发表于[bendyworks.com](https://bendyworks.com/blog/a-month-of-flutter)。*

最近，尤里·沙克德写了一篇鼓舞人心的文章，讲述了他连续一个月每天发表博客的经历。我决定把他的伟大想法和我成为更好的[flutt](https://flutter.io)开发者的目标结合起来。**在 12 月份，我将构建一个 Flutter 应用程序，并写一篇关于这个过程的每日博文。**

## [](#my-goals)我的目标

*   成为更好的 Flutter 开发者
*   练习撰写简洁的博客文章
*   练习编写项目需求

## [](#what-i-am-building)我在建造什么

对于这个挑战，我将建立 Birb，一个专门针对鸟类的图片分享社交网络。一个鸟类的 Instagram，如果你愿意的话。

由于我还没有建立应用程序，我实际上不知道有多少会在 12 月完成。以下是我计划中的 [MVP](https://en.wikipedia.org/wiki/Minimum_viable_product) 特点:

*   所有张贴图像的全局流页面
*   用户认证
*   图像发布
*   用户个人资料页面
*   单个图像页面
*   图像页面上的注释

如果我完成了 MVP，我会添加更多的功能。一些潜在的特征，没有特定的顺序:

*   推送通知
*   图片上的#标签
*   A/B 测试
*   主题
*   友情图/关注
*   朋友图像流
*   喜欢图像

## [](#what-to-expect)期待什么

每天，我都会创建一个新的博客来描述我最近的进步。在一个月的时间里，这些小步骤有望积累成一个不错的应用程序。博客帖子大多简短扼要，涵盖一个小部件或功能。

由于这将是一个正在进行的开发项目，我将尝试不同的模式，不要期望每件事第一次都能完美实现。一路上会有重构和糟糕的选择。然而，我将遵循最佳实践，如编写测试、运行 [CI](https://travis-ci.org/) 、林挺等。

如果你是专业发展工作的新手，你可以遵循[流程](https://github.com/abraham/birb/projects/1)并深入了解项目是如何运行的。我将编写故事和实现功能，我的同事[珀尔](https://twitter.com/pblatteier)将审查进展和代码。

## [](#technologies)技术

我计划在 Birb 中使用的一些技术:

*   颤动(咄)
*   数据库的火灾
*   用于用户身份验证的 Firebase Auth
*   用于托管映像的 Firebase 云存储
*   用于无服务器计算的 Firebase 云功能
*   面向美学的材料设计
*   设备媒体 API
