# 你是 CTO 吗？

> 原文:[https://dev.to/wparad/are-you-a-cto-457](https://dev.to/wparad/are-you-a-cto-457)

> 所以，我们都是 CTO。既然没有一致的行业定义，这对我们每个人意味着什么？

[![](../Images/afdb6261f50096079f4faa9c13c0ae79.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--epkEqPVc--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/960/0%2AgBnt2QlCqZrRqawB.jpg)

这不是我第一次看到这个问题，也不是我第一次有答案。但我决定借此机会把它写下来。

在我开始之前，我想先快速切入正题。越来越多的公司正在成为科技公司，我很难在这里列出一个今天没有以某种方式使用它的公司。我知道它们存在，但是你真的能想到一个不用某种设备的吗？这可能是吹毛求疵，但一旦你在工作中使用设备，它现在就是你业务的一部分。如何设置设备，如何与客户和同事沟通，现在是一个重要方面。如果没有它，你将无法工作，因此，我认为这个建议适用于每个人。

我目前的看法是， **CTO 现在是一个过时的术语**,原因有二。

### [](#technology-as-a-differentiator)技术作为差异化优势

技术不再只是组织的一小部分，它渗透到每一个地方。在每一个角落，你都会发现它在那个地区是独一无二的，与其他任何地方都不一致。有一个统一的观点并不意味着什么。在一家科技公司中，对于每台设备的使用方式或所有软件的使用策略，不可能有单一的愿景。

当技术还是公司的一个小 IT 部门时，让 CTO 来管理核心技术职能更有意义。

> 我们需要向公司注入技术，以超越我们的竞争对手

我这辈子都没听过这句话，技术也不再是竞争优势的工具。当技术在我们的社会和商业中还没有普及的时候，它就是区分器。你可以制造更多的产品，让它更快，内部沟通，外部沟通，等等。但现在这已经不再是创新了，你不会因为“他们以前听说过那个东西”就把某人带进你的公司来管理技术。大家都知道什么是技术。

此外，在您的组织中，越来越多的人可以获得知识并了解技术，不再需要某个特定的人来推动。如果我听说今天公司里有一个人*比技术人员*更了解技术，我会感到震惊。

### [](#fixed-organizational-structure)固定的组织结构

公司不再有固定的组织结构。相反，你有*个层次*的组织。其中最底层我称之为**业务**，公司有一个**董事会**，管理公司所有的**业务**。在历史**业务**中，你可能会发现标准的 C 级高管拥有相同的头衔，但随着时间的推移，业务结构已经发生了变化。我们不再总是在每个组织中都有相同的结构，这是显而易见的。在非常大的组织(> 8000 人)中，公司在商业市场的不同方面面临着许多挑战，通常每个商业领域都有一位首席执行官。

将首席技术官在公司董事会中的角色称为“T1”，将首席技术官在 T2 公司中的角色称为“T3”，将首席技术官在初创公司中的角色称为“T3”已经没有意义了。从例子中可以明显看出，但我无法想象董事会的首席技术官会比董事会中的其他高管更多地接触技术。出于同样的原因，我认为小型创业公司的 CTO 只不过是随意授予第一位非 CEO 的联合创始人的一个空洞的头衔。他们正在进行核心开发，与那里的其他工程师没有什么不同。这没有什么错，但很明显，不是同一个角色。这些不可能都是 CTO。

如果我们看看企业、董事会、团队或任何你称之为你的**团队**的高管，在每一个层面都需要问责制和在一个领域拥有深厚技能的多元化成员(也称为 T 型)。从根本上说，团队对股东、董事会、工程总监、团队领导或你的客户负责。(也许你有一个有趣的组织，你的一些成员对政府、地球或我们人类负责，但通常不是这样)

### [](#who-is-your-team)你的团队是谁？

问责必须是一个实体/个人，所以我会选择一个词来定义首席问责官或首席行政官，除此之外，你有一个拥有一般技能、知识和能力的高管团队。因为我们都是技术专家，所以不要有 CTO，而是专注于我们最了解的领域。你的团队成员将需要你所负责的许多领域的专业知识，无论是业务环境、软件开发、架构、创新、财务等。而在初创阶段，这就没那么有趣了，因为知道你应该成为哪个领域的专家才是最重要的。在**董事会**和**业务**层面，这一点更加清晰。

快速搜索会启发你最佳的团队规模，但那是一个不同的话题。总共是 5 2 人，最多 7 人。在一个团队里，在你的班里，在你的会议室里。使用这个硬性限制，你会把你的专长放在什么地方？最近我听说越来越多的首席产品官、首席创新官和其他一些有趣的人物。有首席**人类**、**幽默、**和**快乐**官员，正如可能有首席**环保联络员**、首席**人工智能战略家**或首席**数据分析师**。因为我知道我的团队都是专家技术人员，我真的不想浪费我团队的一个关键职位在*T*字上。
