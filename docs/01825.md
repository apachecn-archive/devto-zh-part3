# 是的，你应该有代码之外的爱好

昨晚，我看到了推特用户 Amycruz 的一条精彩推文:

> ![Amycruz 👩‍💻 👩‍💻 👩‍💻 profile image](../Images/d2d1648eca04f50d6746d4c20cab9d0c.png)👩‍💻 👩‍💻 👩‍💻[@ amarachiamaechi](https://dev.to/amarachiamaechi)![twitter logo](../Images/4c8a2313941dda016bf4d78d103264aa.png)要成为一名优秀的开发人员，你不必把 99%的时间都花在编写、阅读关于编码的东西上。做其他活动。它有助于更有创造性和丰富你的知识。

这是一个伟大的观点。我看到许多开发者在谈论处理网上数量惊人的信息有多难。一方面:*感觉有点竞争*。谁能最快学会最新的框架，并构建一个超级酷的 app 并开始赚钱？另一方面:因为害怕落后，很难选择以什么顺序学习哪些东西。

让我们首先承认，我们永远也学不会所有的东西。首先，不可能。其次，没必要。学习你想要的工作所需的技能。然后，您可以根据需要填充空白。然而，像我一样的所有学习语言的人可能想知道如何将额外的学习融入到这种心态中。答案是平衡。

你必须平衡学习和其他活动。没有平衡，你就有精疲力竭的危险，就像大多数事情一样。所以，找点爱好吧！我个人喜欢编织、钩针编织、十字绣和阅读回忆录。创造性的爱好是非常有益的。你可以结识有相同爱好的新朋友，在工作面试中有有趣的事情来谈论自己，并暂时从压力大的事情中清理你的大脑。

你有什么非代码爱好想要分享吗？掉个评论！:)
