# 程序员可能的职业道路

> 原文:[https://dev . to/rlxdprogrammer/possible-career-paths-as-programmer-4fkm](https://dev.to/rlxdprogrammer/possible-career-paths-as-programmer-4fkm)

在前几周，我写了很多关于程序员职业发展可能性的文章。文章第一篇"[如何选择适合自己的公司？](https://dev.to/rlxdprogrammer/how-to-choose-the-right-comapany-for-you-1bk5)“我试图概述不同类型的公司。在那之后的“[程序员的自由水平](https://dev.to/rlxdprogrammer/levels-of-freedom-as-a-programmer-3m9n)”中，我写了作为程序员独立工作的机会。这一次，我想总结一下作为一个典型公司的员工的职业发展机会。

如果你在一家公司工作，你正在发展自己，并且越来越好，迟早你会到达一个点，你需要选择你的载体的方向。这些职业道路在大多数大公司都存在，但是它们可能有不同的名称。如果确定了三条主要的职业道路:团队经理、项目经理和技术专家。但除了这三个职位，我还想提一下其他一些不太可能的职位:质量经理、技术销售同事和培训师。我认为这些是主要的定位机会。对于较小的公司来说，这可能是因为它们没有被严格分开。我也试着给每个职位一个简短的描述来帮助你决定哪一个最适合你。

# [](#the-technical-expert)技术专家

一个程序员通常的发展大概是这样的:初级开发人员，高级开发人员还有这里来的是什么？通常这是你需要决定是继续编程还是专注于其他事情的时候。如果你的决定是编程，那么技术专家可能是一个很好的途径。技术专家也经常超越软件架构师的角色。这意味着他们正在设计新的软件模块，并从技术角度控制整个开发:进行代码审查，支持技术决策，在技术问题上支持其他同事。安静的技术专家也经常深入一个主题，并在该领域获得真正的专家，并在公司全球范围内支持特殊领域的同事。

但是你需要知道技术专家也不是整天都在编码:他们需要参加几次会议，和其他人讨论，支持项目经理等等。如果你想在技术上深入这个话题，这是你的正确道路，但不要忘记:除了伟大的技术技能，你还需要良好的软技能。

# [](#the-project-manager)项目经理

关于项目经理的角色我已经在文章[中写过如何理解经理？](https://dev.to/rlxdprogrammer/how-to-understand-the-managers-1p7h)。他们负责项目的时间和成本规划和跟踪。他们与所有利益相关者(客户、高层管理人员、开发团队等)进行沟通。)并且他们正在创建大量的文档(主要是计划和报告)。他们通常需要参加很多会议。项目经理的生活压力很大，因为你要负责整个项目或子项目。你需要很好的沟通和压力处理技巧。你工作中最常用的程序是 Excel 和 PowerPoint。通常，项目经理不会深入参与技术主题，但是如果经理对项目的技术方面有一个粗略的了解，这是很有用的。开始时，你很可能只负责一个较小的子项目，但后来你可以接管一个项目或一个特定客户的所有项目。在这种情况下，你将不再与开发团队有任何直接的联系。

# [](#the-team-manager)团队经理

关于团队经理的角色，我也在文章[中写过如何理解经理？](https://dev.to/rlxdprogrammer/how-to-understand-the-managers-1p7h)。他们负责拥有一个技术上合适的、动机良好的项目，能够成功超越所有项目。为此，团队经理需要参加与团队成员的面谈、一对一会议，需要为团队定义愿景和目标并跟踪它们，接下来还需要与项目经理和更高级别的管理人员保持联系。我认为最大的挑战是与你的所有团队成员进行沟通，给他们一个良好的动力，激励他们进行自我发展，做出高质量的工作。为此，你需要非常好的人际交往技巧。尽管团队经理没有深入参与开发，但也需要对团队正在做的事情有一个高层次的了解。如果你选择这个方向，很可能首先你只负责团队的一小部分(5-6 人)，然后是整个团队(15-30 人)，最后你有机会更上一层楼，负责一群团队或整个部门。

如果你认为你能解决人与人之间的所有冲突，那对你来说是正确的选择。

# [](#the-technical-sales-colleague)技术销售同事

软件公司的销售负责公司业务的延伸。去寻找新的机会和新的项目。此外，还要为要做的项目计算出正确的价格。对于这种活动，他们需要一些技术支持，这些技术支持能够识别公司的技术实力，能够理解项目机会的技术内容并提出进一步的特征，能够对项目所需的开发时间和所需的技能进行高水平的评估。这些通常是技术销售同事的任务。如果你对商业更感兴趣，但你仍然想利用你的技术知识，这对你来说也是一个机会，然而这作为一种职业道路在大多数公司并不突出。

# [](#the-quality-manager)质量经理

质量经理是负责产品质量的人。为了能够做到这一点，质量工程师需要支持开发过程的优化，并关注所有开发团队都遵循该过程，尤其是其质量相关的部分(审查、文档和测试)。为了做到这一点，质量经理通常通过项目经理与不同的开发团队保持联系，并定期控制项目文档。对于这个职位，你需要足够精确和注重质量。

# [](#the-trainer)驯兽师

大多数大公司都为员工举办培训课程。它们要么由公司的同事在内部完成，要么由外部公司完成。在这两种情况下，这些培训都是由专业培训师进行的。培训的主题可以是技术性的，也可以是软技能(沟通、时间管理、压力处理等)。).这也是一个将自己定位于这个方向并成为专业培训师的机会。这并不是在每种情况下都要花费你 100%的时间，所以你可以在接下来的其他活动中做这件事(比如成为一名技术专家)。
