没有一个框架是完美的，但是我一直觉得 Rails 团队在记住应用程序开发是什么方面做得很好。不是它*可能*是什么样子，或者它*理想中*是什么样子，而是它实际上是什么样子。

我不是要批评任何特定的技术，尤其是 React——这可能是自 jQuery 以来最令人惊叹的前端 web 开发技术。但是这个错误显然已经在这个项目以及其他项目中不时地出现。

记住作为用户是什么感觉。核心团队应该有一个多样化的应用程序团队列表，这些团队有不同的需求来指导他们。他们也可能希望以合同工的身份轮换到不同的组织，以便尝一尝这种药的味道。
