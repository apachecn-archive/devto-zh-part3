# 开发人员回顾#10:本周前 7 项，已讨论

> 原文：<https://dev.to/dangolant/devto-review-10-top-7-of-the-week-discussed-f2l>

> 在你看到笔记之前，我想提一下我们现在已经在 iTunes 上了！这意味着无论你使用什么样的播客应用程序，你都应该能够看到我们。我们仍在搞清楚这一部分，所以最新的一集可能需要一点时间来展示，但你可以通过倾听任何你错过的对话来填补这段时间。如果你想要直接的 RSS 链接，你可以点击[这里](https://pcr.apple.com/id1444162833)。

在看望家人一段时间后，Malik 和我本周回来了(比以往任何时候都更好)，我们聚在一起谈论本周关于 dev.to 的 7 篇热门文章。

在我们的第一篇文章中，[“对程序员有用的资源”](https://dev.to/sahilrajput/useful-websites-for-programmers-36k)， [Sahil](https://dev.to/sahilrajput) 分享了一份 Github 上的优秀学习资源列表...嗯，我们以前用过“详尽无遗”这个词，但这次真的名副其实。Malik 和我讨论了一般和特定内容的价值，我们检查出的一些链接，以及人们为什么继续写作。

我们继续讨论[“如何让 Python 与 3 行代码并发”](https://dev.to/rhymes/how-to-make-python-code-concurrent-with-3-lines-of-code-2fpe)，由[押韵](https://dev.to/rhymes)，其中涵盖了仅使用 STDlib 工具将发出 1000 个请求所需的时间减少 7，以及并发执行的复杂性。Malik 和我讨论了并发性、标准库的价值以及围绕重新实现标准功能的语言文化。

在[“node _ modules 问题”](https://dev.to/leoat12/the-nodemodules-problem-29dc)，[莱昂纳多](https://dev.to/leoat12)将 NPM 管理依赖关系的方式与他在 Maven、Gradle 和 Nuget 上的其他经历进行了比较，以及文化(*主题警报*)如何在深层`node_modules`目录中发挥作用。Malik 和我将此与我们对 STDlibs 的讨论以及我们自己引入不必要的依赖的经历联系起来。

[安德鲁·戴维斯](https://dev.to/restoreddev)为我们带来了[《在编程时展现善意》](https://dev.to/restoreddev/showing-kindness-while-programming-52a7)，他在其中讨论了良好文化的基本组成部分:个人习惯。Malik 和我带着 Andrew 的概念运行，并讨论将移情*付诸行动*，庆祝胜利，以及在沟通时责任由谁承担。

在[“编写良好的单元测试:一步一步的教程”](https://dev.to/ice_lenor/writing-good-unit-tests-a-step-by-step-tutorial-14kj)， [Elena](https://dev.to/ice_lenor) 给出了确保你的测试健壮的一步一步的步骤，从黑盒到白盒，到质疑基本原则，使用一个计算世界距离的示例工具。Malik 和我讨论了每种方法的权衡，多少测试是足够的，以及我们有多信任被测系统。

接下来，我们讨论[“5 个致命的 Docker 陷阱😱-对于新用户"](https://dev.to/uilicious/5-fatal-docker-gotchas----for-new-users-2o87)。 [Eugene Cheah](https://dev.to/picocreator) 讲述了他和他的公司在过去一年中遇到的常见码头陷阱，并提供了从简单到“只有在不得已的情况下”的缓解选项。本着尤金帖子的精神，我们分享自己的 Docker #失败。

最后，我们讨论了[“一年前我根本无法做到这一点，但我刚刚构建了我的第一个 Web 应用程序。在这里！”](https://dev.to/jmschuurmans/a-year-ago-i-never-would-have-been-able-to-do-this-but-i-just-built-my-first-web-app-here-it-is-1534)。 [Jeremy Schuurmans](https://dev.to/jmschuurmans) 讲述了他长达一年的编写软件之旅，以及这是如何让他在 7 天内创作出第一个个人项目的。Malik 和我讨论了 Jeremy 的方法，他对其背景影响的思考，以及我们自己写个人项目的经历。

我们跳过本周的“主持人的选择”，但是我们强烈建议你看看 [Grigor 在](https://dev.to/grigorkh) [HTTP/3](https://dev.to/grigorkh/what-is-http3--4pib) 上的帖子。
