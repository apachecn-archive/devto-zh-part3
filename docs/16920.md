## 一个简短的背景故事

我从 20 世纪 90 年代末开始制作网站。我在 2007 年到 2014 年间休息了一段时间，至少可以说，网络开发生态系统已经发生了很大的变化。我一直使用的工作流程对我来说非常清晰，并且符合我的目的。

#### 我习惯什么:

*   超文本标记语言
*   JavaScript(用于效果和交互)
*   半铸钢ˌ钢性铸铁(Cast Semi-Steel)
    *   我在 2014/2015 年为此添加了 SASS
*   一个 php 驱动的 CMS*(Craft/Perch late)*和 PhPMyAdmin 来管理我的 SQL
*   一个共享托管解决方案(1 和 1)，我可以指向域和存储数据

有了这些技术，我可以制作模板来呈现 CMS 中的内容。我的客户对这种方法很满意，我可以使用这种工作流程交付解决方案。然而，我是在去年开始关注 React/GraphQL 运动的。最近我一直在关注盖茨比。

## 我所做的了解和学习

我浏览了他们网站上的 React 教程，和 GraphQL 教程一样，阅读并理解了这些技术是什么的粒度主题。我已经熟悉了包管理器。

通过我的正规 CS 教育，我已经用汇编语言编写了一个文本编辑器，用 C 语言编写了 IPC，但是我更困惑和沮丧的是用 react 构建一个作品集网站(把 Gatsby 作为我的堆栈)，并且在这个过程中没有什么可以展示的。我经常迷失在几十个部分解释的主题中，与其他相关主题没有联系。我感觉自己淹没在竞争框架开发者的推销中，他们都想让我尝试他们的*棒极了的*新技术，他们都做同样的事情——同样的事情？这是什么？我想做很棒的网站。我想从设计转向我用这项技术创建的面向公众的网站。

我不喜欢这么愤世嫉俗。我更喜欢高效和乐观。我感到精疲力尽、沮丧和被利用。

## 审视自己内心

我有足够的理性去理解，如果行业领导者使用这些新的 web 开发技术来为自己谋利，那么我一定是弄错了。我想的不对。作为一名独立学习者，我已经耗尽了自己的资源。哈尔普！

## 崩溃求助

在这一点上，我崩溃了，并寻求帮助。我试图找到一个高层概述如何使用 react 和依赖于它的各种技术的流程图框图*(无论在任何给定时刻都是什么)*，但没有找到任何东西。

#### 核心问题我有

*   托管(面向公众)？我不能在运行时将包安装到我的共享主机计划中。我不能忍气吞声。没有 root 访问权限。我用什么代替？我在家里运行服务器吗？
*   是单靠核心 React 库不够强大吗？我需要安装一个 CMS 或者其他的包来让 react*有用吗*，php 本身并不能提供任务 x、y 和 z 所需的所有抽象吗？
*   最重要的:有什么(最好是免费的)教程/学习资源推荐？也许我的搜索没有返回最好的结果。
