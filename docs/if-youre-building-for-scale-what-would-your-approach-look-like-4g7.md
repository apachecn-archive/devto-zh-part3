# 如果你是为“规模”而建，你的方法会是什么样的？

> 原文：<https://dev.to/aizkhaj/if-youre-building-for-scale-what-would-your-approach-look-like-4g7>

最近，我被忽略了一个机会，因为他们想要一个有生产经验的人，为 10 万或更多的用户群大规模构建一些东西。虽然这是一个很好的反馈，但我很难想象“规模”。我的意思是，我目前在一家用户群大约为。30 万，但我没有建立它的前端/后端，让它走了这么远。

规模建设到底是什么意思？它在前端和后端是什么样子的？

当我想到扩展时，我想到的是开发运维，确保您的服务器能够处理“扩展”并使用缓存(Redis)设置分布式数据库。

那么，如果明天你要开始一个项目，全栈工作，你如何确保它的可伸缩性，以便它可以处理第二天大量涌入的用户(极端的例子很多)？
