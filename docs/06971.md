# 我开始写博客的原因是

> 原文:[https://dev.to/bubbit/the-reason-i-will-start-blogging-4k8m](https://dev.to/bubbit/the-reason-i-will-start-blogging-4k8m)

在过去的 2 到 3 年里，我脑海中一直浮现的一个问题是，当我成为一名开发人员时，我是否应该开始写博客来记录我的学习经历？

# [](#why-should-i-do-it)我为什么要做？

如果你浏览不同的博客网站，从 [Medium](https://medium.com/) 到 [Dev .到](http://dev.to/)，显然你在这些网页上找到的答案会是肯定的，博客为你创造奇迹:

*   提高你的写作技巧
*   提高你的批判性思维
*   组织你的思想和学习
*   向广大观众推销自己

# [](#why-didnt-i-do-it-before)我之前为什么不做？

在我的脑海里，我也多次用是的，我应该回答这个问题。但是我从来没有按照那个答案去发布和发表任何东西。在不同的文件存储系统上有一些过期的页面，从工作相关的到个人的帖子等等，但是没有任何东西会出现在博客网站上。除此之外，另一个问题总是被问到:我应该把我的博客放在哪里？我应该写我自己的博客吗？随之而来的总是长时间的沉默和犹豫不决，有时还会创建一个入门应用。

从我的角度来看，它总是回到同一个众所周知的场景，完成事情的问题。我开始热情地写博客文章的大纲，就像我现在正在做的一样(当然，如果你正在读这篇文章，这将是无效的...)然后留待另一天来完成这篇文章并阅读它，但这另一天永远不会出现，或者在未来很久之后，当这篇文章过时并需要调整以变得有价值时。

# [](#why-will-i-do-it-this-time)为什么这次我会这么做？

在我的工作中，我们必须为每一年制定我们想要如何发展自己的目标，并设定那一年可以实现的目标。正如我之前提到的，写博客在过去几年里一直在我的脑海中，它也一直在我的目标的“待办事项”中，或者作为另一个目标的潜在部分。它总是以没有被选作其他目标而告终。今年，我将改变这种情况，看看是否通过增加额外的动力(更好的年终评估)，我将坚持下去，并真正开始写博客，以看看上述关于提高自己的说法是否属实。

# 我会写些什么呢？

我发现总是很难选择一个主题，你可以在博客上写一些人们可能会感兴趣并发现实际有用的东西，所以我把它附在我今年的另一个目标上。我们刚刚开始在工作中采用 [Ansible](https://www.ansible.com/) ，对很多人来说这是一项新技术，所以我决定参加一个在线课程来了解它。在收集了这些知识之后，我本可以决定自己使用这些知识，并与他人分享课程的链接，但这样我就不会在我认为我需要最大改进的领域提高自己，向他人解释简单易懂的事情。这也和一句名言密切相关:“如果你不能简单地解释它，那你就理解得不够好。”。根据这个[帖子](https://skeptics.stackexchange.com/questions/8742/did-einstein-say-if-you-cant-explain-it-simply-you-dont-understand-it-well-en)，它被误认为是爱因斯坦的语录，而且从来不是他说的，但是我仍然喜欢它。

所以，我最终挑战了自己，为工作中的同事们创建了一个内部研讨会，但接下来我将创建一系列博客，这些博客将与研讨会的不同步骤保持一致，并希望能帮助人们寻找可行的知识。除了研讨会的步骤之外，我还将对我将在研讨会中使用的技术堆栈做一个概述，该技术堆栈基于 Lit 前端，NodeJS API 作为后端(仍未决定使用哪个服务器基础)，最终结果将是一个具有内置分析和数据库的网站，当然这已经做过多次了。

所以我在这里，基本上回答了过去几年我一直在问自己的问题，我是否应该开始写博客，是的，我会尝试一下，我会看看在未来的几个月里它会把我带到哪里。
