# [](#why-its-important-to-stay-informed)为什么保持消息灵通很重要

关注新闻是成为一个有弹性的 R 用户的一个重要部分，因为它帮助你解决你不知道你有过的问题(也许你会看到一个包，可以取代你的一个繁琐的脚本！)以及你将会遇到的问题(有一天，你将需要制作一张地图，并感谢你已经隐约知道`sf`的存在)。显然，保持警觉会有所帮助，因为您了解了工具及其应用。但它也有帮助，因为它让你了解人和组织！了解谁在做什么以及某个主题的进展发生在哪里是非常有价值的，因为当你以后有意识地寻找某个东西时，它会帮助你更容易地找到你的方向。

# 从哪里获得你的 R 新闻补丁

这里有一个可能并不详尽的列表，列出了从哪里可以获得 R 新闻。挑选更符合你的兴趣和工作流程的，不要犹豫，在评论中告诉我其他的信息来源！

## [](#infinite-information-flows-twitter-github-and-devto)无限信息流:Twitter、GitHub、dev.to

Twitter 和 GitHub 都提供了某种时间表，dev.to 也是如此。

*   在 Twitter 上，有一个由(重新)发布的内容(或喜欢的内容)组成的时间线，这要感谢 Twitter 新的随机方面...)据你所知。您可以通过选择要关注的帐户来选择自己的冒险！还有#rstats feed，要么是按流行度排序的最近推文，要么是按发布时间排序的所有推文。我喜欢浏览后者看到有趣的问题，新闻或应用的软件包，R-Ladies 吹嘘等。在我看来，良好体验的关键在于

    *   屏蔽你不喜欢的垃圾邮件和帐户。特别是，保持沉默的大数据 blabla 公司开始污染饲料的习惯，你会更容易看到宝石。
    *   不是一个完美主义者。有人告诉我，看到血流不停是很有压力的。好吧，平静下来，你不会什么都读，你会没事的。
    *   练习！我知道我变得更加深思熟虑，浏览内容的速度也更快了。有趣的是，这可能意味着内容制作者需要明智地选择表情符号、插图等。来引起人们的注意，但这不是今天的话题。
*   在 GitHub 上，我只使用我关注的账户提供的时间线和我观看的报道:创建或分叉的报道，新的发布，人们互相关注(当你看到长时间的朋友互相关注时，有点可爱！)，repos 主演。它可以帮助发现新的代码和新的人。顺便说一下，在你的 GitHub 账户中添加一个头像和最少的信息通常是至关重要的，并且对于你在被某人跟踪后在某人的时间线上获得的几秒钟屏幕时间是有用的:如果你的账户是空的，什么也没有读，否则，人们会看到“简·多伊开始跟踪计算机科学家阿达·洛芙莱斯”，这比“简·多伊开始跟踪 alvlc”更有信息量。

请注意，GitHub 时间轴上没有广告...还没有。

## [](#news-aggregators)新闻聚合器

这是我编的一个类别，我把这些信息来源放在这里，它们没有 Twitter 那么多，因为已经更有组织性和策划性了。

*   R 邮件列表可能是你的事情，无论是特定领域的邮件列表，还是更一般的 R-announce 邮件列表。

*   R Weekly 是一份很棒的时事通讯，按类别组织，便于浏览和选择你感兴趣的内容。第一部分，亮点，是由 R 周刊团队投票的几个链接组成的。你可以通过 RSS 订阅、电子邮件订阅，或者关注[的 Twitter 账户](https://twitter.com/rweekly_org)。R Weekly 的内容来源于 R Weekly 订阅的 RSS 提要和请求。注意，如果你想知道如何让*你的*内容引起人们的注意，你应该看看[我这篇关于博客推广](https://dev.to/2018/07/16/soapbox/)的文章。

*   rOpenSci 双周刊简讯如之前所写，非常棒。它包括关于 rOpenSci 包的新闻，它们在科学文章和博客帖子中的应用，rOpenSci 博客上最近内容的提醒，以及本周以来的相关新闻(这是我如何阅读 dev.to 的)，以及对维护者的呼吁。

*   R 博客作者聚集 R 博客的内容，在他们的网站上重新呈现全部内容。这是发现有趣博客的一种方式。

*   您可以创建自己的新闻聚合器，由 RSS(或 JSON？)你喜欢的博客和时事通讯。我没有这方面的常规，但听说过 Feedly 和 co。

## [](#r-conferences-and-meetups)R 会议和聚会

r 会议(星期六，用户！、RStudio::conf、LatinR、ConectaR 等。)和 meetups(R-女士 meetups，R 用户群)将让你了解新的发展，并结交 R 朋友。如何知道哪些是正在发生的？

*   r 周刊有一个关于即将到来的事件的部分。

*   RStudio 社区论坛上有一个关于事件[的每周帖子。](https://community.rstudio.com/c/irl)

*   Jumping Rivers 维护着一个 R 会议和本地用户组的列表。

现在，*参加*活动显然很棒，但你也可以充分利用网上的资源！

*   会议*节目*可以让你很好地了解新的、流行的和有趣的东西。尤其是你在几个节目里看到的话题。

*   大会网站或志愿者经常提供幻灯片和链接(这是由 Karl Broman 维护的 RStudio::conf 2019 的[链接集)。](https://github.com/kbroman/RStudioConf2019Slides)

*   如果你幸运的话，你甚至可以找到你感兴趣的谈话的录音！见 [Rstudio::conf 2019 视频](https://resources.rstudio.com/rstudio-conf-2019)、[周六开普敦 2018 录音](https://capetown2018.satrdays.org/#programme)、[用户！2018 视频](https://www.youtube.com/channel/UC_R5smHVXRYGhZYDJsnXTwg/videos)。获取信息的廉价方式！

# [](#what-to-do-with-the-information)如何处理这些信息

现在你知道了如何获取大量信息，你该如何利用它呢？

## [](#assess-it)评估一下！

评估你得到的信息，就像你评估任何一条新闻一样，也许可以问问你的 R 朋友们是怎么想的。如果犹豫要不要尝试一个新的套餐，一定要做一个小小的评估来评估它是否值得你花时间和信任。评估一个包不是这篇文章的主题，但是看看这个 rOpenSci unconf 项目中使用的[标准](https://github.com/ropenscilabs/packagemetrics)，[杰夫·莱克关于他如何决定信任一个 R 包的文章](https://simplystatistics.org/2015/11/06/how-i-decide-when-to-trust-an-r-package/)， [rOpenSci 打包指南](https://ropensci.github.io/dev_guide/building.html)。

## [](#store-it-somehow)不知何故储存起来

也许你不需要任何系统，因为你有很好的记忆力，也许你确实需要一个。我不是一个组织教练，所以只会抛出一些链接/想法。弄清楚什么最适合你是非常重要的，这会让你花在阅读新闻上的时间变得值得。

*   Joseph Stachelek 写了如何用他的 Twitter 收藏夹制作一个仪表板。

*   你可以搜索[你的 GitHub 明星回购](https://github.com/maelle?tab=stars)，所以明星回购有点像书签。

*   Suzan Baert 为她自己和任何感兴趣的人维护着一个有趣链接的公共宝库。

*   Lise Vaudor 用法语写了一篇关于 *se 纪念品 de tout*T3 的博文，其中包括一张她办公桌旁的小抄照片。

## [](#share-it)分享一下！

或许不用说，如果你喜欢你阅读或使用的东西，那就把它发布到你的社交网络上，帮助信息流动。:-)

# [](#conclusion)结论

在这篇文章中，受我最近的 R-Ladies 讲座的启发，我总结了我对为什么、在哪里以及如何了解 R 新闻的看法。我在我的讲座中包含了这个主题，因为有人问我“但是你是怎么知道这个的？!"并回答“哦，我只是简单的看了一下”。在几次类似的谈话后，我意识到分享我的个人经历可能会有用。我希望你找到自己的最佳作息，发现令人敬畏的东西...并且在需要的时候记住它！
