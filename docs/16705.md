# 团队项目成功的 3 个关键

> 原文:[https://dev . to/kjwong 3/3-keys-to-successful-team-projects-2jo 5](https://dev.to/kjwong3/3-keys-to-successful-team-projects-2jo5)

新年是反思过去的成功、失败以及如何在未来做得更好的大好时机。这些年来，我参与了一些团队项目。作为开发 [Kachi](https://kachi.app) 的一部分，我一直在思考哪些事情进展顺利，哪些事情我们下次可以做得更好。毕竟，项目是软件制造和改进的方式。以下是我想到的一些关键因素。

## [](#1-a-clear-goal)1。明确的目标

[![Improving code quality is good, but we still need to ship!](../Images/8f02d683bcb22cef5b0d41315788da08.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Lx0iYlfx--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/zidopvjkyv1g8zdvqtvx.jpg)

这看起来是显而易见的，但是对你要完成的目标有一个清晰的想法是成功项目的第一步。无论是对客户的特性还是对代码库的重构，确保有清晰的、可测量的、书面的目标是至关重要的，即使对于较小的项目也是如此。

这将有助于:

### [](#get-buy-in)获得买入

参与一个项目是一个很大的承诺。人们有其他的方式来打发时间。因此，让团队成员感觉到项目是重要的并且值得努力是很重要的。没有人愿意花几个小时在他们觉得是浪费时间的事情上！有一个明确的目标可以让每个人看到最终的好处是什么。

有一次，我试图说服另一个部门的副总裁，我们需要改变我们软件的配置方式。虽然他同意在他的团队中分配一些工程师，但他们最终从未有时间进行工作。如果我更清楚地传达了对他的团队有什么好处，就会更容易获得真正的认同！

### [](#make-decisions)做决定

当目标明确时，在两种不同的结果之间做出选择会容易得多。例如，我们正在评估不同的日志服务供应商。一个是市场上的新玩家，他们没有足够的安全控制。因为我们需要维护我们自己的安全合规性，所以决定使用哪个供应商变得容易多了。

### [](#clarify-expectations)阐明期望

一个长项目快结束的时候，告诉你的客户他们终于可以做 X 了，结果他们脸上露出困惑的表情，告诉你他们真的想要 y，这是一种很糟糕的感觉。以书面形式明确你的目标，并得到客户的批准，这将使所有人都保持一致。

## [](#2-effective-communication)2。有效沟通

作为一名工程师，与和其他人交谈相比，坚持写代码更容易。如果你不编码，这个项目就永远不会完成，对吗？但是有时候通过交流可以节省很多时间和精力。

以下是联系其他团队成员的一些好时机:

1.  受阻。也许你没有你需要的一切，或者需要帮助理解一些事情。无论如何，让团队知道可以帮助你找到解决方案。
2.  上班会迟到。有时候很难承认自己落后了。但是如果你不告诉任何人，不仅你会迟到，而且其他人也无法适应，也许会引起更多的问题。最好的办法是提前了解情况。
3.  工作完成了。把票标成关闭的感觉很好。但是取决于你的团队如何工作，这可能还不够。除非需要它的人知道它，否则工作不是真正完整的！
4.  需求变化。有时一个重要的特性变得不必要了。有时需要新的 API。每当需求改变时，每个人都知道新的期望是至关重要的。

## [](#3-respectful-team-environment)3。相互尊重的团队环境

[![rude.](../Images/0e0d8c3331f260a97dd870f1294ebec8.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--135l2TKe--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ltonwxlcnbk52zzuj8a3.jpg)

如果人是机器人，尊重就没多大关系了。但现实是，如果人们感觉不到被重视和尊重，他们的表现就不会很好。这不是向房间里最聪明的人低头，而是意识到每个人都在贡献自己的时间和专长。每个人都应该放心大胆地提出问题和想法。这是从每个人身上获得最大收益，并为项目获得最佳结果的唯一方法。

我曾与技术人员和非技术人员、有经验的老兵和新毕业生、受过或没有受过正规训练的人、不同种族、性别和性取向的人一起在团队中工作，让每个人都感到舒适和安全的参与总是会带来最好的结果。

### [](#conclusion)结论

这是我成功团队项目的三个关键。专业软件开发是通过项目进行的，所以让项目成功是构建优秀软件的关键。你发现了什么使你的项目成功？
