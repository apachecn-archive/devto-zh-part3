# 筹码中的杰克

> 原文:[https://dev.to/molly/jack-of-the-stack-5d4d](https://dev.to/molly/jack-of-the-stack-5d4d)

在最近的 Web 应用程序[# dev discuse](https://twitter.com/search?src=typd&q=%23DevDiscuss)中，出现了对全栈开发人员的讨论。从那以后，讨论深入到描述开发人员及其关注点的最佳方式。

液体错误:内部

许多人认为他们了解整个堆栈，但并不精通所有这些。这导致了短语“栈顶”的产生。最终，话题转向描述其他开发人员类型。工程师的普遍概念是最先被抛弃的概念之一。

[![](../Images/8b42aceafbf3ce51718e7d86a05a262c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--TtesaTbK--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/9fuqvr3ruqiy04t3576g.jpg)

从那以后，我们四个人试图用字母表来描述不同的开发人员，谈话就开始走下坡路了。这是我和麦克斯、、、劳里、、詹姆斯想到的！

### [](#a-academic)A -学术

坚持 CS 纯度高于一切的开发者。

### [](#b-bootstraper)B - Bootstrap'er

坚持所有网站都应该在前端使用 Bootstrap 的开发者。

[![](../Images/caf142aaa65fe0b997cd4aef73b4da7f.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s---rylMFMA--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/m9a564ln574c3lrkc1j3.jpg)

### [](#c-c-dev)C - C 开发

不想学习其他任何东西的开发人员，因为 C 是他们学习的起点。

### [](#d-devto)D - Dev.to

经常在 DEV 上发帖的开发者。

### [](#e-everything)E -一切

拥有大量领域的一般知识，但也有一些领域他们有更深入的知识的开发人员。这是一个更现实的全栈开发者定义。

### [](#f-f-it)F - F*它

内联编写 CSS 的开发人员，因为 F 一切！

[![](../Images/1f5b8a5ff9c93f545474bc2ed00aa679.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--F5ZzU9hU--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/uwf5phx0ob7px92udz73.jpg)

### G-gif

开发商总是有一个 GIF 与他们的公关！
[![](../Images/beaae4df735785a3e6aeed3999117830.png)T3】](https://res.cloudinary.com/practicaldev/image/fetch/s--Vj3fEnrl--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/aknlnnpbn7lg3o2pgs0f.gif)

### [](#h-hater)仇恨者

只谈论他们有多讨厌其他编程语言的开发人员。

### 我可以做到这一切！

想亲力亲为的开发者忍者。神话中的 10 岁以下的人可能会带来比他们解决的问题更多的问题。

### [](#j-javascripter)J - Javascript'er

认为从数据库到前端的一切都应该用 JavaScript 的开发人员。即使这意味着取代一切！

[![](../Images/9ddf0439d42961f22f0b838e3980e9bc.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--BUr5t541--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/jhtnf9piyyqczp9a9934.jpg)

### [](#k-kick-the-can)K -踢罐子

开发人员总是坚持拖延时间，在以后的公关中解决问题。

### [](#l-learner)L - Learner

正确地知道所有事情的开发人员是关于学习的，并且不断地尝试学习他们所能学习的所有事情。

### [](#m-mac)M - Mac

开发者除了 Mac 之外什么都不会做，他们不明白为什么有人会用 Windows。不要让他们从 Linux 开始。

[![](../Images/d2954b9e3cc29b203274e097a56db19e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s---UdJe43e--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/tweq3ksa8keyixt03hwk.jpg)

### [](#n-narrow-minded)N -小心眼

专注于某个特定堆栈的开发人员到了不计后果的地步。他们不想与任何新事物有任何关系。

### [](#o-on)O - O(n)

痴迷于一切 O(n)的开发者。

### [](#p-perfection)P -完美！

开发人员编写最干净的代码，包括描述性的名称、注释和精彩的自述文件。

### [](#q-quirky)Q -古怪

开发者大部分时间都围绕着同样的技术转，只有一个令人不安的偏差。

### [](#r-recursion-recursion-recursion)R -递归。递归。递归...

开发人员想出了所有事情的递归解决方案。

[![](../Images/fc8958e9b06bd0e6e118fa71b00137ad.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--cBC-Hpd_--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ecan474n599tpoy0h0rn.jpg)

### [](#s-sassy)非常时髦

相信一切都需要 mixin 的开发者！

### [](#t-topic-expert)T -话题专家

拥有大量一般知识，但同时也是某个特定主题的专家的开发人员。

### [](#u-unicorn)优麒麟

主全栈开发者，可能并不存在。

[![](../Images/5834ee46cfb3d2677b4a961623312624.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--z6iy5C6K--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/5tfoi91azklag5qf3jqg.jpg)

### [](#v-view-challenged)V - View 受到挑战

开发人员决定将所有代码内联到视图中，这样 HTML 就变成了一团乱麻，没人能读懂。

### [](#w-what-next)W -接下来呢？！

不断寻找下一个闪亮的新技术的开发人员。

[![](../Images/3ce8fb0f195dc3e87e415d560902e02c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--uJyMa6VB--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/9zw0ygju1ryjyblueaqq.gif)

### [](#x-x-marks-the-spot)X - X 标记斑点

开发人员对每一个 PR 的错误语义吹毛求疵，认为在合并之前必须准确无误。

### [](#y-y-isnt-this-accessible)Y - Y 这不是无障碍吗？

开发者关注可访问性。它们如此重要，应该在字母表中排在前面，对每个单词都如此重要，你几乎会认为它应该是一个元音。

[![](../Images/05dfcd2e909a08363092d65460846a3b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s---bOf12RN--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/vfj9lwbrp7gw2j6452g3.jpg)

### z-zzzz😴

开发人员痴迷于一种没人知道的晦涩难懂的语言，但他们希望每个人都学习。这足以让你入睡。

### ；

开发人员不能把他们的注意力从编程上移开，在一天结束时，他们会忘记列表，比如句子，不是以分号结尾的；

那么，你能和哪个开发商联系起来呢？！如果你能和他们中的许多人联系起来，你可能只是一个“堆栈中的杰克！”

非常感谢我的合作者们，他们帮助我想出了所有这些不同的 dev！强烈推荐给他们每个人一个关注！🤗
