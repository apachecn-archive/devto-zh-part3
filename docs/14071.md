# 美德总有邻居

> 原文:[https://dev . to/sandordargo/virtue-always-have-neighbors-59mc](https://dev.to/sandordargo/virtue-always-has-neighbours-59mc)

孔子写道“美德从不孤独；它总是有邻居。”他的意思是良好的行为和良好的思维是会传染的。

这是真的吗？总的来说，我没有答案，但是根据我的经验，它的工作方式差不多就是这样。

当我 5 年前加入我现在工作的公司时，我对软件开发几乎一无所知。我不断收到大量关于我的拉取请求的评论，通常都是相同的。例如，我发现很难注意不通过值传递变量，而是使用一个&符号甚至 const 关键字。写`const std::string& param`而不是简单的写`std::string param`多恐怖啊！

三件事起了作用。当然是时间。问为什么。当我明白为什么要把这些额外的字符放在那里时，事情就变得简单了。此外，我看到我的同事们都在言行一致。他们的良好行为具有感染力。

然后我在我的团队中承担了一些自动化任务，我开始学习和使用 Python。我很快了解到团队中有一个人拥有相当扎实的 Python 技能。Samuel 是一名 C++开发人员，但他最喜欢 Python。一开始我很痛苦，因为我写的东西对他来说都不够好。坦率地说，事后看来，我写的东西在 Pythonic 的角度来说是垃圾，但它完成了工作。这对我来说只是下一步——编写 Pythonic 代码。我非常感谢他在我的代码中指出了所有这些细节。

让 Samuel 审查我的代码，并让我有机会审查他写的东西，帮助我很快跃上了一个新的高度。我可以向 Sam 感谢我 Python 技能的基础。他的 Pythonic 式思维和良好的编码行为具有感染力。

在那个团队里，我真的开始重视好的思维。我也明白了拥有一个没有编码经验的人并不意味着世界末日。一个人可以通过——不太多的时间——变得更好，尤其是如果基础已经存在的话。应该是你在新兵训练营或你的计算机科学学位之后做的项目。

在那个团队中，拉取请求的质量、代码评审中发起的讨论的深度、设计会议帮助我培养了一种职业自豪感，我把这种自豪感带到了后来的团队中。一开始，我在新岗位上工作时，并没有改变人们思维和工作方式的明确意图。我只是以我认为正确的方式做了我的工作——当然，如果你不停滞不前，而是不断学习，情况会不断变化。过了一段时间，我发现我的做事方式有助于志同道合的人在日常工作中互相支持。

就在那时，我意识到一加一远远大于二。如果你不是一个人在谈论某件事，更多的人会相信它，并实际遵循你的建议。我们可以创造一种学习的文化，并吸引一些没有正确输入的有才华的程序员，我们互相帮助变得更好。怎么会？通过在编码道场的刻意练习，通过观看干净的编码视频，通过为彼此做代码评审，当然还有许多其他方法。没有什么比看到有人因为你帮助成长的社区的提升力量而变得更好了。

我看到良好的行为在其他项目中也具有传染性。有一次，我只加入了几个月，就把 C++ API 移植到 Python。为了理解我必须做什么，我实际上必须研究用 Java 编写的服务器部分。这显然不是我见过的最糟糕的代码，但它也表明编写它的人没有把代码质量和可测试性作为他们工作的一个重要方面。也许是因为时间压力，也许不是。不用说，他们早就去做别的项目了。

我知道让开发人员长期完善代码而从不交付是不受欢迎的。它们对业务不利，因为它们从不交付。我不是其中之一。另一方面，我也看到人们是如何打发时间的。这里一支烟，那里一杯咖啡，一条推特，一份脸书邮报，午餐时间差不多到了。众所周知，我们无法在中午到三点之间高效地工作，如果是午餐时间的话，在三点到五点半之间，当我们不喝咖啡的时候，我们可能会写几行代码。

很明显，我夸张了。很多。但这里面很难说没有道理。我从来没有抽过烟，但是我曾经因为其他一些刚刚提到的浪费时间的行为而感到内疚，尤其是那些漫无目的的，冗长的咖啡会议。但是我下定决心，我不再浪费时间。最可能的原因是，作为父母，我拥有的时间少了，我不得不更加珍惜我的时间。

回到我良好行为的主题，我决定花 25 分钟( [1 pomodoro](http://sandordargo.com/blog/2018/02/28/setting-yourself-up-to-succeed) )来解决我在前面几段提到的服务器部分的技术债务。有些人会尖叫说这个项目没有要求它。没错。他们让我们在你最喜欢的社交媒体平台上漫无目的地滚动吗？他们要求我们这么做了吗...不，我没有继续，如果你想的话你已经明白了。

在一个半月的时间里，我几乎每天都做一个小的拉请求，实际上清理了大约 80%的技术债务——至少根据 SonarQube 的说法。在项目结束时，我发现其他人也开始清理一些存储库。清理技术债务具有传染性。不久他得到了提升。不是因为我，是他做了这项工作，但看到这一点很好...也许我对他有好的影响。

当我离开那个项目/团队时，我正在寻找下一个项目，我的想法是找到一个还不太好的地方，但我可以帮助改善环境。我不只是想加入一个看起来已经很棒的团队。我比一个我已经认识并非常尊敬的人晚了大约六个月来到我的新团队。我们带着同样的意愿来到这里。已经赢了一半了。我发现独自改变事情非常困难，但正如我所说，两个人有更好的机会。

一年过去了，我还是觉得有动力。事实上，我比以往任何时候都更有动力。我能看到我们工作方式的许多好的变化。我们编写更多的测试，我们考虑可维护性，我们在代码评审中问更多的问题，伙伴评审更少，如果需要，我们为自己和团队挺身而出。这也意味着我们有时不得不进行一些不太愉快的谈话，但这不是问题。正如人们所说，一个人的讨论越困难，他或她就会越成功。

我分享了我的例子，因为我在自己的职业生涯中体验到良好的行为是如何传染的，以及孔子是多么正确。我从两方面都经历过。我既看到了也表现出了良好的行为。即使在今天，这也是我职业生涯中最激励我的事情之一。我身边的人树立了一个好榜样。我很感激他们，我会通过表现良好来回报他们的服务。

在我看来，那是你能做的最好的。跟随并展示一个好的例子。抬起你的头，记住:美德总有邻居。
