# 促成开源

有一部分让我感到惊讶，大部分人发现对开源做出贡献是一个挑战。

该文章指出:

阻碍开发人员参与开源的两个最突出的障碍是不知道从哪里开始，以及怀疑他们是否有合适的技能。

在我看来，像 Dev.to 这样的项目在 Github 上有很好的文档记录。尽管我可能对 Ruby 不是很了解，但我确信任何人都能理解，因为最终最重要的是基本的编程原则。

话虽如此，仍有许多项目应该遵循相同的标准。这是我不得不承认的地方，需要做更多的事情来鼓励开发者做出贡献。

你对此有什么看法？你同意这些障碍存在吗？有哪些可以改进的地方？请在下面的评论中告诉我。

## 原因三

我一点也不惊讶的第三个原因是:

公司不给员工贡献时间也是一个重要因素。

也许是时候让更多的公司每周分配一天时间，让开发人员可以花 X 的时间在开源项目上，或者只是学习一些新的东西。

这是我们在 Root Codex 做的事情，我们鼓励我们的开发人员每周贡献时间来做一些宠物项目。

当涉及到贡献您的代码时，您是否遇到过或目前正面临这样的障碍？

你为开源做贡献吗？如果是，是哪个项目？

作为一个社区，我们能做些什么来帮助其他人克服这些障碍吗？
