# 2019 发展解决方案

## 最近我觉得自己很有目标

自从去年找到一份新工作后，我感到特别有活力。和我一起工作的人是比我好得多的开发人员，这不是一件坏事。他们教了我很多东西，我觉得我自学的教育缺少了一些让我变得更全面的技能。

我想出了一些我想在 2019 年学习的东西，这样我可以在我们的应用开发中了解更多，并且我可以在更好地规划应用和组件方面带来更多。

### [](#better-foundation-in-react)更好的基础在反应

10 月份刚学会 React。我从没想过我能学会它，实际上我的新工作教会了我。我花了大约两周的时间看完这些视频，然后花了一个月的时间对自己的技能感到自信。所以，现在在一月份，我正在努力重温我可能忽略的初学者原则。然后，我想让自己适应更高级的功能。这可能是一种奇怪的方式，但生活会很快来到你身边。

### [](#learn-redux)学习 Redux

我对 Redux 知之甚少，我的同事负责创建动作和其他 Redux 功能。我希望能够帮助和制作我的组件，为连接到 Redux 和获取数据做准备。在生活的总体规划中，至少从基本上来说，这似乎是件好事。我并不期望成为一名专家，但能够为个人项目连接一个系统对我来说是一种胜利。

### [](#learn-about-apis-and-nodejs)了解 API 和 NodeJS

我已经使用 NodeJS 和 Express 制作了基本的 API，但是我还想学习更多。这不是列表中最重要的事情，但我真的喜欢学习发展的各个方面。我对后端的工作理解得越好，我对与前端同步的感觉就越好。

### [](#understand-technical-side-of-authentication)了解认证的技术层面

这听起来很傻，但我从未真正考虑过身份验证。现在我的眼界开阔了，我决心学习更多关于 JWT (jsonWebTokens)、OAuth 和任何其他认证系统或版本的知识。

### [](#make-better-tests-jest-amp-enzyme)做更好的测试——Jest&酵素

这对我来说是最重要的。我的测试只是检查组件是否渲染并且没有崩溃。我希望在这一领域取得进展，因为拥有强大测试的组件来捕捉我所犯的任何可能影响我构建的项目的错误将是非常棒的。

### [](#learn-graphql)学习图解法

这似乎是一项即将到来的技术，即将爆发，所以我想保持一些最新的。我不确定我现在会把它用在哪里，但是如果人们会对它感兴趣，那么我就必须关注它。
