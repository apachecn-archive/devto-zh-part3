# 我是如何开始学习一项新技术的

> 原文:[https://dev . to/quantum sheep/how-I-started-learning-a-new-technology-10aa](https://dev.to/quantumsheep/how-i-started-learning-a-new-technology-10aa)

有很多技术，有各种各样的需求:编程语言、软件、操作系统等等...但是如果你想学习一项新技术呢？因为什么原因？在这篇文章中，我们将看到我是如何学习一项新技术的，从而给你一个如何开始的想法。

# [](#why)为什么

这是一个事实:有些技术更适合做其他技术做不到的事情，反之亦然。虽然有些技术可能是相同的，但也有可能导致向另一种技术转变的差异。

例如，编程语言是一个语法、性能和社区的问题，而软件是因为它们的有用性或设计而被使用的；操作系统是用来做他们能做的事情的(Windows 有活动目录，Linux 有完全的定制，等等)...).

工作机会也是学习技术的一个重要原因。

[![](../Images/487210d926dcb2e5d1b2599e6a12157a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--1Q0Rcth9--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/fm724mg388kgz1qmry8x.gif)

但是，让我学会一门技术的，大多是炒作。我知道大多数人认为跟风不是一件好事，但是谁知道呢？Go 只是一种趋势，并成为最常用的编程语言之一，同样的事情也适用于区块链。这只是一个想学习这项技术的问题，因为它很受欢迎，如果它受欢迎，它肯定是一个好技术。

# [](#how)如何

技术文档也将成为你的生活、教程和论坛。学习一件事情并没有真正的好方法，这取决于你的决心和你如何学得更好。

书籍是大多数人的首选。就个人而言，做一些有趣的小项目是我学习技术的方式，但它让我远离理论。

# [](#what-did-i-learned)我学到了什么？

我选择学习的技术是[水晶编程语言](https://crystal-lang.org/)。为什么？因为这是一种趋势，而且有 Ruby 的语法。另外，我最近开始学习如何使用 Arch Linux，完全是为了好玩。

事实是，我不想停留在一种技术上，从长远来看这很无聊。因为我的好奇心，我每天都想发现一些东西。

# [](#conclusion)结论

也许在这篇文章之后，你会开始学习一种新技术，出于任何好的理由。或者也许你会创造一些其他人会学习的东西，因为这是一个奇妙的想法。

发现新事物并让世界进化是让我们感动的事情，需要像那样继续下去。

而你:你是如何以及为什么开始学习某样东西的？
