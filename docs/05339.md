# 从程序员的角度看 UI 设计

> 原文:[https://dev . to/areknawo/ui-design-from-a-a-programmer-s-perspective-cle](https://dev.to/areknawo/ui-design-from-a-programmer-s-perspective-cle)

**这篇文章摘自[我的博客](https://areknawo.com)，所以请务必查看更多最新内容😉**

我一生中做过一些设计，你知道吗？😉先是一些 **3D 美术**，建模，还有雕塑，然后是**矢量图形**，标志设计等等。我一直知道什么在视觉上对我有吸引力，并一直试图达到与该品牌其他专业人士相同的水平。我知道我有自己的**品味**(你可以从这篇博客的设计中窥见一斑)，但我没有掌握正确的技能。我一直希望我的设计更好。我的私人要求从未得到满足。我就是**没定过**。然后，**编程**进来了...

学习如何编码是一个令人愉快的过程。我以前在各种设计和其他软件方面的经验绝对有帮助。我学得很快，最终感觉自己找到了自己的位置。知道如何编码打开了一个广阔的世界**的可能性**(即使它只是一些人可能会说的 JS🤨)最终创造出我一直想要的东西。我没有被提供的软件和工具所限制。我能够从零开始创造任何东西。**编程是一门艺术**，正如一些人可能会说的，它是一种**的精神状态**。然而，作为一个自由职业的网络开发者，一个想自己做所有东西的人，我被认为必须再次设计一些东西...

# [](#the-visual-part)视觉部分

至于 web 开发的可视化部分——**HTML**&**CSS**，我必须承认——我真的没有给他们太多关注。我专注于学习 JS，把 HTML 和 CSS 放在后面一点。当然，我最终也研究了它们，但是，我仍然不太习惯使用它们。我很容易操作他们的功能和语法，但没有达到我想要的水平。我敢打赌，作为任何语言的程序员，在完成你的**杰作**代码时，你肯定有过这种***【wow】*****效果** ✨。别人是否同意你的感受并不重要，最重要的是你创造的东西让你快乐。因为这就是人们最初创造任何东西的原因，不是吗？而且，虽然我对 JS 至少有过几次同样的感觉(有些立刻变暗了🤨)，HTML 和 CSS 我就不能这么说了。总是有一些不理想的东西——错误的排列、位置、颜色、尺寸等等。我就是做不出让我真正满意的东西...

让我们加速到最近的时间。现在，当我在创建我的下一个大型 OSS 项目时。而且，你猜怎么着——这需要我设计一些 **UI 组件**！我从来没有安定过，不是吗？ **UI 设计**——我知道的一些事情可能会让我陷入对**越来越好**的不断需求的永无止境的循环中。这样的问题有解决方法吗？肯定有一个——**随它去**。😅但是，让我们更有创造性(*讽刺*)，采用更复杂的解决方案- **让别人给你带路**。

# [](#design-system)设计系统

[![](../Images/e44f470cece41a41a1e39089d2999582.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--b4cwMnpL--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://areknawo.com/content/images/2019/04/mio-design_assets_1z0dsWbLqJkzTGVbmaJViiaNnh4mK-aoP_design-theming-2x1-large.png)

在 UI 设计中，我们要找的东西叫做 [**设计系统**](https://medium.muz.li/what-is-a-design-system-1e43d19e7696) ，或者更简单的名字——**风格指南**。顾名思义，它的基本形式是一套规则、规范和其他东西，简单地告诉你该做什么。它们作为其他人实现和使用它们的参考点。如果你曾经使用最新的框架和库进行过一些 web 开发，或者只是喜欢 Android，那么你很有可能听说过 Google 的[**Material Design**](https://material.io/)(MD)——它最近非常火！是的，它是众多设计系统中的一个。它随着 Android Lollipop 的推出而广受欢迎，并以其时尚悦目的视觉效果扩展到其他移动设备和谷歌产品，最终来到 concur web。现在，几乎所有针对不同 UI 库和框架的顶级组件库，都是基于 Material Design 的，或者至少在或大或小的程度上受到了它的启发。

但是，我们不要深究下去，回到正题。我需要创建一套属于我自己的 UI 组件。我认为这只是一个逻辑思维的结果🤔在过去，当我制作一些不错的网站时，我经常使用它和不同的库([**Vue titify**](https://vuetifyjs.com/)用于 Vue，以及 [**Material-UI**](https://material-ui.com/) 用于 React)并建立了一些相当好看的网站。问题是，他们并不是真的，你知道，很棒。也许只有一些。其余的都是基于我的标准**模板**-工具栏、抽屉、登陆页面等等。当然，它们看起来很棒，但它们并没有真正让我满意。

# [](#abstraction)抽象

[![](../Images/baa5ccf76ec546e9e0b6a678a0834450.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--luBB7YvB--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://images.unsplash.com/photo-1524840127331-f2ca773ec28f%3Fixlib%3Drb-1.2.1%26q%3D80%26fm%3Djpg%26crop%3Dentropy%26cs%3Dtinysrgb%26w%3D1080%26fit%3Dmax%26ixid%3DeyJhcHBfaWQiOjExNzczfQ)

所以，我选择了 MD，去谷歌官方文档寻找这种神奇，然后我开始阅读...或者说在上面跳来跳去。我在看一些**组件**，通用**指南**，以及其他不同的东西，没有太多的顺序。从我的第一印象来看，我确定这些文档真的很完整，写得很好，并有不同组件的数字规格-对我来说正是正确的事情。然后我开始读它...像真的一样。和...我猜你知道接下来发生了什么。我在读啊读。关于我应该/不应该做的事情，这个**抽象环境**如何运作，阴影，度量，以及——最后——组件。抽象概念、信息和图表的数量让我崩溃。我意识到 MD 不仅仅是我想要的严格的数字规格。所以，在几次尝试和失误之后，我继续前进，寻找更好的解决方案。

在继续这个故事之前，让我们分析一下上面的事实对我这个程序员和**想成为设计师的人**意味着什么。首先，我想要**超清晰的规格**，它就像一个孩子一样引导我穿过一条名为*设计*的拥挤街道。我只是希望一切都像编程世界一样简单明了。不像我说的程序员局限于纯逻辑思维，像很多业外的人喜欢思考。其实完全相反！他们非常有创造力，他们创造的艺术——代码——本身就是一件杰作。程序员不断努力用更新的架构、工具、原则和更多的东西推动他们的创作。⚡——就像 UI 设计部门的**视觉效果**，不断变得越来越好(时不时会有一些缺点)。只是这种代码艺术与大多数艺术作品有些不同。它的主要目标是清晰、易读，并尽可能以最好、最简洁的方式完成。写得好的代码读起来应该像一本好的科学书籍。我想在 UI 设计中采用同样的原则，这是一个很大的错误。

UI 设计不同于代码。它的创建是为了给最终用户留下深刻印象，而不是其他程序员。这意味着**视觉愉悦**🤗出色、直观，并尽可能以最佳方式表达自己的品牌。这就是为什么我真的很钦佩那些创造出这些 [**不可思议的设计**](https://dribbble.com/) (包括程序员)的人，因为他们真的很擅长他们正在做的事情。而且，当你意识到这并不是一件容易的事情时，你会更加印象深刻。你需要有**品味**、**知识**来使用正确的工具、需要**表现力**来将你的想法转化为现实事物，等等。可能我有点过饱和了，(或者只是想让自己感觉好一点😅)但是编程也是一样的。当然，几乎每个人都可以学习如何编码——只需要一些逻辑思维。但是，当你进入更复杂的层次时，事情就开始变得复杂了。在这里，你需要理解编程的真正潜力，以及它是如何工作的。就像设计一样——编程也是一项抽象的活动！一层又一层，从机器码和汇编程序到高级语言和软件架构中使用的不同概念的过渡——这就是编程的本质。编程是一种抽象。

所以，我想要明确的规格，没有找到他们与 MD，所以我继续前进！我真的梦想有一个设计系统，它实际上只有清晰、**详细、**、**数字规格**的开源文档，随时可以被任何愿意这样做的人实现(在这种情况下就是我)。是啊，白日梦...首先，几乎所有的设计系统都有自己的实现，通常在 React 中。这当然没什么不好！我完全同意设计系统需要有自己的官方实现，所以我可以接受(即使这意味着重新发明轮子😕).那时，一些风格指南是开放的，但同时也是封闭的。他们提供了在真实应用中使用的代码样本，但没有提供组件本身，这实际上让我怀疑我是否被法律允许复制他们的设计。由于我不想深究太多，我没有真正考虑这些选项中的任何一个。最后，我发现的所有系统都有自己不同的规则和**抽象准则**。正如我所说的，我还没有完全准备好理解所有这些概念，所以这样的系统对我来说也不是正确的选择。你可能已经猜到了，这让我别无选择。

# [](#weird-idea)怪异的想法

我浪费了大量的时间和精力去搜索那些风格指南。但是，我有吗？一路走来，我找到了一些[](https://designsystemsrepo.com/design-systems/)**[**不同的设计系统**](https://adele.uxpin.com/) 、资源和一些 [**真正鼓舞人心的东西**](https://www.uplabs.com/) 。事实上，我看到过如此赏心悦目的设计，以至于我内心的设计师被夸大了。甚至到了我...决定自己制造部件。**

 **现在，这样的决定，毕竟，我以前经历过似乎很漂亮...最坏的情况是糟糕和混乱。但是，现在的我和那时的我有所不同，你可能在整个故事中没有注意到。在这个时间点上，我对我想要创建的东西有了一个真实的图片。🖼:我把我的设计过程**比作编程**。我清楚地列出了我想做的事情。我想创造一些程序员和设计师都容易使用和**访问的东西。我希望这是非常简单的事情，不需要很好的设计技巧或者品味。并且，希望我能从我身上挤出这些设计师来创造这样一个东西。😅**

但是，除了上面提到的愿望清单，我还有一些新的经历。你没想到我会白手起家，对吧？我有很棒的资源来做我的设计。伟大的材料设计组件**尺寸规格**、**色彩创意**来自其他伟大人物的不同创作，以及**交互功能**来自真正给我留下深刻印象的设计系统的其余部分。有灵感真好。它很好，因为它让你创造出更好的东西！🙌

# [](#just-a-thought)略一思索

这些只是我的一些想法——一个逻辑思考者和一个程序员，他有一些愿望也想成为一个伟大的设计师。我个人认为，不断想成为更好的自己是一件好事。编程和设计并不像你想象的那么遥远。你只需要...**稍微改变一下你的观点**。我认为一点点灵感可能会帮助任何人发展他们自己的设计品味，结果，至少在设计方面变得更好一点。也许我说的不是一些超高水平的杰作之类的，但至少在设计方面有点经验肯定会帮助你成为一名更好的 web 开发人员...反之亦然。还有，设计就是好玩！
