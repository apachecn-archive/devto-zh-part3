# 让你的工作更有趣的 5 种方法

> 本文原载于 2014 年我的个人博客。这些建议经受住了时间的考验，成为我工作决策的基础，对我和我的客户都有好处。这就是为什么我决定在 mkdev 上重新发布它，这样它就可以接触到更多的读者。

这一天刚刚开始，但我已经到了离我办公室最近的地铁站。我溜达到面包店，买了一个羊角面包(巧克力的，如果你想知道这个的话。)然后我走向我的办公室，一只手拿着一个羊角面包，由一位年轻的土耳其女士精心包装，另一只手拿着 mp3 播放器。

在办公室，我以同样缓慢的方式泡了些茶，然后在办公桌前坐下。我平静地喝着茶，嚼着羊角面包，但在那后面隐藏着我体内不断增长的能量流。我迫不及待地开始工作。一些刺激而危险的冒险在我前面。但也有一个简短的早间简报，在那里我不得不谈论昨天所有的娱乐任务。我的工作日甚至还没有开始，但我已经准备好行动了，即使没有任何咖啡或阅读新闻。

过了 10-11 个小时左右，我环顾四周。办公室里有门卫。外面天渐渐黑了。除了我之外，办公室里只有项目经理。我的肚子在咕咕叫，因为我饿了，再也无法思考。又一天我热爱的工作结束了，该回家了。

但三四个月前情况完全相反。我过去常常在早上的简报会上迟到，因为我在那里无话可说。在我必须完成工作的前一个小时，我会停止任何活动，开始无情地拖延。是的，我甚至不害怕感到羞耻。又一天充满了无聊的任务和修复 bug，这种越来越强烈的烦恼感让我更想回家。离开，离开这间办公室。我曾经毫无激情地投入其中，期待着一天的结束。

### [](#the-project-50-for-developers)项目 50 为开发商

与此同时，我在读汤姆·彼得斯的《50 号工程》,结果发现它真的很鼓舞人心。彼得斯的风格非常简洁和感性，他所有的书都是待办事项清单和行动呼吁。核心信息是，你不应该浪费时间，而是要努力把你做的每件事都变成一次有趣的冒险。他的另一个伟大的想法是没有坏项目。你可以随时接受大家已经放弃的任务，并把它们变成令人惊叹的项目。如果你在做一些无聊的工作，没人会关心你到底在做什么。这意味着你有机会做一些意想不到的事情。

读完这本书后，我得出结论，除了我，没有人能让我的工作变得更令人兴奋。是的，目前团队中的气氛相当令人沮丧，所有的任务都显得单调而愚蠢。但这是我不享受生活中每一个小时的理由吗？嗯，我想没有。每天八小时是我生活中很大的一部分，所以我最好把我的抱怨变成娱乐。那么，我们怎样才能让你的日常生活变得有趣和快乐呢？

### [](#1-transform-your-bug-fixing-routine-into-tasks)1。将你的 bug 修复程序转化为任务

当你被告知要修复某个 bug 时，我们都经历过。挺小挺烦的。您可以在一个小时或更短的时间内修复它，然后您必须等待您的同事进行代码审查。可能要花很多时间。可能是正在进行部署，或者他们正在测试一些功能，因此无法部署。因此，一个小错误会导致几天的等待，我们都会完全失去注意力。Bug？哪个 bug？啊，这个？嗯，是的，我修好了...我想是的。

你可能一天要面对五次这样的 bug，甚至更多。对此你该怎么办？将 bug 转化为成熟的任务。例如，您可以重构您发现错误的那部分代码。甚至完全重写这个讨厌的控制器。也有可能根本不需要有缺陷的特性，最好在与管理团队进行简短讨论后将其删除。

### [](#2-turn-your-boring-tasks-into-mini-projects)2。把你无聊的任务变成迷你项目

所有的袋子都修好了，但是剩下的任务一点都不令人兴奋？你想做一些大的事情，而不是把数据导出到 CVS？你知道，没得选择。未来会有大事发生，但现在你需要专注于当前的任务。但是你知道吗？你可以把你无聊的，或者小的任务变成全面的迷你项目，这可能会花费更多的时间和精力，但是会给你带来很多乐趣。

一些任务可能与其他即将开始的大项目有关。如果系统的某个部分已经自动化，有些可能根本就不存在。你也可以重新制定任务，使初始条件相同，但实际上你在做一些更全局的事情。我曾经把更新 gems 的次要版本变成了更新 Ruby on Rails 4:)

仍然不知道如何做到这一切？我也不知道。那些无聊的任务是你的，我对它们一无所知。但是我们可以在下面的评论中讨论它们，如果你愿意，也许我们可以一起找到一个方法。

### [](#3-convince)3。说服

当然，如果你愿意，你可以在你的技术领导背后独立地改变任务。但你应该做好准备，最终有人可能不喜欢这样。因此，你最好能找到一些支持你观点的盟友和论据，并给出时间框架。有了这些，你可以去要求管理团队给你更多的时间来完成任务，让你把它做得更大。重要的是，如果你被拒绝了，那并不意味着你至少不应该尝试。为你所做的事情道歉比请求许可更好。

### [](#4-learn)4。学习

你觉得自己好像被逼到了绝路，因为没有办法让这个特殊的任务变得更大？那就把它当成学习的机会吧。您可以了解一些新技术的要点，或者深入研究代码。把你的工作变成学习，即使是平淡无奇的事情也会变得更有趣。只要你发现新的东西，一切都好。

### [](#5-quit-your-job)5。辞去你的工作

如果你没有做到上述任何一点，仍然不喜欢你的工作，你最好考虑辞职。不开心的程序员不仅对他或她是一件坏事，对公司本身也是。当然，首先是对程序员来说。

请记住，这种情况在编程市场上很少见。提供的工作机会比真正的程序员还多。在它改变之前，你有机会找到一个有挑战性的项目。

我自己遵循了这里给出的所有建议。从那以后，我的任务不再无聊。我开始做自己喜欢的事情，处理有趣的事情，快速发展自己的技能。

记住，你的自我成长只掌握在你自己手中。让你的工作日(不仅仅是)更加令人兴奋也是。团队领导、经理或其他任何人都不会只给你可怕的任务来让你开心。所以努力吧，改变你的工作流程，直到你完全满意为止！

你目前的哪些任务可能被认为是枯燥和无用的？你如何让它们变得更有趣，为什么你还没有这样做？
