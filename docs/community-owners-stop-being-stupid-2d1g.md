# 小区业主，别犯傻了！

> 原文：<https://dev.to/omarkdev/community-owners-stop-being-stupid-2d1g>

一个有影响力的人开始说傻话，人们开始把它当成法律，这有多大问题？

我最近读了一篇文章，谈到了杨奇煜·普登西耶(Symfony 的创造者)和泰勒·奥特威尔(Laravel 的创造者)的基准比较。您可以查看文章:

泰勒·奥特威尔对杨奇煜关于 Symfony 基准测试的说法是否正确？

读了这篇文章，我想，与其他社区(主要是前端)相比，一个 PHP 社区是如何的不团结，这是由于所有者自己的过错。

令人难以置信的是，只有在我们的地区，我们才能在一个地区看到专门的社区，就像由家具制造商组织的螺丝爱好者俱乐部会很奇怪一样。一方面，这非常酷，因为它促进了特定领域的进化，但当这些领域开始相互竞争，忘记了目的，即让语言更容易为所有人所用时，这就成了一个大问题。

### 基准真的是优先考虑的吗？

如果基准测试真的是重中之重，那么在本文中我们将讨论用 C 语言编写的框架。

毫无疑问，基准测试会影响我们的选择，但我相信对于大多数开发人员来说，这不是优先考虑的事情。有成千上万的其他因素影响选择，例如:学习曲线、劳动力市场、敏捷性、设计模式、文档、设计、社区等等。为什么我们不仅仅讨论这些问题呢？还是他们不谈论这些问题？

> 在基准测试中，应用程序运行的环境比自身的技术影响更大。

#### “但是有一个市场，他们在里面打架”

我同意，不管他们多么奇迹般地把愚蠢放在一边，一起做事，你们还是有比较的，因为他们是不同的产品。但是，如果他们真的需要在市场上战斗，并且基准是他们的首要任务，他们为什么不与 Code Igniter 竞争呢？

基本上，当这种情况发生时，everything community 总是说“技术是手段而不是目的”，他们忽略并试图证明他们的草比其他的更绿。最糟糕的是，当你的追随者开始把它当成真理时，事情看起来或多或少是这样的:

> 你的框架应该支持语言进化，而不是相反。

所以[泰勒·奥特威尔](https://medium.com/u/d694c31ce941)和[杨奇煜·普朗西耶](https://medium.com/u/4ecc86a54ac5)(以及其他社区所有者)，停止愚蠢的争斗，为社区推广更多的东西，这样我们就会有更多的人使用他们的工具，从而使它们进化。
