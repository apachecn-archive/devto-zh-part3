# 为什么你应该写更多的三个理由

> 原文：<https://dev.to/marek/three-arguments-for-why-you-should-write-more-1no6>

让我试着说服你多写点。

我不是说多写*代码*。我是说为人类写作。

作为一名开发人员，擅长写作有很大的好处。这一切都需要练习。

你不一定要成为一个好作家才能写作。事实上，恰恰相反:你首先要写很多，然后随着时间的推移，它变得越来越容易。

擅长写作根本不是写高质量的东西。质量不重要。你可以写一篇蹩脚的博客，这绝对比没有博客好。

练习写作的目标不是写出高质量的材料。目标是高效快速地将你的想法融入文本。这是一种首先属于你的技能，其次才是属于读者的技能。

你写得越多，就越容易打开一张空白的纸来写。

你可能会告诉我:我不是作家，我是开发者。我没有时间写作。好吧，那么这篇文章是给你的。让我们用三个理由来解释为什么你应该这么做。

## 为合作者争论

写作是提升你作为合作者的价值的最有力的方式。

大多数软件团队不会因为他们写了多少代码而遇到瓶颈。他们受到团队成员之间通信带宽的限制。

对于任何开发团队来说，这里有一个很好的策略:过度沟通。过度沟通是不会出问题的，因为你会因为误解或者沟通不畅而消除很多问题。

以下是你可以做的与写作相关的重要事情:

*   与团队的其他成员共享进度报告
*   编写教程和入职清单，帮助新团队成员快速上手
*   描述你遇到的障碍以及你的团队如何解决它们
*   通过写下修复、权衡和经验教训，使错误报告更好

如果你的一些团队成员是远程的，或者如果你的团队成员有时在家工作，那么书面的沟通就变得至关重要。

如果你在办公室进行面对面的谈话，而事情没有被写下来，那么缺席的团队成员将会错过重要的信息。

松散消息适用于短期的短暂交流。但是如果你希望你的团队认真对待协作，写下真正的事情。我指的是 Slack 之外:在知识库、wiki 或文档库中。一个你必须以结构化的方式展示你的想法的地方。

## 为竞争对手的一场争论

也许合作不能让你有动力。

也许你很有竞争力，你现在更关注职业发展。

作为一名开发人员，擅长写作是脱颖而出的一种非凡方式。

沟通是一项被开发人员忽视的技能。从我就此进行的许多对话中，我怀疑许多开发人员进入这一行正是因为他们更愿意与计算机交互而不是与人交互。如果你很有竞争力，想要在职业生涯中获得成功，那么写作会是一个巨大的优势。

开个博客。写下你的项目。与世界分享你正在学习的东西。这些都是职业发展的有力策略。这就是你被关注的方式，也是你进入梦想公司的方式。

## 内向者的一种论调。

由于缺乏更好的术语，我将这种观点称为“内向者的观点”,但这并不是我真正的意思。

这是我真正的意思。如果以上两个论点都不能为你做到，那就自己写吧。

软件开发的很大一部分是解决问题。解决问题始于首先识别和描述你面对的是什么样的问题。

描述问题。写出来。这将迫使你真正思考你已经知道的，你需要发现的，以及如何把这些碎片拼凑起来。

令人惊讶的是，一旦你把大多数问题写出来，它们看起来就变得简单多了。有时你甚至在把问题写下来之前就想出了解决方案。

这就像橡皮鸭调试的概念。和某人讨论一个问题是有帮助的，因为讨论的行为帮助你的大脑描绘出正在发生的事情，并让你更接近解决问题的方法。

写作甚至比和橡皮鸭说话还要好。它给了你一个问题的记录，你可以在将来返回。你可以和其他面临类似问题的人分享。这是一个巨大的胜利。

## 你写什么其实并不重要

我试着列出了写作的三个角度，来吸引可能与你对话的不同动机。

我学到了一些东西:如果你想擅长写作，你写什么其实并不重要。就写吧。

实践累积起来。让我重申一下:不要太担心你写作的质量。这种担心导致了文思枯竭。我喜欢赛斯·戈丁在这方面的建议。他说:像说话一样写作。没人会有语塞。

不知道写什么？从自己的日常经历中挑选事情。写下你今天学到的东西。

没地方贴？将其发布到[开发到](https://dev.to)。

如果你每天或每周都这样做，我保证你的写作能力会提高得很快。这都是关于数量和一致性。
