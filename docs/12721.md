# 评论的重要性

> 原文：<https://dev.to/ispirett/the-importance-of-commenting-2gjl>

我开始写代码已经有一年了，虽然我知道注释的重要性，但我并没有把它看得那么重要。我最近开始了一个大的个人项目，这个项目正在慢慢变成一个创业项目。

大约三个星期深入代码，到处都是文件，解决一些什么复杂的问题(可能比我应该使用的代码更多)但是作为新手的所有好处，对吗？)我开始意识到一些事情。每天当我坐下来为代码添加功能时，我都要花费比需要更多的时间来筛选以前编写的代码，在这里我意识到，我没有做足够的注释。

我本可以避免那些头痛的事情，因为我不得不在文件之间来来回回，试图重新叙述我当时的想法。你能想象我会节省多少时间吗？如果我只是注释，我会在代码中走多远？

现在说那是旧闻感觉很好。对我的代码发表评论帮助我成倍地提高了效率。我现在能够浏览 400 多行代码，并且很容易知道每个代码块做什么。

你知道，有时候当你得到一个问题的紧急解决方案，你只是想实施它，然后继续前进，但是，相信我。从长远来看，花时间对你的代码进行适当的注释是值得的，尤其是对于更大的项目。

快乐的评论乡亲们！
