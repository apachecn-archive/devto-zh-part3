# 你不是你写的代码

> 原文：<https://dev.to/ggenya132/you-are-not-the-code-you-write-2h6l>

听说你作为一个人的价值不是来自你的劳动产出，这是一种普遍的情绪。

我们的工作是我们身份的核心部分，为我们的专业成就，有时是个人成就感到自豪是很重要的，但我真诚地希望，当我们是谁的生存问题成为焦点时，我们都有比我们的职业更多的东西要回答。

像技术领域的许多其他人一样，我没有传统的四年制计算机科学学位；我是 java 训练营的产物，所以我会第一个承认，当我把自己的知识基础与一些同事的进行比较时，我觉得自己特别容易患上冒名顶替综合症。除此之外，我倾向于投身于我所追求的任何手艺或爱好(目前是攀岩),所以我强烈地理解成为你所创造的艺术的渴望，无论你选择用什么媒介来表达自己。

说了这么多，我仍然无法理解我的一些同行，他们的全部自我意识都依赖于成为一个“优秀的程序员”。我将讲述泰勒(名为 changed)的故事，他是一名新兵训练营的学生，在我们队列的最初四周期间，他非常激烈地斗争，以使自己成为开发人员 guy guru tech dude 业余爱好者 coder genius。他想给人的印象是，他显然是某种代码奇才——他通过询问大多数学生最初知识范围之外的技术问题来施展这种魅力，并吹嘘自己在几年前作为计算机科学专业的学生已经做过一些编码工作。

然而，泰勒的问题是，他辛苦赢得的自豪感非常脆弱，当其他有才华的学生开始获得一些动力并适应编程世界时，这种自豪感开始迅速破灭。我清楚地记得泰勒在课程中向一位极其耐心的讲师抱怨他的“排名”下降。我必须承认，当我在新兵训练营的时候，我对泰勒和他的困境没有一点同情；然而，在这个行业呆了一段时间后，我已经能够理解泰勒和许多像他一样的人，也许我们都是自己强加的有时无法实现的卓越愿望的囚犯。

在你的内心，有许多无限的身份等待被发现，每一个都充满了细微差别和复杂性。为什么要把你所有的价值都塞进这个令人垂涎的绰号里——为什么要把你所有的价值都放在一个不可能限制完美的期望后面。意识到你心目中的了不起的工程师是一个移动的目标，是不断变化的地平线上海市蜃楼的微光。你可以拉近你的技能和你对巅峰表现的想法之间的距离，但这种方法本质上是渐进的。你和你的优秀感之间总会有一个差距，不管这个差距有多小。拥抱这种距离，和我一起说，足够好就足够好，期待生活中没有编码的东西。至于我自己，我只想多爬一些(字面上的)墙。

我毫不怀疑这个想法和讨论可以推广到许多职业和“策划身份”。请和我一起讨论你是否在工作场所或学校有过类似的经历。
