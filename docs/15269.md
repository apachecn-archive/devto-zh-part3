# 6 个最广泛使用的角度概念

> 原文:[https://dev . to/meenakshi 052003/6-最广泛使用的-angularjs-concepts-3pl3](https://dev.to/meenakshi052003/6-most-widely-used-angularjs-concepts-3pl3)

这篇文章描述了 web 开发中六个最广泛使用的 AngularJS 概念。这些是开发者在各种社区和论坛上提出或回答最多的一些主要话题。

在所有六个主题中，该指令通过取代其他所有主题锁定了第一名。由于主要是我和我的一些同事收集和整理了这些数据，所以我们可能漏掉了一些。因此，欢迎您添加任何进一步的主题，让我们知道它在顶部的原因。

以下是 Angular 开发者最喜欢的六大主题列表:

a)指令
b)范围对象
c)Ng-重复
d)角度 UI &引导
e)路由
f)服务

现在让我们回顾一下为什么上述领域比其他领域更受欢迎。

# [](#a-directive)a)指令

这是 AngularJS 框架的基石和最受关注的特性。以下是一些关键原因:

## [](#i-highly-reusable)i)高度可重复使用

指令作为模块的一部分创建一次。之后，在定义新模块时，在模块中使用它会将它作为一个依赖项注入，并在 web 页面上任何需要的地方提到这些指令。

## [](#ii-readability)ii)可读性

人们可以给指令起任何直观的名字，以增加代码的可读性和可用性。

## [](#iii-dry-approach)iii)干法

与模板一起，指令成为一个更致命的特性。它通过在每个需要的地方重用相同的 HTML 代码而不是到处复制它来消除代码冗余。

# [](#b-scope-object)b)范围对象

范围曾经是程序员第二喜欢的话题，对于那些使用 Angular 第一版维护应用程序的人来说也是如此。范围的概念对于依赖注入是至关重要的。它既强大又复杂。

# c)Ng-重复

ng-repeat 特性为 AngularJS 增加了更多功能，因为它是一个关键特性，消除了在连续迭代中重新执行 HTML 代码所需的服务器端逻辑。有了 ng-repeat，根据需要重复 HTML 代码就变得容易多了。

# [](#d-angular-ui-amp-bootstrapping)d) Angular UI &自举

Angular 缺乏的一个方面是它不能产生一个单一的解决方案来创建一些具有事件功能的伟大 UI。人们仍然需要使用一些 UI 框架，如 bootstrap，Kendo-UI 来创建一个现代和时尚的 UI。这个缺口可以通过使用 Angular UI 和 bootstrap 来填补。

AngularUI 捆绑了一些非常吸引人的功能集，用于增强路由、网格工具、angular 编辑器插件和引导模块等。

# [](#e-routing)e)路由

Angular 中的路由负责创建单页面应用程序。由于 Angular 使得使用路由创建单页面应用程序变得非常简单，因此它自动成为一个显著的特性。一个人需要精通这个概念，以充分利用它的应用。web 开发人员喜欢使用它。

# [](#f-service)f)服务

当角度应用变得更大时，它会变得更加复杂。在这种情况下，最好将控制器中的一些业务逻辑外部化为可重用的对象。服务的概念在角度组件中提供了高级别的模块化和可重用性。这些服务是即插即用型的，可以通过使用依赖注入(DI)特性嵌入到其他模块中。Angular 具有以下支持 DI 的元素:

*   控制器
*   服务

创建服务的一个突出技术是使用工厂配方方法，也称为服务配方。
