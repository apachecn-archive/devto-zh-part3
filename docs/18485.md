> 原文:[https://dev . to/陈格/TIR-a-call-for-rust-2019-roadmap-blog-posts-1p5j](https://dev.to/chenge/tir-a-call-for-rust-2019-roadmap-blog-posts-1p5j)

> 都快 2019 年了！因此，Rust 团队需要为 Rust 明年的发展制定一个路线图。从最高层面来看，Rust 的开发流程如下所示:

1.  Rust 社区在博客上写下了他们想看到的东西。
2.  核心团队阅读这些帖子，并产生一个“路线图 RFC”，这是一个关于下一年开发的建议。
3.  RFC 被广泛讨论，并根据反馈进行修改，最终被接受。
4.  该 RFC 成为下一年接受或推迟 RFC 的指南。
