# 吉托特，两周后

> 原文:[https://dev.to/dimensi0n/gitote-two-weeks-after-2jd9](https://dev.to/dimensi0n/gitote-two-weeks-after-2jd9)

> 两周前我看到一个关于这个新`open source end-to-end software development platform with built-in version control, issue tracking, code review` : [这个帖子](https://dev.to/gitote/gitote-has-been-launched-m35)。我报名去看看[这个新平台](https://gitote.in)是什么样子的。两周后我可以说出我的想法。

⚠️:我只有 15 岁，我的英语不是很好，我想提高它，所以如果你注意到语言错误，请在评论中说出来。⚠️

# [](#good-points)好点

*   这是一个新的开源软件

我不知道这是不是一个论点，但在我看来，这非常重要。我喜欢开源替代方案。这是基于 [Gogs](https://gogs.io/) 这是自我主机。

*   简单的用户界面

我注意到的第一件事是用户界面，如此简单，如此容易理解，好吧，有些人可能会说这是 Github 设计的复制粘贴，但这对我来说不是问题。

*   能见度

Gitote 是出于对社会的关注而制作的。有一个探索页面绝对令人惊叹，我发现了很多好项目，尤其是[**J**ust**A**other shell](https://gitote.in/sahilrajput/Jall)或 [Full Tilt Bash](https://gitote.in/TheRedSpy15/Full-Tilt-Bash) 。

# [](#negative-points)负分

*   没有单元测试服务

这对我来说是个大问题，我需要单元测试，但是现在如果你想使用单元测试服务，你必须在 Gitlab/Github 上创建一个镜像。编辑:Gitote 正在建立一个 drone-ci 实例。

*   我不知道，但我必须找到其他的缺点

# [](#conclusion)结论

我不是说你应该用它来代替 Gitlab/Github/Bitbucket，但是你还是应该试试 Gitote。我忘了说我是 Gitote 的远程实习生，但是我说了我认为好的或坏的所有事情。

目前在 Gitote 上没有很多人，但我认为流行只是时间问题。
