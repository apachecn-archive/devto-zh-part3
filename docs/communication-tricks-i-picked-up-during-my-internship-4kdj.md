# 我学会的提高沟通技巧

> 原文：<https://dev.to/maniflames/communication-tricks-i-picked-up-during-my-internship-4kdj>

这是我在 UpReach 实习的最后一周，我玩得很开心！在 UpReach 摄影亭的前端、后端甚至硬件组件上工作教会了我很多东西。然而，最令人兴奋的事情是我在柏林住了五个月。然而，在另一个国家工作也有不利的一面，比如语言障碍。尽管没有一个人，包括我在内，是以英语为母语的，但英语是最流行的语言。这是因为我能读一点德语，但不会说。最重要的是，我经常不得不与项目管理和销售人员交谈，他们并不总是了解产品背后的技术细节。有四个技巧可以改善我与同事的沟通，我很想与大家分享。

# 1。记录对话

照相亭的生意很快。我不得不不时参加一些简报会，讨论针对特定客户的特殊配置和开发。很多信息会在很短的时间内抛给我。通过在简报中记下重要的细节和关键词，我能够跟踪对话。这个技巧听起来像是显而易见的事情，但是它会让你惊讶忘记记笔记是多么容易。记笔记不仅在当下对你有帮助，从长远来看也是如此。小细节会带来大的不同，在某些时候你会忘记，阅读过去的笔记真的会有帮助。

# 2。推迟你的意见

如果你一次又一次地经历了某些模式，很容易得出结论。在此之前，请确保您收到了*所有*必须传达的信息。对于项目经理来说，一些看似微小的细节可能会对您必须构建的内容产生很大的影响。有人可能会提出一个似乎不适合这种情况的解决方案。在你决定实施一件“更好”的事情之前，先问问这个人为什么要这么做。点点头，做你认为正确或更快的事情可能会适得其反，耗费你更多的时间。一定要确保你理解全部的上下文。当你对大局有所了解时，交流你的顾虑和选择。

# 3。以适当的复杂程度解释技术

每当你必须解释事物的技术方面时，要根据听众的需求来调整你的解释。作为开发人员，很容易忘记每天会遇到多少首字母缩写词和抽象概念。如果你和一个不是开发人员的人交谈，你很可能不得不降低行话的语气。确保行话和技术细节的数量适合你的谈话对象。通常要花时间才能弄清楚一个人对细节的适应程度。UpReach 的一些项目经理倾向于尽可能远离行话。其他人对技术了解很多，可以很容易地理解我。如果你能够用不同的复杂程度来解释自己，那么与不同的部门交谈就会容易得多。

# 4。确保每个人都在同一页上

假设每个人在简报结束前都在同一页上是一个巨大的错误。很容易忘记或误解一些事情。在集体结束简报之前，大声总结重要的结论，并询问是否遗漏了什么。在每个人都同意后，陈述每个人从这一点开始要做什么。从长远来看，这可以节省大量的时间和沟通失误。

清晰的沟通是任何领域工作中非常重要的一部分。希望这四个技巧能让你和同事之间的交流更加顺畅。我仍然需要在所有这些方面变得“优秀”,但仅仅是尝试就已经能带来很大的不同。
