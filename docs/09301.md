这是一个由四部分组成的系列实践教程，涵盖了基于我为我的训练营学生写的练习的 Git 工作流程。我发现它们在课堂上很成功，希望它们作为文章是有用的。

*   [学习 Git 工作流基础知识](http://jarednielsen.com/learn-git-workflow-basics/)

*   [学习 Git 工作流集中协作](http://jarednielsen.com/learn-git-workflow-centralized/)

*   [学习 Git 分支工作流程](http://jarednielsen.com/learn-git-branch-workflow/)

*   [学习 Git Fork &拉请求工作流](http://jarednielsen.com/learn-git-fork-pull-request/)

基础教程假设安装了 Git(或 Git Bash ),并且您熟悉(但不一定精通)命令行。

独自学习协作 Git 工作流很难，所以集中式和分支式教程是为两个人一起工作而编写的。你不需要一个伙伴来跟随，但是你将不能完成所有的操作，除非你配置单独的 GitHub 账户。

Fork & Pull 请求教程使用了一个有助于开源项目的真实示例。
