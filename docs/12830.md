# 一次勇敢的冒险和一只听起来脱离上下文的鸟。

> 原文：<https://dev.to/techiev2/a-brave-adventure-and-a-bird-that-sounds-out-of-context-33n4>

为什么银行会说他们不能处理大量现金，因为他们不能识别你？很困惑，对吧？嘿！推特，一定要听。

今天早些时候，上传截图到 Twitter 手机时发生了一个有趣的情况。这是一个反复出现的问题，它打开了一个阅读指纹和安全知识的渠道。请继续读下去，看看哪里出了问题，哪里是有趣的失误。

前几次我尝试在 Twitter mobile 上上传时，由于一个大图片大小的消息，事情失败了。而且，对于配置文件图像来说，有些确实很大，每张大约 12MB。在某些情况下，他们每个人都使用勇敢的网络浏览器，这已经成为我这些天的默认设置。说到默认，你也应该迁移；完美的选择。

切入正题，我没有做任何情境研究者会做的事情；尝试其他浏览器或图像。坦率地说，为这个测试下载一个浏览器和登录是越来越麻烦的事情。所以很久以前就应该清晰的结束了，直到一段时间以前，当一个 50kb 左右的截图会失败。

即使对我这样的懒骨头来说，这也令人感到惊讶，不是吗？大约 50kb 的图像怎么可能大到需要调整大小？引用 Twitter 的话，“一个或多个图像超过了大小限制，无法调整大小”。Twitter 用这个消息来回应是相当奇怪的。

这一次，在没有另一个浏览器的情况下，我尝试了一些别的东西。关掉了所有的保险栓，瞧！上传很顺利。接下来，打开了除指纹保护之外的所有东西，一切都很好。一旦开启，事情又会失败。

我不明白，当根本问题在别处时，Twitter 为什么会说图像才是问题所在。在处理服务错误时，这可能是极端情况中的极端。但是，这并不能保证显示不相关的错误消息来迷惑用户。

在这一点上，挖掘勇敢的来源，看看到底发生了什么引擎盖下。退一步说，事情很有趣。如果你想进一步了解什么是指纹保护，【Brave browser 的这个是个好地方。
