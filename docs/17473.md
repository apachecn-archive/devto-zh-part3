## [](#the-magic-of-writing)写作的魔力

写作有一种魔力。除了它是一项改变世界的发明，让我们人类超越了生物记忆的限制，并通过亿万年来进行交流。

除此之外，写作就像和你自己对话，像反思但大声说出来，一些更有意识的东西。就像如果独自坐着，专注于一张白纸，或黑暗的屏幕，让我们直接接触到我们最深处的自我，而仅仅是思考本身是无法触及的。

这种深度的特质在与他人交流信息时也非常有用。我不认识你，但我经常有这样的对话，我想向一个人传达一个想法，但在这种情况下，我无法说出我真正的意思，或者错过了部分细微差别。我经常在事后很快反思，我认为“我应该说 X”，“这就是我真正的意思”，但那时已经太晚了。另一个人在阅读时也更容易倾听，分心的事情更少，对环境和下一次反驳的关注更少。

因此，写作是传达你想法的绝佳媒介:一方面，它给你一个完美的环境，让你找到你想以最纯粹的形式分享的真相，另一方面，它给你一个亲密的、全神贯注的读者。

准备好开始写作了吗？

## [](#book-conception)书籍构思

大多数事情都是从一个想法开始的，一个唠叨的微小的想法变成一个耳语，变成一个声音，可能会消失，也可能会发展成一种内心不可阻挡的写那本书的需要。很好！你需要所有的热情和激情来写你的书！

所以在你写书之前，有两件事你需要决定。大部分其他的事情你可以在进行的过程中弄明白，但是这两个是你的书成功的核心:为什么和 T2 做什么。

### [](#before-you-start-why-are-you-writing-a-book)开始之前:你为什么要写书？

**为什么 <sup id="fnref1">[1](#fn1)</sup> 你正在写这本书**是必不可少的，因为它将定义你成功的衡量标准，并在写书的漫长旅程中让你保持动力。

具体的**为什么**不如花时间去想重要。**为什么写这本书？**我想要实现什么？有没有更好的方法来实现这一点？

慢慢来，这将是你开始投入宝贵时间前的最后一个关卡。以下是一些可能的原因:

*   因为我热爱创造力和创造
*   因为我想建造一个杰作，一些美丽的东西
*   赚钱谋生
*   兼职赚点外快
*   因为我喜欢写作
*   因为我想帮助别人
*   要了解特定主题的更多信息
*   提高我的写作技巧
*   成为思想领袖
*   推进我的事业
*   被公认为某一领域的专家
*   变得不朽
*   成名

所有这些都是可供选择的有效理由。花些时间反思并选择对你、你的生活和你的职业生涯重要的。投入一些思考，因为**为什么**将是贯穿整个写作过程的指路明灯。

对我来说，是对创造的热爱和对创造非凡和令人愉快的事物的渴望。我在这个地球上已经有一段时间了，做了很多事情，做了很多项目，但我总是觉得我没有控制力、技能或时间来制作一些真正非凡的东西。

JavaScript-mancy 系列是我最大的努力，尝试去做一些不平凡的事情，给那些经历它的人带来快乐和喜悦。我并没有忽视写作的其他许多好处，比如帮助他人、学习、谋生、建立声誉等等，但对我来说，这些都是创造伟大事物的副作用。

那么，**为什么要写一本书**？

### 你在写什么？

厉害！既然你已经确定了*为什么*，下一步就是考虑**你想写什么**。我认为在选择**写什么**时要遵循的最重要的原则是:

**写一些你关心的事情**。

为什么？因为写作是一个漫长、艰苦、折磨人的过程。写一些你不关心的事情，你会有一段不愉快的时间来积聚能量把它看完。你可能会有一段痛苦的写作时间，当事情变得艰难或者有更好的事情发生时，你可能会放弃写作。

**为什么**决定了**对什么**的强烈程度。想通过写作谋生？然后写一些人们真正想要或需要的东西。想在某个特定领域成名吗？然后写一本关于该领域一个重要话题的书。你知道要点了。

在大多数情况下，选择一个要写的主题不会是一个非常精心的选择，而是一个有机的选择。在你的职业发展中，你会与某些技术打交道，成为一名专家，写关于它们的文章会感觉像是自然的选择。在这种情况下，需要考虑以下几点:

*   你想写主题 X，但是已经有很多关于它的书了。不要感到气馁。竞争其实是一件好事，它说明你的书是有市场的。与竞争对手相比，你需要花费一些额外的时间让你的书变得特别和新颖。
*   **你想写一个话题 Y，但是没有关于它的书**。不要感到气馁。利基市场可能非常有利可图，尤其是对低成本的自助出版图书而言。如果你是唯一一个写了这方面的书的人，也更容易被认为是专家。
*   是什么让你与众不同？ **你正在写的这本书有什么特别之处？**你想写的那本书最特别的地方在于，它是基于你的经历，是你写的。你有什么不容易拥有的知识或经历？试着在你的书中印上你的性格和个性，让你的文字成为你自己，你就会拥有其他书所没有的额外的*。*
*   **写框架？平台？编程语言？模式？**不同的话题有不同的有效期。软件开发是一个超快节奏的行业，事情变化很快。有些事情变化比其他事情快。例如，框架的寿命非常短，而编程语言的寿命通常更长。选择题目时，用你的**为什么**作为指路明灯。

我的什么是 JavaScript-mancy 系列，这个选择是非常有机的。几年来，我一直在写一个充满幻想的博客，当时我在 JavaScript 方面做了很多工作。中对 JavaScript 的普遍看法。NET 社区非常负面，我看到了改变这种状况的机会，那就是向每个人展示 JavaScript 确实很棒。因此 JavaScript+Fantasy = JavaScript-mancy。

所以现在我们已经有了**为什么**和**什么**，我们几乎准备好开始了。只剩下一件事，你是自己出版还是尝试与出版商合作？

### [](#publisher-or-selfpublishing)出版社还是自助出版？

我们现在这个时代是牛逼的。互联网、万维网、计算机的极端可用性和可负担性，以及总体技术的进步几乎使一切都民主化了。制作和销售音乐、新闻、制作广播或电视节目、写作和出版书籍，所有这些不久前只有少数人可以做的事情，今天所有愿意付出努力的人都可以做。

在你开始写书之前，你需要做出选择。你会独自一人独自穿越自助出版书籍的孤独之路吗？或者你和一个成熟的有经验的出版商合作？

我的大部分经历都是写自己出版的书，因此，这也是我在这些文章中关注的。但是为了便于讨论，让我们为这两种方法做一个案例，并强调我所看到的它们的优缺点:

#### [](#publisher-pros)发布者赞成

*   你有一个专业团队可以帮助你
*   你的出版商将为你制定一个时间表，并帮助你遵循它。如果你是一个拖延者，这很好。
*   您可以更轻松地外包打样、营销、发行、封面设计、排版、翻译等工作。专注于写作
*   经销商和实体店的更广泛分销
*   你可以提前付款

#### [](#publisher-cons)出版者反对

*   低版税
*   对时间表和书本身的控制和自由更少
*   你需要向出版商推销你的想法，或者有出版商联系你，他们可能会向你推荐一本书

#### [](#selfpublishing-pros)自助出版专业人士

*   更高的版税，更高的利润
*   完全控制和完全创作许可
*   自由设定自己的时间表

#### [](#selfpublishing-cons)自助出版的弊端

*   你只能靠自己了(尽管你可以组建一个团队)
*   你需要做所有的事情(你可以把事情外包出去，但是你需要扮演项目经理的角色)
*   你需要建立自己的分销渠道

总的来说，如果你和出版商合作，你作为团队的一员，可以专注于写作，外包其他部分，赚取较低的版税，但有可能通过出版商的分销渠道接触到更多的读者。如果你选择自助出版，你将拥有完全的控制权、更高的利润，而且你需要自己解决所有问题。

对于 JavaScript-mancy 系列，我希望拥有完全的控制权和完全的创作许可。我喜欢设计东西，所以我非常喜欢做封面艺术。我也想有一个非常灵活的时间表，因为我有一份全职工作，通常会做很多事情。考虑到所有这些因素，自助出版对我来说是正确的选择。我尝试了 LeanPub，它引起了我的共鸣，所以我选择了自助出版。

正因为如此，本系列的下一篇文章将集中讨论我在过去几年里学到的关于如何写一本自己出版的书的所有东西。**接下来是书籍执行**，以及写一本自出版书籍的所有实际细节。
