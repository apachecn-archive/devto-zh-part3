# 工程师、机械师、工匠

> 原文：<https://dev.to/wuz/engineer-mechanic-craftsperson-13gb>

我刚刚更换完一辆汽车的电池，这让我开始思考我们作为开发者所扮演的不同角色。我们经历了如此多不同的阶段和角色，以至于很难缩小任何一个*的范围。*

当我想到这辆车和我觉得可以轻松搞定的东西时，我开始意识到我在任何时候扮演的一些角色。在任何特定的一天，任何特定的时间，我可能是一名工程师，一名机械师，一名工匠或者三者的结合。

让我解释一下。

## 工程师

传统上，工程师是设计和构建问题解决方案的人。机械工程师为机械产品创造解决方案，航空工程师为飞行创造解决方案。作为一名开发人员(通常被称为软件工程师)，我的工作经常涉及为业务需求设计解决方案——在今天的世界中，这几乎总是一个软件。

当扮演这个角色时，开发人员扮演架构师和计划者的角色。他们决定使用什么工具，实现什么集成，以及最终产品应该是什么样子。在与设计师、项目经理或客户合作时，他们可能会在各个部分上做出妥协，但在一天结束时，他们会决定将哪些部分放入软件以及它们将去往何处。

“作为工程师的开发人员”输入业务关注点、项目需求和代码限制，并输出一个(有希望的)工作软件。

## 机械师

开发人员承担的第二个角色是机械师。机械师在汽车、设备或其他系统离开规划阶段后对其进行维护。除了召回之外，不可能再有大的改变。机械师的工作是解决工程师遗漏的问题或由于磨损而引入的问题。

有了这个角色，开发人员就充当了保持软件运行的角色。当工程软件开始崩溃的时候——可能是由于变化的标准，过时的依赖，或者客户希望他们的软件做别的事情——开发人员现在扮演了机械师的角色。就像机械师维护和改进汽车一样，开发人员也经常维护和改进一个软件。无论他们是升级 Wordpress，修复弹出的 bug 还是添加额外的功能，这个角色都是关于小的改变。开发人员经常会在工程师和机械师的角色之间快速切换，构建新的东西，然后在用户面前维护和改进它。

“作为机械师的开发人员”输入一个软件，然后输出一个类似的软件，按照规范进行微调和固定。

## 工匠

最后，开发人员经常扮演工匠的角色。工匠在特定领域发展他们的技能——也许是木工或电子。他们的手艺可以是业余爱好，也可以是全职工作，但通常他们会努力提高自己的技能。他们花时间思考如何以不同的方式使用工具，或者如果他们调整流程的某个部分会发生什么。

在这个角色中，开发人员致力于提高他们自己、他们的个人技能以及其他开发人员的技能。就像工匠可能会尝试一项新技术或构建一个新工具一样，开发人员通常会学习一门新语言或设计一个新工具用于他们的开发。担任这一角色的开发人员可能会学习 Webpack 的工作方式，学习一门新的编程语言，或者在他们的个人网站上工作。同样，开发人员可能经常在工程师、机械师和工匠之间转换角色。通常，一个新的开发人员可能会在这个角色上花更多的时间，学习磨练他们的技术——然而，一个熟练的开发人员也会继续成长为一个工匠。

“作为工匠的开发人员”输入他们当前的技能，并输出一些新的理解，或者为他们自己，或者为整个生态系统。

## 连续显影

这些角色不是静态的，这将开发人员与这些职位和角色的许多其他版本区分开来。开发人员不断地从一个角色转换到另一个角色，这取决于他们的资历、团队组成、当前兴趣等等。“持续开发人员”知道何时需要每一个角色，并尽他们所能满足技能要求。
