# 调整与改造

> 原文:[https://dev.to/jessekphillips/tweak-vs-revamp-4keh](https://dev.to/jessekphillips/tweak-vs-revamp-4keh)

这是一个困难的问题，对于已建立的软件解决方案和发布过程，你如何适应一个改进？

Docker 正走在前台，它的管理软件即将完成 mindshare。开发人员构建许多产品来帮助解决一个开发问题(dev.to 公布了一些)。

其中许多解决了实际问题，并可能有益于现有和未来的开发，但这可能是一项大工程。对于我的个人项目，我可能会避免购买任何东西，但如果我向公司提出建议，考虑所有适当的选项是很重要的。

因为我不会在个人项目中触及这些，所以我不想浪费一个解决问题的试验期，因为我没有为财务提前足够长的时间计划我的需求，所以没有得到批准就放弃解决方案。

我正在做出正确的选择，但有时我想知道是否有应该做的评估，我没有时间做，至少现在还没有。
