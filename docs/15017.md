# 我使用 GitHub 桌面应用程序是为了一件事。我这样做错了吗？

> 原文:[https://dev . to/Ben/I-use-the-github-desktop-app-for-one-thing-am-I-doing-this-wrong-1i 00](https://dev.to/ben/i-use-the-github-desktop-app-for-one-thing-am-i-doing-this-wrong-1i00)

当我去查看一个不能纯粹根据代码评估的 PR 时，我会点击以下链接:

[![](../Images/545f6855e9952b1291141279d8950a92.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--HhYLT5fI--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cl.ly/22d4b8e211b5/%255Bf3d1a9246da555505da17b175ac13f4e%255D_Image%25202019-01-26%2520at%252012.55.07%2520PM.png)

我发现这是检查分支和给工作应用一个手动外观的最无缝方式。否则我会使用 git CLI，但我发现这是目前为止我最喜欢的方式。

但在我看来，这似乎是错的。我一直在运行这个应用程序，就是为了这个唯一的目的。似乎*关闭了*，因为我必须一直滚动到页面底部并点击一个文本链接。如果这是*合适的*行为，对我来说这个按钮看起来更像旁边的“压缩并合并”按钮是有意义的。

我知道这对我有用，但我只是好奇我是否错过了什么。
