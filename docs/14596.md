# 红宝石和仙丹是完美的阴阳吗？

> 原文：<https://dev.to/joshhadik/are-ruby-and-elixir-the-perfect-yin-and-yang-425l>

过去几周我已经开始为找工作做准备了，你知道这意味着什么… **算法！**

我每天花大约 45 分钟来解答一些常见的面试问题，我从一开始就努力去做的事情之一就是为每个问题寻找多种独特的解决方案。

起初，我只是用 Ruby，因为它是我最强的语言。但是昨天我决定尝试一些不同的东西。

我不会只在 Ruby 中寻找多种解决方案，而是将每个问题解决两次。一次是红宝石，一次是长生不老药。

我这样做才两天，但我已经知道这是我做过的最好的决定之一。

对于那些不知道的人来说，Ruby 和 Elixir 是两种在美学上非常相似的语言，但在幕后的行为却完全不同。

Ruby 本质上是一种**面向对象的**语言，严重依赖于**迭代**和**变异**来解决问题

**Elixir** 是一种**函数式**语言，依靠**递归**和**模式匹配**。

每天用两者来解决问题，迫使我从两个完全不同的角度来看待同一个问题，并使用两套完全不同的工具来解决它。

从长远来看，**我认为这样做会让我成为一个更多维度的开发人员**，我认为这是区分初学者和专业人员的关键技能之一。

无论如何，这让我对整个 Ruby / Elixir 生态系统以及两者之间的交叉进行了很多思考，我意识到**这两者几乎是完美的阴阳两极**。

他们的运作方式截然相反，但有很多相似之处和密切的关系。其中一个是弱的，另一个是强的，如果你能掌握两者，你会成为一个更强的开发者。

你们觉得……红宝石和仙丹是完美的阴阳吗？

还有其他两种语言符合这种模式吗？
