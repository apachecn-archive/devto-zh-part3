# 康卡斯特正在代理所有未加密的内容

> 原文:[https://dev . to/BD morin/Comcast-is-proxing-all-encrypted-content-34nh](https://dev.to/bdmorin/comcast-is-proxying-all-unencrypted-content-34nh)

我最初把这个贴在原本不安全的平台上， [facebook](https://www.facebook.com/bdmorin/posts/2188976868082331) 。我应该编辑这个语法，但我只是想敲出来，因为你知道...工作/工作。

我不能忍受#康卡斯特，没有一个认识我的人觉得这很奇怪，我总是对他们咆哮。事实上，康卡斯特是这么多人的唯一选择，这是可笑的悲哀。我的工作要求我经常上网，我做了大量的安全研究和一般研究。

今天，我发现了一个安全呆子能找到的最可怕的东西。康卡斯特正在迫使所有未加密的流量通过康卡斯特代理服务器。我没有选择，没有人问我，也没有人通知我(我确定长达 938429 页的 TOS 提到过)。这使得康卡斯特能够将他们想要的任何东西注入到你未加密的网页浏览中。

如果你想了解这些插孔在做什么的技术细节，请参见这里:[https://gist . github . com/BD morin/7 BD 16 b 34 cf 75 c 0 f 6 DD 56155301793 c 4d](https://gist.github.com/bdmorin/7bd16b34cf75c0f6dd56155301793c4d)

我测试了一个流行的网站，tvmaze.com(一个只有 http 的网站)，有和没有 VPN，在 HTML 交付的区别是康卡斯特 HTML 注入，其中包括第三方资产调用，分析跟踪等。

我想保护我的整个网络(包括我家里的所有人)免受这种绝对不可接受的间谍活动，然而这变得很糟糕，因为作为断线钳，我们使用流媒体服务，而网飞和 Hulu 不支持 VPN。这些服务会主动阻止 VPN，因为观众可能会出现在不同的地理位置(ODIN 禁止你不是授权区域)，所以如果我通过 VPN 运行我的整个房子，那么我们将无法使用流媒体服务。

我一直在考虑为任何端口 80 的流量部署一个本地强制代理，强制通过我的网关而不是康卡斯特的 VPN 连接。几乎每个流媒体服务都使用 HTTPs，所以这不会欺骗流媒体服务。

这种咆哮的目的是羞辱康卡斯特，而不是他们关心消费者。当你在网上浏览时，你可能会不断地看到 VPN 的广告，这就是为什么，当你的本地服务提供商劫持你的内容并在它到达你之前修改它时，你绝对不能相信它的原因。问问中国，你所有的流量在到达你之前都被监控和修改是什么感觉。在你有机会阅读原版之前，康卡斯特可能会改变任何东西。如果 Comcast 获得了 browers 接受的 CA，他们就可以劫持您的 HTTPS 连接，这在这一点上是完全可以理解的。

使用 Cloudflare 等 web 应用防火墙服务的网站也面临这种风险。Cloudflare 检查进出源服务器的所有流量，因此它是一个可以修改、跟踪和潜在阻止内容的单点。如果黑帽攻击 Cloudflare，数千家电子商务企业可能面临流量被监听的风险。康卡斯特也是一样，如果(以及何时)他们被入侵，他们可以修改你的流量，让你看到别人想让你看到的东西。

不要相信任何人。尤其是像康卡斯特这样毫无价值的公司。
