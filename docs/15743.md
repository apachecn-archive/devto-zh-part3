# 星期五爆炸#57

> 原文:[https://dev.to/horia141/friday-blast-57-2p15](https://dev.to/horia141/friday-blast-57-2p15)

[困难问题简短指南(2018)](https://www.quantamagazine.org/a-short-guide-to-hard-problems-20180716/)——大多数人都熟悉 P 和 NP *复杂性类别*，至少通过计算机科学核心的`P vs NP`问题。这里涵盖了一堆额外的复杂类——BPP、BQP、PH、PSPACE 和 EXPTIME 以及它们之间的关系。有时它们是明确的，有时我们有强有力的理由相信它们是相同的，而有时领域是开放的。

[从头开始介绍模型树(2018)](https://towardsdatascience.com/introduction-to-model-trees-6e396259379a) -计算机科学启发的 ML 模型(决策树和神经网络[1])的一个更酷的方面是它们经常是“可组合的”。这里，决策树中使用的空间划分方法与叶子处的线性回归相结合，以产生比“经典”常数模型更强大的模型。

[AWS、MongoDB 和开源的经济现实(2019)](https://stratechery.com/2019/aws-mongodb-and-the-economic-realities-of-open-source/) -和往常一样，这是 Stratechery 在技术商业方面的一篇好文章。这次的重点是开放源码。在过去的 2-3 年里，开源基础设施领域的技术公司经历了一段艰难的时期，这是一件*的事情*,因为云提供商正在以一种更方便、更便宜的方式提供他们的系统。Docker、ElasticSearch、Redis、MongoDB、Hadoop 以及它们的后台公司就是最好的例子。这篇文章主要关注 MongoDB 和 Mongo Inc .但是它告诉我们。这一领域的公司如果想要成功，就需要向价值链上游移动。我计划很快就此写点自己的东西。

[数据库复制入门(2017)](https://www.brianstorti.com/replication/) -数据库环境下数据复制的各种方法。很好，它既着眼于 SQL 和 NoSQL 解决方案，又着眼于各自如何解决问题。

[一天数十亿条消息——Yelp 的实时数据管道(2016)](https://engineeringblog.yelp.com/2016/07/billions-of-messages-a-day-yelps-real-time-data-pipeline.html)——描述了 Yelp 为组织其系统而部署的令人印象深刻的架构。基于 Kafka 的中央数据仓库并不是那种*未来科技*，我已经在这里发布过几次这样的方法。但是内置了流跟踪、文档和代码集成的中央模式存储非常好。
