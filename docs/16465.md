# 函数式编程之旅

> 原文:[https://dev . to/yordiverkroost/the-journey-into-functional-programming-4385](https://dev.to/yordiverkroost/the-journey-into-functional-programming-4385)

我们生活在一个由面向对象编程语言主导的世界。有时很容易忘记世界不是平的，地平线上还有更多。不仅仅是你用过的 OOP 范例。

或者至少，对我来说是这样。我曾经接触过“编程的另一面”，但那是在大学里。在那里，我使用函数式编程语言，如 Haskell、OCaml 和 Lisk。与面向对象编程相比有很大不同，但它有自己的优势。

但你知道会怎样。当你找到一份软件开发的工作时，你开始使用面向对象的语言。这正是我的情况。我沉迷于 C#和 Java 之类的语言，忘记了另一个世界。和新西兰一样，你听说过，也在图片中看到过。但这也是一个你从未去过也永远不会去的地方。

直到最近。

我的一个朋友提到他的团队使用函数式语言。老实说，这些天我越来越多地听到函数式语言。不仅是高中生和学者，在商业环境中也是如此。所以我问自己:“再深入研究一下那些函数式语言怎么样？”看看炒作的内容是什么。不会有坏处的，对吧？

为了学习函数式编程范例，你需要一门语言来练习。为了简单起见，我选择了我朋友也使用的语言。如果你有一个你认识的人，当你遇到困难时，你可以向他提问，这总是一件好事。

那种语言就是[灵丹妙药](https://elixir-lang.org/)。

对我来说，学习一门新编程语言的最好方法大致如下:

1.  通过浏览文档来了解这门语言的基础。你可以在这里找到长生不老药的资料。
2.  跟随(视频)课程。有偿的，因为那样会让你退学的时候后悔。我选择了 Udemy 上的[完全药剂和凤凰训练营。](https://www.udemy.com/the-complete-elixir-and-phoenix-bootcamp-and-tutorial/)
3.  自己尝试这种语言。我想在完成 Udemy 课程后这样做，并且已经有了一些我可以追求的想法

我想知道的是“函数式”和“面向对象”有什么不同。在网上快速搜索后，我了解到大多数资料是这样解释的:

在**面向对象编程**中，你编写*如何*做某事，而在**函数式编程**中，你编写*做什么*做什么。换句话说:面向对象编程需要你一步一步地写代码，才能达到目标。函数式编程要求你告诉编译器执行什么函数来达到目标。

看待差异的另一种方式是更仔细地检查两种范式的命名。面向对象编程允许您声明和改变映射真实世界的对象。函数式编程允许您声明具有输入和输出的函数。但是这些函数对外部世界和使用它们的环境一无所知。这意味着函数式编程，函数是纯粹的。同样的投入总会给你同样的产出。

另一种比较方式是在*状态*下。面向对象编程是有状态的，而函数式编程是无状态的。

让我们看一个例子来说明面向对象和函数式的区别。下面两段(伪)代码定义了一个学生，然后打印出他的名字。

## [](#objectoriented)面向对象

```
class Student 
{
  string Name;
  int Age;

  Student(string name, int age)
  {
    Name = name;
    Age = age;
  }

  PrintName()
  {
    Console.WriteLine(name);
  }
}

var student = new Student("Bob");
student.PrintName(); 
```

## [](#functional)功能性

```
module Student
{
  print_name(student)
  {
     IO.puts Enum.at(student, 1)
  }
}

student = ["Bob", 19] 
```

面向对象的代码将一个*学生*对象建模为现实世界中的对应物。此外，任何与学生相关的逻辑都是*学生*类的一部分。

每次调用`PrintName()`函数都会返回不同的结果。例如，首先初始化一个新的*学生*，他的名字是*鲍勃*。然后打印姓名。之后，您可以更改名称的值，然后再次打印该名称。当你调用同一个函数时，你会得到不同的结果。这说明`PrintName()`功能不纯。

对于相同的输入，`print_name()`函数的结果总是相同的。在这里，您没有使用对象的内部状态来打印名称。你实际上是把整个*学生*数组作为那个函数的输入。

还要注意，我们不能使用实际的对象来表示学生。所以我们使用一个数组来保存学生的姓名和年龄。然后，我们获取数组的第一个元素(我们知道它保存了名称)并打印它。每次我们给同一个*学生*作为`print_name()`函数的输入，它将返回相同的输出。换句话说，这个函数是纯函数。

这是面向对象编程和函数式编程之间差异的一个缩影。这些对我来说是功能领域的第一步。我希望这篇文章也能给你一个很好的初步概述。
