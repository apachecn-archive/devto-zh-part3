# 8base 如何工作—无服务器、GraphQL、CLI 等等！

> 原文:[https://dev . to/8 base/how-8 base-works server less-graph QL-CLI-and-more-4 ICF](https://dev.to/8base/how-8base-worksserverless-graphql-cli-and-more-4icf)

在过去的十年中，随着多层系统成为规范，后端和前端开发人员之间的分离已经出现。在这段时间里，后端开发人员专注于构建业务逻辑，并通过 REST APIs 向前端开发人员提供数据。与此同时，前端开发人员已经将注意力集中在用户界面上。他们对用户感同身受的能力使他们有独特的资格与业务人员互动，以满足他们的需求。

前端开发人员的标志是精通 JavaScript。2018 年 StackOverflow 开发者调查显示，几乎 70%的开发者精通 JavaScript，这使得它成为世界上使用最广泛的编程语言。

同一调查显示，JavaScript 在最受欢迎的编程语言中名列前茅，近 62%的人说他们喜欢它。TypeScript 是 JavaScript 的衍生物，得分更高，为 67%。

8base 是专门为拥有 JavaScript 和 TypeScript 编码技能的人设计的，他们需要这种超能力来构建端到端软件，而无需其他类型的专业人员来帮助大多数解决方案。

让我们通过一个典型的软件开发生命周期来说明 8base 如何让前端开发人员像全栈开发人员一样构建，像经验丰富的 DevOps pro 一样操作:

**10 种方式 8base 使前端开发人员能够像全栈开发人员一样进行构建**

**1。设计**
***工作在布局和流程上，而不是图纸上。***

用户界面设计者对于在 8base 上构建什么有完全的灵活性；然而，8base 提供了其 Boost UI 工具包，以帮助设计人员快速构建漂亮、可用和响应迅速的用户界面原型，这些用户界面具有通用 UI 组件，还可以智能地与 8base 后端进行对话。尽管设计人员和开发人员受益于使用 Boost，但他们可以不受限制地自由使用其他 UI 组件。通常，前端开发人员不需要使用 Boost 和其他 UI 工具包的设计人员的帮助，就可以构建美观实用的用户界面。

**2。架构**
***避免教条式的讨论，使用最佳实践并立即实现工作价值。***

8base 基于云计算中最现代的概念，提供了一个可扩展的最佳实践架构。前端开发人员可以放心，构建的应用受益于世界一流的架构，而不必涉及技术架构师。

**3。数据库基础设施的部署**
***数据库技术的选择不再像过去那样重要。推动商业价值更重要。***

8base 按需供应 Amazon Web Services (AWS)数据库，无需开发人员了解部署和配置 AWS 基础架构的所有复杂情况。开发人员永远不需要担心是否有足够的容量，因为 8base 会自动扩展数据库以满足其应用程序的需求。出于安全目的，数据经过完全加密，并保存在一个完全冗余、容错且经常备份的环境中。

**4。文件基础架构的部署**
***文件大多独立于结构化业务数据，当您可以轻松加入它们时，巨大的价值就会释放出来。***

8base 还按需提供 AWS 文件存储功能。这种基础设施也是安全的、不受限制的，并具有嵌入式智能，能够实现开箱即用的复杂文件操作，如图像大小调整、缩略图、光学字符识别、面部识别和全文搜索。

**5。部署计算和网络基础设施**
***难道计算和网络资源不应该像电力一样为用户所用吗？***

8base 按需提供 AWS 无服务器计算和网络功能，无需开发人员了解配置 API 网关、配置路由器、协调部署甚至设置 AWS 帐户的复杂性。实质上，开发人员可以利用无限的计算和网络能力来支持前端请求或运行后端功能。

**6。数据库和文件存储设计和定义**
***前端开发人员可以使用 8base 工具设计自己的数据模型。***

8base 提供其数据构建工具集，使前端开发人员(他们可能不具备复杂的数据架构或数据建模技能)能够轻松定义健壮的数据模型。数据模型包括存储在数据库中的结构化数据以及可以轻松连接到结构化数据的非结构化文件。

8base 还提供了本地数据查看和编辑功能，允许安全、快速地操作和测试数据。

7 .**。身份管理、基于角色的安全和用户管理**
***安全在 2019 年不是可选的–8 base 自动锁定数据，让开发人员无法忘记。***

身份管理、基于角色的安全性和用户管理通常是开发人员在每次编写应用程序时所做的多余工作。这种努力不仅浪费，而且容易暴露安全漏洞。

8base 提供完整的身份管理解决方案，支持三十多种不同的单点登录方法，如果需要，还包括开发人员自己的身份验证提供程序。

8base 提供了全面的基于角色的访问控制，可以自动封装数据模型中的每个表、字段和文件。开发者加速平台允许在构建的应用的用户界面中配置和管理功能级许可。

8base 通过允许配置 API 访问令牌来支持来自外部服务的经过身份验证的调用，从而支持与其他系统的安全集成。

最后，8base 为构建的应用程序提供了完整的角色和用户定义以及相关的管理屏幕。

**8。后端开发**
***我们真的需要给后端开发人员增加编写数据库查询的负担吗？***

我们设计了 8base，以便前端开发人员可以利用他们最喜欢的 IDE，并使用他们通常使用的语言 JavaScript 和 TypeScript 来创作服务器端函数。我们选择 JavaScript 是因为它是世界上最流行的编程语言，70%的开发人员都精通它。其他平台已经开发了自己的专有编程语言，这产生了大量的前期培训需求和大规模的持续供应商锁定。

8base 提供了强大的 CLI(命令行界面),允许开发人员编写自定义逻辑并在几秒钟内完成部署。服务器端逻辑立即可用，并可以通过 GraphQL 或 REST 公开，在响应数据事件时触发或按计划执行。解析器、触发器、webhooks 和调度任务都受支持。

逻辑执行完全无需服务器，根据需求实时创建和销毁计算资源。这意味着开发者和客户不必在流量低的时候为闲置的服务器付费，或者在流量高的时候增加更多的服务器。

**9。前端开发**
***允许前端开发人员完全控制流程。***

开发人员可以根据自己选择的技术构建定制的前端用户界面。8base 的 SDK(软件开发工具包)目前支持 React 和 React Native 然而，我们将很快发布对 Swift (iOS)、Android、Angular 和 Vue.js 的支持。开发人员可以利用 8base Boost 或其他 UI 工具包来构建完全响应的用户界面。

8base 提供了现有的最强大的 GraphQL API，允许前端开发人员直接在代码中创建他们需要的确切数据的查询，而不是依赖后端开发人员来完成。在单个查询中，开发人员可以从多个表甚至数据源取回数据，同时应用高级过滤、分页和全文搜索。前端开发人员可以利用流行的 ApolloClient 来使用 API 以及来自充满活力的 GraphQL 生态系统的其他开源库。8base 附带了一个强大的 API Explorer，允许开发人员使用 GraphQL 快速采样数据。我们相信 GraphQL 远远优于传统的 REST 或 SOAP APIs。8base 是同类平台中少数使用 GraphQL 的平台之一。

10。生产环境的持续管理
***如果系统只是随着需求的增长而运行和扩展，而没有临时保姆会怎样？***

大多数企业都配备团队来管理他们的存储和计算基础架构。无论基础架构是在内部还是在云中，都是如此。8base 运行和扩展以满足应用程序需求，而不需要企业为这些角色配备人员。

8base 是由开发者为开发者设计的。8base 旨在彻底改变企业构建和托管现代 web 应用程序的能力，尤其是那些满足大型企业需求的应用程序。

我们期待密切关注客户的需求，并快速响应他们的要求。
