# 当你坐下的时候，你永远不会站起来。

> 原文:[https://dev . to/reliefs/when-you-sit-down-you-never-get-up-4h 38](https://dev.to/reliefs/when-you-sit-down-you-will-never-get-up-4h38)

这是我从出生开始就见证的事情。当一个人坐在电脑前，在那里度过他的一生，如果他幸运的话，他会在 60 岁的时候起床。这不是我的故事，因为我花在电脑上的时间比我一生的时间都多。这就是了。

电脑是一个有趣的东西，因为你很容易成为它的奴隶。这就是大多数设备的操作系统是如何被编程的，所以在你退出之前，如果你没有意识到这一点，你就无法退出这个循环。那么，你如何得出结论，意识到自己陷入了困境..首先你要看看你正在做的任务。

每个输入的命令都需要时间，我可以把你一整天的工作编入一个命令中。所以你每天花 12 个小时做的工作我可以在几秒钟内完成。

那么这里的问题是什么呢？真正的问题是不理解，我们为什么要编程，计算机是如何计算的。编程就是把几个命令变成一个命令，看这都归结为{承担我做的所有任务} > 1 次点击。你如何到达目的地并不重要，重要的是只要点击一下，它就能完成你所有的任务。

我写这篇文章花了 20 分钟，这是浪费时间，我可以轻松地编写一篇完整的人工智能文章，你看不出区别，这个人工智能每毫秒创建一篇文章。

写文章不是我的工作，所以我还没有为此编写一个机器人，但是机器做工作，让机器工作，而不是你。

要理解如何看待计算机，一个人所需要的只是学习一门关于基础计算机科学的课程，但大多数程序员甚至不会这样做，我们的计算机如此之快，然而我们的操作系统、程序和网络开发却远不如 1900 年编写的代码。那么，为什么我们会有这个问题呢..如果我戴上我的开发者眼镜，这很容易，我们一层一层地放上去，大多数开发者从来不剥洋葱。因为他们不那样做，他们永远也不会理解计算机是如何工作的，真的就这么简单。

让我给你举个例子，我们有两个灯泡..开表示是，关表示否

```
10

yes > no

input(name)

if name is steffan > login

else do nothing.

A machine would read something like

if name is steffan > 1

else 0 
```

如果我在你家里有一个灯泡，在街上看，你会看到每次灯泡亮着的时候都是斯蒂芬，每次关着的时候都是别人。
