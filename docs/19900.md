# 编码并不适合除你之外的所有人

> 原文：<https://dev.to/michaelashefor/coding-is-not-for-everyone-but-you-e6j>

编码不容易，但完全值得

对于一个一直在从事与编码无关的工作，同时又试图编码的人来说，我必须说这真的很有挑战性，但却非常值得。

为什么要学编码？是因为这是下一份高薪工作吗？事实上，学习如何编码是完全可以的，因为你正试图获得高薪并享受生活，但我想让你知道，你会变得痛苦，并在看到下一个高薪工作时立即离开。

但是，如果这是一种激情，即使你几乎没有报酬，你也能看到自己在做这件事，并且乐在其中，那么，无论如何，要为一些非常具有挑战性的障碍做好准备，你的安慰将来自真正的快乐。编码本身是广泛的，不仅仅局限于网站开发。从移动应用开发到网页设计和开发(前端和后端)。你需要明确自己想学什么，从什么开始。什么都学没关系，但是你不需要一次学完所有的东西。你需要选择一条路并坚持走下去。这样你会更快地从挫折中解脱出来。

#### 下面是一些小技巧:

现在就开始:开始编码从来没有不好的时候，但是有一个正确的时间，这个时间就是现在，马上。活到老，学到老。只要你能投入时间和精力，没有什么是太难的。任何说编码太难的人都是在向你投射他/她的不安全感和不足。请拒绝他们！

设定你自己的速度:我想让你知道，你不是在和任何人竞争。是的，有目标和指标是可以的，但是根据你的进度设定现实的目标是很重要的。远离不健康的比赛。

有一个导师或推荐人:没有人是孤岛，每个人都需要帮助。让你周围的人都愿意提供帮助，永远不要害怕寻求帮助。其他人已经遇到了你现在面临的任何问题。自救，求助。

**学会依赖:**自己想办法解决问题。利用互联网、你的教程和任何你能做的来提高自己。这样你会成为一个更好的开发者和解决方案提供者。

持续练习:养成一种习惯，根据你的时间表，每天或定期练习你所学的东西。为你的练习设定一个时间表，并请坚持下去。如果没有，你会发现自己一遍又一遍地学习同样的东西

**学会休息:**总是找时间休息和做有趣的活动，因为在一天结束的时候，如果你崩溃了，你的编码之旅就到此为止了。
