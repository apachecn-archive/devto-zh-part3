# 犯错:我的第一个编程大错

> 原文:[https://dev . to/wuz/making-missons-my-first-big-programming-Lund-48 m2](https://dev.to/wuz/making-mistakes-my-first-big-programming-blunder-48m2)

我开始用一种有点非传统的方式编程。我妈妈是我家乡的图书管理员，所以在高中时，我开始在那里做学生听差。我对电脑很在行，并认为自己懂得比实际多得多。所以当我听说图书馆正在设计和建立一个新网站时，我想也许我能帮上忙。我已经在图书馆的计算机实验室工作了(从那里我学到了很多东西，也许还有未来的帖子)，所以学习和工作新网站是很自然的下一步。我主动提供帮助，我的老板礼貌地答应了。

我马上就开始了。这个网站是在 Wordpress 上建立的，使用一个已经存在的主题作为起点。我们有很多定制的 HTML 和插件一起工作来支持网站的大部分功能。可以说，“架构”看起来是这样的:

当不可避免的事情发生时，我已经在工地上工作了大约一个月。我修改了一个插件，但是网站*不喜欢它。有东西坏了，我很快意识到知道足够多的东西坏掉和知道足够多的东西修好之间的区别。*

我打破了一切。

我开始思考我接下来的步骤。我们没有使用版本控制——我不认为任何在网站上工作的人知道那是什么。我们通过 FTP 上传到“测试版”网站的服务器。没有办法恢复旧版本，甚至无法恢复我所做的更改。我禁用了插件，删除了代码，尝试了一切我能想到的方法来回到工作状态。似乎什么都不管用。

在短暂的恐慌之后，我知道这最终是我的责任。我决定带着一个新计划去找我的老板:

1.  解释发生了什么事
2.  希望我没有被解雇/从网站项目中移除
3.  承诺建立一个新的网站

幸运的是，我的老板很棒——她明白这对我来说是一个很好的学习机会。她允许我继续做这个项目，所以我从头开始。我从错误中吸取了教训。我开始阅读关于创建 Wordpress 主题的内容，而不是从一个现有的主题开始这个项目。我不再那么依赖插件来实现功能。通过这份工作，我获得了很多关于编程的知识，并最终达到了我今天的位置——成为一名全职软件工程师。

我仍然会犯错误(尽管通常它们的破坏性较小)。图书馆还是用我建的网站。我有一个很棒的故事要与新开发人员分享。

所以请记住:
新发展——你会犯错误。努力使他们和解。

雇主/经理/上司——错误正在发生。你可以通过生气和解雇员工来回应，或者你可以把一个错误变成一个可教的时刻，成为你团队中一个新工程师的所有权和骄傲。

你编程最大的错误是什么？你是如何回应的？发生了什么事？
