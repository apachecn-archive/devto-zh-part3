# 我的第一个栏杆脚手架

> 原文：<https://dev.to/saral/my-first-rails-scaffold-3gfn>

上周我完成了我的第一个应用程序。这个应用基本上是我的个人博客。我希望在未来的日子里保持和增加它的功能。不过今天，我开始了一个新项目——待办事项清单。我为这个项目做的第一件事就是`rails g scaffold todo_list title:string description:text`。有了它，我第一次使用了脚手架。不过有一件事我想做，我真的想了解脚手架是如何工作的。

以下是我对脚手架工作原理的理解。使用上面的命令创建新的脚手架后，它为我做了几件事。

1.  它在 config 文件夹的`routes.rb`文件中为我生成了路线。在我的编辑器中查看`routes.rb`文件，我可以在文件上看到`resources :todo_list`。

2.  它生成了一个 TodoList 模型，在通过“rails db:migrate”迁移数据库之后，数据库就可以使用了。该模型有一个标题和描述，因为我在创建脚手架时已经设置了这两个参数。

3.  Scaffold 还为 CRUD 方法生成了所有控制器。这意味着从一开始我就能够执行条目的创建、读取、更新和删除。

4.  最后，所有的视图都生成了，视图有一个带有标题和描述的表单。此外，由于控制器已经定义了 CRUD 方法，我能够创建、删除、编辑和查看我的条目。

就这么简单。我上周的全部工作用一句话来概括。😵😵😵😵

尽管如此，上周对我来说是一次从零开始构建一切的伟大学习经历，本周当我深入研究 scaffold 生成的代码时，我将真正读懂代码的字里行间并理解它的作用。
