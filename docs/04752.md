# 你和你的工作都不需要完美

> 原文:[https://dev . to/ashleemboyer/not-you-not-you-work-has-to-10p 4](https://dev.to/ashleemboyer/neither-you-nor-your-work-has-to-be-perfect-10p4)

*这篇文章也在[我的个人博客](https://ashleemboyer.com/neither-you-nor-your-work-has-to-be-perfect)上结束了。*

让我们面对现实:你做的任何东西都不会是绝对完美的。有些人可能不喜欢最终产品。也许你可以从你的博客文章中删除一两个段落或者改写几个句子。您的代码可能会在几个不同的地方进行重构。如果你擦掉几处污迹，那幅画可能看起来更干净。

我们都知道一旦我们开始掉进这些兔子洞，项目会发生什么。“我会花几分钟来解决这个小问题，”如果你不小心的话，可能会变成几个小时或更长时间。根据我对代码的经验，这可能需要更长的时间，因为当你试图清理一些函数时，你可能会意外地破坏一些完全不同的东西。

显然，凡事都有时间和地点。项目让你经历不同阶段的压力。有时候，你没有机会把代码做得像你想要的那么漂亮。功能性几乎总是更重要，因为你的客户可能永远不会看到你的代码。他们所关心的是他们的产品运行良好。所以，到了打孔的时候，尽量不要太担心代码是否完美。

相反，快速记下代码哪里需要帮助，以及如何做才能让它变得更好。希望你以后可以回去进行改进，但是如果不是这样，你可以把这些笔记传给下一个人，或者用它们来解释你从项目经历中学到的东西。人们期望你学到的东西比什么都多。他们会重视你分享的经验。
