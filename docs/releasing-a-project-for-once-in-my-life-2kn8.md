# 在我的生命中发布一个项目！

> 原文：<https://dev.to/jake0lacey/releasing-a-project-for-once-in-my-life-2kn8>

### 为什么这对我这么重要？

我已经发展了大约 7 年，在我职业生涯的开始，我会有非常简单的想法，非原创的想法，如博客引擎或在线商店。我也不知道如何“正确地”部署应用程序。所以在我的天真无邪中，我能够相对快速地做出一些东西，并向全世界发布。

自我意识到这一点，所以我职业生涯的最初目标之一是理解基本原理，并学习如何“正确地”做事。

这有一个缺点...我只是不再发布消息了！我有想法，几乎每个月都有一个，但由于这样或那样的原因，这个想法会失去动力和死亡。老实说，我的 Github 帐户看起来像 28 天后的教堂场景。我觉得为了让它足够强大，我需要做的事情太多了，所以坦白地说，我放弃了。

* * *

所以在新年到来之前，我承诺在假期发布一个我正在做的项目！[https://twitter.com/Jake0Lacey/status/1079126414610309120](https://twitter.com/Jake0Lacey/status/1079126414610309120)

* * *

我想分享我的经历，以及我所做的使这项努力与众不同的事情，并希望能帮助其他人。

接下来我会把它分成几个不同的部分。

*   我在为自己建造这个
*   设计和范围
*   技术方法
*   有责任
*   做得比完美更好

### 我在为自己建造这个

这是一个很大的动力，以前的项目在技术上很有趣，但我不是目标观众，所以他们会死，因为我没有人和他们一起玩。如果你是为自己制作应用程序，这真的很有帮助，它会促使你找到使用它的方法。

### 设计和范围

因为我知道我想使用它，我需要找到 MVP，这是一个足够好的想法，所以我可以在第一天开始使用它！

我所做的是坐下来设计我想要的样子，然后去掉一些特性，这样我就只需要担心一个特性，它必须是一个很好的基础，可以用来构建和迭代其他所有东西。

这让我能够专注于我必须构建的东西，不多也不少。

### 技术手段

我的方法非常激进，如果你在练习一项技术或技能，我不建议这样做，但如果你想快速获得释放，那就去做吧。

我想扩展一下我所说的积极的意思，我的意思是技术部门，重复，差的性能&不欢迎单元/集成测试。为什么？因为这种东西只有在想法成功并且人们使用它的时候才重要，然后我就可以转移我的注意力了。

这也不意味着我只是假设事情工作，我做了测试哈哈。

### 问责

以前的想法当我想发布一些东西时，我会写在笔记本上，并在我完成 x，y 和 z 功能的日期上。然而，我发现这真的没有任何意义，因为日期可以很容易地移动。

所以为了确保你有责任感，我会说你会和别人分享，我在 Twitter 上分享了我的，并去了一些积极的支持，这让我觉得有决心完成工作！

### 做得比完美更好

我喜欢发布一些看起来像我想象的那样或者 100%的时间工作的东西，没有性能问题，有 100%的测试覆盖率，有 CI/CD，自动缩放，你明白这一点。

完美是一个方向，而不是终点。

能够重复一个已经发布的想法，比花几个月的时间在一个想法上，并不断问自己它是否准备好了，感觉要好得多。
