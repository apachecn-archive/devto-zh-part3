# 作为开发人员学习设计

> 原文:[https://dev.to/ukutaht/learning-design-as-a-developer-155h](https://dev.to/ukutaht/learning-design-as-a-developer-155h)

我经常发现自己在做一些自己都不知道该怎么做的事情。我在市场营销、文案，尤其是 UX/UI 设计方面大费周章。作为一个一生中没有设计过任何东西的缺乏想象力的开发人员，我发现下面的一些技巧对尝试设计网页很有帮助。

### [](#1-read-dont-make-me-think)1。阅读[不要让我思考](https://www.amazon.co.uk/Dont-Make-Think-Revisited-Usability/dp/0321965515)

这本书基本上奠定了网页设计的常识基础。这不是一本新书，但它是一本经典著作是有原因的。这里列出的原则可以直接应用到你为网页设计的任何东西上。推荐给任何想从事设计的开发人员。

### [](#2-copy-designs-you-like)2。复制你喜欢的设计

创造力的秘密在于知道如何隐藏你的来源。假设我需要在我的登录页面上添加一个价格部分。我*可以*盯着一张空白的画布看一整个下午，等待创意来袭击我，或者我可以在[运球](https://dribbble.com/)上搜索“*定价页面*，立即看到数百个定价页面。这几乎适用于任何东西，除非你正在设计的东西确实是独一无二的，这是非常罕见的。

### [](#3-start-without-code)3。无代码启动

一旦你有了几个你喜欢的例子，就开始画吧。我更喜欢从纸上或者素描开始。有些人喜欢马上钻研代码，但是我发现这种方法倾向于将我锁定在我想到的第一件事情上。因为代码很难修改，所以我不会像在草图或纸上那样摆弄它。在纸上，你可以很快想出 5 个不同的选项，给别人看，然后开始排除。用代码很难做到这一点。

### [](#4-lose-your-ego-and-get-some-feedback)4。丢掉自我，获得一些反馈

我发现很难和别人分享我的设计作品。我觉得我的工作会受到评判，我的技能不足会暴露出来。我相信大多数制造商都经历过这些感觉，不幸的是，正如你可能已经知道的，没有治愈它们的方法。我告诉自己，我只是一个初学者，我应该没有任何技能，以此来控制自己的情绪。这就是为什么我要求反馈:变得更好。反馈会暴露你的缺点，但是不管有没有人指出，这些缺点都会存在。

### [](#5-read-refactoring-ui)5。阅读[重构 UI](https://refactoringui.com/book/)

这本书我才看了一半，却学到了这么多！对于想要提高设计技能的开发人员来说，这是多么令人惊奇的资源啊。这本书包括了无数关于 UI 原理的具体技巧。它将帮助你开始创建设计系统，并大大加快你的速度。我会在你对设计有所练习后读它。毕竟，这是关于重构 UI 而不是从头开始创建 UI。

### [](#start-designing)开始设计！

我最后的建议是去做。一开始你可能会很糟糕，但是每个初学者都是如此。当一个开发人员说他们在设计方面很差劲时，他们通常认为他们缺乏设计的天赋。这就像说你不会弹钢琴，因为你天生就没有这种技能。太荒谬了。不是每个人都能成为莫扎特，但任何人都可以学会在钢琴上演奏更简单的歌曲，并让它听起来相对不错。这就是我试图用我的设计技巧达到的目标。我不会赢得设计奖。我不会靠设计谋生。我只想做得足够好来度过难关。
