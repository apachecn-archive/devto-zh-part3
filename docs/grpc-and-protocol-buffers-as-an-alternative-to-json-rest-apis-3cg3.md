# gRPC 和协议缓冲区作为 JSON REST APIs 的替代

> 原文：<https://dev.to/franzejr/grpc-and-protocol-buffers-as-an-alternative-to-json-rest-apis-3cg3>

gRPC 是一个开源的远程过程调用框架，协议缓冲区是一种序列化结构化数据的机制。两者都是由 Google 开发的，并在他们的内部和公共 API 中使用。思科和网飞等其他大公司已经从这项关键任务应用技术中受益。

在这篇文章中，我们将学习 gRPC 和 Protobuf 的核心特性，并与 JSON REST APIs 进行比较。
