# 为什么做一个好的程序员几乎完全不需要编程。

> 原文：<https://dev.to/chris_bertrand/why-being-a-good-programmer-barely-requires-programming-at-all-3ngf>

是的，你没看错。要成为一名优秀的程序员，你不需要擅长编程。

我知道你在想什么，那毫无意义！等一下，听我说完。

当你想到一个你认识的优秀程序员时，你脑海中浮现的第一个特征是什么？坚定，聪明，平易近人？冷静，安静，还是认真？也许是别的什么，但我敢打赌不是…一个真正优秀的程序员。这是为什么呢？

程序员在日常工作中需要很多技能。这种训练不再是坐在黑暗的地下室里独自研究代码。事实上，人际交往能力是一个优秀程序员应该具备的最重要的能力之一。敏捷方法需要不同部门和个人之间的大量沟通。你将被要求每天至少检查一次，可以进行计划、改进、追溯；不断交流奋斗、改进和进展顺利的事情。

过去那种预先获得任务和规格，然后任由自己处理的日子已经一去不复返了。你可能会和你的用户接触，问一些关于你的进展的困难问题，并被要求把信息反馈给你的产品负责人/经理。

你需要与你的上司和团队讨论你的设计，因为你很可能不会独自完成这项工作。测试人员会要求更新，质疑你的单元测试是否可信和充分，你的问题跟踪器需要不断更新，让所有人都能看到讨论。

事实上，拥有心理学学位可能比计算机科学学位更有助于你理解如何与这些不同的人打交道。

所以，是的，编程在某些时候是需要的，但是一般来说，我们 90%的人将要做的事情不需要任何革命性的编码专业知识。

在大多数情况下，项目失败的原因是因为没有对需求进行适当的讨论和细化，或者团队没有就对他们的要求进行适当的沟通。很少是因为你不是一个足够好的程序员。
