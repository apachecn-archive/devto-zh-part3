 **## 本地和混合应用

本地应用程序是根据操作系统的指导原则使用所需语言开发的移动应用程序。举个例子，当你在 Android 上用 Java 写代码或者在 iOS 上用 Swift 或者 Objective-C，那么这个应用就是一个原生 app。

因为原生应用程序是明确为操作系统而构建的，所以速度更快。原生移动应用也能提供顶级的用户体验。React 原生框架用于创建原生应用。

另一方面，混合应用是通过 WebView 固定在原生移动应用中的网站。它们是使用可以在不同平台上运行的 CSS、JavaScript 和 HTML 等 web 技术开发的。

这些应用看起来像本地应用，但具有网站的特征。混合应用程序利用 Cordova 和 PhoneGap 工具来访问 iOS、Android 等平台上的功能。Ionic 框架用于开发混合移动应用。

## 反应型天然和离子型骨架的比较

**技术/语言堆栈**

这些框架基于不同的技术栈方法。React Native 采用“学习一个，到处跑”的方法，而 Ionic 采用 web 技术来开发混合应用程序。

美国顶级应用开发者利用 React Native 在平台的限制和指导下开发原生应用。该框架使用 UI 原生组件来帮助移动应用开发公司为用户提供无缝的应用。

另一方面，Ionic 框架使用最少的代码库来开发应用程序，并使用 Angular 来平滑和加速这个过程。一个有角度知识的移动应用程序开发公司可以方便地构建用户友好的应用程序。

**性能**

在性能方面，React Native 是一个用于大型项目的更稳定的框架。这个框架产生了成绩优异的应用程序。另一方面，Ionic 框架为与原生应用功能无关的项目提供了更快的应用开发环境。

当涉及本机代码时，Ionic 框架可能会有性能滞后。使用 Ionic 框架的 [**移动应用程序开发公司**](https://dedicateddevelopers.com/mobile-application-development/) 必须下载不同的插件来访问原生功能。

**插件和社区**

React Native 背后的社区支持非常强大——这个框架也有不同的插件可用。React 本地社区总是互相帮助解决他们可能遇到的任何问题。

有一个成熟的社区支持 Ionic 框架并解决任何问题。然而，这个框架缺少大量的插件来获得原生特性——只有 PhoneGap 和 Cordova 插件在使用。这意味着使用这些框架的移动应用程序开发公司可以加入这些社区进行无缝应用程序开发。

**支持的平台**

这两个跨平台支持 iOS 和 Android。此外，Windows 通用平台支持 Ionic 2。这意味着一家拥有 Ionic 2 的网络和移动开发公司可以为 Windows 手机开发移动应用。

**设计**

一家 [**网络和移动开发公司**](https://dedicateddevelopers.com) 必须利用一个预先开发的组件来设计使用 Ionic 框架的应用程序。React Native framework 中没有设计组件，这使得它对于关注应用设计的人来说不是一个好的选择。

## 包扎

没有人能告诉你这些框架中哪一个更好，因为有不同的因素供你在做决定前考虑。在选择最佳框架时，平台、团队规模、成本等因素起着巨大的作用。根据应用程序的用途，选择要使用的框架是一项相对简单的任务。
