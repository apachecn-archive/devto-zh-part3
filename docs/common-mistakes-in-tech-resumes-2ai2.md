# 科技简历中的常见错误

> 原文：<https://dev.to/kaydacode/common-mistakes-in-tech-resumes-2ai2>

最近，我们在互联网上发布了一篇简历评论[开发人员发布](https://dev.to/kaydacode/resume-review-1oei)和[推文](https://twitter.com/kaydacode/status/1072635945357336581?s=21)。好吧——我们没有断网，但我的通知肯定炸了。总的来说，看着社区互相帮助是一件令人惊奇的事情。我在假期前复习了我所能忍受的，并将继续复习，试图抓住任何漏掉的部分。

在我审阅过的许多评论和我从其他评论者那里看到的评论中，我整理了一份最常见错误的列表。

## 没有准备好的 PDF 简历

招聘人员和经理会通过电子邮件和纸质文件来传递你的简历。用打印机打印一下，确保黑白效果不错，而且没有任何地方被剪掉。

## 仅仅依靠一个网站

尽管拥有一个作品集网站或完整的 LinkedIn 个人资料非常有用，但准备好一份可以发送电子邮件或打印的文档更为重要。通过这个练习，我看到了招聘人员和经理在招聘一个职位时都经历了什么，这并不美好。我不想点击你网站的 5 个页面来获得你的体验，我不想浏览你的 GitHub 项目来看看你知道什么语言，我也肯定没有心情看你的 10 秒加载动画。你完全可以(也应该)在简历上提供你的网站/ GitHub 的网址，这样别人就可以更深入地了解你的经历。尽管:你在简历中包含或链接的任何内容都必须让雇主看到。作为一名招聘经理，如果你去了一个破损或基本不完整的网站，你会怎么想？一位评论者甚至提到，他在查看投资组合时会加载 JS 控制台来查找 JavaScript 错误！偷偷摸摸，偷偷摸摸；)

## 每 5 年(但最好是 10 年)一页

简历旨在快速回顾你的技能、经验和教育程度。它不应该是对你过去 20 年生活的深入探究。这里有一些减少页数的小窍门:

*   仅列出相关经验/项目
*   使用要点，而不是完整的句子..记住容易浏览
*   把你的简历放在你想要的工作上，而不是你现有的工作上
*   尝试不同的布局，有助于把你的简历想象成一个网站。你将如何优化一个空白页面来适应所有内容而不需要滚动？

## 或者，利用半页以上

如果你是这个领域的新手，很难知道该写什么。但是，这也给了你很大的空间来吹嘘你做了多么酷的东西。你上过什么课？你建过哪些团体项目？你使用过什么语言，或者目前正在学习什么语言？你熟悉哪些技术？饭桶？SVN？npm？所有的缩写？！这是你突出你的成就、你正在学习的东西和你感兴趣的东西的机会。在科技领域，你最大的技能是能够学习新事物。显摆一下！

## 只列出你觉得舒服的事情

你简历上列出的任何事情都是面试中被问到的话题。例如:如果你在简历中列出了 JavaScript，因为你为一个也使用 JavaScript 的网站创建了 CSS 和 HTML(但你没有接触 JS)，我要求你解释一下在 JS 中定义一个常数，而你不能，我会认为你的其他技能也是一个谎言。话虽如此，尽管我一度对 Java 非常熟悉，但现在情况不同了，你也不会在我的简历中找到它。首先，不要撒谎或夸大你的技能，为你的成功做好准备。
