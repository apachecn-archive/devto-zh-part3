# 作为设计者的设备

> 原文:[https://dev.to/metamn/devices-as-designers-1og7](https://dev.to/metamn/devices-as-designers-1og7)

当我读到设计的黄金时代即将到来时，我知道有些事情不对劲。我总是意识到即将到来的重大事件的后果。它首先来自《纽约时报》,然后来自各个地方。

IBM 正在招聘数千名设计师。公司都采用设计思维。创业公司是由设计师创立的。银行和品牌正在购买整个设计工作室。凯鹏华盈报道和奥赖利雷达仅仅是#设计。

一切都围绕着设计，但一切看起来和工作都是一样的。也许我们并不真正理解什么是设计。

## [](#frank-chimero-david-hockney-zaha-hadid)法兰克奇美罗，大卫·霍克尼，扎哈·哈迪德

2015 年，我正在阅读弗兰克·奇梅罗的《网络的纹理》。

首先我不明白。这是一篇很长的文章——一篇关于网页设计有多糟糕以及如何修复的优雅的咆哮——我当时心情很愤怒。我刚刚建立了我的第一个，[孤独星球的 Rizzo](https://rizzo.lonelyplanet.com/styleguide/design-elements/colours) 样的生活方式指南，它迷失在噪音中。

尽管这是 arms 的壮举，但没有人关心——由一个人而不是一个团队完成，从零开始重新思考，而不是从插件和生成器混合在一起，开源并可供其他前端开发人员使用。

意识到样式指南是一个简单的炒作——我在 Twitter 上看到甚至德勤也提供样式指南的广告，现在都没有了——令人失望。

今年，我又读了弗兰克的文章，了解了大卫·霍克尼从独立的部分创造艺术的方法，并得到了它。很明显，(网络)设计的未来是拥抱未知，在不断变化的画布上创造艺术，从手表大小到摩天大楼。

我立即开始了一个新项目，一个博客，来更好地分析和理解这项技术。我正在尝试针对同一内容库的不同受众。Hockney 风格，从一个数据源组装同一主题的两个视图。

然后扎哈·哈迪德死了。像往常一样，在这种情况下，我试图通过回顾艺术家的工作和生活来表达我的敬意。我被这位小女士如何建造一些不可思议的美丽，如此不真实的地方所吸引，人们似乎羞于堆积起来。

当你阅读爱丽丝·罗斯托恩的文章时，你会发现哈迪德的建筑对每一位游客来说是多么的特别:

> 您可以选择自己的方式来浏览空间及其内容

...heureka 立刻打动了我。这种体验是个人的，而不是集体的。

## [](#devices-as-designers)作为设计者的设备

> 设备选择自己的方式来组合空间和内容

设备在设计师的帮助下创造体验。设备选择如何将组成设计的组件放在一起。我们的任务是设计单独的内容块，为它们配备元数据——添加语义——然后每个设备将根据其独特的硬件和软件特性以及最佳知识组装页面和网站。

听起来人工智能，听起来很疯狂，但很简单。这完全是关于语义，一点历史和思想的彻底改变。

## [](#semantics)语义

我第一次听说这个概念是在高中的第一堂计算机科学课上。我们被告知代码有两个同等重要的主要特征:语法和语义。语法似乎是自然的，语义让我暗自高兴。我立刻想到，如果代码有语义，那么它可以成为我的朋友。我笑了。

从那以后，糟糕的语义被遗忘了，直到语义 HTML5 再次出现。我早些时候也看过几次，但是被遮住了，我认不出来。想想 CORBA。UML。YAML。

语义 HTML 是天赐之物。为网站写代码减少了一半的认知负荷。从那以后，我写了真正的语义 HTML，它在 T2 HTML 5 的大纲视图中表现得很好。我很开心。

在我的博客上工作，思考如何设计和开发一个完美的网站——我发现一切都围绕着语义。

内容被转换成对设计者更有用的数据和元数据；它配备了对机器人有意义的微数据；它被组织成一个(BEM)文件夹结构，任何开发人员都可以访问；它被组件化，以获得意义，更有思想。甚至[动画](http://alistapart.com/article/motion-with-meaning-semantic-animation-in-interface-design)现在也有语义了。

我们已经开始为我们所做的和工作的每件事赋予意义，因为我们已经意识到我们的工作，为网络设计和开发是非常艰巨的。我们可能需要帮助来避免网络落到我们头上。

让我们的工作变得更简单，让更多的人快乐。尤其是那些抱怨 99%的网站看起来都一样的人。让我们来玩一下创建网站的想法，网站就像扎哈·哈迪德的建筑和大卫·霍克尼的绘画一样，是自我组装的。
现在我们有手表、平板电脑、笔记本电脑、台式机、电视、投影仪屏幕，但未来我们会有成千上万的屏幕，这是任何媒体都无法理解的。

在某个阶段，我们将需要人工智能在我们的网页设计和开发过程中。不知道*什么时候* AI 能取代我们；我认为我们可以暂时保持我们的创新优势。如果不是在所有领域，至少在告诉机器我们的内容和我们想要完成的目标方面。向内容添加语义，根据我们的最大知识范围来塑造内容组件，并让机器为不同的人、不同的场景和用途组装整体。

## [](#history-unite)历史团结起来！

让我们的内容在状态的叠加中既死又活。让我们专注于描述和设计所有这些状态，并让机器为我们在工作中无法意识到的观察者舰队组成一幅画面。

让我们再次回顾，并*调和*几千年前的进步和认知意味着什么，以及现在意味着什么。让我们思考一下*如何看待*的作品，看待事物的作品，关于理解的过程。

[![Observer - Observation - Object](../Images/688d62fe4d346c3c16daa783755ea325.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--JU46T1Ly--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/b38r49mrsjagabbvn3ga.png)

我们都知道有一个观察者在观察一个物体。观察者能看到什么取决于他的观察能力和物体的属性。在历史上，有两种完全不同的方法来增进理解和定义进步。几千年前，当观察者将其观察能力(视觉、听觉、嗅觉、触觉)降低到零，并与物体融为一体时，这就是内卷化——向内的过程。

从那以后，我们有了进化，我们发展了我们的观察能力——向外发展——像物联网一样发展到一个极端的程度。

你看，这都是关于观察；随着内卷化，我们有了[赫尔墨斯三重神](https://www.goodreads.com/author/quotes/272885.Hermes_Trismegistus)成为神；随着进化，我们正在构建我们自己的云，我们自己的神就住在那里。

既然设计是观察，那就让我们看看如何在这个特殊情况下，在我们称为网络的可爱宇宙中参与/进化。

## [](#supercontent)超能

为了切入，让我们用所有的状态装备所有的内容，让它以叠加的方式静止，在那里它什么也不显示，等待一个观察者(访问者)和它的观察技能集(设备)显示它自己。

每个访问者将从我们的内容中看到什么取决于它的心理能力(好奇心)和设备能力(背景)。我们可以信任他们的能力——这些与我们无关。

通过这一步，我们让访问者和设备(人工智能)来决定他们是否想发展或参与我们的内容。这不再是一个哲学问题，而是个人意愿的问题。在这样一个互联时代，没有人愿意负责删除这些重要的观察方法之一。

留给我们，设计师和开发人员的是专注于内容状态，编写人工智能算法，使设备变得如此智能，能够从任何内容和状态组装任何图片。

## [](#what-good-is-it)有什么好处？

当前的设计/设计思维专注于观察，而忽略了观察者和对象。

你想知道新闻吗？整个西方媒体是一根管子。无论你需要什么，你都会得到同样的东西。你必须努力寻找新的来源和灵感。阅读 rt.com，即使它很有趣，但至少有不同的观点。或者获取*地下*博客上关于中国用户界面的文章。

你反对，创造性的东西？你迷失在噪音中，变得麻木。现在算法决定你值不值得看。

看到转折了吗？算法不是让你发现，而是强迫你看到同样的东西。

让我们将整个观察过程民主化。让人们以自己的方式而不是通过镜头的圣杯来看待和使用事物。我们设计师必须展示物品的所有能力，观察者不能忍受我们认为是最佳的东西。让我们不要像上帝一样行事，而是以一种解放的方式使用我们的力量。

## [](#an-example)举个例子

把内容想象成代码。配有句法和语义。由可重复使用、可组合的微小零件自下而上构建而成。存储在一个全局存储库中，在那里它可以被访问和分叉；可搜索到最小的细节；哪里有信用，哪里就有信用；即使是巨人也被追究责任。

这值得吗？当然，以[无法想象的方式](http://www.theverge.com/2016/3/30/11331014/microsoft-windows-linux-ubuntu-bash)。代码和内容想要[免费](http://kk.org/books/what-technology-wants/)。

现在来看看[一个搜索看起来如何](https://www.google.ro/search?q=content+wants+to+be+free)当内容像我们今天知道的那样经典，以及[另一个搜索](https://github.com/search?q=code+is+free)内容仍然不被视为代码，但搜索引擎是为代码设计的。

从谷歌我们没有得到任何语义只是一个简单的文本列表在屏幕上。

从 Github 我们得到了大量有趣的信息。“存储库”标签列出了谁(公司等。)对这个搜索短语最感兴趣，“代码”标签列出了可以找到这个内容块的确切位置(想想文章段落)，“问题”显示了人们对它不清楚的地方。你可以看到这个项目最后更新的时间，花了多少时间创建它，有多少人重复使用和喜欢它，谁是作者和贡献者。

现在你可以想象把注意力集中在物体上是什么样子了。

还有更多。我们没有谈论我们最喜欢的主题——用户界面和体验设计。英雄在哪里？我们应该在某个地方增加一份汉堡菜单吗？我们可以在“问题”上使用视差滚动吗？

那些被“设备是设计者”弄得恼火的人，或者问留下了什么样的创意，或者思考他们职业的消失:别担心。

担心创造力的方式是愚蠢的。让我们给创意更多的来源，更多不同的受众，更多的空间，更大的发挥空间。其他的也会随之而来。
