# 截至 2018 年我不知道的事情

> 原文:[https://dev . to/craignicol/things-I-don-know-as-of-2018-4f 25](https://dev.to/craignicol/things-i-dont-know-as-of-2018-4f25)

去年年底，丹·阿布拉莫夫谈到了他不知道的事情。这是一个有用的反思框架，作为一个有特权的、有经验的开发人员，我很乐意谈论这一点，希望它可以帮助其他人理解有经验的开发人员并不是什么都知道。

我会重复他关于冒名顶替综合症和 Dunning-Kruger 的说法，以及看到我认识的或我追随的真正聪明的开发人员不得不不断证明他们的资格，仅仅因为他们不是 cis，异性恋，健全的白人男子，这是多么糟糕。如果我处在支持你或支持你的位置，我会尽力这样做。你的经历绝对是我不知道的事情，但是我可以听。

在我的 2018 年回顾文章中，我讲述了一些我足够了解的事情。我不得不关心 DevOps、容器和管道，因为它们解决了我的团队面临的实际问题。我喜欢图形，因为三角学和分形是我进入编程的原因，但我在网络工作，研究架构、安全、接口和数据。这就留下了一个广阔的世界，这还不包括未知的未知。

### [](#ruby)红宝石

我已经尝试过几次，但对我来说它看起来还是老派的 Perl。大量标点符号偶尔被领域语言打断。这对我来说听起来是个好主意——几乎完全分离了关注点——但是我一直觉得学习曲线太陡了，所以我退回到 Python 和 C#。

### [](#css)CSS

从 Mosaic 还是浏览器的时候我就开始建网站了。我可以用 CSS 设计东西。在绿色背景上放置红色文本(你这个怪物)，或者在屏幕顶部固定一个导航栏并不难，但是看看那些花哨的网格布局，或者视差滚动效果，或者 CSS 图标，我不知道从哪里开始。我不知道如何对它进行单元测试，所以我的工具在这里不起作用。还好我和一些非常聪明的 CSS 开发人员一起工作。

### [](#markdown)降价销售

我经常使用降价。对于 README，对于[架构决策](https://github.com/joelparkerhenderson/architecture_decision_record)，对于起草博客文章，但是我总是需要谷歌如何插入一个链接，或者一个表格，或者一张图片。每次我都觉得自己像个白痴。

### [](#gulp-npm-and-webpack)gulp、npm 和 webpack

我知道这些是我构建 React 时必不可少的工具。我知道他们把我的代码转换成浏览器运行的东西。我知道依赖性管理是如何工作的([如果你喜欢](https://groups.google.com/forum/#!forum/horn-development)，我可以在喝饮料的时候让你厌烦，但是我只使用过这些预先配置在模板或现有代码库中的 JavaScript 工具，所以我不能从头开始创建 JS 或 TypeScript 构建管道，或者修复一个坏的管道。

我真的无法告诉你任何关于这些如何与 Grunt、Yarn、bower、sass、scss 或其他任何东西一起融入生态系统的智能信息。只要我能写一个脚本来安装和运行它们，我就很高兴不必深究它们。我所关心的是它是否能运行，以及依赖关系是否安全。

不过，我想今年我需要学习更多关于这些工具的知识。

## [](#you-dont-need-to-know-everything)你不需要什么都知道

你会下定决心去学习更多，不管是每周还是每年。要学的东西比你有时间学的要多，这就是为什么我们要按术语来工作。选择一个接下来要学的东西，因为它对你很重要，因为它很有趣，因为你的项目需要它，或者因为它是赚钱的地方。忘记其余的，直到你越过了邓宁-克鲁格曲线，你对这个新事物有了足够的了解，可以看到所有可能的下一步要学的东西。
