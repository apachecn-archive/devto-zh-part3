# 游戏开发很难

> 原文:[https://dev.to/balconygames/game-development-is-hard-1bpc](https://dev.to/balconygames/game-development-is-hard-1bpc)

在过去的几周里，我和我的朋友们在空闲时间一起从头开始制作我们的游戏，我们在这方面没有经验。让我来描述一下它是如何开始的。

我和我的朋友呆在漂亮的酒吧里，喝着冰镇葡萄酒。我们在谈论 30 岁以后的生活:)，以及我们在游戏机上玩游戏的时候有多棒。这些天我们没有足够的时间玩。我是后端开发人员，他是会计软件的开发人员(但开始学习 javascript 并找到了第一份在前端领域工作的工作)。我讲述了我在学生时代学习 Macromedia Flash 制作小游戏的故事。

然后我朋友说(伊戈尔):

*   你知道我有一个朋友(维塔利)在杂货店工作，但他也想制作游戏。

它开始了。3 个人来自不同的地区。

我们决定使用 Unity，因为它看起来是目前构建游戏最快的方式。

我开始关注后端部分和 Unity 的 HTTP 客户端来与服务器对话。

1.  这个游戏的创意是什么？好吧，...这个想法是关于...，与朋友交谈，从教程中获取任何内容并构建它。看起来不错，让我们继续制造产品。
2.  从哪里获得资产？好吧，...这很难，但在 Unity store 中搜索资产。我们有黑色星期五价格，45 美元，我们有用户界面资产。
3.  从哪里获得玩家资产？好吧，...谷歌它，发现资产，与所有者交谈，获得批准和更新游戏中弹出的信息。完成了。
4.  测试测试吧。
5.  怎么挣钱？我们可以赚取，我们增加了应用内购买和观看广告来继续，但它没有破坏游戏性。没有它你仍然可以生活，它不会影响游戏。
6.  获取反馈。我们使用陀螺仪控制。看起来我们需要改变它。
7.  目前，它在 google play 市场 [Sweety Jumper - Android](https://play.google.com/store/apps/details?id=com.balconygames.sweetyjumper) 。
8.  现在，我们没有营销或广告来推广应用程序，这将需要花费更多的钱，但希望如果人们喜欢它，它不会需要花费) :)
9.  我建议使用 gameanalytics.com 来查看你的用户(DAU，毛)，至少你会看到你的播放统计。

在其他文章中，我将尝试描述初学者使用 Unity 和游戏开发的主要瓶颈。
