# 获得每小时或每个项目的报酬？

> 原文：<https://dev.to/simoroshka/getting-paid-per-hour-or-per-project--4ic5>

自从我的第一份工作(在学校暑假期间将一些数据输入 excel 表格)以来，我一直在问自己这个问题:“如果工作能更快完成，那么按小时计酬有意义吗？”。

比如你做导游，拿时薪确实有道理。但是，如果你正在做一项能产生具体结果的既定任务，为什么在大多数情况下，你会获得小时报酬呢？

我总是做作业又快又好。早点开始并且有效率是有意义的，因为我会有更多的空闲时间。我比其他人更快，在数据表中输入行。正如你所猜测的，它没有得到回报，感觉超级不公平和愚蠢(青少年的想法)。

如果你速度快、效率高，最终你做同样多的工作得到的报酬会更少。因为你可能仍然被期望全职工作，所以你可能会过度劳累而筋疲力尽。集中工作要求更高。

追求速度的动力消失了。你开始保存能量。以小时为单位思考，而不是结果。你不会有更多的空闲时间或更多的钱。如果你能保持这样的速度，也许一年后你会得到提升。

另一方面，如果你从每个项目中获得报酬，对天数(而不是小时数)有一个好的估计是非常重要的。)你需要完成它，这样你就有足够的钱继续下去。众所周知，这在软件领域很难做到。大多数时候你不知道这个项目要花多少时间才能完成，除非它是标准的。因此，即使从开发人员的角度来看，按工作时间向开发人员支付报酬也是合理的。

15 年过去了，我还在和自己辩论这个话题。

很高兴听到你的经历。

*   你在不计算你工作时间的软件公司工作过吗？还是只有你是自由职业者才有这个可能？
*   你是“短跑运动员”还是“稳健”的开发者？你练深功吗？你每天/每周能完成多少小时？
*   你喜欢哪种薪酬类型？或许两者都有，视情况而定？
