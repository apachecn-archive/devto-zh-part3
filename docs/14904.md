# 为什么有些开发者对 ide 如此过敏？

> 原文:[https://dev . to/water link/why-some-developers-so-allergy-ides-24hl](https://dev.to/waterlink/why-are-some-developers-so-allergic-to-ides-24hl)

无数次我遇到喜欢更简单编辑器的开发人员。我说的“更喜欢”是指他们对任何想法都有强烈的负面感觉，尤其是那些需要花钱购买的想法。

现在，值得一提的是，在过去的 20 年里，我使用编码工具的过程是这样的:

记事本-> Turbo Pascal 编辑器-> Delphi-> Visual Studio-> Gedit-> Sublime Text-> Vim-> Emacs-> Emacs(邪恶模式)-> Atom -> Vim -> IntelliJ Idea(暂定)。

当我参加一次软件工艺会议时，就已经开始使用 IDE 了。我和另一个开发人员搭档，他向我展示了一些对我来说看起来像魔术一样的东西。

基本上，我对我选择的工具感到怀疑，因为我面前的这个 IDE(在一个有经验的用户手中)可以让我以目前的 5-10 倍的速度进行开发。

现在的问题是:**为什么我们，开发者，会对一个可以大大提高我们生产力的付费工具有如此负面的感觉？**

大家讨论一下！
