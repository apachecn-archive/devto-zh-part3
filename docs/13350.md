# QA 工程师的软件测试任务、过程和工具

在生活中，你遇到过多少次由于某人的错误、员工的无能、未知的风险而导致的最后关头的问题？项目启动延迟？

软件开发也是如此。如果一些功能没有被完全测试，一些错误被遗漏，产品没有在所有设备上被检查——麻烦正在酝酿。

## 质量保证工程师的任务

测试人员扮演的角色不亚于程序员。即使开发人员编写了高质量的代码，在设备上运行了应用程序，并看到一切都在工作，完整的功能系统仍然很遥远。

为什么一些应用程序(甚至看起来很好很有质量)在发布后会得到大量的负面评论？通常原因之一是应用程序在不同设备上的外观或工作方式不同。

由于市场上充斥着各种各样的设备，这种情况变得越来越糟糕。假设有人在一个旧的华为上推出了产品，并在上面看到了一个丑陋的布局。

或者它是由一个沙特阿拉伯的用户安装的，他什么都不懂，因为这个国家的文本是从右向左读的，而且界面不是根据这种文化特性定制的。

你可能听说过可口可乐广告活动因此而失败，但要知道，这不是真的。几年前，这个案例传遍了整个互联网，许多作者将其列入最著名的广告活动失败名单，但它没有得到任何可信来源的证实。

那么，谁来检查系统在所有条件下的运行情况呢？查找 bug 和漏洞？考虑各种不典型的情况？更重要的是，确保产品符合客户要求？测试人员正是负责这些任务的人。

QA 工程师的目标是实现正确的系统功能，防止缺陷，提高软件质量，并在整个项目中为最终客户提供质量信息。

测试人员还必须确保软件解决方案能够按预期工作。为此，他/她使用各种类型的测试将实际的系统行为与预期的行为进行比较。

## 测试人员的任务

*   识别 bug 和错误，描述并发送修订

*   检查实施的改进

*   测试产品，直到达到预期的结果

*   在所有必要的设备和屏幕分辨率上测试软件解决方案

*   检查各种条件下的产品性能，包括关键系统功能的使用，并针对不同的非典型用例进行测试

*   测试产品是否符合要求

*   解决所有异常情况，确保产品功能符合预期

## 功能测试生命周期如下所示:

*   程序员交付开发的特性进行测试

*   QA 测试实现的特性

*   QA 为每个检测到的错误创建一个错误报告

*   设置了错误和功能的优先级

*   产品调试——程序员修复错误并准备构建

*   功能测试。如果发现了错误，就重复测试和调试过程。当一切正常时，该特性被标记为已实现。

## 一些有用的软件测试工具

*   **Postman+Newman+Jenkins**–这个组合用于自动化 API 测试。

*   **Cucumber+Appium**——应用于自动化手机 app 测试。

*   –允许测试人员监控 HTTP/HTTPS 流量。

*   **[PICT](https://www.pict.com.pk/)**–一个非常方便的测试被检查参数值(及其组合)的工具。

*   **Apache JMeter，yandex ank**–支持测试应用性能。

*   JUnit，NUnit–用于编写单元测试。

*   ——用于测试请求和服务。

*   **[像素完美](https://chrome.google.com/webstore/detail/perfectpixel-by-welldonec/dkaagdgjmgdmbnecmcefdhjekcoceebi?hl=en)**——帮助测试布局的浏览器扩展。

*   **[打嗝套件](https://portswigger.net/burp)**——测试系统安全性的绝佳工具。

*   **清单**–QAs 可以专门为项目需求编制清单，也可以使用已经准备好的清单，根据任务进行定制。

*   **[Fabric crash lytics](https://get.fabric.io/)**——一款在团队内部共享构建和收集用户统计数据的流行工具。

*   **[邮差](https://www.getpostman.com/)**——一套测试 API 的工具。它有助于在将 API 集成到客户端应用程序之前测试功能。它还支持创建 API 文档、编写和运行测试，以及用测试值替换来自服务器的真实数据。
