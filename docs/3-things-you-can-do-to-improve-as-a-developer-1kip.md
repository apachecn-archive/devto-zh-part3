## 我们解决了！但是我们解决了什么？

作为开发人员，我们每天都面临着复杂的问题。好消息是我们天生就是解决问题的人。但是我们在解决什么问题呢？

> 解决错误问题的方法，无论多么聪明，都不能解决正确的问题。

作为技术人员，我们的工作是确保我们真正理解问题，这样我们才能提供适当的帮助。

这不仅是开发者成功的关键因素，也是有效发展开发者关系的关键部分。

那么我们如何识别和解决问题呢？这里有三件你可以做的事情来瞄准和回答正确的问题。

## 提问

你有意见。当然，你知道你知道什么。你的经历成就了你现在的地位，这是非常宝贵的。要骄傲！

现在，当有人问你一个关于技术的问题时，你如何回答？你的意见马上出来了吗？你确定你知道他们在问你什么吗？这个问题真的是这个人想知道的吗？你知道为什么这个人会问这个特别的问题吗？

OMG！那要考虑很多！(是的，它是)

想象一下下面的场景。

> *他们:*我该用什么 JavaScript 框架？

你会怎么回答？我们可以说反应，Vue 或 Angular -但是我们会帮助那个人吗？也许吧。他们为什么问这个问题？让我们想象一下，如果你问“为什么”，对话会是这样的。

> 你:为什么这么问？
> 
> 在使用 gobbledeegook.js(是的，这是我编的)之后，我的团队成员在为我们的新项目选择哪一个更新的框架上伤透了脑筋。
> 
> *你:*你为什么考虑转行？
> 
> 他们:我们很了解 gobbledeegook.js，但我们听说其他人都更好。
> 
> *你:*你怎么分类比较好？你现在用的有什么问题？
> 
> *他们:*我们目前的解决方案不能解决我们的问题。我们需要它。
> 
> 你:这个问题是什么，为什么对你项目至关重要？
> 
> *他们:*因为我们目前的应用程序加载缓慢，我们希望它在 2 秒内加载。
> 
> 你:今天怎么这么久？

好吧，这是一个有点虚构的故事，但你得到的想法。最初的问题是提问者长期积累的一系列经验的结果。换句话说，他们有一个你没有的环境！

> 这个练习的一个很好的资源是学习更多关于[5 个为什么](https://en.wikipedia.org/wiki/5_Whys)

好消息是，你可以通过问“为什么”来获得上下文。一旦你了解了这个背景，你就离真正帮助提问者更近了一步。有时，他们甚至会一边和你说一边解决问题。

最终，问“为什么”会把你引向一个与最初的问题不同的方向。

如果你在开发人员关系中工作，这种深刻的洞察力在提供产品反馈和帮助客户方面是非常宝贵的。

## 倾听和观察

虽然听起来很简单，但实际上可能很难。好吧，别对着我做鬼脸。想象你正在帮助某人写代码。当他们打字时，你看着他们的笔记本电脑，你看到他们打字的方式与你不同。他们在使用之前没有定义这个函数。现在，他们转向另一组收集用户输入的代码。他们为什么不修复这个功能？现在他们正在编写 Dockerfile 和硬编码环境变量。这让你很困扰。你会阻止他们跳进去吗？你是做什么的？

仔细观察某人如何工作和解决问题可能是最有启发性的学习方法之一。我们以这样的方式对待事物是有原因的。如果我们的目标是帮助人们解决问题，我们必须首先了解他们是如何思考的。想想他们在想什么。

我们可以观看、聆听并提出开放式问题，例如

*   您希望 docker 容器在哪里运行？
*   你将如何测试你的功能？
*   当这个事件在你的应用中发生时，用户如何知道该做什么？

请注意，这些问题并没有给出是或否的答案。相反，它们旨在激发他人分享他们的想法，告诉他们为什么以及如何思考。有了这些知识，我们就可以学习他们如何工作和解决问题的共同模式。

也许，仅仅是也许，我们可以将这种学习转化为对我们软件的反馈，让他们更容易找到解决方案。

如果有人用头撞墙，那不是墙的错。我们不移动墙壁。相反，我们解决了导致他们撞到头的问题。

了解人们为什么以及如何解决问题的最好方法是真正倾听。

## 主动寻求帮助

我们处在一个奇妙的职业领域。技术人员每天都在奋斗和解决问题，我们与同事分享这些奋斗和解决方案。那是一个社区。一个有益的社区。

你可能以前听过类似下面这句话的话，所以请原谅我。

> 不要为他们没有的问题提供解决方案，*找到他们确实有的问题*并为之精心设计解决方案。

一旦我们理解了人们并与他们建立了联系，这就变得容易多了。这有一个很大的副作用，让我们所有人内心感觉良好。

## 接下来呢？

当你解决了一个问题，分享它。找出你喜欢的交流方式，并拥有它。你可能更喜欢写作、录制视频、创建工具如 VS 代码扩展、创建 OSS 或在播客上谈论它。分享的方式由你决定。

所有这些都适用于开发软件、开发人员关系或解决任何技术问题的团队。

当我们问问题时，我们倾听，我们观察，然后我们可以接受反馈并真正开始提供有用的解决方案。

*   你是怎么提问的？
*   你是如何倾听和观察的？
*   你如何寻求帮助他人？
