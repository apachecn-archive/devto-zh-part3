几年前，当我听到有人为开源项目做贡献时，我总是很害怕😥。我认为一个人在投稿前必须完全了解一大堆。我不知道这是一个谬论。

对贡献开源的一个常见误解是你需要贡献代码。👻

在 Github 上有很多项目你可以在不了解任何代码的情况下参与。让我们意识到，不仅仅是软件开发人员为开源项目做出贡献。我对项目的第一个贡献是修复项目自述文件中的一个类型错误。合并的时候我好开心。

开源提高了你现有的技能，帮助你认识很多人，教人，审查代码和指导初学者。

*开始为开源做贡献的步骤*

这些步骤对我非常有效，我知道如果你没有任何贡献，它会帮助你做出你的第一份贡献。要投稿，你必须浏览 Github 上不同项目的问题，有很多开源项目你可以投稿，Javascript 库，Gatsby，Npm 包，Ruby Gems 等等。

点击问题，然后点击标签并通过`beginner friendly`或`good first issue`进行过滤，您会看到许多容易让您开始的错误，并在几分钟内提出拉请求。
你也可以在自述文件中搜索错别字，并提出一个公关来纠正它，这样做将使你对项目做出贡献，你甚至可能被添加为项目的贡献者，并增加你的个人资料。自从我做出了第一份贡献，我不再害怕为不同的项目做出贡献或在 Github 上解决问题。
看到要解决的问题或难题后，在 Github 上分叉项目并创建一个新的分支来解决问题，然后提出一个拉取请求。加油🎉🎉，你做出了你的第一个贡献。

* * *

由于我是一名 Ruby on Rails 开发人员，所以在这里我会有一点偏见。如果你是一名 Rails 开发者，并且想为开源做出贡献，请访问这个[网站](http://www.opensourcerails.com/),查看流行的 Rails 开源项目。有一个网络我希望你能考虑加入，那就是敏捷风险投资。在[敏捷创投](https://agileventures.org)，你参与了很多项目，从 Rails 到流行的 Javascript 框架，比如 Vue 和 React。

敏捷投资让你感觉像在家里一样，你会看到很多新手，有很多结对会议来解除你的障碍，并与来自世界各地的开发人员合作。不要害怕为开源项目做贡献，修复一个错别字或增加一行代码。一切都很重要💪🏻。
