## 金姆，你现在在忙什么？

我目前在伦敦生活和工作，即将结束我的商业技术角色。嘶…我有一些惊人的消息要分享；作为创客奖学金项目的一部分，我即将成为一名软件工程师。我将于 2019 年 2 月 4 日星期一开始我的预科课程。感谢创客联谊计划，我也将开始我童年的梦想并得到资助！为你们所做的出色工作向制造商大声欢呼吧！**我以为这个梦想已经结束了，因为我没有获得计算机科学学位。我错了！伦敦的技术社区令人惊叹，并很快揭穿了这个神话。**事实证明，我从未真正停止过“编码”,它只是隐藏在我大学期间的气候/环境建模工作、我工作中的 Microsoft Excel 功能以及我对等待解锁的计算机的兴趣之中。地理和软件工程实际上有很多共同点；将问题分解成组成部分和系统的能力——思考给了我巨大的帮助。

离开了我的舒适区，我既兴奋又紧张。我想这是很自然的，因为我会有新的习惯，新的环境，有机会认识新的人，接受新的概念和想法。现在结束我目前的商业技术机会，并把那一章抛在身后，这是非常令人难过的；尽管我非常热衷于投身于我的软件工程事业！

## 你为什么选择在大学学地理？毕业后做了什么？

我不知道我大学毕业后想做什么；18 岁对于做决定来说太年轻了，是的，那时候确实感觉像是世界末日。在我的成长过程中，大学是唯一合乎逻辑的选择。我在医学和法律之间转换，觉得必须适应人们对我期望的“理想”。最后，我选择了地理，因为我很好奇是什么让这个世界运转。我觉得地理很好地融合了艺术和科学，揭示了环境、社会、经济和技术的相互作用。作为牛津大学和 UCL 大学的校友，我获得了地理学士学位(2014 年)和环境建模硕士学位(2015 年)。

毕业后，我还是不知道什么职业适合我；我的许多朋友都参加了很棒的研究生项目，而且报酬丰厚，但当时，我觉得自己似乎“失败了”。

我没有拿自己和别人比较，而是重新振作起来，打造自己的道路。我最终找到了一份能源行业分析师的全职工作，并为一家市场情报公司撰写了大量报告，同时还兼职做家教。我的直觉认为这份工作不适合我，6 个月后，我进入了一家可持续发展咨询公司，专注于加快城市可持续建筑管理的发展和生产力，我在那里呆了 3 年。

## 你第一次遇到代码是什么时候？

我写了第一行代码，同时试图找到一种更好的方法来为我 2011 年的项目提取一些气候数据；那时候，编码仅仅是我得到最终答案的工具。在过去的几年里，我发现自己想要更多地了解一些东西是如何工作的，而不仅仅是接收最终输出。

## 编码对你意味着什么？

好问题。如今，我把编码视为一种艺术形式和一种思维方式——一种哲学。我从来没有想过我会出于好奇去 Foyles 书店找 Jonathan Rasmusson 写的《敏捷武士:敏捷大师如何交付伟大的软件》,这本书是我用来支持我的公司引入敏捷产品和项目管理策略的。时至今日，我已经能够融合生活各方面的技能。到目前为止，我喜欢巩固我的知识，每天改进我的编码，挑战我自己！我受到启发，希望有一天能教别人编程，并自愿为科技行业的多样性和社会流动性贡献自己的技能。我也看到自己在帮助负责任的企业成长，这符合我的使命，即利用编码和技术的力量推动社会、环境和经济变革。

## 是什么促使你转行成为一名软件工程师？

回顾我在开始时的引用，我想与童年的自己联系起来，以摆脱我的头脑认为我应该做的事情，寻找我 T2 真正想做的事情，并将它与我的激情联系起来。我对可持续发展和智能城市充满热情，也对技术如何将人与环境结合在一起，改变人们对环境的感知和居住方式充满热情。与此同时，工作中的机会和对了解技术内部运作的好奇心让我意识到软件工程职业非常适合我。

### 工作中的机遇

我在可持续发展咨询公司的朋友是一群很棒的人，他们激励我每天去工作，做到最好。在咨询团队担任初级可持续发展顾问一年后，我有机会提出一些想法来发展公司的专有技术解决方案，该解决方案专注于可持续发展的房地产行业。该解决方案采用数据驱动的方法来支持商业房地产行业客户的环境数据管理和报告。

我非常感激公司对我的信任，给我机会成为他们新技术团队的一员。那时候这几乎是一张白纸，感觉有些东西是对的。虽然我犹豫着要不要离开我的可持续发展咨询工作，但我为自己能成为某件特别事情的一部分而欢欣鼓舞。

在两年多的时间里，我掌握了敏捷的概念，并通过实施敏捷产品/项目管理策略、业务需求收集和规范，与团队一起从事可持续性软件的软件开发。我学习了 scrum 迭代软件开发以及如何使用 Atlassian 的 JIRA 工具。

### 四周一片肃穆！

凭借我的多学科背景，我既会说商业语言，又会说技术语言。我身兼数职，将技术的细节汇集到商业层面的有意义的东西上。每当一个 bug 出现时，我都会暗自享受，因为这意味着我可以和开发人员聊天，试着调试一些东西，并问他们一些问题。我在工作中获得了很多乐趣，每个人都非常支持我，并提出了很多好主意。当你改变用户实现技术驱动的可持续发展的方式时，身兼数职是一项挑战，但也是值得的。

### 朝九晚五之外的机会

我很幸运，在家门口就能接触到一个活跃的技术社区。外面有很多很棒的社区，这里有一些我必须了解的很棒的社区:

### 代码第一:女生

2017 年 9 月，我加入了《代码第一:女生 Web 开发初级班》，从此一去不返。我记得每周的一个晚上下班后，我飞奔到 Twitter 英国总部，筋疲力尽，但对编码感到如此兴奋和充满活力。我每周开一次会，内容涉及 HTML、CSS、JavaScript、jQuery、GIT、GitHub 协作、开发概念、Twitter Bootstrap 和响应式 web 开发，甚至有机会参与一个小组项目。

一结束，我就注册了 Code First: Girls Advanced Ruby 课程，在那里我了解了 Ruby 编程、Sinatra 框架、GET/POST 请求、开发概念、使用 Mailgun 的自动电子邮件、外部 API 和部署。我开始了解 Heroku 云托管服务上的应用程序部署，并探索了 Twitter API。

进入创客对我来说就像家一样，我想象自己有一天会在那里。我实际上考察了英国的几个编码训练营，但被费用吓住了。幸运的是，我偶然发现了创客的奖学金项目并申请了！这是我做过的最好的决定。我的面试很有挑战性；尽管这是我参加过的最愉快的面试之一。

## 你能给想改变职业的人什么建议？

花时间想想*为什么*和*你想如何*改变。转行总是有风险的，是一种机会成本；如果你认为这是值得的，那就跳下去，花时间放松自己。每个人都不一样，你比任何人都了解自己。改变不会在一夜之间发生，你必须有**的毅力**和**的耐心**才能坚持到底。如果你在某件事上失败了，感到沮丧是很自然的，但是要重新振作起来。如果不适合你，那就找个让你**勾**的。成功没有失败；它们携手并进。记得放松自己，多休息！我这么说是因为我经历了太多次的精疲力竭！最重要的是，生活中有些事情是不如意的；这就是生活。
