# 2019 年软件工程师的 19 个小技巧

> 原文：<https://dev.to/emmabostian/19-tips-for-software-engineers-in-2019-4okm>

快到 2019 年了，编程界有一些惊人的事情正在发生。但是对于许多新开发人员来说，投身于编码事业的前景可能是令人生畏的。

有这么多的技术可以学习，有这么多的语言可以选择。因此，我编制了一份 2019 年软件工程师 19 条建议的清单(没有双关语)。

快乐编码🤩💻

# 1。不要气馁

科技行业是最激烈、最不稳定的行业之一，投入其中会让人不知所措。我每天努力记住的是，每个人在他们编码生涯的某个时刻，都是从我开始的。所有科技行业的领导者都曾经是初学者。

有些人比其他人学得快吗？是的。但是你不能让这阻止你开始。

有一个很好的理由让人们渴望从事编码职业:你可以在互联网上自学任何东西...免费的！通常科技职业的生活方式比其他职业更舒适。

重要的是要记住，虽然有大量的资源，但你不需要学习所有的东西。从基础开始。

第一步是决定你是喜欢从前端(你与之交互的应用程序的用户界面)还是后端(处理数据)开始你的职业生涯。一旦你选择了你的利基，从基础开始。

如果你选择了前端开发，那就钻研 HTML、CSS 和 JavaScript。不要认为这些技能是理所当然的。虽然乍看起来很简单，但是对 web 开发基础的深入了解将会让你成为一名软件工程师。大多数公司更愿意看到一个拥有强大“普通”技能的候选人，而不是一个对流行框架/库非常了解的人。

如果你决定走后端开发的路线，试试像 Python 或 Java 这样的语言。这些都是科技行业非常令人垂涎的技能。

但是不要气馁。你会有完全不知所措的日子。每个人都从你开始的地方开始了他们的旅程。它会变得更容易。

# 2。学会阅读&破译密码

当我开始我的编码之旅时，我认为理所当然的技能之一是阅读其他开发人员代码的能力。这是一项真正重要的技能，因为它允许您在许多不同的代码库中有效地工作。

你不需要知道一种编程语言的所有细微差别，就可以破译代码在做什么。

有很多方法可以学习如何阅读代码。查看一些开源项目。通读文档和源代码，看看是否能描述出发生了什么。你会慢慢开始意识到有可以改进的地方，也许你甚至会开一个公关！

学会分析代码并确定它是否是以最有效的方式编写的。

学习阅读和审查代码是一项需要时间的技能，但是非常值得努力。

# 3。找到你的学习方法

人类以不同的方式学习。有些人在编写代码时能最好地保留信息。其他人通过阅读或观看视频学得最好。

我更喜欢看在线教程和阅读文档来填补空白。

我不是那种可以启动一个沙盒应用程序，然后编码的人。我喜欢遵循一个有指导的教程，然后扩展我的想法。

找到你的学习方法。尝试不同的资源和平台。

一旦你找到了你的学习方法，你会发现你学得更有效率。

# 4。参与社区活动

我直到大约 3 年后才加入在线技术社区，开始了我的编码之旅。这使我处于极大的劣势。

我与冒名顶替综合症做了很大的斗争，发现很难保持动力。但是一旦我在 Twitter 和博客平台上发现了一个开发者社区，我对学习投入了更多。

找一个开源项目。在推特上发布你的编码项目。将您的项目推送到 GitHub。找到一种参与的方式，你会很快与编码建立更亲密的关系。

# 5。平衡理论与实践

为了成为一名有效的程序员，你必须理解某些范式背后的理论(在一定程度上)。

例如，如果您想理解为什么一个嵌套的 For 循环不如同一范围内的两个 for 循环高效，您需要理解 Big-O 符号。

我们生活在一个不需要计算机科学学位就能在科技行业取得成功的时代。也就是说，CS 学位教授的一些概念非常有价值(例如算法、运行时分析等)。).

你不需要学习所有的计算机科学理论，但是你应该理解为什么一个解决方案比另一个更好的理论。

# 6。不要拿自己和别人比较

你的职业生涯永远不会跟其他开发者走完全一样的路。你不应该拿自己和别人比较。

你是独一无二的，你提供的技能将为你量身定制。

仅仅因为一个行业领导者正在学习 Ruby on Rails，并不意味着你必须跟着学。找到让你对编程感兴趣的技能，并尽你所能去学习它们。

学习一项技术没有预期的时间框架。正如我在技巧 3 中提到的，人们用不同的方法学习，因此人们学习的速度也不同。

仅仅因为你花了三个月的时间学习 JavaScript，并不意味着你比一个一个月就学会的女孩更糟糕。

# 7。参与开源

开源社区正在蓬勃发展，为什么不参与进来呢？如果你需要一种方法来提高你的代码阅读技能，那就去找一个开源项目吧！

你可以查看一下 [GitHub 开源](https://github.com/open-source)社区来开始！

您甚至可以开始自己的开源项目！去年九月，我创立了自己的开源项目，[编码蔻驰](https://github.com/Coding-Coach/coding-coach)，这是一次令人大开眼界的经历。

如果您想参与其中并向一些优秀的开发人员学习，请随时查看！

# 8。对不舒服感到舒服

科技行业在不断变化，这意味着总有新技能需要学习。

虽然乍一看这可能令人生畏，但它也提供了许多发现新技术的机会(你怎么会感到无聊呢？！).

提高编码技能的最好方法是处理让你害怕的任务。你可以在工作中通过自愿完成一项具有挑战性的任务来做到这一点。但是你也可以在业余时间利用新技术来做这件事。

你必须适应不舒服的感觉。你不需要知道所有的答案。

# 9。不要害怕问问题

最终你会碰到一个你不知道怎么解决的问题。堆栈溢出不能解决问题！

重要的是要认识到什么时候你需要寻求帮助。如果你试图解决一个问题，并且走了所有可行的路，是时候寻求帮助了。

寻求帮助不会让你看起来软弱；这是我至今仍在纠结的事情。

如果你花了几个小时试图找出你的变量抛出 ReferenceError 的原因，寻求帮助。

# 10。让你周围的人帮助你成长

编码很难。总有一天你会觉得自己不适合做软件工程师。

因此，你必须和信任你的人在一起。找一群能让你建立自尊的核心家人或朋友，和那些不能建立自尊的人说再见。

# 11。一次专注于一件事

一心多用是确保平庸的最快方法。人类不能有效地一心多用。

一次选择一件事来专注。尽你所能去学习它，或者完成它。一旦完成，继续下一件事。

不要试图同时处理几个话题。

# 12。查看更大的图片-所有这些技术如何结合在一起

在您的编码生涯中，您将学习许多不同的技术。确定哪种技术解决哪种任务是非常令人困惑的。

我喜欢画复杂问题的图表。如果我的任务是构建一个应用程序，我喜欢把它分成几个域。例如，我知道我在前端需要 HTML、CSS 和 JavaScript。也许我会决定使用 React 作为 JS 库。也许，如果我需要这个应用程序伸缩，我会选择 Redux 进行状态管理。

了解不同技术的最佳使用案例。你什么时候会选择一个而不是另一个？它们是如何组合在一起的？

这些是你应该学会回答的重要问题。

# 13。找到你的工具箱——哪些工具最适合这项工作？

如今，开发人员并不缺少可用的工具。我能立刻说出至少五个我过去工作过的想法。

可用工具的数量可能会非常多。做一些研究。确定哪种工具适合您。您可能想问的一些问题是:

*   该工具是否得到积极维护？
*   是否有一个蓬勃发展的开发者社区在使用它(如果你需要帮助的话)？
*   配置有多难？
*   集成这个工具会对性能产生负面影响吗？
*   没有工具我能解决这个任务吗？

一个工具的存在并不意味着你需要使用它。

# 14。保持良好的工作/生活平衡

你不需要花一整天的时间来编码。事实上，这样做是不健康的。

你必须保持健康的工作/生活平衡，否则你会遭受精疲力竭的后果。

如果你从醒来的那一刻一直到晚上睡觉的那一刻都在做一件事，你会很快对它产生反感。

通过为自己和其他爱好留出时间，你将与编码保持一种健康的关系。

# 15。学习设计的基础知识

如果你正在构建一个面向用户的应用程序，你应该知道设计的基础。我不是说你需要学习如何使用素描或 Adobe Illustrator，但学习 UX 设计的基础知识将积极影响你的应用程序。

你不能假设用户会像你一样使用你的应用程序。事实上，你根本不应该对用户行为做任何假设。

理解某些 UI 原则背后的推理，以便为您的应用程序确定最佳解决方案。

例如，人们发现方形边缘的按钮实际上比圆形边缘的按钮更适合用户，因为它们包含更多的像素。

学会这些基础知识，你就可以确信你的 web 应用会大放异彩。

# 16。找一个导师

找导师很难，但很有必要。从职业建议到编码最佳实践，你可以学到宝贵的技能。

在这个领域里，找一个你欣赏的人来指导你不会有什么坏处。他们能说的最糟糕的话就是不！

记住，随着时间的推移，重新评估你的导师，以确保这对你们双方都有利。

# 17。建立一份出色的投资组合&简历

投资组合和简历是你在面试过程中给雇主的第一印象。当务之急是展现你最好的一面。

修正拼写和语法错误。确保所有内容的格式一致且适当。

如果你想知道更多关于写一份出色简历的细节，请随意查看我之前的[博文](https://levelup.gitconnected.com/how-to-write-a-stand-out-technical-resume-6cce4721cd8c)。

# 18。快速失败和经常失败

失败是主观的。如果你犯了错误，从中吸取教训。不要两次犯同样的错误。仅仅因为某事“失败”并不意味着它没有价值。

历史上最成功的人创作了大量的作品，然而他们只被少数人认可。

你不必为你开发的每个项目从头到尾编写漂亮的网站。专注于底层技术，你可以担心以后让它看起来更好。

# 19。继续前进

总有一天你会想放弃编码。不要。这是一个艰难的职业，每个人都患有冒名顶替综合症。

你并不孤单。随着时间的推移，编码会变得越来越容易。

每天坚持，你会好起来的。

继续走。
