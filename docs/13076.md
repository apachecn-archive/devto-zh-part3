# 编码，蓝领工作？

> 原文:[https://dev.to/bananabrann/coding-a-blue-collar-job-pfn](https://dev.to/bananabrann/coding-a-blue-collar-job-pfn)

在农村养鸡是我日常生活的一部分。我不得不承认，搬进城市，投入编码工作:驱使美国农村人走向成功的决心和努力，与我在成功的开发人员身上看到的完全一样。

> 如果你从未听说过这个术语，需要体力劳动的工作通常被称为“蓝领”，而且自 1924 年以来就一直如此称呼。这些工作主要可以细分为两类:*非熟练*，如采矿和仓库，以及*熟练*，如电工和焊工。非技术和技术工作蓝领一直是美国经济的主要驱动力，而技术蓝领刺激了中产阶级的增长。

当我们接近某项技能时，我们的心态会极大地影响我们成长的快慢，甚至最终产品的质量。这种心态可以是效率，成长，质量，或者任何可以驱动你和交易的价值。我认为蓝领工作和编码有相似之处，因为它们都是基于有形的物质产出，而不是律师和会计师等工作的智力或效率产出。

当评估编码市场和蓝领工作市场的相似性时，推理可以更进一步。随着我们的老一代人退休，对熟练劳动力的需求空前高涨，公司告诉我们他们愿意补偿。让我们看看水管工，在 T2，芝加哥和纽约的平均年薪是 7 万美元。据报道，对于更有经验的水管工来说，这个数字[高达 20 万美元](https://nypost.com/2016/08/03/city-plumber-made-more-in-ot-than-de-blasios-total-salary/)！我相信你已经听说过同样的事情发生在开发人员身上。用头撞键盘是一种勇气，不是每个人都具备的，并且很难将有能力和有技能的开发人员安置到劳动力中。很少或没有经验的开发人员正在获得 7 万美元/年或更多的工作，随着更多高级和有经验的职位在全国范围内收入超过六位数，天空正在成为极限。

这些数据并不意味着进入软件开发的每个人都有一大笔钱。就像蓝领工作一样，要收获自己的劳动果实，你的技能和进步必须由你自己负责。

纽约和芝加哥成功的水管工的思维模式和开发人员的思维模式几乎是一样的，我相信在提高我们的代码技能方面，可以从那些用手的人身上学到一些东西。
