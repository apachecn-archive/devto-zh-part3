# 为创新选择正确的标准

> 原文：<https://dev.to/jessekphillips/picking-the-right-standards-for-innovation-o2h>

标准是好的，我不想给人留下我不相信标准的印象。但是，如果标准化的重点瞄准了错误的东西，就会有麻烦。标准做法是每次使用标准时都要重新评估，包括本标准。

[标准会阻碍创新吗](http://www.innovationmanagement.se/imtool-articles/do-standards-hinder-innovation/)。我会说这属于一切范畴，一切都要适度...甚至适度(正如我妻子喜欢说的)。如果你选择标准是因为它们是标准的，那你就错了。你选择它们是因为你知道标准的存在是有原因的吗...你能阐明那个原因吗？

根据定义，标准不是创新，如果你有一个核心租户，你的公司就是创新者...你最好不要做标准的最佳实践。这并不意味着标准不应该存在，而是意味着你希望人们打破标准。你希望这些标准受到挑战。你不希望标准是我们一直以来的做法。

[咆哮]以我的名义提出的论点是，该代码已经存在多年，因为担心随着 selenium 的变化，它会有问题。这不太好，“我们一直是这样做的”不是一个合理的借口。这造成了巨大的冲突，创新，标准，不像以前那样做，建立可重复的实践，但永远不再做。
