# Java，C++，还是 C#？？？

> 原文:[https://dev.to/doylecodes/java-c-or-c-1a8l](https://dev.to/doylecodes/java-c-or-c-1a8l)

好吧。我认为自己是一个新的开发者。我主要从事 Node、Javascript 和 React 方面的工作，并且一直希望成为一名开发人员。

在我寻找新职业的过程中，我当然一直在寻找发展我的技能和学习更多的东西。当我在网上看的时候(当然，这可能只是我的领域)，我看到了很多空缺职位。NET 和许多后端工作寻找与 Java 的经验。

## [](#the-question)提问

我的问题是，作为一个认为自己在 Javascript/Node 前沿非常强大的人，接下来要学习的更有益的编程语言是什么？学习 Java、C++或 C#会让我成为更好的程序员吗？ *

## [](#here-are-my-first-thoughts-for-each)以下是我对每一个的最初想法:

*   Java:它无处不在，所以看起来这不仅是一项技能，也是一份简历。我第一次面试搞砸了，因为那是一个后端 Java 职位。(我申请了一个 Node/JS 前端的职位)

*   **C++** :和 C 关系密切(以我的理解)。显然，这对于基于 c 的其他语言很有帮助。而且，我听说这很好，因为它不会自动管理内存之类的东西，所以它让你成为一名更好的开发人员，因为你需要比使用其他语言更多地了解该语言的工作原理。

*   **C#** :有点像 C？这也是。NET 堆栈，它似乎有很多工作。

*我不一定是为了马上找到工作而寻找最好的语言。我试着想得更长远些。由于我上面所说的，这种想法引导我走向 C++，但是我不知道这些语言中的任何一种，我希望有人能有更多的见解！我宁愿学习一门技能可以全面应用的语言，而不是某个特定的东西，但后来我被锁定在这一点上。

或者...也许走完全相反的方向，和斯威夫特一起走？！
