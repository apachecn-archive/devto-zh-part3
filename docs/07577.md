# 2019 年学习测试自动化的 17 大资源

> 原文:[https://dev . to/lambdatest/top-17-resources-to-learn-test-automation-in-2019-3155](https://dev.to/lambdatest/top-17-resources-to-learn-test-automation-in-2019-3155)

缺乏培训是测试人员的主要障碍。通常，每当一个新项目需要时，在一个组织中工作的测试人员会突然被迫学习一个新的框架或者一个自动化工具。您可能会不知所措，不知道如何学习测试自动化，从哪里开始，以及如何掌握 web 应用程序和移动应用程序的测试自动化。

幸运的是，有大量的资源可以帮助您解决如何学习测试自动化的问题，并指导您学习自动化测试的基础知识，从而熟练掌握 web 应用程序和移动应用程序的测试自动化。在这篇文章中，我们将涵盖测试自动化的 17 大资源，包括博客、研讨会和在线视频教程等等，以帮助您通过深入的教程来温习自动化测试的基础知识。我相信每个测试人员，不管是初学者还是有经验的，都应该遵循它们。

## [](#email-newsletters)电子邮件简讯

时事通讯是测试自动化保持更新自动化测试世界中发生的最新事情的最佳资源之一。如果你订阅了一个，它会每天发送一封邮件，里面有来自所有顶级自动化测试网站的最好的新闻、文章和博客文章的摘要。你所需要做的就是，提供你的电子邮件地址并订阅一个。首先，你可以订阅由测试部发布的官方周报。他们每天发布有趣的博客文章，可以教你自动化测试的基础知识等等，尤其是如果你是自动化测试领域的新手。

## [](#workshops-and-other-learning-platforms)工作坊等学习平台

任何一种新技术，尤其是与编码和脚本相关的技术，最好是在亲自动手编码的时候学习。对于自动化来说幸运的是，有相当多的工作坊，你可以在学习的同时并行地进行项目工作。让我们来看看其中的一些。

### [](#rest-assured-workshop-ontestautomation)放心工场—[OnTestAutomation](https://www.ontestautomation.com/)

Bas Dijkstra 是一位经验丰富的测试自动化顾问和培训师，他分享了他的专业知识，并通过他网站上的几门课程帮助新候选人掌握 API 测试。

[![OnTestAutomation](../Images/b3cd23ac64241e8e10f8d139813fa2e8.png)T4】](https://res.cloudinary.com/practicaldev/image/fetch/s--UmboQJ7l--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.lambdatest.com/blog/wp-content/uploads/2019/02/pasted-image-0-8-1-1.png)

虽然，有很多课程提供 API 测试，但是大部分都集中在一个特定的工具上。然而，Dijkstra 提供的课程教你关于

→在 [API 测试](https://www.lambdatest.com/blog/everything-you-need-to-know-about-api-testing/?utm_source=dev&utm_medium=Blog&utm_campaign=ar-03-080419eu&utm_term=OrganicPosting)时你必须注意的事情。
→API 在现代软件中的作用。
→涵盖所有可用于自动化 API 测试的工具。最后但同样重要的是，您可以使用放心库来学习 API 测试。

您还可以在 GitHub 上找到他的工作室，在那里您可以在学习 API 测试的同时获得实践经验，从而提高自动化测试的基础知识。

### [](#explore-with-postman-amber-race)与邮差一起探索——[琥珀族](https://github.com/ambertests/explore-with-postman)

Amber Race 是测试协会的著名演讲者，他让[探索性测试人员](https://www.lambdatest.com/blog/9-must-have-skills-to-master-exploratory-testing/?utm_source=dev&utm_medium=Blog&utm_campaign=ar-03-080419eu&utm_term=OrganicPosting)意识到他们可以通过这个研讨会使用的自动化工具。在 GitHub 上可以免费获得[与邮递员一起探索](https://github.com/ambertests/explore-with-postman)研讨会，在这里你可以覆盖基础知识，并开始体验自动化 API [探索性测试](https://www.lambdatest.com/blog/exploratory-testing-all-about-discovery/?utm_source=dev&utm_medium=Blog&utm_campaign=ar-03-080419eu&utm_term=OrganicPosting)。

### [](#citrus)[柑橘](https://citrusframework.org/docs/home/)

[![Citrus](../Images/c3324a3bed8453d754481ed0f0c5db9a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--TePTck2s--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.lambdatest.com/blog/wp-content/uploads/2019/02/pasted-image-0-9-1.png)

企业应用程序集成通常通过不同的接口分阶段完成。然而，如果接口没有适当的通信，可能会导致许多不可预见的错误和业务错误。这就是为什么集成测试很重要。Citrus 是一个开源框架，在这里您可以找到如何学习测试自动化的答案，以便对任何数据格式或消息传递协议进行集成测试。您可以免费下载该框架，并从文档开始，在那里您可以深入了解如何使用该框架执行自动化集成测试。
