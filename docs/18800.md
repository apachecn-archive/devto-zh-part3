# 使用 CSS 网格创建响应式网页设计

> 原文:[https://dev . to/independent/using-CSS-grid-to-create-responsive-web-design-1fa 8](https://dev.to/independnt/using-css-grid-to-create-responsive-web-design-1fa8)

如果你对网页设计很熟悉，那么你应该知道用 CSS 完成你的设计真的只是一个中间地带。我们仍然需要适应平板电脑和手机的用户体验。现在，超过 50%的 web 活动都是在移动平台上处理的，所以现在，移动优先的设计应该成为主流。在我完成了网上作品集的设计后，我马上意识到这一点，我可以补充说，这看起来非常棒。但是，在手机上拉起来呢？它看起来像热垃圾。那是我了解媒体询问的重要性的地方。根据屏幕大小改变 CSS。现在，在 React 和使用样式化组件以及 CSS grid 的深入研究中，我发现自己又一次处于通常由查询引导方向的分水岭。幸运的是，CSS grid 有方便的工具，称为自动填充和自动适应属性，可以减轻一些以前繁琐的任务。

**媒体查询**

以前，如果我希望我的应用程序或页面有响应性，我必须单独选择窗口大小，以及我希望与之一起缩放的 css，特别是关于具有单独项目的行/列。例如，它看起来像这样:

```
@media only screen and (max-width: 767px) {
    body { font-size: 16px;}
    section { padding: 30px 0;}

    .row { padding: 0 4%; }
    .col {
        width: 100%;
        margin: 0 0 1% 0;
    }

@media only screen and (max-width: 1200px) {
  .row { padding: 0 2%;}
} 
```

在这里，我选择宽度并指定具体的行填充和宽度，这些都可以工作，并且肯定会使站点更具响应性，但最近在使用 react 并拥有一个与 grid 一起工作的页面时，我遇到了自动适应和自动填充，这使我的生活变得容易多了。

**自动适应**

以一个应用程序为例，它获取一个项目并将其放入网格格式的列中。

```
export const CoinGridStyled = styled.div `
  display: grid;
  grid-template-columns: repeat(5, 1fr);
  grid-gap: 15px;
  margin-top: 30px;
` 
```

这里我们有 5 列，每一列占页面的一小部分，不考虑大小。即使我们扩大或缩短窗口，每行仍会有 5 个项目，只是简单地调整大小。但是如果我们把它改成这样的自动适应:

```
 grid-template-columns: repeat(auto-fit, minmax(130px, 1fr)); 
```

现在，这一行将总是适合它所能适合的。因此，扩展窗口，每行将产生 5 个以上的项目，这取决于大小，直到 130 像素，也会这样做。所以现在，随着窗口的扩大，我的行将有最少 2 个项目(最大 130 像素)或最多 10 个项目。但这意味着，即使我只有一个项目，它也会拉伸该项目以适应页面长度。“更精确地自动调整”通过扩展当前可用的列，使它们占据所有可用的空间，从而将它们调整到空间中。因此，在这种情况下，我喜欢自动填充。

**自动适应**

在这里，我们将包括这段代码:

```
 grid-template-columns: repeat(auto-fill, minmax(130px, 1fr)); 
```

现在，有了 auto-foll，它可以“尽可能多的列填充行”。所以现在，物体没有被拉伸。空间是有限的，每个物体占用的空间都是必需的。

现在有了网格，我们将不再需要媒体的询问，感谢上帝。
