正如你可能从标题中想象的那样，我写这篇文章的时候正坐在从柏林到汉堡的火车上。对于那些从未去过德国的人...我们这里的火车上确实有 WIFI，但与你想象的相反，它真的很差。(如果它有时足够快，您只能获得 200mb 的流量< =这要感谢可以绕过的 mac 地址随机化)

等等，什么？第一世界工业国家德国火车上 WIFI 不好？是的，甚至在我乘火车去泰国旅行的时候，我的 WIFI 比我在德国火车上体验过的要好很多。这有两个主要因素:

*   整体移动网络不佳...如果你离开大城市，大部分时间你甚至没有 Edge(是的，孩子，比 2G 慢)或移动网络连接。好难过！
*   火车上的廉价硬件。实际上，火车上的调制解调器是标准的 3G 调制解调器，您也可以购买作为移动热点设备。当然，它们的功能更强大一点，但它们不是为这种特殊的使用情况而设计的:以高比率连接到新基站。让调制解调器在高速列车上做到这一点，实际上是一个相当艰巨的技术挑战。但是我们有 2019 年...考虑把人送上火星...正如我们在其他国家看到的，这个问题显然已经解决了。也许更多的钱投资在这里会更好。

**但是关于这里的 WIFI 已经说得够多了(顺便说一句，目前不存在)**

好吧，对不起，还有一个想法:环顾四周，我看到许多人在漂亮的套房里用笔记本电脑工作。想象一下，他们每小时挣 60 欧元，他们需要两倍的时间来完成一项任务，因为 WIFI 太弱了。假设一列火车上有 100 个(保守计算)这样的人。因此，仅从柏林到汉堡的这一趟旅行(2 小时)就浪费了 60 * 100 * 2 = 12000 欧元的人力资本....最好不要告诉任何公司支付他们的员工这次旅行的火车费和“工作时间”。

## [](#actually-this-article-is-about-tech)其实这篇文章是关于科技的

我不是第一次经历这种情况，但为什么这次我会如此激动，以至于我决定写一篇关于这个话题的博客？作为一名网络开发人员，我目前正在做一个实时聊天项目( [gramchat.me](https://gramchat.me) -请不要客气，如果我真的能在这里工作的话，登陆页面就会完成)，我想在这次旅行中完成登陆页面&的文档。

现在我感觉自己坐在这里，我的笔记本电脑，通常是支付我房租的设备，只不过是一个废弃的黑匣子....几乎我所有的工作流程都需要互联网，**我不能离线工作**。grrrrr

怎么会这样呢？通常我总是在有好的 WIFI 或移动网络的地方(柏林大城市生活),所以一些坏习惯溜了进来:

*   **开发工作**
    *   谷歌字体
    *   需要配置的支付网关
    *   文档(在 stackoverflow 之前怎么会有人写软件？)
    *   用于及时下载依赖项的打包工具
    *   Github 问题和组织委员会
    *   后端基础设施建立在 AWS lambda 之上(不能离线测试)
*   **娱乐**
    *   电影在网飞上映
    *   音乐在 spotify 上
    *   我主要阅读博客文章和网络文章(通过[Hackero](https://hackero.co)；))
*   **通信**
    *   电报/WhatsApp/电子邮件
*   **信息**
    *   作为非母语人士，我很难写这篇文章，因为我不会使用谷歌翻译
*   **...以此类推**

* * *

短暂中断:由于其他问题，我不得不换乘另一列(慢一点的)火车。这个根本没有无线网络...所以现在是下一关。

* * *

我坐在这里，基本上有三个选择:

*   与 LMMS 一起创作电子音乐，我几周前下载了它，但不知道如何使用它
*   **在 Go** 中编码一些东西。感谢 Goland 在标准 lib 文档中的出色构建！
*   写这篇文章抱怨德国的火车状况，抱怨我是如此依赖一种我认为像空气一样自然的资源

所以我在这里写这篇文章:D

## [](#prevent-such-a-situation-in-the-future)防止将来出现这样的情况

所以最大的失败是我没有为离线使用我的设备做好准备。那我以后要怎么做才能防止这种情况发生呢？技术问题需要技术解决方案:

*   **娱乐**
    *   音乐:至少有一些我喜欢的播放列表可以离线使用
    *   电影:事实上，我认为看几个小时电影没什么大不了的= >让我专注于工作
    *   获得一个离线的“以后再读”系统。不久前，我使用了 wallabag，并将在我所有的设备上重新安装它。
*   **通信**
    *   你实际上对此无能为力...所以这里没有什么需要改进的
    *   如果你没有可以离线使用的电子邮件和信息客户端，你应该给自己买一个。(Telegram 有一个漂亮的独立桌面)至少能够通过存档的电子邮件/聊天进行搜索是很好的
*   **信息**
    *   那就离线字典吧
    *   有没有一个 Firefox/Chrome 插件可以将我访问的所有网页保存到离线存储器中？这样我就可以回到我的历史中，访问我以前访问过的页面...如果没有，我可能会编码一个。
*   **开发工作**
    *   有很多不同的离线代码文档系统。我确实选择了 [zeal](https://zealdocs.org/) ,因为它可以在 Linux 上运行，并且是独立的(其他的可以在浏览器中运行，因为我大部分时间都在私人模式下冲浪，它们对我不适用，因为我至少每隔几天就要清除本地存储)
    *   离线 PHP 服务器= >实际上很容易。你知道 PHP 有内置服务器吗？`php -S localhost:8080`
    *   AWS lambda 离线测试框架？还不知道该怎么做...也许这是另一篇博客的好话题
    *   github 实际上有一些支持离线发布的客户端。我会给他们一个尝试
    *   在本地缓存/保存 web 资源。总的来说也许是个好主意..最好不要把谷歌作为你项目的依赖，因为他们会滥用你发给每个访问者的数据
    *   有一个(非常古老的)StackOverflow 转储，它可能会在一些工具中搜索它...会很棒的。(但可能会占用大量磁盘空间)

* * *

哦，女孩，另一件事出现了:我必须出示我的火车票，这是一个 PDF 格式的电子邮件附件...我从没下载过。这是怎么回事...没有网络，我的生活变得疯狂。**下载您的重要机票/文件**

* * *

所以总的来说，这次旅行让我明白了我是多么依赖互联网，我应该改变这一点。请把这篇文章看作是正在进行的工作，因为当我对离线工具有了更多的了解和经验后，我会更新和添加它们。

总的来说，有一个主要的学习方法:**下载东西，而不仅仅是在浏览器中打开。**(我的大学 pdf 也是如此...从来没有下载它们离线使用，所以我没有研究没有)

如果有人处于这种情况下，他或她自己发现了其他有帮助的工具，如果你能与我分享，我会很高兴，这样我就可以将它们引入我的堆栈并更新这篇文章。

所以现在我希望我的移动热点上的 Edge 互联网连接足以上传这篇文章

祝你玩得开心！

西蒙

我发现的另一件事是:检查你的机器上有哪些应用程序在使用互联网，这样如果你的带宽很低，这个重要的资源就不会被一个打开的 Skype 之类的软件吸走。
