到 2021 年，与网络犯罪相关的损失预计将达到每年 6 万亿美元。每天大约有 24，000 个应用程序因恶意内容而被阻止。迫使组织关注移动应用安全性的原因有很多。它可能是勒索软件攻击、恶意软件、特洛伊木马、鱼叉式网络钓鱼、数据泄露或任何病毒。公司每年因网络犯罪损失数百万美元。手机 app 总数超过 2000 亿。

因此，对于一家在线企业来说，执行[移动应用测试](https://www.pcloudy.com)以维持客户的信任是至关重要的。虽然建立一个设备实验室来测试各种设备上的应用程序是有问题的。这将意味着巨大的投资。pCloudy 可以为您解决这个问题，因为它提供了一个设备云，可以在 5000 台设备上执行各种类型的浏览器组合。它加载了未来功能，如人工智能测试机器人“Certifaya”和“FollowME”，这为手动应用测试带来了并行性。

移动应用测试包括认证、授权、数据安全、黑客漏洞、会话管理等。下面我们来盘点一下 2019 年热门的评分最高的移动 app 安全测试工具。

1.  Qark
    Qark 是一款开源静态代码分析工具，它提供了有关 android 应用程序相关安全风险的信息，并对该问题进行了简要描述。Qark 代表快速 android 审查工具包，它用于 Android 识别 APK 文件和源代码中的安全漏洞。它以 APK 的形式创建一个用于测试的定制应用程序，并识别潜在的问题。它不仅会生成关于潜在漏洞的报告，还会提供如何解决问题的信息。

2.  (OWASP) Zed 攻击代理
    开放 web 应用安全项目(Open web application security project，简称 OWASP)是一家专注于提高软件安全性的非营利组织。ZAP 是跨平台的，它在客户端和你的网站之间创建了一个代理。当你浏览网站的所有功能时，它会捕捉你的动作。然后，它会攻击您的网站并生成漏洞报告。

3.  Android Debug Bridge
    ADB 或 Android Debug Bridge 是一个命令行工具，用于与运行在 Android 上的设备进行通信。它提供了一个终端界面来控制你的 Android 设备连接到一台电脑使用 USB。ADB 可用于运行 shell 命令、传输文件、安装/卸载应用程序、重启等。您还可以使用这些命令备份和恢复您的 android 设备。ADB 有三个组成部分，因为它是一个客户端-服务器程序。首先是客户端，你的 android 设备所连接的电脑。我们将从这台计算机通过 USB 电缆发送命令。第二个组件是 daemon，它是一种运行在计算机和设备上的服务，允许设备接受和执行命令。第三个组件是服务器，它是一个管理客户端和守护进程之间通信的软件。

4.  Drozer
    Drozer 是一个针对 android 安全评估的攻击框架，它允许您扮演一个应用程序的角色，并通过底层操作系统和 android 进程间通信机制与其他应用程序进行交互。要使用 Drozer 执行安全评估，用户必须在控制台上运行命令，然后 Drozer 将它们发送到设备以执行相关任务。

5.  MobSF(移动安全框架)
    移动安全框架是一款开源的[自动化安全测试工具](https://www.pcloudy.com/mobile-app-testing-on-multiple-devices/)，适用于 Android、iOS 和 Windows 平台。它为移动应用安全测试执行静态和动态分析，解决 web 服务的问题。MobSF 支持二进制和压缩源代码。它支持使用 API fuzzer 的 API 安全测试。

结论
测试人员可以根据应用的类型和他们需要执行的安全测试的类型，选择这些移动应用安全测试工具中的任何一个。这些工具各有利弊。由您决定哪种工具适合您的移动应用程序。
