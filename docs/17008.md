# [](#where-i-am-currently-%F0%9F%97%BA)我目前在哪里🗺

说到 web 开发，我不能说我是中级。我对这个行业也不陌生。但是当谈到我应该使用什么时，我有时会在不同的东西之间翻转。

目前，我是 Laravel + Vue 用户。我从 PHP 开始，尝试了 Meteor🌠由于幻想在前端和后端都使用单一语言，最终在 Laravel 上使用 PHP(但加入了 Vue)。老实说，我可能最终还是会回到 Node，这取决于我能找到什么来使用它。

但是我想知道其他人使用什么栈，为什么他们倾向于坚持使用它们。

# [](#preempting-a-flame-war)抢占一场火焰战争🔥

我故意*而不是*问为什么你*不*使用不同的语言或堆栈。我遇到过很多关于 PHP vs. Rails vs .的争论(我倾向于回避)。NET w/ C#，以及动态与强类型语言。我相信它们不仅各有各的位置，而且不同的语言和范式更适合不同的人。个性和心态总是在这类决定中发挥作用。

不管怎样，我仍然很好奇是什么吸引其他人使用他们的工具集，以及他们倾向于用这些工具从事什么类型的项目。
