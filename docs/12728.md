# 阅读技术博客(方法)

> 原文:[https://dev.to/solaomi/reading-tech-blogs-how-to-16ka](https://dev.to/solaomi/reading-tech-blogs-how-to-16ka)

有很多精彩的科技博客，但是你不可能把它们都读完。那么，你如何决定谁该读，多久读一次呢？我发现自己阅读 teach 博客有两个原因:

1.  我需要一个问题的帮助或者一个我不理解的解决方案的解释。
2.  我开始对这个人和他们的写作技术或其他方面感兴趣。

### [](#problemsexplanation)问题/解释

第一个原因“谁”很简单，你只需谷歌一下你的问题，使用任何对你有用的东西。这种情况发生的频率取决于你，也许你只遇到过一次或几次这样的问题，当你开始谷歌博客的链接或堆栈溢出时，就会立即弹出(* ***咳咳*** *如何居中 div)。

### [](#interesting-people)有趣的人

围绕第二个原因的问题有点复杂，我倾向于根据这个人的受欢迎程度来划分我所做的事情。

对于最受欢迎的(想想保罗·格拉厄姆，DHH，彼得诺维格等。)，我其实并不直接阅读/跟进他们的任何博客或文章。他们通常对技术对话有着如此强大的影响力，以至于他们所有的作品都被其他人过滤掉，在他们自己的作品中引用或引述。我通常只是读他们的东西，因为它是以上述方式自然出现的。

对于那些比较受欢迎的人来说，我通常会做两件事。首先，我订阅他们的时事通讯，阅读他们在电子邮件中发送的任何感兴趣的东西。我还保留了一个博客的 tabs 文件夹，这样我就可以回去阅读他们的旧帖子。我目前正在读《艾瑞·阿德里诺昆》、[戴夫·塞德迪亚](https://daveceddia.com/)和[丹·阿布拉莫夫](https://overreacted.io/)，但我通常会根据我阅读的时间长短和/或我目前的学习重点来更换。

对于最不受欢迎的文章，我倾向于在它们来的时候阅读，不管我是在 twitter、reddit 还是论坛上找到的。由于内容较少，我倾向于不跟踪这些博客。

### [](#conclusion)结论

这基本上总结了我从科技博客中过滤所有信息的方法。你们是如何从博客中获取技术信息的？
