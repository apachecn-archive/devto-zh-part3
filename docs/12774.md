# 为什么信息构建在 UX 进程中是必要的？

> 原文:[https://dev . to/lambdatest/why-information-architecture-in-UX-process-is-a-必需品-2jci](https://dev.to/lambdatest/why-information-architecture-in-ux-process-is-a-necessity-2jci)

通过你的网站传达信息是一项敏感而关键的任务。您的 web 应用程序可能有很多要传达的内容，但如果您的内容结构不够好，那么这些信息将会以很高的反弹率对您进行反击，最终影响您的 SEO(搜索引擎优化)。设计良好的应用程序和网站，内容适当，更容易探索和理解。但它们不是瞬间产生的。以最终用户更容易浏览的方式构建网站的导航和内容，需要大量的规划、研究和时间。一旦信息发布，你还需要确保它以像素完美的方式显示给你的目标受众，而不管浏览器的差异。你可以通过执行[跨浏览器测试](https://goo.gl/seQqdQ)来确保。记住，表达很重要。

信息架构是构建内容的科学。这是一门在开始设计和开发之前组织网站的艺术，以确保它支持可查找性和可用性。当你将一个网站与一个建筑进行比较时，你会更容易理解，在这个建筑中，建筑师创建了一个包括平面图、楼层位置、门、窗等的蓝图。

类似地，UX 的信息架构(用户体验)创建了一个包含信息层次的蓝图。所以你知道什么需要显示在哪里，对终端用户来说多少才算多？

## [](#clearing-the-fog-between-information-architecture-from-technical-architecture)从技术架构中拨开信息架构之间的迷雾

虽然，一个信息架构师可能有杰出的技术专长，但反过来就不一定了。从 UX 的角度来看，技术团队并不是设计网站信息架构的正确选择。它们需要有经验的信息架构师的指导。

这是因为一个网站的设计是基于一个用户怎么想的，而不是技术团队怎么想的。信息架构师将通过评论、场景、用户访谈清楚地了解用户的思维方式，从而决定如何设计网站。

有时候，信息架构师需要技术架构师的帮助来解决一些后端或功能相关的问题。当网站的信息架构和技术架构携手工作时，结果就是一个满足用户所有需求的 web 应用程序。

## [](#principles-of-information-architecture)信息架构原理

架构师应该对功能有清晰的认识，并对要展示的内容有完整的指导。丹·布朗是 UX 的一名信息架构师，他提出了设计一个优秀网站所需的 8 条原则。

*   **物体原理**:内容就像一个活的有机体。它们有自己独特的行为、生命周期和属性。

*   **选择原则**:选择的数量要尽可能少。多了引起混乱。然而，这个概念是假设的，取决于网站的复杂性。如果网站内容太大，你肯定会要求多选。

*   **披露原则**:一旦用户更深入地挖掘网站，就应该让他们预览将要探索的信息。该信息可以以文本内容的元数据或图像和视频内容的一些快照或元描述的形式给出。

*   **范例原则**:在描述类别中提到的内容的同时，也需要举例。你可以使用一个有效设计的导航系统来安排具体内容或标签的例子。

*   **前门原理**:应该假设 50%的用户会使用不同于主页的入口点。网站的结构应该保证无论用户打开哪个页面，他们都可以流畅地浏览整个网站。

*   **多重分类原则**:网站内容应该分类，以增加用户对网站内容的清晰感。例如，如果您要设计的网站是一个博客，并且您有不同类型的内容，您将需要创建多个不同类别的子页面，而不是将所有的博客文章聚集在一个页面中。

*   聚焦导航的原则:网站的导航应该保持简单，混合不同的东西不是一个好主意。导航应该保持这样一种方式，用户只需通过阅读名称就可以清楚地了解菜单项。

*   **增长原则**:设计一个可扩展的网站以确保布局不会受到内容扩展的困扰是至关重要的。这将确保内容增长时布局不会受到影响。

## [](#pointers-to-an-effective-information-architecture-for-ux)指向 UX 有效信息架构的指针

网站的信息架构被认为是基础步骤。创建文档是最好的基础。在规划网站的信息架构时，请记住以下几点。

*   **明确公司的目标**:在开始计划之前，决定你想用你的网站做什么，你想如何实现你的目标。与利益相关者召开会议，找出答案。

*   **搞清楚用户的目标**:首先，你需要搞清楚会访问你网站的目标受众。创建场景，与可能的用户讨论，并找出用户将如何使用你的网站，出于什么目的。

*   **研究你的竞争对手**:研究与你相似的网站，找出他们是如何从 UX 的角度规划信息架构的。找出你的竞争对手的网站是否更容易导航，他们如何显示关键信息，以及使其有吸引力的东西。记住流行的网页设计是很重要的。以下是【2019 年的 19 个网页设计趋势。

*   **绘制网站地图**:一旦计划和内容准备好了，就该考虑用户如何访问这些内容了。适当地设计出不太复杂的导航，帮助用户轻松浏览你的网站。规划网站地图时，将你的内容分组并相应地绘制出来。

*   **跨浏览器测试**:就[跨浏览器兼容性测试](https://goo.gl/qics1M)而言，您的信息架构行为可能因浏览器而异。如此努力地管理网站的信息架构，却只能眼睁睁地看着它因为浏览器的差异而失败，这将是非常令人失望的。

## [](#value-of-information-architecture)信息架构的价值

如今，内容价值低的网站被脸书和谷歌屏蔽了。重要的是以这样一种方式设计网站，让用户发现内容是有价值的。然而，有时即使有了所有正确的内容，网站也无法将其正确地交付给最终用户。从 UX 的角度来看，背后的原因是信息架构规划不当。对于一个有价值的信息架构，记住以下几点是很重要的。

*   **用户的值**

根据研究，网站的信息架构必须满足用户的以下需求。

*   网站的内容应该是用户需要的东西。
*   内容应该激励用户，因为有时用户并不确切知道他们想要什么。
*   搜索结果应该是海量的，因为用户希望看到尽可能多的信息。
*   用户希望找到他们以前搜索过的项目。

*   **战略价值**

一个成功的网站信息架构可以在许多与业务相关的关注点之间建立一个联盟。企业主通常会考虑大局。他们更关心利润率、费用、客户群和其他因素。一个在工作中技术过硬的信息架构师会创建一个工作流，解决所有者和利益相关者的所有问题，并且为设计目标带来一定程度的清晰性，逐渐为整个产品策略做出贡献。

*   **商业价值**

如果客户，以及最终用户，对网站不满意，这意味着业务的损失。让我们来看看在企业价值中起关键作用的事实。

*   如果一个网站的内部内容有不良的信息架构，员工要花费大量的时间去获取信息。这降低了生产率。
*   如果客户不容易找到他们想要的产品，网站的销售可能会下降，导致商业利润和损失。
*   注册页面应该仔细设计。如果 UX 不好，用户或客户在签约前会三思。从 UX 的角度来看，信息构建在获取新成员方面发挥着重要作用。
*   网站信息架构在搜索引擎优化中也起着至关重要的作用。价值较低的内容或重复的内容会降低网站排名。

*   **技术价值**

网络上发生的一切都是信息技术神奇作用的结果。信息架构师的工作类似于抽象艺术，其价值只有当开发人员在特定产品中实现工作流时才能实现。作为一名信息架构师，您必须确保开发人员或设计人员在浏览您的工作流时能够获得清晰的感觉。

*   概念和语义建模。
*   与将要实现你的设计的团队交流。
*   关心数据库设计者。在实现之前，他们应该清楚地理解您设计的模型。

*   **设计值**

网站信息架构师的工作只是创建网站地图和页面流。下一阶段由设计师处理，他在开始编码阶段之前创建一个网站实际应该是什么样子的现实原型。您的工作流程应该解决某些问题，以确保设计人员在创建原型时不会遇到任何问题。

*   对设计师有意义。必须更容易理解网站的行为。
*   设计师应该了解用户将如何浏览网站。元素，它们的位置，从一个页面到另一个页面的流动应该是简明扼要的。

## [](#breaking-the-myths-of-information-architecture-in-ux-process)打破 UX 进程中信息架构的神话

这里有一些我经常听到的关于网站信息架构的常见误区。我还会谈到实际情况，以帮助消除围绕这个话题的迷雾。

信息架构与用户体验相同
完全不同。网站的信息构建是一个以各种方式满足用户需求的方式设计网站的过程。而用户体验是一个人在使用一个应用程序或产品时的感受。

无论是设计一个单页文件夹，还是一个带有复杂导航系统的 web 应用程序，信息架构师的主要工作都是处理复杂的信息，并以用户易于理解或导航的方式创建内容结构。他们通过执行可用性测试、使用完全以用户为中心的设计方法以及研究用户角色来做到这一点。

UX 的设计者不仅考虑导航和理解应用程序内容的容易程度，还考虑网站如何吸引用户。UX 设计师的主要工作是使用以用户为中心的设计，创造一个目标受众觉得有吸引力的产品，并长期保持参与，而不会感到厌倦或不耐烦。

基本的区别在于，一旦用户浏览了一个网站并存在，他可能会说两件事

*   这很容易
*   这太棒了，信息架构师的工作不需要他考虑网站是否给用户留下了永久的印象，简而言之，他们不需要考虑用户体验。但是 UX 设计师需要考虑 UX 和信息架构。

为了更容易理解，我们来看一个例子。如果你比较一下 Zomato 和 Bookatable，你会发现它们都以一种简单的方式提供了相同的功能。然而，在 Booktable 中，除了简单易用之外，该应用程序的设计和效果也非常有趣。

[![](../Images/859bec1b1c29c3fe3ac686aceef68dec.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--8n8AQ3Li--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.lambdatest.com/blog/wp-content/uploads/2019/02/bookatable-1-1.png)

不仅如此，两者的工作流程也有区别。信息架构师必须研究用户的需求、他们的目标，并进行一种竞争分析。然后他们创建页面流、站点地图和线框。一些可用性测试可能会被执行，并被传递到下一个阶段，在那里设计师做他们的工作。

另一方面，UX 设计师通过信息架构师创建的线框和网站地图，并致力于最终用户的感受和情感需求。比起布局和结构，他们更关注交互模型。通俗地说，可以说，如果你考虑一座建筑，信息建筑形成了基础、支柱和结构框架，而 UX 创造了墙壁、屋顶、绘画和建筑的其余部分。

用户只需点击几下鼠标就能到达他们的目标内容
这是另一个误解，很久以前人们开始使用拨号连接，并习惯于对速度慢感到沮丧。网站的目标是引导用户尽可能少的点击就能到达目的地。

然而，用户体验不在于一个人到达目的地有多快，而在于他是否享受这个旅程。你可以想保持多少次点击就保持多少次，但是你必须保持每次点击之间的内容对终端用户来说非常令人兴奋和吸引人。
[![lambdatest](../Images/5ca15e434455da9426a2aad75efd7d47.png)](https://goo.gl/83GMZL) 
**站点要有逻辑结构化**
人们有这样一种误解，认为一个站点要有逻辑地设计。作为一名信息架构师，我经常以我认为正确的方式设计网站，有时甚至会遭到我的主管的责骂。然而，他们忘记了一个事实，人类不是以一种逻辑的方式工作的。我们做决定，无论是好是坏不是基于逻辑，而是基于我们的感觉。

当你在万圣节逛杂货店时，你可能没有意识到南瓜被放在蔬菜架上。尽管从技术上来说，南瓜是水果而不是蔬菜。这是因为人们有南瓜是蔬菜的概念。
[![](../Images/efa09ddeda98a921af72b3ff6d42f52b.png)T3】](https://res.cloudinary.com/practicaldev/image/fetch/s--VkAcxzCh--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.lambdatest.com/blog/wp-content/uploads/2019/02/pumpkin-1-1.png)

也就是说，从 UX 角度来看，信息架构也是如此。网站的设计和工作流程应该基于终端用户的心智模型，而不是任何预先定义的逻辑。

**提供选项时要靠近 7！**
另一个常见的误区是，你应该在导航中拥有不超过规定数量的选项。这个神话起源于乔治·米勒写的一篇心理学论文，指出普通人在短时间内最多能记住 7 个项目。另一项研究表明，数字是 4，这就是为什么软件激活的序列号以及信用卡和借记卡号码被分组为 4 个一组。

其次，你不需要记住网站的选项。你可以直接在屏幕上看到。制定这些规则是为了避免像可用性测试这样耗时的阶段。然而，只有当您的页面在导航菜单中有了所需数量的项目时，它才会让用户满意。

**不分清主次，可能导致厄运**
人们有一种让所有人都满意的倾向，这种倾向往往以谁都不满意而告终。早先，对网站内容进行优先排序是一个严格的禁忌。你可以看到许多在线购物应用程序按照字母顺序对产品进行分类，这导致客户花费大量时间来搜索他们需要的东西。

然而，趋势正在改变。如果你看到像亚马逊，甚至网飞和 YouTube 这样的网站，你会发现这些内容不是按字母顺序排列的，而是根据你的浏览历史分类排列的。这样更容易找到你真正需要的东西，而不是你正在寻找的东西。

人们说得对，知识少可能有风险。与其盲目相信理论和规则，不如在实际应用之前研究一下理论是否可行。

## UX 流程中高效信息架构的关键

像所有其他技术一样，信息架构也有某些最佳实践，遵循这些实践将会产生一个设计完美的线框。我们来详细讨论一下。

*   不用说，你应该在这个人身上投入大量的时间。在从事网站信息架构的工作时，我总是花时间研究用户的需求，分析相似类型的网站，找出哪些是好的，哪些是坏的，最重要的是，如果可能的话，与朋友或同事讨论他们希望新网站有什么。

*   从 UX 的角度来看，这是信息架构最重要的阶段之一。通常人们使用笔和纸来完成，但是当工作流程交付给设计者时，这只会导致混乱。最好选择一个流行的线框工具，它允许你创建动态的线框和原型。这里有顶级的免费网络用户界面模拟工具。

*   **内容盘点**:在设计布局之前，你需要考虑你的网站会包括哪些元素。创建将包含在项目中的构件列表。这些元素可以包括文档文件、作者信息、元标签、描述和关键字、图像以及像音频和视频这样的媒体元素。拥有一个目录将有助于设计者以一种内容适合的方式来设计页面结构。他们也可以连接元素，使网站看起来完整。

*   **数据建模模式**:有一些特定的数据模型模式可以帮助你建立一个有效的信息架构。

*   **单页**:适合信息量有限的网站。像个人信息和单一产品广告这样的网站可以使用单页模型来设计。

*   扁平结构:更适合除了主页之外几乎没有其他页面的网站，每个页面都有同等的重要性。一个投资组合或一个有 4 或 5 个类别的小型博客网站可以使用这种模式来设计。

*   **索引**:大型网站中最常见的模式。主页是带有导航菜单的主页，用户可以使用导航菜单访问其他页面。以不偏不倚的方式对待每一个子页面，具有同等的优先级。使用这个模型可以设计一个电子商务应用程序。

*   **严格层次模式**:当子页有自己的子页或子元素时使用。对于复杂的站点，这种模型是合适的。
    一个常见的例子是一个组织为其员工提供的官方门户，员工可以使用该门户导航到他们需要的网站，如休假管理、财务、工资单等。
    [![](../Images/5d0bd89e4bd8125dd19a27d35e676ff0.png)T6】](https://res.cloudinary.com/practicaldev/image/fetch/s--L8B4f12z--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://www.lambdatest.com/blog/wp-content/uploads/2019/02/pasted-image-0-43.png)

*   **共存层次结构**:这适用于可以从多个父页面或更高级别的页面访问子页面的网站。如果你的网站有很多可能相互重叠的信息，这种模式是合适的。
    T3![](../Images/4d898e7117976ca8b7c1f1f4df2f1ec8.png)T5】

*   **标注系统**:标注是用寥寥数语表示海量数据的艺术。我经常看到在一个集群页面中设计了大量的数据，足以让用户感到困惑。但是，这些数据可以压缩到标签中，使页面看起来更简单，更容易被最终用户处理。
    例如，让我们假设你正在创建的页面有大量的联系信息，比如电话、邮箱、地址等等。不要把所有这些信息都放在主页上，而是创建一个“联系”按钮，点击它，用户将获得所有需要的信息。您创建的按钮充当了一个标签。

*   分类法(Taxonomy):分类法指的是对信息进行分组，使网站看起来分类良好，便于最终用户浏览。
    例如，如果你正在设计一个购物应用程序，如果你把所有的产品放在一起，产品页面会看起来是聚集在一起的。取而代之的是制作小标签，比如服装、电子产品、家居用品等。并将产品分类排列在每个标签中。

*   **消除认知开销**:认知负荷可以定义为一个正常人脑所能处理的负荷量。在重新设计网站时，我经常遇到冗长的表单，需要用户在一个页面上填写太多的数据。如此多的动作会让用户感到紧张。
    在这种情况下，可以通过使用线性流将整个表单分成多个更小的部分来避免认知开销，只有在前一部分被正确填充时，才会显示一个部分。想知道更多吗？阅读我们在[上的博客什么是设计中的认知开销以及如何减少它](https://goo.gl/zvTmGj)？

*   **审计**:你的网站是一个内容的集合，是使用一个定义良好的网站信息架构开发的。然而，如果您不知道您拥有什么样的信息包，那么无论您是从头开始设计网站还是重新设计网站，规划一个设计良好的架构都会变得很困难。
    在您开始评估信息架构之前，与您的团队进行一次审核会议，并根据您需要什么和不需要什么来整理内容。

用户体验和决定网站未来的功能一样重要。就像混凝土是建筑的基础一样，信息架构也做同样的工作。精心规划网站的信息架构，使内容结构良好且易于浏览，不仅对最终用户是一种享受，从 SEO 的角度来看也是有利可图的。
