# 现代网络上的移动导航

> 原文：<https://dev.to/megazear7/mobile-navigation-on-the-modern-web-c89>

长期以来，网站一直使用顶部对齐的导航布局。去任何网站，控件和导航几乎总是在页面的顶部。有时它会在一个固定的位置，这样当用户向下滚动的时候，导航就会出现。除此之外，偶尔网站会有一个左对齐的导航系统。这些通常是上下文特定的；左侧导航的内容通常取决于您在网站的哪个页面。

这种左手导航方案在移动设备上效果不好，因为水平空间非常珍贵。然而，顶部对齐导航很容易过渡到移动设计。这很有趣，因为应用程序开发已经走上了完全相反的方向。人们早就知道，将应用程序中最常用的导航元素放在屏幕顶部会让用户很难接触到。随着手机变得越来越大，这个问题也越来越严重。因此，移动开发者将他们应用的主要导航功能放在了屏幕的底部。打开任何一个应用程序，你可能会发现事实就是如此。

手机网站设计也该效仿了。很多时候，如果一家公司有一个网站和相应的应用程序，移动版本的网站将更多地从桌面体验而不是应用程序体验中获得用户体验的设计。这通常是因为网站的移动和桌面版本的实现是整体链接的，而应用程序通常是完全不同的。不同的开发周期可能用于每一个，并可能导致不同的设计。不同的利益相关者和产品所有者可能会参与进来，这又会导致转移。

然而，长话短说，就设计而言，移动网站体验应该更紧密地联系到应用程序体验，而不是桌面体验。这方面最突出的例子是移动网站导航。事实上的标准应该是把它放在页面的底部。

我只是看了几个应用程序和移动网站的例子，它们都陷入了同样的陷阱。该网站将导航放在移动屏幕的顶部，紧随桌面版本，应用程序将导航放在屏幕的底部，紧随在移动设备上效果最好的应用程序。
