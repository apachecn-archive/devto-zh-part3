# 星期五爆炸#58

> 原文:[https://dev.to/horia141/friday-blast-58-57m6](https://dev.to/horia141/friday-blast-58-57m6)

[名称、级别和校准(2018)](https://lethain.com//perf-management-system/)——威尔·拉森在构建科技公司*职业阶梯*的道路上的思考。这对于大多数技术组织来说都是常见的东西——几个梯子，梯子上的几个级别，以及每个级别的清晰指南。

[剖析一个随机数发生器(2018)](https://www.masswerk.at/nowgobang/2018/anatomy-of-an-rng)——*有史以来第一款*电脑游戏——太空战的 RNG 密码解析！从 1959 年开始。10 行左右的代码会产生很多问题。

[Titus 上的自动缩放生产服务(2018)](https://medium.com/netflix-techblog/auto-scaling-production-services-on-titus-1f3cd49f5cd7) -人们如何开始使用网飞的 Titus 来完成更复杂的工作，以及他们如何与 AWS 集成来简化自动缩放。这是一个很好的教训——使一些事情变得容易——在这种情况下，在容器中运行非关键服务——将导致广泛的采用，人们试图以更多更广泛的方式使用该服务。所以在技术上专注于做好*的某件事*，然后在此基础上成长，这很好。

[一刀切的数据库并不适合任何人(2018)](https://www.allthingsdistributed.com/2018/06/purpose-built-databases-in-aws.html)——亚马逊首席技术官沃纳·威格尔的见解。很好地概述了各种类型的数据库(以及 AWS 在这一领域能提供什么)。像 Capital One、Expedia 或 Zynga 这样的公司如何在这些东西的基础上构建一些真正的怪物系统也很不错。

[高级语言中的结构化并发性(2018)](http://250bpm.com/blog:124)——结合我在这里写的一些关于并发性的文章，这篇文章处理协程以及它们在高级语言中的行为——活变量和类似的程序构造。此外，取消一如既往地弹出和*是超级重要的*。
