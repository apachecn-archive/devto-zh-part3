## [](#the-new-design)新设计

[![Screenshot of homepage from 2019 design](../Images/7d5fdda5d3cafe831a3263607b83e39e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ddIArdIk--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/llpx1l6ix48z8i46ts70.png)

你可能注意到的第一件事是黑暗主题。我决定转行是因为:

😎它创造了一种干净的美感。

📖它可以在白天和晚上阅读。

🤷‍♂️:呃，我只是真的更喜欢黑色主题。

第二件事是*是的，我终于有一个标志*🎉！我实际上是在 2018 年 1 月做的，花了整整一年才最终使用它，这太疯狂了。我缺乏适当的图形设计技能，所以我很高兴它变得如此简单和聪明。有一个标志应该真的有助于我创造一个更容易识别的品牌。

好吧，所以我的旧设计的主要问题是，由于信息和动作的泛滥，有人访问我的主页时会不知道该做什么。我的意思是:

*   主导航有模糊的图标。电子邮件图标会把你带到时事通讯或联系页面吗？所有这些只是为了给只与博客相关的搜索框腾出空间。
*   我的描述只告诉你我是一个 web 开发人员，在哪里工作。任何想要真正了解我的人都必须知道点击导航中的人物图标才能进入关于我的页面，老实说，我希望大多数人那时已经离开了网站。
*   你可以看到我的文章和项目列表，但是为什么一个新的访问者会在意呢？我更希望他们先更多地了解我，然后他们可以通过访问相应的页面对我的内容表现出兴趣。
*   社交图标很小，隐藏在页脚。我想我只是没有像现在这样重视社交媒体。我也讨厌我有这么多链接，好像我在所有这些平台上都很活跃(我没有)。

新的设计解决了所有这些问题；我去掉了我的“关于”页面、单个项目页面、主页上的最新文章和项目、搜索等等。任何新来的人现在都可以登陆我的主页并快速了解我。导航只有另外 3 页，你可以在那里找到我的项目，播客和文章。最棒的是，我的主要社交媒体资料都链接在文件夹上方。

我前面提到过，这两种设计具有相似的品质。这是因为我相信创造永恒的设计，而不是时尚的设计，所以每次重新设计的改进都有助于品牌的发展。

> 我相信创造永恒的设计，而不是时尚的设计，所以每次重新设计的改进都有助于品牌的发展。

我喜欢在设计中加入一些其他方面，比如微妙的动画，希望你能注意到:

☀️:当你加载一个页面时，我的 logo 中的“阳光”会升起或消失。

👋我的主页上的手势和笔记本表情动画，但只有一次。

🚀我主页上的火箭表情符号有无限飞行的动画效果。

↩️:我的项目、播客和文章页面上的卡片有摇摆效果。

最后，我在文章设计上下了很大功夫。我只是想让内容尽可能的可读，我从[重构 UI 书](https://refactoringui.com/book/)中获得了一些线索。我试图将字符长度保持在 80 左右，并在每个字符、行和部分之间留出足够的空间。因为我对自己的平台有完全的控制权，而不是像 Medium 这样的第三方平台，所以我创造一个令人愉快的阅读体验来让人们回到我的博客是非常重要的。

## [](#the-tech-stack)科技栈

虽然我喜欢谈论设计，但我是一名开发人员。那么，让我们通过查看技术堆栈来了解有趣的东西。注意，如果你想仔细阅读代码，整个网站[在 GitHub](https://github.com/sunnysingh/sunnysingh.io) 上是开源的。

### [](#gatsby-amp-react)【盖茨比】&反应

我已经在使用 Node.js 中内置的名为 [Metalsmith](https://metalsmith.io/) 的静态站点生成器，但它缺少我最近在 [Gatsby](https://www.gatsbyjs.org/) 中遇到的许多功能。

以下是我决定与盖茨比重修旧好的主要原因:

*   它是用我已经熟悉的技术构建的: [Node.js](https://nodejs.org/en/) 和 [React](https://reactjs.org/) 。React 的可重用组件模型对于可维护性非常重要。
*   我不必过多考虑性能。许多最佳实践已经融入其中，这样我就可以专注于网站的建设。
*   代码和内容的热重新加载意味着我可以非常快速地对网站进行更新，而不必等待完全重建。

### [](#styledcomponents)💅样式组件

因为我已经在使用 React，这意味着我也可以利用我最喜欢的样式库， [styled-components](https://www.styled-components.com/) 。关于 CSS-in-JS 是否是一个好的样式化方法，仍然有 [Twitter 争论](https://twitter.com/hashtag/cssinjs?lang=en)，但是在几个项目中使用它之后，我已经决定它是。

对于样式化组件，我只需创建已经附加了样式的组件。它是作为普通样式表包含的完整 CSS，所以媒体查询、关键帧等等。照常工作。哦，是的，我们可以抛开 Sass 和其他预处理程序，现在我们只处理 JavaScript。

### [](#graphql-amp-mdx)📖GraphQL & MDX

我所有的文章都是用 [MDX](https://mdxjs.com/) 写的，用 [GraphQL](https://graphql.org/) 检索。像 React 一样，GraphQL 已经被 Gatsby 使用，这给了我一个很好的机会去熟悉它。现在它只是获取本地降价文件，但如果我想的话，我可以很容易地引入一个完整的 CMS。

MDX 非常酷，非常适合。它允许我在我的 Markdown 文件中包含 React 组件，这是我目前嵌入 YouTube 视频、CodePen 笔和 tweets 的方式。还没有什么新奇的东西，但是我的文章中还有一个可运行的代码演示组件(没有第三方嵌入)。

### [](#socialshares-feather-amp-react-kawaii)📣&卡哇伊

我认为还有一些其他的杂七杂八的图书馆值得一提。

当然，如果我不使用我自己的[社交分享](https://socialshar.es/)社交分享按钮，这将是一种犯罪，你可以在我的文章底部看到这些按钮。
