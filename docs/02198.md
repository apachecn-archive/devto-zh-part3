1.  以逻辑方式对代码块进行分组:

    *   将变量放在它们被使用的地方。
2.  使用短名称:

    *   将变量命名为“a”或“b”也可以。
    *   但是…我失去了上下文，对吗？…不要！如果你遵守规则 1，没有问题，你将减少代码的冗余和认知负荷。
3.  使用不言自明的名称:

    *   没有“mVal”、“tRows”等…
4.  有意使用新台词:

    *   就像你把一个文本分成段落一样，让你的代码中的新行传达一些信息。
