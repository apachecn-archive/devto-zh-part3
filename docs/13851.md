# 20 与您实际使用的代码扩展

> 原文：<https://dev.to/vip3rousmango/vs-code-extensions-youll-actually-use-46gp>

我注意到了几个 VS 代码扩展的帖子，除了几个之外，我对提到的扩展彻底失望了，所以我想我应该编译一下我个人使用的插件列表。

我认为这些对于任何想要充分利用 Visual Studio 代码的主要前端开发人员都是有帮助的。其中一些你已经有了——因为它们是平台上最受推荐的扩展——但是其他的你可能没有。

### 易于阅读

*   美化--通过缩小和适当间隔你的可怕的标记，使 HTML，CSS 和 JS 非常容易阅读。

*   [更好的注释](https://marketplace.visualstudio.com/items?itemName=aaron-bond.better-comments)——在你的代码中创建更人性化的注释。我都用这个。的。时间。对于我们的开发团队来说，这是一个强制性的扩展。

*   [书签](https://marketplace.visualstudio.com/items?itemName=alefragnani.Bookmarks) -它帮助你在代码中导航，在重要的位置之间轻松快速地移动。我将它与 MetaGo 结合使用，这样在编码时几乎不用鼠标。

*   [MetaGO](https://marketplace.visualstudio.com/items?itemName=metaseed.metago) -超快的光标移动/选择，专注于键盘使用。这永远改变了我使用 VS 代码的方式。说真的。

*   [日志文件高亮器](https://marketplace.visualstudio.com/items?itemName=emilast.LogFileHighlighter)——顾名思义，给出 VSCode。日志文件支持，所以你可以真正地阅读这些日志转储没有你的眼睛出血。

*   [指南](https://marketplace.visualstudio.com/items?itemName=spywhere.guides) -现在你不需要折叠和打开所有奇怪的元素来找出嵌套。这真的很棒，上面提到的美化和彩虹括号，下面。

*   [彩虹括号](https://marketplace.visualstudio.com/items?itemName=2gua.rainbow-brackets) -突出显示您当前所在的括号组，并为其余部分涂上不同的颜色，以便于识别位置。如果你是一个重度 JS 开发者，这是一个很好的扩展。

*   [图像预览](https://marketplace.visualstudio.com/items?itemName=kisstkondoros.vscode-gutter-preview) -在边框和悬停时显示图像预览。所以关键是检查我是否引用了正确的图像/图标。

*   [GitLens](https://marketplace.visualstudio.com/items?itemName=eamodio.gitlens) -肯定是使用最多的扩展之一；Gitlens 是 VS 代码的必备工具，它让 Git 体验变得更好。将这个与[配对。gitignore](https://marketplace.visualstudio.com/items?itemName=codezombiech.gitignore) 和[。diff](https://marketplace.visualstudio.com/items?itemName=rafaelmaiolla.diff) 扩展良好的覆盖范围。

### 更快编码

*   CSS Peek -灵感来自括号中一个类似的功能，叫做 CSS Inline Editors。VS 代码中我最喜欢的括号特性之一。

*   我们 lint 我们的 JS，为什么不把 LESS/SASS/CSS 也 lint 呢？非常适合快速清理松散的 CSS。

*   当然，你有 Gulp，Webpack，NPM，Grunt，但有时你想用实时浏览器重载实时编译/传输你的 SASS/SCSS 文件到 CSS 文件。这就是它的作用。

*   [Live Server](https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer) -最好的本地开发服务器，具有静态&动态页面的 live reload 特性(甚至 PHP！).

*   [Version Lens](https://marketplace.visualstudio.com/items?itemName=pflannery.vscode-versionlens) -将指定 package.json 的依赖项/开发依赖项更新至最新版本。推荐人:[米哈伊尔](https://dev.to/qm3ster)

*   DotENV -增加支持和突出显示。env 文件——我非常依赖的推送到 Heroku，Netlify 等...

### 漂亮截图

*   突出代码，用你的代码主题的颜色抓拍一个非常漂亮的截图。非常适合教程或文档，并且您希望提供代码示例。

### 多个实例

*   使用一个私有的 gist 来保存一个设置文件，这样你就可以在多个 VS 代码实例之间同步设置、代码片段、主题、文件图标、启动、按键绑定、工作区和扩展。我用这两个简单的命令来保持我的笔记本电脑、工作桌面和家庭桌面同步。

*   [EditorConfig](https://marketplace.visualstudio.com/items?itemName=EditorConfig.EditorConfig) -用在中找到的设置覆盖用户/工作区设置。编辑器配置文件。我用它来在每个项目的基础上为开发团队执行特定的规则。

所以你有了它，而我有很多其他插件更具体地针对我使用的环境(前端 Web + Azure Cloud ),希望你会发现这些扩展在你日常使用 VS 代码时很有用！

让我知道你是否已经在使用它们，以及你是喜欢还是讨厌它们。或者，我可能错过了一些扩展，应该尝试一下！
