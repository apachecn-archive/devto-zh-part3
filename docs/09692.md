# npm 对 2019 年的 JavaScript 预测

> 原文:[https://dev . to/graphql editor/JavaScript-predictions-for-2019-by-NPM-pie](https://dev.to/graphqleditor/javascript-predictions-for-2019-by-npm-pie)

在 [Node+JS Interactive 2018](https://events.linuxfoundation.org/events/node-js-interactive-2018/) 活动的演讲中， [npm](https://www.npmjs.com/) 的联合创始人兼首席执行官对 web 开发领域的发展方向、JavaScript 的未来等做出了一些预测。他还介绍了 npm 社区中使用的一些开发工具，讨论了最佳实践、后起之秀框架和那些逐渐被遗忘的框架。这些预测是根据 16，000 名开发人员对 JavaScript 使用趋势和最佳实践的反馈意见做出的，前四项预测是:

### [](#you-will-abandon-one-of-your-current-tools)你将放弃你目前的一个工具

没有什么是永恒的，这对于 JavaScript 中的框架和工具来说非常正确。受欢迎程度的平均高峰阶段持续三到五年，然后是缓慢的下降，因为组织必须维护他们现有的应用程序，但已经为新项目转移到较新的框架。对于开发人员来说，这基本上意味着学习新的框架比过于坚持他们当前的工具要好。

### 你团队中的某个人会带来打字稿...

...你应该心存感激！对 16000 多名开发人员的调查显示，46%的受访者使用微软的 [TypeScript](https://www.typescriptlang.org/) ，这是一个 JavaScript 超集，可以编译成常规 JavaScript。热衷于这种转换率的主要原因可能是 TypeScript 通过类型检查提供了额外的安全性。在 2019 年采用 TypeScript 可以给你带来真正的价值，尤其是如果你是一个更大团队的成员。

[![](../Images/36ce6189c34ce4e2d8b0c8cb72f95df6.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--kY7Wyd5C--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/7lyslrx0pqmy1lyahcyy.gif)

### [](#react-will-be-the-dominant-framework)React 将成为主导框架

尽管其增长在 2018 年有所放缓，但 [React](https://reactjs.org/) 仍继续主导网络领域。60%的 npm 调查受访者表示他们正在使用 React。npm 预测，2019 年将有更多的人使用 React 构建 web 应用程序，这将导致更多的教程、案例和 bug 被修复。

### 你需要学习 GraphQL

2019 年将是 [GraphQL](https://graphql.org/) 年。开发者很有可能在 2019 年及以后的新项目中使用它。GraphQL 由它最受欢迎的客户端库 Apollo 跟踪，继续流行。 [GraphQL](https://graphqleditor.com/) 它将成为 2019 年内的一支技术力量。
