# 你不够资格申请软件开发职位吗？

> 原文:[https://dev . to/charlesstover/are-you-under-qualified-to-apply-a-software-development-position-1kd 4](https://dev.to/charlesstover/are-you-under-qualified-to-apply-to-a-software-development-position-1kd4)

Reddit 用户 [orduk](https://www.reddit.com/user/orduk) 问道:“[当你申请现在的职位时，你有多不合格？](https://www.reddit.com/r/webdev/comments/blab5o/how_under_qualified_were_you_when_you_applied_for/)

关于什么是“合格”有很多误解众所周知，招聘广告有着可笑的最低要求:使用一种语言的经验要比它存在的时间长，对初级职位的高级要求，或者对应届大学毕业生的经验要求。许多大三学生觉得“你需要经验来获得经验”的障碍太真实了，这并不奇怪。

我想澄清一件事:

## [](#you-should-always-be-under-qualified)你应该永远在合格之下。🎯

用户 [vidro3](https://www.reddit.com/user/vidro3) 对 orduk 问题的顶部评论很好地反映了开发人员的体验:“大胆地假设我现在能够胜任我目前的职位。”

你的工作给你两次报酬——一次是金钱，一次是知识。这不仅适用于大三学生；这也适用于老年人。如果你只是通过其中一种方式获得报酬，那么你的报酬就太低了。如果你在工作中没有进步，你的职业发展就会受到阻碍。那么，根据定义，你一定是“不合格”，否则你的个人成长就会停滞不前。

这改变了人们对合格的看法。如果你希望在工作中成长，你的公司也希望你在工作中成长，那么你不可能完善或了解你工作的所有方面。你必须在一个能让你成长的水平上被聘用:你必须比合格的低*。这实际上意味着不合格是资格等级。*

## [](#red-flags)红旗🚩

好的科技公司会雇佣“不合格”的人，因为这个人有发展潜力。当一家科技公司想要一个资历过高的人时，这是一个危险信号。他们不希望个人学习任何新的东西或提高自己。“这就是你被雇佣的级别，这就是你永远的身份。不要指望加薪或升职，因为你的工作不会改变，你的绩效也永远不会提高。”

你应该总是不能胜任你申请的任何工作。候选人最大的卖点不是他们广博的知识，而是他们学习自己不知道的东西的渴望和能力。如果一个雇主把你作为一个初级员工投资，他们希望最终有一个对公司忠诚的高级员工。

## [](#underqualified-is-qualified)不合格就是合格💼

永远不要把“最低要求”当成一堵墙。无论如何都要申请，哪怕只是为了面试经历。我现在已经做了 17 年的 web 开发人员，我仍然很少满足所有的最低要求。

为了回答 orduk 的问题，我目前是世界上最大的云服务提供商的前端工程师。除了用于个人文件存储的 OneDrive，我一生中从未使用过云服务。我甚至没有使用过有助于我发展事业的服务；我总是自己托管我的应用程序。我没有使用无服务器的经验。直到面试前两周，我才开始学习系统设计——因为他们告诉我要为面试学习。我不知道大多数云服务是如何工作的，但我在工作中学习，除了薪水，我获得的知识是额外的补偿，这将使我作为一个人、一名开发人员和我的职业生涯得到发展。

## [](#its-about-selling-yourself)讲的是推销自己。💸

作为一名自学成才的前端工程师，没有学位是我最大的障碍之一。我花了很长时间才学会如何推销自己。我知道很多，我有很大的学习潜力，但我没有信心或软技能来知道如何在面试中展示这种潜力。我读的是最低要求的“大专学历”，放弃了。如果我当时知道我现在知道的，我会在我的职业生涯中走得更远！为了学习如何面试而去面试是我能做的最好的事情之一。即使我被拒绝了，我也从那些面试中了解到了什么是需求，我了解到了那些东西。然后，他们不再关心我的学位，而开始关心我:A)充满激情，B)渴望学习，C)深入研究资料以获得表面以下的理解。

最低要求不是最低要求。不合格是意料之中的。软技能是最大的财富，而不是学位或百科知识。你需要经验来获得经验是真的，但是知道如何将你的自学或实践经验作为真正的经验来推销才是从实践面试中获得的自信。

## [](#confidence-speaks-volumes)自信能说明问题🧠

举个例子，一个不太自信的陈述是，“我向 NPM 发布了我的一些 JavaScript。”面试官知道你懂 JavaScript 和 NPM。没关系。

一种自信的说法是，“我设计了一个可重用的组件，它可以扩展到所有的客户用例。对于 GitHub 问题，我让客户参与进来，以满足他们的需求，优先考虑功能，并维持我对他们的服务。”面试官知道你可以与不同专业水平的用户交流技术想法，可以设计跨团队工作的代码，可以考虑边缘情况，可以优先考虑功能，可以提供长期支持和维护。👌

我说的是两种情况下的相同代码。这两种说法来自同一水平的资格。一个是*卖自己*另一个是*卖空自己*。模拟面试对我来说创造了奇迹。我能够了解公司想要什么(“讨论一次你与上司发生冲突的时候，以及你是如何解决的。”)，在家练习谈论我的技能，没有压力或焦虑，然后充满信心地再次面试。冲洗并重复。

## [](#interviewing-is-winwin)面试是双赢🏆

面试不是一个你因为没有被录用而失败的测试。如果你没有被录用，你仍然可以学到一些东西。你面试的时候应该*用*这家公司。确定对他们来说什么是重要的。离开面试时，你应该比进入面试时更有见识。面试失败后，你会成为一名更好的开发人员，因为你知道一些前一天不知道的事情——你知道一些受欢迎的技术，你知道一些更优先的软技能，希望你知道接下来该采取什么步骤来获得你所缺乏的技能。

坚持下去，在每一个障碍中看到机会。
