# 未来证明自己是工程师的 5 个极其重要的技能🙌

> 原文：<https://dev.to/veebuv/5-extremely-important-skills-to-future-proof-yourself-as-an-engineer-4h0k>

每周一我都会思考和想象我在 5 年后会做什么，是的，我计划今天做得更好，但是这个计划也应该包括明天。

我告诉我在 five2one.com.au 的所有工程师，我们今天就需要这样做，这样我们的客户和我们都可以受益于工具的调整，使我们能够构建更好的代码和解决更大的问题。

以下是我对未来证明自己是一名工程师的 5 项极其重要的技能的看法:

## 1。学会与远程团队成员一起工作，并持有可延展的观点

没错，不管你喜不喜欢，远程工作将会继续发展。每个人都想拥有旅行和移动的灵活性，即使你不是那种人(我也是其中之一)，也要准备好提高你的沟通技巧来适应这种趋势。为什么？人们喜欢它，公司也喜欢它，成本更低，如果它是一家全球性公司，工程师们会一直保持清醒。检查出 gitlab 以及 zapier(惊人的公司，远程)。

第二部分是保持可塑的观点，坚定你的观点直到证明是错误的，自我让你一事无成，向其他人学习，我从一个比我小 4 岁的前音乐家新兵训练营毕业生那里学到了很多，所以不要坚持你的观点，以效率为目标-不要总是对的(你会学到更多-免费)

## 2。在创建辅助项目时，练习学习云架构/微服务/无服务器构建

是的，对于初创企业和大公司来说，服务器和孤立的机箱时间变得越来越昂贵。开始学习如何使用 docker 和 ECS 来创建代码部署模式。理解 lambda/serverless 基础设施是如何工作的——因为您在这些环境中用于开发的模式相对不同。从我看到的招聘信息来看，每个人都想要这种类型的工程师，而且这只会越来越多——所以让你自己占得先机吧！

## 3。将 ML/AI 与开发结合起来，我们处于从前端/后端到 web 开发+ ML 的“全栈”的最前沿

人工智能的确是个时髦词，但你难道不明白很多人是通过乘风破浪获得成功的吗？走在曲线的前面，学习 ML/AI 如何工作，这比编程更多的是数学，但老实说，一旦你开始，它并不那么难- Mark Frign Cuban 正在学习如何做 ML 算法-你也需要。术语“全面启动”将很快从 FE/BE 转移到 Fullstack/ML 甚至 ML/Devops

## 4。学习产品，你将学习在构建产品时的股份所有权和责任

对于这一点，我再怎么强调也不为过，作为工程师，我们的目标是实现愿景，也就是说，为客户/为喜欢我们产品并对其赞不绝口的人制造垃圾。了解客户是如何思考的，了解如何让企业赚钱，用你的执行技能来授权企业，如果你是一个技术项目经理，你在当今的环境中几乎是不可触及的

## 5。永远不要把自己局限在一种语言上，学习几种语言，保持一种主要的语言，但是要学会在多种语言中交叉合作

这是我刚开始的时候最大的错误，我只关注 Python 和 JS。世界上有如此多令人难以置信的语言——毫无疑问，它们都是一样的——然而，新的语言通常是为了解决旧语言造成的或没有解决的障碍而构建的。这意味着您将接触到智能模式，这将使您当前的编程技能迅速扩展。我的意思是，这是一个著名的说法，丹·阿布罗莫夫的 redux 方法完全来自榆树。节点的异步等待在 C#领域很久以前就存在了。

当你把大脑扩展到新的语言和模式时，你就把自己放在了火箭飞船上。
