# 作为一个群体观看会议讨论

> 原文:[https://dev . to/matmooredev/watching-conference-talks-as-a-group-552 e](https://dev.to/matmooredev/watching-conference-talks-as-a-group-552e)

本周在工作中，我组织了一次午餐会议，与其他开发人员一起观看了一次会议讨论。

我运行它是为了看看是否有足够多的人有兴趣把它作为一个常规的事情来做。一群人出现了，所以我打算继续下去！

这个想法非常类似于读书俱乐部——我们一起看一个演讲，然后我们谈论它。我的灵感来自我上一份工作中类似的事情。我们称之为“我们喜欢的会议讨论”,我认为它本身就是受到我们喜欢的论文的启发。

我真的很喜欢这种形式，因为我不太可能亲自参加很多会议演讲，尽管网上有很多精彩的演讲，但很难找到时间观看。但是，如果你把它变成一件社交的事情，就更容易为事情腾出时间，而且这是一个讨论我们在工作中遇到的一些问题的好机会。这也是一个很好的方式去了解你的直属团队之外的人。

# 我是如何组织的

我最初犹豫要不要组织任何活动，因为我不确定人们是否会感兴趣。但是我的一个同事向我提到了经营某种读书俱乐部的想法，所以我提出了观看会议讨论的想法。她对要看的东西有一些好主意，知道至少会有一个人出现鼓励我为它定一个时间。

最大的障碍是在办公室里找一个地方让我们见面，并有一个显示器来播放谈话。但是我和我的经理讨论了这个问题，他建议我们可以使用一块空地。他还建议，如果噪音是一个问题，我们可以在笔记本电脑上同步，然后戴上耳机观看。这最终不是一个问题，但它是一个很好的备份选项。

我还决定在周五运行它，因为我们的办公室那时非常安静(许多人在家工作)，所以更容易获得空间。

我在我们的开发者 slack 频道上发布了几次，首先是让人们提前几天知道它正在发生，然后在周五早上再次发布，然后是我们开始时的最后一次提醒。我们中的两个人是远程加入的，所以他们在 youtube 上观看，然后通过一个空闲的电话加入我们的讨论。奇迹般地，我们没有音频/技术问题，所以一切都很顺利。

我们观看的演讲是来自澳大利亚 RubyConf 的[驯服没有微服务的 Monoliths 】,这引发了一场关于我们自己的服务如何构建的讨论，以及确定在哪里引入代码边界的困难。我们还考虑了讨论中讨论的特定问题是否受到 Rails 框架的鼓励，这使得传递 ActiveRecord 模型变得非常容易。我们都来自不同的技术背景，所以讨论除了 ruby 之外的不同编程语言是很有趣的。](https://www.youtube.com/watch?v=-ovkkvvTiRM)

# [](#whats-next)接下来是什么

我希望将此作为一件常规的事情，但是我还没有决定多久运行一次。我可能会试着每周做一次，看看效果如何。

我已经建立了一个 trello 板来收集我们可以一起观看的其他讲座的想法，所以我将在接下来的几个会议中分享这些想法并挑选一些。

到目前为止，我只邀请了我所在部门的人(我们坐在自己的楼层，所以我们和其他人有点隔离)，但如果有足够的兴趣，我们可以邀请所有开发人员。这可能意味着要找到一个更大的空间，所以我要等等看事情会如何发展。

我还想为它取一个很酷的名字，因为到目前为止我一直称它为“技术交流俱乐部”,这并不太吸引人。如果你有任何想法让我知道！
