# HFOSS litreview1

> 原文：<https://dev.to/jks7743/hfoss-litreview1-i5m>

西蒙·塔瑟姆(Simon Tatham)的《如何有效地报告 bug》是我最近看到的一篇关于在合作项目中如何有效地报告 bug 的文章。Tahtham 是创建 PuTTY(一个常用的免费 SSH 客户端)的自由软件程序员。在这篇文章中，Tatham 对如何向程序员报告错误给出了非常详尽的解释。虽然我有很多调试我和其他人的代码的经验，但我还没有考虑过如何恰当地向我的同事阐述这些问题。我认为这篇文章可以帮助那些在与他人交流问题上有困难的人，以及那些想更有效地调试自己代码的人。

在文章的一个部分，标题是“所以我试着…”，Tatham 提出了一个许多人在代码中遇到问题时不会实践的概念，约束。Tatham 声称，当遇到一个 bug 时，许多人会立即开始尝试修复它，而不考虑所有事情。这可能会使一个简单的问题变得更糟。他提倡在做一些不可逆的事情之前进行适当的计划和研究。虽然我确实认为冰层并不总是像 Tatham 在他的文章中所说的那样薄，但我同意，在匆忙解决问题之前，停下来想想发生了什么，并制定出一个解决方案，这将对我解决问题的过程有很大帮助。

虽然我提到的前一节更多地是关于调试我自己的代码，但本文的其余部分非常关注向其他人的项目报告 bug。这篇文章很冗长，但它很好地描述了作为用户或合作者报告错误的正确方法。Tatham 讨论了如何向需要修复 bug 或帮助您找出某些东西不适合您的人提供简明、有用的信息。他希望人们提供如何重现问题的完整步骤，并非常具体地说明当事情开始出现问题时你会看到什么。
