# 2018 年回顾

> 原文:[https://dev.to/philnash/2018-in-review-3gnk](https://dev.to/philnash/2018-in-review-3gnk)

我从来不喜欢写一年的回顾。今年，我发现自己不仅在阅读，还在和别人的评论做比较。我意识到这不是特别有帮助，所以决定在 2019 年底给自己一些东西来比较自己。因此，这是我第一年的回顾。

以下是今年我引以为豪的一些事情:

## [](#professional)专业

2018 年 7 月，我在 Twilio 做了 4 年的开发者传道者。在这段时间里，这是一个真正有趣、有趣、艰难、累人和令人信服的角色。我的第五个年头已经过去一半了，我很高兴分享我的开发经验，帮助其他开发人员，并成为 Twilio 社区的一员。

## [](#development)发展

[GitHub 告诉我 2018 年我做了 582 次贡献](https://github.com/philnash?tab=overview&from=2018-12-01&to=2018-12-31)。这些贡献很好地结合了个人项目，例如 Twilio 应用程序和对其他开源项目的贡献。他们还发布了 5 个库；4 个用 Ruby，一个用 JavaScript。他们是:

*   [pwned](https://github.com/philnash/pwned) :用 Ruby 访问 pwned 密码 API 的库
*   用于压缩 Jekyll 站点中文本文件的三个 Ruby 库:
    *   [jekyll-gzip](https://github.com/philnash/jekyll-gzip)
    *   杰基尔-佐普利
    *   杰基尔-布罗特利
*   [web-share-wrapper](https://github.com/philnash/web-share-wrapper) :用 web 共享 API 替换共享链接的 web 组件

我也开始为 Bitly API 恢复我的 [RubyGem，因为它已经过时了。虽然我需要花更多的时间，但到目前为止进展缓慢。](https://github.com/philnash/bitly/)

在工作中，我接管了[授权设计库](https://github.com/twilio/authy-devise)的维护工作。这个库使得在[设计](https://github.com/plataformatec/devise)中用 [Twilio 的 Authy API](https://www.twilio.com/docs/authy) 实现双因素认证变得很容易。今年有一些很好的机会来改进这个库，尤其是在测试方面。测试一个插入依赖 Rails 的库的库有一些有趣的问题。

### [](#helping)顾不上

我喜欢帮助解决堆栈溢出问题。在 2018 年底，我的声誉是 37，030，这是我非常自豪的。2018 年我留下了 345 个答案，其中 169 个被接受。我还留下了 918 条问题/答案评论，要么询问更多信息，要么跟进。

我很幸运地将此作为我工作的一部分，因为它会占用我相当多的时间。我的大部分答案都是针对标有“Twilio”的问题，尽管你会对开发人员在应用程序中包含 Twilio 时遇到的各种问题感到惊讶。

### [](#learning)学习

在“派对特别晚”新闻中，我在 2018 年设法花了一些时间开始学习反应和角度。我从来不喜欢只在前端工作的应用程序(除非绝对必要，否则没有 JavaScript 你就不能主持 WebRTC 视频聊天！)和框架倾向于鼓励这一点。随着时间的推移，这种情况已经发生了变化，坦率地说，React 和 Angular 都太大了，不能再忽视了(可以说，在 2018 年之前也是如此)。

我很喜欢启动我的学习，2019 年可能会有更多这样的活动。当我发现新事物时，请留意更多的博客帖子。

## [](#writing)写作

这让我想到了今年的写作。在 2018 年之前，我于 2017 年 7 月在[我的博客](https://philna.sh/blog/2017/07/12/two-tests-you-should-run-against-your-ruby-project-now/)上写了最后一篇文章。我打算在 2018 年的写作中更加一致，这在今年的前 3/4 年有点效果。我在我的博客和 Twilio 博客上的最后一篇文章都是在十月份。尽管如此，结果并不都是坏的。我写道:

*   philna.sh/blog 上的 9 个帖子
*   Twilio 的 11 个职位
*   1 个用于[翻车保护杆](https://rollbar.com/blog/top-10-ruby-on-rails-errors/)的客座博文

总共 21 个帖子很好，但我希望今年能更好。更多的一致性再次成为主要目标。谢天谢地，我已经准备好了一些草稿和项目。

## [](#speaking)说话之间

[2018 年我讲了 12 次](https://philna.sh/speaking/history/)。与 2017 年相比，这是一个明显的下降，甚至比 2016 年更进一步。与此同时，我搬到了澳大利亚，那里的活动更少，附近的国家也更少，我很幸运地在过去做了一些演讲。

也就是说，说话更多的是质量而不是数量。我很幸运被选中在这些精彩的会议上发言:

*   [柏林 JSConf EU](https://2018.jsconf.eu/speakers/)
*   [墨尔本的网络方向代码](https://www.webdirections.org/code/speakers/phil-nash.php)
*   [NDC 悉尼](https://ndcsydney.com/talk/service-workers-beyond-the-cache/)
*   墨尔本 Voxxed Days
*   [Angular Conf 澳洲](https://www.angularconf.com.au/#speakers)(学 Angular 今年有回报了！)
*   墨尔本发展节

JSConf EU 无疑是我在这里的亮点，我多年来一直试图让一个演讲被接受，当我收到确认电子邮件时，我非常高兴。这个演讲也是我做过的最重要的演讲之一，主要是关于开发者在网络上使用推送通知和权限的问题。这可能也是我想到的最好的演讲题目。你可以在 YouTube 上看到[“激进的网络应用”。](https://www.youtube.com/watch?v=uo-UOvq3-0Y)

我也在当地集会上展示了很多乐趣。2018 年包括在 [MelbJS](http://melbjs.com/) 、[Web Meetup](https://www.meetup.com/the-web/)、 [RoRo Melbourne](https://www.meetup.com/Ruby-On-Rails-Oceania-Melbourne/) 和 [RoRo Sydney](https://www.meetup.com/Ruby-On-Rails-Oceania-Sydney/) 的会谈。

## [](#travel)旅行

2018 年，我去了悉尼、布里斯班、霍巴特、新加坡、普吉岛、巴厘岛、伦敦、柏林、纽约、圣地亚哥和旧金山。11 个不同的地方和 6 个国家(除了澳大利亚)。

我确实在那里走了几英里。我喜欢旅行，这既是工作的一部分，也是我喜欢度假的原因。这可能会很累，尤其是在工作的时候，但是如果有足够的恢复时间，这种长途旅行是可以持续的。能够回到欧洲参加 JSConf EU，并借此机会拜访我在英国的朋友和家人，也是一件非常棒的事情。

我期待着在 2019 年探索世界上更多新的地方。

## [](#personal)个人

2018 年是我在澳大利亚生活的第一个整年。这是美妙的一年，墨尔本是一个了不起的城市，有美味的食物，有趣的啤酒，很多事情可以做，最重要的是热情友好的人们。凯莉和我非常喜欢这里，今年我们在这里买了房子！这一点，再加上我获得了永久居留权，意味着我们将在墨尔本享受更长时间的生活。
