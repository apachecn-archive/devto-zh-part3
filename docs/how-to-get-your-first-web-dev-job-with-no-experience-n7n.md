# 如何在没有经验的情况下找到你的第一份网站开发工作

> 原文：<https://dev.to/flippedcoding/how-to-get-your-first-web-dev-job-with-no-experience-n7n>

一旦你学会了 web 开发，并且对语言有了很好的理解，就是时候了。是时候开始使用你的新技能开始新的职业生涯了。一开始这很吓人，因为你可能觉得自己还没准备好。就像生活中的其他事情一样，有一个策略可以帮助你更快地找到新工作。

## 准备好投资组合

如果你除了在某个课堂上做过的项目之外，没有任何 web 开发经验，那是行不通的，除非这些项目可以变成有价值的东西。值得投资的项目是展示你真正了解的东西。前端应该是完美的，后端也应该是完美的，如果你想要全栈的话。

这也不仅仅是为用户谈论它的外观和功能。你的代码也应该尽可能的干净和高效。至少，去掉任何注释掉的代码。所有的最佳实践都应该存在，你应该使用你的库的最新版本或者你申请的公司使用的版本。

你应该瞄准 3-5 个投资组合项目，为潜在雇主做好准备。使用 Git 是让他们看到你的代码库的最简单的方法，但是如果这不是一个选择，那就有一个他们可以访问的网站。这应该是你最高级、最复杂、最完美的作品，而且应该很容易让他们接触到。

## 准备好一些参考资料

至少，要有人准备好谈论你的职业道德。可能是以前的同事，也可能是非常了解你的人(与你无关)。因为你以前没有任何经验，任何潜在的雇主可能都不会指望你的推荐人谈论你的编程能力。

这里是你可以让他们震惊的地方。主动为你社区中的某个人做一个公益网站。这对你俩都是双赢。他们得到一个很好的网站，如果你做得好，你会得到一个投资组合项目和一个参考。

他们不会谈论你的代码有多棒，但是他们会谈论那些在 web 开发人员中很难找到的东西。比如良好的沟通技巧和项目管理。

## 展示你的非技术技能

除非你只是一个编程天才，否则作为一个笨拙、反社会的程序员的日子已经结束了。对于我们这些需要工作的人来说，你的非技术技能可能比技术技能更重要。

首先，如果人们觉得他们不能和你交谈，他们可能不会雇佣你。愿意把事情讲透，解释清楚，不要居高临下。另一件对你有帮助的事情是拥有令人难以置信的时间管理技能。每个人都喜欢你按时完成工作。

当你去面试、网上聊天或做其他任何能让你接触到你想工作的公司的事情时，展示你真实的个性。请记住，他们雇用你不仅仅是因为你会敲键盘，因为他们可以让任何人都这么做。

## 确保你了解面试的基本概念

如果你没有准备好，面试会摧毁你的很多自信。去面试之前，再通读一遍职位描述。思考他们使用的技术堆栈，并确定您是否能够回答一些基本的使用案例。

没人会问你 if 语句和布尔值。他们会问你如何编写代码来解决大多数情况下的问题。如果你不能解释如何做像数据绑定这样的事情，或者解释你如何从一个不同的类中继承一些方法，那就去学习这些东西的意思。

当你试图将面试问题从网站翻译成英语时，了解一些基本术语将会对你有很大帮助。另外，如果你真的做了一些好的投资组合项目，你会知道你在说什么。只要用正确的词就行了。

## 甘愿奉子成婚

在没有经验的情况下得到你的第一件东西是很难的，但也不是不可能的。有一个职位等着你。不过，它正等着你去完成它。这可能意味着你一天要做 3-10 份申请。

这没什么神奇的。你不断申请，不断提高，直到被录用。这将需要一些时间，你可能会开始怀疑所有这些额外的工作是否值得。坚持住。它确实变得更好。最棒的是，在你找到第一份工作后，找到下一份更好的工作就不会那么困难了。

如果你坚持不懈地申请工作，参加面试，寻求反馈，尽你所能，你会得到第一份工作。这不像电影里那样光鲜亮丽，但会让你得到工作。在这个过程中，你会学到一些技巧和捷径来更好更快地操作这个过程。

我知道，浏览招聘启事时，仅仅看到许多启事上都有“5 年以上的工作经验”会令人沮丧。不要气馁，因为如果你放弃了，你真的什么也找不到。继续筛选所有的申请，申请你有资格申请的所有东西。
