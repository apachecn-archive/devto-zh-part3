# 使用 Chrome 开发者工具

> 原文:[https://dev . to/flipped coding/using-the-chrome-developer-tools-19f 0](https://dev.to/flippedcoding/using-the-chrome-developer-tools-19f0)

每个开发人员都需要知道如何在他们选择的浏览器中使用开发人员工具，不管是哪种浏览器。它们都有相同的基本功能，所以一旦你知道如何在一个浏览器中使用开发工具，你就可以在所有的浏览器中使用它们。

没有人真正谈论开发者工具中的不同标签。这是你应该知道如何使用或者只是想出办法的事情之一。我至少会给你一个基本的解释，告诉你标签是什么，什么时候你应该使用它们。就我个人而言，我是一个 Chrome 迷，所以我所有的开发工具参考都是关于 Chrome 的。继续操作，在浏览器中按 F12，调出开发者工具。

## [](#elements)元素

当你有一个输入框不能和其他的对齐时，这就是你首先要做的实验。Elements 选项卡将显示您打开开发者工具的页面的所有 HTML。您将能够看到类名、id 和任何注释。

Elements 选项卡最棒的一点是你可以使用 CSS。有一个栏可以让你编辑页面上任何 HTML 元素的样式。这是解决所有奇怪的 CSS 问题的一个很好的方法，不需要为颜色变化这样简单的事情重新编译代码。

你甚至可以改变你的显示器来模拟手机屏幕。这确实有助于移动样式，因为你通常在笔记本电脑上看不到移动样式。Elements 选项卡是您应该真正熟悉的选项卡之一，用于 HTML 和 CSS 调试相关的所有事情。

## [](#console)控制台

通常你会在开发工具的底部看到一个小控制台。控制台选项卡是该选项卡的完整页面版本。在这里，您可以看到已加载页面的任何错误或输出。

如果您正在调试您的代码，您可以检查变量值，执行函数，甚至通过使用控制台更改变量值。它不会泄露太多信息，所以你必须有一些你正在寻找的特定行为。

大多数时候，你可能会把控制台放在开发工具的底部，所以这个标签并不是最重要的东西。如果你需要看到更多的控制台屏幕，这很方便。

## [](#sources)来源

这可以说是开发工具中最有用的选项卡。在“源代码”选项卡中，您可以查看、编辑为页面加载的代码文件，并向其中添加断点。当你在追踪一个 JavaScript bug 时，没有什么比这更好的了。这是您在页面运行时访问整个前端代码的方式。

只有在进行本地调试时，查看文件才真正有用。否则你会看到很多抽象的代码，看起来就像胡言乱语。但这是您可以放置断点来跟踪调用堆栈并找到值的来源的地方。

这也是您可以看到发送到 API 或数据库的值的地方。在所有标签中，这将是你花费最多时间的地方。这也是为什么你可能会让你的控制台停靠在开发者工具的底部。

## [](#network)网络

您是否曾经收到过一个随机的错误消息，而您在 Sources 选项卡中却找不到它？然后，您可能需要在网络选项卡中查找任何错误。由于 Sources 选项卡只显示前端代码，如果没有发现错误，最好检查一下后端问题。

“网络”选项卡会告诉你是否有任何错误的 API 调用。如果您的任何后端方法有错误，Network 选项卡将帮助您找到它们。您将能够看到发送到 API 的请求头，以查看是否有任何值看起来不正常。

不过，您在这里看不到任何后端代码。这将帮助您找出从哪里开始寻找后端，但您仍然需要做一些挖掘。

## [](#performance)表现

如果您的页面一直超时，并且您已经查看了一些其他可能的问题，请尝试此选项卡。在 Performance 选项卡中，您可以开始记录加载或卸载资源需要多长时间。如果您遇到页面超时问题，这将非常有用。

你必须记住在开始调用函数之前点击 Record，否则你将没有任何数据可以查看。它会显示每个调用所用时间的明细，还会显示代码的每个部分执行所用的时间。

老实说，Performance 选项卡并没有被经常使用。但是当它被使用时，它是一个查看位置以提高代码效率的很好的选择。

## [](#application)申请

当您有一个处理 cookies 或本地存储的页面时,“应用程序”选项卡最有用。您可以转到该选项卡，查看页面中的每个 cookie。如果您需要查看更新未显示的情况，可以查看缓存或会话存储中的内容。

此外，如果用户权限是使用 cookies 设置的，这也是测试不同用户权限的一种简单方法。它只是使改变值变得容易，而不需要做很多额外的工作。这也是一个清理你的缓存的好地方。

这是另一个你可能不会经常使用的标签，但是如果你知道如何使用它，它在某些情况下会非常强大。

你们中的一些人可能知道 Chrome 开发工具中有三个标签:审计、内存和安全。这些很少被使用，以至于其他的标签掩盖了它们。当然，在一些特殊的情况下，它们可能会被用到，我只是还没有遇到足够多的情况。
