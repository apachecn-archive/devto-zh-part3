# 我的编码起源故事

> 原文：<https://dev.to/cdw9/my-coding-origin-story-587e>

故事由[https://dev.to/kelly/whats-your-coding-origin-story-544m](https://dev.to/kelly/whats-your-coding-origin-story-544m)提示

从 1996 年我第一次有了一台电脑开始，我就一直在挖掘系统中我能找到的所有部分，看看我能做出什么样的改变和定制。这确实导致了不止一次的电脑完全无法使用，但是我能够学习一些 DOS 命令来使电脑再次正常运行。

我早期的 web 开发经历发生在 Angelfire 和 Geocities 上。我学了一些 HTML 来建立一个有一些丑陋的重复背景图片的网站，来展示我在互联网上找到的 MIDIs 和照片。

在高中的这个时候，我对音乐非常感兴趣，渴望成为一名制作人。当我没有被大学音乐系录取时，这个梦想就被搁置了，我有一段时间没有申报。我一直想辅修计算机科学。我不愿意把它作为专业，因为我不想整天坐在电脑前写代码。

大二的一天，我的辅导员告诉我这所大学的一门课程:教育技术和电信。这本小册子强调了几门必修课:摄影、多媒体、摄像，还有一些选修课，比如网络和游戏开发。我几乎当场就签了名，想知道为什么这不是每个人都想做的事情！

我喜欢 ITT 和 CS 的几乎所有课程。Java 入门对我来说超级简单。尽管我艰难地完成了第二个 Java 类，但我坚持了下来，在第三个类中，一切都开始变得明朗起来。我的专业侧重于网络开发。

大学毕业后，我找到了一份初级开发人员的工作。在那份工作中，我可以花时间学习 CSS，这在我的大学课程中是没有的。我不得不学习如何让风格在所有浏览器上工作一致，当时主要是 IE6 和 Firefox。正是在这里，我也开始学习 Javascript，它非常接近 Java，我明白它是如何工作的。

我能够精通前端开发，即使是在离开并远程完成所有工作，与同事没有太多互动之后。我的丈夫鼓励我在我能挣更多钱的地方找一份新工作，我希望能找到更多样化的工作。我在一家名为六英尺高(Six Feet Up)的小型初创公司找到了一份工作，从此一蹶不振。

这份新工作让我接触到了许多新技术，幸运的是我学得很快。到目前为止，我从未接触过内容管理系统、版本控制或 Python。作为一名模板开发人员，我不需要学习 Python，但是我在工作中接触了很多。

最终我决定学习 Python。当我们的高级开发人员去另一家公司工作时，我突然有了更多写 Python 的工作。这让我能够过渡到后端开发人员。还记得我在大学里提到过我不认为我会喜欢整天写代码吗？这就是我现在做的，我喜欢它。
