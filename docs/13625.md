# 博客文章:在哥本哈根发表演讲

> 原文:[https://dev . to/Jonas bn/blog-post-giving-a-talk-at-Copenhagen js-ll7](https://dev.to/jonasbn/blog-post-giving-a-talk-at-copenhagenjs--ll7)

我最近开始参加我的本地 **JavaScript** 用户组 **CopenhagenJS** 的会议。我绝不是 JavaScript 程序员，但当我开始评估和使用 Node 时，它们引起了我的注意，我使用的几乎所有开发者播客都推荐*离开那里*。

上周，一个有 3 个空缺职位的演讲人招聘活动开始了。

所以我想我不妨试一试，我已经参加了几次聚会，也许是时候回报一下了。我以前在国内外的本地用户组、研讨会和会议上做过演讲，但大多数都是围绕着 **Perl** 展开的。实际上，2012 年 4 月我在哥本哈根给[做过一次演讲](https://www.slideshare.net/jonasbn/stackato-v3)。

由于我不能做关于**节点**或 **JavaScript** 的教育演讲，我想我应该提供一些别的东西，我在很大程度上是一个新手，当谈到 **JavaScript** 和**节点**时，我仍然在试验和学习。

然而，我作为一个开源开发者/贡献者已经有很长时间了，也许我可以在这方面提供一些东西。所以我想了想我最近一直在做的事情，我决定做 **Markdownlint** 。

我向组织者承诺为我的演讲提供一个项目列表，这样他们就可以决定我的演讲是否有趣，所以这就是我的项目列表。

*   Markdown -什么是 Markdown？
*   Markdownlint -什么是 linter？对比 ESLint？
*   Markdownlint——Ruby 和节点实现——的简短而不准确的历史
*   Markdownlint 提供了什么？-一致性和正确性
*   林挺规则及其例外
*   工具链和编辑器集成- VSCode 和 Sublime Text
*   额外材料与 CI/CD 的整合- Travis CI 示例

这只是一个草稿，当幻灯片成型时，我可能会添加新的部分或跳过一些建议，最有可能的是添加内容。

我在开源项目中广泛使用了 **Markdownlint** ,这很好，因为这样就可以很容易地展示真实的例子，对我个人来说，这是一个很好的机会来组织我的笔记、实践和工具的使用。

我将为你们中感兴趣的人发布幻灯片，我可能会用我的演示文稿的文字来补充幻灯片，我发现如果你有时间，这是一个很好的练习，因为它通常可以评估信息流和演示结构。

祝我好运，让我的演讲被接受。
