# Web 组件:正确的方法。项目重启！

> 原文:[https://dev . to/web Padawan/web-components-the-right-way-project-re launch-20d 7](https://dev.to/webpadawan/web-components-the-right-way-project-relaunch-20d7)

到目前为止，Web 组件是最有争议的规范集之一。有人称它们为“失信”，有人[预测](https://frontendmasters.com/books/front-end-handbook/2018/2018.html)2018 年它们仍会“潜伏”等待开发者的兴趣。Web 组件走向采用的道路并不平坦。

此外，这条路真的很长。自从迪米特里·格拉兹科夫和亚历克斯·罗素最初的想法被证明以来，已经过去 7 年多了。现在到 2018 年底，Chrome、Safari、Firefox 都有实现。微软，包括 [VSCode 团队](https://github.com/w3c/webcomponents/issues/776)，也在致力于此。

组件化未来的想法激发了开发人员的热情。随处可见的早期采用者勇敢地在生产中使用 Web 组件(并发现兔子洞有多深)。他们分享痛苦和知识。

那是 4 年前，2014 年 12 月 1 日，Sindre Sorhus 将 [commit](https://github.com/sindresorhus/awesome/commit/07b9c6a8b3cc8ede102bc1fc6f2450994ef79326) 推送到他的“棒极了”列表中，用新项目更新它。在其他名单中，有一个名为“网络组件:正确的方式”的新名单，由蜜桃红·奥尔蒂斯在 2014 年 10 月的几个月前创建。

2014 年即将开始的炒作浪潮失败了，有一段时间不相信。Web 组件并没有像 React 那样迅速地进入浏览器，未来还不明朗，征服了人们的思想——戏剧性的故事，嗯。在某个时候,“正确的方式”列表变得很长时间没有维护。

你问我，现在怎么办？你猜对了: [Web 组件:正确的方式](https://github.com/mateusortiz/webcomponents-the-right-way)再次变得明亮和闪亮——Web 组件也是如此！我很高兴地宣布，从现在开始，作为一名合作者和维护者，我将尽最大努力更新列表并添加精彩的内容。

现在，我需要你们的帮助和社区的力量让它变得更好。我们需要一个集中的知识库，供任何学习如何构建 Web 组件的人使用。我们可以一起努力。不对，已经有了规范、博客文章和教程的链接——期待更多。

除了很容易找到的“介绍”级别的博客帖子，我们需要真正的用例——而不是营销宣传的形式，因为有时会发生这种情况。让我们收集不管具体的堆栈如何都可以使用的最佳实践，而不是挖掘特定于框架的细节。

如果你正在寻找一个地方来观察 Web 组件正在发生的事情，而不关注 w3c 的问题，那么这个 repo 是一个不错的选择。另外，如果你有一些要讨论的话题或值得列入清单的项目，请随时提交你的[问题](https://github.com/mateusortiz/webcomponents-the-right-way/issues/new)。当然，欢迎公关！
