# 更好的跨浏览器测试技巧

> 原文:[https://dev . to/lambdatest/top-tips-for-better-cross-browser-testing-1khe](https://dev.to/lambdatest/top-tips-for-better-cross-browser-testing-1khe)

Web 开发人员通常偏向于浏览器。他们可以争论几个小时，为什么他们最喜欢的浏览器的最新版本是所有 web 开发项目的最佳版本。

很多时候，他们在自己最喜欢的浏览器中工作，假设他们开发的代码也可以在其他浏览器上运行。但是他们系统中没有的其他浏览器呢？他们开发的代码也适用于所有浏览器吗？

在这里，跨浏览器兼容性测试开始发挥作用。正如我们所知，浏览器、操作系统和设备每天都在发展，因此最好将跨浏览器测试作为日常活动的一部分，以确保最佳的最终用户体验。

与此同时，与所有这些工具保持同步，并确保您的 web 应用程序能够按照预期运行，在质量上没有任何差异和妥协，这对于在这个互联网世界中触摸天空至关重要。你可以在这里阅读更多关于[什么是跨浏览器测试以及我们为什么需要它的信息](https://goo.gl/sXEamT)。

我做了细致入微的分析，并提出了一些在执行跨浏览器测试时要记住的技巧。

## [](#target-browser-os-configurations)目标浏览器操作系统配置

在开始测试 web 应用程序之前，首先要做的是找到测试 web 应用程序的浏览器。每个浏览器都有许多版本，一些浏览器如 Chrome 和 Firefox 至少一个月更新一次。大多数科技公司支持最新版本的浏览器，但我们不能离开仍在使用旧版本浏览器的用户群。这将限制我们对不同版本的浏览器进行测试。

发现不同屏幕分辨率的浏览器、浏览器版本和操作系统配置的另一种方法是数据采样。当我们的网站公开时，我们使用不同的工具，如谷歌分析或 Splunk 来跟踪用户数据。我们了解用户浏览器、浏览器版本、移动设备和操作系统的使用情况，并列出最常用的配置，以便更专注于测试。

你也可以阅读这篇文章来获得[为跨浏览器测试](https://bit.ly/2WZ6y4B)选择正确的浏览器列表的提示。

## [](#make-list-of-browsers-as-per-user-taste)根据用户喜好制作浏览器列表

监控用户浏览器的使用是为贵公司获得最佳跨浏览器测试体验的最有效和最关键的因素。您可以提取用户浏览器和设备消耗报告，并根据用户基数确定它们的优先级。在对你的 web 应用进行跨浏览器测试之前，创建一个不同参数的浏览器操作系统配置列表，比如优先级、使用率、可用性等。

## [](#use-smart-tech-ways)使用智能科技方式

在开始浏览器兼容性测试之前，不要做任何假设。对于不同的浏览器元素，不同的浏览器表现出不同的方式。让我们举一个例子:一个数据选择器浏览器属性将在 Chrome 上打开并正常工作，但呈现方式不同，并且在 FireFox 上的月份导航器中有一个错误。

在公开发布之前，请查看此[跨浏览器测试清单](https://goo.gl/M9PfMW)。

## [](#emulators-and-virtual-machines)仿真器和虚拟机

跨新旧浏览器操作系统组合测试网站对于跨浏览器兼容性是必要的。我们可以使用模拟器或虚拟机进行测试，两者都有自己的好处。

许多[基于云的工具](https://goo.gl/BH3hed)提供了一组不同配置的模拟器，可以在所有浏览器版本上复制网站的确切外观和感觉。
你可以在这些模拟器上测试你的网络应用，只需要很少的努力和微不足道的预算。

或者，虚拟机更可信，因为它们被设计为使用特定的浏览器版本。这将让我们了解该网站如何面向利基用户。

## [](#mobile-first-world)移动第一世界

在过去的 20 年里，我们看到了移动应用的指数级增长。从 1998 年的 3.18 亿增加到 2018 年的 77.4 亿。移动用户正在广泛征服互联网世界。考虑到这一点，每个移动第一 IT 公司都非常关注不同移动设备上的移动用户体验。对他们来说，跨设备测试总是处于最高优先级。

每天都有新的移动设备以不同的屏幕尺寸和视窗发布，与它们同步并设置内部基础设施进行测试并不是一种有效的方式。了解了这些问题，许多公司提供了一个基于云的在线平台，在这个平台上，您可以启动所需配置的移动设备，并轻松测试您的网站和 web 应用程序。

您还可以使用谷歌浏览器[检查您的网站](https://goo.gl/Kmj3kD)在不同屏幕尺寸和视窗的不同移动和平板设备上的响应性。

## [](#dont-underestimate-tablets)别小看平板电脑

平板设备覆盖了全球约 4%的互联网市场份额(来源:statcounter.com)，拥有广泛的用户基础。8%的美国人使用平板设备来满足互联网需求。在不同屏幕尺寸和分辨率的不同平板设备上查看您的网站对于获得出色的用户体验非常重要

## 自动化测试

所有设置与用户浏览器操作系统的使用？我想到的第一个问题是如何在所有配置上测试网站？

浏览器-操作系统组合超过 2000 个，对每个配置手动执行测试是一项非常痛苦且重复的任务。

自动化 web 应用程序测试有助于解决这个问题。这是很容易执行，它节省了很多时间。通过自动化测试，您可以编写测试脚本，并在不同的浏览器和操作系统组合中测试您的网站。有许多在线自动化测试工具，它们提供浏览器、操作系统和移动设备

## [](#test-before-going-live)上线前测试

最佳实践是在公开发布之前进行跨浏览器测试。当您的 web 应用程序托管在本地服务器上时，请始终对其进行测试。这对于保持网站良好的用户体验非常有帮助。当你使你的网站活跃起来时，这可以避免你意想不到的失误。

## [](#take-care-of-accessibility-too)也要考虑可访问性

让每个人都能访问你的网站吗？这是一个非常有趣的讨论，不同类型的人可以成为你的用户。可能是一个听不进去的人，一个看不见的男孩或者一个色盲的人，使用屏幕阅读器来阅读你的文本的人，或者有一些运动障碍的人使用键盘和快捷键等非鼠标方法来使用网络。有必要让每个人都能访问你的网站。因此，确保每个用户都能访问你的网站是'[可访问性测试](https://goo.gl/A8hQY7)

## [](#use-appropriate-tool-for-testing)使用合适的工具进行测试

要进行测试，必须使用最好的工具。然而，发现这一点是一个艰难的决定。但是市场上有许多可用的测试平台，选择一个适合您业务的平台是一个至关重要的决定。也要看你的要求。
