如果你像我一样，是一名经验丰富的软件开发人员，没有计算机科学学位，以下是你如何从脸书这样的顶级科技公司找到工作的方法。

## [](#background)背景

在申请脸书大学之前，我已经在总部位于休斯顿的咨询公司 Poetic Systems 工作了三年。如果你想更多地了解我，这是我的 Linkedin。

## [](#timeline)时间轴

*   05/11/2017 我提交了我的脸书职业简历。
*   一名脸书招聘人员联系了我，我们安排了电话面试。
*   09/26/2017 电话面试。
*   2017 年 9 月 29 日招聘人员通知我，我通过了电话面试，我们安排了现场面试。
*   10/16/2017 现场面试。
*   2017 年 10 月 23 日我被告知脸书需要一次额外的电话面试来决定我的情况，我们安排了另一次电话面试
*   2017 年 10 月 30 日电话面试。
*   2017 年 11 月 9 日招聘人员告诉我，我被录用了。

## [](#find-the-right-job)找到合适的工作

我应聘的工作是“软件工程师，企业”。在我之前的公司，我接触过不同种类的网络应用。我真正引以为豪的一个项目是定制企业应用程序构建器。我认为我在这方面的专业知识可以给我一个更好的机会，因为这个项目与他们想要招聘的人相关。

## [](#write-a-concise-resume)写一份简洁的简历

这是我提交给[脸书职业](https://www.facebook.com/careers/)的[简历](https://docs.google.com/document/d/1NYSpOMyNElvszKvXj628ifn0jerD7nR_2EOy-1Esznw)。我在谷歌的好朋友[杰瑞·史](https://www.linkedin.com/in/guangjie-jerry-shi-0ab14b38/)在我的简历上帮了我很多。我学到的一件重要的事情是我的简历应该是一页纸。招聘人员没有足够的时间检查我参与的每个项目。我需要通过将我的价值浓缩成快速可消费的信息来帮助招聘人员。

我有一个在 Linkedin 上记录项目的习惯。这帮助我准备了我的简历。此外，在亲自面试之前，我回顾了我以前的项目，以便提出我解决的有趣问题和我学到的教训。准备工作让我平静下来，增强了我的信心。招聘人员和面试官都在一定程度上询问了我以前的经历。能够谈论一些有趣的项目对我来说绝对是一个优势。

## [](#interviews-preparation)面试准备

[![Leetcode](../Images/16504cbcb1ad5a00c2d33ae2193bdd3d.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--1KJjah-s--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://trentyang.com/assets/image/self-taught-to-facebook/leetcode.png)

### [](#read-an-algorithem-book)读一本算法书

我没有计算机科学学位，但我确实读过一本算法书(罗伯特·塞奇威克和凯文·韦恩的《算法》)，甚至在我的第一份工作之前，就在 Coursera 和 Udacity 上完成了几门计算机科学课程。我肯定会推荐任何软件工程师去读一本算法书，尤其是当你没有计算机科学学位的时候。这不仅对面试非常有帮助，如果一个软件工程师想开发相当复杂的程序，他/她绝对需要知道基础知识。

### [](#preparation-strategy)准备策略

我主要用 leetcode 准备面试问题，因为他们有真正的**面试问题。Geeksforgeeks 是我用来了解算法实现基础的另一个网站。**

2016 年 10 月左右开始准备编码面试。最初我在准备 2017 年 3 月左右的谷歌面试。真是千钧一发，但不幸的是我没有被录用。

*   前两周，我的准备策略是死记硬背。我试着自己“有机地”解决问题。这绝对不是正确的心态。这种方法太费时间了。我没有学到太多东西，因为我只是试图自己解决这些问题。

*   大约两个星期后，我意识到我无法用我的核心策略解决大量的问题(300 个是我的目标)。我决定从 leetcode 论坛上阅读解决方案。leetcode 论坛绝对是个金矿！顶级解决方案非常优雅，许多解决方案包含详细的解释，一些作者对一类问题进行了总结和概括！

*   大约一个月后，我意识到这些问题有不同的主题，例如，动态规划，二分搜索法等。我可以涵盖更多类型的问题来检查每一个主题，而不是仅仅试图通过它们的索引号来解决问题，这在事后看来似乎有点武断。所以我从每个题目开始解决几个问题。

在解决 leetcode 问题和重读算法书来提高我的基本算法技能的过程中，有很多来回。我觉得我对算法的理解有一个艰难的过程。当我第一次阅读这本书并学习课程时，我从未用真正的代码解决过大量的算法问题。leetcode 挑战是一项长期而艰巨的任务。

算法训练是一个耗时的过程。我有一份全职工作要做，还有一个新生婴儿要照顾。幸运的是，我的家人非常支持我！我婆婆和我妻子负责家里的大部分家务。我真的很感激！

最后，我用 python 和 Java 对前 400 个问题进行了两次以上的编码。因此，面试中的编码问题对我来说并不困难。一些面试问题实际上与 Leetcode 中的问题非常相似。

### Java 中的代码

如果你还不熟悉 C 或 C++的话，我会推荐 Java。原因如下:

*   在我的脸书面试中，语言并不重要。但这对我的谷歌面试很重要。在谷歌，核心语言，如 Java、C 或 C++，是我在上次面试中才意识到的必备技能。
*   Java 包含了非常广泛的内置数据结构。例如，TreeMap 类是红黑树的一个实现。这种数据结构不是 Javascript、Python 或 Ruby 内置的。
*   大多数在顶级科技公司工作的人都拥有计算机科学学位，而且由于他们所受的教育，他们通常都知道 Java。使用 Java 对他们来说更容易。

### [](#what-i-could-have-done-better)我本可以做得更好

*   花更少的时间停留在一个问题上，更快地从论坛获得解决方案(花 15 分钟没有任何线索是寻求帮助的好迹象)
*   花更多的时间在论坛上阅读解决方案，因为最好的解决方案非常优雅，值得学习，即使我自己解决了问题
*   花更多的时间回顾以前解决的问题，而不是试图征服更多的问题，系统地回顾，并在回顾时遵循专注于算法的计划，而不是再次编码解决方案，这真的很耗时
*   购买订阅并使用按公司分类功能

### [](#talk-out-loud-in-interviews)在面试中大声说话

当然，我不可能猜到面试中会有什么问题，但我可以准备一个策略。要记住的一件重要事情是，面试是为了评估你的能力。帮助面试官的最好方法就是大声说出来。无论你在想什么，你都应该把它表达出来。在白板上用最有效的算法正确编码是非常重要的，但是在面试中传达我们的思维过程也是非常有价值的，因为这给面试官发送了更多有用的信号，尤其是当你不擅长编码的时候。

## [](#interviews-process)面试流程

[![Interview from https://media.giphy.com/media/FcTdv3xQKhCIE/giphy.gif](../Images/577b2a78b68124e860a4f171a2b7a33a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--o8eVYeMX--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://trentyang.com/assets/image/self-taught-to-facebook/interview.gif)

### [](#phone-interview)电话面试

电话面试是一个筛选过程。看来脸书利用这次面试来迅速筛选掉不合格的候选人。我被要求在像 Google Docs 这样的编码环境中解决一个算法问题。这次面试对我来说不太容易改变。

### [](#onsite-interview)现场面试

通过电话面试后，我被邀请参加现场面试。因为我住在休斯顿，我不得不飞到门洛帕克去参加面试。脸书安排并支付了整个行程，他们真是太慷慨了。

以下是现场面试当天的日程安排:

*   系统设计面试
*   编码面试
*   与一位工程师共进午餐(午餐期间我没有被评估)
*   编码面试
*   编码面试
*   编码面试

每次采访持续约 45 分钟。整个过程非常紧张。

#### [](#coding-interviews)编码面试

编码面试类似于白板上的 leetcode 问题。我认为我在其中三个方面做得还不错。对于其中一个，我提出了几个非最优的解决方案，但最终被引导到最优的解决方案。

#### [](#system-design-interviews)系统设计面试

除了经历了这个的[，我没有为系统设计面试做太多准备。](https://www.hiredintech.com/system-design)

*   一个原因是，通过阅读文章很难准备一个非常互动、流畅和相对高水平的对话。
*   另一个原因是我有几个项目。这次面试对我来说很顺利。

### [](#additional-phone-interview)附加电话面试

额外的电话采访很少见。这次面试的原因是为了收集更多的信号，因为招聘委员会没有就招聘决定达成一致。这次面试和我第一次接受电话面试一模一样。

### [](#tips-during-interviews)面试时的小贴士

在压力下很容易紧张，风险也很高。这次与其说我紧张，不如说我兴奋。以下是我减少焦虑的方法:

*   做好准备。有一个计划总是一件好事，即使这个计划最终没有成功。
*   放松点。被拒真的不是什么大事。你可能已经有了一份你喜欢的工作。你随时可以再次申请。被拒绝时不要气馁(我以前就被谷歌拒绝过)。这里面有很多运气成分，包括面试问题、面试官和公司的招聘计划。
*   合作一点。面试官的目标是尽可能准确地评估你的能力。告诉面试官你的想法，当你陷入困境时，倾听微妙的暗示。
