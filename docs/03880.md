# 熟能生巧

> 原文:[https://dev.to/dclements9/practice-makes-perfect-1](https://dev.to/dclements9/practice-makes-perfect-1)

六周很快就过去了，尤其是当你正在学习一项技能，让你越来越深地陷入一个无尽的兴奋世界。回想起来，我很难相信我已经学到了这么多。

大约两周前，我开始了我的 CLI 项目，以为我知道我需要什么来完成它。但是我想巩固这些知识。熟能生巧。但是创造性的实践填补了你不知道的空白。

该项目需要抓取一个网站，为用户提供信息，并有一个命令行界面与程序进行交互。我选择抓取一个活动日程来为用户创建一个议程( [GDG 发展节 2018](https://devfest.gdgcapitalregion.com/) )。在正确地学习迭代、过滤和将我收集的结果附加到一个对象时，我最好的朋友是 Pry。

我可以不停地说我对牧师的爱。但是我会试着分享一些我最喜欢的命令。虽然 Pry 最根本的好处是从本质上“撬开”代码，但它还有更多。

如果您熟悉 Unix 系统上的终端，您可能使用过' ls '命令来列出文件和文件夹。Pry 也有类似的东西。“ls”命令将显示您撬开的位置的可能方法和变量的列表。虽然这个命令本身看起来微不足道，但是当它与其他命令结合使用时，就会变得非常强大。

这就引出了我的下一个命令，“播放”。更具体地说，“play -l”将播放接下来指定的行。该命令还提供了一种相对简单的方法来进入或绕过块和迭代。当它与“ls”结合使用时，您可以随时跟踪变量和方法的变化。

我的最后一个命令与其说是一个实际的命令，不如说是一个如何使用命令的提示。在“do”块之外，“exit”将从程序中退出。但是在内部，它可以提供关于该块经历的每次迭代的有价值的信息，提供非常需要的调试支持。

总的来说，这是我对这个项目所了解的非常小的一部分。但是我会把它留到另一篇文章中。旅程继续！
