## [](#hi-there)你好！

首先，我想我应该自我介绍一下。我是瑞安。没有人关心这个东西，所以我长话短说，但基本上，我是一名七年级教师，希望成为一名开发人员。我想更多地融入社区，因为我是自学的。这很难做到，因为我有一个家庭和另一份全职工作。没有太多的时间去外面的世界和人们面对面的交流。所以，我来了！

* * *

## [](#the-things-i-wasted-my-time-learning-and-why-it-made-me-a-better-developer)我浪费时间学习的东西*(以及为什么它让我成为更好的开发者)*

从很多方面来说，这是一个点击诱饵的标题，但实际上，作为一个自学成才的人(基本上都是在去年左右)，我已经回顾了一些事情，并认为，“这是一个巨大的浪费。”同时，我觉得这也让我成为了一个更好的开发者。也许我只需要用我花在开发这些方面的时间来解决一些内部问题，但我发现以下是一些事情...考虑到我花了这么多时间学习它们，这并不是很有用。

**1。我记得花了大约两周的时间做各种有关 Treehouse.com 的教程。我用过吗？没有。事实上，我认为我真正从中得到的唯一东西是在听 [Syntax.fm](http://ww.syntax.fm) 的一集时的一些笑声和共鸣，当时他们正在讨论它是不是真的不再存在了。**

我能从那几周学习吞咽中学到什么？它出现的时候很好，而且它是通向现在更有用的其他构建工具的重要垫脚石，比如 Webpack。我可以带走所有开发人员都知道的教训——事物是不断变化的。

**2。jQuery**
我确信如果有大量的人阅读这篇文章，我会让很多人感到不安，但是现在似乎对于任何一个构建现代 web 应用的人来说，你都不会去使用 jQuery。看起来 jQuery 的绝大多数特性现在都已经融入了普通的 JS，我真的认为即使使用 jQuery 更容易，更好的选择仍然是使用普通的 JS。

也就是说，我认为知道 jQuery 如何工作让我成为一名更好的开发人员，因为它仍然无处不在。了解基础知识并能够获得许多似乎基于它的 API，或者以类似的方式运行，这很好。例如，我的一个朋友正在使用 Wix 构建他的个人网站，他想要一些文本淡入淡出的功能。很简单。在 Wix API 中，我发现它们不只是让你直接编辑 CSS...根本...)而且我发现它的结构非常像 jQuery。所以，那时候了解 jQuery 是件好事。

**3。自举**
到处都是。它让网络看起来都一样。定制很痛苦。可能是我最大的时间浪费。我认为学习 Bootstrap 可能是最大的浪费，因为它分散了我的注意力，使我无法花时间去学习已经存在的更好的东西，比如 flexbox 和 grid。这也耽误了我学习如何简单地建立自己的命名约定，以及如何使用好的 ole CSS 创建自己的主题或布局。

为什么我很高兴我学到了这些？好问题。也许是因为整个列系统仍然存在于许多地方，或者也许只是为了更好地了解如何为更大的项目命名 CSS 类。

## [](#so-whats-the-point)那么，有什么意义呢？

我想写这篇文章的主要目的是回顾和反思我所学到的一些东西。当然，也许大多数时候我发现这些东西并不适用于我现在选择做的事情，但是意识到仍然有一些应用程序可以让我受益于 gulp、jquery 或 bootstrap 的知识是很有帮助的。也许你偶然看到这篇文章，而你也刚刚开始编码。如果你像我一样正在学习自由代码营的课程，你应该跳过这些吗？也许吧。或者也许你应该提前知道这不是你需要成为一个**大师**的事情。

如果我作为一个“开发人员”知道一件事的话，那就是你总是在学习新的东西，而把你已经学到的其他东西放在一边。我这样做已经有一年了，而且已经非常成熟了。你需要有一个成长的心态(正如我们在《教学之地》中所说的)和足够的勇气。回顾我的“无用的学习”教会了我继续学习更多。偶尔，你可能会意识到你花时间学习了一些你不经常(或从不)使用的东西，但是如果你发现自己在那个领域，你总是可以寻找积极的一面，从长远来看，学习而不使用的*可以让你成为一个更有弹性的开发人员。*
