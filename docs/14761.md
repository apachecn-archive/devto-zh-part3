# CSS、LESS 和 SASS 之间的区别

> 原文:[https://dev . to/flipped coding/the-differences-between-CSS-less-sass-5a 74](https://dev.to/flippedcoding/the-differences-between-css-less--sass-5a74)

为一个网站编写样式表是一门艺术。你可以用各种不同的方式操作 HTML 元素，重要的是尽可能保持代码的整洁，以避免那些奇怪的样式问题。这就是 LESS 和 SASS 的用武之地。

## CSS 和少&萨斯的区别

知道 CSS 和 LESS 和 SASS 的基本区别很重要。LESS 和 SASS 是 CSS 代码的预处理程序。它们允许您在样式表中使用变量和一些逻辑。由于您可以在样式表中编写实际的代码，因此您的工作表变得动态，这对响应式设计非常有用。

仅用 CSS，你就必须在你想要的地方写出你想要的样式。您的 CSS 文件一旦加载就不会更新。你写的就是你得到的。如果你想硬编码你的风格，这很好。预处理器只是使它更容易做到这一点。

## [](#about-preprocessors)关于前置处理器

预处理器只是一个根据你写的代码为你写 CSS 的程序。通过使用预处理器，您可以做一些事情，比如使用继承选择器和混合。常规 CSS 没有这些选项。

预处理程序使得通读你的样式表和弄清楚所有东西是如何联系在一起的变得更加容易。这使得维护和更新你的样式变得更容易，而不需要做大量的挖掘来找出为什么图像向右多了三个像素。

## [](#less)少了

LESS 代表更精简的样式表。当您希望在样式表中使用 JavaScript 时，这通常是项目的选择。使用 LESS 与编写常规 CSS 没有太大区别。有几个小插件可以让你更容易处理 CSS，但是它们很容易上手。

通过使用变量、混合甚至计算，在 LESS 文件中重用 CSS 是很简单的。如果你对应用程序的风格有很深的了解，你可以选择编辑 JavaScript 文件来减少控制。通常你不会想接触 JavaScript 文件，但是四处看看也无妨。

## [](#sass)萨斯

SASS 基本上与 LESS 相同，只是它用在 Rails 应用程序中。它代表语法上令人敬畏的样式表。你可以用这些叫做 gems 的东西来安装它，我会让一个 Ruby/Rails 开发者告诉你所有的细节。最主要的是，它是另一个 CSS 预处理器，让你更容易编写 CSS。

你仍然有你的混合和变量和所有的花式嵌套的东西。像 LESS 一样，SASS 也完全向后兼容常规 CSS 文件。使用这两种预处理器中的任何一种都不会花费您很长时间。

## [](#css)CSS

不管你用什么样的前置处理器，你都不能忽视一件事，那就是知道如何编写真正的 CSS。当你开始研究预处理器时，你应该对 CSS 有一定程度的掌握。您应该能够让您的 web 应用程序看起来像设计一样，而无需使用预处理器。

你也应该知道如何构建一个 CSS 文件。这将有助于你减少或 SASS 文件易于阅读，这样你就不会像疯了一样在树上跳来跳去。没有什么比了解基础知识更重要的了，CSS 就是你所掌握的基础知识。

希望这有助于理解 CSS 与 LESS 和 SASS 的区别。他们真的都在做同样的事情。他们只是做的有点不同。然后是 SCSS，它是从萨斯派生出来的。我以前在一个非 Ruby 项目中使用过它，但公平地说，这并不是最棒的项目。除了 Ruby/Rails，还有人使用 SCSS 吗？
