# 技术不能创造一个平台，但它确实有所帮助

> 原文：<https://dev.to/oddevan/technology-cannot-make-a-platform-but-it-does-help-j9c>

网络就是为了分享内容而存在的。第一个网络浏览器也是一个网络编辑器。从那以后，程序员们一直致力于使发布更容易、更好的方法。因此，新平台可以建立在现有技术的基础上。

## 关于技术的本质及其作为整体的一部分的位置的一个简短的旁白

很容易认为正确的技术会改变一切。不知何故，正确的代码将使所有旧的蓝色问题消失，我们将幸福地生活在我们的新天堂。

人们很容易忘记，Posterous 是在老布鲁登峰造极的时候存在的。它幸运地拥有更好的技术，包括专用的网址缩写和通过电子邮件发布的能力。老蓝可以说拥有低劣的技术。但它赢了。正确的技术与正确的设计和正确的人在正确的时间走到了一起，瓶子里的闪电击中了。

改变事情需要的不仅仅是好的技术。它需要好的设计，好的时机，以及对要解决的问题的好的理解。但是正确的技术可以使变革成为可能。当我们谈论可以实现新平台的技术时，记住这一点很重要。

## 界面热

所以，对于这篇文章，让我们来看看一些接口。这些也被称为“协议”或“标准”这里的大意是，一群人已经用技术语言写下了一件*事情*应该如何完成。其中最明显的是管理 web 浏览器和服务器如何相互通信的 HTTP 标准。

我们还没有谈论代码，只是我们可以使用它的方式。

### [下载](https://oembed.com)

这就是把`https://www.youtube.com/watch?v=dQw4w9WgXcQ`变成嵌入式代码，让你所有的朋友都讨厌你的原因。它包括几个步骤:

1.  博客从用户处获取 URL。
2.  Blog 也在 oEmbed endpoint 中查找 URL
    *   将 URL 与已知端点列表进行匹配，或者
    *   在页面的`head`中寻找特定的`link`标签。
3.  Blog 点击 oEmbed 端点并获取将 URL 中的内容嵌入页面所需的代码。

虽然这可能最初是针对视频网站的，但它已经发展到涵盖所有类型的网站，包括 Old Blue 自己。该标准的维护者在其网站上有一个使用 oEmbed 的站点列表(不全面)。

**要点:** Reblogs 对老 Blue 有效，因为一切仍在平台内发生。老蓝能够实施归属，并保持社交信息流回内容的原始海报，无论它离原始海报有多远。然而，在开放的互联网上，“转载”和简单的抄袭之间的区别可能很难看出。然而，*嵌入来自其他博客的*帖子的能力，可以在保持属性和社会信息的同时，重新创造重新写博客的想法。

### RSS / [JSON Feed](https://jsonfeed.org/)

RSS 已经使用了近 20 年，允许其他网站和程序从博客和其他定期更新的网站读取更新。它发展得很慢，但它的简单性使它保持相关性，即使大多数互联网用户没有意识到他们正在使用它。

今天，虽然谷歌阅读器已经关闭，但 RSS 阅读器仍然可以在像 [Feedly](https://feedly.com/) 和 [Feed 牧马人](https://feedwrangler.com/)这样的服务中找到。它还被用来填充苹果新闻应用程序中的故事。不过，最普遍的是，它被用来向众多听众发送每一集播客。

JSON Feed 采用与 RSS 相同的原理，但是使用更多的 JSON 而不是 XML 作为主要语法。这使得这种格式看起来更容易理解，并且有助于使这种格式在某些边缘情况下更有弹性。

要点:老蓝的仪表盘允许你关注平台上的其他博客。一个去中心化的平台需要一个标准的方法来关注其他博客，并且它已经存在了。

### oAuth

这是允许外部第三方应用程序绑定回平台的认证流程。到目前为止，如果不使用这个将一个应用程序或网站连接到另一个应用程序或网站，很难在互联网上存在。无论是与脸书一起登录手机游戏，还是将老蓝连接到 Twitter，每个人都熟悉“应用程序需要连接许可”

要点:任何社交网络都不可能存在于真空中，至少现在不会了。任何新平台都需要利用与其他网络的连接，即使只是为了交叉发布帖子。

### [网站提及](https://indieweb.org/Webmention)

web reference 是一个新标准，它允许回复他人的帖子自动相互链接。它模仿了社交网络上的类似功能。

**要点:**同样，获得通常在单一平台上发现的相同社交信息，对于让一个去中心化的平台感觉像一个“正常”的社交网络来说是至关重要的。这是一个相对较新的标准，必须小心确保垃圾邮件和骚扰不会淹没系统。

## 当一个计划走到一起

这些技术中没有一项能让一个新平台获得成功。即使他们都在一起也不能保证成功；事实上，如果没有很好地整合不同的部分，最终的结果会更糟。糟糕多了。

IndieWeb 正在使用这些技术中的许多技术，并且所有这些技术都有可以被任何平台使用的开源代码。正在努力使这些技术更容易获得和使用。给我留下特别深刻印象的是[微博](https://micro.blog)平台，它采用了许多这些技术和其他技术，并使它们成为 Twitter 的可信替代品。

一个新的平台必须知道这些技术是如何相互作用的。正如我前面提到的，Old Blue 的成功不在于它的技术实力，而在于它如何利用该技术来满足用户的目标。任何旧 Blue 的潜在替代品都需要走同样的路:选择正确的技术，并以最佳方式展示，让人们能够理解并有效地使用它。

设计*是*一个难题。
