# 检查你的节点？

> 原文:[https://dev.to/kymiddleton/check-your-nodes-2epn](https://dev.to/kymiddleton/check-your-nodes-2epn)

**Node.js** 是一个开源的跨平台 JavaScript 运行时环境，旨在让 JavaScript 在浏览器或网页之外运行。它是一个通用的工具，可以用于各种目的，包括资产编译、脚本编写、监控、创建桌面应用程序，最显著的是作为 web 服务器的基础。JavaScript 可用于服务器端，因为节点重用 JavaScript。这意味着前端开发人员可以自己构建整个服务器。节点的其他优势:

*   有许多插件可以轻松扩展它的功能。
*   创建一个完整的工作服务器只需要几行代码就可以完成，这样可以加快实现速度。
*   单线程异步模型允许同时处理多个请求，不会出现瓶颈或降低性能。

**设置节点工程**
*快速参考指南*

*   打开收藏夹代码编辑器
*   在代码编辑器中打开命令行

```
`mkdir <projectName>`
`cd <projectName>` 
```

*   使用以下命令在项目文件夹的根目录中安装节点包:

```
`npm init` 
```

这一步将初始化文件夹并创建`package.json`文件。

*   安装完成后，程序将提示一系列问题，以收集有关项目的上下文信息，如项目名称、版权、版本、描述、入口点文件名、作者等。
*   不要求填写这些字段，可以使用`enter`键跳过。
*   当出现`Is this okay? (yes)`提示时，通过输入`yes`然后按`enter`键来响应。

*   在文件夹中寻找`package.json`文件。

*   打开文件以查看和熟悉内容。

*   JSON 文件夹的语法是一个 JavaScript 对象。

*   接下来，在根文件夹的同一个目录中运行以下命令:

```
`npm install express` 
```

这将创建`node_modules`子文件夹。

*   `node_modules`文件保存了特定项目的所有模块。

*   返回并查看`package.json`文件

    *   将会出现一个新对象`dependencies`，其中应该会列出`Express`。
    *   任何新的依赖关系都将被添加到`package.json`文件中。

**对现有项目采取的步骤:**
当处理已经包含`package.json`文件的现有项目时，运行以下命令:

```
`npm install` 
```

这将评估`package.json`文件的内容，并在需要时安装额外的包。

**接收现有项目时要采取的步骤:**

```
`npm install` 
```

这将评估`package.json`文件的内容，并在需要时安装额外的包。

**不共享节点模块文件夹**

这是需要记住的关键一步，尤其是在项目合作的时候。

*   节点模块文件夹必须**而不是**共享。
*   当`npm install`命令运行时，它根据特定计算机的需要编译文件。
*   当协作或共享项目文件时，其他参与者将需要在他们计算机上的项目文件夹的根目录中安装命令`npm install`。
*   请记住，不带包名的`npm install`命令会评估`package.json`文件中现有的依赖项，并安装所需的依赖项。
