**放空进料**

我认为我注意到的一件大事是，人们主动提供建议，往往是为了让建议者而不是接受者感觉良好，这通常不适合接受者的水平。

我在网上发布了一些带有侮辱性的帖子，得到了很多对这些事情的“解释”。例如，我最近既有了 Git 的概念，又有了对什么是拉请求的解释，这虽然对刚开始的人有帮助，但对那些做了大量拉请求并且已经做了很多年的人来说一点也不相关。还有很多不请自来的代码审查，我甚至有自己的代码解释给我听！我知道我的代码做什么，为什么，这就是我写它的原因！

我真的很享受 **[我 90%的工作是...](https://dev.to/david_j_eddy/90-of-my-job-is-8ie)** 的提示，引出了一些很棒的答案。再好不过了 [@buphmin](https://dev.to/buphmin) 在此回复:
