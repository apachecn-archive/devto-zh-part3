# 为客户部署 web 应用程序的正确形式？

> 原文：<https://dev.to/benjiboy13/the-correct-form-for-deploying-a-web-application-for-your-customers-565n>

## 我们需要担心的是什么？

通常在 MERN 堆栈应用或 MEAN stack 应用中，我们需要担心三件事，DNS(域名)，将托管我们的 React/ANgular 应用(如 herocku)的服务器，以及我们将在云上运行的数据库。

这三样东西可能由完全不同的公司管理，例如，你可以从 Go-Daddy 获得域名，从 Herocku 获得主机，从 MongoDB Atlas 获得数据库

## 免费吗？

当然不是，每家公司都将向你收取服务费用，每家公司的价格都不一样，当然你不会为这些服务付费，因为归根结底，你开发的应用程序并不适合你使用，嗯，但我该如何向客户解释所有这些是如何工作的？谁来管理这家公司的所有账目和到期日？，是我的工作还是客户的工作？

## 我对这一切有什么看法？

我个人认为，我是负责管理所有客户账户和到期日的人，作为额外费用的交换，类似于...管理价格。

我只是一个初级开发人员，所以我还没有真正为任何人部署一个实际的应用程序，我现在正在为一个非常小的机构开发一个非常小的 web 应用程序，但不是一个实际的工作，我只是在那里做社会服务，因为我的大学迫使我这样做

所以，尽管部署我正在开发的这个应用程序还为时过早，但正如你所看到的，我开始担心这个部署的事情，我真的不知道如何像一个真正的专家一样处理它，你有什么想法？
