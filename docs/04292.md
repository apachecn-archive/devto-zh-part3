我仍然很惊讶我们能够完成一项伟大的活动。过去的这个周末，我们在拉各斯组织了有史以来第一次 JAMstack 聚会，并取得了巨大的成功。JAMstack 社区已经在世界各地兴起，作为一个爱好者，我已经从我在互联网上看到的所有事件中获得了 FOMO，包括 T2 jam stack 会议。所以我决定和我的共同组织者[威廉·伊莫](https://twitter.com/iChuloo)一起在尼日利亚的拉格斯开一个聚会。我们开始为定于 5 月 4 日举行的第一场活动制定计划。

## [](#planning-the-event)策划事件

我已经有了组织和策划活动的想法，因为我以前主持过一次聚会。当然，如果没有威廉·伊莫的帮助，我不可能完成这件事。我们必须锁定日期、活动主题、演讲者、地点和赞助商。感谢 Netlify 给了我第一次推动，他们赞助了我们的 meetup 页面订阅，当时我还负担不起。我们也有几个赞助商，使这次活动取得了成功。图像管理平台 [Cloudinary](http://cloudinary.com) 和云通信平台 [Twilio](https://www.twilio.com/) 也以某种方式赞助了此次活动，对此我们表示感谢。

Natterbase 的软件开发人员 Oluwasetemi Ojo 发表了题为“**用真实用例理解 jam stack**”的演讲。他给了我们一份 Freecodecamp 和 Gatsby Store 目前如何使用 JAMstack 来支持他们的架构的分析报告。[他的幻灯片可以在这里找到](https://understand-jamstack.netlify.com/#0)。

## [](#lightning-talks)闪电会谈

我们还与 Prosper Otemuyiwa 进行了两次闪电对话。第一个演讲是关于 JAMstack API 的。在这里，Prosper 揭开了 JAMstack 中 A 的神秘面纱。他给出了在用 JAMstack 构建时我们可以利用的非常有用的 API 示例。你可以在这里找到他的幻灯片。

液体错误:内部

第二位演讲者是来自尼日利亚埃努古的媒体开发专家 Ekwuno Obinna。他做了一个题为“T2”的演讲，你的 JAMstack 应用程序真的是 JAMstack 吗？“在他的演讲中，他概述了我们可以使用 Cloudinary 更好地处理图像和视频等媒体的方法。[这是他的幻灯片](https://twitter.com/Obinnaspeaks)。

液体错误:内部

在 Obinna 的演讲之后，Prosper 给了我们他的第二个闪电般的演讲，关于我们如何在简单的地方设计更快的网络体验。他提供了一些技巧和窍门，告诉我们如何在网上为用户建立快速的加载体验。他的幻灯片可以在这里找到。

我们今天的最后一项议程是由威廉·伊莫致闭幕词。他谈到了我们的每一个赞助商:Twilio、T2、Netlify 和 T4，以及他们提供的服务。我们还有 3 分钟的时间给任何有东西要向观众推销的人，这包括工作机会、想法和像我一样为你的创业做广告的机会。这件事到此结束。我们用剩下的时间建立关系网，还为客人准备了贴纸、食物和饮料。
