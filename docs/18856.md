# 恶意软件数据科学的首次冒险

> 原文:[https://dev . to/ello wrence/first-adventure-in-malware-data-science-426m](https://dev.to/ellowrath/first-adventure-in-malware-data-science-426m)

我做软件开发的时间不长，但我喜欢这份工作。在我的日常工作中，我在 Flask 中做后端工作，但我一直在寻找一个令人兴奋的在家工作的项目，一些可以扩展我的技能并教会我一些新东西的项目。也许让我变得有点脏。从十几岁开始，我就对恶意软件着迷。然而，这是一个我没有如我所愿探索的好奇心。因此，当没有淀粉出版社出售时，我找到并预订了约书亚·萨克斯和希拉里·桑德斯的《恶意软件数据科学》。

我刚开始读第四章，目前为止我很喜欢。也许我完成后会写一篇评论。与此同时，我想在博客上快速介绍一下我在第二章中遇到的一个小问题，以及我是如何解决它的(这是我的问题，与作者无关)。

作者的代码都是 python2.7，我更习惯 3。不过，没什么大不了的，对吧？我是专业人士，我能搞清楚。不，这不是我的问题。

当我下载样本代码和恶意软件文件时，我错过了一个小细节。zip 文件的标题是*malware _ data _ science _ entry points _ redated . zip*

我在第 2 章中尝试的任务是打印出反编译的恶意软件，从入口点的地址开始。我完成了 python 脚本的移植(在 print 语句中插入括号是不明智的)并运行它。什么都没发生。

我在想，我做错了什么？我查了文件。我通读了 pefile 的 github 上的 readme。我查了顶点集团的文件。我做的一切都是对的。我甚至下载了作者的有用的 Ubuntu vm，上面已经有了所有的代码和数据。它在那里工作。但不是在我的 fedora vm 上。

这让我沮丧了很长时间。我不记得什么时候“entrypoints_redacted”吸引了我的目光，但当时我觉得有点傻。我稍微修改了一下我的脚本，打印出入口点的地址，它被可爱地命名为 0xcc00ffee。当你把这个地址提供给反汇编器时，你什么也得不到，因为那个地址(它是一个偏移量，对吗？)非常大(3.5 GB)，文件本身的大小几乎是 631k。

我运行了他们的虚拟机，并从那里打印出了入口点地址。低级的 0x121ba(或者 python 中的 74170)。把它输入到我的脚本中，瞧，我得到了反汇编代码。这并不完全是书上说的那样，但它确实是作者的虚拟机上的工作代码所说的那样，所以我想我做对了一些事情。

我期待着在这本书里深入挖掘。
