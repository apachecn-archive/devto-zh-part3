# 什么是配置文件以及它们是如何工作的

> 原文：<https://dev.to/flippedcoding/what-config-files-are--how-they-work-2plm>

那些。配置文件可以是魔术箱。您知道它需要在那里，如果它丢失或损坏，您的项目将无法工作。对于一个新手，有时甚至是有经验的开发人员来说，最可怕的事情之一就是弄乱。配置文件。

对于初学者来说，确切地知道. config 文件是什么会有所帮助。你可能已经猜到了，config 是 configuration 的缩写。这些文件包含项目开始时需要的任何初始设置或参数。web.config 文件中设置的一个示例是如何处理项目中的路由。您可以通过使用重写行来做到这一点。

是什么让？配置文件让开发人员紧张的事实是，配置文件中的一个错误会导致项目失败。并不是说你不能回到过去撤销你所做的改变，但它可能发生在你想不起来你做了什么的关键时刻。

当涉及到你的代码时，你总是想知道你在做什么，尽管在配置文件的情况下，你需要知道你在做什么。现在没有一种超级严格的方式来编写配置文件。你可以用 JSON、XML、YAML 或者它们的组合来写一些。这只是取决于你的项目是如何设置的。

关于配置文件，你必须知道的另一件事是，一些程序只在启动时加载配置文件，而另一些程序会定期检查配置文件。了解如何使用配置文件将有助于您了解如何更好地使用它们，并有助于优化您的项目。

在项目开始时拥有正确的设置和参数是项目成功运行的唯一途径。所以在你改变你的配置文件之前，确保你知道你在做什么。他们将创造或破坏一个项目。
