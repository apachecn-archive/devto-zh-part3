# 什么是代码审查？

> 原文:[https://dev.to/kymiddleton/what-is-a-code-review-4gpa](https://dev.to/kymiddleton/what-is-a-code-review-4gpa)

你有没有在做一个项目时变得过于专注，以至于你周围的世界都崩溃了，而你却没有注意到？我喜欢那些时刻，作为一名新的 web 开发人员，这是我感觉一切都在一起，并且我在一个项目或一项功能上取得了重大进展的时候。当然，不利的一面是，当某个东西坏了或者不按预期的方式工作时，尽管我尽了最大努力集中注意力，我的眼睛还是无法分辨逗号和句号之间的区别。这就是代码审查发挥作用时候了！

代码评审是为协作者评审和批准代码变更的过程，协作者不是被推动的变更的作者。它基本上是由在同一个项目中工作的同行用一个受保护的`master`分支进行的代码检查。

代码评审*需要*认真对待。不要像漫画中描述的那样对待它！
T3![Code Review](../Images/ed380baf8fd0b8b094aa0a03655f0f11.png)T5】

代码审查有巨大的好处。首先，代码评审节省了时间，并有助于预先简化开发过程。最终，节省的前期时间减少了质量保证团队以后的工作量。第二双眼睛来检查你盯着看了几天的东西，可以快速识别简单的错误，甚至捕捉到可能在测试或生产中未被发现的错误。我们都知道时间就是金钱，节省的时间也是节省的金钱。

在我看来，代码评审过程也促进了协作。我与他人合作得越多，我学到的也越多。对我来说，看着别人编码、解决问题和做代码回顾是很有趣的。它提供了一个见证他人编码方法和思维过程的机会。对于任何人来说，这都是一个无价的机会，可以让他们学习如何识别常见问题，或者使用有用的快捷方式编写更简洁的代码。
