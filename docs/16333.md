# 我们要多久才能获取卫星数据？如果 AWS 如愿以偿，这一天不会太久...

> 原文:[https://dev . to/David _ j _ eddy/how-long-until-we-can-all-access-satellite-data-now-long-if-AWS-have-way-4f J1](https://dev.to/david_j_eddy/how-long-until-we-can-all-access-satellite-data-now-long-if-aws-has-its-way-4fj1)

Cross 从我的[博客](https://blog.davidjeddy.com)发布。

想象一下，你在一家木材公司工作。这家木材公司需要一种方法来降低观察和跟踪他们种植树木所用的数百万英亩土地的成本。传统上，必须派人去物业，记录树木的生长速度、健康状况和数量。这一成本是公司向其他市场机会扩张计划中的主要预算因素。

你知道什么最棒吗？如果你的公司可以使用现有的轨道上的商业观测卫星。你认为那会很棒。但是，即使发射成本降低，地面物理基础设施也会很昂贵。下行链路硬件、位置操作、命令和控制。糟糕，本来会很棒的。

输入“[地面站](https://aws.amazon.com/ground-station/)”；'...全面管理的地面站即服务(GSaaS)卫星通信、下行链路和处理卫星数据...。如果你在想“没门；这一定是个恶作剧。”，那么你不是一个人。当最初的公告在[发布时，博客文章中有一个乐高模型](https://aws.amazon.com/blogs/aws/aws-ground-station-ingest-and-process-data-from-orbiting-satellites/)作为硬件演示。这是一个恶作剧，对不对？
