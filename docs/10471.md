### [](#introduce-yourself-who-are-you-where-do-you-work)自我介绍！你是谁？你在哪里工作？

我的名字是玛丽·唐维尔。到目前为止，我已经参与开发人员社区的建设 10 多年了。我有新闻背景，但因为在 O'Reilly Media 实习而进入了科技世界。我爱上了科技社区，从此再也没有回头！

### 谁或什么让你进入了科技行业？

我喜欢说我是偶然进入科技行业的，但实际上，当我还是个孩子的时候，我爸爸就让我对科技产生了兴趣。他是纳帕汽车配件分公司的控制员，偶尔会把新电脑带回家，在办公室测试。他让我妹妹和我玩了一个早期的触摸打字游戏，我迷上了。当我在高中的时候，我写诗(哪个高中生不写诗？)并想做一个网站来正式发表其中的一些诗作。雅虎网站在当时是一个东西，但唯一的选择是即插即用，这不适合我的风格。我父亲鼓励我钻研 HTML，理解事情为什么会这样，几个星期后，我就有了第一个功能性网站。

直到我在 O'Reilly Media 实习时，我才开始钻研技术，除了偶尔在网站上闲逛。我被安排在公共关系部门，负责书籍推广，并向记者分发评论副本。然而，作为一名新闻专业的学生，我想理解我们正在宣传的主题，而不仅仅是重写封底然后继续下一本书。在写新闻稿之前，我研究了每个主题 JavaScript 和 CSS 是如何协同工作的？为什么监控和警报如此重要？Hadoop 适合做什么？

很快，我就有了一个不错的思维导图，知道哪些主题适合其他主题，哪些主题属于完全不同的领域，以及为了了解更多应该问什么问题。我的下一个任务是了解我们为什么出版我们正在出版的东西。谁做的这些决定？为了确保我们做出正确的决定，他们从我们的读者那里收集了哪些信息？

我坚持不懈，继续问这些问题，直到奥赖利的总裁劳拉·鲍德温给了我一个机会，让我担任社区经理的职务，并找到我的问题的答案。她开启了我通往开发者关系世界的旅程，当我抓住这个机会时，我甚至没有意识到这一点。

从那以后，我与世界各地有影响力的人和思想领袖一起工作，了解对他们来说什么是重要的，以及他们认为科技行业将走向何方。在我工作过的公司内部，我已经成为他们的支持者，确保他们的意见被提出来，即使他们不能亲自表达。

这些社区激发了我对开发者关系的热情——我怎样才能最好地为他们服务并帮助他们发挥潜力？-但这些天来，我更关注投资于开发者关系的公司和以此为职业的个人。

我注意到人们明显缺乏对这些开发者关系项目给企业带来的价值的理解。我与 Persa Consulting 的目标是通过为开发人员关系团队提供资源，以及帮助公司利益相关者了解通过培养与您的技术社区的关系可以获得的真正价值，推动行业向前发展。

### 《发展周刊》对你的职业生涯产生了怎样的影响？

DevRel Weekly 是我第一次单独为开发者关系从业者社区提供资源(Community Pulse，一个月度播客，已经发展了 3 年多)。虽然这本书还在进行中，但它不会在几个月内发布，这是一个提供持续每周内容的机会。

这也是我跟上行业趋势的好方法。通过每周收集几十个 RSS 提要和整理数百个链接，我每周花几个小时筛选与 DevRel 相关的内容。我正在分析第一年简讯的数据，并计划发布一份年度报告，根据点击率以及每周出现在我盘子里的文章数量，提供关于哪些主题发表得最多以及人们最感兴趣的信息。

DevRel Weekly 的目的是突出和放大人们围绕开发者关系这一主题发布的精彩内容。找到好的 DevRel 相关内容是很困难的——没有多少人发表相关内容，那些发表的人也经常不使用有帮助的关键字，让其他人很容易找到。

为此，DevRel Weekly 每周四为 DevRel 专业人员和公司利益相关者提供一份精选的文章、招聘信息和活动列表，节省他们在网上搜索最新信息的时间和精力。相反，他们可以专注于如何发展团队，证明他们的商业价值，并与他们的社区建立关系。

### [](#congrats-on-your-recent-book-how-did-that-come-about-what-impact-are-you-hoping-for)祝贺你的新书！那是怎么发生的？你希望有什么影响？

[*开发者关系的商业价值*](https://amzn.to/2EslVv7) 一直是爱的劳动，这是肯定的！它始于 2017 年 15 倍规模的一次对话，我们四位业内人士表达了为开发者关系专业人士创造更多资源的愿望。朱莉·冈德森、杰森·易、纳森·哈维和我，口头上答应和出版社的编辑路易斯·科里甘合写一本书。我很幸运在工作周有专门的时间写这本书(谢谢，SparkPost！)并于 2017 年 7 月接管了整个项目。

这本书的读者是双重的:前半部分是针对利益相关者或决策者，他们需要理解为什么开发者关系是一项有价值的工作。这已经成为一个流行词，每个人都认为他们需要“一个 DevRel”(让我想起“我需要一个 DevOps！”几年前的狂热)但是很少有人知道一旦他们雇佣了一个人该做什么。前几章帮助这些利益相关者理解他们需要做些什么来建立一个可持续发展的开发者关系团队，以及如何从一开始就建立成功的团队。我花时间在跟踪哪些指标和如何设定目标上，以及雇佣谁和每个角色能带来什么。

这本书的后半部分讲述了在你雇佣了一个人之后该做些什么。它涉及更多的实践方面——在会议上发起和发言，如何与公司的其他团队合作，如何建立一个在线社区，等等。

我在出版业工作了太多年，没有想到我会通过出版一本书而致富；)相反，我专注于为我周围的人提供有价值的内容。像这样的推文([https://twitter.com/rachfrieee/status/1082771940903796736?s=21](https://twitter.com/rachfrieee/status/1082771940903796736?s=21) )对我来说是令人难以置信的满足，因为我知道我投入了一年多的项目是真正在帮助别人。

### [](#how-do-you-think-the-developer-relationsindustry-will-change-over-the-coming-years-and-how-might-that-affect-your-role)您认为开发人员关系/行业在未来几年会发生怎样的变化，这对您的角色会产生怎样的影响？

我真诚地希望 2019 年是 DevRel 获得成功的一年。这个词已经流行了至少两年，从 2000 年代中期开始流行起来。工作机会比以往任何时候都多，越来越多的专业人士正在寻找关于开发者关系的日常信息。但是董事会成员和利益相关者仍然有许多问题需要回答，包括这些团队的价值(也称为 ROI)在哪里。

我希望 2019 年和 2020 年将是我们开始乘风破浪，放松一点的年份，而不是不断地对那些可能对开发商关系有错误印象的人采取防御措施。

### 迄今为止，在你的科技生涯中，你学到的最难的一课是什么？

我需要做我为自己做的事。不是为了我的老板…不是为了我的朋友…甚至不是为了我的社区。如果对我来说不重要，那就不会有回报，我不会觉得自己完成了什么，而是会寻找下一件我需要做的事情，向其他人证明自己。

保护自己免于精疲力尽是我必须学习的另一大课。我已经就这个问题做了几次演讲(社区/DevRel 聚焦:[https://vimeo.com/179488000](https://vimeo.com/179488000)和一般:[https://www.youtube.com/watch?v=2ppmFCk5XXE](https://www.youtube.com/watch?v=2ppmFCk5XXE))，强烈地感觉到我们需要将这种趋势扼杀在萌芽状态。这些天，我小心翼翼地确保我的日子是平衡的，并且总是有一件事，不管多小，给我“净能量”换句话说，一旦我完成了那个活动，我会比开始时有更多的精力。

### 工作之外，你有什么爱好吗？你认为它们对你的科技事业有任何帮助吗？

实际上，我正在努力把兴趣爱好加进去！多年来，我一直热爱阅读，如饥似渴地阅读书籍。我正努力把小说重新纳入我的日常日程，“为了好玩。”

我最近也开始跑步了。我已经有几年没有坚持跑步了，但我可以肯定地说我的精力和专注能力有所不同。即使不是跑步，锻炼对我来说每天都有很大的精神影响！
