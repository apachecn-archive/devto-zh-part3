## [](#rationale)理🤔

过度工程一直是许多开发人员的心头之恨。我们工程师有这种倾向，陷入混乱中，添加一些不错的特性或架构来支持可能在遥远的未来发生的事情。为了避免这种情况，我们为项目创建了约束条件并解决了这些约束条件。

但是如果我们能逃出去一次呢？如果我们做对面的**呢？**

这就是这个挑战的来源。

## [](#the-challenge)挑战💪

挑战在于创建一个**待办事项应用**并将其过度设计到极限。

*   用你所知道的最夸张的技术制作它。
*   使用你能使用的最奇特的技术。
*   用最夸张的架构来构建它。
*   以最疯狂的用户体验进行设计。

**[叉这个项目](https://github.com/johnpaulada/overengineeringchallenge)** ，让你的想象力尽情驰骋。你能做的没有限制。

**只是 OVERENGINEER。**

你有许可证做这件事。

## [](#example)举例📜

以下是我做这个挑战时的想法:

*   使用苦艾酒创建酏剂 GraphQL API
*   使用 ReasonReact 创建 web 应用程序前端
*   使用 Kubernetes 将 web 应用程序缝合在一起
*   使用 Flutter 或 bs-react-native 创建移动应用程序
