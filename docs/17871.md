# 设计的民主化

> 原文：<https://dev.to/metamn/democratization-of-design-3428>

WordPress 是一家与脸书、Twitter、Tumblr、Shopify 同时诞生的公司，如今为 30%的网站提供支持。该公司本月推出了一个新版本的网络应用，该应用具有一个名为 Gutenberg 的改变游戏规则的功能。

Gutenberg 是一个新的网页编辑器，使人们能够可视化地设计自己的网站。

这并不是第一次尝试让人们自由设计自己的网站。Atavist、Wix、ReadyMag、Jux 之前都曾尝试过，但一如既往，在创新和失败之后，当尝试真正成功时，会有一种动力。

这种势头就是现在。由 WordPress 通过其巨大的影响力和生态系统，以及脸书创造的革命性工具 React 提供。

这两个放在一起不能错过让每个人都成为设计师。

登录 WordPress，把一些叫做 Gutenberg blocks 的 React 组件放在一起，瞧，几个小时后你的网站就准备好了。

## 地震

当每个人都是设计师时，背景会发生翻天覆地的变化。一些经典的软件工作——UI/UX 设计师、前端和后端开发人员——将和整个行业一起沉没。

这些角色及其工具将不再需要，或者更少需要。人们将使用一种新的设计工具——浏览器！—他们会实时使用它。

他们不会启动 Photoshop 或 Sketch 来设计，而是等待上帝/奇迹/另一个开发者来实现他们的创造。

他们不会打开浏览器，进入后端、仪表板、内容管理系统或我们今天所知道的电子商务引擎。他们将在前端进行实时的*所见即所得*的改变。

取代单一的、不兼容的、现有的系统、服务器和应用程序，原子的、高度交互和可互操作的组件将无处不在地运行在微服务架构上。

组件。UI/UX 设计、前端代码和后端支持的混合，以自我维持的方式打包。你只需要把内容放到上面，然后选择一种风格来展示。仅此而已。

## 痛苦不堪

设计的民主化是从 WordPress 开始的，这将会给这个行业的一半带来痛苦，就像人工智能夺走工作岗位一样。

取代数百万个主题、数百个视觉主题构建器、数十个内容管理和电子商务系统的将是一些组件框架，它们彼此兼容，使用通用设计系统，如谷歌的 Material Design 或来自另一个主要参与者的另一个系统。

我个人对此很高兴。我们的行业人口过剩，严重扭曲，许多江湖骗子和一夜成名的像素推手每天都在重新发明同一个轮子，隐藏在冠冕堂皇的头衔后面，为大笔金钱驱动全球中产阶级化。是时候冷静下来，面对现实了。

## 场景

我真正关心的是，没有设计教育和审美意识的广大公众将如何设法设计出一个好看的、完成的网站？

我们如何避免所有的网站看起来都一样？

厌倦了社交网络的大众将很快开始离开孤岛，寻找一些原创的东西来表达自己，即使是以喜欢为代价。

他们会涌向 WordPress 选择设计系统；挑选一些组件和配色方案；一个下午他们就会欣喜若狂地想出解决办法。第二天，他们会醒来，意识到 a)他们不会赢得下一个奖项，b)他们的作品看起来几乎和隔壁的家伙一样。

在接下来的短短几年里，我们能做些什么来满足他们呢？我们不希望每个新网站看起来都一样，也不难看。我们不想让人们在对社交网络失望之后，立即对网络失望。

## 智能组件

我猜这场比赛是为了创造智能组件，而这些组件不会被外行人滥用来破坏相关的设计系统。

或者:创建基于智能组件的高度可定制的设计系统，即使在严重误用的情况下也能保持其统一性。

我们不想要另一个主题福里斯特或地理城市。

几年前，我写过一篇类似的科技小说文章，名为《作为设计师的设备》( Devices as Designers )( T1 ),在这篇文章中，我在思考由人工智能组装成一个系统的高度智能组件，这些组件从手表、网站到摩天大楼上的互动装置都有展示。

我们离那不远，也不近。但还是要走这条路。代替设备，人们将以一种不太智能的方式完成组装界面的任务，迫使我们拿出更智能的组件和设计系统。

我们的新工具首先要教会人们如何设计。认识到他们真正想要的不是在画布上扔出另一个滑块和两个带有杀手英雄的渐变，而是传递一种信息。

当这些新发现的设计火焰喷射器——我们称之为组件——被付诸实践时，我们应该想出设计向导来框定日常设计师的冲动，并首先推动创建可用的内容，然后以协调的方式让狂暴发生。

这场竞赛的赢家将是那些教育用户、迫使他们坚持最佳实践的公司，在基础设置完成后，用新颖的、基于人工智能的组件释放他们的用户，而不是再次用滑块、英雄和汉堡菜单淹没他们。

是的，我们将不得不为我们的组件配备语义，使它们能够被作为设计师的用户以及由人工智能提供动力的底层技术栈理解，从而带来娱乐，或许还有新的启示。
