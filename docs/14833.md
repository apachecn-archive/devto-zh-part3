这立刻让我想起了马克·纳特几年前的一篇精彩分析的一部分。他推断，谷歌与网络(和标准机构)的紧密联系，使得网络作为一个平台，成为了脸书不希望面临的风险。他说，React 是将浏览器抽象化的一个步骤。

Gramshackle 追踪了谷歌和脸书的关系，并概述了他认为脸书想要篡夺谷歌所需要的条件。他认为 React Native 是一种在移动平台上添加抽象层的方法，GraphQL 是 REST 的替代品。他们甚至可能针对 HTTP 本身，考虑到它对这个世界的重要性，这是一个可怕的想法。

我认为这是完全合理的，甚至是可能的。JS-JS-JS vs HTML-CSS-JS 仅仅是将浏览器抽象化，作为一个公司试图在一个由不是他们的参与者主导的生态系统中茁壮成长的结果。

所以我同意，这感觉像是一场关于 web 未来的战争，标准机构被用作武器(注意:GraphQL 和 JSX 是标准，但不是 W3C)，但不是针对 web 开发人员，也不是基于团队规模(我们只是夹在中间)。但这是一场我们以前见过的战争。Mac 还是 PC？随之而来的是浏览器抽象出了操作系统，以及其他有助于浏览器作为一个平台取得成功的因素。谷歌赢得了这场战争。现在，脸书正在我们最新一轮的科技战争中使用类似的剧本。

如果你不知道脸书的意图，那么他们对多年来网络平台发展的践踏绝对令人沮丧。目前缺乏网络的替代品会让他们看起来像是一群疯子。

但我注意到最新一代技术的惊人的文档和易用性，我相信这是因为创作者真的真的希望你使用他们的东西。以 [GatsbyJS](https://www.gatsbyjs.org/) 和他们惊人的教程系列为例。我承认潜在的复杂性令人压抑，但它是为了让一个人能够掌握工具并有效地使用它们。

我认为正在发生的一切都在缩小大团队和小团队之间的差距，我很高兴看到行业朝着这个方向发展。
