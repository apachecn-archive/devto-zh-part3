# 对反对可访问性的五个(错误)论点的反驳

> 原文：<https://dev.to/maxwell_dev/comebacks-for-five-wrong-arguments-against-accessibility-5g5j>

正如我和其他几位 a11y 倡导者所知，让别人关注可访问性并不容易。考虑到许多道德和商业上的理由，这是应该的，但事实并非如此。我在网上多次重复这些论点。有时是直接的回应，有时是痛苦的尖叫。还有很多工作要做。

为了让这更容易，我决定提前写一些答案。以下是我听到的反对可访问性的五个常见论点，以及我的反驳。我写下它们，这样下次有人在网上缠着我们要答案时，我和其他人可以链接到它们，以节省我们的时间和精力。

我们开始吧！

## 1)我们的大多数用户不需要可访问性

这是我在网上看到的最常见的回答，几乎总是来自白人。这是为什么呢？我的猜测是，从表面上看，这似乎是合乎逻辑的——如果大多数人不使用它，为什么要包括它呢？易接近性就像秘密的鳄鱼坑。很少有人在他们的公寓里使用它们，所以没有公寓提供它们。这是有道理的，也让找地方住变得更加困难。

但这是基于一个错误的想法，即无障碍环境只帮助那些有持续或明显残疾的人。一些人确实从一个可访问的网站中受益更多，但是每个人都在某种程度上受益。

我写的一篇关于可访问性的文章中的一个重点是记住压力案例。压力案例不仅仅是我们认为的残疾，比如失明。它们包括影响每个人的临时或环境条件。即使是享有特权、身体健全的应用程序的受众也会处理压力情况，例如:

*   折断一只手臂
*   受损或退化的视力
*   在意想不到的屏幕尺寸上使用应用程序
*   光线不足或刺眼
*   正在接受妨碍他们认知功能的药物治疗
*   使用糟糕的无线网络，因为他们在躲避强盗，而这些强盗没有落入鳄鱼陷阱

当有人指出他们的受众是忽视可访问性的原因时，他们就错了。* *该群体现在不需要可访问的网站，但他们会需要。在某种程度上每个人都有。* *不建立一个可访问的网站意味着你可能会疏远所有人(并失去他们的业务)。

## 我们会在需要的时候添加它

坦率地说，我认为这只是一个完全避免可及性的借口。我一直告诉自己，我会更换鳄鱼坑的假地毯，但我从来没有这样做，看看我现在在哪里！(请不要看，因为我需要保持低调)。

回到正题，**推迟可访问性是行不通的，因为它需要在早期就成为优先事项。如果可访问的标记和样式添加得太晚，每个网站的许多部分都需要重做。在这一点上，有人会说“我们现在不能改变这些组件，所以我们必须跳过这一步。”如果有人抱怨网站在线时的可访问性，那也是一样的反应。“它会改变太多的设计，所以我们什么也做不了。”**

这就像有人把一个房间漆成血的颜色，如果它让人们毛骨悚然，他们会改变颜色。当人们爬出来时，同一个人说重新粉刷房间是一个太大的改变，难以管理。这是一种逃避，让人们责怪(通常是)假装的带宽不足，而不是他们做出的选择。

这就是为什么尽早获得可访问性认同非常重要。越晚把它列为优先事项，就越难在不破坏你所建立的基础上继续工作。

这是站不住脚的，因为它违背了编程的核心原则:自动化枯燥、重复的任务。比较以下两个选项，告诉我哪个听起来更有效。

1.  花一两个小时来建立一个可访问的界面
2.  让你的客服人员每个月浪费几个小时处理投诉。他们重复同样的互动，使原本基本的过程变得冗长和令人沮丧。不打电话的客户会放弃，去找竞争对手。

是啊，第二个不理想。当可访问的界面可以节省时间和精力时，时间和精力都浪费了。这对客户、你的同事和你的底线都是残酷的。

## 4)太难了

许多开发人员告诉我，语义 HTML 和测试压力案例“太难处理了”

这很容易被推翻，因为可访问性可能很难，但作为一名 web 开发人员也很难。习惯就好。

好的 CSS 是坚韧的。设计一个数据库是困难的。驯服鳄鱼而不惊动你的房东是很困难的。但是做艰难的事情是我们的工作。

JavaScript 很难，但是开发者知道学习它的规则和权衡是值得的。可访问性是相同的，只是有不同的规则和权衡。

在最近的一个工作项目中，很难制作一个可访问的下拉菜单，因为它需要:

*   使用鼠标和键盘展开和折叠
*   展开时只有键盘导航的菜单项
*   在手机上向右显示
*   不与语义标记中的其他元素冲突，包括徽标链接和标题
*   不会因动态菜单项而中断

处理这种和其他可访问性需求是很困难的。但我们的工作是让它变得容易理解。我们做了研究，写了代码，投入了时间，并使它工作。因为这是我们该死的工作。

## 5)没关系

这是最严厉的回答之一:他们根本不在乎可访问性。我将跳过我在这个回答中发现的道德问题，例如:

*   将大量的人视为“不值得”他们的网站或项目
*   互联网的前提是如何更好地获取信息和交流。不可访问的网站破坏了它的伟大之处。

不过，我不会用这些论点来反驳那些不关心可访问性的人。他们就像通过鳄鱼陷阱在死亡中幸存下来的人。他们的思想是愤怒的，一成不变的，没有镇定剂改变它是毫无意义的。因此，在我完善我的工具，通过 Twitter 向人们发布它们之前，我会节省精力。

更好的回答是，从商业角度来看，无障碍网站也很有帮助。无法访问的网站打击了这些人最关心的东西:他们的钱包。

我在以前的可访问性文章中也谈到了这些，但是三个商业好处是:

1.  更大的受众，意味着更多的客户和数据。不可访问的网站迫使用户参与竞争。
2.  客户服务资源减少。你可以把这些再投资到扩大公司和网站的功能上。
3.  防止因无法访问的网站而被起诉。对违反《美国残疾人法》的网站提起的诉讼越来越多。

不管他们对可访问性的感觉如何，没有人能忽视它对利润的影响。

我更喜欢用“不要做别人的鸡巴”来说服人们。但我早就知道有些人不关心道德或善待其他群体。关于金钱的争论是一个不幸但有效的解决方法。

## 包装完毕

下次有人反对可访问性时，请记住这篇文章，无论是在线还是当面。我鼓励你把它作为参考(或者给他们发一个链接以节省更多时间)。

我们的时间最好花在富有成效的任务上，比如提高我们的可访问性知识，构建更好的界面，以及摆脱那些需要摆脱鳄鱼咬伤的人。说真的，他们会在几个月内痊愈。继续前进。
