首先:什么是 SQL 注入？

根据 OWASP(开放 web 应用安全项目)的说法，SQL 注入是:
“SQL 注入攻击包括通过从客户端到应用的输入数据插入或“注入”SQL 查询”([https://www.owasp.org/index.php/SQL_Injection](https://www.owasp.org/index.php/SQL_Injection))

这是什么意思？简单来说，这是一个注入(意味着输入命令或查询)到一个领域与 SQL 代码，允许网站数据库的操作。

那么它到底是做什么的呢？

如果你的网站容易受到 SQL 注入病毒的攻击，那么它可能会很危险，原因如下。如果成功，SQL 注入可以:

1.  利用和读取数据库中的敏感数据。如果操作正确，SQL 注入可以从数据库中读取敏感数据，例如电话号码、地址，甚至密码(通常 SQL 命令会收集所有这些数据。所以，所有的密码，所有的地址等等。甚至是管理员！！！)
2.  另一件事是它有可能搞乱*实际的*数据。这是非常直接的，但基本上它将能够插入/更新/删除。(**注意:如果黑客没有意识到他们在做什么，这可能是无意中发生的。他们可以输入一个 SQL 注入命令，而不熟悉数据库，并可能删除特定数据或全部数据！插入或更新也是如此。另一个你应该保护自己免受 SQLI 攻击的原因！**)
3.  扰乱数据库管理系统。SQLI 也可以扰乱数据库管理系统。它有可能激活管理命令(如关闭 DBMS)，它还可以从 DBMS 中恢复文件，甚至可以向操作系统发出命令。
