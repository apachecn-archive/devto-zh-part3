# 什么是事业所有权？它对我有什么帮助？

> 原文：<https://dev.to/jamesmh/thoughts-on-career-ownership-4bbc>

职业所有权只是谈论掌控自己职业的另一种方式。

这是关于认识到你的职业是一个有机的东西，你可以培育和成长，忽略甚至破坏它。

# 两类

每当我谈到“职业”这个词时，我喜欢把它分成两大类:**声誉和能力。**

是的，在你的生活/职业生涯中，特定的工作对你的职业生涯有一定的影响。例如，如果你在过去 10 年里做过 3 份营销工作，那么就你的职业生涯而言，你是一名营销人员，职业是市场营销。

然而，我喜欢把你的职业生涯想象成一段旅程——一段仍有未知潜力的旅程。

转向不同行业的潜力(正如我看到许多人所做的)。成为特定行业领导者的潜力。等等。

鉴于人们从一个行业跳槽到另一个行业实际上很常见，我认为关注声誉和能力是最重要的。无论你从事什么工作或行业，这些都会伴随你。

# 能力

我在职业生涯早期经历的一件令人震惊(嗯，也许不那么令人震惊)的事情是，我曾经共事过的高级开发人员甚至架构师显然不是因为他们能力太强，而是因为他们仅仅有 X 年的经验，并且认识合适的人。

当时，我告诉自己，我真的不想仅仅因为我有 X 年的工作经验就担任领导或高级职位。我想拥有做好这份工作的技能和经验。

这是激励我开始[电子邮件简讯以帮助开发人员在职业生涯中成长的一个重要想法](https://eepurl.com/gdIV5X):我希望你们成长为高级和领导角色，因为你们有能力和技能来做这项工作！

注意:认识合适的人和有经验确实有所帮助——但这两者是相辅相成的。

就你的职业生涯而言，如果你能拥有在你的行业中取得成功所需的能力，那么你就拥有了一块基石。

换句话说，你可以把工作做好。

# 名声

而名声，更多的是别人对你的看法。

❔:别人认为你是一个有能力的软件开发员吗？

❔:你善良吗？

❔有帮助吗？

❔智能？

❔做得好吗？

❔产生结果了吗？

❔:你能学得很快吗？

❔:你擅长指导和教导他人吗？

如果你有做好一名开发人员所需的技能，其他人怎么会知道呢？

他们如何证实这一点？

## 建立你的声誉

建立声誉只是谈论人们是否知道你的价值、你有能力做什么以及你是谁的一种方式。

换句话说，一般来说，建立声誉就是帮助他人了解你的能力。

让我们强调这一点:

> 建立声誉就是帮助别人了解你的能力。

越多的人知道你和你带来的价值，你就越能为社区提供更多的认可。

这会给你带来更多的机会😉。

这里有一些方法可以帮助你开始成长，并向他人展示你带来的价值:

✔博客

✔评论行业领袖在社交媒体上的帖子

✔参加用户组和聚会，结识新朋友

✔在一个用户群中演讲

✔通过推特或在社交媒体上发帖说出你的想法

✔建设公共可用项目

✔ YouTube 视频

✔写了一本书

✔只是尽你所能帮助别人

# 一个你可能想尝试的例子？

我做过的一件事是在 LinkedIn 上找到过去的同事和经理，给他们发信息询问他们过得怎么样。

接下来，我会给他们一些关于我最近在做什么的句子。

最后，我会让他们知道，如果他们有任何需要，我随时都愿意和他们聊天。

这样做给我带来了一些很好的机会。在某些出版物上发表客座博文，获得一些兼职工作等等。

## 被人“顶礼膜拜”

这是确保你在周围人心目中处于“顶端”的一种方法。

想象一下，前经理或前同事跳槽到另一家公司，现在需要找到一个真正擅长 X、Y 和 Z 的可靠开发人员(无论你的显著优势是什么——你应该推广的优势！).如果你已经在这个人的脑海里，猜猜这个人会和谁联系这个机会？**你！**

如果你想确保你的职业能够经受住生活的风暴，并帮助你获得那些令人敬畏的工作，那么你需要掌控一切！
