# 我如何通过手机应用程序赚到 2000 美元

> 原文:[https://dev . to/claudiobernasconi/how-I-earn-2000-with-a-mobile-app-29cj](https://dev.to/claudiobernasconi/how-i-earned-2000-with-a-mobile-app-29cj)

我通常会写博客。NET 相关的主题。一个朋友问我，为什么我从来不写关于我的手机应用程序的博客，我是如何用它赚到 2000 美元以上的。嗯，我猜是真的，我从来没有谈过，也从来没有写过([除了 2015 年的这篇文章](https://www.claudiobernasconi.ch/2015/04/02/10-reasons-why-you-should-upgrade-your-existing-android-application/))。

我必须承认，我的手机应用程序不是. NET 应用程序，而是用 Java 编写并在谷歌 Play 商店发布的 Android 应用程序。

终于到了揭示所有让我变得富有的秘密的时候了，这样你也可以变得富有——我在开玩笑。这篇文章提供了我在编程、营销、销售和重写我的第一个移动应用程序方面的经验。

我还分享了我在这个过程中学到的东西，我会再次做同样的事情，以及在另一个移动应用项目中我会做什么不同的事情。

# [](#did-i-really-earn-2000)我真的赚了 2000 美金吗？

先说钱。是的，我在谷歌 Play 商店用一个简单的移动应用程序赚了 2000 多美元。让我们更深入地探究一下钱都进了我的口袋。

回到 2010 年 11 月，我发布了我的第一个(也是唯一一个)手机应用程序。在最初发布后，我在 2017 年春天进行了几次更新和一次完整的重写。

如你所见，我在发布后的最初几个月赚了大部分利润。在最初的 14 个月里(2010 年底和 2011 年全年)，我完成了大约 1480 美元的销售额。

总的来说，我的销售额约为 2179 美元(T1)，这是 1525 美元(T3)的净利润(T2)。

这也意味着从 2012 年到今天，我只赚了大约 700 美元。我想和你分享一些我学到的东西:

1.  第一次启动时，确保**你的应用程序运行正常，功能丰富**。当然，你可能会使用敏捷开发实践，但是你的应用程序应该能够工作，并具有最小的功能集。我的应用被推向市场，因此，在最初的几个月里，我获得了很多下载量。如果应用程序不能解决实际问题，人们就不会为它付费，商店也不会向新人推荐我的应用程序。

2.  开发一个应用程序不仅仅是写它的第一个版本并且永远不要再碰它。它不是像房地产或其他类似的赚钱机器。我不关心应用程序推出后，这是一个错误。我建议在应用程序发布后，只要它还在商店里，就计划开发它。我只做了几个错误修正，但我从未做出任何更重大的改进，也从未添加额外的功能。我想这就是为什么仅仅一年后**销售额就大幅下降**的原因。

3.  **你可以赚点外快**，但你不会因为一个 app 而变得富有。当然，如果你用整个工程团队和资金建造下一个优步，前途无量。

但是如果你是一个没有团队的独立开发者，你在空闲时间开发你的应用，不要指望从中赚那么多钱。我不得不说，每年 1000 美元对我来说很不错，我从来没有期望从中赚到超过几美元。

# [](#the-app-i-made)我做的 app

我制作了瑞士停车场指南。打开 app 就能看到哪些停车场开放，还有多少车位。

*更多截图可上 [Google Play]((https://play.google.com/store/apps/details?id=ch.claudiobernasconi.carParking)) 。*

除了列表，我还包括了一个显示所有停车场的地图视图，如果你点击一个停车场，你可以使用谷歌地图开始导航。

该应用程序的一次性价格为 1.95 美元。我自己不喜欢订阅，我想大多数消费者也不喜欢。因此，我决定用一个有前期费用的付费应用。2 美元在我的国家是一个小数目，我甚至可以提高价格，但我从来没有这样做过。

# [](#how-i-got-the-idea)我怎么会有这个想法

我想为自己创造一些东西，当我去瑞士不同的地方时，可以帮助我找到停车场。在城市里，经常有交通堵塞，我厌倦了寻找停车机会。我想跳进我的车，开到离我的目的地最近的停车场。

我从中学到的是:

*   心中有一个明确的目标是至关重要的。你的应用应该为用户解决一个问题。这个问题有多难解决并不重要；app 解决问题是必不可少的。

*   完成这个项目的动机是能够为我解决现实世界的问题，并学习如何开发移动应用程序。在这个应用程序发布的时候，移动应用程序是相当新的，我想获得一些开发移动应用程序的经验。

# [](#how-i-developed-the-app)我是如何开发这个应用的

回到过去，Eclipse 和 Android 开发工具(ADT)是必经之路。我在我的电脑上安装了那个软件，我已经准备好了。

构建一个应用程序，我需要知道的一切就是我已经掌握的编程技能和 Android 平台的在线文档，当然，还有测试应用程序的 Android 设备。过去，我用的是一部 HTC Desire T1，这是我的第一部智能手机。

# [](#the-rewrite-in-2017)重写于 2017 年

几年后，我回到这个项目，并考虑更新这个应用程序。我不得不使用新的谷歌地图 API。旧的 API 正在退役，还有其他平台的变化，比如物理按钮从设备上消失了。

我回到代码库，让我告诉你；使用 Eclipse 和 ADT 进行开发是一件痛苦的事情。2010 年工具很差，7 年都没有改善。至少不是我想用的东西。我习惯了 Visual Studio，它发展了很多，提供了更多的功能。

是时候试试新的安卓工作室了。我安装了 Android Studio，并决定这个应用程序太老了，我要从头开始写。我打开 IDE 并开始工作。除了少数资源，我完全重写了应用程序，以适应当前的平台要求。

当我完成了这个项目并发布了我的应用程序的第二版时，我希望它能再次获得牵引力，并实现一些销售。正如我在上面描述的那样，事情并没有按照计划进行。

# [](#things-i-would-consider-for-a-new-project)我会为新项目考虑的事情

进入一个新项目，我会考虑以下关键经验:

*   确保解决一个现实世界的问题。如果应用程序解决了一个问题，它就有一个潜在的市场。

*   你的 app 要很好的解决问题。商店里有数百万个应用程序，你希望人们安装你的应用程序。它必须脱颖而出，做得更好、更快，或者以一种独特的方式。标记人员会将这一点称为[独特的销售主张(USP)](https://en.wikipedia.org/wiki/Unique_selling_proposition) 。

*   你可以像我上面详细解释的那样赚钱。人们愿意花钱购买能解决他们问题的应用程序。当然，可能会有免费的应用程序，如果你不想赚钱，你可以创建一个免费的应用程序。但是如果你想销售你的应用，确保定价正确。在我的例子中，我以 1.95 美元的价格出售我的应用程序，这个价格对我来说相当不错。我不得不承认，我从来没有试过更高的价格。确保测试什么最适合你，当你的应用是新的时候，不要犹豫经常改变价格。

*   完成一个项目，比如从头到尾开发一个应用程序，帮助我理解了软件的项目生命周期。从规划到设计、开发和营销，我必须自己做所有的事情，我学到了很多。这些知识对我理解我日常工作所在公司的业务流程帮助很大。

# [](#technology-choices)技术选择

如果我今天创建一个新的移动应用程序，我会尝试 Xamarin。微软在 2016 年收购了 Xamarin。微软收购 Xamarin 之前，它的开发非常昂贵。收购后，大部分资源，如 Xamarin 大学，对开发者免费开放。

此外，我为 Android 平台编写的 Java 代码的一个缺点是，该应用程序只能在 Android 智能手机和平板电脑上运行。我的目标一直是把这款应用程序带到 iPhone 上，但没有 Mac，我无法编译这款应用程序，而且我还必须用另一种编程语言重写代码。

Xamarin 和 Xamarin Forms 提供了一个引人注目的平台来开发一个单一的代码库，交叉编译到 Android 或 iOS，两者都为各自的平台提供了一个本机应用程序。

# [](#conclusion)结论

这篇文章解释了我在 2010 年创建我的第一个(也是唯一一个)移动应用的方式和原因。它还揭示了我从中赚了多少钱，以及用一个简单的应用程序赚钱是可能的。

如果我遇到另一个我想解决的问题；我会考虑再写一个新的应用程序。这是一次有趣的经历，额外的几美元帮助我支付了大学期间的账单。

虽然用 Java 写 Android 应用还是可以的，但是作为一个有经验的人。作为一名热衷于 C#编程的. NET 开发人员，我会在我的下一个移动应用中尝试使用 [Xamarin 和 Xamarin Forms](https://docs.microsoft.com/en-us/xamarin/xamarin-forms/) 。

如果你也写了一个应用程序，请在评论中告诉我你使用了哪些技术，以及你的结果是什么。
