# 一年技术工作

> 原文：<https://dev.to/alivenotions/one-year-in-tech-22g2>

这篇文章是关于我在大学毕业后一年半的职业生涯中所学到和意识到的一些事情。

这些事情是对未来的提醒——我，因为不遵循它们是非常容易的，所以我是在和自己说话，就像这篇文章的普通读者一样。

## 成为好社区的一部分

参与社区是你做任何工作的巨大动力。当你遇到困难时，社区会帮助你，让你集思广益，并让你的工作成为一种社交活动。

这个社区可以是你当地的社区，通过 meetups，Discord 频道，Twitter，Reddit 等等。我个人注意到的一件有趣的事情是，当你不想做某事时，让自己参与讨论(只是通过阅读它们)可以极大地推动你自己开始尝试事情。去年我加入了几个社区。我在大部分时间里都是一个倾听者，并将寻求以不同的方式做出更多的贡献。

吸引我加入 React 的原因之一是这个令人惊叹的社区。查看一下 React flux discord 频道，Twitter 上的社区(React 核心团队，瑞安·弗洛伦斯，肯特·c·多兹等人)，YouTube 上的 funfunfunfunction 频道等。

顺便说一句， [MPJ](https://twitter.com/mpjme) 是我跟随他之后开始做的许多开明思考的最大原因之一。你绝对应该去看看他！

## 对为什么感兴趣

仅仅知道一个事物的 API 是很好的，但是试图理解这个概念的推理可能更有成就感。它有助于你了解为什么某样东西会被创造或发展，以及它在更大的图景中是如何适应的。这也让你意识到你所学的很多东西可以很容易地转移到其他各种事情上。这可以让您专注于构建解决方案，而不是一直担心工具。

我发现的另一个好处是，它让人们更容易对图书馆(或任何人)的决定感同身受。对为什么事情会以这种方式进行感兴趣会鼓励你不要很快放弃事情(我注意到很多人都这样做，很粗鲁)，因为在你形成观点之前，你会渴望知道背后的原因。

## 尝试跨栈学习

这并不意味着你试图学习世界上的一切。我认为那无论如何是不可能的。这甚至不意味着你试图了解你目前正在做的项目的一切。但是我认为理解应用程序的不同活动部分如何相互作用以及信息如何从一个模块传递到另一个模块是非常有益的。

这不仅使与不同团队和人员的对话变得更容易、更全面，还能让你对团队中的每个人产生巨大的共鸣，因为你意识到了他们工作的重要性。

这不是为了填充简历，而是为了进行更丰富的对话。

## 射击较重的东西

读那些你没有 100%得到的东西。或者目前超出你技能水平的事情。你可能只能理解其中的 50%,甚至更少，但它满足了你的好奇心，让你在内容中挣扎。你会做你自己的研究，寻找东西，问你自己的原创问题。

这也是一个令人谦卑的练习，因为你会意识到有些事情你不理解，必须努力去理解。但是在兴奋的感觉和说你很烂的感觉之间有一条细微的界限。每当你要阅读/学习更重的东西时，就把它当作一个自由判断区。要知道在任何时候停下来都是可以的，如果你从整个练习中拿走了哪怕一件东西，那就是胜利！

这可能是整篇文章中最个人化的一点，对你们中的许多人来说可能不一样。

## 有自己的副业

从大学走向工作的最大现实打击是意识到很多工作并不总是那么有趣。你最终会做很多无聊的事情，有时只是做一些随意的事情，或者很多次没有被分配任何工作。我很多大学开始工作的朋友也面临这种情况。

这可能会让你开始紧张，并给人一种不满足感，尤其是如果你在这个领域还不太成熟的话(通常我们刚开始的时候都是这样)。

我发现业余时间拥有自己的东西对健康的精神状态绝对至关重要。这并不一定意味着只是在工作之余编码。你可以沉迷于你的爱好，学习新的东西等等。

你的办公室工作不能定义你是谁，但它仍然是你一天中的一大块。它的缺点是不稳定，随时都可能改变。但是做自己的事情在你的掌控之中，你选择什么带给你满足感。

## 有些日子心情不好是正常的

有很多次，当我没有达到自己的期望时，我觉得自己很糟糕。不管是与编码、音乐还是诗歌等等有关，我想很多人都会时不时地有这种感觉。

我想告诉你，有这样的感觉是完全自然的。出去，做你喜欢的事。请对自己宽容些。伴随着没有效率的感觉的是这种强烈的无价值感，因为你觉得自己根本没有进步。请对自己宽容些。

人们在一段时间内学习和提高，但我们通常无法评估这种进步，这使我们低估了自己。深呼吸，一两天后再来讨论这个问题。一次承担一天的负担。不多不少。

如果你觉得这种低落的坏情绪会持续一段时间，**请**去和某人谈谈，寻求帮助。抑郁症是真实存在的，是一件非常丑陋的事情。

## 睡眠

这可能是一件需要严格自律的事情，因为即使是相对良好的意图也会妨碍睡眠。从事某项工作并富有成效的诱惑也让我们觉得我们应该熬夜并继续工作。

我们觉得我们正在努力工作，磨练我们的技能，但从长远来看，这只会阻碍我们的进步。我过去在大学经常熬夜，过了一段时间后，我意识到我只是一直感觉很累。一天中的任何时候我都不完全健康。我已经认真地开始有规律的睡眠，我感觉更清新，我的注意力水平也提高了。你需要找到适合自己的睡眠量，有些人需要 6，有些人需要 8。在这件事上不要听别人的。

如果你的睡眠时间比你身体需要的少 2 小时，你就不会节省 2 小时，相反，你是在用 2 小时来换取一天中较低的效率。

## 看大局

关心你写的代码是一件非常好的事情。它允许您编写可读和可维护的代码。但是一旦关闭编辑器，您就需要停止关心它。要意识到编写代码只是交付产品的一部分。

这有两条路可走。首先是把你写的代码放在心里。这使得获取反馈变得困难，代码审查看起来像是惩罚。有些人知道的比你多，这些人从不同的角度看待这件事(想想你对你朋友的关系提出建议是多么明智，但你自己却很糟糕)。代码只是实现最终目标的实现细节，而最终目标就是你试图构建的产品。

第二种情况是，当我们不关心编写测试和文档，匆忙完成部署工作流程，轻视讨论解决方案等等，只喜欢编写笨拙的代码和尽可能快地推送功能时，我们没有意识到更大的前景(经理和程序员都深受其害)。

我一直在这两种情况下挣扎，尤其是前一种，我注意到它不仅阻碍了你正在构建的产品的质量，而且也使你难以成长。

## 去你的好奇心带你去的地方

在当今世界，决定学什么是一个困难的选择。每个人都有关于你应该做什么或走哪条路的意见，这些意见中的许多都应该得到尊重。你可以选择下一步学习什么，基于当前的趋势或基于就业市场空间等。但这不应该阻止你追随你的好奇心。

我并不提倡失业和学习一门过时的技术，因为那不实际。但是一旦你有了工作，想学点什么，就去你的好奇心想去的地方吧。

有时候，满足求知欲是如此重要，以至于没人付钱给你也没关系。对于技术来说是这样，对于你想学的其他技能来说也是如此。很多时候，我担心学习一些东西在将来不会有任何回报。这是一个等待发生的遗憾。

此外，学习任何东西都会帮助你成长，让你成为一个全面发展的人。努力成为那样的人。

## 坚持>其他一切

这是我不得不通过艰难的方式学习的东西。在我职业生涯的前 2-3 个月，我会尝试在不同的垂直领域学习。无论那天听起来有多酷，我都会一试身手。购买课程、观看 youtube 播放列表、阅读系列教程等等。虽然这样做没有错，但痛苦的是这种做法不会留在脑海中，很快就会消失。

我几乎不记得任何 python，尽管我买了两门课程，完成了其中一门课程的 80%,包括三四个项目。我开始忘记去年 4 月刚刚完成的机器学习课程的细节。

其核心原因是缺乏坚持。少学点，但坚持不懈地学习，比每天花 12 个小时做不同的事情要好得多。在相当长的时间内投入到你想学的东西中。
