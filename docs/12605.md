# 我今年第三次重新设计了我的作品集网站！

> 原文:[https://dev . to/sdu duzog/I-redid-my-portfolio-site-今年第三次-25ee](https://dev.to/sduduzog/i-redid-my-portfolio-site-for-the-third-time-this-year-alone-25ee)

2018 年最后一个季度是 dev.to 在我的职业生涯中最有影响力的一年。更棒的是，因为这是我在大学队的最后一年。我受到启发，开始了一个[项目](https://github.com/sduduzog/slim-launcher) **，并实际完成了它**，同时听从项目建议，在我的作品集上炫耀。

## [](#from-vuejs-to-a-jekyll-generated-noscript-site)从 Vue.js 到一个 Jekyll 生成的‘noscript’站点

我带着一些由 Vue.js 支持的有趣动画的文件夹进入了一年，但作为我今年决心的一部分，我想开始写博客。我关闭了我的网站，用 Jekyll 和它的默认主题重新创建了它。没有任何 javascript。

## [](#from-default-minima-theme-to-more-custom-design)从默认的极简主题到更加定制的设计

好吧，我觉得 Jekyll 提供极简作为他们的入门主题，不太重也不太复杂，可以定制，这很棒。我把主页改成了我觉得最能代表我的地方。*在某个时候，我的作品集网站在一次工作面试中被打开，当时我正在投影仪上*

# [](#now-this)现在这个！

为了重新设计，我创建了一个新的分支。我删除了回购协议上的所有内容，并从一个新的 index.html 文件开始，从那里开始我的工作。和失业🤣，我已经在我的笔记本电脑前花了 10 个多小时更新我的网站，现在在推送更改后几分钟写这篇文章。

[在这里查看我不完整的作品集](https://sduduzog.github.io/)并告诉我你的想法。当然，它还没有完成，我需要为我的项目添加一个部分和一个链接到我的简历。我还必须为我的博客文章重新设计页面。我会使用任何 Jekyll 主题，但我觉得这样会从网站上带走我的个性。

更有趣的是，我在学习 vim 的同时也在做这件事。人们真的需要看看这篇文章，对于像我这样的初学者来说，因为“*退出 vim* ”的笑话，他们不敢迈出这一步，退出 vim 实际上比我想象的要容易得多😳
