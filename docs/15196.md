# 为什么表单验证很重要

> 原文：<https://dev.to/flippedcoding/why-form-validation-is-important-37mj>

我有一个挑战给你。去你最喜欢的网站看看有多少网站上没有表格。我会等的。如果你真的发现一些网站没有任何表格，把它们放在评论里，因为我只在超级老的网站上见过。

我的观点是表单无处不在。它们有不同的形状和大小，负责各种信息。这就是为什么很容易忽略创建表单需要多少验证的原因。

## 什么事

如果您将用户输入保存到数据库中，您最好确保您确切地知道该输入会带来什么。实现这一点的方法是使用验证让用户以您需要的格式输入信息。我相信你已经得到了无效的电子邮件地址的错误。

这是因为你需要以某种格式输入你的电子邮件才能通过验证测试。这也扩展到了常规的文本框。这就是为什么你不能把你的地址放在电话号码应该放的地方。这也是为什么你不能输入一个没有正确位数的信用卡号码。

## 为什么很重要

您需要表单验证作为一种安全措施。表单很容易成为黑客的目标，因为我们都知道它们以某种方式连接到数据库。让人们把他们想要的任何东西放到你的表单中，作为一个开始，你会受到 SQL 注入的攻击，这可能会变得更高级。

除了将您从黑客手中解救出来，表单验证也将您从用户手中解救出来。人们会做一些奇怪的事情。有人会因为喜欢你的表单而在表单中输入一个荒谬的值，从而导致你的网站崩溃。

表单验证的重要性远不止这些，但这是我见过的破坏项目的两个原因。

## 工作原理

谢天谢地，表单验证并不太难。有大量优秀的库，大多数框架，无论是前端还是后端，都有某种内置的验证。真的只是一堆支票。

电话号码是真实的号码吗？地址是否具有您期望的格式，它是否是一个真实的位置？用户是否在你需要的所有字段中输入了信息？你要确保你回答了所有这些小问题。

老实说，如果您需要一些真正基本的验证，您可能可以使用一些 if-else 语句、switch 语句和一些正则表达式。但是验证兔子洞有多深你就挖多深。

## 不同的使用方法

**前端**

如你所知，前端就是用户看到的和与之交互的东西。这就是为什么一个好的验证块包括你显示给用户的消息。这也是抵御不良表单数据的第一道防线。如果你能在前端做一个检查，那不是一个坏主意。

这样，在数据得到验证之前，后端永远不会受到影响，从而节省了时间。因此，您可以专注于定制错误消息，让您的用户知道他们需要修复什么。你能做的最糟糕的事情就是不告诉用户发生了什么。

表单验证与用户体验和安全性一样重要。让用户像你希望的那样轻松地输入数据，而且大多数时候他们都会这样做。如果他们不这样做，那么他们将得到一个容易理解的错误消息。

**后端**

这是你在数据库前的最后一道防线。在对数据库进行任何更改之前做一些额外的验证是没有坏处的，有时您只能在后端做这些。另外，您可能有一些更强大的工具来帮助处理数据的验证。

在后端进行验证也使它更加安全。没有人可以打开开发工具，深入研究代码，找出如何击败你的验证。

在时间紧迫的情况下，很容易回头进行表单验证。在您处理表单时，请继续进行处理。这会节省你以后的一些返工，而且当你把它展示给你的老板或客户时，它会让你的应用程序看起来更加完美。
