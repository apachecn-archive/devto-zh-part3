## [](#github)GitHub

GitHub 已经是开源世界的领导者，拥有 2000 万个知识库，并且还在增加。开发者第一次可以免费使用 GitHub 进行他们的私人项目，每个资源库最多可以有三个合作者。许多开发人员希望在公开发布之前，使用私有回购来进行辅助项目或私下尝试，以鼓励越来越多的开发人员参与开源工作。

根据他们最新的博客，很明显微软的最终目标是重新引导 GitHub 用户使用他们的 Azure 服务。

在 GitHub 上关注我: [mittalyashu](http://github.com/mittalyashu/)

## [](#gitlab)GitLab

尽管 GitLab 来自相同的背景，但它仍然有一种独特的远程工作方式，并以手册的形式开放知识和指导方针，他们遵循这些来运营公司。

尽管 GitLab.com 最近超过了 1000 万个存储库，而且像 Drupal、CMake、GNOME、Xorg、Debian 这样组织都在使用 GitLab 的自托管版本。

当人们听到 GitHub 在免费计划中提供无限私有库的嗡嗡声时，他们仍然不相信，因为 GitLab 仍然通过免费提供 CI 和部署功能而获胜。

甚至在 [CodeCarrot](https://www.codecarrot.net/) 我们也使用 GitLab 为我们的客户构建产品。

### [](#bitbucket)BitBucket

BitBucket 因免费提供无限的私有存储库和管道特性而广受欢迎。与 GitLab 相比，我会说 GitLab 仍然是赢家，因为 BitBucket 仅提供 50 分钟/月的构建时间，而 GitLab 在共享 runners 上提供 2000 分钟/月的构建时间。

如果你喜欢自助服务，你可能会发现一次性付款更好。

我仍然不知道 BitBucket.org 储存库的数量。

你对此有什么看法？你更喜欢用 GitHub，BitBucket 还是 GitLab 哪个？
