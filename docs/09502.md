# 你在新技术中失败的 3 个原因

> 原文:[https://dev . to/larryfoobar/you-will-never-learn-something-236](https://dev.to/larryfoobar/you-will-never-learn-something-236)

开发者写代码，设计师画界面，建筑师规划建筑，工程师造东西。各种专家做不同种类的事情。但是他们都有什么共同的行为呢？

这个动作叫做**学习。学习是你需要从事的每项工作的必经阶段。但是要真正学到东西，你需要学会如何学习。**

通常，在开始工作之前，我们会花大约 10-18 年的时间来学习(学校、大学、硕士学位等)。但事实上，这种做法可能会在未来误导我们。

<center>**Meet Peter. Peter is a back-end web developer.**</center>

![Meet Peter Developer](../Images/8c08a028c17c6b08313e3953502810a4.png)

他用 Python 写代码，擅长数学和数据结构。他喜欢探索新技术，并希望扩展他的开发堆栈。最近彼得开始对机器学习(ML)表现出兴趣。他被一些使用 ML 构建的新的流行项目迷住了。他肯定决定开始学习它。由于他的日常工作不需要 ML，他想出了一个抽象的想法，即建立一些副业。

彼得有一个计划。这对他来说很好:

1.  下载(或购买)几本关于这个主题的顶级书籍。阅读它们。
2.  在 Coursera/Udemy/Lynda/等注册。报名参加一个关于这个话题的课程。完成它。获得证书。
3.  观看 Youtube 上关于该主题的一系列讲座。整整 160 个小时。
4.  草拟一个 ML 项目的想法。
5.  在 GitHub 上开发一个项目
6.  尽情享受吧！

熟悉吧？但是为什么会以这样的方式发生呢？

至少有几个原因:

1.  彼得习惯于这样做。从童年开始，我们的学校、大学、亲戚和朋友都教他这样做。“先学习，后实践”的模式是他唯一可能的选择。
    T3![learning](../Images/95ce894880e49ac0c5de85a829a5b986.png)T5】

2.  对新知识的恐惧。投入到所有这些新知识中，Peter 意识到他还有多少知识不知道，以及学习的道路有多长。理解这一点并不能激励彼得。
    T3![fear of knowledge](../Images/39aaccfbdb0c85b8147c95b524819ae9.png)T5】

3.  提前学习是不行的。一个人不可能在几个月的时间里吸收大量的新理论知识，然后才开始在实践中运用这些知识。学习不是这样的。知识需要巩固和立即应用。
    T3![rocko's knowledge](../Images/0f777eab6b751589a85501a6c89061bd.png)T5】

但是，彼得能做什么呢？

长话短说，彼得只需要立即开始做事，而不需要花很多时间在学习上。无论是*学习*还是*做*的过程都应该被分成小块，并组合成一个完整的过程，从而给彼得想要的结果。
彼得需要做范式转换:

<center>"*First Learning And Then Doing*"
→
"*Doing by Learning And Learning By Doing*":</center>
