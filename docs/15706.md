# 每个人都想反应自然。我还应该学习原生 iOS 吗？

> 原文：<https://dev.to/evandeaubl/everybody-wants-react-native-should-i-still-learn-native-ios-4n02>

移动开发者的市场比以往任何时候都大。你开始学习 iOS 开发的原因很有可能是为了得到一份全职的开发应用程序的工作。

如果这是你的动机，当你看到许多公司要求的一个主要要求时，你可能已经看过求职公告板，并对你的决定感到后悔:

反应自然。

即使你真的喜欢原生 iOS 开发，你也不知道学习 React Native 是否会让你更有市场。“我的意思是，毕竟 React Native 还是移动开发，iOS 和 Android 开发我都可以自称，再加上 JavaScript 的知识。这很好，对吗？”

**“为了让我得到报酬的平台，我可以放弃我热爱的平台，对吗？”**【暗示灵魂的微小碎片在里面死去】

我在这里说，你不需要。人们对原生 iOS 开发非常感兴趣，即使你根本不学习 React Native，找工作也不会有问题。

* * *

有许多 iOS 开发商店对在其应用程序中保留 iOS 和 Android 的“原生感觉”更感兴趣。另外，还有一些备受瞩目的应用，如 AirBnB，曾经使用 React Native，然后决定原生应用是更好的选择。

长话短说:仍然有很多理由选择 native 而不是 React Native。只要这些原因仍然存在——我不认为它们会消失——就会有对原生 iOS 开发者的需求。

我看到重复出现的一个重要原因是:跨平台框架来来去去。我是一家公司的开发主管，在那里我们最初使用类似的跨平台框架 Titanium 来构建我们的移动应用程序。我们最终转向了两个平台的原生应用，因为我们想添加许多 Titanium 很难支持的功能。最终，整个行业对钛的兴趣也是如此。

React Native 已经积累了相当多的追随者，但永远无法保证第三方依赖关系将与底层平台保持同步，或者保持适合您的需求。对于一家公司来说，这是坚持本地开发的一个非常有说服力的理由。

但也许你并不认为 JavaScript 有那么差，还是决定选择 React Native。了解原生 iOS 开发仍然会让你比一般的 React 原生开发者更有价值:

*   许多 React 原生应用是混合的:要么是带有 React Native 中一些屏幕的 iOS 核心，要么是带有原生 iOS 代码扩展的 React 原生核心。了解 iOS 让你更有能力弥合两个平台之间的差距。
*   如果 React Native 中的东西不像预期的那样工作，您可以依靠您的 iOS 知识来帮助您了解可能出错的地方以及如何调试它。
*   如果你已经掌握了 iOS 的知识，学习 React Native 会更容易，因为你已经熟悉了 Native 级别的概念。

只要 iOS 存在，iOS 开发就会存在，而且 iOS 似乎不会很快走向任何地方。如果你热爱原生 iOS 开发，现在仍然是把它作为职业追求的大好时机。
