# 大家好，我是阿曼达&这是我在 Lambda 学校的全栈开发项目一周后所学到的东西

> 原文：<https://dev.to/amlane/hi-i-m-amanda-this-is-what-i-ve-learned-after-1-week-at-lambda-school-s-full-stack-dev-program-580h>

编码很难。做好受挫的准备，即使你通常擅长学习新事物。即使是在该领域工作了几年的高级开发人员，仍然很难完全理解某些概念。允许自己成为一名初学者，并记住——努力！=不可能。

走出你的舒适区。如果你内向，尝试结对编程。如果你觉得 HTML & CSS 很舒服，那就多花点时间学习 JavaScript 吧。如果你不挑战自己，你就不会成长。

**你用过 React 吗？反应原生？少呢？萨斯？Node.js？大教堂。c？C++？Python？姜戈？露比？Ruby on Rails？你会自举吗兄弟？**不要难受。我也没有。

不要把你的旅程和任何人相比。每个学生加入 Lambda 时都有不同程度的专业知识，但只有那些即使在逆境中仍保持决心的人，才能留下一大堆 Web 开发人员。

学习编码的唯一方法是通过重复和应用。没有捷径可走。如果你想学习如何用几种计算机编程语言写作，你必须学习和练习。我建议你马上开始一个个人项目，在那里你可以应用你每周学到的东西。

与其他喜欢学习编码的人建立联系。Twitter 上有一个我一无所知的世界。在 Lambda 学校的第二天，我承认在#100DaysOfCode 挑战中失败了，我想我必须放弃才能真正专注于学校。我醒来时收到了社区中其他开发人员的几条消息，他们为我加油，鼓励我继续前进。

**听说过[《橡皮鸭》](https://rubberduckdebugging.com/)吗？**你在解释问题的过程中，有多少次去问一个问题只是为了自己回答？我的朋友，这就是橡皮鸭解决问题的方法。对鸭子有不合理的恐惧？一只来自 [tokidoki](https://www.tokidoki.it/collections/toys-collectibles) 的小猫也很好。
