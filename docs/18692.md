### [](#easyflipviewpager%E2%80%8A%E2%80%8Athe-flip-animations-for-your-viewpager)EasyFlipViewPager —查看页面的翻转动画

#### [](#a-behind-the-scenes-story-of-an-open-source-android-library-to-create-book-amp-card-flipping-animations-in-viewpager)一个开源 Android 库的幕后故事，在 ViewPager 中创建图书&卡片翻转动画

大约 2 年前，2017 年 1 月，我在 Github 上发布了我的第一个开源库 [**EasyFlipView**](https://github.com/wajahatkarim3/EasyFlipView) 。当时我正在为一个客户做一个英语词汇类型的 app。一项特别的任务是制作抽认卡，一面是英语单词，另一面是意思。我想把它做得像一张真正的可以翻转的卡片。一个简单的谷歌搜索给了我大量简单的代码片段和资源来创建翻转动画，我把它放在了应用程序中。客户很喜欢它，我继续开发应用程序。

项目结束后，我和另一个客户有了一次有趣的邂逅。他间接激励我做开源贡献。我想快速试验一下，在我的动力消失之前在 Github 上放点东西。然后这个翻转的动画打动了我。差不多准备好了。我所要做的就是上传到 jCenter 上，并在 Github 上创建一个好的自述文件。

这看起来很容易，但却是一项艰巨的工作。我花了几天时间完成并放到 Github 上。我不太了解开源或者如何营销图书馆等等。但我做到了。我相信这是初学者的运气，图书馆开始得到星星和叉子。很快我穿越了 500 颗星星。这是一次疯狂又神奇的经历。这让我沉迷于开源。我在[这篇文章](https://android.jlelse.eu/my-journey-to-500-stars-on-github-b881cf2bd18)上更详细地分享了我的经历。

[Github 上新手从 0 到 500 颗星的旅程](https://android.jlelse.eu/my-journey-to-500-stars-on-github-b881cf2bd18)

但是星叉并不是任何开源 Android 库的有效度量。这些指标告诉创建者其他开发人员对这个库的兴趣。但真正的衡量标准是有多少下载量和多少应用程序使用它。

但是为什么我要一直告诉你 EasyFlipView 呢？标题说的是别的。对吗？EasyFlipView 有一个问题已经公开了一段时间。问题是增加了通过触摸和手势手动翻转视图的功能。

<figure>[![](../Images/ea947b4a9c810407e0a46c986cfc1379.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--MdoSauZZ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2AGxiJxzyoPoFzvyGxwcogFw.png)

<figcaption>[手动翻转版面](https://github.com/wajahatkarim3/EasyFlipView/issues/15)来自 [EasyFlipView 库](https://github.com/wajahatkarim3/EasyFlipView/)</figcaption>

</figure>

我想修复它，但由于多种原因无法做到。所以，最近我开始在一个基于书籍的应用程序上工作，我喜欢好的用户界面/UX，我想在应用程序中添加书籍翻页动画。我使用 ViewPager 来处理书的页面，所以我有了这个想法来为书的翻页效果创建一个自定义的 PageTransformer。我在 [Uplabs](https://medium.com/u/559b0105cd29) 从作者的名字得到了灵感设计，我想创建一个库，它会给我这样的演示。

[明星刘在 UpLabs 的 Onboarding 书店演示](https://www.uplabs.com/posts/onboarding-animation-expore)

幸运的是，创建这个库也帮助我解决了 EasyFlipView 库上的一个[长期存在的问题。所以对我来说，这是一个双赢的局面。现在唯一剩下的事情就是创建图书馆。我对 PageTransformer 或 ViewPager 的动画了解不多。我开始在 GitHub、StackOverflow 和无数博客上寻找。但是我没有发现任何非常有用的东西。我有一些代码片段可以在 ViewPager 中进行页面转换。所以经过两天的实验，我成功地制作了一个简单但不太流畅的翻页动画。我编写了更多的代码，调整了一些值，增加了一些定制，瞧。成功了。总的来说，我花了大约一周的时间，每天工作 2-3 个小时。下一个任务是为开发人员创建一个易于使用和易于理解的界面库。](https://github.com/wajahatkarim3/EasyFlipView/issues/15)

> 根据我的经验，我知道创建库会让你成为更好的开发人员，因为你不是在为不关心代码的应用程序用户编写代码。相反，你是在为使用你的代码的开发人员写作。因此，你必须编写易于使用和易于其他开发人员理解的代码。

从定制的 PageTransformer 类代码到 jCenter 上可用的可重用库，我又花了大约 7-8 天的时间。你可能想知道为什么这么长时间？毕竟，它只是[上传到 Bintray](https://android.jlelse.eu/publishing-your-android-kotlin-or-java-library-to-jcenter-from-android-studio-1b24977fe450) 上。嗯，没那么容易，因为这一步不涉及代码。

> 文档对于任何开发者来说都是最枯燥的工作。有很多好的开源项目从来没有得到好的反馈或用户，因为他们的文档很差。一个好的自述文件是任何开源项目的灵魂。
