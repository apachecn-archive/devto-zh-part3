# 当我第一次点击 Javascript 时

> 原文:[https://dev . to/ma5ly/when-JavaScript-first-clicked-for-me-1e 24](https://dev.to/ma5ly/when-javascript-first-clicked-for-me-1e24)

我清楚地记得在我知道如何写 Javascript 之前的一段时间。我可以做 HTML 和 CSS，也可能偶尔插入 JQuery 代码片段，但是只要我看到变量或函数，我就会厌恶地把目光移开。这些莫名其妙的字母、符号和数字是什么？

几年来，我忽略了它。只要我的抱负没有超越制作一个漂亮的小网站或建立一个博客，我就没问题。有几次，这里或那里的教程引诱我进入 Ruby on Rails 或其他编程之类的东西。但是他们通常一停止详细描述我应该做的每一件事就把我吓跑了。

但我知道这种行为不能继续下去。如果我真的想在网上谋生，我应该学习 Javascript 语言。如果我真的想做一个应用程序，那是不可能的。于是我做了一次终极尝试。

[![Me laughing in pain](../Images/4f133a7c2f696e0167d0ba2a94ffe56b.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--17enYi0c--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/yfkn2gsxa2ljtbql3487.jpg) 
*当我第一次尝试阅读编程书籍时，我一直对我不理解的 95%的文本笑得很不自信。*

实际上，这意味着我尝试阅读 Javascript 书籍(没有点击)、阅读博客文章(小点击)、观看我的一位同事编写高级代码(有趣，但没有真正点击)和观看视频教程(大点击！).

对我来说，最终理解发生了什么的起点是理解函数。不知何故，函数、自变量、括号、参数和返回值把我搞糊涂了。怎么可能呢，对吧？并不是有多种方法来定义一个函数...

但是一旦我决定操纵我的思维，认为定义一个函数只有一种真正的方法，我就进行了第二步思考。我开始把一个函数想象成一个小的指令集，当你调用它的时候，浏览器就会执行它。

对我来说很有意义。在学习的早期阶段，这才是最重要的。下一个问题是我可以给这个函数什么样的指令。我会向谁下达这些指令。

查询选择器似乎是答案的一部分。结果是(我想我是在 Wes Bos 的一个视频中第一次看到这一点)，你可以通过它们的 ID 获取一个 HTML 元素。然后，您可以将该元素保存为变量，您可以随意命名它。

但是真正的突破发生在我了解`console.log`和浏览器控制台本身的时候。我真的不能夸大这个发现对我的重要性。

因为现在我可以试错了:*嗯，现在我的变量是空的，现在它有了某种值*。仅仅是因为能够到处玩并获得直接反馈，我就能够预测当我做某些事情时会发生什么。

当你抓住足够的东西来得到结果时，事情开始变得有趣。所以当我把变量填满后，我就准备好让我的函数来处理它们了。操纵他们的整个生命，如果你愿意的话。

就在那一刻，我读到的一些东西又回到了我的脑海中。我可以识别我的变量持有不同类型的数据，基于此，我可以做不同的事情。有数组、字符串、数字等等。

我所要做的就是足够频繁地使用谷歌搜索，我就能找到我可以使用的方法。不是说我现在觉得一切都很好，也不是说我觉得自己像一个真正的程序员，但这是我第一次真正相信我能学会所有这些编程的东西。或者不管我当时怎么想编程。

正是这些精神上的飞跃让我不得不放弃。如果没有任何技术背景，编程的世界可能会令人望而生畏，即使这个词有所有的知识和耻辱。但是，当一些事情开始发生变化，你意识到这很有挑战性，但并非不可能，你就可以真正开始学习了。

现在，当我和某人说话时，我从来不会告诉他们编程很难或者只是为了# thenerds。我总是说这只是你可以学习的另一种技能。比如会计，滑板或者画画。如果他们真的感兴趣，我会推荐他们去 YouTube。去看看人们在做什么，解释什么，看看这是不是你自己想做的事情。

回想起来，当我还处于完全不知所措的阶段时，是看别人写代码的视频真正帮助了我。看到某人正在做的事情的结果和现场评论的结合对我来说非常有效。书籍和博客只是在后来才帮助了我。

这就是我伟大旅程的开始。对你来说，事情第一次`.click()`是什么时候？
