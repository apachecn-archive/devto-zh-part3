# 学习新技能的可持续方法

> 原文:[https://dev.to/joshhadik/rapid-reps-一种易于管理的学习新技能的方式-1pf](https://dev.to/joshhadik/rapid-reps---a-manageable-way-to-learn-new-skills-1pf)

三个多月前，我决定在我的日常生活中增加一个新习惯，我称之为“快速重复”，让我告诉你，这是我做过的最好的决定之一！

当时，这只是一个理论。我有一个关于如何快速学习新技能的想法，大致基于[间隔重复](https://en.wikipedia.org/wiki/Spaced_repetition)的想法。我现在已经是快速重复的第十二周了，我可以毫无疑问地说，这很有效。老实说，我认为我在过去的三个月里学到的东西比我开始做这件事之前的三年还多。

# [](#what-are-rapid-reps)什么是快速重复？

这个想法很简单:

选择一项任务，每天重复一周。

这个任务应该足够小，你可以在合理的时间内完成。(我喜欢把我的 rapid reps 设定为 45 分钟，但是你可以选择最适合你的时间。)

在一周结束的时候，选择一个新的任务，重新开始！

您可能需要完成的一些任务示例:

1.  在您想要学习的任何新框架中构建一个单一资源 CRUD 应用程序
2.  用脸书、推特、谷歌等来练习设置 OAuth。
3.  构建一个框架应用程序，并将其部署到您想要学习如何使用的任何云服务提供商。
4.  使用您最感兴趣的 CI 工具为简单的应用程序设置自动化 CI/CD 流程。

# [](#why-are-rapid-reps-effective)快速代表为什么有效？

为了简单起见，我列出了我认为这个习惯如此有益的三个主要原因，但我知道如果我想的话，我还可以列出更多的原因。

你记住了工作流程。

到周末，您可能不记得您编写的每一行代码的确切语法，但我保证下次您必须实现类似的东西时，无论是几天后还是几个月，您都会对一般工作流和这样做所需的一系列步骤有更好的了解，这将为您节省大量时间和大量错误(以及修复这些错误的大量时间)。)

你会发现错误。

在一周的某个时候，你肯定会遇到一两个(或五十个)错误，这实际上是一件非常好的事情，因为它给了你一个没有压力的环境来学习这些错误意味着什么以及如何修复它们。你猜怎么着？当你在现实世界中遇到同样的错误时，面对截止日期和股东的所有压力，以及对打破一切的巨大恐惧，你会知道如何修复它们！

你建立了一个示例代码的档案。

说真的。每天当我开始编码时，我做的第一件事就是打开我的 Rapid Reps 文件夹，把它扔到一个外部监视器上。它给了我一个很好的合并库，里面有大量关于如何实现不断增长的工具和技术列表的例子。最好的部分是，如果你选择做同样的事情，这将是你写的所有代码，这意味着(希望)这是你能够理解的代码，比一些随机的答案更容易。

# [](#a-challenge)挑战

我挑战你完成一周的快速重复！

用这周剩下的时间选择一项任务。想一想你一直想学的东西，但是你从来都没有时间...也许是 Docker，也许是 CI，也许是 Vue，也许是任何东西！一旦你心中有了一个主题，选择一个你下周可以快速重复完成的小任务，尽量简单！

这样，下周一，你就可以开始你的第一周快速练习了！

如果你决定接受挑战，这里有[十条建议](https://dev.to/joshhadik/ten-tips-to-get-the-most-out-of-rapid-reps-480n)可以让你在第一周收获最大。
