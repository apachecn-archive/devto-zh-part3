# [](#talk-formats)👉🏻谈话格式

您知道演示有几种不同的风格(或“类型”)吗？以下是几种流行的格式。

*   ⚡ **闪电演讲**这是我个人最喜欢的演讲，是为在最短时间内产生最大影响而设计的简短(5 - 15 分钟)演讲。把你的陈述精简到一小段时间就抓住了你想要表达的本质。准备和练习这些谈话会更容易。**在我看来，这是最好的会议类型**。为什么？因为人们往往注意力持续时间短，所以我怀疑更短的时间会产生更大的影响。

液体错误:内部

*   🗣 **分组会议**这是课堂式演讲的常见形式，你的发言时间从半小时到 75 分钟不等。我个人认为 45 分钟是一个“最佳时间点”,我不喜欢更长的格式，但要为这个范围内的任何事情做好准备。

# 使用 Azure 函数处理 CSV 文件并将数据导入 Azure SQL

[![No longer maintained](../Images/4d3d66d77bcfa26a571bbef4912625cc.png)T2】](https://camo.githubusercontent.com/d145942b0088c2b2c79fa7f6cdfab315ccf72f3bab078edb1abef8c196754cac/68747470733a2f2f696d672e736869656c64732e696f2f6d61696e74656e616e63652f6e6f2f32303230)

提取、转换和加载(ETL)是一种常见的业务功能，通常对于不同系统之间的集成是必要的。传统的方法通常包括设置专用的 FTP 服务器，然后部署计划的作业来解析文件并翻译它们以供业务使用。无服务器架构使这项工作变得更加容易，因为文件上传时可以触发触发器。Azure Functions 是专注于特定问题的小段代码，非常适合处理 ETL 之类的任务。

本教程演示了如何使用 Azure SQL Database-as-a-Service 产品并结合 Azure 功能，将一个遗留流程迁移到云中 100%运行。您从本地开发环境开始，端到端地构建和测试流程，然后创建基于云的资产并部署到 Azure。

*   👩🏻‍🏫开放空间我喜欢提供开放空间的会议！这些是临时演示区域。通常你会在一个开放的空间注册一个演示，甚至是一个讨论的想法。你被分配了一个时间段，然后任何感兴趣的人都会过来合作。它通常比完整的演示更亲密，压力更小。如果你不确定自己是否准备好登台，可以考虑在会议上尝试一下。我经常发现话题包括支持有抱负的演讲者。
