# 井字游戏-另一个浏览器游戏

> 原文：<https://dev.to/chillsunfire/tic-tac-toe--another-browser-game-2i3i>

** *免责声明:我不是这里任何东西的附属机构，只是分享我的真实意见。* **

为了继续编写代码，我找到了另一个浏览器游戏来玩。这不像我的喂蛇游戏那样有趣，但是对于检查代码并试图找出一种不同的(也许更好的)方式来编写它来说，这是一个很好的练习。

和以前一样，我从 Udemy 的课程“JavaScript 和 jQuery 中的项目”开始。这个想法相当简单——点击一个方块，将其更改为 X 或 O(它们根据回合数交替出现),当一行中有 3 个相同的标记时，就宣布获胜。它还具有检测平局游戏的能力，如果你想在游戏正式结束前重新开始，它还提供重置功能。

#### 我从哪里来？

作为参考，下面是我跟随视频之后 JavaScript 的样子:[首先提交 bitbucket](https://bitbucket.org/sunfireweb/tictactoe/src/48918780cbc230f616ad7eb055c944e33d6a2b53/script.js?at=master)

有很多带有 AND 和 OR 的 If 语句，有几行是重复的。感觉笨拙而迅速。所以我问自己，我该如何改变这一切？

首先，由于最初的版本是在 2014 年写的，并且有比我认为必要的更多的文件夹，我清理了路径并更新了 jQuery CDN 链接。

然后我说让我们去掉匿名函数，设置一些实际的可重用函数。我还定义了一个成功的行是什么样子的，并列出匹配的变量。从那里我能够创建`check4win()`函数的外壳。

我还做了一些重新整理。最初的版本在单击处理程序中设置了几乎所有的内容。当我创建函数和定义变量时，我将它们移动到有意义的层次——一些在页面加载之前就在最顶端，而另一些在更深处。这花了一点时间，因为我必须确保函数中的所有内容都在正确的时间被访问。你看，JavaScript 的棘手之处在于 DOM(文档对象模型——也就是被加载和可访问的页面元素)。如果您试图访问尚未实际创建的 DOM 元素，或者在页面完全加载之前，您会得到一个错误。所以我不得不继续测试，直到一切都按正确的顺序进行。

最后，我准备做一些测试。玩了一次，然后开始一个新游戏，才意识到上一个游戏结束时获胜条件尚未清除，所以当我单击新游戏的第一个方块时，它会给我相同的获胜弹出窗口。必须解决这个问题…

#### 我要去哪里？

然后，在今天停下来之前，我想我会看看它在电话上是如何播放的。所以我迅速将文件添加到我的网络主机上，并从我的手机上调出[hitsaru.com/tictactoe](http://hitsaru.com/tictactoe)，才意识到白板太小了。简单的解决方法:只要确保在索引文件的`<head>`的`<meta>`标签中设置了`meta name="viewport" content="width=device-width, initial-scale=1.0"`。我不太了解如何改进移动网站，但据我所知，它运行得很好。

最重要的是，我将此作为练习使用 git 发布变更的机会。如果你感兴趣的话，我的 Bitbucket 上有完整的回购协议[。](https://bitbucket.org/sunfireweb/tictactoe/src/master/)

这就是我今天下午有趣的小项目。就像我说的，我知道井字游戏没有喂蛇有趣，但是如果你能看一下并告诉我你的想法，我会很感激。绝对欢迎提出改进建议。
