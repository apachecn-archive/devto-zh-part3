# Vim 并不适合所有人。以下是如何让它成为你的

> 原文：<https://dev.to/chhschou/vim-isnt-for-everyone-here-is-how-to-make-it-yours-301b>

我说过 *Vim 并不适合所有人*吗？我第一次尝试时确实有这种感觉。经过多年几次失败的尝试，我终于让 Vim 成为我的日常司机。下面我从自己的经历中总结了一些建议，供刚开始使用 Vim/Neovim 的人参考。

1.  从小处着手，太多太快是我过去学习 Vim 失败的一个常见主题。只需要 6 个键就可以向左(`h`)/向右(`l`)/向上(`k`)/向下(`l`)、删除(`x`)和插入文本(`i`)。我花了两个星期才习惯了前面提到的 6 个键，并戒掉了用鼠标的习惯。

2.  现在使用 Vim
    实际上，最初不需要在终端或 GVim(窗口中的 Vim)中设置 Vim。大多数编辑器/IDE 都有一个等待启用的 Vim 仿真。

3.  最初避免在压力下使用它
    专业工作即使不加入 Vim 也已经够难的了。当我在我的个人项目中使用 Vim 时，我获得了更大的成功，我只是用我已经知道的东西或者在一个会话中添加一两个命令来享受这种体验。我用这种方法在几周内缓慢但稳定地扩展了我的 Vim 肌肉记忆。

4.  触摸打字很重要
    触摸打字几乎是先决条件。我从来没有见过任何人在不能相当准确地触摸类型的情况下能够很好地使用 Vim。在使用 Vim 时，我的触摸打字出现了弱点，我在会话开始时增加了 5 分钟的有针对性的打字练习，直到我感到舒适为止。

5.  在定制之前先体验默认的键映射
    有些人主张尽早推广定制，理由是低效的键映射会产生糟糕的肌肉记忆，从而对生产力产生长期影响。然而，我发现默认的键映射对于大多数任务来说已经足够了，尤其是对于 Neovim。我也不喜欢定制，除非键图真的让我很烦，而且我觉得更舒服的键图还没有被有用的东西占用。

以上是我在过去多次学习 Vim 的失败尝试后，觉得对我有用的一些方法。我现在保存着一张学习表，上面有我想学习的命令。当我把命令序列存储在肌肉存储器中时，我会把它从我的学习单中删除，以保持它是最新的。

维明快乐！
