# 再次坠入爱河

> 原文：<https://dev.to/anotherdevblog/falling-in-love-again-2leo>

从前，回到学校，我发现了[云 9](https://aws.amazon.com/cloud9/?origin=c9io) 。一个在你的网络浏览器中给你一个 IDE 的网站？和根终端访问您自己的 Linux 虚拟机？我在天堂。我在工作中有一台结实的机器，但对于我的学校工作，我只有一台旧的翻新的 Chromebook。但我喜欢我的旧 Chromebook，不想花一大笔钱买一台合适的笔记本电脑，只是为了每天背着书包去学校，以便为我的 C 或 Java 课程编译代码。另外，我需要 Python 来写这篇文章和我在 Jekyll 上运行的另一个博客。和我兼职做的 Echo 和 Google Home 动作的节点。Cloud 9 完全符合这一需求。我可以为每个堆栈创建一个新项目，然后直接在我的 Chromebook 上编辑。

但实际上，在 Cloud 9 上编辑和在合适的 IDE 上编辑是不一样的。自动完成功能极其有限，尽管该应用程序的性质迫使我在命令行方面做得更好，但当凌晨 2 点，您仍在处理 C 任务时，您并不真正关心您是否真正擅长于通过命令行遍历目录，或者从“真正的”IDE 会立即发现的错别字中解析出模糊的 GCC 编译错误。

我试着用面包丁双启动我的 Chromebook，但开销太大了，我甚至在安装编辑器之前就开始耗尽驱动器空间(就像我说的，这是一个旧的 Chromebook)。我几乎放弃了，买了一台合适的笔记本电脑，但最终，我还是坚持了下来。一部分是为了省钱，一部分只是为了说，我设法用一台 Chromebook 完成了我的 4 年 CS 学位。:)

现在，我已经不上学了，还一直在用九朵云。为什么？嗯，部分是因为那台旧的 Chromebook 仍然是我在家里的首选笔记本电脑，部分是因为我只是觉得安装 RVM 和 Python 来写博客很奇怪。或者安装和配置 Node，这样我就可以随时更新我的 Google Home 操作。这样的例子不胜枚举。我不想在我的个人笔记本电脑上堆积 5 种语言的 30 个框架的 50 个版本。

但与此同时，我也渴望原生环境/IDE 能够提供的美好事物。即使是我的博客，撇开潜在的杰基尔胆量不谈，也不过是 Markdown、YAML 和 web frontend。拥有真正的智能感知就好了。和扩展。和断点调试。以及体面的编辑器带来的所有其他好处(比如我的宝贝儿子 VS Code)。即使只是降价预览也不错。有一点，我想知道:既然 VS Code 是一个运行在 Node 上的电子应用程序，那么在某个 Linux 容器上创建一个 VS Code 的 web 服务实例会有多难呢？

然后昨天我听到在 [CodingBlocks](https://www.codingblocks.net/) 的家伙提到这个叫做[编码器](https://coder.com/)的新东西。这正是我梦寐以求的:一个 web 可访问的容器，它直接在浏览器中运行 VS 代码的完整实例。我简直不敢相信自己的耳朵。我头晕目眩。那天我跑回家，打开我的笔记本电脑，创建了一个免费账户。是的，它是免费的。太牛逼了。这是我所希望的一切。

我创建了一个免费帐户，克隆了我的 GitHub 托管的博客，安装了 RVM、Python 和 Jekyll，阅读了要绑定和...就在那里！为了庆祝，我用 Coder 写了这篇文章，用的是已经 7 年的 Chromebook。

我恋爱了。一个合适的 IDE。在我的浏览器中。我所有的扩展，任务运行能力等等。在一个容器里，我可以随意吹走和重新水合。太美了。

我将继续与程序员一起玩，因为他们通过他们的公共阿尔法工作。接下来我要做的是。我已经注意到用户界面中的一些错误，这些错误肯定会暴露出来，但是到目前为止，没有什么会真正妨碍正常的工作流程。到目前为止，我完全被迷住了。
