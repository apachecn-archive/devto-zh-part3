## Linux

许多人对 Linux 又爱又恨。有数百种不同的发行版，每一种都有自己的关于 Linux 操作系统应该做什么和如何工作的想法。我不会在这里涉及任何具体的回购，但那将在下一篇文章中！

### 优点

1.  非常可定制
2.  强大的工具支持
3.  轻量级，可以在大多数硬件上运行

### 弊

1.  配置可能需要很长时间
2.  你可以永远花时间去解决你自己造成的小问题
3.  你是后援小队

## MacOS

MacOS 是苹果电脑上运行的操作系统。

### 优点

1.  专为运行它的硬件而设计。
2.  强大的工具支持/支持大多数 Linux 工具
3.  苹果的大力支持

### 弊

1.  昂贵的硬件！计划报废。
2.  一些为 Linux 构建的服务只运行了一半
3.  如果你不喜欢它的外观，你也无能为力

## 窗口

对于开发者软件来说，Windows 是一个有争议的选择，除非你在运行 C#等语言的 Windows 服务器上工作。最近，随着 Github 的收购和用于 Linux 的 Windows 子系统的引入，Windows 的未来看起来很光明。

### 优点

1.  运行在*吨*的硬件上
2.  微软最近加倍投资开发人员！
3.  现在可以用 Linux 的 Windows 子系统运行很多软件

### 弊

1.  购买许可证可能会很贵
2.  不良工具支持的遗留问题
3.  需要特殊软件来运行基本工具

# 我该如何选择

现在我们已经介绍了一些常见的操作系统，让我们讨论一下在选择使用哪个操作系统时需要考虑的一些事情。

## 扩展性

当考虑一个好的操作系统时，你需要考虑的一件事就是可扩展性。开发人员在其职业生涯中使用了如此多的工具，尽管跨平台工具变得越来越普遍，但是您希望确保您可以按照您希望的方式设置您的开发环境。一些好的基准是使用软件包管理器的能力(现在大多数操作系统上都有)，访问命令行，以及一些基本的、可编辑的系统级首选项。

## 易用性

除了可扩展性，一个好的操作系统应该兼顾易用性。很难在两者之间找到一个很好的中间点，但是除非你真的喜欢摆弄工具设置和安装，否则你可能想要一个能让这些事情变得非常简单的操作系统。我个人喜欢在我的 Archlinux 机器上工作——一切都和我配置的一样，没有任何东西是我没有添加的。然而，当出现问题时，我可能需要很长时间来修复它。如果事情发生了，你要确保你的工具在那里支持你。

## 硬件

最后，在考虑选择什么操作系统时，硬件是一个大问题。大多数 Linux 发行版可能可以安装在大多数机器上，但如果你想在非苹果电脑上运行 MacOS，你会有一段不好的时间。Windows 可能会在大多数地方运行，但许可证可能会很贵！考虑你的开发机器的外形和用例。我的一个朋友每天都带着一台大型游戏 Windows 笔记本电脑去工作——他运行多个虚拟机和驱动器去工作，所以这对他来说很有意义。然而，我很多天都是步行去上班，并且经常在咖啡店工作，所以我需要一些便于携带和在很多地方使用的东西。虽然这些考虑事项不是操作系统的一部分，但您使用的操作系统很可能会以某种方式由运行它的硬件决定。

# 无论你选择什么，让它成为你的

最终，您对开发环境的选择是高度个人化的，并归结于许多不同的因素。找到一些让你日复一日快乐工作的事情，你将会做出一个好的选择。不要听信那些拥护“一个真正的操作系统”的人。花些时间尝试不同的操作系统。
