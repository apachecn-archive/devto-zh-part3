# NextJS 及其天才创意

> 原文：<https://dev.to/revskill10/nextjs-and-its-genius-ideas-5bda>

人人都爱反应堆。

JS 框架每周都会推出新的框架。但是当我们谈到服务器端渲染框架时，选择仍然有限。

我玩过 VueJS、FusionJS 和 NextJS 以及其他一些“较小的”SSR 框架，我选择了 NextJS，原因如下。

1.  基于文件的路由。

啊哈，我不需要编写任何代码来使我的 React 应用程序可路由！不仅是客户端路由，服务器端路由也是如此。我称之为关于如何完成路由的古老而天才的想法。

这里的好处是，你将免费获得基于路由的代码分割！

1.  getInitialProps

在旧版本的 ReactJS 中，我们有`getInitialState`，我相信 React 开发者会非常想念它。而 NextJS“窃取”了这个想法，让我们的 React 组件在服务器端呈现它的初始数据！

如果你还没有使用 NextJS，今天就试试吧！
