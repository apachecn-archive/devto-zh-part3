# 理想的颜色工具

> 原文:[https://dev.to/metamn/an-ideal-color-tool-57ik](https://dev.to/metamn/an-ideal-color-tool-57ik)

为网站和应用程序生成调色板的工具总是受欢迎的。然而，一个完全适合网络的产品却不见了。

## [](#the-basic-need)基本需要

我们至少需要两种颜色来创建一个用户界面。文本颜色和背景颜色。

这些可以很容易地选择；我们只需要确保它们的色彩对比度足够好，让所有类型的观众都清晰可见，甚至是那些有视觉障碍的观众。

两种颜色对于博客来说已经足够好了。

当出现更多内容类型时—行动号召按钮；多种视觉部分，如英雄、特写、评价——两种颜色可能还不够。

## [](#a-monochromatic-palette)单色调色板

有了这两种颜色，我们还能得到另外几种很好的颜色组合。首先是一个倒置的配色方案，然后是中间的许多其他方案。

这些组合对于一个基本的登录页面或一个简单的营销网站来说已经足够好了。

英雄是用反色设置的，英雄上的按钮/行动号召用默认颜色。后来，不太重要的部分可以在颜色之间，例如在黑/白原色的情况下是灰色。

## [](#1-color)+1 颜色

用一个生动的行动呼吁按钮来打破单色配色方案，真是令人耳目一新。这就是为什么用户界面倾向于建立在两种以上的颜色上。

当一种新的颜色被添加到两种基本颜色中时，要么需要支配它们，要么被它们支配。

行动号召应该主导现有的配色方案，使其内容更接近观众。

不太重要的内容部分应该用主色隐藏在背景中。

理论上说暖色占主导，冷色下沉。在经典色轮上，暖色在左边:红色、橙色、黄色——冷色在右边:紫色、蓝色、绿色。

## [](#color-properties)颜色属性

要为现有的单色配色方案挑选完美的红色，我们可能需要数学。有很多红色——根据什么标准，你可以肯定*红色是这个方案的完美红色？*

为了使颜色可以量化，使用了颜色模型和颜色空间，如 RGB 或 CMYK。

每个型号都描述了具有特定属性的每种颜色，例如:

*   发光度——一个绝对属性
*   温度——一个绝对属性
*   饱和度-描述颜色中有多少个`gray`的相对属性
*   亮度——描述颜色中有多少个`white`和`black`的相对属性
*   对比度-第三种颜色的相对属性

有了这些属性，*完美的红色*可以被重新配制成*一种与当前配色方案具有相同饱和度和亮度的红色，并且与背景色的对比度为 4.5*。

色彩模型可以专门针对印刷(Pantone、CMYK)或显示器屏幕(RGB、HSL)等媒介进行定制。他们可能会带来惊喜。在 CMYK 中`blue`与`yellow`混合给出`green`，在 RGB 中给出`gray`。

为屏幕用户界面选择合适的颜色模型很重要。

## [](#an-ideal-tool)理想的工具

### 1—可计算的颜色模型

创建完美配色方案的理想工具首先要有一个可计算、可编程的颜色模型。

人们应该能够使用代码生成一致的调色板，并以思考其他编程工件(如数据和算法)的同样方式思考颜色。

### 2—预载内容类型

一个理想的工具应该预装不同的内容类型。对于一个简单的博客，一个简单的登录页面，一个完整的产品营销网站，等等。

有了它们，可以进行现场测试，看看配色方案是如何工作的。

图形艺术家有他们自己的工具，比如 Adobe Color Wheel，这可能很适合设计海报和传单，但肯定不足以直观地设计基于文本的用户界面。

Paletton 认识到了这个缺陷，并增加了一个功能，可以在真实的网页场景中预览调色板。然而，太可怕了。

[谷歌颜色工具](https://material.io/tools/color/)要好得多。这是一个好的开始，但没有给出足够的细节。

### 3—单色调色板生成器

一旦我们选择了两种主要颜色，我们应该能够看到这些颜色有多少额外的颜色对。

谷歌的颜色工具给了我们两个提示(亮、暗)，但这里有更多的玩法。

### 4—特殊色轮

最后，我们需要一个色轮来帮助我们选择现有单色方案之外的暖色和冷色。

Paletton 对生成的调色板有这样一种微调机制，但不会将色轮设置为只能拾取相应新颜色的状态。
