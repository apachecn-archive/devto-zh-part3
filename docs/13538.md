[![weswedding profile image](../Images/4c64d726cb8b72cad8ba0749559591a7.png) ](/weswedding) [ Weston Wedding ](/weswedding) • [<time datetime="2019-02-06T19:34:25Z" class="date-short-year"> Feb 6 '19 </time> • Edited on <time datetime="2019-02-06T19:35:28Z" class="hidden m:inline-block date-no-year">Feb 6</time>](https://dev.to/weswedding/comment/8id9) 

通常“无头”意味着一个软件没有图形用户界面。

要理解什么是“无头”CMS，你必须理解，从历史上来说，CMS 不仅仅是“管理内容”

CMS 通常有一个“管理端”或“后端”,用于向 CMS 输入数据和文件。有些甚至让你定义特定种类的内容；比如“博客帖子。”博客文章有文本和图像，可能还有其他东西，比如标签或类别。

然后，CMS 有一个面向公众的“正面”,大多数网站访问者都会看到。这是由 CMS 提供的某种“主题”层或“模板”层生成的。它获取文本、图像和类别，并生成 HTML 以使博客文章显示在浏览器中。

Headless 是指从 CMS 中剥离“正面”层。CMS 不再生成一个完整的网站，它已经被简化为仅仅保存内容。这取决于另一个软件对 CMS 中的数据做些什么。

[![somedood profile image](../Images/44bc5cdbe5a3d3e67f77d2bb2993d61f.png) ](/somedood) [ Basti Ortiz ](/somedood) • [<time datetime="2019-02-09T13:26:43Z" class="date-short-year"> Feb 9 '19 </time> • Edited on <time datetime="2019-02-10T02:13:26Z" class="hidden m:inline-block date-no-year">Feb 10</time>](https://dev.to/somedood/comment/8jk5) 

你的父亲(家长班)给你，学生/儿子(子类 A)，一本写得很模糊的关于加减法的教材。因为你的父亲不能给你一堂关于这个话题的课，你被困在一个你不知道如何解决的问题上。你看到你的兄弟(子类 B)也很困惑。

这时你决定向你父亲寻求指示。不幸的是，出于某种原因，你父亲也不知道如何解决这个问题。

这就是为什么你的父亲去问你的祖父(家长类的家长)。最后，你的祖父理解了这个问题，并告诉你的父亲如何解决它，他后来告诉你如何自己解决它。

你弟弟也去找你爸爸问他问题，反过来导致你爸爸又去问你爷爷问题。解释之后，你父亲终于走近你哥哥，回答了他的问题。

*简而言之，这就是**原型继承**，如果当前对象还没有某个属性或方法的实现，那么该属性或方法的实现就被委托给一个“父/祖先类”。*
