# Git 和其他通信工具

> 原文:[https://dev . to/jessekphillips/git-and-other-communication-tools-60l](https://dev.to/jessekphillips/git-and-other-communication-tools-60l)

我在 Git 上的上一篇文章试图关注你的历史的最终目标。但是我已经可以听到唠叨的回答“我们有其他的工具来交流这些事情。”

最有可能的是，你正在利用某种形式的缺陷跟踪系统，这些系统帮助你组织正在改变的东西，并反馈已经完成的东西。这些工具非常重要，因为它们提供外部输入、未来所需工作的记录，并促进规划工作。它们还可以简化漂亮的图形和开发周期。

在我看来，这些工具使得清楚地传达提交历史中的变更变得更加重要。很有可能你的 git 提交可以和你的 bug 跟踪系统集成在一起，不仅可以对问题/bug 留下评论，还可以解决/关闭它们。

如果你读过我的上一篇文章，我提到在开发一个特性的过程中，可能会发现并修复一些额外的 bug。当您将这个修复分解成它自己的提交时，您会开始想，“在这个提交中我引用了什么问题号？”这个问题的答案可能是，“我需要一个新的问题编号来分配这个提交。”否则，如果您正在进行代码审查，审查者将会查看您的提交，并询问“这项工作完成的记录在哪里，我没有看到分配的问题编号？”

通过交流，我们对交流的内容和地点有了更好的制衡和期望。
