# 为什么对 Web 应用程序进行跨设备测试很重要

> 原文:[https://dev . to/lambdatest/why-it-important-to-perform-cross-device-testing-of-your-web-application-od2](https://dev.to/lambdatest/why-it-is-important-to-perform-cross-device-testing-of-your-web-application-od2)

世界上大多数人使用 800 X 600 显示器访问网站的日子已经一去不复返了。目前只有台式机有 5-6 分辨率，算上 2k 也有 4k 的。除此之外，还有平板电脑，最重要的是手机。智能手机是目前访问网站的主要媒介。非常重要的一点是，你的网站应该在各种设备分辨率下都能响应并正常运行。在这篇文章中，我们将讨论为什么跨多种设备测试你的网站是重要的。

## 移动优先方法

如今，所有组织都选择了[移动优先方法](https://webmasters.googleblog.com/2018/03/rolling-out-mobile-first-indexing.html)。他们正在开发或重新设计内部网站以及客户网站，以便新网站能够在所有移动设备上正常运行。为了跟上你的竞争对手，理想的是你必须测试你的网站，以确保他们在移动设备上正常工作。

## 谷歌推荐

自 2015 年以来，谷歌一直根据网站在多种设备上的友好程度，在其搜索结果中对网站进行排名。从那时起，组织开始优化他们的网站，以实现[跨设备兼容性](https://www.lambdatest.com/blog/what-is-cross-browser-compatibility-and-why-we-need-it/)。目前，谷歌已经领先一步，推出了移动优先索引。移动设备上的兼容性已经成为网站在搜索结果页面上排名的主要标准。理想的情况是，你设计的网站应该兼容所有设备，尤其是手持设备和智能手机。

## 多任务处理是新趋势

消费者现在使用多种设备来执行他们的任务。让我们看看网飞和亚马逊。它们广泛应用于桌面和移动设备，在每种设备中，它们的布局都是不同的。如果您的应用程序对于不同的设备也有不同的用户界面，那么您应该在将应用程序发布到生产环境之前对它们进行适当的测试。

## 日益增多的设备

虽然由于苹果的设备控制政策，iOS 设备的数量有限，但开源的 Android 没有这种限制。有数百种 Android 设备，每一种都有自己的分辨率，几乎每天都有新设备推出。测试您的应用程序是否能在所有目标设备上正常工作，而不会导致任何布局问题，这一点非常重要。

## 设备使用因区域而异

让我们假设你正在创建一个应用程序，由于缺乏时间；你只选择在 iPhones 上测试。如果您所在区域的大多数用户都是 iPhone 用户，这没什么问题。但是，如果你打算走向国际，这将是一个问题。由于有些地方大多数用户使用 Android 或 Windows phone，理想的做法应该是研究各种用户区域，你计划将这些区域作为目标，整理出一个设备列表，并在所有这些设备上运行测试。

## 云平台让测试更容易

通常，测试人员会因为耗时和缺少设备而跳过跨设备测试。目前跨设备测试并没有那么难。像 [LambdaTest](https://www.lambdatest.com/) 、 [Browserstack](https://www.browserstack.com/) 、 [Crossbrowsertesting](https://crossbrowsertesting.com/) 这样的云平台使得跨多个实时设备和浏览器同时测试你的网站变得更加容易。你所需要做的就是打开你的浏览器，选择设备，点击你的网址，然后开始测试。

## 用户很容易不满意

普通用户在一个网站上花费的时间不到一分钟。在这段时间里，他们决定是继续使用这个网站，还是换一个有相同目的的网站。猜测用户使用什么设备访问你的网站是不可能的。因此，执行跨设备兼容性测试以确保您为改善用户体验而加入的所有最新功能在所有设备上都可以正常工作是非常重要的。

## 无数的浏览器

早先，Internet Explorer 是唯一一个可以在桌面上访问网站的浏览器。目前，有数百种用于桌面和移动设备的浏览器，每一种都有自己呈现 CSS 和 HTML 代码的方式。一个在桌面版谷歌浏览器上运行良好的网站可能会导致移动版火狐浏览器崩溃。这就是为什么[跨浏览器测试](http://www.lambdatest.com)和跨设备测试对于确保用户的不间断浏览会话非常重要。

## 更新和新发布的技术

一个网站的平均寿命是 3 到 5 年。新的 web 开发技术定期发布，确保开发更快、更兼容的网站。如果你想跟上你的竞争对手，你必须重新设计你的网站，以便在市场上保持领先地位。然而，新法规可能会导致某些设备损坏。因此，每次重新设计某个模块或整个网站时，执行跨设备兼容性测试是很重要的。

如果你是一个个人开发者或者是一个专门提供 web 解决方案的组织的所有者，上述事实会让你意识到跨设备兼容性测试的重要性。所以，继续进行测试，为客户提供一个他们可以从任何地方访问的完美网站。
