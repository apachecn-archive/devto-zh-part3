# 使用 Uploadcare 优化自动驾驶仪上的图像(以提高页面加载速度)

> 原文：<https://dev.to/rsedykh/key-image-optimizations-to-maximize-your-google-pagespeed-score-a-guide-by-pagedetox-2hfp>

**总结:使用 CDN 的 URL 指令优化图像的分步指南，以消除 Google PageSpeed 图像相关问题。**

当你用[谷歌页面速度洞察](https://developers.google.com/speed/pagespeed/insights/)分析一个网站时，它通常
会指出几个需要修复的图像问题:

1.  下一代格式
2.  图像尺寸
3.  响应性
4.  惰性装载

我们分析了 1 万个用户网站，发现:

*   只有极小比例的用户使用下一代图像格式，仅这一项就能导致文件大小下降 25-34%(来源:[谷歌](https://developers.google.com/speed/webp/docs/webp_study))。
*   几乎每个压缩图像都可以进一步缩小，而不会明显降低视觉质量。
*   问题往往是关于一些巨大的图像；修复这些问题可以显著缩短加载时间。
*   通常没有延迟加载(当图像只在滚动到某个位置时才加载)。
*   许多被分析的网站要么不提供响应图像体验，要么提供较差的响应图像体验。

消除这些问题将提高页面加载速度，降低跳出率，并改善用户体验。

如果你不知道具体要做什么，解决这些问题是令人沮丧的。以下是优化图片的终极指南**三步**:

1.  *优化尺寸&格式* —在不显著降低视觉质量的情况下消除多余的图像重量。
2.  *使用响应图像* —为每个设备和屏幕提供合适的图像。
3.  *添加延迟加载* —仅加载用户正在查看的图像。

## 简版指南

*   用`srcset`代替`src`。
*   使用 CDN 的 URL 指令创建图像版本。
