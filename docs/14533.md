# 您需要二进制存储库吗？

> 原文:[https://dev.to/jfrog/do-you-need-a-binary-repository-2al](https://dev.to/jfrog/do-you-need-a-binary-repository-2al)

[![](../Images/f91c65acd3aec1594f26619e2ba26081.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--9BbKruHe--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://greenido.files.wordpress.com/2019/01/screen-shot-2019-01-30-at-1.47.34-pm.png%3Fw%3D696)

任何开发人员[都知道](https://12factor.net/codebase)你必须有一个源代码库(例如 Git ),但是我时常会被问到“为什么我需要一个二进制库”？

下面是简短的回答:

**更快、更安全的软件开发**–如今，任何公司都是软件公司，最好的公司每天/每小时都会发布更新。快速推送更新的能力是一个真正的竞争优势。当你的团队中几乎没有工程师的时候，你希望避免每次*“NPM 安装”*“获取整个互联网”。

二进制回购会给你缓存这些库的选项，并确保你使用的是正确的库(而不是被[黑掉的](https://www.theregister.co.uk/2018/07/12/npm_eslint/)库)。从开发人员的角度来看，这对他们的生产力是一个很大的提升，因为它节省了开发期间和每次构建的时间。更好的是，从 DevOps 的角度来看，控制所有包/库(并扫描它们以寻找**漏洞**)的能力是一个巨大的优势。它使内部工程团队能够更好地控制发布，因为他们对“组成”发布版本的所有内容都完全透明(例如，质量、性能、安全性、许可证等)。查看 [12 因素应用宣言](https://12factor.net/dependencies)了解更多依赖关系(声明和隔离)。

* * *

**提高生产力—**通过从一个中央(受管理的)位置共享库和组件，您可以为开发周期带来秩序。如今，有如此多的第三方库供您在任何项目中使用，您必须有一种有效的方法来管理它们。二进制存储库是一个合适的工具。它将防止不同的团队使用不同的库(例如，5 个不同的库来解析 JSON 等)来完成相同的任务，并将为您提供一种更好的方法来管理您的策略、许可和部署。由于在每个包/库和工件上包含丰富的元数据，您可以使用这样的 repo 更有效地管理您的版本。

总之，使用二进制回购将提高你的产品:质量、安全性和速度。

*   **规模**–在过去的 10 年里，Artifactory 为世界上一些要求最苛刻的(也是最大的)公司提供服务。如果你需要像思科、网飞、谷歌和其他公司那样的规模，它会满足你的需求。它将多站点复制和高可用性作为产品不可或缺的一部分。

*   **[DevOps 工具集](https://jfrog.com/devops-tools/)** **集成**–你有成熟的 CI/CD 管道吗？很好。由于 Artifactory 是通用的，它将在几分钟(而不是几天)内与您的管道和技术堆栈连接。这也是实现自动化并开始改善您当前开发运维状态的一个很好的方式。

*   **完整的工件** **生命周期**–我们知道你的大部分构建/工件在很长一段时间内都不会有用，有很多选项可以“清理”和控制你的开发团队的产出。如果没有一个中心位置来管理它，将会耗费您的时间和金钱(例如，在查找每个项目的谁在使用哪个库时，会出现存储浪费和混乱)。

*   **混合解决方案**–以防您需要一个针对内部站点和云的解决方案。它已经融入到产品中了。

*   **高级存储管理**t——由于 Artifactory 使用校验和库，它知道如何避免重复。每个库/工件的每个版本将只保留一次。这不仅节省了存储空间，还节省了时间(和金钱)，因为这是一种更高效的工作方式。
