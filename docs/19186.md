令人兴奋的消息(无论如何对我来说)，这些播客本周被自动标记。这意味着我的工作量减少了，尽管结果远非完美。

我并不为我为实现这一目标而编写的代码感到自豪，这是非常典型的[“原型”](https://blog.codinghorror.com/new-programming-jargon/)风格。

为了获得标签列表，我...

1.  从 StackOverflow 中的[标签开始，5000+](https://data.stackexchange.com)
2.  加入了同义词(可能不正确),试图去掉太具体的标签，2700+
3.  通过 [qit](https://qit.cloud/) 运行所有 2700+项，丢弃少于 3 个结果- 800+的项
4.  手动通读列表，删除不明确或不感兴趣的术语(如“产品”或“文本”)，并将它们添加到黑名单文件- 259 个标签
5.  添加了一些遗漏的术语- 262 个标签！

这仍然比我想要的要多，但是还有很多改进的空间，一旦我把它重构到一个更好的地方，添加同义词就很容易了。

代码都在 [github](https://github.com/codingblocks/tech-podcasts) 上，尽管目前还没有得到很好的分解。(记住，“原型生产”)

下周我希望专注于清理，迭代标签，并最终回到 [qit](https://github.com/codingblocks/podcast-app) 。
