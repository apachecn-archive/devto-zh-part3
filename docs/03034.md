# 我希望在开始 Web 开发之前就知道的事情

> 原文：<https://dev.to/flippedcoding/things-i-wish-i-knew-before-i-started-web-development-1eb8>

在我做 web 开发之前，我是一名工程师。我以前做过软件，主要是机器人和机器学习，所以我认为 web 开发会是一个有趣的转变。我认为这是编程，我喜欢这样做，为什么不呢？

这是我做过的最好的决定之一，但有几件事我希望在我做出改变之前就知道。

## 这与数学无关

从我的编程背景来看，我认为会有很多数学知识。我认为 web 开发使用了所有这些疯狂的算法，我必须能够将这些算法翻译成代码。如果我试着去做，我也不会错得更离谱。

Web 开发与其说是一门科学，不如说是一门艺术，我花了一段时间才明白这一点。处理一项任务总有不止一种方法，多种方法是最佳选择。要成为一名优秀的开发人员，解决问题的技能比数学技能更重要。找到问题的根源或清楚地定义一个新特性是一种比试图做复杂的数学运算更重要的技能。

## 你必须经常与人交谈

老实说，我认为既然 web 开发人员主要写代码，我就不必和任何人交谈。看起来我只是需要有人告诉我他们想要做什么，然后我一个人去工作一周。我真的很期待我的第一份网络开发工作！

然后我了解了敏捷开发和客户需求。当然有些日子我不和很多人说话，但是没有一天我不和任何人说话。问题出现了，项目范围更新了，客户改变了需求。良好的沟通技能和良好的技术技能一样，都是优秀开发人员的定义。其实沟通能力更重要。

如果你想找一份与他人互动有限的工作，那么网站开发并不适合。这可能是对我最大的震惊，因为所有关于开发者的刻板印象。不管媒体如何描述我们，开发人员都是最友好、乐于助人和健谈的人。

## 你真的是边走边学

任何工作都有一定程度的这种情况，但是对于 web 开发来说有点不同，因为变化可能是剧烈的。编程语言和框架一直在更新。甚至 ide 也会更新，这可能会改变您的正常工作流程。你的公司可以决定从 Angular 转向 React，或者从 MVC 转向。网芯。

或者你可以找一份新工作，他们使用完全不同的技术。最重要的是你理解了底层的概念和架构，接下来的问题就是弄清楚如何用一种特定的语言来做到这一点。作为一名网络开发人员，你必须学会如何在不错过最后期限的情况下学习。

这可能是一些简单的事情，比如当你有机会的时候阅读文章或者从事个人项目来巩固那些概念和语言特定的语法。做好大量谷歌搜索的准备，不要感到尴尬。高级开发人员仍然需要一直查找资料。

## 你通常不会从头开始一个项目

基于教程和课程通常教授 web 开发的方式，我认为我将从事的每个项目都将是全新的。除了为人们制作的一些网站，我还没有做过新的网站。大多数公司已经有现成的软件，他们只是想让你更新和维护它。

这就是为什么理解 web 开发的核心概念如此重要。你将会阅读其他人的代码，试图找出他们做了什么以及为什么要这样做。知道如何找到遗留代码也是一项重要的技能。

我在 web 开发生涯的最初几个月里很迷茫。当你不理解其中的一些核心概念时，浏览所有的代码是令人生畏的。如果你真的很期待展示你新的 web 应用构建技能，这也可能会有点令人失望。

## 没有任何真正的规定

这个让我措手不及。在工程领域，你必须满足某些规定，否则你的设计会被退回或完全废弃。这项工作受到严密监控，你需要的签名数量令人难以置信。在 web 开发中，有一些最佳实践。这是一组使 web 开发更容易的规则，但是它们可以有不同的解释。

例如，不是每个公司都以同样的方式使用敏捷。不是每个公司都花时间写测试，或者他们跳过他们觉得不必要的部分。当然，有一些像 SOX 和 HIPPA 这样的规则，但不是针对软件架构和代码实现方式的。

您可能需要批准一个拉取请求，它可能会因为一些更改而被退回，但是这个过程远没有我想象的那样规范或强制。Web 开发真的就像一门艺术，因为每个人做事情的方式都有所不同。这取决于你使用的语言，你工作的环境，和你一起工作的人。

这些只是我希望一开始就知道的一些事情。现在有更多的资源，但是没有一个真正花时间来解释一个专业开发人员每天到底在做什么。希望这给任何初学者更多一点的洞察力。

当你成为一名网站开发者时，有没有什么让你感到惊讶的事情？当你得到第一份工作时，你有没有完全失去期望？尤其是当你从一个完全不相关的领域转行的时候。
