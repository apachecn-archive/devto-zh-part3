# 当陷入长期项目时，如何跟随软件行业的变化

> 原文：<https://dev.to/proticm/how-to-follow-the-software-industry-changes-when-stuck-on-a-long-term-project-ohm>

# 简介

不管你是雇员还是在家工作的自由职业者，在你的职业生涯中都会有一个时刻，你会以某种方式结束某种长期项目。

在开始的时候，这个项目会看起来很酷，很吸引人，就像其他任何新事物一样，尤其是如果你以前没有使用这项技术的经验。你将会非常兴奋地工作，学习新的东西。

然后你花了六个月的时间...然后一年，又一年...这感觉就像你坐在长椅上，看着很多很酷的东西从你身边经过...类似于，哦，棱角分明...走了，Vue...走了，反应过来...走了，。网络核心...不见了。

# 软件行业的步伐

我很久以前就爱上了这个软件，这要感谢我的大学教授，我知道这个行业可能会迅速变化，因此，如果你不跟随它，时间会消耗你，你的知识会过时。我们职业的学习部分是永无止境的过程。我永远不会忘记他说过的那句话:*“如果你不打算学到职业生涯的最后一刻，现在就离开这个行业”*。

请想一想，在过去的几年中，该行业发生了多少变化和增加，尤其是在前端领域？创建了多少新的 JavaScript 库？与你现在需要的知识相比，几年前你需要知道什么来设置你的网络应用？

这里的关键部分是，我们需要热爱我们所做的事情，我们需要做的与此相关的一切都变得超级容易。

# 主观地学习和通过做讲座

我们都知道我们的工作会因为所有的截止日期和客户要求而变得很苛刻，但有时我们可以在这里或那里抓住一点空闲时间。如果这是少量的时间，你可以用它来学习一些你感兴趣的东西或者是主流的东西。在此期间，您可以安装您最喜欢的 IDE，并使用流行的框架之一安装 hello world 应用程序。尽可能多地尝试各种配置，尝试一下，看看输出是什么。记住你这样做是为了学习，因此没有什么是被禁止的。把代码放到 GitHub 上。这样你就可以从另一台机器上访问这个项目，另外，你还可以学习 git。

另一个很好的方法是做讲座。有时，你可以和你的同事一起建立一个小教室，在那里你可以选择一个主题，通过收集所需的知识来准备自己，并向小组展示。

在演示过程中，任何人都可以问你问题。通过回答与否，你将看到你准备得有多充分，以及你在学习过程中错过了什么。

您可以创建一个时间表，其中每个讲座将由小组中的不同人员进行演示。这是一起学习和增加整个团队平均知识的好方法。

# 在家学习-开源项目

这种方式是最常见的，也是学习新事物的唯一方式。这种学习方式的主要好处是你不依赖于时间。你可以阅读文章，慢慢学习，并根据你的需要多次预装你的测试溶液。没有人催促你，打破你的循环。

在对你所掌握的新技术足够熟悉之后，你应该尝试用它来创造一些有用的东西。将解决方案设置为开源项目。这是从那些对你所使用的技术了如指掌的人那里获得一些反馈的好方法，除了对你的项目进行总体评论，他们可能还会抛出一些技术评论。

遵循这些原则，我创建了这个库。

# 知识库

不要偷懒，创建自己的知识库。保存工作代码的例子，写下解释，标记有用的链接...所有这些在将来都会派上用场，这一点可以肯定。你将把自己从在 Google 和 StackOverflow 上一遍又一遍地搜索相同内容的挣扎中解救出来，而且你将拥有用你自己的话写的解释和评论，这将帮助你理解代码。

你和你的同事建立了一个 GitHub repo，每次发现新的有用的东西时，你们每个人都可以更新它。

# 博客

读，读，读。选择一个博客，或者你最喜欢的作者，关注他们写的东西。

但是，始终要努力理解你正在阅读的内容。不要只是不聚焦的浏览一篇文章。既然你已经打开了，你可能对这个话题感兴趣，所以你不妨试着理解地阅读它。

通过理解你刚刚读到的内容，你可以参与讨论，发表评论，指出可以做得更好或描述得更好的部分...浏览一篇文章，在阅读时看着可爱的女孩/男孩在外面走过，这只是浪费时间，对你没有任何好处...当然是在学习和提升知识方面:)。

就是这样。在遵守行业标准时，我会努力遵循一些指导原则。
