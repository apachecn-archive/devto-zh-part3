# d 符合标准稳定性

> 原文：<https://dev.to/jessekphillips/d-does-meet-standard-stability-lja>

我从 2006 年开始用 D 编码。我见过很多它的成长和痛苦。挑战之一是开箱即用的体验。

回到过去，你会得到一个 zip 文件并解压它。这太简单了。编译器会编译你给它的文件，这是令人困惑的，为什么我需要解释编译器编译了什么。

通常，Visual Studio 是 Windows 开发人员所期望的开箱即用稳定性的范例。通常我同意这一点，事情通常是可行的。

但是最近我被寄予了更多的合作期望。我发现人们无法适应更新。任何问题都是更新的错误，当有更新问题时，通常是因为许多需要的部分中只有一个被更新。库依赖停滞不前。

当着眼于 web 开发前沿时。构建是语言、工具、冲突框架的集合，编译时间比编译语言要长。

如果使用 D，我不会建议避免这一点，但我可以自信地说，开发世界的其余部分也没有解决问题。
