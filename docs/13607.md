# 编程的许多世界

> 原文:[https://dev . to/Josh hadik/mapping-out-the-many-the-world-of-programming-1 NBA](https://dev.to/joshhadik/mapping-out-the-many-worlds-of-programming-1nba)

我记得我刚开始学长生不老药的时候。那是一年半以前的事了。

那时，我正快乐地生活在用 Ruby 进行面向对象编程的世界里，在这个世界里，一切对我来说都有意义。我知道如何使用对象、方法、类和继承，一切都感觉很好！

然后就发生了。在一个决定命运的夜晚，在我越来越渴望尝试长生不老药之后，我决定冒险进入未知的领域，进入函数式编程的世界。一开始很害怕，也很陌生，但也很兴奋。一个建立在功能、流程、管理者和并发性之上的世界。一切都是新的，一切都令人兴奋。

在我的冒险之前，我处于一种幸福的无知状态，许多初学者都曾发现自己处于这种状态。我以为我都知道了。我已经掌握了面向对象的世界，在那短暂的一瞬间，我既天真又幸运地相信我成功了。我已经掌握了编程。

哦，我错了。

那晚的旅程改变了我。它让我大开眼界。我怎么会天真到相信自己在短短两年内掌握了一切？外面还有多少未知的世界？编程的宇宙到底有多大？

所以现在，差不多两年后，我决定开始新的冒险。不要去征服每一个世界，甚至不要去拜访每一个世界，一生中没有足够的时间去做这些。

号码

我只是希望描绘出编程的许多世界。

# [](#the-many-worlds-of-programming)编程的多个世界

这篇文章背后的想法是，开发被分成许多不同的“世界”,就像你在视频游戏中看到的那样。我们作为开发人员可以从一个世界到另一个世界学习新技能(或者坚持一个世界并掌握它)

我想谈谈我所接触过的(或者至少听说过的)一些不同的编程世界，以及构成那个世界的一些关键部分。

我不是这些领域的专家，我可能会把某些东西放在你不同意的地方。如果你不同意列表中的任何内容，请在评论中告诉我。此外，这还不完整，我肯定我已经错过了至少一半的编程的关键世界。我很想让这成为一个“社区”的努力来绘制所有的世界。

## [](#basic-programming)基本编程

这是编程的最基础层次，几乎每个程序员，不管他们对什么领域感兴趣，都从这里开始。它由编程逻辑的关键组件组成。

**关键部件**

1.  数据类型
2.  变量
3.  收集
4.  基本操作
5.  条件逻辑
6.  循环

## [](#object-oriented-programming)面向对象编程

可能是开发中最流行的范例。面向对象编程是一种编程方法，通过这种方法，系统和应用程序基于不同对象的使用以及它们如何相互作用。

**关键部件**

1.  班级
2.  方法
3.  目标
4.  遗产
5.  多态性
6.  抽象
7.  包装

## [](#functional-programming)功能编程

函数式编程正在兴起，它有助于消除面向对象编程的一些主要缺陷，例如基于外部因素(内存状态)或线程死锁的测试方法的不可预测性。函数式编程的基础是通过使用数学函数(1:1 的输入/输出比)对数据进行操作。)

**关键部件**

1.  功能
2.  处理
3.  监督者
4.  并发
5.  递归

## [](#low-level-programming)低级编程

低级编程是我不完全熟悉的东西，因为我主要和高级语言打交道。低级编程的思想是，你以类似于处理器处理数据的方式来处理数据，而没有新时代语言的所有抽象。级别越低，为您处理的复杂性就越少，因此您必须自己担心诸如内存分配、指针、垃圾收集甚至位操作之类的事情。

**关键部件**

1.  存储器分配
2.  两颗北极指极星
3.  位运算
4.  装配
5.  机器代码
6.  二进制的

## [](#theoretical-computer-science)理论计算机科学

我可以很容易地称这个世界为“准备面试”这个世界由你在大学期间学到的许多东西组成，并且比实际将代码付诸实践更多地涉及编程背后的理论和数学。

我是在训练营学会编程的，所以我缺乏这方面的专业知识，我肯定我错过了相当多的核心部分。如果你认为有什么需要补充的，请告诉我！

**关键部件**

1.  数据结构(集合/堆栈/队列/链表/树等)。)
2.  算法(二分搜索法/冒泡排序等。)
3.  大 O 符号
4.  空间与时间复杂性

## [](#testing)测试

测试是确保代码质量保持一致的最重要的事情之一。测试代码是一种通过特定的输入或交互来模拟代码中某些功能或特性的使用，并确保代码按照您的预期运行的方法。

**关键部件**

1.  单元测试
2.  集成测试
3.  模拟和存根
4.  工厂
5.  基准

## [](#data-management)数据管理

数据管理是处理如何为应用程序或服务存储数据的领域，而不是处理如何计算数据的领域。在这个世界中，您会发现许多不同的独特的数据存储方法——每种方法都最适合特定的情况，您可以使用这些方法来加快对数据的访问，还可以使用一些模式来确保数据的完整性并避免数据丢失。

**关键部件**

1.  结构化查询语言
2.  关系数据库
3.  指数
4.  NoSQL 数据库
5.  图形数据库
6.  内存中数据存储区(Redis)
7.  数据完整性

## [](#data-transfer)数据传输

数据传输处理如何通过互联网或其他网络在设备之间传输数据。许多开发人员坚持使用高级协议和模式，这些协议和模式使用抽象来隐藏正在发生的许多内部事情，但是当您深入这个世界时，您可以学到很多东西！

**关键部件**

1.  OSI 模型层
2.  传输控制协议
3.  超文本传送协议
4.  会话/cookie
5.  SSL / HTTPS
6.  休息
7.  GraphQL

## [](#devops)DevOps

DevOps 现在有点流行，这是有原因的！web 开发过程中最大的瓶颈之一是让运行在开发人员本地计算机上的代码能够在服务器上运行，并对全世界开放！由于环境的差异，开发人员在最终部署他们的代码时经常会面临许多问题和错误。DevOps 是一个致力于最佳实践和工具的编程分支，您可以使用它们来确保从本地代码到实时代码的过渡尽可能无缝。

**关键部件**

1.  12 因素应用程序
2.  饭桶
3.  CI/CD
4.  自动化工具
5.  码头工人
6.  库伯内特斯

## [](#operations)操作

操作就是你的代码如何在野外运行。在您的本地机器上，您可以经常下载一些程序，以便于存储数据或运行您的代码。然而，在生产中，你必须依靠定制服务器或各种云服务来确保用户可以访问你的应用程序，并持续运行。

**关键部件**

1.  主机和域名
2.  AWS /谷歌云/数字海洋/ Heroku
3.  Linux 操作系统
4.  嘘
5.  防火墙和安全性
6.  日志

## [](#frontend-fundamentals)前端(基础)

Frontend 这个词通常用来描述 web 上应用程序的客户端环境。换句话说，前端是用户实际看到的东西。在过去，这通常意味着 HTML 文件和一些 CSS，以及一点 JavaScript 来处理一些动画和对后端的调用。从那以后发生了很多变化，但是这个世界仍然和以前一样适用，并且会教给你很多基础知识，帮助你成长为一名开发人员。

**关键部件**

1.  超文本标记语言
2.  半铸钢ˌ钢性铸铁(Cast Semi-Steel)
3.  java 描述语言
4.  JQuery/AJAX

## [](#frontend-new-age)前端(新时代)

最近，前端开发经历了从 HTML、CSS 和 JavaScript 的旧时代到现在的彻底转变。随着许多工具、标准和框架的开发或实现，这种情况已经发生，这些工具、标准和框架通过处理许多您过去必须手工编码的常见特性或问题来加速开发。

**关键部件**

1.  反应 vs Vue vs 角度
2.  网络包
3.  引导/物化
4.  EMCAScript 6+
5.  渐进式网络应用
6.  SPA 与 SSR
7.  Redux

## [](#architecture)建筑

*感谢[@劳里](https://dev.to/laurieontech)为这个世界出谋划策！*
