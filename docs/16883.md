# 如何看待 Visual Studio Live Share 多用户编辑？

> 原文：<https://dev.to/zspencer/what-do-you-think-of-visual-studio-live-share-for-multi-user-editing-45m8>

你好戴夫。托维亚人！

我正在研究 Visual Studio Live Share 作为一种机制，在不需要太多屏幕共享的情况下进行一些低延迟的远程配对。(哪个慢！)

我很好奇您在以下过程中遇到了哪些挑战:

*   设置它
*   授予访问权限
*   长时间使用它
*   在会话过程中添加新成员/删除新成员

过去，我尝试过类似的工具，我遇到的问题是:

*   “哦，不，它停止工作，但它是无形的”
*   “等等，测试失败了？哦！他们运行的是尚未更新的版本！”
*   “是的，所有的代码还没有在你的电脑上，因为我们还没有在编辑器中打开它们……”

我的希望是，随着微软(他们传统上非常擅长开发工具)对多用户代码编辑器的支持，它将比我过去使用的那些意图良好但资金不足的业余项目更加稳定可用。
