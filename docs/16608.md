# 如何不在面试和招聘技术领域的初级员工时表现糟糕？

> 原文:[https://dev.to/keziyahl/how-to-not-suck-at-interviewing-招聘-初级技术人员-1b40](https://dev.to/keziyahl/how-to-not-suck-at-interviewing--hiring-juniors-in-tech-1b40)

(最初发布于[科技大三学生](https://juniorsintech.com)。)

上周，我在 Twitter 上问了两个问题:

1.  你被一份工作拒绝的具体原因是什么？

2.  即使被拒绝，你有过哪些积极的、[有益的面试经历？](https://twitter.com/KeziyahL/status/1080883639146160130)

当你几乎没有经验的时候，在科技行业找一份工作是很困难的。但是你们中的一些人(我指的是招聘经理、人力资源和任何参与招聘初级技术人才的人)让它变得不必要的困难、令人心碎，甚至带有歧视性。

(当然，这是假设你不是一开始就完全反对雇佣初级人才。)

所以，根据我得到的回复，这里有一些招聘技术行业新人的注意事项。

# [](#1-experience-level)1。经验水平

不要:因为没有足够的经验而拒绝初级候选人。

如果你招聘的是初级职位，你应该知道初级职位的人也会申请，对吗？好的。那你为什么因为我们没有足够的经验而拒绝我们呢？即使不是初级角色，为什么要浪费别人的时间面试他们几个星期，只告诉他们不够有经验？你只要看看他们的简历就能明白这一点。

*   在推特上， [@megantiffany12](https://twitter.com/megantiffany12/status/1080602934700851201) 写道:“申请了全栈职位，完成了 3 次面试。被告知我(1)没有足够的后端经验，以及(2)他们正在寻找高级开发人员。”
*   [@karaluton](https://twitter.com/karaluton/status/1080530510894510081) :“有一次我参加了第三轮面试，有人告诉我，尽管我是代码挑战赛的佼佼者之一，但我还是太年轻了”
*   “在一家大银行参加了 4 轮开发人员职位的面试后，我被告知其他候选人比我更有经验。”正如@taeluralexis [回复](https://twitter.com/TaelurAlexis/status/1080528682634989568)的那样，如果你要这样浪费我们的时间，至少给我们钱吧。
*   [@Dayhaysoos](https://twitter.com/Dayhaysoos/status/1080521811203424257) :“一份工作说他们需要一个有 6 个月经验的 React 本地开发人员，我有 2 年经验，申请，电话面试，然后被告知我没有足够的经验..我做开发人员大概有 3、4 年了。”那这家公司到底想要什么？
*   匿名:他们被一份需要 0-1 年经验的工作拒绝了。为什么？因为他们没有足够的经验。”大约 30 分钟后，他谈到我的项目看起来令人印象深刻，这个角色需要的是我“显然可以做”的事情，再次声明他正在寻找有 0-1 年经验的人，他以“但你没有足够的经验”结束了面试。在你申请更多的职位之前，先把简历填好。”“有人让它有意义。
*   [@mario_p_vargas](https://twitter.com/mario_p_vargas/status/1082186774812610560) :“在一次招聘活动中(针对小开发人员)，一名招聘人员告诉我们，他正在寻找“更多的火力”，而我们的团队项目“太初级了”。“再说一次，这是在一次招聘活动上……为小开发人员”。

**做:如果你只是打算雇佣一个更有经验的人，**那么就有一个单独的角色，有一个单独的工作描述。还有，给大三学生写更好的职位描述。这里有一篇关于[如何做到这一点的好文章](https://tessakriesel.com/mentoring/writing-better-junior-web-developer-job-descriptions/)。

# [](#2-requirements)2。要求

不要:根据 1)不在工作描述中，2)对工作无关紧要，3)甚至不可能的事情来评估候选人。

*   在 Twitter 上， [@mmgfrog](https://twitter.com/Mmgfrog/status/1081174603068858369) 说:“去参加了一个前端面试，但马上就被问及 SQL 查询/连接，很慌张。”不好意思，SQL 知识什么时候变成前端的东西了？
*   在 2017 年底的一次采访中，我在电话屏幕上被问及统计和其他一般的高级数学问题。我不在乎这是不是一个硬性要求(不是，我通过了电话筛选)，或者他们只是想了解一下我的背景知识。我觉得这完全没有必要，让我毫无理由地紧张。
*   [@rodrigoelp:](https://twitter.com/rodrigoelp/status/1081284000763637761) ”被称为“看起来你对 x 技术没有至少 2 年的经验”。这项技术自宣布以来有 1 年时间，自向公众开放以来有 6 个月时间。
*   [@iamask09](https://twitter.com/iamask09/status/1080725932485226496) :“我被一家顶尖的科技公司拒绝了。他们有一次编码测试，我只用了一半的时间就成功完成了，我被拒绝了，只是因为他们不喜欢我的毕业院校。”
*   @ WSmithNY:“3 天后他打电话给我，向我道歉，因为他被迫雇用一个测试分数更高的人，尽管他认为我是最适合这份工作的人。测试被高估了。”

做:根据与职位描述实际相关的事情来评估候选人。

就像我之前说的，为初级员工写更好的职位描述。此外，如果 JD 中有不需要的部分，明确地说出来。我们很多人都放弃了工作申请，因为我们认为我们需要满足每一个要求。(如果你是一个正在读这篇文章的候选人，你不会！无论如何都要申请。)

# [](#3-tests)3。试验

不要:大多数白板测试，以及只是 WTF 的测试。

*   [@paulmcclean](https://twitter.com/paulmcclean/status/1080923915931791360) :“有人让我在白板上画一张流程图，告诉我如何泡茶。我忘记把茶包包括在内了。”
*   [@anerdydev](https://twitter.com/anerdydev/status/1080561072136957952) :“当我申请一家 FAANG 公司时，我的招聘人员说一切都是伟大的文化智慧，但当谈到白板时，我花了比预期更长的时间来回答这些问题，尽管我能够非常正确地回答它们。”因为开发人员的日常生活包括尽快回答白板问题，对吗？

做:与实际工作相关的有用测试。

*   2017 年末，我申请了一家使用 React 的公司，所以我的代码测试是编写一个 React 组件。**看，没那么难**。
*   [@radhikamorabia](https://twitter.com/radhikamorabia/status/1080884994267103232) :“使用非白板方式评估编码能力。1)清除带回家的练习。我知道这很戏剧化，但我完全相信这是评估低年级学生的最佳方式。2)与资深人士讨论代码库。他们会评估你问问题的能力。”

# [](#4-feedback)4。反馈

不要:模糊的、糟糕的、无用的或其他废话的反馈(如果有的话)。

*   设计师@gintel 写道:“有人告诉我，他们觉得和他们一起工作会很无聊。据他们说，我在港口的创意作品比我在那里的任务要多。”(一般生活小贴士:也许不要告诉别人他们对某事的感受。)
*   @browncdonovan 写道:“当我缺乏开发经验时，通常是这样的:我们决定向另一个方向前进，我们决定与更符合我们要求的其他候选人一起前进，或者在仔细考虑后，我们决定这次不前进。”这对任何人都没有帮助。
*   [@isaacandsuch](https://twitter.com/isaacandsuch/status/1080556263597998080) :“大三的时候，一封一行的邮件，拒绝了一个入门级的 web dev 职位。我们不能雇用你，因为你对互联网了解不多。“候选人缺乏迷因知识，还是……？继续说:“这是在一次面对面的面试之后，面试官睡着了，他的助手不得不接手。在一个地下混凝土办公室里，每个人看起来都很痛苦。回想起来，很高兴我没有得到那份工作。"
*   [@o_dafydd](https://twitter.com/o_dafydd/status/1080878939667660800) :“几年前，有人告诉我，他们正在寻找一个对他们可以“更好地塑造”的事物有更少意见的人。”所以我不应该对自己的职业有什么看法。明白了。

我们知道我们还有很多要学习和努力的地方。我们想知道这些东西是什么，以便在下次面试中做得更好。所以请给我们反馈。

*   @ThugDebugger 写道:“改变我生活的采访是脸书和亚马逊。我进入了他们的最后一轮，他们的开发团队说“你很棒。但还没达到我们的水平。阅读克罗克福德的 JS 好的部分，然后回来找我们”。那本书让我的 JS 技能提高了 1000 倍”。
*   匿名:他们必须做一个前端工作的测试，没有通过。但他们得到了非常具体、有用的反馈，说明他们做得好的地方，比如使用 Flexbox 和良好的变量命名，以及他们可以做得更好的地方，比如在代码中使用未使用的变量，以及不分离关注点。
*   在为我前面提到的那家公司进行 React 代码测试时，我能够将我的工作提交给一个开发人员，并获得关于需要改进的地方的详细反馈。在我改进了我的代码之后，我能够再次从开发人员那里得到反馈。
*   @taeluralexis: “一次我最终失败但却很享受的面试是我不得不做一个紧张的带回家的项目。在现场，我们检查了代码，尽管解决方案“比他们最资深的工程师的工作要好”，但他问了一些问题，迫使我进行创造性的思考。我喜欢这次采访的原因是因为它为我提供了一个全新的编码视角。对于一些公司来说，这不是关于你的代码是否能正常工作。他们更关心工程师，以及根据需求&规范的变化来修改和精炼代码的能力。”她面试没通过，但还是收获了很多，真的很爽。

# [](#5-support-transparency-trust-respect-humanity)5。支持、透明、信任、尊重、人性

不要:让我们觉得自己像狗屎。

我不确定我知道如何表达这一点。如果面试没有地狱般的感觉不是很好吗？如果候选人没有感觉一直被蒙在鼓里？如果他们在整个过程中没有感觉到自己是完全独立的？如果公司认可候选人的才华，而不强迫他们去证明自己的价值？如果我们像对待人类一样对待彼此呢？

保持透明&支持我们。把我们当人看！相信我们的工作。无论如何，你都应该在工作中这样做。那么为什么人们在面试过程中不这么做呢？

*   [@Hoeyboey](https://twitter.com/Hoeyboey/status/1080898932711010310) 写道:“当有带回家的编码挑战时，明确时间准则，甚至给我评分表，这样我就知道对我的期望是什么(比如 rubocop 通过了吗:2 分’之类的东西)。”
*   @AireddySuprith :“我最后一轮面试的一天是跟随一名软件工程师一整天。我们结对编程，并允许我编写单元测试。请记住，这是在我收到录用通知之前……我参加了 3 轮面试，每一轮他们都会在每次面试前几天发给我学习和准备的材料。他们对我很坦诚，并在整个过程中一直让我了解情况。招聘人员非常友好，配合我的日程安排。”
*   在推特上，[我写了](https://twitter.com/KeziyahL/status/1080886336289153026):“当然还有我现在在 [@Backstage_Cap](https://twitter.com/Backstage_Cap) 的工作。对我来说最突出的是，我不必费力去证明我能编码。[@ arlanwahere](https://twitter.com/ArlanWasHere)在网上看了我的作品集，看到了我的作品，想和我通电话。”

# [](#6-discrimination-culture-fit-amp-otherwise-being-unfair)6。【文化契合】&歧视，否则就是不公平

**不要:嗯……不要歧视，不要对考生不公平。**

*   [@EmberFirehair](https://twitter.com/EmberFirehair/status/1080580230249472000) :“还有一次我没有足够的经验——尽管他们雇佣的那个家伙比我经验少。”
*   “我为一个家伙工作，他在我开始工作一年后坦率地承认我得到了这份工作，因为另一个候选人是印度人。我感觉糟透了。之后就开始寻找新的工作。”
*   [@GriyoPikliz](https://twitter.com/GriyoPikliz/status/1080524842431139842) :“我的一个阿尔及利亚朋友有一个阿尔及利亚硕士，我们一起参加了一个新兵训练营，每一份工作在他完成第三次面试后，他们告诉他，他不能被录用，因为缺乏美国经验”
*   [@ch1nux](https://twitter.com/ch1nux/status/1080800156138831873) :“我试过申请的最差的大公司，我被拒绝了，因为我来自委内瑞拉。招聘人员认为我与政府有关系……他们没有道歉……”
*   [@the_happy_coder](https://twitter.com/the_happy_coder/status/1080899562481598465) :“我被拒绝了，因为我说“我今天还有一个机会回复我，所以在我对这个提议做出决定之前，出于尊重，我想我应该听听他们的意见。"他们说我“动力不足”，撤回了他们的提议。"
*   [@the_happy_coder](https://twitter.com/the_happy_coder/status/1080899751032307713) 分享了另一个经历:“我也有一个地方给我回电话说‘我们认为你没有 10000 小时的经验，所以我们不能给你这份工作。不过，我们可以给你一份无薪实习。然后我说，“很抱歉，但我现在需要关注健康保险带来的机会。”他们说“你现在不是失业了吗？你现在有医疗保险吗？”这很有侵犯性。"

哦，我投票赞成在招聘时禁用“文化契合度”这个词。说某人不符合文化是没有帮助的，而且**我们知道你真正的意思。**

*   [@MxOwl](https://twitter.com/MxOwl/status/1080521112688066560) :“有一次，我大三的时候，因为“文化契合度”(都是直白公子哥儿)，在通过所有白板测试后被拒绝了。”
*   [@NerdyAndNatural](https://twitter.com/NerdyAndNatural/status/1080520384938762240) :“我总是得到‘不符合文化’的答案，这个答案对我来说既不合理，也不代表我的技能。”我回答说:“有时候我怀疑那是不是“不是白人”的暗号。
*   [@missvallord](https://twitter.com/missvallord/status/1080994142765776896) :“那个我自己也有。在疯狂和失望了一段时间后，我最终感到欣慰，因为我不想与独角兽、忍者和兄弟们一起工作。我想与解决复杂问题的聪明人一起工作。”
*   [@1queer1](https://twitter.com/1queer1/status/1080525101156704256) :“不符合文化”——尽管告诉我我是一个多么优秀的候选人，多么令人愉快，我拥有他们想要的一切——只要它不是在一个奇怪的身体里(或有一个奇怪的头脑)。"

做:认识到你有偏见，并采取行动。

你和你团队中的其他人可能在某种程度上对招聘有偏见。雇人帮你解决这个问题。说真的。有人拿工资来帮助做这种事情。没有理由不在公司内部解决这个问题。否则，你可能会传递非常好的人才，更不用说(如果你在美国)做非法的狗屎。

最后，聘用大三学生。我们很有才华，很棒，渴望学习，比你想象的要能干得多。如果你想在科技时事通讯的[上发布一份工作，请告诉我。](https://juniorsintech.com/)

我的低年级同学们，坚持住。尤其是如果你想在科技行业找到第一份工作。我知道你能做到。
