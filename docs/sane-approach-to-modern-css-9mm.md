# 现代 CSS 的合理方法

> 原文：<https://dev.to/bcncodeschool/sane-approach-to-modern-css-9mm>

作为我们的 JavaScript 全栈训练营的一部分，教授 CSS 已经有几年了，这有助于了解一些初学者最常见的困惑。虽然 CSS 不是一种编程语言，但对一些人来说，它似乎比 JS 更令人沮丧。以下是我们定期给所有学生的最常见的建议:

## 保持简单。

没必要把事情过分复杂化。如果你需要做的只是水平居中你的文本，只需使用`text-align: center`。无需使用 Flexbox 或 margins 或`position`。有时候，简单地浏览一些你喜欢的网站，并从那里复制一大块 CSS 到你的样式表中是很有诱惑力的。嗯，有可能你会有 90%的多余的 CSS，而你并不真正需要。另外，这对 T2 学习 CSS 一点帮助都没有。

## 浮动、伸缩、网格——选择什么

随着 CSS 变得越来越大，有时选择正确的方式会让人不知所措。我们的一般建议是在顶层元素上使用网格来创建布局——页眉、页脚、列、侧栏。那么对于嵌套元素，Flex 似乎是对齐的最佳选择，尤其是对于垂直居中。还有浮动——嗯，在大多数情况下，你甚至不需要它们，这很好。对于初学者来说，根据浮动相对于兄弟/父元素的行为来控制浮动可能有点困难。

## 在你的 CSS 文件中有一些结构

在 CSS 文件中组织样式的一个好方法是从元素选择器开始。然后为这些类中的元素添加类选择器和组合选择器。之后，把你的 id 选择器。我们还注意到，对于一些学生来说，按照他们在 DOM 中的顺序组织 CSS 选择器更自然。这似乎也是一个很好的方法，因为如果你知道你的`footer`在 HTML 的底部，向下滚动到 CSS 文件的底部来找到页脚的样式是很自然的。

## 为较大的应用程序将 CSS 分成几个文件

将应用程序的许多组件组织成一个良好的结构，拆分 CSS 文件也是有意义的。所以，比方说，你可以把你的风格应用到你的网站的一些特定区域到单独的 CSS 文件中，比如 header.css，footer.css，sidebar.css 等等。它不仅可以帮助你减小样式表的大小，防止它变得太大，还可以让你的组件在 js 和 css 文件完全隔离的情况下更加可重用。

## 内联与外部

使用 React 时，您可能会将 css 放在 JS 中，这种情况下的样式被浏览器视为内联样式，因此当您要应用 CSS 文件中的一些全局样式时，如果您的选择器将 JS 生成的内联 CSS 中的元素作为目标，它们将被忽略，因为内联样式将优先。请记住这一点，不要在“它不起作用”时感到沮丧。

## 长度单位

虽然像素看起来可能是最好的选择来建立你的布局，就像你在设计模型中一样，并且在你的屏幕上看起来很棒，但是节省你的时间，马上从相对单位开始。

在某些情况下，像素可能是最好的选择，例如，限制内容容器的`max-width`,这样它就不会在更大的屏幕上变得太大而降低可读性，但通常情况下，像素会因为视窗大小的微小变化而破坏您的布局。因此`%`、`em`和`vh, vw`在几乎所有情况下都更合适。

## 媒体查询

有这么多用户代理，我们如何为媒体查询选择断点，我们应该有多少个？再次强调，保持简单。桌面版、平板电脑版和移动版将帮助你不至于发疯，同时拥有一个反应灵敏的网站。请记住，您只需要为媒体查询中的选择器添加实际正在更改的属性，而不是所有属性。

最后一个终极建议——你的网站在不同的浏览器和设备上看起来不一样是没问题的，不要试图拥有绝对完美的版本。首先是内容和功能。用户应该能够阅读你为他们准备的所有内容，并与你的网站互动。专注于此！
