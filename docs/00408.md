# 作为一名开发人员，这幅图如何改变了我

> 原文:[https://dev . to/nikitachernykh/how-this-drawing-changed-me-as-a-developer-hhf](https://dev.to/nikitachernykh/how-this-drawing-changed-me-as-a-developer-hhf)

今天我想和大家分享这个小图是如何成为一个功能性的 chrome 扩展，以及它如何改变了我这个开发者。一天，当我浏览互联网时，我发现我完全忘记了时间。我认为如果我能追踪我在网上冲浪浪费了多少时间，那将是一个很棒的主意。为了解决这个问题，我决定开发最终成为互联网迷的 chrome 扩展。

在 Instagram 上关注我，看看我现在在做什么。👀

## **它是如何开始的**

我从 jQuery 开始。我希望我可以使用 React 或其他流行的框架，但我必须用我当时所知道的东西来实现它。我很想学习其他的库和框架，但是没有项目来应用它们。我是那种总是试图从我所做的事情中获取价值的学习者。每当我开始学习一些东西的时候，我喜欢把它应用到我遇到的现实世界的问题中。我不喜欢仅仅因为“每个人”都认为知道很棒就去学习东西——我想通过找出能帮助我解决问题的最佳方法来学习。

在某一点上，jQuery 变得杂乱无章，我开始考虑改用 AngualrJS。转换很有挑战性，但更重要的是，它向我介绍了一个缩放问题。现在我有太多的代码要处理。我在角度控制器中有不必要的逻辑，我的代码是重复的，感觉没有条理。

所以我决定清理一下。

## **小步骤**

我把网虫分在不同的功能里，把和那个功能相关的东西都放在同一个文件夹里。
[![](../Images/cb890fa5ed9ef258ccb25c7e8f015f5f.png)T3】](https://res.cloudinary.com/practicaldev/image/fetch/s--0r_Em_hv--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/c1nn0sze8kkrmdpm00a4.png)

对于应用程序逻辑，我把所有东西都放在 background.js(在后台文件夹中)中，我把它们转换成一个模块，我计划将来把它拆分成更小的模块。Browserfy 帮我打包了所有东西，我添加了 Jest 进行测试。

## 学习测试——我的“啊哈！”时刻

在着手这个项目之前，我总是害怕测试，我看不到其中的价值。“为什么我要写两遍代码”，我想。现在我知道为什么它对我没有意义了。我看不到价值，因为我早期的许多项目都很小，而且很短暂。

当我日复一日地在“网络迷”上工作时，我注意到我开始害怕对项目的某些部分进行修改，仅仅因为它们已经在工作了。我害怕，因为我不再相信我的代码了！这就是为什么测试是“必须”的原因！在用测试编写并覆盖了我 70%的代码之后，对代码重构的恐惧消失了。我变得更加自信，可以专注于新功能。

## **这个项目如何改变了我**

我不再是以前的那个开发者了。现在，当开始一个项目时，我试着把这些重要的知识记在心里:

1.  早期的决定会对未来产生积极和消极的影响。仔细考虑你决定使用什么语言和框架。你试图解决的问题是什么？你使用的工具是否正确？

2.  编写可重用的代码。如果可能的话，将你的代码分解成更小的部分，重用公共的部分来保持代码的干燥(不要重复)和可读性。

3.  测试！编写测试不仅能帮助你更好地理解你的代码，还能让你有信心做出改变，而不用担心意外引入 bug。

4.  不要害怕犯错误！我用这句话作为我编码“快速移动和打破东西”的指南。这个想法是，如果你从来没有打破任何东西，你可能走得不够快。

5.  找人给你反馈。即使你有 5-10 个用户可以激励你为他们开发最好的应用程序！

## **网虫教会我什么**

对我来说，最大的收获是计划的重要性。当你用一个代码构建一个新的东西时，你经常会被卡住。有时候多到很容易气馁。我意识到我被卡住了，因为我的任务太大了，以至于无法开始。我学会了花更多的时间去计划和把事情分成更小的部分。这种方法在我做《网虫》的时候帮了我很多次，没有它，我永远也不会完成它。

最后，我明白了找到我做这个项目背后的原因是多么重要。对我来说，没有目标，我就失去了动力，没有动力，就没有更多的乐趣。谁喜欢做自己不喜欢的事情？有时候你不得不这样做，但是如果你有选择的话会更好。我的动力是做一些对人们有帮助或节省时间的事情。

当我想通这一点时，我头脑中的一切都清楚了，我知道我下一步要做什么。
