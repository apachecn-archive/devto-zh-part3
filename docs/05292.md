### [](#my-experience-with-agile)我的敏捷经验

敏捷开发是我们在 Flatiron 开发软件的方式。我们在整个项目中实施了测试驱动开发、看板、站立会议和线框图。我用这些方法来构建我的每个主要项目。

### [](#what-is-agile-development)什么是敏捷开发？

我最近听了 Javascript Jabber 关于这个话题的播客。这里的是这一集的链接。这一集的嘉宾是杰出的敏捷开发者 James Shore。

James Shore 花了相当多的时间讨论敏捷开发的演进。它创建于 2001 年，伴随着敏捷宣言的出版。如果你对这个话题感兴趣，我强烈推荐这个播客。

### [](#safe-coaching)安全辅导

回到 meetup 和 SAFe，Robert 描述了教练框架，并分享了他作为 SAFe 教练的工作经历。我特别喜欢听到经理和教练之间的区别。

### [](#final-thoughts)最后的想法

敏捷软件开发是一种实践，我将在整个开发过程中继续实践。我对练习为 React 程序编写更多的测试特别感兴趣。
