# 信息安全思维

这篇文章最初发表在我的博客上。

* * *

在对黑客、恶意软件、入侵检测、社会工程和所有定义信息安全领域的有趣和有趣的事情感兴趣后，我决定喝喷泉里的水。我冒险决定投资它，并最终认真对待它。我开始学习更多，阅读更多，并参加了 ctfs。

在解决了 [hackthebox](https://hackthebox.eu) 上的几个简单机器后，我开始意识到一些事情:

*   你总是在学习
*   你永远是个菜鸟
*   分享是关怀。我得到了如此多的帮助和支持，以至于不以某种方式“回报”会感觉很奇怪
*   安全是一种心态

这件事是最重要的，我想就此再详细阐述一下。为什么安全应该是一种心态？不就是用技术很容易解决的事情吗？最终，技术会被利用，对吗？

嗯，是也不是。有很多利用代码中的弱点的漏洞。但是，最重要的攻击媒介之一(如果不是最重要的话)是坐在键盘前的人。问问每一个有经验的社会工程师(或者大多数孩子)——人类想要帮助，他们想要让你的生活变得更容易。所以意识到人性是安全意识的一部分。要意识到，有人可能想利用你。不是你面前的电脑，不是你桌上的智能手机。你是目标。因为有时候询问密码比利用漏洞更容易。

安全性思维的另一部分影响着开发人员。开发者(作为一个角色)是非常面向目标的。他们想把事情做好，实现另一个特性，并且非常有条理。编写测试，编写文档——在此之前，用户故事被分解成任务，必要的信息被提取出来。所有这些都是过程的一部分，定义了程序员脑海中的图像。一个图像，其中充满了需要做什么——有时还有如何做。也许他们心里也有一些测试用例。这可能非常危险，因为开发人员可能只关注预期的行为、预期的输入等。但是所有的边缘案例呢？从个人经验来说，这种情况经常发生。owasp top-10 并非偶然包含许多容易缓解的内容。2019 年处理 sql 注入真的不是技术问题……

但是，即使有安全意识的开发人员，这也还没有解决。最优秀的安全意识开发团队可能会被产品所有者(或者创建用户故事/规范的人)误导，或者严重限制他们的能力，因为他们只关注特性。尤其是当时间或预算不足时(这种情况总是存在)，安全性很容易被忽略(“如果我们不看，一切都是安全的，对吗？”)或被视为竞争劣势。创造一个安全的产品既困难又昂贵。至少比完全忽视安全性要昂贵得多。只要客户不把他们的钱带到其他地方去惩罚那些明显不关心他们安全的组织，这方面就不会有任何改变。

考虑到客户，我们找到了下一个安全思维能够带来回报的群体:付费客户。如果丑闻、漏洞和安全漏洞不足以让你放弃一些公司和产品，那么，为什么要改变呢？如果我们不用钱投票，什么都不会改变，每个人的数据最终都会公之于众。

总结一下

*TL；*
博士安全感是一种心态。开发人员、产品所有者和客户可以从开发他们使用的产品的安全性中获益——否则我们将面临越来越多的安全漏洞。

为了创建安全的软件，参与该过程的每个人都可以受益于意识到安全不是一个成本因素，而是一个质量问题。
