# JavaScript 社区中我最喜欢的老师

> 原文:[https://dev . to/emmabostian/my-favorite-teachers-in-the-JavaScript-community-53h](https://dev.to/emmabostian/my-favorite-teachers-in-the-javascript-community-53h)

科技世界中有如此多的资源，很难对其进行分类并为你选择合适的资源。有这么多的课程、博客和播客可供选择，所以我列出了 JavaScript 和前端开发社区中一些我最喜欢的老师。欢迎在下面留下你的建议！

# [](#brian-holt)[布莱恩霍尔特](https://twitter.com/holtbt)

[![Brian Holt](../Images/3194dec202a8b4dd05641dbd7e638291.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--etwj7ZoX--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/99hfy87ok29th0179cg2.png)

布莱恩有许多令人难以置信的、简单易懂的关于[前端大师](https://frontendmasters.com/)的课程。他的 React 课程将在几个小时内把你从初学者变成专家。另外，他是一个真正的好人。

# [](#kahlil-lechelt)[卡希尔·勒切特](https://twitter.com/kahliltweets)

[![Kahlil Lechelt](../Images/25b36977ea8c225b2eed95113280df17.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Nof7HXv1--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/y0fm2x78328kptb9phep.png)

我对这个建议有点偏见，因为 Kahlil 是我的同事，然而我在 LogMeIn 的一年里从他那里学到了很多东西。Kahlil 在德国卡尔斯鲁厄经营一个 JavaScript meetup，并在他的个人网站上发布与 T2 相关的博客。

# [](#dan-abramov)[丹·阿布拉莫夫](https://twitter.com/dan_abramov)

[![Dan Abramov](../Images/485b0dcb1e4b9b97d7d24507e325d986.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ckYez4go--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/c23uv7yexigxs99kbr95.png)

丹出现在这个名单上有什么奇怪的吗？他是 Twitter 上最平易近人、最受欢迎的 React 开发者之一。他的教学风格非常简单易懂，当他不知道什么的时候也不怕承认。我强烈推荐订阅[他的简讯](https://overreacted.io/)。

# [](#sarah-drasner)[萨拉·德拉斯纳](https://twitter.com/sarah_edo)

[![Sarah Drasner](../Images/e19e0278dfa9354d022085d4464195c3.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--WnKLhfdd--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/awk912fxi35dcwkidu2x.png)

Sarah 是开发人员社区中另一位令人难以置信的老师。她关于 Vue 和 SVG 动画的教程对我来说是无价的。她还写了一些关于 CSS 技巧的博文，我强烈推荐你去看看。

# [](#john-lindquist)[约翰·林奎斯特](https://twitter.com/johnlindquist)

[![John Lindquist](../Images/114c78e6ea030a81def76a4aef2c521a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--8ioqqYTa--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/6kiwy24ka342yfccypje.png)

约翰是你见过的最好、最聪明的人之一。他是受欢迎的教程网站 [Egghead.io](http://egghead.io/) 的联合创始人之一，并且一直在制作优秀的内容。他的教程很小，很容易理解。

# [](#jacob-schatz)[雅各布·沙茨](https://twitter.com/jakecodes)

[![Jacob Schatz](../Images/d9cf132f47c9eba0c1e164003146c12e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--6Ot0TcaC--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/tpecwsbak2d1lg6pjj0a.png)

我第一次听到 Jacob 在阿姆斯特丹会议上的演讲时，我被震撼到了。他能够以一种完全有趣的方式分解设计模式，这是一个以前密集的话题。另外，他非常有魅力，会让你整个演讲充满乐趣！

# [](#ali-spittel)[阿里·斯皮特](https://twitter.com/ASpittel)

[![Ali Spittel](../Images/d3b4cd0a253d7d1ea54424ea86cc80f6.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Gb4UWGM7--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/bktrf939acuq21n1oslj.png)

阿里上这个榜也就不足为奇了。她在 Dev.to 上发布了令人惊叹的博客[，为初学者提供了很棒的内容。此外，她在科技界有着积极的影响。](https://dev.to/aspittel)

# [](#kyle-simpson)[凯尔·辛普森](https://twitter.com/getify)

[![Kyle Simpson](../Images/5803c4cbae35971fa1e90dbf372c1a19.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--7NdrnZS8--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/83t019tytyg62mj2fuu3.png)

凯尔·辛普森是最终帮助我理解 JavaScript 的老师。他有令人惊叹的前端硕士课程，并在 GitHub 上免费发布了他的[你不知道的 JavaScript](https://github.com/getify/You-Dont-Know-JS) 系列。他是一个真正的 JavaScript 大师，一个全面的好人。

# [](#kent-dodds)[肯特·多兹](https://twitter.com/kentcdodds)

[![Kent Dodds](../Images/55d749ee85c77baab515e6f52a70dff1.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--yPL_XegJ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/fz43alksd9rwvi85nly7.png)

肯特擅长教授许多不同的东西。你可以在 [Egghead.io](http://egghead.io/) 以及[前端大师](http://frontendmasters.com/)上找到他。他的内容非常容易理解，如果你有任何问题，他真的很有帮助。

# [](#rick-hanlon)[瑞克·汉隆](https://twitter.com/rickhanlonii)

[![Rick Hanlon](../Images/2c49d26a7bfd60c4bb810145342c05cf.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--195iU9Nu--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/xh274aosym2qbx5kiiky.png)

我第一次看到 Rick 在阿姆斯特丹演讲，他的演讲让我印象深刻。他用 Jest 谈论测试，并以一种全新的贝莱尔王子格式这样做。他不仅超级有趣，而且让学习复杂的主题变得轻而易举。

# [](#joe-previte)[乔普雷维特](https://twitter.com/jsjoeio)

[![Joe Previtte](../Images/0a9040c55270f49962b46ff26c4da3f7.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--oEDCsOt6--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/c9681738z9h4n021kveq.png)

Joe 在 Egghead.io 上做了很棒的教程。他的 Figma 设计课程让学习原型设计变得很容易。Joe 在 Twitter 上非常活跃，总是散发着积极的气息。

# [](#eve-porcello)[夏娃波尔切洛](https://twitter.com/eveporcello)

[![Eve Porcello](../Images/1e2d6c3ddc2a967a0a9946ac90d7b881.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--SkAy5wPt--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ok85we7q1sd33vi0iym7.png)

Eve 在 Egghead.io 上制作了关于 GraphQL 的惊人教程。我在 Lynda.com 的[上过她的很多课。她教 React、Webpack 等等！](https://www.lynda.com/Eve-Porcello/1874244-1.html)

# [杰森·兰斯多夫](https://twitter.com/jlengstorf)

[![Jason Lengsdorf](../Images/63e909697d606338430cbcd07adc9ad3.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--BS9xz8gc--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/f6wlospup5bdqvw68i85.png)

我认识杰森已经好几年了；我们在 IBM 一起工作。自从我们认识以来，我可以告诉你，和他一起工作是一种享受。他为 Gatsby(我最喜欢的 React 静态站点技术)进行了精彩的现场直播，并且总是乐于提供帮助。

# [](#lindsey-kopacz)[林赛科帕奇](https://twitter.com/LittleKope)

[![Lindsey Kopacz](../Images/7014f4d5f3720a73e2f81bcffdb6cd41.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ftH7_RTj--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/bh9gvnthu5k3s1yhdn29.png)

Lindsey 在可访问性领域知识渊博。她在确保我的组件符合 W3C 可访问性方面给予了极大的帮助。她在自己的[个人网站](https://www.a11ywithlindsey.com/)上发布博客，还在 [Egghead.io](https://egghead.io/instructors/lindsey-kopacz) 上开设课程。

# [](#greg-sidelnikov)[格里格·西德尼科夫](https://twitter.com/js_tut)

[![Greg Sidelnikov](../Images/829ecea39fe1421dfca7e8f7092e803a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--8XUdT55P--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/sm9ce3zvhm6dga4prmn5.png)

你可能知道格雷格是推特上的 JavaScript 老师。他制作令人难以置信的内容(免费！).他出版了几本书，包括 [JavaScript 语法](http://www.javascriptgrammar.com/)和[CSS 视觉词典](https://www.amazon.com/CSS-Visual-Dictionary-Greg-Sidelnikov/dp/1983065633)。

这些只是在我的编码之旅中帮助过我的少数人。请在下面留下你最喜欢的老师！
