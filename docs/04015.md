偶尔，我偶然发现一份设计文档中有“直觉”这个词。它可能出现在一个新的项目简介中，或者一个要求“我们的用户需要直观和最好的体验”的需求文档中它可以在公司的设计原则剧本中找到，位于“令人愉快”一词附近，并尽可能留下解释的空间。

> “直觉”这个词的问题在于，它对不同的人有不同的含义:对我来说是直觉的产品，对我父亲来说可能不是直觉的。

没有人把 I 字写下来，是因为幼稚，或者没有好的意思。他们写这篇文章是为了提醒人们，在整个设计过程中，简单的体验比不太重要的事情更重要。

当然，直到最后期限临近，商业 KPI 变得更加激进。

### [](#intuitive-shouldnt-be-a-design-goal)直觉不应该是设计目标

你见过哪个产品需求文档专门概述某个产品应该*而不是*直观？

作为客户或商业利益相关者，易用性是对任何产品的基本期望。任何使用起来不直观的东西都可能影响转换率、参与度和任何其他使产品在逻辑上可行的指标。

作为一名设计师，易用性应该是你工作的一个品质，而不是你在设计中插入或删除的一个特性。

### [](#breaking-down-the-meaning-of-intuitive)分解直观的意思

“直觉”这个词的问题在于，它对不同的人有不同的含义:对我来说是直觉的产品，对我父亲来说可能不是直觉的。

找到直观的东西取决于太多的因素，设计师无法标准化这个术语的真正含义:用户的年龄、性别和性别、文化背景、技术成熟度、过去使用类似产品的经验，等等。

广义来说，直观仅仅意味着**易于使用**。

但是，如果你更深入地了解人们在开始概述“感觉直观”的体验时的真正意思，你会发现一套非常基本的设计原则，如果你是 UX 或 UI 设计师，这应该听起来非常熟悉:

*   **可见性:**屏幕上的重要元素是否可见？一个元素越明显，用户就越有可能知道它们以及如何使用它们。
*   反馈:一旦用户采取了行动，他们是否清楚采取了什么行动，该行动完成了什么？
*   **约束:**界面是否限制了用户可以采取的动作集合？无尽的可能性经常让用户感到困惑。
*   **启示:**设计是否帮助人们理解*如何*与某些元素互动？启示是给用户每个组件如何工作的“线索”。
*   **可发现性:**是否有可能通过与一个物体的互动来弄清楚如何使用它？
*   **定位:**在任何给定的时间点，用户是否知道他们在产品结构中的位置？他们知道如何后退和前进吗？
*   期望:在采取行动之前，行动的结果是否符合用户对结果的期望？
*   **效率:**产品在帮助用户完成一项任务时是否高效？是否有提高效率、缩短流程或加快结果的空间？

关于这个话题的更多信息，请阅读唐·诺曼的《日常事物的设计》——任何数码设计师的必读之作。

希望下一次在对话中出现“直觉”这个词时，你会有更多的信息来帮助理解对方真正的意思。
