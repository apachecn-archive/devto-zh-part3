# 2019 年顶级 JavaScript 框架

> 原文:[https://dev . to/lambdatest/top-JavaScript-frameworks-for-2019-491k](https://dev.to/lambdatest/top-javascript-frameworks-for-2019-491k)

今天，世界范围内使用了大量的编程语言，每种语言都有自己的用途、复杂性、优点和怪癖。然而，毫无疑问，JavaScript 在网络上留下了不可磨灭的持久印象，连续第六年成为世界上最受欢迎的编程语言。

[Stack Overflow 2018 年度开发者调查](https://insights.stackoverflow.com/survey/2018/)涉及超过 100，000 名受访者，表明全球近 70%的开发者认为 JavaScript 是他们最喜欢的编程语言，远远超过了 Java、Python、PHP 和 C#等主流竞争对手。

JavaScript 广泛流行和占主导地位的关键原因在于它的多功能性和灵活性。JavaScript 不仅被视为传统客户端使用的网络语言，而且现在也与原生移动应用开发、桌面应用、[自动化机器学习](https://www.lambdatest.com/blog/machine-learning-for-automation-testing/)、[渐进式网络应用](https://www.lambdatest.com/blog/testing-a-progressive-web-application-with-lambdatest/)等一起用于服务器端应用。

有数百个 JavaScript 框架和库可供您使用，而且每个月还会推出几十个，这使得为您的下一个项目选择正确的框架成为一项令人生畏的任务。为了帮助你，我整理了 2019 年 11 大 JavaScript 框架列表。我根据 HackerRank 的 StateOfJs 调查提供的统计数据编制了这个列表，并参考了 SimilarTech 的使用统计数据。

## [](#front-end-frameworks)前端框架

### [](#1-react)1。反应

[![image](../Images/4bb59f0ea37cc933066713f58f87ab6c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--xkwo9sPx--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image6-4-1.png)

毫无疑问，React JS 在我们的 2019 年顶级 JavaScript 框架列表中排名第一。React 是一个高效的声明式 JavaScript 库，用于构建交互式 ui。它由脸书工程师乔丹·沃克创造，于 2013 年 5 月发射。在过去的 5 年里，React 超越了竞争对手 Angular JS，牢牢确立了自己的统治地位。React 用于为每个状态创建可重用的用户界面组件，这些组件可以自动更新和呈现数据，而无需重新加载页面。请记住，React 是一个库，而不是技术上的框架。它主要用于处理 web 和移动应用程序的 MVC(模型视图控制器)模型中的“V”(视图)层。除了 web 应用程序，React 还可以用于创建 android 和 iOS 都支持的原生移动应用程序。凭借如此多的贡献，React JS 在 2018 年统治了其他框架，我相信 [React JS 也将在 2019 年继续统治](https://www.lambdatest.com/blog/will-reactjs-continue-to-dominate-in-2019/)。

**使用统计**

*   React 为网络上的 475，000 个网站提供支持。
*   目前有 64.8%的 javascript 开发者在使用 React。
*   在 JS 调查中排名第一的最受欢迎的前端 javascript 框架。
*   github——120.5k 星，21k 叉，1200 贡献者。

[![image](../Images/a7b6e78ca5d31a7846147a01bcd28f4d.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--XGOmflFf--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image9-6-1.png)

**为什么 REACT 是 2019 年的顶级 JavaScript 框架之一？**

*   **高性能**–React 以其高效率和灵活性而闻名。它可以很容易地与其他框架集成，没有任何麻烦。可用于客户端和服务器端。
*   **丰富的资源**–react 由脸书维护，网上有大量的文档和资源，这使得 react 的学习曲线比它的主要竞争对手 AngularJS 平滑得多。
*   **向后兼容**–使用 CodeMods 从旧版本过渡或迁移到新版本相当容易。
*   **易于维护的组件结构**–基于组件的架构有助于提高代码的可重用性和大型项目的易维护性。
*   **丰富的社区**–React 在 GitHub 上有近 1300 名贡献者，比其他任何库/框架都多。
*   **单向数据流**–react via flux controls 中的单向/单向向下数据绑定有助于确保对子元素结构的任何更改都不会影响父元素结构

**使用 REACT 的缺点**

*   与 Vue JS 相比，许多开发人员可能会发现 react 过于复杂和臃肿。对于小规模项目来说，coms 与 react 保持一致所增加的复杂性可能是不必要的
*   JSX——JSX 的使用增加了另一层复杂性。JSX 是一个预处理程序，它为 JS 增加了 XML 语法扩展。尽管 JSX 有助于以更安全、更快速的方式进行代码反应，但对于新开发人员来说，这很难掌握。
*   **对组装工具的需求**–React 需要大量的组装工具才能正常工作，并与其他库和框架兼容。
*   **SEO 问题**–众所周知，使用 react 构建的网页会面临搜索引擎爬虫/机器人的索引问题。

**杰出用户**
网飞、雅虎、Airbnb、Instagram、脸书、WhatsApp、Sberbank、PayPal、微软、BBC

### 2。视图(view)

[![](../Images/88dbf8903b8af4edd23c766e8ec51229.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--QC4LF3YF--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image5-5-1.png)

Vue Js 在 2019 年的顶级 JavaScript 框架中排名第二。Vue 是一个开源的 javascript 框架，旨在简化 UI 设计。它由尤雨溪创作，于 2014 年发布。过去几年见证了 Vue 受欢迎程度的爆炸式增长，在 2018 年 JS 状态调查中，Vue 的增长突飞猛进，超过了谷歌的竞争对手 AngularJS 框架。

Vue 吸引人的关键点是它的进步性，这有助于它适应开发者的需求。它建立在 angular 和 react 等其他框架和库的概念之上，并提供了一个包含所有最佳特性和实践的紧凑轻量级解决方案。

**使用统计**

*   64k 网站由 Vue 提供支持。
*   目前有 28.8%的 javascript 开发者在使用 vue。
*   在 JS 调查中排名第二的最受欢迎的前端 javascript 框架。
*   github——125k 星，18k 叉，253 个贡献者。

[![image](../Images/2156146dc6d483c94a784f73adb501ef.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s---7KTEvPa--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image13-3-2.png)

**为什么是 VUE。JS 是 2019 年最顶尖的 JavaScript 框架之一？**

*   **快速配置** : Vue 有一个内置的 MVC 模型，与 angular 和 react 相比，它使配置变得快速简单。
*   **灵活性**:Vue 的灵活性和模块化特性有助于具有其他框架和库经验的开发人员以最小的努力快速采用 Vue。
*   **轻松学习曲线**:Vue 的学习曲线比 Angular 和 react 容易多了。只需要 HTML 和 JavaScript 的知识。
*   轻量级尺寸:与 angular 和 react 等所有其他主流 JavaScript 框架相比，Vue 是高度轻量级且尺寸紧凑的。Vue 的压缩版只有 18 kb 大小。
*   **集成能力** : Vue 可用于构建 spa 和大规模复杂应用程序，也可与 Laravel、Symfony Django 等服务器端框架轻松集成。
*   **转换**:当在 DOM 中添加或删除元素时，Vue 可以很容易地将转换添加到元素中。

**使用 VUE 的缺点**

*   支持不佳:Vue 没有像 angular 和 react 那样为新用户提供大量的资源。考虑到相对较低的市场份额，刚接触 Vue js 的开发者可能找不到足够的支持和资源。
*   小社区:Vue 开发者社区的很大一部分来自非英语的东欧国家。社区参与度过去相当低，但预计这种情况将在 2019 年发生变化，因为截至 2018 年底，Vue 的受欢迎程度大幅上升。

**知名客户**
StackOverflow、9gag、Adobe、小米、Grammarly、阿里巴巴

### [](#3-angular)3。有角的

[![](../Images/5bc3db7a393409c5b4716513de54aafd.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--pi8hXspz--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image20-2.png) 
Angular 是一个基于 typescript 的前端 web 应用框架。AngularJS 或 Angular 1 是由谷歌工程师 Misko Hevery 和 Adam Abrons 在 2010 年推出的，而 angular 2 是在 2016 年推出的。

Angular 可以说是最古老的现代 javascript 框架，它允许用户构建动态交互式网站。尽管是最古老的， [Angular JS 仍然被认为是全球大量开发者的第一选择]((https://www.lambdatest.com/blog/angularjs-a-developers-first-choice/)),这就是为什么它是 2019 年顶级 JavaScript 框架之一。

Angular 使开发人员能够构建高效和健壮的单页面应用程序，而不是静态的 HTML 网站。Google 彻底修改了最初的框架，发布了一个完全不同的改进的 typescript Angular 2。Angular 2 引入了基于组件的架构、改进的依赖注入、日志服务和组件间通信。

最新的当前版本 7 于 2018 年 9 月推出，每 6 个月发布一次新的更新。版本 7 在性能、角度材料和 CDK、对 Typescript 版本 3.1、RxJS 6.3、节点版本 10 的依赖性方面引入了巨大的改进。还引入了虚拟滚动和内容投影。

**使用统计**

*   网络上 35 万个网站都是 AngularJS/Angular2+版本的。
*   目前有 23.9%的 javascript 开发者在使用 Angular。
*   在 JS 调查中排名第三的最受欢迎的前端 javascript 框架。
*   github——44.5k 星，11.5k 叉，840 贡献者。

[![](../Images/3034d3ddae842047c0b0846d541bb096.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--IAVl2ZlM--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image11-5-1.png)

为什么 ANGULAR 是 2019 年最顶尖的 JavaScript 框架之一？

*   **数据绑定**除了 AngularJS 中的双向绑定，angular 2 和 beyond 还引入了 3 种其他类型的数据绑定，即单向属性绑定、事件绑定和插值。
*   **增强的 RXJS** 在 Angular 中使用 RXJS 库来处理异步数据调用。它允许在执行的同时独立处理事件。这导致了闪电般快速的编译时间。
*   **丰富的资源和支持**:谷歌提供了丰富的资源宝库，每 6 个月就会推出新的更新和改进。
*   **支持渐进式网络应用** : Angular 是第一个为渐进式网络应用的[开发集成特性的框架。](https://www.lambdatest.com/blog/planning-to-move-your-app-to-a-pwa-all-you-need-to-know/)
*   **长期支持**angular 最大的优势之一是它有谷歌这样的巨头支持，谷歌承诺长期支持 Angular。Igor Minar 和 Steven Fuin 在 ng-conf 2017 主题演讲中重申了他们的承诺。
*   **原生移动应用**:由 angular 提供支持的 NativeScript 支持为 iOS 和 android 开发原生移动应用。

**角度问题**。

*   **语法难** : Angular 在 angular2 及以后采用 TypeScript 后，语法复杂度有了很大的飞跃。然而，对于任何从普通 JS 或 jQuery 转型的人来说，这是一条陡峭的学习曲线。
*   **迁移**:从较旧的 angular 版本迁移，尤其是从 AngularJS 迁移到 angular 7 非常复杂。
*   **陡峭的学习曲线**:在三个顶尖的 javascript 前端框架中，Angular 的学习曲线最陡峭，并且遭受了非常高的放弃率。angular 的大量用户已经转向 Vue 和 react。
*   **MVC 知识**:使用 angular 的一个至关重要的先决条件是需要对模型视图控制器架构有一个透彻的理解。

**知名客户**
Upwork，Freelancer，Udemy，YouTube，Paypal，Nike，Google，Telegram，Weather，iStockphoto，AWS，Crunchbase。

### [](#4-ember)4。余烬

[![](../Images/62ae6a09defda7508fb44437e58cbae4.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--bV_MHDJn--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image19-2-1.png)

2019 年顶级 javascript 框架列表中的第四个框架是 Ember.js. Ember 是一个开源前端 JavaScript 框架，用于构建健壮的 web 应用程序。Ember 最初在 2010 年被称为 SproutCore framework。2011 年 12 月，jQuery 的智囊 Yehuda Katz 推出了 ember。Ember 是一个基于 MVVM(模型-视图-视图模型)模式原理的固执己见的框架。它配备了微光引擎，提高渲染速度。突出的特性包括链接两个属性的数据绑定和调试错误的检查工具。

**使用统计**

*   网络上的 7k 网站由 Ember 提供支持。
*   目前有 6.3%的 javascript 开发者在使用 Ember。它的受欢迎程度在过去几年里停滞不前。
*   在 JS 调查中排名第四的最受欢迎的前端 javascript 框架。
*   github——20k 星，4k 叉，750 贡献者。

[![](../Images/b273e759840035ed1aea42b9fc863f73.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--PistPXOo--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image10-5-1.png)

**为什么 EMBER 是 2019 年最顶尖的 JavaScript 框架之一？**

*   Ember 模板(Ember templates):Ember 最显著的特点之一是模板可以帮助创建令人惊叹的 UI。模板是用手柄语言编码的，这大大减少了必要的代码。一旦对模板的基础数据进行审核，模板就会自动更新。
*   Ember CLI : Ember-CLI 是一个命令行插件，与 Ember stack 打包在一起。Ember-CLI 为 Sass/Less、CoffeeScript、Handlebars 等技术提供支持。
*   Ember 附带了检查工具来帮助调试
*   **约定优于配置** : Ember 基于约定优于配置(CoC)。这有助于开发人员更专注于功能和更快地构建应用程序，更少担心规划高性能

**余烬的问题**

*   **失去市场份额** : Ember 在过去的 5 年里经历了巨大的停滞，未来将会变得黯淡无光。像网飞和 Airbnb 这样的大公司已经抛弃了 ember，转向了 react 这样的其他框架。
*   **缺乏最新的资源**:由于社区不断缩小，Ember 提供的支持非常少，并且受到过时资源的困扰。
*   过于复杂，无法在小型项目中实施。花费太多时间进行配置和使用，降低了工作效率。
*   与 react 和 Vue 相比，ember 非常复杂和难学，尤其是对初学者来说。

**突出用户**
Apple music、Groupon、Vine、Twitch、Chipotle、Atlas.io、sitepoint

### [](#5-polymer)5。聚合物

[![image](../Images/a9be10e385a2e6ef1a48581156624da5.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--rjwZCMeE--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image1-10-1.png) 
我们 2019 年顶级 JavaScript 框架列表中的最后一个前端框架是 Polymer.js，Polymer 由谷歌的一个工程师团队创建，并于 2015 年推出。它是一个开源的 JavaScript 库，用于使用可重用的 web 组件创建跨浏览器兼容的 web 应用程序。所有主要的谷歌服务都是使用聚合物构建的，即 YouTube、Google play 音乐、Google play 电影、Allo、Google I/O 和 Google Earth。Polymer 允许开发人员使用 HTML 和 CSS 创建自己的组件，并使用 JavaScript 添加交互。

**使用统计**

*   互联网上的 4k 网站都是由聚合物驱动的。
*   目前有 3.8%的 javascript 开发者在使用 Polymer。
*   在 JS 调查中排名第六的最受欢迎的前端 javascript 框架。
*   github——20k 星，1.9k 叉，140 贡献者。

[![image](../Images/0020d79fc662cda4ee9224426b913ee5.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--RKUhrkph--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image12-4.png)

**为什么 POLYMER 是 2019 年最顶尖的 JavaScript 框架之一？**

*   Polymer.js 使开发人员能够构建他们自己的定制 HTML 元素。
*   **闪电般的速度**——与其他前端 JavaScript 框架相比，polymer clocks 速度极快，尤其是在 chrome 和 safari 浏览器上。
*   支持单向和双向数据绑定。
*   **跨浏览器兼容性** : polymer 的设计特别考虑了跨浏览器兼容性测试

**聚合物的问题**

*   与同类产品不同，它缺乏服务器端渲染。
*   **资源不丰富**:与 react Vue 和 angular 相比，polymer 在网络上的可用资源很少，社区很小，这阻碍了它的流行。

**大客户**

youtube、play music、Electronic Arts、ING、可口可乐、麦当劳、IBM、通用电气等谷歌应用

### [](#6-express)6。表达

[![](../Images/9685735a3b7815b19d793c707662f784.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--WQ--Cl1w--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image3-5-1.png)

就后端而言，2019 年排名靠前的 JavaScript 框架是 Express.js，Express 是 Node.js 的开源标准框架，Express 用于创建服务器端的 web 应用和 API。它因其简单性、灵活性、可伸缩性和极简主义受到全球 node js 开发人员的好评。Express JS 由 TJ Holowaychuk 于 2010 年 11 月推出，主要与 Angular 或 react 一起使用，并在 MEAN 或 MEARN 堆栈中形成服务器端框架层。

**使用统计**

*   100k 网站由 Express.js 提供支持。
*   64.7%的 javascript 开发者经常使用 Express.js。
*   在 JS 调查中排名第一的最受欢迎的后端 javascript 框架。
*   github——42k 星，7k 叉，240 贡献者。

[![](../Images/b02abf401cd6b70a2c575bd99c8c07d6.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--qfRr3Xtv--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image15-2-1.png)

**为什么 EXPRESS 是 2019 年最顶尖的 JavaScript 框架之一？**

*   广泛支持:如前所述，在所有后端 javascript 框架中，express.js 拥有世界上最大的用户群体。它有大量的资源可以帮助初学者快速入门。
*   **插件**:与被糟糕的插件功能困扰的 meteor.js 不同，Express 支持各种各样的强大插件来协助开发者。
*   健壮(Robust): express . js 不仅是极简和轻量级的，而且不容易出错。
*   **可重用性**:express . js 内置路由器，促进代码可重用性，加快工作流程。

**快递问题**

*   **繁琐的性质** : Express.js 开发人员需要参与大量繁琐的手工任务，这会延长开发周期。
*   **没有内置的错误处理** : Express 缺少一个关键特性，它没有内置的错误处理，这使得调试代码非常耗时，并降低了开发时间。

**大客户**

IBM、优步、Paypal、网飞、Airbnb、沃尔玛、ebay、Linkedin

### [](#7-next)7。然后

2019 年最受欢迎的 JavaScript 框架列表中的第二个后端框架是 js。下一个是服务器端渲染工具，它为 React 赢得了通用后端框架的地位。Next.js 由 Zeit Startup 创建，于 2017-2018 年推出。除了服务器端渲染，Next.js 还提供客户端渲染以及自动代码分离和热代码重载。Next.js 被认为是结束 express.js 主导地位的潜在挑战者。

**使用统计**

*   世界各地的 7k 网站都由 Next.js 提供支持。
*   目前有 8.6%的 javascript 开发者在使用 Next.js。
*   在 JS 调查中排名第二的最受欢迎的后端 javascript 框架。
*   github——33.7k 星，3.7k 叉，604 贡献者。

**为什么 NEXT 是 2019 年的顶级 JavaScript 框架之一？**

*   配备了默认的服务器端呈现。
*   **自动代码拆分** : Next 支持自动代码拆分。这提高了页面重新加载的速度，对更好的 serp 排名非常有益
*   **零点设置** : Next 要求零点设置，开箱即可投入生产。
*   CSS 支持:Next.js 打包了 styled-jsx，它提供了完整的 CSS 支持
*   **支持热代码重载**:只加载那些被修改过的 DOM 元素，而不是整个页面。

**下一个问题**

*   **缺乏支持**:由于 next JS 还不成熟，推出仅一年多，它缺乏更成熟和年长同行享有的支持水平。

### [](#8-meteor)8。流星

[![](../Images/68e08cc74c4b9a17e9443e9f443016ce.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--GxkSi9Ae--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image14-2.png)

Meteor JS 是由 Meteor Development Group 开发的开源后端 JavaScript 框架。Meteor 是用 Node JS 编写的，用于构建 android 和 iOS 的跨平台 web 和移动应用程序。Meteor 于 2012 年 1 月推出，稳定版于 2018 年 5 月推出。最初，流星是在 2011 年以 Skybreakin 的名字发布的。Meteor 配备了 Blaze engine，但也可以与 Angular 和 React 等流行的前端库/框架一起使用。

对于开发人员来说，最大的挫折之一就是需要掌握最新的多种语言来构建简单的应用程序。Meteor 通过将前端和后端封装到一种语言中——普通 JavaScript——消除了这种负担。此外，由于它的模块化库结构，Meteor 很容易设置和使用。通过提供所有这些好处，meteor 是 2019 年的顶级 JavaScript 框架之一。

**使用统计**

*   19k 网站由 Meteor 提供动力。
*   2019 年有 4.5%的 javascript 开发者在使用 Meter。
*   在 JS 调查中排名第四的最受欢迎的后端 javascript 框架。
*   github——40.7k 星，5k 叉，400 贡献者。

[![image](../Images/25f3d13dc76f4cb4f289a0993d2449f0.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Bgq0zUM9--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image18-2.png)

**为什么说 METEOR 是 2019 年最顶尖的 JavaScript 框架之一？**

*   **简单性** : Meteor 将客户端前端和服务器端后端合并成一种语言——纯 JavaScript。这使得 meteor 在所有其他框架中最容易被初学者理解。
*   **丰富的包和库** : meteor 提供了大量的官方社区库和智能包，它们高度可靠，是最新的，可以增强功能并加快开发过程。
*   **实时测试工具** : Meteor 有自己的内置测试工具 velocity，可以与最流行的 JavaScript 框架集成，用于测试，如 Mocha、Jest 和 Jasmine。
*   有帮助的社区 : Meteor 有一个巨大的社区，提供广泛的教程资源和源源不断的文档，帮助初学者轻松采用这个框架。
*   **MongoDB** : Meteor 支持 MongoDB 数据库，比 SQL 之类的传统关系数据库好很多。
*   动态重新加载:Meteor 支持在每次代码修改时动态重新加载页面，并且只重新加载那些被修改的 DOM 元素，而不是整个页面。
*   原生移动应用:使用 Cordova 集成，Meteor 可以为 Android 和 iOS 构建健壮的原生跨平台移动应用。

**流星的问题**

*   品牌化–随着大多数用户转向其他更受欢迎的框架，如前端的 react angular vue js 和后端的 express.js 和 next.js，Meteor 的受欢迎程度迅速下降。
*   与竞争对手相比，Meteor 的集成较差且复杂。不适合新手。
*   仅支持 **MongodB noSQL 数据库**而不是非常流行和广泛使用的关系 SQL 数据库。

**大客户**

马自达、宜家、霍尼韦尔、mtv、efounders、tech stack、hazeorid

### [](#9-jest)9。他来了

[![](../Images/c651e61251f8e5252a85d95f3b28f420.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Kga1JS6T--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image7-5.png)

Jest JS 是脸书在 2017 年创建的开源 JavaScript 测试库。尽管 Jest 主要用于测试 React 和 React 原生应用，但 Jest 仍然是一个通用的测试工具，可以适应任何 JavaScript 库或框架，使其成为 2019 年顶级 JavaScript 框架的一个伟大选择。与 Mocha 不同，Jest 可以通过最少的配置开箱即用，是快照测试的理想选择。Jest 的另一个显著特性是它能够与 Babel 无缝集成，并支持所有 JavaScript 衍生语言，如流行的 TypeScript。

**使用统计**

*   大约 30k-40k 的网站正在使用 Jest 框架进行 javaScript 测试。
*   目前有 39.6%的 javascript 开发人员和测试人员在使用 Jest。
*   在 JS 调查中排名第一的最受欢迎的 javascript 测试框架，紧随其后的是 Mocha。
*   github——23k 星，3.2k 叉，800+贡献者。

[![](../Images/dc7a904116e878e811ba7c0c9ed391cc.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--WVvjatOB--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image16-2.png)

**为什么 JEST 是 2019 年的顶级 JavaScript 框架之一？**

*   **闪电般的速度** : Jest 速度极快。与 Mocha Jest 不同，Jest 运行并行测试来减少测试时间并最大化性能。
*   **不需要配置** : Jest 开箱即用，不需要像 mocha 那样复杂的设置配置。Jest 是在开发人员使用 create-react-app 创建他们的 react 应用程序时配置的。测试可以放在 *test* 文件夹中，也可以用. spec.js/.test.js 扩展名命名
*   嘲弄库–Jest 自带嘲弄库，这是它相对于 mocha 的一个巨大优势。它还有内置的间谍和匹配器。
*   **易用性**:在所有的 JavaScript 测试库和框架中，比如 Mocha 和 Jasmine，Jest 的简单性和易用性仍然是无与伦比的。它是专门为担心 JavaScript 测试的开发人员创建的。

**笑话的问题**

*   **不成熟** : Jest 是最年轻的测试库，与其成熟的竞争对手相比。它可能不适合需要各种测试的大型项目。
*   **支持差**:由于 Jest 还很年轻，相比 Mocha 或 Jasmine，Jest 的社区支持和工具资源相对较少。像 webstorm started 这样的流行 IDE 直到最近才开始支持。

**知名客户**
脸书、Instagram、Twitter、纽约时报、Glympse、Airbnb、Spotify

### [](#10-mocha)10。摩卡

[![](../Images/64fcccae6f460cfde125b890cf12e9ee.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--u2Rwsk9p--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/image4.jpg) 
Mocha 是 2019 年另一个顶级 JavaScript 框架之一。Mocha 是一个基于 Node.js 的非常流行和健壮的 JavaScript 测试框架。Mocha 因其丰富的特性和简单的异步测试能力而广受赞誉。与 Jest 不同，Mocha 是一个成熟的老框架，有可靠的支持和工具。Mocha 还支持各种各样的现代 web 浏览器和 JS 库。然而，Mocha 明显比 Jest 慢，这促使 Airbnb 等大客户放弃 Mocha，转而使用 Jest。

**使用统计**

*   大约 3 万个网站正在使用 Mocha 框架进行 javaScript 测试。
*   39.5%的 javascript 开发人员和测试人员定期使用 Mocha。
*   在 JS 调查中排名第二的最受欢迎的 javascript 测试框架。
*   github——17k 星，2.3k 叉，400+贡献者。![](../Images/cbe012e0821e0fb13ee081b8c2aa02e5.png)

**为什么 MOCHA 是 2019 年的顶级 JavaScript 框架之一？**

*   高灵活性:与 Jest 自带的大量模仿库不同，mocha 允许用户根据自己的需要和偏好自由选择任何库或框架。
*   **异步测试**:与同行不同，Mocha 支持异步测试。在较新的 mocha 更新中，用户可以省略测试完成所需的回调，以加快测试过程。
*   **安装方法** : Mocha 提供多种安装方法，本地或全球安装。它还可以用于在客户端的浏览器中运行测试。
*   **支持多种浏览器**:说到浏览器支持，摩卡依然无与伦比。它还具有运行浏览器特定测试的功能。
*   **与断言库的集成**:通过将 Mocha 与断言库集成，测试人员可以大大加快他们的测试工作流程。Chai 是目前最流行的断言库。

**摩卡的问题**

*   **广泛的配置**:Jest 崛起并超越 mocha 成为最流行的测试框架的最大原因是因为 Mocha 需要设置广泛的配置。与 Jest 不同，它没有自带的模仿库，因此不能开箱即用。快照测试:使用截图测试功能需要集成一个名为 chai-jest-snapshot 的独立库。
*   **快照测试**:为了使用截图测试功能，需要集成一个名为 chai-jest-snapshot 的独立库。

Asana、coursera、Typeform、selluki、Droplr、Onedio

### [](#11-jasmine)11。茉莉

[![](../Images/6ef78ab61d6a7d02884b3dc99d125f90.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--FD0Zw-nc--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/pasted-image-0-30.png)
2019 年 11 大 JavaScript 框架的最后一名是 Jasmine。Jasmine 是一个开源的 javascript 测试框架，它没有依赖性，可以用来测试任何种类的 JavaScript 代码。jasmine(JsUnit 的继任者)由 Pivotal Labs 构建，于 2010 年推出，受 RSpec 影响很大。因为它是基于行为驱动开发(BDD)的，Jasmine 不把整个应用程序作为一个整体来测试，而是把测试分解成小的单元或语句。与 Mocha 不同，Jasmine 配备了自己的断言库，还提供了实现测试替身的“间谍”。

**使用统计**

*   网络上大约有 20k-25k 个网站正在使用 Jasmine 框架进行 javaScript 测试。
*   28.5%的 javascript 开发人员和测试人员在使用 Jasmine。
*   在 JS 调查中排名第三的最受欢迎的 javascript 测试框架。
*   github——14k 星，2k 叉，180+贡献者。

[![](../Images/cab1c2f452fce9c353b5c0674788132b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--oaVrpKAL--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn.shortpixel.ai/client/q_glossy%2Cret_img%2Cw_800%2Ch_180/https://www.lambdatest.com/blog/wp-content/uploads/2019/01/pasted-image-0-31.png)

**为什么 JASMINE 是 2019 年的顶级 JavaScript 框架之一？**

*   Jasmine 不依赖于任何其他框架或库——它没有依赖性。

*   不需要任何 DOM 来运行测试。Jasmine 复制了它需要运行测试来检查 JavaScript 代码的 DOM 部分。因此，它能够在几秒钟内运行数百个测试，是最快的 javascript 测试框架之一，在某些情况下甚至超过了 Jest。

*   易于阅读的描述性语法和低复杂性。

*   Jasmine 仍然是在 AngularJS 中运行测试的最受欢迎的选择。

*   可以用最少的配置进行开箱即用的部署，并且有自己的断言库，可以很容易地进行扩展。

成长黑客、Coderus、Ubiqua、Infoshare
