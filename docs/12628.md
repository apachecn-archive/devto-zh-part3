# 🇹🇼✈️带戴夫和我去台湾，✈️🇹🇼

> 原文:[https://dev.to/xenoxdev/-taking-dev-to-taiwan-with-me-2c 77](https://dev.to/xenoxdev/-taking-dev-to-taiwan-with-me--2c77)

这个月我一直在台湾旅行，探索这个美丽的国家，吃了大量的海鲜，并结交了新朋友。所以在我踏上旅程之前，我计划在这里参加一些聚会。虽然这里没有太多的活动，但我发现了一个名为 [Wizard Amigos](https://www.meetup.com/WizardAmigos/) 的团体，这是一个由个人组成的游牧社区，帮助新开发人员学习 Javascript。

[![](../Images/a1247920d2da2d72b483d65e87789027.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ebsge9Jm--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://scontent.ftpe7-3.fna.fbcdn.net/v/t1.0-9/17499395_10155580147441416_24121990061902149_n.jpg%3F_nc_cat%3D103%26_nc_ht%3Dscontent.ftpe7-3.fna%26oh%3D2a2b93d3664377bc6662eeae553a0546%26oe%3D5CEBA39F)

他们的任务立刻引起了我的共鸣。所以我参加了他们的两次聚会，希望能帮助塑造一些年轻人的思想。我指导他们如何在 Electron 中制作桌面应用程序，以及如何在 node.js 中制作 CLI 应用程序。GitHub 上的 CLI 应用程序 **devtocli** 。

巧合的是，大约在同一时间，本被邀请作为嘉宾参加产品搜索电台。在播客中，他谈到了 Dev.to 是如何出现的，以及这个平台背后的核心思想是什么。他还谈到了以身作则([，这是我从一开始就钦佩的](https://dev.to/sarthology/why-devto-is-winning-over-hashnode-27nj))，他对开源的热爱，以及如何管理远程团队，还有其他有趣的东西。我强烈推荐你们去看这一集。

听播客让我想起为什么我离开了 [Medium](http://medium.com/@Sarthaksharma0) ，甚至有一段时间是生产力的顶级作者，而开始写 Dev.to。这是因为 Dev.to 是一个纯粹意义上的社区。人们*倾听*你，*回应*你，而且*准备好*帮助你。

你如何让这样一个强大的社区变得更加强大？当然是通过加入更多志同道合的人！这正是我在巫师朋友聚会上所做的。要了解他们的组织在做什么，你可以查看他们的[网站](http://wizardamigos.com)或者只查看下面的这张截图。

[![](../Images/da9e6101cc2f48514028aafd076296fd.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--LDEiWSRB--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://i.imgur.com/B2WX8wC.png)

总之，我邀请他们所有人加入 Dev.to。我认为他们的使命会引起我们这里所有人的强烈共鸣，因为他们试图帮助初学者开发者，这就是 Dev.to 的目的，对吗？向他们展示这个平台真是太棒了，我可以从他们的眼神中看出他们有多兴奋！哦，最棒的部分是告诉他们关于 Dev.to 的联合创始人[杰斯](https://dev.to/jess)和*台湾人*——美国软件开发商。他们非常自豪地发现了这一点。(相信我，杰西，他们真的是。)
