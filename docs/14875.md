# 一个日记平台的日记- 1:追随你的梦想

> 原文：<https://dev.to/simoroshka/diaries-of-a-diary-platform---1-follow-your-dreams-1b1m>

# [T1】简介](#intro)

在过去的一年半时间里，我一直在开发一个在线日记平台——非常私人的博客。这个系列将是这个旅程中非常个人化的故事。

# 一场梦

2017 年 9 月，我做了一个梦。我度过半生的在线日记平台永远地倒下了。我不停地刷新页面，却一遍又一遍地看到“此网站无法访问”的信息。多年来，我一直在记录思想、情感、梦想、抱负、失败、爱情故事，几乎所有的东西，现在它不见了。我在那里交到了非常棒的朋友，现在我无法联系他们中的一半。在社交网络存在之前，我们就已经是一个社交网络了。多年来，它是整个互联网上最安全、最舒适的地方。现在一切都没了，噗。

我醒来时，一切都还在线，还在工作。同一个旧网站，看起来和十年前一模一样。但是这个梦让我想知道这个可爱的地方会持续多久。我给它 2-4 年前，业主将失去任何剩余的一点兴趣。

# 社会支持的力量

当然，我写了一篇关于此事的悲伤、沉思的帖子。我在思考为什么没有真正的替代品:如今你可以在社交网络、专业平台(如 dev.to)和独立博客之间做出选择。几乎没有任何地方鼓励以任何形式写任何随机的东西，让其他随机的人找到它并开始真诚的对话，这往往可以导致友谊。日记，不是博客。网上，不是私人的，至少不完全是。思想、表达、身份的自由，以及免受“智能”算法和监视的自由。美好的旧时光。

我表达了一个希望，希望有一天有人可以再做一个“像这样的”，然后我们一起搬到那里。“也许甚至会是我，更有经验和知识”。

我的朋友们忽略了最后一点，我们开始讨论如何不仅仅是一样而是更好，我们需要保留什么，我们应该放弃什么，以及添加或改变什么真的很好。受到启发，我说:“好的，我会试着玩这个，应该会很好玩”。也没那么难，对吧？..

# 无知的力量

当时，我在小型团队和项目中有大约 2 年的随机 web 开发经验，在一个大项目中有 4 个月的 React 工作经验。“我知道 Node 和 React，我有几个关于从头开始构建应用程序的在线课程，这应该足够了”。我在谷歌上搜索了 SPA、PWA、SSR、JWT 和许多其他聪明的缩写和单词。我花了几周时间试图理解如何构建最好的项目配置，以及使用什么框架。我无法决定是应该使用 NoSQL 还是回到关系数据库。认证似乎很难做到正确。我的脑海里没有任何清晰的画面，只有一些片断，和许多许多的想法。

然而，感觉好像我就快到了，我只需要通过一些瓶颈，之后就会变得很容易。也许再过一个月。那些在线课程在十几个小时内完成，对吗？

我什么都不知道，我甚至不知道。另一方面，如果我知道这段旅程会有多漫长和艰难，我就不会迈出这第一步。

# 梦想成真

而且不是好的方面。在我做了那个梦的两个月后，网站的所有者宣布这将是，现在被戏剧性地称为，“最后的秋天”。许多人绝望、恐慌，寻找可能的替代方案(记住，没有任何替代方案)。我振作起来，发了一篇关于我的努力和未来计划的帖子。它像病毒一样传播开来，新的篇章开始了。
