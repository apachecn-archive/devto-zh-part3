DevOps 目前非常热门，我认识的大多数朋友、同事和高级开发人员都在努力成为一名 DevOps 工程师，并将自己标榜为他们组织中的 DevOps 冠军。

虽然我确实承认 [DevOps](https://javarevisited.blogspot.com/2018/09/10-devops-courses-for-experienced-java-developers.html) 的好处，它与改进的软件开发和部署直接相关，但根据我有限的经验，我可以说**这不是一件容易的工作**。在这么多的工具和实践中选择正确的道路是非常困难的。

作为一名 Java 博客作者，我的许多读者经常问我一些问题，比如如何成为一名 DevOps 工程师，我应该学习哪些工具？我应该遵循哪些实践？对于一个 DevOps 的人来说，学习 Maven 和 Jenkins 是必须的吗？Docker 和 Kubernetes 怎么样？基础设施自动化是 DevOps 的一部分吗？我应该学习[厨师](https://click.linksynergy.com/fs-bin/click?id=JVFxdTr9V80&subid=0&offerid=323058.1&type=10&tmpid=14538&RD_PARM1=https%3A%2F%2Fwww.udemy.com%2Fchef-fundamentals-a-recipe-for-automating-infrastructure%2F)、[木偶](http://bit.ly/2yc6O4D)还是[可回答的](http://bit.ly/2ygZXa3)只是我不断想到的一些问题。

我努力尝试用我最少的经验来回答这些问题，但是我不能用一种简单的令人敬畏的和可重复使用的方式记下它们，但是，不要担心。

今天，我将与您分享一个非常棒的资源，它将帮助您成为您一直想成为的 DevOps 工程师，即 **2022 DevOps 工程师路线图**。

昨天我在网上漫不经心地浏览时，看到了 [Kamranahmedse](https://github.com/kamranahmedse) 的[这个](https://github.com/kamranahmedse/developer-roadmap)优秀的 GitHub 页面，它展示了成为前端开发人员、后端开发人员、全栈 web 开发人员以及最后但同样重要的**devo PS 工程师**的一些有用的路线图。

这个路线图在任何意义上都是令人敬畏的，因为它不仅强调了 [DevOps 工程师](https://javarevisited.blogspot.com/2018/09/10-devops-courses-for-experienced-java-developers.html)的角色，还告诉了您需要学习哪些工具和技术来覆盖该领域。

最重要的是，漂亮的颜色在视觉上很有吸引力(你不喜欢黄色和带有蓝色线条的奶油色吗？)，这样你就可以拿一份打印出来的东西贴在办公桌上，方便参考。

虽然路线图在某种意义上是好的，它告诉你要学什么，但它没有告诉你如何学习和在哪里学习。为了完成路线图，我分享了一些有用的在线课程，有免费的也有付费的，这样你就可以学习和改进你想要的工具或领域。顺便说一句，如果你买了我在帖子上推荐的课程或书籍，我可能会得到补偿，这取决于它是免费的还是付费的。

### [](#3-learn-to-live-in-terminal)**3。学会在航站楼生活**

### [](#4-networking-and-security)**4。网络和安全**

为了成为一名优秀的 DevOps 工程师，你必须了解基本的网络和安全概念，如 DNS、OSI 模型、 [HTTP](https://javarevisited.blogspot.com/2013/07/how-ssl-https-and-certificates-works-in-java-web-application.html) 、HTTPS、FTP、SSL、TLS 等。

### [](#8-learn-to-monitor-software-and-infrastructure)**8。学习监控软件和基础设施**

除了设置和部署之外，监控是 DevOps 的另一个重要方面，这也是 DevOps 工程师了解基础设施和应用监控非常重要的原因。
