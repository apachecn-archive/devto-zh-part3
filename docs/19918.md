# 偏爱工具是糟糕的工程学

> 原文:[https://dev . to/samjarman/favoring-tools-is-bad-engineering-1 jom](https://dev.to/samjarman/favouring-tools-is-bad-engineering-1jom)

今天我想谈谈糟糕的工程师，特别是他们的一个方面——解决方案偏好。但是首先，让我再次提醒你工程学是如何工作的。

**你遇到了问题**——这可能是任何事情——你可能需要一个新的数据库，一个新的前端框架，一个新的包管理器，或者甚至更抽象，你可能需要一种新的方式在方便的时候向用户显示数据。见鬼，它甚至可能不是软件，你需要水从水龙头里出来，你需要一个新的淋浴设备，你需要一座跨河的桥，你需要为一个偏远地区提供电力。

然后，你**深入这个问题。**你看子问题，*对这个领域的独特约束*，你的公司，你的人和团队，你的未来计划，你过去的计划，等等。这些都很重要。

从那里，你尝试一些解决方案。通过小型概念验证(POC)或实验，您依次尝试了您能想到的每种解决方案，最终您会看到最有效的解决方案。您可能会排除一些，因为它们不适合您的独特约束。你可能会进一步测试其他人。这是一些解决方案的知识可以派上用场的地方，稍后会有更多的介绍。

从那里，你解决关系。如果一个解决方案看起来和另一个几乎一样，你就继续下去。找到其他独特的约束，直到一个明确的赢家出现。从那里，您可以继续实施解决方案。如果您发现问题，那么您的 POC 不够好，您需要后退一步。它发生了。挺好的。

就是这样，这是一个优秀工程师解决问题的简单方法。

现在，我们来谈谈什么是糟糕的工程。

糟糕的工程进入了问题解决阶段，偏向于你最喜欢的潜在解决方案。这是绝对荒谬的，但它一直在发生，尤其是在软件工程领域，尤其是在网上。这在网上变得太普遍了，看了超级超级难过。让我列出一些我认为是糟糕工程的标志的短语:

*   “我们现在只使用 Swift”

*   “我更喜欢椰子而不是迦太基”

*   “你不用打字稿！LOL？？？?"

*   " VS 代码是最好的文本编辑器！"

*   “哦，是优步的”

*   “您应该在您的下一个项目中使用架构模式”

*   “只有当你的堆栈中有卡夫卡时，你才能扩展”

*   “我们如何在这方面使用人工智能？”

*   “它是 X，但是使用了区块链”

*   “Lol 你用的不是 ES2018？”

*   “哈！你还用 PHP？!"

我想比这更进一步。我想说**甚至*像*这样的技术或工具都是不好的工程。**如果你喜欢它，在解决问题时你会对它有偏见，并且你会因此成为一个糟糕的问题解决者(又名工程师)。当然，我发现 Swifts 语法*令人愉快*，或者*可读性*，或者*可维护性*。但最终，当它是解决问题的最佳方式时，我会使用它。在 iOS 应用的背景下，如果有适当的限制，Swift 可能会输给其他可能的解决方案，从任何东西到 Objective-C、React Native、移动网站、地图上更好的谷歌列表，到在当地投放一个简单的传单。

我们是工程师。我们解决问题。我们不会带着一箱解决方案走来走去，然后把它们扭曲成我们遇到的问题。我们着眼于手头问题的独特约束，并从那里出发。

“但是 Sam，我擅长 X，这就是我喜欢它的原因，这样我可以更快地做事情，企业也因此喜欢我”。那很好。擅长某个特定的工具或技术是可以的。但是，只有当约束正确时，这才是一个好的解决方案。可能倾向于这些情况的一些(*常见*而非*常量*)约束是:

*   你已经有了一个 X 代码库，所以你添加了更多的 X 代码库

*   交付日期临近和/或成本必须保持在较低水平——所以在寻找解决方案时，你要强调你/你的团队当前的技能。作为一名优秀的工程师，你会警告管理层不按正确的方式做事的长期风险。参见:*创业公司*。

*   该项目是短期的，如技术演示，临时使用或类似。

所以，总的来说，有几个要点我想讲清楚。

1.  先说问题。努力消除你的偏见，用良好的科学和工程原则进行公正的评估。

2.  排除实用主义的东西，但注意那些偏见。但是，没必要用 Cobol 写前端框架。

3.  留意博客和在线内容销售解决方案，作为对你独特问题的解决方案。它*可能是*一个解决方案，但我可以保证它不是*所有人*的解决方案。Dev 上的一些人。对此感到内疚！我自己包括在过去。

4.  深入研究并掌握一项技术是没问题的，但最好的工程师或开发人员在他们的头衔中没有工具名称。高级 Java 开发人员在我听来就像高级 PVC 管道工一样可笑。最好从问题入手，找到解决问题的最佳方案。

5.  一些专注于某个工具的博客和在线内容通常是为了让你更熟练地使用它们，但要警惕那些你现在知道是智力不诚实和糟糕的工程的断言。

祝你好运！如果我们不断地审视问题并提出最佳解决方案，未来将会是令人惊叹的。如果我们继续使用我们现在拥有的东西，我们将会停滞不前。我们不希望这样。
