# 让我们飞向 Google I/O Pt。3 -主题演讲和新朋友

> 原文：<https://dev.to/bennypowers/let-s-fly-to-google-i-o-pt-3-keynotes-and-new-friends-1aoo>

我不想报道我参加的活动的技术内容(你可以现场直播或观看录像)，我更想在 dev.to 上与大家分享我作为与会者的经验，所以下面是一个 I/O 新手的一些零散笔记。

在等待主题演讲开始的时候，人群中每隔几分钟就会爆发出一阵欢呼声，我会伸长脖子去看 12 个巴西人、15 个印度人、8 个伊拉克人或 4 个新西兰人骄傲地挥舞着他们的国旗。会议上的种族和民族多样性令人振奋。我为代表以色列和犹太人民感到自豪。如果我再来，也许我还会带一面旗子。

参加会议的极少数人戴着可笑的帽子。他们做得对。我为他们人生选择的智慧而折服。

安 AI DJ 和他的人类搭档 DJ Nao 开启了主题演讲。人类更好。喜欢人工智能闪光灯上的塑料斗篷。视觉效果太棒了。总而言之，非常有价值。

有人租了一架飞机飞过主题演讲，横幅上写着“谷歌控制不是隐私# SAVELOCALNEWS”。那是一些高质量的阴影，就在那里。

## 关于主题演讲的想法

在 TCAV 的演讲中看到一些关于解决 ML 模型中偏倚的工作真的很有趣。隐私自动删除功能太棒了。“删除”实际上是指“删除”吗？。在搜索和地图中隐姓埋名也很神奇，但另一方面，gboard 上的联邦学习让我的蜘蛛感官兴奋不已。不过，也许这条船已经起航了。

我真的很喜欢关于 web 平台特性的新闻，比如图像延迟加载、更快的 v8 解析等等。这方面的大新闻是搜索引擎中的常青铬。这将导致在生产中更多地采用 web 标准。对我来说，另一个亮点是 CI 中的灯塔绩效预算。我心中有几个项目可以使用 Firebase 性能监控和自动化灯塔审计。

展示的新网络平台功能看起来真的很棒。在 paint holding、门户和 web 包之间，将会出现一些真正引人注目的用例。有一天，Google 否决了一整类 SSR 用例，但随后又推出了一个全新的用例。

我有很多关于传送门的问题。当孩子接管时，Javascript 上下文会发生什么变化？你有新的`document`和`window`参考吗？XSS 和 XSRF 的含义是什么？你能控制动画吗？这里有很多我想吃的东西。

P2P 上的 Web 打包？是的，请！我还没有深入研究这个特性。我想知道其他供应商的实现情况如何。

Paul Lewis 关于 [Web Perception Toolkit](//perceptiontoolkit.dev) 的演讲非常精彩。我对条形码扫描功能有很多想法，我喜欢作者可以用结构化数据标记来定义它的用途。还有一个 WASM 多菲勒！

从地址栏安装 PWA 很酷。我超级热衷于卸载 Slack.app，改用 PWA。

网络上的双工看起来很酷。我想知道它是否能突破暗影 DOM 的界限。

## 新朋友

在去音乐节的路上，我非常担心在一个有成千上万人参加的活动中遇到新的人。然而，在那一刻，只需要一点努力和友好的微笑就能让对话进行下去，我在第一天就遇到了一群非常有趣的人。

Lars Knudsen 是 Polymer slack 社区的成员，和我一样，他也是网络标准的积极倡导者。在开发者大会上，我有幸坐在他旁边。谈论 Chrome 的新热点很有意思。

演讲结束后，我和 Adam Argyle 逛了逛，他在 Google 做 CSS 开发。我昨天发现了我的一个新朋友，我们就框架变动、工程标准和网络工作的不确定性进行了一次闲聊。

## 分组会议

之后，我参加了两个会议:一个叫做“从头开始现场编码 ML 模型”,另一个是关于 UX 的声音设计。ML 的演讲对我来说有点牵强，但我有足够的 ML 背景，所以我没有完全迷失。我的主要收获之一是，谷歌提供了具有不同抽象级别的 API 和产品，因此对于许多用例来说，没有必要自己设置和训练模型，而是使用更抽象的工具来完成工作。我一直在与我的姐姐 Zeesy 合作，她是一位住在多伦多市区的艺术家，从事一些围绕人工智能的项目，我认为这些工具可能会为他们提供很多价值。

在康纳·奥沙利文在 UX 做[声音设计的演讲](//material.io/guidelines/sound)时，我对[路缘切割效应](https://medium.com/@mosaicofminds/the-curb-cut-effect-how-making-public-spaces-accessible-to-people-with-disabilities-helps-everyone-d69f24c58785)有了深入的了解。尽管我身体健全，英语是我的母语，但我仍然使用了会议的实时字幕屏幕，它们帮助我理解了演示。太多时候，我们认为 a11y 是绝对的，但事实是每个人都有 a11y 问题，这些问题在一个人的一生中甚至在一天中不断变化。这就是说，在可访问性方面的任何投资都是值得的。
