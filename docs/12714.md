# #100DaysOfCode -中途

> 原文:[https://dev.to/seanmfox/100daysofcode-半途而废-7ke](https://dev.to/seanmfox/100daysofcode---half-way-there-7ke)

在一个半空/半满的世界里，我倾向于后者，并倾向于乐观。正因为如此，当我考虑到我已经连续写了 50 天的代码时，我的感觉更像是“哇，这太快了！”，而不是“等等，我还有 50 多天呢？!"我将把这归因于我如何面对挑战。自从上次报告我的[进度](https://dev.to/seanmfox/100daysofcode---first-quarter-finished-1g1)以来，这里是我从 100 天的代码中收集到的一些教训。

### [](#balance-of-projects-and-learning)平衡项目和学习

在过去的几个月里，我一直在全力以赴地创建项目，以展示我的求职作品集。即使当我开始进入#100DaysOfCode 时，我仍然在迭代这些应用程序。在过去的几周里，我变得更有意识地不仅仅停留在功能爬行的跑步机上，而是花一些时间去学习。当 React 挂钩发布时，我决定花一些时间来弄清楚它们。我还参加了 CS50 项目，只是为了扩大我的知识面，让我的大脑在计算机科学的基础上做一些努力。这种日常习惯的改变对我真的很有好处。到目前为止，我的一部分感觉我需要不断地从事我的兼职项目。现在，我可以暂时离开他们，以便成长为一名更全面的开发人员。

### 没关系，有些日子就像水中慢跑

几周前，我开始做一些水中慢跑。顾名思义，它基本上是在一个池中运行的。不过，我很快就发现，不管你踩得多用力，你都不会很快到达任何地方。正是在其中的一次会议上，我与编码产生了联系。在一些晚上结束的时候，我不一定觉得我在我的编码之旅中取得了重大进展。尤其是当一个晚上不可避免地要致力于解决一个由人为错误(也就是我)导致的错误时。这有点像浪费时间，因为没有真正穿越“距离”。在需要把每天我所完成的事情用语言表达出来的时候，我意识到即使那些“无用”的时间也会结出一些果实。即使感觉进展缓慢，也有成长。

### [](#inspiration-from-others)来自他人的灵感

我越是了解别人为#100DaysOfCode 所做的事情，我就越能从开发和工程的大计划中学到新的东西。我甚至开始列出我听说过并想探索的技术。我不一定打算掌握它们，但随着对它们的熟悉，我可以与更多的人对话。有如此多令人惊奇的事情可以改善我们的世界，我想看看有什么工具可以让这一切成为可能。

还有 50 天，之后还有继续连胜的可能，我谦卑地相信我的努力会成功。当我达到 75 天时，如果一切按计划进行，当第 100 天时，我一定会发布更新。
