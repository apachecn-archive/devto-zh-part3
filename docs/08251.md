# 开始前的一些笔记

同样，这些阶段不一定与职位相对应。它们是对开发人员的技能、知识和整体能力的衡量。

它们确实会根据你所处的领域而变化——网络、IOT、数据分析等等。

虽然具体的工具、技术甚至架构模式可能有所不同，但一般原则都适用于广泛的领域。

# 中级开发人员的第一指标

我想从一个更具反思性的指标开始，它可以帮助你衡量你是否接近或处于职业生涯的这个阶段。

这绝不是一个“确定”的指标。但我认为这是大多数中级开发人员都经历过并将会经历的。

我怀疑你们中大多数更有经验的人会同意😊。

因此...这是什么？

你好奇吗？

在这里！

中级开发人员通常有一种心理冲动或“冲动”,认为他们实际上知道了他们需要知道的一切。

也许他们觉得他们关于如何最好地设计一个系统的观点是“完整的”

也许他们觉得 Javascript 是他们唯一需要和想要的编程语言。

不管是什么，总会有某种程度的感觉，好像你已经“找到”了银弹。你现在的观点终于是“唯一的”

# 不好吗？

不，这不是不好的感觉。这并不意味着他们傲慢或卑鄙。

它就这样发生了🤷‍♂️.

这是你作为开发人员成长旅程的一部分。

# 为你的事业进入青春期

用非常容易理解的术语来说:

中级阶段对你的职业生涯来说就像步入青春期。

青少年倾向于开始相信他们比父母懂得更多。

他们倾向于实验，并对自己的信仰保持大胆。

他们开始走自己的路。

这就是中级开发者。他/她开始独立思考。

这太棒了！这意味着他们已经积累了足够的知识和经验，可以“将点连接起来”，思考设计模式、最佳实践等问题。以独特和创造性的方式。

可以说，他们开始“打破规则”。

就像爵士音乐“打破了”传统音乐理论的规则一样，中级开发人员也在玩跳出框框的思维。

# 一些特质

让我们看看一个中级开发人员如何将初级开发人员的技能和知识提升到一个新的水平。

当然，中级开发人员需要学习一些“新”的东西，并变得有经验！

*   他们现在已经很好地掌握了如何将不同的系统(API、模块、包等)连接在一起。)
*   他们应该很好地理解如何实现一个可行的系统
    *   对于 web 开发人员来说，这可能包括以下内容:
        *   MVC，MVVM 等等。
        *   重构
        *   OOP 原则
        *   其他范例，如函数式编程
        *   应用程序部署
        *   能够拥有自己的功能
        *   开始不时地与客户沟通
    *   对于数据库开发人员，它可能包括:
        *   分身术
        *   索引
        *   分片
        *   性能调整
        *   高级查询(cte、窗口功能、应用等。)
        *   其他范例如 NoSQL、基于键/值的数据库、分布式数据库等。
*   被视为对团队有价值的贡献者
*   精通工具(ide，其他开发工具等)。)
*   开始习惯性地使用:
    *   设计模式
    *   代码气味💩和重构
*   等等。

# 一些建议

好吧-有趣的部分！

你想成为中级开发人员吗？或者你觉得你可能已经在这个阶段的某个地方了？

如果你正处于职业生涯的这个阶段，这里有一些我认为非常有用的建议:

## 获得高知名度的项目

这不是为了让你出名。是这样的:

1.  你可以通过参与一个重要或高价值的项目快速获得经验
2.  你可以在你的组织中成为一个把事情做好的人

如果你能积累经验，并在你的组织中为人所知(这不是名气，而是知名度)，那么当新的机会出现时，你就会开始受到吸引。

## 跳上坚硬的东西

类似于上一点，除非你的舒适区被推开，否则你是学不会的。

而且，你会被视为攻击公司或社区重要事物的人。

这将建立信任、声誉和信心。

## 找到导师

这一点怎么说都不为过。你一个人做不到。

你需要有人给你出主意，并能向你学习。尤其是在这个阶段。

你需要意识到，你容易出现本文开头提到的“青春期”特质。

导师可以帮助你确保某些想法或实验不会走得太远，如果需要的话，还可以把你拉回来。

这可能是你团队中的某个人。

可能是社区里的某个人。

它没有**有**成为一个明确的契约。但是，它可以！

你只需要找到一个你信任的人(或者更可能是一些人),你可以向他提出你的问题和想法！

此外，向这个人寻求资源、建议、项目，甚至是帮助他们获得成功的故事。
