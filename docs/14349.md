# #100DaysOfCode -第一季度结束

> 原文:[https://dev.to/seanmfox/100daysofcode-第一季度结束-1g1](https://dev.to/seanmfox/100daysofcode---first-quarter-finished-1g1)

1 月 1 日。这是一个新的开始、新的冒险和新的一年带给我们所有人的决心的时刻。随着 2019 年初的临近，我知道我想在今年尝试一些不同的东西。我花了 2018 年的大部分时间在场外观察开发社区，犹豫是否要加入#CodeNewbie、# DevDiscuss 或其他各种各样的标签。我要把这归因于我的内向，加上一点冒充者综合症。因此，为了打破这种现状，我说服自己是时候通过参与 Twitter 上的#100DaysOfCode 来加强我的社交游戏了。

如果你不熟悉这个概念，规则是非常简单的——一百天一个小时的代码，并在推特上发布当天取得的进展。这种参加某种连续比赛的做法对我来说并不陌生。我曾尝试过几次跑步，但通常都放弃了，因为我的身体对无尽的英里感到厌恶，而不是缺乏纪律性。我对完成这项任务最大的担忧是生活会发生——有时候，找一个小时坐下来做些有成效的事情是不可行的。我知道在某个时候，我需要接受这个现实，但我想当我到达那里时，我会跨过那座桥。

现在一月即将结束，我已经过了 25 天了——哇哦！到目前为止，这是一次很有启发性的经历。为了有助于总结，在结束这个项目的第一个季度后，这里有一些想法。

首先，我现在有一个代码日志...算是吧！回顾我的推文，我可以看到过去我花了很多时间在做什么。由于不是写日记的类型，每天发短信对我来说是可行的。它不必过于冗长或过于深入。接吻有时是最明智的方式。

第二，我不得不在每天结束时问自己，“自我，你今天完成了什么？”有时候，这个答案真的很容易确定。对其他人来说，更多的是接受我花了一整个晚上的时间寻找错误信息的解决方案。这两种类型的日子对开发人员来说都是常态。

第三，体会到了 CodePen 和免费代码营的美好。在我不得不在学校呆到很晚的日子里，能够在编辑器上跳一会儿并完成 JavaScript 审查是无价的。虽然不是理想的设置，但它能满足我的需要。

但是，尽管#100DaysOfCode 有这么多好处，还是有一两个领域我还在努力。其中一个方法就是在睡觉前记得发微博。当我结束晚上的代码时，已经是我打算停止的时候了。那时我所能想到的就是尽我所能睡觉。结果，我的大部分推文第二天就发出去了——哎呀！

另一个事实是，在不久的将来，我将再次投入马拉松训练(该死的伤病！).在我的家庭是我的第一要务的情况下，我需要在跑英里和跑代码之间分配的有限时间将变得更具挑战性。虽然不太理想，但是将我一天中需要的时间分段可能是我最好的选择。

总的来说，我对这一百天的冒险相当满意。我真的很期待在同一旅程中与其他开发者建立新的联系。当我完成一半的时候，我一定会再给你一次更新——那只有几个星期了！

编码快乐！
