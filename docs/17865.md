# 无头 CMS 是结构化 CMS

> 原文：<https://dev.to/metamn/headless-cms-is-structured-cms-49bn>

我们都知道内容管理系统的未来是无头的:内容是在某个专门的系统中创建的，该系统不一定负责将其交付给最终用户。

到目前为止，这两个方面是捆绑在一起的。您已经在 WordPress 仪表盘中创建了一篇文章，并使用 WordPress 主题再次显示给您的用户。

将来，你可以在任何地方的任何系统中创建内容，并且仍然能够在你的网站上显示，就像它是在你自己的网站后端创建的一样。这就是所谓的无头语义分布式网络。我们都需要光明的未来。

为什么会有这种变化？

网络的本质是 napstering:通过消除中间商来分解铁板一块的垄断结构。新出生的蛋鸡将形成一个新的市场，新的领导者，稍后将再次 napstered。有人可能会说，熵定律。

WordPress 成为了一个庞然大物和垄断者，因为它控制了服务器、数据、内容和表示层。它被许多新市场如微服务而不是服务器抢占了；联合数据，而不是一个单一的中央数据库；更好的视觉内容设计者和编辑者，如 Wix、Contentful 和 co；React 和其他组件框架，而不是前端的主题。

在 Smashing Magazine 上阅读 Knut Melvæ的《结构化内容管理系统无头项目的战略》是一个启示，Knut Melvæ是这一领域最有趣的新参与者之一 Sanity.io 的幕后人
