# 学习。去吧。教书。

> 原文:[https://dev.to/lssweatherhead/learn-do-teach-3225](https://dev.to/lssweatherhead/learn-do-teach-3225)

2015 年 2 月，细雨蒙蒙，我和一群苏格兰女人在旧金山参观一些大型科技巨头，如 Pinterest、脸书和谷歌。在那里，我们被邀请去观看视频。

对于那些不知道的人来说，IDEO 是一家位于帕洛阿尔托(以及其他地方)的数字产品设计公司，他们是以人为中心的设计的鉴赏家。他们的意识形态的一些例子包括:

*   雇佣一位 96 岁的老太太来设计和制造一系列产品来养老。
*   拥有一个完整的[游戏实验室](http://www.ideotoylab.com/)会让你内心的孩子想要在豆袋上睡觉，玩魔法。
*   建设梦幻般的项目，如时髦的停车位分配聊天机器人。

在我们的访问中，真正让我印象深刻的一件事是他们对员工发展的口头禅:**学-做-教**。从那以后，我一直把它作为自己职业生涯的路标。

其核心思想很简单，他们问:

> 对于你所拥有的每一项技能，它在“学-做-教”的尺度上处于什么位置？

我知道，当我有未完成的学习来补充我已经积极涉猎的东西时，我是最快乐的，但这因人而异。有一些范式(比如 C#。NET)，我永远不会觉得我完全“知道”，但总是工作，并有能力教学方面的...！它总是视情况而定，并且总是一个流动的模型——作为一名开发人员的乐趣在于你永远不能真正停止学习！<3

随着 2018 年的临近，现在可能是整理你自己的学习或教导的好时机。你的下一步是什么？
