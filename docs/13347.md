## [软件工程的冲击](https://jaironlanda.com/blog/Blog%20Post/what-is-software-engineering)

在信息技术的世界里；软件是基于需求和技术升级自发进化的最普遍的实体。然而，并不是所有的软件都面向目标受众；有些是不完整的，令人不快的，过时的，维护不善的，等等。幸运的是，其他一些人把很大一部分财富拉给了他们的供应商(问问比尔·盖茨)。如果每个软件都被指派了一个好的软件工程师，那么失败的案例将会在图表的底部摇摆，因为工程师将会确保以下的事情；

1.  客观性:只有优秀的软件工程师才能消除客户提出的软件需求中的任何模糊之处，并提出有效且经过验证的交付。
2.  可靠性:软件工程师最擅长任务关键型系统，在这类系统中，运行期间不允许出现故障。因此，他们确保他们构建的系统是健壮、可靠、安全和智能的。空间技术和金融管理系统是优秀软件工程的少数受益者。
3.  效率:他们确保软件是经济的和友好的。从而用最少的资源提供最好的系统。这需要合理的分析来实现…感谢软件工程师。
4.  可用性:这也包括确保软件对目标受众是用户友好和有效的。能够迅速满足项目目标可以将软件的状态从无用变为有用。
5.  可维护性:我们听说可重用的代码和模型提高了相关性，并有助于加速开发、简化集成和支持维护/升级。许多软件经受住了时间的考验，因为他们的软件工程师提供了简单的更新/升级以及维护。
