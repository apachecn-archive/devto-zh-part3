# [](#what-is-toxicdevrant)什么是瘾君子？

有时软件开发是令人沮丧的，不管是因为工具、组织还是我们与之互动的人。尽管我们试图变得积极和有建设性，有时我们只是需要发泄一下，知道有人在听。当这种情况发生时，毒理专家是你同情的听众。

这就像那些支持团体中的一个，人们谈论那些使他们受到创伤的事情，而其他人点头并小声表示同意。我在电视上看到的，看起来是个好主意。唯一的区别是，这是在互联网上，我们承认主要是抱怨第一世界的问题。

我们是软件开发人员，我们更愿意解决问题，而不是抱怨。但是偶尔，问题是找到合适的地方去抱怨，这样我们可以感觉更好，并回到解决问题上来。问题解决了。

# [](#what-toxicdevrant-isnt)什么毒物不是

*   寻找技术问题答案的地方。有这样的网站。
*   用于社交、求职或招聘的网站。这些东西都有网站。
*   讨论任何事情的论坛。
*   任何人的博客。
*   专业帮助。

ToxicDevRant 旨在满足特定的需求，而不是成为一个模糊、笼统、包罗万象的论坛，讨论任何与开发相关的主题，或者与其他网站已经做得很好的内容重叠。

它与类似[TheDailyWTF.com](http://thedailywtf.com/)及其相关论坛有何不同？那个网站(我猜)更倾向于提供娱乐。如果这个论坛也能提供一点，那当然很好，但它的主要目的是让开发人员释放一些有害的沮丧、愤怒或悲伤。

# [](#the-rules)规则

*   要么支持，要么沉默。你的意见是有价值的，但是还有其他的网站，开发者可以向你索取，你也可以给他们。用评论或点赞表示同情，或者默默不同意。
*   我不需要提及钓鱼或骚扰，因为之前的规则已经排除了这两种行为。
*   不要试图解决问题。解决问题很牛逼。有这样的网站。当有人被听到时，需要被听到的问题就解决了。
*   紧扣主题，避免闲聊。发表你的咆哮。其他人会很舒服地发表他们的咆哮，而不会觉得他们打断了谈话。没有转折，没有中断。你不用举手。只是咆哮。
*   尊重他人的匿名。不要说出公司、客户、同事或项目的名称。
*   保持 PG -轻度 PG 的语言，而不是 PG-13。保持干净。
*   不要贬低群体，包括但不限于种族、性别、角色或使用特定语言或工具编程的人。我想只要他们是真正的纳粹分子，贬低他们是可以的，但这是题外话，所以也不要这样做。不要把任何人比作纳粹。
*   不要讨论政治、宗教或社会经济意识形态。甚至不要暗示它。这同样适用于任何支持或反对社会公正问题或计算平台偏好的事情。有很多地方可以讨论这些东西。我们团结一致。
*   不要提议或讨论做任何违法的事情，包括但不限于暴力和故意破坏。

我认为这涵盖了一切，我错了。

这些规则是否过于严格和限制？也许吧。让我知道你的想法。我担心的是，正如在几乎任何新闻文章的评论部分所观察到的那样，显然为自己的毒性而活的不健康的人会赶走所有其他人。此外，大多数这些规则都是样板文件。

我会试着用常识来调节。别笑了。当我不可避免地意识到这是多么的天真时，我会尝试别的东西。

# [](#suggestions)建议

*   保持匿名。
*   在你自己的时间和你自己的互联网连接上使用这个网站。有时公司会屏蔽某些域名。他们应该阻止这个。
*   想吼就吼，但不要一直吼，因为那会让人沮丧。偶尔抱怨可能会让我们感觉更好——这就是我们的想法——但它不能代替建设性的行动。卸下负担，感觉更好，然后停下来做伟大的事情。
*   这不是博客，所以不要担心有什么原创的东西要说。发泄和别人一模一样的事情是可以的。或者只是同意。或者加上自己的经历。
*   你提供的任何娱乐价值都是受欢迎和赞赏的，但这不是重点。
