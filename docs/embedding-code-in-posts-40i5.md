# 在帖子中嵌入代码

> 原文：<https://dev.to/moopet/embedding-code-in-posts-40i5>

当我在这里和其他像 Medium 这样的网站上阅读文章时，我有时会看到人们上传他们作品的照片。

像 [Carbon](https://carbon.now.sh) 这样的服务可以让你粘贴代码并得到一个漂亮的图片版本。

看起来很可爱。它颜色很好。它有很好的对比度。它总是让人觉得你在用苹果电脑。

它仍然完全不可接近。

你不能合理地将一大段代码放入图像的`alt`文本中。在你选择有限的托管平台上，你通常不能聪明地用`figure`和`code`让人们选择他们想要的。

然而，这些平台本身也提供了代码嵌入选项，带有特定于语言的语法突出显示。在 markdown 中，这就像将代码粘贴到三个反勾号中一样简单。您可以通过在开始标记后直接包含语言类型来强制语法突出显示特定的语言。

不幸的是，我想不出让 dev.to 显示一系列反勾号的方法，即使通过各种不同的方式来避开它们。出于这个例子的目的，假设`'`是一个反斜杠，mmkay？

```
'''
This will be enclosed in <pre> tags.
'''

'''js
// This will be highlighted as if it was javascript
''' 
```

Enter fullscreen mode Exit fullscreen mode

生成的代码仍然被大量标记以处理颜色和间距，但它更容易理解:

*   样式表可以在不同的视窗尺寸下处理它，而不会以非常小的文本结束。
*   它将考虑您在浏览器中设置的字体大小覆盖。
*   您可以使用自己的字体配置将其复制并粘贴到您自己的编辑器中。

这适用于所有人，而不仅仅是视力有限的人:

*   在回复帖子时，你可以*在原始代码的相关部分复制*,而不必同时打开两个浏览器，全部手动完成。
*   你可以*将代码粘贴*到你自己的编辑器中并进行测试。
*   你可以打开你的浏览器/窗口管理器/操作系统的时髦的新黑暗模式，如果你幸运的话，这个网站的风格会尊重你的设置。
*   你可以*搜索*包含某些半记忆位代码的帖子，而不期望主机站点有一个百万美元的 OCR 人工智能系统准备从其试图接管世界的常规工作中节省 CPU 周期。

其他第三方代码主机呢，比如 [Codepen](https://codepen.io) 等？

他们也很好。尤其是如果

*   你在一个允许你嵌入脚本但不支持代码高亮的网站上(比如一个托管的博客)
*   您的代码是一个功能完整的示例，可以作为演示运行，而不仅仅是作为代码片段显示。
*   你很乐意添加对另一个服务的依赖，这一切都需要(更多的故障点，潜在的费用，等等。)
