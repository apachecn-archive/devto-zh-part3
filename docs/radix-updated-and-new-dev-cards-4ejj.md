# 基数更新和新的开发卡！

> 原文：<https://dev.to/radixdlt/radix-updated-and-new-dev-cards-4ejj>

因此，就[开发](https://www.radixdlt.com/roadmap)而言，我们正越来越接近我们的技术上线(定于 2019 年第四季度)目标——节点运行器软件接近完成 95%,资产支持令牌 API 完成 90%,实用令牌工具包落后一点，完成 70%。

为了定义我们的开发生产力，我们在 Radix 使用 Scrum/Agile 方法。我们使用用户故事来定义需要交付什么，并通过我们的冲刺速度来衡量我们的生产力，冲刺速度是在平均两周的冲刺中完成的故事点的数量。从现在开始，我们决定与大家分享我们的知识库中包含哪些 [Sprint](https://docs.radixdlt.com/alpha/develop/sprint-reviews) 。

我们还发布了两个有趣的用例——一个 [POS 卡系统](https://docs.radixdlt.com/alpha/learn/use-cases/card-payment-system)，它不仅可以用于简单的交易——它是一个稍微复杂一点的解决方案，可以洞察消费习惯，还可以存储忠诚度积分。

我们还准备了 Radix 品牌的开发者卡——如果你想用这些卡来开发应用，请告诉我，或者通过任何社交媒体账户联系 Radix。

查看这些卡片[这里](https://twitter.com/radixdlt/status/1123512208154136576)

第二个用例是一个 [IOT](https://docs.radixdlt.com/alpha/learn/use-cases/internet-of-things) 解决方案，专注于探索总账上温度控制物流系统的可能性。
