# 星期五前端:咬大小的 CSS 掘金版

> 原文：<https://dev.to/kball/friday-frontend-bitesized-css-nuggets-edition-l85>

本周的时事通讯有很多非常棒的 CSS 小块，展示了一些非常酷的效果。让我看看你的想法！

还有一些很棒的 JavaScript 基础文章，介绍了用 web 组件构建的跨框架 UI 工具包，等等。尽情享受吧！

最佳，

来自 ZenDev 的 KBall

#### CSS&SCS

##### [如何创建和动画旋转覆盖](https://tympanus.net/codrops/2019/04/18/how-to-create-and-animate-rotated-overlays/)

这是一个在屏幕上隐藏和显示内容的超级酷的效果，本教程使它超级简单。如果你使用基于 html 的幻灯片，或者使用旋转木马或其他形式的隐藏和显示内容，你会想要通读一遍。

##### [淡出姐弟在 CSS 上盘旋](https://www.trysmudford.com/blog/fade-out-siblings-css-trick/)

非常聪明的小金块在悬停时淡出元素的兄弟。我以为进入它会使用兄弟选择器，但是那个解决方案有一些漏洞，而他们的解决方案没有。爱死了！

##### [如何创建多行文字背景效果](https://codyhouse.co/blog/post/multi-line-text-background)

另一个很小的教程，展示了一个非常酷的悬停背景效果，你可以用在内联元素(比如链接)上，即使它跨越了多行。

##### [35 个启发性的 CSS 动画范例](https://medium.muz.li/inspiring-examples-of-css-animation-2b2a5dd464e0)

一大堆 CSS 动画例子，每一个都是代码笔，所以你可以看到它是如何完成的。非常有趣！我想我最喜欢的是鼓声和在绳子上摇摆的猫。

##### [嵌套网格像嵌套表格吗？](https://www.rachelandrew.co.uk/archives/2019/04/12/are-nested-grids-like-nested-tables/)

随着所有关于[子网格即将到来](https://www.smashingmagazine.com/2018/07/css-grid-2/)的讨论，值得思考它是什么和不是什么。最肯定的是不会回到嵌套表的时代…虽然这听起来有点像是可能的，但在本文中，Rachel Andrew 分析了根本的区别。

#### JavaScript

##### [JavaScript 异步等待](https://zellwk.com/blog/async-await/)

异步很难。对于新的 JavaScript 开发人员来说，这是最难的主题之一，有时意想不到的行为甚至会让我们中最有经验的人感到困惑。async 和 await 关键字给了我们一些很好的语法糖，使它看起来更容易接近，但也带来了一些陷阱，如果你不小心的话，可能会撞到你。这篇文章做了一步一步的讲解。

##### [JavaScript 中的 reducer 是什么？](https://www.robinwieruch.de/javascript-reducer/)

对一个随着像 Redux 这样的库的兴起而变得越来越流行的概念的可靠的“基础”解释。如果你看到到处都是“reducer”这个词(也许你自己也写了一些)，但是不知道它到底是什么意思，看看这个吧。

##### [如何使用 JavaScript 控制台:超越 console.log()](https://medium.freecodecamp.org/how-to-use-the-javascript-console-going-beyond-console-log-5128af9d573b)

这太疯狂了。我经常使用 console.log，尤其是当我试图理解一个我不熟悉的代码库中正在发生的事情时。我多少意识到了用 CSS 做样式日志这样的事情的能力…但是 console.table 呢？这太酷了！

##### [包装一个普通的 JavaScript 包用于 React](https://scotch.io/tutorials/wrap-a-vanilla-javascript-package-for-use-in-react)

这是一篇非常好的长篇文章，讲述了如何在 React 中使用香草包。这让我想起了我写的一篇类似的文章，内容是将 JavaScript 集成到 Vue 中。通读这些内容会让您更好地理解 React 是如何工作的，并且在您下次想要使用没有集成到您选择的 web 框架中的库时，它也是您工具箱中的一个有用工具。

##### [2019 年你应该知道的 9 个 Web 组件 UI 库](https://blog.bitsrc.io/9-web-component-ui-libraries-you-should-know-in-2019-9d4476c3f103)

这真的很棒。我已经看到大多数 UI 库分裂成特定于框架的组件库或仅 CSS(或主要是 CSS)工具包，这些工具包避免了大规模的 JavaScript，但是这些库利用 web 组件来拥有它们的蛋糕并吃掉它。但是要注意大小…例如，基于材质的文件看起来像是 300k 的 javascript。基于聚合物的要小得多(并且是基于组件的)，但是依赖于聚合物。

#### 其他牛气

##### [包容性隐藏](https://www.scottohara.me/blog/2017/04/14/inclusively-hidden.html)

很好地总结了你可以选择的隐藏和显示内容的各种方式以及辅助技术。我真的很喜欢这本书，因为它不仅涵盖了技术，还深入到了你可能选择这个或那个的原因。

##### [TDD 改变了我的人生](https://medium.com/javascript-scene/tdd-changed-my-life-5af0ce099f80)

我倾向于对完全的 TDD 持怀疑态度，特别是当涉及到需求快速变化的产品时，但是这是我所听到的声音更大、更有说服力的支持论点之一。

##### [新进 Chrome 74](https://developers.google.com/web/updates/2019/04/nic74)

我在这里不经常包括浏览器更新，但这一次是因为有一个非常有趣的大 JavaScript 变化。在 Chrome 中，私有类字段现在是默认开启的！那是相当大的！

##### [利用资源提示优化性能](https://www.smashingmagazine.com/2019/04/optimization-performance-resource-hints/)

类似于我们上个月链接的一篇[帖子](https://3perf.com/blog/link-rels/)，但更容易理解，它通过向浏览器暗示它应该如何提前加载资源来优化性能。
