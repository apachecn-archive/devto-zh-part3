# (几乎)没有鼠标:浏览器

> 在这篇文章中，我将以 Firefox 浏览器的快捷方式为例。它们中的大部分也可以在其他基于 webkit 的浏览器上运行。给定快捷方式的前缀可能因操作系统而异。

最初发布在[媒体](https://medium.com/@bargord11/going-almost-mouseless-browsers-e83c7f48b046)上。

## [](#why-even-bother)何必多此一举呢？

我写这篇文章的原因是，在“标准”导航(主要是基于鼠标的导航)的性能方面，我看到了一个需要改进的领域。

当然，并不是每一个软件都有快捷方式，这绝对没问题。我想象使用一个没有鼠标的图形设计应用程序可能会有问题并且效率低下。但是大多数时候，given 程序共享其中的一小部分。

最好的例子可能是你的浏览器。

在本文中，您将学习足够的快捷方式来执行最常见的任务，如标签和窗口管理或强大的文本编辑。它们被分成 4 节课，所以你可以循序渐进地学习。

当你把导航控制权交给键盘后，你将(可能)永远不会回头。
能够做上述事情而无需:

*   将手从键盘上移到鼠标上
*   放置光标
*   让你的手回到键盘上极大地改善了我的工作流程和速度。当您在行中执行许多操作时，这种方法最有效。节省的时间越来越多！

## [](#process-of-learning)学习的过程

我学习一套新捷径的方法是:

1.  写下几个你想学的捷径
2.  把它们放在一个显眼的地方——你办公桌旁边的壁纸/便条/便利贴
3.  练习，直到你对它们感到舒服为止。

然后从第一点开始重复整个过程。一段时间后，你的肌肉记忆会让它成为你的不二法门。

## [](#tab-management)标签页管理

**一级**

*   打开新标签页— `Ctrl + T`
*   关闭当前标签页— `Ctrl + W`
*   恢复上次关闭的标签页— `Ctrl + Shift + T`
*   打开一个新窗口— `Ctrl + N`
*   重新加载页面— `Ctrl + R`

这五个命令应该已经提高了您的选项卡管理速度。作为奖励点，您可以刷新当前标签，并恢复以前关闭的标签，神经保存！

**二级**

*   转到第 n 个标签— `Alt + N`(适用于第 1-8 个标签)
*   转到最后一个选项卡— `Alt + 9`
*   将当前选项卡向左移动— `Ctrl + Shift + PgUp`
*   将当前选项卡向右移动— `Ctrl + Shift + PgDn`
*   焦点地址行— `Ctrl + L`

有了这五个快捷键，你就可以聚焦给定的标签页并在标签页间移动。额外的一点是，访问地址行变得更容易了。

**三级**

*   转到页面上的下一个可聚焦元素— `Tab`
*   转到页面上的上一个可聚焦元素— `Shift + Tab`
*   返回上一页— `Ctrl + [`
*   前进到下一页— `Ctrl + ]`
*   打开一个新的私人窗口— `Ctrl + Shift + P`

到了这个级别，您将能够在页面上的链接、输入和其他交互元素之间跳转。此外，您可以在当前选项卡浏览历史中来回导航。想开一个新的私人窗口吗？你已经得到它了。

**第四级**

*   转到下一个单词— `Ctrl + Arrow Right`(编辑文本时)
*   转到上一个单词— `Ctrl + Arrow Left`
*   转到最后一个单词— `Ctrl + A + Arrow Right`
*   转到第一个单词— `Ctrl + A + Arrow Left`
*   打开历史记录— `Ctrl + H`

这组快捷键会让你在文本导航时更加熟练。

如果你熟悉 VI/VIM，我推荐你使用一个模拟其按键绑定的扩展。你可能会为你的浏览器找到一个合适的。虽然通常不是一对一的映射。
