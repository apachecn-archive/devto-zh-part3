# 创业公司在开发移动产品时犯的错误

> 原文：<https://dev.to/brightdevs/mistakes-startups-make-when-developing-their-mobile-product-21lc>

如果你曾经在早期创业公司工作过，那么我肯定你知道这种感觉——你的移动应用程序，一个你已经紧张工作了几个月的概念，终于进入开发阶段。事情开始变得很现实了，不是吗？

一方面，你从所有的兴奋中兴奋起来。另一方面，你也可能对项目的范围感到害怕。尤其是，如果你是移动应用程序开发游戏的新手。

为了帮助你使这成为一个高效和明智的过程，我们列出了一些移动应用开发中最常见的错误。这些是:

错误 1:没有验证市场潜力

在初始阶段，设置一个成功的移动应用需要大量的前期工作。在你做出决定之前，彻底验证你的应用在市场上的潜力。有很多方法可以做到这一点:焦点小组，市场研究调查，或咨询专业机构，等等。如果你不能确定你的应用解决了市场上真正的需求，你就有失败的风险。

错误 2:没有建立(或尊重)MVP

对于一些初创公司的创始人来说，理解哪些功能对该应用的第一个版本是绝对重要的，哪些属于“值得拥有”的类别，可能相当困难。

这就是与经验丰富的移动应用程序开发团队合作将会创造奇迹的地方。在你的团队开始开发应用程序之前，告诉他们组织一次所谓的范围界定会议。预期的结果应该是就 MVP 达成一致——推出应用程序首个工作版本所需的关键功能列表。

但是那些可能促使你中途考虑更新你的 MVP 的意外情况呢？在您坚持任何改变之前，请记住它们可能会扰乱整个开发过程。这可能会导致延误，有时还会浪费财政资源。

错误 3:在太多平台上开发你的应用

有时候，在创业初期专注于一个平台是值得的。有太多成功的移动应用程序只在 iOS 或 Android 上运行。这很可能是因为市场研究表明，潜在用户使用一个平台比使用另一个平台多。与你的首席技术官或你将与之合作的软件公司谈谈——他们肯定会向你推荐一个最有效、最合适的解决方案。

错误 4:微观管理(或忽视)你的开发人员

作为一个初创公司的创始人，你可能会想与你的团队保持全天候的联系，这往往会适得其反，将你的开发人员的注意力从任务上引开。相反，你可能非常自信你的团队处理得很好，以至于你认为没有必要出席会议。告诉你的团队联系的频率对开发过程最有利。

错误 5:不关注应用分析

一旦应用上线，许多早期创业公司就会犯错误，将全部注意力放在应用商店评论上，同时低估应用分析的作用。事实上，监控应用程序的性能、用户行为、安装和保持率(以及许多其他因素)都应该被视为对后 MVP 阶段的产品开发有害。市场上有一整套分析工具，请让您的团队决定哪一个最能满足您的需求。

摘要

总结一下，借鉴别人的经验。记住上面的启动错误，你会避免许多耗时的障碍。这反过来会让你把所有的精力集中在成功的应用商店发布上。祝你好运！
