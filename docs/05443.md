# 我的新博客帖子工作流程

> 原文:[https://dev . to/therealdanvega/my-new-blog-post-workflow-4ih 6](https://dev.to/therealdanvega/my-new-blog-post-workflow-4ih6)

嗯，我已经在我的新网站上写了几个月的博客了，我想我会分享我是如何写所有博客帖子的。我真的很高兴我的写作过程，它让我可以在任何电脑上写作，并快速从想法到文章。在这篇文章中，我将谈一谈我过去是如何写文章的，以及我在新的工作流程中寻找什么。

## [](#writing-requirements)写作要求

如果你一直关注我，你会知道我是用 WordPress 管理器写文章的。我会说我的 WordPress 的帖子都在使用经典编辑器，但是我是少数几个真正喜欢 Gutenberg block 编辑器的人之一。

我为个人项目和工作写了很多文档。这些天我写的几乎所有东西都降价了，所以就需求而言，我只有几个:

*   干净和不受干扰的写作
*   降价
*   拼写和语法检查

### [](#clean-amp-distraction-free-writing)干净&分心自由写作

当我坐下来开始写作时，我很容易分心，所以对我来说，我需要一个无分心区。如果我在笔记本上写东西，这意味着背景中没有电视，我可以一个人呆在一个舒适的地方收集我的想法。

如果我在一个文本编辑器应用程序中写东西，文档需要是唯一打开的东西。当我在浏览器中工作时尤其如此。如果有任何标签打开或其他东西的诱惑就在那里，我可能会被拉向另一个方向。

### [](#markdown)降价销售

我真的很喜欢 Markdown，因为它是最容易在平台之间转移的格式。我清楚地记得，当我开始在 Markdown 写作时，无论如何，我都不明白为什么人们会喜欢它。现在，我无法想象用其他形式写作。对我来说，用 Markdown 写作比其他任何东西都更有成效。

### [](#spelling-amp-grammar-checking)拼写&语法检查

我觉得高效的一个重要方面是了解自己的优势和劣势。如果你能花更多的时间在你的优势上，少花点时间担心你的弱点，你会更有效率。我在拼写和语法方面非常糟糕，这正是我需要帮助的地方。所以，除非有人自愿成为我的全职编辑，否则我将依靠工具和服务来帮助我。

## [](#visual-studio-code-markdown)Visual Studio 代码降价

在过去的 6 个月左右的时间里，我一直用 Visual Studio 代码在 markdown 中全职编写文档、教程和练习。在大多数情况下，我真的很喜欢它，但有一个问题是我的交易破坏者。

我是[语法](https://www.grammarly.com/)的超级粉丝，几乎在浏览器的所有地方都使用它。这可能是为了给重要的社交媒体状态更新写一封重要的电子邮件。这是我的使命，我无法想象没有它的写作。

事实证明，在 Visual Studio 代码中，没有办法使用语法。所以我过去做的是用 visual studio 代码编写一个完整的 markdown 文档，然后将文本内容复制并粘贴到 Google 文档或 Grammarly 上一个您可以使用的编辑器中。

你可以想象这不是很有效率，我真的厌倦了这样做。我曾多次在 Twitter 上联系 Grammarly，请他们为 Visual Studio 代码添加支持。我仍然希望有一天这能成为现实，但在那之前，我一直在寻找另一种方法。

另一个并不妨碍交易的组件是降价预览窗口，但它有时会困扰我。我不喜欢打开我的减价文件，也不喜欢需要一个预览窗口来查看结果。

## [](#notion)观念

观念是我一直听说的工具之一，但我就是不明白。现在我承认他们在过去的一年里做了一些重大的改进，但是我记得我试了很多次，说我就是不明白。

我不断尝试的原因是，我不断在播客和推特上听说它，我不断问自己他们知道什么我不知道的。事实证明，这是一种你不可能真正喜欢玩 5 分钟的工具。

在我花了一个周末的时间建立概念后，我真的开始看到它背后的力量。这是我使用的许多工具的替代品，成为我需要的一切的一站式商店。下面是我的个人观念工作空间截图。如果你对我如何在几乎所有事情上使用概念感兴趣，请告诉我，我会整理一个帖子或视频，向你展示我是如何使用它的。

### [](#blogging-on-notion)博客上的观念

我非常喜欢 concept 的一个特点是，你可以使用一组数据，并有多个视图。如果你不太确定那是什么意思，没关系，我现在就来解释一下。

我为博客文章设置了一个看板，在这里我可以看到我所有的想法，下一步是什么，我现在正在做什么以及已经完成了什么。对于我的写作过程来说，有一个地方可以让你快速地为一篇博客文章添加一些笔记、图片或样本代码，这是至关重要的。

[![Kanban board](../Images/7368364f2efa8ea204584a97e31ede68.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--Yd-pm9ae--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/7mw6rmi8os839kg36sdb.png)

我在 Asana 做了类似的事情，但是在那里我必须创建一个新的内容日历来查看什么时候会被发布。

在概念中，你可以为每篇新文章设置一些属性，本质上这只是一个新页面。这允许我定义我期望这篇文章发布的发布日期。

[![Blog Post Custom Properties](../Images/5b8e12dea301d84197f7961a299df8d9.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--4bRL6hgh--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/im2m1180q3ecqkvdnv5z.png)

到目前为止没有什么新的东西，但他是观念的真正力量发挥作用的地方。您可以为同一组数据创建不同的视图。因此，现在我可以为博客文章创建一个新的日历视图，并且基于发布日期，我可以获得我所有博客文章的日历视图。这真的帮助我保持正轨，让我知道我发表了什么，什么时候。

[![Calendar View](../Images/4cd7edd1082e34bfd3cbc7cb96b7fba9.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--gHcQ6KKD--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/b30xajv256itbuk0s6yp.png)

我也喜欢使用这个视图来查看我是否过于频繁地发布了某个主题。我试着把我的帖子分开，这样就不会集中在一个话题上，这真的很有帮助。

### [](#markdown-amp-block-editor)Markdown &块编辑

我真的很享受写作的过程。你做的每件事都是一个块，所以当你开始写作时，它是一个文本块。如果你按下回车键，你知道你要添加一个新的标题(或任何块类型)，你只需输入一个正斜杠(/)，你会得到一个块类型列表。这只是在创建一个页面时可以使用的块类型的一个例子。

*   文本
*   页
*   待办事项列表
*   标题 1、2 和 3
*   项目符号、编号和切换列表
*   引用
*   链接到页面
*   提到一个人或一页
*   日期或提醒
*   数据库(这里是一个大的块列表)
*   图像、书签、视频、音频
*   密码
*   文件
*   嵌入(如推文、Gists、谷歌地图等)

我的大部分博文都是文本、图片和代码。所以我真的很欣赏使用这些内容类型是如此容易。这是以前博客文章中的一段代码。我喜欢开箱即用的代码格式，以及确定代码是什么语言的能力。

[![Syntax Highlighting](../Images/e0ac7ed21ecd2e1e3f1c40f2a1565734.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--rpurvN1D--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/f5ij6go6n4xfesv3w2ba.png)

写作过程中我喜欢的另一个很好的部分是，因为它是基于网络的，我可以在任何电脑或手机上写作。是的，我有时会用手机写一些我不想忘记的东西，或者给文章添加一个链接，以便我记得以后引用它。

### [](#grammarly-support)语法上支持

记得我说过语法支持是必须的吗？好消息是，它的工作，有点。当我第一次开始用观念写作时，它根本不起作用。Chrome 插件会启动并向我显示建议，但每当我试图应用这些建议时，idea 中的一些东西就会启动并恢复到原来的样子。另一个问题是，插件有时需要一段时间才能发现问题。

奇怪的是，就在这个星期，它开始为我工作。有时我不得不硬刷新页面，但当我得到语法建议时，我现在可以悬停在它们上面并接受更改。我联系了“观念”，问他们是否修好了什么，他们没有。

虽然这有点像工作，但并不像我希望的那样工作，这是我绝对需要前进的东西，所以我希望他们正在努力。

### [](#export-as-markdown)导出为降价

当我的博客文章准备好了，我就去右上角，点击 3 个小点，然后导出为 Markdown。这可能会令人困惑，因为还有一个选项可以将所有内容导出为降价。这个特性是在页面中嵌入页面的情况下使用的。

[![Export as Markdown](../Images/f33965d2d5070a0decbcfe43fe37a1e2.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--EOpQnaxX--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/1ko6tpeyraxkzcct6wgs.png)

关于导出，我真正欣赏的是我在 Markdown 中有我所有的代码和图像，并且我准备好发布。在那里，我将使用我在这里写的自定义生成器[来生成一篇新的博文。然后，我将我所有的图片复制到我的新帖子文件夹中，并将所有的降价信息复制到生成器创建的新降价文件中。](https://www.danvega.dev/blog/2019/04/23/gridsome-blog-post-generator)

### [](#whats-missing)少了什么

这个过程远非完美，有些事情做得不好，有些事情我希望他们能添加进来。正如我前面提到的，语法支持对我来说是必须的，我希望他们能解决这个问题。“以降价形式导出”是可行的，但并不完美。

代码块不以 Markdown 格式导出，所以它们周围没有语言定义的反勾。这意味着在我发帖之前，我必须检查并手动修复这些问题，这有点繁琐和烦人。

导出过程的另一部分是，当我创建一个表时，它不会以 Markdown 格式导出一个简单的表。它认为这个表是一个数据库，因此将它导出为另一个页面，但这对我不起作用。

## [](#cross-posting-on-devto)在开发目标上交叉过账

写博客过程的最后一步是将我的文章交叉发布在 https://www.dev.to 上。如果你不熟悉 DEV，这是一个很棒的开发者社区。我发现，当我在 Reddit 这样的平台上发布文章时，我因为是垃圾邮件发送者而遭到了很多人的憎恨。我的意思是我的文章是为了帮助别人，这个过程的哪一部分是垃圾邮件？

DEV 不仅允许我发布文章，还让我从社区中获得了一些非常棒的反馈。那里的每个人都很好，我从来没有在任何其他平台上参与过。我通常会在它出现在我的博客上一两天后发布，但当我这样做时，DEV 允许我为每个帖子的前面添加一个规范的 URL。

我还用 1000w x 492h 的尺寸创建了我所有的博客文章封面图片，因为这是 DEV 想要的尺寸。这个尺寸也很适合我的博客，但是现在我有了两个平台的封面图片。如果你还没有检查过 [DEV](https://dev.to) ，我强烈推荐你这样做。

因为你可以添加自定义属性到每个帖子，我甚至添加了一个开发发布日期，所以我知道这将在那里上线。
