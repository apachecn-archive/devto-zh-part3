# 如何选择要自动化的测试用例

> 原文:[https://dev . to/jaswankaur/how-to-choose-the-test-cases-to-automated-3 EEG](https://dev.to/jaswantkaur/how-to-choose-which-test-cases-to-automate-3eeg)

与手工测试相比，Web 应用程序测试自动化有很多优势。首先，它改善了结果和质量，增加了可靠性，并减少了结果的偏差。此外，它加速了过程，增加了覆盖测试，并且最终可以提高软件的整体质量。良好的自动化可以提高出版物的质量，增加测试覆盖面，降低测试成本，如果测试得当，还可以及早发现错误。

测试自动化中的一个主要问题是选择要自动化的测试用例。但是有人怎么知道所有的测试用例都是自动化的呢？

基本的规则是，您必须理解单个测试用例自动化将会带来的好处。

让我们看看其他一些关于如何选择自动化测试用例的技巧和规则。

## [](#ten-test-cases-you-should-automate)你应该自动化的十个测试用例

### [](#1-repetitive-test-runs)1。重复测试运行

这是黄金法则。如果测试运行是重复的，您应该将其自动化。这会节省你宝贵的时间和精力。此外，如果这是您认为将来会经常运行的测试，那么自动化是一个非常好的想法。每次你都会得到更准确的结果。

### [](#2-highrisk-test-cases)2。高风险测试案例

如果涉及的风险很高，您应该自动化测试用例。根据潜在的人为错误确定测试用例的优先级。

### [](#3-critical-parts-of-a-web-application)3。Web 应用程序的关键部分

自动化 web 应用程序的关键部分是一个聪明的想法。你想避免任何人为错误的可能性，为了安全起见。对于可能危及整个 web 应用程序的测试用例来说尤其如此。

### [](#4-extensive-tests)4。广泛的测试

输入大量数据会让人精疲力尽。此外，在手动测试过程中出错的几率也会增加。这就是为什么意味着大型数据集的测试通常是自动化的。主要是指那些需要填写长表格的测试用例。

### [](#5-evaluate-the-pros-and-cons)5。评估利弊

通过评估，我们暗示了完成自动化过程或者手动进行测试所需的时间。如果你不准备在自动化过程中投入时间，也就是说，如果过程本身比手工测试花费更多的时间，常识告诉你自动化是不值得的。对于非重复性测试来说尤其如此。

### [](#6-three-golden-tests)6。三个黄金测试

当谈到自动化时，有三个测试用例你不应该质疑:回归测试、冒烟测试和健全性测试。为什么？因为应用程序的每个版本都需要测试。

要执行这些测试，有各种工具可供您使用。您可以利用像 [LambdaTest](https://goo.gl/ERMuWJ) 这样的工具，帮助您测试 2000 多种浏览器和操作系统组合。您可以使用 LambdaTest 执行手动和自动化测试。

### [](#7-if-you-can-answer-the-magical-question-why)7。如果你能回答神奇的问题“为什么？”

不要仅仅因为你可以就自动测试用例。这会给你带来弊大于利。如果这对应用程序、你的团队或组织有益，那就去做吧。你的目标应该是快速反馈、精确和更好的整体结果。但是，不要忘记，Selenium 在编写脚本时需要很深的知识。

手动测试有时是更好的选择，尤其是如果您不是熟练的 Selenium 用户。

记住“为什么”这个问题的答案谈到自动化，千万不要说“因为我可以”

### [](#8-complex-cases)8。复杂的案件

自动化复杂的测试用例几乎是一个规则。同样，当涉及到复杂性时，您确实希望安全起见并节省您的时间。

### [](#9-performance-testing)9。性能试验

我们通常选择自动化性能测试，比如负载测试和压力测试。

### [](#10-long-testing)10。长期测试

如果一个案件需要通宵处理，你应该自动处理。

## [](#when-not-to-automate)当不自动化时

这里有一个关于何时避免自动化的提示。如果您有一个还没有被彻底探索的测试用例，那么进行手工测试总是一个更好的选择。否则，您可能会得到错误的结果。请记住，准确性和有效的结果，以及有见地的测试，是您的最终目标。

总之，在选择要自动化的测试用例时，坚持这三个原则:总是自动化耗时的测试、重复的测试、难做的测试和与风险相关的测试。这将有助于您将注意力集中在只能手动完成或者应该手动完成的测试用例上，同时为处理无法自动化或者无法由机器完成的更重要的任务留出一些宝贵的时间。
