我喜欢这个想法，我想尝试一下。所以事不宜迟，我 2019 年的主题是:

# [](#principals)校长

我做开发已经有一段时间了。我做过一些不同类型的工作，主要是咨询或产品团队。我开始意识到的一件事是，好的开发始于好的原则。我仍在努力完善和发现所有这些原则，但这里有一些是我目前正在追求的。

### [](#general)通用

1.  持续指标推动持续改进
2.  观点不是事实。
3.  不怪，修。

### [](#code)代码

1.  不要让你的代码做太多。
2.  光盯着代码是修不好的。
3.  [什么都写两遍](https://dev.to/wuz/stop-trying-to-be-so-dry-instead-write-everything-twice-wet-5g33)
4.  总是评论你的抽象
5.  为删除而优化

### [](#writing)写作

1.  为自己写作，为读者编辑

### [](#relationships)关系

1.  关闭循环。不要让联系消失。

### [](#mentorship)师友

1.  导师将与你分享他们的经验，为你的知识库添砖加瓦。
2.  导师会利用他们的关系网帮助你成功实现目标。

### [](#some-other-goals)其他一些目标

1.  每周至少在 dev.to 上写一次
2.  在科技会议上发言
3.  释放一些我一直在开发的副业项目的工作
4.  围绕学习成为一名开发人员发布 3 个系列(第一个正在编写中！)
