# 铁轨！我来了(第四天)

> 原文：<https://dev.to/saral/rails-here-i-come-day-4-3g17>

按照计划，我今天开始为我的博客创建一个用户登录和注册页面。这些需要在不同的阶段进行认证，我打算从头开始构建，不使用任何 gem。尽管事后看来，我是否使用 gem 并没有太大的区别，因为，无论哪种方式，我都很难真正理解和理解注册、用户认证和会话的概念。

因此，今天的帖子将更多地讨论过程，我的努力而不是实际的代码本身，因为说实话，即使我能够让身份验证工作，我也不知道它是如何实现的，因此，谈论实际的代码没有意义。

虽然从昨天开始我就取得了成就(如果我可以这么说的话),但这一天开始得很好。我创建了用户页面和注册页面，并开始在不同的 git 分支中工作。我遵循了使我能够创建帖子并将其存储在数据库中的流程。按照这个过程，我能够在我的数据库中注册一个用户，然后我意识到——我不知道如何对认证位进行编码。我四处寻找教程，找到了一个可以帮助我从头开始创建注册和登录页面的教程。“我真的会试着理解这些代码”，我想。但是唉！这是不可能的。我无法理解正在发生的事情。有那么几个瞬间，我确实觉得自己已经理解了这个概念，但后来我又不太确定。因此，我决定明天再试一次。我真的需要从字里行间去理解发生了什么。此外，我还觉得我在没有任何 Ruby 知识的情况下尝试了 Rails。所以，我决定也去看看露比。同时，我会继续开发我的应用程序并构建它，我也会花时间学习 Ruby 基础知识。我相信这能帮我理清头绪。

如果你对这个 Ruby on Rails 新手有什么建议，我洗耳恭听。谢谢你
