# VS 代码遗漏了一个重要的 WebStorm 特性

> 原文：<https://dev.to/mokkapps/vs-code-misses-an-important-webstorm-feature-5f28>

我真的很喜欢 VS Code，但是我错过了 WebStorm 的一个特性:

[代码检查](https://www.jetbrains.com/help/webstorm/code-inspection.html)

> 检查不仅检测编译错误，还检测不同的代码效率低下。每当你有一些不可及的代码，未使用的代码，非本地化的字符串，未解决的方法，内存泄漏，甚至拼写问题，你会很快找到它。

你知道我如何在 VS 代码中检测不可及的代码、未使用的代码或未解析的方法吗？

基本上，我只知道使用 TSLint 或 ESLInt，使用它们的未使用变量规则，但这还没有这么强大。例如，它们不跟踪未使用的方法。
