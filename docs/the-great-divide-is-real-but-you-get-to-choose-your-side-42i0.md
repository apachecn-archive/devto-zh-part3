# 巨大的分歧是真实的，但是你可以选择你的立场

> 原文：<https://dev.to/sirtimbly/the-great-divide-is-real-but-you-get-to-choose-your-side-42i0>

最近的一些文章对我产生了影响，它们对于我们行业正在沸腾的这场辩论的背景很重要。我推荐阅读雷切尔·安德鲁斯的《大鸿沟》、《T2》、《HTML CSS 和我们正在消失的入口点》，以及克里斯蒂安·海尔曼的《HTML 是编译目标》，还有《T4》。

实际上，我对 CSS 和 HTML 被对待的方式并不感到愤怒。我感到担心。而且，这种担心是正常的，是关于身份和别人对我的看法。我担心我在 web 应用程序开发方面是愚蠢和短视的。我担心我并不是自己想象中的那种网页设计师。

我担心维护的身份是什么？我最初是做 photoshop 文件的 HTML 和 CSS 剪辑的。我和另外一名设计师以及几名“程序员”在一家小公司工作。我们为小企业建立了数百个网站。自 1999 年以来，我一直在为自己和其他人制作网站，但当我大学毕业并开始“职业生涯”时，我真正成长为一名 CSS 专家，以及所有需要通过艰苦的方式才能发现的浏览器小怪癖和技巧。

大约在 2006 年，我说服了团队中的其他人从表格布局转向纯 CSS。浮起来，伙计。漂浮物...*颤抖*

我在大学里上过计算机科学课，以及所有的设计和通信课，但这并不是我真正喜欢的东西。我首先认为自己是一名设计师，但我被期望将我的 photoshop 设计变成真正灵活的网页模板，可以处理 CMS 可能放入其中的任何内容。在浏览器中做设计工作对我来说很有趣。尽管在大学期间尝试过几次 PHP，但服务器端编程的神秘魔力仍然超出了我的适应范围。最终，我开始在 Flash 小部件和“富互联网应用程序”中编写交互脚本。我能够将这些知识应用到浏览器中的 javascript 上。大约在那个时候，jQuery 出现了，让一切变得更好。

由于一些奇怪的职业转变，2010 年我发现自己是一个企业后端 web 开发团队的开发人员。我发现自己在快速学习新技能，用 C#和 ASP.Net MVC 进行后端编程，数据库设计，架构图，大型企业软件开发生命周期过程。

在过去的 20 年里，我发现我真正想要的一直是建立令人敬畏的体验，这种体验很容易建立，也很容易分享。HTML 和 CSS 从来都不是“简单的入口点”，它们是你需要学会操作以在屏幕上得到想要的输出的奇怪的技术上的复杂语法。当我需要的时候，学习成为一名程序员是必须有机发生的事情。现在，我已经在过去的 5 年中深入研究了 javascript 应用程序的编码，并编写了 9 年的后端服务器应用程序，我可以说，我不在乎将 HTML 和 CSS 视为任何特别“珍贵”的东西。它们只是告诉浏览器如何呈现 UI 所需的结构化数据格式。如果我们有工具层，可以从 javascript 动态创建所有必要的 HTML 和 CSS，那么对我来说真的不重要。我们仍然可以让应用程序变得可访问，我们仍然可以让它变得有性能，我们仍然可以在开发人员工具中打开它，查看所有的 DOM 节点和样式属性。

我不认为用 JS 写你所有的 CSS 是一个好主意。我认为我们仍然可以构建有用的和可重用的 CSS，利用级联，而不是害怕它。我们可以通过理解来处理全局范围的本质，而不是仅仅用一个随机 ID 覆盖所有内容来创建隔离。CSS 的建立是有原因的，知道原因并理解如何使用这些方面是值得的。但是，这并不意味着颠覆对 CSS 如何工作的通常预期总是错误的。原子 CSS 是一个很好的例子。

我一直坚持前端开发人员的身份，即使是在我专注于设计或服务器端开发的时候。现在，我自己对前端开发人员的看法更符合新一代 JS 工程师的更大观点，他们构建单页面应用程序，并在方便的时候绕过 HTML 和 CSS，我想知道我是否目光短浅，对自己的技术不负责任。我关心负责任地对待网络，从长远的角度看，我们的工作可以用几十年。如果我们用 JS 工具链构建一个内容网站，我们需要确保它输出静态的 HTML 页面，这些页面在加载时会被 JS 增强——就像 [Nuxt](https://nuxtjs.org) 所做的那样。

我认为现代工具链和 SPA 开发框架非常令人兴奋和有用。我对事件循环、调用堆栈、虚拟 dom 和功能代码的了解比我预想的要多。追求学习我所做的是我的选择。我有特权、运气和极大的热情来帮助我到达我所在的分界线的那一边。我可以告诉你，这不再是一个巨大的鸿沟，它只是令人兴奋地看着丰富多样的 web 开发方法，深情地回忆我过去站过的山峰，并展望未来，那里有更多令人兴奋的攀登等待着我。
