# 客户安全问卷:VRA 两步走

> 原文：<https://dev.to/tedcarstensen/customer-security-questionnaires-the-vra-two-step-1k2k>

这篇文章最初由乔治·查马利斯撰写，发表在 heavybit.com 上。

供应商风险评估(vra)是在销售周期中最糟糕的时候弹出的令人讨厌的安全问卷。

不要让冗长的电子表格和令人生畏的网络表单欺骗了你:vra 不是一个严格开发、客观分析的安全测试。相反，它们更像是让你约会对象专横的父母相信你的意图是高尚的，你会在宵禁前回来。这是一场外交和印象的游戏，你最好的办法是穿着得体，有礼貌，然后尽快离开。

向 CTO 倾销 vra 的常见方法浪费了宝贵的时间，并限制了您的机动能力。你需要一种方法来安抚你的评论家，取悦你的冠军，并为你的竞争设置一个高标准。你需要用最少的时间和精力来完成这一切。

你需要的是 VRA 两步舞。

这是怎么做的。接下来是背景和角色，点击这里查看[门票](https://docs.google.com/spreadsheets/d/1o8fC939ZldHr5qYAU1jtpNA2y9KI19ok4guxHP-OoJc/edit?usp=sharing)，点击这里查看[小贴士](https://docs.google.com/document/d/1hRKSG9G9ar052MackxuvFW80Wii91wK9Ul6nGk8Zm2M/edit)。

#### 第一步。武装你的冠军

在 VRA 流程开始之前，销售主管会给你的拥护者发送一份代笔的框架文档，描述你的服务是什么，他们将如何使用它，风险，以及一些安全亮点。该文档有助于您的拥护者在内部推介您的产品，并且应该发送给评审者——为 VRA 流程的其余部分提供有用的背景信息。

#### 第二步。吸引你的评论者

销售主管负责管理与审核者的所有互动。他们会在每次互动中保持乐观，并承诺在两周内完成回复。你的评论者肯定是一个工作过度的专业人士，永远不会得到足够的尊重。节省他们的时间，给他们一些善意；它会创造奇迹。

#### 第三步。填写 VRA

销售主管将 VRA 传递给验证者(一个关于 VRA 的游戏[0])。审核者应该是一个团队成员，有足够的时间在两周内完成表格，有足够的技术知识来理解材料，有足够的外交知识来处理模糊、不相关或疯狂的问题。验证者将利用公司的安全政策、之前完成的 vra、提示和技巧列表以及他们自己来之不易的经验，在截止日期前(或之前)将完成的调查问卷发送给销售人员。

#### 【根据需要】退回给 CTO

验证者不知道答案的问题应该转给首席技术官，他会从技术团队那里找到正确的答案。当这种情况发生时，验证者应该以更新的策略或提示和技巧列表的形式获取答案。

#### 【根据需要】澄清

如果评审者对您的回答有“顾虑”，销售主管将联系首席技术官，首席技术官将尽一切努力解决任何误解。作为其中的一部分，首席技术官可能需要把审核者扔到公共汽车下面，向审核者大量道歉，并哀叹如今很难找到好的帮助……无论如何都要得到批准。

## VRAs:他们从哪里来，去哪里

很有可能你苦苦思索的上一个 VRA 是十多年前由一个委员会创建的，该委员会迫于行业的合规性要求，对第三方风险采取了一些措施。委员会成员有非技术性的头衔，如合规、保险、官员和先生。经过深思熟虑，该团队一致认为，设计一款为他们公司量身定制的 VRA 的最佳方式是复制并粘贴某人在上一份工作中使用过的问卷内容。

从那以后，回收的问卷每年都要接受审查，委员会的新成员必须添加一些东西，以表明他们正在参与。像所有的委员会一样，这些贡献是由善意、恶意、盲目的信念以及这样一种信念共同驱动的，即在任何人意识到事情有多糟糕之前，他们就会退休。[1]

这一连串的添加(而不是删除)就是为什么 vra 充满了从样板文件到不一致、混乱、过时、不知情和不相关的问题。不可避免地，委员会中的一些人决定他们的贡献将是通过将它迁移到一个企业 web 应用程序来“简化”该过程，该应用程序曾经是顶级的，但今天肯定会在每个 web 浏览器上中断，除了 Windows XP 附带的 Internet Explorer 版本。

如果你曾经在填写 VRA 时觉得自己快疯了，那是完全可以理解的——这些问卷是电子表格形式的官僚精神分裂症。

如果你认为那不好，想象一下你不得不日复一日地复习它们。

很有可能你的评论者正处于公司组织结构图中一个高度政治化分支的最底层。他们可能是法律部门下的安全团队，或风险部门下的技术团队，或保险部门下的合规团队…无论他们在哪里，都将在下一次重组中被转移。

审查者本身可能是刚从大学毕业的人，或者是坚定的专业人士，或者是技术爱好者或技术恐惧者，或者是安全专家，或者仅仅是缺乏安全感，渴望升职或退出。唯一一致的是，他们会工作过度，报酬过低，并且知道他们承担了所有犯错的风险，却没有得到任何正确的回报。

个人和政治变量的结合意味着绝对没有办法知道在任何给定的时间对任何给定的客户 VRA 审查过程会有多严格。一家价值十亿美元的金融机构可能只会让你粗略地看一眼，而一家成立五年的 SAAS 公司需要几个月的时间，需要从详细的背景调查到血祭的一切。

虽然有人试图创建标准化问卷调查或外包审查过程，但这些策略并没有流行起来。SOC 2 或 ISO 27001 等合规制度越来越受欢迎，开始融入这一流程，但支付合规报告是否会取代 vra 或被视为未来的赌注，这是一个悬而未决的问题。VRA 的挑战不会消失，你需要相应地制定策略。

## 角色扮演

以下是对贵公司将参与 VRA 进程的每位参与者的一些说明。参与的每个人通读这些笔记，以了解团队其他成员将要做什么。

### 销售线索

好消息是，您已经在销售周期中走得足够远，您的潜在客户愿意花费时间和金钱让您通过安全审查流程。你现在的角色是 VRA 流程的代言人——除了请首席技术官来澄清事情之外，你将处理所有的来回工作。

#### 武装你的冠军——取景 Doc

发送[框架文件](https://drive.google.com/open?id=1bkIbyHeZkqcRYseoxtyFyRKrlqrw97KqcFjbt6etEcU)是一场赌博，但由于审查者将在没有它的情况下让你通过最高级别的审查，这是一场下跌有限而上涨很多的赌博。

VRAs 缺少的一个主要东西是关于你的公司实际做什么的任何种类的上下文。没有这一点，审查者将被迫认为你对公司(以及他们的工作)构成了生存威胁，并将让你接受绝对最高级别的审查。

框架文档给了你一个机会，通过预先添加上下文，你可以让评审者降低他们的专业偏执，理想地给你一个更容易通过评审过程的路径。额外的好处是——冠军还可以从文档中提取部分内容，用于在内部简报、演示等中宣传你的公司。

你对框架文档的定制程度取决于你，但花一分钟时间用你的拥护者的声音来表达它，并列出他们将如何使用它的细节，这可以让他们更容易地将它作为他们拼凑的东西来播放。

#### 吸引你的评论者——保持积极的态度

无论过程中发生了什么，你与评论者的交流都应该带着愉快、积极的尊重。您的方法应该始终假设，如果有任何问题，它很可能是一个误解，您很乐意通过找到更好的答案或请 CTO 来帮助澄清来解决。

#### 始终在收货时打开 VRAs

在向审阅者承诺两周内完成之前，您需要确保文档没有损坏，或者您的在线门户证书有效。

#### 迅速做出反应

评论者讨厌他们发来一个 VRA，却没有收到任何回复。打开 VRA 后，回复一封感谢信，让他们知道你已经打开了它，并计划在两周内还给他们。

### 验证者

恭喜你，你被任命为公司审核人。这是一份费力不讨好、令人讨厌的工作，但对你的公司来说，能够达成交易绝对至关重要。你将从销售团队和首席技术官身上卸下一大块重担，因此他们至少都欠你一杯啤酒。

#### 2 周是进取但可能的

这之所以可行，是因为 vra 之间有大量的重复——当你填完四五个问题时，你就已经回答了 90%的问题。剩下的 10%将是奇怪的东西，你需要根据具体情况来处理。

#### 拍摄完成早

不要等到两个星期的最后期限才填写文档，你永远不知道什么时候会遇到需要技术团队回答的问题。早点结束会让你的公司看起来很棒。

#### 根据需要向首席技术官求助

会有你不知道答案的问题。你需要带他们去找首席技术官寻求答案，然后首席技术官会把你介绍给技术团队中合适的人，并确保那个人会及时回答你(你很容易被吹掉，而首席技术官不会)。

#### 建立一个提示列表

你的建议清单是一份内部文件，你应该记下你想出的使这个过程有效的具体策略和战略。这是一个有用的提醒，可以交给下一个接任者。[下面是一些入门的小技巧](https://docs.google.com/document/d/1hRKSG9G9ar052MackxuvFW80Wii91wK9Ul6nGk8Zm2M/edit)。

#### 请记住——这是非常主观的

你没有被比作某种安全的黄金理想——你被比作其他所有人……这是一个非常低的标准。

### CTO

你一直说你有比填这些该死的表格更重要的事情要做。你说得对。诀窍是建立一个流程，将它们卸载给其他人。这里有一套[帮助你在公司内部建立 VRA 两步](https://docs.google.com/spreadsheets/d/1o8fC939ZldHr5qYAU1jtpNA2y9KI19ok4guxHP-OoJc/edit?usp=sharing)的入场券。

#### 在验证者寻求帮助时给予支持

如果验证者被卡住了，你需要把他们交给你团队中合适的人来得到答案。确保你的团队成员知道验证者有截止日期，他们需要尽快回复。大多数回答应该不会超过几分钟。

#### 澄清=拥有问题，快速解决问题

每当你被拉去帮助销售主管处理评审者送来的问题时，你应该准备好把它当作一个大误会，你和你的团队将承担所有责任——即使问题是模糊的或者评审者不知道他们在说什么。

低调行事，把一切都揽在自己身上，这是你搞清楚评论者想让你看到什么的最好办法。一旦他们给了，尽快给他们，然后继续你的生活。

## Outro:为 VRAs 道歉

VRA 过程中最令人恼火的一点是，这并不是一个坏主意。第三方服务确实带来了重大的持续风险，即公司的敏感数据可能被恶意黑客窃取或意外暴露在互联网上。

强制将安全性作为企业合同的先决条件的法规和遵从性制度，比安全行业在过去三十年里说的或做的任何事情都更能推动计算机防御。黑客的生存风险是真实存在的，但监管罚款或无法达成交易的财务风险是一个更有效的激励因素。

尽管抱怨 VRA 流程很容易(而且非常容易)，但我们每天都从为降低第三方风险而强制实施的安全控制中受益。是的，由委员会设计，由天知道是谁审查的 VRA 进程的方法有问题，但在我们拿出更好的东西之前，这是我们得到的最好的。

同时，还有 VRA 两步舞。100%保证不会每次都完美地工作，人和过程之间有太多的差异。它承诺的是提供一种标准的、高效的方法，并具有最大的灵活性。这种灵活性是必不可少的，因为你要通过每个客户的独特怪异的 VRA 挑战。

祝你好运。
