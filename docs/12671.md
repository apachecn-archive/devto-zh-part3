# HYWCT:维护

> 原文：<https://dev.to/flippedcoding/hywct-maintenance-247d>

这是四部分系列的最后一部分:你的网站是如何组成的。到目前为止，您已经了解了前端和后端代码、服务器及其工作原理，以及互联网的工作原理。你知道制作一个网站并让它在网上可用的所有事情。你只需要知道最后一件事。

如果你有一个网站，你希望人们不断回来，你必须准备好做一些维护工作，并更进一步。保持网站的更新和安全只是维护的一个方面。好的文档是维护和搜索引擎优化的一部分。这里有一些维护网站的方法。

## 更新

在某个时间点，你的网站至少会有*次更新。最常见的更新是添加新功能。随着你的网站开始吸引更多的用户，他们会向你展示你可以改进的地方。一种方法是让你的网站更容易使用。也许不是让人们在常规搜索框中输入，而是让搜索框为他们做某种类型的提前预测。*

功能并不是更新的唯一途径。任何时候你的网站上有真正的用户，他们注定会以一种不可预见的方式破坏一些东西。修复 bug 也是你更新代码的原因之一。可能是颜色变化这样的小事，也可能是像你的支付表格不工作这样的大事。不管怎样，错误是 web 开发周期的一部分。

其他可能需要更新的是您的代码库使用的库。库也会经历更新，这可能会改变它们处理更新代码的方式。保持你所有的库是最新的，以防止任何潜在的安全漏洞，并从中获得最佳性能(大部分时间)。

## 文档

文档唯一令人痛苦的时候是当你找不到它或者它写得不好的时候。维护你的网站的一部分是了解你的网站是如何运作的。随着时间的推移，这些知识会因为不同的项目或者仅仅是因为你前一天晚上睡觉而褪色。留下足够的文档，这样你或其他人就能弄清楚网站是如何运作的。

在网站上留下如何通过注册过程或如何通过任何其他过程的注释。这个文档主要是为你准备的，所以它不必是一个有着完美语法或特定格式的正式文件。它只需要有足够的有用信息。

你的文档可以是一堆带箭头的图片，如果这能帮助你或者一个新人知道下一步该做什么的话。用你自己的语言写文档会让你更容易更有效地工作，因为你会确切地知道该做什么。当你开始一个新项目时，把你的文档想象成你希望拥有的指令。

## SEO

这对于希望走得更远的开发者来说是一个额外的奖励。如果你懂一点 SEO 或者搜索引擎优化，你可以让自己成为一个更有价值的 web 开发者。SEO 是一个致力于帮助网页在搜索结果中排名更好的领域。作为一个网站开发者，这并不需要太多额外的工作，但是它可以对网站的访问量产生巨大的影响。

一旦你有了所有可用的内容，你就可以开始关注内容中的特定关键词。你会想选择一个人们可能会搜索的关键词或关键短语，然后调整你的页面以增加额外的强调。你可以做的一件事是看关键词密度。根据内容的长度，你的关键词不应该在文本中出现超过 3-5 次。

另一件你可以留意的事情是移动响应能力。搜索引擎会降低移动布局糟糕的页面的排名。确保您的网站易于在不同尺寸的设备上浏览。您可以在任何浏览器中使用开发者工具来快速检查网站的大小。甚至有预设的手机和平板电脑尺寸可供选择。

你要考虑的最后一件事是 HTML 的语义。当这些网络爬虫访问你的网站时，他们使用语义来找到他们正在寻找的一些数据。如果你没有使用像

<main>和

<aside>这样的标签，你可能会错过一些简单的优化。</aside>

</main>
