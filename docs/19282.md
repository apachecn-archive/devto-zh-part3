# 你正在开发的产品是如何构建的？

> 原文：<https://dev.to/tiagomagalhaes/hows-the-product-youre-working-on-architected-4jpn>

首先，我想向这个社区介绍我自己。我已经有一些不错的文章可以在以后找到时间阅读，并在这里发表了一些评论。

我来自葡萄牙，我从出生起就住在那里，已经有 3 年的专业经验，其中将近 10 年是在尝试摆弄 ASP(没有. net)。在那门课失败后，在重试时，PHP 出现了，AMP 栈一直陪着我，直到我开始工作并开始学习 AngularJS(不接触 Java 后端)。一年后，我换了家公司，开始在甲骨文臭名昭著的 HTP 公司 AngularJS integrated 正常工作。在另一次切换和一个仍在进行中的附带项目后，我发现自己删除了 Angular 中的 JS，并在 Postgres 中使用 EAN 堆栈，同时在访问 API 的一侧使用原生移动应用程序。

这篇文章的目的是分享你正在使用的产品的架构。这种互动将展示一个产品或一个 SaaS 可以有多么多样化和复杂，并将帮助我和你看到你正在做什么栈和集成，并看到和评估我们的产品架构是否有改进，我认为我的可以改进。
