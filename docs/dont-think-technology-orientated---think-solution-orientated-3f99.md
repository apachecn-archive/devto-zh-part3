# 不要以技术为导向，要以解决方案为导向

> 原文：<https://dev.to/lschultebraucks/dont-think-technology-orientated---think-solution-orientated-3f99>

[第一次发表在我的主页上](https://lasseschultebraucks.com/life/work/2018/12/04/dont-think-technology-orientated-think-solution-orientated.html)

我一直有一个问题，我想每个开发人员都有，那就是有大量多样且有趣的技术。它很大，你可以每天从一个地方跳到另一个地方。

昨天我和一位资深开发人员进行了一次非常有趣的对话。我们谈论了各种各样的事情，包括技术和技术的快速变化。我告诉他，我总是在一个项目上遇到问题，我只是在玩了几个星期后跳到另一个项目，开始使用一种新技术，如框架或新的编程语言，然后跳到另一个项目。他回答说，这可能是错误的方式，我应该表现得更加面向解决方案，而不是面向技术。

他这么说是什么意思？学习不同的技术，不同的编程语言有错吗？

当然不是。但是，以技术驱动而不是以解决方案驱动可能是错误的。我的意思是，总是跟上技术并保持在宣传列车上是非常好的，但一旦这非常困难，你将永远不会成为任何方面的专家。我与之交谈的高级开发人员称自己为全栈，但不是因为他总是跟上最新和不同的 JavaScript 框架。他将新技术应用于想要解决的问题，从而构建真正的解决方案，而不是仅仅玩玩新技术。

接下来，他还建议我，如果我对自己的项目没有想法，就参与开源项目。作为一名开发人员，你应该为开源软件做贡献的原因有很多，我不想在这里赘述，而是想指出几点。凭借你的专业知识，你可以产生巨大的影响，即使对你来说这是一个琐碎而简单的任务。你不要把你的工作给别人，你会从开源中学到很多。

总而言之，这次谈话让我明白了两点:首先，要以解决方案为导向，而不是以技术为导向。即使你想成为一名全栈开发人员，它也会进一步提升你的技能。其次，作为一名初级开发人员，为开源做贡献是非常重要的。这基本上是一个双赢的局面。你会学到新的东西，你会创造真正的价值，你会引起潜在雇主的注意。

这篇文章应该会提醒我，在对话之后，我明白了一些要点。我希望这也能帮助其他想要学习和成长的年轻开发人员。
