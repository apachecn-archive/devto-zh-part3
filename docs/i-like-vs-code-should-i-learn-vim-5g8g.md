# 我喜欢 VS 代码。我应该学 Vim 吗？

辛德雷·索胡斯开了一家 ama 回购公司，有人问他[这个](https://github.com/sindresorhus/ama/issues/568)问题:

我花了一分钟写了一些关于 vim 的想法，并在对海报的回复中摆脱了令人生畏的“学习 Vim”的感觉。剧透一下，我觉得值得了解一下 vim。

嘿 [@anishmittal2020](https://github.com/anishmittal2020) ，我今年 23 岁，2018 年 5 月刚刚毕业，获得计算机科学学位，所以我不知道你是否认为我是“年轻的”之一，但这里有一些想法。我个人大约 80%的时间使用 VS 代码，20%的时间使用 Vim。以下是我认为 Vim 有价值的原因:

## Vim 非常适合远程编辑

随着时间的推移，您会发现自己处于使用 GUI 编辑器(如 VS Code 或 Sublime Text)不容易、不可行和/或不可能的情况。例如，通过 SSH 在远程机器上工作。当然，在某些情况下，你可以使用 SFTP 或 SCP 或 rsync 或其他什么来使用 VS 代码进行远程编辑，但如果你只是试图进行简单的修改，那就要做很多工作。

例如，如果您需要快速更改系统的`hosts`文件，该怎么办？如果您知道 Vim，那么您只需 SSH 到系统并运行`vim /etc/hosts`，保存您的更改，就完成了。同样，您可以同步目录，甚至使用类似`sshfs`的东西进行远程挂载，但这是一个重量级的解决方案。此外，您可能会发现，出于安全考虑，您不能这样直接同步或挂载。

同样的想法也适用于 docker 容器，在您重新构建 docker 映像之前，您可能希望在容器内进行更改以验证它。

## 你不必一下子学会 Vim

我认为很多人因为不能退出的笑话而被“学习”Vim 所困扰。显然不仅如此，Vim 还有点复杂。但是对于它给你的所有能力，你可以很容易地开始使用它。

*   要退出，就是`:q!`。仅仅知道这个命令，我们就可以了解一些关于 Vim 的事情:命令以`:`开始，可以被缩写为`quit -> q`，可以被强制为`!`。
*   保存(写)一个文件，就是`:w filename`，它教会我们也可以给命令参数。
*   要退出写，就是`:wq`，我们了解到命令可以链接在一起。
*   `hjkl`是移动键，`w`是“单词”,`b`是“后退单词”。
*   点击`i`进入插入模式，在这里你可以打字。按下`esc`回到命令模式，此时按键执行动作而不是打字。

在 5 个要点中，您已经学习了很多关于 Vim 的知识，并且知道编辑文件的基本知识。从这里开始，你只需要通过使用它来练习，这确实是学习东西的最好方法。如果你熟悉命令行，并且已经经历了学习的过程，那也是一样的。你尽可能多地使用终端，在你不知道的时候想出如何做事情。

如果你不知道，有一个 30 分钟的简短教程，可以帮助你学习一些关于 vim 的更高级的东西，比如替换模式、运动等等。它让你在适当的位置进行修正，这样你就可以边走边练习。我强烈推荐，但还是那句话，你没必要一下子掌握所有的东西。Vim 很灵活，所以有很多东西您可能永远不会用到。VS 代码也是如此，我敢打赌你不会担心学习它的所有细节。

## Vim 有很多插件，很难管理它们

VS 代码扩展市场是 VS 代码最好的部分之一，因为它可以很容易地找到根据您的需求定制编辑器的方法。根据我的经验，Vim 有大部分相同的插件，尽管管理它们更具挑战性。大多数情况下，你使用类似于`vim-plug`的插件管理器，并将配置存储在你的`.vimrc`中。

## VS 代码不到处跑(还？*)

这又回到了远程编辑方面，但有所改变。我刚买了一台 iPad Pro，作为旅行时的超便携电脑(我真的讨厌在机场拖着笔记本电脑😅).我依赖 SSH 和我的远程服务器来真正工作(主要是编码和配置更改)。本质上，iPad 变成了服务器的远程终端，我在那里工作(与大型机时代没有什么不同😄)所以我唯一的选择是使用 SSH 的编辑器。有很多编辑，但我更喜欢 Vim！
