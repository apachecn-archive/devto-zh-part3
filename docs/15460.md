# Git 用于数据存储

> 原文 https://dev.to/jessekphillips/git-for-data-storage-1j9h

这绝不是什么原创的想法。我读过一些东西，比如 NoSQL 数据库的 [Git。](https://www.google.com/amp/s/www.kenneth-truyers.net/2016/10/13/git-nosql-database/amp/)

为什么还要考虑 git？数据的历史可能非常有价值，git 使用许多工具来跟踪变更，以审查这些变更。我认为如果我直接使用数据库而不是通过文件系统，我可以获得一些好处。

您不希望考虑使用需要复杂连接的高度交叉引用数据的 git。Git 被设计成高效地存储数据并且不重复，因此它可以查询历史和该历史的元数据。

我仍然在思考这个用例，它要求历史是最有价值的。但是有一点是肯定的，数据必须在文件系统之外工作，因为 git 原语存储、blobs 和树直接映射到文件系统。

我最初的应用想法是测试结果和工件。分支对于环境可能是有用的，但是管理与其他环境的关联可能是有用的，但是很困难。我们将会看到我是否尝试走得更远。
