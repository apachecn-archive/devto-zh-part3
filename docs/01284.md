# 使用 Testsigma 实现移动应用自动化的 3 个步骤

> 原文:[https://dev . to/pmnagabhushanam/3-steps-to-automate-mobile-applications-using-test sigma-5ee 7](https://dev.to/pmnagabhushanam/3-steps-to-automate-mobile-applications-using-testsigma-5ee7)

在这篇文章中，我们将通过三个简单的步骤学习如何使用 Testsigma 自动化移动应用。

可以使用各种平台和测试框架来自动化移动应用程序。

*   使用 AndroidSDK/UIAutomator2 的 Android 应用
*   使用 Xcode-swift / XCUITest 的 iOS 应用程序

让我们一步一步来看。

## 第一步

保持自动化所需的应用程序准备好上传到 Testsigma 实验室。安装在测试移动设备上(本地混合执行)。该应用程序也可以位于本地机器或服务器上，可通过互联网上的直接链接访问。你可以在这里阅读更多关于 Testsigma 中执行模型的内容。

Testsigma 实验室是 Testsigma 的专用设备云，有 100 多种真实的网络、移动设备和不同版本的 Android OS、iOS 平台可供选择来执行测试。

Testsigma 提供的其他云实验室解决方案包括 Sauce Labs 和 BrowserStack。

要上传移动应用程序，在上传页面中导航到测试开发>上传
，点击右上角的上传按钮。

## 第二步

开始编写测试步骤。

如果你是在 [Testsigma](https://testsigma.com) 中创建测试步骤的新手，请参考下面解释这一点的短片链接。在 Testsigma 中编写自动化测试非常容易，因为每个人都可以很容易地编写和理解它们，因为测试是用简单明了的英语编写的。Testsigma 使用 NLP 语法来轻松地自动化复杂的测试，并且不涉及任何编码！

让我们编写启动移动应用程序的步骤，点击应用程序上的一些元素。
