# 追溯性反模式

> 原文：<https://dev.to/goyo/retrospective-antipatterns-374a>

嘿！嘿！你！是的，你。过来这里。让我们谈一谈。

先说清楚，我们不是因为你有罪才在这里的。(我们谈论的真正原因是，我在我的机器上开始了一个非常长的数据提取，我厌倦了等待，直到它结束。)

因此...我最近参加了你的一些回顾会，我有点失望，你知道吗？这就像...你们太聪明了，有这么大的潜力。尽管如此，还是有一些事情让我抓狂...

首先，**呻吟**。

我明白了。经过 X 周的努力工作，你终于有机会发泄你的沮丧，谈论不好的事情。太好了，来吧，尽情释放吧。但是要确保你在 10-20 分钟内完成，这取决于整个会议的长度。

你看，发泄不是会议的真正意义，或者至少不是唯一的意义。我们作为一个团队正在进行回顾以改进，这很难通过一两个小时的抱怨来完成。在某一点上，你需要把注意力从感觉转移到潜在的问题和它们潜在的解决方案上。

单子上的第二件事...关注其他事情和其他人。

> 啊，另一个队做(不)这个做那个！
> 
> 啊，我们继承的这个愚蠢的遗产代码！
> 
> 啊，我们缺少的这个工具，或者它没有像我们期望的那样工作！
> 
> 啊，这些公司流程让我们慢了下来！
> 
> 啊，这些不断变化的要求！

你看到这里的模式了吗？啊，等等，我已经破坏了答案。

坏消息是，把责任放在其他人和外部因素上不会推动你的项目前进，也不会改善你的团队工作方式。

好消息是，至少你找到了一些痛点。在大多数情况下，你可以直接将这些转化为你的团队可以解决的事情:

> 我们与另一个团队的合作不够有效。
> 
> 我们在日常工作中重构不够努力。
> 
> 我们没有确保我们有合适的工具来完成这项工作。
> 
> 我们还没有弄清楚如何有效地配合公司的流程，并且/或者我们没有尽全力去改变无效的流程。
> 
> 我们没有足够短的反馈回路。

听着，这并不是说你和你的团队要为你的项目/公司发生的每一件坏事负责。我只是说，在回顾中，你应该想办法:

*   解决你能影响的不好的事情
*   尽管有些事情你无法影响(无论是暂时的还是永久的),但还是要有效地工作

明白吗？太好了。

既然你已经有了正确的、以改善为导向的心态，我的下一个观点应该很明显了。

**你。应该。避免。[自行车脱落](https://en.wiktionary.org/wiki/bikeshedding)。真的。**

人们很容易在小的、不重要的细节上发生争执——标签或空格，这个库或那个库，这个命名约定或那个命名约定。哦，我喜欢！让我们尽可能地打磨这个宝贝吧！

龙躺在某个地方，写在一张单独的便条上，或者，也许，已经被遗忘，只是在一句话中提到？那很难。这需要...思考！

简而言之，你在回顾中的工作就是识别这条龙，并想出杀死它的计划。如果没有足够的时间，小的农民纠纷和问题可以事后解决或根本不解决。

全身长的...好吧！很高兴和你谈话，下次见！
