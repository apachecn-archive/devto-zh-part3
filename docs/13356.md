# HYWCT:服务器

> 原文:[https://dev.to/flippedcoding/hywct-servers-301b](https://dev.to/flippedcoding/hywct-servers-301b)

这是四部分系列的第二部分:你的网站是如何组成的。对大多数人来说，服务器有点神秘，但其实不必如此。如果你知道你在看什么，你想做什么，它们不会比代码更复杂。今天没有一个在线网站不是托管在某种服务器上的，所以了解一点是有好处的。

你的服务器处理所有没有人真正考虑过的后台事务，比如数据库和你处理流量的能力。了解这些组件是如何使您的网站在线可用的，可以帮助您预防未来的问题，因为您知道要查找什么。

## [](#database)数据库

似乎新闻中总是有关于数据泄露或大公司被黑的消息。他们通常指的是数据库。数据库保存着你的网站的所有用户信息。这包括密码、用户名、信用卡、社会安全号码以及任何其他可以在您的网站上输入的信息。

这就是为什么保护您的数据库非常重要。您的后端代码是您的第一道防线。一旦你克服了这一点，你还可以做一些其他的事情来保护你的珍贵数据。

## [](#security)安全

服务器可以用来设置很多安全措施，防止黑客入侵你的东西。有一件事你可以做，以帮助保持你的网站和数据安全，是有一个单独的服务器为您的数据库和您的网络应用程序。大多数时候，人们试图通过将数据库和 web 应用程序的代码放在同一个服务器上来节省一点时间和金钱。当你这样做的时候，黑客更容易得到他们需要的一切，因为他们只需要黑一台服务器。

保持任何新补丁的更新对于保护您的站点也是至关重要的。当你看到你的服务器有新的更新时，最好去做。我知道更新有时可能不确定，但最好冒这个险，而不是冒被黑的风险，因为人们知道某个特定的弱点。

一个简单但容易忽视的安全预防措施是确保数据库的安全控制是启用的。通常，您的数据库管理员(DBA)会担心这些事情，但是知道它们的存在不会有什么坏处。默认情况下，安全控制通常是启用的，这只是需要检查的事情之一。

另一件有助于保证服务器安全的事情是一个好的防火墙。web 应用程序防火墙可以防止交叉脚本攻击和 SQL 注入攻击。可能已经有了某种数据库防火墙，但是拥有一个 web 应用程序防火墙可以防止人们通过你的网站入侵你的数据库。使用防火墙来真正限制谁可以访问您的服务器。

## [](#scaling-up)放大比例

为了稍微改变一下思路，我们来谈谈如何使用服务器来扩展你的网站。每台服务器都有有限的资源。CPU 和 RAM 只能处理这么多的活动，而实际上它们无法处理更多的活动。您必须预测随着时间的推移您预期的流量，并相应地选择服务器。

如果您需要扩展服务器，您有几个选择。如果你的主机包允许升级，你应该不会有任何问题。您将为额外的资源支付更多费用，但您不必担心任何配置更改。如果你没有能力升级，你可能需要一个新的服务器。

“新”服务器可以是您当前链接到其他服务器以共享资源的服务器。您可以安装更多的处理能力或对数据库进行分区。作为一名 web 开发人员，您可能不太需要担心这一点。但是如果你在小公司工作或者你为自己工作，知道如何扩展服务器是很好的。

这是对什么是服务器以及它们如何在你的网站的大计划中工作的简要概述。幕后发生的事情太多了，会让你大吃一惊。请记住，服务器只是一台用来托管网站的计算机。真的就这些了。
