# 我给自己定下的奇怪规则让我找到了工作

> 原文：<https://dev.to/healeycodes/the-weird-rules-i-set-myself-that-got-me-a-job-2g9m>

以下不是好建议。只是*建议*。甚至这也是一种延伸。

我过去真的很喜欢那些生产力黑客。没有零时差，番茄，写下你一天的目标，开始一项任务五分钟，等等。最终，它们合并成一份规则文件。我想成为一名程序员。我已经消化了所有我能接受的建议。博客和阅读材料，复杂的图表，用你还不知道的时髦词汇描述你的职业生涯。我说，分析麻痹够了，然后在伦敦的一家小咖啡店里打出了一些规则。对这些，我卡住了。

下面的部分描述了一些规则和结果(面试，工作机会),但是请注意，我的成功可能与这里的每一个字都没有关系。

### 每天用两种语言写代码

对我来说，这就是 JavaScript 和 Python。我的目标是全栈和后端位置。如果我被困在项目中，我会从头开始用两种语言编写数据结构。重复帮助他们坚持下来。我想在没有任何准备的情况下，用任何一种语言编写相对干净的代码，如果我被叫去面试的话。如果你的目标是学习，解决对你来说具有挑战性的问题永远不会是浪费时间。

像 [LeetCode](https://leetcode.com/) 和 [HackerRank](https://www.hackerrank.com/) 这样的网站有一些问题，你可以对它们进行排序，让它们变得越来越难。重要的是你意识到目标不是正确答案——而是*理解*。在你的知识边缘推进应该产生迷你学习会议，在那里你追逐解决方案的不同实现，以便你理解为什么和如何。我发现个人博客总是有最好的分解。

### 看书自己抱着课本睡觉

如果你看过我的其他文字，你就知道我不会停止推荐 *Grokking 算法*。我选择了我喜欢的计算机科学教科书，这些教科书不一定会对我找工作有帮助。这些书不一定要有代码或数学，例如*代码:隐藏的语言..*或*神话中的人月*。我在找工作时做的很多事情都是为了进入正确的空间。说起来很尴尬，但是我做了我想象中一个好的软件工程师会做的事情。我想这就像想成为一名优秀的篮球运动员，然后花训练后的时间站在中心球场盯着篮筐。有时候休息一下也不错，你知道的。

### 每天学习面试问题琐事

我并没有试图去记忆，那样会感觉很廉价，我只是试图理解这些问题意味着什么，以便指导我的学习。他们被问及原因。琐事问题实际上是关于异步编程和事件循环。我必须知道答案，并且能够谈论它所代表的主题。这些会议首先是可怕的，因为它们进一步揭露了我不知道的一切(现在仍然不知道！).

你不会经常不得不调用`delete`，但是知道它做什么和*不做是很好的(我在一次采访中被问到这个问题)。像`most popular JavaScript interview questions`这样的搜索会给你找到我所说的列表。这些问题可能会变得相当深奥，所以你的里程可能会有所不同。*

 *### 三个具有即时交互的可测试项目

三个人感觉是个好数字。我逆向工作——首先想出简历上项目的简介和要点。建设项目排在第二位。他们每个人都需要一个测试套件，例如`npm test`、`pytest`。在我看来，优秀的工程师会测试一切。我想我仍然相信这一点。尽早编写测试可以加速开发的后期阶段。

我说的即时交互是什么意思？我猜想有人会在我简历的每个超链接项目上花大约 4 秒钟——大多数人甚至不会点击。我在我所在地区的数字海洋 [Droplet](https://www.digitalocean.com/products/droplets/) 上主持了这三个项目。Digital Ocean 也有一些很好的 Linux 入门指南。我不想使用免费选项，比如 Heroku 的免费层，不使用时休眠，当访客到来时必须唤醒——我的四秒钟就这样没了。

一旦他们到达页面，我希望他们能够立即与项目的核心思想互动，例如，不需要注册或授权。我会浏览一下我的项目，让你有个更好的想法。

##### CodePen 战队克隆

我过去(现在仍然)非常喜欢结对编程，并为远程协作编码创建了一个低延迟的解决方案。我克隆了 CodePen 的设计，但通过 WebSocket 将显示数据传输到整个房间。队友可以在大约 25 毫秒内看到实时编辑。房间也可以分叉。当到达主页时，*创建房间*按钮在最前面和中间。

##### 游戏堵塞高分 API

这是我为游戏开发者运行了几年的免费解决方案，只需调用我们的 RESTful API 就可以立即建立一个 highscores 后端。在加载主页时，您会看到您的 API 键，并且有预填充了这个键的链接来测试这些特性。

##### 轻量级谷歌分析克隆

我尽可能以最基本的方式克隆了谷歌分析，允许人们推出自己的分析。这个角度是隐私友好的，人们只被跟踪 24 小时。我在简历上链接了演示页面，上面有弹出的动画图，显示了页面浏览量、独立访问者、平均访问量。页面时间，跳出率。

回到规则上来。

### 一天看十份简历

人们到处张贴科技简历，但我发现质量最高的是在 [r/cscareerquestions](https://www.reddit.com/r/cscareerquestions/) 上。在所有的学习之后，你是否得到面试机会几乎完全取决于你的简历。阅读所有这些简历有助于我理解我自己的简历在与什么竞争。它给了我版面、页面设计甚至项目的灵感。我也帮忙指出我看到的语法错误。我花时间在他们身上——有时甚至把他们打印出来，并用笔做上标记。这种行为是有历史先例的，亨特·S·汤普森用打字机打完了《了不起的盖茨比》和《永别了，武器》(还是用打字机打的!)。).

### 求职信与真实联系

在英国，求职信似乎更有益。我把公司在你的求职信中提供的小文本框也算在内。我努力写一些真实的东西，因为我真的相信真诚胜过一切。我喜欢阅读该公司的技术博客，如果他们有，如果没有，我经常检查他们网站的源代码，并试图找到有趣的东西进行评论。我甚至发现过一次泄漏的 API 密钥。在我的大部分面试中，都会提到求职信中的话题。

### 电话面试的节奏和计划

事先把所有东西都打印出来，放在床上。招聘启事、我的个人简历、公司简历、面试官的 LinkedIn(尽管，不要提到这个)——这些都帮助我了解了他们的背景，看到他们的脸让人放松。这提醒了我，人们几乎是一致友好的，希望你成功。我总是在房间里踱来踱去接受电话采访。我时而踱步，时而平躺。

### 前一天前往面试地点

我前一天去了面试的地方，这样我的旅程就不会有压力了。我想象自己去面试，得到工作，并在那里工作——就像我要开始新的通勤一样。当我走进房间时，熟悉这个区域让我感觉更舒服，这才是真正的意义所在。一个小时左右的自我快照。然而，没有什么能让我做好在大楼里迷路的准备，这实际上发生了两次。

跟着我申请，最后电话面试率 10%，都是邀请我亲自面试。不幸的是，我在这篇文章中展示了我对测试项目同样痴迷的兴趣。
