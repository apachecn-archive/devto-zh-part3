# 回到过去:处理后退按钮和慢速连接

> 原文:[https://dev . to/jean Carl/going-back-in-time-dealing-with-the-back-button-and-slow-connections-52o 4](https://dev.to/jeancarl/going-back-in-time-dealing-with-the-back-button-and-slow-connections-52o4)

随着越来越多的内容通过 JavaScript 动态加载，它变得越来越引人注目，并可能扰乱用户体验，甚至扰乱用户的位置感。

例如，我正在浏览 Twitter 时间线，点击了一个链接，将我带离了时间线。我的大脑活动可能出现了失误，但当浏览器加载我的第一次点击时，我瞥见了时间轴上的其他东西，引起了我的兴趣。我迅速按下后退按钮，显然对我所看到的比我要去的地方更感兴趣。唉，内容已经不在了。

在下面的屏幕截图中，广告是动态加载的，并在页面呈现时发生变化。

于是我靠近一点看了看，思考了一下刚刚经历的用户体验。浏览器上的后退按钮意味着(或者曾经意味着)我应该能够回到我所在的地方，并且大概从我离开的地方继续。但是，由于几个原因，这是一个有缺陷的期望。首先说一下时间。根据我点击后退的速度(现在与一小时后)，可能会有新的内容添加到时间线中。动态内容的危险。这可能会改变内容或我在页面上的位置，带来不同步的体验。

时间轴的另一个体验是自动分页。您可以在这里向下滚动页面，新内容会自动追加，从而创建一个永无止境的内容列表/流。对于那些懒得点击“下一步”按钮等待的人来说，这是个好主意。然而，后退按钮也打破了这种体验。如果你在返回历史的路上(呃，在很多页内容的页面上)，那一页是如何处理将数百个条目重新加载到列表中并神奇地让你回到你离开的地方的呢？

哦，一想到这个我就头疼。我想我们已经到了宇宙自我坍缩的时刻。

我感到不安的最后一件事是，我们仍然没有向 web 开发人员灌输分心的艺术。有了 200mbp 的互联网连接，他们可以构建和测试 web 体验，web 开发人员认为事物加载如此之快，以至于他们不需要为较慢的连接构建过渡。过渡就像旋转的轮子，直观地显示某事(希望)正在发生，例如计算机正在思考。事实上，这是当年的情况，但今天，我们有足够快的计算机，所以我们不需要担心，对不对？

不对。如果你去一个互联网接入不稳定的地方，你很快就会发现那里有相当多的人没有快速(甚至是可靠)的互联网连接。你也不必离开你的硅谷泡沫。乘坐 Caltrain 子弹快车，尝试使用移动热点。参差不齐的报道(令人惊讶的是帕洛阿尔托有这样的问题)会让浏览最简单的网站成为令人沮丧的经历。不幸的是，我们在美国没有稳固的互联网连接，网络开发者应该明白这个事实。

现在，我承认我把很多责任推到了一些我承认我过去也犯过的错误上。时间、精力和其他因素使得开发人员不太重视这些小的 UX 问题。但是这里有一些事情可以减轻这种体验。

首先，纺车。等等吧。是的，给用户一个视觉上的提示，这个过程比预期的要长，但是仍然有事情在做。

将 target="_blank "添加到链接中。这将打开一个新的浏览器窗口或标签。用户可以很容易地关闭那个标签页并返回到原来的地方。不需要加载任何内容或找出他们在哪里。这就像一个零再工程的努力，令人惊讶的是这个技巧没有被更多地使用。

最后一个可能有点棘手，但是请尝试缓存内容。例如，为浏览器添加一个 HTTP 头，以便将内容保留更长时间。或者将其存储在本地存储器中。

避免给用户带来不愉快的体验，让他们停止正在做的事情。也许再过二十年我们就能解决这个问题。
