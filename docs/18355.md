# 用户

> 原文:[https://dev.to/mortoray/the-user-88e](https://dev.to/mortoray/the-user-88e)

编程最重要的一个方面是用户。在我们的所有讨论中，即使是操作系统的最低技术细节，也要记住这一点。我们需要编程的唯一原因是为了用户。

用户是谁？因为他们是我们最终服务的人，我们需要了解他们是谁，他们不是谁。在最抽象的意义上，用户是任何使用技术来实现目标的人。

这是摄影师润色他们的照片。这是一个游戏玩家在控制器后面放松。是朋友在电话里互相交谈。这是一个会计在电子表格中输入数字。成千上万的人浏览社交媒体，或者少数人组织周末出游。这是一个饥饿的人在周六深夜点了一个汉堡。这是一位音乐家正在对他们的新杰作进行最后的润色。这是医学研究人员试图创造一种新的疫苗。

该榜单关注的是做实事的人。我有意忽略了很多我认为是二级用户的人。这些人为其他人生产工具，为最终用户服务。这些是使用技术和库的管理员和程序员。我们是用户，但是一个特殊的阶层。我们的需求需要得到满足，但最终我们需要迎合主要用户。

注意，我在这里没有提到软件。用户不关心软件、硬件或两者之间的任何东西。用户有一个高级的、真实世界的任务。许多组件需要协同工作来帮助它们。分别考虑硬件、软件和进程对用户是有害的。

理想情况下，用户的思想永远不会离开他们的应用领域。他们不担心他们正在使用的比特。它无缝融合，成为他们的流动的一部分。除非我们能像用户一样思考，否则没有办法提供这种流动。我们需要同理心来理解他们的问题。任何时候我们做出决定，我们都必须考虑它对那些用户的影响。挫折感来自期望的不匹配，不一定是缺陷。

此外，我们需要考虑我们的用户有什么能力。他们不像我们一样精通计算机，同样，他们的思想不需要离开他们的领域。进入用户的大脑让我们使用他们能理解的术语、符号和过程。用户不应该必须阅读这本书来使用我们制造的工具。

超越知识，不是所有的人生来都是一样的。不仅仅是残疾，微小的变化，比如手指的大小，都会使应用程序难以使用。我们没有相同的反应时间，也没有相同的鉴赏能力，甚至不能分辨颜色。在一个万物相连的世界里，我们必须考虑所有人。即使只留下几个也是不公平的。

创造好的软件需要理解我们的用户。他们是谁？他们能做什么？他们想做什么？对于所有的开发团队来说，这远远超出了编程的范畴。我们开发软件的原因只有一个，它需要始终处于我们思想的最前沿。

用户。
