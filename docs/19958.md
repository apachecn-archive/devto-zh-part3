# 我如何免费主持我的流行词博客

> 原文:[https://dev . to/nathanheffley/how-I-host-my-buzz word-powered-blog-for-free-2c 77](https://dev.to/nathanheffley/how-i-host-my-buzzword-powered-blog-for-free-2c77)

我的[博客](https://www.nathanheffley.com/)是我喜欢称之为“流行语驱动的”,因为它是我想尝试的新技术的试验场。在目前的形式下，它是一个由[盖茨比](https://www.gatsbyjs.org/)开发的单页[反应](https://reactjs.org/)应用。整个网站都是预先呈现的静态页面，所以即使用户没有启用 JavaScript，他们也会获得通过 HTTP 请求加载每个页面的常规体验。但是，如果用户确实启用了 JavaScript，该网站将变成一个单页渐进式网络应用程序，即使他们的互联网连接不稳定或完全无法访问，该应用程序也将继续工作。

简而言之，它是一个移动优先、支持离线、预渲染的静态单页渐进式 web 应用程序，使用 Gatsby 生成的 React。

我希望，所有这些时髦的词语都能带来美妙的用户体验。在第一次访问时，该网站在大约一秒钟内加载，只下载了大约 500KB 的数据(这只是半兆字节，而目前的平均网站是几兆字节)。既然应用程序已经加载，后续的页面加载实际上是即时的。只需要传输几个字节的数据来确保页面是最新的。即使用户离开了网站，将来又回来，整个网站的加载速度会更快。不到一秒钟的时间来渲染网站，并且每个页面只有几百个字节来确保一切都是最新的。

使用我的网站不仅对我的读者来说很容易，对我来说也很容易。运行一个静态网站比其他传统的博客平台如 Wordpress 要容易得多，也安全得多。使用静态站点的最大障碍是拥有 CMS 客观上比将新文件提交到 git 存储库或上传到服务器更加用户友好。这就是为什么我决定只用一个 CMS 来处理我博客的内容。如果你查看我网站的代码(在 GitHub 上的[，你会注意到我的帖子都不在那里。这是因为当构建网站的静态页面时，它可以从](https://github.com/NathanHeffley/blog) [Contentful](https://www.contentful.com/) 中提取数据，并使用这些信息来生成页面。这让我可以开源我的博客框架，而不用放弃我为它写的所有内容的版权。

这让我想到了如何免费托管我的内容的第一部分...

## [](#free-content-management)免费内容管理

Contentful 允许您使用各种字段创建不同的内容类型。这对于像博客这样的东西来说是完美的。任何时候我想写一篇新的博客文章，我只需登录 Contentful，创建一篇新文章并填写所有字段，一旦我发布了这篇文章，它就开始生效。因为 Gatsby 只在构建站点时从 API 中提取数据，而且只在发生变化(代码或内容)时才构建站点，所以我永远不用担心会遇到任何速率限制。我是唯一一个为我的博客写作的人，我当然没有几十个内容类型或几千个帖子，这让我可以轻松地留在他们免费计划的限制内。

通常，在 Contentful 这样的外部服务上编辑内容不会导致托管在不同服务器上的网站更新。但是由于 webhooks 的强大功能，每当我在 Contentful 上保存更新时，它都会向我的托管平台发送一条消息来重建网站。

这让我想到了我如何免费托管我的整个网站的最后一部分...

## [](#free-static-site-hosting)免费静态站点托管

如果你想托管静态网站，Netlify 是一个很棒的服务。最大的限制是免费计划只允许一个账户，但如果你是一个独行侠开发者(或者共享网飞账户的那种人)，这没问题。如果这个限制对你来说不是问题，并且没有任何额外的特性是必须的，那么你就不会对使用 Netlify 有任何抵触。一个站点可以直接从你的 git 仓库中获取，他们的服务器会运行你需要运行的任何构建命令来构建这个站点。网站建成后，他们会将静态文件部署到他们的 CDN 上。

每当您对您的 git 存储库的主分支(或您从其部署的任何分支)进行新的提交时，Netlify 将自动重建您的站点并部署它。

更重要的是在 Contentful 中发布新内容时进行更新。正如我提到的，每当我保存新内容时，就会调用一个 webhook。更具体地说，Netlify webhook 是用一个触发站点重建的密钥调用的。Gatsby 获取新内容并为其构建页面，并在添加页面时更新任何需要更新的内容。
