# 你需要知道的 7 个基本 Linux 命令

> 原文:[https://dev . to/real _ ishan/7-essential-Linux-commands-you-needle-to-know 3151](https://dev.to/real_ishan/7-essential-linux-commands-you-need-to-know-3151)

一个亲密的朋友和导师曾经告诉我，你只需要知道大约 10 个命令就可以正确地使用 Linux。在使用了 10 年 Linux 服务器之后，我可以自信地说这个数字是相当准确的。

然而，如果你只是在寻找一个小项目的基本操作，你可以做得更少。在本文中，我将介绍 7 个 Linux 命令，您将在 99.99%的情况下使用它们。

> **快速提示**:我这里只介绍最常用的。如果您想了解更多，请在您的终端中键入`man <command>`来查看手册(您可以退出使用 q)

## [](#file-navigation)文件导航

### [](#ls)ls

`ls`是一个简单的命令，列出当前目录下的所有文件。如果您继续操作，在您的(macOS 或 Linux)终端中键入`ls`,您将看到您的主文件夹中的文件列表。

就其本身而言，`ls`并没有给出太多的信息。我大多用 3 个参数，像这样:`ls -lah`。这些论点是这样的:

1.  `l`:这是小写的‘ell’，不是‘ei’。它将以长格式显示列表，包括文件权限(现在不用考虑这些)和许多其他细节
2.  `h`:默认情况下，Linux 会在`ls -l`列表中以字节显示文件大小，没有任何单位。使用`h`参数将使这些文件大小可读。
3.  `a`:显示所有隐藏文件(linux 中以`.` [dot]开头的文件)。

### [](#cd)cd

你很可能从 Windows 中知道`cd`。它允许您导航到新文件夹(目录)。

快速 3 步用法:

1.  `cd folder-name`切换到名为`folder-name`的文件夹
2.  `cd`转到您的个人文件夹(默认情况下您登录时所在的文件夹，通常是`/home/<username>`
3.  `cd ..`转到父文件夹

**快速绕开路径** : cd 可以采用相对路径和绝对路径。绝对路径以`/`或`~`开头，例如`/etc/hosts`或`/usr/local/bin`。

相对路径是相对于当前文件夹的。如果你在`songs`文件夹中，并且其中有另一个文件夹`maroon_5`，`cd maroon_5`会将当前活动的文件夹切换到`maroon_5`，但只在`songs`文件夹中。

## [](#file-management)文件管理

这些命令对于复制、移动和删除文件非常有用。作为一般规则，请记住这些命令遵循以下格式:

`command <source> <destination>`

### [](#cp-amp-mv)cp & mv

`cp <source> <destination>`将源文件复制到目的地。

*快速举例:*您的主目录中有一个名为`notes.txt`的文件，您想将它移动到文件夹`all_notes`中。那你就用`cp notes.txt all_notes/`

如果您尝试复制一个文件夹(Linux 中的目录)，这将会失败。要复制文件夹，必须使用`r`参数。要将文件夹`notes_2018`复制到文件夹`all_notes`中，您将使用`cp -r notes_2018 all_notes`

移动文件而不是复制，在 windows 中考虑剪切和粘贴。遵循与`cp`相同的格式。

### [](#rm)rm

`rm`删除文件或目录。就像`cp`一样，如果你想删除文件夹，你需要指定`-r`参数。

**附言**我这里不涉及编辑文件。但是如果你需要这样做，你可以在 Ubuntu 类发行版上使用`vi <filename>`或者(更喜欢)`nano <filename>`。不过要小心`vi`:

## [](#interacting-with-servers)与服务器交互

### [](#ssh)ssh

`ssh`允许您登录远程服务器。如果您需要登录到 Linux 服务器，您需要:

1.  它是 IP 地址或主机名
2.  服务器的有效用户名和密码

使用`ssh <username>@<server>`开始认证。在大多数情况下，服务器会提示您输入密码，如果密码有效，您将会登录。

> 快速提示:一旦完成，您可以使用`exit`从远程服务器注销。

### [](#scp)scp

当您想要在不同的服务器或机器之间传输文件时,`scp`非常方便。

当您的笔记本电脑上有一些文件，并且想要将它们移动到远程服务器时，您将使用它。用`scp <local-file> <username>@<server>:<path>`就行了。例如，如果在`/Users/sam/secret.txt`您的 Mac 上有一个文件，并且您想将它复制到位于`192.168.0.1`的安全服务器的文件夹`/tmp`中，您将像这样使用 scp:

`scp secret.txt <server-username>@192.168.0.1:/tmp`
