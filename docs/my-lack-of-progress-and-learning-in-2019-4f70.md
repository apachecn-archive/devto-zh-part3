# 2019 年我的进步和学习的不足

> 原文：<https://dev.to/rjpsyco009/my-lack-of-progress-and-learning-in-2019-4f70>

随着 2019 年的到来和 2018 年的结束，我真的意识到最后一年对我来说是多么没有成效。当然，在完成 IT 学士学位的过程中，我收获了很多。我对此感觉很好，但是我做过的最好的编程只是在我上学的时候，从那以后我再也没有上过 Java 课...二月？

我必须找出哪里出了问题。在写论文和建立文档技能的过程中，我有了完成作业的愿景，完成了我的顶点...只是编码。

那么为什么没有发生呢？

希望你能从我过去 7 个月的经历中学到一些有价值的东西。

**注意**:这篇文章主要是针对像我这样没有开发或技术工作的大三学生....还没有。
这是给那些必须花时间在家的你的。也许你是单身。也许你和我一样有家庭。但是你在找工作，只是想提高你的技能，或者像我一样两者都有一点。我希望我能通过我的不成功提供一些见解。以下是我的想法。

### **-什么工作-**

我从 Ruby 开始了我的编程之旅。我这么说是因为这是我遇到的第一种语言。它激励我去追求我的教育和追求软件开发的概念。不知道从何说起，就 googled 了一下“该学什么编程语言”，你大概也能猜到我当时的感受。在这个领域你可以做很多事情，但是我选择了 Java，因为那是我的教育系统选择的语言。很艰难。我做了一些基本的应用程序，学习了一些概念，比如输入验证和基本的 GUI 构建。我玩得很开心。不过说实话，我希望在拿到学位后能更上一层楼，比如 python，这样我就能在相对较短的时间内学到尽可能多的东西。我仍然希望如此。

你必须意识到你想用编程做什么。也许没什么特别的，但我认为选择一门语言或研究领域是一个好的开始。不管你的选择是什么，也不管这门语言有多难掌握，你很可能会学到很多可以用于另一门语言、工作或项目的相同概念。毕竟，如果你每天花在语言和框架上的时间有限，就像我决定开始这一切的头几个月所做的那样，你将一事无成。如果你每天花超过 1 或 2 个小时，你也许可以一次更好地应付多个学科。

### **-如何工作-**

现在是 2019 年，学习编码必须成为互联网上增长最快的概念之一。有大量的资源可供你选择。重要的是要明白，我们学习和工作的方式都不一样。你必须了解你的思想和习惯。在过去的两年里，我养成了上班前早起，然后编码的习惯。对你们中的一些人来说，做一个夜猫子会更有益，甚至可以在午休时编码。我想说每天花些时间练习至少是一个好的开始。可能要一个小时。它可能是十五分钟。仅仅是让你的思维进入程序员的思维方式，就会帮助你更熟悉你当时正在学习的概念。

### **-与谁共事-**

这是我的败笔。除了在大学老师指导下的学习，我其实没有太多的社团。当然，我会评论某人的帖子，或者在 twitter 上关注一个开发者...但是如果你不经常联系并真正讨论你学到的东西，你的成长将会严重受阻。开发人员在地下室里是一个臭烘烘、孤独的书呆子的刻板印象已经一去不复返了。软件编程是关于与他人合作来创造令人惊奇的东西。在 [GitHub](//github.com) 、 [dev.to](https://dev.to) 、甚至现实生活中的*上接触当地的编码社区。如果你身边没有，试着开始一个吧！你可能会发现，像我一样，即使在你可能居住的农村地区，也有其他程序员在寻求联系和协作。在这一点上，我对自己大喊大叫，所以这是为了你和我更好地与他人联系。*

### **-什么不行-**

去年我经历了很多失败的尝试，我不愿意从中吸取教训，我认为这在很大程度上导致了我精疲力尽，陷入了冒名顶替综合症，并且长时间不编码，仅仅因为我觉得自己是个失败者。我发现的一个大趋势是尝试 Twitter 上的[100 days ofcode](https://www.100daysofcode.com/)，如果你向下滚动我的 feed，你会看到它的证据。规则是你每天都要编码 100 天，有很多细节我就不多说了。除了偶尔空闲的一天，我脑子里有一个想法，如果我错过了一天，我绝对要重新开始，否则有人会发现我，指责我欺骗挑战。我会失败，会不好意思再发一条#day1 进度的推特，会太投入以至于根本不会编码。这在 2018 年的某个时间点重复了至少 8 次。当挑战的目标是让你去编码时，我不得不走到现在这一步，在这里我明白我因为放弃和对不一致的感觉而浪费了多少潜力。

这不是挑战的错。没有从错误中吸取教训是我的错...读者可能会多次听到这句话，你必须*应用*你从这些错误中学到的东西。要知道，每一次失败，都有机会重新站起来，从头再来。编程很难。许多比我更有经验的开发人员可能会更好地强调这一点。这并不意味着你应该退出。没有任何理由！如果这意味着在挑战中休息一下，或者甚至修改规则来帮助你成功...动手吧！明白什么不行，做点不一样的。这看起来是一个简单的概念，但我不得不通过我的固执来学习。

我知道我今年想要什么。作为一名开发人员取得进步。去读更多的代码，写更多的代码。加入我，让你的 2019 年成为进步的一年，无论你的目标是什么。
