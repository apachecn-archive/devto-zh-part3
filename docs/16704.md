# 开发者热身:重建网站，分解它，建立它

> 原文:[https://dev . to/nika codes/warm-up-for-developers-recreating-a-website-breaking-it-down-building-it-up-27dn](https://dev.to/nikacodes/warm-up-for-developers-recreating-a-website-breaking-it-down-building-it-up-27dn)

热身对于任何类型的剧烈运动都是必不可少的。运动员——无论是职业运动员还是业余运动员——在进行全面锻炼之前，都会先做伸展运动，然后从轻微的身体运动开始。音乐家们会在尝试下一首作品之前，用音阶和调音来预热他们的乐器和附件。艺术家们在投入当前的努力之前会先做热身草图。所有这些团体不仅仅是在准备他们的身体，他们也在为即将到来的更大的任务准备他们的思想。他们在尝试更艰难、更费力的挑战之前已经进入了一种节奏。在某些情况下，为了防止受伤，热身是必要的。

与前面提到的实践一样，编码也受益于不断的实践和热身。对任何网站开发者来说，一个很好的热身就是重建一个已经存在的网站。根据你的水平，你可以增加一些挑战来扩展你的代码技能。

> ### 专业提示
> 
> 尝试添加时间限制。一个完整的网站一小时能做多少？三十分钟？你想让你的模拟网站有多实用？

重建一个网站似乎是一项艰巨的任务，尤其是对新手来说。如果你是一个试图重建现有网站或从头开始创建网站的人，你可能已经意识到，即使是最简单的网站也可能需要数百行代码。

我希望通过这篇文章来帮助你浏览一个网站，快速分解它的视觉组件，并且(希望)让这个练习变得简单一点。

## 热身

所以你已经选好了你的网站，确定它对你的热身目标来说既不太难也不太容易，你已经准备好工作了！但是你从哪里开始呢？

对于本文的例子，我将查看 [Medium](https://medium.com/) 的主页并对其进行分解。

这看起来可能很简单，但是一旦你开始钻研代码，你就会意识到它是由 div 中的 div 组成的。其实你可能觉得是 [divs 一直往下](https://en.wikipedia.org/wiki/Turtles_all_the_way_down)。

你可能首先意识到的是，Medium 是一个无限滚动的网站。没有“结束”,但是只要你愿意滚动，它就会不断地产生文章。就本文的目的而言，我将建议添加一个页脚，并将内容区域限制在 10 篇文章以内。

## 打破它

当你第一次看网页的时候，把所有的内容看作一个整体。

<center>![](../Images/7aa01eb21ceee591f5384e1b724091cd.png)</center>

它是从网页的一边一直到另一边吗？当你一路向下滚动时，网页的顶部和底部是否有很大的距离？

对于 Medium，整个内容周围有明显的边界，所以在您的`<body>`标签中，您将使用 CSS 或 SASS 创建一个通用的`<div>`容器，它将是这些特定的维度。

那很容易。但是现在我们已经开始看盒子里面所有的图片和文字，对吗？没有。因为，我们还可以把这个大盒子拆成小盒子。

<center>![](../Images/eb6091812029748e8c60434d39a0d8b7.png)</center>

这更容易管理，我们已经在第一个更大的`<div>`容器中创建了三个更容易定义的`<div>`标签。

事实上，你的代码——或者伪代码——看起来应该非常类似于:

```
<body>
    <div id=”container”>
        <div id=”header”>
            Header
        </div>
        <div id=”content”>
            Content
        </div>
        <div id=”footer”>
            Footer
        </div>
    </div>
</body> 
```

所有样式和定位都将在连接到 div id 的 CSS 或 SASS 文件中完成。我建议在创建时设置每个`<div>`的尺寸和位置。当你继续添加更多的元素时，这使得解决问题变得更加容易。

现在，我们将分解每个部分的单个组件。记住:这是一路向下。

对于热身或练习来说，在代码中如何称呼每个部分并不重要。你必须记住你给哪个部分或分区命名了什么。如果您不得不回去修复某些东西或进行调试，那么确定以哪个 div 为目标将是一场噩梦。

我还将指出，特色部分实际上是三列，第三列被分成两个以上的部分。这完全取决于你愿意放弃多小的自己。

## 在实践中

利用这些练习和热身来试验一下，与多少个`<div>`或`<section>`一起工作你会感到舒服。多少才算多？你的代码什么时候开始变得难以理解？

这些问题以及更多问题的答案将取决于你作为一名程序员的喜好。

当你觉得你已经把你的网站分解成最小的、最容易管理的部分——关键词是容易管理——你就可以开始添加内容了。

我不建议在这个练习中过多地复制网站内容。尤其是如果你选择了一个页面定期更新的网站。然而，你需要插入文字和图片(如果网站有图片),以确保你是诚实的娱乐。

## 最后的想法

像任何肌肉一样，当你有规律地练习技能(比如编码)时，大脑会表现得更好。即使你没有完全完成，建立一个网站并尝试重新创建它会提高你作为一个程序员的技能；让你熟悉不同网站的不同布局；并为你提供一个开放的沙盒，让你在安全的环境中实验和挑战自我。

我希望这篇文章有助于让分解和建立一个网站变得更容易。
