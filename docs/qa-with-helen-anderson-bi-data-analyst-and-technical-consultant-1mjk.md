***交给海伦...**T3】*

### **1)请简单介绍一下您自己，您是如何进入数据领域的，您的数据之旅到目前为止是怎样的？**

你好！我是**海伦·安德森**，我是 [Xero](https://www.xero.com) 数据服务团队的数据分析师和技术顾问。我支持 Xero 的分析师社区进行代码审查，建立他们需要的数据集，维护他们工作中使用的数据库，并为初级分析师提供指导。

到目前为止，我的数据之旅并不是最传统的。

我在大学没有学过计算机科学，甚至没有学过 IT。获得商业学位毕业后，我进入了供应链分析领域。我真的很喜欢解决如何在正确的时间把正确的东西给正确的人的难题。根据顾客的需求、运输时间表，以及我的第一份工作——从新西兰向世界各地运输苹果——以及种植者估计他们会收获多少，制定一个计划。尽管我们做任何事情都使用 Excel，但它让我走上了现在的道路。

从那以后，我在新西兰皇家芭蕾舞团工作，根据剧院的座位配置进行收入预测，在伦敦的 T2 天美时为欧洲设计手表制造，在 T4 的破冰船平衡世界各地的库存。

三年前，当我加入 Xero 时，我开始从 Excel 转向 SQL 编码。我开始在营销团队做分析师，为电子邮件活动拉名单，做活动后分析。当谈到使用 SQL 时，我很晚才开始玩这个游戏，但我已经被吸引住了。现在，我在 BI 团队工作，负责支持那些初级和非技术分析师，他们是三年前的“我”,同时在项目中提高我的技术技能，为新的数据库平台构建数据模型。

### 在你目前的数据岗位上，你的一天是怎样的？(你使用哪些工具和语言？大团队/小团队/独狼？基于办公室还是远程？)

在我的职业生涯中，我第一次加入了一个由分析师、开发人员和工程师组成的大团队。数据服务团队大约有 30 人。这与我迄今为止担任的大多数角色中唯一的分析师有很大的不同。

当我第一次加入这个团队时，冒名顶替综合症对我打击很大。

当你是唯一的分析师时，你会做你知道的事情，因为除了 StackOverflow 没有人可以问。我反复思考这些感觉，并向我周围有多年经验的人寻求帮助和学习。我们有一个令人难以置信的团队文化，每个人都很乐意提供帮助，我们通过每月一次的大奖来庆祝每个人的成功。

我们有一个处理临时数据请求的分析师团队，一个构建报告中使用的模型的 BI 团队，一个维护数据管道和平台的数据工程团队，以及确保自助式可视化平台用户友好的 MicroStrategy 顾问。

因为我的角色是支持业务团队中的分析师，所以我可以做一点点事情。有时我会在 Redshift 中构建一个自定义数据集来支持分析师的工作，向他们展示如何使用 Microstrategy 来展示他们的报告，或者对将数据移动到我们维护的专用 Aurora 数据库以支持他们工作的气流作业进行故障排除。

### 通过博客，你已经拥有了一大批追随者。您认为数据专业人员在职业生涯的各个阶段公开分享他们的所作所为和所学有多重要？

上次我找工作的时候，我开始在自己的 Wordpress 网站上写博客。不足为奇的是，这里并没有挤满新的游客。

在看到一位同事在 Medium 上写博客后，我权衡了一下自己的选择，决定投身于 Dev.to。我真的很高兴我这样做了。我不像社区中的许多人那样是传统的 web 开发人员或软件工程师，但是我仍然可以提供一些东西。

我最初的几个帖子是“列表”，很容易阅读，但感觉有点太 Buzzfeedy。我重新评估了语气，找到了一个更像对话的声音。就像我和同事谈论一个技术主题一样，但是不需要重新创建技术文档。

博客在很多方面都是有益的。

你在强化你对所选技术或工具的学习或理解。所以即使你只获得了屈指可数的浏览量，你仍然做了一些有价值的事情。

这都是你独特的观点。这并不意味着你必须了解一个话题的所有内容。这就是为什么你在写博客，而不是重写技术文档。

即使你是初学者，解释你如何学习如何使用一个新工具，你的观点也很重要。你永远不知道谁会偶然发现你的帖子，并发现你对某个主题的解释有助于他们找到合适的位置。

### **4)您认为自己的数据职业接下来会走向何方？以个人贡献者的身份发展你的技术技能，还是进入一个更加基于管理的角色？**

我非常幸运能加入一个鼓励这两者的团队。现在，我正在积累我对 PostgreSQL 特性的知识，因为我支持的分析师已经将这种特性转移到了 Aurora 数据库。我正在努力更多地了解加载数据的数据管道和我们用来构建自己感兴趣的基础设施的 AWS 服务，并更多地了解我们的 DevOps 和数据工程师的工作。

随着分析师社区的发展，我期待着为初级分析师提供更多的支持，并将很快在我的第一次技术演讲中进行公开演讲。即使我没有转行做管理，我仍然能够支持和教导我周围的人。

### 你给有抱负的数据科学家的第一条建议是什么？

成为一名优秀的数据分析师或科学家不仅仅是粗制滥造 SQL 和熟悉数据库。重要的是要学会如何倾听利益相关者的意见，并确定他们从报告或仪表板中需要什么。在学习沟通技巧、解释数字背后的故事以及以最终用户最容易理解的方式呈现数据方面投入同等的精力。

通过磨练这些技能，以及建立模型和你的技术技能，你会走得很远。
