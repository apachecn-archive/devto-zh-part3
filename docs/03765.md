# 我从哪里开始？先说体验。

> 原文：<https://dev.to/yiddishninja/where-do-i-start-start-with-the-experience-26c3>

[![Carnival](img/3625268bb5af56461e0dc8ab85b99a5f.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--V_PGvjXQ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/yovttix1t7vkugv6j3g1.jpg)

在我最近加入的一个开发人员社区网站上，一名新成员询问从哪里开始为他们的学校制作一个约会应用程序。我给出了以下建议。

> 从让他们报名开始。这是怎么回事？
> 
> 他们是如何建立个人资料的？里面有什么？
> 
> 他们个人资料中的哪些数据是用来将他们与其他人进行匹配的？
> 
> 你对那个数据的匹配算法是什么？你向他们展示了他们比赛的哪些数据？
> 
> 他们是如何联系到自己感兴趣的比赛的？
> 
> 无论是 1999 年的 Match.com 还是 2019 年的最新约会应用，这些问题都普遍适用。
> 
> 一旦你有了这些答案，你的应用程序的架构和支持它的后端服务就会变得更加明显。

任何 app 都是从体验开始的。客户通过你的 app 的旅程是什么样的？

从客户第一次接触你的应用开始。根据客户看到的内容，他们下一步该做什么的线索，以及他们如何完成一项任务，比如签约，写出步骤。不要跑题谈论他们可能进行的所有可能的附带旅行。从头到尾绘制一个简单易行的旅程。然后再多图几张，足够覆盖一个简单的 beta。

不要拘泥于设计适合你所开发的技术的客户体验。设计您的技术，为您想要的客户体验提供动力。从体验开始，技术就会显现出来。
