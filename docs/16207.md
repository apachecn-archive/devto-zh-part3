# 教育机器人

> [https://dev . to/third nexus 6/机器人-教育-31l2](https://dev.to/terceranexus6/robtica-educativa-31l2)

当时是瑞士的夏天，为 14 ~ 18 岁的青少年学生开设了一门“T0”机器人和编程课程。其中一个小组正在给我展示他们的评估演示，主题是什么样的机器人会用他们在课堂上学会使用的免费硬件来建造，以及对技术部分的简要解释。我的学生们非常激动地向我展示了一个民防战斗机器人，只不过是一个变压器“*免费*。他们给我设计了一个原型用箭头标记了伺服马达传感器胸口的火石。正如各种项目向我展示的，不同班级团体的发明，我们笑了很多，但最重要的是，很多人对**在家里进行自己的项目**感兴趣。

一个月前，他曾在格林纳达大学的一个校园里参加过一个类似的课程，但仅限于年龄相似的女孩。我的小组必须**利用 arduino(自由硬件)构建指挥和视频游戏**，研究一种游戏界面、物理指挥界面的设计，具有基本的挑战性；就像所有电子游戏一样。结果很有趣，在我们解决常见编程和 arduino 连接错误的时候，很多关于电子游戏的评论都被发表了。

我的学生比西班牙那边的学生小一点。通过视频呼叫，我们致力于 arduino，学习编程、基本网络安全和安全导航。但是他的主要兴趣是硬件，所以我们把他的烧成灰。我们通过一点一点的练习来学习，每次练习时都会庆祝，直到我们得到了一些更大的项目，比如带纸板和仆人的机器人手。有反复出现的错误，但我们从代码和硬件两方面都学到了‘t0’【debug gear】，我们总是问自己同样的问题。*"∞电缆这是否布置得很好？"，"∞我们漏了一点吃了吗？"，“我们是否连接了正确的端口？”*。

我在一个机器人班做了一个紧急补课，学生年龄在五到十岁之间。 **Arduino 是我们的主要工具**，但最重要的是了解电子不是魔术。我在一个小黑板上画了膝盖的基本连接，用各种颜色。我们正在研究世界上可以用电子、传感器和演员之间的区别做些什么。并且我们设法用 scratch(一种旨在学习编程逻辑的编程语言)和 arduino，**做了一个小小的红绿灯，有时你需要寻找三四种方式来解释或问同样的问题，但最终我们能够理解**，结果是庆祝所有的团体都有一个 led 红绿灯

技术存在于我们生活的每一部分，是我们生活的社会。它在世界上有分量，了解技术有助于我们了解如何在其中发挥作用。**计算不再仅仅是技术专长的问题，而是世界的一个基本部分**。新一代人是在显示器和技术的包围下诞生的，必须认识到，这是通过逻辑和电子技术而不是魔术来实现的。我们会惊讶地发现数字原住民能教给我们的一切，但为此，我们必须向他们展示基础。幸运的是，现在有许多工具可以帮助这项工作，这些工具已经得到清理，使计算机更接近基础教育。当我学习编程时，我不得不在网上找一门课程，我不得不反复复习，因为我不太明白-我...。一些比我更有经验的人更难，也许有一本基本手册和一个频谱。幸运的是，几年后在高中，我的计算机老师澄清了我的疑问，并记录了我的知识，通过他巨大的耐心和干劲，他作出了巨大的努力来教我们 c(现有最不友好的编程语言之一)。

很容易认为，我们从一开始就很难理解的事情，是向一个孩子解释这一点是可行的，但我在讲课中学到的是**在教学上的居高临下最好离她远点**。令人惊讶的是，他们有能力了解到他们有一些孩子，而且有了适当的工具，他们能够理解那些对我们来说代价更大的概念。其中一些工具是我想在本文中介绍的，重点是机器人。

没有编程，电子和机器人就没有意义。这一切都是一项挑战，因为在机器人教学中需要考虑两个不同(虽然是相关的)部分:**编程和硬件**。编程，如果我们基本上保持不变，这是合乎逻辑的。同样的逻辑使我们建造一条道路，过马路或玩游戏，只是用一个有规则的定义而不是抽象的概念。但是直觉起着重要作用，它不仅是理论，而且是一种表达我们已经可以推断的东西的方式。当试图通过正式的编程语言(如 c、C++、python 或类似语言)来表达这种逻辑时，我们可能会遇到一堵附加的困难墙，那就是了解该语言的结构。就像西班牙人学习一种语言时，我们学习中文的难度更大，中文的结构与法语的结构完全不同，我们对法语的了解程度也不同。为此，几年前出现了**【scratch】**，这种编程语言旨在学习直观编程。

[Scratch](https://scratch.mit.edu/) 是由块组成的，这些块作为片片排列在一起，并标有“重复同时...”等基本编程动作，“如果……”，“不做……”和类似的东西。不同的功能以不同的颜色区分，因此您可以快速了解正在构建的程序类型，并且可以轻松地通过 d 抽象来识别错误。它虽然简单，但非常灵活，允许从核心方案到更复杂的结构。当使用 scratch 达成松散时，切换到上述任何一种语言并跳过正规计算要容易得多，因为它们最终都是基于它们最基本的结构中相同的规则。**回到硬件**、arduino 等类似的微控制器可以用 scratch 编程，极大地方便了教育者的工作。一旦学生们了解了编程的基本动态，我们就进入了下一个挑战，他们了解到同样的**逻辑如何影响有形的东西**，例如 arduino。最简单的开始方法是通过运动或光。如果只有一个基本 arduino 套件，最简单的方法就是关闭并打开 led。它需要最少的连接，并且非常图形化。

但是，在机器人教学开发的许多工具中，根据学生的年龄和知识，有广泛的选择。一个很常见的例子是**scornbot**。我第一次发现[学校机器人](http://escornabot.com/web/)是在格林纳达大学计算机工程技术学校开设的教育机器人课程中。他们给我们看了一个有轮子和五个按钮的小机器人。按钮是右、左、上、下、开始。然后他们把一个大格子放在地板上，有一个正方形开始，我们可以选择一个正方形结束。**机器人的想法是用按钮设计一条从开始网格到结束的路**。这一幕很有趣，因为我们的学生对教学感兴趣，他们和几个小时前教我们操作系统、代数或编程的老师坐在一起玩儿童机器人。但是这个想法变得很清楚，我们不需要数学的高概念来编程，这样一个简单的想法迫使你抽象、思考一个计划、行动、纠正和寻找不同的选择。

[![](../Images/7c732c8c1fef5233527303c4cc972a5b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--laTNFbVd--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://orballoprinting.com/398-thickbox_default/pipop-escornabot.jpg)

当你把这个玩具介绍给一个幼儿教师时，建议他们编造一个伴随机器人的故事。例如，一个宇航员及其机器人在一个裸体星球上迷路了，不得不从网格中躲避洞穴进化，直到到达能够发出信息的基地；一辆想到达目的地并在途中停下来买水的汽车；一个试图穿过花园的同性恋。极限在于教育者的想象力，但最终通过游戏，最小的人能够学习基本的编程方法。

还有其他的玩具能让我们到达类似的东西，例如最近我试了“**【Scotty go！**。"[史考特走！](http://scottiego.vicensvives.com/es/el-juego/)是模仿涂鸦块的卡片组。其想法是建立功能和程序，然后使用应用程序扫描和启动从仪表板构建的程序。app 是一个游戏，围绕着一个可爱的角色，一个叫 scottie 的机器人在地球上强制着陆，需要修理他的船。这就需要通过一些需要 Scratch 编程知识的测试。一开始，它会引导你自己的应用程序，逐渐给你的想象力留下更多的工作。游戏玩得很好，因为虽然编程逻辑与 scratch 的数字版本相同，但他们有有形的纸板部件，可以触摸、分散在桌面上或移动，有助于更快、更轻松地学习。对于儿童和小学儿童来说，当他们能够与手互动时，他们更容易记住概念，而且他们也有游戏形式的目标。

我最近在[采访](https://elpais.com/elpais/2019/01/02/escaparate/1546438642_004492.html)中评论道，用 arduino 和其他灵活的玩具/微控制器如[微比特](https://microbit.org/co-uk-landing/)进行教学的好处之一是可以定制的。如果我们的小女儿喜欢恐龙，我们可以通过创建一个简单的机器人和仆人，打开和关闭纸板恐龙的嘴来找到链接。如果我们的孩子对魔法感兴趣的话你可以找个方法做一根魔杖只要动一下就能激活一盏灯。所有这一切都是因为自由硬件使我们能够从头开始寻找和学习其他人以前完成并一步一步共享的各种项目。也许这需要我们从孩子身上学习，如果我们以前不懂得编程或电子，但幸运的是有很多材料。在[这个存储库](https://github.com/terceranexus6/clases_con_rafa/tree/master/apuntes_curso_nuevo)里，我把我所有的课堂资料都放在网上了。在[这个博客](https://sosarduinoprojects.wordpress.com/)中，有和 arduino 白皮书相同的项目，也有一些提示，而[在](https://thearduinoassistant.wordpress.com/)这里也有。这些是一些西班牙语的，如果你懂英语的话，还有更多。

归根结底，我们非常幸运的是，有一些工具是为从小就能教授和学习机器人和计算机而设计的。这是一个很好的机会，可以开始与最小的孩子学习一些新的东西，或者无论如何，给他们一个机会通过游戏了解他们的世界是如何运作的。
