# 到处都是开发技术是好是坏？

> 原文：<https://dev.to/leoat12/is-it-good-or-bad-to-be-all-over-the-place-with-dev-technologies--1k47>

2018 年对我来说是疯狂的一年，这是我作为开发人员职业生涯的第一年，在此之前，我更多地处理网络基础设施和管理。我是如何从一个系统管理员的职业发展到开发人员的，这是另外一个故事了。在开始真正从事开发工作之前，我从来没有深入过开发领域，我还在大学学习，准备毕业，没有太多的空间，特别是因为这不是我的第一选择。但是我被开发世界吸引住了，一年前，我在这里写了一篇回顾，并就我同时学习许多东西的渴望征求了一些建议。

让我们从头开始，我被一个 Java Web 开发人员的职位录用了，而且，我必须承认，在面试前几天，我对 Spring Framework 一无所知，我是说真的，我并不因为说了实话而感到羞耻。我在大学时就知道 Java，过去也开发过简单的 web 应用程序，但那是用 C#编写的，尽管 MVC 的概念是一样的。这帮助我快速理解了 Spring、JAX-RS 和其他 Java 框架。幸运的是，我被录用了，我开始在公司内外深入研究 Spring。Spring Boot、春季安全、春季数据、春季会议等。不过，我还有很多要学的。

同时，该公司也是 AWS 的合作伙伴，我们通常在 AWS 上部署我们的应用程序，对其服务的了解是至关重要的。于是，我也开始以获得认证为目标学习 AWS。经过几个月的学习，我通过了认证考试，现在我是一名认证开发人员。它对我很有帮助，让我有信心提出对 AWS 现有内容的改进，很高兴看到这一点有了结果。

只是这已经是需要消化的大量知识了，Spring 和 AWS 有一些共同点:它们都很庞大，有很多模块、服务、复杂性和细节，从简单的 web 应用到安全、数据处理等等。我还花了一些精力来理解 AWS 上的 DevOps，这并不是考试的一部分。我也喜欢无服务器，因此花了很长时间用 AWS Lambda 和 DynamoDB 做实验。

因为我在用 DynamoDB 做实验，所以我也看了一下 MongoDB，看看它们有什么不同，我发现 DynamoDB 在我的实验中有一些限制，我不得不看一看其他解决方案，看看它们是否更符合我的用户情况。现在我正在尝试 Elasticsearch 和 Logstash，也很喜欢，在我看来，它会对未来的项目产生非常好的影响。

然而，在我进入当前的 Elasticsearch vibe 之前，我也有前端 vibe，我担心我的 HTML，CSS 和框架知识，如 Angular，VueJs 和 React，我还发了一篇关于[全栈开发](https://dev.to/leoat12/be-or-not-to-be-a-full-stack-developer-2ko)的帖子，我过去和现在都担心如何成为一名全栈开发人员。我第一次使用 Angular 的经历并不愉快，但最近，在花了这么多时间制作 web 应用程序之后，我在 Spring 和模板引擎(如 JSP 和百里香叶)方面的许多经验被证明对更好地理解 Angular 非常有用，现在我回到 Angular 并为个人项目开发 UI，也许很快我会在这里谈到这一点。

除此之外，我现在还在试验 Flutter，这是谷歌最新的移动开发框架。今年我还用 Java、NativeScript 和 Xamarim 进行了原生 Android 开发！颤振一直是一个惊喜，我认为这种氛围将持续到明年。

所以，在经历了这一切之后，我认为有可能理解我的担忧是什么:我到处都是！我从数据库、移动和 web 框架、云提供商等方面尝试了很多东西，但没有深入其中任何一个，除了 Spring 和 AWS，这实际上是我日常职业生活的一部分。

你对此有什么看法？我应该更专注吗？实验这么多不深入可以吗？你有什么经验来处理我们现在拥有的大量技术，以及像我们这样的 IT 人天生就有的好奇心？
