# 我应该做那个测试作业吗？

> 原文：<https://dev.to/asizikov/thoughts-on-test-assignments-3o0j>

测试作业并不是一件好事。其实完全相反。这可能是招聘过程中发生的最糟糕的事情之一。

总的来说，招聘过程在很多方面都存在问题。不太清楚如何雇佣一名工程师。测试我们技能的新方法来来去去。它告诉我们，在 30-90 分钟的时间内检查受访者的技能几乎是不可能的。

我不打算谈论关于编译器的各种难题或棘手的问题，也不打算在这里讨论任何压力面试实践。今天是关于测试作业的。

# 从公司的角度来看

招聘是一个非常昂贵的过程，而且也不太可靠。几个小时的采访一点也不免费:

*   一两个工程师进行面试的一个小时
*   面试前准备 20-30 分钟，面试后整理结果
*   招聘人员的时间

如果你把它加起来，你会得到一个很好的数字。期待企业削减成本是合理的。有很多方法可以让它变得更便宜:电话筛选和测试在行业中非常典型。

有一种方式比其他方式更突出。给候选人的任务。一般来说，这是一段很无聊的代码，可能会有一些技巧。有时，你不得不在电话筛选阶段之前就这样做，作为额外的过滤器。大部分时间你都是在现场查房前服用的。

通常假设你会花费(我应该说浪费吗？)几个小时完成作业。这是招聘经理或工程师做出的估计，而且是基于你对每项技术和任务的每个方面都很熟悉的假设。

检查你的代码需要多长时间？嗯，大约 15-20 分钟。看起来不公平，对吧？

没错。一家公司试图把费用记在你的名下。他们不知道如何在简短的面试中评估你的技能和经验，他们也不知道如何通过电话来评估，因此你需要花几个小时来说服他们只是谈谈。

顺便说一句，一个工作是评估你的任务的人是不会付出太多努力的。他们有足够多的事情要做，他们真的不喜欢转移注意力。想象一下一遍又一遍地看着相同的代码。他们通常会快速浏览一份清单。

# 一个考生的观点

除非你的目标是像谷歌、亚马逊或脸书这样的顶级科技公司(顺便说一句，他们不会真的给出测试任务)，否则你不会渴望得到一份特定的工作。绝大多数公司都在做类似的事情:将表单输入存储在数据库中。嗯，算是吧，但你知道我的意思。

对于考生来说，考试作业不是很有用。一方面，它们常常彼此相似，另一方面，它们很难被重用。每家公司都试图让它有点特别。你必须开始一个新的项目，配置所有的部分，选择一个 ORM 或建立一个 linter，一遍又一遍。

说到...你多久在工作中开始一个新项目？就像从一开始。你知道，一个`git init`和`File->New Project`。在做各种各样的测试任务时，我已经这样做了几十次，在工作中也有很多次。大多数时候，你要么加入一个现有的项目，要么开始一个项目，并长期坚持下去。所以，老实说，在我看来这不像是相关技能。

# 你的时间成本是多少？

想想吧。你超龄一天有多少工时？如果你年轻且单身，你甚至可能拥有全部 24 种。你不能长时间保持这种状态，但你可以坚持一两天。然而，当你长大或有了家庭，情况就不同了。你喜欢什么？每天 12-14 个小时。一天剩下的时间你做家务和睡觉。

你必须花八到十个小时在工作和通勤上。那给你留下大约四个小时的空余时间。所以招聘经理会利用这宝贵的四个小时。他们想让你把所有可用的资源都花掉，这样他们这边就能省点时间了。

一个公司有多少资源？那里有数十名工程师、经理和人力资源专家。如果我们谈论一个中小型公司。所以他们有数百个工时可以支配。额外的 20-40 分钟来评估候选人似乎并不太重要。

这是一个明显的不对称过程。你的时间比公司少，但前期要花更多的资源。

我真的不知道我们该怎么办。这取决于每个人。如果你有信心，如果你珍惜你的时间，并且你正在申请一家超龄公司，你可以拒绝把任务带回家。有很多方法可以做到这一点，你可以要求他们支付你的时间。你可以让他们先看看你的投资组合。给他们看看你的博客，你的公共项目，如果你有的话。让他们下载一个你做的 app，先试用一下。你觉得怎样都行。

然而，你可能无法拒绝。你可能只是没钱了，或者你可能在谈论一份理想的工作，你不想因为骄傲而毁了它。我明白。

尽管我建议你在某个地方发表作业。如果我们谈论代码，就让它成为 GitHub 吧。、或运球、或在你领域内专家中流行的任何其它地方。首先要明确一点:你要发表它，因为你拥有你的工作成果。除非他们想买下它。

希望有一天你会得到类似的任务。给他们回个链接就行了。这可能行得通，而且可能会节省你一些时间。

# 不会那么差吧？

良好的...让我们看看。你还记得那时候你花了很多时间，做了一个完美的东西，整洁而又精致？之后你就被雇佣了，没有任何问题。呃，那只是个梦。

不幸的是，通常一个完美的结果只是一个和你见面的理由。这就是真正的面试的地方。他们会问你一些问题，以确保你是作者。有时候这感觉像是一个不必要的步骤。他们可以通过电话给你一个简单的 15-20 分钟的测试。

想象一下，你发现你不喜欢办公室、团队或公司的其他任何东西。你知道，有些事情只有在面对面的时候才能发现。为了达到这个目标，你已经花了几个小时在测试作业上。真是浪费...

支持考试作业的又一个理由。他们应该让有时间参加博客或 OSS 贡献等公共活动的候选人与有家庭的人或由于工作合同限制而不能发布任何代码的人平等。老实说，我不知道这有什么用。如果你没有时间做你喜欢的项目，为什么你会有时间做测试作业呢？

你怎么想？你经常在实际的现场面试前看到一个带回家的作业吗？
