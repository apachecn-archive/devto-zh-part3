# Ionic 与 NativeScript - JavaScript 移动框架比较

> 原文：<https://dev.to/manojrawat77/ionic-vs-nativescript---javascript-mobile-frameworks-comparison-2mn>

### 比较以确定每个框架的优势

应用程序和网站的开发方式已经彻底改变了。由于系统和方法的发展，用更通用的语言来构建软件，在相同的环境中工作的困难更少了。开发人员努力创建了几个框架，在这些框架中他们可以使用 JavaScript 来控制原生 iOS 和 Android 应用程序。有不同的框架可用于开发移动应用程序，其优势集中在每个项目的需求和特征上。

因此，区分哪一种最适合您的应用非常重要。要了解哪种框架适合您的项目，您必须确定要使用的编程语言，以及它将如何影响应用程序的性能。

在这一点上，有必要记住，当我们谈论原生应用程序时，我们指的是那些为特定平台(Android、iOS 或 Windows Phone)构建的应用程序。这使得开发更有效，应用程序的整体性能更好、更可靠。然而，原生应用的问题在于，必须为每个平台开发不同的版本，因为编程语言表示其他系统下的不兼容代码。这种努力增加了许多倍的开发成本。

尽管混合应用程序通过 C#或 JavaScript 等语言满足了这一需求，但它们仍然依赖于原生浏览器，这也降低了它们在复杂任务中的速度和性能。出于这个原因，今天我们将比较两个 Javascript 框架 Ionic 和 NativeScript，以评估每个框架根据手机需求提供的优势。

### 离子的

它是一个开源 SDK，利用 HTML5、CSS 和 JavaScript 提供的优势，为混合移动应用程序的开发而创建。Ionic 的性能非常出色，它允许创建能够在 iOS、Android、Windows 和网页上运行的应用程序。由于这个原因，它在印度的应用程序开发公司中需求量很大。

Ionic 依靠用户界面来提供对安装在主机上的操作系统的功能的访问，并从由所需的附加资源(如 Cordova 或 PhoneGap)支持的浏览器运行。

### 原生脚本

它是一个开源框架，能够使用 JavaScript 或任何其他兼容的语言(如 TypeScript)创建完全本机的应用程序。使用 NativeScript 开发的应用程序使用相同的 API，就好像它们是在 Xcode 或 Android Studio 中创建的一样，这是该框架被印度多家软件开发公司广泛使用的一大原因。

### 易于使用

任何有应用程序的工作环境都需要 web 开发的基础知识，尤其是 Javascript，然而，对于那些没有什么经验的人来说，学习处理每个框架可能会有很多困难。虽然 Ionic 显然为用户提供了更多的指导，但与 NativeScript 相比，它更容易使用，也有许多有用的文档可以帮助，而 native script 并不建议没有经验的用户使用。

### 表演

虽然 Ionic 更容易学习，但它的性能在创建应用程序时会大大降低，尤其是那些需要复杂任务的应用程序(例如游戏)，因为它们必须在 WebView 中运行。另一方面，NativeScript 在应用程序的性能方面提供了更大的优势，与 Ionic 相比，差异是显而易见的，尽管这些的权重要大得多，因为它是一个原生环境。

### 商业知名度

Ionic 无疑在市场上击败了 NativeScript，有很多应用程序、插件和亮点，如谷歌地图，而竞争对手提供了近 1000 个插件的目录。

### 代码库的有用性

让我们记住，Ionic 是一个为开发包装应用程序而创建的框架，因此，在重用代码库时具有很大的优势。然而，NativeScript 与 Angular 团队一起构建了一个能够从单个项目开发应用程序的方案，从而简化了程序员的工作。

**结论**

根据项目的需要，这两个框架都提供了有用的优势，这就是为什么了解要开发的应用程序的类型以做出正确的决策是很重要的。
