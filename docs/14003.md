# 为什么你应该给…思想？

> 原文:[https://dev . to/Irina _ scur tu/why-you-should-give-a-think-2h 94](https://dev.to/irina_scurtu/why-you-should-give-a-thought-2h94)

**开发人员注意事项**

久而久之，我们对我们正在进行的项目、团队、管理、技术、领域和我们周围的一切都很满意。

我们发现自己在日常工作中有足够的快乐，不再寻求挑战。时间飞逝，变得稀缺，我们学习的意愿减弱…

但是，几年后，当技术趋势发生变化，而我们不久前停止学习时，会发生什么呢？

术语“高级开发人员”将有其他的价值，除了…老戴夫？

很多经验丰富、对工作充满热情的开发人员不敢站出来举办研讨会或讲座，也不敢参与教学活动，因为他们认为自己没有公开演讲的技能，或者他们感兴趣的话题不够酷。嗯……他们肯定有很多专业知识，你猜怎么着？

**不练，那些技能是无法完善的！**

说实话，‘资深 dev’应该是指比你落后 xx 年以上。资历也应该由你愿意分享你的知识，培养你周围的人，以及你参与教学和指导他人的程度来决定。

你应该承担起以身作则的责任。你不应该等着别人带头，参与进来，向别人展示这一点，如果你可以，他们也可以成为灵感。

我们大多数人都参与面试未来的同事，我们经常抱怨找不到合适的人，因为他们的专业知识不合适。

你有没有想过帮助别人成长，让他们在你的面试中做更好的准备？或者更好的是……决定其他人加入你的公司仅仅是因为**你在那里，**因为你可能会对他们的职业道路产生影响？

> 对知识的投资总是会有最大的回报。
> 
> 本杰明·富兰克林

技术诀窍是一种即将到期的资产。新技术一出现，或者新版本一发布，它就过时了，所以你可以试着推迟它的到期日。

**参与**的第一步，是**成为技术社区**的一部分，在那里你可以有一个完美的环境来围绕技术或项目类型改变信息、想法、经验和“经验教训”,为什么不呢，结识新朋友。

这种环境让任何成员都有机会了解其他人在特定情况下的做法，他们面临的问题是什么，选择或应用一种解决方案比另一种解决方案有什么好处。

是的，我们都知道所有的问题都可以在谷歌上找到答案，但即便如此:和真人见面不是很酷吗？一个你可以提问、辩论或为什么不可以的社区，一个你可以围绕商业领域或具有社会影响的项目进行头脑风暴的社区？

实际上，一个社区可以教你如何不重新发明轮子，并停止做别人的错误，学习最佳实践，书面或不书面的，发现新的行业趋势，技术，并发展你的技能。

**技术社区是人们和他们的激情之间的联系**，这超越了他们今天工作的公司。

出于这个原因，你应该加入**右社区，**并通过参加他们的聚会来投资它。

什么样的社区适合你？

嗯，首先，它应该是一个由人而不是由公司驱动的社区。由一家公司发起的“meetup”之类的活动通常是出于招聘或营销目的，并且通常会直接受到预算的影响。无论这些“聚会”背后的议程是什么，meetup/社区都是关于人、学习、分享知识的意愿、成长、更进一步、结识新朋友、领导力、志愿服务和许多事情，但不是一家公司。
