# 无代码工具被忽视的好处

> 原文:[https://dev . to/coda/the-overlapped-benefit-of-no-code-tools-41p 3](https://dev.to/coda/the-overlooked-benefit-of-no-code-tools-41p3)

我不是产品设计师、创业企业家或工程师。七年前，我无意中戴上了所有这些帽子，因为我想创造一些东西。一些我可以在网上给朋友看的东西。能产生收入的东西。最重要的是，我可以把*称为我自己的*。

我一直是一个 Excel 迷，并决定创建一个由硅胶制成的物理键盘盖，将向用户教授 Excel 键盘快捷键。我可以做 100 件不同的事情来开始这项努力，但在我看来，最重要的第一步是设计产品。那是 2012 年，我本来可以拿起 Adobe Illustrator 或其他一些线框图工具，但我最终拿起了一个我已经知道如何使用的工具:PowerPoint。

没错，好老的微软 Powerpoint。我已经在工作中一直使用它，并认为知道如何使用 PowerPoint(尽管它不是为产品设计而设计的)的好处将超过像 Adobe Illustrator 这样更合适的工具的学习曲线。几个月后，我有了第一批库存，建立了一个基本的网站，我的业务开始运行。

### [](#building-for-the-community)为社区建设

[![](../Images/89cd72d91d0678a4fe5af75772ceee3b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s---VZmIF9B--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2AemRJnHyPG0NLGQSx4FYuTA.jpeg)

就在我推出我的产品的时候，社交媒体开始兴起，我看到了一个产品如何成为“病毒”对于我自己的产品，我经历了一些，但对我来说更有趣的是我接触到的社区。我开始写博客，给论坛投稿，创建在线课程，反馈循环让我坚持下去。我意识到我的业务的主要焦点不是产品，而是围绕它形成的社区。我认为许多创客在构思项目时都考虑到了一个特定的社区。这都是关于找到一个特定的问题，并建立一个解决该问题的解决方案，对不对？虽然社区建设很棒，但就我个人而言，我想从我的项目中得到更多。

我写了[一篇文章](https://makermag.com/2019/01/12/taking-things-apart/)关于为什么拆开东西是一个制造者的重要技能。我认为，制造商也喜欢把不同的“零件”组装在一起，创造出比零件总和更大的东西。这是缺失的部分。

无论怎么想象，我都不认为自己是艺术家。但是当我在 PowerPoint 中用方框和圆圈按钮创建产品设计时，这个过程的一个方面让我感觉自己像一个艺术家。说起来很奇怪，但是 PowerPoint 让我觉得自己很有创造力。如果你是一个创造者，这种创造性的破坏、实验和重建可能是你在使用像 PowerPoint 这样简单的工具时的普遍感觉。

对于那些非技术人员来说，我相信这些无代码工具给了你一种前所未有的表达方式。Ryan 指出“你不再需要成为一名程序员来在互联网上构建东西，这赋予了一批来自不同背景和视角的新的创造者。”

为了更进一步，这些无代码工具为制造者释放了一种新的技能:创造力。正如 Ryan 所说，只有一小群软件工程师在学会如何编码后能够在互联网上进行创作。现在，所有背景和经历的创客都可以感受到从设计一直到发布网站或应用程序的创意是什么样的。

无代码工具的一个副产品是它们给工作场所的制造者带来的创造力。如果你以前在一家公司工作过(或者现在在一家公司工作)，你可能会对你的 IT 部门强迫你使用的工具感到沮丧。我之前写过一篇关于我的公司让我使用的一个非常旧的时间追踪软件的文章。该软件不仅笨重，而且更新速度很慢，因为需要从他们合作的供应商那里获得新的许可证。那是一片混乱。

如今，许多员工通过使用无代码工具构建自己的*版本*的时间跟踪软件来解决这个问题，这样软件就可以完全按照他们想要的那样工作，并且他们可以在不需要 it 帮助的情况下轻松更新。我认为 Google Sheets 和 Excel 是第一批无代码工具，大多数人使用它们来构建他们需要的工具，而不需要代码。

想想你在 Google Sheets 中看到的所有日程跟踪、招聘跟踪和项目跟踪。这些都是微型应用程序，可以用一些 Excel 公式的基本知识来构建。无论您是业务分析师还是产品设计师，现在您都有能力在公司内部进行创新，使用大量的非代码工具构建应用程序。您可以构建适合您工作流程的应用程序，而不是依赖它来购买定义您工作方式的昂贵的 SaaS 应用程序。

### [](#nocode-stacks)无代码栈

如果你以前使用过 Excel 或者 Google 工作表，恭喜你！你是一个使用无代码工具的制造者！您可能想知道如何使用一些非代码工具来释放自己的创造力，而这要从了解哪些非代码工具开始。Ryan 的[文章](https://medium.com/@rrhoover/the-rise-of-no-code-e733d7c0944d)提到了一些，Colin Winhall 也在[这篇文章](https://makermag.com/2019/01/05/no-code-movement/)的 Maker Mag 上列出了一些工具。

构建您使用的无代码工具的一种方法是在您的项目中需要有“堆栈”。栈是从软件工程师和系统架构师那里借来的概念。例如，过去许多 web 开发人员使用的常见堆栈是 [LAMP](https://en.wikipedia.org/wiki/LAMP_(software_bundle)) 。 **L** inux 作为操作系统， **A** pache 作为你的服务器， **M** ySQL 作为你的数据库， **P** HP 作为你的编程语言。对于用无代码工具构建的数亿开发者来说，堆栈看起来像什么？

当您开始构建自己的应用程序时，需要注意一些关于无代码堆栈的事情:

1.  **有些工具可能已经过时了** —在 Bram 的无代码堆栈中，Gmail 和 Google Analytics 已经存在很长时间了。当与无代码工具结合使用时，它们会变得更加强大。一些例子:当一个新行被添加到 Google Sheets 时，Zapier 可以在 Google Analytics 中运行一个报告来返回数据。在[结尾](https://coda.io/?utm_campaign=maker-mag-feb2019&utm_medium=blog&utm_source=makermag)中，当你点击文档中的一个按钮时，你可以自动[给某人发送一封电子邮件](https://help.coda.io/connect-your-doc/packs/buttons-and-packs#gmail---send-email-or-create-draft)。
2.  **混合搭配工具** —这可能是许多制造商的既定目标，但你应该不断尝试不同的工具，以找到适合你的堆栈的工具。随着 API 和[数据的开放](https://dev.to/coda/increasing-productivity-by-opening-your-data-with-apis-2jc6)，大多数平台优先考虑与其他服务的集成，而不是围墙花园的方法。这意味着，如果您想与另一个工具集成，您正在使用的无代码平台有可能具有与其他工具的即插即用能力。
3.  **无代码工具倾向于“多合一”** —这似乎与最后一个要点相矛盾，但无代码工具之所以工作得这么好，是因为它们不依赖于其他应用程序的良好运行。如果你正在制作 we 网站，并使用 LAMP stack，如果你仅仅依靠 Linux 或 MySQL 做*的话，你真的很难做任何事情。您需要栈的所有四个部分来启动和运行您的站点或应用程序。今天的无代码工具(从原始的 Excel/Google Sheets 到现代的 [Coda](https://coda.io/?utm_campaign=maker-mag-feb2019&utm_medium=blog&utm_source=makermag) )将数据库、计算和前端都放在了平台本身。用于通知、自动化和任务管理的插件使得在同一个无代码平台上管理您的堆栈变得更加容易。*

### [](#corporations-are-taking-notice)企业正在注意

在无代码领域，很少看到独角兽的诞生，因为无代码工具仍处于起步阶段。制造商仍在为他们自己的项目构建和测试平台，但在大型财富 500 强公司注意到这一点之前，你可能会在你的公司里一直使用一些过时的甲骨文或 SAP 的时间跟踪软件。“没有人会因为选择 IBM 而被解雇，”他们说。

但是大公司已经注意到了无代码运动。有一个名为 OutSystems 的“低代码”平台，它在去年筹集了 3 . 6 亿美元的风险投资，巩固了他们的独角兽地位。该公司于 2001 年在葡萄牙成立，可能几经周折才走到今天。

这对在无代码平台上构建应用程序的开发者来说意味着什么？如果你决定在一家大公司做全职工作，你的技能很受欢迎。“精通微软办公软件”或“了解 XYZ 客户关系管理系统”已经成为过去。无论你去哪里，制造者的适应性、创造性和用低代码工具解决问题的技能现在和将来都是需要的。你知道什么比在简历上看到你知道微软 Office 更让雇主兴奋吗？一份你已经启动的项目、网站和应用程序的清单，表明你可以从头开始创业。

### [](#a-perfect-environment-for-creativity)创造创造力的完美环境

正如 Ryan、Bram 和 Ben 向我们展示的那样，使用无代码工具进行构建显示了比以往任何时候都多的制造者有了发挥他们创造力的途径。不仅仅是绘画或创作歌曲方面的创造力，而是营销、产品设计和运营方面的创造力。如果你在运营部门工作，你上一次在管理库存或传递客户请求方面感到有创造力是什么时候？很有可能你在工作中制作的超级光滑的 Excel 电子表格是唯一给你许可去创建你自己的和你公司其他人可以使用的东西的东西。

对于创客一代，无论你是朝九晚五还是自由职业者，这些无代码工具都允许你修补和试验，直到你为你的用例构建出完美的工具。这里的关键是，你必须愿意修补和试验，当然偶尔也会失败。

在纪录片《寻找伟大》中，导演讨论了“无组织游戏”的概念，以及规则如何阻止孩子长大后成为伟大的运动员:

> 最近有一项关于刚刚赢得世界杯的德国足球队的研究，研究了入选国家队的球员和比他们低一级的球员的发展道路。他们能想到的唯一大的不同是，那些进入国家队的球员年轻时有更多的时间在无组织的小范围比赛中……并继续以更多的无组织比赛进入职业队。—寻找伟大

关于这一点如何适用于制造商，我的观点是:继续建造、试验和创造，没有关于你能制造什么的指导方针或规则。当我们“玩”无代码平台时，我们训练自己比同龄人更有创造力。
