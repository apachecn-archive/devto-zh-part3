# 您软件开发生涯的各个阶段

> 原文：<https://dev.to/jamesmh/the-stages-of-your-software-development-career-2dd>

你有没有想过你职业生涯的下一步应该是什么？

怎么知道接下来要学什么工具、原理或者编程语言？

我想与你们分享一个简单的框架，它将帮助处于职业生涯任何阶段的开发人员能够导航到下一步和升级。

# 重要注意事项

每一步都代表了你职业生涯中的一个大致“位置”。

在每一步之间，都有一些过渡，需要积累某些领域的知识和经验。

每一步都建立在前一步的基础上。

这些不一定与职位相对应。

拥有“高级开发人员”头衔的人实际上可能只是技能和知识方面的初级开发人员！

是的，有时候人们被提升仅仅是因为他们在这个领域有 X 年的工作经验，而不是因为他们有能力胜任所需要的技能和知识。

或者，也许他们为同一家公司工作了很长时间，以至于他们被赋予了一个“更好”的头衔。

我相信你们大多数人也看到过这种情况。

当然，我不得不说:*每个阶段之间没有真正的“硬”线。*

这只是展望你职业生涯的一种方式，因此更容易整体管理和把握。这是一组有用的概念——仅此而已！

让我们开始吧！

# 1:编码员

一个程序员基本上是一个爱好者。

编码员的特点是:

*   了解编程语言语法——甚至可能非常了解！
*   可以自己创建小项目，如视频游戏、应用程序等。
*   优秀的问题解决者
*   独自工作
*   他们的工作没有报酬(不专业)

# 2:初级开发人员

初级开发人员类似于刚起步的程序员。

但是，他们现在进入 IT 领域是作为一名受薪的专业人员，他们与其他专业人员组成的团队一起工作。

很快，一个初级开发人员很可能会设法在一个对他/她来说太大而无法理解的代码库中工作。他们必须学会如何将问题分解成更小、更易处理的任务。

他们将开始学习如何将不同的系统连接在一起——比如客户机-服务器交互或从服务器端代码连接到数据库。

# 3:中级开发人员

他/她是一个有能力的团队成员和贡献者。

他们已经学会了如何构建基本的系统——比如一个 web 应用程序。

在这个阶段，开发人员正在学习的一些技能是:

*   了解基本系统架构
*   能够很好地使用专业工具
*   对设计模式、代码味道等有基本的了解。
*   了解如何作为团队的一部分很好地工作

# 4:高级开发人员

高级开发人员变得非常熟练和博学，以至于他/她周围的人都把他们看作是指导者和一般的“关键人物”。

他们通常拥有一些专业的高级编程知识，能够使用高级技术和模式构建更复杂的系统。

作为一个例子，高级 Web 开发人员的一些特征是:

*   对设计模式、代码味道、重构等有所了解。
*   了解架构范例，如领域驱动设计、事件源、CQRS 等。
*   一些服务器部署和维护经验，关于负载平衡、连接池等基础知识。

是的-那是一大堆东西！

## 关于不同侧重点的说明

这个例子着眼于一个高级网络开发人员的特质。虽然其中的大部分都是相当通用的，并且在不同的焦点上共享，但是对于来自*的高级数据分析师*、*的高级 IOT 开发人员*等来说，它们可能是不同的。

例如，*面向数据库的高级开发人员*需要更多关于数据库管理、编程等方面的知识和技能。

# 5:首席开发人员

首席开发人员专注于为他们的开发团队提供高层次的指导。

但更重要的是，他们是开发团队和组织中其他面向业务的部门之间的桥梁或连接器。

查看[这篇文章](https://dev.to/jamesmh/whats-the-difference-between-a-senior-and-lead-developer-1pl2)了解更多细节。

# 6:技术带头人

一个技术领导者已经成为某个特定领域的专家，并且通常有一个非常有针对性的品牌/定位来说明他们擅长什么。

这可能是特定的编程语言、web 安全、react 前端开发、开发人员指导，甚至是特定的行业！

他们是社区的建设者，拥有一群将他们视为直接或间接导师的追随者。

他们是支持某种特定事业或想法的思想领袖。

他们公开谈论他们的专业领域或他们支持的想法。

简而言之，他们是技术专家和非常熟练的沟通者。
