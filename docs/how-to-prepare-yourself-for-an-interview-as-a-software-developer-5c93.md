# 作为软件开发人员如何准备面试？

> 原文：<https://dev.to/rlxdprogrammer/how-to-prepare-yourself-for-an-interview-as-a-software-developer-5c93>

# 简介

参加工作面试真的很不容易，压力也很大，但如果你没有一个好的人际网络，这几乎是获得梦想工作的唯一途径。你在工作面试中的表现可以决定你是否会得到这份新工作，从技术角度来看，你需要保持健康，你还需要注意你的沟通方式。只要做一个简单的准备，你就能提高胜算。接下来所有的观点都是基于我自己的经历。我参加了双方的许多面试:作为面试官和候选人。在这篇文章中，我试图总结我的经历。

每个公司的申请过程都不一样，但是弄清楚什么在等着你是个好主意。面试可以只是一个简短的介绍，重点是你在寻找什么。这可能是一次他们关注你个性的面试。也可以是技术面试。它可以是这三者的任意组合。所以，如果你收到了面试邀请，但根据邀请，你仍然不清楚面试的主题是什么，你可以随意提问。你也应该澄清面试的语言。

# 如何准备不同类型的面试？

如果是面试，人力资源合作伙伴很可能会关注你在寻找什么和你的个性。在这种情况下，你不应该期待深刻的技术问题。很有可能你会浏览你的简历，你需要谈谈你的经历。所以在参加面试之前，一定要通读你自己的简历，试着为自己介绍所有你以前工作过的地方，关注你在那里真正做了什么，你支持了哪些项目，你使用了哪些技术，你有哪些职责。你应该谈谈。之前每个位置 2 分钟。也可以有关于研究的问题，然后你需要告诉你的研究是什么，你论文的主题是什么。还有一件更重要的事情:你肯定会被问到你真正在寻找什么。对于这个问题你需要有一个明确的答案。比如“我正在寻找一个 C++开发人员的职位，在那里我也可以处理软件架构”。接下来是一些典型的人力资源问题。老实说，对我来说这一直是最困难的部分。因为我对谈论技术话题没有问题，但是回答像:你为什么比对方候选人强，或者你的强项和弱项是什么这样的问题对我来说真的很难。所以我建议你收集可能的人力资源问题(你可以在网上找到很多收集的问题，有一些回答什么的想法)，从中列出一个清单，写下你的答案。这样在面试时你就不需要时间去思考了。

大多数公司也有技术面试。但是一些公司正在合并人力资源和技术面试。在准备技术面试的时候，再读一遍你申请的职位描述和简历中的主要技能。当然，你不可能在几天内学会所有的东西，但是如果你在面试前检查这些话题，会很有帮助。对于所有的技术话题，都有一些流行的面试问题。就像如果你申请一份 C++开发人员的工作，我几乎可以肯定会有关于智能指针和多线程的问题。在准备阶段，试着关注这些话题。对于所有的技术话题，你都可以在网上找到面试问题集，通读一遍并尝试回答，如果你不能回答谷歌一下。如果你仍然感到不舒服，我的建议如下:申请有类似描述的工作，但你对该公司并不真正感兴趣(它离你的地方太远，你听到了不好的反馈等。).所以，如果你想找一份 java 开发人员的工作，申请一些 java 开发人员的职位，即使他们对你来说并不那么感兴趣。然后你可以参加一些面试，在那里你绝对没有风险，所以你可以毫无压力地尝试自己，你可以弄清楚在那个领域有哪些典型的问题。

在面试之前，总是设法找到一些关于公司活动的信息。如果您在考虑他们可能面临的技术挑战或他们正在使用的技术之前先思考一下，会有很大帮助。

# 面试时如何表现？

面试时保持冷静和自信。面试官可能会试图让你感到不舒服，但尽量保持冷静。让你感觉不舒服的典型问题是“你为什么想换工作？”、“你为什么比其他候选人强？”、“你和现在的老板、同事关系如何？”。如果你在面试前准备好这些问题的答案是很好的。他们应该诚实，但政治正确。因此，回答“我讨厌我的老板，因为他是个混蛋”不是一个好主意。更好的说法是:“有时候，让我的老板相信我的想法有点困难。”。

如果你已经在面试了，试着给出你最好的答案，保持冷静和自信。但是有一点很重要:不要羞于给出下面的答案:对不起，我不知道，我没有这方面的经验。所以你可以为面试官节省很多时间，为自己节省一些不舒服的分钟。在大多数面试中，如果你不知道所有的事情，这不是问题，但是如果你说了一些完全不真实的事情，那就有问题了。这在我身上也发生了上千次。当我引导面试并问候选人:“你有 QT 信号槽模型的经验吗？”我通常得到的回答是:是的。然后对于这个问题:“它是如何工作的？”我得不到明确的回答，只是一些屁话。所以这是要避免的，最好是说:“不，对不起”，然后没问题，让我们谈论一些其他的话题。

还有一点:工作面试就像约会:双方都试图确保你们是匹配的。所以你需要展示你最好的一面:提及你能做的一切，但不要撒谎，他们会意识到，你可能会遇到一些非常不舒服的情况。另一方面，你需要检查公司是否适合你的需求。在面试的第一部分，让面试官提问，但之后通常面试官会问你是否还有其他问题。是的，你应该有！问职位，问公司，问用的技术。这样你就可以对这个机会有一个清晰的认识，你就可以决定是否要加入他们。另一方面，这会让面试官感觉更好，让他们觉得你对这个职位真的感兴趣。

面试那天，不要再制造任何压力。找一些相配的衣服。在这一点上，你应该遵循公司的着装规范。事先检查他们的网站，如果他们有照片，检查他们的员工在工作中穿什么。最重要的是:及时赶到那里。最好在面试开始前 20-30 分钟就计划好你的到达。很多事情都可能发生:附近没有停车位，交通堵塞，你的火车晚点，你找不到地址等等。但是如果你来得太早，就不要进入大楼。利用这段时间在大楼里走一走，检查所有可能感兴趣的东西:哪里有停车位，有可以吃饭的地方等等。在面试开始前 10 分钟左右进入大楼。去接待处介绍你自己，告诉她你是来面试的。永远不要忘记面试官的名字(如果邀请中有)。在面试的开始，大多数情况下你会被问及是否需要水。说好就好。通过喝水，你总能在面试中赢得一些真正有用的时间。与此同时，你可以考虑一下答案。关于赢得时间:如果你知道答案，但你需要思考几秒钟，你可以开始胡说。好吧，这可能不是最正确的词，但我在这里的意思是这样的句子:“是的，我必须在我以前的项目中解决这样一个问题，找到解决方案非常复杂……”同时你有几秒钟的时间来制定你的答案。或者再举一个例子:问题是锁在多线程中是如何工作的。然后你从什么是锁定以及为什么它是必要的开始，最后你可以告诉它是如何工作的，但是在你的介绍中你有一些时间来制定你的答案。

最后但同样重要的是:在面试中，你的技术知识不是唯一要检查的东西，你的态度、个性和你的沟通方式也很重要！诚实，豁达，看起来有动力，那是成功的一半。

# 关于薪水的大问题

面试中最有趣的部分之一是薪水。通常这个话题只会在面试结束时出现。让面试官提出话题。如果这个话题没有出现，不要担心，也许它只会在下一轮面试中出现。关于这个答案:如果我在寻找新的工作，我总是首先决定我能接受的最低工资是多少。当我被问及我的工资需求时，我通常会说比这个数字多 10-20%,但我要说的是我很灵活。所以，如果公司能满足我对这个职位的期望，我将会超过我的最低期望，如果不能，还有进一步讨论的空间。对我来说，这个策略很有效。如果你不知道什么是现实的，使用谷歌:有几页可以告诉你在你所在的城市某个特定职位的平均工资是多少！

# 总结
