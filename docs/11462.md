对离开一份舒适的软件工程工作并冒险进入未知领域是什么样的一种开放的见解。

在过去的三年里，我在三家不同的公司工作过；在这篇文章中，我将概述我离开我做过的最好的工作之一的真实收益和损失。

这是我在这些公司的时间和经历的个人描述，它不应该以任何方式影响你对公司本身或在那里工作的人的看法，这只是我的经历的回忆。

### 我的工程时间表

汤森路透: **2012 年(未覆盖/大学行业安置)**

Net-A-Porter 研究生计划:**2014–2018**

SkyScanner:**2018–2019**

怀特哈特 GB : **2019 年至今**

### 网-波特

#### 背景

我于 2014 年以毕业生的身份加入 NET-A-Porter(以下简称 **YNAP** )。在这美妙的 4 年里，我成为了一名软件工程师，这要感谢那里那些了不起的人和导师的指导，也要感谢那里不同的范例、逻辑/商业问题和我有幸接触到的经历。

我真心热爱我的工作。我会一觉醒来，期待着进去——我知道很多人在职业生涯中没有足够的幸运来体验这一点——但我对工作场所的崇拜开始减少，我开始考虑我的职业前景。问题是，我已经变得喜欢这里的人和舒适，我真的很害怕，因为种种原因，我无法在其他工作岗位上生存下去:

*   我没有从事另一份工作的技能(最好的情况是冒名顶替综合症)
*   我不会像在这份工作中那样与其他同事建立有意义的关系
*   其他工作对我的期望要高得多，午睡很舒服
*   “我会非常想念这里的人”
*   如果技术不是最先进的呢？如果有意义，NAP 很乐意测试出血边缘
*   我真的不具备(再次)从事另一份工作的技能

这家公司是我职业生涯的一个伟大起点；作为一名毕业生，我尝试了不同的堆栈和团队(Java、Scala、Node、React、Vue、Swift、Perl、Ruby)，并最终在公司最重要的团队之一中担任了 Node/React 的角色。在接下来的几年里，我参与了许多大型项目，产生了真正的影响。

除了我面临的巨大技术挑战，我还设法体验了在一家真正拥有伟大文化的公司工作的感觉；同事们实际上是彼此的朋友，下班后/上班期间/下班前的社交聚会是常态；每个人都很棒，我喜欢那里的人。这在很大程度上是因为在招聘阶段非常重视确保加入的人和公司一样重视同样的事情。

> 有趣的事实:你知道吗，从统计数据来看，工作文化最差的公司是那些把它作为卖点的公司。这是因为员工一开始就对未来抱有很高的期望，其中大部分从未实现

#### 危险的暗示

好日子持续了几年，直到事情开始发生变化。在一些大规模的业务和技术方向变化之后，我的日常工作并不像我曾经发现的那样深刻——我不会在这里详细说明，但我不再对即将到来的工作类型和我预见的公司(以及后来的我自己)正在做的工作类型感兴趣。人们开始看到这种转变，并开始离开，这不应该让任何人感到震惊，也不能总是归咎于公司的问题，一些人只是想在几年后改变一下环境。

除此之外，我一直在自己的时间里摆弄令人兴奋的不同语言和书库，并发现自己真的想在一个更专注于后端的角色中工作，特别是我爱上了 Golang——这里的一个小问题是，在这一点上，我的许多专业经验并不适合后端角色，除非薪水大幅下降；我有一些，但我缺乏数据库经验和其他传统的交易技巧。称之为贪婪吧，但我想成为一名不同类型的工程师，而不大幅削减我的收入。

#### 继续前进

于是 2017 年 9 月开始另找工作。幸运的是，YNAP 给了我一个很好的简历职位，这要归功于我的一些同事在科技界的一些令人难以置信的工作，我收到了来自许多不同公司的大量面试邀请；这也是我的冒名顶替综合症，实际上我已经做了一些很好的工作，赢得了一些黑客日，创建了一些相当成功的辅助项目，受过良好的培训，并被告知我是一个好的开发人员。当我意识到他们提供的不是我想要的(或认为我想要的)时，我拒绝了一些提议？)技术/产品方面。我也很害怕，我不停地为那些给我提供职位或面试机会的公司找借口，一拖就是几个月。

2018 年 4 月，在工作中经历了几个转折点后，我终于鼓起勇气离开了，并在 SkyScanner 找到了一份全栈开发人员的工作，这似乎符合要求。

最后没有，不过没关系。

### SkyScanner

我加入并立即开始在一个小型自治团队中工作，该团队专注于在 React-Native 中重新创建移动应用程序的一部分。该团队积极性很高，对我们将要从事的领域和堆栈有很好的了解，项目看起来很酷。

我表现如何？

*   ✅新技术要求
*   ✅大幅加薪
*   Good team ✅
*   后端工作？❌

相当不错吧？算是吧。

[https://medium . com/media/CFF 7d 4 a5 b 868 c 807 c 4740d 95 e 5466903/href](https://medium.com/media/cff7d4a5b868c807c4740d95e5466903/href)

你看 SkyScanner 给我提供了什么职位他们不清楚我会做什么样的技术，并强调我接受后会被安排到一个与我的技能相匹配的合适团队。我让他们知道我真的不想做 Java，仅仅是因为我讨厌用它工作，我不想再被困在前端；他们似乎在船上，他们的其他技术栈似乎合适。

#### 最初的想法

当我第一次到那里时，团队(由没有严格 JS 背景的新员工组成)正在完成他们构建的 React 原生原型的 MVP，以查看堆栈是否符合项目要求。这是有意义的，多平台原生应用需要更少的工程师，该项目不需要 RN 无法提供的任何利基平台功能，我们仍然可以让它看起来很好。经过几个月的学习 React-Native(如果你在 web 上做过 React，学习曲线非常小)和创建新版本的应用程序后，结果是团队需要一个后端系统来代理和聚合我们为我们的应用程序发出的调用。

> 太酷了，我终于可以做一些后台工作了，并且获得了我渴望已久的体验。

由于他们的背景，这个团队想用 Java 来构建它。令我沮丧的是，这很有意义，符合目的，而且 Java 有公司的强大支持，所以工具和帮助都是现成的。好吧，很公平，我可以加入，尽管我试图得到像 Golang 或 Elixir 或 **<插入 CVDD 技术栈这里>** 在图片中。

兴奋地终于开始一些后端工作，我发现该公司有一个“开发人员支持”团队。他们的工作是围绕开发人员经验和千篇一律的构建创建脚本和工具。只需运行一个 CLI 命令，就可以准备好一个安全的、日志记录就绪的 CI/CD、AWS dev/prod 和 terraformed 应用程序，这是非常有用和高效的。

尽管这很有成效，但令我失望的是，我无法学习任何关于设置过程的知识，因为它已经为我按下按钮完成了，根本没有运营知识或 Java 技能可供学习。在我看来，大量的知识来自于从零开始建立系统，你学习事物是如何耦合的，事物在哪里生存，以及事物如何相互作用。

在 Java 端要做的实际工作也不是我想要的——我想在分布式系统上工作，构建大规模工作的微服务，优化数据库查询，分析数据流等等。这个应用程序很简单，它将请求代理到几个内部系统，并告诉应用程序如何运行——这是我以前在其他堆栈中没有做过的事情。忽略这一点，团队由来自各种背景的工程师组成，你猜怎么着，软件工程师固执己见，如果他们中的一些人只想用 Java 工作，他们可能只会拿起 Java 入场券，所以你猜谁负责与想写像 enterprise Java 这样的 JavaScript 的开发人员一起做大部分的 React 工作？

你猜对了。

[https://medium . com/media/408 db 45 ebdb 83 f 608 e 78 DD 68 af 3 fcf 17/href](https://medium.com/media/408db45ebdb83f608e78dd68af3fcf17/href)

好吧，我对技术不满意，那我自己开发的产品呢？大约 6 个月后，我们发布了一个我们计划好的应用程序版本，由于高层的一些决定，我们做了几次调整。这款应用还不错，我们以敏捷的方式频繁发布更新——但我仍然停留在 React Native 的工作岗位上，同时我觉得我在其他领域的技能正在下降——我不再配置 webpack(这真的是一件好事)，我没有做任何 web 性能调整，我没有做任何节点或后端工作，我失去了接触 AWS 的机会。在我看来，手机也是一个限制性更强、更烦人的发布平台。总的来说，我觉得作为一名开发人员我停滞不前了。

除了这个问题，我还意识到我团队中的许多其他开发人员并没有真正参与进来——虽然这不应该影响我，但当你的同事每天都指出(非常有效的)缺陷时，很难投入到团队和产品中。

#### 那么这份工作给了我什么？

自信和清晰(以及加薪)。

React 原生技能很好，我仍然在业余时间用它们来创建小应用程序。但我从这份工作中获得的最有价值的东西是，换工作不一定是可怕的。我通过了更清晰和一些切实的简历技能，以及加薪。

完美吗？不，但这是一次学习经历。

我和人们建立了紧密而有意义的联系吗？不完全是，我交了一些朋友，其中一些我仍然保持着联系，但是我从来没有想到我会和那些我开始职业生涯的四年多来在每个工作场所的人建立起同样的关系。这本来是一个很好的奖励，但没有必要——我仍然和重要的人保持联系，朋友来自生活的各个方面。

我只工作了 10 个月就离开了，但那是我的决定，因为我已经意识到我想从一份工作中得到什么，不想要什么。我需要一个至少能让我探索后端开发和处理数据库的角色。我需要一些不同的东西来保持我的兴趣。我需要一个我真正关心的产品。

经过一个月左右的寻找新工作(和一个简短的承包想法)，我发现了一个可爱的初创公司，它希望真正帮助人们，并赋予怀特哈特新一代世界领导人权力。

### WhiteHat

写这篇文章时，我只在 WhiteHat 呆了 1.5 个月，这是一个充满挑战的环境，我必须更快、更有效地学习，这是因为初创企业的性质，也是因为整个技术堆栈都是在 Elixir、Elm 和 PostgresDB 中构建的，除了我从 ES6 和 Redux 中收集的信息，我从未做过任何函数式编程。

但是我在学习，我在与人交往，我在通过一个快速成长和扩张其领域的小公司的眼睛来看待工程生活，我很兴奋。这个 50 人左右的小公司前途光明，我认为我们可以做大事。

除了渴望进入一个全新的技术堆栈并获得一些后端工作的经验，我实际上对我们能够实现的目标和我们正在做的事情持乐观态度，这不仅仅是向负担得起的人出售昂贵的东西，事实证明这对我很重要。

### 外卖

跳槽很恐怖。尤其是如果你已经在那里呆了一段时间，并与那里的人和你在那里的记忆产生共鸣。但是如果你发现自己不再有挑战性，不再感兴趣，不再学习或者不再关心你正在做的工作，那么也许是时候去别处看看了。

没必要像你想的那么可怕。

其他公司也在做不可思议的事情。仅仅因为你现在的公司做了一些很好的事情并不意味着其他人没有，也不意味着你会在其他地方错过。

SkyScanner 有一个令人难以置信的 devops 设置，它真的让我大吃一惊。它也有一些非常有才华的人在许多不同的团队中工作，它试图建立一种客观上良好的工作文化——即使我对此有些问题。

Net-A-Porter 在其前端技术团队中对细节有着非常敏锐的眼光，并由一些出色的产品、技术和交付线索领导。我的老团队对测试、性能和用户体验的钦佩和动力是巨大的，它帮助我成为了今天的开发人员。它也是我所知道的一些最不可思议的人的家。

但是你通过移动来学习，这能帮助你克服恐惧。我不再像以前那样害怕寻找新的机会，因为我现在知道，虽然草不总是更绿，但它仍然会给你学习的经验，教会你一些你不知道的关于这个行业和你自己的事情。

#### 轮到你了

如果你确切地知道你想做什么，什么让你兴奋，并且你已经收集或正在学习所需的技能，那么你已经成功了一半。

软件工程师是幸运的——目前市场非常好，技术创新每天都在带来新的机会。只要确保当你在寻找他们的时候，你真的质疑什么对你来说是重要的。

在面试中，询问文化，询问他们你真正要做什么样的工作，看看你是否能和已经在那里工作的开发人员谈谈。深入了解他们如何发布，如何构建，路线图中有什么，他们如何获得对产品的反馈，他们如何进行 A/B 测试，他们通常如何测试，真正深入了解。

还要记住，当你被面试时，你也在面试他们。不要低估自己，要意识到这家公司需要你，否则他们不会花时间去了解。

他们有需求，但你也有。

#### TL；速度三角形定位法(dead reckoning)

通过换工作，你释放了自己的恐惧，害怕其他工作没有那么好，害怕你交不到那么多朋友，也不会过得那么开心。你可能不会，但每次搬家都会让你接触到新的思维方式、新的技术平台、不同的工作文化和加薪，因为说实话，人们在乎钱是因为钱很重要。动起来，你得到的不仅仅是停留(一般来说)。

对你想去的地方要挑剔，知道你想从工作中得到什么。是产品、技术还是金钱？知道自己想要什么，学习技能，不要因为自己优秀到可以去别的地方工作就依恋一个地方，即使你不相信自己是。
