# SQL 真的是数据科学职业的游戏规则改变者吗？

> 原文：<https://dev.to/alanhylands/is-sql-really-a-game-changer-for-data-science-careers-2oa8>

## 在这件事上，我将全力以赴。

SQL 是数据分析职业生涯中你必须掌握的最基本技能。我们可以争论到底 R 是否比 Python 更好，争论到面红耳赤。或者说 SAS 和 Java 是不是死了。或者说 Spark 比 Azure 有用。这些都不重要，除非你摘下了挂在最低处的果子，并安全地放在你的技能篮子里。

### 那就是 SQL。

## 一些相关的统计研究。

数据科学家 [Jeff Hale](https://www.linkedin.com/in/jeff-hale-99a7877/) 最近就数据科学家最需要的技能进行了一项出色的分析。他为 [KD 掘金](https://www.kdnuggets.com/2018/11/most-demand-skills-data-scientists.html)写了这篇文章，对于有抱负的和当前的数据分析师来说都很值得一读。

Jeff 搜集了 LinkedIn、Indeed、Monster、SimplyHired 和 AngelList 上的招聘信息，看看哪些术语在各种数据科学相关的职位空缺中出现得最频繁。他只在美国境内搜索，并对“数据科学家”“[关键词]”进行精确匹配搜索。

对我来说，最有趣的部分是他对各种技术技能的排名。对于任何在过去 6-12 个月浏览过数据科学职位空缺的人来说，Python、R 和 SQL 分别排名第一、第二和第三不会让人感到惊讶。

## 但是你不是说 SQL 最重要吗？

您可能会认为我在这里用一项研究表明 SQL 在列表中仅排在第三位来打击自己的论点。如果超过 70%的列表需要 Python 技能，超过 60%的列表需要 R 知识，难道我们不应该首先关注这些吗？我觉得你不应该。原因如下。

1.  SQL 更容易从头开始学习基础知识。即使您只学习了 SELECT 语句的一些基本用法，您也已经领先于 99%的 Excel 骑师，这些骑师除了复制/粘贴和 VLOOKUPs 之外，从不费心去做其他事情。
2.  SQL 跨越了更广阔的数据科学世界的所有领域。想当分析师？学习 SQL 从数据仓库中选择你需要的数据。想做数据科学家，跑机器学习？在进入算法之前，使用 SQL 清理你的数据。作为一名数据工程师，你喜欢构建数据管道吗？SQL。SQL。SQL。SQL。
3.  RDBMS 数据库将继续存在。数据库世界里总会有闪亮的新对象来分散我们的注意力。(我在看你 MongoDB。)但绝大多数公司和在线应用仍然运行在 MySQL、SQL Server 和 PostgreSQL 三巨头上。你不需要和他们一起工作，就能注意到他们的名字会让你大吃一惊。 ***提示**:三个字母长，和前传押韵。*

然而，在杰夫的分析中，SQL 的另一个元素排在第三位，这是由[的](https://datamovesme.com/2018/12/03/getting-into-data-science-faqs/)克里斯汀·凯雷尔在杰夫的分析中提出的。(是的，我知道，这就像一路下跌的海龟，分析师们互相分析彼此的工作。)

## SQL 有多重要？

Kristin 问 SQL 有多重要，并惊讶地发现它只排在第三位。她认为这可能是因为招聘公司认为 SQL 非常基础，是招聘数据科学家的先决条件。

我完全同意。在招聘数据分析师时，如果他们没有展示出 SQL 知识，我会立即扔掉面前的简历。这甚至不是它是否值得的问题。这是一开始就必须的。

当我进入数据世界时，我很幸运，因为我来自一个 web 和软件开发背景。我一直在构建医疗保健和电子商务领域的桌面和在线应用程序，其中一个基本部分就是成为我自己的开发 DBA。

我不仅设计和开发表结构，还在开发和生产阶段编写和优化查询和存储过程。小团队需要[全面的技能](https://simpleanalytical.com/generalists-real-data-science-unicorns/)并且知道 SQL 是我最依赖的技能之一:日复一日。

## SQL 是一个超级大国。

当我第一次在当前的工作场所开始做数据分析师时，我只是从开发人员的角度继续编写 SQL。当我编写子查询和手工编写 SQL 时，我团队中的其他分析师看着我，好像我是某种女巫。他们仍然使用微软 Access 中的拖放式查询设计器。虽然我不知道有什么不同，但它帮助我迅速起步，并在早期产生了很大的影响。

辩护的理由几乎已经准备好了。

我在互联网论坛上经常看到这样的问题，我的回答总是以同样的方式开始和结束。如果你想从事数据分析工作，不学习 SQL 会给你自己带来很大的麻烦。在你的职业生涯中，继续建立在这个基础上，但是请学习基础知识。

SQL 是不断赠送的礼物。

你不会后悔的。我保证。
