#### CSS&SCS

##### [](#stacked-borders)[【堆叠的“边界”](https://css-tricks.com/stacked-borders/)

超级有趣看看如何在你的元素上创建奇特的边框，结合框阴影，轮廓，边框，甚至一些背景剪辑。这很有趣，让我想到了吸引人们注意页面元素的可能性。

##### [](#sophisticated-partitioning-with-css-grid)[用 CSS 网格进行复杂的划分](https://v6.robweychert.com/blog/2019/03/css-grid-sophisticated-partitioning/)

有趣的是，您可以通过各种方式划分网格，然后根据 n-child 和其他相关说明符进行划分。在显示内容的方式上创造一个非常有机的感觉集合。

##### [](#why-cant-i-set-the-font-size-of-a-visited-link)[为什么我不能设置被访问链接的字体大小？](https://jameshfisher.com/2019/03/08/why-cant-i-set-the-font-size-of-a-visited-link/)

##### [](#redesigning-our-docs-part-4-building-a-scalable-css-architecture)[重新设计我们的文档——第四部分——构建可扩展的 CSS 架构](https://blog.algolia.com/redesigning-our-docs-part-4-building-a-scalable-css-architecture/)

选择基于 Taliwind.js 的“效用优先”CSS 方法的有趣案例研究。我已经听到越来越多的关于这些方法作为 BEM 风格方法的替代方法的消息…我自己还没有在一个严肃的项目中尝试过，但是这些论点越来越有道理。此外，如果你对《顺风》感兴趣，请和创作者一起查看这一集！

##### [CSS 中的 If 语句和 for 循环](https://www.quirksmode.org/blog/archives/2019/03/if_statements_a.html)

试图将 CSS 的概念和隐喻与更传统的软件开发范例联系起来的有趣外观。如果你是一名 JavaScript 开发人员，并且正在努力研究 CSS，这绝对是一件有趣的事情。

#### [](#javascript)JavaScript

##### [](#7-tricks-with-resting-and-spreading-javascript-objects)[用静止和传播 JavaScript 对象的 7 招](https://blog.bitsrc.io/6-tricks-with-resting-and-spreading-javascript-objects-68d585bdc83)

我喜欢在 JavaScript 中使用 rest/epread 操作符的工效学(万一 rest/spread 对您没有意义，这是我们的小趣事)。事实上，如果你想要一个快速和肮脏的介绍，看看我去年写的一篇文章。但是如果你已经对这些基础知识相当熟悉了，这篇文章有一些超级酷的技巧，你可以用它们来扩展。爱死了！

##### [](#why-you-shouldnt-use-momentjs)[为什么不该用 Moment.js…](https://inventi.studio/en/blog/why-you-shouldnt-use-moment-js)

标题比文章更有观点——这篇文章对 Moment 的问题或挑战进行了分解，但也介绍了不同的数据库在不同维度上的表现。

##### [](#reduce-reduce-reduce)[减少，减少，减少](https://www.matthewgerstman.com/tech/reduce-reduce-reduce/)

当我处理列表时，我喜欢 reduce，但是每次我都必须查找它在 JavaScript 中是如何工作的，并仔细考虑它。本文深入研究了 reduce，并展示了如何使用它来创建一组有用的函数，如“groupBy”和“keyBy”。

##### [](#react-component-types-a-complete-overview)[React 组件类型:完整概述](https://www.robinwieruch.de/react-component-types/)

对随着时间的推移在 React 生态系统中成长起来的各种类型的组件进行了详细的介绍。我特别喜欢这篇文章的一点是，它给出了为什么存在所有这些不同组件类型的历史背景，因此那些多年没有在 React 中编码的人可以更好地理解他们今天看到的各种选项背后的一些原因。

##### [](#writing-resilient-components)[编写弹性组件](https://overreacted.io/writing-resilient-components/)

我看到有人发推文说，每当丹·阿布拉莫夫写一篇博客，一半的前端社区就会损失 4 个小时来阅读和消化它。这有点言过其实，但也有一定的真实性。这是一篇会让你思考很多的长文…而且绝对 100%值得。无论你使用什么样的框架，它都将改善你编写组件的思路。
