# 10 个小小的决心，在 2019 年成为更好的开发者

> 原文：<https://dev.to/antjanus/10ish-teeny-tiny-resolutions-to-become-a-better-developer-in-2019-54f5>

受文章[快乐 2019 年的 19 个微小的决心](https://medium.com/@AllisonGauss/19-teeny-tiny-resolutions-for-a-happier-2019-5f068f840c67)的启发，我想浏览一下作为一名开发人员可以做的一些很棒的小事的列表，以成为一名更好的开发人员。这不是一个要学习的算法、要解决的技术等的列表。这是一份清单，希望不会花太多时间，但会有很大帮助。

## 1。参加 [FreeCodeCamp 无障碍课程](https://learn.freecodecamp.org/responsive-web-design/applied-accessibility)

可访问性是一件大事。这在开发社区也很热门，如果你知道你在说什么的话，这是一件大事。不仅如此，课程本身非常快，不会对你做的事情有太大的改变，它只会帮助你专注于使用正确的 HTML 标记，教你关于可访问性的新 HTML 标记(我不知道`accesskey`是个东西！)，总的来说，让您更加了解关于可访问性的问题。

## 2。赶上一个前同事

当我们离开工作时，有时很容易完全离开与我们一起工作的人，那里的环境等等。但是，我发现和以前一起工作的人聊聊很有启发性，也很有帮助。

除了一个有趣的社交电话(我仍然强烈推荐)，保持联系对一个人的职业生涯非常重要。你永远不知道什么时候你会找新工作，或者什么时候你会找一个你认识并信任的人加入你的公司。

加分点:那个前同事不需要是开发者！

## 3。清洁你的键盘

拿些压缩空气把那些垃圾都吹出来！拿块棉签和一些酒精，把那些键擦干净。或者只是查查怎么做。那个键盘可能需要清洗一段时间了。

## 4。观察某人使用你不熟悉或你讨厌的操作系统，看看他们为什么喜欢它

这听起来可能很奇怪，但是我发现听听为什么人们想要使用一个平台而不是另一个平台是非常重要的。我是一名 Windows 用户，但我经常使用 Linux (Ubuntu + Arch)和 MacOS。了解为什么用户更喜欢一个，也有助于理解你的用户和其他开发者。

如果你有时间，也可以在网络浏览器上试试。为什么有人用 IE Edge？还是火狐？或者说为什么人们会坚持使用 Chrome？

## 5。观看一个关于你感兴趣的技术的在线讲座

文档试图教导，而文章倾向于指导(如在 teach 中)或者争论为什么你应该使用一种技术。但是，如果你想了解这项技术是如何工作的，它是用来做什么的，或者人们是如何使用这项技术编写代码的，可以去看看一个讲座。

[查看节目会谈报告](https://github.com/hellerve/programming-talks)中的精华部分

## 6。了解 RSI(重复性压力损伤)

事实上，更多地了解你的工作如何影响你的健康。几年前，我的手腕和前臂开始疼痛，因为我使用的键盘不适合我。在某种程度上，它迫使我停止工作，因为我的手臂太疼了。

如果你发现自己在工作后有背痛、手腕痛或其他疼痛，你可能想看看如何改善你的工作环境，使之成为一个更健康的环境。

## 7。了解代码编辑器的 1 个新的高级用户功能

学习一个编辑器的超级用户特性集的所有特性确实是不合理的，但是学习特性会比你想象的更有帮助。

我是一名 VS 代码用户，我学到的一个特性是，当我在一个变量上按下`Ctrl+F2`(在 Windows 上)时，我可以编辑所有出现的变量名。我曾经使用 Find & Replace，它不够精确，你不能只查找/替换所有出现的内容。

在那里，如果你有 VS 代码，你已经完成了这个。

## 8。了解 Flexbox 或 CSS 网格

我之前撒谎了。这是我想让你们学习的一项技术。至少，如果你是任何类型的 web 开发人员。一旦你掌握了窍门(而且不会花太长时间！)，你会发现自己到处都在用。

我最近开始为我的网站重新设计原型，flexbox + grid 让这种体验变得更好。如果你感兴趣，这是不断变化的结果。

我在这里给你放了一些链接:

*   网格花园 -一个非常简单的测试和学习 CSS 网格的方法。尽管它缺少 CSS 网格中最重要的特性之一`grid-template-areas`
*   我在 CSS grid 上制作了几个视频，如果你想看的话。有[响应博客网站布局的 CSS 网格](https://www.youtube.com/watch?v=1MOqDTp6Vwk)和[CSS 网格的基本介绍](https://www.youtube.com/watch?v=V48baOnbWNA&t=1754s)
*   Flexbox Froggy -由 Grid Garden 的同一批人制作，Flexbox Froggy 是另一个以非常有趣的方式学习和测试 Flexbox 的工具

## 9。发布包

不管你选择什么语言，去发布一个包吧！如果有您倾向于在项目之间复制/粘贴的代码，不管它有多小或多简单，制作一个包，在您选择的平台上发布它，并开始使用它。
