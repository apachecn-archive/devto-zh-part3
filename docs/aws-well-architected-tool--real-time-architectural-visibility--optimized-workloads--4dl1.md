几个月前，我们写了一篇关于 AWS 良好架构评审的 [重要性的博客。这篇文章讲述了在不同场景下进行这项练习的重要性。随着 AWS 在 2018 re:Invent 上宣布推出](http://blog.totalcloud.io/aws-well-architected-review-should-i-or-not/) [AWS 架构良好的工具](https://aws.amazon.com/well-architected-tool/) ，我们兴奋不已，并试用了它。

对于那些不知情的人来说，AWS 架构良好的工具是一个为当今的云架构师设计的自助式免费审计工具。它的目标是帮助用户基于 AWS 良好架构的框架构建安全、高性能、弹性和高效的应用程序基础设施。

该服务根据一组问题的输入来审查预定义工作负载的状态。问题是关于 ***卓越运营、安全性、可靠性、性能效率和成本优化*** 。它将审查结果与已确立的 AWS 最佳实践进行比较，并生成一份可下载的报告，帮助云架构师设计和规划他们的工作负载以实现最优化。

## 下一步:通过实时架构可见性验证您的网络、安全性和成本答案

作为一名解决方案架构师或云架构师，您肯定希望根据工作负载的当前状态来验证您的答案。比方说，你需要回答安全部分的一个问题，这个问题问***‘你如何保护你的网络？’*** 此问题下的一些清单要求您验证工作负载，以实现网络保护、限制暴露、控制所有层的流量等。为此使用一些具有架构可见性的实时数据怎么样？

在像 AWS 这样的动态环境中，资源由几个团队成员跨项目按需提供、扩大或缩小。因此，在您不知情的情况下，您的工作负载状态发生变化的可能性很高。访问用于验证的实时数据或定期基础设施审计发挥着关键作用。

TotalCloud 的网络视图和安全组视图通过特定工作负载中的端到端环境提供实时架构可见性。验证网络保护、限制暴露、控制所有层的流量等问题的答案。有了这两种观点就变得轻而易举了。因为该平台在实时拓扑结构上提供了可视化提示，从而更容易定位漏洞。
