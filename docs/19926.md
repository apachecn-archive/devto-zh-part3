# 如果你想成为一名高于平均水平的网络安全研究人员，请提问

> 原文:[https://dev.to/sinxloud/web-security-researcher-questions-EPN](https://dev.to/sinxloud/web-security-researcher-questions--epn)

当你开始练习安全时，你必须通过提问来对你的学习过程进行艰难的调整。

*   **这是什么 Bug？**
*   这个 bug 是如何工作的？
*   这个 bug 为什么会工作？

提问是 web 安全成功的关键，你几乎每次都需要问为什么问题(为什么这个有用，为什么这个没用)并自己回答它们。

**如果你发现一个 bug**我是怎么发现这个 bug 的？这个 bug 为什么会存在？

**如果没有发现 bug**为什么我没有发现这个 bug？我做错了什么？

你必须知道为什么你找不到那个 bug。也许你能在虚拟机上的实践中找到错误，但是你不能在实际的网站上找到错误。

**你要知道为什么**。可能你正在测试的网站已经实现了一个很好的安全机制。所以你必须了解他们做对了什么。一旦你知道了更多正确做事的方法，你就会自动知道如何去做错误的事情。下次当你去测试另一个网站时，你就会知道这些人缺少什么。

臭虫赏金猎人的常见问题是，一旦你发现一只臭虫，你就会兴奋过度。你想很快得到报酬，你的注意力转移到金钱上，在 200 美元的赏金下你停止了逻辑思考。因此，有时你的 bug 会有更大的影响。只有你多思考问题，你才能获得 2000 美元甚至 20000 美元的奖金。

请记住，作为一名网络安全研究人员，您会使用常规的日常物品，并想到恶意使用这些物品的方法。因此，提问可以让你从鸟瞰的角度看网站，找到容易发现的安全漏洞，而不会造成严重影响。

您需要了解 web 开发世界中正在发生的一切。

如果你想成为一名高于平均水平的网络安全研究者，你必须更仔细、更深入地了解网站使用的不同技术是如何结合在一起的。

**所以问问题会帮助你做到这一点**
