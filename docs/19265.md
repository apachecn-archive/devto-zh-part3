# 教育编程语言

> 原文:[https://dev . to/stereo booster/educational-programming-language-2155](https://dev.to/stereobooster/educational-programming-language-2155)

让我们来谈谈专门为教育设计的编程语言。这是一个概述，而不是深入的探讨——我将简单地谈论一些方面，也许会在单独的帖子中写更多。

## [](#logo-1967)标志，1967 年

Logo 是 Lisp 的一种方言，被设计成一种学习工具。西蒙·派珀特等人在 1967 年创作了第一版 Logo。《标志》是建构主义方向的首批作品之一。

*边注*:建构主义是由 Papert 发明的教育理论，它是让·皮亚杰的思想——建构主义的发展。Papert 在日内瓦与 Piaget 一起工作。还有类似的观点:皮亚杰建构主义，维果茨基社会建构主义。我在这里做一个手波状的解释，不一定是完全正确的。

关键思想是，知识是由每个学习者构建，当学习者主动与某个主题互动并自己探索它时，它就会发生。因此，所有教师试图通过解释或强迫背诵一些定义来传授知识的努力都是无效的。当学生愿意学习时，学习就发生了，教师的任务是创造一个环境，让学生进行实验，与主题进行互动，并建立自己的知识。

[![](../Images/af6af07e696ecc1dd7c2d1b70e60426b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--vll_WorO--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/g349114cjuduxqig6btn.png)

最流行的 Logo 环境涉及到海龟，学习者可以通过提供一系列指令，也就是算法来控制它。通过这种方式，学习者可以实验并观察海龟对不同算法的反应。

Logo 提供了清晰的错误信息，为学习者降低了障碍。

这个标志可以:

```
fowad
I don't know how to fowad 
```

LISP 会这样做(取决于方言):

```
(fowad)
fowad: undefined;
cannot reference an identifier before its definition 
```

在 REPL，学习者可以一个接一个地执行命令，他们不需要一次写完所有程序。

该徽标使用带有隐式强制的动态类型系统，例如，字符串可以自动转换为数字，这为孩子们降低了障碍，但正如我们所知，如果在大型语言中大量使用，如 JavaSctipt 或 PHP，它可能会反咬一口。

该标志使用了少量的特殊符号，所以它的文字看起来像自然语言，这也降低了壁垒。

**关键特性**:交互性(继承自 LISP)、友好的错误消息、专为降低入学门槛而设计

**关键范式**:命令式(学习者直接下命令)、过程式、强制动态键入
**年龄**:学前、学校

### [](#further-development)进一步发展

在 90 年代，标志的概念被扩展到了物理设备上。这个项目被称为[乐高标志](https://web.archive.org/web/20160802205923/http://ucls.uchicago.edu/students/projects/1994-95/Lego-Logo/ProjectDescription.html)。有可能用标志指令控制现实世界的“机器”海龟。

### [](#scratch-2002)刮刮乐，2002

[https://www.youtube.com/embed/tEWFDJSmWcw](https://www.youtube.com/embed/tEWFDJSmWcw)

2010 年，Papert 的想法获得了新生——编程语言 Scratch 诞生了。它利用了与 Logo 相同的想法，但增加了一些内容。

它使用**可视块**而不是文本来表示代码指令，这降低了障碍，更重要的是，没有办法出现语法错误——错误的块不会组合在一起。

它使用**事件驱动的**范例来处理交互性。我猜，这个功能已经存在于 Lego-Logo 中了。利用事件，学生可以创造游戏、测验或让机器人对动作做出反应。

Lego-Logo 项目看似死亡，但有一个现代的替代方案——[Scratch+Arduino](http://s4a.cat/)。

参见:麻省理工学院的 Scratch 3.0(T1)，哈佛大学的使用 Scratch 的计算机入门课程(T2)

一个潜在的有趣想法是能够在块表示和文本表示之间切换，类似于他们在 [Luna-Lang](https://www.luna-lang.org/) 中所做的。

## [](#pascal-1970)帕斯卡，1970 年

1970 年，N. Wirth 发表了 Pascal 语言，它是 Algol X 的变体，基本上包含了所有 Wirth 的思想，但 Algol 并不接受。

附注:沃思和埃德格·迪克斯特拉、东尼·霍尔、彼得·兰丁、约翰·麦卡锡等人都是 Algol 68 委员会的成员。

1976 年 N. Wirth 写了一本书《算法+数据结构=程序》，内容涵盖:基础数据结构、排序、递归算法、动态信息结构、语言结构和编译器。在最后一部分，他展示了如何用 400 行代码实现迷你类 Pascal 语言编译器。

它是本文中唯一一种具有静态类型系统的语言。静态类型系统似乎创建了一个更高的进入壁垒——你需要声明对象的类型，你可以混合不同类型的对象，你需要引入集合来推理类型等等。另一方面，帕斯卡有非常简单明了的和在某种程度上有限的类型系统，这正是柯尼根所批评的。

**关键范式**:命令式、结构化、程序式、静态式系统。
**年龄**:学校，学院

### [](#further-development)进一步发展

这是帕斯卡的一个小家谱:

```
Pascal → Modula → Modula-2 → Modula-3
   |               ↳ Oberon → Oberon-2
   ↳ Objective Pascal → Delphi 
```

据我所知，Pascal 现在在教育中没有广泛使用。

从某种意义上说，围棋可以被认为是帕斯卡的继承者。

> Go 主要属于 C 家族(基本语法)，有来自 Pascal/Modula/Oberon 家族的**重要输入(声明、包)，加上一些来自受东尼·霍尔 CSP 启发的语言的想法，比如 Newsqueak 和 Limbo(并发)。
> ——[Go FAQ](https://golang.org/doc/faq)**

它有一个简单的静态系统，接受类似 Kernighan 的批评。Pascal 因为不支持变长字符串而受到批评，Go 因为缺少多态性而受到批评。

## [](#scheme-1975)方案，1975 年

Scheme 是 Lisp 的一种方言，由小杰拉德·让伊·萨斯曼·盖伊·l·斯蒂尔在 70 年代发明。

他们基本上“重新发现”了λ演算——当 1958 年麦卡锡创建 LISP 时，他听说过λ演算，但还没有研究过它。但是苏斯曼和斯蒂尔看到了 Lambda 的潜力，并对语言进行了修改以使其工作，这样他们就打开了从理论到实践的大门。

*旁注*:西蒙·派珀特是苏斯曼的博士生导师。
*边注 2* :他们开发方案的研究对象之一是[卡尔·休伊特](https://www.youtube.com/watch?v=7erJ1DV_Tlo)开发的演员模型，灵感来自 SmallTalk 的细胞理念。

1985 年哈罗德·艾贝尔森，杰拉德·让伊·萨斯曼，朱莉·苏斯曼出版了一本书《计算机程序的结构和解释》，这基本上是一门麻省理工学院 6.001 课程。

这是麻省理工学院 6.001 的一节课，也是我最喜欢的一段话:

[https://www.youtube.com/embed/2Op3QLzMgSY](https://www.youtube.com/embed/2Op3QLzMgSY)

> 对这个行业来说，计算机科学是一个可怕的名字。首先，它不是一门科学。它可能是工程，也可能是艺术，但我们实际上会看到，计算机所谓的科学实际上与魔法有很多共同点……所以它不是一门科学。它也不是真的非常关于计算机。它与计算机无关，就像物理学与粒子加速器无关，生物学与显微镜和培养皿无关。它与计算机无关，就像几何学与使用测量仪器无关一样。
> 
> —哈尔·阿伯尔森

**关键特性**:交互性(又是 Lisp)，一个最小的核心，允许从零开始构建更复杂的概念(这就是为什么 LISP 中有这么多 DSL)
**关键范例**:主要是功能性的，但命令式风格是可能的，动态键入
**年龄**:大学。我想通过一些改进，障碍可以降低。

### [](#further-development)进一步发展

Scheme(和其他 Lisp 方言)在工业界(例如 Clojure)和教育界(例如 Racket)仍然被广泛使用，但是传说中的 6.001 在新课程中被弃用，取而代之的是 6.00 (Python)、6.01 和 6.02。

> 苏斯曼指出，工程师现在例行公事地为复杂的硬件编写代码，他们并不完全理解(而且由于商业机密，往往无法理解。)在软件层面上也是如此，因为编程环境由具有巨大功能的巨大库组成。据苏斯曼说，他的学生花了大部分时间阅读这些图书馆的手册，以找出如何将它们缝合在一起以完成工作。他说今天的编程“更像科学。你拿起这个图书馆，戳它。你写程序戳它，看看它做什么。你说，‘我能调整它来做我想做的事情吗？’".SICP 的“综合分析”观点——用更小、更简单的部分构建更大的系统——变得无关紧要了。如今，我们通过戳来编程。
> ——[为什么麻省理工学院不再教 SICP](https://web.archive.org/web/20160504164044/http://www.posteriorscience.net/?p=206)

## [](#smalltalk-1980)SmallTalk，1980

SmallTalk 于 1980 年公开发布，但它的开发开始得更早。

1968 年，艾伦·凯听了马文·明斯基关于现代教育以及皮亚杰和派珀特思想的演讲。此外，Kay 在 Papert/Minsky 实验室看到了 Logo 语言的应用。

1972 年，艾伦·凯发表了论文[“适合所有年龄段儿童的个人电脑”](https://www.mprove.de/diplom/gui/Kay72a.pdf)(又名 Dynabook)。有消息称，他早在 1968 年就有这个想法，但在这种情况下，我想提醒一下，平板电脑的第一个原型之一是在《2001:太空漫游》(1968)中展示的。

凯想创造一种简单的语言，让孩子们能够使用。这种语言应该伴随着 Dynabook。同年，他编写了 SmallTalk 的第一个原型(SmallTalk-72)，它从 Simula、LOGO、LISP、FLEX 和他之前的工作中吸取了一些想法。

他们用儿童和未经训练的成年人做了一些实验。实验表明，人们很容易完成简单的任务，但在更复杂的任务上却停滞不前，甚至在他们看到解决方案后也无法分解。

他们认为也许继承可以用于简化——专家可以为复杂的问题编写解决方案，用户可以通过继承重用它。但事实证明这也是一个令人困惑的概念。

虽然最初的目的是为儿童开发一种编程语言，但在这个过程中，艾伦·凯和他的实验室发明了“纯”面向对象的概念。Alan Kay 创造了术语面向对象编程和 OOP 风格的 6 个基本原则，如一切都是对象，对象通过消息进行通信(这是受生物细胞和网络中的计算机的启发)。

*边注*:关于 OOP 是谁发明的，OOP 是关于什么的，有一些有争议的观点。有些人认为 Simula 是第一种 OOP 语言，但它是一种带有对象的更过程化的语言，最接近的祖先是 C++，它应该是更好的 C 语言。我相信 OOP 有两个不同的“流派”，但这个主题是另一篇文章。Kay 的 OOP 更接近演员模型(它启发了演员模型)。

他们能够构建个人计算机的工作原型和有效的语言，但他们似乎失去了语言用于教育的最初想法。凯试图在 1976 年“重置”它。

1979 年，乔布斯看到了他们的成果，印象非常深刻，但在 1980 年，施乐决定停止这项研究。研究人员决定向公众发布 SmallTalk。

即使 SmallTalk 是为儿童设计的，它似乎也没有达到最初的目的。本文是我对[原文艾伦凯的文章](http://worrydream.com/EarlyHistoryOfSmalltalk)的解读，不要轻信我的话——阅读原文。

**关键范例** : OOP，动态类型系统

### [](#further-development)进一步发展

由于授权复杂，SmallTalk 并没有击中主流(至少没有 C、C++、Java 那么多)。后来开放实现出现了，像 Squeak 和 Pharo。据我所知，最受欢迎的祖先(至少精神上)是 Ruby。它是用于教育的一点，例如[你好 Ruby](https://www.helloruby.com/meetruby) 和 [Hackety Hack](https://web.archive.org/web/20160902010851/http://www.hackety.com:80/) 。

后来，在 2005 年，艾伦·凯和西蒙·派珀特重新提出了 Dynabook 的想法，并参加了项目**每个孩子一台笔记本电脑(OLPC)**——这是一款低端设备，面向大多数人没有电脑的贫穷国家。

## [](#afterwords)后记

如果你想了解更多关于 CS 教育的关键人物，你可以阅读[对计算机科学教育的杰出贡献](https://sigcse.org/sigcse/programs/awards/outstanding)。尼古拉斯·沃斯、哈罗德(哈尔)艾贝尔森和艾伦凯都在名单中。

我还要提到 Bret Victor，他是许多开发人员的巨大灵感来源。在其他想法中，他也有一些关于教育的想法，比如[可学编程](http://worrydream.com/#!/LearnableProgramming)和[发明原理](https://jamesclear.com/great-speeches/inventing-on-principle-by-bret-victor)。

我相信计算机文化应该像我们教孩子读、写和做数学一样来教。如果他们愿意，这些孩子可以成为程序员，以满足[“下一个蓝领工作”](https://dev.to/stereobooster/developer-is-the-next-blue-collar-job-269b)的需求。

很高兴地说，原始 Papert 和凯的想法在这里。最新的 OLPC 是一款带有触摸屏和太阳能电池的平板电脑，可以运行 Scratch。但是仍然有很多未解决的问题，例如，Scratch 教授命令式编程，一旦孩子开始学习函数式范式，老师就会说旧的模式不好，比如赋值、突变、for 循环等等。这是错误的，就好像他们会在前三年教你散文，然后介绍诗歌，解释散文很烂。相反，我们应该教什么是计算，计算可以用不同的方式完成，你需要根据任务选择一种方法。

最后一点:我说的是为教育设计的编程语言，这和为研究设计或使用的语言是不一样的。Python 广泛用于研究，尤其是数据科学。Haskell 和 ML-family 语言广泛用于 CS 研究。
