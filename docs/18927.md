# 自动化:🤖使用 GitHub 的机器人来提高你的生产力

> 原文：<https://dev.to/pierre/automation--use-githubs-bots-to-increase-your-productivity--30j5>

在我写这篇文章的时候，我很惊讶地发现只有少数拥有大型项目的 GitHub 用户有效地使用 GitHub 机器人来管理他们即将到来的问题和 PRs(只要看看每个机器人的安装数量)。

然而，他们可以节省你以前的时间和清理，并帮助你的项目更好的贡献者。

## 最佳“必备”机器人(我的前五名)

1.  [锁定](https://probot.github.io/apps/lock/)
2.  [陈旧的](https://probot.github.io/apps/stale/)
3.  [合并后，自动删除 PRs 上的分支](https://probot.github.io/apps/delete-merged-branch/)。
4.  [当 PR 或问题包含默认标题或空描述时，请求信息](https://github.com/apps/request-info)。
5.  我最喜欢的一个💖(目前仅限测试版访问)。
6.  压缩你的 repo 的所有图像(甚至不需要一个 yml 配置文件来运行)。

现在我会尽量简短🙂但希望它已经有所帮助！💪
