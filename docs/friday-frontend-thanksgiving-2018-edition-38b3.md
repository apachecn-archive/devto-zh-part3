#### CSS&SCS

##### [排除将有望像更强大的网格友好浮动一样](https://css-tricks.com/exclusions-will-hopefully-be-like-more-powerful-grid-friendly-floats/)

与[相关的是，这篇文章](https://v6.robweychert.com/blog/2018/11/css-grid-editorial-layouts/)上周介绍了 CSS Grid 做得不好的地方，这涉及到 CSS 排除的概念，目前只在微软浏览器中可用。我喜欢这样简单地将块元素放入文本环绕的内联上下文中。我也可以想象这个游戏以一种非常有趣的方式与[外形——外部](https://css-tricks.com/almanac/properties/s/shape-outside/)属性一起玩。

##### [为什么不能同时使用函数式 CSS 和常规 CSS？](https://css-tricks.com/why-cant-we-use-functional-css-and-regular-css-at-the-same-time/)

功能 CSS(像[超光速粒子](https://tachyons.io/)这样的东西，或者像`.margin-bottom-20px`这样的类)往往会引发强烈的反应——许多人要么喜欢它，要么讨厌它，这导致它感觉像是一个要么全有要么全无的决定。本文着眼于混合搭配的效果，在项目的大部分时间里仍然使用更传统的 CSS，同时使用一些功能性的 CSS 类。

##### [硬核 CSS calc( )](https://medium.com/buildit/hardcore-css-calc-bdfb0162993c)

另一个非媒体查询驱动的响应趋势的例子。我喜欢这篇文章是因为它带你经历了这个过程，从布局问题开始，通过推导所需的数学，一直到 CSS。

##### [大括号像素](https://alistapart.com/article/braces-to-pixels)

非常酷的一步一步的浏览过程，从你作为开发人员编写的 CSS 规则，到计算规则，层叠，CSS 对象模型，布局和绘画。

这太棒了！一个 UI 模式库，你可以用 JavaScript 创建，完全用 HTML 和 SCSS 实现。有些还不足以在生产中使用，但有大量的例子可以，每个例子都包括使用说明，突出显示它们是否可以用于生产，还可以使用链接让您知道是否有任何浏览器支持问题。

#### JavaScript

##### [用 Vue.js 构建交互式信息图](https://www.smashingmagazine.com/2018/11/interactive-infographic-vue-js/)

非常酷的案例研究使用 Vue.js 和 Greensock 构建基于 SVG 的交互式信息图。我喜欢！

##### [2018 年 JavaScript 现状调查](https://2018.stateofjs.com/)

2018 年 JavaScript 调查结果出来了，这里有很多非常有趣的数据。人们津津乐道的一个大而有趣的事情是 Angular 的受欢迎程度远远落后于 React 和 Vue。也就是说，不要把这些结果当作真理——它们是未经标准化的原始报告，而且似乎没有做出任何努力来获得一个超级有代表性的样本。

##### [用 React 钩子复用简单代码](https://blog.bitsrc.io/simple-code-reuse-with-react-hooks-432f390696bf)

React 挂钩声称的好处之一是在 React 中重用有状态代码的更简单的方法。这篇文章通过构建一个非常简单的定制钩子来看看它是如何工作的。不得不说，我越觉得这是一种方法，我就越喜欢它。看起来它在 Vue 中也很容易使用。

##### [虚拟 DOM 慢。见见记忆中的世界。](https://medium.freecodecamp.org/the-virtual-dom-is-slow-meet-the-memoized-dom-bb19f546cc52)

这很有趣。有人写了一个新的 JavaScript 库，他有一个新的方法，他们声称具有虚拟 DOM 的人体工程学优势，同时速度快了大约 30 倍。显然作者不是没有偏见的，但是如果这被证明是正确的，并且他们能够在不牺牲现代框架的许多其他细节的情况下实现这种水平的好处，我希望虚拟 DOM 方法已经被广泛接受，这将会随之而来。

##### [React 中渲染道具概述](https://css-tricks.com/an-overview-of-render-props-in-react/)

渲染道具已经成为 React 开发中的主流模式之一。如果您还没有遇到或掌握这种设计模式，这是一个很好的入门读物，可以让您对这种方法有一个大致的了解，并有一个使用它的例子。

#### 其他牛气

##### [用块状思维代替块状思维的含义](https://www.smashingmagazine.com/2018/11/implications-blocks-blobs/)

现在古腾堡即将投入生产(从上周开始更正，它实际上应该在 11 月 27 日发货——谢谢 Amy 告诉我最新消息！)，我们开始看到更多关于这将如何改变 Wordpress 世界的思考。这是一个很好的问题，说真的，如果你不在 Wordpress 的世界里，但你有一些反应能力，你真的应该看看这里，因为这里将会有一个主要的淘金热。

##### [Chrome Dev 峰会 2018 亮点](https://bitsofco.de/chrome-dev-summit-2018/)

Chrome Dev 峰会在上周举行，我们看到了亮点和视频。这很好地总结了一些重大公告。或者，如果你有很多时间，可以查看一下所有课程的 Youtube 播放列表。

##### [为什么使用大于“0”的`tabindex`值是不好的](http://www.karlgroves.com/2018/11/13/why-using-tabindex-values-greater-than-0-is-bad/)

一位可访问性顾问写了一篇有趣的文章，讲述了使用 tabindex 来提高可访问性是如何导致网站的可访问性和可维护性下降的。

##### [TensorSpace.js](https://tensorspace.org/)

理解神经网络的三维可视化框架。内置于 Tensorflow 之上的 JavaScript。延续基于 JavaScript 的机器学习工具的趋势，使学习和理解 ML 变得更加容易和容易。在操场上有一些有趣的例子。

##### [渐进式网络应用:一个令人信服的类似本土的网络应用案例](https://blog.bitsrc.io/progressive-web-apps-a-convincing-case-for-native-like-apps-on-the-web-8732bfa07945)

降低 PWAs 的价值主张。查看纯 Web 和纯本地应用程序各自的缺点，然后深入研究 PWAs 如何在减轻这些缺点的同时获得它们各自的优势。还包括一个简单的教程，让你开始与你的第一个 PWA。
