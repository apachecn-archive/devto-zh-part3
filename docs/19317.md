# 有时候你必须制造客户需要的产品，而不是你想要的产品...

> 原文:[https://dev . to/this _ Frankie/有时你必须构建客户需要的产品 3pje](https://dev.to/this_frankie/sometimes-you-have-to-build-the-product-the-client-needs-the-one-you-want-3pje)

所以我们有一个情况特殊的客户。他们有一个网站，但他们的广告人，不知何故拥有网站的权利和所有这一切，基本上给了他们最后通牒。支付我们的服务费用，否则我们将在月底停止你的工作。

看起来很直接，但问题是广告公司确实履行了合同。长话短说，他们需要一个网站。

我们模仿一些东西，基于我们找到的一个模板，把它交给设计师，让它与众不同。我们用这种方式让其他客户也这样做。建立几个模板，我们可以相对快速地制作和刻录，然后每次我们这样做都会变得越来越容易。

我喜欢这个主意！我想全力以赴，我想给他们所有他们能想到的选择。你想要一个英雄元素？多大的？什么颜色？什么标题？什么内容？我给了他们一切。快进两周，两天后是最后期限，是时候放内容了。我在这里或那里修复一些东西，我还没有做好网站客户端的准备(modx 有模板变量，你需要正确订购，以便 McD 的前端更清晰)，但内容正在放入。哦，有时候浏览器就是不能连接到服务器。

所以这是临界灾难，我们检查日志，它有几兆字节，所以我的任务是解析它并找出问题所在。

现在我不是一个 PHP 开发人员，我永远不会自称是。我喜欢 JavaScript，我喜欢能够一起做同步代码和同步代码。很好玩，很有意思。事实证明，同步代码有一些奇怪的地方。

他们的方式是我们的 cms 解析模板，它会首先呈现块和片段，然后是模板字段，然后是逻辑。

现在，我刚刚建立了一个完整的网站，由非常复杂的组件组成，运行得非常好，他们依赖于...很多逻辑。基本上，MODx 已经构建了片段和块，现在模板域要求 cms 返回并删除基于逻辑的片段和块。

这导致了一系列错误，遗憾的是，这些错误并没有全部解决，我不得不花费昨天和今天的所有时间来解析我自己的代码(你最好相信我会在开始编写下一个模板之前编写文档),并将我漂亮的组件提取到 MODx 可以以合理的方式解析的逻辑更少的组件中。

网站及时上线了吗？嗯，在我写这篇文章的时候(晚上 8:55)，考虑到他们是在西海岸时间，我们还有几个小时。

让我痛苦的是，如果我只是建立了客户需要的网站，我几周前就已经完成了。相反，我试着变得聪明，不去想客户。选中此字段，如果为空，则选中此字段，如果为空，则选中此字段，以此类推。接吻真的很有帮助。

我的狗在对我哀叫，让我注意它，但希望我吸取了教训。
