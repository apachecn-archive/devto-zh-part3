# 什么是开发者社区项目？

> 原文:[https://dev . to/dzello/what-are-developer-community-projects-1i1l](https://dev.to/dzello/what-are-developer-community-projects-1i1l)

[![Top community projects](../Images/f5111ddf92424b6b71143cdbdb3ffe68.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s---6qHTq4u--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://res.cloudinary.com/dzello/image/upload/f_auto%2Cq_auto%2Cw_800%2Cc_scale/developermode/promo-3)

> 成功的开发者社区不是通过营销建立起来的，有时尽管有营销也能成功。
> ——[汤姆·温特沃斯](https://tomwentworth.com/a-primer-on-developer-marketing-47d792d67586)

每个以开发者为中心的公司都面临着一个难题:你需要接触到你的受众，但是传统的 B2B 营销策略是无效的。

众所周知，开发者忽略了传统的 B2B 营销活动，如白皮书、门控内容、网络研讨会、冷外联和付费广告。开发人员从其他开发人员那里学习新工具和技术的方式通常是 T2。来自一家公司的任何“闻起来像营销”的信息都可能被忽视或遭到怀疑。

> 出于需要，开发商已经对传统的营销策略产生了免疫力。——[新的拥王者](https://thenewkingmakers.com/)斯蒂芬·奥格雷迪著

所以为了引起注意，专注于开发者的公司需要像开发者一样思考和行动。那是什么意思？这一切都是为了获得更多的实践机会。

[![Sweden Civictech hackathon](../Images/1336a7fc07dcc09397351010c790554c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--646tSVIl--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://res.cloudinary.com/dzello/image/upload/f_auto%2Cq_auto%2Cw_700%2Cc_scale/developermode/civictechsweden-hackathon)

开发人员通过实践来学习。他们经常通过写博客、发微博、在 DEV 上发帖来与同龄人分享他们新获得的知识🙌，在 GitHub 上发表演讲和发布代码。专注于发展的公司可以做同样的事情，为他们的产品或服务创造更多的知名度。

当一个单独的开发人员为他们的同事创造一些令人兴奋的东西时，这通常被称为一个兼职项目。当一家专注于发展的*公司*创建了一个项目来激励他们的社区时，我们称之为**社区项目**。他们是与厌倦了通常的营销噪音的开发者联系的好方法。

> 开发人员社区项目鼓励黑客和探索者在低风险的实践环境中学习新技术。

这些项目为开发人员社区提供价值，同时为母公司带来业务成果。然而，它们可能很难实现，不仅需要编写代码，还需要更大的跨职能努力，通常包括设计、产品和营销。

为了帮助揭开这个过程的神秘面纱，我们收集了一些最**成功和引人注目的开发者社区项目的例子，**以及它们共有的关键标准列表。我们还详细说明了一些好处，这些好处使它们成为向开发人员进行营销和销售的公司的首选工具。

## [](#community-project-criteria)社区项目标准

社区项目有不同的形状和大小，但有几个突出的共同属性:

*   对社区的价值
*   深度和互动性
*   独特的品牌和智能包装

### [](#value-for-the-community)为社区值

这是一个社区项目最重要的标准。不仅仅是不加掩饰地让开发者免费试用你的服务，它必须是有用的。

增加价值的一个方法是创建一个教育资源，比如 T2 JWT。来自 Auth0 的 IO。JWT。IO 教导开发人员如何以一种直接而彻底的方式使用 JSON web token(jwt ),并且它通过包含一个交互式调试器增加了一个全新的价值维度。

另一种给出价值的方式是制作一个解决问题的工具，比如 T2 的文档搜索。DocSearch 使得搜索开源项目的文档网站成为可能，解决了发布到 web 上的文档的历史难题。它对任何开源项目维护者都是完全免费的，Algolia 工程师亲自帮助新维护者入门。

[![Algolia DocSearch](../Images/be57df85eb206fbda891dfe90f1b7450.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--sNlNrd31--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://res.cloudinary.com/dzello/image/upload/f_auto%2Cq_auto%2Cw_800%2Cc_scale/developermode/algolia-docsearch-screenshot)

第三种增加价值的方式是创建一个**开源库、插件或框架**来帮助开发者构建他们的应用。看看 Stripe 和他们的[华丽的开源页面](https://stripe.com/open-source)就知道这种方法正在全面展开。这些项目中的许多都有自己的开发者社区，并且是 Stripe 特别强大的认知生成器和品牌建设者。

[![Stripe Open Source](../Images/95a74c6ce0bb356bb351ab798f01d0a6.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--r5mEjI2T--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://res.cloudinary.com/dzello/image/upload/f_auto%2Cq_auto%2Cw_800%2Cc_scale/developermode/stripe-open-source-screenshot)

### [](#depth-and-interactivity)深度和互动性

**开发人员边做边学，**因此，一个可以克隆并在本地运行的代码库将比一个静态的博客帖子更吸引人，也更有效。动手越多越好。

如果一个社区项目的目的是教学，最好有一个演示，并有一步一步的指导。一些最好的项目用一个有趣的数据集或一些有趣的 gif 来增加趣味。

然后是 TwilioQuest。 [TwilioQuest](http://twilio.com/quest) 将学习使用 Twilio API 的体验转变为一个完整的角色扮演游戏，包含复古图形、自定义头像、个性化任务和经验值。

[![TwilioQuest](../Images/6db51f5367ab0e9c15ea0011b7e7bb15.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--iHocbx23--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://res.cloudinary.com/dzello/image/upload/f_auto%2Cq_auto%2Cw_800%2Cc_scale/developermode/twilio-twilioquest-screenshot)

“TwilioQuest 是一款互动、自定进度的游戏，在这里你可以学习如何 Twilio。TwilioQuest 介绍 Twilio 产品、功能和概念，同时奖励您经验值和战利品。”👾

另一种获得深度的方法是展示多种技术的协同工作，这是 Stream 的方法，当时他们创建了名为 Cabin T1 的 React T4 Redux 教程。不仅仅是一堂简单的课，Cabin 还是一个学习 *6 种不同 API*的平台。每个 API 都有自己独特的教程，并为最终产品添加了有用的功能:这是一个成熟的图像共享应用程序，开发人员可以通过学习课程来构建它。

[![Stream Cabin](../Images/4f397f71544922bc3c6e0d2c69cb2855.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--LN7WefDw--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://res.cloudinary.com/dzello/image/upload/f_auto%2Cq_auto%2Cw_auto%2Cc_scale/developermode/stream-cabin-screenshot)

### [](#unique-identity-and-smart-packaging)独特的身份和智能包装

社区项目通常有自己的品牌、标志和颜色，通常与公司的品牌联系在一起，但又足够独立，使体验感觉过于商业化，并引起怀疑。一个社区友好的身份也将使项目对参与者和贡献者更有吸引力。

项目品牌化的一个很好的例子是 Mailjet 的 MJML，这是一个开源项目，可以很容易地创建响应性电子邮件设计。MJML 代表“Mailjet 标记语言”，但在登录页面上，公司引用是微妙的，该项目有自己独特而有吸引力的视觉标识。事实上，母品牌仅在登录页面的页脚处提及。**这种方法给人的印象是，通过 MJML，Mailjet 更关心为开发者提供一个有用的工具，而不是推广其品牌或推动注册。**

[![MJML](../Images/65da07560fbe8b968e13c8c9d9279475.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--vG3oJUSw--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://res.cloudinary.com/dzello/image/upload/f_auto%2Cq_auto%2Cw_800%2Cc_scale/developermode/mailjet-mjml-screenshot)

但是包装不仅仅是品牌和设计。专注于开发人员的分析公司 [Redmonk](https://redmonk.com/) 描述了软件打包的[概念](https://redmonk.com/rstephens/2017/02/15/monkigras2017/)，因为它关系到安装和使用的便利性、文档的质量以及它的故事被讲述和理解的程度。

**优秀的社区项目擅长打包和交付出色的整体开发人员体验，因为它们具有:**

*   清晰的最新文档，包括自述文件
*   包含常见使用案例教程的入门指南
*   清晰的投稿指南和行为准则
*   报告错误或获得帮助的已知途径，例如问题和 PRs
*   获取项目更新的方式，例如通过 twitter、聊天或电子邮件

一个包装良好的社区项目是 GitHub 的 Brandon Keepers 的 [Probot。自定义徽标、文档和社区构建的应用程序库都激发了信心，并使其看起来有趣且易于投入。](https://probot.github.io/)

[![GitHub Probot](../Images/0c28272b65c704ed7cb414c377cada5b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--_7YW34KV--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://res.cloudinary.com/dzello/image/upload/f_auto%2Cq_auto%2Cw_800%2Cc_scale/developermode/github-probot-screenshot)

## [](#benefits-for-companies)为公司带来的好处

当公司建立社区项目时，开发人员带着新的工具和知识离开，但是他们不是唯一的受益者。一个公司成功的社区项目将惠及它的每一个团队。它为营销和销售团队提供了在博客、会议和销售电话上谈论的有趣话题。对于每个团队来说，它建立了值得骄傲的士气，并增加了每个人对开发人员-客户如何思考的理解。

让我们更详细地看一下以下每一项优势:

*   用户获取、激活和保留
*   提前几年接触开发人员
*   鼓舞人心的内容、产品和文化
*   建立社区声誉
*   帮助招聘顶尖工程师

### [](#drive-user-acquisition-activation-and-retention)驱动用户获取、激活和留存

当开发人员发现一个很酷的项目时，他们自然会对项目背后的人感到好奇。这种好奇心经常会让他们点击查看制造它的公司。

> **“JWT。IO 现在是 Auth0 流量和注册人数的首选。每个月都有数百万开发者访问它。”**——马丁·贡托弗尼卡斯在他的博客上写道

另一个严重的流量驱动因素是 Netlify 的 StaticGen，它跟踪静态站点生成器的流行程度。排行榜经常变化，但 Netlify 的脚本使网站保持最新，**鼓励开发者经常回来查看。几年前，StaticGen 是一个热门的权威，是一个具有巨大 SEO 价值的社区项目的例子。**

[![Netlify Staticgen](../Images/878498dfa8129434d8e1d23a0c1cfb42.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--2cRnA6ea--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://res.cloudinary.com/dzello/image/upload/f_auto%2Cq_auto%2Cw_800%2Cc_scale/developermode/netlify-staticgen-screenshot)

### [](#reach-developers-years-ahead-of-time)提前几年到达开发商

Cristiano Betta 在他的文章 *[理解开发者](https://betta.io/blog/2017/02/23/understanding-developers/)* 中，根据他们的*动机*和*坚持的水平来考虑开发者与新工具或技术的关系。*

他称之为*黑客*和*探险者*的群体最有可能为了学习或好奇而尝试新工具，而**不是**因为他们有一个确切的问题要解决。到目前为止，他们代表了最大的潜在新用户群体。

[![Hackers and explorers](../Images/5f69d1ff5502b6b84f858a5b96b2255a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--H8augbdN--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://res.cloudinary.com/dzello/image/upload/f_auto%2Cq_auto%2Cw_700%2Cc_scale/v1548295593/developermode/developer-motivation-persistence)

社区项目帮助公司与这两个群体建立联系，拓宽营销渠道，并创造机会让开发者提前几个月或几年拿到工具。当开发人员最终发现自己有资格推荐或购买公司的产品时，公司为尽早与开发人员建立联系并保持头脑清醒所做的工作将会得到回报。

### [](#inspire-content-product-and-culture)激发内容、产品和文化

**社区项目往往是长期的、常青的，**这使它们成为开发友好型内容的重复出现的机会，这些内容可以在博客、社交媒体、播客、时事通讯中重新利用，甚至在销售场合作为提供价值的对话开场白。

只要看看 MailJet 是如何将 MJML 的影响扩展到其所有渠道的。他们已经写了大约 20 篇包含 MJML 的文章，从基本的项目更新到全面的教程，涵盖的主题有"[如何用 MJML](https://www.mailjet.com/blog/news/templating-language-tutorials-abandoned-cart-email/) 编写废弃的购物车电子邮件。"

[![Mailjet blog](../Images/7cbecf02dcc511555d7a0639d5abb8c9.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--D6trrhxf--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://res.cloudinary.com/dzello/image/upload/f_auto%2Cq_auto%2Cw_800%2Cc_scale/developermode/ss-mailjetblog)

有时，社区项目如此成功，以至于导致产品衍生，这正是 MailJet 在发布其 [MJML 桌面应用](https://www.mailjet.com/blog/news/mjml-gets-its-own-desktop-app/)时所做的。

最后但同样重要的是，社区项目影响内部文化。他们是新的东西，公司里的每个人都会对此感到兴奋，从中受益，并通常为之做出贡献。**这全面增加了开发人员的同理心**，提高了工程师和员工在面对客户时的效率。

> “开源项目确实对整个 Algolia 产生了影响，它们改变了我们的开发方式，并真正鼓舞了社区。” — [Vincent Voyer](https://twitter.com/vvoyer) ，Algolia 首席开发支持工程师

### [](#build-community-reputation)建立社区信誉

有价值的贡献可以建立信任，让公司成为开发者社区的积极参与者。仅仅为了赞助或推销他们的产品而参加每一次会议并不是这样。

一个活跃的社区记录会让接下来的一切变得更容易，比如获得高调的赞助、主题演讲、被介绍给有影响力的人，以及获得与更高级别的公司合作的机会。

Algolia 通过免费赠送 DocSearch 和其他计划，包括 [Yarn package search](https://blog.algolia.com/yarn-search-javascript-packages/) 和对[财政支持开源项目的承诺](https://stories.algolia.com/why-were-backing-opencollective-and-webpack-39e39838f18a)，与数百个开源维护者建立了关系。

### [](#recruit-top-engineers)招聘顶尖工程师

软件工程师希望为重视社区参与并为员工提供学习和贡献机会的公司工作。事实上，**根据 2018 StackOverflow 开发者调查， [56%的专业开发者为开源](https://insights.stackoverflow.com/survey/2018/#developer-profile-contributing-to-open-source)** 做出了贡献。

尽管 Stripe 的开源页面对潜在的 T2 用户很有吸引力，但最受影响的观众可能是那些正在考虑下一步去哪里工作的工程师。正如“由条纹”类别的存在所暗示的那样，**公众参与由备受尊敬的公司支持的开源项目是一份受欢迎的简历和职业生涯建设者**。

但是 Stripe 并不孤单。尽管 Airbnb 的客户不是开发者，但他们有一个令人印象深刻的页面来展示他们对社区的贡献，这无疑有助于他们吸引顶级人才，他们声称[“开源是我们在 AirBnb 所做的核心。”](https://airbnb.io/projects/)

## [](#start-a-community-project)开始一个社区项目

社区项目将宝贵的资源、良好的用户和开发人员体验以及独特的身份结合起来，为开发人员提供有用的东西，在这个过程中自然会提高项目创建者的意识。这是双赢的局面。
