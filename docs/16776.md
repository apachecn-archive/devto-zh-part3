# 从我第一次创业开始

> 原文：<https://dev.to/grepliz/starting-at-my-first-startup--3agd>

没有人喜欢改变。不舒服。挺吓人的。它让人不安地向未知跋涉。但是当我发现自己站在职业生涯的岔路口时，我明白了。我必须改变。行业不会一成不变，人也不应该一成不变。

我对软件行业并不陌生。我在大型公司和中型公司工作过。我在相当成功的公司建立了舒适的职业生涯，并从有趣的项目中学到了很多。我在上一个公司工作了 12 年，虽然一切都很顺利，但我内心有个小小的声音一直提醒我不要太安逸。

当我得到在 SKUR 工作的机会时，我欣喜若狂。我一直想为一家初创公司工作，并喜欢能够为一家基础公司做出贡献的想法。头两个星期是大量新信息的旋风。无论是在技术方面还是在领域方面都是如此。我了解了 AWS EC2 实例、S3 桶和代码管道。我浏览了 LAMP stack 应用程序的架构，阅读了 Python 代码，提交了对网站的修改，并给了 UX 一些建议。我了解到，许多扫描构成了一个点云，所有扫描默认为米。对某些人来说，这可能只是许多奇怪的词，对我来说，它们代表着打破我以前的舒适，开始与 SKUR 这样令人兴奋的公司一起旅行。

有人告诉我，建筑行业是采用技术最慢的行业之一，公司刚刚开始探索创新，这将推动他们更上一层楼。我很高兴能为这一目标做出贡献，也很高兴能沿着这条路走下去。

这篇文章最初发表在 SKUR 公司博客上。
