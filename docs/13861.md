# 开发者能从艺术课程中学到什么

> 原文:[https://dev . to/hzlmn/what-can-developer-learn-from-taking-art-classes-2hp 8](https://dev.to/hzlmn/what-can-developer-learn-from-taking-art-classes-2hp8)

大家好👋几个月前，我决定尝试一些新的东西，开始参加一个学术绘画班。我总是对令人惊叹的艺术品印象深刻，但我从来没有试图自己做一些事情。我最后一次尝试绘画是在学生时代，我从未想过我会再次尝试😃。

对我来说，这一切都是从阅读[保罗·格拉厄姆](http://www.paulgraham.com/)的经典[黑客与画家](https://www.amazon.com/Hackers-Painters-Big-Ideas-Computer/dp/1449389554)书开始的。在其中一个章节中，作者在艺术和“黑客”之间画了一个清晰的平行线，并提出了有力的声明，即开发人员的工作在大多数情况下更接近艺术而不是科学领域。

我决定自己检查一下，在与艺术家交流和探索这个领域这么长时间后，我发现大部分创意人员的心态与技术人员没有太大区别。更重要的是，我们在思考问题和解决问题的方式上有很多共同之处。这是我到目前为止做的一些关键笔记。

# [](#master-fundamentals-books)掌握基础知识📚

[![](../Images/e8a04ba21b4c748619ae87d512e6ebf2.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--aQSXk1k3--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://instagram.fdnk1-2.fna.fbcdn.net/vp/ceff6739a57919a00d2b48d324637db5/5CE6B26F/t51.2885-15/e35/32788416_791106627766187_1868360251783249920_n.jpg%3F_nc_ht%3Dinstagram.fdnk1-2.fna.fbcdn.net)

首先，这似乎是显而易见的，但是想想所有这些每个月甚至每个星期都会出现的闪亮的新技术和工具。人们很容易迷失在炒作中，但大多数情况下，它们都有共同的基本概念。

任何优秀的艺术家都是从探索简单的形状开始，以形成一个坚实的基础，只有在这之后，才能制作一些更难的物体，因为最复杂的物体只是简单图元的良好组合。

这是真正与编程相关的，而不是学习新的流行框架等。试着想想它实现了什么概念，做了什么基础研究。

# [](#ignore-details-on-first-stages-pointup)忽略第一阶段☝️的细节

[![](../Images/605e0819f015a9c890d24965a86d8fa5.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--nCmVDfWn--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/nn1cqj9k0ypzzraqu2ba.JPG)

当你试图画一些形状时，当你专注于一件事情时，很容易忘记整个物体。我的老师总是告诉我，一个结构良好的对象，即使失去了一些细微的差别，也会比一个有很多细节但没有坚实的组合形式的对象看起来更好。

现在考虑制作软件项目，在第一阶段添加细节(不必要的依赖、过早的优化、复杂的抽象)会增加风险。相反，试着想出一些“可行的”解决方案，即使它还不太好。

> 过早的优化是编程中所有罪恶(或者至少是大部分罪恶)的根源唐纳德·克努特

力求进步，而非完美。随着时间的推移，项目会变得越来越复杂，所以在最初的步骤中添加不必要的东西可能会使其不可伸缩。这就是为什么我认为亲吻是最重要的概念。

# [](#recognize-common-patterns-clipboard)识别常见模式📋

[![](../Images/0b060409176f9f6cfbf92351ac807230.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--DeJUFcUE--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://instagram.fdnk1-2.fna.fbcdn.net/vp/735fe9e2629ef871680edba112133818/5CE67846/t51.2885-15/e35/39304319_2146269228780130_3597159771825766400_n.jpg%3F_nc_ht%3Dinstagram.fdnk1-2.fna.fbcdn.net)

例如，看这幅图，这是一个非常经典的尿布折叠。乍一看，新手可能很难理解从哪里开始，但如果我们仔细观察，我们可以看到这个复杂的形状主要由简单的图元组成，如圆锥，它们之间有平滑的过渡。识别这些图案对艺术家来说非常重要。

这是一个很好的视觉演示，说明了为什么学习[设计模式](https://sourcemaking.com/design_patterns)对于一个优秀的工程师来说是至关重要的。知识，更重要的是，正确检测并在项目中应用它们的能力，将使您的代码更加健壮，更容易理解。

# [](#conclusion)结论

[![](../Images/90ab2e1b35102847ba5cea31933c4a4c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--IU2V6Pp5--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://instagram.fiev10-1.fna.fbcdn.net/vp/2e2bcf5999390f973c3fdfd9871f1671/5CEE06C6/t51.2885-15/e35/49933482_1025602667563468_1841351465186509710_n.jpg%3F_nc_ht%3Dinstagram.fiev10-1.fna.fbcdn.net)

在我短暂的“实验”后，我可以自信地说，探索新领域和从不同角度看待已知事物是非常有用的。学习新技能，即使与你的职业不直接相关，也能让你更好地解决问题，让你对常见问题有新的看法。
