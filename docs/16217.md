# 获得系统当前日期和时间的合法途径是什么？

> 原文：<https://dev.to/ankitutekar/what-is-the-legit-way-to-get-the-current-date-and-time-of-system--mg9>

是的，标题有点奇怪，但是继续读下去，我有一个有趣的故事要讲。最近在工作中，我的任务是实现一个定时器。这相当简单，至少我是这么认为的。用户执行一些操作- >后端触发一个事件- >后端执行一些计算，保存数据和当前日期时间，并返回响应，包括记录的日期时间- >浏览器在收到响应后启动计时器。计时器应该从后端事件发生时开始运行 1 小时。
问题是，我的系统时钟被设置为比实际时钟慢 8 秒。因此，计时器会考虑额外的 8 秒。
我们有服务器端验证，所以即使用户可以在计时器结束后执行操作，我们也做得很好。
我的问题是，除了依靠客户端系统获取当前日期时间，还有其他更好的选择吗？你使用服务器端计时/第三方服务来做这些事情吗？如果是，网络延迟如何？像在线考试系统这样的系统是如何处理的？
