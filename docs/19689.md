# 活下去

> 原文:[https://dev.to/gmeluski/stay-alive-iaj](https://dev.to/gmeluski/stay-alive-iaj)

我在某处读到过，随着你在组织中级别的上升，你所能控制的和你所不能控制的平衡在等式的“你不能”部分聚集得更多。成为一名工程经理，对我来说，成了这个习语受欢迎的一个例子。为了这个特殊的职位，我必须比以前更多地接触外界的影响。

作为一名 IC，我的许多技术知识都是由公司中有较大影响力的人所做决策的结果。一般来说，一个集成电路可能在某项技术上表现出色，但这项技术的选择是由高层人员做出的，他们随后找到了相关人员。集成电路可以就细节争论一整天，但一项技术选择的成败取决于非管理层或执行层的人，这种情况很少见。

很长一段时间，我依赖我的个人网络和同事来了解 web 工程领域的重要发展。通过选择重视应用新技术的组织，很可能会有关于技术的持续内部分享和辩论，推动我的知识向前发展。我很少搜索资源，因为信息是通过去工作和与同事相处的自然过程获得的。

后来我成了工程经理。自上而下的信息将更多地成为管理和组织的倾向，而不是技术的。我也将变得更加孤立于我的直接同事:我有他们，但我们都花更多的时间关注我们团队内部的事情。

担任工程经理也意味着我承担了更多的技术选择和所有权。这意味着我没有太多时间去深入探索技术，而且不得不在“它对我有吸引力[插入高度主观和有争议的理由]”的层面上评估事物。所以我开始建立一些参考资料，我会把它们整齐地放在一起，我可以称之为“基础”顺序——一个类别越早，它就越有可能是一个长期的奖励结构，你可以利用它来建立知识，代价是与当前发生的事情不那么直接相关。

我当然不是这样来找他们的。当我意识到它已经超出了我的理解范围时，这更像是把所有的东西都扔进一堆，并最终以一种结构更合理的方式重塑这一堆。

## [](#books)书籍

如果你花足够长的时间研究任何类型的互联网推荐的自我提升途径，你总是会遇到一些人，他们本质上说*你必须阅读*。我同意，而且是毫不含糊的。这里的其他建议都很棒，但是出版一本书所花费的时间、精力和精力实际上意味着从内容中获取信息和想法。关于如何找到时间或如何提高阅读质量的信息与我在这里要说的是不同的，所以我很乐意把它留在这里:

TL；大卫:你没有选择。你必须阅读。

也就是说，要成为现代编程范例的一部分，远不止 600 页的测试驱动开发之类的东西(尽管它们仍然很重要)。有关于产品开发的书，有关于 UX 的书，有关于管理理论的书，有关于目标系统的书。这是所有的东西，对列表中后面的事情来说都是基本的。没有这个基础，它增加了某种类型的智力混乱的可能性——不知道追逐哪种技术，试图追逐所有的技术，最终失败。

但是，在哪里可以找到把小麦和众所周知的谷壳分开的建议呢？

继续读。

## [](#newsletters-and-articles)时事通讯和文章

很长一段时间，我都找不到一个可靠的及时相关的文章来源。队友会带着一些新的、不同的或前沿的信息出现，而我是第二个(或第三个或第四个)。在组织结构中的提升意味着我必须在知识队列中稍微靠前一点。我必须决定所述技术是否相关或者是否有用，这需要时间。

当我为一家出版物工作时，当时许多产品人员的口头禅是这个叫做时事通讯的小东西。然后我意识到，是的，科技简讯确实存在！正如我开始欣赏基于新闻的时事通讯的魔力一样，基于技术的时事通讯甚至更好，因为它们包含了可操作的信息。

所以有人在宇宙中，把所有这些令人难以置信的相关和有趣的文章集中到一个地方，所以我不再需要一点一点地寻找它们。不仅仅是一个人，也不仅仅是一个主题。正是众多的主题突然变得与我的角色密切相关。所以我订阅了。又订阅了。又订阅了一些。直到我有一个堵塞的邮箱，我别无选择，只能放弃其中的一些。

## [](#twitter)碎碎念

啊。是的。精神错乱。拒绝被驯服的混沌，肆无忌惮地、充满激情地向宇宙蔓延。除非你愿意在应用程序中点击一两次以上，并开始创建列表。这也是我从记者艾伦·鲁斯布罗格写的一本名为《再玩一遍》的书上看到的。同样值得注意的是，这是一个类别支持另一个类别的另一个例子

列表+ Tweetdeck？是啊！有一些东西没有 twitter 账户，但是有 RSS 订阅，这需要做一些事情来在 twitter 订阅中获取他们的信息。最终，我建立了一个新的 twitter 账户，可以使用 IFTTT 发布最新的 RSS，然后将这个账户添加到相关的 Twitter 列表中。这是一个保持在更直接、更令人兴奋的事物之上的好方法，而不必去追逐事物。我是说，就好像信息一直在向我涌来。毫无疑问，很有启发性。

[![](../Images/de753251f9d3e99d8b8bdb5ed2944010.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--sMXuTYdS--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/http://meluski.com/assets/images/tweetdeck.png)

此外，我养成了把我的清单放在 Tweetdeck 左侧的习惯，把我的主要时间线越推越远，被遗忘在它应该在的地方。不是因为它不好(尽管它可能会因为一天或一周的事件而变坏)，而是因为它是一个几乎无限的吸引人、有时令人愤怒的信息流。我喜欢跟随聪明、有趣、固执己见的人。我喜欢政治和治理，所以你可以看到持续关注我的时间表而不是我的清单会让我偏离我需要做的事情(了解与我的工作相关的事情，记得吗？).所以，是的，我已经策划了占据我屏幕左侧的列表。

说到技术，关注某些项目或这些项目的主要贡献者是值得的。一般来说，谈到发展，我喜欢选择那些正在做我觉得有趣的事情的公司和人。如果商业想法是有趣的，那么他们的产品和技术想法也可能是有趣的，或者至少值得一谈。最终，这就是我在这个层次之外所寻找的。
