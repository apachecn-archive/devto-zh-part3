# 从初级开发人员的角度写作

> 原文:[https://dev . to/http junkie/writing-from-a-beginner-developers-perspective-30l 0](https://dev.to/httpjunkie/writing-from-a-beginner-developers-perspective-30l0)

当你写软件开发或以任何方式试图教别人任何与技术相关的话题或学习新的或外国的概念时，从初学者的角度来写和教是必要的，这样你就不会疏远或遗漏你的读者或学生可能需要的重要信息。试着把自己放在一个从头开始的人的位置上，记住在你的旅程中绊倒你的事情，这些事情有时会非常小，看起来微不足道，但如果你把它们遗漏掉，会对某人的学习产生巨大的影响。

## [](#why-from-a-beginners-level-perspective)为什么从初学者的层面来看？

首先，它让我可以和读者一起学习。作为一个最近才从 Angular 转向 React 的人，从这个角度来写肯定对我有好处。此外，我觉得我可以巩固自己的基础知识，并思考其他人在同一话题上会遇到的问题。有了 React，我就进入了这个领域，因为钩子在 16.x alpha 版本中开始变得可用。我知道我需要做的是从一个初学者的角度重新开始，并了解钩子如何对那些初学者有益，但这也意味着回去理解在钩子出现之前 React 应用程序是如何编写的，这样我就可以在这个问题上有一定的权威性。

我认为，从 react Fiber 开始，React 经历了真正的转变。我很惊讶 React 团队是如何为 React Hooks 的发布做好准备的，这个发布大约发生在一周前。在我的文章中，我说我认为钩子计算得很好，我认为它们为 JavaScript 社区和 React 提供了很多。它们允许轻松地共享代码，并以更简洁和干净的方式挂钩到其他逻辑，同时将该逻辑与其他关注点分开。我很喜欢鱼钩，并且认为没有鱼钩的人是真的睡在鱼钩上。

当 Hooks 发布的时候，我刚刚尽可能多地学习了 React，这很有趣，因为我知道我需要回去更多地学习基于类的 React，而不是直接投入进去。我讨厌的一件事是，当有人写“新的闪亮的东西”，但不能解释为什么它很棒，以及我们来自哪里。我不仅希望能够做到这一点，还希望能够回答一些棘手的问题，比如为什么我认为它们如此伟大，以及它们将如何帮助我们编写更好的应用程序。对我来说，这是一种脚踏实地的方式。我也知道最好是从初学者的角度来写 React，考虑到要开始使用钩子，就意味着要从我们如何编写和组合 React 组件开始。

我知道，我可以利用这个机会，不仅进一步了解 React，还可以推广我所倡导的产品。我要做的事情之一是开始将我们简单的产品演示转换成 React Hooks。这帮助我通过一遍又一遍地做来记忆我学到的东西。写作也为我做到了这一点。然后是时候开始学习如何用 React Hooks 重构一个实际的应用程序了，[Kent C . Dodds](https://www.youtube.com/user/kentdoddsfamily)在 YouTube 上有一个很棒的视频，但是我突然找不到了。如果有，我会贴出来。

顺便说一句，我还想说，这些年来，我在开发人员社区里有很多我尊敬的人，Kent 是其中最平易近人和最好的一个。这对我来说非常重要，我想感谢他和其他人(陈宸和肯·惠勒也想到了)超级平易近人，或者只是关心并与我们这些凡人互动。我对 JavaScript 的大量学习通常是从这些内容创建者的视频或博客文章开始的。

起初，我并没有意识到我会陷入一个多么令人惊奇的话题。到目前为止，我写的关于 Progress 的大部分东西都是 React Hooks。我是从一个真正的“初级水平”开始接触这个话题的，我希望能够将这些知识打包，让其他人更容易接触到。并不是说 [ReactJS](https://reactjs.org/docs) 的医生们还没有这样做，但我仍然觉得，利用我所使用的所有资源来学习它们，并为那些想要走捷径的人浓缩起来，是很有价值的(免责声明:生活中没有真正的捷径)。虽然文档是最好的资源，也是任何人在学习 Hooks 时应该首先查看的地方，但是我觉得如果我从初学者的角度学习这个主题，我可以为我的博客读者提供一种非常容易理解的方式来了解 React 中这个惊人的新概念，特别是如果他们是 React 或 JavaScript 的新手。同样，我认为了解一个主题的最好方法是写下它，我不在乎其他人是否在做同样的事情，也不在乎我是否经常引用他们的东西。它仍在增进我对这个问题的了解。
