# 如何为您的项目找到好的库

> 原文:[https://dev . to/flipped coding/how-you-can-find-good-libraries-for-your-projects-5 ghj](https://dev.to/flippedcoding/how-you-can-find-good-libraries-for-your-projects-5ghj)

当你选择图书馆时，既有太多的，也有不够的。对任何一个人来说，有太多太多的事情需要去了解，也没有足够多的事情让你放心去相信。那么你怎么知道哪些是好的呢？当您为项目选择库时，需要考虑以下几点。

## [](#how-long-has-the-library-been-around)图书馆存在多久了？

每个人都知道目前正在发生的科技热潮。这基本上就像新的淘金热。这就是为什么你必须检查一个图书馆存在了多久。每天都有那么多新的东西冒出来，我知道其中隐藏着一些伟大的东西。尽管当你为客户做一个项目时，你不能做太多的实验。

你需要一些你知道有用的东西。也许几年后新的会被证明是伟大的，但是现在你需要一些有一点历史的东西。看评论和评论。看看过去两年人们对图书馆的评价，你就能感受到使用它的好处。

## [](#when-was-the-last-update)最后一次更新是什么时候？

对于已经存在一段时间的库来说，这是你必须关注的事情之一。有时候你以为自己已经挖到了金子，却发现上一次有人碰它是在 2009 年。更糟糕的是，如果在项目中使用这个库之前没有检查，现在所有东西都坏了，因为它是用已经不存在的东西写的。不像发生在我身上的那样…

无论如何，你要确保在过去的一年中至少有一次更新。最好是过去 6 个月，但有时图书馆不需要每隔几个月更新一次。只要你能看到有人不让代码变成历史课，那么它就应该很好用。

## [](#is-it-compatible-with-your-project)与你的项目是否兼容？

您必须确保该库使用的语言适合您的项目。我知道这听起来很蠢，人们也不会不去检查，但这确实发生了。你可能会被一个库的美丽描述所吸引，忘记它是用 C++编写的，而你使用的是 JavaScript。

它不仅仅是检查语言兼容性。这是您项目的正确版本吗？它使用了正确的语法还是过时了？您可以偶然发现最老版本的 project 的完美库，但这对您没有帮助。你需要一个与你的版本兼容的库，否则奇怪的事情会发生，而且很可能发生在错误的时间。因此，请确保该库与您正在使用的库相匹配。

## [](#is-there-good-documentation)有没有好的文档？

无论做什么，都离不开文档。如果你第一次使用一个库，那么你需要一些好的文档。首先，你能找到任何文件吗？如果你不能从创建图书馆的人那里找到任何东西，这是一个不好的迹象。你可能必须学会如何通过纯粹的意志力来使用它。

找到一个有好文档的库是必须的。当您发现那些奇怪的错误似乎无缘无故地发生，并且您知道代码的其余部分以前是正常的，该文档将帮助您修复这些错误。当你想把这个库推得比它的现成文档更远时。你需要它！

## [](#is-there-a-community-around-it)周围有社区吗？

不管一个图书馆有多棒，总会有一个奇怪的问题，你似乎发现了，只有另外三个人知道。你能找到那三个人吗？一个健康的社区是一个图书馆将会得到良好支持的标志。即使建立图书馆的人不再支持它，一个健康的社区通常会接管它。

另外，有一个特定的地方让其他人帮助你调试代码总是好的。我们都遇到过堆栈溢出中的一个人帮助拯救了我们的生命。这就是你希望在社区中看到的情况。

## [](#what-are-the-close-alternatives)有哪些比较接近的备选方案？

凡事都有某种替代。在图书馆的世界里，你可以很容易地找到 1000 个图书馆在做同样的事情。他们只是做的稍微不同。这就是为什么一旦你找到一个你有点喜欢的，看看其他几个图书馆是有好处的。结果可能是你喜欢的库正是你需要的，但是看看另外一两个库会让你对自己的决定更有信心。

四处看看需要一点额外的努力，但相信我，它可以节省你几天的工作和一些尴尬。但是不要走得太远。看 1-3 个备选方案再做决定。最糟糕的事情将会发生，那就是你必须将你的项目恢复到一个不同的版本。确保在下载新库之前提交所有内容！

如果你研究这些东西，你将更有可能找到一些你可以使用多年的好的库。不要害怕尝试其中的一些，看看它们在你的项目中是如何工作的。这就是 GitHub 的用途。我花了一段时间才找到一个好的日期时间库。时区如此复杂真是疯狂。你使用过哪些库来使你的项目更好或更容易？
