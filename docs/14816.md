# 技术选择

> 原文：<https://dev.to/ukutaht/technology-choices-506h>

总的来说，我不认为技术栈对软件项目有太大的影响。尤其是如果你是一个单独的制造商，几乎所有的风险都在销售和营销方面，而不是技术方面。最好的方法通常是选择一些无聊的工具，开始解决客户的问题。

然而，我也非常热衷于开发工具，我非常喜欢研究编程语言和不同的数据库。在这篇文章中，我将回顾到目前为止我为[似是而非的](https://plausible.io)所做的主要技术选择。

### 仙丹+凤凰

我觉得大部分程序员都有自己喜欢和习惯的默认语言或生态系统。对我来说，那是灵丹妙药。我第一次尝试是在 2013 年我看到迪夫·托马斯谈到它的时候。从那时起，我开始了 Elixir Koans 项目，并在许多商业项目中使用了 Elixir。

这是一种奇妙的语言，设计时对细节给予了如此多的关注。有些人可能认为它是新的和现有的语言，但对我来说，这是一个可以依赖的成熟技术。毕竟，Erlang VM 从 1986 年就出现了，而且已经非常成熟了。

我对 picking Elixir 的唯一担心是，它不是一种超高性能的数据处理语言。如果似是而非是成功的，它将不得不吸收大量的流量。我是说很多。所有的流量也需要进行分析，以提取有用的信息，如浏览器版本和参考来源。我不知道长生不老药能在多大程度上解决这个问题，但我们会拭目以待。我可能必须在某个时候用低级语言编写一个合适的摄取管道，但这将是一个很大的问题，因为这意味着产品是成功的！

### PostgreSQL

再说一次，当谈到持久性时，这几乎是我的默认选择。Postgres 非常成熟，经过深入研究和广泛的实战检验。这是一个美丽的开源项目，他们每年都在做得更好！

如果似是而非真的起飞了，它必须吸收大量的流量，我可能不得不转而使用 InfluxDB 。intrusion 是专门为高写入速度和基于时间的聚合查询而构建的时标数据库。它非常适合分析应用程序。另一方面，有一个叫做[时标](https://www.timescale.com/)的 Postgres 扩展做同样的事情，看起来也很强大。加上是 Postgres！

### 顺风 CSS

我决定用一些新的东西来代替传统的、无聊的 css 编写方式。Tailwind 是一种非常独特的编写 HTML+CSS 的方式。查看他们的网站,了解更多关于他们实用至上的方法。

我花了几天时间才到达顿悟时刻，但一旦我做到了，就没有回头路可走了。与我尝试过的其他方法相比，我可以用更少的时间获得更好看的设计。我肯定会在未来的项目中再次使用它。

### 服务器渲染的 HTML +零星 JS

似乎现在每个新项目默认都是单页应用。我想开发人员已经习惯了 React 和 Angular 这样的工具，以至于忘记了如何使用它们。

我选择了经典的基础设施:在服务器上提供 HTML，在前端加入 javascript 以实现交互性。我甚至放弃了 jQuery，因为按照现代 web 标准，脱离普通 Javascript 相当容易。如果标准还没有得到很好的支持，您总是可以包含几个聚合填充。到目前为止，我唯一引入的 javascript 库是 Chart.js

我对这种方法非常满意，它让事情变得简单，网站加载速度也很快。与现代水疗相比，传统的服务器端应用程序中的活动部分更少，这使得它们更容易推理。

### 外部工具

到目前为止，我已经集成了以下外部工具:

*   [Heroku](https://www.heroku.com/) 主持
*   [哨兵](https://sentry.io/)进行错误追踪
*   [邮件枪](https://www.mailgun.com/)用于发送电子邮件
*   [用于 DNS、SSL 和 CDN 的 Cloudflare](https://www.cloudflare.com/)

我目前唯一付费的是 Heroku，只是为了确保 dyno 不会休眠。
