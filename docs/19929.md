# 一年的软件开发经验

> 原文：<https://dev.to/flaviabastos/one-year-as-a-software-developer-3612>

过去的一周标志着我成为全职软件开发人员一周年。我认为停下来反思过去的 12 个月是很重要的，所以这是我的“第一年回顾”。你可以[在“关于”页面](https://flaviabastos.ca/about/)阅读更多关于我的背景，但这里是 TL；博士:我是一名自学成才的软件开发人员。

###### 今年学到的东西

**事情 1:保持记录的价值**

我在上班的第一天就打开了 Google Docs，从那以后，我一直在记录我学到了什么，我在努力做什么，我的同事如何帮助我，如何解决一个特定的问题，如何解决一个问题，我第一次提交代码库等等。这非常有价值，原因如下:

1.  这是我与经理两周一次的一对一会谈的主要参考资料

2.  我有一个关于我过去如何故障诊断和解决类似问题的参考(我已经无数次使用这个文档了！)

3.  我可以看到我里程碑(我的第一次承诺，我帮助一个资深同事的时候，等等)并庆祝它们

4.  我可以回顾过去某个时候我在努力解决的问题，并意识到这个问题/概念不再那么困难了

5.  给予同事反馈

我让这份文件一直开着，这样就很容易找到，我可以在事情发生时写下来，这样就不会有忘记任何东西的风险。我还发现，即使我可能花了几天时间来解决问题和学习某个特定的主题，几个星期后，我将不再记得这些信息，除非我回到我的笔记。所以，未来的我，继续记笔记吧！

**第二件事:新技术概念和工具**

今年我学到的技术(日常相关的)东西的数量相当多。事实上，我可以说我每天都学到新的东西！这里是我在过去一年中做的一些很酷的事情(无论是在工作中还是在业余项目中):GIT、Docker、shell script、IPv4 和 IPv6、websockets、TCP-server、测试框架、REST APIs、产品分析等等。有些我不得不从头学起(shell 脚本！)，有些我建立在一些以前的知识(GIT)上，有些我仍然模糊不清(tcp-servers)。然而，学习是没有止境的，这太棒了！

**第三件事:事情并没有变得更容易**

做一名开发人员从来都不容易。你学到了一些东西，这些东西不再是一个谜，但你还有一长串的谜要去挖掘。我认为除了我之外的每个人都完全了解每时每刻发生的事情，所以我会在一个问题上“放弃”,请求同事的帮助，完全期待他们来到我的办公桌前，在一两分钟内解决问题。那从未发生过。事实上，每次我们都会一起盯着我的屏幕看一段时间。所以，这是一个教训:总会有一些事情让我苦苦挣扎。随着我了解的越来越多，具体情况会有所变化，但这份工作的本质就是不了解。没有人什么都知道，那是不可能的。我们都必须接受这一点。

**事情四:求助**

每个人都需要帮助，尤其是在刚开始的时候。例如，我学会了尽我所能地做研究和探索，并且不羞于寻求帮助，甚至在会议中举手说我不明白的事情。一个关键的问题是如何寻求帮助。我可能会在未来写更多关于这方面的内容，但有一种错误的方式(“我的东西不起作用”)和一种正确的方式(“我需要做 x。到目前为止，我尝试了 w，y 和 z，得到了这些错误/响应。请问有人能帮我吗？”).所以，寻求帮助是可以的。

**事情五:时间是有限的资源；明智地对待你的开销**

bug 需要修复，特性需要编写，测试需要测试，哦，不要忘记查看刚刚发布的关于 flash-new-thingy 的新文章。可能会让人不知所措。今年早些时候，我努力寻找足够的时间来做我认为应该做的事情，直到我最终意识到我必须优先考虑。我认为这里仍然有一些改进的空间。

事情 6:每件事看起来都太难了，直到你开始着手去做

真实故事:我们有一个相当长的 bug 清单要处理，而这个领域的潜规则是“你总是选择清单上最重要的 bug 来处理”。我会阅读错误描述，并认为不可能，呃-呃，我不知道那个问题是什么，对我来说解决它太难了，也许更资深的人会更适合。这种情况每时每刻都在发生。现实是我别无选择(另外，每个问题似乎都同样复杂！)当我开始排除故障、阅读代码和进行研究时，事情开始变得明朗起来，最终我能够解决这个问题。然而，转到下一个问题时，我又回到了“做不到”的心态。我学到的教训是，我们不知道的每件事都很难，但一旦我们开始探索，思考就会变得有意义。
