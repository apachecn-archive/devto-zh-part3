# 帮助我一路走来的资源，前端和后端...

> 原文:[https://dev . to/Sarah Ahmed 626/front-back-end-but-am-just-a-beginner-277d](https://dev.to/saraahmed626/front-back-end-but-am-just-a-beginner-277d)

我的旅程一团糟！来自计算机科学的学术背景帮助我更快地理解了一些概念，但在大学时代，我对此没有任何兴趣，也没有花太多精力去了解它。因此，作为一名开发人员，我现在所拥有的技能只能归功于自学。
*不是每个计算机专业的毕业生都是程序员！*一个事实。
我非常兴奋地开始开发一个真实世界的项目，但是怎么做呢？在哪里？什么时候？

### [](#test-projects)测试项目

由于除了未完成的尝试、挑战、例子和教程之外，我没有任何编程经验，当我第一次开始工作时，我被要求阅读有关 MVC、C#和。net，我开始阅读这个和那个，更多的教程，文章，视频，没有时间限制，我只需要尽可能多地阅读和实现。大约 3 周后，我收到了我的第一个项目:一个电子商店。在我开始开发的团队成员的一些基础帮助下，接下来我接受了第二个项目，是一个医院预约系统。
项目的主要目的不是让我完成这两个项目，当然，一个初学者甚至有经验的开发人员都不会在给我的这么短的时间内完成这样的系统，目的是通过做来实践，弄脏我的手，衡量我的技能水平和性能。
我对事情的进展感到非常惊讶，随着每个问题的出现，我要么通过面对问题、思考解决方案并进行研究来学习，要么通过实际解决问题来学习，在这两种情况下，我都得到了知识的支持，这些知识是我从之前未完成的尝试中获得的，我实际上获得了知识和技能，但我不知道自己已经拥有了它们。

### [](#backend-development)后端开发

在两次失败的 100 天代码挑战尝试中，我通过以下方式使用 Django 框架学习 python:**Eric Matthes 的 Python 速成班**举例来说，当时我并不觉得自己有所进步，但后来当我在. NET 中实现 MVC 时，这一点浮出了水面。
这是我真正喜欢编程的一点，一旦你理解了一种语言或框架中的概念，以后就很容易在另一种语言中理解。编程基础也是如此:数组、类、对象、循环..我以前在 PHP、Java、Python、JS 中练习过，当我开始使用 C#时，我甚至不用看它，它就在那里。

### [](#front-end-development)前端开发

作为一个“大部分”左脑的人，可以看出我和前端不会成为很好的朋友，但团队的代码是做前端和后端。那时我在想:好吧，后端你知道概念和语法，你应该构建一个二三...etc 具体的东西和一切都是好的，前端是另一方面！技术、语法、概念看起来很简单，但在后端没有明显的指导方针，所有这些技术都需要混合在一起，但如何混合呢？另外，你需要建立的是完全取决于你没有明显的边界线，它需要想象力和创造力的火花！

#### [](#css-and-html)CSS 和 HTML

FreeCodeCamp 课程:从零到拥有一个由我从头构建的作品集:)
响应式网页设计课程创造了奇迹，因为我不知道从哪里开始，但通过应对挑战，在完成课程之前，我开始编辑我的医院预约项目设计，并获得了信心，知道是的！我实际上可以做一些前端任务，我非常喜欢这个简单、优雅的设计。
在我完成课程后，我做了 5 个最低要求的项目，只是为了通过挑战，并获得证书，是的，这很俗气，但你知道，一个人需要鼓励！然后我对个人作品集项目更感兴趣，所以我做了很多修改来完善它。结果很好，我又一次想:哦，哇，这看起来像一个真正的网页，而且是我自己从头开始做的！我的信心增加了很多。

#### [](#javascript)JavaScript

起初，我在 100 天的代码中学习了 FCC JavaScript 和数据结构课程，知道我已经完成了大多数挑战，但我对 JavaScript 的用途没有丝毫概念，也不知道它应该放在项目代码中的什么位置？！这就是为什么我不认为它适合像我这样的完全初学者，在这件事上让我受益最大的资源是 [MDN - JavaScript](https://developer.mozilla.org/en-US/docs/Web/JavaScript) 我会给它一百万个赞。

### [](#first-task-as-a-software-developer)作为软件开发人员的第一项任务

随着时间的推移，3 个月后我开始感到无聊，并希望参与到一个真实世界的项目中，所以我停止了我的测试项目，并完全停止了练习。几个星期后，我收到了我的第一个真正的项目任务，我非常高兴，因为我需要知道在这一切之后我将如何表现。任务是添加图形到现有的销售和营销项目+完成报告和这些报告的过滤器。
我表现得非常好，在那一点上我知道我需要停止担心，理解这个概念，实践它，放松，因为当你以后需要它时，它会在那里等着你去使用它。

### [](#a-supportive-community)一个支持的社区

twitter 上的代码标签 100 天社区过去是，现在仍然是游戏规则的改变者，这是我了解大多数有用资源的地方，twitter 上的开发人员社区通常非常支持、鼓励和友好，参与该社区。今天:)

### [](#summary)总结

在编程过程中，你永远不知道你已经掌握了所有这些技能，直到你开始开发一个项目并开始动手，你会对结果感到惊讶。学会学习，是的，你没看错学会如何学习，理解概念，语法并不重要，它们最终都会走到一起。
