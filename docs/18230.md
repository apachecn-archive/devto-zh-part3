# 抵抗和学习编码

> 原文:[https://dev . to/vaidotas/resistance-and-learning-to-code-2bgh](https://dev.to/vaidotas/resistance-and-learning-to-code-2bgh)

## [](#what-is-resistance)什么是阻力？

你知道那种坐下来写代码，最后在 YouTube/Twitter/Egghead/plural sight 上看教程和“学习”的感觉吗，感觉很有效率，但实际上没有写任何*新的*代码？有人称之为教程地狱，有人说简直就是拖延症。

还有一个词可以形容它——反抗。

史蒂文·普雷斯菲尔德在他的书《艺术的战争》中把抵抗描述为一种自我产生、自我延续的东西，一种感觉，一种当一个人想象自己是谁(作家、画家、企业家)时所经历的拉力，但是他们并没有采取措施去实现这一愿景。

在文章的第一段，他描述得很好:“你是一个不写作的作家，一个不画画的画家，还是一个从不创业的企业家？那你就知道什么是反抗了。”把这个调整到手边的话题——“你是一个不编程的程序员吗？”

## [](#recognizing-the-pull-of-resistance)认清阻力的拉力

作为一个在职业之间转换的人，从已知到未知，从我擅长的事情到令人恐惧的事情，每天当我坐下来写一行代码时，我都会感觉到阻力的拉力。你可能也是。

*   如果你的时间被 Twitter 的引力劫持，并自我辩解说你将在自我实现的无限延伸中真正学到一些东西，这是一个迹象。
*   如果你经常被新的和闪亮的技术分心，并且有几十个半生不熟的个人项目永远见不到阳光，这是另一个迹象。
*   如果你经常在谷歌上搜索开发人员的工资，并试图弄清楚职业转换是否值得，是否是一个明智之举，那么，你基本上就是我。

写这个帖子，从某种意义上来说，是阻力告诉我这样做，而不是我的工作。你看这个帖子也是一样。它与我们内心的恐惧一致，害怕成功，害怕努力工作，害怕追求我们想要追求的东西。

## [](#conquering-resistance)征服抵抗

通过训练，对抗我们头脑中的这种力量变得更加容易。然而，这并不意味着它会消失。每天我们都会想出一个借口去做一些工作之外的事情，探索一些与工艺无关的事情，而不是工艺本身。我们会找到理由，而且是非常好的理由。

我找到的一个答案是对这门手艺的全职投入。不是浅尝辄止，而是在头脑中与自己签订牢不可破的契约。史蒂文·普雷斯菲尔德称之为“转向专业”。

不管怎样，我们每天都去上班。我们[每天写代码](https://johnresig.com/blog/write-code-every-day/)。我们关注长期收益，而不是短期满足。投入工作才是重要的，成功是副作用。

## [](#rules-for-success)成功规则

1.  每天按时去上班。在那段时间不要徘徊，保持专注，练习这门手艺。
2.  忽略别人告诉你的风险、不确定性或选择。如果你觉得活着，沉浸在编码的实践中，听听那个声音。
3.  不要只关注即时指标，如推文点赞、博客文章分享、下载量或工作机会。把你的注意力放在工作本身和实现未来目标的必要步骤上。成功会随后到来，这与你的工作无关。
4.  完成你陈述的内容，继续下一个目标。不要回头去猜测自己。
