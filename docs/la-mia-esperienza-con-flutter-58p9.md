# 我的颤栗经历

> 原文：<https://dev.to/grandemayta/la-mia-esperienza-con-flutter-58p9>

谷歌刚刚发布了其框架的 1.0 版，用于开发名为[浮动](https://flutter.io/)的本机应用程序。

我一直是一个应用程序爱好者，在我的工作过程中，我参与了很多项目，在这些项目中，我偶然发现了[为不同的公司从头开始创建应用程序](https://milano.repubblica.it/cronaca/2015/06/22/foto/playground-117440499/1/#2)。

[![Pisapia con la palla a spicchi lancia la app dei 120 playground di Milano](img/6706e961faa76e58d0228cc987b5ebf9.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--Osd-jKzW--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/5od4hkdfi9kw1buq6x2j.jpg)

大多数情况下，我一直使用角度/反应、Css 和 Html 等 web 技术，然后在框架的帮助下，比如 Apache Cordova 或 [Phonegap](https://phonegap.com/) 创建[混合应用程序](https://developer.telerik.com/featured/what-is-a-hybrid-mobile-app/)。这种方法的问题在于，外观&常常感觉到，可用性有时也会受到严重影响，尤其是在功能不太强大的设备上。

接着 Facebook 上出现了[反应原生](https://facebook.github.io/react-native/)，微软上出现[xamparin](https://visualstudio.microsoft.com/xamarin/)。
二者具有相同的承诺，允许开发者在 Android、IOS 和 Windows Phone (RIP)上使用相同的代码。

React Native AirBnb 在 2018 年的最大用户[宣布](https://medium.com/airbnb-engineering/sunsetting-react-native-1868ba28e30a)出于错误和各种问题等各种原因告别 Facebook 家庭框架。

还是在 2018 年谷歌宣布[发布首个 Beta 版颤振](https://proandroiddev.com/google-i-o-2018-flutter-recap-6f06545efd74)。阿里巴巴立即看到新框架的潜力，采用它来开发5000 万用户的应用程序。

我个人用 Flutter 创建了两个应用程序，它们现在在 Android 商店。学习曲线很快，因为它总是采用组件逻辑，就像 React 或 Angular 一样，即使它们被称为 Widget。
它使用的编程语言是[【dart】](https://www.dartlang.org/)，因此面向 Java 或 C#用户的对象比较简单。他们还引入了集成了材料设计指南和烹饪指南的现成 UI 组件，从而加快了开发速度。文件仍有待改进，但我已走上正轨。
